# KiriKiri 历史与生态

## 1. 什么是 KiriKiri？

在视觉小说（Visual Novel）和美少女游戏（Galgame）的开发领域，KiriKiri（吉里吉里）是一个不可逾越的里程碑。它不仅是一个引擎，更是一个时代的代名词。

### 创始人：W.Dee (渡邊哲也)
KiriKiri 由日本开发者 **W.Dee**（真名渡邊哲也）发起并主导开发。他在 20 世纪 90 年代末意识到，当时市面上的脚本引擎要么过于简陋，无法支持复杂的演出效果；要么过于封闭，开发者难以进行深度的底层定制。于是，他决定开发一套既易于上手又具有极强扩展性的引擎，这便是 KiriKiri。

### 诞生背景：视觉小说的黄金时代
21 世纪初是日本 Galgame 行业的黄金时期。当时的玩家不再满足于简单的文字和静态立绘，他们追求动态的背景、华丽的转场效果、复杂的分支剧情以及高画质的影音表现。KiriKiri 的出现正好填补了这一技术空白。它允许开发者通过简单的脚本实现极高质量的视觉呈现，极大地推动了行业的技术进步。

---

## 2. 版本的演进：从 1 到 Z

KiriKiri 经历了几次重大的版本迭代，每一个版本都代表了当时技术的最高水准。

### KiriKiri (1.x)
最初的版本奠定了引擎的基础框架，确立了以 TJS 语言为核心的逻辑处理机制。

### KiriKiri2 (2.x)
这是应用最广、最为经典的内核版本。KiriKiri2 引入了更强大的插件机制和渲染优化，支持分层图像合成。许多至今仍被玩家津津乐道的商业大作（如 *Fate/stay night* 的早期 PC 版）均是基于此版本开发的。

### KiriKiriZ (3.x)
由于 KiriKiri2 的代码库逐渐老化，难以适应现代操作系统（如 Windows 10/11）和 64 位环境的需求，W.Dee 和社区发起了 **KiriKiriZ** 项目。KiriKiriZ 主要是对原版 KiriKiri2 的重构与现代化更新，保持了极高的向下兼容性，目前已在 GitHub 上开源（wamsoft 组织）。

## 3. 核心架构：TVP 与 KAG

要理解 KiriKiri，必须分清它的三个层次：

### TVP (T Visual Presenter)
这是引擎的核心展示系统，负责底层任务。
- **任务**：窗口管理、图像渲染、音频解码、输入响应、文件读取等。
- **比喻**：TVP 就像是一台先进的播放机，它知道如何把数据转换成屏幕上的像素和扬声器里的声音。

### TJS2 (Think and Joy Script 2)
这是 KiriKiri 自创的脚本语言。
- **特点**：语法非常接近 JavaScript 和 C++。它是一种面向对象的解释型语言，负责游戏的逻辑控制。
- **作用**：定义角色的属性、计算好感度、控制 UI 的动态逻辑等。

### KAG (KiriKiri Adventure Game) 脚本框架
这是一个建立在 TJS2 之上的“游戏模板”或“脚本框架”。
- **目的**：让非编程人员也能写游戏。
- **语法**：使用特殊的标记语言（如 `[l]` 表示等待点击，`[cm]` 表示清除文字）。
- **现实意义**：90% 的 KiriKiri 游戏开发者实际上是在写 KAG 脚本，而不是直接写 TJS2 代码。

## 4. 技术特色：为什么它如此成功？

### 插件化架构
KiriKiri 最强大的地方在于它的 **插件系统**。原版引擎通过 `.dll` 文件扩展功能。
- **图像插件**：支持更多图片格式（如 WebP、PNG10、PSD）。
- **音频插件**：支持更高级的音频格式（如 OGG、FLAC、WAV）。
- **系统插件**：提供更多底层 API，甚至可以在游戏内嵌入网页浏览器或视频播放器。

### XP3 归档格式
KiriKiri 使用 `.xp3` 格式将游戏的脚本、图片、音频等资源打包。这种格式支持分段压缩和简单的加密，极大地优化了游戏的加载速度和防盗版能力。

### 多层渲染与复杂特效
相比于早期竞品，KiriKiri 允许开发者自由定义“层”（Layer）。你可以有无数层背景、立绘、对话框，甚至可以通过 TJS2 直接操作像素，实现类似 Photoshop 的混合模式（如滤色、正片叠底等）。

## 5. 生态：那些你熟悉的“神作”

如果你曾经玩过日系 Galgame，你大概率已经接触过 KiriKiri。

- **TYPE-MOON**：著名的 *Fate/stay night* 和 *Fate/hollow ataraxia* 早期 PC 版正是使用 KiriKiri2 开发。
- **Key**：许多早期的作品也有 KiriKiri 的身影。
- **柚子社 (Yuzusoft)**：在从自研引擎转向 KiriKiri 后，其作品（如《天神乱漫》）展现了极高的视觉和系统交互水准。

直到今天，虽然 Unity 和 Ren'Py 逐渐崛起，但在日本商业 Galgame 市场，KiriKiriZ 依然是许多老牌大厂的首选。

---

## 6. 社区与开源

KiriKiri 的生命力源于其开放性。
- **Wamsoft**：这是引擎的官方组织，他们在 GitHub 上开源了 KiriKiriZ 的核心代码以及许多重要的官方插件。
- **开发者社区**：尽管主要集中在日本，但全球的汉化组和独立开发者为 KiriKiri 贡献了大量的翻译补丁和兼容性修复插件。

KAG (KiriKiri Adventure Game) 是建立在 TJS2 脚本语言之上的一个高层框架。TJS2 是一门完整的面向对象语言（类似于 JavaScript/C++），负责游戏逻辑。而 KAG 提供了一套更贴近创作者习惯的指令集，让非程序员只需通过简单的标记符号就能实现对话展示、图片切换和分支跳转。

</details>

2. **KiriKiri 处理游戏资源主要使用的归档格式是什么？它有什么优点？**

<details>
<summary>查看答案</summary>

主要使用的归档格式是 **XP3**。它的优点包括：支持将大量零散文件打包成单文件，减少磁盘读写次数；支持资源压缩以减小游戏体积；支持简单的加密以防止资源被随意提取。

</details>

3. **KiriKiriZ 相比 KiriKiri2 最大的改进是什么？**

<details>
<summary>查看答案</summary>

KiriKiriZ 主要解决了现代操作系统的兼容性问题。它重构了底层代码，支持 64 位系统环境，支持现代的 C++ 标准，并保持了对 KiriKiri2 游戏脚本和资源的极高兼容性。

</details>
