# 启动流程全链路

理解 KrKr2 如何从一个二进制文件启动并进入到游戏场景，对于进行底层调试和性能优化至关重要。本章将追踪从 OS 入口点到脚本引擎开始运行的全过程。

## 1. 启动流程概览

KrKr2 的启动可以分为三个阶段：平台入口引导、Cocos2d 环境搭建、以及引擎业务初始化。

```text
[ 平台入口 ] -> platforms/*/main.cpp
      |
[ 应用控制 ] -> TVPAppDelegate::run()
      |
[ 引擎初始化 ] -> applicationDidFinishLaunching()
      |      1. SDL 初始化 (SDL_SetMainReady)
      |      2. 记录主线程 ID (TVPMainThreadID)
      |      3. Cocos2d Director & GLView 创建
      |      4. 平台适配 (窗口样式/设计分辨率)
      |      5. 语言配置 & UI 扩展加载
      |
[ 场景启动 ] -> TVPMainScene::CreateInstance() -> runWithScene()
      |
[ 任务分发 ] -> scheduleOnce (延迟启动文件选择器或直接加载游戏)
```

## 2. 核心入口：AppDelegate.cpp

真正的初始化逻辑集中在 `cpp/core/environ/cocos2d/AppDelegate.cpp`。让我们看看关键的代码片段及其职责：

```cpp
// 关键代码段 (基于 AppDelegate.cpp)
bool AppDelegate::applicationDidFinishLaunching() {
    // 1. 初始化底层多媒体抽象层
    SDL_SetMainReady();

    // 2. 记录主线程，KrKr2 是重度多线程设计的
    TVPMainThreadID = std::this_thread::get_id(); // 行 33 左右

    // 3. 初始化 Cocos2d 核心控制器
    auto director = Director::getInstance();
    auto glview = director->getOpenGLView();
    if(!glview) {
        // ... 创建并初始化 GLView ...
    }

    // 4. Windows 平台特有的窗口样式处理 (行 45-47)
#if (CC_TARGET_PLATFORM == CC_PLATFORM_WIN32)
    HWND hwnd = glview->getWin32Window();
    LONG style = GetWindowLong(hwnd, GWL_STYLE);
    style |= (WS_THICKFRAME | WS_MAXIMIZEBOX); // 允许缩放和最大化
    SetWindowLong(hwnd, GWL_STYLE, style);
#endif

    // 5. 环境配置与 UI 初始化 (行 70+)
    TVPInitUIExtension(); // 初始化特定平台的 UI 扩展
    LocaleConfigManager::GetInstance()->Initialize(); // 多语言支持

    // 6. 创建并运行第一个场景
    auto scene = TVPMainScene::CreateInstance();
    director->runWithScene(scene);

    // 7. 延迟执行：显示文件选择器或启动默认脚本 (行 80+)
    director->getScheduler()->scheduleOnce([](float dt){
        TVPGlobalPreferenceForm::Initialize();
        // 此处通常会根据启动参数决定显示“打开文件”对话框还是直接进游戏
    }, 0.5f, "InitializeApp");

    return true;
}
```

## 3. 跨平台入口的差异

虽然 `AppDelegate` 是跨平台的，但启动它的"第一行代码"位于 `platforms/` 目录下：

- **Windows**: `platforms/windows/main.cpp` 使用 `WinMain`。由于 Windows 桌面应用的特殊性，它还负责加载图标、处理 DPI 缩放。
- **Android**: 通过 JNI 由 Java 层引导。在 C++ 层，它通常由 SDL 提供的入口引导进入，最终调用 `TVPAppDelegate::run()`。
- **Linux**: 一个标准的 `int main()`，初始化简单的 GL 环境后立即跳转到 `AppDelegate`。

## 4. 生命周期管理

KrKr2 的生命周期同样由 `AppDelegate` 控制。Cocos2d 框架提供了几个关键的回调，KrKr2 通过这些回调来控制引擎的暂停和恢复：

- **applicationDidEnterBackground**: 当游戏进入后台（如按下手机 Home 键或最小化窗口）时，引擎会自动暂停音频播放、停止 TJS2 虚拟机的心跳，以节省电力或系统资源。
- **applicationWillEnterForeground**: 当游戏重新回到前台时，恢复音频、刷新 UI 状态。

## 5. 关键设计模式：Director 单例

KrKr2 采用了 Cocos2d 的 **Director** 单例模式。这意味着在整个应用程序的运行期间，只有一个 `Director` 实例负责协调：

1. **场景切换 (Scene Management)**: 所有的 TJS2 画面（包括菜单、对话框、游戏画面）都作为一个个 `Scene` 在 `Director` 的统一调度下进行渲染。
2. **渲染循环 (Rendering Loop)**: `Director` 负责驱动每一帧的渲染和脚本逻辑的推进。
3. **全局参数设置**: 诸如 FPS 设置（KrKr2 默认设置为 60）、分辨率自适应方案等。

## 练习题与答案

1. **选择题**：KrKr2 记录主线程 ID (`TVPMainThreadID`) 的主要目的是什么？
   A. 提高渲染速度
   B. 在多线程环境中确保某些 UI 或关键逻辑在主线程执行
   C. 统计 CPU 使用率
   D. 仅仅是为了打印日志

2. **填空题**：真正的初始化和环境搭建逻辑位于 `cpp/core/environ/cocos2d/` 下的 ______ 文件中。

3. **简答题**：描述在 Windows 平台下，KrKr2 是如何通过代码让窗口支持手动缩放和最大化的。

1. **B**（KrKr2 是一个多线程引擎，但许多操作系统 API 和渲染操作必须在主线程完成，因此需要记录主线程 ID 以进行线程校验和任务同步）。
2. **AppDelegate.cpp**（它是引擎在 Cocos2d 框架下的核心引导逻辑）。
3. **回答**：在 `applicationDidFinishLaunching` 函数中，KrKr2 会先通过 `glview->getWin32Window()` 获取原生窗口句柄 `HWND`，然后通过 Win32 API `GetWindowLong` 获取当前的窗口样式。接着，它将 `WS_THICKFRAME`（允许调整边框大小）和 `WS_MAXIMIZEBOX`（启用最大化按钮）这两个标志位合并到样式中，最后通过 `SetWindowLong` 重新应用这些样式。
