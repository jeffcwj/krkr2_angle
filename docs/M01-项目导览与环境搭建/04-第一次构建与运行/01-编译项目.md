# 编译项目

在上一章中，我们已经配置好了开发环境。现在，我们将进入激动人心的环节：第一次构建 KR2 引擎。KR2 使用 CMake 作为构建系统，并充分利用了 CMake Presets 来简化多平台的编译流程。

## CMake Presets 简要回顾

正如在 [Ch3 环境搭建](../03-环境搭建/README.md) 中提到的，CMake Presets 允许我们将复杂的编译参数（如编译器路径、依赖库位置、编译选项等）预定义在 `CMakePresets.json` 文件中。这意味着你不需要记忆冗长的命令行参数，只需指定一个“预设名称”即可。

## Windows 平台编译

Windows 是 KR2 的主要开发平台。我们推荐使用 Visual Studio 2022 提供的工具链。

### 1. 打开开发人员命令行
为了确保编译器路径正确，请从“开始”菜单搜索并打开：
- **x64 Native Tools Command Prompt for VS 2022**
- 或者 **Developer PowerShell for VS 2022**

### 2. 克隆项目
如果你还没有下载源码，请执行：
```bash
git clone <repo_url>
cd KR2
```

### 3. 使用脚本一键构建
项目根目录下提供了便捷脚本：
```cmd
./scripts/build-windows.bat
```
该脚本会自动调用 CMake 命令完成配置与构建。

### 4. 手动编译命令
如果你想了解脚本背后的细节，或者需要自定义参数，可以手动执行以下两条命令：

**配置阶段：**
```bash
cmake --preset="Windows Debug Config" -DDISABLE_TEST=ON
```
这里 `-DDISABLE_TEST=ON` 用于跳过单元测试的编译，以加快首次构建速度。

**构建阶段：**
```bash
cmake --build --preset="Windows Debug Build"
```

### 5. 输出位置
编译成功后，可执行文件位于：
`out/windows/debug/bin/krkr2/krkr2.exe`

## Linux 平台编译

在 Linux（如 Ubuntu 22.04+）上，编译过程同样被脚本化。

### 1. 执行构建脚本
```bash
chmod +x ./scripts/build-linux.sh
./scripts/build-linux.sh
```
该脚本不仅会编译项目，还会将必要的 `libfmod.so` 音频库安装到系统的 `/usr/lib` 目录下，以确保运行时能正确加载。

### 2. 输出位置
编译产物位于：
`out/linux/debug/bin/krkr2/krkr2`

---

## macOS 平台编译

macOS 平台目前推荐使用手动命令，确保你已安装 Xcode Command Line Tools。

### 1. 编译命令
```bash
cmake --preset="MacOS Debug Config"
cmake --build --preset="MacOS Debug Build"
```

### 2. 输出位置
编译完成后，你会得到一个标准的 macOS App Bundle：
`out/macos/debug/bin/krkr2/krkr2.app`

---

## Android 平台编译

KR2 采用跨平台核心，因此 Android 编译需要结合 CMake 和 Gradle。

### 1. 编译 Android APK
在根目录执行：
```bash
./platforms/android/gradlew -p ./platforms/android assembleDebug
```
这会调用 NDK 工具链编译 C++ 核心，并打包成 Android APK 文件。

### 2. 输出位置
`platforms/android/out/android/app/outputs/apk/debug/*.apk`

---

## 替代方案：Docker 编译

如果你不想在本地安装复杂的工具链，可以利用 Docker 镜像。

### 1. Linux 构建镜像
```bash
docker build -f dockers/linux.Dockerfile -t linux-builder .
```

### 2. Android 构建镜像
```bash
docker build -f dockers/android.Dockerfile -t android-builder .
```

## 构建进阶说明

### 1. Debug vs Release
- **Debug**：包含完整调试符号，运行慢但适合开发。
- **Release**：开启 O2 优化，移除调试日志，性能极佳。

### 2. 首次构建时间预期
由于引入了大量外部库并使用 vcpkg。
- **首次构建**：需下载编译 boost, ffmpeg, skia 等，耗时 **30-60 分钟**。
- **增量构建**：仅需 **1-5 秒**。

### 3. ccache 加速
KR2 的 CMake 会自动探测系统中的 `ccache`。如果安装了此工具，它会自动启用编译缓存，极大缩短后续编译时间。

1. 最常见的是 `cmake` 报错找不到 `cl.exe`（MSVC 编译器）或无法检测到 target architecture。
