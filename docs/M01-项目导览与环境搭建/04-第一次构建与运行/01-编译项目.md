## Configure 阶段详解

构建分两步：

1. Configure：解析 CMake 并生成构建系统。
2. Build：执行编译和链接。

示例：

```bash
cmake --preset="Linux Debug Config"
cmake --build --preset="Linux Debug Build"
```

### Configure 做了什么

以 `Linux Release Config` 为例：

1. 选择 `Ninja` 生成器。
2. 设置输出目录 `out/linux/release`。
3. 设置 `CMAKE_BUILD_TYPE=Release`。
4. 读取 `CMakeLists.txt` 的 option 和平台分支。
5. 生成 `CMakeCache.txt` 与 `build.ninja`。

### 变量覆盖规则

优先级：

1. 命令行 `-D`
2. preset `cacheVariables`
3. `CMakeLists.txt` 默认 option

示例：

```bash
cmake --preset="Windows Release Config" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
```

## 构建类型对比

| 类型 | 优化 | 调试信息 | 场景 |
|---|---|---|---|
| Debug | 低 | 多 | 开发调试 |
| Release | 高 | 少 | 发布测试 |
| RelWithDebInfo | 高 | 中 | 性能 + 排查 |

结合 `CMakeLists.txt`：

- Debug 定义 `DEBUG`、`_DEBUG`
- 非 Debug 定义 `NDEBUG`

## 并行编译参数

推荐：

```bash
cmake --build --preset="Linux Debug Build" --parallel
cmake --build --preset="Linux Debug Build" --parallel 8
```

说明：

- `--parallel` 自动并发。
- `--parallel N` 指定并发。

## 编译优化技巧

### ccache

`CMakeLists.txt` 自动探测：

```cmake
find_program(CCACHE_PROGRAM ccache)
if (CCACHE_PROGRAM)
    set(CMAKE_C_COMPILER_LAUNCHER "${CCACHE_PROGRAM}")
    set(CMAKE_CXX_COMPILER_LAUNCHER "${CCACHE_PROGRAM}")
endif ()
```

### 关闭不必要目标

```bash
cmake --preset="Linux Debug Config" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
```

## 日志分析

按阶段看错误：

1. Configure
2. Compile
3. Link

信号：

- `Could NOT find` -> Configure
- `error: xxx.cpp:line` -> Compile
- `LNK2019` / `undefined reference` -> Link

## Android 特殊流程

Android 入口是 Gradle：

```text
gradlew assembleDebug/assemble
  -> externalNativeBuild(CMake)
    -> NDK 编译
      -> 打包 APK
```

关键变量：

- `ANDROID_SDK` / `ANDROID_HOME`
- `ANDROID_NDK`
- `VCPKG_ROOT`

## 动手实践

### Step 1：检查 CMake

```bash
cmake --version
```

预期：显示 `cmake version` 且版本不低于 3.28。

### Step 2：列出 preset

```bash
cmake --list-presets
```

预期：显示当前主机可用的 preset 列表。

### Step 3：执行 Debug Configure

```bash
cmake --preset="Windows Debug Config" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
cmake --preset="Linux Debug Config" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
cmake --preset="MacOS Debug Config" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
```

预期：出现 `Build files have been written to`。

### Step 4：执行 Debug Build

```bash
cmake --build --preset="Windows Debug Build" --parallel
cmake --build --preset="Linux Debug Build" --parallel
cmake --build --preset="MacOS Debug Build" --parallel
```

预期：出现 `[n/m]` 进度并最终链接成功。

### Step 5：检查产物

- `out/windows/debug/bin/krkr2/krkr2.exe`
- `out/linux/debug/bin/krkr2/krkr2`
- `out/macos/debug/bin/krkr2/krkr2.app`

### Step 6：执行 Release

按你的主机平台执行对应命令：

```bash
cmake --preset="<Windows/Linux/MacOS Release Config>" -DENABLE_TESTS=OFF -DBUILD_TOOLS=OFF
cmake --build --preset="<Windows/Linux/MacOS Release Build>" --parallel
```

预期：`out/<platform>/release` 下生成对应产物。

### Step 7：Android Debug APK

```bash
./platforms/android/gradlew -p ./platforms/android --console=plain --no-daemon assembleDebug
```

预期：日志出现 `BUILD SUCCESSFUL`。

### Step 8：体验增量编译

```bash
cmake --build --preset="Linux Debug Build" --parallel
```

预期：编译文件数明显减少，耗时低于首次全量构建。

## 本节小结

- 桌面端使用 CMake Presets + Ninja。
- Android 使用 Gradle 入口。
- `-D` 会覆盖 preset/default 值。
- 构建类型和并发参数直接影响调试与速度。
