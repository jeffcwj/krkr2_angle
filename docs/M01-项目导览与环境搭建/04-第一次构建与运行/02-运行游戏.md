# 运行游戏

完成项目的编译后，我们就拥有了 KR2 引擎的可执行文件。接下来，我们需要通过它来运行一个真实的 KiriKiri 引擎游戏。

## 获取游戏文件

KR2 引擎本身不包含任何游戏数据。你需要准备一份合法的 KiriKiri 游戏副本。

### 当前支持的游戏
目前 KR2 处于开发阶段，由于引擎架构的巨大差异，当前官方仅针对以下一款游戏进行了完整适配：
- **游戏名称**：天神乱漫 - LUCKY or UNLUCKY!?
- **发行日期**：2009年5月29日
- **引擎版本**：KiriKiri 2

请确保你已经获取了该游戏的正版授权文件。

### 典型的文件结构
一个标准的 KiriKiri 游戏目录通常包含以下内容：
- `data.xp3`：核心游戏脚本和素材归档文件
- `plugin/`：各种 `.dll` 扩展插件目录（KR2 会在内部寻找等效的实现）
- `config.ksc`：启动配置
- 其他 `.xp3` 归档（如 `bgm.xp3`, `voice.xp3` 等）

## Windows 运行

在 Windows 上运行游戏有多种方式：

### 方式 1：双击运行（默认文件选择器）
直接前往编译输出目录并双击 `krkr2.exe`：
`out/windows/debug/bin/krkr2/krkr2.exe`

启动后，程序会弹出一个文件选择对话框。请在对话框中定位并选择游戏所在的主目录。

### 方式 2：命令行指定路径
如果你在调试过程中希望直接加载游戏，可以通过命令行参数传入路径：
```cmd
krkr2.exe "C:\Games\TenshinRanman"
```

## Linux 运行

在 Linux 平台上，请确保你已经按照上一节的要求安装了 `libfmod.so`。

### 命令行运行
```bash
./out/linux/debug/bin/krkr2/krkr2 "/home/user/games/TenshinRanman"
```
如果未指定路径，同样会弹出基于系统的原生文件选择器。

---

## macOS 运行

你可以通过 Finder 直接打开编译好的 `.app` 程序包：
`out/macos/debug/bin/krkr2/krkr2.app`

或者通过终端运行：
```bash
open out/macos/debug/bin/krkr2/krkr2.app --args "/Users/name/Games/TenshinRanman"
```

## Android 运行

在 Android 平台上，我们需要将 APK 安装到设备上。

### 1. 安装 APK
确保你的设备已连接并启用了开发者模式：
```bash
adb install platforms/android/out/android/app/outputs/apk/debug/app-debug.apk
```

### 2. 准备游戏资源
将游戏目录（如 `tenshinranman/`）整体复制到 Android 设备的存储中，建议路径为：
`/sdcard/Documents/Games/TenshinRanman`

### 3. 运行并选择目录
启动安装好的 **KR2** 应用。在出现的界面中，使用内置的文件选择器导航到上述路径并点击“选择”。

## 运行时日志

KR2 使用 `spdlog` 进行日志记录。为了监控引擎的工作状态，你可以查看控制台输出。

### 日志器类别
- **core**：核心逻辑日志（渲染、输入、系统交互等）。
- **tjs2**：脚本引擎执行日志（变量、函数调用、错误信息）。
- **plugin**：扩展插件加载和执行日志。

### 控制台输出示例
在启动时，你会在命令行终端中看到如下信息：
```text
[2024-03-17 10:00:00] [core] [info] KR2 Engine initializing...
[2024-03-17 10:00:00] [core] [info] AppDelegate: applicationDidFinishLaunching
[2024-03-17 10:00:01] [tjs2] [info] TJS2 VM starting execution...
```
