## 渲染管线全局架构

KrKr2 的渲染管线采用**延迟合成（Deferred Composition）**架构。图层修改不会立即触发渲染，而是标记脏区域，在主循环的适当时机统一合成。

### 管线全景图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            TJS 脚本层                                        │
│   layer.left = 100;  // 触发 SetLeft() → Update() → AddUpdateRegion()      │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         更新区域累积阶段                                      │
│   LayerManager::UpdateRegion (tTVPComplexRect) 累积所有脏矩形                │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼ 主循环调用 UpdateToDrawDevice()
┌─────────────────────────────────────────────────────────────────────────────┐
│                         完成（Completion）阶段                               │
│   Primary->CompleteForWindow(this)                                          │
│     ├── BeforeCompletion()         // 预处理                                │
│     ├── InternalComplete2[_GPU]()  // 实际渲染                              │
│     │     ├── Draw[_GPU]()         // 递归绘制图层树                        │
│     │     │     ├── DrawSelf()     // 绘制自身内容                          │
│     │     │     └── child->Draw()  // 递归子图层                            │
│     │     └── DrawCompleted()      // 混合到目标缓冲区                      │
│     └── AfterCompletion()          // 后处理                                │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         最终呈现阶段                                         │
│   NotifyLayerImageChange() → DrawDevice → 显示设备                          │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 更新触发机制

### 脏区域标记入口

当 TJS 脚本或引擎内部修改图层属性时，会调用 `Update()` 方法标记脏区域：

```cpp
// cpp/core/visual/LayerIntf.cpp 第 2170-2183 行
void tTJSNI_BaseLayer::SetPosition(tjs_int left, tjs_int top) {
    if(Rect.left != left || Rect.top != top) {
        bool visible = GetVisible() || GetNodeVisible();
        if(IsPrimary() && (left != 0 || top != 0))
            TVPThrowExceptionMessage(TVPCannotMovePrimary);
        
        if(visible)
            ParentUpdate();  // 标记旧位置为脏
        
        Rect.set_offsets(left, top);  // 实际移动
        
        if(Parent)
            Parent->NotifyChildrenVisualStateChanged();
        SetToCreateExposedRegion();
        
        if(visible)
            ParentUpdate();  // 标记新位置为脏
    }
}
```

**关键观察**：移动操作会**两次**调用 `ParentUpdate()`——旧位置和新位置都需要重绘。

### LayerManager 脏区域管理

```cpp
// cpp/core/visual/LayerManager.h 第 48-65 行
class tTVPLayerManager : public iTVPLayerManager, public tTVPDrawable {
private:
    tTVPComplexRect UpdateRegion;         // 累积的脏区域
    tTVPComplexRect UpdateRegionForCompletion;  // 用于本次完成的区域
    
public:
    void AddUpdateRegion(const tTVPRect &rect) {
        // 将矩形合并到更新区域
        UpdateRegion.Or(rect);
        
        // 防止区域碎片化
        if(UpdateRegion.GetCount() > TVP_UPDATE_UNITE_LIMIT) {
            UpdateRegion.Unite();  // 合并为最小包围矩形
        }
    }
    
    tTVPComplexRect &GetUpdateRegionForCompletion() {
        // 交换区域，避免渲染期间新更新干扰
        std::swap(UpdateRegion, UpdateRegionForCompletion);
        UpdateRegion.Clear();
        return UpdateRegionForCompletion;
    }
};
```

**设计要点**：

1. **tTVPComplexRect**：支持复杂形状的区域管理（多个不相交矩形）
2. **碎片防护**：当区域数量超过阈值时自动合并
3. **双缓冲区域**：渲染期间使用独立区域，新更新不影响当前帧

### CompleteForWindow() 完成流程

```cpp
// cpp/core/visual/LayerIntf.cpp 第 7310-7337 行
void tTJSNI_BaseLayer::CompleteForWindow(tTVPDrawable *drawable) {
    // 1. 预处理
    BeforeCompletion();
    
    // 2. 通知更新区域已固定
    if(Manager)
        Manager->NotifyUpdateRegionFixed();
    
    InCompletion = true;
    
    // 3. 通知开始位图合成
    if(Manager)
        Manager->GetLayerTreeOwner()->StartBitmapCompletion(Manager);
    
    try {
        // 4. 执行实际渲染（GPU 或软件路径）
        if(IsGPU()) {
            InternalComplete2_GPU(Rect, drawable);
        } else {
            InternalComplete2(Manager->GetUpdateRegionForCompletion(), drawable);
        }
    } catch(...) {
        if(Manager)
            Manager->GetLayerTreeOwner()->EndBitmapCompletion(Manager);
        throw;
    }
    
    // 5. 结束位图合成
    if(Manager)
        Manager->GetLayerTreeOwner()->EndBitmapCompletion(Manager);
    
    InCompletion = false;
    
    // 6. 后处理
    AfterCompletion();
}
```

## GPU vs 软件渲染路径选择

### IsGPU() 判断逻辑

```cpp
// cpp/core/visual/LayerIntf.cpp 第 70-75 行
static bool IsGPU() {
    static bool isGPU = !TVPIsSoftwareRenderManager() &&
        !IndividualConfigManager::GetInstance()->GetValue<bool>(
            "ogl_accurate_render", false);
    return isGPU;
}
```

**判断条件**：

| 条件 | 结果 |
|------|------|
| `TVPIsSoftwareRenderManager() == true` | 强制软件渲染 |
| `ogl_accurate_render == true` | 精确渲染模式，使用软件路径 |
| 其他情况 | 使用 GPU 路径 |

### 两条路径对比

| 特性 | GPU 路径 (`Draw_GPU`) | 软件路径 (`Draw`) |
|------|----------------------|------------------|
| **更新粒度** | 整个 Rect | 精细的 ComplexRect |
| **缓存利用** | 简化处理 | 完整的 CacheBitmap 机制 |
| **条带分割** | 无 | 自适应条带高度（16-256 行） |
| **适用场景** | 移动端、简单场景 | 桌面端、复杂合成 |

## 软件渲染详解：InternalComplete2()

### 条带分割策略

```cpp
// cpp/core/visual/LayerIntf.cpp 第 7190-7276 行
void tTJSNI_BaseLayer::InternalComplete2(tTVPComplexRect &updateregion,
                                         tTVPDrawable *drawable) {
    // 查询排除区域（如视频叠加层）
    if(Manager)
        Manager->QueryUpdateExcludeRect();
    
    // 遍历所有脏矩形
    tTVPComplexRect::tIterator it = updateregion.GetIterator();
    while(it.Step()) {
        tTVPRect r(*it);
        r.add_offsets(Rect.left, Rect.top);  // 转换到父坐标系
        
        if(TVPGraphicSplitOperationType != gsotNone) {
            // 计算最佳条带高度
            tjs_int oh;
            if(GetVisibleChildrenCount() || InTransition) {
                tjs_int rw = r.get_width();
                if(rw < 40)       oh = 256;
                else if(rw < 80)  oh = 128;
                else if(rw < 160) oh = 64;
                else if(rw < 320) oh = 32;
                else              oh = 16;  // 8核 CPU 每核 2 行
            } else {
                oh = r.get_height();  // 无需分割
            }
            
            // 按条带分割绘制
            for(tjs_int y = r.top; y < r.bottom; y += oh) {
                tTVPRect opr;
                opr.left = r.left;
                opr.right = r.right;
                opr.top = y;
                opr.bottom = std::min(y + oh, r.bottom);
                
                Draw(drawable, opr, false);  // 绘制条带
            }
        } else {
            Draw(drawable, r, false);  // 不分割
        }
    }
    
    updateregion.Clear();
}
```

**条带分割的意义**：

1. **CPU 缓存友好**：窄条带更容易放入 L1/L2 缓存
2. **多核并行**：每个条带可独立处理
3. **自适应**：根据宽度动态调整高度

### 分割模式

## 递归绘制流程

### Draw() 方法核心逻辑

```cpp
// cpp/core/visual/LayerIntf.cpp 第 6794-7050 行
void tTJSNI_BaseLayer::Draw(tTVPDrawable *target, const tTVPRect &r,
                            bool visiblecheck) {
    // 1. 可见性检查
    if(visiblecheck && !IsSeen())
        return;
    
    // 2. 求交区域
    tTVPRect rect = r;
    if(!TVPIntersectRect(&rect, rect, Rect))
        return;  // 无交集
    
    CurrentDrawTarget = target;
    ParentRectToChildRect(rect);  // 转换到子坐标系
    
    // 3. 处理过渡效果
    if(InTransition && TransWithChildren) {
        TransDrawable.Init(this, target);
        target = &TransDrawable;  // 重定向到过渡处理器
    }
    
    // 4. 判断是否使用缓存
    bool totalopaque = (DisplayType == ltOpaque && Opacity == 255);
    
    if(GetCacheEnabled() && 
       !(InTransition && !TransWithChildren && DivisibleTransHandler)) {
        // ===== 缓存路径 =====
        DrawWithCache(target, rect);
    } else {
        // ===== 非缓存路径 =====
        if(GetVisibleChildrenCount() == 0) {
            // 5a. 叶子节点：直接绘制自身
            tTVPRect pr = rect;
            pr.add_offsets(Rect.left, Rect.top);
            DrawSelf(target, pr, rect);
        } else {
            // 5b. 分支节点：处理重叠和暴露区域
            DrawWithChildren(target, rect, totalopaque);
        }
    }
    
    CurrentDrawTarget = nullptr;
}
```

### DrawSelf() 绘制自身内容

```cpp
// cpp/core/visual/LayerIntf.cpp 第 6388-6439 行
void tTJSNI_BaseLayer::DrawSelf(tTVPDrawable *target, tTVPRect &pr,
                                tTVPRect &cr) {
    if(!MainImage) {
        // 无位图：若类型为 ltOpaque，填充中性色
        if(DisplayType == ltOpaque) {
            tTVPBaseTexture *temp =
                tTVPTempBitmapHolder::GetTemp(cr.get_width(), cr.get_height());
            try {
                tTVPRect bitmaprect = cr;
                bitmaprect.set_offsets(0, 0);
                CopySelf(temp, 0, 0, bitmaprect);  // 填充中性色
                target->DrawCompleted(pr, temp, bitmaprect, DisplayType, Opacity);
            } catch(...) {
                tTVPTempBitmapHolder::FreeTemp();
                throw;
            }
            tTVPTempBitmapHolder::FreeTemp();
        }
        return;
    }
    
    // 调整到图像局部坐标
    cr.add_offsets(-ImageLeft, -ImageTop);
    
    if(InTransition && !TransWithChildren && DivisibleTransHandler) {
        // 处理非子树过渡
        tTVPBaseTexture *temp =
            tTVPTempBitmapHolder::GetTemp(cr.get_width(), cr.get_height());
        try {
            tTVPRect bitmaprect = cr;
            bitmaprect.set_offsets(0, 0);
            DoDivisibleTransition(temp, 0, 0, cr);
            target->DrawCompleted(pr, temp, bitmaprect, DisplayType, Opacity);
        } catch(...) {
            tTVPTempBitmapHolder::FreeTemp();
            throw;
        }
        tTVPTempBitmapHolder::FreeTemp();
    } else {
        // 正常绘制：直接提交 MainImage
        target->DrawCompleted(pr, MainImage, cr, DisplayType, Opacity);
    }
}
```

### DrawCompleted() 混合回调

```cpp
// cpp/core/visual/LayerIntf.cpp 第 7084-7165 行
void tTJSNI_BaseLayer::DrawCompleted(const tTVPRect &destrect,
                                     tTVPBaseTexture *bmp,
                                     const tTVPRect &cliprect,
                                     tTVPLayerType type, tjs_int opacity) {
    // 处理 Binder 类型或透传情况
    if(DisplayType == ltBinder ||
       (MainImage == nullptr && DirectTransferToParent)) {
        tTVPRect _destrect(destrect);
        _destrect.add_offsets(Rect.left, Rect.top);
        CurrentDrawTarget->DrawCompleted(_destrect, bmp, cliprect, type, opacity);
        return;
    }

    if(bmp != UpdateBitmapForChild) {
        if(MainImage == nullptr) {
            // 特殊优化：无主图像时的透明处理

        } else {
            // 标准混合
            BltImage(UpdateBitmapForChild, DisplayType,
                     destrect.left - UpdateOfsX, destrect.top - UpdateOfsY,
                     bmp, cliprect, type, opacity);
        }
    }
}
```

## 临时缓冲区管理

### tTVPTempBitmapHolder 单例

```cpp
// cpp/core/visual/LayerIntf.cpp 第 80-211 行
class tTVPTempBitmapHolder {
private:
    static tTVPTempBitmapHolder *Instance;
    
    // 嵌套分配跟踪
    struct TempEntry {
        tTVPBaseTexture *Bitmap;
        int NestCount;
    };
    std::vector<TempEntry> Pool;
    int CurrentNest;
    
public:
    static tTVPBaseTexture *GetTemp(tjs_uint w, tjs_uint h) {
        if(!Instance)
            Instance = new tTVPTempBitmapHolder();
        return Instance->InternalGetTemp(w, h);
    }
    
    static void FreeTemp() {
        if(Instance)
            Instance->InternalFreeTemp();
    }
    
private:
    tTVPBaseTexture *InternalGetTemp(tjs_uint w, tjs_uint h) {
        CurrentNest++;
        
        // 查找可复用的缓冲区
        for(auto &entry : Pool) {
            if(entry.NestCount == 0 &&
               entry.Bitmap->GetWidth() >= w &&
               entry.Bitmap->GetHeight() >= h) {
                entry.NestCount = CurrentNest;
                return entry.Bitmap;
            }
        }
        
        // 创建新缓冲区
        tTVPBaseTexture *bmp = new tTVPBaseTexture(w, h, 32);
        Pool.push_back({bmp, CurrentNest});
        return bmp;
    }
    
    void InternalFreeTemp() {
        // 释放当前嵌套层的缓冲区
        for(auto &entry : Pool) {
            if(entry.NestCount == CurrentNest)
                entry.NestCount = 0;
        }
        CurrentNest--;
    }
    
    // 空闲时压缩池
    void Compact() {
        // 移除未使用且超大的缓冲区
        Pool.erase(
            std::remove_if(Pool.begin(), Pool.end(),
                [](const TempEntry &e) {
                    return e.NestCount == 0 && 
                           e.Bitmap->GetWidth() * e.Bitmap->GetHeight() > 
                           TVP_TEMP_BITMAP_MAX_SIZE;
                }),
            Pool.end());
    }
};
```

**设计亮点**：

1. **嵌套计数**：支持递归绘制场景下的多级分配
2. **大小匹配**：优先复用足够大的缓冲区
3. **自动压缩**：应用切后台时释放过大缓冲区

### 使用模式

```cpp
// 典型使用模式（RAII 风格）
void SomeRenderFunction() {
    tTVPBaseTexture *temp = tTVPTempBitmapHolder::GetTemp(width, height);
    try {

    } catch(...) {
        tTVPTempBitmapHolder::FreeTemp();  // 确保异常时释放
        throw;
    }
```

## GPU 渲染路径：Draw_GPU()

```cpp
// cpp/core/visual/LayerIntf.cpp 第 6516-6635 行
void tTJSNI_BaseLayer::Draw_GPU(tTVPDrawable *target, int x, int y,
                                const tTVPRect &r, bool visiblecheck) {
    if(visiblecheck && !IsSeen())
        return;
    
    tTVPRect rect;
    if(!TVPIntersectRect(&rect, r, Rect))
        return;
    
    x += rect.left - r.left;
    y += rect.top - r.top;
    
    tTVPRect rctar(rect);
    rctar.set_offsets(x, y);
    
    CurrentDrawTarget = target;
    ParentRectToChildRect(rect);
    
    // 处理过渡
    if(InTransition && TransWithChildren) {
        TransDrawable.Init(this, target);
        target = &TransDrawable;
    }
    
    if(Opacity < 255 || (InTransition && TransWithChildren)) {
        // 需要临时缓冲区的情况
        if(GetVisibleChildrenCount() == 0) {
            DrawSelf(target, rctar, rect);
        } else {
            // 分配临时缓冲区
            bool useTemp = !GetCacheEnabled();
            if(useTemp) {
                UpdateBitmapForChild = tTVPTempBitmapHolder::GetTemp(
                    rect.get_width(), rect.get_height());
            } else {
                UpdateBitmapForChild = CacheBitmap;
            }
            
            // 复制自身内容
            if(MainImage != nullptr)
                CopySelfForRect(UpdateBitmapForChild, 0, 0, rect);
            
            // 递归绘制子图层
            TVP_LAYER_FOR_EACH_CHILD_BEGIN(child) {
                if(!child->Visible) continue;
                
                tTVPRect chrect;
                if(!TVPIntersectRect(&chrect, rect, child->Rect))
                    continue;
                
                // 设置子图层更新参数
                UpdateOfsX = 0;
                UpdateOfsY = 0;
                UpdateRectForChildOfsX = chrect.left - child->Rect.left;
                UpdateRectForChildOfsY = chrect.top - child->Rect.top;
                
                child->Draw_GPU((tTVPDrawable *)this, 
                               chrect.left, chrect.top, chrect);
            }
            TVP_LAYER_FOR_EACH_CHILD_END
            
            // 提交合成结果
            rect.set_offsets(0, 0);
            target->DrawCompleted(rctar, UpdateBitmapForChild, rect,
                                 DisplayType, Opacity);
            
            if(useTemp)
                tTVPTempBitmapHolder::FreeTemp();
        }
    } else {
        // 完全不透明且无过渡：简化路径
        if(GetVisibleChildrenCount() == 0) {
            DrawSelf(target, rctar, rect);
        } else {
            DrawnRegion.Clear();
            DrawSelf(target, rctar, rect);
            
            TVP_LAYER_FOR_EACH_CHILD_BEGIN(child) {
                if(!child->Visible) continue;
                
                tTVPRect chrect;
                if(!TVPIntersectRect(&chrect, rect, child->Rect))
                    continue;
                
                child->Draw_GPU(target, x, y, rect);
            }
            TVP_LAYER_FOR_EACH_CHILD_END
        }
    }
    
    CurrentDrawTarget = nullptr;
}
```

**GPU 路径特点**：

1. **简化区域计算**：不使用 ComplexRect，直接处理整个 Rect
2. **直接透传**：完全不透明时无需临时缓冲区
3. **坐标传递**：通过 x, y 参数累积偏移

```cpp
            else if(TVPIsTypeUsingAddAlpha(destlayertype))
                met = bmAlphaOnAddAlpha;
            else
                met = bmAlpha;
            break;

            hda = hda || TVPIsTypeUsingAlphaChannel(destlayertype);
            met = bmAdd;
            break;

    }
```

```cpp
class UpdateRegionDebugger {

            FrameUpdates.push_back(*it);
        }

        if(!Enabled || FrameUpdates.empty()) return;
        
        // 在更新区域边缘绘制红色矩形
        for(const auto &rect : FrameUpdates) {
            DrawRectOutline(buffer, rect, 0xFFFF0000);  // 红色边框
        }
        
        // 输出调试信息
        TVPAddLog(TJS_W("Frame update regions: ") + 
                  ttstr(FrameUpdates.size()) + TJS_W(" rects"));
    }
    
private:
```

```cpp
// render_profiler.cpp
#include <chrono>

class RenderProfiler {
private:
    struct PhaseTime {
        std::chrono::microseconds Total{0};
        int Count = 0;
    };
    
    std::map<std::string, PhaseTime> Phases;
    std::chrono::high_resolution_clock::time_point PhaseStart;
    std::string CurrentPhase;
    
public:
    void BeginPhase(const std::string &name) {
        EndPhase();  // 结束上一阶段
        CurrentPhase = name;
        PhaseStart = std::chrono::high_resolution_clock::now();
    }
    
    void EndPhase() {
        if(CurrentPhase.empty()) return;
        
        auto now = std::chrono::high_resolution_clock::now();
        auto duration = std::chrono::duration_cast<std::chrono::microseconds>(
            now - PhaseStart);
        
        Phases[CurrentPhase].Total += duration;
        Phases[CurrentPhase].Count++;
        
        CurrentPhase.clear();
    }
    
    void PrintReport() {
        TVPAddLog(TJS_W("=== Render Pipeline Profile ==="));
        
        for(const auto &[name, time] : Phases) {
            float avgUs = static_cast<float>(time.Total.count()) / time.Count;
            TVPAddLog(ttstr(name.c_str()) + TJS_W(": avg ") + 
                     ttstr(std::to_string(avgUs)) + TJS_W(" us"));
        }
    }
    
    void Reset() {
        Phases.clear();
    }
};

// 集成到渲染流程
void ProfiledCompleteForWindow(tTJSNI_BaseLayer *layer,
```

```cpp
        tTVPLayerType LayerType;
        tjs_int Opacity;
    };

public:
    RecordingDrawable(tTVPDrawable *realTarget) : RealTarget(realTarget) {}
    
    tTVPBaseTexture *GetDrawTargetBitmap(const tTVPRect &rect,
                                         tTVPRect &cliprect) override {
        Operations.push_back({DrawOp::GetTarget, rect, ltOpaque, 255});
        return RealTarget->GetDrawTargetBitmap(rect, cliprect);
    }
    
    tTVPLayerType GetTargetLayerType() override {
        return RealTarget->GetTargetLayerType();
    }
    
    void DrawCompleted(const tTVPRect &destrect, tTVPBaseTexture *bmp,
                       const tTVPRect &cliprect, tTVPLayerType type,
                       tjs_int opacity) override {
        Operations.push_back({DrawOp::DrawComplete, destrect, type, opacity});
        RealTarget->DrawCompleted(destrect, bmp, cliprect, type, opacity);
    }
    
    void PrintOperations() const {
        TVPAddLog(TJS_W("=== Draw Operations ==="));
        for(size_t i = 0; i < Operations.size(); i++) {
            const auto &op = Operations[i];
            ttstr msg;
            if(op.OpType == DrawOp::GetTarget) {
                msg = TJS_W("GetTarget: ");
            } else {
                msg = TJS_W("DrawComplete: type=") + ttstr((int)op.LayerType) +
                      TJS_W(" opacity=") + ttstr(op.Opacity) + TJS_W(" ");
            }
            msg += TJS_W("rect=(") + ttstr(op.Rect.left) + TJS_W(",") +
                   ttstr(op.Rect.top) + TJS_W(",") + ttstr(op.Rect.right) +
                   TJS_W(",") + ttstr(op.Rect.bottom) + TJS_W(")");
            TVPAddLog(msg);
        }
    }
    
    void Clear() { Operations.clear(); }
};
```

## 本节小结

1. **延迟合成架构**：图层修改只标记脏区域，在 `UpdateToDrawDevice()` 统一渲染
2. **双路径设计**：GPU 路径（`Draw_GPU`）简化处理，软件路径（`Draw`）精细优化
3. **递归绘制模式**：`Draw()` → `DrawSelf()` → `DrawCompleted()` 形成完整调用链
4. **条带分割优化**：根据宽度自适应条带高度，利用 CPU 缓存和多核
5. **临时缓冲区池化**：`tTVPTempBitmapHolder` 支持嵌套分配，避免频繁内存操作
6. **混合模式丰富**：支持 26 种图层类型，包括 Photoshop 兼容模式

### 题目 2：GPU vs 软件路径差异

下列哪些说法正确？（多选）

A. GPU 路径使用 `tTVPComplexRect` 进行精细区域追踪

### 题目 3：临时缓冲区泄漏场景

以下代码存在什么问题？如何修复？

```cpp
void RenderLayer(tTVPDrawable *target, const tTVPRect &rect) {
    tTVPBaseTexture *temp = tTVPTempBitmapHolder::GetTemp(rect.get_width(), 
                                                          rect.get_height());
    
    ProcessBitmap(temp);  // 可能抛出异常
    target->DrawCompleted(rect, temp, rect, ltAlpha, 255);
    
    tTVPTempBitmapHolder::FreeTemp();
}
```

<details>
<summary>查看答案</summary>

**问题**：如果 `ProcessBitmap()` 抛出异常，`FreeTemp()` 不会被调用，导致嵌套计数错误。

**修复方案**：使用 try-catch 确保异常时也释放：

```cpp
void RenderLayer(tTVPDrawable *target, const tTVPRect &rect) {
    tTVPBaseTexture *temp = tTVPTempBitmapHolder::GetTemp(rect.get_width(), 
                                                          rect.get_height());
    try {
        ProcessBitmap(temp);
        target->DrawCompleted(rect, temp, rect, ltAlpha, 255);
    } catch(...) {
        tTVPTempBitmapHolder::FreeTemp();  // 异常时释放
        throw;  // 重新抛出
    }
    tTVPTempBitmapHolder::FreeTemp();
}
```

```cpp
class TempBitmapGuard {
    bool Released = false;
public:
    tTVPBaseTexture *Bitmap;
    TempBitmapGuard(tjs_uint w, tjs_uint h) 
        : Bitmap(tTVPTempBitmapHolder::GetTemp(w, h)) {}
    ~TempBitmapGuard() { if(!Released) tTVPTempBitmapHolder::FreeTemp(); }
    void Release() { tTVPTempBitmapHolder::FreeTemp(); Released = true; }
};

void RenderLayer(tTVPDrawable *target, const tTVPRect &rect) {
    TempBitmapGuard guard(rect.get_width(), rect.get_height());

    target->DrawCompleted(rect, guard.Bitmap, rect, ltAlpha, 255);
```
