## 图层属性传播机制

KiriKiri2 的图层系统采用**树形传播**模型，当父图层的某些属性发生变化时，会自动影响所有子图层的状态。这种设计确保了视觉一致性和交互正确性。

### 可见性状态传播

图层的可见性通过 `Visible` 属性控制，但实际可见性还取决于父图层链的状态：

```cpp
// 文件：cpp/core/visual/LayerIntf.h 第 299-310 行
class tTJSNI_BaseLayer {
private:
    bool Visible;  // 图层自身的可见性标志
    
public:
    bool GetVisible() const { return Visible; }
    void SetVisible(bool st);
    
    // 检查父图层链是否都可见（不检查透明度）
    bool GetParentVisible() const;
    
    // 节点可见性 = 父链可见 && 自身可见
    bool GetNodeVisible() {
        return GetParentVisible() && Visible;
    }
    
    // 是否实际"可见"（可见且不透明）
    bool IsSeen() const { return Visible && Opacity != 0; }
};
```

**关键概念区分：**

| 方法 | 含义 | 用途 |
|------|------|------|
| `GetVisible()` | 仅返回自身的 Visible 标志 | 获取脚本设置的值 |
| `GetParentVisible()` | 检查所有祖先的 Visible | 判断是否被父层隐藏 |
| `GetNodeVisible()` | 父链 + 自身的综合可见性 | 用于渲染判断 |
| `IsSeen()` | 可见且有不透明度 | 用于命中测试和绘制优化 |

### SetVisible 的传播实现

当调用 `SetVisible(true/false)` 时，系统执行以下操作：

```cpp
// 文件：cpp/core/visual/LayerIntf.cpp（简化实现）
void tTJSNI_BaseLayer::SetVisible(bool st) {
    if(Visible != st) {
        // 1. 先更新脏区（隐藏前或显示后都需要重绘）
        if(Visible) {
            Update(false);  // 隐藏前标记脏区
        }
        
        // 2. 更新可见性标志
        Visible = st;
        
        // 3. 通知父图层子节点状态变化
        if(Parent) {
            Parent->NotifyChildrenVisualStateChanged();
        }
        
        // 4. 重建裸露区域信息
        SetToCreateExposedRegion();
        
        // 5. 显示后标记脏区
        if(Visible) {
            Update(false);
        }
        
        // 6. 处理焦点转移（如果被隐藏且持有焦点）
        if(!st && Manager) {
            if(Manager->GetFocusedLayer() == this) {
                Manager->SetFocusTo(GetNextFocusable(), true);
            }
        }
    }
}
```

### 子节点可见计数优化

为了避免每次渲染都遍历整棵子树，系统维护了 `VisibleChildrenCount` 缓存：

```cpp
// 文件：cpp/core/visual/LayerIntf.h 第 262-269 行
class tTJSNI_BaseLayer {
private:
    tjs_int VisibleChildrenCount;  // -1 表示需要重新计算
    
public:
    tjs_int GetVisibleChildrenCount() {
        if(VisibleChildrenCount == -1)
            CheckChildrenVisibleState();  // 惰性计算
        return VisibleChildrenCount;
    }
    
    void CheckChildrenVisibleState();        // 遍历计算可见子节点数
    void NotifyChildrenVisualStateChanged(); // 失效缓存
};

// 失效通知实现
void tTJSNI_BaseLayer::NotifyChildrenVisualStateChanged() {
    VisibleChildrenCount = -1;  // 标记为需要重新计算
    SetToCreateExposedRegion(); // 裸露区域也需要重建
    
    // 向上传播（父图层的统计也受影响）
    if(Parent) {
        Parent->NotifyChildrenVisualStateChanged();
    }
}
```

**触发 NotifyChildrenVisualStateChanged 的场景：**
- 子图层 Visible 变化
- 子图层 SetHasImage 变化
- 子图层位置/尺寸变化（SetLeft/Top/Width/Height）
- 子图层被添加/移除

## 启用状态传播

`Enabled` 属性控制图层是否响应用户输入。与 Visible 类似，它也有节点级别的计算：

```cpp
// 文件：cpp/core/visual/LayerIntf.h 第 684-692 行
class tTJSNI_BaseLayer {
private:
    bool Enabled;      // 自身启用标志
    bool EnabledWork;  // 用于检测状态变化的工作变量
    
public:
    void SetEnabled(bool b);
    bool GetEnabled() const { return Enabled; }
    bool ParentEnabled();   // 检查父链是否都启用
    
    // 节点启用 = 自身启用 && 父链启用 && 不被模态层禁用
    bool GetNodeEnabled() {
        return GetEnabled() && ParentEnabled() && !IsDisabledByMode();
    }
};
```

### SetEnabled 的事件通知

启用状态变化时，系统会触发 `onNodeEnabled` / `onNodeDisabled` 事件：

```cpp
// 文件：cpp/core/visual/LayerIntf.cpp 第 3768-3795 行
void tTJSNI_BaseLayer::SetEnabled(bool b) {
    if(Enabled != b) {
        // 1. 保存整棵子树的当前启用状态快照
        if(Manager)
            Manager->SaveEnabledWork();
        
        try {
            Enabled = b;
            
            if(Enabled) {
                // 变为启用：检查是否可以获得焦点
                if(Manager)
                    Manager->CheckTreeFocusableState(this);
            } else {
                // 变为禁用：移除焦点
                if(Manager)
                    Manager->BlurTree(this);
            }
        } catch(...) {
            if(Manager)
                Manager->NotifyNodeEnabledState();
            throw;
        }
        
        // 2. 对比快照，触发状态变化事件
        if(Manager)
            Manager->NotifyNodeEnabledState();
    }
}

// 递归保存启用状态快照
void tTJSNI_BaseLayer::SaveEnabledWork() {
    EnabledWork = GetNodeEnabled();  // 保存当前节点启用状态
    
    // 递归保存子节点状态
    TVP_LAYER_FOR_EACH_CHILD_NOLOCK_BEGIN(child)
        child->SaveEnabledWork();
    TVP_LAYER_FOR_EACH_CHILD_NOLOCK_END
}

// 递归触发启用状态变化事件
void tTJSNI_BaseLayer::NotifyNodeEnabledState() {
    bool en = GetNodeEnabled();
    
    // 只在状态真正变化时触发事件
    if(Owner && !Shutdown && EnabledWork != en) {
        if(en) {
            static ttstr eventname(TJS_W("onNodeEnabled"));
            TVPPostEvent(Owner, Owner, eventname, 0, 
                         TVP_EPT_IMMEDIATE, 0, nullptr);
        } else {
            static ttstr eventname(TJS_W("onNodeDisabled"));
            TVPPostEvent(Owner, Owner, eventname, 0, 
                         TVP_EPT_IMMEDIATE, 0, nullptr);
        }
    }
    
    // 递归通知子节点
    TVP_LAYER_FOR_EACH_CHILD_BEGIN(child)
        child->NotifyNodeEnabledState();
    TVP_LAYER_FOR_EACH_CHILD_END
}
```

## 事件触发机制

图层系统通过 `Fire*` 系列方法将底层输入事件转换为 TJS2 脚本可处理的事件。

### TVPPostEvent 事件投递

所有 Fire* 方法最终都调用 `TVPPostEvent` 来投递事件：

```cpp
// 事件投递函数原型
void TVPPostEvent(
    iTJSDispatch2* source,     // 事件源对象
    iTJSDispatch2* target,     // 事件目标对象
    const ttstr& eventname,    // 事件名称（如 "onClick"）
    tjs_uint32 tag,            // 事件标签（通常为 0）
    tjs_uint32 flags,          // 事件标志
    tjs_int numparams,         // 参数数量
    tTJSVariant* params        // 参数数组
);

// 常用事件标志
#define TVP_EPT_IMMEDIATE    0x0001  // 立即投递（同步执行）
#define TVP_EPT_DISCARDABLE  0x0002  // 可丢弃（队列满时丢弃旧事件）
```

### 命中测试实现

```cpp
// 文件：cpp/core/visual/LayerIntf.cpp 第 3196-3276 行

bool tTJSNI_BaseLayer::_HitTestNoVisibleCheck(tjs_int x, tjs_int y) {
    if(HitType == htMask) {
        // 使用主图像的 Alpha 通道进行命中测试
        if(MainImage) {
            tjs_int px = x - ImageLeft, py = y - ImageTop;
            if(px >= 0 && py >= 0 && 
               px < (tjs_int)MainImage->GetWidth() &&
               py < (tjs_int)MainImage->GetHeight()) {
                if(HitThreshold <= 0)
                    return true;  // 阈值为 0，任何位置都命中
                
                // 获取该像素的 Alpha 值
                tjs_uint32 cl = MainImage->GetPoint(px, py);
                if((tjs_int)(cl >> 24) < HitThreshold)
                    return false;  // Alpha 低于阈值，不命中
                else
                    return true;
            } else {
                return false;  // 超出图像范围
            }
        } else {
            // 图层没有图像，视为全透明
            if(x >= 0 && y >= 0 && 
               x < Rect.get_width() && y < Rect.get_height()) {
                return (0 >= HitThreshold);  // 阈值为 0 时命中
            }
            return false;
        }
    } else if(HitType == htProvince) {
        // 使用 Province 图像进行命中测试
        if(ProvinceImage) {
            tjs_int px = x - ImageLeft, py = y - ImageTop;
            if(px >= 0 && py >= 0 && 
               px < (tjs_int)ProvinceImage->GetWidth() &&
               py < (tjs_int)ProvinceImage->GetHeight()) {
                tjs_uint32 cl = ProvinceImage->GetPoint(px, py);
                return (cl != 0);  // Province 值非 0 则命中
            }
            return false;
        }
        return false;
    }
    
    return true;  // 默认命中
}

// 带脚本回调的命中测试
bool tTJSNI_BaseLayer::HitTestNoVisibleCheck(tjs_int x, tjs_int y) {
    bool res = _HitTestNoVisibleCheck(x, y);
    
    if(res) {
        // 调用 onHitTest 事件进行额外判断
        if(Owner && !Shutdown) {
            OnHitTest_Work = true;
            
            tTJSVariant param[3];
            param[0] = x;
            param[1] = y;
            param[2] = true;  // 默认命中结果
            static ttstr eventname(TJS_W("onHitTest"));
            TVPPostEvent(Owner, Owner, eventname, 0, 
                         TVP_EPT_IMMEDIATE, 3, param);
            
            res = OnHitTest_Work;  // 脚本可修改此值
        }
    }
    
    return res;
}
```

### 获取最前端图层

从图层树中找到指定坐标处最前端（Z-Order 最高）的可响应图层：

```cpp
// 文件：cpp/core/visual/LayerIntf.cpp 第 3279-3352 行

bool tTJSNI_BaseLayer::GetMostFrontChildAt(
    tjs_int x, tjs_int y,
    tTJSNI_BaseLayer **lay,
    const tTJSNI_BaseLayer *except,
    bool get_disabled) 
{
    // 1. 可见性检查
    if(!Visible)
        return false;
    
    // 2. 坐标转换（从父坐标系到本地坐标系）
    x -= Rect.left;
    y -= Rect.top;
    
    // 3. 矩形边界检查
    if(x < 0 || y < 0 || 
       x >= Rect.get_width() || y >= Rect.get_height())
        return false;
    
    tjs_int ox = x, oy = y;
    
    // 4. 逆序遍历子节点（后绘制的在前）
    {
        tObjectListSafeLockHolder<tTJSNI_BaseLayer> holder(Children);
        tjs_int i = Children.GetSafeLockedObjectCount() - 1;
        for(; i >= 0; i--) {
            tTJSNI_BaseLayer *child = Children.GetSafeLockedObjectAt(i);
            if(!child) continue;
            
            // 递归检查子节点
            bool b = child->GetMostFrontChildAt(x, y, lay, except, get_disabled);
            if(b) return true;
        }
    }
    
    // 5. 排除项检查
    if(except == this)
        return false;
    
    // 6. 自身命中测试
    if(HitTestNoVisibleCheck(ox, oy)) {
        // 检查是否被禁用或被模态层遮挡
        if(!get_disabled && (!GetNodeEnabled() || IsDisabledByMode())) {
            *lay = nullptr;
            return true;  // 命中但不可交互
        }
        *lay = this;
        return true;
    }
    
    return false;
}
```

### 焦点遍历

```cpp
// 文件：cpp/core/visual/LayerIntf.cpp 第 3598-3631 行

tTJSNI_BaseLayer *tTJSNI_BaseLayer::_GetNextFocusable() {
    tTJSNI_BaseLayer *p = this;
    tTJSNI_BaseLayer *current = this;
    
    // 获取 Z-Order 中的下一个图层
    p = p->GetNeighborBelow(true);  // true = 循环
    if(current == p) return nullptr;
    if(!p) return nullptr;
    
    current = p;
    do {
        // 检查是否可聚焦且加入焦点链
        if(p->GetNodeFocusable() && p->JoinFocusChain)
            return p;
    } while(p = p->GetNeighborBelow(true), p && p != current);
    
    return nullptr;
}

// 带事件通知的版本
tTJSNI_BaseLayer *tTJSNI_BaseLayer::GetNextFocusable() {
    FocusWork = _GetNextFocusable();
    
    // 触发 onSearchNextFocusable 事件，允许脚本覆盖
    if(Owner && !Shutdown && (!FocusWork || FocusWork->Owner)) {
        static ttstr eventname(TJS_W("onSearchNextFocusable"));
        tTJSVariant param[1];
        if(FocusWork)
            param[0] = tTJSVariant(FocusWork->Owner, FocusWork->Owner);
        else
            param[0] = tTJSVariant((iTJSDispatch2 *)nullptr);
        TVPPostEvent(Owner, Owner, eventname, 0, 
                     TVP_EPT_IMMEDIATE, 1, param);
    }
    return FocusWork;
}
```

```cpp
void tTJSNI_BaseLayer::FireFocus(tTJSNI_BaseLayer *prevfocused,
                                  bool direction) {
    if(Owner && !Shutdown) {
        static ttstr eventname(TJS_W("onFocus"));
        tTJSVariant param[2];
        if(prevfocused)
            param[0] = tTJSVariant(prevfocused->Owner, prevfocused->Owner);
        else
            param[0] = tTJSVariant((iTJSDispatch2 *)nullptr);
        param[1] = direction;  // true=向前, false=向后
        TVPPostEvent(Owner, Owner, eventname, 0, 
                     TVP_EPT_IMMEDIATE, 2, param);
    }
}
```

```cpp
void tTJSNI_BaseLayer::DefaultKeyDown(tjs_uint key, tjs_uint32 shift) {
    if(!Manager) return;
    
    bool no_shift_downed = !(shift & TVP_SS_SHIFT) && 
                           !(shift & TVP_SS_ALT) &&
                           !(shift & TVP_SS_CTRL);
    
    // Tab / 右 / 下：下一个可聚焦图层
    if((key == VK_TAB || key == VK_RIGHT || key == VK_DOWN) &&
       no_shift_downed) {
        Manager->FocusNext();
    }
    // Shift+Tab / 左 / 上：上一个可聚焦图层
    else if((key == VK_TAB && (shift & TVP_SS_SHIFT) &&
             !(shift & TVP_SS_ALT) && !(shift & TVP_SS_CTRL)) ||
            key == VK_LEFT || key == VK_UP) {
        Manager->FocusPrev();
    }
    // Enter / Esc：传递给父图层
    else if((key == VK_RETURN || key == VK_ESCAPE) && no_shift_downed) {
        if(Parent) {
            if(Parent->GetNodeEnabled())
                Parent->FireKeyDown(key, shift);
        }
    }
}
```

```cpp
            layerStartX = left;
            layerStartY = top;

    function onMouseMove(x, y, shift) {
        if(dragging) {
            // 计算新位置
            var newLeft = layerStartX + (x - dragStartX);
            var newTop = layerStartY + (y - dragStartY);
            setPos(newLeft, newTop);
        }
    }
    
    function onMouseUp(x, y, button, shift) {
        if(button == mbLeft) {
            dragging = false;
        }
    }
    
    function onMouseLeave() {
        // 鼠标离开时停止拖拽（防止鼠标快速移动丢失）
        dragging = false;
    }
}
```

```cpp
// ScalableImage.tjs
class ScalableImage extends Layer {
    var baseScale = 1.0;
    var currentScale = 1.0;
    var imageLayer;
    
    function ScalableImage(window, parent, imagePath) {
        super.Layer(window, parent);
        setSize(400, 400);
        
        // 子图层显示实际图像
        imageLayer = new Layer(window, this);
        imageLayer.loadImages(imagePath);
        imageLayer.setSizeToImageSize();
        centerImage();
    }
    
    function centerImage() {
        imageLayer.setPos(
            (width - imageLayer.width) / 2,
            (height - imageLayer.height) / 2
        );
    }
    
    function onTouchScaling(startDist, curDist, cx, cy, flag) {
        if(flag == 0) {
            // 开始缩放
            baseScale = currentScale;
        } else if(flag == 1) {
            // 缩放进行中
            var scale = baseScale * (curDist / startDist);
            scale = Math.max(0.5, Math.min(3.0, scale));  // 限制范围
            
            if(scale != currentScale) {
                currentScale = scale;
                applyScale();
            }
        }
        // flag == 2 时缩放结束，无需特殊处理
    }
    
    function applyScale() {
        var newWidth = int(imageLayer.imageWidth * currentScale);
        var newHeight = int(imageLayer.imageHeight * currentScale);
        imageLayer.setSize(newWidth, newHeight);
        centerImage();
    }
}
```

### 题目 1：属性传播顺序

当调用 `parentLayer.visible = false` 时，以下哪些事件/操作会发生？按顺序排列。

A. 子图层的 GetNodeVisible() 返回值变化
B. 触发脏区更新
C. 调用 NotifyChildrenVisualStateChanged
D. 子图层自身的 Visible 属性变化

<details>
<summary>查看答案</summary>

正确顺序：**B → (属性修改) → C → A**

1. 首先调用 `Update(false)` 标记脏区（B）
2. 修改 `Visible = false`
3. 调用 `Parent->NotifyChildrenVisualStateChanged()`（C）
4. 子图层的 `GetNodeVisible()` 现在返回 false（A），因为 `GetParentVisible()` 返回 false

**D 不会发生**：父图层设置 visible=false 不会改变子图层自身的 Visible 属性，只影响 GetNodeVisible() 的计算结果。

### 题目 3：实现复合命中区域

实现一个 L 形图层，只有 L 形区域响应点击。提示：使用 onHitTest 事件。

```cpp
// LShapedLayer.tjs
class LShapedLayer extends Layer {
    // L 形的尺寸参数
    var totalWidth = 100;
    var totalHeight = 100;
    var armWidth = 30;     // L 形臂宽
    var armHeight = 30;
    
    function LShapedLayer(window, parent) {
        super.Layer(window, parent);
        setSize(totalWidth, totalHeight);
        
        // 绘制 L 形
        face = dfAlpha;
        fillRect(0, 0, width, height, 0x00000000);  // 透明背景
        
        // 竖直部分（左侧）
        colorRect(0, 0, armWidth, height, 0xFF4080FF, 255);
        
        // 水平部分（底部）
        colorRect(0, height - armHeight, width, armHeight, 0xFF4080FF, 255);
    }
    
    function onHitTest(x, y, hit) {
        // 检查是否在 L 形区域内
        var inVerticalArm = (x >= 0 && x < armWidth && 
                             y >= 0 && y < totalHeight);
        var inHorizontalArm = (x >= 0 && x < totalWidth && 
                               y >= totalHeight - armHeight && 
                               y < totalHeight);
        
        hit = inVerticalArm || inHorizontalArm;
    }
    
    function onClick(x, y) {
        System.inform("L 形区域被点击！位置: (" + x + ", " + y + ")");
    }
}
```

**关键点：**
- onHitTest 的第三个参数 `hit` 是引用，修改它会影响最终的命中结果
- L 形由两个矩形区域组成：竖直臂和水平臂
- 点击透明区域（L 形外部）不会触发 onClick
