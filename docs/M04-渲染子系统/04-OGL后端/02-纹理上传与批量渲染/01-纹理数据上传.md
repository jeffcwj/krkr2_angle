## 纹理上传概述

在 OpenGL 渲染系统中，纹理数据需要从 CPU 内存传输到 GPU 显存才能被渲染管线使用。KrKr2 的 OGL 后端将这一过程封装为两个核心函数：

| 函数 | 用途 | 调用时机 |
|------|------|----------|
| `InternalInit` | 初始化纹理，分配显存并上传初始数据 | 创建新纹理时 |
| `InternalUpdate` | 局部更新纹理数据 | 纹理内容变化时（如动画、文字渲染） |

```
┌─────────────────────────────────────────────────────────────────┐
│                    纹理生命周期                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   创建纹理                                                       │
│      │                                                          │
│      ▼                                                          │
│  ┌─────────────────┐                                            │
│  │  InternalInit   │  ──────▶  分配显存 + 上传初始数据           │
│  └────────┬────────┘                                            │
│           │                                                      │
│           ▼                                                      │
│  ┌─────────────────────────────────┐                            │
│  │     纹理正常使用（渲染/采样）      │                            │
│  └────────┬────────────────────────┘                            │
│           │                                                      │
│           ▼                                                      │
│  ┌─────────────────┐                                            │
│  │ InternalUpdate  │  ◀──────  内容更新（动画帧、文字变化等）      │
│  │   (可多次调用)   │                                            │
│  └────────┬────────┘                                            │
│           │                                                      │
│           ▼                                                      │
│  ┌─────────────────┐                                            │
│  │   销毁纹理       │  ──────▶  释放显存                         │
│  └─────────────────┘                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 纹理初始化：InternalInit

### 函数签名与职责

`InternalInit` 是纹理对象的初始化入口，负责：

1. 根据纹理格式选择正确的 OpenGL 像素格式
2. 设置 `GL_UNPACK_ALIGNMENT` 对齐参数
3. 调用 `glTexImage2D` 分配显存并上传数据
4. 记录内部尺寸用于后续坐标计算

### 源码分析

```cpp
// 来源：cpp/core/visual/ogl/RenderManager_ogl.cpp 第 913-961 行
void InternalInit(const void *pixel, unsigned int intw, unsigned int inth,
                  unsigned int pitch) {
    // 第一步：根据纹理格式选择像素格式
    GLenum pixfmt = GL_RGBA;        // OpenGL 像素格式（源数据）
    GLenum internalfmt = GL_RGBA;   // OpenGL 内部格式（GPU 存储）
    
    switch(Format) {
        case TVPTextureFormat::Gray:
            pixfmt = GL_LUMINANCE;       // 灰度图：单通道亮度
            internalfmt = GL_LUMINANCE;
            break;
        case TVPTextureFormat::RGB:
            pixfmt = GL_RGB;             // RGB：3 通道无 Alpha
            internalfmt = GL_RGB;
            break;
        case TVPTextureFormat::RGBA:
            pixfmt = GL_RGBA;            // RGBA：4 通道完整格式
            internalfmt = GL_RGBA;
            break;
        default:
            break;
    }

    // 第二步：根据 pitch 对齐设置 GL_UNPACK_ALIGNMENT
    // pitch & 7 获取最后 3 位，判断对齐情况
    switch(pitch & 7) {
        case 0:
            glPixelStorei(GL_UNPACK_ALIGNMENT, 8);  // 8 字节对齐
            break;
        case 2:
        case 6:
            glPixelStorei(GL_UNPACK_ALIGNMENT, 2);  // 2 字节对齐
            break;
        case 4:
            glPixelStorei(GL_UNPACK_ALIGNMENT, 4);  // 4 字节对齐
            break;
        default:
            glPixelStorei(GL_UNPACK_ALIGNMENT, 1);  // 无对齐要求
            break;
    }

    // 第三步：检查内存压力，必要时释放缓存
    TVPCheckMemory();
    
    // 第四步：绑定纹理对象
    _glBindTexture2D(texture);
    
    // 第五步：上传纹理数据到 GPU
    glTexImage2D(GL_TEXTURE_2D, 0, internalfmt, intw, inth, 0, pixfmt,
                 GL_UNSIGNED_BYTE, pixel);

    // 第六步：记录内部尺寸，用于 UV 坐标计算
    internalW = intw;
    internalH = inth;
    
    // 统计显存使用量
    _totalVMemSize += internalW * internalH * getPixelSize();
    
    CHECK_GL_ERROR_DEBUG();  // 调试模式下检查 GL 错误
}
```

### 关键概念解析

#### 1. 像素格式（Pixel Format）

| 格式 | 通道数 | 每像素字节 | 用途 |
|------|--------|-----------|------|
| `GL_LUMINANCE` | 1 | 1 | 灰度图、遮罩、Alpha 通道 |
| `GL_RGB` | 3 | 3 | 不透明图像（背景） |
| `GL_RGBA` | 4 | 4 | 带透明通道图像（角色立绘） |

#### 2. GL_UNPACK_ALIGNMENT 对齐参数

`GL_UNPACK_ALIGNMENT` 控制 OpenGL 如何从 CPU 内存读取像素数据。默认值为 4，表示每行数据按 4 字节对齐。

```cpp
// 对齐检测逻辑解析
switch(pitch & 7) {  // pitch & 7 等价于 pitch % 8
    case 0:  // pitch 能被 8 整除 → 8 字节对齐
        glPixelStorei(GL_UNPACK_ALIGNMENT, 8);
        break;
    case 4:  // pitch 能被 4 整除但不能被 8 整除 → 4 字节对齐
        glPixelStorei(GL_UNPACK_ALIGNMENT, 4);
        break;
    case 2:
    case 6:  // pitch 能被 2 整除但不能被 4 整除 → 2 字节对齐
        glPixelStorei(GL_UNPACK_ALIGNMENT, 2);
        break;
    default: // pitch 是奇数 → 无对齐（1 字节）
        glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
        break;
}
```

**常见错误**：如果对齐设置错误，会导致图像出现斜纹或错位：

```
正确对齐：                     错误对齐（应该是 1，设成了 4）：
┌──────────────┐              ┌──────────────┐
│ ABCDABCDABCD │              │ ABCDABCD│BCD │
│ ABCDABCDABCD │   →          │ ABCDA│CDABCD │
│ ABCDABCDABCD │              │ AB│DABCDABCD │
└──────────────┘              └──────────────┘
```

## 纹理更新：InternalUpdate

### 局部更新的必要性

对于可变纹理（如文字渲染、动画），完全重新上传整个纹理效率低下。`InternalUpdate` 支持局部更新：

- 只更新变化的区域
- 减少 CPU→GPU 数据传输
- 适用于文字图层、动态 UI

### 源码分析

```cpp
// 来源：cpp/core/visual/ogl/RenderManager_ogl.cpp 第 963-1007 行
void InternalUpdate(const void *pixel, int pitch, int x, int y, int w, int h) {
    // 如果纹理被缩放过，先恢复原始尺寸
    if(_scaleW != 1.f || _scaleH != 1.f) {
        RestoreNormalSize();
    }
    
    const unsigned char *src = (const unsigned char *)pixel;
    GLenum pixfmt;
    _glBindTexture2D(texture);
    unsigned int pixsize = Format & 0xF;  // 从格式枚举提取像素大小
    
    // 选择像素格式并设置对齐
    switch(Format) {
        case TVPTextureFormat::Gray:
            pixfmt = GL_LUMINANCE;
            glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
            break;
        case TVPTextureFormat::RGB:
            pixfmt = GL_RGB;
            glPixelStorei(GL_UNPACK_ALIGNMENT, 4);
            break;
        case TVPTextureFormat::RGBA:
            pixfmt = GL_RGBA;
            glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
            break;
        default:
            return;
    }
    
    bool rearranged = false;
    
    // GL_UNPACK_ROW_LENGTH 扩展处理
    if(GL_CHECK_unpack_subimage) {
        // 扩展可用：直接设置行宽度，无需复制数据
        glPixelStorei(GL_UNPACK_ROW_LENGTH, pitch / pixsize);
    } else if(pitch != w * pixsize) {
        // 扩展不可用且数据非连续：需要手动重排数据
        rearranged = true;
        unsigned int arrpitch = w * pixsize;
        unsigned char *arr = new unsigned char[arrpitch * h], *p = arr;
        for(int i = 0; i < h; ++i, p += arrpitch, src += pitch)
            memcpy(p, src, arrpitch);
        src = arr;
    }
    
    // 执行局部更新（核心调用）
    glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h, pixfmt, GL_UNSIGNED_BYTE, src);
    
    // 清理状态
    if(GL_CHECK_unpack_subimage)
        glPixelStorei(GL_UNPACK_ROW_LENGTH, 0);
    if(rearranged)
        delete[] src;
    
    CHECK_GL_ERROR_DEBUG();
}
```

### glTexImage2D vs glTexSubImage2D

| 函数 | 用途 | 是否分配显存 |
|------|------|-------------|
| `glTexImage2D` | 初始化纹理，设置尺寸和格式 | ✅ 分配 |
| `glTexSubImage2D` | 更新已有纹理的部分区域 | ❌ 使用已有显存 |

```cpp
// glTexImage2D：创建 512x512 纹理
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, 512, 512, 0, GL_RGBA, 
             GL_UNSIGNED_BYTE, initialData);

// glTexSubImage2D：更新 (100,100) 位置的 64x64 区域
glTexSubImage2D(GL_TEXTURE_2D, 0, 100, 100, 64, 64, GL_RGBA,
                GL_UNSIGNED_BYTE, updateData);
```

## GL_UNPACK_ROW_LENGTH 扩展详解

### 问题背景：非连续内存布局

在实际应用中，图像数据通常不是紧密排列的。例如：

```
┌───────────────────────────────────────┐
│ 实际内存布局（pitch > width * pixelSize）│
├───────────────────────────────────────┤
│                                       │
│  +-------------------+----+           │
│  │ 图像像素数据       │ 填充 │  ← 行 0   │
│  +-------------------+----+           │
│  │ 图像像素数据       │ 填充 │  ← 行 1   │
│  +-------------------+----+           │
│  │ 图像像素数据       │ 填充 │  ← 行 2   │
│  +-------------------+----+           │
│                                       │
│  实际宽度: width * 4 字节 (RGBA)        │
│  行间距 (pitch): 通常为 16/32 字节对齐   │
│                                       │
└───────────────────────────────────────┘
```

这种布局常见于：
- 从更大图像截取的子区域
- 内存对齐优化的位图（Windows BMP）
- 视频帧数据（FFmpeg AVFrame）
- DMA 传输优化的缓冲区

### 传统解决方案 vs 扩展方案

#### 方案一：手动复制（无扩展时）

当 `GL_EXT_unpack_subimage` 扩展不可用时，必须手动将数据重排为紧密布局：

```cpp
// 检测到需要重排
if(pitch != w * pixsize) {
    // 分配临时缓冲区（紧密排列）
    unsigned char *arr = new unsigned char[w * pixsize * h];
    unsigned char *dst = arr;
    const unsigned char *src = pixel;
    
    // 逐行复制有效数据，跳过填充字节
    for(int i = 0; i < h; ++i) {
        memcpy(dst, src, w * pixsize);  // 只复制有效像素
        dst += w * pixsize;             // 目标紧密排列
        src += pitch;                    // 源跳过填充字节
    }
    
    // 上传紧密排列的数据
    glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h, pixfmt, 
                    GL_UNSIGNED_BYTE, arr);
    delete[] arr;
}
```

**缺点**：
- 额外的内存分配（`new`）
- 额外的内存复制（`memcpy` 循环）
- 对大纹理影响显著

#### 方案二：使用扩展（高效）

`GL_EXT_unpack_subimage` 扩展允许直接告诉 OpenGL 源数据的行间距：

```cpp
if(GL_CHECK_unpack_subimage) {
    // 告诉 OpenGL 每行的像素数（不是字节数！）
    glPixelStorei(GL_UNPACK_ROW_LENGTH, pitch / pixsize);
    
    // 直接上传，GL 会正确跳过填充
    glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h, pixfmt, 
                    GL_UNSIGNED_BYTE, pixel);
    
    // 恢复默认值（0 表示紧密排列）
    glPixelStorei(GL_UNPACK_ROW_LENGTH, 0);
}
```

**优点**：
- 零额外内存分配
- 零额外复制
- GPU 驱动优化处理

### 扩展检测实现

```cpp
// 来源：cpp/core/visual/ogl/RenderManager_ogl.cpp
// 扩展检测宏
#define GL_CHECK_unpack_subimage TVPCheckGLExtension("GL_EXT_unpack_subimage")

// 扩展检测函数实现
bool TVPCheckGLExtension(const char *name) {
    static std::set<std::string> extensionSet;
    static bool initialized = false;
    
    if(!initialized) {
        // 解析 GL_EXTENSIONS 字符串（空格分隔）
        const char *ext = (const char *)glGetString(GL_EXTENSIONS);
        if(ext) {
            std::istringstream stream(ext);
            std::string token;
            while(stream >> token) {
                extensionSet.insert(token);
            }
        }
        initialized = true;
    }
    
    return extensionSet.count(name) > 0;
}
```

### 平台差异

| 平台 | GL_EXT_unpack_subimage 支持 | 备注 |
|------|----------------------------|------|
| Windows (Desktop GL) | ✅ 核心功能（GL 1.1+） | 无需检测 |
| Linux (Desktop GL) | ✅ 核心功能 | 无需检测 |
| macOS (Desktop GL) | ✅ 核心功能 | 无需检测 |
| Android (OpenGL ES 2.0) | ⚠️ 需要扩展 | 部分 GPU 不支持 |
| Android (OpenGL ES 3.0+) | ✅ 核心功能 | ES 3.0 内置 |
| iOS (OpenGL ES) | ⚠️ 需要扩展 | 已废弃，建议用 Metal |

## 本节小结

- **InternalInit** 负责纹理创建和初始数据上传，需正确设置像素格式和对齐参数
- **InternalUpdate** 支持局部更新，避免完全重传提升性能
- **GL_UNPACK_ALIGNMENT** 必须与源数据的行对齐匹配，否则图像会出现错位
- **GL_UNPACK_ROW_LENGTH** 扩展可高效处理非连续内存，但需运行时检测
- 移动平台（OpenGL ES 2.0）可能不支持该扩展，需要兼容代码路径

## 练习题与答案

### 题目 1：GL_UNPACK_ALIGNMENT 设置

**问题**：一个 100x100 的 RGB 图像（3 字节/像素），每行末尾有 2 字节填充（pitch = 302）。应该如何设置 `GL_UNPACK_ALIGNMENT`？

<details>
<summary>查看答案</summary>

**分析**：
- 图像宽度：100 像素
- 每像素字节：3 (RGB)
- 有效行字节：100 * 3 = 300
- 行间距 pitch：302
- 填充：302 - 300 = 2 字节

**计算对齐**：
```cpp
pitch & 7 = 302 & 7 = 302 % 8 = 6
```

6 对应 `case 2: case 6:`，应设置：
```cpp
glPixelStorei(GL_UNPACK_ALIGNMENT, 2);
```

**验证**：302 能被 2 整除（302 / 2 = 151），设置正确。

</details>

### 题目 2：扩展不可用时的处理

**问题**：当 `GL_EXT_unpack_subimage` 扩展不可用时，如何高效上传一个 pitch > width * pixelSize 的纹理子区域？

<details>
<summary>查看答案</summary>

**方案**：手动重排数据为紧密布局后上传

```cpp
void uploadSubregion(const uint8_t* srcData, int srcPitch,
                     int x, int y, int w, int h, int pixelSize) {
    // 检查是否需要重排
    int dstPitch = w * pixelSize;  // 紧密布局的行字节数
    
    if(srcPitch == dstPitch) {
        // 已经是紧密布局，直接上传
        glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                       GL_RGBA, GL_UNSIGNED_BYTE, srcData);
        return;
    }
    
    // 分配临时缓冲区
    std::vector<uint8_t> temp(dstPitch * h);
    
    // 逐行复制
    const uint8_t* src = srcData;
    uint8_t* dst = temp.data();
    for(int row = 0; row < h; ++row) {
        memcpy(dst, src, dstPitch);
        src += srcPitch;  // 源：跳过填充
        dst += dstPitch;  // 目标：紧密排列
    }
    
    // 上传重排后的数据
    glPixelStorei(GL_UNPACK_ALIGNMENT, 1);  // 使用最小对齐
    glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                   GL_RGBA, GL_UNSIGNED_BYTE, temp.data());
}
```

**优化建议**：
- 使用内存池避免频繁 `new/delete`
- 对于固定大小的更新区域，可预分配缓冲区
- 考虑使用 SIMD（SSE/NEON）加速 memcpy

</details>
