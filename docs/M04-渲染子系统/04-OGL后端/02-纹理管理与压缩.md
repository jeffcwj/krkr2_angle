## 4.2.1 iTVPTexture2D 接口

`iTVPTexture2D` 定义在 `RenderManager.h` 第 99-147 行，是所有纹理对象的抽象基类。它采用引用计数管理生命周期，同时提供 CPU 端和 GPU 端两套访问接口。

引用计数是纹理管理的核心机制。纹理对象可能被多个图层共享（例如多个子图层引用同一张背景图），引用计数确保最后一个使用者释放时才真正销毁 GL 纹理资源。

```cpp
// 源码位置：RenderManager.h:99-147
class iTVPTexture2D {
public:
    // ---------- 生命周期管理 ----------
    virtual void AddRef() = 0;      // 增加引用计数
    virtual void Release() = 0;     // 减少引用计数，归零时销毁

    // ---------- 尺寸查询 ----------
    virtual unsigned int GetWidth() = 0;    // 逻辑宽度（原始图像尺寸）
    virtual unsigned int GetHeight() = 0;   // 逻辑高度
    // 注意：GL 纹理的实际尺寸可能大于逻辑尺寸（NPOT 对齐）

    // ---------- CPU 端像素访问 ----------
    // 获取第 y 行的只读指针（用于软件渲染回读）
    virtual const void* GetScanLineForRead(unsigned int y) = 0;
    // 获取第 y 行的可写指针（用于 CPU 端修改纹理）
    virtual void* GetScanLineForWrite(unsigned int y) = 0;
    // 标记区域已修改，下次 GPU 使用前需重新上传
    virtual void Update(const tTVPRect& rc) = 0;
    // 单像素访问
    virtual unsigned int GetPoint(int x, int y) = 0;
    virtual void SetPoint(int x, int y, unsigned int color) = 0;

    // ---------- 属性查询 ----------
    virtual bool IsStatic() = 0;    // 静态纹理（创建后不再修改）
    virtual bool IsOpaque() = 0;    // 不透明纹理（可优化为 RGB 格式）
    virtual float GetScale() = 0;   // 缩放因子（half 模式为 0.5）

    // ---------- GPU 端访问 ----------
    // 返回平台适配的纹理对象（cocos2d::Texture2D*）
    virtual void* GetAdapterTexture() = 0;

    // ---------- 资源回收 ----------
    virtual void RecycleProcess() = 0;  // 周期性调用，释放长期未使用的 CPU 缓存
};
```

### 静态 vs 可变纹理

KrKr2 区分两类纹理，因为它们的内存策略完全不同：

| 特性 | 静态纹理 | 可变纹理 |
|------|----------|----------|
| 创建后修改 | 不允许 | 允许（通过 GetScanLineForWrite） |
| CPU 缓存 | 可在上传 GPU 后释放 | 必须保留（用于回读和重新上传） |
| 典型用途 | 背景图、立绘、UI 素材 | 图层画布、临时绘制目标 |
| 压缩支持 | 可使用 ETC2/PVRTC | 不支持（需要随机写入） |
| 显存优化 | IsOpaque 时使用 RGB（省 25%） | 始终 RGBA |

## 4.2.2 tTVPOGLTexture2D 实现

`tTVPOGLTexture2D` 是 OpenGL 后端的标准纹理实现，定义在 `RenderManager_ogl.cpp` 第 862-1145 行。它封装了一个 GL 纹理名称（`GLuint texture`）和可选的 CPU 端像素缓存。

### 核心数据结构

```cpp
// 源码位置：RenderManager_ogl.cpp:862-920
class tTVPOGLTexture2D : public iTVPTexture2D {
private:
    GLuint texture;          // GL 纹理名称
    bool IsCompressed;       // 是否为压缩格式（ETC2/PVRTC）
    TVPTextureFormat::e Format;  // 像素格式
    unsigned int Width, Height;   // 逻辑尺寸
    unsigned int internalW, internalH;  // GL 纹理实际尺寸（可能因 NPOT 对齐而更大）
    std::vector<uint8_t> PixelData;     // CPU 端像素缓存
    float _scaleW, _scaleH;  // 缩放因子（half 模式下为 0.5）

    // cocos2d 适配器（延迟创建）
    class AdapterTexture2D : public cocos2d::Texture2D {
        // 将 GL 纹理名称包装为 cocos2d 纹理对象
    };
    AdapterTexture2D* _adapterTexture;
};
```

### GL 纹理创建流程

纹理创建在构造函数中完成，核心调用链是 `glGenTextures → glBindTexture → glTexImage2D`：

```cpp
// 简化的构造流程（源码位置：RenderManager_ogl.cpp:925-980）
tTVPOGLTexture2D::tTVPOGLTexture2D(
    const void* pixel, int pitch,
    unsigned int w, unsigned int h,
    TVPTextureFormat::e format)
{
    Width = w; Height = h;
    Format = format;

    // 1. 生成 GL 纹理对象
    glGenTextures(1, &texture);
    glBindTexture(GL_TEXTURE_2D, texture);

    // 2. 设置纹理参数
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

    // 3. 选择内部格式
    GLenum glFormat, glInternalFormat;
    switch (format) {
    case TVPTextureFormat::Gray:
        glFormat = GL_LUMINANCE;
        glInternalFormat = GL_LUMINANCE;
        break;
    case TVPTextureFormat::RGB:
        glFormat = GL_RGB;
        glInternalFormat = GL_RGB;
        break;
    case TVPTextureFormat::RGBA:
    default:
        glFormat = GL_RGBA;
        glInternalFormat = GL_RGBA;
        break;
    }

    // 4. 上传纹理数据
    glTexImage2D(GL_TEXTURE_2D, 0, glInternalFormat,
                 w, h, 0, glFormat, GL_UNSIGNED_BYTE, nullptr);

    // 5. 如果提供了像素数据，立即更新
    if (pixel) {
        InternalUpdate(pixel, pitch, 0, 0, w, h);
    }
}
```

### 纹理数据上传（InternalUpdate）

`InternalUpdate` 负责将 CPU 端像素数据传输到 GPU。这里有一个关键的跨平台差异：`GL_UNPACK_ROW_LENGTH` 扩展。

```cpp
// 源码位置：RenderManager_ogl.cpp:1000-1060
void tTVPOGLTexture2D::InternalUpdate(
    const void* data, int pitch,
    int x, int y, int w, int h)
{
    glBindTexture(GL_TEXTURE_2D, texture);

    if (GL_CHECK_unpack_subimage) {
        // 方式A：有 GL_EXT_unpack_subimage 扩展
        // 直接设置行跨距，一次调用上传所有数据
        int bytesPerPixel = (Format == TVPTextureFormat::Gray) ? 1 : 4;
        glPixelStorei(GL_UNPACK_ROW_LENGTH, pitch / bytesPerPixel);
        glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                        GL_RGBA, GL_UNSIGNED_BYTE, data);
        glPixelStorei(GL_UNPACK_ROW_LENGTH, 0);  // 恢复默认
    } else {
        // 方式B：无扩展支持（某些 OpenGL ES 2.0 设备）
        // 需要逐行上传或先重新排列数据到紧凑布局
        int rowBytes = w * 4;
        if (pitch == rowBytes) {
            // 行跨距恰好等于行宽度，直接上传
            glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                            GL_RGBA, GL_UNSIGNED_BYTE, data);
        } else {
            // 重新排列到紧凑内存
            std::vector<uint8_t> compact(w * h * 4);
            const uint8_t* src = (const uint8_t*)data;
            for (int row = 0; row < h; ++row) {
                memcpy(&compact[row * rowBytes], src + row * pitch, rowBytes);
            }
            glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                            GL_RGBA, GL_UNSIGNED_BYTE, compact.data());
        }
    }
}
```

### 跨平台差异

| 平台 | GL_UNPACK_ROW_LENGTH | NPOT 纹理 | 最大纹理尺寸 |
|------|---------------------|-----------|-------------|
| Windows (OpenGL 3.x) | ✅ 核心功能 | ✅ 完全支持 | 通常 16384 |
| Linux (OpenGL 3.x) | ✅ 核心功能 | ✅ 完全支持 | 通常 16384 |
| macOS (OpenGL 3.2) | ✅ 核心功能 | ✅ 完全支持 | 通常 16384 |
| Android (GLES 2.0) | ❌ 需要扩展 | ⚠️ 部分支持 | 通常 4096-8192 |

## 4.2.3 超大纹理分块（tTVPOGLTexture2D_split）

当图像尺寸超过 `GL_MAX_TEXTURE_SIZE` 时（Android 设备常见限制为 4096×4096），引擎使用 `tTVPOGLTexture2D_split` 将图像分割为多个瓦片纹理。

分块策略的判断逻辑在 `CreateStaticTexture2D` 中：

```cpp
// 源码位置：RenderManager_ogl.cpp:2815-2828
static iTVPTexture2D* CreateStaticTexture2D(tTVPBitmap* bmp) {
    tjs_uint w = bmp->GetWidth(), h = bmp->GetHeight();

    // 宽度超过最大纹理2倍 且 支持 unpack_subimage → 使用分块
    if (w > GetMaxTextureWidth()) {
        if (w > GetMaxTextureWidth() * 2 && GL_CHECK_unpack_subimage) {
            return new tTVPOGLTexture2D_split(bmp);
        }
    }
    // 高度超过最大纹理2倍 → 使用分块
    else if (h > GetMaxTextureHeight() * 2) {
        return new tTVPOGLTexture2D_split(bmp);
    }

    // 未超限 → 普通纹理
    return _CreateStaticTexture2D(
        bmp->GetBits(), w, h, bmp->GetPitch(),
        bmp->Is32bit() ? TVPTextureFormat::RGBA : TVPTextureFormat::Gray,
        bmp->IsOpaque);
}
```

`tTVPOGLTexture2D_split` 的核心设计是**按需加载瓦片**：

```cpp
// 源码位置：RenderManager_ogl.cpp:1160-1300（简化）
class tTVPOGLTexture2D_split : public iTVPTexture2D {
private:
    tTVPBitmap* SourceBitmap;  // 保持对原始位图的引用

    // 瓦片缓存：key = (tileX << 16 | tileY)
    struct GLTextureInfo {
        GLuint texName;
        int tileW, tileH;
    };
    std::map<uint32_t, GLTextureInfo> CachedTexture;

    int TileW, TileH;         // 每个瓦片的尺寸
    int TilesX, TilesY;       // 水平/垂直瓦片数

public:
    // 按需获取指定瓦片的纹理
    GLuint GetTileTexture(int tileX, int tileY) {
        uint32_t key = (tileX << 16) | tileY;
        auto it = CachedTexture.find(key);
        if (it != CachedTexture.end()) {
            return it->second.texName;
        }

        // 缓存未命中 → 从原始位图裁剪并创建新纹理
        int srcX = tileX * TileW;
        int srcY = tileY * TileH;
        int w = std::min(TileW, (int)Width - srcX);
        int h = std::min(TileH, (int)Height - srcY);

        GLuint tex;
        glGenTextures(1, &tex);
        // ... 上传裁剪区域的像素数据 ...

        CachedTexture[key] = {tex, w, h};
        return tex;
    }

    // 当分块不可行时，回退到单张缩小纹理
    iTVPTexture2D* AsSingleTexture() {
        // 使用 cv::resize 将整张图缩小到 MaxTextureSize 以内
        // 返回单张纹理（有质量损失）
    }
};
```

瓦片化的过程可以用图表说明：

```
原始图像: 8192 × 4096 像素
GL_MAX_TEXTURE_SIZE = 4096

分块结果：
┌─────────────┬─────────────┐
│ 瓦片(0,0)    │ 瓦片(1,0)    │
│ 4096×4096   │ 4096×4096   │
└─────────────┴─────────────┘

每个瓦片独立上传为一个 GL 纹理
渲染时根据可见区域只加载需要的瓦片
```

**常见错误1：忘记瓦片间的坐标转换**

```cpp
// ❌ 错误：直接使用原始坐标访问分块纹理
glTexSubImage2D(..., globalX, globalY, ...);  // 坐标可能超出瓦片范围

// ✅ 正确：转换为瓦片局部坐标
int tileX = globalX / TileW;
int tileY = globalY / TileH;
int localX = globalX % TileW;
int localY = globalY % TileH;
GLuint tex = GetTileTexture(tileX, tileY);
glBindTexture(GL_TEXTURE_2D, tex);
// 使用 localX, localY 访问
```

## 4.2.4 纹理优化策略

OGL 后端在 `InitGL()` 中根据配置选择纹理创建策略：

```cpp
// 源码位置：RenderManager_ogl.cpp:2879-2899
void InitGL() {
    // 默认策略：原始尺寸
    _CreateStaticTexture2D = CreateStaticTexture2D_normal;

    // 读取用户配置
    std::string method = IndividualConfigManager::GetInstance()
        ->GetValue<std::string>("ogl_compress_tex", "none");

    if (method == "half") {
        // 策略1：半尺寸（2x 缩小）
        _CreateStaticTexture2D = CreateStaticTexture2D_half;
    } else if (method == "etc2") {
        // 策略2：ETC2 压缩（Android 推荐）
        if (TVPIsSupportTextureFormat(GL_COMPRESSED_RGB8_ETC2)) {
            _CreateStaticTexture2D = CreateStaticTexture2D_ETC2;
        }
    } else if (method == "pvrtc") {
        // 策略3：PVRTC 压缩（iOS/某些 GPU）
        if (TVPIsSupportTextureFormat(GL_COMPRESSED_RGBA_PVRTC_4BPPV1_IMG)) {
            _CreateStaticTexture2D = CreateStaticTexture2D_PVRTC;
        }
    }
}
```

四种纹理策略的对比：

| 策略 | 显存占用 | 画质 | CPU 开销 | 适用场景 |
|------|----------|------|----------|----------|
| normal | 100% | 无损 | 最低 | 桌面平台、显存充裕 |
| half | 25% | 2x 缩小 | 低（仅缩放） | 大背景图，移动端节省显存 |
| ETC2 | 12.5-25% | 有损压缩 | 高（CPU 编码） | Android OpenGL ES 3.0+ |
| PVRTC | 12.5% | 有损压缩 | 高（CPU 编码） | iOS/PowerVR GPU |

### ETC2 压缩流程

```cpp
// 源码位置：RenderManager_ogl.cpp:2700-2746（简化）
static iTVPTexture2D* CreateStaticTexture2D_ETC2(
    const void* dib, tjs_uint w, tjs_uint h,
    tjs_int pitch, TVPTextureFormat::e fmt, bool isOpaque)
{
    // 1. 先检查是否为纯色（可用 1×1 纹理替代）
    iTVPTexture2D* ret = CreateStaticTexture2D_solid(dib, w, h, pitch, fmt);
    if (ret) return ret;

    // 2. 检查不透明性（决定使用 RGB 还是 RGBA 格式）
    if (!isOpaque) {
        isOpaque = TVPCheckOpaqueRGBA((const tjs_uint8*)dib, pitch, w, h);
    }

    // 3. 尺寸对齐到 4 的倍数（ETC2 块大小为 4×4）
    int tw = (w + 3) & ~3;
    int th = (h + 3) & ~3;

    // 4. CPU 端编码
    uint8_t* pixel; size_t pvrsize; GLenum texfmt;
    if (isOpaque) {
        pixel = (uint8_t*)ETCPacker::convert(dib, w, h, pitch, true, pvrsize);
        texfmt = GL_COMPRESSED_RGB8_ETC2;       // RGB 8bpp → 4bpp
    } else {
        pixel = (uint8_t*)ETCPacker::convertWithAlpha(
            dib, w, h, pitch, pvrsize);
        texfmt = GL_COMPRESSED_RGBA8_ETC2_EAC;  // RGBA 32bpp → 8bpp
    }

    // 5. 创建压缩纹理（直接传给 glCompressedTexImage2D）
    ret = new tTVPOGLTexture2D_static(
        pixel, pvrsize, texfmt, w, h, tw, th, 1, 1);
    delete[] pixel;
    return ret;
}
```

### 纯色优化

一个非常巧妙的优化：如果整张图片的所有像素颜色相同，只需创建 1×1 的纹理，配合 `GL_CLAMP_TO_EDGE` 采样模式即可正确渲染任意大小的区域。这在视觉小说中很常见（纯色背景、遮罩层）。

```cpp
// CreateStaticTexture2D_solid 的核心逻辑
static iTVPTexture2D* CreateStaticTexture2D_solid(
    const void* dib, tjs_uint w, tjs_uint h,
    tjs_int pitch, TVPTextureFormat::e fmt)
{
    if (TVPCheckSolidPixel(dib, w, h, pitch)) {
        // 整张图是纯色！创建 1×1 纹理即可
        uint32_t color = *(const uint32_t*)dib;
        return new tTVPOGLTexture2D_solid(color, w, h);
        // 逻辑尺寸仍为 w×h，但 GL 纹理只有 1×1
        // 节省 w*h*4 - 4 字节显存
    }
    return nullptr;  // 不是纯色，走正常路径
}
```

**常见错误2：纹理格式与像素数据不匹配**

```cpp
// ❌ 错误：KrKr2 内部使用 BGRA 格式，但传给 GL 的格式标记为 RGBA
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, w, h, 0,
             GL_RGBA, GL_UNSIGNED_BYTE, bgraData);
// 结果：红蓝通道互换，画面色彩异常

// ✅ 正确做法之一：使用 GL_BGRA 扩展（桌面 GL 支持）
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, w, h, 0,
             GL_BGRA, GL_UNSIGNED_BYTE, bgraData);

// ✅ 正确做法之二：在 shader 中交换通道
// gl_FragColor = texture2D(tex, coord).bgra;
```

**常见错误3：释放纹理后继续使用**

```cpp
// ❌ 错误：Release 后继续使用纹理
iTVPTexture2D* tex = mgr->CreateTexture2D(bmp, 0);
tex->Release();
mgr->OperateRect(target, nullptr, ..., tex, ...);  // 悬空指针！

// ✅ 正确：确保引用计数覆盖整个使用周期
iTVPTexture2D* tex = mgr->CreateTexture2D(bmp, 0);
mgr->OperateRect(target, nullptr, ..., tex, ...);
tex->Release();  // 使用完毕后再释放
```

### 实践2：模拟分块纹理的瓦片坐标计算

```cpp
#include <cstdio>
#include <algorithm>

// 模拟瓦片坐标转换（不需要 GL 上下文）
struct TileInfo {
    int tileX, tileY;    // 瓦片索引
    int localX, localY;  // 瓦片内局部坐标
    int tileW, tileH;    // 该瓦片的实际尺寸
};

TileInfo GetTileAt(int globalX, int globalY,
                   int imageW, int imageH, int maxTexSize)
{
    int tileW = maxTexSize;
    int tileH = maxTexSize;
    int tileX = globalX / tileW;
    int tileY = globalY / tileH;

    TileInfo info;
    info.tileX = tileX;
    info.tileY = tileY;
    info.localX = globalX % tileW;
    info.localY = globalY % tileH;
    info.tileW = std::min(tileW, imageW - tileX * tileW);
    info.tileH = std::min(tileH, imageH - tileY * tileH);
    return info;
}

int main() {
    // 模拟 8192×4096 图像，GL_MAX_TEXTURE_SIZE = 4096
    int imageW = 8192, imageH = 4096, maxTex = 4096;

    // 查询点 (5000, 2000) 在哪个瓦片
    auto info = GetTileAt(5000, 2000, imageW, imageH, maxTex);
    printf("全局坐标 (5000, 2000) → 瓦片(%d,%d) 局部(%d,%d) 瓦片尺寸(%d×%d)\n",
           info.tileX, info.tileY,
           info.localX, info.localY,
           info.tileW, info.tileH);
    // 输出：瓦片(1,0) 局部(904,2000) 瓦片尺寸(4096×4096)
    return 0;
}
```

| PvrTcEncoder 接口 | `visual/ogl/pvrtc.h` | 1-6 |

## 本节小结

1. `iTVPTexture2D` 使用引用计数管理生命周期，提供 CPU/GPU 双端访问接口
2. `tTVPOGLTexture2D` 通过 `glGenTextures` + `glTexImage2D` 创建 GL 纹理，`InternalUpdate` 处理像素上传的 pitch 对齐问题
3. 超大纹理通过 `tTVPOGLTexture2D_split` 分割为瓦片，按需加载，未渲染的瓦片不占用显存
4. 四种纹理策略（normal/half/ETC2/PVRTC）通过函数指针在 `InitGL` 中切换，用户可通过配置选择
5. 纯色检测优化将整张纯色图替换为 1×1 纹理，在视觉小说场景中可大幅节省显存

## 练习题与答案

### 练习1：显存计算

**问题**：一张 1920×1080 的 RGBA 背景图在不同策略下分别占用多少显存？假设 ETC2 RGB 的压缩比为 6:1，ETC2 RGBA 为 4:1。

<details>
<summary>参考答案</summary>

- **normal**：1920 × 1080 × 4 = 8,294,400 字节 ≈ 7.9 MB
- **half**：960 × 540 × 4 = 2,073,600 字节 ≈ 2.0 MB（原来的 25%）
- **ETC2 RGB**（若不透明）：1920 × 1080 × 4 / 6 ≈ 1,382,400 字节 ≈ 1.3 MB
  - 更精确计算：ETC2 RGB 每 4×4 块占 8 字节 → (1920/4) × (1080/4) × 8 = 480 × 270 × 8 = 1,036,800 字节 ≈ 1.0 MB
- **ETC2 RGBA**（含透明）：(1920/4) × (1080/4) × 16 = 480 × 270 × 16 = 2,073,600 字节 ≈ 2.0 MB
- **纯色**（如果恰好是纯色）：1 × 1 × 4 = 4 字节

</details>

### 练习2：分块策略分析

**问题**：为什么 `CreateStaticTexture2D` 只在宽度超过 `MaxTextureSize * 2` 时才使用分块，而不是超过 `MaxTextureSize` 就分块？

<details>
<summary>参考答案</summary>

当图像仅略微超出 `MaxTextureSize`（如 5000×3000 在 4096 限制下）时，分块会产生 2×1=2 个瓦片。但此时引擎有更好的选择：

1. **缩小到限制内**：5000→4096 的缩放只损失 18% 分辨率，对视觉小说背景来说通常可接受
2. **分块的额外开销**：每个瓦片需要独立的 GL 纹理对象、独立的绘制调用、瓦片间的缝合处理

只有当图像尺寸远超限制（>2x）时，缩小的质量损失才不可接受，此时分块才是更好的选择。代码中的 `* 2` 阈值就是这个权衡点。

另外注意代码中还有一个条件：`GL_CHECK_unpack_subimage`。没有这个扩展时，分块上传效率很低（需要逐行拷贝），所以即使超过 2x 也不分块，而是回退到缩小策略。

</details>

### 练习3：调试纹理问题

**问题**：如何诊断"画面显示为黑色但无 GL 错误"的问题？列出至少 3 种可能的原因。

<details>
<summary>参考答案</summary>

画面黑色但无 GL 错误的常见原因：

1. **纹理未正确上传**：`glTexImage2D` 的 `data` 参数为 `nullptr`（只分配了空间但忘记调用 `InternalUpdate`），或 `pitch` 计算错误导致上传了错位的数据

2. **纹理坐标错误**：UV 坐标全为 (0,0)，导致所有采样点都落在左上角的单个像素上；或缩放因子（`_scaleW/_scaleH`）错误导致 UV 映射到纹理外部

3. **混合模式错误**：`glBlendFunc` 设置为 `(GL_ZERO, GL_ZERO)` 会产生全黑输出；或者渲染方法的 shader 中 alpha 计算错误

4. **渲染目标未绑定**：FBO 绑定了错误的纹理，或者渲染完成后忘记切换回屏幕帧缓冲

5. **纹理格式不匹配**：用 `GL_LUMINANCE` 格式创建纹理但传入 RGBA 数据，只有 R 通道有效且映射到灰度

调试步骤：
```cpp
// 使用 glReadPixels 检查渲染结果
GLubyte pixel[4];
glReadPixels(100, 100, 1, 1, GL_RGBA, GL_UNSIGNED_BYTE, pixel);
printf("读回像素: R=%d G=%d B=%d A=%d\n",
       pixel[0], pixel[1], pixel[2], pixel[3]);
```

</details>
