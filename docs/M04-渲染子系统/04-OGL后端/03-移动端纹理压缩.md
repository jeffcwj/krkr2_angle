## 为什么移动端需要纹理压缩

### 带宽与内存的双重瓶颈

移动设备的 GPU 与桌面级 GPU 有本质差异：

| 指标 | 桌面 GPU (RTX 3080) | 移动 GPU (Adreno 650) | 差距倍数 |
|------|--------------------|-----------------------|----------|
| 显存带宽 | 760 GB/s | 51 GB/s | **15x** |
| 显存容量 | 10 GB | 共享内存 6-8 GB | 受系统限制 |
| 功耗预算 | 320W | 5-10W | **32-64x** |
| 散热能力 | 主动散热 | 被动散热 | 有限 |

对于一张 1024×1024 RGBA 纹理：

```
未压缩大小 = 1024 × 1024 × 4 bytes = 4 MB
```

如果游戏有 100 张这样的纹理，仅纹理数据就占用 **400 MB** 显存，远超移动设备承受能力。

### 压缩格式对比

```
┌─────────────────────────────────────────────────────────────────────┐
│                    移动端纹理压缩格式对比                            │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────┤
│   格式       │  压缩率     │  质量       │  Android    │    iOS      │
├─────────────┼─────────────┼─────────────┼─────────────┼─────────────┤
│ 未压缩 RGBA │    1:1      │   最佳      │     ✓       │     ✓       │
│ ETC1        │    6:1      │   中等      │     ✓       │     ✗       │
│ ETC2/EAC    │    4:1~6:1  │   较好      │     ✓       │     ✗       │
│ PVRTC       │    8:1      │   中等      │     ✗       │     ✓       │
│ ASTC        │   可变      │   最佳      │   部分✓     │   A8+✓     │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┘
```

**核心结论**：
- **Android 首选 ETC2**：OpenGL ES 3.0+ 强制支持
- **iOS 首选 PVRTC**：PowerVR GPU 原生支持，性能最优
- **ASTC 是未来趋势**：但老设备兼容性不足

## ETC 压缩详解

### ETC 格式基础

ETC（Ericsson Texture Compression）是 OpenGL ES 的标准纹理压缩格式：

| 版本 | 全称 | 特点 | OpenGL 常量 |
|------|------|------|-------------|
| ETC1 | Ericsson Texture Compression 1 | 仅 RGB，无 Alpha | `GL_ETC1_RGB8_OES` |
| ETC2 | Ericsson Texture Compression 2 | RGB + 可选 Alpha | `GL_COMPRESSED_RGB8_ETC2` |
| EAC | Ericsson Alpha Compression | 专用 Alpha 通道 | `GL_COMPRESSED_RGBA8_ETC2_EAC` |

### ETC2 压缩原理

ETC2 将图像划分为 4×4 像素块，每个块压缩为 **8 字节**（64 位）：

```
4×4 像素块（RGB，无 Alpha）
┌───────────────────────────────────────┐
│ 原始数据：4 × 4 × 3 bytes = 48 bytes  │
│ 压缩后：8 bytes                       │
│ 压缩比：48 / 8 = 6:1                  │
└───────────────────────────────────────┘

4×4 像素块（RGBA，含 Alpha）
┌───────────────────────────────────────┐
│ 原始数据：4 × 4 × 4 bytes = 64 bytes  │
│ 压缩后：16 bytes（8 RGB + 8 Alpha）   │
│ 压缩比：64 / 16 = 4:1                 │
└───────────────────────────────────────┘
```

### ETC 压缩核心算法

ETC 压缩的关键在于 **基色（Base Color）** 和 **修正表（Modifier Table）** 的选择：

```cpp
// 来源：cpp/core/visual/ogl/etcpak.cpp 第 633-637 行
// 8 种修正表，每种有 4 个修正值
const int32 g_table[8][4] = {
    { 2, 8, -2, -8 },       { 5, 17, -5, -17 },
    { 9, 29, -9, -29 },     { 13, 42, -13, -42 },
    { 18, 60, -18, -60 },   { 24, 80, -24, -80 },
    { 33, 106, -33, -106 }, { 47, 183, -47, -183 }
};
```

**压缩流程**：

```
输入：4×4 RGBA 像素块
  ↓
1. 检测是否为纯色块 → 快速路径压缩
  ↓
2. 计算 4 个 2×4 子块的平均颜色
  ↓
3. 选择最佳基色编码模式（Individual 或 Differential）
  ↓
4. 对每个像素选择最接近的修正值索引（2 位）
  ↓
5. 打包为 64 位压缩块
```

**纯色块检测**（快速路径）：

```cpp
// 来源：cpp/core/visual/ogl/etcpak.cpp 第 836-846 行
static uint64 CheckSolid(const uint8 *src) {
    const uint8 *ptr = src + 4;
    for(int i = 1; i < 16; i++) {
        if(memcmp(src, ptr, 4) != 0) {
            return 0;  // 非纯色块，需要完整压缩
        }
        ptr += 4;
    }
    // 纯色块，直接编码颜色值
    return 0x02000000 | (uint(src[0] & 0xF8) << 16) |
           (uint(src[1] & 0xF8) << 8) | (uint(src[2] & 0xF8));
}
```

### ETC Alpha 压缩（EAC）

EAC（ETC2 Alpha Compression）使用独立的 8 字节存储 Alpha 通道：

```cpp
// 来源：cpp/core/visual/ogl/etcpak.cpp 第 156-335 行
void compressBlockAlphaFast(uint8 *data, uint8 *returnData) {
    int alphasum = 0;
    int maxdist = -2;
    bool solid = true;
    
    // 计算 16 个像素的 Alpha 平均值
    for(int x = 0; x < 16; x++) {
        alphasum += data[x];
    }
    int alpha = (int)(((float)alphasum) / 16.0f + 0.5f);
    
    // 计算最大偏差
    for(int x = 0; x < 16; x++) {
        if(abs(alpha - data[x]) > maxdist)
            maxdist = abs(alpha - data[x]);
    }
    
    // 纯色 Alpha 快速路径
    if(maxdist == 0) {
        memset(returnData, 0, 8);
        returnData[0] = alpha;
        return;
    }

}
```

### ETC2 使用示例

```cpp
#include "etcpak.h"
#include <vector>
#include <cstdint>

// 将 RGBA 图像压缩为 ETC2+EAC 格式
std::vector<uint8_t> CompressToETC2(
    const uint8_t* pixels,  // BGRA 像素数据
    int width,              // 宽度（必须是 4 的倍数）
    int height,             // 高度（必须是 4 的倍数）
    bool hasAlpha           // 是否包含 Alpha 通道
) {
    // 计算 pitch（每行字节数）
    int pitch = width * 4;
    size_t compressedSize = 0;
    void* compressed = nullptr;
    
    if (hasAlpha) {
        // 使用 ETC2 + EAC 压缩（16 bytes/block）
        compressed = ETCPacker::convertWithAlpha(
            pixels, width, height, pitch, compressedSize
        );
    } else {
        // 仅 RGB，使用 ETC2（8 bytes/block）
        compressed = ETCPacker::convert(
            pixels, width, height, pitch, true, compressedSize
        );
    }
    
    // 复制到 vector 并释放原始内存
    std::vector<uint8_t> result(
        static_cast<uint8_t*>(compressed),
        static_cast<uint8_t*>(compressed) + compressedSize
    );
    free(compressed);
    
    return result;
}

// 上传 ETC2 纹理到 OpenGL
void UploadETC2Texture(
    GLuint texture,
    const std::vector<uint8_t>& compressedData,
    int width,
    int height,
    bool hasAlpha
) {
    glBindTexture(GL_TEXTURE_2D, texture);
    
    GLenum format = hasAlpha 
        ? GL_COMPRESSED_RGBA8_ETC2_EAC    // 16 bytes/block
        : GL_COMPRESSED_RGB8_ETC2;         // 8 bytes/block
    
    glCompressedTexImage2D(
        GL_TEXTURE_2D,
        0,                                  // mip level
        format,
        width,
        height,
        0,                                  // border (must be 0)
        static_cast<GLsizei>(compressedData.size()),
        compressedData.data()
    );
    
    // 设置采样参数
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
}
```

## PVRTC 压缩详解

### PVRTC 格式基础

PVRTC（PowerVR Texture Compression）是 Imagination Technologies 为 PowerVR GPU 设计的专有格式：

| 模式 | 压缩比 | 每像素位数 | OpenGL 常量 |
|------|--------|-----------|-------------|
| PVRTC 4bpp | 8:1 | 4 bits | `GL_COMPRESSED_RGBA_PVRTC_4BPPV1_IMG` |
| PVRTC 2bpp | 16:1 | 2 bits | `GL_COMPRESSED_RGBA_PVRTC_2BPPV1_IMG` |
| PVRTC2 | 改进版 | 4 bits | `GL_COMPRESSED_RGBA_PVRTC_4BPPV2_IMG` |

**关键特性**：
- **仅 iOS 设备支持**（PowerVR GPU）
- **必须是 2 的幂次方尺寸**
- **必须是正方形**（width == height）
- **原生支持 Alpha 通道**

### PVRTC 压缩原理

PVRTC 采用独特的 **双低频信号插值** 方法：

```
┌────────────────────────────────────────────────────────────────────┐
│                    PVRTC 4bpp 压缩原理                             │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  传统块压缩：每个 4×4 块独立编码                                    │
│  ┌────┐ ┌────┐ ┌────┐                                              │
│  │ A  │ │ B  │ │ C  │  → 块边界可能出现明显接缝                     │
│  └────┘ └────┘ └────┘                                              │
│                                                                    │
│  PVRTC 双线性插值：颜色在块之间平滑过渡                             │
│                                                                    │
│        ColorA          ColorA          ColorA                      │
│           ●───────────────●───────────────●                        │
│           │               │               │                        │
│           │   插值区域    │   插值区域    │                        │
│           │               │               │                        │
│           ●───────────────●───────────────●                        │
│        ColorB          ColorB          ColorB                      │
│                                                                    │
│  每个 4×4 块存储：                                                  │
│  - ColorA（14-15 bits）：低频颜色 A                                 │
│  - ColorB（14-15 bits）：低频颜色 B                                 │
│  - Modulation（32 bits）：每像素 2 位权重索引                       │
│                                                                    │
│  解码时：Pixel = lerp(ColorA, ColorB, weight)                      │
│  权重来自相邻 4 个块的双线性插值                                    │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

```cpp
struct PvrTcPacket {

    ColorRgb<int> GetColorRgbB() const;

    void SetColorB(const ColorRgb<unsigned char> &c);
    void SetColorA(const ColorRgba<unsigned char> &c);
    void SetColorB(const ColorRgba<unsigned char> &c);
    
    // 双线性插值因子表

    // 调制权重表

};
```

### 调制权重

PVRTC 使用 2 位索引从预定义权重中选择混合比例：

```cpp
// 来源：cpp/core/visual/ogl/pvrtc.cpp 第 373-385 行
// 权重格式：{ colorA权重, colorB权重, alphaA权重, alphaB权重 }
const unsigned char PvrTcPacket::WEIGHTS[8][4] = {
    // Mode=0 的权重
    { 8, 0, 8, 0 },   // 00: 100% ColorA
    { 5, 3, 5, 3 },   // 01: 62.5% ColorA + 37.5% ColorB
    { 3, 5, 3, 5 },   // 10: 37.5% ColorA + 62.5% ColorB
    { 0, 8, 0, 8 },   // 11: 100% ColorB
    
    // Mode=1 的权重（带 Alpha 穿透）
    { 8, 0, 8, 0 },   // 00: 100% ColorA
    { 4, 4, 4, 4 },   // 01: 50% 混合
    { 4, 4, 0, 0 },   // 10: 50% 颜色，Alpha=0（穿透）
    { 0, 8, 0, 8 },   // 11: 100% ColorB
};
```

### Morton 码排列

PVRTC 使用 **Morton 序（Z-order）** 存储压缩块，而非传统的行优先顺序：

```cpp
// 来源：cpp/core/visual/ogl/pvrtc.cpp 第 44-74 行
constexpr unsigned short MORTON_TABLE[256] = {
    0x0000, 0x0001, 0x0004, 0x0005, 0x0010, 0x0011, 0x0014, 0x0015,
    0x0040, 0x0041, 0x0044, 0x0045, 0x0050, 0x0051, 0x0054, 0x0055,
    // ... 完整表格见源码
};

static unsigned GetMortonNumber(int x, int y) {
    return MORTON_TABLE[x >> 8] << 17 | MORTON_TABLE[y >> 8] << 16 |
           MORTON_TABLE[x & 0xFF] << 1 | MORTON_TABLE[y & 0xFF];
}
```

**Morton 码的优势**：
- 提高缓存局部性
- 更好的纹理采样性能
- 便于 Mipmap 生成

### PVRTC 压缩核心流程

```cpp
// 来源：cpp/core/visual/ogl/pvrtc.cpp 第 724-812 行
void EncodeRgba4Bpp(void *result, const RgbaBitmap &bitmap) {
    assert(bitmap.GetBitmapWidth() == bitmap.GetBitmapHeight());  // 必须正方形
    assert(BitUtility::IsPowerOf2(bitmap.GetBitmapWidth()));      // 必须 2 的幂
    
    const int size = bitmap.GetBitmapWidth();
    const int blocks = size / 4;
    const int blockMask = blocks - 1;
    
    PvrTcPacket *packets = static_cast<PvrTcPacket *>(result);
    
    // 第一遍：为每个块计算颜色边界框
    for(int y = 0; y < blocks; ++y) {
        for(int x = 0; x < blocks; ++x) {
            ColorRgbaBoundingBox cbb;
            CalculateBoundingBox(cbb, bitmap, x, y);
            
            PvrTcPacket *packet = packets + GetMortonNumber(x, y);
            packet->usePunchthroughAlpha = 0;
            packet->SetColorA(cbb.min);  // 边界框最小值 → ColorA
            packet->SetColorB(cbb.max);  // 边界框最大值 → ColorB
        }
    }
    
    // 第二遍：计算每个像素的调制值
    for(int y = 0; y < blocks; ++y) {
        for(int x = 0; x < blocks; ++x) {
            const unsigned char (*factor)[4] = PvrTcPacket::BILINEAR_FACTORS;
            const ColorRgba<unsigned char> *data = 
                bitmap.GetData() + y * 4 * size + x * 4;
            
            uint32_t modulationData = 0;
            
            for(int py = 0; py < 4; ++py) {
                for(int px = 0; px < 4; ++px) {
                    // 获取相邻 4 个块的颜色
                    // ... 双线性插值计算 ...
                    
                    // 选择最佳调制值
                    int projection = (v % d) * 16;
                    int lengthSquared = d % d;
                    if(projection > 3 * lengthSquared)  modulationData++;
                    if(projection > 8 * lengthSquared)  modulationData++;
                    if(projection > 13 * lengthSquared) modulationData++;
                    
                    modulationData = BitUtility::RotateRight(modulationData, 2);
                    factor++;
                }
            }
            
            PvrTcPacket *packet = packets + GetMortonNumber(x, y);
            packet->modulationData = modulationData;
        }
    }
}
```

### PVRTC 使用示例

```cpp
#include "pvrtc.h"
#include <vector>
#include <cstdint>
#include <cmath>

// 将 RGBA 图像压缩为 PVRTC 4bpp 格式
std::vector<uint8_t> CompressToPVRTC(

        pixels,

        width, 
        height, 
        isOpaque
```

## ASTC 压缩详解

### ASTC 格式基础

ASTC（Adaptive Scalable Texture Compression）是 ARM 开发的下一代纹理压缩格式：

| 块大小 | 压缩比 (RGBA) | 每像素位数 | 质量 |
|--------|--------------|-----------|------|
| 4×4 | 4:1 | 8.00 bpp | 最佳 |
| 5×4 | 5:1 | 6.40 bpp | 很好 |
| 5×5 | 6.25:1 | 5.12 bpp | 好 |
| 6×5 | 7.5:1 | 4.27 bpp | 较好 |
| 6×6 | 9:1 | 3.56 bpp | 中等 |
| 8×5 | 10:1 | 3.20 bpp | 中等 |
| 8×6 | 12:1 | 2.67 bpp | 一般 |
| 10×5 | 12.5:1 | 2.56 bpp | 一般 |
| 10×6 | 15:1 | 2.13 bpp | 较差 |
| 8×8 | 16:1 | 2.00 bpp | 较差 |
| 10×8 | 20:1 | 1.60 bpp | 差 |
| 10×10 | 25:1 | 1.28 bpp | 差 |
| 12×10 | 30:1 | 1.07 bpp | 很差 |
| 12×12 | 36:1 | 0.89 bpp | 很差 |

**核心优势**：
- **可变块大小**：质量/大小权衡灵活
- **统一 128 位块**：所有块大小都压缩为 128 位
- **支持 HDR**：可压缩 HDR 纹理
- **支持 3D 纹理**：体积纹理压缩

### KrKr2 的 ASTC 实现

项目使用移植自 [astcrt](https://github.com/daoo/astcrt) 的实时压缩器：

### ASTC 压缩原理

ASTC 的核心是 **自适应分区（Partitioning）** 和 **端点编码（Endpoint Encoding）**：

```
┌────────────────────────────────────────────────────────────────────┐
│                      ASTC 压缩块结构（128 位）                      │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │ 配置位（11 bits） │ 分区 │ 端点颜色 │ 权重数据              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                    │
│  配置位：                                                          │
│    - 块模式：决定权重网格大小和精度                                │
│    - 分区数：1-4 个分区                                           │
│    - 颜色端点模式：LDR/HDR、量化级别                               │
│                                                                    │
│  分区：                                                            │
│    - 将块内像素分组（最多 4 组）                                   │
│    - 每组共享一对端点颜色                                          │
│    - 使用种子生成分区模式（1024 种预定义模式）                     │
│                                                                    │
│  端点颜色：                                                        │
│    - 每个分区 2 个端点（起点和终点颜色）                           │
│    - 支持多种量化级别（3-256 级）                                  │
│                                                                    │
│  权重数据：                                                        │
│    - 每个像素一个权重（或使用双平面）                              │
│    - 权重在端点间插值                                              │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 平台选择策略

### 运行时格式检测

```cpp
#include <string>

enum class TextureCompression {
    None,       // 不压缩
    ETC2,       // Android OpenGL ES 3.0+
    PVRTC,      // iOS PowerVR
    ASTC        // 新设备通用
};

TextureCompression DetectBestCompression() {
    // 检查 OpenGL 扩展
    const char* extensions = 
        reinterpret_cast<const char*>(glGetString(GL_EXTENSIONS));
    std::string extStr(extensions ? extensions : "");
    
#ifdef __APPLE__
    // iOS: 优先使用 PVRTC
    if (extStr.find("GL_IMG_texture_compression_pvrtc") != std::string::npos) {
        return TextureCompression::PVRTC;
    }
    // 新 iOS 设备支持 ASTC
    if (extStr.find("GL_KHR_texture_compression_astc_ldr") != std::string::npos) {
        return TextureCompression::ASTC;
    }
#endif

#ifdef __ANDROID__
    // Android: 检查 OpenGL ES 版本
    GLint majorVersion = 0;
    glGetIntegerv(GL_MAJOR_VERSION, &majorVersion);
    
    // ASTC 优先（如果支持）
    if (extStr.find("GL_KHR_texture_compression_astc_ldr") != std::string::npos) {
        return TextureCompression::ASTC;
    }
    
    // OpenGL ES 3.0+ 必须支持 ETC2
    if (majorVersion >= 3) {
        return TextureCompression::ETC2;
    }
#endif
    
    return TextureCompression::None;
}
```

### 纹理加载器集成

```cpp
class CompressedTextureLoader {
public:
    struct TextureData {
        std::vector<uint8_t> data;
        int width;
        int height;
        GLenum format;
        bool compressed;
    };
    
    static TextureData Load(
        const uint8_t* rawPixels,  // BGRA
        int width,
        int height,
        bool hasAlpha
    ) {
        TextureData result;
        result.width = width;
        result.height = height;
        
        auto compression = DetectBestCompression();
        
        switch (compression) {
        case TextureCompression::ETC2: {
            size_t compressedSize = 0;
            void* data;
            if (hasAlpha) {
                data = ETCPacker::convertWithAlpha(
                    rawPixels, width, height, width * 4, compressedSize
                );
                result.format = GL_COMPRESSED_RGBA8_ETC2_EAC;
            } else {
                data = ETCPacker::convert(
                    rawPixels, width, height, width * 4, true, compressedSize
                );
                result.format = GL_COMPRESSED_RGB8_ETC2;
            }
            result.data.assign(
                static_cast<uint8_t*>(data),
                static_cast<uint8_t*>(data) + compressedSize
            );
            free(data);
            result.compressed = true;
            break;
        }
        
        case TextureCompression::PVRTC: {
            // PVRTC 要求正方形、2 的幂次方
            if (width != height || (width & (width - 1)) != 0) {
                // 回退到未压缩
                result.data.assign(
                    rawPixels, 
                    rawPixels + width * height * 4
                );
                result.format = GL_RGBA;
                result.compressed = false;
            } else {
                size_t compressedSize = (width * height) / 2;
                result.data.resize(compressedSize);
                PvrTcEncoder::EncodeRgba4Bpp(
                    rawPixels, result.data.data(), 
                    width, height, !hasAlpha
                );
                result.format = hasAlpha 
                    ? GL_COMPRESSED_RGBA_PVRTC_4BPPV1_IMG
                    : GL_COMPRESSED_RGB_PVRTC_4BPPV1_IMG;
                result.compressed = true;
            }
            break;
        }
        
        case TextureCompression::ASTC: {
            int blocksX = (width + 3) / 4;
            int blocksY = (height + 3) / 4;
            result.data.resize(blocksX * blocksY * 16);
            ASTCRealTimeCodec::compress_texture_4x4(
                rawPixels, result.data.data(), width, height
            );
            result.format = GL_COMPRESSED_RGBA_ASTC_4x4_KHR;
            result.compressed = true;
            break;
        }
        
        default:
            // 不压缩
            result.data.assign(
                rawPixels, 
                rawPixels + width * height * 4
            );
            result.format = GL_RGBA;
            result.compressed = false;
            break;
        }
        
        return result;
    }
};
```

### 实践 1：实现简单的 ETC2 压缩器

创建一个最小化的 ETC2 纯色块压缩器：

```cpp
#include <cstdint>
#include <cstring>

// 简化版 ETC2 纯色块编码器
struct ETC2Block {
    uint8_t data[8];
    
    // 编码纯色块（所有像素相同颜色）
    void EncodeSolid(uint8_t r, uint8_t g, uint8_t b) {
        // Differential 模式，两个子块颜色相同
        // 使用 RGB555 量化
        uint8_t r5 = r >> 3;  // 5 bits
        uint8_t g5 = g >> 3;  // 5 bits
        uint8_t b5 = b >> 3;  // 5 bits
        
        // 编码基色
        data[0] = (r5 << 3) | (r5 >> 2);  // R
        data[1] = (g5 << 3) | (g5 >> 2);  // G
        data[2] = (b5 << 3) | (b5 >> 2);  // B
        
        // 使用表 0，修正值 0
        data[3] = 0x02;  // differential = 1, flip = 0, table = 0,0
        
        // 所有像素使用索引 0（无修正）
        data[4] = 0x00;
        data[5] = 0x00;
        data[6] = 0x00;
        data[7] = 0x00;
    }
};

void CompressSolidColorTexture(
    uint8_t r, uint8_t g, uint8_t b,
    int width, int height,
    uint8_t* output
) {
    int blocksX = (width + 3) / 4;
    int blocksY = (height + 3) / 4;
    
    ETC2Block block;
    block.EncodeSolid(r, g, b);
    
    // 所有块使用相同编码
    for (int i = 0; i < blocksX * blocksY; i++) {
        memcpy(output + i * 8, block.data, 8);
    }
}
```

### 实践 2：比较压缩格式质量

```cpp
#include <cmath>

// 计算 PSNR（峰值信噪比）
double CalculatePSNR(
    const uint8_t* original,
    const uint8_t* decompressed,
    int width, int height
) {
    double mse = 0.0;
    int pixelCount = width * height;
    
    for (int i = 0; i < pixelCount * 4; i++) {
        double diff = original[i] - decompressed[i];
        mse += diff * diff;
    }
    
    mse /= (pixelCount * 4);
    
    if (mse == 0) return INFINITY;
    
    double maxVal = 255.0;
    return 10.0 * log10((maxVal * maxVal) / mse);
}

// 压缩质量测试
void CompareCompressionQuality(
    const uint8_t* original,
    int width, int height
) {
    printf("压缩质量对比 (%dx%d):\n", width, height);
    printf("----------------------------------------\n");
    
    // ETC2 压缩
    size_t etc2Size = 0;
    void* etc2Data = ETCPacker::convertWithAlpha(
        original, width, height, width * 4, etc2Size
    );
    
    // ETC2 解压
    std::vector<uint8_t> etc2Decoded(width * height * 4);
    int blocksY = height / 4;
    ETCPacker::decodeWithAlpha(
        etc2Data, etc2Decoded.data(), width * 4, height, width / 4, blocksY
    );
    
    double etc2PSNR = CalculatePSNR(original, etc2Decoded.data(), width, height);
    printf("ETC2+EAC: 大小 %zu bytes, PSNR %.2f dB\n", etc2Size, etc2PSNR);
    
    free(etc2Data);
    
    // PVRTC 压缩（仅正方形 2 的幂纹理）
    if (width == height && (width & (width - 1)) == 0) {
        size_t pvrtcSize = (width * height) / 2;
        std::vector<uint8_t> pvrtcData(pvrtcSize);
        PvrTcEncoder::EncodeRgba4Bpp(
            original, pvrtcData.data(), width, height, false
        );
        
        // PVRTC 解压需要额外实现，此处省略
        printf("PVRTC 4bpp: 大小 %zu bytes\n", pvrtcSize);
    }
    
    // ASTC 压缩
    int blocksX = (width + 3) / 4;
    int astcBlocksY = (height + 3) / 4;
    size_t astcSize = blocksX * astcBlocksY * 16;
    std::vector<uint8_t> astcData(astcSize);
    ASTCRealTimeCodec::compress_texture_4x4(
        original, astcData.data(), width, height
    );
    printf("ASTC 4x4: 大小 %zu bytes\n", astcSize);
    
    // 原始大小
    printf("未压缩: 大小 %d bytes\n", width * height * 4);
}
```

## 本节小结

- **ETC2/EAC** 是 Android 平台的首选格式，OpenGL ES 3.0+ 强制支持
- **PVRTC** 是 iOS 平台的首选格式，但要求正方形且尺寸为 2 的幂次方
- **ASTC** 是未来趋势，质量最佳，但需要较新设备支持
- KrKr2 项目包含三种格式的实时压缩器，可根据目标平台动态选择
- 纹理压缩可显著减少显存占用和带宽消耗，是移动端性能优化的关键

```cpp
bool CanUsePVRTC(int width, int height) {
    // 必须正方形
    if (width != height) return false;
    // 必须是 2 的幂次方
    if (width == 0 || (width & (width - 1)) != 0) return false;

int AdjustForPVRTC(int width, int height) {

    while (pot < maxDim) pot <<= 1;

}
```

### 题目 3：实现压缩格式自动选择

编写一个函数，根据运行平台和纹理特性自动选择最佳压缩格式：

<details>
<summary>查看答案</summary>

```cpp
#include <string>

struct TextureInfo {
    int width;
    int height;
    bool hasAlpha;
    bool requiresHighQuality;  // 是否需要高质量（如 UI 元素）
};

enum class Platform {
    Android,
    iOS,
    Desktop
};

enum class CompressionFormat {
    Uncompressed,
    ETC2,
    ETC2_EAC,
    PVRTC_RGB,
    PVRTC_RGBA,
    ASTC_4x4,
    ASTC_6x6,    // 更高压缩率
    ASTC_8x8     // 最高压缩率
};

CompressionFormat SelectBestFormat(
    const TextureInfo& texture,
    Platform platform,
    bool hasASTCSupport
) {
    // 桌面平台通常不需要压缩
    if (platform == Platform::Desktop) {
        return CompressionFormat::Uncompressed;
    }
    
    // iOS 平台
    if (platform == Platform::iOS) {
        // 优先检查 ASTC 支持（A8 及更新芯片）
        if (hasASTCSupport) {
            if (texture.requiresHighQuality) {
                return CompressionFormat::ASTC_4x4;
            }
            return CompressionFormat::ASTC_6x6;
        }
        
        // 回退到 PVRTC
        bool canUsePVRTC = (texture.width == texture.height) &&
                           ((texture.width & (texture.width - 1)) == 0);
        if (canUsePVRTC) {
            return texture.hasAlpha 
                ? CompressionFormat::PVRTC_RGBA 
                : CompressionFormat::PVRTC_RGB;
        }
        
        // PVRTC 不可用，使用未压缩
        return CompressionFormat::Uncompressed;
    }
    
    // Android 平台
    if (platform == Platform::Android) {
        // 优先 ASTC（如果支持）
        if (hasASTCSupport) {
            if (texture.requiresHighQuality) {
                return CompressionFormat::ASTC_4x4;
            }
            // 非关键纹理使用更高压缩率
            return CompressionFormat::ASTC_8x8;
        }
        
        // 回退到 ETC2（OpenGL ES 3.0+ 强制支持）
        return texture.hasAlpha 
            ? CompressionFormat::ETC2_EAC 
            : CompressionFormat::ETC2;
    }
    
    return CompressionFormat::Uncompressed;
}

// 使用示例
void Example() {
    TextureInfo characterTexture = {
        .width = 512,
        .height = 512,
        .hasAlpha = true,
        .requiresHighQuality = true
    };
    
    TextureInfo backgroundTexture = {
        .width = 1024,
        .height = 1024,
        .hasAlpha = false,
        .requiresHighQuality = false
    };
    
    // Android 有 ASTC 支持
    auto charFormat = SelectBestFormat(
        characterTexture, Platform::Android, true
    );
    // 结果：ASTC_4x4（高质量 + Alpha）
    
    auto bgFormat = SelectBestFormat(
        backgroundTexture, Platform::Android, true
    );
    // 结果：ASTC_8x8（背景图可以用更高压缩率）
    
    // iOS 无 ASTC（A7 芯片）
    auto iosFormat = SelectBestFormat(
        characterTexture, Platform::iOS, false
    );
    // 结果：PVRTC_RGBA（512×512 正方形，可用 PVRTC）
}
```
