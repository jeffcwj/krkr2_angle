## 过渡系统总体架构

KrKr2 的过渡效果系统（Transition System）负责在两个图层画面之间执行平滑切换，是视觉小说引擎的核心视觉功能之一。该系统采用**提供者-处理器**（Provider-Handler）分离架构：

```
┌─────────────────────────────────────────────────────────┐
│                    TJS2 脚本层                           │
│  layer.beginTransition(%[method:"crossfade", time:500]) │
└────────────────────────┬────────────────────────────────┘
                         │ 查找注册的 Provider
┌────────────────────────▼────────────────────────────────┐
│              过渡注册表 (Hash Table)                      │
│  "crossfade" → tTVPCrossFadeTransHandlerProvider         │
│  "universal" → tTVPUniversalTransHandlerProvider         │
│  "scroll"    → tTVPScrollTransHandlerProvider            │
└────────────────────────┬────────────────────────────────┘
                         │ Provider::StartTransition()
┌────────────────────────▼────────────────────────────────┐
│              过渡处理器 (Handler)                         │
│  tTVPCrossFadeTransHandler                               │
│  - StartProcess(tick)  → 计算当前相位                     │
│  - Process(data)       → 混合像素                        │
│  - EndProcess()        → 判断是否结束                     │
└────────────────────────┬────────────────────────────────┘
                         │ 通过 RenderManager 混合
┌────────────────────────▼────────────────────────────────┐
│              渲染后端 (OGL / Software)                    │
│  TVPGetRenderManager()->OperateRect(method, ...)         │
└─────────────────────────────────────────────────────────┘
```

这种设计的核心理念是**关注点分离**（Separation of Concerns）：
- **Provider**（提供者）：工厂角色，负责解析脚本参数、创建 Handler 实例
- **Handler**（处理器）：执行角色，负责逐帧计算混合结果
- **Engine**（引擎）：驱动角色（在 LayerIntf 中），负责定时调用 Handler 的 StartProcess/Process/EndProcess

## 核心接口定义

### 过渡类型枚举

```cpp
// transhandler.h — 过渡模式定义
// tTVPTransType：决定过渡如何影响源和目标图层

enum tTVPTransType {
    ttSimple   = 0,  // 简单过渡：目标图层直接在自身上绘制
                     // 不需要源图层的当前画面，只在目标上叠加
    ttExchange = 1   // 交换过渡：需要源图层(src1)和目标图层(src2)
                     // 同时使用两个画面进行混合
                     // crossfade、universal、scroll 都使用此类型
};

// tTVPTransUpdateType：决定更新回调的粒度

enum tTVPTransUpdateType {
    tutDivisible     = 0,  // 可分割更新：引擎按区块调用 Process()
                           // Handler 每次只处理一个矩形区域
                           // 适合：全屏过渡（crossfade、universal）
    tutDivisibleFade = 1,  // 可分割淡入淡出：特化的 Divisible
                           // 引擎可以优化为全局 Alpha 混合
    tutGiveUpdate    = 2   // 给予更新：Handler 自行决定更新哪些区域
                           // Handler 调用回调函数通知引擎需要更新的矩形
                           // 适合：局部过渡（只影响部分区域）
};
```

### 基础接口层次

过渡系统的接口继承关系如下：

```
iTVPBaseTransHandler           ← 所有过渡处理器的基类
├── iTVPDivisibleTransHandler  ← 可分割类型（引擎驱动）
└── iTVPGiveUpdateTransHandler ← 自主更新类型（Handler 驱动）

iTVPTransHandlerProvider       ← 所有过渡提供者的基类
```

```cpp
// transhandler.h — 基础过渡处理器接口

class iTVPBaseTransHandler {
public:
    virtual tjs_error AddRef() = 0;    // 引用计数增加
    virtual tjs_error Release() = 0;   // 引用计数减少（0 时析构）

    // 设置选项（可在过渡运行中动态修改参数）
    virtual tjs_error SetOption(
        iTVPSimpleOptionProvider *options) = 0;

    // 开始一帧的处理
    // tick: 当前时间戳（毫秒级 tjs_uint64）
    // 返回 TJS_S_TRUE 表示过渡仍在进行
    // 返回 TJS_S_FALSE 表示过渡已完成
    virtual tjs_error StartProcess(tjs_uint64 tick) = 0;

    // 结束一帧的处理
    virtual tjs_error EndProcess() = 0;

    // 生成最终画面（过渡完成时调用）
    virtual tjs_error MakeFinalImage(
        iTVPScanLineProvider **dest,      // 输出：最终画面
        iTVPScanLineProvider *src1,       // 输入：原始画面
        iTVPScanLineProvider *src2) = 0;  // 输入：目标画面
};
```

## 过渡注册表系统

### 哈希表存储

过渡提供者通过名称注册到一个全局哈希表中，TJS2 脚本通过名称查找对应的提供者：

```cpp
// TransIntf.cpp — 全局注册表

// 使用 tTJSRefHolder 封装引用计数管理
typedef tTVPRefHolder<iTVPTransHandlerProvider>
    tTVPTransHandlerProviderHolder;

// 全局哈希表：名称 → Provider
static tTJSHashTable<ttstr, tTVPTransHandlerProviderHolder>
    TVPTransHandlerProviders;

// 懒初始化标志
bool TVPTransHandlerProviderInit = true;
```

### 注册 API

```cpp
// TransIntf.cpp — 注册和注销

void TVPAddTransHandlerProvider(iTVPTransHandlerProvider *pro) {
    // 从 Provider 获取名称
    const tjs_char *cname;
    if (TJS_FAILED(pro->GetName(&cname)))
        TVPThrowExceptionMessage(TVPTransHandlerError,
            TJS_W("tTVPTransHandlerProvider::GetName failed"));

    ttstr name = cname;

    // 检查重名
    if (TVPTransHandlerProviders.Find(name))
        TVPThrowExceptionMessage(TVPTransAlreadyRegistered, name);

    // 添加到哈希表（自动 AddRef）
    tTVPTransHandlerProviderHolder holder(pro);
    TVPTransHandlerProviders.Add(name, holder);
}

void TVPRemoveTransHandlerProvider(iTVPTransHandlerProvider *pro) {
    const tjs_char *cname;
    pro->GetName(&cname);
    TVPTransHandlerProviders.Delete(ttstr(cname));
}
```

### 查找与懒初始化

```cpp
// TransIntf.cpp — 查找过渡提供者

iTVPTransHandlerProvider *TVPFindTransHandlerProvider(
    const ttstr &name)
{
    // 首次调用时注册内置过渡效果（懒初始化）
    if (TVPTransHandlerProviderInit) {
        TVPRegisterDefaultTransHandlerProvider();
        TVPTransHandlerProviderInit = false;
    }

    // 在哈希表中查找
    auto *holder = TVPTransHandlerProviders.Find(name);

    if (!holder) {
        // 找不到指定的过渡效果 → 回退到 crossfade
        static bool showed = false;
        if (!showed) {
            TVPShowSimpleMessageBox(
                TVPFormatMessage(TVPCannotFindTransHander, name),
                TJS_W("Warning"));
            showed = true;  // 只显示一次警告
        }
        // 降级为 crossfade（保证不崩溃）
        holder = TVPTransHandlerProviders.Find(TJS_W("crossfade"));
    }

    iTVPTransHandlerProvider *pro = holder->GetObjectNoAddRef();
    pro->AddRef();
    return pro;
}
```

```cpp
// TransIntf.cpp — 注册三种内置过渡效果

static void TVPRegisterDefaultTransHandlerProvider() {
    iTVPTransHandlerProvider *prov;

    // 1. 淡入淡出（crossfade）
    prov = new tTVPCrossFadeTransHandlerProvider();
    TVPAddTransHandlerProvider(prov);
    prov->Release();  // 哈希表已持有引用

    TVPAddTransHandlerProvider(prov);
    prov->Release();
```

### iTVPSimpleOptionProvider

```cpp
// TransIntf.cpp — 选项提供者实现

class tTVPSimpleOptionProvider : public iTVPSimpleOptionProvider {
    tjs_int RefCount;
    tTJSVariantClosure Object;  // TJS2 字典对象的闭包
    ttstr String;               // 字符串缓存

public:
    // 以数值方式获取选项
    tjs_error GetAsNumber(const tjs_char *name, tjs_int64 *value) {
        tTJSVariant val;
        tjs_error er = Object.PropGet(0, name, nullptr, &val, nullptr);
        if (TJS_FAILED(er)) return er;
        if (val.Type() == tvtVoid) return TJS_E_MEMBERNOTFOUND;
        if (value) *value = val.AsInteger();
        return TJS_S_OK;
    }

    // 以字符串方式获取选项
    tjs_error GetAsString(const tjs_char *name, const tjs_char **out) {
        tTJSVariant val;
        tjs_error er = Object.PropGet(0, name, nullptr, &val, nullptr);
        if (TJS_FAILED(er)) return er;
        String = val;  // 持有字符串缓冲区
        if (out) *out = String.c_str();
        return TJS_S_OK;
    }
};
```

```cpp
class tTVPSimpleImageProvider : public iTVPSimpleImageProvider {
public:
```

## 过渡生命周期

一个完整的过渡效果从创建到销毁经历以下阶段：

```
阶段 1: 创建
  脚本调用 layer.beginTransition(options)
    → TVPFindTransHandlerProvider("crossfade")
    → Provider.StartTransition(options, ...)
    → 创建 Handler 实例
    → 返回 Handler + TransType + UpdateType

阶段 2: 运行（每帧重复）
  引擎定时器触发
    → Handler.StartProcess(currentTick)
      → 计算当前 Phase（0 ~ PhaseMax）
    → 对每个脏区域：Handler.Process(data)
      → 混合 Src1 和 Src2 的像素
    → Handler.EndProcess()
      → 返回 TJS_S_TRUE（继续）或 TJS_S_FALSE（完成）

阶段 3: 完成
  EndProcess() 返回 TJS_S_FALSE
    → Handler.MakeFinalImage(&dest, src1, src2)
      → 通常返回 src2（过渡后的画面）
    → Handler.Release()
    → 清理资源
```

### 时间驱动的相位计算

所有内置过渡效果都基于**时间相位**（Phase）驱动：

```cpp
// TransIntf.cpp — CrossFade 的 StartProcess

tjs_error tTVPCrossFadeTransHandler::StartProcess(tjs_uint64 tick) {
    if (First) {
        First = false;
        StartTick = tick;  // 记录过渡开始时间
    }

    // 线性插值计算相位
    // Phase 范围：0 ~ PhaseMax（默认 255）
    Phase = static_cast<tjs_int>(
        (tick - StartTick) * PhaseMax / Time);

    if (Phase >= PhaseMax)
        Phase = PhaseMax;  // 钳位到最大值

    return TJS_S_TRUE;
}

// EndProcess：判断过渡是否结束
tjs_error tTVPCrossFadeTransHandler::EndProcess() {
    if (Phase >= PhaseMax)
        return TJS_S_FALSE;  // 过渡完成
    return TJS_S_TRUE;       // 继续
}
```

## 动手实践：分析过渡的数据流

以下代码模拟了过渡系统的核心数据流：

```cpp
// transition_lifecycle_demo.cpp
// 演示过渡系统的创建-运行-销毁生命周期

#include <cstdint>
#include <iostream>
#include <string>
#include <unordered_map>
#include <memory>
#include <functional>
#include <chrono>
#include <thread>

// 简化的过渡处理器基类
class ITransHandler {
public:
    virtual ~ITransHandler() = default;
    virtual bool StartProcess(uint64_t tick) = 0;
    virtual void Process(uint32_t* dest, const uint32_t* src1,
                         const uint32_t* src2, int width) = 0;
    virtual bool EndProcess() = 0;
};

// 简化的淡入淡出处理器
class CrossFadeHandler : public ITransHandler {
    uint64_t startTick_ = 0;
    uint64_t duration_;
    int phase_ = 0;        // 当前相位 (0-255)
    bool first_ = true;

public:
    explicit CrossFadeHandler(uint64_t duration) : duration_(duration) {}

    bool StartProcess(uint64_t tick) override {
        if (first_) {
            first_ = false;
            startTick_ = tick;
        }
        // 线性插值：elapsed / duration * 255
        phase_ = static_cast<int>(
            (tick - startTick_) * 255 / duration_);
        if (phase_ > 255) phase_ = 255;

        std::cout << "  Phase: " << phase_ << "/255" << std::endl;
        return true;
    }

    void Process(uint32_t* dest, const uint32_t* src1,
                 const uint32_t* src2, int width) override {
        for (int i = 0; i < width; i++) {
            // 逐通道线性插值
            uint32_t s1 = src1[i], s2 = src2[i];
            uint32_t r = ((s1 & 0xFF) * (255 - phase_)
                       + (s2 & 0xFF) * phase_) / 255;
            uint32_t g = (((s1 >> 8) & 0xFF) * (255 - phase_)
                       + ((s2 >> 8) & 0xFF) * phase_) / 255;
            uint32_t b = (((s1 >> 16) & 0xFF) * (255 - phase_)
                       + ((s2 >> 16) & 0xFF) * phase_) / 255;
            dest[i] = r | (g << 8) | (b << 16) | 0xFF000000;
        }
    }

    bool EndProcess() override {
        return phase_ < 255;  // true=继续, false=完成
    }
};

// 简化的过渡提供者注册表
class TransRegistry {
    using Factory = std::function<
        std::unique_ptr<ITransHandler>(uint64_t time)>;
    std::unordered_map<std::string, Factory> providers_;

public:
    void Register(const std::string& name, Factory factory) {
        providers_[name] = std::move(factory);
    }

    std::unique_ptr<ITransHandler> Create(
        const std::string& name, uint64_t time)
    {
        auto it = providers_.find(name);
        if (it == providers_.end()) {
            std::cout << "警告: 未找到过渡 '" << name
                      << "', 回退到 crossfade" << std::endl;
            it = providers_.find("crossfade");
        }
        return it->second(time);
    }
};

int main() {
    // 1. 注册过渡效果
    TransRegistry registry;
    registry.Register("crossfade", [](uint64_t time) {
        return std::make_unique<CrossFadeHandler>(time);
    });

    // 2. 创建过渡（模拟脚本调用）
    auto handler = registry.Create("crossfade", 500); // 500ms

    // 3. 准备源数据
    const int W = 4;
    uint32_t src1[W] = {0xFFFF0000, 0xFFFF0000,  // 红色画面
                        0xFFFF0000, 0xFFFF0000};
    uint32_t src2[W] = {0xFF0000FF, 0xFF0000FF,  // 蓝色画面
                        0xFF0000FF, 0xFF0000FF};
    uint32_t dest[W];

    // 4. 运行过渡（模拟引擎帧循环）
    auto start = std::chrono::steady_clock::now();
    bool running = true;
    int frame = 0;

    while (running) {
        auto now = std::chrono::steady_clock::now();
        uint64_t tick = std::chrono::duration_cast<
            std::chrono::milliseconds>(now - start).count();

        std::cout << "帧 " << frame << " (tick=" << tick << "ms):"
                  << std::endl;

        handler->StartProcess(tick);
        handler->Process(dest, src1, src2, W);
        running = handler->EndProcess();

        // 打印第一个像素的颜色
        std::cout << "  像素[0] = 0x" << std::hex << dest[0]
                  << std::dec << std::endl;

        frame++;
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }

    std::cout << "过渡完成! 总帧数: " << frame << std::endl;
    return 0;
}
```

## 常见错误及解决方案

### 错误 1：过渡效果名称拼写错误导致静默降级

```
症状：指定了一个自定义过渡名称，但实际播放的是 crossfade
原因：名称拼写错误或插件未正确注册
```

KrKr2 的设计是找不到指定过渡时自动降级为 crossfade，且只弹一次警告。解决方案：

```cpp
// 检查过渡是否注册成功
void VerifyTransitionRegistered(const ttstr &name) {
    auto *holder = TVPTransHandlerProviders.Find(name);
    if (!holder) {
        // 明确报错，而非静默降级
        TVPThrowExceptionMessage(
            TJS_W("过渡效果 '%1' 未注册，请检查插件是否加载"),
            name);
    }
}
```

### 错误 2：过渡时间设为 0 导致除零错误

```
症状：layer.beginTransition(%[method:"crossfade", time:0]) 导致崩溃
原因：Phase 计算中 Time 作为除数
```

KrKr2 的保护措施 — 强制最小时间为 2ms：

```cpp
// TransIntf.cpp — crossfade Provider 中的保护
tjs_int64 time;
er = options->GetAsNumber(TJS_W("time"), &time);
if (TJS_FAILED(er))
    TVPThrowExceptionMessage(TVPSpecifyOption, TJS_W("time"));
if (time < 2)
    time = 2;  // 防止除零和极短过渡的数值问题
```

### 错误 3：源画面尺寸不匹配

```
症状：crossfade 时画面错位或崩溃
原因：src1 和 src2 的尺寸不同
```

```cpp
// TransIntf.cpp — Provider 中的尺寸检查
if (src1w != src2w || src1h != src2h)
    TVPThrowExceptionMessage(
        TVPTransitionLayerSizeMismatch,
        ttstr((tjs_int)src2w) + TJS_W("x") + ttstr((tjs_int)src2h),
        ttstr((tjs_int)src1w) + TJS_W("x") + ttstr((tjs_int)src1h));
// 错误信息示例: "过渡源画面尺寸不匹配: 800x600 vs 640x480"
```

## 本节小结

- KrKr2 过渡系统采用 **Provider-Handler** 分离架构，Provider 是工厂，Handler 是执行器
- 过渡效果通过**名称注册到全局哈希表**，支持懒初始化和查找失败降级到 crossfade
- 两种更新模式：**Divisible**（引擎按区块驱动）和 **GiveUpdate**（Handler 自主报告更新区域）
- 生命周期：StartTransition → [StartProcess → Process × N → EndProcess] × 帧数 → MakeFinalImage
- 相位计算基于**时间线性插值**：Phase = elapsed × PhaseMax / Duration

## 练习题与答案

### 题目 1：为什么过渡注册表使用懒初始化而不是在程序启动时注册？

<details>
<summary>查看答案</summary>

懒初始化（Lazy Initialization）有以下好处：

1. **启动性能**：程序启动时不需要创建过渡对象，减少启动耗时。很多游戏场景不会用到过渡效果。

2. **初始化顺序安全**：过渡提供者的创建可能依赖 RenderManager 等组件。如果在启动时注册，需要精确控制初始化顺序。懒初始化在首次使用时触发，此时所有依赖组件已就绪。

3. **插件兼容**：第三方插件可以在内置效果注册之前或之后注册自己的过渡效果。如果内置效果在程序启动时就注册，可能与插件的注册时序冲突。

代码中通过 `TVPTransHandlerProviderInit` 布尔标志实现：首次调用 `TVPFindTransHandlerProvider` 时检查该标志，如果为 true 则调用 `TVPRegisterDefaultTransHandlerProvider()` 注册内置效果，然后将标志设为 false。

</details>

### 题目 2：解释 tTVPDivisibleData 中为什么需要分别存储 Left/Top 和 DestLeft/DestTop？

<details>
<summary>查看答案</summary>

`tTVPDivisibleData` 中的坐标分为两组：

- `Left, Top`: 区域在**全局画面坐标系**中的位置，用于从规则图像等辅助资源中读取对应像素
- `DestLeft, DestTop`: 区域在**目标缓冲区坐标系**中的写入位置
- `Src1Left, Src1Top`: 区域在**源画面 1 坐标系**中的读取位置
- `Src2Left, Src2Top`: 区域在**源画面 2 坐标系**中的读取位置

需要分开存储是因为**图层可能有偏移量**。当引擎将一个大画面分割成多个区块时，每个区块在不同坐标系中的位置可能不同：

```
全局坐标 (Left=100, Top=50)
  → Src1 可能从 (Src1Left=0, Src1Top=0) 开始读
  → Src2 可能从 (Src2Left=100, Src2Top=50) 开始读
  → Dest 写入到 (DestLeft=100, DestTop=50)
```

这种设计允许过渡处理器在不同大小、不同偏移的图层之间正确混合，同时支持 scroll 过渡中源画面的平移效果。

</details>

### 题目 3：如果要添加一个"旋转过渡"效果，需要修改哪些文件？给出实现框架。

<details>
<summary>查看答案</summary>

只需修改一个文件 `TransIntf.cpp`（或创建新文件），添加以下内容：

```cpp
// 1. 定义旋转过渡处理器（继承 crossfade 基类复用时间管理）
class tTVPRotateTransHandler : public tTVPCrossFadeTransHandler {
    typedef tTVPCrossFadeTransHandler inherited;
    float angle_;  // 当前旋转角度

public:
    tTVPRotateTransHandler(iTVPSimpleOptionProvider *options,
                           tTVPLayerType layertype, tjs_uint64 time)
        : tTVPCrossFadeTransHandler(options, layertype, time, 360)
        , angle_(0.0f) {}

    void Blend(tTVPDivisibleData *data) override {
        // 根据 Phase (0-360) 计算旋转角度
        angle_ = Phase * 3.14159f / 180.0f;
        // 使用旋转变换混合 src1 和 src2
        // （实际实现需要 2D 旋转矩阵 + 双线性插值）
    }
};

// 2. 定义旋转过渡提供者
class tTVPRotateTransHandlerProvider
    : public tTVPCrossFadeTransHandlerProvider {
public:
    tjs_error GetName(const tjs_char **name) override {
        if (name) { *name = TJS_W("rotate"); return TJS_S_OK; }
        return TJS_E_FAIL;
    }

    iTVPBaseTransHandler *GetTransitionObject(
        iTVPSimpleOptionProvider *options,
        iTVPSimpleImageProvider *imagepro,
        tTVPLayerType layertype,
        tjs_uint src1w, tjs_uint src1h,
        tjs_uint src2w, tjs_uint src2h) override
    {
        tjs_int64 time;
        options->GetAsNumber(TJS_W("time"), &time);
        if (time < 2) time = 2;
        return new tTVPRotateTransHandler(options, layertype, time);
    }
};
```
