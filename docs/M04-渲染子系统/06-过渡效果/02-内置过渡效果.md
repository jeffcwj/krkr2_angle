## CrossFade — 淡入淡出

CrossFade 是最基础的过渡效果，通过在两个画面之间做**线性 Alpha 插值**实现平滑渐变。

### 核心混合逻辑

CrossFade 的 `Blend` 方法根据目标图层的 Alpha 类型选择不同的混合方法：

```cpp
// TransIntf.cpp — CrossFade 的 Blend 实现
void tTVPCrossFadeTransHandler::Blend(tTVPDivisibleData *data) {
    iTVPRenderMethod *method;
    int opa_id;

    // 根据图层类型选择不同的混合方法
    if (TVPIsTypeUsingAlpha(DestLayerType)) {
        // Alpha 图层：保留混合后的 Alpha 通道
        static iTVPRenderMethod *_method =
            TVPGetRenderManager()->GetRenderMethod(
                "ConstAlphaBlend_SD_d");  // _d = dest alpha
        method = _method;
    } else if (TVPIsTypeUsingAddAlpha(DestLayerType)) {
        // 加法 Alpha 图层：使用加法混合
        static iTVPRenderMethod *_method =
            TVPGetRenderManager()->GetRenderMethod(
                "ConstAlphaBlend_SD_a");  // _a = additive alpha
        method = _method;
    } else {
        // 普通图层：标准 Alpha 混合
        static iTVPRenderMethod *_method =
            TVPGetRenderManager()->GetRenderMethod(
                "ConstAlphaBlend_SD");    // SD = src/dest 双源
        method = _method;
    }

    // 设置当前混合不透明度 (Phase: 0~255)
    static int _opa_id = method->EnumParameterID("opacity");
    method->SetParameterOpa(_opa_id, Phase);

    // 构建源纹理数组（两个源画面）
    tRenderTexRectArray::Element src_tex[] = {
        tRenderTexRectArray::Element(
            data->Src1->GetTexture(),    // 源画面 1
            tTVPRect(data->Src1Left, data->Src1Top,
                     data->Src1Left + data->Width,
                     data->Src1Top + data->Height)),
        tRenderTexRectArray::Element(
            data->Src2->GetTexture(),    // 源画面 2
            tTVPRect(data->Src2Left, data->Src2Top,
                     data->Src2Left + data->Width,
                     data->Src2Top + data->Height))
    };

    // 通过 RenderManager 执行 GPU 混合
    TVPGetRenderManager()->OperateRect(
        method,
        data->Dest->GetTextureForRender(),  // 目标纹理
        nullptr,                             // 无裁剪
        tTVPRect(data->DestLeft, data->DestTop,
                 data->DestLeft + data->Width,
                 data->DestTop + data->Height),
        tRenderTexRectArray(src_tex));
}
```

### 三种图层类型对比

| 图层类型 | 混合方法名 | 行为 |
|---------|-----------|------|
| 普通 | `ConstAlphaBlend_SD` | 忽略 Alpha 通道，仅混合 RGB |
| Alpha | `ConstAlphaBlend_SD_d` | 同时混合 RGBA 四个通道 |
| 加法 Alpha | `ConstAlphaBlend_SD_a` | Alpha 通道使用加法混合 |

### 相位边界优化

```cpp
// TransIntf.cpp — Process 方法中的边界优化
tjs_error tTVPCrossFadeTransHandler::Process(
    tTVPDivisibleData *data)
{
    if (Phase == 0) {
        // 完全是源画面 1 — 直接指向 Src1，无需混合
        data->Dest = data->Src1;
        data->DestLeft = data->Src1Left;
        data->DestTop  = data->Src1Top;
    } else if (Phase == PhaseMax) {
        // 完全是源画面 2 — 直接指向 Src2
        data->Dest = data->Src2;
        data->DestLeft = data->Src2Left;
        data->DestTop  = data->Src2Top;
    } else {
        // 中间阶段 — 执行混合
        Blend(data);
    }
    return TJS_S_OK;
}
```

这是一个重要的性能优化：在过渡的起点和终点，直接复用源画面的纹理引用，跳过混合计算。

## Universal — 万能过渡

Universal 过渡是 KiriKiri 引擎最强大的过渡效果，通过一张**灰度规则图像**（Rule Image）控制画面的切换顺序。规则图像中每个像素的亮度值决定该位置在过渡过程中何时开始切换。

### 工作原理

```
规则图像（灰度，0-255）：
┌─────────────────┐
│ 0   50  100 150 │  ← 左侧先切换（值小），右侧后切换（值大）
│ 25  75  125 175 │
│ 50  100 150 200 │
│ 75  125 175 255 │
└─────────────────┘

Phase = 0   : 全部显示 Src1
Phase = 100 : 规则值 < 100 的像素显示 Src2，其余显示 Src1
Phase = 200 : 规则值 < 200 的像素显示 Src2
Phase = 255 : 全部显示 Src2
```

### Universal 过渡处理器

```cpp
// TransIntf.cpp — Universal 过渡处理器

class tTVPUniversalTransHandler : public tTVPCrossFadeTransHandler {
    tjs_int Vague;                   // 模糊度（过渡边缘宽度）
    iTVPScanLineProvider *Rule;      // 规则图像
    iTVPRenderMethod *Method;        // GPU 渲染方法
    tjs_int MethodPhaseID;           // Phase 参数 ID
    tjs_int MethodVagueID;           // Vague 参数 ID

public:
    tTVPUniversalTransHandler(
        iTVPSimpleOptionProvider *options,
        tTVPLayerType destlayertype,
        tjs_uint64 time,
        tjs_int vague,
        iTVPScanLineProvider *rule)
        // PhaseMax = 255 + vague（扩展范围以覆盖模糊区域）
        : tTVPCrossFadeTransHandler(options, destlayertype,
                                     time, 255 + vague)
    {
        Vague = vague;
        Rule = rule;
        Rule->AddRef();  // 持有规则图像引用

        // 根据图层类型选择 GPU 渲染方法
        if (TVPIsTypeUsingAlpha(DestLayerType)) {
            Method = TVPGetRenderManager()->GetRenderMethod(
                "UnivTransBlend_d");
        } else if (TVPIsTypeUsingAddAlpha(DestLayerType)) {
            Method = TVPGetRenderManager()->GetRenderMethod(
                "UnivTransBlend_a");
        } else {
            Method = TVPGetRenderManager()->GetRenderMethod(
                "UnivTransBlend");
        }

        // 获取参数 ID 并设置初始 Vague 值
        MethodPhaseID = Method->EnumParameterID("phase");
        MethodVagueID = Method->EnumParameterID("vague");
        Method->SetParameterInt(MethodVagueID, vague);
    }

    ~tTVPUniversalTransHandler() override {
        Rule->Release();  // 释放规则图像
    }
};
```

### Universal 的 Blend 方法

Universal 的混合使用三个源纹理 — Src1、Src2 和 Rule：

```cpp
// TransIntf.cpp — Universal Blend
void tTVPUniversalTransHandler::Blend(tTVPDivisibleData *data) {
    // 构建三源纹理数组
    tRenderTexRectArray::Element src_tex[] = {
        // Src1 — 过渡前画面
        tRenderTexRectArray::Element(
            data->Src1->GetTexture(),
            tTVPRect(data->Src1Left, data->Src1Top,
                     data->Src1Left + data->Width,
                     data->Src1Top + data->Height)),
        // Src2 — 过渡后画面
        tRenderTexRectArray::Element(
            data->Src2->GetTexture(),
            tTVPRect(data->Src2Left, data->Src2Top,
                     data->Src2Left + data->Width,
                     data->Src2Top + data->Height)),
        // Rule — 规则图像（灰度）
        tRenderTexRectArray::Element(
            Rule->GetTexture(),
            tTVPRect(data->Left, data->Top,
                     data->Left + data->Width,
                     data->Top + data->Height))
    };

    TVPGetRenderManager()->OperateRect(
        Method,
        data->Dest->GetTextureForRender(), nullptr,
        tTVPRect(data->DestLeft, data->DestTop,
                 data->DestLeft + data->Width,
                 data->DestTop + data->Height),
        tRenderTexRectArray(src_tex));
}
```

```cpp
// TransIntf.cpp — Universal Provider 的参数处理

iTVPBaseTransHandler *tTVPUniversalTransHandlerProvider::
    GetTransitionObject(/*...*/)
{
    // 1. 获取过渡时间（必选）
    tjs_int64 time;
    er = options->GetAsNumber(TJS_W("time"), &time);
    if (TJS_FAILED(er))
        TVPThrowExceptionMessage(TVPSpecifyOption, TJS_W("time"));
    if (time < 2) time = 2;

    // 2. 获取模糊度（可选，默认 64）
    tjs_int64 vague;
    er = options->GetAsNumber(TJS_W("vague"), &vague);
    if (TJS_FAILED(er))
        vague = 64;  // 默认 64 级模糊

    // 3. 获取规则图像路径（必选）
    const tjs_char *rulename;
    er = options->GetAsString(TJS_W("rule"), &rulename);
    if (TJS_FAILED(er))
        TVPThrowExceptionMessage(TVPSpecifyOption, TJS_W("rule"));

    // 4. 加载规则图像（8bpp 灰度）
    iTVPScanLineProvider *scpro;
    er = imagepro->LoadImage(
        rulename,
        8,          // 8bpp 灰度
        0x02ffffff, // 色键
        src1w, src1h, &scpro);
    if (TJS_FAILED(er))
        TVPThrowExceptionMessage(
            TVPCannotLoadRuleGraphic, rulename);

    // 5. 创建处理器
    auto *ret = new tTVPUniversalTransHandler(
        options, layertype, time,
        static_cast<tjs_int>(vague), scpro);
    scpro->Release();  // Handler 已 AddRef
    return ret;
}
```

### Vague（模糊度）的作用

Vague 参数控制过渡边缘的平滑程度：

```
Vague = 0  : 硬切换（锐利边缘）
  规则值 < Phase → 完全显示 Src2
  规则值 >= Phase → 完全显示 Src1

Vague = 64 : 柔和过渡（64 级渐变）
  规则值 < Phase - 64 → 完全显示 Src2
  规则值 > Phase → 完全显示 Src1
  中间区域 → 按比例混合

Vague = 255 : 最大模糊（等效于 crossfade）
```

```cpp
// PhaseMax = 255 + Vague
// 这确保了即使 Vague 很大，过渡也能完整覆盖所有像素
// 例如 Vague=64 时 PhaseMax=319
// Phase 从 0 递增到 319 才算完成
```

## Scroll — 滚动过渡

Scroll 过渡模拟画面滚动效果，支持四个方向和三种停留模式的组合。

### 方向与停留模式

```cpp
// transhandler.h — 滚动过渡参数

enum tTVPScrollTransFrom {
    sttLeft   = 0,  // 从左向右滚动
    sttTop    = 1,  // 从上向下滚动
    sttRight  = 2,  // 从右向左滚动
    sttBottom = 3   // 从下向上滚动
};

enum tTVPScrollTransStay {
    ststNoStay   = 0,  // 无停留：两个画面同时移动
    ststStayDest = 1,  // 目标停留：Src2 不动，Src1 被推出
    ststStaySrc  = 2   // 源停留：Src1 不动，Src2 推入覆盖
};
```

### 滚动偏移计算

```cpp
// TransIntf.cpp — 滚动过渡的偏移计算
void tTVPScrollDivisibleData::Blend(
    tTVPScrollTransFrom from,
    tTVPScrollTransStay stay,
    tjs_int phase)
{
    tjs_int imagewidth, imageheight;
    Src1->GetWidth(&imagewidth);
    Src1->GetHeight(&imageheight);

    tjs_int src1left = 0, src1top = 0;
    tjs_int src2left = 0, src2top = 0;

    switch (from) {
        case sttLeft:  // 从左向右滚
            if (stay == ststNoStay)
                // 两个画面都移动
                src1left = phase,
                src2left = phase - imagewidth;
            else if (stay == ststStayDest)
                // Src2 固定，Src1 不动
                src2left = phase - imagewidth;
            else if (stay == ststStaySrc)
                // Src1 固定，被 Src2 推出
                src1left = phase;
            break;

        case sttRight:  // 从右向左滚
            if (stay == ststNoStay)
                src1left = -phase,
                src2left = imagewidth - phase;
            else if (stay == ststStayDest)
                src2left = imagewidth - phase;
            else if (stay == ststStaySrc)
                src1left = -phase;
            break;

        case sttTop:    // 从上向下滚
            if (stay == ststNoStay)
                src1top = phase,
                src2top = phase - imageheight;
            else if (stay == ststStayDest)
                src2top = phase - imageheight;
            else if (stay == ststStaySrc)
                src1top = phase;
            break;

        case sttBottom: // 从下向上滚
            if (stay == ststNoStay)
                src1top = -phase,
                src2top = imageheight - phase;
            else if (stay == ststStayDest)
                src2top = imageheight - phase;
            else if (stay == ststStaySrc)
                src1top = -phase;
            break;
    }

    // 计算源矩形并处理优先级裁剪
    tTVPRect rdest(Left, Top, Width + Left, Height + Top);
    tTVPRect rs1(src1left, src1top,
                 imagewidth + src1left, imageheight + src1top);
    tTVPRect rs2(src2left, src2top,
                 imagewidth + src2left, imageheight + src2top);

    // StayDest: Src2 有优先级，裁剪 Src1 的重叠区域
    // StaySrc:  Src1 有优先级，裁剪 Src2 的重叠区域
    // ...

    // 使用 TVPSLPCopyRect 将两个画面复制到目标
    tTVPRect d;
    if (TVPIntersectRect(&d, rdest, rs1)) {
        tjs_int dl = d.left - Left + DestLeft;
        tjs_int dt = d.top  - Top  + DestTop;
        d.add_offsets(-src1left, -src1top);
        TVPSLPCopyRect(Dest, dl, dt, Src1, d);
    }
    if (TVPIntersectRect(&d, rdest, rs2)) {
        tjs_int dl = d.left - Left + DestLeft;
        tjs_int dt = d.top  - Top  + DestTop;
        d.add_offsets(-src2left, -src2top);
        TVPSLPCopyRect(Dest, dl, dt, Src2, d);
    }
}
```

## 动手实践：实现完整的万能过渡模拟

```cpp
// universal_transition_demo.cpp
// 演示 Universal 过渡的完整工作原理

#include <cstdint>
#include <cstring>
#include <iostream>
#include <cmath>
#include <vector>

// 生成规则图像：从左到右的线性渐变
void GenerateLinearRule(uint8_t* rule, int width, int height) {
    for (int y = 0; y < height; y++) {
        for (int x = 0; x < width; x++) {
            // 从左(0) 到右(255) 线性渐变
            rule[y * width + x] = static_cast<uint8_t>(
                x * 255 / (width - 1));
        }
    }
}

// 生成规则图像：从中心向外的圆形渐变
void GenerateCircularRule(uint8_t* rule, int width, int height) {
    float cx = width / 2.0f, cy = height / 2.0f;
    float maxDist = std::sqrt(cx * cx + cy * cy);
    for (int y = 0; y < height; y++) {
        for (int x = 0; x < width; x++) {
            float dx = x - cx, dy = y - cy;
            float dist = std::sqrt(dx * dx + dy * dy);
            rule[y * width + x] = static_cast<uint8_t>(
                std::min(255.0f, dist / maxDist * 255.0f));
        }
    }
}

// Universal 过渡混合（单帧）
void UniversalBlend(
    uint32_t* dest,
    const uint32_t* src1,
    const uint32_t* src2,
    const uint8_t* rule,
    int width, int height,
    int phase,       // 当前相位 (0 ~ 255+vague)
    int vague)       // 模糊度
{
    for (int i = 0; i < width * height; i++) {
        int ruleVal = rule[i];

        // 计算该像素的混合比例
        int blend;
        if (vague == 0) {
            // 硬切换
            blend = (ruleVal < phase) ? 255 : 0;
        } else {
            // 柔和过渡
            int low  = phase - vague;
            int high = phase;

            if (ruleVal <= low)
                blend = 255;     // 完全显示 Src2
            else if (ruleVal >= high)
                blend = 0;       // 完全显示 Src1
            else
                // 线性插值：(high - ruleVal) / (high - low) * 255
                blend = (high - ruleVal) * 255 / vague;
        }

        // 逐通道混合
        uint32_t s1 = src1[i], s2 = src2[i];
        uint32_t r = ((s1 & 0xFF) * (255 - blend)
                   + (s2 & 0xFF) * blend) / 255;
        uint32_t g = (((s1>>8)&0xFF) * (255-blend)
                   + ((s2>>8)&0xFF) * blend) / 255;
        uint32_t b = (((s1>>16)&0xFF) * (255-blend)
                   + ((s2>>16)&0xFF) * blend) / 255;
        dest[i] = r | (g<<8) | (b<<16) | 0xFF000000;
    }
}

int main() {
    const int W = 8, H = 4;

    // 源画面：红色和蓝色
    std::vector<uint32_t> src1(W*H, 0xFFFF0000); // 红
    std::vector<uint32_t> src2(W*H, 0xFF0000FF); // 蓝
    std::vector<uint32_t> dest(W*H);
    std::vector<uint8_t> rule(W*H);

    // 生成从左到右的规则图像
    GenerateLinearRule(rule.data(), W, H);

    std::cout << "=== 规则图像 ===" << std::endl;
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++)
            printf("%3d ", rule[y*W+x]);
        std::cout << std::endl;
    }

    // 模拟不同相位的过渡
    int vague = 64;
    for (int phase = 0; phase <= 255 + vague; phase += 80) {
        UniversalBlend(dest.data(), src1.data(), src2.data(),
                       rule.data(), W, H, phase, vague);

        std::cout << "\n=== Phase=" << phase
                  << "/" << (255+vague) << " ===" << std::endl;
        for (int x = 0; x < W; x++) {
            uint32_t p = dest[x]; // 第一行
            int r = p & 0xFF, b = (p>>16) & 0xFF;
            // 红=src1, 蓝=src2
            if (r > 200) std::cout << "红 ";
            else if (b > 200) std::cout << "蓝 ";
            else std::cout << "混 ";
        }
        std::cout << std::endl;
    }

    return 0;
}
```

### TVPSLPCopyRect 的 GPU 版本

滚动过渡使用的矩形复制也已迁移到 GPU：

```cpp
// TransIntf.cpp — GPU 加速的矩形复制
static void TVPSLPCopyRect(
    iTVPScanLineProvider *destimg, tjs_int x, tjs_int y,
    iTVPScanLineProvider *srcimg, const tTVPRect &srcrect)
{
    // 使用 RenderManager 的 "Copy" 方法
    static iTVPRenderMethod *method =
        TVPGetRenderManager()->GetRenderMethod("Copy");

    tRenderTexRectArray::Element src_tex[] = {
        tRenderTexRectArray::Element(
            srcimg->GetTexture(), srcrect)
    };

    TVPGetRenderManager()->OperateRect(
        method,
        destimg->GetTextureForRender(), nullptr,
        tTVPRect(x, y,
                 x + srcrect.get_width(),
                 y + srcrect.get_height()),
        tRenderTexRectArray(src_tex));
}
```

## 常见错误及解决方案

### 错误 1：Universal 过渡中规则图像尺寸不匹配

```
症状：过渡画面出现拉伸或错位
原因：规则图像的尺寸与源画面不同
```

规则图像在加载时会被缩放到 src1 的尺寸（`src1w × src1h`）。但如果缩放导致质量下降，过渡边缘会不平滑。解决方案：确保规则图像分辨率与游戏窗口分辨率一致。

### 错误 2：Scroll 过渡方向参数超出范围

```
症状：传入 from=5 导致画面不动或显示异常
原因：枚举值只有 0-3
```

```cpp
// 安全的参数校验
tTVPScrollTransFrom from;
er = options->GetAsNumber(TJS_W("from"), &value);
if (TJS_FAILED(er) || value < 0 || value > 3)
    from = sttLeft;  // 越界时回退到默认值
else
    from = (tTVPScrollTransFrom)value;
```

## 本节小结

- **CrossFade** 通过线性 Alpha 插值实现淡入淡出，支持三种图层类型，在 Phase=0 和 PhaseMax 时有指针直传优化
- **Universal** 使用灰度规则图像控制切换顺序，Vague 参数控制边缘模糊度，PhaseMax = 255 + Vague
- **Scroll** 支持 4 方向 × 3 停留模式 = 12 种组合，MaxPhase 取决于画面尺寸
- 所有内置过渡已从 CPU 像素操作迁移到 GPU 加速（通过 RenderManager::OperateRect）

## 练习题与答案

### 题目 1：设计一个规则图像，使得 Universal 过渡呈现"从四角向中心收缩"的效果。描述图像内容并给出生成代码。

<details>
<summary>查看答案</summary>

需要生成一个**中心亮、四角暗**的规则图像。由于规则值小的区域先切换到 Src2，四角（暗=0）先切换，中心（亮=255）最后切换。

```cpp
void GenerateCornerToCenter(uint8_t* rule, int w, int h) {
    float cx = w / 2.0f, cy = h / 2.0f;
    // 使用到最近边缘的距离（Chebyshev 距离）
    for (int y = 0; y < h; y++) {
        for (int x = 0; x < w; x++) {
            // 到四角的最小距离 → 四角最暗
            float distToEdge = std::min({
                (float)x, (float)y,
                (float)(w-1-x), (float)(h-1-y)
            });
            float maxDist = std::min(cx, cy);
            rule[y * w + x] = static_cast<uint8_t>(
                distToEdge / maxDist * 255.0f);
        }
    }
}
// 效果：四角先切换到 Src2 → 逐渐向中心推进 → 中心最后切换
```

</details>

### 题目 2：Scroll 过渡中 `stay=ststStayDest` 时为什么需要裁剪 Src1 的矩形？

<details>
<summary>查看答案</summary>

当 `stay=ststStayDest` 时，Src2（目标画面）保持不动，Src1（源画面）被推出屏幕。两者可能有重叠区域。

如果不裁剪 Src1 的矩形，在重叠区域内 Src1 会被绘制在 Src2 之上（因为绘制顺序是先 Src1 后 Src2），导致 Src2 被遮挡。但 `StayDest` 的语义是 Src2 不动且始终可见。

因此需要裁剪 Src1，去除与 Src2 重叠的部分：

```cpp
if (stay == ststStayDest) {
    // Src2 有优先级，裁剪 Src1
    if (from == sttLeft || from == sttRight) {
        // 水平方向：调整 Src1 的左右边界
        if (rs2.right >= rs1.left && rs2.right < rs1.right)
            rs1.left = rs2.right;   // Src2 右边在 Src1 内部
        if (rs2.left >= rs1.left && rs2.left < rs1.right)
            rs1.right = rs2.left;   // Src2 左边在 Src1 内部
    }
    // 垂直方向类似
}
```

这确保了在屏幕上 Src2 始终在前，Src1 只显示未被 Src2 覆盖的部分。

</details>
