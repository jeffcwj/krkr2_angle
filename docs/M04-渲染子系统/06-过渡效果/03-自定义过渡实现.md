## Provider-Handler 实现模式

### 回顾：两层架构

过渡系统采用 Provider-Handler 分离模式：

- **Provider（提供者）**：工厂角色，负责解析 TJS 脚本传入的参数字典，创建对应的 Handler 实例
- **Handler（处理器）**：执行角色，负责在每帧中根据当前进度（curtain 值）完成实际的像素混合

```
TJS 脚本调用 transition(...)
       │
       ▼
  TVPFindTransHandlerProvider(name)
       │ 查找注册表
       ▼
  Provider::GetHandler(option_dict)
       │ 解析参数，创建 Handler
       ▼
  Handler::StartProcess()  →  Handler::Process()×N  →  Handler::EndProcess()
       │                          │
       │                          ▼
       │                    每帧混合像素
       ▼
  delete Handler
```

### 核心接口定义

实现自定义过渡效果前，必须了解两个核心接口。源码位于 `cpp/core/visual/transhandler.h`：

```cpp
// --- iTVPDivisibleTransHandler: 过渡处理器接口 ---
// 源码: transhandler.h 第 31-80 行
class iTVPDivisibleTransHandler {
public:
    virtual ~iTVPDivisibleTransHandler() {}

    // 设置过渡参数（层尺寸等）
    virtual void SetStage(
        iTVPDivisibleTransHandlerProvider *provider,
        tjs_int stage_idx,      // 阶段索引（通常为 0）
        tjs_int stage_num       // 阶段总数（通常为 1）
    ) = 0;

    // 获取最大相位值（过渡总帧数）
    virtual tjs_int GetMaxPhase() = 0;

    // 开始一帧处理
    virtual tjs_int StartProcess(tjs_int phase) = 0;

    // 执行实际混合（核心方法）
    virtual void Process(
        tTVPDivisibleTransHandlerInfo *info  // 混合信息
    ) = 0;

    // 结束一帧处理
    virtual void EndProcess() = 0;

    // 释放
    virtual void Release() { delete this; }
};

// --- iTVPDivisibleTransHandlerProvider: 过渡提供者接口 ---
// 源码: transhandler.h 第 83-107 行
class iTVPDivisibleTransHandlerProvider {
public:
    virtual ~iTVPDivisibleTransHandlerProvider() {}

    // 工厂方法：创建 Handler
    virtual iTVPDivisibleTransHandler *GetHandler(
        tjs_int stage_count,    // 阶段数量
        iTJSDispatch2 *options  // TJS 字典，包含过渡参数
    ) = 0;

    // 提供选项元数据（告知脚本端有哪些参数）
    virtual void GetOptionProvider(
        iTVPSimpleOptionProvider *provider
    ) = 0;

    // 提供图像元数据（告知需要哪些额外图像，如 rule 图）
    virtual void GetImageProvider(
        iTVPSimpleImageProvider *provider
    ) = 0;

    // 释放
    virtual void Release() { delete this; }
};
```

## 实战：实现"百叶窗"过渡效果

我们实现一个"百叶窗"（Blinds）过渡效果——画面被分成若干水平条带，每个条带逐渐从源图像 1 过渡到源图像 2。

### 第一步：定义 Handler

```cpp
// BlindsTrans.h — 百叶窗过渡 Handler
#pragma once
#include "transhandler.h"

class tTVPBlindsDivisibleTransHandler
    : public iTVPDivisibleTransHandler {
private:
    // 过渡参数
    tjs_int NumBlinds;       // 百叶窗条带数量（默认 8）
    tjs_int MaxPhase;        // 最大相位值
    tjs_int CurPhase;        // 当前相位
    tjs_int LayerWidth;      // 图层宽度
    tjs_int LayerHeight;     // 图层高度

    // Provider 引用（用于获取图层尺寸）
    iTVPDivisibleTransHandlerProvider *Provider;

public:
    tTVPBlindsDivisibleTransHandler(tjs_int numBlinds)
        : NumBlinds(numBlinds > 0 ? numBlinds : 8)
        , MaxPhase(255)
        , CurPhase(0)
        , LayerWidth(0)
        , LayerHeight(0)
        , Provider(nullptr) {}

    // 设置阶段参数
    void SetStage(
        iTVPDivisibleTransHandlerProvider *provider,
        tjs_int stage_idx,
        tjs_int stage_num
    ) override {
        Provider = provider;
    }

    // 返回最大相位值（0~255）
    tjs_int GetMaxPhase() override {
        return MaxPhase;
    }

    // 开始一帧处理：返回需要处理的区域数
    tjs_int StartProcess(tjs_int phase) override {
        CurPhase = phase;
        return 1;  // 单区域处理
    }

    // 核心混合逻辑
    void Process(
        tTVPDivisibleTransHandlerInfo *info
    ) override;

    // 结束一帧
    void EndProcess() override {}

    void Release() override { delete this; }
};
```

### 第二步：实现 Process 方法

`Process` 方法是过渡效果的核心。它接收一个 `tTVPDivisibleTransHandlerInfo` 结构，包含源图像 1、源图像 2、目标缓冲区等信息：

```cpp
// BlindsTrans.cpp — Process 核心实现
#include "BlindsTrans.h"
#include "RenderManager.h"

void tTVPBlindsDivisibleTransHandler::Process(
    tTVPDivisibleTransHandlerInfo *info
) {
    // 获取图层尺寸
    tjs_int w = info->Width;
    tjs_int h = info->Height;

    if(w == 0 || h == 0) return;

    // 计算每个条带的高度
    tjs_int blindHeight = h / NumBlinds;
    if(blindHeight < 1) blindHeight = 1;

    // 计算当前相位下每个条带已展开的像素行数
    // phase 从 0 到 MaxPhase（255）
    tjs_int revealedRows =
        (tjs_int)((tjs_int64)blindHeight * CurPhase / MaxPhase);

    // GPU 路径：使用 RenderManager 的矩形操作
    iTVPRenderManager *rm = TVPGetRenderManager();

    for(tjs_int i = 0; i < NumBlinds; i++) {
        tjs_int y0 = i * blindHeight;
        tjs_int y1 = y0 + revealedRows;
        if(y1 > h) y1 = h;

        // 已展开区域：复制源图像 2（Src2）
        if(revealedRows > 0 && y0 < h) {
            tTVPRect srcRect(0, y0, w, y1);
            tTVPRect dstRect(0, y0, w, y1);
            // 复制 Src2 到目标
            info->Dest->OperateRect(
                &dstRect, info->Src2, &srcRect,
                nullptr, nullptr,
                rmPsCopy,     // 直接复制模式
                255           // 完全不透明
            );
        }

        // 未展开区域：保留源图像 1（Src1）
        tjs_int y2 = y0 + blindHeight;
        if(y2 > h) y2 = h;
        if(y1 < y2) {
            tTVPRect srcRect(0, y1, w, y2);
            tTVPRect dstRect(0, y1, w, y2);
            info->Dest->OperateRect(
                &dstRect, info->Src1, &srcRect,
                nullptr, nullptr,
                rmPsCopy, 255
            );
        }
    }

    // 处理最后不足一个完整条带的剩余行
    tjs_int lastY = NumBlinds * blindHeight;
    if(lastY < h) {
        tTVPRect srcRect(0, lastY, w, h);
        tTVPRect dstRect(0, lastY, w, h);
        // 根据进度决定使用 Src1 还是 Src2
        auto src = (CurPhase > MaxPhase / 2)
            ? info->Src2 : info->Src1;
        info->Dest->OperateRect(
            &dstRect, src, &srcRect,
            nullptr, nullptr,
            rmPsCopy, 255
        );
    }
}
```

### 第三步：定义 Provider

Provider 负责解析 TJS 字典参数并创建 Handler：

```cpp
// BlindsTrans.cpp — Provider 实现
class tTVPBlindsTransHandlerProvider
    : public iTVPDivisibleTransHandlerProvider {
public:
    // 工厂方法
    iTVPDivisibleTransHandler *GetHandler(
        tjs_int stage_count,
        iTJSDispatch2 *options
    ) override {
        // 从选项字典中读取 "numBlinds" 参数
        tjs_int numBlinds = 8;  // 默认值
        if(options) {
            tTJSVariant val;
            if(TJS_SUCCEEDED(options->PropGet(
                TJS_MEMBERMUSTEXIST,
                TJS_W("numBlinds"),  // 参数名
                nullptr, &val, options)))
            {
                numBlinds = (tjs_int)val.AsInteger();
            }
        }

        // 创建并返回 Handler
        return new tTVPBlindsDivisibleTransHandler(
            numBlinds
        );
    }

    // 提供选项元数据
    void GetOptionProvider(
        iTVPSimpleOptionProvider *provider
    ) override {
        if(provider) {
            // 通知脚本端：我们接受 "numBlinds" 参数
            provider->AddOption(TJS_W("numBlinds"),
                TJS_W("8"));  // 默认值字符串
        }
    }

    // 提供图像元数据（百叶窗不需要额外图像）
    void GetImageProvider(
        iTVPSimpleImageProvider *provider
    ) override {
        // 不需要 rule 图或其他辅助图像
    }

    void Release() override { delete this; }
};
```

## 参数解析的完整流程

### 从 TJS 字典到 C++ 参数

项目源码中内置过渡效果的参数解析遵循统一模式。以 `TransIntf.cpp` 中 CrossFade 的参数解析为例（第 92-134 行）：

```cpp
// 源码: TransIntf.cpp 第 92-134 行
// CrossFade Provider 的 GetHandler 方法
iTVPDivisibleTransHandler *GetHandler(
    tjs_int stage_count,
    iTJSDispatch2 *options
) override {
    // 1. 设置默认值
    tjs_int layertype = 0;  // ltAlpha

    // 2. 从选项字典读取参数
    if(options) {
        tTJSVariant val;
        if(TJS_SUCCEEDED(options->PropGet(
            TJS_MEMBERMUSTEXIST,
            TJS_W("layertype"), nullptr,
            &val, options)))
        {
            layertype = (tjs_int)val.AsInteger();
        }
    }

    // 3. 根据参数创建不同的 Handler
    switch(layertype) {
        case 0:   // ltAlpha
            return new tTVPCrossFadeDivisibleTransHandler(
                ltAlpha);
        case 1:   // ltAddAlpha
            return new tTVPCrossFadeDivisibleTransHandler(
                ltAddAlpha);
        case 2:   // ltOpaque
            return new tTVPCrossFadeDivisibleTransHandler(
                ltOpaque);
        default:
            return new tTVPCrossFadeDivisibleTransHandler(
                ltAlpha);
    }
}
```

## 注册系统深度解析

### 注册表内部结构

过渡效果的注册使用全局哈希表。源码位于 `TransIntf.cpp` 第 21-62 行：

```cpp
// 源码: TransIntf.cpp 第 21-40 行
struct tTVPTransHandlerProviderHolder {
    ttstr Name;
    iTVPDivisibleTransHandlerProvider *Provider;

    tTVPTransHandlerProviderHolder(
        const ttstr &name,
        iTVPDivisibleTransHandlerProvider *provider
    ) : Name(name), Provider(provider) {}
};

// 全局注册表（动态数组，不是哈希表）
static std::vector<tTVPTransHandlerProviderHolder>
    TVPTransHandlerProviders;

// 内置效果是否已注册标记
static bool TVPTransHandlerProvidersInit = false;
```

### 查找逻辑与回退策略

引擎查找过渡效果时使用 `TVPFindTransHandlerProvider`。源码位于 `TransIntf.cpp` 第 42-62 行：

```cpp
// 源码: TransIntf.cpp 第 42-62 行
iTVPDivisibleTransHandlerProvider *
TVPFindTransHandlerProvider(const ttstr &name) {
    // 延迟初始化：首次调用时注册所有内置效果
    if(!TVPTransHandlerProvidersInit) {
        TVPTransHandlerProvidersInit = true;

        // 注册三个内置效果
        TVPAddTransHandlerProvider(
            TJS_W("crossfade"),
            new tTVPCrossFadeTransHandlerProvider());
        TVPAddTransHandlerProvider(
            TJS_W("universal"),
            new tTVPUniversalTransHandlerProvider());
        TVPAddTransHandlerProvider(
            TJS_W("scroll"),
            new tTVPScrollTransHandlerProvider());
    }

    // 遍历查找匹配名称
    for(auto &holder : TVPTransHandlerProviders) {
        if(holder.Name == name)
            return holder.Provider;
    }

    // 未找到：输出警告并回退到 crossfade
    static bool warned = false;
    if(!warned) {
        TVPAddLog(TJS_W("Warning: transition '")
            + name
            + TJS_W("' not found, using crossfade"));
        warned = true;
    }
    return TVPFindTransHandlerProvider(
        TJS_W("crossfade"));
}
```

注意几个关键设计点：
1. **延迟初始化**：内置效果直到第一次查找时才注册，避免启动开销
2. **回退策略**：未知名称不会崩溃，而是回退到 crossfade 并输出一次性警告
3. **线性查找**：注册表使用 `std::vector` 线性遍历，因为过渡效果数量通常很少（<20），性能不是瓶颈

## 跨平台注意事项

过渡效果的实现涉及 GPU 渲染操作，不同平台有差异：

| 平台 | GPU API | 注意事项 |
|------|---------|----------|
| Windows | OpenGL 3.x / DirectX | 确保 GL 状态通过 Cocos2d-x 缓存调用 |
| Linux | OpenGL 3.x | Mesa 驱动可能不支持某些纹理格式 |
| macOS | OpenGL 2.1（已废弃）| Apple 已废弃 OpenGL，未来需迁移到 Metal |
| Android | OpenGL ES 2.0/3.0 | 纹理尺寸受 `GL_MAX_TEXTURE_SIZE` 限制 |

对于纯 CPU 路径的过渡实现（如不使用 `OperateRect`），四个平台行为一致：

```cpp
// 跨平台安全的 CPU 混合路径示例
void ProcessCPU(tjs_uint32 *dest,
                const tjs_uint32 *src1,
                const tjs_uint32 *src2,
                tjs_int width,
                tjs_int phase,
                tjs_int maxPhase)
{
    // 使用 tvpgl 混合函数（所有平台一致）
    tjs_int opacity = phase * 255 / maxPhase;
    TVPConstAlphaBlend_SD(
        dest,    // 目标缓冲区
        src1,    // 源图像 1
        src2,    // 源图像 2
        width,   // 像素数
        opacity  // 不透明度 0-255
    );
}
```

## 动手实践

### 练习：实现"对角线擦除"过渡

实现一个从左上角到右下角的对角线擦除效果。分界线沿对角线方向移动，分界线上方显示新图像（Src2），下方显示旧图像（Src1）。

完整步骤：

1. 创建 `DiagonalWipeTrans.h` 和 `DiagonalWipeTrans.cpp`
2. Handler 的 `Process` 方法中，计算每行像素的分界点
3. Provider 解析 `direction` 参数（支持 4 个方向）
4. 注册到引擎

核心 Process 实现参考：

```cpp
void tTVPDiagonalWipeTransHandler::Process(
    tTVPDivisibleTransHandlerInfo *info
) {
    tjs_int w = info->Width;
    tjs_int h = info->Height;

    // 对角线位置（从 0 到 w+h）
    tjs_int diagPos =
        (tjs_int)((tjs_int64)(w + h) * CurPhase / MaxPhase);

    for(tjs_int y = 0; y < h; y++) {
        // 该行的分界 x 坐标
        tjs_int splitX = diagPos - y;
        if(splitX < 0) splitX = 0;
        if(splitX > w) splitX = w;

        // 左侧（已擦除）：显示 Src2
        if(splitX > 0) {
            tTVPRect src(0, y, splitX, y + 1);
            tTVPRect dst(0, y, splitX, y + 1);
            info->Dest->OperateRect(
                &dst, info->Src2, &src,
                nullptr, nullptr, rmPsCopy, 255);
        }
        // 右侧（未擦除）：显示 Src1
        if(splitX < w) {
            tTVPRect src(splitX, y, w, y + 1);
            tTVPRect dst(splitX, y, w, y + 1);
            info->Dest->OperateRect(
                &dst, info->Src1, &src,
                nullptr, nullptr, rmPsCopy, 255);
        }
    }
}
```

## 常见错误及解决方案

### 错误 1：GetMaxPhase 返回 0 导致除零

```
症状：过渡开始后立即崩溃，调用栈指向 phase 计算
原因：GetMaxPhase() 返回了 0
```

修复：确保 `GetMaxPhase()` 始终返回正数：

```cpp
tjs_int GetMaxPhase() override {
    // 错误：return 0;
    // 正确：至少返回 1
    return MaxPhase > 0 ? MaxPhase : 255;
}
```

### 错误 2：Process 中未处理全部像素区域导致残影

```
症状：过渡期间某些区域显示黑色或上一帧残影
原因：Process 方法没有覆盖目标缓冲区的所有像素
```

修复：确保每一帧中目标缓冲区的所有区域都被写入：

```cpp
void Process(tTVPDivisibleTransHandlerInfo *info) override {
    // 必须确保 (0,0)-(w,h) 范围内每个像素都被写入
    // 即使某些区域"不需要变化"，也要从 Src1 复制过来
    tTVPRect fullRect(0, 0, info->Width, info->Height);
    // 先填充 Src1 作为基底
    info->Dest->OperateRect(
        &fullRect, info->Src1, &fullRect,
        nullptr, nullptr, rmPsCopy, 255);
    // 再在需要的区域覆盖 Src2
    // ...
}
```

### 错误 3：Provider 注册后 TJS 脚本找不到

```
症状：TJS 报错 "transition method not found" 或回退到 crossfade
原因：注册函数没有在引擎初始化时调用，或名称大小写不匹配
```

修复：
1. 确保注册函数在 `TVPTransHandlerProvidersInit` 初始化块中调用
2. 名称字符串精确匹配（区分大小写）

```cpp
// 在 TransIntf.cpp 的初始化块中添加
if(!TVPTransHandlerProvidersInit) {
    TVPTransHandlerProvidersInit = true;
    // 内置效果...
    TVPAddTransHandlerProvider(
        TJS_W("crossfade"), new tTVPCrossFadeTransHandlerProvider());
    // 添加自定义效果
    TVPAddTransHandlerProvider(
        TJS_W("blinds"),  // 必须与 TJS 中的 method 完全一致
        new tTVPBlindsTransHandlerProvider());
}
```

## 本节小结

- 自定义过渡效果需要实现 **Provider**（工厂）和 **Handler**（执行器）两个类
- Handler 的 `Process` 方法是核心，接收 `tTVPDivisibleTransHandlerInfo`，完成源图像混合
- Provider 的 `GetHandler` 方法解析 TJS 字典参数，`GetOptionProvider` 提供参数元数据
- 注册通过 `TVPAddTransHandlerProvider(name, provider)` 完成
- 引擎对未知过渡名称有回退策略（crossfade + 一次性警告）
- GPU 路径使用 `OperateRect`，CPU 路径使用 `tvpgl` 函数，两者均跨平台兼容

### 题目 1：实现一个"旋转百叶窗"过渡效果

在标准百叶窗基础上，让每个条带不是简单地展开，而是从中心向两侧展开（类似窗帘拉开）。写出 Handler 的 `Process` 方法核心逻辑。

<details>
<summary>查看答案</summary>

```cpp
void tTVPRotatingBlindsHandler::Process(
    tTVPDivisibleTransHandlerInfo *info
) {
    tjs_int w = info->Width;
    tjs_int h = info->Height;
    tjs_int blindHeight = h / NumBlinds;
    if(blindHeight < 1) blindHeight = 1;

    // 从中心展开：计算半展开量
    tjs_int halfReveal =
        (tjs_int)((tjs_int64)(blindHeight / 2)
        * CurPhase / MaxPhase);

    for(tjs_int i = 0; i < NumBlinds; i++) {
        tjs_int blindTop = i * blindHeight;
        tjs_int blindCenter = blindTop + blindHeight / 2;

        // 中心展开区域：Src2
        tjs_int revealTop = blindCenter - halfReveal;
        tjs_int revealBot = blindCenter + halfReveal;
        if(revealTop < blindTop) revealTop = blindTop;
        if(revealBot > blindTop + blindHeight)
            revealBot = blindTop + blindHeight;
        if(revealBot > h) revealBot = h;

        // 上方未展开区域：Src1
        if(revealTop > blindTop) {
            tTVPRect s(0, blindTop, w, revealTop);
            tTVPRect d(0, blindTop, w, revealTop);
            info->Dest->OperateRect(
                &d, info->Src1, &s,
                nullptr, nullptr, rmPsCopy, 255);
        }
        // 中心展开区域：Src2
        if(revealTop < revealBot) {
            tTVPRect s(0, revealTop, w, revealBot);
            tTVPRect d(0, revealTop, w, revealBot);
            info->Dest->OperateRect(
                &d, info->Src2, &s,
                nullptr, nullptr, rmPsCopy, 255);
        }
        // 下方未展开区域：Src1
        tjs_int blindBottom = blindTop + blindHeight;
        if(blindBottom > h) blindBottom = h;
        if(revealBot < blindBottom) {
            tTVPRect s(0, revealBot, w, blindBottom);
            tTVPRect d(0, revealBot, w, blindBottom);
            info->Dest->OperateRect(
                &d, info->Src1, &s,
                nullptr, nullptr, rmPsCopy, 255);
        }
    }
}
```

关键点：每个条带计算中心位置 `blindCenter`，然后 `halfReveal` 从中心向两侧扩展。

</details>

### 题目 2：为什么 TransIntf.cpp 中过渡注册使用 `std::vector` 而不是 `std::unordered_map`？

<details>
<summary>查看答案</summary>

原因有三：

1. **数量极少**：一个游戏通常只注册 3-10 种过渡效果，线性查找的 O(n) 在 n<20 时比哈希表更快（没有哈希计算和桶分配开销）
2. **查找频率低**：过渡效果只在场景切换时查找一次，不是每帧调用。即使是 O(n) 也完全可以接受
3. **历史原因**：原始 KiriKiri2 代码风格倾向于简单数据结构。`std::vector` 配合 `ttstr` 比较已足够

如果需要支持大量自定义过渡（>50种），可考虑改用 `tTJSHashTable<ttstr, iTVPDivisibleTransHandlerProvider*>`，与图像加载器注册表类似。

</details>

### 题目 3：如果需要让百叶窗过渡支持"垂直条带"模式，需要修改哪些部分？

<details>

需要修改三个部分：

1. **Provider**：添加 `direction` 参数解析

```cpp
iTVPDivisibleTransHandler *GetHandler(
    tjs_int stage_count,
    iTJSDispatch2 *options

}
```

2. **Handler 构造函数**：保存 direction 参数

3. **Process 方法**：根据 direction 切换计算轴

```cpp
void Process(tTVPDivisibleTransHandlerInfo *info) override {
    tjs_int w = info->Width, h = info->Height;
    if(Direction == 0) {
        // 水平条带（原有逻辑，按行分割）
        tjs_int blindH = h / NumBlinds;
        // ... 原有代码 ...
    } else {
        // 垂直条带（按列分割）
        tjs_int blindW = w / NumBlinds;
        if(blindW < 1) blindW = 1;
        tjs_int revealedCols =
            (tjs_int)((tjs_int64)blindW
            * CurPhase / MaxPhase);

        for(tjs_int i = 0; i < NumBlinds; i++) {
            tjs_int x0 = i * blindW;
            tjs_int x1 = x0 + revealedCols;
            if(x1 > w) x1 = w;
            // 已展开列：Src2
            if(revealedCols > 0 && x0 < w) {
                tTVPRect s(x0, 0, x1, h);
                tTVPRect d(x0, 0, x1, h);
                info->Dest->OperateRect(
                    &d, info->Src2, &s,
                    nullptr, nullptr,
                    rmPsCopy, 255);
            }
            // 未展开列：Src1
            tjs_int x2 = x0 + blindW;
            if(x2 > w) x2 = w;
            if(x1 < x2) {
                tTVPRect s(x1, 0, x2, h);
                tTVPRect d(x1, 0, x2, h);
                info->Dest->OperateRect(
                    &d, info->Src1, &s,
                    nullptr, nullptr,
                    rmPsCopy, 255);
            }
        }
    }
}
```
