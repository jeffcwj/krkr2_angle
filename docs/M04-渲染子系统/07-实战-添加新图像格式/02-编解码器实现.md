# 7.2 编解码器实现

## 本节目标

- 掌握 TGA 格式的二进制结构（头部、像素数据、RLE 压缩）
- 实现完整的 `TVPLoadTGA()` 解码器，正确调用 `SizeCallback` / `ScanLineCallback`
- 实现 `TVPSaveAsTGA()` 编码器，将 BGRA 位图写入 TGA 文件
- 实现 `TVPLoadHeaderTGA()` 快速头部解析，返回宽高与色深信息
- 理解像素格式转换（灰度→BGRA、RGB→BGRA、RGBA→BGRA）

## 7.2.1 TGA 格式二进制结构

TGA（Truevision Graphics Adapter）是最简单的光栅格式之一，非常适合作为学习案例——无需引入第三方库，纯二进制解析即可实现。

### TGA 头部结构（18 字节）

```cpp
// 示例 1：TGA 头部定义
// 文件：LoadTGA.h（新建）
#pragma pack(push, 1)
struct tTGAHeader {
    tjs_uint8  idLength;        // 图像标识符长度（通常为 0）
    tjs_uint8  colorMapType;    // 颜色表类型：0=无, 1=有
    tjs_uint8  imageType;       // 图像类型：2=RGB, 3=灰度, 10=RLE-RGB, 11=RLE-灰度
    // 颜色表规格（5 字节）
    tjs_uint16 colorMapOrigin;  // 颜色表起始索引
    tjs_uint16 colorMapLength;  // 颜色表条目数
    tjs_uint8  colorMapDepth;   // 每条目位数（15/16/24/32）
    // 图像规格（10 字节）
    tjs_uint16 xOrigin;         // X 原点（通常为 0）
    tjs_uint16 yOrigin;         // Y 原点（通常为 0）
    tjs_uint16 width;           // 图像宽度（像素）
    tjs_uint16 height;          // 图像高度（像素）
    tjs_uint8  bitsPerPixel;    // 每像素位数：8/24/32
    tjs_uint8  descriptor;      // 图像描述符，bit5=上下翻转标志
};
#pragma pack(pop)
```

> **概念解释**：`#pragma pack(push, 1)` 强制编译器按 1 字节对齐结构体，确保与文件中的二进制布局完全一致。如果不指定对齐方式，编译器可能在字段之间插入填充字节，导致 `sizeof(tTGAHeader)` 大于 18 字节。读取文件时一次 `ReadBuffer` 就能拿到完整头部，无需逐字段解析。

### 图像类型速查

| `imageType` | 含义 | 压缩 | 像素格式 |
|:-----------:|:----:|:----:|:--------:|
| 2 | 真彩色 | 无 | BGR / BGRA |
| 3 | 灰度 | 无 | 8bit Gray |
| 10 | 真彩色 | RLE | BGR / BGRA |
| 11 | 灰度 | RLE | 8bit Gray |

```
┌─────────────────────────────────────────┐
│  TGA 文件结构                            │
├──────────┬──────────────────────────────┤
│  0-17    │ tTGAHeader (18 字节)          │
│  18..    │ Image ID (idLength 字节)      │
│  ...     │ Color Map (如有)              │
│  ...     │ Pixel Data (未压缩/RLE 压缩)  │
└──────────┴──────────────────────────────┘
```

## 7.2.2 实现 TVPLoadTGA 解码器

解码器是格式支持的核心。它必须：
1. 读取 18 字节头部 → 获取宽高、色深、压缩方式
2. 调用 `SizeCallback` 通知框架图像尺寸
3. 逐行读取/解压像素数据 → 格式转换为 BGRA → 调用 `ScanLineCallback`

### 完整解码器实现

```cpp
// 示例 2：TVPLoadTGA 完整实现
// 文件：LoadTGA.cpp（新建）
#include "tjsCommHead.h"
#include "GraphicsLoaderIntf.h"
#include "MsgIntf.h"
#include "LoadTGA.h"

// RLE 解压辅助：将 RLE 编码的像素流解压到 destBuf
// bytesPerPixel：每像素字节数（1/3/4）
// pixelCount：本行总像素数
static void DecompressRLERow(
    tTJSBinaryStream* stream,
    tjs_uint8* destBuf,
    int bytesPerPixel,
    int pixelCount)
{
    int decoded = 0;
    while (decoded < pixelCount) {
        tjs_uint8 packet;
        stream->ReadBuffer(&packet, 1);

        int count = (packet & 0x7F) + 1; // 像素重复/原始数量
        if (packet & 0x80) {
            // Run-Length 包：1 个像素值重复 count 次
            tjs_uint8 pixel[4];
            stream->ReadBuffer(pixel, bytesPerPixel);
            for (int i = 0; i < count && decoded < pixelCount; i++, decoded++) {
                memcpy(destBuf + decoded * bytesPerPixel,
                       pixel, bytesPerPixel);
            }
        } else {
            // Raw 包：count 个不同像素值
            for (int i = 0; i < count && decoded < pixelCount; i++, decoded++) {
                stream->ReadBuffer(
                    destBuf + decoded * bytesPerPixel,
                    bytesPerPixel);
            }
        }
    }
}

// 像素格式转换：将任意格式源行转换为 BGRA（32 位）
// srcBpp：源像素字节数（1=灰度, 3=BGR, 4=BGRA）
static void ConvertRowToBGRA(
    const tjs_uint8* src,
    tjs_uint32* dest,
    int width,
    int srcBpp)
{
    for (int x = 0; x < width; x++) {
        switch (srcBpp) {
        case 1: {
            // 灰度 → BGRA：R=G=B=Gray, A=255
            tjs_uint8 g = src[x];
            dest[x] = (0xFF << 24) | (g << 16) | (g << 8) | g;
            break;
        }
        case 3: {
            // BGR → BGRA：补 A=255
            tjs_uint8 b = src[x * 3 + 0];
            tjs_uint8 g = src[x * 3 + 1];
            tjs_uint8 r = src[x * 3 + 2];
            dest[x] = (0xFF << 24) | (r << 16) | (g << 8) | b;
            break;
        }
        case 4: {
            // BGRA → 直接拷贝（TGA 原生 BGRA 与 KrKr2 内部一致）
            memcpy(&dest[x], &src[x * 4], 4);
            break;
        }
        }
    }
}

void TVPLoadTGA(void* formatdata,
                void* callbackdata,
                tTVPGraphicSizeCallback sizecallback,
                tTVPGraphicScanLineCallback scanlinecallback,
                tTVPGraphicMetaInfoPushCallback metainfopushcallback,
                tTJSBinaryStream* src,
                tjs_int keyidx,
                tTVPGraphicLoadMode mode)
{
    // ① 读取头部
    tTGAHeader header;
    src->ReadBuffer(&header, sizeof(tTGAHeader));

    // 验证图像类型
    if (header.imageType != 2 && header.imageType != 3 &&
        header.imageType != 10 && header.imageType != 11) {
        TVPThrowExceptionMessage(
            TJS_W("Unsupported TGA image type: %1"),
            ttstr((tjs_int)header.imageType));
    }

    // 跳过 Image ID
    if (header.idLength > 0) {
        src->SetPosition(src->GetPosition() + header.idLength);
    }

    // 跳过颜色表（简化实现不支持调色板 TGA）
    if (header.colorMapType == 1) {
        int cmapBytes = header.colorMapLength *
                        ((header.colorMapDepth + 7) / 8);
        src->SetPosition(src->GetPosition() + cmapBytes);
    }

    int w = header.width;
    int h = header.height;
    int bpp = header.bitsPerPixel / 8; // 每像素字节数
    bool rle = (header.imageType == 10 || header.imageType == 11);
    bool topDown = (header.descriptor & 0x20) != 0;

    // ② 通知框架图像尺寸
    sizecallback(callbackdata, w, h);

    // ③ 分配行缓冲区
    std::vector<tjs_uint8> rowBuf(w * bpp);
    std::vector<tjs_uint32> bgraRow(w);

    // ④ 逐行读取并转换
    for (int y = 0; y < h; y++) {
        int destY = topDown ? y : (h - 1 - y);

        if (rle) {
            DecompressRLERow(src, rowBuf.data(), bpp, w);
        } else {
            src->ReadBuffer(rowBuf.data(), w * bpp);
        }

        // 根据加载模式处理像素
        if (mode == glmGrayscale) {
            // 灰度模式：取第一字节作为灰度值
            tjs_uint8* line = (tjs_uint8*)scanlinecallback(
                callbackdata, destY);
            if (!line) continue;
            for (int x = 0; x < w; x++) {
                if (bpp == 1) {
                    line[x] = rowBuf[x];
                } else {
                    // 从 BGR(A) 计算亮度：Y = 0.299R + 0.587G + 0.114B
                    int b = rowBuf[x * bpp + 0];
                    int g = rowBuf[x * bpp + 1];
                    int r = rowBuf[x * bpp + 2];
                    line[x] = (tjs_uint8)(
                        (r * 77 + g * 150 + b * 29) >> 8);
                }
            }
            scanlinecallback(callbackdata, -1); // 通知行完成
        } else {
            // 正常模式（glmNormal）：转换为 BGRA
            ConvertRowToBGRA(rowBuf.data(), bgraRow.data(), w, bpp);

            // 处理色键透明（keyidx >= 0 时）
            if (keyidx >= 0) {
                tjs_uint32 keyColor = (tjs_uint32)keyidx | 0xFF000000;
                for (int x = 0; x < w; x++) {
                    if ((bgraRow[x] | 0xFF000000) == keyColor) {
                        bgraRow[x] = 0x00000000; // 透明
                    }
                }
            }

            tjs_uint32* line = (tjs_uint32*)scanlinecallback(
                callbackdata, destY);
            if (!line) continue;
            memcpy(line, bgraRow.data(), w * sizeof(tjs_uint32));
            scanlinecallback(callbackdata, -1);
        }
    }

    // ⑤ 推送元信息（可选）
    if (metainfopushcallback) {
        metainfopushcallback(callbackdata,
            TJS_W("format"), TJS_W("TGA"));
        metainfopushcallback(callbackdata,
            TJS_W("bits_per_pixel"),
            ttstr((tjs_int)header.bitsPerPixel).c_str());
        if (rle) {
            metainfopushcallback(callbackdata,
                TJS_W("compression"), TJS_W("RLE"));
        }
    }
}
```

> **概念解释**：`keyidx`（色键索引）是 KrKr2 图像加载系统的关键特性。当用户在 TJS 脚本中指定色键颜色时（如 `0x00FF00` 绿色），框架将该值传入 `keyidx` 参数。解码器需要将所有匹配该颜色的像素设为完全透明（`0x00000000`），从而实现"扣色"效果。这在早期不含 Alpha 通道的素材中非常常用。

> **概念解释**：`scanlinecallback(callbackdata, -1)` 的语义是"通知框架当前行写入完毕"。第二参数为 `-1` 时不返回行指针，仅作为信号。这一设计让框架可以在每行写入后立即进行后处理（如 Alpha 预乘、颜色空间转换），而非等整张图完成后批量处理。

## 7.2.3 实现 TVPSaveAsTGA 编码器

编码器相对简单——TGA 的 BGRA 字节序与 KrKr2 内部格式完全一致，无需额外转换。

```cpp
// 示例 3：TVPSaveAsTGA 编码器
void TVPSaveAsTGA(void* formatdata,
                  tTJSBinaryStream* dst,
                  const ttstr& mode,
                  tjs_int width,
                  tjs_int height,
                  tTVPGraphicSaveScanLineCallback scanlinecallback,
                  void* callbackdata)
{
    bool save32 = (mode == TJS_W("tga") || mode == TJS_W("tga32"));
    bool save24 = (mode == TJS_W("tga24"));
    bool saveGray = (mode == TJS_W("tga8"));

    int bpp = save32 ? 4 : (save24 ? 3 : 1);

    // 构造 TGA 头部
    tTGAHeader header;
    memset(&header, 0, sizeof(header));
    header.imageType = saveGray ? 3 : 2; // 灰度或真彩色
    header.width     = (tjs_uint16)width;
    header.height    = (tjs_uint16)height;
    header.bitsPerPixel = (tjs_uint8)(bpp * 8);
    header.descriptor   = 0x20; // 上→下行序

    if (save32) {
        header.descriptor |= 0x08; // 8 位 Alpha
    }

    dst->WriteBuffer(&header, sizeof(tTGAHeader));

    // 逐行写出像素数据
    std::vector<tjs_uint8> rowBuf(width * bpp);

    for (int y = 0; y < height; y++) {
        const tjs_uint32* srcLine =
            (const tjs_uint32*)scanlinecallback(callbackdata, y);
        if (!srcLine) continue;

        if (save32) {
            // BGRA → 直接写出
            dst->WriteBuffer(srcLine, width * 4);
        } else if (save24) {
            // BGRA → BGR（丢弃 Alpha）
            for (int x = 0; x < width; x++) {
                tjs_uint32 px = srcLine[x];
                rowBuf[x * 3 + 0] = (tjs_uint8)(px);       // B
                rowBuf[x * 3 + 1] = (tjs_uint8)(px >> 8);  // G
                rowBuf[x * 3 + 2] = (tjs_uint8)(px >> 16); // R
            }
            dst->WriteBuffer(rowBuf.data(), width * 3);
        } else {
            // BGRA → 灰度
            for (int x = 0; x < width; x++) {
                tjs_uint32 px = srcLine[x];
                int b = px & 0xFF;
                int g = (px >> 8) & 0xFF;
                int r = (px >> 16) & 0xFF;
                rowBuf[x] = (tjs_uint8)((r * 77 + g * 150 + b * 29) >> 8);
            }
            dst->WriteBuffer(rowBuf.data(), width);
        }
    }
}
```

> **概念解释**：`header.descriptor` 的 bit5 控制行序方向。设为 `0x20` 表示从上到下存储（与屏幕坐标一致），设为 `0` 表示从下到上（BMP 风格）。KrKr2 内部位图统一使用上到下行序，因此保存时设置 bit5 可避免额外翻转操作。同时 bit3-0 表示 Alpha 通道位数（`0x08` = 8 位 Alpha），对于 32 位 TGA 必须设置此标志。

## 7.2.4 实现 TVPLoadHeaderTGA 头部解析

头部解析函数用于快速获取图像元信息（宽、高、色深），无需加载全部像素数据。框架在 `TVPGetImageInfo` 等 TJS 函数中调用它。

```cpp
// 示例 4：TVPLoadHeaderTGA 头部解析
void TVPLoadHeaderTGA(void* formatdata,
                      tTJSBinaryStream* src,
                      iTJSDispatch2** dic)
{
    // 读取 18 字节头部
    tTGAHeader header;
    if (src->Read(&header, sizeof(tTGAHeader)) != sizeof(tTGAHeader)) {
        return; // 文件太短，非 TGA
    }

    // 简单验证：imageType 必须是已知类型
    if (header.imageType != 2 && header.imageType != 3 &&
        header.imageType != 10 && header.imageType != 11) {
        return; // 不是我们支持的 TGA 类型
    }

    // 宽高合理性检查
    if (header.width == 0 || header.height == 0 ||
        header.width > 16384 || header.height > 16384) {
        return;
    }

    // 色深检查
    if (header.bitsPerPixel != 8 &&
        header.bitsPerPixel != 24 &&
        header.bitsPerPixel != 32) {
        return;
    }

    // 创建 TJS 字典对象，填入元信息
    *dic = TJSCreateDictionaryObject();
    tTJSVariant val;

    val = (tjs_int)header.width;
    (*dic)->PropSet(TJS_MEMBERENSURE, TJS_W("width"), nullptr, &val, (*dic));

    val = (tjs_int)header.height;
    (*dic)->PropSet(TJS_MEMBERENSURE, TJS_W("height"), nullptr, &val, (*dic));

    val = (tjs_int)header.bitsPerPixel;
    (*dic)->PropSet(TJS_MEMBERENSURE, TJS_W("bpp"), nullptr, &val, (*dic));

    bool hasAlpha = (header.bitsPerPixel == 32);
    val = hasAlpha ? TJS_W("true") : TJS_W("false");
    (*dic)->PropSet(TJS_MEMBERENSURE, TJS_W("has_alpha"), nullptr, &val, (*dic));
}
```

## 7.2.5 与 PNG 解码器的对比

理解 TGA 与 PNG 的实现差异有助于掌握通用编解码器模式：

| 对比维度 | PNG（LoadPNG.cpp） | TGA（LoadTGA.cpp） |
|:--------:|:-----------------:|:-----------------:|
| 第三方库 | libpng（必需） | 无（纯手工解析） |
| 压缩算法 | DEFLATE（zlib） | RLE 或无压缩 |
| 内存分配 | `PNG_malloc` 自定义分配器 | `std::vector` 自动管理 |
| 行读取 | `png_read_row()` 一次解压一行 | 手动循环读取/解压 |
| 颜色转换 | libpng 内置 `png_set_bgr()` 等 | 手动 `ConvertRowToBGRA()` |
| 错误处理 | `setjmp/longjmp` 跳转 | 异常（`TVPThrowExceptionMessage`） |
| 头部解析 | `png_get_IHDR()` API | 直接读 18 字节结构体 |

```cpp
// 示例 5：PNG vs TGA 错误处理对比
// PNG 风格（setjmp）：
if (setjmp(png_jmpbuf(png_ptr))) {
    png_destroy_read_struct(&png_ptr, &info_ptr, NULL);
    TVPThrowExceptionMessage(TJS_W("PNG read error"));
}

// TGA 风格（C++ 异常）：
try {
    src->ReadBuffer(&header, sizeof(tTGAHeader));
} catch (...) {
    TVPThrowExceptionMessage(TJS_W("TGA read error"));
}
// TGA 不需要 setjmp 是因为没有使用 C 风格的第三方库，
// 可以直接使用 C++ 异常机制，代码更清晰安全。
```

### 练习：为 TGA 解码器添加 RLE 压缩保存

在 `TVPSaveAsTGA` 中添加 `tga_rle` 模式，使用 RLE 压缩输出：

```cpp
// 练习代码框架
static void WriteRLERow(tTJSBinaryStream* dst,
                        const tjs_uint8* row,
                        int width, int bpp)
{
    int x = 0;
    while (x < width) {
        // 检查连续相同像素
        int runStart = x;
        while (x + 1 < width && x - runStart < 127) {
            if (memcmp(row + x * bpp, row + (x + 1) * bpp, bpp) != 0)
                break;
            x++;
        }
        x++; // 包含当前像素

        if (x - runStart > 1) {
            // RLE 包
            tjs_uint8 packet = 0x80 | ((x - runStart) - 1);
            dst->WriteBuffer(&packet, 1);
            dst->WriteBuffer(row + runStart * bpp, bpp);
        } else {
            // 收集 Raw 包（连续不同像素）
            int rawStart = runStart;
            while (x < width && x - rawStart < 128) {
                if (x + 1 < width &&
                    memcmp(row + x * bpp,
                           row + (x + 1) * bpp, bpp) == 0) {
                    break; // 开始重复，结束 Raw 包
                }
                x++;
            }
            tjs_uint8 packet = (x - rawStart) - 1;
            dst->WriteBuffer(&packet, 1);
            dst->WriteBuffer(row + rawStart * bpp,
                           (x - rawStart) * bpp);
        }
    }
}
```

## 本节小结

- TGA 头部仅 18 字节，`#pragma pack(1)` 可直接映射为结构体
- RLE 解压核心：高位 bit7 区分 Run-Length 包与 Raw 包，低 7 位为数量
- 像素格式转换需处理三种情况：灰度（1B）→ BGRA、BGR（3B）→ BGRA、BGRA（4B）直拷
- 色键 `keyidx` 需在格式转换后、写入 scanline 前处理
- 保存器利用 TGA 与 KrKr2 同为 BGRA 字节序的特性，32 位模式可零拷贝写出
- 头部解析函数应快速失败（文件太短/类型不支持时直接 return），不抛异常

## 常见错误及解决方案

### 错误 1：结构体对齐导致头部读取错位

**症状**：宽度/高度读出异常大的值（如 65536+ 或负数），图像完全乱码。

**原因**：未使用 `#pragma pack(1)`，编译器在 `colorMapType` 和 `imageType` 之间插入了 1 字节填充。

```cpp
// ❌ 错误：默认对齐
struct tTGAHeader {
    tjs_uint8  idLength;
    tjs_uint8  colorMapType;
    // 编译器可能在此插入 padding！
    tjs_uint8  imageType;
    tjs_uint16 colorMapOrigin; // 偏移变成 4 而非 3
    // ...
};

// ✅ 正确：强制 1 字节对齐
#pragma pack(push, 1)
struct tTGAHeader { /* ... */ };
#pragma pack(pop)
```

### 错误 2：行序翻转遗漏

**症状**：图像上下颠倒。

**原因**：TGA 默认行序是从底到顶（类似 BMP），但未检查 `descriptor` 的 bit5。

```cpp
// ❌ 错误：总是假设从上到下
for (int y = 0; y < h; y++) {
    scanlinecallback(callbackdata, y); // 永远是正序
}

// ✅ 正确：根据 descriptor 决定行序
bool topDown = (header.descriptor & 0x20) != 0;
int destY = topDown ? y : (h - 1 - y);
scanlinecallback(callbackdata, destY);
```

### 错误 3：RLE 跨行解压

**症状**：RLE 压缩的 TGA 图像后半部分出现错位条纹。

**原因**：某些 TGA 编码器的 RLE 包会跨越行边界，但我们的解码器假设每行独立。

```cpp
// ✅ 修复：在 DecompressRLERow 中严格限制 decoded < pixelCount
// 并确保主循环调用位置正确（每行重新开始解压）
for (int y = 0; y < h; y++) {
    DecompressRLERow(src, rowBuf.data(), bpp, w);
    // 即使上一行未消耗完一个 RLE 包，流位置已正确推进
}
```
