# 1.3 线程模型详解

## 本节目标

深入理解 KrKr2 视频播放器的三线程架构。掌握 BasePlayer 主线程、CVideoPlayerVideo 视频解码线程和 CVideoPlayerAudio 音频解码线程的协作模式，理解线程间的消息传递、同步机制和生命周期管理。

---

## 1. 三线程架构总览

KrKr2 视频播放器采用经典的三线程流水线架构，这种架构是多媒体播放器的标准设计模式。三个线程各自承担不同的职责，通过消息队列进行通信，实现解封装与解码的并行处理。这种设计的核心优势在于：解封装（I/O 密集）和解码（CPU 密集）可以同时进行，充分利用多核 CPU 的并行能力。

三线程架构的设计哲学是"关注点分离"——每个线程只关注自己的职责，通过消息队列实现松耦合通信。这使得任何一个线程的修改不会影响到其他线程的逻辑。

### 2.2 线程退出

```cpp
// 源码: movie/ffmpeg/VideoPlayer.cpp
// 主线程的安全退出流程

void BasePlayer::CloseFile() {
    // 1. 设置终止标志
    m_bAbortRequest = true;
    
    // 2. 唤醒可能在等待的消息队列
    m_messenger.Abort();
    
    // 3. 等待主线程退出
    // CThread::StopThread() 内部调用：
    //   Windows: WaitForSingleObject(m_ThreadHandle, INFINITE)
    //   POSIX:   pthread_join(m_thread, NULL)
    StopThread();
    
    // 4. 销毁子播放器
    // 子播放器的析构函数会等待各自的线程退出
    delete m_VideoPlayerVideo;
    m_VideoPlayerVideo = nullptr;
    
    delete m_VideoPlayerAudio;
    m_VideoPlayerAudio = nullptr;
    
    // 5. 关闭解封装器
    delete m_pDemuxer;
    m_pDemuxer = nullptr;
    
    // 6. 关闭输入流
    delete m_pInputStream;
    m_pInputStream = nullptr;
}
```

## 3. 视频解码线程（CVideoPlayerVideo）

CVideoPlayerVideo 是视频解码的工作马——它在独立线程中持续从消息队列中取出视频数据包，解码为 YUV 帧，然后通过 CRenderManager 输出到显示层。这个线程是整个播放器中最计算密集的部分。

视频解码线程的设计核心是一个消息驱动的循环：它不是被动地等待数据，而是主动地从消息队列中拉取数据包进行处理。这种"拉模式"使得流量控制自然地与解码速度匹配。

### 3.1 线程主循环

```cpp
// 源码: movie/ffmpeg/VideoPlayerVideo.cpp
// CVideoPlayerVideo::Process() — 视频解码线程主循环

void CVideoPlayerVideo::Process() {
    while (!m_bAbortRequest) {
        // 从消息队列中获取消息（阻塞等待）
        CDVDMsg* pMsg;
        int result = m_messageQueue.Get(&pMsg, 
            DVD_TIME_BASE / 4);  // 超时: 250ms
        
        if (result == MSGQ_TIMEOUT) {
            // 超时 — 继续等待
            continue;
        }
        
        if (result != MSGQ_OK) {
            // 队列被中止 — 退出线程
            break;
        }
        
        // 根据消息类型处理
        switch (pMsg->GetMessageType()) {
            
        case CDVDMsg::DEMUXER_PACKET: {
            // ── 解码视频帧 ──
            CDVDMsgDemuxerPacket* msgPkt = 
                static_cast<CDVDMsgDemuxerPacket*>(pMsg);
            DemuxPacket* pPacket = msgPkt->GetPacket();
            
            // 发送压缩数据到解码器
            bool ret = m_pVideoCodec->AddData(*pPacket);
            
            // 从解码器获取解码帧
            ProcessDecoderOutput();
            break;
        }
        
        case CDVDMsg::GENERAL_FLUSH: {
            // ── 刷新缓冲区（Seek 时触发） ──
            m_pVideoCodec->Reset();
            m_renderManager->Flush();
            m_stalled = false;
            break;
        }
        
        case CDVDMsg::GENERAL_EOF: {
            // ── 文件结束 ──
            // 从解码器中取出剩余的帧
            ProcessDecoderOutput();
            break;
        }
        
        default:
            break;
        }
        
        pMsg->Release();
    }
}
```

### 3.2 解码输出处理

```cpp
// 源码: movie/ffmpeg/VideoPlayerVideo.cpp
// ProcessDecoderOutput() — 处理解码器输出的帧

void CVideoPlayerVideo::ProcessDecoderOutput() {
    while (true) {
        // 尝试从解码器获取一帧
        DVDVideoPicture picture = {};
        int decodeResult = m_pVideoCodec->GetPicture(&picture);
        
        if (decodeResult == VC_PICTURE) {
            // 成功获得一帧，输出到渲染管理器
            OutputPicture(&picture);
        } 
        else if (decodeResult == VC_BUFFER) {
            // 解码器需要更多输入数据
            break;
        }
        else if (decodeResult == VC_ERROR) {
            // 解码错误，跳过这一帧
            break;
        }
    }
}

// OutputPicture() — 输出解码帧到渲染层
void CVideoPlayerVideo::OutputPicture(DVDVideoPicture* pPicture) {
    double pts = pPicture->pts;
    
    // 1. 计算帧丢弃需求
    int dropReq = CalcDropRequirement();
    if (dropReq > 0 && pts < m_clock.GetClock()) {
        // 这一帧已经迟到了，丢弃它
        m_iDroppedFrames++;
        return;
    }
    
    // 2. 等待渲染缓冲区可用
    // 如果帧缓冲区满，阻塞等待渲染器消费一帧
    double timeout = 0.5;  // 最多等 500ms
    if (!m_renderManager->WaitForBuffer(timeout)) {
        // 超时，可能渲染器卡住了
        return;
    }
    
    // 3. 将帧添加到渲染队列
    m_renderManager->AddVideoPicture(*pPicture);
    
    // 4. 翻转页面（将帧标记为待显示）
    m_renderManager->FlipPage();
}
```

## 4. 音频解码线程（CVideoPlayerAudio）

CVideoPlayerAudio 在独立线程中处理音频数据包的解码和输出。与视频线程类似，它也使用消息驱动的循环模式。音频线程的特殊之处在于它需要维护精确的时间同步——音频通常作为主时钟的参考源。

音频线程的一个关键特点是它的输出是连续的：视频帧可以被丢弃，但音频断续会被人耳立即察觉。因此音频线程的设计更注重连续性和低延迟。

### 4.1 线程主循环

```cpp
// 源码: movie/ffmpeg/VideoPlayerAudio.cpp
// CVideoPlayerAudio::Process() — 音频解码线程主循环

void CVideoPlayerAudio::Process() {
    while (!m_bAbortRequest) {
        CDVDMsg* pMsg;
        int result = m_messageQueue.Get(&pMsg, 
            DVD_TIME_BASE / 4);  // 超时: 250ms
        
        if (result == MSGQ_TIMEOUT)
            continue;
        if (result != MSGQ_OK)
            break;
            
        switch (pMsg->GetMessageType()) {
            
        case CDVDMsg::DEMUXER_PACKET: {
            // ── 解码音频帧 ──
            CDVDMsgDemuxerPacket* msgPkt = 
                static_cast<CDVDMsgDemuxerPacket*>(pMsg);
            DemuxPacket* pPacket = msgPkt->GetPacket();
            
            // 解码并输出 PCM 数据
            OutputPacket(pPacket);
            break;
        }
        
        case CDVDMsg::GENERAL_FLUSH: {
            // ── Seek 刷新 ──
            m_pAudioCodec->Reset();
            m_audioClock = 0;
            m_syncState = IDVDStreamPlayer::SYNC_STARTING;
            break;
        }
        
        case CDVDMsg::GENERAL_EOF: {
            // ── 文件结束 ──
            // 将解码器中剩余的音频数据输出
            DrainDecoder();
            break;
        }
        
        default:
            break;
        }
        
        pMsg->Release();
    }
}
```

### 4.2 音频同步处理

```cpp
// 源码: movie/ffmpeg/VideoPlayerAudio.cpp
// OutputPacket() — 解码音频并处理同步

void CVideoPlayerAudio::OutputPacket(DemuxPacket* pkt) {
    // 1. 发送到解码器
    int consumed = m_pAudioCodec->AddData(*pkt);
    
    // 2. 获取解码后的 PCM 数据
    uint8_t* audioData = nullptr;
    int audioSize = 0;
    double pts = pkt->pts;
    
    while (m_pAudioCodec->GetData(&audioData, &audioSize)) {
        
        // 3. 同步处理
        // 计算当前音频 PTS 与主时钟的偏差
        double clockError = pts - m_clock.GetClock();
        
        if (m_syncType == SYNC_DISCON) {
            // 不连续同步模式：通过调整时钟来纠正偏差
            if (fabs(clockError) > DVD_MSEC_TO_TIME(10)) {
                // 偏差 > 10ms，调整时钟
                m_clock.ErrorAdjust(clockError);
            }
        } else if (m_syncType == SYNC_RESAMPLE) {
            // 重采样同步模式：通过改变音频采样率来纠正
            // 偏差为正 → 减慢音频（降低采样率）
            // 偏差为负 → 加速音频（提高采样率）
            double ratio = 1.0 + clockError * 0.01;
            m_pAudioCodec->SetResampleRatio(ratio);
        }
        
        // 4. 输出到音频设备
        m_pAudioDevice->AddPackets(audioData, audioSize);
        
        // 5. 更新音频时钟
        double duration = (double)audioSize / 
            (m_pAudioCodec->GetSampleRate() * 
             m_pAudioCodec->GetChannels() * 
             sizeof(int16_t));
        m_audioClock = pts + duration;
    }
}
```

## 5. 线程间通信

### 5.1 CDVDMessageQueue

三个线程之间通过 CDVDMessageQueue 进行通信。这是一个线程安全的消息队列，支持优先级排序和超时等待。消息队列是线程间唯一的通信通道——线程之间不直接共享可变状态，所有交互都通过消息传递完成。

```cpp
// 源码: movie/ffmpeg/Message.h + MessageQueue.h
// 消息队列的核心接口

class CDVDMessageQueue {
public:
    // 放入消息（发送端调用）
    // priority: 优先级（值越大优先级越高）
    void Put(CDVDMsg* pMsg, int priority = 0);
    
    // 取出消息（接收端调用）
    // timeout: 最大等待时间（微秒），0 = 非阻塞
    // 返回 MSGQ_OK, MSGQ_TIMEOUT, MSGQ_ABORT
    int Get(CDVDMsg** pMsg, unsigned int timeout);
    
    // 中止队列（唤醒所有等待的 Get 调用）
    void Abort();
    
    // 刷新队列（清空所有消息）
    void Flush();
    
    // 获取当前数据大小
    int GetDataSize() const;
    
private:
    std::list<DVDMessageListItem> m_messages;  // 消息链表
    CCriticalSection m_section;                 // 互斥锁
    CEvent m_hEvent;                            // 条件变量
    bool m_bAborted;                            // 中止标志
};
```

### 5.3 线程安全机制

```cpp
// 源码: movie/ffmpeg/Thread.h
// 线程同步原语

// 临界区（互斥锁）
typedef std::mutex CCriticalSection;

// 自动锁（RAII 风格）
typedef std::unique_lock<std::mutex> CSingleLock;

// 事件（条件变量 + 互斥锁）
class CEvent {
public:
    void Set();                        // 触发事件
    void Reset();                      // 重置事件
    bool WaitMSec(unsigned int ms);    // 等待事件（带超时）
    
private:
    std::condition_variable m_cond;
    std::mutex m_mutex;
    bool m_signaled;
};

// 使用示例：线程安全的消息入队
void CDVDMessageQueue::Put(CDVDMsg* pMsg, int priority) {
    CSingleLock lock(m_section);  // 加锁
    
    // 按优先级插入到合适位置
    auto it = m_messages.begin();
    while (it != m_messages.end() && it->priority >= priority)
        ++it;
    m_messages.insert(it, {pMsg, priority});
    
    // 通知等待的消费者
    m_hEvent.Set();
    
    // lock 析构时自动解锁（RAII）
}
```

## 6. 线程生命周期管理

### 6.1 线程创建

```cpp
// 源码: movie/ffmpeg/Thread.h/cpp
// CThread 线程基类

class CThread {
public:
    void Create();                  // 创建并启动线程
    void StopThread(bool wait = true);  // 停止线程
    bool IsRunning() const;         // 检查线程是否在运行
    
protected:
    virtual void Process() = 0;     // 纯虚函数，子类实现线程逻辑
    bool m_bAbortRequest = false;   // 终止请求标志
    
private:
    std::thread m_thread;           // C++ 标准线程
};

// Create() 的典型实现
void CThread::Create() {
    m_bAbortRequest = false;
    m_thread = std::thread([this]() {
        Process();  // 调用子类重写的 Process()
    });
}

// StopThread() 的安全退出
void CThread::StopThread(bool wait) {
    m_bAbortRequest = true;  // 通知线程退出
    
    if (wait && m_thread.joinable()) {
        m_thread.join();     // 等待线程退出
    }
}
```

### 6.2 优雅关闭

```cpp
// 线程优雅关闭的完整流程
// 确保数据不丢失，资源正确释放

void BasePlayer::GracefulShutdown() {
    // 1. 停止解封装（不再向子播放器发送数据包）
    m_bAbortRequest = true;
    
    // 2. 通知子播放器即将关闭
    // 发送 GENERAL_EOF 让它们处理完缓冲区
    if (m_VideoPlayerVideo) {
        m_VideoPlayerVideo->SendMessage(
            new CDVDMsg(CDVDMsg::GENERAL_EOF));
    }
    if (m_VideoPlayerAudio) {
        m_VideoPlayerAudio->SendMessage(
            new CDVDMsg(CDVDMsg::GENERAL_EOF));
    }
    
    // 3. 中止子播放器的消息队列
    // 唤醒所有阻塞在 Get() 上的线程
    if (m_VideoPlayerVideo)
        m_VideoPlayerVideo->AbortMessageQueue();
    if (m_VideoPlayerAudio)
        m_VideoPlayerAudio->AbortMessageQueue();
    
    // 4. 等待子播放器线程退出
    if (m_VideoPlayerVideo)
        m_VideoPlayerVideo->StopThread();
    if (m_VideoPlayerAudio)
        m_VideoPlayerAudio->StopThread();
}
```

1. KrKr2 视频播放器使用三线程架构：主线程（解封装+控制）、视频解码线程、音频解码线程
2. 线程间通过 CDVDMessageQueue 消息队列通信，避免直接共享可变状态
3. 消息类型包括数据包（DEMUXER_PACKET）、控制命令（FLUSH/EOF/SEEK）等
4. CThread 基类使用 `std::thread` 实现跨平台线程创建，通过 `m_bAbortRequest` 标志控制退出
5. 线程安全通过 `std::mutex`（CCriticalSection）和 `std::condition_variable`（CEvent）实现
6. 优雅关闭流程确保子播放器处理完缓冲区后再退出，避免数据丢失

## 常见错误及解决方案

### 错误 1：线程退出时死锁

```
程序卡住在 StopThread()，无法退出
```

**原因**：子播放器线程阻塞在 `m_messageQueue.Get()` 上，`m_bAbortRequest = true` 无法唤醒它。

**解决方案**：
1. 在调用 `StopThread()` 之前，先调用 `m_messageQueue.Abort()` 唤醒等待的线程
2. 确保消息队列的 `Abort()` 方法正确设置了 `m_bAborted` 标志并通知条件变量

### 错误 2：消息队列内存泄漏

```
Valgrind: definitely lost: CDVDMsgDemuxerPacket
```

**原因**：消息队列中的 CDVDMsg 对象使用引用计数管理生命周期，`Flush()` 时未正确释放。

**解决方案**：
1. 确保 `Flush()` 方法对所有消息调用 `pMsg->Release()`
2. 确保 `Get()` 返回的消息在处理完毕后调用 `pMsg->Release()`

### 错误 3：音视频解码线程CPU占用过高

```
CVideoPlayerVideo 线程 CPU 使用率 100%
```

**原因**：消息队列 `Get()` 的超时值设置为 0（非阻塞），线程在空转。

**解决方案**：
1. 确保 `Get()` 的超时值大于 0（如 `DVD_TIME_BASE / 4` = 250ms）
2. 当没有数据时线程应该挂起等待，而不是轮询

## 练习题与答案

### 练习 1：为什么使用消息队列而不是直接函数调用？

<details>
<summary>查看答案</summary>

1. **线程安全**：消息队列内部使用互斥锁保护，避免了直接跨线程调用导致的竞态条件
2. **解耦**：发送方和接收方不需要知道对方的实现细节，只需遵循消息协议
3. **异步处理**：发送方不需要等待接收方处理完成（非阻塞 Put），实现真正的并行
4. **流量控制**：消息队列可以设置容量上限，当队列满时发送方自然减速
5. **优先级支持**：控制消息（Seek/Flush）可以设置高优先级，优先于数据包被处理

</details>

### 练习 2：分析以下线程退出顺序的问题

```cpp
// 错误的退出顺序
delete m_VideoPlayerVideo;  // Thread #2 销毁
delete m_VideoPlayerAudio;  // Thread #3 销毁
StopThread();               // Thread #1 停止
```

<details>
<summary>查看答案</summary>

这个退出顺序有严重问题：

1. **Thread #1 还在运行**：BasePlayer 主线程的 `Process()` 可能正在通过 `m_VideoPlayerVideo->SendMessage()` 发送消息
2. **悬垂指针**：删除子播放器后，主线程中的指针成为悬垂指针，访问会导致段错误/崩溃
3. **正确顺序应该是**：
   - 先停止主线程（`StopThread()`），确保不再向子播放器发送消息
   - 然后停止子播放器线程（`AbortMessageQueue()` + `StopThread()`）
   - 最后删除子播放器对象

</details>

### 练习 3：如果视频解码速度跟不上播放速度，系统会如何应对？

<details>
<summary>查看答案</summary>

系统有多层应对机制：

1. **帧丢弃（CVideoPlayerVideo）**：`CalcDropRequirement()` 检测到延迟帧过多时，跳过解码部分帧
2. **解码器跳过（CDVDVideoCodecFFmpeg）**：设置 `AVCodecContext->skip_frame` 跳过非关键帧的解码
3. **渲染器跳过（CRenderManager）**：`PrepareNextRender()` 跳过已经过期的帧，直接显示最新的帧
4. **流量控制（BasePlayer）**：如果视频队列已满但消费很慢，BasePlayer 会暂停读取新数据包，给解码器更多处理时间

这些机制协同工作，确保即使在 CPU 负载高的情况下也能维持播放的流畅性（可能降低画质但不卡顿）。

</details>
