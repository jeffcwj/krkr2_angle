## 解码器架构概览

KrKr2 的解码器子系统沿袭自 XBMC/Kodi，采用**抽象基类 + 工厂模式**的经典设计。视频和音频各有一个抽象基类（`CDVDVideoCodec` / `CDVDAudioCodec`），FFmpeg 实现是默认的软件解码后端，同时预留了硬件解码接口。

### 类继承关系图

```
CDVDVideoCodec (抽象基类，定义 Open/Decode/GetPicture 等纯虚函数)
├── CDVDVideoCodecFFmpeg (FFmpeg 软件解码，项目主力解码器)
│   └── IHardwareDecoder (内嵌硬件解码接口，支持 VDPAU/DXVA/VAAPI/VTB/MMAL)
├── CDVDVideoCodecAndroidMediaCodec (Android 硬件解码，条件编译)
├── CDVDVideoCodecAmlogic (Amlogic 芯片硬件解码，条件编译)
└── CDVDVideoCodecOpenMax (OpenMax IL 解码，条件编译)

CDVDAudioCodec (抽象基类，定义 Open/Decode/GetData 等纯虚函数)
├── CDVDAudioCodecFFmpeg (FFmpeg 软件解码，默认音频解码器)
└── CDVDAudioCodecPassthrough (音频直通，透传比特流到外部解码器)
```

工厂类 `CDVDFactoryCodec` 负责根据 `CDVDStreamInfo` 中的编解码器 ID 和平台特性，自动选择并创建合适的解码器实例。

## CDVDVideoCodec 抽象基类

`CDVDVideoCodec` 定义在 `VideoCodec.h` 中（343 行），是所有视频解码器的基类。它规定了一组纯虚函数，任何视频解码器实现都必须遵守这套接口契约。

### 核心接口方法

```cpp
// 源文件: cpp/core/movie/ffmpeg/VideoCodec.h 第 191-342 行
class CDVDVideoCodec {
public:
    CDVDVideoCodec(CProcessInfo &processInfo)
        : m_processInfo(processInfo) {}

    virtual ~CDVDVideoCodec() = default;

    // 打开解码器，返回 true 表示成功
    // hints: 流信息（编解码器 ID、宽高、extradata 等）
    // options: 附加选项（渲染格式列表、surfaces 数量等）
    virtual bool Open(CDVDStreamInfo &hints,
                      CDVDCodecOptions &options) = 0;

    // 解码一帧数据
    // pData: 压缩数据指针，可为 nullptr（表示 flush）
    // iSize: 数据大小
    // dts/pts: 解码/显示时间戳（DVD_TIME_BASE 单位）
    // 返回: VC_* 标志位组合
    virtual int Decode(uint8_t *pData, int iSize,
                       double dts, double pts) = 0;

    // 重置解码器状态（seek 时调用）
    virtual void Reset() = 0;

    // 获取解码后的图片数据
    virtual bool GetPicture(DVDVideoPicture *pDvdVideoPicture) = 0;

    // 通知解码器当前帧是否会被丢弃
    // bDrop=true 时解码器可跳过 IDC/环路滤波以节省 CPU
    virtual void SetDropState(bool bDrop) = 0;

    // 返回解码器名称（如 "ff-h264"）
    virtual const char *GetName() = 0;

    // 获取解码统计信息（用于帧丢弃决策）
    virtual bool GetCodecStats(double &pts,
                               int &droppedFrames,
                               int &skippedPics) {
        droppedFrames = -1;
        skippedPics = -1;
        return false;
    }

    // 设置解码器控制标志
    // DVD_CODEC_CTRL_HURRY: 催促解码器尽快输出
    // DVD_CODEC_CTRL_DROP: 当前帧将被丢弃
    // DVD_CODEC_CTRL_DRAIN: 排空解码器缓冲
    virtual void SetCodecControl(int flags) {}

protected:
    CProcessInfo &m_processInfo;  // 进程信息，用于记录解码器状态
};
```

### Decode() 返回值标志位

`Decode()` 方法的返回值是一组可组合的标志位，定义在 `VideoCodec.h` 第 171-189 行：

| 标志位 | 值 | 含义 |
|--------|-----|------|
| `VC_ERROR` | 0x01 | 解码出错，不会返回其他消息 |
| `VC_BUFFER` | 0x02 | 解码器需要更多数据 |
| `VC_PICTURE` | 0x04 | 解码器已产出一帧图片 |
| `VC_USERDATA` | 0x08 | 发现用户数据（如隐藏字幕） |
| `VC_FLUSHED` | 0x10 | 解码器状态已丢失，需重新启动 |
| `VC_DROPPED` | 0x20 | 帧已被解码器丢弃 |
| `VC_NOBUFFER` | 0x40 | FFmpeg 的 GetBuffer 失败 |
| `VC_REOPEN` | 0x80 | 解码器请求重新打开 |

这些标志位可以组合使用。例如 `VC_PICTURE | VC_BUFFER` 表示"已产出一帧图片，同时还需要更多数据"。

```cpp
// 典型使用模式（CVideoPlayerVideo::Decode 内部）
int iDecoderState = m_pVideoCodec->Decode(pData, iSize, dts, pts);

if (iDecoderState & VC_ERROR) {
    // 解码出错，处理错误
}
if (iDecoderState & VC_BUFFER) {
    // 解码器需要更多数据，继续读取
}
if (iDecoderState & VC_PICTURE) {
    // 有解码后的图片可用
    DVDVideoPicture picture;
    m_pVideoCodec->GetPicture(&picture);
    OutputPicture(&picture);
}
if (iDecoderState & VC_REOPEN) {
    // 解码器要求重新打开（硬件解码失败回退到软解）
    m_pVideoCodec->Reopen();
}
```

### DVDVideoPicture 输出结构

解码器通过 `GetPicture()` 填充 `DVDVideoPicture` 结构体，这是视频帧在解码器和渲染器之间传递的核心数据容器：

```cpp
// 源文件: cpp/core/movie/ffmpeg/VideoCodec.h 第 50-123 行
struct DVDVideoPicture {
    double pts;  // 显示时间戳（秒），用于 CVideoPlayer 追踪 pts
    double dts;  // 解码时间戳

    // 使用 union 支持多种像素数据来源
    union {
        struct {
            uint8_t *data[4];      // YUV 平面数据指针
            int iLineSize[4];       // 每个平面的行字节数（stride）
        };
        struct { DXVA::CRenderPicture *dxva; };       // DXVA 硬件解码
        struct { VDPAU::CVdpauRenderPicture *vdpau; }; // VDPAU 硬件解码
        struct { VAAPI::CVaapiRenderPicture *vaapi; }; // VAAPI 硬件解码
        struct { struct __CVBuffer *cvBufferRef; };     // VideoToolbox
        struct { CDVDMediaCodecInfo *mediacodec; };     // Android MediaCodec
        // ... 其他硬件解码后端
    };

    unsigned int iFlags;          // DVP_FLAG_* 标志位组合
    double iRepeatPicture;        // 重复显示次数（隔行视频）
    double iDuration;             // 帧持续时间
    unsigned int iFrameType : 4;  // 帧类型: I=1, P=2, B=3

    // 色彩空间信息
    unsigned int color_matrix : 4;    // 颜色矩阵（BT.601/709等）
    unsigned int color_range : 1;     // 1=全范围色彩
    unsigned int color_primaries;     // 色域原色
    unsigned int color_transfer;      // 传输特性

    // 量化参数表（后处理滤镜使用）
    int8_t *qp_table;
    int qstride;
    int qscale_type;              // MPEG1/MPEG2/H264

    // 尺寸信息
    unsigned int iWidth;          // 图片宽度
    unsigned int iHeight;         // 图片高度
    unsigned int iDisplayWidth;   // 无黑边的显示宽度
    unsigned int iDisplayHeight;  // 无黑边的显示高度

    ERenderFormat format;         // 渲染格式（YUV420P 等）
};
```

`DVDVideoPicture` 的 `union` 设计是 Kodi 遗产的精华之一：同一块内存可以存储软件解码的 YUV 数据指针，也可以存储硬件解码的不透明句柄，通过 `format` 字段来区分当前帧的实际数据类型。

## CDVDVideoCodecFFmpeg 详解

`CDVDVideoCodecFFmpeg` 是项目中实际使用的视频解码器实现，封装了 FFmpeg 的 `avcodec_decode_video2()` API。它支持软件解码和通过 `IHardwareDecoder` 接口进行硬件加速解码。

### Open() 初始化流程

`Open()` 方法（`VideoCodecFFmpeg.cpp` 第 266-435 行）执行以下步骤：

```cpp
// 简化后的初始化流程
bool CDVDVideoCodecFFmpeg::Open(CDVDStreamInfo &hints,
                                CDVDCodecOptions &options) {
    m_hints = hints;     // 保存流信息以供 Reopen() 使用
    m_options = options;

    // 1. 构建支持的像素格式列表
    m_formats.clear();
    for (auto it = options.m_formats.begin();
         it != options.m_formats.end(); ++it) {
        AVPixelFormat pixFormat =
            CDVDCodecUtils::PixfmtFromEFormat(*it);
        if (pixFormat != AV_PIX_FMT_NONE)
            m_formats.push_back(pixFormat);
        // YUV420P 额外添加 YUVJ420P（全范围 YUV）
        if (*it == RENDER_FMT_YUV420P)
            m_formats.push_back(AV_PIX_FMT_YUVJ420P);
    }
    m_formats.push_back(AV_PIX_FMT_NONE); // 终止符

    // 2. 查找 FFmpeg 解码器
    AVCodec *pCodec = avcodec_find_decoder(hints.codec);
    if (pCodec == nullptr) return false;

    // 3. 分配并配置 AVCodecContext
    m_pCodecContext = avcodec_alloc_context3(pCodec);
    m_pCodecContext->opaque = (void *)this;  // 回调中使用
    m_pCodecContext->get_format = GetFormat;  // 像素格式协商回调
    m_pCodecContext->codec_tag = hints.codec_tag;

    // 4. 设置线程模型
    if (!hints.software) {
        // 尝试硬件解码（VDPAU/DXVA/VAAPI/VTB/MMAL）
        bool tryhw = false;
        #ifdef HAVE_LIBVDPAU
        tryhw = true;
        #endif
        // ... 其他硬件平台检测

        if (tryhw) {
            m_decoderState = STATE_HW_SINGLE; // 单线程硬件解码
        } else {
            // 多线程软件解码
            int num_threads = TVPGetProcessorNum() * 3 / 2;
            num_threads = std::max(1, std::min(num_threads, 16));
            m_pCodecContext->thread_count = num_threads;
            m_decoderState = STATE_SW_MULTI;
        }
    }

    // 5. 设置基本参数
    m_pCodecContext->coded_height = hints.height;
    m_pCodecContext->coded_width = hints.width;
    m_pCodecContext->bits_per_coded_sample = hints.bitsperpixel;

    // 6. 拷贝 extradata（SPS/PPS 等编码参数）
    if (hints.extradata && hints.extrasize > 0) {
        m_pCodecContext->extradata_size = hints.extrasize;
        m_pCodecContext->extradata = (uint8_t *)av_mallocz(
            hints.extrasize + FF_INPUT_BUFFER_PADDING_SIZE);
        memcpy(m_pCodecContext->extradata,
               hints.extradata, hints.extrasize);
    }

    // 7. 打开解码器
    while (avcodec_open2(m_pCodecContext, pCodec, nullptr) < 0) {
        // 降低分辨率重试
        if (pCodec->max_lowres < m_pCodecContext->lowres) {
            m_pCodecContext->lowres = pCodec->max_lowres;
            if (avcodec_open2(m_pCodecContext, pCodec, nullptr) >= 0)
                break;
        }
        avcodec_free_context(&m_pCodecContext);
        return false;
    }

    // 8. 分配 AVFrame 缓冲
    m_pFrame = av_frame_alloc();         // 最终输出帧
    m_pDecodedFrame = av_frame_alloc();  // 解码中间帧
    m_pFilterFrame = av_frame_alloc();   // 滤镜输出帧

    return true;
}
```

### 线程模型决策

解码器的线程数量决策是一个重要的性能考量点：

```cpp
// 源文件: VideoCodecFFmpeg.cpp 第 311-348 行
// 线程数 = CPU 核心数 * 1.5，最少 1 个，最多 16 个
int num_threads = TVPGetProcessorNum() * 3 / 2;
num_threads = std::max(1, std::min(num_threads, 16));
m_pCodecContext->thread_count = num_threads;
m_pCodecContext->thread_safe_callbacks = 1;
```

| 平台 | 典型 CPU 核心数 | 解码线程数 | 说明 |
|------|----------------|-----------|------|
| Windows x64 | 8 核 | 12 线程 | 高性能桌面 |
| macOS arm64 | 8-10 核 | 12-15 线程 | Apple Silicon |
| Linux x64 | 4-8 核 | 6-12 线程 | 服务器/桌面 |
| Android arm64 | 4-8 核 | 6-12 线程 | 移动设备 |

注意：当启用硬件解码时（`STATE_HW_SINGLE`），FFmpeg 使用单线程模式，因为硬件解码器本身是并行的。

### Decode() 解码流程

`Decode()` 方法（第 555-726 行）是解码的核心入口：

```cpp
int CDVDVideoCodecFFmpeg::Decode(uint8_t *pData, int iSize,
                                 double dts, double pts) {
    int iGotPicture = 0, len = 0;

    if (!m_pCodecContext) return VC_ERROR;
    if (pData) m_iLastKeyframe++;

    // 1. 硬件解码器优先处理
    if (m_pHardware) {
        int result;
        if (pData || (m_codecControlFlags & DVD_CODEC_CTRL_DRAIN)) {
            result = m_pHardware->Check(m_pCodecContext);
            result &= ~VC_NOBUFFER;
        } else {
            result = m_pHardware->Decode(m_pCodecContext, nullptr);
        }
        if (result) return result;
    }

    // 2. 构造 AVPacket
    m_dts = dts;
    m_pCodecContext->reordered_opaque = pts_dtoi(pts);

    AVPacket avpkt;
    av_init_packet(&avpkt);
    avpkt.data = pData;
    avpkt.size = iSize;
    // 时间戳从 DVD_TIME_BASE 转为 AV_TIME_BASE
    avpkt.dts = (dts == DVD_NOPTS_VALUE)
        ? AV_NOPTS_VALUE
        : dts / DVD_TIME_BASE * AV_TIME_BASE;
    avpkt.pts = (pts == DVD_NOPTS_VALUE)
        ? AV_NOPTS_VALUE
        : pts / DVD_TIME_BASE * AV_TIME_BASE;
    avpkt.flags = AV_PKT_FLAG_KEY; // 始终标记为关键帧

    // 3. 调用 FFmpeg 解码
    len = avcodec_decode_video2(m_pCodecContext,
                                m_pDecodedFrame,
                                &iGotPicture, &avpkt);

    // 4. 错误处理
    if (m_decoderState == STATE_HW_FAILED && !m_pHardware)
        return VC_REOPEN;  // 硬件解码失败，请求回退到软解

    if (len < 0) return VC_ERROR;
    if (!iGotPicture) return VC_BUFFER; // 需要更多数据

    // 5. 帧丢弃检测
    int64_t framePTS =
        av_frame_get_best_effort_timestamp(m_pDecodedFrame);
    m_dropCtrl.Process(framePTS,
        m_pCodecContext->skip_frame > AVDISCARD_DEFAULT);

    // 6. 关键帧检测
    if (m_pDecodedFrame->key_frame) {
        m_started = true;
        m_iLastKeyframe = m_pCodecContext->has_b_frames + 2;
    }
    if (!m_started) {
        av_frame_unref(m_pDecodedFrame);
        return VC_BUFFER; // 还没遇到关键帧，跳过
    }

    // 7. 滤镜处理或直接输出
    int result;
    if (m_pHardware) {
        av_frame_move_ref(m_pFrame, m_pDecodedFrame);
        result = m_pHardware->Decode(m_pCodecContext, m_pFrame);
    } else if (m_pFilterGraph && !m_filterEof) {
        result = FilterProcess(m_pDecodedFrame);
    } else {
        av_frame_move_ref(m_pFrame, m_pDecodedFrame);
        result = VC_PICTURE | VC_BUFFER;
    }

    return result;
}
```

### GetPicture() 输出图片

`GetPicture()` 方法（第 880-915 行）将 `m_pFrame` 的数据填充到 `DVDVideoPicture` 中：

```cpp
bool CDVDVideoCodecFFmpeg::GetPicture(
    DVDVideoPicture *pDvdVideoPicture) {
    // 硬件解码走单独路径
    if (m_pHardware)
        return m_pHardware->GetPicture(
            m_pCodecContext, m_pFrame, pDvdVideoPicture);

    // 确定像素格式
    AVPixelFormat pix_fmt = (AVPixelFormat)m_pFrame->format;
    pDvdVideoPicture->format =
        CDVDCodecUtils::EFormatFromPixfmt(pix_fmt);

    // 填充公共字段（宽高、DAR、时间戳、色彩空间等）
    if (!GetPictureCommon(pDvdVideoPicture))
        return false;

    // 拷贝 YUV 数据指针
    for (int i = 0; i < 4; i++)
        pDvdVideoPicture->data[i] = m_pFrame->data[i];
    for (int i = 0; i < 4; i++)
        pDvdVideoPicture->iLineSize[i] = m_pFrame->linesize[i];

    return true;
}
```

注意这里**没有拷贝像素数据本身**，只拷贝了指针。这意味着 `DVDVideoPicture` 中的数据只在下一次 `Decode()` 调用之前有效。上层播放线程必须在此期间完成帧的消费（上传到纹理或拷贝到帧缓冲）。

### SetDropState() 帧丢弃优化

当播放器判定某帧将被丢弃时，会调用 `SetDropState(true)` 让解码器跳过不必要的计算：

```cpp
// 源文件: VideoCodecFFmpeg.cpp 第 447-471 行
void CDVDVideoCodecFFmpeg::SetDropState(bool bDrop) {
    if (m_pCodecContext) {
        if (bDrop && m_pHardware && m_pHardware->CanSkipDeint()) {
            // 硬件解码器支持跳过去隔行处理
            m_requestSkipDeint = true;
            bDrop = false;
        } else {
            m_requestSkipDeint = false;
        }

        if (bDrop) {
            // 跳过非参考帧的解码（节省大量 CPU）
            m_pCodecContext->skip_frame = AVDISCARD_NONREF;
            m_pCodecContext->skip_idct = AVDISCARD_NONREF;
            m_pCodecContext->skip_loop_filter = AVDISCARD_NONREF;
        } else {
            // 恢复正常解码
            m_pCodecContext->skip_frame = AVDISCARD_DEFAULT;
            m_pCodecContext->skip_idct = AVDISCARD_DEFAULT;
            m_pCodecContext->skip_loop_filter = AVDISCARD_DEFAULT;
        }
    }
}
```

FFmpeg 的 `AVDISCARD_NONREF` 会跳过所有非参考帧（B 帧），只解码 I 帧和 P 帧，大幅降低 CPU 负载。这是在视频播放落后于音频时的重要补救措施。

### CDropControl 帧丢弃检测器

```cpp
void CDVDVideoCodecFFmpeg::CDropControl::Process(
    int64_t pts, bool drop) {
    if (m_state == INIT) {
        // 收集前 10 帧的 PTS 差值来计算平均帧间隔
        if (pts != AV_NOPTS_VALUE &&
            m_lastPTS != AV_NOPTS_VALUE) {
            m_diffPTS += pts - m_lastPTS;
            m_count++;
        }
        if (m_count > 10) {
            m_diffPTS = m_diffPTS / m_count;
            if (m_diffPTS > 0) {
                m_state = VALID;
                m_count = 0;
            }
        }
    } else if (m_state == VALID && !drop) {
        // PTS 跳跃超过 20%，说明有帧被丢弃
        if (std::abs(pts - m_lastPTS - m_diffPTS) >
            m_diffPTS * 0.2) {
            m_count++;
            if (m_count > 5) {
                // 连续 5 次异常，重置状态机
                Reset(true);
            }
        } else {
            m_count = 0;
        }
    }
    m_lastPTS = pts;
}
```

状态机分两个阶段：
1. **INIT 阶段**：收集前 10 帧的 PTS 差值，计算平均帧间隔
2. **VALID 阶段**：检测实际 PTS 差值与平均值的偏差，超过 20% 则判定为丢帧

`GetFormat()` 是一个静态回调函数，由 FFmpeg 在解码器初始化时调用，用于协商输出像素格式。这也是硬件解码器注入的关键入口点：

```cpp
// 源文件: VideoCodecFFmpeg.cpp 第 104-230 行
enum AVPixelFormat CDVDVideoCodecFFmpeg::GetFormat(
    struct AVCodecContext *avctx,
    const AVPixelFormat *fmt) {
    CDVDVideoCodecFFmpeg *ctx =
        (CDVDVideoCodecFFmpeg *)avctx->opaque;

    // 非硬件解码模式，使用默认格式
    if (ctx->m_decoderState != STATE_HW_SINGLE) {
        return avcodec_default_get_format(avctx, fmt);
    }

    // 遍历 FFmpeg 支持的格式列表，尝试创建硬件解码器
    const AVPixelFormat *cur = fmt;
    while (*cur != AV_PIX_FMT_NONE) {
        #ifdef HAVE_LIBVDPAU
        if (VDPAU::CDecoder::IsVDPAUFormat(*cur)) {
            VDPAU::CDecoder *vdp = new VDPAU::CDecoder(...);
            if (vdp->Open(avctx, ...)) {
                ctx->SetHardware(vdp);
                return *cur;
            }
        }
        #endif
        // DXVA, VAAPI, VTB, MMAL 类似...
        cur++;
    }

    // 所有硬件解码器都失败，回退到软件解码
    ctx->m_decoderState = STATE_HW_FAILED;
    return avcodec_default_get_format(avctx, fmt);
}
```

```cpp
// 源文件: AudioCodecFFmpeg.cpp 第 37-113 行
bool CDVDAudioCodecFFmpeg::Open(CDVDStreamInfo &hints,
                                CDVDCodecOptions &options) {
    AVCodec *pCodec = nullptr;

    // DTS 音频特殊处理：优先使用 dcadec 解码器
    if (hints.codec == AV_CODEC_ID_DTS && allowdtshddecode)
        pCodec = avcodec_find_decoder_by_name("dcadec");

    // 通用查找
    if (!pCodec)
        pCodec = avcodec_find_decoder(hints.codec);
    if (!pCodec) return false;

    // 分配 AVCodecContext
    m_pCodecContext = avcodec_alloc_context3(pCodec);

    // 配置音频参数
    m_pCodecContext->channels = hints.channels;
    m_pCodecContext->sample_rate = hints.samplerate;
    m_pCodecContext->block_align = hints.blockalign;
    m_pCodecContext->bit_rate = hints.bitrate;
    m_pCodecContext->bits_per_coded_sample = hints.bitspersample;

    // 默认 16 位
    if (m_pCodecContext->bits_per_coded_sample == 0)
        m_pCodecContext->bits_per_coded_sample = 16;

    // 拷贝 extradata
    if (hints.extradata && hints.extrasize > 0) {
        m_pCodecContext->extradata = (uint8_t *)av_mallocz(
            hints.extrasize + FF_INPUT_BUFFER_PADDING_SIZE);
        m_pCodecContext->extradata_size = hints.extrasize;
        memcpy(m_pCodecContext->extradata,
               hints.extradata, hints.extrasize);
    }

    // 打开解码器
    if (avcodec_open2(m_pCodecContext, pCodec, nullptr) < 0) {
        Dispose();
        return false;
    }

    m_pFrame1 = av_frame_alloc();
    return true;
}
```

### CDVDAudioCodecFFmpeg::Decode() 解码

```cpp
// 源文件: AudioCodecFFmpeg.cpp 第 121-169 行
int CDVDAudioCodecFFmpeg::Decode(uint8_t *pData, int iSize,
                                 double dts, double pts) {
    int iBytesUsed;
    if (!m_pCodecContext) return -1;

    // 构造 AVPacket
    AVPacket avpkt;
    av_init_packet(&avpkt);
    avpkt.data = pData;
    avpkt.size = iSize;
    avpkt.dts = (dts == DVD_NOPTS_VALUE)
        ? AV_NOPTS_VALUE
        : dts / DVD_TIME_BASE * AV_TIME_BASE;
    avpkt.pts = (pts == DVD_NOPTS_VALUE)
        ? AV_NOPTS_VALUE
        : pts / DVD_TIME_BASE * AV_TIME_BASE;

    // 调用 FFmpeg 解码
    iBytesUsed = avcodec_decode_audio4(
        m_pCodecContext, m_pFrame1, &m_gotFrame, &avpkt);

    if (iBytesUsed < 0 || !m_gotFrame) {
        return iBytesUsed;
    }

    // 修复某些编解码器的 channels 问题
    if (m_pFrame1->channels == 0)
        m_pFrame1->channels = av_frame_get_channels(m_pFrame1);

    // 防止消耗超过输入大小的数据
    if (iBytesUsed > iSize) iBytesUsed = iSize;

    // 提取矩阵编码信息（环绕声）
    if (m_pFrame1->nb_side_data) {
        for (int i = 0; i < m_pFrame1->nb_side_data; i++) {
            AVFrameSideData *sd = m_pFrame1->side_data[i];
            if (sd->type == AV_FRAME_DATA_MATRIXENCODING) {
                m_matrixEncoding =
                    *(enum AVMatrixEncoding *)sd->data;
            }
        }
    }

    // 更新输出格式
    m_format.m_dataFormat = GetDataFormat();
    m_format.m_channelLayout = GetChannelMap();
    m_format.m_sampleRate = GetSampleRate();

    return iBytesUsed;
}
```

```cpp
    if (m_channels == m_pCodecContext->channels &&
        m_layout == m_pCodecContext->channel_layout)
        return;  // 无变化则跳过

    m_channels = m_pCodecContext->channels;
    m_layout = m_pCodecContext->channel_layout;
```

## AVFilter 视频滤镜管线

`CDVDVideoCodecFFmpeg` 还内置了 FFmpeg 的 libavfilter 支持，用于去隔行、旋转等后处理操作。

### FilterOpen() 初始化滤镜图

```cpp
// 源文件: VideoCodecFFmpeg.cpp 第 917-1025 行
int CDVDVideoCodecFFmpeg::FilterOpen(
    const std::string &filters, bool scale) {
    if (m_pFilterGraph) FilterClose();
    if (filters.empty() && !scale) return 0;
    if (m_pHardware) return 0; // 硬件解码不用软件滤镜

    // 创建滤镜图
    m_pFilterGraph = avfilter_graph_alloc();

    // 创建 buffer 源滤镜（输入端）
    AVFilter *srcFilter = avfilter_get_by_name("buffer");
    char args[128];
    sprintf(args, "%d:%d:%d:%d:%d:%d:%d",
            m_pCodecContext->width,
            m_pCodecContext->height,
            m_pCodecContext->pix_fmt,
            /* time_base, sample_aspect_ratio */);
    avfilter_graph_create_filter(
        &m_pFilterIn, srcFilter, "src", args, nullptr,
        m_pFilterGraph);

    // 创建 buffersink 汇滤镜（输出端）
    AVFilter *outFilter = avfilter_get_by_name("buffersink");
    avfilter_graph_create_filter(
        &m_pFilterOut, outFilter, "out", nullptr, nullptr,
        m_pFilterGraph);

    // 设置输出像素格式白名单
    av_opt_set_int_list(m_pFilterOut, "pix_fmts",
                        &m_formats[0], AV_PIX_FMT_NONE,
                        AV_OPT_SEARCH_CHILDREN);

    // 解析滤镜字符串（如 "yadif=1:-1" 或 "transpose=1"）
    if (!filters.empty()) {
        avfilter_graph_parse_ptr(m_pFilterGraph, filters.c_str(),
                                  &inputs, &outputs, nullptr);
    } else {
        // 仅格式转换，直接连接输入输出
        avfilter_link(m_pFilterIn, 0, m_pFilterOut, 0);
    }

    // 配置滤镜图
    avfilter_graph_config(m_pFilterGraph, nullptr);
    return result;
}
```

### 支持的滤镜操作

| 滤镜名 | 用途 | 参数示例 |
|--------|------|---------|
| `yadif` | 去隔行 | `yadif=0:-1`（半速率）/ `yadif=1:-1`（全速率） |
| `transpose` | 旋转 | `transpose=1`（90°）/ `transpose=2`（270°） |
| `vflip,hflip` | 180° 旋转 | 垂直+水平翻转 |

```cpp
        AVCodec *pCodec = avcodec_find_decoder(codecId);
        if (!pCodec) {
            printf("错误: 找不到解码器 ID=%d\n", codecId);
            return false;
        }

        m_pCodecContext = avcodec_alloc_context3(pCodec);
        m_pCodecContext->coded_width = width;
        m_pCodecContext->coded_height = height;

        // 拷贝 extradata（SPS/PPS 等）
        if (extradata && extrasize > 0) {
            m_pCodecContext->extradata_size = extrasize;
            m_pCodecContext->extradata = (uint8_t *)av_mallocz(
                extrasize + AV_INPUT_BUFFER_PADDING_SIZE);
            memcpy(m_pCodecContext->extradata,
                   extradata, extrasize);
        }

        // 设置多线程
        m_pCodecContext->thread_count = 4;

        if (avcodec_open2(m_pCodecContext, pCodec, nullptr) < 0) {
            printf("错误: 无法打开解码器\n");
            avcodec_free_context(&m_pCodecContext);
            return false;
        }

        m_pFrame = av_frame_alloc();
        printf("解码器已打开: %s (%dx%d)\n",
               pCodec->name, width, height);
        return true;
    }

    // 类似 CDVDVideoCodecFFmpeg::Decode()
    int Decode(uint8_t *pData, int iSize,
               double dts, double pts) {
        if (!m_pCodecContext) return -1;

        AVPacket avpkt;
        av_init_packet(&avpkt);
        avpkt.data = pData;
        avpkt.size = iSize;

        int ret = avcodec_send_packet(m_pCodecContext, &avpkt);
        if (ret < 0) return -1;

        ret = avcodec_receive_frame(m_pCodecContext, m_pFrame);
        if (ret == AVERROR(EAGAIN)) return 0;  // 需要更多数据
        if (ret < 0) return -1;

        return 1;  // 有帧输出
    }

    // 类似 CDVDVideoCodecFFmpeg::GetPicture()
    bool GetPicture(SimpleVideoPicture *pic) {
        if (!m_pFrame || !m_pFrame->data[0]) return false;

        for (int i = 0; i < 4; i++) {
            pic->data[i] = m_pFrame->data[i];
            pic->linesize[i] = m_pFrame->linesize[i];
        }
        pic->width = m_pFrame->width;
        pic->height = m_pFrame->height;

        int64_t bpts = m_pFrame->best_effort_timestamp;
        if (bpts != AV_NOPTS_VALUE)
            pic->pts = (double)bpts * DVD_TIME_BASE
                       / AV_TIME_BASE;
        else
            pic->pts = (double)DVD_NOPTS_VALUE;

        return true;
    }

    void Close() {
        av_frame_free(&m_pFrame);
        avcodec_free_context(&m_pCodecContext);
    }

private:
    AVCodecContext *m_pCodecContext;
    AVFrame *m_pFrame;
};

int main() {
    // 演示解码器创建流程
    SimpleVideoDecoder decoder;

    // 模拟 H.264 解码器打开
    if (decoder.Open(AV_CODEC_ID_H264, 1280, 720,
                     nullptr, 0)) {
        printf("解码器创建成功！\n");
        printf("这模拟了 CDVDFactoryCodec::CreateVideoCodec()"
               " 的工作流程\n");
```

## 常见错误及解决方案

### 错误 1：解码器打开失败

```
症状: CDVDVideoCodecFFmpeg::Open() 返回 false
原因: avcodec_find_decoder() 返回 nullptr，或 avcodec_open2() 返回负值
```

**排查步骤：**
1. 检查 `hints.codec` 是否为有效的 `AVCodecID`（如 `AV_CODEC_ID_H264`）
2. 确认 FFmpeg 编译时包含了对应的解码器（`./configure --list-decoders | grep h264`）
3. 检查 `extradata` 是否正确（H.264 需要 SPS/PPS，HEVC 需要 VPS/SPS/PPS）

### 错误 2：解码卡在 VC_BUFFER 不输出画面

```
症状: Decode() 持续返回 VC_BUFFER，但 iGotPicture 始终为 0
原因: 解码器缓冲了 B 帧，等待更多参考帧才能输出
```

**解决方案：**
- 继续喂入数据，直到解码器积累足够的参考帧
- 对于 H.264 流，通常需要发送 `has_b_frames + 1` 个包才能得到第一帧输出
- 如果在 seek 后出现，确保先 Reset() 再从关键帧开始喂数据

### 错误 3：硬件解码回退到软解后画面异常

```
症状: 画面花屏或绿屏
原因: STATE_HW_FAILED 后 GetFormat() 回退到软解，但线程模型未调整
```

**解决方案：**
- `VC_REOPEN` 返回值触发 `Reopen()`，会完全重建解码器
- 检查 `m_decoderState` 是否从 `STATE_HW_SINGLE` 正确切换到 `STATE_SW_MULTI`

## 本节小结

- KrKr2 的解码器采用**抽象基类 + 工厂模式**，`CDVDVideoCodec` 和 `CDVDAudioCodec` 定义接口契约，`CDVDFactoryCodec` 根据平台和流信息自动选择实现
- `CDVDVideoCodecFFmpeg` 封装 `avcodec_decode_video2()`，支持软件多线程解码和通过 `IHardwareDecoder` 接口进行硬件加速
- `CDVDAudioCodecFFmpeg` 封装 `avcodec_decode_audio4()`，包含采样格式映射和声道布局构建
- 视频解码器通过 `DVDVideoPicture`（YUV 数据指针 + 元信息）向上层传递帧数据，音频解码器通过 `DVDAudioFrame`（PCM 数据 + 格式信息）传递
- `CDropControl` 状态机检测实际帧丢弃，`SetDropState()` 通过 FFmpeg 的 `AVDISCARD` 跳过非参考帧以节省 CPU
- 工厂模式按照**硬件专用解码器 → FFmpeg 内部硬件加速 → FFmpeg 软件解码**的优先级链选择解码器

## 练习题与答案

### 题目 1：分析 Decode() 返回 VC_REOPEN 的触发条件

阅读 `VideoCodecFFmpeg.cpp` 中 `Decode()` 方法的相关代码，回答以下问题：
1. `VC_REOPEN` 在什么条件下返回？
2. 上层代码（`CVideoPlayerVideo`）收到 `VC_REOPEN` 后会执行什么操作？
3. 为什么这个机制对硬件解码回退至关重要？

<details>
<summary>查看答案</summary>

1. `VC_REOPEN` 在 `Decode()` 方法的第 610-611 行返回：

```cpp
if (m_decoderState == STATE_HW_FAILED && !m_pHardware)
    return VC_REOPEN;
```

条件是：`m_decoderState` 为 `STATE_HW_FAILED`（硬件解码器在 `GetFormat()` 中所有候选都失败后设置），且 `m_pHardware` 为空（已清理硬件解码器引用）。

2. 上层代码收到 `VC_REOPEN` 后，会调用 `m_pVideoCodec->Reopen()`（`VideoCodecFFmpeg.cpp` 第 745-750 行）：

```cpp
void CDVDVideoCodecFFmpeg::Reopen() {
    Dispose();
    if (!Open(m_hints, m_options)) {
        Dispose();
    }
}
```

这会完全销毁当前解码器并重新创建。由于 `m_decoderState` 已经是 `STATE_HW_FAILED`，重新 `Open()` 时不会再尝试硬件解码，而是直接进入 `STATE_SW_MULTI` 多线程软件解码模式。

3. 这个机制至关重要是因为：硬件解码器可能在任何时候失败（驱动问题、不支持的编码参数、显存不足等），通过 `VC_REOPEN` 机制可以透明地回退到软件解码，保证视频播放不中断。

</details>

### 题目 2：为什么音频解码器不需要 SetDropState()？

视频解码器有 `SetDropState()` 方法来跳过帧解码，但音频解码器没有类似的方法。请从音视频解码的特性差异角度解释原因。

<details>
<summary>查看答案</summary>

音频解码器不需要 `SetDropState()` 的原因有以下几点：

1. **音频不能丢帧**：人耳对音频中断极其敏感，即使丢失几毫秒的音频数据也会产生明显的"卡顿"或"咔嗒"声。相比之下，丢弃几帧视频（尤其是 B 帧）在视觉上几乎察觉不到。

2. **音频解码成本低**：音频解码的 CPU 开销远低于视频解码。一帧 1080p 视频解码可能需要几毫秒，而一帧音频解码通常在微秒级别。因此不需要通过跳过解码来节省 CPU。

3. **音频是时间基准**：在 A/V 同步中，音频通常是主时钟参考。视频帧可以丢弃来追赶音频，但音频不能丢弃。如果音频落后，A/V 同步模块会通过调整重采样率（`SYNC_RESAMPLE`）或插入/删除小量采样来修正，而不是丢弃整帧。

4. **帧间依赖差异**：视频中 B 帧可以安全丢弃（不被其他帧参考），但音频帧通常是顺序依赖的，跳过一帧会导致后续帧解码错误。

</details>

### 题目 3：实现一个自定义的 CDVDVideoCodec 子类

假设你需要为 KrKr2 添加一个 "空解码器"（NullCodec），用于性能测试。它接受任何编解码器 ID，但不实际解码，只生成纯黑帧。请写出完整的头文件和实现。

```cpp
class CDVDVideoCodecNull : public CDVDVideoCodec {
public:
    CDVDVideoCodecNull(CProcessInfo &processInfo)
        : CDVDVideoCodec(processInfo),

    ~CDVDVideoCodecNull() override {
        delete[] m_blackFrame;
    }

    bool Open(CDVDStreamInfo &hints,
              CDVDCodecOptions &options) override {
        m_width = hints.width > 0 ? hints.width : 1280;
        m_height = hints.height > 0 ? hints.height : 720;

        // 分配纯黑 YUV420P 帧
        int ySize = m_width * m_height;
        int uvSize = (m_width / 2) * (m_height / 2);
        m_blackFrame = new uint8_t[ySize + uvSize * 2]();

        // Y 平面全 0 = 黑色
        memset(m_blackFrame, 0, ySize);
        // U/V 平面全 128 = 色度中值（无色）

        return true;

               double dts, double pts) override {
        m_lastPts = pts;
        m_frameCount++;
        return VC_PICTURE | VC_BUFFER;
    }

    bool GetPicture(DVDVideoPicture *pic) override {
        memset(pic, 0, sizeof(DVDVideoPicture));
        pic->iWidth = m_width;
        pic->iHeight = m_height;
        pic->iDisplayWidth = m_width;
        pic->iDisplayHeight = m_height;
        pic->format = RENDER_FMT_YUV420P;
        pic->pts = m_lastPts;
        pic->iFlags = DVP_FLAG_ALLOCATED;

        int ySize = m_width * m_height;
        int uvW = m_width / 2;
        pic->data[0] = m_blackFrame;
        pic->data[1] = m_blackFrame + ySize;
        pic->data[2] = m_blackFrame + ySize + uvW * (m_height / 2);
        pic->iLineSize[0] = m_width;
        pic->iLineSize[1] = uvW;
        pic->iLineSize[2] = uvW;

        return true;
    }

    void SetDropState(bool bDrop) override {}
    const char *GetName() override { return "null"; }

private:
    int m_width;
    int m_height;
    int m_frameCount;
    double m_lastPts = 0.0;
    uint8_t *m_blackFrame;
```
