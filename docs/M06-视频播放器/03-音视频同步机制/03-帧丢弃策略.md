## 为什么需要帧丢弃

在视频播放过程中，解码线程（CVideoPlayerVideo）需要将每一帧解码后的画面及时交给渲染管线显示。但在以下场景中，解码速度可能跟不上显示需求：

- **CPU 性能不足**：低端设备解码 1080p 视频时，单帧解码耗时超过帧间隔（如 30fps 视频的帧间隔为 33.3ms）
- **突发性负载**：系统后台任务抢占 CPU 时间片，导致解码线程被短暂挂起
- **复杂视频帧**：I 帧（关键帧）的解码时间通常是 P/B 帧的 3-10 倍
- **Seek 后追赶**：用户拖动进度条后，解码器需要从最近的关键帧开始解码，积累了大量待解码帧

如果不处理这些迟到帧，会出现以下问题：

```
时间轴: ──────────────────────────────────────────────►
音频:    |帧1|帧2|帧3|帧4|帧5|帧6|帧7|帧8|帧9|帧10|
视频:    |帧1|..帧2..|..帧3..|帧4|..帧5..|帧6|       ← 解码慢
显示:    |帧1|  空  |帧2|帧3 |  空  |帧4 |帧5|       ← 卡顿

问题：视频落后音频，画面卡顿，音画不同步
```

帧丢弃策略的核心思想是：**宁可跳过几帧画面，也要让视频追上音频时钟**。人眼对于丢失少量帧的感知远小于对音画不同步的感知——人耳能感知到约 20ms 的音画偏差，但短暂丢失 1-2 帧（33-66ms 的画面缺失）通常不会被注意到。

## KrKr2 的三层帧丢弃架构

KrKr2 的帧丢弃机制分为三个层次，从解码到渲染逐层递进：

```
┌─────────────────────────────────────────────────────────┐
│                    帧丢弃三层架构                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第一层：解码层 (CDropControl)                            │
│  ┌─────────────────────────────────┐                    │
│  │ CDropControl::Process()         │                    │
│  │ - 统计 PTS 间隔，检测丢帧模式    │                    │
│  │ - INIT → 收集 10+ 样本          │                    │
│  │ - VALID → 检测 >20% PTS 偏差    │                    │
│  └──────────────┬──────────────────┘                    │
│                 │ 信息传递                                │
│                 ▼                                        │
│  第二层：决策层 (CalcDropRequirement)                     │
│  ┌─────────────────────────────────┐                    │
│  │ CalcDropRequirement()           │                    │
│  │ - 综合 decoder stats + render   │                    │
│  │   stats 做出丢弃决策             │                    │
│  │ - 返回 EOS_VERYLATE 等标志位    │                    │
│  │ - SetDropState() 控制解码器      │                    │
│  └──────────────┬──────────────────┘                    │
│                 │ 丢弃的帧                               │
│                 ▼                                        │
│  第三层：渲染层 (PrepareNextRender)                       │
│  ┌─────────────────────────────────┐                    │
│  │ PrepareNextRender()             │                    │
│  │ - 跳过队列中已过期的帧           │                    │
│  │ - m_lateframes 追踪迟到程度      │                    │
│  │ - 容差收紧：>6 帧迟到则无容差    │                    │
│  └─────────────────────────────────┘                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 三层协作的信号流

```
解码器 ──GetCodecStats()──► CalcDropRequirement() ──SetDropState()──► 解码器
                                    │
渲染器 ──GetStats()────────►        │
                                    │
                                    ▼
                           Process() 主循环
                           设置 codecControl 标志
                                    │
                                    ▼
                           OutputPicture()
                           WaitForBuffer()
                                    │
                                    ▼
                           PrepareNextRender()
                           跳过过期帧
```

## 第一层：CDropControl — 解码层丢帧检测

`CDropControl` 是 `CDVDVideoCodecFFmpeg` 的内部类，负责在解码层面统计帧间隔并检测丢帧模式。它是一个简单但精巧的状态机。

### 状态机设计

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 61-102 行

// CDropControl 有两个状态
enum { INIT, VALID };

// 状态机成员变量
int64_t m_lastPTS;    // 上一帧的 PTS
int64_t m_diffPTS;    // PTS 间隔的累积和（用于求平均）
int     m_count;      // 样本计数器
int     m_state;      // 当前状态：INIT 或 VALID
```

### INIT 阶段：收集帧间隔样本

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 73-88 行
void CDVDVideoCodecFFmpeg::CDropControl::Process(int64_t pts, bool drop) {
    if (m_state == INIT) {
        // 当前帧和上一帧的 PTS 都有效时，累加间隔
        if (pts != AV_NOPTS_VALUE && m_lastPTS != AV_NOPTS_VALUE) {
            m_diffPTS += pts - m_lastPTS;  // 累加 PTS 差值
            m_count++;                      // 增加样本数
        }
        // 收集超过 10 个样本后，计算平均帧间隔
        if (m_count > 10) {
            m_diffPTS = m_diffPTS / m_count;  // 求平均帧间隔
            if (m_diffPTS > 0) {
                // 平均帧间隔有效，进入 VALID 状态
                m_state = CDropControl::VALID;
                m_count = 0;  // 重置计数器，用于后续异常检测
            }
        }
    }
    // ... VALID 状态处理见下文
    m_lastPTS = pts;  // 更新上一帧 PTS
}
```

**设计要点：**
- 需要 **至少 10 个样本** 才能得到可靠的平均帧间隔。对于 30fps 视频，这意味着约 333ms 的采集时间
- 对 `AV_NOPTS_VALUE`（即 `0x8000000000000000`，FFmpeg 的无效 PTS 标记）进行过滤，跳过无效样本
- `m_diffPTS > 0` 检查确保计算出的平均帧间隔为正值（排除逆序 PTS 的异常情况）

### VALID 阶段：检测丢帧

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 89-102 行
} else if (m_state == VALID && !drop) {
    // 仅在非丢弃模式下检测——如果正在丢弃，PTS 跳变是正常的
    if (std::abs(pts - m_lastPTS - m_diffPTS) > m_diffPTS * 0.2) {
        // 当前帧间隔偏离平均值超过 20%
        m_count++;        // 累加异常计数
        if (m_count > 5) {
            // 连续 5 次异常，说明帧率已经发生变化
            // 重置状态机，重新学习新的帧间隔
            Reset(true);
        }
    } else {
        m_count = 0;  // 正常帧，重置异常计数
    }
}
```

**20% 偏差阈值的含义：**

```
30fps 视频：平均帧间隔 = 33.3ms
20% 偏差阈值 = 33.3ms × 0.2 = 6.67ms
允许范围：33.3ms ± 6.67ms = [26.6ms, 40.0ms]

如果两帧间隔为 66.6ms（跳过了一帧），偏差 = |66.6 - 33.3| = 33.3ms
33.3ms > 6.67ms → 检测到丢帧！

如果两帧间隔为 35ms（轻微抖动），偏差 = |35 - 33.3| = 1.7ms
1.7ms < 6.67ms → 正常，不触发
```

### Reset 函数的 init 参数

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 63-71 行
void CDVDVideoCodecFFmpeg::CDropControl::Reset(bool init) {
    m_lastPTS = AV_NOPTS_VALUE;  // 总是清除上一帧 PTS

    if (init || m_state != VALID) {
        // 完全重置：回到 INIT 状态
        m_count = 0;
        m_diffPTS = 0;
        m_state = INIT;
    }
    // 如果 init=false 且当前是 VALID，保留学习到的帧间隔
    // 只清除 lastPTS，这样 Seek 后可以快速恢复检测
}
```

**两种 Reset 场景：**

| 场景 | init 值 | 行为 |
|------|---------|------|
| Seek / Flush | `false` | 保留 `m_diffPTS`，快速恢复 VALID 状态 |
| 帧率变化（连续 5 次异常） | `true` | 完全重置，重新学习帧间隔 |
| 流切换 | `true` | 新视频流，需要重新学习 |

## 第二层：CalcDropRequirement — 丢弃决策

`CalcDropRequirement()` 是帧丢弃的核心决策函数，它综合解码器和渲染器的状态信息，输出丢弃指令。

### 返回值标志位

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.h 第 19-22 行
#define EOS_ABORT        1   // 致命错误，中止输出
#define EOS_DROPPED      2   // 帧已被丢弃
#define EOS_VERYLATE     4   // 严重迟到，请求丢弃下一帧
#define EOS_BUFFER_LEVEL 8   // 缓冲区水位低，需要加速解码
```

这些标志位可以通过按位或组合使用。例如 `EOS_DROPPED | EOS_BUFFER_LEVEL` 表示帧已被丢弃，同时缓冲区水位偏低。

### CalcDropRequirement 完整解析

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp 第 1013-1090 行
int CVideoPlayerVideo::CalcDropRequirement(double pts) {
    int result = 0;
    int lateframes;
    double iDecoderPts, iRenderPts;
    int iSkippedPicture = -1;  // 解码器跳过的帧数
    int iDroppedFrames = -1;   // 解码器丢弃的帧数
    int iBufferLevel;
    int queued, discard;

    m_droppingStats.m_lastPts = pts;  // 记录最后一次处理的 PTS

    // ① 获取解码器统计信息
    if (!m_pVideoCodec->GetCodecStats(
            iDecoderPts, iDroppedFrames, iSkippedPicture))
        iDecoderPts = pts;  // 获取失败则使用当前 PTS
    if (iDecoderPts == DVD_NOPTS_VALUE)
        iDecoderPts = pts;

    // ② 获取渲染器统计信息
    m_renderManager.GetStats(lateframes, iRenderPts, queued, discard);
    iBufferLevel = queued + discard;  // 总缓冲帧数 = 排队中 + 待丢弃

    // ③ 检查缓冲区水位
    if (iBufferLevel < 0)
        result |= EOS_BUFFER_LEVEL;     // 异常：缓冲区为负
    else if (iBufferLevel < 2) {
        result |= EOS_BUFFER_LEVEL;     // 缓冲区不足 2 帧，需要加速
    }

    // ④ 处理解码器报告的丢帧/跳帧
    if (m_bAllowDrop) {
        if (iSkippedPicture > 0) {
            // 解码器跳过了某些帧（如 B 帧被跳过）
            CDroppingStats::CGain gain;
            gain.frames = iSkippedPicture;
            gain.pts = iDecoderPts;
            m_droppingStats.m_gain.push_back(gain);
            m_droppingStats.m_totalGain += gain.frames;
            result |= EOS_DROPPED;
        }
        if (iDroppedFrames > 0) {
            // 解码器丢弃了某些帧
            CDroppingStats::CGain gain;
            gain.frames = iDroppedFrames;
            gain.pts = iDecoderPts;
            m_droppingStats.m_gain.push_back(gain);
            m_droppingStats.m_totalGain += iDroppedFrames;
            result |= EOS_DROPPED;
        }
    }

    // ⑤ 减去已经"兑现"的增益
    // 当渲染器的 PTS 超过了某个增益记录的 PTS，
    // 说明那次丢帧的效果已经体现，可以扣除
    while (!m_droppingStats.m_gain.empty() &&
           iRenderPts >= m_droppingStats.m_gain.front().pts) {
        m_droppingStats.m_totalGain -= 
            m_droppingStats.m_gain.front().frames;
        m_droppingStats.m_gain.pop_front();
    }

    // ⑥ 计算净迟到程度
    int lateness = lateframes - m_droppingStats.m_totalGain;
    // lateframes: 渲染器报告的迟到帧数
    // m_totalGain: 已经丢弃但效果尚未体现的帧数
    // lateness > 0: 仍然迟到，需要继续丢弃

    if (lateness > 0 && m_speed) {
        result |= EOS_VERYLATE;  // 通知调用者：需要丢弃下一帧
    }
    return result;
}
```

### CDroppingStats — 丢帧增益追踪器

`CDroppingStats` 是一个辅助类，用于追踪"已经决定丢弃但效果尚未在渲染端体现"的帧数。这是一个精妙的设计——防止重复丢帧。

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.h 第 24-37 行
class CDroppingStats {
public:
    void Reset();
    void AddOutputDropGain(double pts, int frames);

    struct CGain {
        int frames;     // 丢弃的帧数
        double pts;     // 丢弃发生时的 PTS
    };
    std::deque<CGain> m_gain;   // 增益记录队列
    double m_totalGain;         // 总增益（未兑现的丢弃帧数）
    double m_lastPts;           // 最后处理的 PTS
};
```

**增益追踪的工作原理：**

```
时间轴：────────────────────────────────────────────►
         t1        t2        t3        t4
         │         │         │         │
决策层：  丢弃3帧   │         │         │
         gain={3,t1}│        │         │
         │         │         │         │
渲染层：  │    显示帧A  显示帧B    │
         │    pts=t0   pts=t2    │
         │         │         │         │
增益计算：│         │   iRenderPts=t2 > t1
         │         │   扣除 gain={3,t1}
         │         │   totalGain -= 3
         │         │         │         │
         │    totalGain=3  totalGain=0  │
```

如果没有这个增益追踪机制，CalcDropRequirement 会持续看到 `lateframes > 0` 而不断请求丢帧，导致过度丢弃。

### Process() 主循环中的丢弃执行

CalcDropRequirement 返回的标志位在 `Process()` 主循环中被处理：

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp 第 388-428 行
// （位于 DEMUXER_PACKET 消息处理分支内）

bRequestDrop = false;
iDropDirective = CalcDropRequirement(pts);

// ① EOS_VERYLATE：请求解码器丢弃下一帧
if (iDropDirective & EOS_VERYLATE) {
    if (m_bAllowDrop) {
        bRequestDrop = true;  // 允许丢弃
    }
}

// ② 设置编解码器控制标志
int codecControl = 0;
if (iDropDirective & EOS_BUFFER_LEVEL)
    codecControl |= DVD_CODEC_CTRL_HURRY;     // 催促解码器加速
if (m_speed > DVD_PLAYSPEED_NORMAL)
    codecControl |= DVD_CODEC_CTRL_NO_POSTPROC; // 快进时跳过后处理
if (bPacketDrop)
    codecControl |= DVD_CODEC_CTRL_DROP;        // 来自上层的强制丢弃

m_pVideoCodec->SetCodecControl(codecControl);

// ③ 记录丢弃的帧
if (iDropDirective & EOS_DROPPED) {
    m_iDroppedFrames++;   // 总丢帧计数
    iDropped++;           // 连续丢帧计数
}

// ④ 安全阀：如果队列已空或倒放，停止丢弃
if (m_messageQueue.GetDataSize() == 0 || m_speed < 0) {
    bRequestDrop = false;
    m_iDroppedRequest = 0;
    m_iLateFrames = 0;
}

// ⑤ 上层强制丢弃优先级最高
if (bPacketDrop)
    bRequestDrop = true;

// ⑥ 通知解码器是否丢弃下一帧
m_pVideoCodec->SetDropState(bRequestDrop);
```

**codecControl 标志位含义：**

| 标志 | 值 | 含义 | 解码器行为 |
|------|---|------|-----------|
| `DVD_CODEC_CTRL_HURRY` | 缓冲区低 | 加速解码 | 跳过后处理滤镜、降低质量 |
| `DVD_CODEC_CTRL_NO_POSTPROC` | 快进模式 | 禁用后处理 | 跳过去块效应、锐化等滤镜 |
| `DVD_CODEC_CTRL_DROP` | 强制丢弃 | 直接丢弃该帧 | 不输出解码结果 |

### m_bAllowDrop 的控制

`m_bAllowDrop` 是一个重要的开关，控制是否允许主动丢帧。在原版 Kodi 中，这个标志由帧率检测结果控制——只有当帧率被准确计算出来后，才允许丢帧（避免在帧率未知时误丢帧）。

```cpp
// 在 KrKr2 中，CalcFrameRate() 被整体 #if 0 禁用
// ResetFrameRateCalc() 同样被禁用
// 因此 m_bAllowDrop 的初始值 false 将保持不变

// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp 第 61 行
m_bAllowDrop = false;  // 构造函数中初始化

// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp 第 898-908 行
void CVideoPlayerVideo::ResetFrameRateCalc() {
#if 0   // ← 整体禁用！
    // ... 原版 Kodi 代码 ...
    m_bAllowDrop = (!m_bCalcFrameRate && ...) || ...;
#endif  // 因此 m_bAllowDrop 永远不会被设为 true
}
```

**重要发现：** 在 KrKr2 中，`m_bAllowDrop` 始终为 `false`！这意味着 `CalcDropRequirement()` 中 `if (m_bAllowDrop)` 分支永远不会执行，解码器的主动丢帧逻辑在 KrKr2 中被间接禁用了。视频播放的帧丢弃主要依赖渲染层的 `PrepareNextRender()`。

> **KrKr2 特化说明：** 视觉小说的视频通常是预渲染的过场动画（OP/ED），分辨率和编码复杂度适中，在目标平台上一般不会出现严重的解码性能问题。因此禁用主动丢帧是一个合理的简化。

## 第三层：PrepareNextRender — 渲染层帧跳过

`PrepareNextRender()` 是渲染管理器中最关键的帧调度函数，负责从帧队列中选取下一个要显示的帧。当检测到队列中有过期帧时，它会跳过这些帧。

### 完整源码解析

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoRenderer.cpp 第 1141-1222 行
void CRenderManager::PrepareNextRender() {
    if (m_queued.empty()) {
        // 队列为空，回到 IDLE 状态
        m_presentstep = PRESENT_IDLE;
        m_presentevent.notify_all();
        return;
    }

    // ① 计算当前屏幕上应该显示的时间点
    double frameOnScreen = m_dvdClock.GetClock();
    double frametime = 1.0 / g_graphicsContext.GetFPS() * DVD_TIME_BASE;
    // frametime = 一个显示刷新周期（如 60Hz → 16.67ms → 16667µs）

    // ② 计算总延迟补偿
    // 显示延迟 + 驱动内部缓冲（假设驱动提前 1 帧）
    double totalLatency = DVD_SEC_TO_TIME(m_displayLatency)
                        - DVD_MSEC_TO_TIME(m_videoDelay)
                        + 2 * frametime;
    // totalLatency ≈ 0 - 0 + 2×16667µs ≈ 33334µs（约 33ms）

    // ③ 计算渲染目标时间点
    double renderPts = frameOnScreen + totalLatency;
    // renderPts = "现在开始渲染的帧，到显示在屏幕上时，时钟走到这里"

    double nextFramePts = m_Queue[m_queued.front()].pts;
    if (m_dvdClock.GetClockSpeed() < 0)
        nextFramePts = renderPts;  // 倒放时跳过 PTS 比较

    // ④ VSync 时钟同步校正（可选）
    if (m_clockSync.m_enabled) {
        double err = fmod(renderPts - nextFramePts, frametime);
        m_clockSync.m_error += err;
        m_clockSync.m_errCount++;
        if (m_clockSync.m_errCount > 30) {
            double average = m_clockSync.m_error / m_clockSync.m_errCount;
            m_clockSync.m_syncOffset = average;
            m_clockSync.m_error = 0;
            m_clockSync.m_errCount = 0;
            // 反馈给主时钟进行微调
            m_dvdClock.SetVsyncAdjust(-average);
        }
        renderPts += frametime / 2 - m_clockSync.m_syncOffset;
    } else {
        m_dvdClock.SetVsyncAdjust(0);
    }

    // ⑤ 核心逻辑：选择要显示的帧
    if (renderPts >= nextFramePts || m_forceNext) {
        auto iter = m_queued.begin();
        int idx = *iter;
        ++iter;
        while (iter != m_queued.end()) {
            // 容差计算：迟到程度越严重，容差越小
            // m_lateframes <= 6: 容差 = 0.98 × frametime
            // m_lateframes >  6: 容差 = 0（完全不容忍）
            double x = (m_lateframes <= 6) ? 0.98 : 0;
            if (renderPts < m_Queue[*iter].pts + x * frametime)
                break;  // 这个帧还没到时间，停止跳过
            idx = *iter;  // 跳过当前帧，选择更新的帧
            ++iter;
        }

        // ⑥ 将跳过的帧移入丢弃队列
        while (m_queued.front() != idx) {
            requeue(m_discard, m_queued);  // free → queued → discard
            m_QueueSkip++;                 // 跳帧计数器
        }

        // ⑦ 更新迟到帧计数
        int lateframes = (renderPts - m_Queue[idx].pts) 
                        * m_fps / DVD_TIME_BASE;
        if (lateframes)
            m_lateframes += lateframes;  // 累加迟到
        else
            m_lateframes = 0;            // 不迟到，立即归零

        // ⑧ 切换到 FLIP 状态，准备显示
        m_presentstep = PRESENT_FLIP;
        m_discard.push_back(m_presentsource);  // 当前帧进丢弃队列
        m_presentsource = idx;                  // 选中新帧
        m_queued.pop_front();
        m_presentpts = m_Queue[idx].pts - totalLatency;
        m_presentevent.notify_all();
    }
}
```

### 容差收紧机制详解

这是 PrepareNextRender 最精巧的部分——根据迟到程度动态调整容差：

```
m_lateframes 的含义：
- 负值 (-1)：初始化状态
- 0：当前不迟到
- 1-6：轻微迟到，给予 0.98 × frametime 的容差
- >6：严重迟到，容差为 0（零容忍）

容差的效果：

【轻微迟到时（m_lateframes <= 6）】
  renderPts < pts + 0.98 * frametime  → 不跳过
  
  例：60fps，frametime = 16.67ms
  如果帧 PTS 超前 renderPts 最多 16.33ms (0.98 × 16.67)，
  认为该帧"还来得及"，不跳过。
  这给了驱动内部缓冲一个合理的处理窗口。

【严重迟到时（m_lateframes > 6）】
  renderPts < pts + 0 * frametime  → 不跳过
  即：renderPts < pts  → 不跳过
  
  只有帧 PTS 严格在未来才保留，
  任何已经到达或过期的帧都会被跳过。
  这是"紧急追赶模式"。
```

用数字示例说明：

```
场景：60fps 视频，frametime = 16.67ms
当前 renderPts = 100.0ms

队列中的帧：
  帧A: pts = 83.3ms  (迟到 16.7ms)
  帧B: pts = 100.0ms (刚好到时)
  帧C: pts = 116.7ms (尚未到时)

【m_lateframes = 3（轻微迟到）】
  检查帧B: renderPts(100) < pts(100) + 0.98*16.67(16.3) = 116.3?
  100 < 116.3 → true → 不跳过帧A，选择帧A显示
  结果：显示帧A（虽然迟到，但在容差范围内）

【m_lateframes = 8（严重迟到）】
  检查帧B: renderPts(100) < pts(100) + 0 = 100?
  100 < 100 → false → 跳过帧A
  检查帧C: renderPts(100) < pts(116.7) + 0 = 116.7?
  100 < 116.7 → true → 不跳过帧B，选择帧B显示
  结果：跳过帧A，显示帧B（快速追赶）
```

### m_lateframes 的累加与归零

```cpp
// 迟到帧计数的计算
int lateframes = (renderPts - m_Queue[idx].pts) * m_fps / DVD_TIME_BASE;
```

这个公式将 PTS 差值转换为帧数：

```
renderPts - pts = 时间差（微秒）
× m_fps = 乘以帧率
÷ DVD_TIME_BASE = 除以 1000000（微秒→秒的转换已在乘法中体现）

例：renderPts 比 pts 早 33ms，30fps 视频
lateframes = 33000 * 30 / 1000000 = 0.99 → int 截断 = 0
不算迟到！精确到帧的判定。

renderPts 比 pts 早 50ms：
lateframes = 50000 * 30 / 1000000 = 1.5 → int 截断 = 1
迟到 1 帧。
```

**归零策略的含义：** `m_lateframes` 要么累加（持续迟到），要么立即归零（恢复正常）。这种"非累加即归零"的设计意味着：一旦视频追上了时钟，之前的迟到历史立即清除，容差恢复到宽松模式。

```cpp

                              int &queued, int &discard) {

    pts = m_dvdClock.GetClock();     // 当前时钟值
```

**`m_lateframes / 10` 的设计意图：** `m_lateframes` 在 `PrepareNextRender()` 中是按帧累加的（可能每帧加 1-2），而 `GetStats` 向外部报告时除以 10，起到了**低通滤波**的效果——避免瞬时的迟到波动导致上游（CalcDropRequirement）过度反应。

## OutputPicture 中的丢弃路径

OutputPicture 是帧从解码器到渲染器的最后一道关卡，包含多个丢弃判定点：

```cpp
// 源码位置：cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp 第 714-871 行
int CVideoPlayerVideo::OutputPicture(const DVDVideoPicture *src, double pts) {
    // ... 配置渲染器 ...
    
    // 丢弃路径 1：快进模式下的预估丢弃
    if (m_speed > DVD_PLAYSPEED_NORMAL) {
        // 估算下一帧到渲染器的时间
        double diff = pts - renderPts;
        double mindiff = DVD_SEC_TO_TIME(1/m_fFrameRate) * (bufferLevel+1);
        if (diff < mindiff) {
            // 该帧到达渲染器时已经迟到，提前丢弃
            m_droppingStats.AddOutputDropGain(pts, 1);
            return result | EOS_DROPPED;
        }
    }
    
    // 丢弃路径 2：解码器标记的丢弃帧
    if (pPicture->iFlags & DVP_FLAG_DROPPED) {
        m_droppingStats.AddOutputDropGain(pts, 1);
        return result | EOS_DROPPED;
    }
    
    // 丢弃路径 3：等待渲染缓冲区超时
    int buffer = m_renderManager.WaitForBuffer(m_bAbortOutput, maxWaitTime);
    if (buffer < 0) {
        m_droppingStats.AddOutputDropGain(pts, 1);
        return EOS_DROPPED;
    }
    
    // 丢弃路径 4：AddVideoPicture 失败
    int index = m_renderManager.AddVideoPicture(*pPicture);
    if (index < 0) {
        m_droppingStats.AddOutputDropGain(pts, 1);
        return EOS_DROPPED;
    }
    
    // 成功路径：提交帧给渲染器
    m_renderManager.FlipPage(m_bAbortOutput, pts, ...);
    return result;
}
```

```cpp
#include <cstdio>
#include <cstdlib>
#include <deque>
#include <cmath>

// 模拟 DVD_TIME_BASE 和相关常量
static constexpr int64_t DVD_TIME_BASE = 1000000;
static constexpr int64_t DVD_NOPTS_VALUE = 0xFFF0000000000000LL;
static constexpr int64_t AV_NOPTS_VALUE = 0x8000000000000000LL;

// 模拟 EOS 标志位
#define EOS_ABORT        1
#define EOS_DROPPED      2
#define EOS_VERYLATE     4
#define EOS_BUFFER_LEVEL 8

// 简化版 CDroppingStats
struct DroppingStats {
    struct Gain {
        int frames;
        double pts;
    };
    std::deque<Gain> gains;
    int totalGain = 0;
    double lastPts = 0;

    void reset() {
        gains.clear();
        totalGain = 0;
    }

    void addGain(double pts, int frames) {
        gains.push_back({frames, pts});
        totalGain += frames;
    }

    // 减去已兑现的增益
    void subtractRealized(double renderPts) {
        while (!gains.empty() && renderPts >= gains.front().pts) {
            totalGain -= gains.front().frames;
            gains.pop_front();
        }
    }
};

// 简化版 CalcDropRequirement
int calcDropRequirement(
    double pts,                // 当前帧 PTS
    int lateframes,            // 渲染器报告的迟到帧数
    int bufferLevel,           // 渲染缓冲区水位
    int skippedPictures,       // 解码器跳过的帧数
    int droppedFrames,         // 解码器丢弃的帧数
    double decoderPts,         // 解码器 PTS
    double renderPts,          // 渲染器 PTS
    bool allowDrop,            // 是否允许丢弃
    DroppingStats &stats       // 丢帧统计
) {
    int result = 0;
    stats.lastPts = pts;

    // 检查缓冲区水位
    if (bufferLevel < 0) {
        result |= EOS_BUFFER_LEVEL;
    } else if (bufferLevel < 2) {
        result |= EOS_BUFFER_LEVEL;
        printf("  [HURRY] 缓冲区水位低: %d 帧\n", bufferLevel);
    }

    // 处理解码器报告的丢帧
    if (allowDrop) {
        if (skippedPictures > 0) {
            stats.addGain(decoderPts, skippedPictures);
            result |= EOS_DROPPED;
            printf("  [SKIP] 解码器跳过 %d 帧\n", skippedPictures);
        }
        if (droppedFrames > 0) {
            stats.addGain(decoderPts, droppedFrames);
            result |= EOS_DROPPED;
            printf("  [DROP] 解码器丢弃 %d 帧\n", droppedFrames);
        }
    }

    // 减去已兑现的增益
    stats.subtractRealized(renderPts);

    // 计算净迟到
    int lateness = lateframes - stats.totalGain;
    printf("  迟到帧数=%d, 未兑现增益=%d, 净迟到=%d\n",
           lateframes, stats.totalGain, lateness);

    if (lateness > 0) {
        result |= EOS_VERYLATE;
        printf("  [VERYLATE] 请求丢弃下一帧\n");
    }

    return result;
}

// 打印结果标志
void printFlags(int result) {
    printf("  结果标志: ");
    if (result & EOS_ABORT) printf("ABORT ");
    if (result & EOS_DROPPED) printf("DROPPED ");
    if (result & EOS_VERYLATE) printf("VERYLATE ");
    if (result & EOS_BUFFER_LEVEL) printf("BUFFER_LEVEL ");
    if (result == 0) printf("(无标志 - 正常)");
    printf("\n\n");
}

int main() {
    DroppingStats stats;
    double fps = 30.0;
    double frameInterval = DVD_TIME_BASE / fps;  // 33333µs

    printf("=== 场景 1：正常播放 ===\n");
    int r1 = calcDropRequirement(
        frameInterval * 10,   // pts = 第10帧
        0,                    // 不迟到
        3,                    // 缓冲区有3帧
        0, 0,                 // 无跳帧/丢帧
        frameInterval * 10,   // decoder pts
        frameInterval * 9,    // render pts = 第9帧
        true,                 // 允许丢弃
        stats
    );
    printFlags(r1);

    printf("=== 场景 2：迟到但有增益补偿 ===\n");
    stats.reset();
    // 先添加一些增益（之前丢了 2 帧）
    stats.addGain(frameInterval * 8, 2);
    int r2 = calcDropRequirement(
        frameInterval * 12,   // pts = 第12帧
        2,                    // 迟到2帧
        1,                    // 缓冲区只有1帧
        0, 0,
        frameInterval * 12,
        frameInterval * 7,    // render 还在第7帧（增益未兑现）
        true,
        stats
    );
    printFlags(r2);

    printf("=== 场景 3：严重迟到，无增益 ===\n");
    stats.reset();
    int r3 = calcDropRequirement(
        frameInterval * 20,   // pts = 第20帧
        5,                    // 迟到5帧！
        0,                    // 缓冲区空
        0, 0,
        frameInterval * 20,
        frameInterval * 15,
        true,
        stats
    );
    printFlags(r3);

    printf("=== 场景 4：解码器主动丢帧 ===\n");
    stats.reset();
    int r4 = calcDropRequirement(
        frameInterval * 25,
        3,                    // 迟到3帧
        2,
        2,                    // 解码器跳过2帧
        1,                    // 解码器丢弃1帧
        frameInterval * 25,
        frameInterval * 22,
        true,                 // 允许丢弃
        stats
    );
    printFlags(r4);

    return 0;
}
```

### 实验 2：模拟 PrepareNextRender 容差收紧

```cpp
#include <cstdio>
#include <deque>

static constexpr double DVD_TIME_BASE = 1000000.0;

struct FrameInQueue {
    int id;
    double pts;
};

void simulatePrepareNextRender(
    double renderPts,       // 渲染目标时间
    double frametime,       // 帧间隔（微秒）
    double fps,             // 帧率
    int lateframes,         // 当前迟到帧数
    std::deque<FrameInQueue> &queue  // 帧队列
) {
    if (queue.empty()) {
        printf("  队列为空，进入 IDLE\n");
        return;
    }

    auto iter = queue.begin();
    int idx = iter->id;
    double selectedPts = iter->pts;
    ++iter;

    int skipped = 0;
    while (iter != queue.end()) {
        // 容差计算
        double x = (lateframes <= 6) ? 0.98 : 0;
        double threshold = iter->pts + x * frametime;

        printf("  检查帧%d (pts=%.1fms): renderPts(%.1f) < threshold(%.1f)? ",
               iter->id, iter->pts / 1000.0,
               renderPts / 1000.0, threshold / 1000.0);

        if (renderPts < threshold) {
            printf("是 → 保留帧%d\n", idx);
            break;
        } else {
            printf("否 → 跳过帧%d\n", idx);
            idx = iter->id;
            selectedPts = iter->pts;
            skipped++;
            ++iter;
        }
    }

    // 更新迟到计数
    int newLate = (int)((renderPts - selectedPts) * fps / DVD_TIME_BASE);
    printf("  选中帧%d, 跳过%d帧, 新增迟到=%d帧\n", idx, skipped, newLate);

    if (newLate)
        lateframes += newLate;
    else
        lateframes = 0;
    printf("  累计迟到帧数: %d (容差模式: %s)\n\n",
           lateframes, lateframes > 6 ? "零容忍" : "宽松");
}

int main() {
    double fps = 30.0;
    double frametime = DVD_TIME_BASE / fps;  // 33333µs

    printf("=== 场景 A：轻微迟到（m_lateframes=2）===\n");
    {
        std::deque<FrameInQueue> q = {
            {1, 80000.0}, {2, 113333.0}, {3, 146666.0}
        };
        simulatePrepareNextRender(
            115000.0,     // 115ms — 稍微超过帧2的 PTS
            frametime, fps, 2, q
        );
    }

    printf("=== 场景 B：严重迟到（m_lateframes=8）===\n");
    {
        std::deque<FrameInQueue> q = {
            {1, 80000.0}, {2, 113333.0}, {3, 146666.0}
        };
        simulatePrepareNextRender(
            115000.0,     // 同样的 115ms
            frametime, fps, 8, q  // 但迟到帧数=8
        );
    }

    printf("=== 场景 C：大量迟到帧堆积 ===\n");
    {
        std::deque<FrameInQueue> q = {
            {1, 50000.0}, {2, 83333.0}, {3, 116666.0},
            {4, 150000.0}, {5, 183333.0}
        };
        simulatePrepareNextRender(
            170000.0,     // 170ms — 帧1-3都过期了
            frametime, fps, 10, q
        );
    }

    return 0;
}
```

### 实验 3：模拟 CDropControl 状态机

```cpp
#include <cstdio>
#include <cstdlib>
#include <cmath>
#include <cstdint>

static constexpr int64_t AV_NOPTS_VALUE = 0x8000000000000000LL;

class CDropControl {
public:
    enum State { INIT, VALID };

    CDropControl() { reset(true); }

    void reset(bool init) {
        m_lastPTS = AV_NOPTS_VALUE;
        if (init || m_state != VALID) {
            m_count = 0;
            m_diffPTS = 0;
            m_state = INIT;
        }
    }

    void process(int64_t pts, bool drop) {
        if (m_state == INIT) {
            if (pts != AV_NOPTS_VALUE && m_lastPTS != AV_NOPTS_VALUE) {
                int64_t diff = pts - m_lastPTS;
                m_diffPTS += diff;
                m_count++;
                printf("  INIT: 样本 #%d, diff=%lldµs, 累计=%lldµs\n",
                       m_count, (long long)diff, (long long)m_diffPTS);
            }
            if (m_count > 10) {
                m_diffPTS = m_diffPTS / m_count;
                if (m_diffPTS > 0) {
                    m_state = VALID;
                    m_count = 0;
                    printf("  → 进入 VALID 状态！平均帧间隔=%lldµs (%.1ffps)\n",
                           (long long)m_diffPTS,
                           1000000.0 / m_diffPTS);
                }
            }
        } else if (m_state == VALID && !drop) {
            int64_t deviation = std::abs(pts - m_lastPTS - m_diffPTS);
            int64_t threshold = (int64_t)(m_diffPTS * 0.2);
            printf("  VALID: 偏差=%lldµs, 阈值=%lldµs",
                   (long long)deviation, (long long)threshold);
            if (deviation > threshold) {
                m_count++;
                printf(" → 异常 #%d", m_count);
                if (m_count > 5) {
                    printf(" → 重置！帧率可能已变化");
                    reset(true);
                }
            } else {
                m_count = 0;
                printf(" → 正常");
            }
            printf("\n");
        }
        m_lastPTS = pts;
    }

    State getState() const { return m_state; }
    int64_t getAvgInterval() const { return m_diffPTS; }

private:
    int64_t m_lastPTS;
    int64_t m_diffPTS;
    int m_count;
    State m_state;
};

int main() {
    CDropControl ctrl;

    printf("=== 阶段 1：学习 30fps 帧间隔 ===\n");
    int64_t interval30 = 33333;  // 30fps
    for (int i = 0; i < 15; i++) {
        ctrl.process(interval30 * i, false);
    }

    printf("\n=== 阶段 2：正常播放（轻微抖动）===\n");
    int64_t base = interval30 * 15;
    for (int i = 0; i < 5; i++) {
        // 添加 ±2ms 的随机抖动
        int64_t jitter = (std::rand() % 4000) - 2000;
        ctrl.process(base + interval30 * i + jitter, false);
    }

    printf("\n=== 阶段 3：丢帧（间隔翻倍）===\n");
    base = interval30 * 20;
    for (int i = 0; i < 8; i++) {
        // 每隔一帧丢一帧 → 间隔变成 66666µs
        ctrl.process(base + interval30 * 2 * i, false);
    }

    printf("\n最终状态: %s\n",
           ctrl.getState() == CDropControl::INIT ? "INIT" : "VALID");

    return 0;
}
```

| `cpp/core/movie/ffmpeg/VideoPlayerVideo.cpp` | 1092-1103 | CDroppingStats::Reset/AddOutputDropGain |

| `cpp/core/movie/ffmpeg/VideoRenderer.cpp` | 1236-1245 | GetStats() — m_lateframes/10 平滑 |

### KrKr2 特化要点

1. **m_bAllowDrop 始终为 false**：因为 `CalcFrameRate()` 和 `ResetFrameRateCalc()` 都被 `#if 0` 禁用，`m_bAllowDrop` 永远不会被设为 `true`
2. **帧丢弃主要依赖渲染层**：虽然解码层的 CDropControl 和决策层的 CalcDropRequirement 代码都存在，但由于 `m_bAllowDrop = false`，实际起作用的是 PrepareNextRender 的帧跳过
3. **CalcFrameRate 完全禁用**：原版 Kodi 通过分析帧时间戳来精确计算帧率，KrKr2 直接使用 demuxer 提供的帧率值

## 常见错误与排查

### 错误 1：过度丢帧导致画面跳跃

**症状：** 视频播放时画面明显跳跃，像是快进

**排查：**
```cpp
// 在 PrepareNextRender 中添加日志
printf("PrepareNextRender: lateframes=%d, QueueSkip=%d, "
       "renderPts=%.3f, nextPts=%.3f\n",
       m_lateframes, m_QueueSkip,
       renderPts / DVD_TIME_BASE,
       m_Queue[m_queued.front()].pts / DVD_TIME_BASE);
```

**可能原因：**
- `totalLatency` 计算过大，导致 `renderPts` 远超实际需要
- 系统时钟不稳定（特别是 Android 上从休眠恢复后）
- `g_graphicsContext.GetFPS()` 返回了错误的显示刷新率

### 错误 2：帧丢弃不生效，视频持续落后

**症状：** 视频画面越来越落后于音频

**排查：**
```cpp
// 检查 m_bAllowDrop 的值
printf("m_bAllowDrop=%d, CalcDrop result=0x%x\n",
       m_bAllowDrop, iDropDirective);

// 检查 GetStats 的返回值
int lf, q, d;
double rpts;
m_renderManager.GetStats(lf, rpts, q, d);
printf("lateframes=%d (raw: %d), queued=%d, discard=%d\n",
       lf, m_lateframes, q, d);
```

**可能原因：**
- KrKr2 中 `m_bAllowDrop` 始终为 false，解码层丢帧不会触发
- 如果需要启用主动丢帧，需要修改构造函数或添加帧率检测回调

### 错误 3：m_lateframes 持续累加导致画质降低

**症状：** 即使视频已经追上时钟，容差仍然处于零容忍模式

**排查：**
```cpp
// m_lateframes 只在当前帧不迟到时归零
// 检查是否存在持续的微小迟到
int lateframes = (renderPts - m_Queue[idx].pts) * m_fps / DVD_TIME_BASE;
printf("当前帧迟到帧数: %d (累计: %d)\n", lateframes, m_lateframes);
// 如果 lateframes 持续为 1（边界值），m_lateframes 会不断累加
```

**解决方案：** 考虑给 `m_lateframes` 添加衰减机制，而不是仅在完全不迟到时归零。

## 本节小结

- KrKr2 的帧丢弃采用**三层架构**：解码层 CDropControl（状态机检测丢帧模式）、决策层 CalcDropRequirement（综合判断是否丢帧）、渲染层 PrepareNextRender（跳过过期帧）
- **CDropControl** 通过收集 10+ 样本学习帧间隔，然后用 20% 偏差阈值检测丢帧事件
- **CalcDropRequirement** 使用 CDroppingStats 增益追踪机制避免过度丢帧——已决定丢弃但效果未体现的帧不会重复触发丢弃
- **PrepareNextRender** 的容差收紧机制是核心：m_lateframes ≤ 6 时保留 0.98 帧时间的容差，>6 时零容忍
- **KrKr2 特化**：`m_bAllowDrop` 始终为 false，解码层主动丢帧被禁用，帧丢弃主要依赖渲染层的帧跳过
- `GetStats()` 中 `m_lateframes / 10` 的除法起到低通滤波作用，避免瞬时波动导致过度反应

## 练习题与答案

### 题目 1：CDropControl 的 20% 阈值计算

对于一个 24fps 的视频（帧间隔约 41667µs），CDropControl 的异常检测阈值是多少？如果某一帧的间隔为 83334µs（恰好两倍），会触发异常检测吗？

<details>
<summary>查看答案</summary>

```
平均帧间隔 = 1000000 / 24 = 41667µs
20% 阈值 = 41667 × 0.2 = 8333µs

帧间隔 83334µs 的偏差 = |83334 - 41667| = 41667µs
41667µs > 8333µs → 是的，会触发异常检测

这个偏差意味着跳过了一帧。CDropControl 会将 m_count 加 1。
如果连续 5 次出现这样的偏差（即连续跳帧），
状态机会重置到 INIT 状态重新学习帧间隔。
```

</details>

### 题目 2：CalcDropRequirement 的增益追踪

假设在 t=100ms 时，CalcDropRequirement 决定丢弃 3 帧（增益记录 {frames=3, pts=100ms}）。之后在 t=130ms 时再次调用 CalcDropRequirement，此时 renderPts=90ms，lateframes=4。净迟到（lateness）是多少？会触发 EOS_VERYLATE 吗？

<details>
<summary>查看答案</summary>

```
增益追踪检查：
  renderPts(90ms) >= gain.pts(100ms)?
  90ms < 100ms → 否，增益尚未兑现

totalGain = 3（仍然有效）

净迟到 = lateframes - totalGain = 4 - 3 = 1

lateness(1) > 0 → 是的，会触发 EOS_VERYLATE

解释：虽然已经丢弃了 3 帧，但由于渲染器还没处理到那个时间点
（renderPts=90ms < gain.pts=100ms），丢帧的效果还没有体现。
扣除未兑现增益后，净迟到为 1 帧，仍然需要继续丢帧。

当 renderPts 最终超过 100ms 时，totalGain 会减 3，
如果此时 lateframes 也降低，净迟到可能变为 0 甚至负数，
EOS_VERYLATE 就不会再触发了。
```

</details>

### 题目 3：PrepareNextRender 容差行为分析

在 60fps 视频（frametime=16667µs）中，队列包含帧 A(pts=200000µs) 和帧 B(pts=216667µs)。renderPts=218000µs。分别在 m_lateframes=4 和 m_lateframes=8 时，哪个帧会被选中显示？各跳过了几帧？

<details>
<summary>查看答案</summary>

```
【m_lateframes = 4（宽松模式）】
  容差 x = 0.98
  检查帧B: renderPts(218000) < pts_B(216667) + 0.98 * 16667(16333)?
  218000 < 233000? → true → 不跳过帧A
  选中帧A(pts=200000)，跳过 0 帧
  
  新 lateframes = (218000 - 200000) * 60 / 1000000 = 1.08 → int = 1
  累计 m_lateframes = 4 + 1 = 5

【m_lateframes = 8（零容忍模式）】
  容差 x = 0
  检查帧B: renderPts(218000) < pts_B(216667) + 0?
  218000 < 216667? → false → 跳过帧A
  没有更多帧了，选中帧B(pts=216667)，跳过 1 帧
  
  新 lateframes = (218000 - 216667) * 60 / 1000000 = 0.08 → int = 0
  m_lateframes = 0 → 归零！恢复宽松模式

总结：
  宽松模式下显示帧A（老帧），迟到继续累加
  零容忍模式下跳过帧A显示帧B（新帧），迟到归零恢复正常
  
这就是容差收紧的威力：在迟到严重时果断跳帧，
快速恢复到正常状态，避免持续的画面卡顿。
```

</details>
