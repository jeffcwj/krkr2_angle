## DVDVideoPicture 结构体详解

### 基本概念

`DVDVideoPicture`（DVD Video Picture，视频画面结构）是 KrKr2 视频播放器中最核心的数据结构之一。它继承自 Kodi/XBMC 的设计，用于在解码器和渲染器之间传递一帧完整的视频数据。这个结构体采用了 union（联合体）设计，使得同一块内存可以存储不同硬件后端的数据引用，这是一种经典的多态数据承载模式。

来源文件：`cpp/core/movie/ffmpeg/VideoCodec.h` 第 50-123 行

```cpp
// 视频帧的核心数据结构（来自 VideoCodec.h 第 50-123 行）
struct DVDVideoPicture {
    double pts;  // 显示时间戳（秒），用于 A/V 同步
    double dts;  // 解码时间戳（秒），B 帧排序用

    union {
        struct {
            uint8_t *data[4];      // YUV 平面数据指针：[0]=Y [1]=U [2]=V [3]=Alpha(未使用)
            int iLineSize[4];      // 每个平面一行的字节数（含 padding）
        };
        struct {
            DXVA::CRenderPicture *dxva;  // DirectX 硬件加速句柄
        };
        struct {
            VAAPI::CVaapiRenderPicture *vaapi;  // VA-API 硬件加速句柄
        };
        struct {
            CDVDMediaCodecInfo *mediacodec;  // Android MediaCodec 句柄
        };
        struct {
            CDVDAmlogicInfo *amlcodec;  // Amlogic 硬件解码句柄
        };
        // ... 还有 VDPAU、OpenMax、MMAL 等后端
    };

    unsigned int iFlags;          // 帧标志位（顶场优先、隔行、已丢弃等）
    double iRepeatPicture;        // 重复显示次数（3:2 pulldown 用）
    double iDuration;             // 帧持续时间
    unsigned int iFrameType : 4;  // 帧类型：I=1, P=2, B=3, D=4
    unsigned int color_matrix : 4; // 色彩矩阵（BT.601/BT.709 等）
    unsigned int color_range : 1;  // 1=全范围 0=有限范围

    unsigned int iWidth;          // 帧宽度（像素）
    unsigned int iHeight;         // 帧高度（像素）
    unsigned int iDisplayWidth;   // 去掉黑边后的显示宽度
    unsigned int iDisplayHeight;  // 去掉黑边后的显示高度

    ERenderFormat format;         // 渲染格式枚举（YUV420P、NV12 等）
};
```

### union 设计的意义

union 是 C/C++ 中的联合体（Union），所有成员共享同一段内存。`DVDVideoPicture` 的 union 包含了多种硬件后端的数据句柄，但在任意时刻只有一种是有效的，由 `format` 字段决定使用哪个成员：

```cpp
// 根据 format 判断使用 union 的哪个成员
void ProcessFrame(DVDVideoPicture& pic) {
    switch (pic.format) {
        case RENDER_FMT_YUV420P:
            // 使用 data[0..2] 和 iLineSize[0..2]
            // 这是纯软件解码的路径
            ProcessYUV(pic.data[0], pic.data[1], pic.data[2],
                       pic.iLineSize[0], pic.iLineSize[1], pic.iLineSize[2],
                       pic.iWidth, pic.iHeight);
            break;
        case RENDER_FMT_DXVA:
            // 使用 pic.dxva 指针，数据在 GPU 显存中
            // Windows DirectX 硬件加速路径
            ProcessDXVA(pic.dxva);
            break;
        case RENDER_FMT_MEDIACODEC:
            // 使用 pic.mediacodec 指针
            // Android 硬件解码路径
            ProcessMediaCodec(pic.mediacodec);
            break;
        default:
            break;
    }
}
```

**KrKr2 中的实际情况**：由于 KrKr2 使用纯 FFmpeg 软件解码（硬件加速工厂在 `FactoryCodec.cpp` 中被 `#if 0` 禁用），实际运行时 `format` 始终为 `RENDER_FMT_YUV420P`，union 中只有 `data[]` 和 `iLineSize[]` 被使用。

### 帧标志位详解

`iFlags` 字段使用位掩码（Bitmask）组合多个标志：

```cpp
// 来自 VideoCodec.h 第 130-155 行
#define DVP_FLAG_TOP_FIELD_FIRST    0x00000001  // 顶场优先（隔行扫描）
#define DVP_FLAG_REPEAT_TOP_FIELD   0x00000002  // 重复顶场（3:2 pulldown）
#define DVP_FLAG_ALLOCATED          0x00000004  // 数据已分配
#define DVP_FLAG_INTERLACED         0x00000008  // 隔行帧
#define DVP_FLAG_DROPPED            0x00000010  // 帧已在解码阶段被丢弃，无实际数据

// 编解码器控制标志（由播放器设置，通知解码器）
#define DVD_CODEC_CTRL_SKIPDEINT    0x01000000  // 跳过去隔行处理
#define DVD_CODEC_CTRL_NO_POSTPROC  0x02000000  // 跳过后处理
#define DVD_CODEC_CTRL_HURRY        0x04000000  // 渲染缓冲即将耗尽，加速输出
#define DVD_CODEC_CTRL_DROP         0x08000000  // 此帧将被丢弃
#define DVD_CODEC_CTRL_DRAIN        0x10000000  // 刷出最后的帧（不再输入新包）

// 使用示例：检查帧是否已被丢弃
void CheckFrame(DVDVideoPicture& pic) {
    if (pic.iFlags & DVP_FLAG_DROPPED) {
        // 帧在解码阶段被丢弃，data[] 指针无效
        // 只有 pts/dts 等元数据可用
        printf("帧被丢弃，PTS=%.3f\n", pic.pts);
        return;
    }
    if (pic.iFlags & DVP_FLAG_INTERLACED) {
        // 隔行帧，可能需要去隔行处理
        bool topFieldFirst = (pic.iFlags & DVP_FLAG_TOP_FIELD_FIRST) != 0;
        printf("隔行帧，%s场优先\n", topFieldFirst ? "顶" : "底");
    }
}
```

## YUV420P 像素格式与平面存储

### YUV 色彩空间基础

YUV 是一种将亮度（Luminance, Y）和色度（Chrominance, U/V）分离的色彩空间。与 RGB 不同，YUV 利用了人眼对亮度变化比色彩变化更敏感的特性，可以在减少色度信息的同时保持视觉质量。YUV420P 中的 "420" 表示色度采样比为 4:2:0，即水平和垂直方向上色度分辨率各减半；"P" 表示 Planar（平面存储），三个分量分别存储在独立的内存区域。

```
YUV420P 内存布局示意图（8x4 像素的帧）：

Y 平面（8x4 = 32 字节，每像素 1 字节）：
┌─────────────────────────┐
│ Y00 Y01 Y02 Y03 Y04 Y05 Y06 Y07 │  ← lineSize[0] 可能 > 8（含 padding）
│ Y10 Y11 Y12 Y13 Y14 Y15 Y16 Y17 │
│ Y20 Y21 Y22 Y23 Y24 Y25 Y26 Y27 │
│ Y30 Y31 Y32 Y33 Y34 Y35 Y36 Y37 │
└─────────────────────────┘

U 平面（4x2 = 8 字节，宽高各减半）：
┌───────────────┐
│ U00 U01 U02 U03 │  ← lineSize[1] 可能 > 4
│ U10 U11 U12 U13 │
└───────────────┘

V 平面（4x2 = 8 字节，宽高各减半）：
┌───────────────┐
│ V00 V01 V02 V03 │  ← lineSize[2] 可能 > 4
│ V10 V11 V12 V13 │
└───────────────┘
```

### lineSize 与 width 的区别

这是一个极其重要的概念。`lineSize`（行字节数，又称 stride 或 pitch）通常 **大于等于** 实际像素宽度。FFmpeg 为了 SIMD（Single Instruction Multiple Data，单指令多数据）对齐优化，会将每行数据填充（padding）到 16 或 32 字节的倍数。

```cpp
// 典型场景：1920x1080 视频
// iWidth = 1920
// iLineSize[0] = 1920（碰巧对齐）或 1952（32 字节对齐填充）
// iLineSize[1] = 960 或 976（色度平面）
// iLineSize[2] = 960 或 976

// 正确的逐行拷贝方式（来自 CodecUtils.cpp 第 52-85 行）
bool CopyYPlane(uint8_t* dst, int dstStride,
                uint8_t* src, int srcStride,
                int width, int height) {
    if (width == srcStride && srcStride == dstStride) {
        // 快速路径：stride 与宽度一致，整块 memcpy
        memcpy(dst, src, width * height);
    } else {
        // 慢速路径：逐行拷贝，只拷贝有效像素，跳过 padding
        for (int y = 0; y < height; y++) {
            memcpy(dst, src, width);  // 只拷贝 width 字节的有效数据
            src += srcStride;         // 源跳过完整行（含 padding）
            dst += dstStride;         // 目标跳过完整行
        }
    }
    return true;
}
```

**常见错误**：直接用 `memcpy(dst, src, width * height)` 拷贝整个平面——如果 `lineSize > width`，拷贝出的图像会出现斜纹或错位。

## 从 AVFrame 到 DVDVideoPicture 的转换

### 解码输出流程

FFmpeg 解码器输出 `AVFrame`，`CDVDVideoCodecFFmpeg::GetPicture()` 负责将其转换为 `DVDVideoPicture`。这个过程在解码线程（`CVideoPlayerVideo::Process()`）中执行：

```
FFmpeg 解码链路：

CDVDMsgDemuxerPacket (包含压缩数据)
        │
        ▼
CDVDVideoCodecFFmpeg::Decode()
        │  调用 avcodec_decode_video2(m_pCodecContext, m_pFrame, ...)
        │  返回 VC_PICTURE 表示有帧可取
        ▼
CDVDVideoCodecFFmpeg::GetPicture()
        │  将 AVFrame 字段映射到 DVDVideoPicture
        ▼
DVDVideoPicture (传递给 OutputPicture)
        │
        ▼
CRenderManager::AddVideoPicture()
        │  调用 CBaseRenderer::AddVideoPicture()
        ▼
TVPMoviePlayer::AddVideoPicture()
        │  拷贝到 BitmapPicture 环形缓冲
        ▼
BitmapPicture (等待主线程消费)
```

### GetPicture 的字段映射

```cpp
// CDVDVideoCodecFFmpeg::GetPicture() 的核心映射逻辑
bool CDVDVideoCodecFFmpeg::GetPicture(DVDVideoPicture* pDvdVideoPicture) {
    if (!m_pFrame) return false;

    // 1. 像素数据指针直接传递（零拷贝）
    // AVFrame::data[i] → DVDVideoPicture::data[i]
    // AVFrame::linesize[i] → DVDVideoPicture::iLineSize[i]
    for (int i = 0; i < 4; i++) {
        pDvdVideoPicture->data[i] = m_pFrame->data[i];
        pDvdVideoPicture->iLineSize[i] = m_pFrame->linesize[i];
    }

    // 2. 尺寸信息
    pDvdVideoPicture->iWidth = m_pCodecContext->width;
    pDvdVideoPicture->iHeight = m_pCodecContext->height;
    pDvdVideoPicture->iDisplayWidth = m_pCodecContext->width;
    pDvdVideoPicture->iDisplayHeight = m_pCodecContext->height;

    // 3. 时间戳：AVFrame::pts → 秒
    // 注意：FFmpeg 的 pts 是以 time_base 为单位的整数
    // 在 Demuxer 阶段已经转换为 DVD_TIME_BASE 单位
    pDvdVideoPicture->pts = m_dts;  // 由 Decode() 传入的 pts 参数
    pDvdVideoPicture->dts = DVD_NOPTS_VALUE;

    // 4. 帧类型映射
    // AVFrame::pict_type → iFrameType
    switch (m_pFrame->pict_type) {
        case AV_PICTURE_TYPE_I: pDvdVideoPicture->iFrameType = FRAME_TYPE_I; break;
        case AV_PICTURE_TYPE_P: pDvdVideoPicture->iFrameType = FRAME_TYPE_P; break;
        case AV_PICTURE_TYPE_B: pDvdVideoPicture->iFrameType = FRAME_TYPE_B; break;
        default: pDvdVideoPicture->iFrameType = FRAME_TYPE_UNDEF; break;
    }

    // 5. 渲染格式
    pDvdVideoPicture->format = m_formats[0];  // 通常为 RENDER_FMT_YUV420P

    // 6. 隔行扫描标志
    pDvdVideoPicture->iFlags = 0;
    if (m_pFrame->interlaced_frame)
        pDvdVideoPicture->iFlags |= DVP_FLAG_INTERLACED;
    if (m_pFrame->top_field_first)
        pDvdVideoPicture->iFlags |= DVP_FLAG_TOP_FIELD_FIRST;

    return true;
}
```

**关键细节**：`GetPicture()` 是零拷贝操作——`data[]` 指针直接指向 FFmpeg 内部的 AVFrame 缓冲区。这些指针在下次调用 `Decode()` 后可能失效，因此后续的 `AddVideoPicture()` 必须在同一帧处理周期内完成数据拷贝。

### 帧分配与释放

```cpp
// 来自 CodecUtils.cpp 第 16-50 行
// 分配一个 YUV420P 帧的内存
DVDVideoPicture* CDVDCodecUtils::AllocatePicture(int iWidth, int iHeight) {
    DVDVideoPicture* pPicture = new DVDVideoPicture;
    if (pPicture) {
        pPicture->iWidth = iWidth;
        pPicture->iHeight = iHeight;

        int w = iWidth / 2;   // 色度平面宽度 = 亮度的一半
        int h = iHeight / 2;  // 色度平面高度 = 亮度的一半
        int size = w * h;     // 单个色度平面大小
        // 总大小 = Y平面 + U平面 + V平面
        int totalsize = (iWidth * iHeight) + size * 2;
        uint8_t* data = new uint8_t[totalsize];

        if (data) {
            // 三个平面在连续内存中依次排列
            pPicture->data[0] = data;                          // Y 起始
            pPicture->data[1] = pPicture->data[0] + (iWidth * iHeight);  // U 起始
            pPicture->data[2] = pPicture->data[1] + size;     // V 起始
            pPicture->data[3] = nullptr;                       // Alpha 未使用

            // lineSize = 实际宽度（无 padding）
            pPicture->iLineSize[0] = iWidth;
            pPicture->iLineSize[1] = w;
            pPicture->iLineSize[2] = w;
            pPicture->iLineSize[3] = 0;
        }
    }
    return pPicture;
}

// 释放帧内存
void CDVDCodecUtils::FreePicture(DVDVideoPicture* pPicture) {
    av_free(pPicture->data[0]);  // 释放连续分配的 YUV 数据
    delete pPicture;              // 释放结构体
}
```

### YUV420P 帧拷贝

```cpp
// 来自 CodecUtils.cpp 第 52-85 行
// DVDVideoPicture → DVDVideoPicture 拷贝（处理不同 lineSize）
bool CDVDCodecUtils::CopyPicture(DVDVideoPicture* pDst, DVDVideoPicture* pSrc) {
    uint8_t *s, *d;
    int w = pSrc->iWidth;
    int h = pSrc->iHeight;

    // 拷贝 Y 平面
    s = pSrc->data[0];
    d = pDst->data[0];
    for (int y = 0; y < h; y++) {
        memcpy(d, s, w);              // 只拷贝有效宽度
        s += pSrc->iLineSize[0];      // 源：按源 stride 前进
        d += pDst->iLineSize[0];      // 目标：按目标 stride 前进
    }

    // 拷贝 U/V 平面（尺寸减半）
    w >>= 1;  // 宽度除以 2
    h >>= 1;  // 高度除以 2

    // 拷贝 U 平面
    s = pSrc->data[1];
    d = pDst->data[1];
    for (int y = 0; y < h; y++) {
        memcpy(d, s, w);
        s += pSrc->iLineSize[1];
        d += pDst->iLineSize[1];
    }

    // 拷贝 V 平面
    s = pSrc->data[2];
    d = pDst->data[2];
    for (int y = 0; y < h; y++) {
        memcpy(d, s, w);
        s += pSrc->iLineSize[2];
        d += pDst->iLineSize[2];
    }
    return true;
}
```

### YV12Image 中间格式

`YV12Image` 是渲染器端的帧数据结构，定义在 `BaseRenderer.h` 第 17-28 行：

```cpp
// 渲染器使用的帧数据格式（BaseRenderer.h 第 17-28 行）
typedef struct YV12Image {
    uint8_t* plane[MAX_PLANES];    // 平面数据指针（MAX_PLANES=3）
    int planesize[MAX_PLANES];     // 每个平面的总字节数
    unsigned stride[MAX_PLANES];   // 每行字节数（= lineSize）
    unsigned width = 0;            // 图像宽度
    unsigned height = 0;           // 图像高度
    unsigned flags;                // 标志位

    unsigned cshift_x;  // 色度水平偏移位数（YUV420=1, 即除以2）
    unsigned cshift_y;  // 色度垂直偏移位数（YUV420=1）
    unsigned bpp;       // 每像素字节数（YUV420P=1, NV12=1）
} YV12Image;
```

`CopyPicture(YV12Image*, DVDVideoPicture*)` 重载负责从解码帧拷贝到渲染器帧：

```cpp
// 来自 CodecUtils.cpp 第 87-129 行
bool CDVDCodecUtils::CopyPicture(YV12Image* pImage, DVDVideoPicture* pSrc) {
    // Y 平面拷贝
    uint8_t* s = pSrc->data[0];
    uint8_t* d = pImage->plane[0];
    int w = pImage->width * pImage->bpp;  // 有效行宽
    int h = pImage->height;

    // 快速路径判断：如果源和目标的行宽、stride 完全一致
    if ((w == pSrc->iLineSize[0]) &&
        ((unsigned int)pSrc->iLineSize[0] == pImage->stride[0])) {
        memcpy(d, s, w * h);  // 一次性拷贝整个平面
    } else {
        for (int y = 0; y < h; y++) {  // 逐行拷贝
            memcpy(d, s, w);
            s += pSrc->iLineSize[0];
            d += pImage->stride[0];
        }
    }

    // U 平面（使用 cshift_x/y 计算色度平面尺寸）
    s = pSrc->data[1];
    d = pImage->plane[1];
    w = (pImage->width >> pImage->cshift_x) * pImage->bpp;
    h = (pImage->height >> pImage->cshift_y);
    // ... 类似的快速/慢速路径拷贝

    // V 平面同理
    return true;
}
```

## BitmapPicture 环形缓冲区

### 结构定义

`BitmapPicture` 是 KrKr2 特有的（非 Kodi 继承的）帧缓冲结构，定义在 `KRMoviePlayer.h` 第 193-215 行：

```cpp
// KRMoviePlayer.h 第 193-223 行
struct BitmapPicture {
    ERenderFormat fmt;       // 存储格式
    union {
        uint8_t* data[4];    // 通用数据指针数组
        uint8_t* rgba;       // RGBA 模式：单指针指向 width*height*4 的缓冲
        uint8_t* yuv[3];     // YUV 模式：三个指针分别指向 Y/U/V 平面
    };
    int width = 0;           // 帧宽度
    int height = 0;          // 帧高度（rgba 模式下 pitch = width * 4）
    double pts;              // 显示时间戳（秒，已转换）

    void swap(BitmapPicture& r);  // 交换两个 BitmapPicture 的数据
    void Clear();                  // 释放内存并清零指针
};

// 环形缓冲管理
// MAX_BUFFER_COUNT = 4
BitmapPicture m_picture[MAX_BUFFER_COUNT];  // 4 个帧槽位
int m_curPicture = 0;    // 消费者读取位置
int m_usedPicture = 0;   // 已使用的帧数量
std::mutex m_mtxPicture;              // 保护缓冲区的互斥锁
std::condition_variable m_condPicture; // 通知消费者有帧可用 / 通知生产者有空位
```

### 环形缓冲工作原理

```
环形缓冲区状态示意（MAX_BUFFER_COUNT=4）：

写入位置 = (m_curPicture + m_usedPicture) & (MAX_BUFFER_COUNT - 1)
读取位置 = m_curPicture

初始状态（空）：
┌───┬───┬───┬───┐
│   │   │   │   │  m_curPicture=0, m_usedPicture=0
└───┴───┴───┴───┘
  ↑
 读/写

写入 2 帧后：
┌───┬───┬───┬───┐
│ A │ B │   │   │  m_curPicture=0, m_usedPicture=2
└───┴───┴───┴───┘
  ↑       ↑
 读      写

消费 1 帧后：
┌───┬───┬───┬───┐
│   │ B │   │   │  m_curPicture=1, m_usedPicture=1
└───┴───┴───┴───┘
      ↑   ↑
     读   写

缓冲区满（写入阻塞等待）：
┌───┬───┬───┬───┐
│ D │ A │ B │ C │  m_curPicture=1, m_usedPicture=4
└───┴───┴───┴───┘
      ↑
     读=写（满）
```

## AddVideoPicture：帧入队的两条路径

### Overlay 模式（YUV 直通）

Overlay 模式下，`TVPMoviePlayer::AddVideoPicture()` 将 YUV 数据直接拷贝到 `BitmapPicture`，不做像素格式转换。来源文件：`KRMoviePlayer.cpp` 第 176-228 行：

```cpp
// KRMoviePlayer.cpp 第 176-228 行
int TVPMoviePlayer::AddVideoPicture(DVDVideoPicture& pic, int index) {
    // 1. 格式检查：只接受 YUV420P
    if (pic.format != RENDER_FMT_YUV420P) return -2;  // 不支持的格式
    if (pic.pts == DVD_NOPTS_VALUE) return 0;          // 无效时间戳，静默丢弃

    // 2. 等待空闲槽位
    if (m_usedPicture >= MAX_BUFFER_COUNT) {
        std::unique_lock<std::mutex> lk(m_mtxPicture);
        m_condPicture.wait(lk);  // 阻塞等待消费者释放槽位
    }
    if (m_usedPicture >= MAX_BUFFER_COUNT) return -1;  // 仍然满，返回失败

    int width = pic.iWidth, height = pic.iHeight;

    // 3. 分配并拷贝 YUV 三个平面
    int yuvwidth[3] = { width, width / 2, width / 2 };
    int yuvheight[3] = { height, height / 2, height / 2 };
    uint8_t* yuvdata[3] = { nullptr };

    for (int i = 0; i < 3; ++i) {
        int size = yuvwidth[i] * yuvheight[i];
        yuvdata[i] = (uint8_t*)TJSAlignedAlloc(size, 4);  // 4 字节对齐分配

        if (yuvwidth[i] == pic.iLineSize[i]) {
            // 快速路径：lineSize == 宽度，直接整块拷贝
            memcpy(yuvdata[i], pic.data[i], size);
        } else {
            // 慢速路径：逐行拷贝，跳过 padding
            uint8_t* d = yuvdata[i], *s = pic.data[i];
            for (int y = 0; y < yuvheight[i]; ++y) {
                memcpy(d, s, yuvwidth[i]);
                d += yuvwidth[i];
                s += pic.iLineSize[i];
            }
        }
    }

    // 4. 写入环形缓冲
    {
        std::lock_guard<std::mutex> lk(m_mtxPicture);
        BitmapPicture& picbuf =
            m_picture[(m_curPicture + m_usedPicture) & (MAX_BUFFER_COUNT - 1)];
        picbuf.Clear();                    // 释放旧数据
        picbuf.width = width;
        picbuf.height = height;
        for (int i = 0; i < 3; ++i) {
            picbuf.yuv[i] = yuvdata[i];    // 移交 YUV 指针所有权
        }
        picbuf.pts = pic.pts / DVD_TIME_BASE;  // 转换为秒
        ++m_usedPicture;                   // 已用帧数 +1
        return MAX_BUFFER_COUNT - m_usedPicture;  // 返回剩余空位数
    }
}
```

### Layer 模式（YUV→RGBA 转换）

Layer 模式下，`VideoPresentLayer::AddVideoPicture()` 使用 `sws_scale` 将 YUV420P 转换为 RGBA 格式。来源文件：`KRMovieLayer.cpp` 第 69-109 行：

```cpp
// KRMovieLayer.cpp 第 69-109 行
int VideoPresentLayer::AddVideoPicture(DVDVideoPicture& pic, int index) {
    if (pic.format != RENDER_FMT_YUV420P) return -2;
    if (pic.pts == DVD_NOPTS_VALUE) return 0;

    // 等待空闲槽位（与 Overlay 相同）
    if (m_usedPicture >= MAX_BUFFER_COUNT) {
        std::unique_lock<std::mutex> lk(m_mtxPicture);
        m_condPicture.wait(lk);
    }
    if (m_usedPicture >= MAX_BUFFER_COUNT) return -1;

    int width = pic.iWidth, height = pic.iHeight;

    // 1. 分配 RGBA 缓冲（每像素 4 字节）
    uint8_t* data = (uint8_t*)TJSAlignedAlloc(width * height * 4, 4);
    int datasize = width * 4;  // RGBA 一行的字节数

    // 2. 创建/复用 SwsContext 进行像素格式转换
    img_convert_ctx = sws_getCachedContext(
        img_convert_ctx,
        width, height, AV_PIX_FMT_YUV420P,  // 源格式
        width, height, AV_PIX_FMT_RGBA,      // 目标格式
        SWS_FAST_BILINEAR,  // 缩放算法（此处无缩放，仅做格式转换）
        nullptr, nullptr, nullptr);

    // 3. 执行转换：YUV420P → RGBA
    sws_scale(img_convert_ctx,
              pic.data, pic.iLineSize,  // 源 YUV 数据
              0, pic.iHeight,           // 从第 0 行开始，处理全部行
              &data, &datasize);        // 目标 RGBA 数据

    // 4. 写入环形缓冲
    {
        std::lock_guard<std::mutex> lk(m_mtxPicture);
        BitmapPicture& picbuf =
            m_picture[(m_curPicture + m_usedPicture) & (MAX_BUFFER_COUNT - 1)];
        picbuf.Clear();
        picbuf.width = width;
        picbuf.height = height;
        picbuf.data[0] = data;   // RGBA 数据只用一个指针
        picbuf.pts = pic.pts / DVD_TIME_BASE;
        ++m_usedPicture;
    }
    return MAX_BUFFER_COUNT - m_usedPicture;
}
```

### 两种模式对比

| 特性 | Overlay 模式 | Layer 模式 |
|------|-------------|-----------|
| 存储格式 | YUV420P（3 个平面） | RGBA（1 个连续缓冲） |
| 内存占用（1920×1080） | 1920×1080×1.5 ≈ 3.1 MB | 1920×1080×4 ≈ 8.3 MB |
| 格式转换 | 无（GPU 上的 YUV Shader 处理） | CPU 上 sws_scale 转换 |
| CPU 开销 | 低（仅拷贝） | 高（色彩空间转换） |
| 渲染方式 | TVPYUVSprite（Cocos2d 节点） | tTVPBaseTexture（引擎纹理） |
| 适用场景 | 全屏/窗口视频覆盖 | 视频作为图层与其他元素混合 |

## WaitForBuffer 阻塞等待机制

`TVPMoviePlayer::WaitForBuffer()` 是解码线程在缓冲区满时的等待接口。来源文件：`KRMoviePlayer.cpp` 第 148-159 行：

```cpp
int TVPMoviePlayer::WaitForBuffer(volatile std::atomic_bool& bStop, int timeout) {
    // 1. 快速路径：有空闲槽位直接返回
    int remainBuf = MAX_BUFFER_COUNT - m_usedPicture;
    if (remainBuf > 0) return remainBuf;

    // 2. 慢速路径：循环等待
    std::unique_lock<std::mutex> lk(m_mtxPicture);
    while (!bStop && MAX_BUFFER_COUNT <= m_usedPicture && timeout > 0) {
        timeout -= 10;
        // 每次等待 10ms，检查条件
        m_condPicture.wait_for(lk, std::chrono::milliseconds(10));
    }
    // 返回值 > 0 表示有空位，≤ 0 表示超时或停止
    return MAX_BUFFER_COUNT - m_usedPicture - 1;
}
```

**调用时序**：
1. `CVideoPlayerVideo::OutputPicture()` 调用 `WaitForBuffer(m_bStop, maxWaitTime)`
2. 如果返回 ≤ 0，帧被丢弃（超时丢帧）
3. 如果返回 > 0，调用 `AddVideoPicture()` 入队

### 实验 2：环形缓冲区模拟

```cpp
#include <cstdio>
#include <mutex>
#include <condition_variable>
#include <thread>
#include <chrono>

#define MAX_BUFFER_COUNT 4

struct SimplePicture {
    int frameId = -1;
    double pts = 0.0;
};

class RingBuffer {
    SimplePicture m_picture[MAX_BUFFER_COUNT];
    int m_curPicture = 0;    // 读位置
    int m_usedPicture = 0;   // 已用数量
    std::mutex m_mtx;
    std::condition_variable m_cond;
    bool m_stop = false;

public:
    // 生产者：写入帧（解码线程调用）
    int Push(int frameId, double pts) {
        if (m_usedPicture >= MAX_BUFFER_COUNT) {
            std::unique_lock<std::mutex> lk(m_mtx);
            m_cond.wait_for(lk, std::chrono::milliseconds(100));
        }
        if (m_usedPicture >= MAX_BUFFER_COUNT) return -1;

        {
            std::lock_guard<std::mutex> lk(m_mtx);
            int writePos = (m_curPicture + m_usedPicture) & (MAX_BUFFER_COUNT - 1);
            m_picture[writePos].frameId = frameId;
            m_picture[writePos].pts = pts;
            ++m_usedPicture;
            printf("[生产] 帧 %d 写入槽位 %d, 已用=%d\n",
                   frameId, writePos, m_usedPicture);
        }
        return MAX_BUFFER_COUNT - m_usedPicture;
    }

    // 消费者：读取帧（渲染线程调用）
    bool Pop(SimplePicture& out) {
        if (m_usedPicture <= 0) return false;
        {
            std::lock_guard<std::mutex> lk(m_mtx);
            out = m_picture[m_curPicture];
            m_picture[m_curPicture].frameId = -1;
            m_curPicture = (m_curPicture + 1) & (MAX_BUFFER_COUNT - 1);
            --m_usedPicture;
            m_cond.notify_all();  // 通知生产者有空位
            printf("[消费] 帧 %d 从槽位读取, 已用=%d\n",
                   out.frameId, m_usedPicture);
        }
        return true;
    }

    void Stop() { m_stop = true; m_cond.notify_all(); }
};

int main() {
    RingBuffer buf;

    // 生产者线程：模拟解码器以 30fps 产出
    std::thread producer([&buf]() {
        for (int i = 0; i < 12; i++) {
            int remain = buf.Push(i, i / 30.0);
            if (remain < 0) printf("[生产] 缓冲满，帧 %d 被丢弃\n", i);
            std::this_thread::sleep_for(std::chrono::milliseconds(33));
        }
    });

    // 消费者线程：模拟渲染器以 24fps 消费（故意慢于生产）
    std::thread consumer([&buf]() {
        std::this_thread::sleep_for(std::chrono::milliseconds(50));  // 延迟启动
        for (int i = 0; i < 12; i++) {
            SimplePicture pic;
            if (buf.Pop(pic)) {
                printf("[渲染] 显示帧 %d, PTS=%.3f\n", pic.frameId, pic.pts);
            }
            std::this_thread::sleep_for(std::chrono::milliseconds(42));
        }
    });

    producer.join();
    consumer.join();
    return 0;
}
```

## 常见错误与解决方案

### 错误 1：忽略 lineSize 导致图像错位

**现象**：视频画面出现对角线方向的条纹或错位，颜色混乱。

**原因**：直接按 `width * height` 做 `memcpy`，而实际 `lineSize > width`。

```cpp
// ❌ 错误写法
memcpy(dst, src_y, width * height);

// ✅ 正确写法
for (int y = 0; y < height; y++) {
    memcpy(dst + y * dstStride, src_y + y * srcLineSize, width);
}
```

### 错误 2：NV12 格式混淆 UV 顺序

**现象**：视频色彩完全偏蓝或偏红。

**原因**：NV12 的 UV 交错顺序是 U0V0U1V1...，如果写成 V0U0V1U1... 则是 NV21 格式。

```cpp
// ❌ 错误：NV21 顺序
*d_uv++ = *s_v++;  // V 在前
*d_uv++ = *s_u++;  // U 在后

// ✅ 正确：NV12 顺序
*d_uv++ = *s_u++;  // U 在前
*d_uv++ = *s_v++;  // V 在后
```

### 错误 3：sws_scale 上下文泄漏

**现象**：视频播放过程中内存持续增长。

**原因**：每帧创建新的 `SwsContext` 但未释放。

```cpp
// ❌ 错误：每帧创建新上下文
struct SwsContext* ctx = sws_getContext(...);
sws_scale(ctx, ...);
// 忘记 sws_freeContext(ctx)

// ✅ 正确：使用 sws_getCachedContext 复用上下文
// KrKr2 的做法（KRMovieLayer.cpp 第 88-91 行）
img_convert_ctx = sws_getCachedContext(
    img_convert_ctx,  // 传入旧上下文，参数不变时复用
    width, height, AV_PIX_FMT_YUV420P,
    width, height, AV_PIX_FMT_RGBA,
    SWS_FAST_BILINEAR, nullptr, nullptr, nullptr);
```

| **Linux** | `posix_memalign(&ptr, 4, size)` 或 `TJSAlignedAlloc` | sws_scale（CPU） | OpenGL `glTexSubImage2D()` |
| **macOS** | `posix_memalign(&ptr, 4, size)` 或 `TJSAlignedAlloc` | sws_scale（CPU） | Metal 或 OpenGL 纹理更新 |
| **Android** | `memalign(4, size)` 或 `TJSAlignedAlloc` | sws_scale（CPU），ARM NEON 可加速 | OpenGL ES `glTexSubImage2D()` |

KrKr2 使用 `TJSAlignedAlloc(size, 4)` 统一了所有平台的对齐分配，内部根据平台选择合适的系统调用。

## 本节小结

- `DVDVideoPicture` 是解码器到渲染器的数据桥梁，union 设计支持多种硬件后端
- KrKr2 实际只使用 `RENDER_FMT_YUV420P` 软件解码路径
- `lineSize` 与 `width` 的区别是帧拷贝中最常见的陷阱
- Overlay 模式直通 YUV 数据，Layer 模式用 `sws_scale` 转为 RGBA
- `BitmapPicture` 环形缓冲（4 槽）通过 mutex + condition_variable 实现线程安全的生产者-消费者模式
- `CDVDCodecUtils` 提供帧分配、拷贝、格式转换的标准化工具函数

### 题目 1：计算 YUV420P 帧的内存占用

给定一个 1280×720 的视频帧，使用 YUV420P 格式且 lineSize 对齐到 32 字节，请计算：
1. Y/U/V 三个平面各自的 lineSize
2. 三个平面各自的总字节数
3. 整帧的总内存占用

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>

int main() {
    int width = 1280, height = 720;

    // Y 平面 lineSize：1280 已经是 32 的倍数
    int yLineSize = (width + 31) & ~31;  // = 1280
    int yTotal = yLineSize * height;     // = 1280 * 720 = 921600

    // U/V 平面：宽高各减半
    int cw = width / 2;   // = 640
    int ch = height / 2;  // = 360
    int uvLineSize = (cw + 31) & ~31;  // = 640（已对齐）
    int uTotal = uvLineSize * ch;      // = 640 * 360 = 230400
    int vTotal = uvLineSize * ch;      // = 640 * 360 = 230400

    int total = yTotal + uTotal + vTotal;

    printf("Y  lineSize=%d, 总大小=%d 字节\n", yLineSize, yTotal);
    printf("U  lineSize=%d, 总大小=%d 字节\n", uvLineSize, uTotal);
    printf("V  lineSize=%d, 总大小=%d 字节\n", uvLineSize, vTotal);
    printf("总内存占用=%d 字节 (%.2f MB)\n", total, total / 1048576.0);
    // 输出：
    // Y  lineSize=1280, 总大小=921600 字节
    // U  lineSize=640, 总大小=230400 字节
    // V  lineSize=640, 总大小=230400 字节
    // 总内存占用=1382400 字节 (1.32 MB)
    return 0;
}
```

对比 RGBA 格式：1280 × 720 × 4 = 3,686,400 字节（3.52 MB），YUV420P 节省约 62.5% 的内存。

</details>

### 题目 2：为什么 AddVideoPicture 中 pts 要除以 DVD_TIME_BASE？

在 `KRMoviePlayer.cpp` 和 `KRMovieLayer.cpp` 中，`AddVideoPicture()` 都将 `pic.pts / DVD_TIME_BASE` 存入 `BitmapPicture::pts`。请解释这个转换的意义。

<details>
<summary>查看答案</summary>

`DVD_TIME_BASE` 的值定义在 `Clock.h` 中：

```cpp
#define DVD_TIME_BASE 1000000.0  // 微秒级精度的时间基准
```

在 KrKr2 的视频播放管线中，时间戳存在两种单位：

1. **内部时间戳**（`DVDVideoPicture::pts`）：以 `DVD_TIME_BASE` 为单位，即微秒级整数。例如 PTS=1500000 表示 1.5 秒。这是从 Demuxer 的 `ConvertTimestamp()` 转换而来的统一内部时间。

2. **显示时间戳**（`BitmapPicture::pts`）：以秒为单位的浮点数。例如 PTS=1.5 表示 1.5 秒。这是给渲染器消费的，因为渲染器用 `dt`（帧间隔，秒）做计时比较。

除以 `DVD_TIME_BASE` 就是将微秒级内部时间转换为秒级显示时间。在 Overlay 模式的 `PresentPicture()` 中可以看到 `m_curpts += dt`（dt 是秒），然后与 `picbuf.pts` 比较，因此两者必须在同一单位下。

</details>

### 题目 3：实现一个带超时的帧等待函数

参考 `TVPMoviePlayer::WaitForBuffer()` 的实现，编写一个泛化的带超时等待函数，在条件满足或超时后返回。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>
#include <mutex>
#include <condition_variable>
#include <chrono>
#include <atomic>
#include <thread>
#include <functional>

// 泛化的带超时条件等待
// 返回值：true=条件满足，false=超时或停止
bool WaitWithTimeout(std::mutex& mtx,
                     std::condition_variable& cond,
                     std::atomic_bool& stop,
                     int timeoutMs,
                     std::function<bool()> predicate) {
    // 快速路径：条件已满足
    if (predicate()) return true;

    std::unique_lock<std::mutex> lk(mtx);
    // 使用 wait_for 的谓词版本，自动处理虚假唤醒
    bool result = cond.wait_for(lk,
        std::chrono::milliseconds(timeoutMs),
        [&]() { return stop.load() || predicate(); });

    return result && !stop.load();
}

// 使用示例
int main() {
    std::mutex mtx;
    std::condition_variable cond;
    std::atomic_bool stop{false};
    int counter = 0;

    // 后台线程：2 秒后设置条件
    std::thread bg([&]() {
        std::this_thread::sleep_for(std::chrono::seconds(2));
        {
            std::lock_guard<std::mutex> lk(mtx);
            counter = 5;
        }
        cond.notify_all();
    });

    printf("等待 counter > 0（超时 3 秒）...\n");
    bool ok = WaitWithTimeout(mtx, cond, stop, 3000,
                              [&]() { return counter > 0; });
    printf("结果: %s, counter=%d\n", ok ? "条件满足" : "超时", counter);

    bg.join();
    return 0;
}
// 输出：
// 等待 counter > 0（超时 3 秒）...
// 结果: 条件满足, counter=5
```

</details>
