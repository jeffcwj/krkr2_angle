## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| Variant | 变体类型 | TJS2 中所有值的统一容器——一个变量可以在不同时刻存储整数、字符串或对象，Variant 就是这个"万能盒子" |
| void 类型 | Void Type | TJS2 中表示"空/未定义"的类型，相当于 JavaScript 的 `undefined`，在 C++ 层枚举值为 `tvtVoid` |
| Octet | 八进制打包字符串 | TJS2 特有的二进制数据类型，用 `<% %>` 语法书写，存储原始字节序列——常用于图像/音频数据的嵌入 |
| 引用计数 | Reference Counting | 一种内存管理策略：每个对象维护一个"被引用次数"的计数器，引用增加时 +1，减少时 -1，降到 0 时自动释放内存 |
| 函数作用域 | Function Scope | 变量的可见范围以函数为边界——在函数内任何位置用 `var` 声明的变量，在整个函数内都可访问（类似 JavaScript 的 `var` 行为） |
| 闭包 | Closure | 函数"捕获"其定义时所在作用域的变量的能力——即使外层函数已返回，内层函数仍可访问那些变量 |
| 标签联合 | Tagged Union | C/C++ 中一种数据结构设计：用一个枚举"标签"标记当前 union 中存储的是哪种类型的数据 |

## 一、TJS2 的六种基本类型

### 1.1 类型总览

TJS2 是一门**动态类型**（Dynamic Typing）语言，这意味着变量本身没有固定类型，类型信息附着在**值**上。在 C++ 引擎内部，所有 TJS2 值都被统一存储在一个叫做 `tTJSVariant` 的类中——这个名字的含义是"变体"，因为它可以在不同时刻承载不同类型的数据。

`tTJSVariant` 支持以下六种类型，每种对应一个枚举值：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 103-110 行
enum tTJSVariantType {
    tvtVoid,     // 空值——对应 TJS2 的 void
    tvtObject,   // 对象——函数、类实例、数组、字典都是对象
    tvtString,   // 字符串——UTF-16 编码的 Unicode 字符串
    tvtOctet,    // 八进制打包——原始二进制字节序列
    tvtInteger,  // 整数——64 位有符号整数（tjs_int64）
    tvtReal      // 实数——64 位 IEEE 754 双精度浮点数（double）
};
```

注意：TJS2 **没有**布尔类型（Boolean）。条件判断使用整数 `0`（假）和非零值（真），这一点与 C 语言相同。

### 1.2 在 TJS2 脚本中体验六种类型

```javascript
// TJS2 脚本示例：六种基本类型的声明与使用

// 1. void — 未赋值或显式赋 void
var a;                    // a 的类型是 void
var b = void;             // 显式赋值为 void
// typeof(a) 返回 "Void"

// 2. Integer — 64位有符号整数
var count = 42;           // 十进制整数
var hex = 0xFF;           // 十六进制（255）
var negative = -100;      // 负数
// typeof(count) 返回 "Integer"

// 3. Real — 64位双精度浮点数
var pi = 3.14159265;      // 浮点数
var sci = 1.5e10;         // 科学记数法（1.5 × 10^10）
// typeof(pi) 返回 "Real"

// 4. String — UTF-16 Unicode 字符串
var greeting = "こんにちは";  // 日文字符串，原生支持
var name = "KiriKiri";        // ASCII 字符串
var interp = "値は%{count}です"; // 字符串插值（TJS2 特有！）
// typeof(greeting) 返回 "String"

// 5. Octet — 二进制数据
var data = <% 48 65 6C 6C 6F %>;  // "Hello" 的 ASCII 字节
var empty_oct = <% %>;             // 空的八进制打包
// typeof(data) 返回 "Octet"

// 6. Object — 对象（函数、类实例、数组、字典等）
var arr = [];                // 空数组（Array 对象）
var dict = %[];              // 空字典（Dictionary 对象）
var func = function() { return 1; };  // 函数对象
// typeof(arr) 返回 "Object"
```

### 1.3 类型检测

TJS2 提供两种方式检测值的类型：

```javascript
// 方式一：typeof 运算符 — 返回类型名称的字符串
var x = 42;
var t = typeof(x);        // t == "Integer"

var y = "hello";
var t2 = typeof(y);       // t2 == "String"

var z;
var t3 = typeof(z);       // t3 == "Void"

// 方式二：instanceof 运算符 — 检查对象是否为某个类的实例
class Animal {
    var name;
    function Animal(n) { name = n; }
}

class Dog extends Animal {
    function Dog(n) { super.Animal(n); }
}

var dog = new Dog("Rex");
var isAnimal = dog instanceof "Animal";  // true
var isDog = dog instanceof "Dog";        // true
var isString = dog instanceof "String";  // false
// 注意：instanceof 的参数是类名的字符串，不是类引用
```

**与 JavaScript 的关键差异**：

| 特性 | JavaScript | TJS2 |
|------|-----------|------|
| typeof 返回值 | `"number"`、`"string"` | `"Integer"`、`"Real"`、`"String"` |
| null 类型 | `typeof null === "object"`（历史遗留 bug） | TJS2 没有 null，用 `void` 代替 |
| undefined | `typeof undefined === "undefined"` | `typeof void === "Void"` |
| instanceof | `obj instanceof ClassName` | `obj instanceof "ClassName"`（用字符串） |
| 布尔类型 | 有 `true`/`false` | 无，用整数 0/非0 代替 |

## 二、C++ 层的 Variant 实现

### 2.1 标签联合（Tagged Union）设计

理解 TJS2 类型系统的关键在于看懂 `tTJSVariant` 的 C++ 实现。它采用了经典的**标签联合**（Tagged Union）设计模式：用一个 `union` 存储不同类型的数据，用一个枚举 `vt`（variant type）标记当前存储的是哪种类型。

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 136-153 行
// 精简后的核心结构

struct tTJSVariant_S {
    // 数据存储——同一时刻只有一个成员有效
    union {
        tTJSVariantClosure_S Object;  // 对象：包含 Object 指针和 ObjThis 指针
        tTVInteger Integer;           // 整数：tjs_int64（64位有符号整数）
        tTVReal Real;                 // 实数：double（64位双精度浮点）
        tTJSVariantString *String;    // 字符串：指向引用计数字符串的指针
        tTJSVariantOctet *Octet;      // 八进制打包：指向引用计数字节数组的指针
    };
    tTJSVariantType vt;               // 标签：标记上面的 union 当前存储的是哪种类型
};
```

为什么用标签联合而不是 C++ 的 `std::variant`？因为 TJS2 诞生于 2000 年，那时 C++ 标准库还没有 `std::variant`（C++17 才加入）。即使在今天，手写的标签联合在性能上也有优势——没有异常安全开销，内存布局更紧凑。

### 2.2 每种类型的内存布局

让我们逐一分析 union 中每个成员的内存布局：

```
tTJSVariant 内存布局（简化视图）：
┌──────────────────────────────────┬───────────┐
│           union (16 bytes)       │ vt (4B)   │
├──────────────────────────────────┼───────────┤
│ tvtVoid:    [   未使用（16B）  ] │ tvtVoid   │
│ tvtInteger: [ tjs_int64 (8B) |pad] │ tvtInteger│
│ tvtReal:    [ double    (8B) |pad] │ tvtReal   │
│ tvtString:  [ String*   (8B) |pad] │ tvtString │
│ tvtOctet:   [ Octet*    (8B) |pad] │ tvtOctet  │
│ tvtObject:  [ Object*(8B)|ObjThis*(8B)] │ tvtObject │
└──────────────────────────────────┴───────────┘
总大小 = 20 字节（#pragma pack(push, 4)）
```

**关键细节**：

1. **Object 类型最大**：它需要存储两个指针（`Object` 和 `ObjThis`），占 16 字节，这决定了 union 的大小
2. **Integer 和 Real 只用 8 字节**：64 位整数和 64 位浮点数各占 8 字节，剩余 8 字节是 padding
3. **String 和 Octet 是指针**：它们只存储一个指向堆上对象的 8 字节指针，实际数据在堆上
4. **`#pragma pack(push, 4)` 强制 4 字节对齐**：这使得整个结构体更紧凑

### 2.3 为什么 Object 需要两个指针？

`tTJSVariantClosure_S` 结构体包含两个成员：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 125-128 行
struct tTJSVariantClosure_S {
    iTJSDispatch2 *Object;    // 函数/方法的实际对象
    iTJSDispatch2 *ObjThis;   // 调用时的 this 上下文
};
```

这个设计是为了支持 TJS2 的**闭包**（Closure）机制。当你把一个对象的方法保存为变量时，需要同时记住：

1. **Object**：方法本身（可调用的函数对象）
2. **ObjThis**：方法的 `this` 绑定（方法属于哪个对象）

```javascript
// TJS2 示例：理解 Object 和 ObjThis 的区别
class Player {
    var hp = 100;
    
    function heal() {
        // 这里的 this 就是 ObjThis
        this.hp += 10;
    }
}

var player = new Player();
var healFunc = player.heal;  
// healFunc 内部：
//   Object  = heal 函数对象
//   ObjThis = player 实例

healFunc();  // 调用时，自动将 player 作为 this 传入
// 等价于 player.heal()

// 如果不保存 ObjThis，调用 healFunc() 时就不知道
// this 应该指向谁——这就是为什么需要两个指针
```

## 三、类型转换规则

### 3.1 隐式类型转换

TJS2 在运算和赋值时会自动进行类型转换。理解这些转换规则对于调试至关重要。

```javascript
// 示例 1：算术运算中的类型提升
var a = 10;         // Integer
var b = 3.5;        // Real
var c = a + b;      // c = 13.5 (Real) — Integer 被提升为 Real

// 示例 2：字符串与数字的加法
var s = "100";      // String
var n = 42;         // Integer
var r1 = s + n;     // r1 = "10042" (String) — 数字被转为字符串拼接
var r2 = +s;        // r2 = 100 (Integer) — 一元 + 将字符串转为数字
var r3 = s * 2;     // r3 = 200 (Integer) — 乘法强制字符串转数字

// 示例 3：void 的转换
var v;              // void
var r4 = v + 1;     // r4 = 1 — void 在算术中被当作 0
var r5 = v + "hi";  // r5 = "hi" — void 在字符串拼接中被当作空字符串

// 示例 4：布尔上下文（条件判断）
if (0) { /* 不执行 */ }        // 0 → false
if (1) { /* 执行 */ }          // 非零整数 → true
if ("") { /* 不执行 */ }       // 空字符串 → 0 → false
if ("hello") { /* 取决于内容 */ } // 字符串先转整数再判断
if (void) { /* 不执行 */ }     // void → false
```

### 3.2 C++ 层的类型转换实现

在 C++ 引擎内部，类型转换由 `tTJSVariant` 的 `AsInteger()`、`AsReal()`、`AsString()` 等方法实现。让我们看看 `AsInteger()` 的完整实现——它展示了从其他类型到整数的转换规则：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 862-879 行
// 将当前 Variant 转换为整数并返回

tTVInteger AsInteger() const {
    switch(vt) {
        case tvtVoid:
            return 0;                          // void → 0
        case tvtObject:
            TJSThrowVariantConvertError(*this, tvtInteger);  // 对象不能转整数，抛异常
        case tvtString:
            return String->ToInteger();        // 字符串 → 尝试解析为整数
        case tvtInteger:
            return Integer;                    // 整数 → 直接返回
        case tvtReal:
            TJSSetFPUE();                      // 设置浮点异常标志
            return (tTVInteger)Real;           // 实数 → 截断为整数
        case tvtOctet:
            TJSThrowVariantConvertError(*this, tvtInteger);  // 八进制打包不能转整数
    }
    return 0;
}
```

同样，`AsReal()` 的转换规则：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 937-955 行
tTVReal AsReal() const {
    TJSSetFPUE();  // 设置浮点单元异常标志

    switch(vt) {
        case tvtVoid:
            return 0;                          // void → 0.0
        case tvtObject:
            TJSThrowVariantConvertError(*this, tvtReal);  // 对象不能转实数
        case tvtString:
            return String->ToReal();           // 字符串 → 尝试解析为浮点数
        case tvtInteger:
            return (tTVReal)Integer;           // 整数 → 提升为 double
        case tvtReal:
            return Real;                       // 实数 → 直接返回
        case tvtOctet:
            TJSThrowVariantConvertError(*this, tvtReal);  // 八进制打包不能转实数
    }
    return 0.0f;
}
```

而 `AsString()` 可以将几乎所有类型转为字符串：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 755-775 行
tTJSVariantString* AsString() const {
    switch(vt) {
        case tvtVoid:
            return nullptr;                                    // void → 空字符串
        case tvtObject:
            return TJSObjectToString(*(tTJSVariantClosure*)&Object); // 对象 → 调用 toString
        case tvtString:
            if(String) String->AddRef();                       // 字符串 → 增加引用计数后返回
            return String;
        case tvtInteger:
            return TJSIntegerToString(Integer);                // 整数 → 格式化为十进制字符串
        case tvtReal:
            return TJSRealToString(Real);                      // 实数 → 格式化为浮点字符串
        case tvtOctet:
            TJSThrowVariantConvertError(*this, tvtString);     // 八进制打包不能转字符串！
    }
    return nullptr;
}
```

### 3.3 完整的类型转换规则表

下面这张表总结了所有类型之间的转换行为：

| 源类型 → 目标类型 | Integer | Real | String | Object | Octet | Bool 上下文 |
|-------------------|---------|------|--------|--------|-------|------------|
| **void** | 0 | 0.0 | `""` (空) | ❌ 异常 | ❌ 异常 | false |
| **Integer** | 原值 | (double)值 | 十进制文本 | ❌ 异常 | ❌ 异常 | 0→false,其余→true |
| **Real** | (int64)截断 | 原值 | 浮点文本 | ❌ 异常 | ❌ 异常 | 0.0→false,其余→true |
| **String** | 解析为整数 | 解析为浮点 | 原值 | ❌ 异常 | ❌ 异常 | 转整数后判断 |
| **Object** | ❌ 异常 | ❌ 异常 | toString() | 原值 | ❌ 异常 | 非null→true |
| **Octet** | ❌ 异常 | ❌ 异常 | ❌ 异常 | ❌ 异常 | 原值 | 非null→true |

**注意**：Octet 类型非常"孤立"——它几乎不能与其他类型相互转换，这是有意的设计。Octet 表示原始二进制数据（如图片字节、音频采样），将二进制数据"转换"为整数或字符串在语义上没有意义，强制转换只会导致隐蔽的 bug。

### 3.4 比较运算的两种模式

TJS2 提供两种比较方式——**普通比较**和**严格比较**，这与 JavaScript 的 `==` 和 `===` 类似：

```javascript
// TJS2 中的两种比较

// 普通比较（==）— 允许类型转换后比较
var a = 42;
var b = "42";
var r1 = (a == b);     // true — 字符串 "42" 被转为整数 42

// 严格比较（===）— 必须类型和值都相同
var r2 = (a === b);    // false — Integer 和 String 类型不同

// void 的比较行为
var v;
var r3 = (v == 0);     // true — void 在比较中被当作 0
var r4 = (v === 0);    // false — void 类型 ≠ Integer 类型
var r5 = (v == void);  // true — 两个 void 相等
var r6 = (v === void); // true — 类型也相同

// 对象的比较
var arr1 = [1, 2, 3];
var arr2 = [1, 2, 3];
var arr3 = arr1;
var r7 = (arr1 == arr2);  // false — 不同的对象实例
var r8 = (arr1 == arr3);  // true — 同一个对象引用
```

在 C++ 层，这两种比较分别对应 `NormalCompare()` 和 `DiscernCompare()` 方法：

```cpp
// NormalCompare —— 普通比较（允许类型转换）
// 如果两个 variant 的"值"相等则返回 true
// 例如 Integer(42) 和 String("42") → true

// DiscernCompare —— 严格比较（类型+值都必须相同）
// 只有类型和值都完全一致才返回 true
// 例如 Integer(42) 和 String("42") → false
```

## 四、变量声明与作用域

### 4.1 var 声明

TJS2 使用 `var` 关键字声明变量，这是唯一的变量声明方式：

```javascript
// 基本声明与赋值
var x = 10;           // 声明并初始化
var y;                // 声明但不初始化，值为 void
var a = 1, b = 2;    // 同时声明多个变量

// 赋值后类型可以改变（动态类型）
var data = 42;        // 此时 data 是 Integer
data = "hello";       // 现在 data 变成了 String
data = [1, 2, 3];    // 现在 data 变成了 Object（数组）
data = void;          // 现在 data 变成了 void

// 类型改变在 C++ 层的实现：
// 每次赋值，tTJSVariant::operator= 会：
// 1. 释放旧值的引用（ReleaseContent）
// 2. 复制新值（CopyRef）
// 3. 更新类型标签 vt
```

### 4.2 函数作用域

TJS2 的变量作用域是**函数作用域**（Function Scope），而不是块作用域。这意味着在 `if`、`for`、`while` 等块内声明的变量，在整个函数内都可以访问：

```javascript
// 函数作用域演示
function testScope() {
    // x 还没声明，但不会报错（变量提升）
    
    if (true) {
        var x = 42;  // 在 if 块内声明
    }
    
    // x 在 if 块外仍然可以访问！
    // 因为 var 的作用域是整个函数，不是 if 块
    System.inform(x);  // 显示 42
    
    for (var i = 0; i < 3; i++) {
        var loopVar = i * 10;
    }
    
    // i 和 loopVar 在 for 循环外仍然可以访问
    System.inform(i);        // 显示 3
    System.inform(loopVar);  // 显示 20（最后一次循环的值）
}
```

**与 JavaScript 的对比**：

| 特性 | JavaScript `var` | JavaScript `let`/`const` | TJS2 `var` |
|------|-----------------|------------------------|-----------|
| 作用域 | 函数作用域 | 块作用域 | 函数作用域 |
| 变量提升 | 有（初始值 undefined） | 有暂时性死区(TDZ) | 有（初始值 void） |
| 重复声明 | 允许 | 禁止 | 允许 |

### 4.3 全局变量与局部变量

```javascript
// 全局作用域
var globalVar = "我是全局变量";

function modifyGlobal() {
    // 访问全局变量
    globalVar = "被修改了";
    
    // 声明局部变量——不影响外部
    var localVar = "我是局部变量";
}

modifyGlobal();
// globalVar 现在是 "被修改了"
// localVar 在此处不可访问

// 使用 global 关键字显式引用全局对象
function useGlobal() {
    var x = 10;  // 局部 x
    
    // 如果同名的全局变量和局部变量冲突
    // 用 global.xxx 显式访问全局变量
    global.x = 100;  // 设置全局 x 为 100
    
    System.inform(x);         // 10（局部）
    System.inform(global.x);  // 100（全局）
}
```

### 4.4 变量提升（Hoisting）

与 JavaScript 的 `var` 一样，TJS2 的变量声明会被"提升"到函数顶部：

```javascript
// 变量提升示例
function hoistingDemo() {
    System.inform(typeof(x));  // "Void" — x 已被提升但未赋值
    
    // 此处 x 已经存在（被提升），但值为 void
    
    var x = 42;
    System.inform(typeof(x));  // "Integer" — 现在 x 有值了
}

// 上面的代码等价于：
function hoistingDemoExpanded() {
    var x;  // ← 声明被提升到函数开头
    
    System.inform(typeof(x));  // "Void"
    
    x = 42;  // ← 赋值留在原位
    System.inform(typeof(x));  // "Integer"
}
```

## 五、闭包与变量捕获

### 5.1 闭包的基本概念

**闭包**（Closure）是函数"记住"它被创建时的环境的能力。在 TJS2 中，内层函数可以访问外层函数的局部变量，即使外层函数已经返回：

```javascript
// 闭包基础示例
function makeCounter() {
    var count = 0;  // 这个变量会被闭包捕获
    
    return function() {
        count++;
        return count;
    };
}

var counter = makeCounter();
// makeCounter() 已经返回了，但 count 变量不会被销毁
// 因为返回的匿名函数"捕获"了它

System.inform(counter());  // 1
System.inform(counter());  // 2
System.inform(counter());  // 3
// 每次调用都在修改同一个 count 变量
```

### 5.2 闭包在 C++ 层的实现

在引擎内部，闭包是通过 `tTJSVariantClosure`（变体闭包）结构实现的：

```cpp
// 源码位置：cpp/core/tjs2/tjsVariant.h 第 125-128 行
struct tTJSVariantClosure_S {
    iTJSDispatch2 *Object;    // 函数对象本身
    iTJSDispatch2 *ObjThis;   // 闭包绑定的 this 上下文
};
```

当 TJS2 创建一个闭包时，编译器会：

1. 将函数体编译为一个 `tTJSInterCodeContext` 对象（字节码上下文）
2. 将该对象包装为 `tTJSVariant`（类型为 `tvtObject`）
3. 在 `tTJSVariant` 的 `Object` 联合成员中存储：
   - `Object` 指针 → 指向函数对象（包含编译后的字节码）
   - `ObjThis` 指针 → 指向捕获的变量环境（外层作用域）

```
闭包在内存中的结构：
┌─────────────────────────────┐
│ tTJSVariant (闭包变量)       │
│ ┌─────────────────────────┐ │
│ │ Object ──→ 函数字节码    │ │    ┌──────────────────┐
│ │ ObjThis ──→ 外层作用域 ──┼──┼──→ │ count: 0 (初始)  │
│ └─────────────────────────┘ │    │ ...其他捕获变量   │
│ vt = tvtObject              │    └──────────────────┘
└─────────────────────────────┘
```

### 5.3 闭包的常见用法

```javascript
// 用法 1：事件回调
class Button {
    var onClick;
    var label;
    
    function Button(lbl) {
        label = lbl;
    }
    
    function click() {
        if (onClick !== void) onClick();
    }
}

function setupUI() {
    var message = "按钮被点击了！";  // 会被闭包捕获
    var btn = new Button("确定");
    
    btn.onClick = function() {
        // 这个匿名函数捕获了外层的 message 变量
        System.inform(message);
    };
    
    return btn;
}

var button = setupUI();
button.click();  // 显示 "按钮被点击了！"

// 用法 2：数据隐藏（模拟私有变量）
function createPlayer(initialHP) {
    var hp = initialHP;    // "私有"变量
    var maxHP = initialHP;
    
    var player = %[];  // 创建空字典作为"对象"
    
    player.getHP = function() {
        return hp;
    };
    
    player.heal = function(amount) {
        hp += amount;
        if (hp > maxHP) hp = maxHP;
    };
    
    player.damage = function(amount) {
        hp -= amount;
        if (hp < 0) hp = 0;
    };
    
    return player;
}

var hero = createPlayer(100);
hero.damage(30);
System.inform(hero.getHP());  // 70
hero.heal(50);
System.inform(hero.getHP());  // 100（不超过 maxHP）
// 外部无法直接访问 hp 和 maxHP
```

## 本节小结

- TJS2 有六种基本类型：void、Integer、Real、String、Octet、Object
- 所有类型在 C++ 层统一存储在 `tTJSVariant` 的标签联合中
- 类型转换遵循明确的规则，Octet 类型几乎不参与转换
- 变量使用 `var` 声明，遵循函数作用域（不是块作用域）
- 闭包通过 `tTJSVariantClosure` 结构实现，同时保存函数对象和 this 上下文
- TJS2 没有布尔类型，用整数 0/非0 代替

### 题目 2：分析 tTJSVariant 内存行为

假设你在调试 KrKr2 引擎，在 `tTJSVariant::operator=` 的赋值操作中打了断点。现在执行以下 TJS2 代码：

```javascript
var x = "hello";  // (1)
x = 42;           // (2)
x = [1, 2, 3];   // (3)
```

请描述每次赋值时，`tTJSVariant` 的 `vt` 字段和 union 中的活跃成员如何变化，以及引用计数管理的操作。

<details>
<summary>查看答案</summary>

```
(1) var x = "hello";
    vt = tvtString
    union 活跃成员：String（指向堆上的 tTJSVariantString 对象）
    引用计数：tTJSVariantString 的引用计数 = 1（x 持有一个引用）

(2) x = 42;
    步骤：
    a. ReleaseContent() 被调用
       → vt 当前是 tvtString
       → String->Release()  // 引用计数 -1，变为 0
       → tTJSVariantString 对象被释放（因为引用计数归零）
    b. vt = tvtInteger
    c. union 活跃成员改为 Integer = 42
    引用计数：Integer 不需要引用计数（它是值类型，直接存在 union 中）

(3) x = [1, 2, 3];
    步骤：
    a. ReleaseContent() 被调用
       → vt 当前是 tvtInteger
       → 整数不需要释放，无操作
    b. vt = tvtObject
    c. union 活跃成员改为 Object
       → Object.Object = 数组对象指针（iTJSDispatch2*）
       → Object.ObjThis = nullptr（数组没有特殊的 this 绑定）
    d. AddRef() 被调用
       → Object.Object->AddRef()  // 数组对象引用计数 +1
    引用计数：数组对象的引用计数 = 1（x 持有引用）
```

关键点：
- String 和 Object 类型需要引用计数管理（因为它们是堆分配的）
- Integer 和 Real 直接存储在 union 中，不需要引用计数
- void 不需要任何引用管理
- Octet 也需要引用计数（类似 String）

</details>

### 题目 3：闭包捕获问题

以下代码的输出是什么？为什么？

```javascript
function createFunctions() {
    var funcs = [];
    
    for (var i = 0; i < 3; i++) {
        funcs.add(function() {
            return i;
        });
    }
    
    return funcs;
}

var fns = createFunctions();
System.inform(fns[0]());  // ?
System.inform(fns[1]());  // ?
System.inform(fns[2]());  // ?
```

<details>
<summary>查看答案</summary>

输出全部是 `3`：
```
3
3
3
```

原因：
- TJS2 的 `var` 是函数作用域，`i` 在整个 `createFunctions` 函数内共享同一个变量
- 三个匿名函数都捕获了**同一个** `i` 变量（不是 `i` 的快照）
- 当 for 循环结束时，`i` 的值已经变成 3
- 调用任何一个函数时，它们都读取同一个 `i`，得到 3

这与 JavaScript 中使用 `var` 的经典闭包陷阱完全相同。

解决方案——使用立即执行函数（IIFE）创建新的作用域：

```javascript
function createFunctions() {
    var funcs = [];
    
    for (var i = 0; i < 3; i++) {
        // 用 IIFE 在每次循环中创建新的作用域
        funcs.add((function(capturedI) {
            return function() {
                return capturedI;  // 捕获的是参数 capturedI，不是外层的 i
            };
        })(i));
    }
    
    return funcs;
}

var fns = createFunctions();
System.inform(fns[0]());  // 0
System.inform(fns[1]());  // 1
System.inform(fns[2]());  // 2
```

在 C++ 层面，每次 IIFE 调用会创建一个新的 `tTJSInterCodeContext` 作为函数执行环境，`capturedI` 作为该环境的局部变量存在于独立的寄存器区域中，因此三个闭包各自捕获了不同的变量。

</details>
