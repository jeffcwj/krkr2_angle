# 字符串与数值处理

## 术语预览

| 术语 | 含义 |
|------|------|
| **TJSParseString** | 字符串字面量解析函数，处理单引号和双引号字符串，支持完整的转义序列和嵌入式表达式语法。内部调用 `TJSInternalParseString` 完成实际的字符扫描和转义处理 |
| **TJSInternalParseString** | 字符串解析的核心内部函数，逐字符扫描字符串内容，处理反斜杠转义序列（`\n`、`\t`、`\xNN`、`\0NNN` 等），并识别嵌入式表达式标记（`&` 和 `${}`） |
| **转义序列（Escape Sequence）** | 以反斜杠 `\` 开头的特殊字符表示法，用于在字符串中插入不可打印字符或特殊字符。TJS2 支持 C 风格转义（`\n`、`\t`、`\r`）和扩展转义（`\xNN` 十六进制、`\0NNN` 八进制） |
| **TJSParseNumber** | 数值字面量解析的入口函数，处理符号（正负）并委托给 `TJSParseNumber2` 进行实际的进制判断和数值解析 |
| **TJSParseNumber2** | 数值解析的核心函数，根据前缀判断进制：`0x` → 十六进制，`0b` → 二进制，`0` → 八进制，其他 → 十进制。对于整数调用 `TJSParseNonDecimalInteger`，对于浮点数调用 `TJSExtractNumber` 或 `TJSParseNonDecimalReal` |
| **非十进制浮点数** | TJS2 独有的语法，允许十六进制、八进制、二进制格式的浮点数字面量，使用 `p` 指数（类似 C99 的 hex float）。例如 `0x1.8p1` 表示 3.0。内部通过手动组装 IEEE 754 位模式实现 |
| **IEEE 754 位模式** | 浮点数的二进制表示标准，TJS2 在解析非十进制浮点数时直接操作 64 位双精度浮点数的符号位、指数位和尾数位，手动组装最终的浮点值 |
| **TJSParseOctet** | 字节序列（Octet）字面量解析函数，处理 `<% hex_bytes %>` 语法，将十六进制字节对转换为 `tTJSOctet` 类型的二进制数据 |
| **TJSParseRegExp** | 正则表达式字面量解析函数，处理 `/pattern/flags` 语法，将其转换为内部格式 `//flags/pattern` 存储。只在语法分析器通过 `SetStartOfRegExp()` 回调后才被调用 |
| **TJSUnescapeBackSlash** | 反斜杠转义处理函数，将字符串中的转义序列（如 `\n`、`\t`）转换为实际的字符值。被 `TJSInternalParseString` 在字符扫描过程中调用 |
| **嵌入式表达式标记** | 在字符串内部标记表达式边界的特殊字符序列：`&` 开始一个以 `;` 结束的表达式，`${` 开始一个以 `}` 结束的表达式。这些标记在字符串扫描阶段被识别，在 GetNext 的状态机中被处理 |

## 1. 字符串解析概述

TJS2 的字符串字面量支持两种引号风格（单引号 `'` 和双引号 `"`）、完整的 C 风格转义序列、以及嵌入式表达式语法。字符串解析由三个函数协作完成：

```
GetToken()
    │
    ├─ 遇到 ' 或 "
    │
    └─ TJSParseString()         ← 外层包装
        │
        └─ TJSInternalParseString()  ← 核心扫描
            │
            └─ TJSUnescapeBackSlash()  ← 转义序列处理
```

### 1.1 单引号 vs 双引号

与 JavaScript 相同，TJS2 中单引号和双引号的字符串在语义上**完全等价**——没有任何行为差异：

```javascript
// 以下两行完全等价
var s1 = "hello world";
var s2 = 'hello world';

// 可以在一种引号内使用另一种
var s3 = "it's a test";       // 双引号内包含单引号
var s4 = 'say "hello"';       // 单引号内包含双引号

// 转义也完全相同
var s5 = "line1\nline2";
var s6 = 'line1\nline2';      // 与 s5 相同
```

### 1.2 TJSParseString 的结构

```cpp
// TJSParseString：字符串解析入口
static tjs_int TJSParseString(
    tjs_char *&ptr,           // 扫描指针（会被修改）
    tTJSVariant &val,         // 输出：解析结果
    bool embeddable           // 是否处理嵌入式表达式标记
)
{
    tjs_char delimiter = *ptr;  // 记录引号类型 (' 或 ")
    ptr++;                       // 跳过开始引号

    ttstr result;                // 结果字符串

    // 调用内部解析函数
    bool has_embeddable = false;
    TJSInternalParseString(ptr, result, delimiter,
                           embeddable, has_embeddable);

    // ptr 现在指向结束引号之后
    val = tTJSVariant(result);
    return T_CONSTVAL;
}
```

## 2. 转义序列详解

### 2.1 基本转义序列

TJS2 支持与 C/C++ 兼容的标准转义序列：

```
转义序列    字符值    说明
────────    ────────  ──────────────────
\\          \         反斜杠本身
\'          '         单引号
\"          "         双引号
\n          0x0A      换行符（LF）
\r          0x0D      回车符（CR）
\t          0x09      水平制表符（Tab）
\f          0x0C      换页符（Form Feed）
\b          0x08      退格符（Backspace）
\a          0x07      响铃符（Bell）
\0          0x00      空字符（NUL）
```

```cpp
// TJSUnescapeBackSlash 的核心实现
static tjs_char TJSUnescapeBackSlash(const tjs_char *&ptr)
{
    // ptr 当前指向 \ 后面的字符
    switch(*ptr) {
        case TJS_W('n'):  ptr++; return TJS_W('\n');   // 换行
        case TJS_W('r'):  ptr++; return TJS_W('\r');   // 回车
        case TJS_W('t'):  ptr++; return TJS_W('\t');   // 制表
        case TJS_W('f'):  ptr++; return TJS_W('\f');   // 换页
        case TJS_W('b'):  ptr++; return TJS_W('\b');   // 退格
        case TJS_W('a'):  ptr++; return TJS_W('\a');   // 响铃
        case TJS_W('\\'):  ptr++; return TJS_W('\\');  // 反斜杠
        case TJS_W('\''):  ptr++; return TJS_W('\'');  // 单引号
        case TJS_W('"'):  ptr++; return TJS_W('"');    // 双引号
        // ... 扩展转义 ...
    }
}
```

### 2.2 十六进制转义 \xNN

`\x` 后跟恰好两个十六进制数字，表示一个 Unicode 码值：

```javascript
// 十六进制转义
var tab = "\x09";          // 制表符，等价于 \t
var space = "\x20";        // 空格
var A = "\x41";            // 字母 A
var smiley = "\xFF";       // 码值 0xFF 的字符
```

```cpp
case TJS_W('x'):
case TJS_W('X'):
{
    // \xNN：两位十六进制
    ptr++;  // 跳过 x/X
    tjs_int code = 0;
    // 读取第一个十六进制数字
    if(TJS_iswxdigit(*ptr)) {
        code = TJSHexNum(*ptr);
        ptr++;
    }
    // 读取第二个十六进制数字
    if(TJS_iswxdigit(*ptr)) {
        code = code * 16 + TJSHexNum(*ptr);
        ptr++;
    }
    return (tjs_char)code;
}
```

注意 `\x` 只读取**最多两个**十六进制数字。这与 JavaScript 的 `\xNN` 行为一致，但不同于 JavaScript 的 `\uNNNN`（TJS2 不支持 `\u` 转义）。

### 2.3 八进制转义 \0NNN

`\0` 后跟最多三个八进制数字，表示一个字符码值：

```javascript
// 八进制转义
var nul = "\0";            // NUL 字符（码值 0）
var bell = "\07";          // 响铃（码值 7），等价于 \a
var tab = "\011";          // 制表符（码值 9），等价于 \t
var max = "\0377";         // 码值 255（八进制 377）
```

```cpp
case TJS_W('0'):
{
    // \0NNN：最多三位八进制
    ptr++;  // 跳过 0
    tjs_int code = 0;
    tjs_int count = 0;
    while(count < 3 && *ptr >= TJS_W('0') && *ptr <= TJS_W('7')) {
        code = code * 8 + TJSOctNum(*ptr);
        ptr++;
        count++;
    }
    return (tjs_char)code;
}
```

### 2.4 未知转义的处理

当反斜杠后跟的字符不是任何已知的转义序列时，TJS2 的处理方式是**保留原字符**（忽略反斜杠）：

```javascript
// 未知转义
var s = "\q";  // TJS2 中等价于 "q"（反斜杠被忽略）
var s = "\z";  // 等价于 "z"
```

```cpp
default:
    // 不是已知的转义字符
    // 直接返回该字符本身（反斜杠被丢弃）
    tjs_char c = *ptr;
    ptr++;
    return c;
```

这与 JavaScript 的严格模式（strict mode）不同——JavaScript 严格模式下未知转义会报错。TJS2 选择了更宽容的处理方式。

## 3. 字符串内的嵌入式表达式标记

### 3.1 标记识别

`TJSInternalParseString` 在扫描字符串内容时，会检查两种嵌入式表达式标记：

```cpp
// TJSInternalParseString 中的嵌入式表达式标记检测
static void TJSInternalParseString(
    tjs_char *&ptr,
    ttstr &result,
    tjs_char delimiter,
    bool embeddable,
    bool &has_embeddable)
{
    while(*ptr && *ptr != delimiter) {
        if(*ptr == TJS_W('\\')) {
            // 转义序列处理
            ptr++;
            result += TJSUnescapeBackSlash(ptr);
        }
        else if(embeddable && *ptr == TJS_W('&')) {
            // &expr; 形式的嵌入式表达式
            has_embeddable = true;
            // 在此处中断字符串扫描
            // 返回到目前为止的字符串片段
            return;
        }
        else if(embeddable &&
                *ptr == TJS_W('$') &&
                *(ptr+1) == TJS_W('{')) {
            // ${expr} 形式的嵌入式表达式
            has_embeddable = true;
            // 在此处中断字符串扫描
            return;
        }
        else {
            // 普通字符
            result += *ptr;
            ptr++;
        }
    }

    // 跳过结束引号
    if(*ptr == delimiter) ptr++;
}
```

### 3.2 标记类型对比

```javascript
// &expr; 语法（Perl 风格）
var msg = @"Hello &name;!";
//               ^    ^
//               |    └─ 分号结束表达式
//               └────── & 开始表达式

// ${expr} 语法（ES6 风格）
var msg = @"Hello ${name}!";
//               ^^    ^
//               ||    └─ } 结束表达式
//               |└───── 变量名
//               └────── ${ 开始表达式

// 两种形式可以混合使用
var msg = @"&greeting; ${name}, score: &score;";
```

### 3.3 与普通字符串的区别

嵌入式表达式标记**只在 `@"..."` 字符串中被识别**。普通字符串中的 `&` 和 `${}` 都是普通字符：

```javascript
// 普通字符串：& 和 $ 是普通字符
var s1 = "price: $100 & tax";   // 字面值："price: $100 & tax"

// 嵌入式字符串：& 和 ${} 触发表达式嵌入
var s2 = @"price: ${100} & tax"; // & tax 也会被当作嵌入式表达式！
// 正确写法：
var s3 = @"price: ${100} \& tax"; // 用 \& 转义
```

这是 `embeddable` 参数的作用——当从普通字符串解析（`"..."`）调用时，`embeddable = false`，标记检测被跳过；当从嵌入式字符串解析（`@"..."`）调用时，`embeddable = true`，启用标记检测。

## 4. 数值解析架构

TJS2 的数值解析由多个函数分层协作：

```
GetToken()
    │
    ├─ 遇到 0-9 或 +/- 后跟数字 或 . 后跟数字
    │
    └─ TJSParseNumber()              ← 入口：处理符号
        │
        └─ TJSParseNumber2()          ← 进制判断与分派
            │
            ├─ 0x... → TJSParseNonDecimalInteger()  (十六进制整数)
            │          或 TJSParseNonDecimalReal()   (十六进制浮点)
            │
            ├─ 0b... → TJSParseNonDecimalInteger()  (二进制整数)
            │          或 TJSParseNonDecimalReal()   (二进制浮点)
            │
            ├─ 0...  → TJSParseNonDecimalInteger()  (八进制整数)
            │          或 TJSParseNonDecimalReal()   (八进制浮点)
            │
            └─ 其他  → TJSExtractNumber()           (十进制)
```

### 4.1 TJSParseNumber：符号处理

```cpp
static tjs_int TJSParseNumber(
    const tjs_char *&ptr,
    tTJSVariant &val)
{
    // 跳过正负号
    bool negative = false;
    if(*ptr == TJS_W('+')) {
        ptr++;
    } else if(*ptr == TJS_W('-')) {
        negative = true;
        ptr++;
    }

    // 委托给 TJSParseNumber2
    tjs_int result = TJSParseNumber2(ptr, val);

    // 如果是负数，取反
    if(negative) {
        if(val.IsInteger()) {
            val = tTJSVariant(-(tjs_int64)val);
        } else {
            val = tTJSVariant(-(tjs_real)val);
        }
    }

    return result;
}
```

### 4.2 TJSParseNumber2：进制判断

```cpp
static tjs_int TJSParseNumber2(
    const tjs_char *&ptr,
    tTJSVariant &val)
{
    // 特殊字面量检查
    if(TJSStringMatch(ptr, TJS_W("true"))) {
        ptr += 4;
        val = tTJSVariant((tjs_int)1);
        return T_CONSTVAL;
    }
    if(TJSStringMatch(ptr, TJS_W("false"))) {
        ptr += 5;
        val = tTJSVariant((tjs_int)0);
        return T_CONSTVAL;
    }
    if(TJSStringMatch(ptr, TJS_W("NaN"))) {
        ptr += 3;
        // 构造 NaN
        tjs_real d;
        *(tjs_uint64*)&d = TJS_IEEE_D_P_NaN;
        val = tTJSVariant(d);
        return T_CONSTVAL;
    }
    if(TJSStringMatch(ptr, TJS_W("Infinity"))) {
        ptr += 8;
        // 构造正无穷
        tjs_real d;
        *(tjs_uint64*)&d = TJS_IEEE_D_P_INF;
        val = tTJSVariant(d);
        return T_CONSTVAL;
    }

    // 进制判断
    if(*ptr == TJS_W('0')) {
        if(*(ptr+1) == TJS_W('x') || *(ptr+1) == TJS_W('X')) {
            // 十六进制：0x...
            ptr += 2;
            return ParseHexNumber(ptr, val);
        }
        if(*(ptr+1) == TJS_W('b') || *(ptr+1) == TJS_W('B')) {
            // 二进制：0b...
            ptr += 2;
            return ParseBinNumber(ptr, val);
        }
        if(TJS_iswdigit(*(ptr+1))) {
            // 八进制：0...
            ptr++;
            return ParseOctNumber(ptr, val);
        }
    }

    // 十进制
    return TJSExtractNumber(ptr, val);
}
```

## 5. 十进制数值解析

### 5.1 TJSExtractNumber

十进制数值解析使用标准的浮点数格式：`整数部分.小数部分e指数部分`

```cpp
static tjs_int TJSExtractNumber(
    const tjs_char *&ptr,
    tTJSVariant &val)
{
    const tjs_char *start = ptr;
    bool is_real = false;  // 是否包含小数点或指数

    // 1. 扫描整数部分
    while(TJS_iswdigit(*ptr)) ptr++;

    // 2. 检查小数点
    if(*ptr == TJS_W('.')) {
        is_real = true;
        ptr++;
        // 扫描小数部分
        while(TJS_iswdigit(*ptr)) ptr++;
    }

    // 3. 检查指数部分（e/E）
    if(*ptr == TJS_W('e') || *ptr == TJS_W('E')) {
        is_real = true;
        ptr++;
        // 可选的正负号
        if(*ptr == TJS_W('+') || *ptr == TJS_W('-'))
            ptr++;
        // 扫描指数数字
        while(TJS_iswdigit(*ptr)) ptr++;
    }

    // 4. 转换为数值
    if(is_real) {
        // 浮点数：使用标准库转换
        tjs_real d = TJS_strtod(start);
        val = tTJSVariant(d);
    } else {
        // 整数：手动转换
        tjs_int64 i = TJS_strtoi64(start);
        val = tTJSVariant(i);
    }

    return T_CONSTVAL;
}
```

### 5.2 十进制数值示例

```javascript
// 整数
var a = 42;           // 十进制整数
var b = 0;            // 零
var c = 1234567890;   // 大整数

// 浮点数
var d = 3.14;         // 标准浮点
var e = .5;           // 省略整数部分（0.5）
var f = 1.;           // 省略小数部分（1.0）

// 科学计数法
var g = 1e10;         // 1 × 10^10
var h = 2.5e-3;       // 0.0025
var i = 1.23E+4;      // 12300.0
```

## 6. 非十进制整数解析

### 6.1 十六进制整数

```javascript
// 十六进制整数（0x 或 0X 前缀）
var a = 0xFF;          // 255
var b = 0x1A2B;        // 6699
var c = 0xDEADBEEF;    // 3735928559
```

```cpp
// TJSParseNonDecimalInteger 的十六进制分支
static tjs_int TJSParseNonDecimalInteger(
    const tjs_char *&ptr,
    tTJSVariant &val,
    int base)  // base = 16
{
    tjs_int64 result = 0;
    while(TJS_iswxdigit(*ptr)) {
        result = result * 16 + TJSHexNum(*ptr);
        ptr++;
    }
    val = tTJSVariant(result);
    return T_CONSTVAL;
}

// TJSHexNum：将十六进制字符转换为数值
static inline tjs_int TJSHexNum(tjs_char ch)
{
    if(ch >= TJS_W('0') && ch <= TJS_W('9'))
        return ch - TJS_W('0');
    if(ch >= TJS_W('a') && ch <= TJS_W('f'))
        return ch - TJS_W('a') + 10;
    if(ch >= TJS_W('A') && ch <= TJS_W('F'))
        return ch - TJS_W('A') + 10;
    return 0;  // 不应该到达
}
```

### 6.2 八进制整数

```javascript
// 八进制整数（0 前缀）
var a = 077;           // 63（八进制 77）
var b = 0755;          // 493（Unix 文件权限）
var c = 010;           // 8（注意不是 10！）
```

```cpp
// TJSOctNum：将八进制字符转换为数值
static inline tjs_int TJSOctNum(tjs_char ch)
{
    return ch - TJS_W('0');  // '0'-'7' → 0-7
}
```

### 6.3 二进制整数

```javascript
// 二进制整数（0b 或 0B 前缀）
var a = 0b1010;        // 10
var b = 0b11111111;    // 255
var c = 0B10000000;    // 128
```

```cpp
// TJSBinNum：将二进制字符转换为数值
static inline tjs_int TJSBinNum(tjs_char ch)
{
    return ch - TJS_W('0');  // '0' → 0, '1' → 1
}
```

## 7. 非十进制浮点数——TJS2 的独特特性

这是 TJS2 数值解析中最复杂也最独特的部分。TJS2 允许十六进制、八进制、二进制格式的浮点数，使用 `p` 指数（类似 C99 的 hex float），并通过**手动组装 IEEE 754 位模式**来实现。

### 7.1 语法格式

```javascript
// 十六进制浮点数
var a = 0x1.8p1;       // 1.5 × 2^1 = 3.0
var b = 0xA.Cp0;       // 10.75 × 2^0 = 10.75
var c = 0x1p10;        // 1 × 2^10 = 1024.0

// 八进制浮点数
var d = 01.4p2;        // 1.5 × 2^2 = 6.0（八进制 1.4 = 十进制 1.5）

// 二进制浮点数
var e = 0b1.1p3;       // 1.5 × 2^3 = 12.0（二进制 1.1 = 十进制 1.5）
```

通用格式：`[前缀]整数部分.小数部分p[±]指数`

- **前缀**：`0x`（十六进制）、`0`（八进制）、`0b`（二进制）
- **整数部分**：对应进制的数字
- **小数部分**：对应进制的数字（可选）
- **p 指数**：以 2 为底的指数（十进制表示）

### 7.2 IEEE 754 双精度浮点数格式

在理解 TJS2 的实现之前，需要了解 IEEE 754 双精度浮点数的位布局：

```
63  62        52  51                                    0
┌─┬──────────┬──────────────────────────────────────────┐
│S│ Exponent │              Mantissa                    │
│ │ (11 bits)│             (52 bits)                    │
└─┴──────────┴──────────────────────────────────────────┘

S = 符号位（0 = 正，1 = 负）
Exponent = 偏移指数（实际指数 + 1023）
Mantissa = 尾数（隐含前导 1，即 1.mantissa）

数值 = (-1)^S × 2^(Exponent - 1023) × 1.Mantissa
```

### 7.3 TJSParseNonDecimalReal 的实现

```cpp
static tjs_int TJSParseNonDecimalReal(
    const tjs_char *&ptr,
    tTJSVariant &val,
    int base)  // base = 2, 8, 或 16
{
    // 1. 确定每个数字对应的位数
    int bits_per_digit;
    switch(base) {
        case 2:  bits_per_digit = 1; break;  // 二进制：1 位/数字
        case 8:  bits_per_digit = 3; break;  // 八进制：3 位/数字
        case 16: bits_per_digit = 4; break;  // 十六进制：4 位/数字
    }

    // 2. 收集整数部分的位
    tjs_uint64 mantissa = 0;
    int mantissa_bits = 0;
    bool has_point = false;
    int point_position = 0;  // 小数点后的位数

    // 扫描整数部分
    while(IsValidDigit(*ptr, base)) {
        int digit = DigitValue(*ptr, base);
        mantissa = (mantissa << bits_per_digit) | digit;
        mantissa_bits += bits_per_digit;
        ptr++;
    }

    // 3. 检查小数点
    if(*ptr == TJS_W('.')) {
        has_point = true;
        ptr++;

        // 扫描小数部分
        while(IsValidDigit(*ptr, base)) {
            int digit = DigitValue(*ptr, base);
            mantissa = (mantissa << bits_per_digit) | digit;
            mantissa_bits += bits_per_digit;
            point_position += bits_per_digit;
            ptr++;
        }
    }

    // 4. 解析 p 指数
    int p_exponent = 0;
    bool p_negative = false;
    if(*ptr == TJS_W('p') || *ptr == TJS_W('P')) {
        ptr++;
        if(*ptr == TJS_W('-')) {
            p_negative = true;
            ptr++;
        } else if(*ptr == TJS_W('+')) {
            ptr++;
        }
        while(TJS_iswdigit(*ptr)) {
            p_exponent = p_exponent * 10 + (*ptr - TJS_W('0'));
            ptr++;
        }
        if(p_negative) p_exponent = -p_exponent;
    }

    // 5. 计算最终指数
    //    实际指数 = p_exponent + (mantissa_bits - point_position - 1)
    //    减 1 是因为 IEEE 754 的隐含前导 1
    int exponent = p_exponent +
                   (mantissa_bits - point_position - 1);

    // 6. 规范化尾数（移除前导零，对齐到 52 位）
    // 找到最高有效位
    int msb = 0;
    for(int i = 63; i >= 0; i--) {
        if(mantissa & ((tjs_uint64)1 << i)) {
            msb = i;
            break;
        }
    }

    // 7. 手动组装 IEEE 754 双精度位模式
    tjs_uint64 ieee_bits = 0;

    // 符号位（此函数只处理正数，符号在外层处理）
    // ieee_bits |= (sign << 63);

    // 指数域（加偏移 1023）
    tjs_int biased_exp = exponent + 1023;
    if(biased_exp <= 0) {
        // 次正规数（denormal）或零
        // 简化处理...
    } else if(biased_exp >= 2047) {
        // 无穷大
        ieee_bits = TJS_IEEE_D_P_INF;
    } else {
        ieee_bits |= ((tjs_uint64)biased_exp << 52);
    }

    // 尾数域（隐含前导 1 不存储）
    // 将 mantissa 对齐到 52 位并去掉前导 1
    if(msb > 52) {
        mantissa >>= (msb - 52);
    } else if(msb < 52) {
        mantissa <<= (52 - msb);
    }
    mantissa &= 0x000FFFFFFFFFFFFF;  // 清除隐含的前导 1
    ieee_bits |= mantissa;

    // 8. 将位模式转换为 double
    tjs_real result;
    *(tjs_uint64*)&result = ieee_bits;
    val = tTJSVariant(result);

    return T_CONSTVAL;
}
```

### 7.4 为什么需要手动组装位模式？

标准的 `strtod()` 只能解析十进制浮点数。对于 `0x1.8p1` 这样的非十进制浮点数，C 标准库在 C99 之前没有现成的解析函数。TJS2 的代码库编写于 2000 年前后，当时大多数 C++ 编译器还不完全支持 C99 的 `strtod` 对 hex float 的扩展。因此 TJS2 选择了直接操作 IEEE 754 位模式来实现。

这种方法的优点是：
- **精确控制**：直接操作位模式，不存在十进制到二进制转换的精度损失
- **平台独立**：不依赖标准库的行为差异
- **支持任意进制**：统一处理十六进制、八进制、二进制浮点数

缺点是：
- **代码复杂**：需要理解 IEEE 754 的细节
- **边界情况多**：次正规数、无穷大、NaN、舍入等都需要处理

### 7.5 非十进制浮点数的实际应用

在实际的 KR2 游戏脚本中，非十进制浮点数很少使用。它主要在以下场景有用：

```javascript
// 精确的二进制分数
var half = 0b0.1p0;     // 精确的 0.5

// 颜色值的精确表示（0-255 映射到 0.0-1.0）
var alpha = 0xCCp-8;    // 0xCC / 256 = 204/256 ≈ 0.796875
                        // 精确值！无十进制转换误差

// 音频采样的位操作
var sample = 0x7FFFp-15; // 最大 16 位有符号整数 / 32768
```

### 8.1 语法

字节序列字面量使用 `<% ... %>` 语法，内部包含十六进制字节对：

```javascript
// 字节序列字面量
var png_header = <% 89 50 4E 47 0D 0A 1A 0A %>;
var jpeg_header = <%FFD8FFE0%>;  // 空格可选
var empty = <%%>;                 // 空字节序列
```

### 8.2 TJSParseOctet 实现

```cpp
static tjs_int TJSParseOctet(
    const tjs_char *&ptr,
    tTJSVariant &val)
{
    // ptr 当前指向 <% 后的位置
    ptr += 2;  // 跳过 <%

    std::vector<tjs_uint8> bytes;

    while(*ptr) {
        // 跳过空白
        while(*ptr == TJS_W(' ') || *ptr == TJS_W('\t') ||
              *ptr == TJS_W('\n') || *ptr == TJS_W('\r'))
            ptr++;

        // 检查结束标记 %>
        if(*ptr == TJS_W('%') && *(ptr+1) == TJS_W('>')) {
            ptr += 2;  // 跳过 %>
            break;
        }

        // 读取一个字节（两个十六进制数字）
        if(!TJS_iswxdigit(*ptr)) {
            // 错误：非法字符
            break;
        }
        tjs_uint8 byte = (tjs_uint8)(TJSHexNum(*ptr) << 4);
        ptr++;

        if(!TJS_iswxdigit(*ptr)) {
            // 错误：不完整的字节
            break;
        }
        byte |= (tjs_uint8)TJSHexNum(*ptr);
        ptr++;

        bytes.push_back(byte);
    }

    // 创建 tTJSOctet 对象
    val = tTJSVariant(bytes.data(), (tjs_uint)bytes.size());
    return T_CONSTVAL;
}
```

## 9. 正则表达式字面量

### 9.1 语法

TJS2 的正则表达式字面量使用与 JavaScript 相同的 `/pattern/flags` 语法：

```javascript
// 正则表达式字面量
var re1 = /hello/;           // 简单模式
var re2 = /\d+\.\d+/;       // 匹配浮点数
var re3 = /pattern/gi;      // 带标志：全局 + 忽略大小写
```

### 9.2 TJSParseRegExp 实现

```cpp
static tjs_int TJSParseRegExp(
    const tjs_char *&ptr,
    tTJSVariant &val)
{
    // ptr 当前指向 / 字符
    ptr++;  // 跳过开始的 /

    ttstr pattern;
    ttstr flags;

    // 1. 扫描模式部分（直到未转义的 /）
    while(*ptr && *ptr != TJS_W('/')) {
        if(*ptr == TJS_W('\\')) {
            // 转义字符——保留反斜杠和下一个字符
            pattern += *ptr;
            ptr++;
            if(*ptr) {
                pattern += *ptr;
                ptr++;
            }
        } else {
            pattern += *ptr;
            ptr++;
        }
    }

    // 2. 跳过结束的 /
    if(*ptr == TJS_W('/')) ptr++;

    // 3. 扫描标志部分
    while(TJS_iswalpha(*ptr)) {
        flags += *ptr;
        ptr++;
    }

    // 4. 转换为内部格式：//flags/pattern
    //    这种格式将标志放在前面，方便后续解析
    ttstr internal = TJS_W("//");
    internal += flags;
    internal += TJS_W("/");
    internal += pattern;

    val = tTJSVariant(internal);
    return T_REGEXP;
}
```

### 9.3 内部存储格式

注意 TJS2 将正则表达式存储为特殊格式的字符串 `//flags/pattern`，而不是 JavaScript 那样的 RegExp 对象。这个格式将标志放在前面，方便 VM 在创建正则表达式对象时快速提取标志：

```javascript
// 源代码                    内部存储
/hello/gi         →         "//gi/hello"
/\d+/             →         "///\d+"         （无标志）
/test/m           →         "//m/test"
```

### 9.4 正则表达式的触发条件

正如前一节所述，TJSParseRegExp 不会被 GetToken 的 `/` case 分支直接调用。它只在语法分析器判断 `/` 应该是正则表达式开始时，通过 `SetStartOfRegExp()` 回调触发。

```
时序：

1. GetToken 扫描到 /
2. 返回 T_SLASH（除法运算符）
3. 语法分析器发现 T_SLASH 在当前语法上下文中不合法
   （例如：if 语句的条件表达式开头不可能是除法）
4. 语法分析器调用 lexer->SetStartOfRegExp()
5. SetStartOfRegExp 将 Current 回退到 PrevPos（/ 的位置）
6. 设置 RegularExpression = true
7. 语法分析器再次调用 GetNext
8. GetToken 检查 RegularExpression 标志
9. 调用 TJSParseRegExp，返回 T_REGEXP
```

## 10. 数字辅助函数

### 10.1 数字字符转换函数族

TJS2 定义了四个数字字符转换函数，每个对应一种进制：

```cpp
// 十六进制字符 → 数值
static inline tjs_int TJSHexNum(tjs_char ch)
{
    if(ch >= TJS_W('0') && ch <= TJS_W('9'))
        return ch - TJS_W('0');            // '0'-'9' → 0-9
    if(ch >= TJS_W('a') && ch <= TJS_W('f'))
        return ch - TJS_W('a') + 10;      // 'a'-'f' → 10-15
    if(ch >= TJS_W('A') && ch <= TJS_W('F'))
        return ch - TJS_W('A') + 10;      // 'A'-'F' → 10-15
    return 0;
}

// 八进制字符 → 数值
static inline tjs_int TJSOctNum(tjs_char ch)
{
    return ch - TJS_W('0');  // '0'-'7' → 0-7
}

// 十进制字符 → 数值
static inline tjs_int TJSDecNum(tjs_char ch)
{
    return ch - TJS_W('0');  // '0'-'9' → 0-9
}

// 二进制字符 → 数值
static inline tjs_int TJSBinNum(tjs_char ch)
{
    return ch - TJS_W('0');  // '0'-'1' → 0-1
}
```

这些函数都被声明为 `inline`，编译器会将它们内联展开，消除函数调用开销。

### 10.2 数字合法性检查

```cpp
// 检查字符是否是指定进制的合法数字
static inline bool IsValidDigit(tjs_char ch, int base)
{
    switch(base) {
        case 2:  return ch == TJS_W('0') || ch == TJS_W('1');
        case 8:  return ch >= TJS_W('0') && ch <= TJS_W('7');
        case 10: return TJS_iswdigit(ch);
        case 16: return TJS_iswxdigit(ch);
        default: return false;
    }
}
```

## 11. 特殊数值常量

### 11.1 NaN 和 Infinity 的位模式

TJS2 通过直接定义 IEEE 754 位模式来创建特殊浮点值：

```cpp
// IEEE 754 双精度特殊值的位模式
#define TJS_IEEE_D_P_NaN   0x7FF8000000000000ULL
//   S=0, Exp=2047(全1), Mantissa=非零 → quiet NaN（正）

#define TJS_IEEE_D_N_NaN   0xFFF8000000000000ULL
//   S=1, Exp=2047(全1), Mantissa=非零 → quiet NaN（负）

#define TJS_IEEE_D_P_INF   0x7FF0000000000000ULL
//   S=0, Exp=2047(全1), Mantissa=0 → 正无穷

#define TJS_IEEE_D_N_INF   0xFFF0000000000000ULL
//   S=1, Exp=2047(全1), Mantissa=0 → 负无穷
```

### 11.2 直接位模式操作 vs 浮点运算

TJS2 使用直接位模式操作而不是浮点运算来创建 NaN 和 Infinity：

```cpp
// TJS2 的方式（直接位模式，保证跨平台一致）
tjs_real d;
*(tjs_uint64*)&d = TJS_IEEE_D_P_NaN;
// d 现在是 NaN

// 不使用的方式（浮点运算，可能有平台差异）
// double d = 0.0 / 0.0;     // 可能触发浮点异常
// double d = HUGE_VAL;       // 可能不是 IEEE 754 无穷
```

直接位模式操作的优点是：
1. **不触发浮点异常**：某些平台（特别是早期 Windows）的浮点异常设置可能导致 `0.0/0.0` 抛出硬件异常
2. **值确定**：保证生成的是 quiet NaN 而不是 signaling NaN
3. **跨平台一致**：不依赖编译器对特殊浮点运算的处理方式

## 练习题与答案

### 练习 1：转义序列解析

**问题**：以下 TJS2 字符串的实际值分别是什么？

```javascript
var a = "\\n";
var b = "\x0A";
var c = "\012";
var d = "\"quotes\"";
var e = "\q\w";
```

<details>
<summary>点击查看答案</summary>

- **a** = `\n`（两个字符：反斜杠和字母 n）。`\\` 被解析为反斜杠字面量，`n` 是普通字符。
- **b** = 换行符（一个字符）。`\x0A` 是十六进制转义，0x0A = 换行符 LF。
- **c** = 换行符（一个字符）。`\012` 是八进制转义，八进制 12 = 十进制 10 = 换行符 LF。
- **d** = `"quotes"`（8个字符）。`\"` 被解析为双引号字面量。
- **e** = `qw`（两个字符）。`\q` 和 `\w` 都不是合法的转义序列，TJS2 丢弃反斜杠，保留后面的字符。

</details>

### 练习 2：非十进制浮点数计算

**问题**：计算以下 TJS2 非十进制浮点数的十进制值：

```javascript
var a = 0x1.0p4;
var b = 0b1.01p2;
var c = 0xF.8p-1;
```

<details>
<summary>点击查看答案</summary>

- **a** = `0x1.0p4`
  - 十六进制 1.0 = 十进制 1.0
  - p4 = × 2^4 = × 16
  - 结果 = 1.0 × 16 = **16.0**

- **b** = `0b1.01p2`
  - 二进制 1.01 = 1 + 0/2 + 1/4 = 十进制 1.25
  - p2 = × 2^2 = × 4
  - 结果 = 1.25 × 4 = **5.0**

- **c** = `0xF.8p-1`
  - 十六进制 F.8 = 15 + 8/16 = 十进制 15.5
  - p-1 = × 2^(-1) = × 0.5
  - 结果 = 15.5 × 0.5 = **7.75**

</details>

### 练习 3：IEEE 754 位模式

**问题**：TJS2 中 `NaN` 和 `Infinity` 的 IEEE 754 位模式分别是 `0x7FF8000000000000` 和 `0x7FF0000000000000`。为什么它们的指数域相同（都是 `0x7FF` = 2047），但一个是 NaN 一个是无穷？

<details>
<summary>点击查看答案</summary>

IEEE 754 标准规定当指数域全为 1（双精度下值为 2047）时：
- 如果尾数域为**全零**，则该值表示**无穷大**（正或负取决于符号位）
- 如果尾数域为**非零**，则该值表示 **NaN**（Not a Number）

具体到 TJS2 的值：
- `0x7FF0000000000000`：指数 = 0x7FF，尾数 = 0x0000000000000（全零）→ 正无穷
- `0x7FF8000000000000`：指数 = 0x7FF，尾数 = 0x8000000000000（非零）→ NaN

进一步地，NaN 分为两种：
- 尾数最高位为 1 → **quiet NaN**（安静 NaN，不触发异常）
- 尾数最高位为 0 → **signaling NaN**（信号 NaN，可能触发异常）

TJS2 使用的 `0x7FF8000000000000` 尾数最高位为 1（`0x8...`），所以是 quiet NaN，不会触发浮点异常。

</details>

### 练习 4：字节序列的解析

**问题**：以下字节序列字面量会被解析为什么？

```javascript
var a = <% 48 65 6C 6C 6F %>;
var b = <%00FF%>;
var c = <% 1 2 3 %>;
```

<details>
<summary>点击查看答案</summary>

- **a** = 包含 5 个字节的序列：`0x48 0x65 0x6C 0x6C 0x6F`，即 ASCII 编码的 "Hello"。

- **b** = 包含 2 个字节的序列：`0x00 0xFF`。空格是可选的，`00FF` 等价于 `00 FF`。

- **c** = 这个会产生**解析错误**。`<% 1 2 3 %>` 中的 `1`、`2`、`3` 各自只有一个十六进制数字，但 `TJSParseOctet` 要求每个字节由恰好两个十六进制数字表示。第一个 `1` 会被读取，然后空格不是十六进制数字，导致字节不完整。正确写法应为 `<% 01 02 03 %>`。

</details>

### 练习 5：正则表达式的内部格式

**问题**：以下正则表达式字面量在 TJS2 内部会被存储为什么格式？

```javascript
var r1 = /test/;
var r2 = /\d+\.\d+/gi;
var r3 = /hello\/world/;
```

TJS2 将正则表达式存储为 `//flags/pattern` 格式：

- **r1** = `"///test"`
  - 标志：无
  - 模式：`test`
  - 内部格式中标志为空，所以是 `//` + `` + `/` + `test`

- **r2** = `"//gi/\d+\.\d+"`
  - 标志：`gi`
  - 模式：`\d+\.\d+`
  - 内部格式：`//` + `gi` + `/` + `\d+\.\d+`

- **r3** = `"///hello\/world"`
  - 标志：无
  - 模式：`hello\/world`（反斜杠和斜杠都保留）
  - 内部格式：`//` + `` + `/` + `hello\/world`

注意模式中的转义序列（如 `\d`、`\/`）在词法分析阶段不被处理——它们原样保留在字符串中，由正则表达式引擎在运行时解释。
