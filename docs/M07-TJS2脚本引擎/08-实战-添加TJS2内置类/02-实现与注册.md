# 02 - 实现与注册

## 术语预览

| 术语 | 含义 |
|------|------|
| **成员注册（Member Registration）** | 使用`RegisterNCM`将C++函数/属性包装为TJS2成员并添加到类对象的字典中 |
| **链式调用（Method Chaining）** | 方法返回对象自身（`objthis`），允许连续调用如`sb.append("a").append("b")` |
| **闭包修正（Closure Fixup）** | 实例创建时，将类级别的方法闭包的objthis从NULL改为具体实例对象 |
| **参数分派（Parameter Dispatch）** | 根据`param[i]->Type()`判断参数的TJS2类型，选择不同的C++处理分支 |
| **成员保证标志（MEMBERENSURE）** | `PropSetByVS`的标志位，如果成员不存在则自动创建 |
| **全局字符串映射（GlobalStringMap）** | TJS2内部的字符串池，`RegisterNCM`通过它将成员名映射到唯一的`tTJSVariantString*` |

## 1. C++核心类完善

上节已给出`tTJS2StringBuffer`的基本设计。在实际编码中，需要补充一些细节：

```cpp
// tjs2StringBuffer.h - 完整版本
#ifndef __TJS2_STRING_BUFFER_H__
#define __TJS2_STRING_BUFFER_H__

#include <string>
#include <cstddef>
#include <algorithm>

// tTJS2StringBuffer: 高效字符串拼接缓冲区
// 使用std::wstring作为底层存储
// 所有方法都是非虚的（无虚函数表开销）
// 线程安全性：无（TJS2是单线程的）
class tTJS2StringBuffer {
private:
    std::wstring buffer_;

    // 辅助方法：整数转宽字符串
    // 手动实现避免std::to_wstring在某些平台的问题
    static void intToWStr(long long val,
                          std::wstring& out) {
        if (val == 0) { out += L'0'; return; }

        bool neg = val < 0;
        if (neg) { out += L'-'; val = -val; }

        // 计算位数
        wchar_t buf[32];
        int len = 0;
        while (val > 0) {
            buf[len++] = L'0' + (wchar_t)(val % 10);
            val /= 10;
        }
        // 反转
        for (int i = len - 1; i >= 0; i--) {
            out += buf[i];
        }
    }

    // 辅助方法：浮点数转宽字符串
    static void realToWStr(double val,
                           std::wstring& out) {
        // 使用snwprintf确保跨平台一致
        wchar_t buf[64];
        #ifdef _WIN32
        _snwprintf(buf, 64, L"%.6g", val);
        #else
        swprintf(buf, 64, L"%.6g", val);
        #endif
        out += buf;
    }

public:
    // === 构造函数 ===

    explicit tTJS2StringBuffer(size_t cap = 64) {
        buffer_.reserve(cap);
    }

    explicit tTJS2StringBuffer(const wchar_t* initial)
        : buffer_(initial ? initial : L"") {}

    // === 修改操作 ===

    void append(const wchar_t* str) {
        if (str) buffer_.append(str);
    }

    void append(const wchar_t* str, size_t len) {
        if (str && len > 0) buffer_.append(str, len);
    }

    void appendInt(long long val) {
        intToWStr(val, buffer_);
    }

    void appendReal(double val) {
        realToWStr(val, buffer_);
    }

    void appendLine(const wchar_t* str) {
        if (str) buffer_.append(str);
        buffer_.push_back(L'\n');
    }

    void appendChar(wchar_t ch) {
        buffer_.push_back(ch);
    }

    void insert(size_t pos, const wchar_t* str) {
        if (!str) return;
        if (pos > buffer_.size()) pos = buffer_.size();
        buffer_.insert(pos, str);
    }

    void remove(size_t start, size_t count) {
        if (start >= buffer_.size()) return;
        if (count == (size_t)-1 ||
            start + count > buffer_.size()) {
            count = buffer_.size() - start;
        }
        buffer_.erase(start, count);
    }

    int replace(const wchar_t* oldStr,
                const wchar_t* newStr) {
        if (!oldStr || !newStr) return 0;
        std::wstring old(oldStr);
        std::wstring rep(newStr);
        if (old.empty()) return 0;

        int count = 0;
        size_t pos = 0;
        while ((pos = buffer_.find(old, pos))
               != std::wstring::npos) {
            buffer_.replace(pos, old.size(), rep);
            pos += rep.size();
            count++;
        }
        return count;
    }

    void clear() { buffer_.clear(); }

    // === 查询操作 ===

    size_t length() const { return buffer_.size(); }
    size_t getCapacity() const {
        return buffer_.capacity();
    }
    bool empty() const { return buffer_.empty(); }

    const wchar_t* c_str() const {
        return buffer_.c_str();
    }

    int indexOf(const wchar_t* str,
                size_t fromIndex = 0) const {
        if (!str) return -1;
        size_t pos = buffer_.find(str, fromIndex);
        return (pos == std::wstring::npos)
            ? -1 : (int)pos;
    }

    std::wstring substring(size_t start,
                           size_t end) const {
        if (start >= buffer_.size())
            return std::wstring();
        if (end > buffer_.size())
            end = buffer_.size();
        if (start >= end)
            return std::wstring();
        return buffer_.substr(start, end - start);
    }

    wchar_t charAt(size_t index) const {
        if (index >= buffer_.size()) return 0;
        return buffer_[index];
    }

    // === 容量管理 ===

    void reserve(size_t cap) {
        buffer_.reserve(cap);
    }

    void shrink() {
        buffer_.shrink_to_fit();
    }
};

#endif
```

> **跨平台要点**：`_snwprintf`是MSVC特有的函数，POSIX平台使用`swprintf`。通过`#ifdef _WIN32`条件编译处理。KrKr2项目已在CMake中定义了`WINDOWS`宏，但标准的`_WIN32`更通用。

## 2. NI类实现

NI类（Native Instance）的实现在上节已给出完整设计。这里补充一些实际编码中的注意事项：

```cpp
// tjs2StringBufferNI.h - 完整NI类实现
#ifndef __TJS2_STRING_BUFFER_NI_H__
#define __TJS2_STRING_BUFFER_NI_H__

#include "tjsNative.h"
#include "tjs2StringBuffer.h"

class tTJS2StringBufferNI : public tTJSNativeInstance {
public:
    tTJS2StringBuffer* sb_;

    tTJS2StringBufferNI() : sb_(nullptr) {
        // 不在构造函数中创建sb_
        // 原因：C++构造和TJS2构造是分离的
        // C++构造由CreateNativeInstance调用（无参数）
        // TJS2构造由Construct调用（有TJS2参数）
    }

    ~tTJS2StringBufferNI() {
        // Destruct通常不需要重写
        // 但为安全起见，确保释放
        if (sb_) {
            delete sb_;
            sb_ = nullptr;
        }
    }

    tjs_error TJS_INTF_METHOD Construct(
        tjs_int numparams,
        tTJSVariant **param,
        iTJSDispatch2 *tjs) override
    {
        // 处理三种构造模式
        if (numparams == 0) {
            // new StringBuffer()
            sb_ = new tTJS2StringBuffer();
        }
        else if (param[0]->Type() == tvtInteger) {
            // new StringBuffer(1024)
            tjs_int cap = (tjs_int)*param[0];
            if (cap < 0) cap = 64;  // 负数用默认值
            sb_ = new tTJS2StringBuffer((size_t)cap);
        }
        else {
            // new StringBuffer("初始内容")
            // 对于任何非整数类型，尝试转为字符串
            ttstr str(*param[0]);
            sb_ = new tTJS2StringBuffer(str.c_str());
        }

        return TJS_S_OK;
    }

    void TJS_INTF_METHOD Invalidate() override {
        // 释放C++资源
        // Invalidate可能在以下时机调用：
        // 1. TJS2 GC回收对象时
        // 2. 脚本调用 invalidate 方法时
        // 3. 引擎关闭时批量清理
        if (sb_) {
            delete sb_;
            sb_ = nullptr;
        }
    }
};

#endif
```

> **Invalidate vs Destruct**：两者都会释放sb_，这不是bug而是防御性编程。正常流程是`Invalidate()`先释放资源，然后`~tTJS2StringBufferNI()`（由Destruct触发）发现sb_已经为nullptr跳过删除。如果某些异常路径跳过了Invalidate，析构函数是最后的安全网。

## 3. NC类与方法注册——完整实现

这是本节的核心——完整的NC类定义，包含所有16个方法和3个属性的注册代码。

```cpp
// tjs2StringBufferClass.cpp - StringBuffer类的完整实现
// 这是唯一需要编译的源文件
// 头文件通过include引入

#include "tjsNative.h"
#include "tjsError.h"      // TJS2错误码定义
#include "tjs2StringBufferNI.h"

// 全局ClassID——每个原生类必须有一个
// -1表示未初始化，TJS_BEGIN_NATIVE_MEMBERS会赋值
static tjs_int32 tTJS2StringBufferClassID = -1;

// ====================================================
// StringBuffer类对象定义
// ====================================================
class tTJS2StringBufferClass : public tTJSNativeClass {
    typedef tTJSNativeClass inherited;

public:
    tTJS2StringBufferClass()
        : tTJSNativeClass(TJS_W("StringBuffer"))
    {
        // ============================================
        // 成员注册块开始
        // TJS_BEGIN_NATIVE_MEMBERS宏展开为：
        //   tTJS2StringBufferClassID =
        //     TJSRegisterNativeClass("StringBuffer");
        // ============================================
        TJS_BEGIN_NATIVE_MEMBERS(StringBuffer)

        // ---- finalize（必须，每个原生类都需要） ----
        TJS_DECL_EMPTY_FINALIZE_METHOD

        // ---- 构造函数 ----
        TJS_BEGIN_NATIVE_CONSTRUCTOR_DECL(
            /*var.name*/ _this,
            /*var.type*/ tTJS2StringBufferNI,
            /*cls.name*/ StringBuffer)
        {
            // 构造参数处理已在NI的Construct中完成
            // 这里不需要额外操作
            return TJS_S_OK;
        }
        TJS_END_NATIVE_CONSTRUCTOR_DECL(StringBuffer)

        // ============================================
        // 核心方法
        // ============================================

        // ---- append(value) → 返回自身 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(append)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            // 根据参数类型分派
            switch (param[0]->Type()) {
            case tvtString:
                _this->sb_->append(
                    ttstr(*param[0]).c_str());
                break;
            case tvtInteger:
                _this->sb_->appendInt(
                    (tjs_int64)*param[0]);
                break;
            case tvtReal:
                _this->sb_->appendReal(
                    (tjs_real)*param[0]);
                break;
            case tvtVoid:
                // void类型追加空字符串（静默处理）
                break;
            default:
                // 其他类型：先转为字符串再追加
                _this->sb_->append(
                    ttstr(*param[0]).c_str());
                break;
            }

            // 返回自身，支持链式调用
            // objthis是当前TJS2对象实例
            if (result)
                *result = tTJSVariant(
                    objthis, objthis);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(append)

        // ---- appendLine(value) → 返回自身 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(appendLine)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams >= 1 &&
                param[0]->Type() != tvtVoid) {
                // 先追加内容
                switch (param[0]->Type()) {
                case tvtString:
                    _this->sb_->append(
                        ttstr(*param[0]).c_str());
                    break;
                case tvtInteger:
                    _this->sb_->appendInt(
                        (tjs_int64)*param[0]);
                    break;
                case tvtReal:
                    _this->sb_->appendReal(
                        (tjs_real)*param[0]);
                    break;
                default:
                    _this->sb_->append(
                        ttstr(*param[0]).c_str());
                    break;
                }
            }
            // 追加换行符
            _this->sb_->appendChar(L'\n');

            if (result)
                *result = tTJSVariant(
                    objthis, objthis);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(appendLine)

        // ---- insert(pos, value) → 返回自身 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(insert)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 2)
                return TJS_E_BADPARAMCOUNT;

            // 参数0：插入位置（整数）
            tjs_int pos = (tjs_int)*param[0];
            if (pos < 0) pos = 0;

            // 参数1：插入内容
            ttstr val(*param[1]);
            _this->sb_->insert(
                (size_t)pos, val.c_str());

            if (result)
                *result = tTJSVariant(
                    objthis, objthis);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(insert)

        // ---- remove(start [, count]) → 返回自身 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(remove)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            tjs_int start = (tjs_int)*param[0];
            if (start < 0) start = 0;

            // count参数可选
            // 如果省略或为void，删除到末尾
            size_t count = (size_t)-1;
            if (numparams >= 2 &&
                param[1]->Type() != tvtVoid) {
                tjs_int c = (tjs_int)*param[1];
                if (c >= 0) count = (size_t)c;
            }

            _this->sb_->remove(
                (size_t)start, count);

            if (result)
                *result = tTJSVariant(
                    objthis, objthis);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(remove)

        // ---- replace(oldStr, newStr) → 替换次数 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(replace)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 2)
                return TJS_E_BADPARAMCOUNT;

            ttstr oldStr(*param[0]);
            ttstr newStr(*param[1]);

            int count = _this->sb_->replace(
                oldStr.c_str(), newStr.c_str());

            // 返回替换次数（不是自身）
            // 因为调用者通常需要知道是否替换成功
            if (result)
                *result = tTJSVariant(
                    (tjs_int)count);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(replace)

        // ---- clear() → void ----
        TJS_BEGIN_NATIVE_METHOD_DECL(clear)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            _this->sb_->clear();

            if (result)
                *result = tTJSVariant();
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(clear)

        // ---- toString() → String ----
        TJS_BEGIN_NATIVE_METHOD_DECL(toString)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            // 将缓冲区内容转为TJS2字符串
            // ttstr会复制内容——这是有意的
            // 返回的字符串是独立的，后续对sb的
            // 修改不会影响已返回的字符串
            if (result)
                *result = tTJSVariant(
                    ttstr(_this->sb_->c_str()));
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(toString)

        // ============================================
        // 查询方法
        // ============================================

        // ---- indexOf(searchStr [, fromIndex]) → 整数 ----
        TJS_BEGIN_NATIVE_METHOD_DECL(indexOf)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            ttstr searchStr(*param[0]);

            size_t fromIndex = 0;
            if (numparams >= 2 &&
                param[1]->Type() != tvtVoid) {
                tjs_int fi = (tjs_int)*param[1];
                if (fi > 0) fromIndex = (size_t)fi;
            }

            int pos = _this->sb_->indexOf(
                searchStr.c_str(), fromIndex);

            if (result)
                *result = tTJSVariant(
                    (tjs_int)pos);
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(indexOf)

        // ---- substring(start [, end]) → String ----
        TJS_BEGIN_NATIVE_METHOD_DECL(substring)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            tjs_int start = (tjs_int)*param[0];
            if (start < 0) start = 0;

            // end参数可选，默认到末尾
            size_t end = _this->sb_->length();
            if (numparams >= 2 &&
                param[1]->Type() != tvtVoid) {
                tjs_int e = (tjs_int)*param[1];
                if (e >= 0) end = (size_t)e;
            }

            std::wstring sub =
                _this->sb_->substring(
                    (size_t)start, end);

            if (result)
                *result = tTJSVariant(
                    ttstr(sub.c_str()));
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(substring)

        // ---- charAt(index) → String ----
        TJS_BEGIN_NATIVE_METHOD_DECL(charAt)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            tjs_int index = (tjs_int)*param[0];
            if (index < 0) {
                // 负索引：返回空字符串
                if (result)
                    *result = tTJSVariant(
                        ttstr(TJS_W("")));
                return TJS_S_OK;
            }

            wchar_t ch = _this->sb_->charAt(
                (size_t)index);

            if (ch == 0) {
                // 越界：返回空字符串
                if (result)
                    *result = tTJSVariant(
                        ttstr(TJS_W("")));
            } else {
                // 返回单字符字符串
                wchar_t buf[2] = { ch, 0 };
                if (result)
                    *result = tTJSVariant(
                        ttstr(buf));
            }
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(charAt)

        // ============================================
        // 容量管理方法
        // ============================================

        // ---- reserve(capacity) → void ----
        TJS_BEGIN_NATIVE_METHOD_DECL(reserve)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            if (numparams < 1)
                return TJS_E_BADPARAMCOUNT;

            tjs_int cap = (tjs_int)*param[0];
            if (cap > 0) {
                _this->sb_->reserve((size_t)cap);
            }

            if (result)
                *result = tTJSVariant();
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(reserve)

        // ---- shrink() → void ----
        TJS_BEGIN_NATIVE_METHOD_DECL(shrink)
        {
            TJS_GET_NATIVE_INSTANCE(
                _this, tTJS2StringBufferNI);

            _this->sb_->shrink();

            if (result)
                *result = tTJSVariant();
            return TJS_S_OK;
        }
        TJS_END_NATIVE_METHOD_DECL(shrink)

        // ============================================
        // 属性定义
        // ============================================

        // ---- length属性（只读） ----
        TJS_BEGIN_NATIVE_PROP_DECL(length)
        {
            TJS_BEGIN_NATIVE_PROP_GETTER
                TJS_GET_NATIVE_INSTANCE(
                    _this, tTJS2StringBufferNI);
                *result = tTJSVariant(
                    (tjs_int)_this->sb_->length());
                return TJS_S_OK;
            TJS_END_NATIVE_PROP_GETTER

            TJS_DENY_NATIVE_PROP_SETTER
        }
        TJS_END_NATIVE_PROP_DECL(length)

        // ---- capacity属性（只读） ----
        TJS_BEGIN_NATIVE_PROP_DECL(capacity)
        {
            TJS_BEGIN_NATIVE_PROP_GETTER
                TJS_GET_NATIVE_INSTANCE(
                    _this, tTJS2StringBufferNI);
                *result = tTJSVariant(
                    (tjs_int)_this->sb_->
                        getCapacity());
                return TJS_S_OK;
            TJS_END_NATIVE_PROP_GETTER

            TJS_DENY_NATIVE_PROP_SETTER
        }
        TJS_END_NATIVE_PROP_DECL(capacity)

        // ---- empty属性（只读） ----
        TJS_BEGIN_NATIVE_PROP_DECL(empty)
        {
            TJS_BEGIN_NATIVE_PROP_GETTER
                TJS_GET_NATIVE_INSTANCE(
                    _this, tTJS2StringBufferNI);
                *result = tTJSVariant(
                    (tjs_int)(_this->sb_->empty()
                        ? 1 : 0));
                return TJS_S_OK;
            TJS_END_NATIVE_PROP_GETTER

            TJS_DENY_NATIVE_PROP_SETTER
        }
        TJS_END_NATIVE_PROP_DECL(empty)

        // ============================================
        // 成员注册块结束
        // ============================================
        TJS_END_NATIVE_MEMBERS

    }  // 构造函数结束

    // CreateNativeInstance: 工厂方法
    // 由tTJSNativeClass::FuncCall(NULL)内部调用
    tTJSNativeInstance* CreateNativeInstance()
        override
    {
        return new tTJS2StringBufferNI();
    }
};
```

## 4. 注册与注销函数

```cpp
// tjs2StringBufferClass.cpp（续）

// 全局类对象指针
static iTJSDispatch2* StringBufferClassObj = nullptr;

// 注册StringBuffer类到TJS2全局命名空间
void RegisterStringBufferClass() {
    // Step 1: 创建类对象
    // 构造函数中的TJS_BEGIN_NATIVE_MEMBERS
    // 会自动完成：
    //   - 分配ClassID
    //   - 注册所有方法和属性
    tTJS2StringBufferClass* cls =
        new tTJS2StringBufferClass();

    // Step 2: 获取TJS2全局对象
    // 全局对象是所有全局变量的容器
    iTJSDispatch2* global =
        TJSGetGlobalObject();

    if (global) {
        // Step 3: 将类对象设置为全局变量
        tTJSVariant val(cls, cls);
        global->PropSetByVS(
            TJS_MEMBERENSURE | TJS_IGNOREPROP,
            TJS_W("StringBuffer"),
            &val,
            global
        );
        // PropSetByVS内部会AddRef cls
    }

    // Step 4: Release我们的引用
    // 全局对象现在持有cls的引用
    cls->Release();

    // 保存指针供注销使用
    StringBufferClassObj = cls;
}

// 从TJS2全局命名空间移除StringBuffer类
void UnregisterStringBufferClass() {
    if (StringBufferClassObj) {
        iTJSDispatch2* global =
            TJSGetGlobalObject();

        if (global) {
            // 从全局删除StringBuffer成员
            global->DeleteMember(
                0,
                TJS_W("StringBuffer"),
                nullptr,
                global
            );
        }

        StringBufferClassObj = nullptr;
        // 注意：不需要Release——注册时已经Release过
        // 全局对象持有的引用会在DeleteMember时释放
    }
}
```

> **引用计数分析**：
> - `new tTJS2StringBufferClass()` → refcount = 1
> - `tTJSVariant val(cls, cls)` → refcount = 2（Variant内部AddRef）
> - `PropSetByVS(...)` → refcount = 3（全局对象内部AddRef）
> - `val`析构 → refcount = 2
> - `cls->Release()` → refcount = 1（只剩全局对象持有）
> - `DeleteMember(...)` → refcount = 0 → 销毁

## 5. 链式调用的实现细节

链式调用（`sb.append("a").append("b")`）的核心是**让方法返回对象自身**。在低级宏方式中，"自身"就是`objthis`：

```cpp
// 链式调用的关键代码
if (result)
    *result = tTJSVariant(objthis, objthis);
//                        ^^^^^^^ ^^^^^^^
//                        对象     context
// tTJSVariant(obj, obj) 创建一个对象引用
// 内部会调用 objthis->AddRef()
// 当Variant被销毁时调用 objthis->Release()
```

> **为什么不能返回`_this`（NI指针）**？因为NI不是iTJSDispatch2，无法被TJS2脚本直接使用。TJS2脚本操作的是TJS2对象（通过objthis引用），NI是隐藏在对象内部的C++数据。

链式调用的完整流程：

```
TJS2: sb.append("a").append("b")
      │
      ├─ sb.append("a")
      │   ├─ 查找sb对象的append成员 → 闭包
      │   ├─ FuncCall(闭包, objthis=sb, ["a"])
      │   ├─ 执行append逻辑
      │   ├─ *result = tTJSVariant(objthis, objthis)
      │   └─ 返回sb对象自身
      │
      └─ [返回值].append("b")
          ├─ [返回值] == sb（同一个对象）
          ├─ 查找sb对象的append成员 → 闭包
          ├─ FuncCall(闭包, objthis=sb, ["b"])
          ├─ 执行append逻辑
          ├─ *result = tTJSVariant(objthis, objthis)
          └─ 返回sb对象自身
```

### 哪些方法应该支持链式调用？

| 方法 | 返回值 | 链式调用 | 理由 |
|------|--------|---------|------|
| `append` | 自身 | ✅ | 最常用的链式场景 |
| `appendLine` | 自身 | ✅ | 与append一致 |
| `insert` | 自身 | ✅ | 允许连续插入 |
| `remove` | 自身 | ✅ | 允许连续删除 |
| `replace` | 替换次数 | ❌ | 调用者需要知道替换了多少次 |
| `clear` | void | ❌ | 清空后通常不再操作 |
| `toString` | String | ❌ | 返回字符串，不是自身 |
| `indexOf` | 整数 | ❌ | 查询方法，返回位置 |
| `substring` | String | ❌ | 提取操作，返回子串 |
| `charAt` | String | ❌ | 查询方法，返回字符 |
| `reserve` | void | ❌ | 容量管理，不需要链式 |
| `shrink` | void | ❌ | 容量管理，不需要链式 |

## 6. 错误处理策略

### 6.1 TJS2错误码

低级宏方式的方法通过返回`tjs_error`（typedef为`tjs_int`）来报告错误。常用的错误码定义在`tjsError.h`中：

```cpp
// 常用TJS2错误码
#define TJS_S_OK            0     // 成功
#define TJS_S_TRUE          0     // 成功（true语义）
#define TJS_E_FAIL         -1     // 通用失败
#define TJS_E_MEMBERNOTFOUND  -2  // 成员未找到
#define TJS_E_NOTIMPL      -3     // 未实现
#define TJS_E_INVALIDPARAM  -4    // 参数无效
#define TJS_E_BADPARAMCOUNT -5    // 参数数量错误
#define TJS_E_INVALIDTYPE   -6    // 类型无效
#define TJS_E_NATIVECLASSCRASH -7 // 原生类崩溃
//   （通常是NI获取失败，对象已被Invalidate）

// 宏：检查是否成功/失败
#define TJS_SUCCEEDED(x) ((x) >= 0)
#define TJS_FAILED(x)    ((x) <  0)
```

### 6.2 StringBuffer的错误处理模式

```cpp
// 模式1：参数数量检查
if (numparams < 1)
    return TJS_E_BADPARAMCOUNT;
// TJS2运行时会将此错误转为TJS2异常
// 脚本端可以用try/catch捕获

// 模式2：参数类型宽容处理
// StringBuffer选择宽容策略：
// 不严格检查类型，尽量转换
// 原因：TJS2是动态类型语言，严格类型检查
// 会让API用起来不自然
switch (param[0]->Type()) {
    case tvtString:
        // 直接使用
        break;
    case tvtInteger:
        // 转为字符串
        break;
    case tvtVoid:
        // 静默忽略
        break;
    default:
        // 尝试toString
        break;
}

// 模式3：范围检查（宽容）
tjs_int pos = (tjs_int)*param[0];
if (pos < 0) pos = 0;  // 修正负数
// 不抛异常，而是修正为合理值
// 原因：与JavaScript的String方法行为一致

// 模式4：NI获取失败
// TJS_GET_NATIVE_INSTANCE 内部处理：
// 如果获取失败（对象已Invalidate），
// 自动返回 TJS_E_NATIVECLASSCRASH
```

### 6.3 防御性编程清单

```
✅ 每个方法开头检查 numparams
✅ NI指针通过 TJS_GET_NATIVE_INSTANCE 获取（自动检查）
✅ sb_ 指针在 Invalidate 后为 nullptr
✅ 字符串参数 null 检查（ttstr 构造可能失败）
✅ 负数索引修正为0
✅ 越界访问返回空值（不崩溃）
✅ result 指针 null 检查（调用者可能不需要返回值）
❌ 不检查 sb_ 是否为 nullptr（依赖 TJS_GET_NATIVE_INSTANCE）
   理由：如果 Construct 成功，sb_ 一定非空
         如果 Invalidate 后访问，TJS_GET_NATIVE_INSTANCE
         会先失败，不会执行到 sb_ 访问代码
```

### 题目1：分析以下append实现的bug

```cpp
TJS_BEGIN_NATIVE_METHOD_DECL(append)
{
    TJS_GET_NATIVE_INSTANCE(_this, tTJS2StringBufferNI);
    if (numparams < 1) return TJS_E_BADPARAMCOUNT;

    ttstr str(*param[0]);
    _this->sb_->append(str.c_str());

    if (result) *result = tTJSVariant(objthis, objthis);
    return TJS_S_OK;
}
TJS_END_NATIVE_METHOD_DECL(append)
```

这个实现有什么问题？

<details>
<summary>查看答案</summary>

**问题分析**：

1. **整数和浮点数的精度丢失**：
   当传入整数（如`sb.append(42)`），`ttstr(*param[0])`会调用tTJSVariant的字符串转换。虽然功能正确，但多了一层不必要的中间字符串分配。对于高频调用，应直接使用`appendInt`/`appendReal`避免临时字符串分配。

2. **void类型处理**：
   当传入void（如`sb.append(void)`），`ttstr(*param[0])`可能产生空字符串"void"或""，取决于TJS2实现。不够明确——正确做法是检查`tvtVoid`后直接跳过。

3. **缺少类型分派**：
   统一走字符串转换丢失了按类型优化的机会。特别是对于整数，`appendInt`使用手动数字转换，比先转`ttstr`再转`wchar_t*`更高效。

**修正后的代码**就是本节中给出的完整append实现（使用switch-case按类型分派）。

</details>

### 题目2：为StringBuffer添加一个`reverse`方法

实现一个`reverse()`方法，将缓冲区中的内容原地反转。需要：
1. 在C++核心类中添加反转逻辑
2. 在NC类中注册TJS2方法
3. 支持链式调用

<details>
<summary>查看答案</summary>

```cpp
// 1. 在 tTJS2StringBuffer 类中添加：
void reverse() {
    if (buffer_.size() <= 1) return;

    size_t n = buffer_.size();
    for (size_t i = 0; i < n / 2; i++) {
        std::swap(buffer_[i],
                  buffer_[n - 1 - i]);
    }
}

// 2. 在 TJS_BEGIN_NATIVE_MEMBERS 块中添加：
TJS_BEGIN_NATIVE_METHOD_DECL(reverse)
{
    TJS_GET_NATIVE_INSTANCE(
        _this, tTJS2StringBufferNI);

    _this->sb_->reverse();

    // 支持链式调用
    if (result)
        *result = tTJSVariant(
            objthis, objthis);
    return TJS_S_OK;
}
TJS_END_NATIVE_METHOD_DECL(reverse)
```

TJS2使用示例：
```javascript
var sb = new StringBuffer("Hello");
sb.reverse();
System.inform(sb.toString()); // "olleH"

// 链式调用
var result = new StringBuffer("abc")
    .append("def")
    .reverse()
    .toString();
// result == "fedcba"
```

注意事项：
- 对于包含代理对（surrogate pair）的Unicode字符串（如emoji），简单的字符级反转会破坏编码。在KrKr2的使用场景（日文视觉小说）中，代理对不常见，但完善的实现应该考虑此问题。
- Windows下wchar_t是2字节（UTF-16），代理对问题更突出。Linux/macOS下wchar_t是4字节（UTF-32），一个wchar_t就是一个完整的Unicode码位。

</details>

### 题目3：解释RegisterStringBufferClass中的引用计数变化

逐行分析以下代码中`cls`对象的引用计数变化，说明每一步引用计数是多少，以及为什么最终不会内存泄漏。

```cpp
void RegisterStringBufferClass() {
    tTJS2StringBufferClass* cls = new tTJS2StringBufferClass();  // (A)
    iTJSDispatch2* global = TJSGetGlobalObject();                // (B)
    tTJSVariant val(cls, cls);                                   // (C)
    global->PropSetByVS(..., &val, global);                      // (D)
    cls->Release();                                              // (E)
    // val析构                                                    // (F)
}
```

<details>
<summary>查看答案</summary>

**逐步引用计数分析**：

```
步骤  代码                                refcount  说明
(A)   new tTJS2StringBufferClass()        1         构造时初始refcount=1
(B)   TJSGetGlobalObject()               1         不影响cls
(C)   tTJSVariant val(cls, cls)           2         Variant内部调用cls->AddRef()
(D)   PropSetByVS(...)                    3         全局对象内部AddRef
(E)   cls->Release()                      2         手动释放我们的引用
(F)   val析构（函数返回时）               1         Variant析构调用cls->Release()
```

**最终状态**：refcount = 1，只有全局对象持有cls的引用。

**不会泄漏的原因**：

1. 当TJS2引擎关闭时，调用`UnregisterStringBufferClass()`
2. `DeleteMember`从全局对象中移除"StringBuffer"条目
3. 全局对象内部Release cls → refcount = 0
4. cls被销毁（~tTJS2StringBufferClass）

**如果忘记调用`cls->Release()`会怎样**？
- 步骤(E)不执行 → 函数结束后refcount = 2
- UnregisterStringBufferClass中DeleteMember → refcount = 1
- cls永远不会被销毁 → **内存泄漏**

这就是为什么注册函数中的`cls->Release()`是必须的——它平衡了`new`时的初始引用。

</details>
