# M07 — TJS2 脚本引擎

## 模块概述

TJS2（T Visual Presenter JavaScript-like Scripting Language 2）是 KiriKiri/吉里吉里引擎的内嵌脚本语言。它的语法与 JavaScript 非常相似——支持动态类型、原型继承、闭包、正则表达式、异常处理等现代特性，但又有自己的独特设计（如 `property` 关键字、八进制打包字符串等）。

在 KrKr2 模拟器中，TJS2 是连接游戏逻辑与引擎底层的桥梁。游戏开发者用 TJS2 脚本（`.tjs` 文件）编写对话逻辑、界面控制、动画效果，而 KAG（KiriKiri Adventure Game，KiriKiri 冒险游戏标记语言）标签最终也会调用 TJS2 函数来执行。可以说，**不理解 TJS2，就无法理解 KrKr2 引擎的运作方式**。

本模块将从 TJS2 的语言设计开始，逐层深入到编译器前端（词法分析、语法分析）、编译器后端（字节码生成）、虚拟机执行、对象系统、以及 C++ 原生绑定。最终，你将能够阅读和修改 `cpp/core/tjs2/` 下的每一个源文件。

## 学习目标

完成本模块后，你将能够：

1. **编写 TJS2 脚本** — 掌握 TJS2 的语法、类型系统、类与继承、闭包、异常处理等核心特性
2. **理解 TJS2 编译流程** — 从源码字符串到 token 流、再到 AST（抽象语法树，Abstract Syntax Tree，编译器内部的树形代码表示）、再到字节码的完整过程
3. **阅读和调试 TJS2 虚拟机** — 理解 `tjsInterCodeExec.cpp` 中的指令分发循环，能定位脚本执行时的问题
4. **理解 TJS2 的对象模型** — 掌握 `tTJSVariant`（变体类型）和 `iTJSDispatch2`（对象接口）的设计，理解动态类型如何实现
5. **将 C++ 功能暴露给 TJS2** — 使用 `tjsNative` 和 NCB 宏系统（一套自动生成 TJS2 绑定代码的 C++ 宏）把 C++ 类和函数注册到 TJS2 脚本环境中
6. **扩展 TJS2 引擎** — 能添加新的内置对象、修改语法规则、优化虚拟机性能

## 术语预览

本模块将涉及以下术语，先有个印象，后续章节会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| TJS2 | T Visual Presenter JS-like 2 | KiriKiri 引擎的内嵌脚本语言，语法类似 JavaScript 但有自己的独特扩展 |
| KAG | KiriKiri Adventure Game | 基于 TJS2 的上层标记语言，用 `@tag` 和 `[tag]` 格式编写视觉小说剧本，最终调用 TJS2 函数执行 |
| tTJSVariant | TJS Variant | TJS2 的万能变量类型，用一个 C++ 联合体（union）保存整数、浮点数、字符串、对象等各种值 |
| iTJSDispatch2 | TJS Dispatch Interface 2 | TJS2 对象系统的核心接口，所有 TJS2 对象都实现这个接口，提供属性读写、方法调用等操作 |
| 字节码 | Bytecode | 编译器将 TJS2 源码翻译成的中间指令序列，由虚拟机逐条执行，而不是直接翻译成机器码 |
| 虚拟机 | Virtual Machine (VM) | TJS2 的字节码执行器，类似 Java 的 JVM——读取字节码指令并执行对应操作 |
| 词法分析 | Lexical Analysis (Lexing) | 将源码字符串切分为 token（标记）序列的过程，如把 `var x = 42;` 切分为 `var`、`x`、`=`、`42`、`;` |
| 语法分析 | Syntactic Analysis (Parsing) | 将 token 序列按语法规则组装成 AST（抽象语法树）的过程，由 Bison 生成的解析器完成 |
| Bison | GNU Bison | 一个 LALR 语法分析器生成器，接受 `.y` 语法文件作为输入，自动生成 C/C++ 的解析器代码 |
| NCB | Native Class Binding | KrKr2 中用于自动生成 TJS2 绑定代码的 C++ 宏系统，省去手写冗长注册代码的麻烦 |
| 引用计数 | Reference Counting | TJS2 对象的内存管理方式——每个对象记录有多少地方在引用它，引用数归零时自动释放 |
| ScriptBlock | Script Block | TJS2 的编译单元——一段 TJS2 源码经过编译后成为一个 ScriptBlock，包含字节码和常量池 |
