# 第一章 第三节：归档格式对比与选择

## 术语预览

| 术语 | 英文 | 含义 |
|------|------|------|
| 归档格式 | Archive Format | 将多个文件打包成单一文件的二进制格式规范 |
| 分段压缩 | Segmented Compression | 将文件数据分成多个独立段分别压缩，支持随机访问 |
| 中央目录 | Central Directory | ZIP 格式末尾存储的完整文件索引表 |
| 魔数 | Magic Number | 文件开头的固定字节序列，用于识别文件格式类型 |
| 提取过滤器 | Extraction Filter | XP3 特有的回调函数，在解压后对数据进行二次处理（如解密） |
| 固实压缩 | Solid Compression | 7z 格式中将多个文件合并后统一压缩，提高压缩率但牺牲随机访问性能 |
| 块结构 | Chunk Structure | XP3 索引区使用的嵌套数据块组织方式，每个块有4字节类型标识和8字节大小 |
| 流式读取 | Streaming Read | TAR 格式从头到尾顺序读取的访问方式，不需要预先加载索引 |
| 字节对齐 | Byte Alignment | 数据在文件中按特定边界（如512字节）对齐存放 |
| 自适应策略 | Adaptive Strategy | 根据资源类型和使用场景自动选择最优归档格式的设计方案 |

## 1. KrKr2 归档格式全景

KrKr2 引擎同时支持四种归档格式，每种都有完整的实现代码。这不是简单的"多格式支持"——每种格式都在特定场景下有不可替代的优势。理解它们的差异，是做出正确技术选型的基础。

### 1.1 四种格式的定位

```
┌─────────────────────────────────────────────────────┐
│                KrKr2 归档格式体系                      │
├─────────────┬──────────────────────────────────────┤
│   XP3       │ 引擎原生格式，功能最完整               │
│             │ 分段压缩 + 提取过滤器 + EXE内嵌        │
├─────────────┼──────────────────────────────────────┤
│   ZIP       │ 通用兼容格式，工具链最丰富              │
│             │ 完整重实现（不依赖系统 zlib）            │
├─────────────┼──────────────────────────────────────┤
│   TAR       │ 轻量顺序格式，适合流式分发              │
│             │ POSIX + GNU 扩展，无压缩开销            │
├─────────────┼──────────────────────────────────────┤
│   7z        │ 高压缩率格式，适合大体积资源             │
│             │ LZMA SDK 集成，固实压缩               │
└─────────────┴──────────────────────────────────────┘
```

**格式识别机制**：KrKr2 的 `TVPOpenArchive()` 工厂函数通过读取文件头部的魔数字节来自动判断格式。魔数是文件开头的固定字节序列，每种格式都有唯一的标识。这意味着引擎不依赖文件扩展名——一个 `.dat` 文件只要内容是合法的 XP3 格式，就会被正确识别和打开。

```cpp
// 源码位置：StorageIntf.cpp — TVPOpenArchive() 工厂函数
// 展示格式自动识别的核心逻辑
tTVPArchive* TVPOpenArchive(const ttstr& name, tjs_int mode) {
    // 第一步：读取文件头部魔数
    tTJSBinaryStream* stream = TVPCreateStream(name, TJS_BS_READ);
    unsigned char header[16];
    stream->Read(header, 16);
    
    // 第二步：按优先级尝试各格式
    // XP3 魔数：11字节 "XP3\r\n \n\x1a\x8b\x67\x01"
    if (memcmp(header, "XP3\r\n \n", 7) == 0) {
        return new tTVPXP3Archive(name, mode);
    }
    
    // ZIP 魔数：4字节 "PK\x03\x04"（本地文件头）
    // 或 "PK\x05\x06"（空归档的中央目录结束标记）
    if (header[0] == 'P' && header[1] == 'K') {
        return new tTVPZIPArchive(name);
    }
    
    // 7z 魔数：6字节 "7z\xBC\xAF\x27\x1C"
    if (memcmp(header, "7z\xBC\xAF\x27\x1C", 6) == 0) {
        return new tTVP7zArchive(name);
    }
    
    // TAR 没有统一魔数，通过头部校验和验证
    // 检查偏移 257 处的 "ustar" 标识（POSIX tar）
    // 或通过计算 512 字节头部的校验和来验证
    if (IsTarArchive(stream)) {
        return new tTVPTARArchive(name);
    }
    
    // 无法识别 → 抛出异常
    TVPThrowExceptionMessage(TVPCannotOpenArchive, name);
    return nullptr;  // 不会执行到这里
}
```

> **注意**：以上代码展示了实际的格式判断逻辑。TAR 格式的识别最复杂，因为 TAR 没有像其他格式那样简洁的魔数标识，需要依靠头部结构校验来确认。

## 2. 格式特性详细对比

### 2.1 特性矩阵

下表是四种格式在 KrKr2 中的**实际实现**对比，不是通用格式规范对比：

| 特性维度 | XP3 | ZIP | TAR | 7z |
|---------|-----|-----|-----|-----|
| **魔数长度** | 11 字节 | 2 字节 (PK) | 无统一魔数 | 6 字节 |
| **索引位置** | 文件尾部 | 文件尾部(中央目录) | 无索引(顺序) | 文件尾部 |
| **压缩算法** | zlib(逐段) | deflate | 无 | LZMA/LZMA2 |
| **加密支持** | 提取过滤器(可扩展) | 无 | 无 | 无 |
| **随机访问** | ✅ O(1)哈希查找 | ✅ 中央目录 | ❌ 顺序扫描 | ⚠️ 视编码器而定 |
| **EXE内嵌** | ✅ 原生支持 | ❌ | ❌ | ❌ |
| **分段压缩** | ✅ 独立段 | ❌ 整文件 | ❌ | ⚠️ 固实块 |
| **段缓存** | ✅ LRU(1MB) | ❌ | ❌ | ❌ |
| **归档句柄缓存** | ✅ 8个槽位 | ❌ | ❌ | ❌ |
| **嵌套归档** | ✅ `>` 分隔符 | ✅ | ✅ | ✅ |
| **流式创建** | 专用XP3Stream | inflate解压 | 偏移+长度包装 | 内存解压或直接 |
| **实现代码量** | ~1200行 | ~2100行 | ~150行 | ~260行 |

### 2.2 逐项深度分析

#### 索引结构对比

**索引结构**决定了"打开归档后如何找到某个文件"的效率。这是归档格式最核心的设计差异之一。不同的索引策略直接影响文件查找速度、内存占用和加载时间。

```cpp
// 示例：四种格式的文件查找路径对比
// 假设归档中有 1000 个文件，查找 "images/bg001.png"

// ========== XP3 查找路径 ==========
// 1. 归档打开时：读取尾部索引 → 解析所有 File chunk → 
//    构建 std::unordered_map<ttstr, tjs_int> 哈希表
// 2. 文件查找时：
void FindInXP3(const ttstr& name) {
    // 步骤 1：规范化文件名（小写 + 统一分隔符）
    ttstr normalized = NormalizeInArchiveStorageName(name);
    // 步骤 2：哈希表 O(1) 查找
    auto it = IndexHashTable.find(normalized);
    if (it != IndexHashTable.end()) {
        int index = it->second;  // 直接得到文件索引
        // 步骤 3：通过索引访问 Items[index] 获取段信息
    }
}
// 复杂度：O(1) 平均，O(n) 最坏（哈希冲突）
// 内存开销：哈希表 + 所有文件名 + 段信息数组

// ========== ZIP 查找路径 ==========
// 1. 归档打开时：定位中央目录结束记录(EOCD) → 
//    读取中央目录 → 构建文件信息列表
// 2. 文件查找时：
void FindInZIP(const char* name) {
    // 遍历中央目录记录列表
    for (int i = 0; i < num_entries; i++) {
        // 逐个比较文件名
        if (strcmp(entries[i].filename, name) == 0) {
            // 找到 → 读取本地文件头 → 解压
            break;
        }
    }
}
// 复杂度：O(n) 线性扫描
// KrKr2 实现中没有为 ZIP 构建哈希表

// ========== TAR 查找路径 ==========
// 1. 归档打开时：从头到尾扫描所有 512 字节头部 →
//    记录每个文件的偏移和大小
// 2. 文件查找时：
void FindInTAR(const char* name) {
    // TAR 在打开时已经扫描了所有头部
    // 通过基类 tTVPArchive 的哈希表查找
    // （因为 TAR 的构造函数中调用了 AddItem）
}
// 复杂度：打开时 O(n)，查找时 O(1)（继承基类哈希表）
// 特殊之处：打开开销大（全量扫描），但查找快

// ========== 7z 查找路径 ==========
// 1. 归档打开时：LZMA SDK 的 SzArEx_Open() 解析头部 →
//    获取文件数量和名称
// 2. 文件查找时：
void FindIn7z(const char* name) {
    // 同样通过基类哈希表查找
    // 7z 构造函数中遍历 LZMA SDK 的文件列表
    // 调用 AddItem 注册到基类
}
// 复杂度：打开时依赖 LZMA SDK，查找时 O(1)
```

#### 压缩策略对比

**压缩策略**直接决定了两个关键指标：文件体积（影响下载和存储成本）和访问速度（影响运行时性能）。四种格式采用了完全不同的压缩思路。

```cpp
// 压缩策略对比的关键区别

// ========== XP3：分段压缩（Segmented Compression） ==========
// 一个文件可以被分成多个段（segment），每段独立压缩
// 优势：只需解压要读取的那一段，不必解压整个文件
struct tTVPXP3ArchiveSegment {
    bool IsCompressed;      // 该段是否被压缩
    tjs_uint64 Start;       // 段在归档中的偏移
    tjs_uint64 Offset;      // 段在原始文件中的偏移
    tjs_uint64 OrgSize;     // 原始大小
    tjs_uint64 ArcSize;     // 归档中的大小（压缩后）
};
// 场景示例：一个 10MB 的大图被分成 10 个 1MB 段
// 如果只需要读取中间部分，只需解压 1-2 个段
// 而不是解压全部 10MB

// ========== ZIP：整文件压缩 ==========
// 每个文件作为整体进行 deflate 压缩
// 读取时必须从头开始解压到需要的位置
void ReadFromZIP(int offset, int length) {
    // 即使只需要文件末尾的 100 字节，
    // 也要从头解压整个文件
    // 因为 deflate 是流式压缩，数据有前后依赖
    unsigned char* full_data = new unsigned char[file_size];
    inflate(compressed_data, full_data);
    memcpy(output, full_data + offset, length);
    delete[] full_data;
}

// ========== TAR：无压缩 ==========
// TAR 本身不做任何压缩，文件以原始形式存储
// 如需压缩，通常整个 TAR 文件再用 gzip/bzip2 压缩
// 但 KrKr2 只支持纯 TAR，不支持 .tar.gz
// 优势：零解压开销，直接读取
// 劣势：文件体积与原始文件相同

// ========== 7z：固实压缩（Solid Compression） ==========
// 同一个"文件夹"（folder，7z 术语）内的文件合并后统一压缩
// 极高的压缩率，但随机访问性能差
void ReadFrom7z(int file_index) {
    SzArEx_Extract(
        &db,           // 归档数据库
        &lookStream,   // 输入流
        file_index,    // 文件索引
        &blockIndex,   // 输出：所属块索引
        &outBuffer,    // 输出：解压后的整块数据
        &outSize,      // 输出：整块大小
        &offset,       // 输出：目标文件在块中的偏移
        &sizeProcessed,// 输出：目标文件大小
        &allocImp,     // 内存分配器
        &allocTempImp  // 临时内存分配器
    );
    // 注意：为了获取一个文件，可能需要解压整个"固实块"
    // 该块可能包含数百个其他文件
    // KrKr2 的优化：如果是单编码器非压缩块，直接创建 TArchiveStream
}
```

## 3. 性能对比测试框架

理论分析只能给出定性结论。要做出准确的技术选型，需要用实际数据说话。下面的测试框架可以定量对比四种格式在不同维度上的表现。

### 3.1 基准测试设计

```cpp
// 示例：归档格式性能对比测试框架
// 编译：g++ -O2 -std=c++17 bench_archive.cpp -o bench_archive
#include <chrono>
#include <cstdio>
#include <cstring>
#include <vector>
#include <string>
#include <random>
#include <algorithm>
#include <functional>
#include <unordered_map>

// 模拟的测试数据结构
struct TestFile {
    std::string name;     // 文件名
    size_t size;          // 文件大小
    const char* category; // 文件类别
};

// 性能计时器
struct BenchTimer {
    using Clock = std::chrono::high_resolution_clock;
    Clock::time_point start_;
    
    void start() { start_ = Clock::now(); }
    
    double elapsed_ms() const {
        auto end = Clock::now();
        return std::chrono::duration<double, std::milli>(end - start_).count();
    }
};

// 测试结果记录
struct BenchResult {
    const char* format;       // 格式名称
    double open_ms;           // 打开归档耗时
    double lookup_ms;         // 单次查找耗时（平均）
    double seq_read_ms;       // 顺序读取耗时
    double random_read_ms;    // 随机读取耗时
    size_t memory_bytes;      // 内存占用
    size_t archive_bytes;     // 归档文件大小
};

// 生成测试用文件集合（模拟典型游戏资源）
std::vector<TestFile> GenerateTestSet() {
    std::vector<TestFile> files;
    
    // 大量小文件（脚本）
    for (int i = 0; i < 500; i++) {
        char name[64];
        snprintf(name, sizeof(name), "scripts/scene_%04d.ks", i);
        files.push_back({name, 2048 + (i % 8192), "script"});
    }
    
    // 中等文件（UI 图像）
    for (int i = 0; i < 200; i++) {
        char name[64];
        snprintf(name, sizeof(name), "images/ui/button_%03d.png", i);
        files.push_back({name, 32768 + (i % 65536), "image"});
    }
    
    // 大文件（背景图）
    for (int i = 0; i < 50; i++) {
        char name[64];
        snprintf(name, sizeof(name), "images/bg/background_%03d.bmp", i);
        files.push_back({name, 1048576 + (i % 2097152), "background"});
    }
    
    // 音频文件
    for (int i = 0; i < 100; i++) {
        char name[64];
        snprintf(name, sizeof(name), "sound/bgm_%03d.ogg", i);
        files.push_back({name, 524288 + (i % 1048576), "audio"});
    }
    
    return files;
}

// 运行测试（伪代码，展示测试方法论）

    printf("测试文件集：%zu 个文件\n", files.size());
    
    // 对每种格式运行相同的测试
    const char* formats[] = {"XP3", "ZIP", "TAR", "7z"};
    BenchResult results[4];
    
    for (int f = 0; f < 4; f++) {

        results[f].format = formats[f];

        // 包含：读取头部 + 解析索引 + 构建内部数据结构
        timer.start();
        // archive = OpenArchive(formats[f], archive_path);
        results[f].open_ms = timer.elapsed_ms();

        timer.start();
        for (int i = 0; i < 1000; i++) {
            int idx = rand() % files.size();
            // archive->FindFile(files[idx].name);
        }
        results[f].lookup_ms = timer.elapsed_ms() / 1000.0;
        
        // 测试 3：顺序读取所有文件
        timer.start();
        for (size_t i = 0; i < files.size(); i++) {
            // auto stream = archive->CreateStream(i);
            // stream->ReadAll();
        }
        results[f].seq_read_ms = timer.elapsed_ms();
        
        // 测试 4：随机读取（模拟实际游戏运行）
        timer.start();
        std::vector<int> order(files.size());
        for (size_t i = 0; i < order.size(); i++) order[i] = (int)i;
        // std::shuffle(order.begin(), order.end(), rng);
        for (int idx : order) {
            // auto stream = archive->CreateStream(idx);
            // stream->ReadAll();
        }
        results[f].random_read_ms = timer.elapsed_ms();
    }
    
    // 输出结果表格
    printf("\n%-8s %10s %10s %10s %10s %10s %10s\n",
        "格式", "打开(ms)", "查找(ms)", "顺序(ms)", "随机(ms)", 
        "内存(KB)", "体积(KB)");
    printf("%-8s %10s %10s %10s %10s %10s %10s\n",
        "----", "--------", "--------", "--------", "--------",
        "--------", "--------");
    for (int f = 0; f < 4; f++) {
        printf("%-8s %10.2f %10.4f %10.2f %10.2f %10zu %10zu\n",
            results[f].format,
            results[f].open_ms,
            results[f].lookup_ms,
            results[f].seq_read_ms,
            results[f].random_read_ms,
            results[f].memory_bytes / 1024,
            results[f].archive_bytes / 1024);
    }
}
```

## 4. 场景化选型指南

### 4.1 决策树

根据项目实际需求，按以下决策树选择格式：

```
需要发布正式游戏？
├─ 是 → 需要资源加密？
│  ├─ 是 → XP3（唯一支持提取过滤器的格式）
│  └─ 否 → 资源总体积超过 2GB？
│     ├─ 是 → 7z（极致压缩率节省下载带宽）
│     └─ 否 → XP3（原生格式，性能最优）
│
├─ 开发/调试阶段？
│  ├─ 需要频繁修改资源？
│  │  ├─ 是 → 直接使用文件系统（不打包）
│  │  └─ 否 → ZIP（工具链支持最好）
│  └─ 需要持续集成打包？
│     └─ TAR（打包最快，无压缩开销）
│
└─ 特殊场景？
   ├─ 热更新补丁 → XP3（AutoPath 补丁覆盖机制）
   ├─ 嵌入 EXE 分发 → XP3（原生 EXE 内嵌支持）
   ├─ 跨引擎资源共享 → ZIP（最通用的格式）
   └─ MOD 社区支持 → ZIP（玩家最熟悉的格式）
```

### 4.2 典型场景分析

```cpp
// 场景 1：正式发布的视觉小说游戏
// 推荐：XP3
// 理由：加密保护 + 分段压缩 + EXE 内嵌 = 完美发布方案

// 配置示例：设置 XP3 提取过滤器进行资源加密
void SetupProductionArchive() {
    // 注册提取过滤器（在主程序启动时调用）
    // tTVPXP3ArchiveExtractionFilter 是函数指针类型
    // 参数：
    //   info->FileHash  — 文件的 adlr 哈希值（用作解密密钥种子）
    //   info->Buffer    — 解压后的数据缓冲区
    //   info->BufferSize — 缓冲区大小
    //   info->Offset    — 当前段在文件中的偏移
    auto filter = [](tTVPXP3ExtractionFilterInfo* info) {

        tjs_uint8* buf = (tjs_uint8*)info->Buffer;
        tjs_int64 size = info->BufferSize;
        
        // 简单的 XOR 解密（实际项目中应使用更强的算法）
        for (tjs_int64 i = 0; i < size; i++) {
            buf[i] ^= (tjs_uint8)(key >> ((i & 3) << 3));
        }
    };
    
    TVPSetXP3ArchiveExtractionFilter(filter);
}
```

```cpp
// 示例：格式迁移工具框架
// 支持任意两种格式之间的转换

#include <cstdio>
#include <cstring>
#include <vector>
#include <string>

// 抽象的归档读取接口
class IArchiveReader {
public:
    virtual ~IArchiveReader() = default;
    virtual int GetFileCount() = 0;
    virtual const char* GetFileName(int index) = 0;
    virtual size_t GetFileSize(int index) = 0;
    virtual bool ReadFile(int index, void* buffer, size_t size) = 0;
};

// 抽象的归档写入接口
class IArchiveWriter {
public:
    virtual ~IArchiveWriter() = default;
    virtual bool AddFile(const char* name, const void* data, 
                         size_t size, bool compress) = 0;
    virtual bool Finalize() = 0;
};

// 迁移引擎
class ArchiveMigrator {
public:
    struct MigrationStats {
        int total_files;          // 总文件数
        int migrated_files;       // 已迁移文件数
        int failed_files;         // 失败文件数
        size_t source_size;       // 源归档大小
        size_t target_size;       // 目标归档大小
        double elapsed_seconds;   // 耗时
    };
    
    // 执行迁移
    MigrationStats Migrate(IArchiveReader* source, 
                           IArchiveWriter* target,
                           bool compress_all) {
        MigrationStats stats = {};
        stats.total_files = source->GetFileCount();
        
        for (int i = 0; i < stats.total_files; i++) {
            const char* name = source->GetFileName(i);
            size_t size = source->GetFileSize(i);
            
            // 读取源文件数据
            std::vector<unsigned char> buffer(size);
            if (!source->ReadFile(i, buffer.data(), size)) {
                printf("  [失败] %s — 读取错误\n", name);
                stats.failed_files++;
                continue;
            }
            
            // 决定是否压缩（已压缩格式不再压缩）
            bool should_compress = compress_all && !IsAlreadyCompressed(name);
            
            // 写入目标归档
            if (target->AddFile(name, buffer.data(), size, should_compress)) {
                stats.migrated_files++;
            } else {
                printf("  [失败] %s — 写入错误\n", name);
                stats.failed_files++;
            }
        }
        
        target->Finalize();
        return stats;
    }
    
private:
    // 判断文件是否已经是压缩格式
    bool IsAlreadyCompressed(const char* filename) {
        const char* ext = strrchr(filename, '.');
        if (!ext) return false;

            ".png", ".jpg", ".jpeg", ".gif",    // 图像

                }

            }
        }
```

```cpp
ArchiveFormat DetectFormat(const char* filename) {
    FILE* fp = fopen(filename, "rb");
    if (!fp) return FORMAT_UNKNOWN;
    
    // 读取前 16 字节
    unsigned char header[16];
    size_t read = fread(header, 1, 16, fp);
    if (read < 6) {
        fclose(fp);
        return FORMAT_UNKNOWN;
    }
    
    // 检查 XP3 魔数（前 7 字节）
    // "XP3\r\n \n" = {0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A}
    const unsigned char xp3_magic[] = {0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A};
    if (read >= 7 && memcmp(header, xp3_magic, 7) == 0) {
        fclose(fp);
        return FORMAT_XP3;
    }
    
    // 检查 ZIP 魔数（前 2 字节 = "PK"）
    if (header[0] == 'P' && header[1] == 'K') {
        // 进一步验证：第 3-4 字节应为有效的 ZIP 签名
        // 本地文件头：03 04
        // 中央目录：01 02  
        // EOCD：05 06
        if ((header[2] == 0x03 && header[3] == 0x04) ||
            (header[2] == 0x05 && header[3] == 0x06)) {
            fclose(fp);
            return FORMAT_ZIP;
        }
    }
    
    // 检查 7z 魔数（前 6 字节）
    const unsigned char sz_magic[] = {'7', 'z', 0xBC, 0xAF, 0x27, 0x1C};
    if (memcmp(header, sz_magic, 6) == 0) {
        fclose(fp);
        return FORMAT_7Z;
    }
    
    // 检查 TAR：seek 到偏移 257，查找 "ustar"
    if (fseek(fp, 257, SEEK_SET) == 0) {
        char ustar[6];
        if (fread(ustar, 1, 5, fp) == 5) {
            ustar[5] = '\0';
            if (strcmp(ustar, "ustar") == 0) {
                fclose(fp);
                return FORMAT_TAR;
            }
        }
    }
    
    fclose(fp);
```

### 练习 2：压缩率估算器

**参考答案**：

```cpp
#include <cstdio>
#include <cstring>
#include <vector>
#include <string>
#include <algorithm>

struct FileEntry {
    std::string name;
    size_t size;
};

struct CompressionEstimate {
    const char* format;
    size_t estimated_size;
    double ratio;
};

// 根据扩展名获取预估压缩率
double GetCompressionRatio(const char* ext, const char* format) {
    // 已压缩格式：几乎无法进一步压缩
    const char* compressed[] = {
        ".png", ".jpg", ".jpeg", ".gif", ".webp",
        ".ogg", ".mp3", ".aac", ".flac",
        ".mp4", ".webm",
        ".zip", ".7z", ".gz", ".xz",
        nullptr
    };
    
    // 文本/脚本格式：高压缩率
    const char* text[] = {
        ".txt", ".ks", ".csv", ".xml", ".json",
        ".tjs", ".html", ".css", ".js",
        nullptr
    };
    
    // 未压缩图像/音频：中等压缩率
    const char* uncompressed[] = {
        ".bmp", ".tga", ".ppm",
        ".wav", ".pcm", ".raw",
        nullptr
    };
    
    // 检查文件类型
    bool is_compressed = false;
    bool is_text = false;
    bool is_uncompressed = false;
    
    for (int i = 0; compressed[i]; i++) {
        if (strcmp(ext, compressed[i]) == 0) { is_compressed = true; break; }
    }
    for (int i = 0; text[i]; i++) {
        if (strcmp(ext, text[i]) == 0) { is_text = true; break; }
    }
    for (int i = 0; uncompressed[i]; i++) {
        if (strcmp(ext, uncompressed[i]) == 0) { is_uncompressed = true; break; }
    }
    
    // 各格式对不同类型文件的预估压缩率
    // 返回值含义：压缩后大小 / 原始大小（越小越好）
    if (strcmp(format, "XP3") == 0) {
        if (is_compressed)   return 0.98;  // zlib 对已压缩数据几乎无效
        if (is_text)         return 0.25;  // zlib 对文本效果好
        if (is_uncompressed) return 0.50;  // 中等效果
        return 0.70;                       // 未知类型保守估计
    }
    if (strcmp(format, "ZIP") == 0) {
        if (is_compressed)   return 0.99;  // deflate 同样无法压缩已压缩数据
        if (is_text)         return 0.28;  // 与 zlib 相当
        if (is_uncompressed) return 0.52;
        return 0.72;
    }
    if (strcmp(format, "TAR") == 0) {
        // TAR 不压缩，只有 512 字节对齐的开销
        return 1.0;
    }
    if (strcmp(format, "7z") == 0) {
        if (is_compressed)   return 0.97;  // LZMA 对已压缩数据略好
        if (is_text)         return 0.15;  // LZMA 对文本效果极好
        if (is_uncompressed) return 0.35;  // 固实压缩效果显著
        return 0.55;
    }
    
    return 1.0;
}

std::vector<CompressionEstimate> EstimateCompression(
    const std::vector<FileEntry>& files) {
    
    const char* formats[] = {"XP3", "ZIP", "TAR", "7z"};
    std::vector<CompressionEstimate> results;
    
    size_t total_original = 0;
    for (auto& f : files) total_original += f.size;
    
    for (const char* fmt : formats) {
        size_t estimated = 0;
        
        for (auto& f : files) {
            // 提取扩展名
            const char* ext = strrchr(f.name.c_str(), '.');
            if (!ext) ext = "";
            
            // 转小写（简化实现）
            char ext_lower[16] = {};
            for (int i = 0; ext[i] && i < 15; i++) {
                ext_lower[i] = (ext[i] >= 'A' && ext[i] <= 'Z') 
                    ? ext[i] + 32 : ext[i];
            }
            
            double ratio = GetCompressionRatio(ext_lower, fmt);
            estimated += (size_t)(f.size * ratio);
        }
        
        // TAR 额外开销：每文件 512 字节头 + 对齐
        if (strcmp(fmt, "TAR") == 0) {
            estimated += files.size() * 512;  // 头部
            for (auto& f : files) {
                size_t padding = (512 - (f.size % 512)) % 512;
                estimated += padding;  // 对齐填充
            }
        }
        
        results.push_back({
            fmt,
            estimated,
            (double)estimated / total_original
        });
    }
    
    return results;
}

int main() {
    // 模拟游戏资源文件列表
    std::vector<FileEntry> files = {
        {"scripts/main.ks",      15000},
        {"scripts/chapter1.ks",  32000},
        {"scripts/chapter2.ks",  28000},
        {"images/bg001.png",     524288},
        {"images/bg002.png",     614400},
        {"images/sprite.bmp",    1048576},
        {"sound/bgm01.ogg",      2097152},
        {"sound/se_click.wav",   65536},
        {"video/opening.mp4",    10485760},
    };
    
    size_t total = 0;
    for (auto& f : files) total += f.size;
    
    printf("文件总计：%zu 个，原始大小：%.2f MB\n\n", 
        files.size(), total / 1048576.0);
    
    auto estimates = EstimateCompression(files);
    
    printf("%-8s %12s %8s  %s\n", "格式", "预估大小", "压缩率", "评价");
    printf("%-8s %12s %8s  %s\n", "----", "--------", "------", "----");
    
    for (auto& est : estimates) {
        const char* comment = "";
        if (est.ratio < 0.7)  comment = "★ 推荐：体积显著减小";
        else if (est.ratio < 0.9) comment = "中等效果";
        else comment = "几乎无压缩效果";
        
        printf("%-8s %10.2f MB %6.1f%%  %s\n",
            est.format,
            est.estimated_size / 1048576.0,
            est.ratio * 100,
            comment);
    }
    
    return 0;
}

// 预期输出：
// 文件总计：9 个，原始大小：14.16 MB
//
// 格式       预估大小   压缩率  评价
// ----       --------   ------  ----
// XP3        13.16 MB   92.9%  几乎无压缩效果
// ZIP        13.23 MB   93.4%  几乎无压缩效果
// TAR        14.17 MB  100.0%  几乎无压缩效果
// 7z         12.47 MB   88.0%  中等效果
//
// 注意：因为测试数据中大部分是已压缩文件（png/ogg/mp4），
// 所以压缩率都不高。如果资源以 BMP/WAV 为主，差异会更明显。
```
