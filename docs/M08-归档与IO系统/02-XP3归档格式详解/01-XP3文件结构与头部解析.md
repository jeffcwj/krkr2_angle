# 第二章 第一节：XP3 文件结构与头部解析

## 术语预览

| 术语 | 英文 | 含义 |
|------|------|------|
| XP3 | XPK3 (XP3) | KiriKiri 引擎原生的归档文件格式，全称 "XPK version 3" |
| 魔数 | Magic Number | 文件开头用于标识格式类型的固定字节序列，XP3 的魔数为 11 字节 |

| segm 子块 | Segment Sub-chunk | 存储文件的分段信息，每段 28 字节，包含压缩标志、偏移和大小 |

| EXE 内嵌 | EXE Embedding | 将 XP3 归档数据附加到可执行文件末尾的技术，实现单文件分发 |
| 索引续读标志 | Index Continue Flag | 索引区标志位的第 7 位，为 1 时表示后续还有更多索引数据需要读取 |

## 1. XP3 格式概述

XP3（全称 XPK version 3）是 KiriKiri/TVP2 引擎的原生归档格式。与通用的 ZIP 或 TAR 格式不同，XP3 是专门为视觉小说游戏的资源管理场景设计的。它的核心设计目标是：**高效随机访问 + 可扩展加密 + 支持 EXE 内嵌**。

XP3 格式经历了两代演进：
- **XPK1/XPK2**：TVP 0.9x 版本使用的旧格式，已不再支持
- **XP3 (XPK3)**：TVP2 使用的现行格式，本节详细分析

### 1.1 XP3 文件整体结构

一个标准的 XP3 归档文件由三个主要部分组成：

```
┌──────────────────────────────────────────────┐
│  XP3 文件整体布局                              │
├──────────────────────────────────────────────┤
│                                              │
│  ┌────────────────────────────────────┐      │
│  │ 头部区（Header）                     │      │
│  │ 偏移 0x00: 11 字节魔数               │      │
│  │ 偏移 0x0B: 8 字节索引区偏移值         │      │
│  │ 总计 19 字节                         │      │
│  └────────────────────────────────────┘      │
│                                              │
│  ┌────────────────────────────────────┐      │
│  │ 数据区（Data Area）                  │      │
│  │ 文件 0 的段数据（可能已压缩）         │      │
│  │ 文件 1 的段数据                      │      │
│  │ ...                                 │      │
│  │ 文件 N 的段数据                      │      │
│  └────────────────────────────────────┘      │
│                                              │
│  ┌────────────────────────────────────┐      │
│  │ 索引区（Index Area）                 │      │
│  │ 1 字节编码标志                       │      │
│  │ 压缩/原始索引数据                    │      │
│  │ 内含所有 File 块                     │      │
│  └────────────────────────────────────┘      │
│                                              │
└──────────────────────────────────────────────┘
```

**关键设计决策**：索引区放在文件尾部而非头部。这使得归档的创建过程可以先写数据、最后写索引，避免了需要预先知道所有文件偏移量的问题。读取时，引擎先读头部获取索引偏移，跳到文件尾部读索引，然后通过索引中记录的偏移去访问数据区。

## 2. 头部区详解

### 2.1 魔数结构

XP3 的魔数总共 11 字节，被刻意分成两部分存储在源码中。分成两部分的原因是：**防止编译器将完整魔数序列存储在可执行文件的初始化数据段中**，避免在程序自身文件中误检测到 XP3 标记。

```cpp
// 源码位置：XP3Archive.cpp 第 217-251 行
// XP3 魔数的完整字节序列及每个字节的设计意图

// 第一部分：8 字节
static tjs_uint8 XP3Mark1[] = {
    0x58,  // 'X'  — 格式名称第一个字符
    0x50,  // 'P'  — 格式名称第二个字符
    0x33,  // '3'  — 格式版本号（ASCII '3'）
    0x0D,  // '\r' — 回车符，与下一个换行符组成 CR+LF
    0x0A,  // '\n' — 换行符
    0x20,  // ' '  — 空格
    0x0A,  // '\n' — 又一个换行符
    0x1A,  // EOF  — DOS 文本结束标记
    0xFF   // 哨兵值（不属于魔数，用于数组结束标记）
};

// 第二部分：3 字节
static tjs_uint8 XP3Mark2[] = {
    0x8B,  // 日文假名编码的一部分
    0x67,  // 日文假名编码的一部分
    0x01,  // 文件结构版本和字符编码标识
    0xFF   // 哨兵值
};

// 运行时拼接为完整的 11 字节魔数
static tjs_uint8 XP3Mark[11 + 1];  // +1 防止 DWORD 对齐越界读取
static bool DoInit = true;
if (DoInit) {
    DoInit = false;
    memcpy(XP3Mark, XP3Mark1, 8);      // 复制前 8 字节
    memcpy(XP3Mark + 8, XP3Mark2, 3);  // 复制后 3 字节
}
```

每个字节都有精心设计的用途：

| 偏移 | 字节值 | 含义 | 设计理由 |
|------|--------|------|---------|
| 0x00 | `0x58` ('X') | 格式标识 | 可读的格式名称 |
| 0x01 | `0x50` ('P') | 格式标识 | 可读的格式名称 |
| 0x02 | `0x33` ('3') | 版本号 | 区分 XPK1/XPK2/XP3 |
| 0x03 | `0x0D` (CR) | 回车符 | 检测 CR→LF 转换损坏 |
| 0x04 | `0x0A` (LF) | 换行符 | 检测换行符被修改 |
| 0x05 | `0x20` (空格) | 空格 | 文本可读区域的一部分 |
| 0x06 | `0x0A` (LF) | 换行符 | 再次检测换行符转换 |
| 0x07 | `0x1A` (EOF) | DOS EOF | 阻止 `type` 命令继续输出 |
| 0x08 | `0x8B` | 日文编码 | 检测字符编码转换损坏 |
| 0x09 | `0x67` | 日文编码 | 同上 |
| 0x0A | `0x01` | 版本+编码 | 高 4 位=结构版本(0)，低 4 位=字符编码(1=BMP Unicode) |

> **为什么检测这么多种"损坏"？** 在早期互联网时代，文件经常通过 FTP 的 ASCII 模式传输，或被文本编辑器以"文本文件"方式打开。这些操作可能自动转换换行符（CR+LF → LF）或修改字符编码。XP3 魔数中嵌入了这些"陷阱字节"，如果文件被错误转换，魔数就不再匹配，能立即检测出问题。

### 2.2 索引偏移字段

紧跟在 11 字节魔数之后的是一个 8 字节的小端序整数，存储索引区在文件中的绝对偏移位置：

```cpp
// 源码位置：XP3Archive.cpp Init() 第 345-353 行
// 读取索引区偏移

// 跳过 11 字节魔数
st->SetPosition(11 + offset);  // offset 是 EXE 内嵌时的基础偏移

// 读取 8 字节小端序整数作为索引区偏移
tjs_uint64 index_ofs = st->ReadI64LE();

// 跳转到索引区
st->SetPosition(index_ofs + offset);
```

```
头部区内存布局（共 19 字节）：

偏移    大小    内容              说明
───────────────────────────────────────────
0x00    11      XP3 魔数          格式标识 + 完整性检测
0x0B    8       索引偏移(uint64)  指向索引区的绝对文件偏移
───────────────────────────────────────────
总计    19 字节
```

### 2.3 完整的头部解析实现

```cpp
// 示例：完整的 XP3 头部解析器
// 编译：g++ -std=c++17 xp3_header.cpp -o xp3_header
#include <cstdio>
#include <cstdint>
#include <cstring>

// XP3 魔数（完整 11 字节）
static const uint8_t XP3_MAGIC[11] = {
    0x58, 0x50, 0x33,   // "XP3"
    0x0D, 0x0A,         // CR LF
    0x20, 0x0A,         // 空格 LF  
    0x1A,               // DOS EOF
    0x8B, 0x67,         // 日文编码检测
    0x01                // 版本 0 + 编码 1 (BMP Unicode)
};

struct XP3Header {
    uint8_t magic[11];          // 魔数
    uint64_t index_offset;      // 索引区偏移
};

// 从字节数组读取小端序 64 位整数
uint64_t ReadLE64(const uint8_t* p) {
    return (uint64_t)p[0]       | ((uint64_t)p[1] << 8)  |
           ((uint64_t)p[2] << 16) | ((uint64_t)p[3] << 24) |
           ((uint64_t)p[4] << 32) | ((uint64_t)p[5] << 40) |
           ((uint64_t)p[6] << 48) | ((uint64_t)p[7] << 56);
}

// 解析 XP3 头部
// 返回值：true 表示成功，false 表示不是 XP3 格式
bool ParseXP3Header(FILE* fp, XP3Header* header, uint64_t base_offset) {
    // 定位到基础偏移（EXE 内嵌时不为 0）
    fseek(fp, (long)base_offset, SEEK_SET);
    
    // 读取 19 字节头部
    uint8_t raw[19];
    if (fread(raw, 1, 19, fp) != 19) {
        printf("错误：文件太小，无法读取完整头部\n");
        return false;
    }
    
    // 验证魔数
    if (memcmp(raw, XP3_MAGIC, 11) != 0) {
        printf("错误：魔数不匹配\n");
        // 逐字节对比，找出具体哪个字节不同
        for (int i = 0; i < 11; i++) {
            if (raw[i] != XP3_MAGIC[i]) {
                printf("  偏移 0x%02X: 期望 0x%02X, 实际 0x%02X\n",
                    i, XP3_MAGIC[i], raw[i]);
            }
        }
        return false;
    }
    
    // 复制魔数
    memcpy(header->magic, raw, 11);
    
    // 解析索引偏移（小端序）
    header->index_offset = ReadLE64(raw + 11);
    
    printf("XP3 头部解析成功：\n");
    printf("  魔数：XP3 (有效)\n");
    printf("  结构版本：%d\n", (raw[10] >> 4) & 0x0F);
    printf("  字符编码：%d (1=BMP Unicode)\n", raw[10] & 0x0F);
    printf("  索引区偏移：0x%llX (%llu 字节)\n",
        (unsigned long long)header->index_offset,
        (unsigned long long)header->index_offset);
    
    return true;
}

int main(int argc, char* argv[]) {
    if (argc < 2) {
        printf("用法：%s <XP3文件路径>\n", argv[0]);
        return 1;
    }
    
    FILE* fp = fopen(argv[1], "rb");
    if (!fp) {
        printf("无法打开文件：%s\n", argv[1]);
        return 1;
    }
    
    // 获取文件大小
    fseek(fp, 0, SEEK_END);
    long file_size = ftell(fp);
    printf("文件大小：%ld 字节\n\n", file_size);
    
    XP3Header header;
    bool ok = ParseXP3Header(fp, &header, 0);
    
    if (!ok) {
        printf("\n尝试检测 EXE 内嵌...\n");
        // 检查是否是 MZ (EXE) 头部
        fseek(fp, 0, SEEK_SET);
        uint8_t mz[2];
        fread(mz, 1, 2, fp);
        if (mz[0] == 0x4D && mz[1] == 0x5A) {
            printf("检测到 MZ 头部，扫描 XP3 标记...\n");
            // 从偏移 16 开始，按 16 字节对齐扫描
            // （与 KrKr2 源码 TVPGetXP3ArchiveOffset 逻辑一致）
        }
    }
    
    fclose(fp);
    return ok ? 0 : 1;
}
```

## 3. EXE 内嵌机制

XP3 格式的一个独特特性是支持**将归档数据附加到 Windows 可执行文件 (.exe) 的末尾**。这样用户只需要一个 .exe 文件就能运行游戏，而不需要额外的 .xp3 文件。

### 3.1 EXE 内嵌的检测原理

```cpp
// 源码位置：XP3Archive.cpp 第 213-306 行
// TVPGetXP3ArchiveOffset() — 检测 XP3 在文件中的起始偏移

bool TVPGetXP3ArchiveOffset(tTJSBinaryStream *st, const ttstr name,
                            tjs_uint64 &offset, bool raise) {
    st->SetPosition(0);
    tjs_uint8 mark[11 + 1];
    
    // 读取前 11 字节
    mark[0] = 0;  // 哨兵值
    st->ReadBuffer(mark, 11);
    
    if (mark[0] == 0x4D /*'M'*/ && mark[1] == 0x5A /*'Z'*/) {
        // 检测到 "MZ" — 这是 Windows PE/DOS 可执行文件的标志
        // XP3 数据被附加在 EXE 的末尾
        // 从偏移 16 开始，按 16 字节对齐扫描 XP3 魔数
        
        bool found = false;
        offset = 16;  // 从偏移 16 开始（跳过 DOS 头）
        st->SetPosition(16);
        
        // 每次读取 256KB 进行扫描
        const tjs_uint one_read_size = 256 * 1024;
        tjs_uint8 *buffer = new tjs_uint8[one_read_size];
        tjs_uint read;
        
        while (0 != (read = st->Read(buffer, one_read_size))) {
            tjs_uint p = 0;
            while (p < read) {
                if (!memcmp(XP3Mark, buffer + p, 11)) {
                    // 找到 XP3 魔数
                    offset += p;
                    found = true;
                    break;
                }
                p += 16;  // 按 16 字节对齐步进
            }
            if (found) break;
            offset += one_read_size;
        }
        delete[] buffer;
        
        if (!found) {
            // EXE 中没有找到 XP3 标记
            if (raise) TVPThrowExceptionMessage(TVPCannotUnbindXP3EXE, name);
            return false;
        }
        
    } else if (!memcmp(XP3Mark, mark, 11)) {
        // 直接就是 XP3 文件（不是 EXE 内嵌）
        offset = 0;
    } else {
        // 既不是 EXE 也不是 XP3
        if (raise) TVPThrowExceptionMessage(/*...*/);
        return false;
    }
    
    return true;
}
```

### 3.2 16 字节对齐的意义

扫描时按 16 字节对齐步进，这个设计基于两个考量：

1. **性能优化**：如果逐字节扫描一个几十 MB 的 EXE 文件，耗时会很长。按 16 字节步进可以减少比较次数约 16 倍
2. **格式约定**：XP3 工具在将归档附加到 EXE 末尾时，会确保 XP3 数据从 16 字节对齐的位置开始

```cpp
// 示例：EXE 内嵌 XP3 的文件结构
// 演示如何构建和解析内嵌归档

#include <cstdio>
#include <cstdint>
#include <cstring>
#include <vector>

// EXE + XP3 的混合文件结构：
//
// ┌──────────────────────────────────┐ 偏移 0
// │ MZ 头部 (DOS header)             │
// │ ...                              │
// │ PE 头部和节表                     │
// │ ...                              │
// │ 代码节 (.text)                    │
// │ 数据节 (.data)                    │
// │ 资源节 (.rsrc)                    │
// │ ...                              │
// ├──────────────────────────────────┤ 对齐到 16 字节
// │ XP3 魔数 (11 字节)                │ ← XP3 起始偏移 (base_offset)
// │ 索引偏移 (8 字节)                  │
// │ 数据区...                         │
// │ 索引区...                         │
// └──────────────────────────────────┘ 文件末尾

// 模拟 EXE 内嵌 XP3 的扫描过程
uint64_t ScanForXP3InEXE(FILE* fp, long file_size) {
    static const uint8_t XP3_MAGIC[11] = {
        0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A, 0x1A,
        0x8B, 0x67, 0x01
    };
    
    // 分配 256KB 扫描缓冲区
    const size_t SCAN_BUF_SIZE = 256 * 1024;
    std::vector<uint8_t> buffer(SCAN_BUF_SIZE);
    
    uint64_t offset = 16;  // 从偏移 16 开始
    fseek(fp, 16, SEEK_SET);
    
    size_t read;
    while ((read = fread(buffer.data(), 1, SCAN_BUF_SIZE, fp)) > 0) {
        // 按 16 字节对齐扫描
        for (size_t p = 0; p + 11 <= read; p += 16) {
            if (memcmp(buffer.data() + p, XP3_MAGIC, 11) == 0) {
                printf("在偏移 0x%llX 处找到 XP3 标记\n",
                    (unsigned long long)(offset + p));
                return offset + p;
            }
        }
        offset += read;
    }
    
    printf("未找到 XP3 标记\n");
    return UINT64_MAX;  // 未找到
}

// 测试程序
int main() {
    printf("=== XP3 EXE 内嵌扫描演示 ===\n\n");
    
    // 构造一个模拟的 EXE+XP3 文件用于演示
    const char* test_file = "test_embedded.dat";
    FILE* fp = fopen(test_file, "wb");
    if (!fp) { printf("无法创建测试文件\n"); return 1; }
    
    // 写入模拟的 MZ 头部
    uint8_t mz_header[256] = {};
    mz_header[0] = 0x4D;  // 'M'
    mz_header[1] = 0x5A;  // 'Z'
    fwrite(mz_header, 1, 256, fp);
    
    // 填充到 16 字节对齐的位置（假设 EXE 部分有 1024 字节）
    uint8_t padding[768] = {};
    fwrite(padding, 1, 768, fp);  // 256 + 768 = 1024
    
    // 在偏移 1024 处写入 XP3 魔数（16 字节对齐）
    uint8_t xp3_magic[11] = {
        0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A, 0x1A,
        0x8B, 0x67, 0x01
    };
    fwrite(xp3_magic, 1, 11, fp);
    
    // 写入索引偏移（指向后面的位置）
    uint64_t index_offset = 1024 + 19 + 100;  // 假设数据区 100 字节
    fwrite(&index_offset, 8, 1, fp);
    
    fclose(fp);
    
    // 重新打开并扫描
    fp = fopen(test_file, "rb");
    fseek(fp, 0, SEEK_END);
    long size = ftell(fp);
    
    printf("测试文件大小：%ld 字节\n", size);
    uint64_t found = ScanForXP3InEXE(fp, size);
    
    if (found != UINT64_MAX) {
        printf("XP3 基础偏移：0x%llX (%llu)\n",
            (unsigned long long)found, (unsigned long long)found);
    }
    
    fclose(fp);
    remove(test_file);  // 清理测试文件
    return 0;
}
```

## 4. 索引区结构

索引区是 XP3 格式的核心，包含了归档中所有文件的元信息。理解索引区的结构是实现 XP3 读取器的关键。

### 4.1 索引区头部

```
索引区开头的字节布局：

偏移    大小    内容                  说明
───────────────────────────────────────────────────────
0x00    1       index_flag            编码标志
                                      低 3 位 (& 0x07)：编码方式
                                        0 = 原始数据（RAW）
                                        1 = zlib 压缩
                                      第 7 位 (& 0x80)：续读标志
                                        0 = 这是最后一块索引
                                        1 = 后面还有更多索引

[如果编码方式 = zlib]
0x01    8       compressed_size       压缩后大小 (uint64 LE)
0x09    8       uncompressed_size     原始大小 (uint64 LE)
0x11    变长    compressed_data       zlib 压缩的索引数据

[如果编码方式 = RAW]
0x01    8       index_size            索引数据大小 (uint64 LE)
0x09    变长    index_data            原始索引数据
```

```cpp
// 源码位置：XP3Archive.cpp Init() 第 356-404 行
// 索引区读取的两种模式

// 读取编码标志
tjs_uint8 index_flag;
st->ReadBuffer(&index_flag, 1);
tjs_uint index_size;

if ((index_flag & TVP_XP3_INDEX_ENCODE_METHOD_MASK) ==
    TVP_XP3_INDEX_ENCODE_ZLIB) {
    // === 模式 1：zlib 压缩的索引 ===
    tjs_uint64 compressed_size = st->ReadI64LE();
    tjs_uint64 r_index_size = st->ReadI64LE();
    
    // 安全检查：大小不能超过 32 位范围（防止内存溢出攻击）
    if ((tjs_uint)compressed_size != compressed_size ||
        (tjs_uint)r_index_size != r_index_size)
        TVPThrowExceptionMessage(TVPReadError);
    
    index_size = (tjs_int)r_index_size;
    indexdata = new tjs_uint8[index_size];
    tjs_uint8 *compressed = new tjs_uint8[(tjs_uint)compressed_size];
    
    // 读取压缩数据
    st->ReadBuffer(compressed, (tjs_uint)compressed_size);
    
    // 解压
    unsigned long destlen = (unsigned long)index_size;
    int result = uncompress(
        (unsigned char *)indexdata, &destlen,
        (unsigned char *)compressed, (unsigned long)compressed_size);
    
    if (result != Z_OK || destlen != (unsigned long)index_size)
        TVPThrowExceptionMessage(TVPUncompressionFailed);
    
    delete[] compressed;
    
} else if ((index_flag & TVP_XP3_INDEX_ENCODE_METHOD_MASK) ==
           TVP_XP3_INDEX_ENCODE_RAW) {
    // === 模式 2：未压缩的原始索引 ===
    tjs_uint64 r_index_size = st->ReadI64LE();
    if ((tjs_uint)r_index_size != r_index_size)
        TVPThrowExceptionMessage(TVPReadError);
    
    index_size = (tjs_uint)r_index_size;
    indexdata = new tjs_uint8[index_size];
    st->ReadBuffer(indexdata, index_size);
    
} else {
    // 未知编码方式 → 格式错误
    TVPThrowExceptionMessage(TVPReadError);
}
```

### 4.2 索引续读机制

XP3 支持索引分成多个部分存储。当索引标志的第 7 位（`0x80`）被设置时，表示读取完当前索引后，还需要回到头部区读取下一个索引偏移。这个设计允许增量更新归档——只需在文件末尾追加新索引，不需要重写整个文件。

```cpp
// 源码位置：XP3Archive.cpp Init() 第 348-502 行
// 索引续读循环

// 读取所有索引部分
while (true) {
    // ... 读取并解析当前索引部分 ...
    
    // 检查续读标志
    if (!(index_flag & TVP_XP3_INDEX_CONTINUE))
        break;  // 没有更多索引了
    
    // 有续读标志 → 继续读取下一部分
    // （循环回到 while 开头，读取新的 index_ofs）
}

// TVP_XP3_INDEX_CONTINUE 的定义
#define TVP_XP3_INDEX_CONTINUE 0x80  // 位于 XP3Archive.h 第 74 行
```

```
索引续读场景示意：

初始归档 v1.0：
┌─────────────┬──────────┬─────────────────────┐
│ 头部(19B)   │ 数据区   │ 索引区(flag=0x01)    │
│ magic+ofs_1→│          │ ← ofs_1              │
└─────────────┴──────────┴─────────────────────┘
                                flag=0x01 表示 zlib 压缩 + 无续读

更新后 v1.1（追加了新文件）：
┌─────────────┬──────────┬────────────────────┬──────────┬─────────────────────┐
│ 头部(19B)   │ 旧数据区 │ 旧索引区(flag=0x81)│ 新数据区 │ 新索引区(flag=0x01)  │
│ magic+ofs_1→│          │ ← ofs_1            │          │ ← ofs_2             │
└─────────────┴──────────┴────────────────────┴──────────┴─────────────────────┘
                                flag=0x81 = zlib + 续读标志
注意：ofs_2 的位置记录在新索引区之前（实际实现中
      可能需要修改头部或在旧索引区结尾存储新的偏移）
```

## 5. Chunk（块）解析

索引数据解压后，是一系列 Chunk（块）的顺序排列。XP3 的块结构是一种简洁的类型-长度-值（TLV，Type-Length-Value）编码。TLV 是一种通用的数据编码方式：先用固定字节表示数据类型，再用固定字节表示数据长度，最后是实际数据。这种结构的优点是解析器可以跳过不认识的块类型，实现前向兼容。

### 5.1 块结构定义

```
单个 Chunk 的二进制布局：

偏移    大小    内容
──────────────────────────
0x00    4       ChunkName      4 个 ASCII 字符的类型标识
0x04    8       ChunkSize      块数据大小（uint64 小端序）
0x0C    变长    ChunkData      块数据（大小由 ChunkSize 指定）
──────────────────────────
总计    12 + ChunkSize 字节
```

### 5.2 FindChunk 实现

KrKr2 使用 `FindChunk` 函数在索引数据中定位指定类型的块。这个函数是一个线性扫描器——从指定的起始位置开始，逐块检查类型名称直到找到匹配的块。

```cpp
// 源码位置：XP3Archive.cpp 第 584-610 行
// FindChunk — 在索引数据中查找指定类型的块

bool tTVPXP3Archive::FindChunk(
    const tjs_uint8 *data,   // 索引数据缓冲区
    const tjs_uint8 *name,   // 要查找的 4 字节类型名
    tjs_uint &start,          // 输入：搜索起始位置；输出：块数据起始位置
    tjs_uint &size)           // 输入：搜索范围大小；输出：块数据大小
{
    tjs_uint start_save = start;  // 保存原始位置（找不到时恢复）
    tjs_uint size_save = size;
    
    tjs_uint pos = 0;
    while (pos < size) {
        // 比较 4 字节类型名
        bool found = !memcmp(data + start, name, 4);
        start += 4;  // 跳过类型名
        
        // 读取 8 字节块大小
        tjs_uint64 r_size = ReadI64FromMem(data + start);
        start += 8;  // 跳过大小字段
        
        tjs_uint size_chunk = (tjs_uint)r_size;
        if (size_chunk != r_size)
            TVPThrowExceptionMessage(TVPReadError);  // 大小溢出
        
        if (found) {
            // 找到目标块
            // start 现在指向块数据的起始位置
            size = size_chunk;
            return true;
        }
        
        // 没找到，跳过这个块的数据，继续搜索下一个
        start += size_chunk;
        pos += size_chunk + 4 + 8;  // 4(类型) + 8(大小) + 数据
    }
    
    // 没有找到，恢复原始位置
    start = start_save;
    size = size_save;
    return false;
}
```

### 5.3 顶层 File 块

索引数据的顶层是一系列 `File` 块，每个 `File` 块描述归档中的一个文件。`File` 块内部嵌套了三个子块：

```
File 块的嵌套结构：

File（顶层块，类型标识 "File"）
├── info（子块，类型标识 "info"）
│   └── 文件基本信息：标志、大小、文件名
├── segm（子块，类型标识 "segm"）
│   └── 分段列表：每段 28 字节
└── adlr（子块，类型标识 "adlr"）
    └── 文件哈希值：4 字节
```

```cpp
// 源码位置：XP3Archive.cpp Init() 第 406-498 行
// File 块的遍历解析

// 块类型名常量（4 字节 ASCII）
static const tjs_uint8 cn_File[] = { 'F', 'i', 'l', 'e' };
static const tjs_uint8 cn_info[] = { 'i', 'n', 'f', 'o' };
static const tjs_uint8 cn_segm[] = { 's', 'e', 'g', 'm' };
static const tjs_uint8 cn_adlr[] = { 'a', 'd', 'l', 'r' };

tjs_uint ch_file_start = 0;
tjs_uint ch_file_size = index_size;

for (;;) {
    // 1. 在索引数据中查找下一个 File 块
    if (!FindChunk(indexdata, cn_File, ch_file_start, ch_file_size))
        break;  // 没有更多 File 块了
    
    // 2. 在 File 块内查找 info 子块
    tjs_uint ch_info_start = ch_file_start;
    tjs_uint ch_info_size = ch_file_size;
    if (!FindChunk(indexdata, cn_info, ch_info_start, ch_info_size))
        TVPThrowExceptionMessage(TVPReadError);  // info 是必须的
    
    // 3. 解析 info 子块（见下一小节）
    // ...
    
    // 4. 在 File 块内查找 segm 子块
    tjs_uint ch_segm_start = ch_file_start;
    tjs_uint ch_segm_size = ch_file_size;
    if (!FindChunk(indexdata, cn_segm, ch_segm_start, ch_segm_size))
        TVPThrowExceptionMessage(TVPReadError);  // segm 是必须的
    
    // 5. 解析 segm 子块（见下一小节）
    // ...
    
    // 6. 在 File 块内查找 adlr 子块
    tjs_uint ch_adlr_start = ch_file_start;
    tjs_uint ch_adlr_size = ch_file_size;
    if (!FindChunk(indexdata, cn_adlr, ch_adlr_start, ch_adlr_size))
        TVPThrowExceptionMessage(TVPReadError);  // adlr 是必须的
    
    // 7. 解析 adlr 子块
    item.FileHash = ReadI32FromMem(indexdata + ch_adlr_start);
    
    // 8. 将文件信息加入列表
    ItemVector.push_back(item);
    
    // 9. 移动到下一个 File 块
    ch_file_start += ch_file_size;
    ch_file_size = index_size - ch_file_start;
    Count++;
}
```

## 6. 子块详解

### 6.1 info 子块

`info` 子块存储文件的基本元信息：

```
info 子块数据布局：

偏移    大小    类型        内容
────────────────────────────────────────
0x00    4       uint32      flags        保护标志
                                         bit 31: 受保护标志
0x04    8       uint64      OrgSize      原始大小（解压后）
0x0C    8       uint64      ArcSize      归档大小（所有段的归档大小之和）
0x14    2       uint16      NameLen      文件名长度（BMP Unicode 字符数）
0x16    变长    uint16[]    Name         文件名（BMP Unicode 编码）
────────────────────────────────────────
总计    22 + NameLen*2 字节
```

```cpp
// 源码位置：XP3Archive.cpp Init() 第 422-437 行
// info 子块解析

tArchiveItem item;

// 读取标志（4 字节）
tjs_uint32 flags = ReadI32FromMem(indexdata + ch_info_start + 0);

// 检查保护标志
// TVP_XP3_FILE_PROTECTED = (1 << 31) = 0x80000000
if (!TVPAllowExtractProtectedStorage &&
    (flags & TVP_XP3_FILE_PROTECTED))
    TVPThrowExceptionMessage(TVPSpecifiedStorageHadBeenProtected);

// 读取原始大小和归档大小
item.OrgSize = ReadI64FromMem(indexdata + ch_info_start + 4);   // 偏移 4
item.ArcSize = ReadI64FromMem(indexdata + ch_info_start + 12);  // 偏移 12

// 读取文件名
tjs_int len = ReadI16FromMem(indexdata + ch_info_start + 20);  // 偏移 20
// 从 BMP Unicode (16位) 转换为内部字符串
ttstr name = TVPStringFromBMPUnicode(
    (const tjs_uint16 *)(indexdata + ch_info_start + 22), len);
item.Name = name;

// 规范化文件名（小写 + 统一分隔符）
if (normalizeName)
    NormalizeInArchiveStorageName(item.Name);
```

### 6.2 segm 子块

`segm` 子块是分段信息的数组，每段固定 28 字节。一个文件可以被分成多个段，每段可以独立选择是否压缩。这种设计使得大文件的随机访问不需要解压整个文件——只需要解压包含目标数据的那几个段。

```
segm 子块数据布局：

每段 28 字节，结构如下：
偏移    大小    类型        内容
────────────────────────────────────────
0x00    4       uint32      flags        段编码标志
                                         低 3 位 (& 0x07)：
                                           0 = 未压缩 (RAW)
                                           1 = zlib 压缩
0x04    8       uint64      Start        段数据在归档文件中的起始偏移
0x0C    8       uint64      OrgSize      段的原始大小（解压后）
0x14    8       uint64      ArcSize      段在归档中的大小（压缩后）
────────────────────────────────────────
总计    28 字节/段

注意：段的 Offset（在解压后文件中的偏移）不存储在 segm 中，
而是在解析时通过累加 OrgSize 计算得出。
```

```cpp
// 源码位置：XP3Archive.cpp Init() 第 450-480 行
// segm 子块解析

// 计算段数量（每段 28 字节）
tjs_int segment_count = ch_segm_size / 28;

// 累计偏移量（段在解压后文件中的位置）
tjs_uint64 offset_in_archive = 0;

for (tjs_int i = 0; i < segment_count; i++) {
    tjs_uint pos_base = i * 28 + ch_segm_start;
    tTVPXP3ArchiveSegment seg;
    
    // 读取段编码标志
    tjs_uint32 flags = ReadI32FromMem(indexdata + pos_base);
    
    if ((flags & TVP_XP3_SEGM_ENCODE_METHOD_MASK) == TVP_XP3_SEGM_ENCODE_RAW)
        seg.IsCompressed = false;  // 未压缩
    else if ((flags & TVP_XP3_SEGM_ENCODE_METHOD_MASK) == TVP_XP3_SEGM_ENCODE_ZLIB)
        seg.IsCompressed = true;   // zlib 压缩
    else
        TVPThrowExceptionMessage(TVPReadError);  // 未知编码
    
    // 段在归档文件中的起始偏移
    // 注意：要加上 offset（EXE 内嵌时的基础偏移）
    seg.Start = ReadI64FromMem(indexdata + pos_base + 4) + offset;
    
    // 段在解压后文件中的偏移（通过累加计算）
    seg.Offset = offset_in_archive;
    
    // 段的原始大小和归档大小
    seg.OrgSize = ReadI64FromMem(indexdata + pos_base + 12);
    seg.ArcSize = ReadI64FromMem(indexdata + pos_base + 20);
    
    item.Segments.push_back(seg);
    offset_in_archive += seg.OrgSize;  // 累加到下一段的偏移
    segmentcount++;
}
```

## 7. 完整的 XP3 解析器实现

将以上所有知识整合，实现一个完整的独立 XP3 解析器：

```cpp
// 示例：完整的 XP3 归档解析器
// 编译：g++ -std=c++17 -lz xp3_parser.cpp -o xp3_parser
// 需要链接 zlib（-lz）
#include <cstdio>
#include <cstdint>
#include <cstring>
#include <vector>
#include <string>
#include <zlib.h>

// ===== 工具函数 =====

uint16_t ReadLE16(const uint8_t* p) {
    return (uint16_t)p[0] | ((uint16_t)p[1] << 8);
}

uint32_t ReadLE32(const uint8_t* p) {
    return (uint32_t)p[0] | ((uint32_t)p[1] << 8) |
           ((uint32_t)p[2] << 16) | ((uint32_t)p[3] << 24);
}

uint64_t ReadLE64(const uint8_t* p) {
    return (uint64_t)p[0]       | ((uint64_t)p[1] << 8) |
           ((uint64_t)p[2] << 16) | ((uint64_t)p[3] << 24) |
           ((uint64_t)p[4] << 32) | ((uint64_t)p[5] << 40) |
           ((uint64_t)p[6] << 48) | ((uint64_t)p[7] << 56);
}

// ===== 数据结构 =====

struct XP3Segment {
    bool is_compressed;
    uint64_t start;       // 归档中偏移
    uint64_t offset;      // 解压后文件中偏移
    uint64_t org_size;    // 原始大小
    uint64_t arc_size;    // 归档中大小
};

struct XP3FileEntry {
    std::string name;
    uint32_t flags;
    uint32_t file_hash;
    uint64_t org_size;
    uint64_t arc_size;
    std::vector<XP3Segment> segments;
};

// ===== Chunk 查找 =====

bool FindChunk(const uint8_t* data, const char* name,
               uint32_t& start, uint32_t& size) {
    uint32_t s_save = start, sz_save = size;
    uint32_t pos = 0;
    
    while (pos < size) {
        bool found = (memcmp(data + start, name, 4) == 0);
        start += 4;
        uint64_t chunk_size = ReadLE64(data + start);
        start += 8;
        
        if (chunk_size > UINT32_MAX) {
            start = s_save; size = sz_save;
            return false;
        }
        
        if (found) {
            size = (uint32_t)chunk_size;
            return true;
        }
        
        start += (uint32_t)chunk_size;
        pos += (uint32_t)chunk_size + 12;
    }
    
    start = s_save; size = sz_save;
    return false;
}

// ===== UTF-16LE → UTF-8 简易转换 =====

std::string UTF16LEToUTF8(const uint16_t* src, int len) {
    std::string result;
    for (int i = 0; i < len; i++) {
        uint16_t c = src[i];
        if (c < 0x80) {
            result += (char)c;
        } else if (c < 0x800) {
            result += (char)(0xC0 | (c >> 6));
            result += (char)(0x80 | (c & 0x3F));
        } else {
            result += (char)(0xE0 | (c >> 12));
            result += (char)(0x80 | ((c >> 6) & 0x3F));
            result += (char)(0x80 | (c & 0x3F));
        }
    }
    return result;
}

// ===== XP3 解析主函数 =====

std::vector<XP3FileEntry> ParseXP3(const char* filename) {
    std::vector<XP3FileEntry> entries;
    
    FILE* fp = fopen(filename, "rb");
    if (!fp) { printf("无法打开文件\n"); return entries; }
    
    // 读取并验证魔数
    uint8_t header[19];
    fread(header, 1, 19, fp);
    
    static const uint8_t MAGIC[11] = {
        0x58,0x50,0x33, 0x0D,0x0A,0x20,0x0A, 0x1A, 0x8B,0x67,0x01
    };
    if (memcmp(header, MAGIC, 11) != 0) {
        printf("不是 XP3 文件\n");
        fclose(fp);
        return entries;
    }
    
    // 读取索引偏移
    uint64_t index_offset = ReadLE64(header + 11);
    printf("索引区偏移：0x%llX\n", (unsigned long long)index_offset);
    
    // 跳转到索引区
    fseek(fp, (long)index_offset, SEEK_SET);
    
    // 读取索引标志
    uint8_t index_flag;
    fread(&index_flag, 1, 1, fp);
    
    int encode_method = index_flag & 0x07;
    printf("索引编码方式：%s\n", encode_method == 1 ? "zlib压缩" : "原始");
    
    // 读取索引数据
    uint8_t* indexdata = nullptr;
    uint32_t index_size = 0;
    
    if (encode_method == 1) {
        // zlib 压缩
        uint64_t comp_size = 0, uncomp_size = 0;
        fread(&comp_size, 8, 1, fp);
        fread(&uncomp_size, 8, 1, fp);
        
        index_size = (uint32_t)uncomp_size;
        indexdata = new uint8_t[index_size];
        uint8_t* compressed = new uint8_t[(uint32_t)comp_size];
        
        fread(compressed, 1, (uint32_t)comp_size, fp);
        
        unsigned long destlen = index_size;
        int ret = uncompress(indexdata, &destlen,
                            compressed, (unsigned long)comp_size);
        delete[] compressed;
        
        if (ret != Z_OK) {
            printf("索引解压失败：%d\n", ret);
            delete[] indexdata;
            fclose(fp);
            return entries;
        }
        printf("索引大小：压缩 %llu → 解压 %u 字节\n",
            (unsigned long long)comp_size, index_size);
    } else {
        // 原始数据
        uint64_t raw_size = 0;
        fread(&raw_size, 8, 1, fp);
        index_size = (uint32_t)raw_size;
        indexdata = new uint8_t[index_size];
        fread(indexdata, 1, index_size, fp);
    }
    
    fclose(fp);
    
    // 解析 File 块
    uint32_t file_start = 0;
    uint32_t file_size = index_size;
    
    while (FindChunk(indexdata, "File", file_start, file_size)) {
        XP3FileEntry entry;
        
        // 解析 info 子块
        uint32_t info_start = file_start, info_size = file_size;
        if (FindChunk(indexdata, "info", info_start, info_size)) {
            entry.flags = ReadLE32(indexdata + info_start);
            entry.org_size = ReadLE64(indexdata + info_start + 4);
            entry.arc_size = ReadLE64(indexdata + info_start + 12);
            uint16_t name_len = ReadLE16(indexdata + info_start + 20);
            entry.name = UTF16LEToUTF8(
                (const uint16_t*)(indexdata + info_start + 22), name_len);
        }
        
        // 解析 segm 子块
        uint32_t segm_start = file_start, segm_size = file_size;
        if (FindChunk(indexdata, "segm", segm_start, segm_size)) {
            int seg_count = segm_size / 28;
            uint64_t offset_acc = 0;
            for (int i = 0; i < seg_count; i++) {
                uint32_t base = segm_start + i * 28;
                XP3Segment seg;
                uint32_t flags = ReadLE32(indexdata + base);
                seg.is_compressed = (flags & 0x07) == 1;
                seg.start = ReadLE64(indexdata + base + 4);
                seg.offset = offset_acc;
                seg.org_size = ReadLE64(indexdata + base + 12);
                seg.arc_size = ReadLE64(indexdata + base + 20);
                entry.segments.push_back(seg);
                offset_acc += seg.org_size;
            }
        }
        
        // 解析 adlr 子块
        uint32_t adlr_start = file_start, adlr_size = file_size;
        if (FindChunk(indexdata, "adlr", adlr_start, adlr_size)) {
            entry.file_hash = ReadLE32(indexdata + adlr_start);
        }
        
        entries.push_back(entry);
        
        // 移动到下一个 File 块
        file_start += file_size;
        file_size = index_size - file_start;
    }
    
    delete[] indexdata;
    return entries;
}

// ===== 主程序 =====

int main(int argc, char* argv[]) {
    if (argc < 2) {
        printf("用法：%s <XP3文件>\n", argv[0]);
        printf("解析 XP3 归档文件并列出所有文件信息\n");
        return 1;
    }
    
    printf("=== XP3 归档解析器 ===\n\n");
    
    auto entries = ParseXP3(argv[1]);
    
    printf("\n共 %zu 个文件：\n\n", entries.size());
    printf("%-40s %10s %10s %6s %10s\n",
        "文件名", "原始大小", "归档大小", "段数", "哈希值");
    printf("%-40s %10s %10s %6s %10s\n",
        "--------", "--------", "--------", "----", "--------");
    
    uint64_t total_org = 0, total_arc = 0;
    for (auto& e : entries) {
        printf("%-40s %10llu %10llu %6zu 0x%08X\n",
            e.name.c_str(),
            (unsigned long long)e.org_size,
            (unsigned long long)e.arc_size,
            e.segments.size(),
            e.file_hash);
        total_org += e.org_size;
        total_arc += e.arc_size;
    }
    
    printf("\n总计：原始 %.2f MB, 归档 %.2f MB, 压缩率 %.1f%%\n",
        total_org / 1048576.0, total_arc / 1048576.0,
        total_org > 0 ? 100.0 * total_arc / total_org : 0);
    
    return 0;
}
```

## 8. 字节序处理

XP3 格式统一使用**小端序**（Little-Endian）存储所有多字节整数。小端序是一种字节排列方式，低位字节存放在低地址处。例如，32 位整数 `0x01234567` 在小端序中的存储顺序是 `67 45 23 01`。

KrKr2 实现了三个从内存中读取小端序整数的辅助函数：

```cpp
// 源码位置：XP3Archive.cpp 第 613-632 行
// 小端序读取辅助函数

// 读取 16 位小端序整数
tjs_int16 tTVPXP3Archive::ReadI16FromMem(const tjs_uint8 *mem) {
    tjs_uint16 ret = (tjs_uint16)mem[0] | ((tjs_uint16)mem[1] << 8);
    return (tjs_int16)ret;
}

// 读取 32 位小端序整数
tjs_int32 tTVPXP3Archive::ReadI32FromMem(const tjs_uint8 *mem) {
    tjs_uint32 ret = (tjs_uint32)mem[0]       | ((tjs_uint32)mem[1] << 8) |
                     ((tjs_uint32)mem[2] << 16) | ((tjs_uint32)mem[3] << 24);
    return (tjs_int32)ret;
}

// 读取 64 位小端序整数
tjs_int64 tTVPXP3Archive::ReadI64FromMem(const tjs_uint8 *mem) {
    tjs_uint64 ret = (tjs_uint64)mem[0]       | ((tjs_uint64)mem[1] << 8)  |
                     ((tjs_uint64)mem[2] << 16) | ((tjs_uint64)mem[3] << 24) |
                     ((tjs_uint64)mem[4] << 32) | ((tjs_uint64)mem[5] << 40) |
                     ((tjs_uint64)mem[6] << 48) | ((tjs_uint64)mem[7] << 56);
    return (tjs_int64)ret;
}
```

**为什么不直接用指针强制转换？** 比如 `*(uint32_t*)mem`？原因有两个：

1. **字节对齐**：`mem` 可能不是 4 字节对齐的地址，在 ARM 等平台上直接访问未对齐地址会触发硬件异常
2. **大端序平台**：如果代码运行在大端序 CPU 上，直接转换会得到错误的值。逐字节组装保证了跨平台正确性

### 题目 1：手动解析 XP3 头部字节

**题目**：给定以下 19 字节的十六进制数据，解析出 XP3 头部信息：

```
58 50 33 0D 0A 20 0A 1A 8B 67 01 A0 05 00 00 00 00 00 00
```

请回答：
1. 魔数是否有效？
2. 文件结构版本号是多少？
3. 字符编码标识是什么？
4. 索引区偏移是多少（十进制和十六进制）？

<details>
<summary>查看答案</summary>

1. **魔数有效**。前 11 字节 `58 50 33 0D 0A 20 0A 1A 8B 67 01` 完全匹配 XP3 标准魔数。

2. **文件结构版本号 = 0**。第 11 个字节（偏移 0x0A）的值是 `0x01`，高 4 位 = `0x0 >> 4 = 0`。

3. **字符编码标识 = 1（BMP Unicode）**。`0x01 & 0x0F = 1`，表示使用 16 位 BMP Unicode 编码文件名。

4. **索引区偏移**：
   - 后 8 字节（小端序）：`A0 05 00 00 00 00 00 00`
   - 计算：`0xA0 + (0x05 << 8) = 160 + 1280 = 1440`
   - 十进制：**1440**
   - 十六进制：**0x05A0**

验证代码：
```cpp
#include <cstdio>
#include <cstdint>

int main() {
    uint8_t data[] = {
        0x58,0x50,0x33,0x0D,0x0A,0x20,0x0A,0x1A,0x8B,0x67,0x01,
        0xA0,0x05,0x00,0x00,0x00,0x00,0x00,0x00
    };
    
    // 版本和编码
    uint8_t ver_byte = data[10];
    printf("结构版本：%d\n", (ver_byte >> 4) & 0x0F);  // 0
    printf("字符编码：%d\n", ver_byte & 0x0F);          // 1
    
    // 索引偏移
    uint64_t offset = 0;
    for (int i = 0; i < 8; i++)
        offset |= ((uint64_t)data[11 + i] << (i * 8));
    printf("索引偏移：%llu (0x%llX)\n",
        (unsigned long long)offset, (unsigned long long)offset);
    // 输出：索引偏移：1440 (0x5A0)
    
    return 0;
}
```

### 题目 2：实现 Chunk 遍历器

**题目**：编写一个函数，遍历并打印索引数据中所有块的类型名称和大小。不需要递归到子块内部。

```cpp
// 函数签名
void ListChunks(const uint8_t* data, uint32_t size);
```

**示例输出**：
```
Chunk "File" — 大小: 156 字节
Chunk "File" — 大小: 203 字节
Chunk "File" — 大小: 89 字节
```

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>
#include <cstdint>
#include <cstring>

uint64_t ReadLE64(const uint8_t* p) {
    uint64_t r = 0;
    for (int i = 0; i < 8; i++)
        r |= ((uint64_t)p[i] << (i * 8));
    return r;
}

void ListChunks(const uint8_t* data, uint32_t size) {
    uint32_t pos = 0;
    int index = 0;
    
    while (pos + 12 <= size) {  // 至少需要 4(类型) + 8(大小)
        // 读取 4 字节类型名
        char name[5] = {};
        memcpy(name, data + pos, 4);
        pos += 4;
        
        // 读取 8 字节大小
        uint64_t chunk_size = ReadLE64(data + pos);
        pos += 8;
        
        printf("[%d] Chunk \"%s\" — 大小: %llu 字节\n",
            index, name, (unsigned long long)chunk_size);
        
        // 检查是否能安全跳过
        if (pos + chunk_size > size) {
            printf("    警告：块数据超出索引范围！\n");
            break;
        }
        
        pos += (uint32_t)chunk_size;
        index++;
    }
    
    printf("\n共 %d 个顶层块\n", index);
}

// 测试
int main() {
    // 构造测试数据：3 个 File 块
    // 每个 File 块内容简化为空（大小为 0）
    uint8_t test_data[36];  // 3 * 12 字节
    int pos = 0;
    
    for (int i = 0; i < 3; i++) {
        memcpy(test_data + pos, "File", 4); pos += 4;
        uint64_t sz = 0;  // 空块
        memcpy(test_data + pos, &sz, 8); pos += 8;
    }
    
    ListChunks(test_data, sizeof(test_data));
    return 0;
}

// 输出：
// [0] Chunk "File" — 大小: 0 字节
// [1] Chunk "File" — 大小: 0 字节
// [2] Chunk "File" — 大小: 0 字节
//
// 共 3 个顶层块
```

### 题目 3：检测并修复损坏的 XP3 魔数

**题目**：某个 XP3 文件通过 FTP ASCII 模式传输后损坏了。写一个程序检测具体损坏了哪些字节，并尝试修复。

提示：ASCII 模式传输最常见的损坏是：
- `0x0D 0x0A` (CR+LF) 被转换为 `0x0A` (LF)
- `0x0A` (LF) 被转换为 `0x0D 0x0A` (CR+LF)

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>
#include <cstdint>
#include <cstring>

static const uint8_t CORRECT_MAGIC[11] = {
    0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A, 0x1A,
    0x8B, 0x67, 0x01
};

// 魔数中每个字节的描述
static const char* BYTE_DESC[11] = {
    "'X'",
    "'P'",
    "'3'",
    "CR (回车符)",
    "LF (换行符)",
    "空格",
    "LF (换行符)",
    "EOF (DOS结束标记)",
    "日文编码高字节",
    "日文编码低字节",
    "版本+编码标识"
};

struct DiagResult {
    bool is_valid;
    int corrupt_count;
    char description[512];
};

DiagResult DiagnoseXP3Magic(const uint8_t* actual, int actual_len) {
    DiagResult result = {};
    
    if (actual_len < 11) {
        result.is_valid = false;
        snprintf(result.description, sizeof(result.description),
            "文件头不足 11 字节（仅 %d 字节）", actual_len);
        return result;
    }
    
    // 逐字节对比
    result.is_valid = true;
    int desc_pos = 0;
    
    for (int i = 0; i < 11; i++) {
        if (actual[i] != CORRECT_MAGIC[i]) {
            result.is_valid = false;
            result.corrupt_count++;
            desc_pos += snprintf(result.description + desc_pos,
                sizeof(result.description) - desc_pos,
                "  偏移 %d: %s — 期望 0x%02X, 实际 0x%02X\n",
                i, BYTE_DESC[i], CORRECT_MAGIC[i], actual[i]);
        }
    }
    
    if (result.is_valid) {
        snprintf(result.description, sizeof(result.description),
            "魔数完全正确");
    }
    
    return result;
}

bool RepairXP3Magic(uint8_t* data, int len) {
    if (len < 11) return false;
    
    printf("修复前：");
    for (int i = 0; i < 11; i++) printf("%02X ", data[i]);
    printf("\n");
    
    // 直接覆盖为正确的魔数
    memcpy(data, CORRECT_MAGIC, 11);
    
    printf("修复后：");
    for (int i = 0; i < 11; i++) printf("%02X ", data[i]);
    printf("\n");
    
    return true;
}

int main() {
    printf("=== XP3 魔数损坏检测与修复 ===\n\n");
    
    // 模拟三种常见的损坏场景
    
    // 场景 1：CR+LF → LF 转换（丢失了 CR）
    printf("--- 场景 1：CR 被移除 ---\n");
    uint8_t damaged1[11] = {
        0x58, 0x50, 0x33,
        0x0A,               // 原本应该是 0x0D（CR 被移除）
        0x0A, 0x20, 0x0A, 0x1A, 0x8B, 0x67, 0x01
    };
    auto diag1 = DiagnoseXP3Magic(damaged1, 11);
    printf("诊断结果：%s\n", diag1.is_valid ? "正常" : "损坏");
    if (!diag1.is_valid) {
        printf("损坏字节数：%d\n%s\n", diag1.corrupt_count, diag1.description);
        printf("推断原因：FTP ASCII 模式移除了 CR 字符\n");
        RepairXP3Magic(damaged1, 11);
    }
    
    // 场景 2：字符编码转换（日文字节被修改）
    printf("\n--- 场景 2：字符编码被转换 ---\n");
    uint8_t damaged2[11] = {
        0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A, 0x1A,
        0xE3, 0x81, 0x01    // 0x8B67 被 Shift-JIS→UTF-8 转换
    };
    auto diag2 = DiagnoseXP3Magic(damaged2, 11);
    printf("诊断结果：%s\n", diag2.is_valid ? "正常" : "损坏");
    if (!diag2.is_valid) {
        printf("损坏字节数：%d\n%s\n", diag2.corrupt_count, diag2.description);
        printf("推断原因：字符编码自动转换修改了日文检测字节\n");
        RepairXP3Magic(damaged2, 11);
    }
    
    printf("\n修复完成。注意：修复魔数后还需验证索引区完整性。\n");
    
    return 0;
}
```
