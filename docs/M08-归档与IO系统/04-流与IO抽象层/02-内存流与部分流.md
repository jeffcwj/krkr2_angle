## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 内存流 | Memory Stream | 将一块连续内存伪装成可寻址的字节流，用统一的 Read/Write/Seek 接口操作 |
| 引用模式 | Reference Mode | 内存流不拥有底层缓冲区，仅持有指针，析构时不释放内存；此模式下禁止写入 |
| 分级增长 | Tiered Growth | 缓冲区扩容时根据当前大小选择不同增量——小文件用小步长避免浪费，大文件用大步长减少 realloc 次数 |
| 部分流 | Partial Stream | 在一个已有流上开一个"窗口"，只暴露指定偏移量和长度范围的数据，外部看来就像独立的流 |
| 所有权转移 | Ownership Transfer | 部分流的构造函数接管底层流的生命周期：析构时自动 delete 底层流 |
| 本地临时存储 | Local Temp Storage | 当文件不在本地文件系统（如在归档包内或网络路径上）时，先复制到本地临时目录，供需要本地路径的 API 使用 |

## 一、tTVPMemoryStream 详解

### 1.1 设计动机

在游戏引擎中，许多场景需要在内存中构建或缓存数据，然后通过统一的流接口传递给其他模块：

```
┌──────────────────────────────────────────────────┐
│ 使用场景                                          │
├──────────────────────────────────────────────────┤
│ ① 序列化：TJS2 对象 → 内存流 → 写入文件           │
│ ② 解压缩：zlib 解压 → 内存流 → 读取解压后数据      │
│ ③ 网络：HTTP 响应体 → 内存流 → 解析器消费          │
│ ④ 测试：构造测试数据 → 内存流 → 传入被测函数       │
│ ⑤ 加密/解密：密文 → 内存流 → 解密后作为明文流使用   │
└──────────────────────────────────────────────────┘
```

如果每个模块都自己管理 `malloc`/`realloc`/指针偏移，不仅代码重复，而且接口不统一。tTVPMemoryStream 将这些操作封装成标准的 tTJSBinaryStream 接口，使内存中的字节块与磁盘文件、归档条目、网络流享有完全相同的操作方式。

### 1.2 类声明分析

以下是 `UtilStreams.h` 第 81-123 行的完整声明：

```cpp
// 源码：cpp/core/base/UtilStreams.h 第 81-123 行
class tTVPMemoryStream : public tTJSBinaryStream {
protected:
    void *Block;       // 指向底层内存缓冲区的指针
    bool Reference;    // true = 引用模式（不拥有内存），false = 自有模式
    tjs_uint Size;     // 当前有效数据大小（字节）
    tjs_uint AllocSize;// 已分配的缓冲区总容量（字节），>= Size
    tjs_uint CurrentPos;// 当前读写位置（字节偏移）

public:
    tTVPMemoryStream();
    tTVPMemoryStream(const void *block, tjs_uint size);
    ~tTVPMemoryStream() override;

    // tTJSBinaryStream 纯虚函数实现
    tjs_uint64 Seek(tjs_int64 offset, tjs_int whence) override;
    tjs_uint Read(void *buffer, tjs_uint read_size) override;
    tjs_uint Write(const void *buffer, tjs_uint write_size) override;
    void SetEndOfStorage() override;
    tjs_uint64 GetSize() override { return Size; }

    // 非流接口的辅助方法
    void *GetInternalBuffer() { return Block; }       // 获取底层缓冲区指针
    void *GetInternalBuffer() const { return Block; } // const 版本
    void Clear();                                      // 清空并释放缓冲区
    void SetSize(tjs_uint size);                       // 手动设置缓冲区大小

protected:
    void Init();  // 初始化所有成员为零/null

    // 可覆写的内存分配器——子类可替换为自定义分配策略
    virtual void *Alloc(size_t size);
    virtual void *Realloc(void *orgblock, size_t size);
    virtual void Free(void *block);
};
```

**关键设计点：**

1. **`Size` vs `AllocSize` 分离**：`Size` 是逻辑大小（有效数据量），`AllocSize` 是物理大小（实际分配的内存量）。两者分离是为了避免每次 Write 都触发 realloc——只要 `CurrentPos + write_size < AllocSize`，就可以直接 memcpy 而无需重新分配。

2. **`Reference` 标志**：区分"我拥有这块内存"和"别人给我一个指针看看"的两种模式。引用模式下 Write 直接抛异常。

3. **虚拟分配器三件套**（Alloc/Realloc/Free）：默认使用 `TJS_malloc`/`TJS_realloc`/`TJS_free`（即标准 C 库的 malloc/realloc/free 的 TJS 封装）。子类可覆写这三个函数来使用内存池、对齐分配、调试追踪等自定义策略。

### 1.3 两种构造模式

tTVPMemoryStream 有两种工作模式，由构造方式决定：

```
┌─────────────────────────────────────────────────────────┐
│                    自有模式 (Owned)                       │
│  tTVPMemoryStream()                                     │
│  或 tTVPMemoryStream(nullptr, size)                      │
│                                                         │
│  ┌────────────────────┐                                  │
│  │  Block = Alloc(sz) │ ← 自己分配内存                   │
│  │  Reference = false  │                                  │
│  │  析构时 Free(Block) │ ← 自己负责释放                   │
│  │  可读可写           │                                  │
│  └────────────────────┘                                  │
├─────────────────────────────────────────────────────────┤
│                   引用模式 (Reference)                    │
│  tTVPMemoryStream(existingPtr, size)                     │
│  （existingPtr != nullptr）                               │
│                                                         │
│  ┌────────────────────┐                                  │
│  │  Block = existingPtr│ ← 使用外部给定的指针             │
│  │  Reference = true   │                                  │
│  │  析构时不释放       │ ← 外部负责内存生命周期            │
│  │  只读（Write 抛异常）│                                  │
│  └────────────────────┘                                  │
└─────────────────────────────────────────────────────────┘
```

对应源码（`UtilStreams.cpp` 第 88-110 行）：

```cpp
// 自有模式：默认构造，缓冲区从空开始
tTVPMemoryStream::tTVPMemoryStream() {
    Init();  // Block=nullptr, Reference=false, Size=0, AllocSize=0, CurrentPos=0
}

// 引用模式或预分配模式
tTVPMemoryStream::tTVPMemoryStream(const void *block, tjs_uint size) {
    Init();
    Block = (void *)block;  // 去 const——内部统一用 void*
    if(!Block) {
        // block 为 nullptr：预分配 size 字节的自有缓冲区
        Block = Alloc(size);
        if(!Block)
            TVPThrowExceptionMessage(TVPInsufficientMemory);
        // Reference 保持 false（自有模式）
    } else {
        // block 不为 nullptr：引用外部内存
        Reference = true;  // 标记为引用模式
    }
    Size = size;
    AllocSize = size;
    CurrentPos = 0;
}

// 析构：只有自有模式才释放内存
tTVPMemoryStream::~tTVPMemoryStream() {
    if(Block && !Reference)  // 有缓冲区 且 不是引用模式
        Free(Block);
}
```

**注意 `const_cast` 设计**：构造函数参数声明为 `const void *block`，但内部赋值给 `void *Block` 时直接做了 C 风格的 `(void *)block` 强制转换。这是因为引用模式下 Write 会抛异常来阻止写入，而不是依赖 const 正确性来保证。

### 1.4 Seek 实现：边界钳位

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 113-140 行
tjs_uint64 tTVPMemoryStream::Seek(tjs_int64 offset, tjs_int whence) {
    tjs_int64 newpos;
    switch(whence) {
        case TJS_BS_SEEK_SET:  // 从起始位置定位
            if(offset >= 0) {
                if(offset <= Size)        // 不能超过有效数据范围
                    CurrentPos = static_cast<tjs_uint>(offset);
                // 否则忽略，CurrentPos 保持不变
            }
            return CurrentPos;

        case TJS_BS_SEEK_CUR:  // 从当前位置偏移
            if((newpos = offset + (tjs_int64)CurrentPos) >= 0) {
                tjs_uint np = (tjs_uint)newpos;
                if(np <= Size)
                    CurrentPos = np;
            }
            return CurrentPos;

        case TJS_BS_SEEK_END:  // 从末尾偏移（offset 通常为 0 或负数）
            if((newpos = offset + (tjs_int64)Size) >= 0) {
                tjs_uint np = (tjs_uint)newpos;
                if(np <= Size)
                    CurrentPos = np;
            }
            return CurrentPos;
    }
    return CurrentPos;  // 未知 whence 值：不做任何操作
}
```

**与 TArchiveStream::Seek 的关键区别**：

| 特性 | tTVPMemoryStream | TArchiveStream |
|------|-----------------|----------------|
| 越界行为 | 静默忽略（CurrentPos 不变） | 钳位到 [0, DataLength] |
| 负位置处理 | 通过 `>= 0` 检查丢弃 | 钳位到 0 |
| 底层操作 | 仅修改 CurrentPos 整数 | 调用 `_instr->SetPosition()` 移动底层流 |
| 性能 | O(1)，无系统调用 | O(1) + 底层 Seek 系统调用 |

tTVPMemoryStream 的 Seek 是纯内存操作——只是修改一个整数值。因为数据已经全部在内存中，不需要像文件流那样通知操作系统移动文件指针。

### 1.5 Read 实现

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 143-153 行
tjs_uint tTVPMemoryStream::Read(void *buffer, tjs_uint read_size) {
    if(CurrentPos + read_size >= Size) {
        // 读取范围超出有效数据：截断为剩余可读量
        read_size = Size - CurrentPos;
    }

    // 直接内存拷贝——这就是内存流比文件流快的根本原因
    memcpy(buffer, (tjs_uint8 *)Block + CurrentPos, read_size);

    CurrentPos += read_size;  // 推进读取位置

    return read_size;  // 返回实际读取的字节数
}
```

Read 的实现非常简洁：一次 `memcpy` 就完成了。与磁盘 IO 需要经过内核缓冲区、页缓存等路径相比，内存流的读取就是一次简单的内存复制。

**边界条件分析：**
- 如果 `CurrentPos == Size`（已到末尾），`read_size` 被截断为 0，`memcpy` 复制 0 字节，返回 0——调用者据此判断数据读完
- 如果 `CurrentPos + read_size > Size` 但 `CurrentPos < Size`，返回 `Size - CurrentPos` 个字节——部分读取

### 1.6 Write 实现与分级增长策略

Write 是 tTVPMemoryStream 最复杂的方法，包含了动态缓冲区增长逻辑：

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 156-195 行
tjs_uint tTVPMemoryStream::Write(const void *buffer, tjs_uint write_size) {
    // ① 引用模式下禁止写入
    if(Reference)
        TVPThrowExceptionMessage(TVPWriteError);

    tjs_uint newpos = CurrentPos + write_size;
    if(newpos >= AllocSize) {
        // ② 需要扩容：根据当前 AllocSize 选择增长步长
        tjs_uint onesize;
        if(AllocSize < 64 * 1024)          // < 64 KB
            onesize = 4 * 1024;             //   → 每次增长 4 KB
        else if(AllocSize < 512 * 1024)    // < 512 KB
            onesize = 16 * 1024;            //   → 每次增长 16 KB
        else if(AllocSize < 4096 * 1024)   // < 4 MB
            onesize = 256 * 1024;           //   → 每次增长 256 KB
        else                                // >= 4 MB
            onesize = 2024 * 1024;          //   → 每次增长约 2 MB（注意是 2024 不是 2048）

        AllocSize += onesize;

        if(CurrentPos + write_size >= AllocSize) {
            // ③ 增长一个步长还不够？直接使用精确大小
            AllocSize = CurrentPos + write_size;
        }

        Block = Realloc(Block, AllocSize);

        if(AllocSize && !Block)
            TVPThrowExceptionMessage(TVPInsufficientMemory);
        // 注释原文："this exception cannot be repaired; a fatal error."
    }

    // ④ 实际写入数据
    memcpy((tjs_uint8 *)Block + CurrentPos, buffer, write_size);

    CurrentPos = newpos;

    // ⑤ 更新逻辑大小（仅当写入超出当前 Size 时）
    if(CurrentPos > Size)
        Size = CurrentPos;

    return write_size;
}
```

**分级增长策略图解：**

```
AllocSize 范围          增长步长      最多浪费        适用场景
────────────────────────────────────────────────────────────
   0 ~  64 KB           4 KB        最多  4 KB      小型配置文件
  64 ~ 512 KB          16 KB        最多 16 KB      中等脚本文件
 512 KB ~ 4 MB        256 KB        最多256 KB      图片/音频数据
   > 4 MB             ~2 MB         最多 ~2 MB      大型资源文件
────────────────────────────────────────────────────────────

                    realloc 频率对比（写入 1MB 数据）：
固定步长 4KB:   256 次 realloc   ← 频繁，性能差
固定步长 256KB:   4 次 realloc   ← 小文件浪费 256KB
分级策略:       ≈ 20 次 realloc  ← 平衡点
```

**为什么是 2024 而不是 2048？** 这是源码中的实际数值（`UtilStreams.cpp` 第 172 行 `onesize = 2024 * 1024`）。看起来像是一个 typo（本应是 2048 即 2 MB），但实际差异只有 24 KB，对性能几乎没有影响。

### 1.7 SetEndOfStorage：截断与收缩

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 198-207 行
void tTVPMemoryStream::SetEndOfStorage() {
    if(Reference)
        TVPThrowExceptionMessage(TVPWriteError);  // 引用模式不允许修改

    Size = CurrentPos;        // 将当前位置作为新的逻辑末尾
    AllocSize = Size;         // 收缩分配量以释放多余内存
    Block = Realloc(Block, Size);
    if(Size && !Block)
        TVPThrowExceptionMessage(TVPInsufficientMemory);
}
```

SetEndOfStorage 的典型用法是"先多分配再截断"：

```cpp
// 使用模式：写入可变长度数据，完成后收缩到实际大小
tTVPMemoryStream stream;
stream.Write(someData, maxPossibleSize);   // AllocSize 可能远大于实际需要
// ... 实际只写了一部分 ...
stream.Seek(actualSize, TJS_BS_SEEK_SET);  // 定位到实际数据末尾
stream.SetEndOfStorage();                   // 截断：Size = actualSize, Realloc 收缩
```

### 1.8 Clear 和 SetSize

```cpp
// Clear：完全重置到初始状态
// 源码：cpp/core/base/UtilStreams.cpp 第 210-214 行
void tTVPMemoryStream::Clear() {
    if(Block && !Reference)
        Free(Block);     // 释放自有内存
    Init();              // 重置所有成员：Block=nullptr, Size=0, etc.
}

// SetSize：手动调整缓冲区大小
// 源码：cpp/core/base/UtilStreams.cpp 第 217-238 行
void tTVPMemoryStream::SetSize(tjs_uint size) {
    if(Reference)
        TVPThrowExceptionMessage(TVPWriteError);

    if(Size > size) {
        // 缩小：截断数据，Realloc 释放多余内存
        Size = size;
        AllocSize = size;
        Block = Realloc(Block, size);
        if(CurrentPos > Size)
            CurrentPos = Size;   // 如果当前位置超出新大小，钳位
        if(size && !Block)
            TVPThrowExceptionMessage(TVPInsufficientMemory);
    } else {
        // 扩大：增加分配量，新区域内容未定义
        AllocSize = size;
        Size = size;
        Block = Realloc(Block, size);
        if(size && !Block)
            TVPThrowExceptionMessage(TVPInsufficientMemory);
    }
}
```

**注意**：SetSize 扩大时不会将新增区域初始化为零。如果需要零初始化，调用者需要自行 memset。

### 1.9 虚拟分配器

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 250-258 行
void *tTVPMemoryStream::Alloc(size_t size) {
    return TJS_malloc(size);      // = malloc(size)
}

void *tTVPMemoryStream::Realloc(void *orgblock, size_t size) {
    return TJS_realloc(orgblock, size);  // = realloc(orgblock, size)
}

void tTVPMemoryStream::Free(void *block) {
    TJS_free(block);              // = free(block)
}
```

这三个方法都是 `virtual` 的，意味着子类可以覆写它们来实现：

- **对齐分配**：某些 SIMD 操作需要 16 字节或 32 字节对齐的内存
- **内存池**：频繁创建销毁小型 MemoryStream 时，用内存池减少系统调用
- **调试追踪**：记录每次分配/释放的大小和调用栈，用于内存泄漏检测
- **计数器**：统计总分配量，当超过阈值时触发内存压缩

## 二、tTVPPartialStream 详解

### 2.1 设计动机

在归档系统中，一个文件包（如 XP3 归档）包含多个文件条目，每个条目占据归档文件中的一个连续区间。要读取某个条目，需要：

1. 打开归档文件的底层流
2. Seek 到该条目的起始偏移
3. 限制读取范围不超过该条目的长度

tTVPPartialStream 将步骤 2-3 封装为一个独立的流对象。对消费者来说，它看起来就像一个从 0 开始、大小为条目长度的独立流：

```
                    归档文件 (底层流)
    ┌─────────────────────────────────────────────┐
    │  Header  │  Entry A  │  Entry B  │  Entry C │
    └─────────────────────────────────────────────┘
    0        100         500        1200        2000

              tTVPPartialStream(stream, 500, 700)
              ┌──────────────────────────┐
              │      Entry B 的视图       │
              │  逻辑偏移 0 = 物理偏移 500 │
              │  逻辑大小 = 700 字节       │
              └──────────────────────────┘
              0                         700
```

### 2.2 类声明分析

```cpp
// 源码：cpp/core/base/UtilStreams.h 第 135-157 行
class tTVPPartialStream : public tTJSBinaryStream {
private:
    tTJSBinaryStream *Stream;   // 底层流指针（拥有所有权，析构时 delete）
    tjs_uint64 Start;           // 窗口在底层流中的起始偏移
    tjs_uint64 Size;            // 窗口大小（字节）
    tjs_uint64 CurrentPos;      // 当前逻辑位置（相对于窗口起始）

public:
    tTVPPartialStream(tTJSBinaryStream *stream, tjs_uint64 start,
                      tjs_uint64 size);
    ~tTVPPartialStream() override;

    tjs_uint64 Seek(tjs_int64 offset, tjs_int whence) override;
    tjs_uint Read(void *buffer, tjs_uint read_size) override;
    tjs_uint Write(const void *buffer, tjs_uint write_size) override;
    // void SetEndOfStorage(); // 使用基类默认行为（抛异常）
    tjs_uint64 GetSize() override;
};
```

**关键设计点：**

1. **只读设计**：Write 方法直接返回 0（不抛异常，但不写入任何数据）。注释中也标注了 "read-only accesses"。
2. **所有权语义**：构造函数的注释明确写道 "the stream given as a argument here will be owned by this instance, and freed at the destruction."（传入的流将被本实例拥有，并在析构时释放）。
3. **`tjs_uint64` 类型**：与 tTVPMemoryStream 使用 `tjs_uint`（32位）不同，PartialStream 使用 64 位无符号整数。这是因为归档文件可能超过 4 GB，窗口的起始偏移和大小需要 64 位表示。

### 2.3 构造与析构：所有权转移

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 264-281 行
tTVPPartialStream::tTVPPartialStream(tTJSBinaryStream *stream,
                                     tjs_uint64 start, tjs_uint64 size) {
    // 接管所有权：析构时会 delete stream
    Stream = stream;
    Start = start;
    Size = size;
    CurrentPos = 0;

    try {
        Stream->SetPosition(Start);  // 立即将底层流定位到窗口起始
    } catch(...) {
        delete Stream;     // SetPosition 失败：释放底层流
        Stream = nullptr;
        throw;             // 重新抛出异常
    }
}

// 源码：cpp/core/base/UtilStreams.cpp 第 284-287 行
tTVPPartialStream::~tTVPPartialStream() {
    if(Stream)
        delete Stream;  // 释放底层流
}
```

**异常安全设计**：构造函数在 `SetPosition` 失败时立即 delete 底层流并将 Stream 设为 nullptr，确保即使构造中途抛异常也不会泄漏。这是 RAII（Resource Acquisition Is Initialization，资源获取即初始化）原则的一种体现——构造失败时，已获取的资源必须立即释放。

### 2.4 Seek 实现：坐标转换

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 290-318 行
tjs_uint64 tTVPPartialStream::Seek(tjs_int64 offset, tjs_int whence) {
    tjs_int64 newpos;
    switch(whence) {
        case TJS_BS_SEEK_SET:
            newpos = offset;
            if(offset >= 0 && offset <= static_cast<tjs_int64>(Size)) {
                newpos += Start;  // 逻辑偏移 → 物理偏移
                CurrentPos = Stream->Seek(newpos, TJS_BS_SEEK_SET) - Start;
                // 物理位置减去 Start = 逻辑位置
            }
            return CurrentPos;

        case TJS_BS_SEEK_CUR:
            newpos = offset + CurrentPos;
            if(offset >= 0 && offset <= static_cast<tjs_int64>(Size)) {
                newpos += Start;
                CurrentPos = Stream->Seek(newpos, TJS_BS_SEEK_SET) - Start;
            }
            return CurrentPos;

        case TJS_BS_SEEK_END:
            newpos = offset + Size;
            if(offset >= 0 && offset <= static_cast<tjs_int64>(Size)) {
                newpos += Start;
                CurrentPos = Stream->Seek(newpos, TJS_BS_SEEK_SET) - Start;
            }
            return CurrentPos;
    }
    return CurrentPos;
}
```

**核心公式：** `物理偏移 = 逻辑偏移 + Start`，`逻辑偏移 = 物理偏移 - Start`

注意一个微妙之处：所有三种 whence 模式最终都调用 `Stream->Seek(newpos, TJS_BS_SEEK_SET)`——即总是将底层流的 Seek 模式转换为绝对定位。这简化了实现，因为不需要让底层流的"当前位置"概念与 PartialStream 的"当前位置"同步。

**与 TArchiveStream Seek 的区别：**

```
tTVPPartialStream::Seek():
    ① 计算物理偏移 newpos = 逻辑偏移 + Start
    ② 调用 Stream->Seek(newpos, TJS_BS_SEEK_SET)  ← 委托给底层流
    ③ CurrentPos = 返回值 - Start                   ← 反算逻辑位置

TArchiveStream::Seek():
    ① 直接修改 CurrentPos（纯整数运算）
    ② 钳位到 [0, DataLength]
    ③ 调用 _instr->SetPosition(CurrentPos + StartPos)  ← 同步底层流
```

两者的区别在于错误处理：PartialStream 信任底层流的返回值来设置 CurrentPos（如果底层 Seek 失败会得到异常结果），而 TArchiveStream 先修改 CurrentPos 再同步底层流。

### 2.5 Read 实现

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 321-331 行
tjs_uint tTVPPartialStream::Read(void *buffer, tjs_uint read_size) {
    if(CurrentPos + read_size >= Size) {
        // 截断到窗口边界
        read_size = static_cast<tjs_uint>(Size - CurrentPos);
    }

    tjs_uint read = Stream->Read(buffer, read_size);  // 委托给底层流

    CurrentPos += read;  // 按实际读取量推进（不是请求量）

    return read;
}
```

注意 CurrentPos 更新使用的是 `read`（底层流实际返回的字节数）而不是 `read_size`（请求量）。这是因为底层流可能因为自身原因（如文件末尾）返回比请求更少的字节。

### 2.6 Write 实现（只读流）

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 334-336 行
tjs_uint tTVPPartialStream::Write(const void *buffer, tjs_uint write_size) {
    return 0;  // 静默忽略写入请求
}
```

Write 不抛异常，只是返回 0 表示"0 字节被写入"。这与 tTVPMemoryStream 引用模式的 Write（抛异常）形成对比。两种策略各有优劣：

| 策略 | 优点 | 缺点 |
|------|------|------|
| 返回 0（PartialStream） | 调用者可以检查返回值优雅处理 | 如果调用者不检查返回值，会静默丢数据 |
| 抛异常（MemoryStream 引用模式） | 不可能忽略，bug 立即暴露 | 调用者必须准备好 catch |

## 三、tTVPPartialStream vs TArchiveStream 全面对比

KrKr2 中有两个"窗口流"类——都是在一个底层流上开一个固定范围的视图。但它们的设计差异反映了不同的使用场景：

```
┌──────────────────────────┐    ┌──────────────────────────┐
│   tTVPPartialStream      │    │     TArchiveStream       │
│   (UtilStreams.h)        │    │   (StorageIntf.h)        │
├──────────────────────────┤    ├──────────────────────────┤
│ 底层流: tTJSBinaryStream │    │ 底层流: tTJSBinaryStream │
│ 拥有所有权 (delete)       │    │ 归档引用 (AddRef/Release)│
│ 只读 (Write 返回 0)      │    │ 只读 (Write 返回 0)      │
│ 通用窗口                 │    │ 归档专用窗口             │
└──────────────────────────┘    └──────────────────────────┘
```

| 对比维度 | tTVPPartialStream | TArchiveStream |
|----------|-------------------|----------------|
| **头文件** | `UtilStreams.h` 第 135-157 行 | `StorageIntf.h` 第 331-381 行 |
| **底层流所有权** | **拥有**：析构时 `delete Stream` | **不拥有**：持有 `tTVPArchive*`，通过 `AddRef/Release` 引用计数 |
| **构造参数** | `(stream, start, size)` | `(owner, offset, length)` — owner 是 tTVPArchive |
| **Seek 实现** | 委托给底层流的 `Seek()` | 纯整数运算 + `_instr->SetPosition()` |
| **越界 Seek** | 条件不满足时不操作 | 钳位到 `[0, DataLength]` |
| **Read 实现** | 委托给底层流的 `Read()` | 委托给底层流的 `ReadBuffer()` |
| **Read 差异** | 允许部分读取（返回实际字节数） | `ReadBuffer` 要求精确读取（不足时抛异常） |
| **Write** | 返回 0 | 返回 0 |
| **使用场景** | 通用流窗口，任何需要限制范围的场合 | 归档系统内部，从归档文件中读取单个条目 |
| **生命周期** | 独立于归档系统 | 与 tTVPArchive 共生（引用计数） |

**为什么需要两个类？**

TArchiveStream 专门为归档系统设计——它持有 tTVPArchive 的引用计数，确保只要还有 TArchiveStream 在使用归档，归档对象就不会被销毁。而 tTVPPartialStream 是一个通用工具，不关心底层流来自哪里。

## 四、tTVPLocalTempStorageHolder 详解

### 4.1 设计动机

某些第三方库（如 UnRAR）需要本地文件系统路径才能打开文件，无法接受抽象的流接口。但 KrKr2 的统一存储系统中，文件可能在 XP3 归档包内、HTTP URL 上、或其他不能直接映射到本地路径的位置。

tTVPLocalTempStorageHolder 解决这个问题：检查文件是否可以直接通过本地路径访问，如果不能，就将文件内容复制到临时目录，然后提供临时文件的本地路径。

```
                   文件可本地访问？
                        │
              ┌─────────┴─────────┐
              │ 是                 │ 否
              │                    │
      LocalName = 本地路径    ┌──────────────────┐
      FileMustBeDeleted=false │ 1. 创建临时文件夹  │
      （什么都不做）           │ 2. 复制文件到临时目录│
                              │ 3. LocalName=临时路径│
                              │ FileMustBeDeleted=true│
                              └──────────────────┘
                                     │
                              析构时自动清理
                              （删文件 + 删文件夹）
```

### 4.2 类声明

```cpp
// 源码：cpp/core/base/UtilStreams.h 第 57-71 行
class tTVPLocalTempStorageHolder {
    bool FileMustBeDeleted;   // 析构时是否需要删除临时文件
    bool FolderMustBeDeleted; // 析构时是否需要删除临时文件夹
    ttstr LocalName;          // 本地文件系统路径
    ttstr LocalFolder;        // 本地临时文件夹路径

public:
    tTVPLocalTempStorageHolder(const ttstr &name);
    ~tTVPLocalTempStorageHolder();

    bool IsTemporaryFile() const { return FileMustBeDeleted; }
    const ttstr &GetLocalName() const { return LocalName; }
};
```

### 4.3 构造函数：智能路径解析

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 28-73 行
#define TVP_LOCAL_TEMP_COPY_BLOCK_SIZE 65536 * 2  // 128 KB 复制块大小

tTVPLocalTempStorageHolder::tTVPLocalTempStorageHolder(const ttstr &name) {
    // 注意：name 必须已经标准化（normalize）！

    FileMustBeDeleted = false;
    FolderMustBeDeleted = false;

    // 尝试获取本地路径
    LocalName = TVPGetLocallyAccessibleName(name);

    if(LocalName.IsEmpty()) {
        // 文件不在本地文件系统上——需要复制到临时目录

        // 创建临时文件夹（保留原文件名很重要）
        LocalFolder = TVPGetTemporaryName();
        LocalName = LocalFolder + TJS_W("/") + TVPExtractStorageName(name);
        TVPCreateFolders(LocalFolder);
        FolderMustBeDeleted = true;
        FileMustBeDeleted = true;

        try {
            // 流式复制：源 → 目标
            tTVPStreamHolder src(name);                              // 打开源
            tTVPStreamHolder dest(LocalName,
                                  TJS_BS_WRITE | TJS_BS_DELETE_ON_CLOSE); // 打开临时文件
            tjs_uint8 *buffer = new tjs_uint8[TVP_LOCAL_TEMP_COPY_BLOCK_SIZE];
            try {
                tjs_uint read;
                while(true) {
                    read = src->Read(buffer, TVP_LOCAL_TEMP_COPY_BLOCK_SIZE);
                    if(read == 0)
                        break;          // 源读完
                    dest->WriteBuffer(buffer, read);  // WriteBuffer 要求全量写入
                }
            } catch(...) {
                delete[] buffer;
                throw;
            }
            delete[] buffer;
        } catch(...) {
            // 复制失败：清理已创建的临时文件和文件夹
            if(FileMustBeDeleted)
                TVPRemoveFile(LocalName);
            if(FolderMustBeDeleted)
                TVPRemoveFolder(LocalFolder);
            throw;
        }
    }
}
```

**设计细节解析：**

1. **保留原文件名**：临时文件使用 `TVPExtractStorageName(name)` 提取原始文件名。这很重要，因为某些库（如 UnRAR）会根据文件扩展名判断格式。

2. **128 KB 复制块**：每次读取 `65536 * 2 = 131072` 字节（128 KB）。这个大小在减少系统调用次数和避免过大内存占用之间取得平衡。

3. **双层异常安全**：外层 try-catch 处理文件操作失败（如临时目录创建失败），内层 try-catch 确保 buffer 在异常时被释放。

4. **TJS_BS_DELETE_ON_CLOSE**：目标流使用了 `DELETE_ON_CLOSE` 标志。这意味着如果目标流对象被销毁（例如复制中途异常），临时文件会被自动删除——又一层异常安全保障。

### 4.4 析构函数：清理临时文件

```cpp
// 源码：cpp/core/base/UtilStreams.cpp 第 77-82 行
tTVPLocalTempStorageHolder::~tTVPLocalTempStorageHolder() {
    if(FileMustBeDeleted)
        TVPRemoveFile(LocalName);      // 删除临时文件
    if(FolderMustBeDeleted)
        TVPRemoveFolder(LocalFolder);  // 删除临时文件夹
}
```

### 4.5 跨平台行为

| 平台 | `TVPGetLocallyAccessibleName` 行为 | 临时目录位置 |
|------|--------------------------------------|-------------|
| Windows | file:// 协议的路径直接返回本地路径 | `%TEMP%` 下的随机目录 |
| Linux | file:// 路径返回本地路径 | `/tmp` 下的随机目录 |
| macOS | file:// 路径返回本地路径 | `NSTemporaryDirectory()` 下 |
| Android | file:// 或 content:// 路径需特殊处理 | 应用缓存目录下 |

### 示例 3：部分流（窗口流）实现

```cpp
#include <cstdio>
#include <cstring>
#include <cstdint>
#include <cstdlib>
#include <cassert>

// 基础流接口
class IStream {
public:
    virtual ~IStream() = default;
    virtual uint64_t seek(int64_t offset, int whence) = 0;
    virtual uint32_t read(void *buffer, uint32_t size) = 0;
    virtual uint64_t getSize() = 0;
};

// 简单内存流（作为底层流）
class MemStream : public IStream {
    const uint8_t *data_;
    uint32_t size_;
    uint32_t pos_ = 0;

public:
    MemStream(const uint8_t *data, uint32_t size)
        : data_(data), size_(size) {}

    uint64_t seek(int64_t offset, int whence) override {
        int64_t newpos;
        switch (whence) {
            case 0: newpos = offset; break;
            case 1: newpos = pos_ + offset; break;
            case 2: newpos = size_ + offset; break;
            default: return pos_;
        }
        if (newpos >= 0 && newpos <= (int64_t)size_)
            pos_ = (uint32_t)newpos;
        return pos_;
    }

    uint32_t read(void *buffer, uint32_t readSize) override {
        if (pos_ + readSize > size_)
            readSize = size_ - pos_;
        memcpy(buffer, data_ + pos_, readSize);
        pos_ += readSize;
        return readSize;
    }

    uint64_t getSize() override { return size_; }
};

// 部分流：模拟 tTVPPartialStream
class PartialStream : public IStream {
    IStream *stream_;   // 拥有所有权
    uint64_t start_;    // 窗口起始偏移
    uint64_t size_;     // 窗口大小
    uint64_t pos_ = 0;  // 当前逻辑位置

public:
    // 构造时接管 stream 的所有权
    PartialStream(IStream *stream, uint64_t start, uint64_t size)
        : stream_(stream), start_(start), size_(size) {
        // 立即定位到窗口起始位置
        stream_->seek(start_, 0);
    }

    // 析构时释放底层流
    ~PartialStream() override {
        delete stream_;
    }

    uint64_t seek(int64_t offset, int whence) override {
        int64_t logicalPos;
        switch (whence) {
            case 0: logicalPos = offset; break;              // SEEK_SET
            case 1: logicalPos = (int64_t)pos_ + offset; break; // SEEK_CUR
            case 2: logicalPos = (int64_t)size_ + offset; break; // SEEK_END
            default: return pos_;
        }
        if (logicalPos >= 0 && logicalPos <= (int64_t)size_) {
            // 将逻辑位置转换为物理位置并定位底层流
            uint64_t physicalPos = start_ + logicalPos;
            stream_->seek(physicalPos, 0);  // 始终用 SEEK_SET
            pos_ = logicalPos;
        }
        return pos_;
    }

    uint32_t read(void *buffer, uint32_t readSize) override {
        // 截断到窗口边界
        if (pos_ + readSize > size_) {
            readSize = (uint32_t)(size_ - pos_);
        }
        uint32_t actual = stream_->read(buffer, readSize);
        pos_ += actual;  // 按实际读取量推进
        return actual;
    }

    uint64_t getSize() override { return size_; }
};

int main() {
    // 模拟一个归档文件，包含 3 个条目
    // 条目布局：
    //   [0..9]    = "HEADER0123"    (10 字节头部)
    //   [10..19]  = "EntryAAAAA"    (Entry A, 10 字节)
    //   [20..34]  = "EntryBBBBBBBBB" (Entry B, 15 字节)
    //   [35..44]  = "EntryCCCCC"    (Entry C, 10 字节)
    const char *archiveData =
        "HEADER0123"       // [0..9]
        "EntryAAAAA"       // [10..19]
        "EntryBBBBBBBBB"   // [20..34]
        "EntryCCCCC";      // [35..44]

    uint32_t totalLen = (uint32_t)strlen(archiveData);
    printf("归档总大小: %u 字节\n\n", totalLen);

    // 为 Entry B 创建部分流：start=20, size=15
    // 注意：MemStream 是 new 出来的，所有权转移给 PartialStream
    auto *partialB = new PartialStream(
        new MemStream((const uint8_t *)archiveData, totalLen),
        20, 15);

    printf("Entry B 部分流:\n");
    printf("  逻辑大小: %llu 字节\n", (unsigned long long)partialB->getSize());

    // 读取 Entry B 的全部内容
    char buf[32] = {};
    uint32_t readCount = partialB->read(buf, sizeof(buf));
    printf("  读取 %u 字节: \"%s\"\n", readCount, buf);
    // 输出: 读取 15 字节: "EntryBBBBBBBBB"

    // Seek 到 Entry B 的第 5 字节
    partialB->seek(5, 0);
    memset(buf, 0, sizeof(buf));
    readCount = partialB->read(buf, sizeof(buf));
    printf("  从偏移5读取 %u 字节: \"%s\"\n", readCount, buf);
```

### 示例 4：自定义分配器的内存流

```cpp
#include <cstdio>
#include <cstring>
#include <cstdlib>
#include <cstdint>

// 模拟 KrKr2 的虚拟分配器机制
// 子类通过覆写 Alloc/Realloc/Free 来自定义内存管理策略
class MemoryStreamBase {
protected:
    uint8_t *block_ = nullptr;
    bool reference_ = false;
    uint32_t size_ = 0;
    uint32_t allocSize_ = 0;
    uint32_t pos_ = 0;

    // 虚拟分配器——子类可覆写
    virtual void *doAlloc(size_t sz) { return malloc(sz); }
    virtual void *doRealloc(void *p, size_t sz) { return realloc(p, sz); }
    virtual void doFree(void *p) { free(p); }

public:
    virtual ~MemoryStreamBase() {
        if (block_ && !reference_) doFree(block_);
    }

    uint32_t write(const void *data, uint32_t writeSize) {
        if (reference_) return 0;
        uint32_t newpos = pos_ + writeSize;
        if (newpos > allocSize_) {
            uint32_t growth = (allocSize_ < 64*1024) ? 4*1024 : 256*1024;
            allocSize_ += growth;
            if (newpos > allocSize_) allocSize_ = newpos;
            block_ = (uint8_t *)doRealloc(block_, allocSize_);
        }
        memcpy(block_ + pos_, data, writeSize);
        pos_ = newpos;
        if (pos_ > size_) size_ = pos_;
        return writeSize;
    }

    uint32_t getSize() const { return size_; }
    uint32_t getAllocSize() const { return allocSize_; }
};

// 追踪分配器：记录每次分配/释放操作
class TrackedAllocStream : public MemoryStreamBase {
    int allocCount_ = 0;
    int reallocCount_ = 0;
    int freeCount_ = 0;
    size_t totalAllocated_ = 0;

protected:
    void *doAlloc(size_t sz) override {
        allocCount_++;
        totalAllocated_ += sz;
        printf("  [追踪] Alloc(%zu)\n", sz);
        return malloc(sz);
    }

    void *doRealloc(void *p, size_t sz) override {
        reallocCount_++;
        totalAllocated_ = sz;  // realloc 后总量 = 新大小
        printf("  [追踪] Realloc(%zu)\n", sz);
        return realloc(p, sz);
    }

    void doFree(void *p) override {
        freeCount_++;
        printf("  [追踪] Free()\n");
        free(p);
    }

public:
    void printStats() const {
        printf("分配统计: alloc=%d, realloc=%d, free=%d, 当前分配=%zu 字节\n",
               allocCount_, reallocCount_, freeCount_, totalAllocated_);
    }
};

// 对齐分配器：确保缓冲区 16 字节对齐（SIMD 优化场景）
class AlignedAllocStream : public MemoryStreamBase {
    static constexpr size_t ALIGN = 16;

    // 将大小向上取整到 ALIGN 的倍数
    static size_t alignUp(size_t sz) {
        return (sz + ALIGN - 1) & ~(ALIGN - 1);
    }

protected:
    void *doAlloc(size_t sz) override {
        sz = alignUp(sz);
#ifdef _WIN32
        return _aligned_malloc(sz, ALIGN);
#else
        void *ptr = nullptr;
        posix_memalign(&ptr, ALIGN, sz);
        return ptr;
#endif
    }

    void *doRealloc(void *p, size_t sz) override {
        sz = alignUp(sz);
#ifdef _WIN32
        return _aligned_realloc(p, sz, ALIGN);
#else
        // POSIX 没有 aligned_realloc，需要手动实现
        void *newPtr = nullptr;
        posix_memalign(&newPtr, ALIGN, sz);
        if (p && newPtr) {
            // 复制旧数据（简化：复制 min(oldSize, newSize) 字节）
            memcpy(newPtr, p, sz);  // 注意：实际应用需要记录旧大小
            free(p);
        }
        return newPtr;
#endif
    }

    void doFree(void *p) override {
#ifdef _WIN32
        _aligned_free(p);
#else
        free(p);
#endif
    }

public:
    bool isAligned() const {
        return block_ && ((uintptr_t)block_ % ALIGN == 0);
    }
};

int main() {
    printf("=== 追踪分配器 ===\n");
    {
        TrackedAllocStream stream;
        // 多次小量写入，观察 realloc 调用模式
        for (int i = 0; i < 20; i++) {
            char data[256];
            memset(data, 'A' + (i % 26), sizeof(data));
            stream.write(data, sizeof(data));
        }
        printf("\n写入 %u 字节后:\n", stream.getSize());
        stream.printStats();
    }
    // 析构时输出 Free()

    printf("\n=== 对齐分配器 ===\n");
    {
        AlignedAllocStream stream;
        char data[100];
        memset(data, 0xFF, sizeof(data));
        stream.write(data, sizeof(data));
        printf("缓冲区对齐状态: %s\n",
               stream.isAligned() ? "16字节对齐 ✓" : "未对齐 ✗");
        printf("分配大小: %u 字节（向上对齐到16的倍数）\n",
               stream.getAllocSize());
    }

    return 0;
}
```

## 本节小结

- **tTVPMemoryStream** 是一个将内存缓冲区包装成 tTJSBinaryStream 接口的工具类，支持两种模式：自有模式（可读写，自动管理内存）和引用模式（只读，不拥有内存）
- **分级增长策略**根据当前缓冲区大小选择不同的扩容步长（4KB/16KB/256KB/~2MB），在减少 realloc 次数和控制内存浪费之间取得平衡
- **tTVPPartialStream** 在一个底层流上创建只读窗口，将物理偏移隐藏为从 0 开始的逻辑地址。它拥有底层流的所有权，析构时自动释放
- **TArchiveStream** 与 tTVPPartialStream 功能类似但设计不同：它持有 tTVPArchive 的引用计数（AddRef/Release），而不是 delete 底层流
- **tTVPLocalTempStorageHolder** 是一个 RAII 工具，自动检测文件是否可本地访问，不可访问时复制到临时目录，析构时自动清理
- 三个类的 **虚拟分配器** 和 **所有权转移** 模式都体现了 C++ RAII 原则和多态扩展设计

### 题目 1：tTVPMemoryStream 的 Seek 行为分析

给定以下操作序列，分析每一步后 `CurrentPos` 和 `Size` 的值：

```cpp
tTVPMemoryStream stream;
// 初始状态: CurrentPos=0, Size=0, AllocSize=0

stream.Write("ABCDEF", 6);
// Step 1: CurrentPos=?, Size=?

stream.Seek(2, TJS_BS_SEEK_SET);
// Step 2: CurrentPos=?, Size=?

stream.Write("XY", 2);
// Step 3: CurrentPos=?, Size=?

stream.Seek(-1, TJS_BS_SEEK_END);
// Step 4: CurrentPos=?, Size=?

stream.Seek(100, TJS_BS_SEEK_SET);
// Step 5: CurrentPos=?, Size=?
```

<details>
<summary>查看答案</summary>

```
Step 1: Write("ABCDEF", 6)
  newpos = 0 + 6 = 6
  需要扩容：AllocSize += 4096 → AllocSize = 4096
  memcpy 写入 "ABCDEF"
  CurrentPos = 6, Size = 6
  缓冲区内容: "ABCDEF"

Step 2: Seek(2, TJS_BS_SEEK_SET)
  offset=2, >= 0 且 <= Size(6) → 条件满足
  CurrentPos = 2, Size = 6（Seek 不改变 Size）

Step 3: Write("XY", 2)
  newpos = 2 + 2 = 4
  4 < AllocSize(4096) → 无需扩容
  memcpy 覆盖 [2..3] 为 "XY"
  CurrentPos = 4
  4 > Size(6)? 否 → Size 不变
  CurrentPos = 4, Size = 6
  缓冲区内容: "ABXYEF"

Step 4: Seek(-1, TJS_BS_SEEK_END)
  newpos = -1 + 6 = 5
  newpos >= 0 且 np(5) <= Size(6) → 条件满足
  CurrentPos = 5, Size = 6

Step 5: Seek(100, TJS_BS_SEEK_SET)
  offset=100, >= 0 但 100 > Size(6) → 条件不满足！
  CurrentPos 保持不变
  CurrentPos = 5, Size = 6
  （注意：内存流的 Seek 不允许定位到超出 Size 的位置）
```

</details>

### 题目 2：实现 tTVPPartialStream 的 SEEK_CUR 和 SEEK_END

源码中 tTVPPartialStream 的 Seek 对 SEEK_CUR 的边界检查条件是 `if(offset >= 0 && offset <= static_cast<tjs_int64>(Size))`，这个条件检查的是 offset 本身，而不是计算后的 newpos。这是否正确？请分析这个实现的行为，并说明是否存在 bug。

<details>
<summary>查看答案</summary>

这确实是一个潜在的 **bug**（或至少是不完善的实现）。分析如下：

```cpp
case TJS_BS_SEEK_CUR:
    newpos = offset + CurrentPos;
    if(offset >= 0 && offset <= static_cast<tjs_int64>(Size)) {
        // 问题：检查的是 offset，而不是 newpos
        newpos += Start;
        CurrentPos = Stream->Seek(newpos, TJS_BS_SEEK_SET) - Start;
    }
    return CurrentPos;
```

**问题场景举例：**

假设 `Size = 100`, `CurrentPos = 90`：
- `Seek(20, SEEK_CUR)`: offset=20, offset >= 0 且 20 <= 100 → **允许**
  - 但 newpos = 90 + 20 = 110，超出窗口范围！
  - 底层流会被定位到 `Start + 110`，超出了窗口边界

- `Seek(-5, SEEK_CUR)`: offset=-5, offset >= 0 → **不满足，被拒绝**
  - 但 newpos = 90 + (-5) = 85，完全合法！
  - 正常的向后移动被错误地拒绝了

**正确的检查应该是：**

```cpp
case TJS_BS_SEEK_CUR:
    newpos = offset + CurrentPos;
    if(newpos >= 0 && newpos <= static_cast<tjs_int64>(Size)) {
        newpos += Start;
        CurrentPos = Stream->Seek(newpos, TJS_BS_SEEK_SET) - Start;
    }
    return CurrentPos;
```

SEEK_END 也有类似的问题：`Seek(0, SEEK_END)` 应该定位到末尾，但 offset=0 在 "offset >= 0 && offset <= Size" 条件下会被允许（这个恰好是对的），而 `Seek(-5, SEEK_END)` 的 offset=-5 不满足 `offset >= 0`，会被错误拒绝。

**结论：** 这是源码中的一个已知 bug——SEEK_CUR 和 SEEK_END 的边界检查条件应该检查计算后的 `newpos` 而不是原始的 `offset`。在实际使用中，因为 PartialStream 主要用于顺序读取（SEEK_SET 定位 + 顺序 Read），这个 bug 很少被触发。

</details>

### 题目 3：设计一个"计数器内存流"子类

编写一个 `CountingMemoryStream` 类，继承 tTVPMemoryStream（或示例中的 `MemoryStreamBase`），要求：
1. 覆写虚拟分配器三件套（Alloc/Realloc/Free）
2. 统计总分配次数和当前分配的总字节数
3. 当累计分配超过某个阈值时打印警告
4. 提供 `printReport()` 方法输出统计信息

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <cstdint>

class CountingMemoryStream {
protected:
    uint8_t *block_ = nullptr;
    bool reference_ = false;
    uint32_t size_ = 0;
    uint32_t allocSize_ = 0;
    uint32_t pos_ = 0;

    // 统计数据
    int allocCount_ = 0;
    int reallocCount_ = 0;
    int freeCount_ = 0;
    size_t currentBytes_ = 0;    // 当前已分配字节
    size_t peakBytes_ = 0;       // 峰值分配字节
    size_t totalAllocated_ = 0;  // 历史总分配字节（累计）
    size_t threshold_;            // 警告阈值

    virtual void *doAlloc(size_t sz) {
        allocCount_++;
        currentBytes_ += sz;
        totalAllocated_ += sz;
        if (currentBytes_ > peakBytes_) peakBytes_ = currentBytes_;
        checkThreshold();
        return malloc(sz);
    }

    virtual void *doRealloc(void *p, size_t sz) {
        reallocCount_++;
        // realloc 后当前字节数 = 新大小（旧的被替换）
        size_t oldSize = allocSize_;  // 近似值
        currentBytes_ = currentBytes_ - oldSize + sz;
        totalAllocated_ += sz;
        if (currentBytes_ > peakBytes_) peakBytes_ = currentBytes_;
        checkThreshold();
        return realloc(p, sz);
    }

    virtual void doFree(void *p) {
        freeCount_++;
        currentBytes_ -= allocSize_;  // 近似
        free(p);
    }

    void checkThreshold() {
        if (totalAllocated_ > threshold_) {
            printf("⚠️  内存警告：累计分配 %zu 字节，超过阈值 %zu 字节\n",
                   totalAllocated_, threshold_);
        }
    }

public:
    explicit CountingMemoryStream(size_t warnThreshold = 1024 * 1024)
        : threshold_(warnThreshold) {}

    ~CountingMemoryStream() {
        if (block_ && !reference_) doFree(block_);
    }

    uint32_t write(const void *data, uint32_t writeSize) {
        if (reference_) return 0;
        uint32_t newpos = pos_ + writeSize;
        if (newpos > allocSize_) {
            uint32_t growth;
            if (allocSize_ < 64*1024) growth = 4*1024;
            else if (allocSize_ < 512*1024) growth = 16*1024;
            else growth = 256*1024;
            allocSize_ += growth;
            if (newpos > allocSize_) allocSize_ = newpos;

            if (!block_)
                block_ = (uint8_t *)doAlloc(allocSize_);
            else
                block_ = (uint8_t *)doRealloc(block_, allocSize_);
        }
        memcpy(block_ + pos_, data, writeSize);
        pos_ = newpos;
        if (pos_ > size_) size_ = pos_;
        return writeSize;
    }

    void printReport() const {
        printf("╔══════════════════════════════════╗\n");
        printf("║     内存流分配统计报告           ║\n");
        printf("╠══════════════════════════════════╣\n");
        printf("║ Alloc 次数:    %8d          ║\n", allocCount_);
        printf("║ Realloc 次数:  %8d          ║\n", reallocCount_);
        printf("║ Free 次数:     %8d          ║\n", freeCount_);
        printf("║ 当前分配:      %8zu 字节    ║\n", currentBytes_);
        printf("║ 峰值分配:      %8zu 字节    ║\n", peakBytes_);
        printf("║ 累计分配:      %8zu 字节    ║\n", totalAllocated_);
        printf("║ 警告阈值:      %8zu 字节    ║\n", threshold_);
        printf("╚══════════════════════════════════╝\n");
    }
};

int main() {
    // 设置 50KB 的警告阈值
    CountingMemoryStream stream(50 * 1024);

    // 写入数据直到触发警告
    char data[1024];
    memset(data, 'X', sizeof(data));

    printf("开始写入数据...\n\n");
    for (int i = 0; i < 100; i++) {
        stream.write(data, sizeof(data));
    }

    printf("\n最终统计:\n");
    stream.printReport();

    /* 预期输出：
       开始写入数据...

       ⚠️  内存警告：累计分配 XXXXX 字节，超过阈值 51200 字节
       ...（可能多次触发）

       最终统计:
       ╔══════════════════════════════════╗
       ║     内存流分配统计报告           ║
       ╠══════════════════════════════════╣
       ║ Alloc 次数:           1          ║
       ║ Realloc 次数:        ~20         ║
       ║ Free 次数:            0          ║
       ║ 当前分配:       ~102400 字节     ║
       ║ 峰值分配:       ~102400 字节     ║
       ║ 累计分配:       ~XXXXXX 字节     ║
       ║ 警告阈值:         51200 字节     ║
       ╚══════════════════════════════════╝
    */

    return 0;
}
```

### 示例 2：分级增长策略可视化

```cpp

#include <cstring>
#include <cstdint>

// 追踪 realloc 调用次数的内存流
class TrackedMemoryStream {
    uint8_t *block_ = nullptr;
    uint32_t size_ = 0;
    uint32_t allocSize_ = 0;
    uint32_t pos_ = 0;
    int reallocCount_ = 0;  // realloc 调用计数

public:
    ~TrackedMemoryStream() { free(block_); }

    // 带分级增长的写入
    uint32_t writeTiered(const void *data, uint32_t writeSize) {
        uint32_t newpos = pos_ + writeSize;
        if (newpos > allocSize_) {
            uint32_t growth;
            if (allocSize_ < 64 * 1024)
                growth = 4 * 1024;
            else if (allocSize_ < 512 * 1024)
                growth = 16 * 1024;
            else if (allocSize_ < 4096 * 1024)
                growth = 256 * 1024;
            else
                growth = 2024 * 1024;  // 注意：与源码一致，2024 而非 2048

            allocSize_ += growth;
            if (newpos > allocSize_)
                allocSize_ = newpos;

            block_ = (uint8_t *)realloc(block_, allocSize_);
            reallocCount_++;
        }
        memcpy(block_ + pos_, data, writeSize);
        pos_ = newpos;
        if (pos_ > size_) size_ = pos_;
        return writeSize;
    }

    // 固定步长写入（对照组）
    uint32_t writeFixed(const void *data, uint32_t writeSize,
                        uint32_t fixedGrowth) {
        uint32_t newpos = pos_ + writeSize;
        if (newpos > allocSize_) {
            allocSize_ += fixedGrowth;
            if (newpos > allocSize_)
                allocSize_ = newpos;
            block_ = (uint8_t *)realloc(block_, allocSize_);
            reallocCount_++;
        }
        memcpy(block_ + pos_, data, writeSize);
        pos_ = newpos;
        if (pos_ > size_) size_ = pos_;
        return writeSize;
    }

    int getReallocCount() const { return reallocCount_; }
    uint32_t getAllocSize() const { return allocSize_; }

    void reset() {
        free(block_);
        block_ = nullptr;
        size_ = 0;
        allocSize_ = 0;
        pos_ = 0;
        reallocCount_ = 0;
    }
};
```
