## 为什么需要类型转换系统？

在 KrKr2 插件开发中，C++ 和 TJS2 是两个完全不同的类型世界：

- **C++ 侧**：有 `int`、`float`、`double`、`std::string`、自定义类等丰富的静态类型
- **TJS2 侧**：所有值都通过 `tTJSVariant`（万能变量类型，类似 JavaScript 的动态类型）传递

当你在插件中注册一个 C++ 方法给 TJS2 调用时，需要解决两个核心问题：

1. **参数传入**：TJS2 脚本传来的 `tTJSVariant` 参数，如何安全地转换为 C++ 方法需要的具体类型（如 `int`、`const char*`）？
2. **返回值传出**：C++ 方法返回的具体类型值，如何打包成 TJS2 能理解的 `tTJSVariant`？

如果没有自动类型转换系统，每注册一个方法都要手写大量的类型检查和转换代码。ncbind 的类型系统正是为了消除这种重复劳动——它通过 C++ 模板元编程（Template Metaprogramming，利用编译器在编译期计算类型信息的技术），让类型转换在编译期自动选择和生成，运行时零开销。

下面这张图展示了类型转换的整体架构：

```
┌─────────────────────────────────────────────────────────────┐
│                    TJS2 脚本世界                              │
│   var result = myPlugin.calculate(42, "hello");              │
│                     ↓ 参数: tTJSVariant[]                    │
│                     ↑ 返回: tTJSVariant                      │
├─────────────────────────────────────────────────────────────┤
│              ncbTypeConvertor 自动转换层                      │
│                                                              │
│   tTJSVariant ──→ SelectConvertorType ──→ int               │
│   tTJSVariant ──→ SelectConvertorType ──→ const char*       │
│   double      ──→ SelectConvertorType ──→ tTJSVariant       │
│                                                              │
│   转换策略:  DirectCopy | CastCopy | Boxing/Unboxing         │
│              | NarrowChar | WideChar | Custom                │
├─────────────────────────────────────────────────────────────┤
│                    C++ 插件世界                               │
│   double MyPlugin::calculate(int a, const char* b) { ... }  │
└─────────────────────────────────────────────────────────────┘
```

## ncbTypedefs — 基础类型别名集合

`ncbTypedefs` 是 ncbind 框架的"类型字典"，定义在 `cpp/core/plugin/ncbind.hpp` 第 28-65 行。它将 TJS2 引擎中散落各处的类型名收集到一个 `struct` 中，方便框架内部统一引用。

### 源码解析

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 28-65 行
struct ncbTypedefs {
    // === 基础类型别名 ===
    typedef tjs_char const*           NameT;      // 名称类型：TJS2 宽字符指针（类似 wchar_t*）
    typedef tjs_uint32                FlagsT;     // 标志位类型：32 位无符号整数
    typedef tjs_int32                 IdentT;     // 标识符类型：32 位有符号整数（用于类 ID）

    // === TJS2 引擎相关类型别名 ===
    typedef MethodCaller              CallerT;    // 方法调用器类型
    typedef tTJSNativeInstanceType    InstanceTypeT;  // 原生实例类型
    typedef tTJSNativeClassForPlugin  ClassObjectT;   // 插件用的 TJS2 类对象类型

    // === 编译期工具模板 ===
    template <typename T> struct Tag { typedef T Type; };       // 类型标签（Tag Dispatch 技术）
    template <int  N> struct NumTag  { enum { n = N }; };       // 数值标签（编译期分支）
    template <bool B> struct BoolTag { enum { b = B }; };       // 布尔标签（编译期分支）

    // === Variant 类型获取 ===
    static inline tTJSVariantType GetVariantType(tTJSVariant const &var) {
        return (const_cast<tTJSVariant*>(&var))->Type();        // 获取 Variant 内部存储的实际类型
    }

    // === 回调类型 ===
    typedef tTJSNativeClassMethodCallback CallbackT; // TJS2 原生方法回调签名

    // === 带实例的回调 ===
    template <class T>
    struct CallbackWithInstance {
        // 回调函数签名：接收返回值指针、参数数量、参数数组、原生实例指针
        typedef tjs_error (*Type)(tTJSVariant *result,
                                   tjs_int numparams,
                                   tTJSVariant **param,
                                   T *nativeInstance);
    };

    // === 编译期类型比较工具 ===
    template <typename A, typename B>
    struct TypeEqual       { enum { Result = false }; };  // 默认：不同类型
    template <typename A>
    struct TypeEqual<A, A> { enum { Result = true  }; };  // 特化：相同类型
};
```

### 关键点说明

**Tag Dispatch（标签分发）** 是 C++ 模板元编程中的经典技术。`Tag<T>` 将类型信息"包装"成一个空结构体，用作函数参数的类型标签，让编译器在重载决议时选择正确的函数版本。例如：

```cpp
// 不使用 Tag Dispatch 的写法——编译器无法区分两个同签名函数
template <typename T>
T convert(tTJSVariant const& var);  // 从 Variant 转为 int？string？编译器不知道

// 使用 Tag Dispatch 的写法——编译器通过参数类型自动选择
template <typename T>
T convert(tTJSVariant const& var, ncbTypedefs::Tag<T> const&) {
    // 编译器根据 Tag<int> 或 Tag<string> 选择不同的特化版本
}

// 调用时：
int    i = convert(var, ncbTypedefs::Tag<int>());     // 选择 int 版本
ttstr  s = convert(var, ncbTypedefs::Tag<ttstr>());   // 选择 ttstr 版本
```

`NumTag<N>` 的用途类似——当需要根据类型的 `sizeof` 大小选择不同处理逻辑时使用。例如 `ncbStringConvertor` 用 `NumTag<sizeof(value_type)>` 来区分窄字符串（`char`，1 字节）和宽字符串（`wchar_t`/`tjs_char`，通常 2 或 4 字节）。

## ncbTypeConvertor — 类型转换核心模板

`ncbTypeConvertor` 是 ncbind 类型系统的心脏，定义在 `cpp/core/plugin/ncbind.hpp` 第 236-358 行。它通过一系列嵌套模板，在编译期自动决定"从类型 A 转换到类型 B 应该使用哪个转换器"。

### 整体架构

```
ncbTypeConvertor
├── Conversion<FROM, TO>         判断 FROM→TO 是否可隐式转换
├── Stripper<T>                  剥离 const/*/& 得到裸类型
├── ToPointer<T>                 统一获取指针（值→&、引用→&、指针→原样）
├── ToTarget<T>                  指针还原为目标修饰形式（*→&、*→*）
├── DirectCopy                   直接赋值策略
├── CastCopy<CAST>               经由中间类型 CAST 做 static_cast
├── SpecialMap<T, IsSrc>         特殊化映射表（记录类型是否有自定义转换器）
└── SelectConvertorType<SRC,DST> 最终选择器——编译期决定用哪个转换器
```

### Conversion — 编译期类型可转换性检测

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 239-258 行
template <typename FROM, typename TO>
struct Conversion {
private:
    typedef char OK;                        // 大小为 1 的类型
    typedef struct { char ng[2]; } NG;      // 大小为 2 的类型
    static OK check(TO) {};                 // 如果 FROM 能隐式转换为 TO，匹配这个重载
    static NG check(...) {};                // 否则匹配可变参数版本
    static FROM wrap() {};                  // 提供一个 FROM 类型的值（仅用于编译期推导）
public:
    enum {
        Exists = (sizeof(check(wrap())) == sizeof(OK)),  // FROM→TO 可转换？
        Same   = false                                    // 是否完全相同的类型？
    };
};
// 完全特化：相同类型
template <typename T>
struct Conversion<T, T> { enum { Exists = true, Same = true }; };
```

这是经典的 **SFINAE**（Substitution Failure Is Not An Error，替换失败不是错误）技巧。编译器在编译期尝试将 `FROM` 传给 `check(TO)` 函数：
- 如果能隐式转换，选择返回 `OK`（大小 1）的重载
- 如果不能，选择返回 `NG`（大小 2）的可变参数重载
- 通过 `sizeof` 比较结果大小，在编译期确定 `Exists` 的值

### Stripper — 类型修饰符剥离器

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 261-265 行
template <typename T> struct Stripper             { typedef T Type; };  // 裸类型：原样
template <typename T> struct Stripper<T*>         { typedef typename Stripper<T>::Type Type; };  // 剥掉 *
template <typename T> struct Stripper<T&>         { typedef typename Stripper<T>::Type Type; };  // 剥掉 &
template <typename T> struct Stripper<const    T> { typedef typename Stripper<T>::Type Type; };  // 剥掉 const
```

递归特化模式：如果类型是 `const MyClass*&`，Stripper 会依次剥掉 `&` → `const` → `*`，最终得到 `MyClass`。这在 Boxing/Unboxing 时特别重要——无论你传入的是 `MyObj`、`MyObj*`、`const MyObj&`，框架都能找到对应的 `ncbInstanceAdaptor<MyObj>`。

### 三种转换策略

ncbind 提供三种基本转换策略，按优先级从高到低：

| 策略 | 类名 | 适用场景 | 示例 |
|------|------|---------|------|
| 特殊化转换 | `SpecialMap` 中注册的自定义转换器 | 有专门的转换逻辑 | `ncbNarrowCharConvertor`（窄字符串）、`ncbNativeObjectBoxing`（对象装箱） |
| 直接拷贝 | `DirectCopy` | 源类型和目标类型可以直接赋值 | `tTJSVariant` → `tTJSVariant` |
| 类型转换拷贝 | `CastCopy<CAST>` | 需要经由中间类型做 `static_cast` | `int` → `tTVInteger` → `tTJSVariant` |

### SelectConvertorType — 编译期自动选择

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 343-357 行（简化版）
template <typename SRC, typename DST>
struct SelectConvertorType {
    // 1. 检查 SRC 是否在 SpecialMap 中有特殊化
    typedef hasSpecial<SRC, true>  SrcSpecialT;
    // 2. 检查 DST 是否在 SpecialMap 中有特殊化
    typedef hasSpecial<DST, false> DstSpecialT;

    // 3. 如果任一有特殊化 → 使用特殊化转换器（DST 优先）
    //    否则 → 使用 DirectCopy（或编译报错）
    typedef typename ifelse<
        SrcSpecialT::Exists || DstSpecialT::Exists,
        specialSelect,      // 有特殊化转换器
        directSelect<SRC, DST>  // 尝试直接拷贝
    >::Type::Type Type;
};
```

整个选择过程完全在编译期完成，运行时没有任何分支判断开销。

## 内置类型转换注册

ncbind 在 `ncbind.hpp` 第 394-463 行通过宏为所有 C++ 基础类型注册了默认转换器。理解这些注册，你就知道哪些类型可以"零配置"直接在插件方法签名中使用。

### 整数类型 — CastCopy\<tTVInteger\>

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 394-401 行
// 所有 C++ 整数类型都通过 tTVInteger（TJS2 内部的 64 位整数）做中间转换
NCB_TYPECONV_CAST_INTEGER(   signed char);    // signed char ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER( unsigned char);    // unsigned char ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(signed int);        // int ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(unsigned int);      // unsigned int ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(  signed short);    // short ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(unsigned short);    // unsigned short ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(signed long);       // long ↔ tTJSVariant
NCB_TYPECONV_CAST_INTEGER(unsigned long);     // unsigned long ↔ tTJSVariant
```

`NCB_TYPECONV_CAST_INTEGER(type)` 展开后等价于：

```cpp
// 展开示例：NCB_TYPECONV_CAST_INTEGER(int) 的效果
// C++ int → tTJSVariant：先 static_cast<tTVInteger>(int_value)，再赋值给 Variant
// tTJSVariant → C++ int：先从 Variant 取出 tTVInteger，再 static_cast<int>(...)
NCB_TYPECONV_SRCMAP_SET(int, ncbTypeConvertor::CastCopy<tTVInteger>, false);
NCB_TYPECONV_DSTMAP_SET(int, ncbTypeConvertor::CastCopy<tTVInteger>, false);
```

`tTVInteger` 是 TJS2 内部的整数类型（通常是 `tjs_int64`，即 64 位有符号整数）。所有 C++ 整数类型都先转为 `tTVInteger`，再存入 `tTJSVariant`，反之亦然。

### 浮点类型 — CastCopy\<tTVReal\>

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 402-403 行
NCB_TYPECONV_CAST_REAL(float);     // float ↔ tTJSVariant（经由 tTVReal）
NCB_TYPECONV_CAST_REAL(double);    // double ↔ tTJSVariant（经由 tTVReal）
```

`tTVReal` 是 TJS2 内部的浮点类型（通常是 `double`）。`float` 会先提升为 `double` 再存入 Variant。

### 布尔类型

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 404 行
NCB_TYPECONV_CAST(bool, bool);     // bool ↔ tTJSVariant（直接 cast）
```

### 字符串类型 — 窄字符串与宽字符串

字符串转换分两类，因为 TJS2 内部使用宽字符（`tjs_char`，通常是 `wchar_t` 或 `char16_t`），而 C++ 常用窄字符（`char`）：

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 445-463 行

// 窄字符串：需要字符编码转换（char* → tjs_char* 需要宽化）
NCB_TYPECONV_NARROW_STRING(         char const*);   // const char* ↔ tTJSVariant
NCB_TYPECONV_NARROW_STRING(  signed char const*);   // const signed char* ↔ tTJSVariant
NCB_TYPECONV_NARROW_STRING(unsigned char const*);   // const unsigned char* ↔ tTJSVariant

// 宽字符串：可以直接拷贝（因为 TJS2 内部就是宽字符）
NCB_TYPECONV_WIDE_STRING(tjs_char const*);          // const tjs_char* ↔ tTJSVariant
```

**重要提示**：窄字符串转换时，ncbind 会分配临时内存来存放宽化后的字符串（见 `ncbNarrowCharConvertor::ToNChar`）。这个临时内存的生命周期仅在一次方法调用期间有效——如果你需要持久保存字符串，应该在方法内部拷贝一份。

### std::string 和 ttstr

对于 `std::string`、`std::wstring` 等 STL 字符串，需要使用 `NCB_TYPECONV_STL_STRING` 宏注册：

```cpp
// 如果你的插件方法使用 std::string 参数，需要在插件代码中添加：
#include <string>
NCB_TYPECONV_STL_STRING(std::string);    // std::string ↔ tTJSVariant
NCB_TYPECONV_STL_STRING(std::wstring);   // std::wstring ↔ tTJSVariant
```

以下类型可以直接在插件方法签名中使用，无需额外注册：

| C++ 类型 | 转换策略 | 中间类型 | 说明 |
|----------|---------|---------|------|
| `int` / `unsigned int` | CastCopy | `tTVInteger` | 32 位整数 |
| `long` / `unsigned long` | CastCopy | `tTVInteger` | 平台相关长整数 |
| `short` / `unsigned short` | CastCopy | `tTVInteger` | 16 位整数 |

| `iTJSDispatch2*` | Dispatch | — | TJS2 对象接口指针 |

## ncbNativeObjectBoxing — 对象装箱与拆箱

对于简单类型（整数、浮点、字符串），上面的 CastCopy/NarrowChar 转换器就够了。但当你需要把一个 **C++ 自定义类的实例** 传给 TJS2 脚本时，就需要 Boxing/Unboxing（装箱/拆箱）机制。

**Boxing（装箱）** 的含义是：将一个 C++ 对象"包装"到 TJS2 的 `tTJSVariant` 中，使 TJS2 脚本可以像操作普通对象一样操作它。**Unboxing（拆箱）** 则是反向操作：从 `tTJSVariant` 中"提取"出原始的 C++ 对象指针。

### 装箱流程

```
C++ 对象 (MyClass*)
       ↓
ncbNativeObjectBoxing::Boxing
       ↓
ncbInstanceAdaptor<MyClass>::CreateAdaptor(ptr)
       ↓  创建一个 TJS2 对象，内部持有 C++ 指针
tTJSVariant（存储 iTJSDispatch2* 指向适配器对象）
       ↓
TJS2 脚本可以调用 myObj.method() 了
```

### 拆箱流程

```
tTJSVariant（内部是 iTJSDispatch2*）
       ↓
ncbNativeObjectBoxing::Unboxing
       ↓
var.AsObjectNoAddRef() → 取出 iTJSDispatch2*
       ↓
ncbInstanceAdaptor<MyClass>::GetNativeInstance(obj)
       ↓  从 TJS2 对象中提取 C++ 指针
MyClass* （原始 C++ 对象指针）
```

### 源码解析

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 503-548 行
struct ncbNativeObjectBoxing {
    typedef tTJSVariant VarT;
    typedef ncbTypeConvertor ConvT;

    /// Boxing：C++ → TJS2
    struct Boxing : public ncbStrictResultConvertor {
        // 根据传入方式（值/引用/指针）决定是否拷贝
        template <typename T> struct box            // 值传递：new 一份拷贝
            { static T* Get(T const &t) { return new T(t); }
              enum { Sticky = false }; };           // 非 Sticky：TJS2 销毁时 delete
        template <typename T> struct box<T&>        // 引用传递：不拷贝
            { static T* Get(T &t) { return &t; }
              enum { Sticky = true }; };            // Sticky：TJS2 不负责 delete
        template <typename T> struct box<T const&>  // const 引用：不拷贝
            { static T* Get(T const &t) { return const_cast<T*>(&t); }
              enum { Sticky = true }; };
        template <typename T> struct box<T*>        // 指针传递：直接使用
            { static T* Get(T *t) { return t; }
              enum { Sticky = false }; };           // 调用者负责生命周期

        template <typename SRC>
        inline void operator()(VarT &dst, SRC src, ncbTypedefs::Tag<SRC> const&) const {
            typedef typename ConvT::Stripper<SRC>::Type ClassT;  // 剥离修饰符得到裸类
            typedef ncbInstanceAdaptor<ClassT> AdaptorT;

            ClassT *p = box<SRC>::Get(src);          // 获取 C++ 对象指针
            bool const s = box<SRC>::Sticky;         // 是否 Sticky
            iTJSDispatch2 *adpobj =
                AdaptorT::CreateAdaptor(p, s);       // 创建 TJS2 适配器对象
            dst = tTJSVariant(adpobj, adpobj);       // 存入 Variant
            adpobj->Release();                       // Variant 已持有引用，释放临时引用
        }
    };

    /// Unboxing：TJS2 → C++
    struct Unboxing {
        template <typename DST>
        inline void operator()(DST &dst, VarT const &src) const {
            typedef typename ConvT::Stripper<DST>::Type ClassT;
            typedef ncbInstanceAdaptor<ClassT> AdaptorT;

            iTJSDispatch2 *obj = src.AsObjectNoAddRef();    // 从 Variant 取出 TJS2 对象
            ClassT *p = AdaptorT::GetNativeInstance(obj, true);  // 提取 C++ 指针
            dst = ConvT::ToTarget<DST>::Get(p);             // 转换为目标修饰形式
        }
    };
};
```

### Sticky 标志详解

`Sticky` 是装箱机制中一个非常重要的概念，决定了 C++ 对象的生命周期归谁管理：

| 传递方式 | Sticky | 含义 | 典型场景 |
|----------|--------|------|---------|
| 值 `T` | `false` | Boxing 时 `new T(t)` 拷贝一份，TJS2 对象销毁时 `delete` | 返回值为值类型 |
| 引用 `T&` / `const T&` | `true` | 不拷贝，直接引用原对象；TJS2 对象销毁时 **不 delete** | 返回成员变量的引用 |
| 指针 `T*` | `false` | 直接使用指针；TJS2 对象销毁时 `delete` | 返回 `new` 出来的对象 |
| const 指针 `const T*` | `false` | 同指针，但去掉 const | 少见场景 |

**常见陷阱**：如果你的 C++ 方法返回 `T&`（引用），Boxing 会设置 Sticky=true，意味着 TJS2 不会 delete 这个对象。但如果原始 C++ 对象先于 TJS2 对象被销毁，TJS2 脚本再访问时就会产生**悬挂指针**（Dangling Pointer，指向已释放内存的指针）。因此，返回引用时必须确保引用目标的生命周期长于 TJS2 对象。

### 注册自定义类型为可 Boxing

要让一个 C++ 类支持自动 Boxing/Unboxing，只需一行宏：

```cpp
// 示例：让 MyRect 类支持作为方法参数和返回值自动转换
class MyRect {
public:
    int x, y, width, height;
    MyRect(int x, int y, int w, int h) : x(x), y(y), width(w), height(h) {}
};

// 在全局作用域注册（通常放在插件 .cpp 文件的顶部）
NCB_TYPECONV_BOXING(MyRect);

// 之后在 ncbind 注册的方法中就可以直接使用 MyRect 作为参数/返回值：
class MyPlugin {
public:
    MyRect getRect() { return MyRect(0, 0, 100, 200); }           // 返回值自动 Boxing
    void setRect(MyRect const& rect) { /* ... */ }                 // 参数自动 Unboxing
    // TJS2 脚本调用：
    // var r = myPlugin.getRect();  // r 是一个 TJS2 对象，内部持有 MyRect
    // myPlugin.setRect(r);          // r 自动 Unboxing 为 MyRect const&
};
```

**前提条件**：`MyRect` 必须已经通过 `NCB_REGISTER_CLASS(MyRect)` 注册为 TJS2 类（将在下一节详讲），否则 `ncbInstanceAdaptor<MyRect>::CreateAdaptor` 会找不到类对象而失败。

## ncbInstanceAdaptor — 实例适配器

`ncbInstanceAdaptor<T>` 是连接 C++ 对象和 TJS2 对象的"桥梁"，定义在 `cpp/core/plugin/ncbind.hpp` 第 119-232 行。每当 TJS2 脚本创建一个已注册的 C++ 类的实例时，实际上创建的是一个 `ncbInstanceAdaptor<T>` 对象，它内部持有一个指向真正 C++ 对象的指针。

### 核心数据结构

```cpp
// 文件: cpp/core/plugin/ncbind.hpp 第 119-148 行（简化）
template <class T>
struct ncbInstanceAdaptor : public tTJSNativeInstance {
    // tTJSNativeInstance 是 TJS2 引擎的原生实例基类
    // 每个 TJS2 对象可以关联一个 tTJSNativeInstance

    NativeClassT *_instance;  // 指向真正的 C++ 对象
    bool _sticky;             // 如果为 true，析构时不 delete _instance

    void Invalidate() override {
        _deleteInstance();    // TJS2 对象被垃圾回收时调用
    }

private:
    void _deleteInstance() {
        if (_instance && !_sticky) delete _instance;  // 仅在非 Sticky 时 delete
        _instance = 0;
        _sticky = false;
    }
};
```

### 关键静态方法

`ncbInstanceAdaptor<T>` 提供了一组静态方法，用于在 `iTJSDispatch2*`（TJS2 对象接口）和 C++ 实例之间互相转换：

```cpp
// 从 TJS2 对象中提取 C++ 实例
static NativeClassT* GetNativeInstance(iTJSDispatch2 *obj, bool err = false);
// 用法：MyClass* ptr = ncbInstanceAdaptor<MyClass>::GetNativeInstance(tjsObj);

// 将 C++ 实例设置到 TJS2 对象中
static bool SetNativeInstance(iTJSDispatch2 *obj, NativeClassT *instance, bool err = false);

// 创建一个包含 C++ 实例的 TJS2 对象
static iTJSDispatch2* CreateAdaptor(NativeClassT *inst, bool sticky = false, bool err = false);
// 用法：iTJSDispatch2* tjsObj = ncbInstanceAdaptor<MyClass>::CreateAdaptor(new MyClass());
```

### 完整示例：手动装箱与拆箱

```cpp
#include "ncbind.hpp"
#include <cstdio>

// 定义一个简单的 C++ 类
class Point {
public:
    int x, y;
    Point(int x, int y) : x(x), y(y) {}
    void print() const {
        std::printf("Point(%d, %d)\n", x, y);  // 输出: Point(10, 20)
    }
};

// 手动装箱：将 C++ Point 对象包装为 TJS2 对象
iTJSDispatch2* boxPoint(int x, int y) {
    Point* p = new Point(x, y);
    // CreateAdaptor 会创建一个 TJS2 对象，内部持有 p
    // sticky=false 表示 TJS2 对象销毁时自动 delete p
    iTJSDispatch2* tjsObj = ncbInstanceAdaptor<Point>::CreateAdaptor(p, false);
    return tjsObj;  // 调用者负责 Release()
}

// 手动拆箱：从 TJS2 对象中提取 C++ Point 指针
void unboxAndPrint(iTJSDispatch2* tjsObj) {
    // GetNativeInstance 从 TJS2 对象的 NativeInstance 中提取 Point*
    // err=true 表示如果提取失败则抛出 TJS2 异常
    Point* p = ncbInstanceAdaptor<Point>::GetNativeInstance(tjsObj, true);
    if (p) {
        p->print();  // 输出: Point(10, 20)
    }
}

// 使用 NCB_TYPECONV_BOXING 后，上面的手动操作由框架自动完成
NCB_TYPECONV_BOXING(Point);
```

## ncbPropAccessor — Dictionary/Array 便捷访问器

`ncbPropAccessor` 是 ncbind 提供的实用工具类，定义在 `cpp/core/plugin/ncbind.hpp` 第 602-819 行。它封装了 `iTJSDispatch2` 接口的繁琐调用，让 C++ 代码可以像操作普通 map/vector 一样读写 TJS2 的 Dictionary（字典，类似 JSON 对象）和 Array（数组）。

### 为什么需要 ncbPropAccessor？

不使用 ncbPropAccessor 时，从 TJS2 Dictionary 中读取一个整数值需要这样写：

```cpp
// 不使用 ncbPropAccessor：啰嗦且容易出错
void readConfigRaw(iTJSDispatch2* dict) {
    tTJSVariant var;
    // PropGet 需要 5 个参数：标志位、键名、hint、输出变量、上下文对象
    tjs_error err = dict->PropGet(0, TJS_W("width"), nullptr, &var, dict);
    if (TJS_SUCCEEDED(err)) {
        int width = static_cast<int>((tjs_int64)var);  // 手动类型转换
        std::printf("width = %d\n", width);
    }
}
```

使用 ncbPropAccessor 后，同样的操作简化为：

```cpp
// 使用 ncbPropAccessor：简洁清晰
void readConfigEasy(iTJSDispatch2* dict) {
    ncbPropAccessor acc(dict);       // 包装 iTJSDispatch2*
    int width = acc.getIntValue(TJS_W("width"), 0);  // 一行搞定，带默认值
    std::printf("width = %d\n", width);
}
```

### 构造方式

`ncbPropAccessor` 提供四种构造方式，适配不同场景：

```cpp
// 1. 从 iTJSDispatch2* 构造（最常用）
// addref=true 时自动增加引用计数，析构时自动 Release
ncbPropAccessor acc1(dispatch2Ptr);

// 2. 拷贝构造
ncbPropAccessor acc2(acc1);  // 共享同一个 dispatch 对象，增加引用计数

// 3. 从 tTJSVariant 构造
ncbPropAccessor acc3(someVariant);  // 自动调用 var.AsObject() 获取 dispatch

// 4. 从全局变量名构造
ncbPropAccessor acc4(TJS_W("System"));  // 获取全局作用域中名为 "System" 的对象
```

### 读取操作

```cpp
void demonstrateReading(iTJSDispatch2* dict) {
    ncbPropAccessor acc(dict);

    // === 按键名读取（Dictionary 操作）===
    int    width  = acc.getIntValue(TJS_W("width"), 800);     // 默认值 800
    double scale  = acc.getRealValue(TJS_W("scale"), 1.0);    // 默认值 1.0
    ttstr  title  = acc.getStrValue(TJS_W("title"), TJS_W("Untitled"));

    // 检查键是否存在
    if (acc.HasValue(TJS_W("optionalKey"))) {
        // 键存在时才读取
    }

    // 检查键是否存在并获取类型
    tTJSVariantType type;
    if (acc.HasValue(TJS_W("data"), nullptr, &type)) {
        // type 可能是 tvtInteger、tvtString、tvtObject 等
    }

    // === 按索引读取（Array 操作）===
    int count = acc.GetArrayCount();  // 获取数组长度
    for (int i = 0; i < count; i++) {
        int val = acc.getIntValue(i, 0);  // 按索引读取
        std::printf("[%d] = %d\n", i, val);
    }

    // 泛型读取（需要显式指定类型标签）
    tjs_int val = acc.GetValue(TJS_W("key"),
                               ncbTypedefs::Tag<tjs_int>());
}
```

## 自定义类型转换器

当 ncbind 的内置转换器不能满足需求时，你可以编写自定义转换器。典型场景包括：

- 枚举类型（需要 `int` 与 `enum` 之间的映射）
- 复合类型（需要拆分/合并多个字段）
- 第三方库的类型（如 `cv::Mat`、`nlohmann::json`）

### 编写自定义转换器的步骤

自定义转换器是一个 `struct`，提供两个 `operator()` 重载——一个负责"C++ → TJS2"，一个负责"TJS2 → C++"：

```cpp
// 示例：为 RGB 颜色结构体编写自定义转换器
struct Color {
    uint8_t r, g, b;
    Color() : r(0), g(0), b(0) {}
    Color(uint8_t r, uint8_t g, uint8_t b) : r(r), g(g), b(b) {}
};

// 自定义转换器：Color ↔ tTJSVariant
// 策略：在 TJS2 侧用一个整数 0xRRGGBB 表示颜色
struct ColorConvertor {
    // C++ Color → TJS2 Variant（装箱方向）
    void operator()(tTJSVariant &dst, Color const &src) {
        // 将 r, g, b 打包为一个 24 位整数
        tjs_int64 packed = (static_cast<tjs_int64>(src.r) << 16)
                         | (static_cast<tjs_int64>(src.g) << 8)
                         | (static_cast<tjs_int64>(src.b));
        dst = packed;  // 存为整数 Variant
    }

    // TJS2 Variant → C++ Color（拆箱方向）
    void operator()(Color &dst, tTJSVariant const &src) {
        tjs_int64 packed = src;  // Variant 自动转为整数
        dst.r = static_cast<uint8_t>((packed >> 16) & 0xFF);
        dst.g = static_cast<uint8_t>((packed >> 8)  & 0xFF);
        dst.b = static_cast<uint8_t>( packed        & 0xFF);
    }
};

// 注册自定义转换器
NCB_SET_CONVERTOR(Color, ColorConvertor);
// 展开后等价于：
// NCB_SET_TOVARIANT_CONVERTOR(Color, ColorConvertor);  // Color → Variant
// NCB_SET_TOVALUE_CONVERTOR(Color, ColorConvertor);     // Variant → Color
```

注册后，`Color` 就可以在插件方法签名中直接使用：

```cpp
class ColorPlugin {
public:
    Color lighten(Color c, int amount) {
        // TJS2 传入整数 0xRRGGBB，自动转为 Color
        c.r = static_cast<uint8_t>(std::min(255, c.r + amount));
        c.g = static_cast<uint8_t>(std::min(255, c.g + amount));
        c.b = static_cast<uint8_t>(std::min(255, c.b + amount));
        return c;  // 返回值自动转为整数 Variant
    }
};

NCB_REGISTER_CLASS(ColorPlugin) {
    NCB_METHOD(lighten);
};
// TJS2 脚本: var brighter = colorPlugin.lighten(0xFF0000, 50);
// brighter 的值为 0xFF3232（红色变亮了）
```

### 单向转换器

有时你只需要单向转换（只作参数或只作返回值），可以使用 `NCB_SET_TOVARIANT_CONVERTOR` 或 `NCB_SET_TOVALUE_CONVERTOR`：

```cpp
// 示例：一个只能作为返回值的"只读"状态信息
struct StatusInfo {
    int code;
    std::string message;
};

// 只注册 C++ → TJS2 方向（只能作为返回值，不能作为参数）
struct StatusToVariant {
    void operator()(tTJSVariant &dst, StatusInfo const &src) {
        // 将 StatusInfo 转为格式化字符串 "[code] message"
        char buf[256];
        std::snprintf(buf, sizeof(buf), "[%d] %s", src.code, src.message.c_str());
        dst = tTJSString(buf);
    }
};
NCB_SET_TOVARIANT_CONVERTOR(StatusInfo, StatusToVariant);
```

## 常见错误及解决方案

### 错误 1：编译报错 "NCB_COMPILE_ERROR_NoImplement"

```
error: invalid use of incomplete type 'struct ncbTypeConvertor::NCB_COMPILE_ERROR_NoImplement'
```

**原因**：你在方法签名中使用了一个 ncbind 不知道如何转换的类型。比如使用了 `std::vector<int>` 但没有注册转换器。

**解决方案**：
1. 如果是自定义类 → 使用 `NCB_TYPECONV_BOXING(MyClass)` 注册
2. 如果是 STL 字符串 → 使用 `NCB_TYPECONV_STL_STRING(std::string)`
3. 如果是其他复杂类型 → 编写自定义转换器并用 `NCB_SET_CONVERTOR` 注册

### 错误 2：运行时崩溃 "Invalid instance type"

**原因**：Unboxing 时，TJS2 对象内部存储的类型与你期望提取的类型不匹配。例如你传入了一个 `Vec2` 对象，但方法参数期望 `Rect`。

**解决方案**：检查 TJS2 脚本中传入的对象类型是否正确。

### 错误 3：运行时崩溃 "No class object"

**原因**：尝试 Boxing 一个尚未通过 `NCB_REGISTER_CLASS` 注册的类型。`CreateAdaptor` 需要类对象来创建 TJS2 实例。

**解决方案**：确保在使用 `NCB_TYPECONV_BOXING(T)` 之前，`T` 已经有对应的 `NCB_REGISTER_CLASS(T)` 注册。

---

## 本节小结

- **ncbTypedefs** 提供统一的类型别名，是整个 ncbind 框架的"类型字典"
- **ncbTypeConvertor** 通过模板元编程在编译期自动选择类型转换策略，支持 DirectCopy、CastCopy、特殊化三种策略
- 基础类型（整数、浮点、布尔、字符串）已预注册，可零配置使用
- **ncbNativeObjectBoxing** 处理 C++ 自定义类的 Boxing/Unboxing，通过 `ncbInstanceAdaptor` 桥接两个世界
- **Sticky 标志**决定 C++ 对象的生命周期归谁管理——值传递时 TJS2 负责 delete，引用传递时不 delete
- **ncbPropAccessor** 封装 `iTJSDispatch2` 接口，简化 Dictionary/Array 的读写操作
- 自定义转换器通过 `NCB_SET_CONVERTOR` 宏注册，需提供双向 `operator()` 重载

### 题目 1：类型转换策略判断

对于以下 C++ 类型，说出 ncbind 使用哪种转换策略，以及中间类型（如有）：

1. `unsigned short`
2. `const tjs_char*`
3. `const char*`
4. `自定义类 Sprite`（已注册 `NCB_TYPECONV_BOXING`）

<details>
<summary>查看答案</summary>

1. `unsigned short` → **CastCopy\<tTVInteger\>**，中间类型为 `tTVInteger`（64 位整数）。先 `static_cast<tTVInteger>(value)` 再赋值给 `tTJSVariant`。
2. `const tjs_char*` → **WideChar（ncbWideCharConvertor）**，无中间类型（TJS2 内部就是宽字符，近似零拷贝）。
3. `const char*` → **NarrowChar（ncbNarrowCharConvertor）**，中间类型为临时分配的宽字符缓冲区。会调用 `ToNarrowStr` 进行编码转换。
4. `Sprite` → **ncbNativeObjectBoxing**（Boxing/Unboxing）。Boxing 时通过 `ncbInstanceAdaptor<Sprite>::CreateAdaptor` 创建 TJS2 对象包装 C++ 指针；Unboxing 时通过 `GetNativeInstance` 提取。

</details>

### 题目 2：编写自定义转换器

为以下枚举类型编写一个自定义转换器，使其可以在 TJS2 脚本中用整数表示：

```cpp
enum class Direction { Up = 0, Down = 1, Left = 2, Right = 3 };
```

要求：
- TJS2 脚本传入整数 0-3，C++ 侧收到 `Direction` 枚举值
- C++ 返回 `Direction` 时，TJS2 侧收到对应整数
- 如果传入的整数超出范围，默认返回 `Direction::Up`

<details>
<summary>查看答案</summary>

```cpp
#include "ncbind.hpp"
#include <cstdio>

// 枚举定义
enum class Direction { Up = 0, Down = 1, Left = 2, Right = 3 };

// 自定义转换器
struct DirectionConvertor {
    // C++ Direction → TJS2 Variant
    void operator()(tTJSVariant &dst, Direction const &src) {
        dst = static_cast<tjs_int64>(src);  // 枚举值转为整数
    }

    // TJS2 Variant → C++ Direction
    void operator()(Direction &dst, tTJSVariant const &src) {
        tjs_int64 val = src;  // 从 Variant 取出整数
        if (val >= 0 && val <= 3) {
            dst = static_cast<Direction>(val);  // 范围内：正常转换
        } else {
            dst = Direction::Up;  // 超出范围：默认值
        }
    }
};

// 注册转换器
NCB_SET_CONVERTOR(Direction, DirectionConvertor);

// 使用示例
class Player {
public:
    Direction facing = Direction::Up;

    void move(Direction dir) {
        facing = dir;
        const char* names[] = {"Up", "Down", "Left", "Right"};
        std::printf("Player moved %s\n", names[static_cast<int>(dir)]);
        // 输出: Player moved Right（当 TJS2 传入 3 时）
    }

    Direction getFacing() const {
        return facing;  // 自动转为整数返回给 TJS2
    }
};

NCB_REGISTER_CLASS(Player) {
    NCB_CONSTRUCTOR(());
    NCB_METHOD(move);
    NCB_METHOD(getFacing);
}
// TJS2 脚本:
// var p = new Player();
// p.move(3);                    // 传入 3，C++ 收到 Direction::Right
// var dir = p.getFacing();      // 返回 3
```

</details>

### 题目 3：ncbPropAccessor 实战

编写一个 C++ 函数，接收一个 TJS2 Dictionary 参数，读取其中的 `"x"`、`"y"`、`"radius"` 字段（都是数值），计算圆的面积（π × r²），并将结果写入同一个 Dictionary 的 `"area"` 字段。要求使用 `ncbPropAccessor`。

<details>
<summary>查看答案</summary>

```cpp
#include "ncbind.hpp"
#include <cmath>
#include <cstdio>

// 函数：读取 Dictionary 中的 x, y, radius，计算面积写入 area
static void calcCircleArea(iTJSDispatch2* dict) {
    ncbPropAccessor acc(dict);

    // 读取字段（带默认值）
    double x = acc.getRealValue(TJS_W("x"), 0.0);
    double y = acc.getRealValue(TJS_W("y"), 0.0);
    double radius = acc.getRealValue(TJS_W("radius"), 1.0);

    // 计算面积
    double area = M_PI * radius * radius;

    // 写入结果
    acc.SetValue(TJS_W("area"), area);

    std::printf("Circle at (%.1f, %.1f), radius=%.1f, area=%.4f\n",
                x, y, radius, area);
    // 输入 {x:0, y:0, radius:5} 时输出:
    // Circle at (0.0, 0.0), radius=5.0, area=78.5398
}

// 将函数附加到 Scripts 全局对象
NCB_ATTACH_FUNCTION(calcCircleArea, Scripts, calcCircleArea);
// TJS2 脚本:
// var circle = %["x" => 0, "y" => 0, "radius" => 5];
// Scripts.calcCircleArea(circle);
// System.inform("area = " + circle.area);  // 输出: area = 78.5398...
```

</details>
