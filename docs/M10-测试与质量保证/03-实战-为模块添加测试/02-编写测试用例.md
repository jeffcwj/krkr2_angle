## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 测试层级 | Test Hierarchy | 用 TEST_CASE 和 SECTION 嵌套形成的树形测试结构，每条路径是一个独立测试 |
| 断言宏 | Assertion Macro | Catch2 提供的验证宏（如 REQUIRE、CHECK），测试代码的核心工具 |
| 测试隔离 | Test Isolation | 每个 SECTION 独立运行，前一个 SECTION 的修改不影响后一个 |
| 包围盒 | Bounding Box | 能包住一组几何图形的最小矩形，do_union 计算的就是包围盒 |
| 左闭右开 | Half-Open Interval | 包含左端点但不包含右端点的区间，如 [0, 10) 包含 0 但不包含 10 |
| 源文件注册 | Source Registration | 在 CMakeLists.txt 的 SOURCES 列表中添加 .cpp 文件名，让 CMake 知道要编译它 |

## 测试文件创建

### 文件结构规划

我们的测试文件将放在现有的测试目录中：

```
tests/unit-tests/core/visual/
├── CMakeLists.txt    ← 修改：在 SOURCES 中添加新文件
├── main.cpp          ← 不修改：已有 spdlog 初始化
└── rect_test.cpp     ← 新建：tTVPRect 的所有测试
```

为什么用 `rect_test.cpp` 而不是 `tTVPRect_test.cpp`？项目的测试命名惯例是用简短的描述性名称（如 `psbfile-dll.cpp`、`tjs.cpp`），不带类名前缀。`rect_test` 简洁且一目了然。

### CMake 配置修改

需要在 `tests/unit-tests/core/visual/CMakeLists.txt` 的 `SOURCES` 列表中添加文件名：

```cmake
# 来源：tests/unit-tests/core/visual/CMakeLists.txt
# 修改前：
set(SOURCES
)

# 修改后：
set(SOURCES
    rect_test.cpp
)
```

这一行修改触发了以下 CMake 自动化流程：

1. `string(REPLACE ".cpp" "" ...)` 提取出目标名 `rect_test`
2. `add_executable(rect_test rect_test.cpp main.cpp)` 创建可执行文件，链接自定义入口
3. `target_link_libraries` 链接 `krkr2core`（提供 `ComplexRect.h`）和 `Catch2::Catch2`
4. `catch_discover_tests` 自动发现所有 `TEST_CASE` 注册到 CTest

> **不需要手动链接 ComplexRect.h**：因为 `krkr2core` 是 INTERFACE 库，它的 `target_include_directories` 会传播给所有链接它的目标。`rect_test` 链接 `krkr2core` 后，就能直接 `#include "ComplexRect.h"`。

### 测试文件头部

每个测试文件都需要包含必要的头文件。以下是 `rect_test.cpp` 的开头：

```cpp
// rect_test.cpp — tTVPRect 结构体的单元测试
// 来源：tests/unit-tests/core/visual/rect_test.cpp

#include <catch2/catch_test_macros.hpp>  // TEST_CASE, REQUIRE, CHECK 等宏
#include "ComplexRect.h"                 // tTVPRect, tTVPPoint 定义

// 注意：不需要 #include "test_config.h"
// 因为 tTVPRect 是纯内存结构，不需要读取测试夹具文件
```

> **为什么不需要 `test_config.h`？** `test_config.h` 提供 `TEST_FILES_PATH` 宏，用于定位磁盘上的测试数据文件。`tTVPRect` 的所有测试都是纯内存操作——构造对象、调用方法、检查返回值——不需要读写文件，因此不需要这个头文件。

## 测试代码组织策略

### TEST_CASE 与 SECTION 的分层

回顾上一节的测试清单，32 个用例分为 7 个方法组。我们用以下层级组织代码：

```
TEST_CASE("tTVPRect 构造函数")
  ├── SECTION("四参数构造")
  └── SECTION("默认构造")

TEST_CASE("tTVPRect 尺寸方法")
  ├── SECTION("get_width 正常值")
  ├── SECTION("get_width 零宽度")
  ├── SECTION("get_width 负宽度")
  ├── SECTION("set_width 正常值")
  └── SECTION("get_height/set_height")

TEST_CASE("tTVPRect 位置方法")
  ├── SECTION("add_offsets 正值")
  ├── SECTION("add_offsets 负值")
  └── SECTION("set_offsets")

TEST_CASE("tTVPRect 重置与状态")
  ├── SECTION("clear")
  ├── SECTION("is_empty 正常矩形")
  ├── SECTION("is_empty 零宽/零高/反转")
  └── SECTION("set_size")

TEST_CASE("tTVPRect 集合运算")
  ├── SECTION("do_union 重叠")
  ├── SECTION("do_union 不相邻")
  ├── SECTION("do_union 空参数")
  ├── SECTION("intersects_with 重叠/不重叠/边缘")
  └── SECTION("included_in 内部/外部/相同/空")

TEST_CASE("tTVPRect 比较运算符")
  ├── SECTION("operator< / operator>")
  └── SECTION("operator== / operator!=")

TEST_CASE("tTVPRect union 内存布局")
  └── SECTION("多种访问方式一致性")
```

**为什么用多个 TEST_CASE 而不是一个大 TEST_CASE？** 每个 `TEST_CASE` 是一个独立的测试入口。如果一个方法组的测试失败了，Catch2 的输出会明确指出是哪个 `TEST_CASE` 出了问题。把所有测试放在一个 `TEST_CASE` 中，失败信息会被淹没在大量输出中。

## 完整测试代码

下面按方法组逐一编写测试代码。每个代码块都是完整的、可直接放入 `rect_test.cpp` 的代码。

### 测试 1：构造函数

```cpp
// ===== 构造函数测试 =====
TEST_CASE("tTVPRect 构造函数", "[rect][constructor]") {
    SECTION("四参数构造函数正确赋值") {
        tTVPRect rect(10, 20, 110, 220);

        // 验证四个字段的值
        REQUIRE(rect.left == 10);
        REQUIRE(rect.top == 20);
        REQUIRE(rect.right == 110);
        REQUIRE(rect.bottom == 220);
    }

    SECTION("四参数构造函数支持负坐标") {
        // 负坐标在屏幕外的图层中很常见
        tTVPRect rect(-50, -30, 100, 200);

        REQUIRE(rect.left == -50);
        REQUIRE(rect.top == -30);
        REQUIRE(rect.right == 100);
        REQUIRE(rect.bottom == 200);
    }

    SECTION("四参数构造函数支持零值") {
        tTVPRect rect(0, 0, 0, 0);

        REQUIRE(rect.left == 0);
        REQUIRE(rect.top == 0);
        REQUIRE(rect.right == 0);
        REQUIRE(rect.bottom == 0);
    }
}
```

> **标签（Tag）的用法**：`"[rect][constructor]"` 是 Catch2 的标签语法。运行测试时可以用 `--tag "[rect]"` 只运行矩形相关测试，或 `--tag "[constructor]"` 只运行构造函数测试。标签是可选的，但在测试数量多时非常有用。

### 测试 2：尺寸方法

```cpp
// ===== 尺寸方法测试 =====
TEST_CASE("tTVPRect 尺寸方法", "[rect][size]") {
    // 创建一个标准矩形，供所有 SECTION 共用
    // 注意：每个 SECTION 都会从这里重新开始执行（测试隔离）
    tTVPRect rect(10, 20, 110, 220);

    SECTION("get_width 返回正确宽度") {
        // width = right - left = 110 - 10 = 100
        REQUIRE(rect.get_width() == 100);
    }

    SECTION("get_height 返回正确高度") {
        // height = bottom - top = 220 - 20 = 200
        REQUIRE(rect.get_height() == 200);
    }

    SECTION("get_width 零宽度矩形") {
        tTVPRect zero_w(10, 20, 10, 220);  // left == right
        REQUIRE(zero_w.get_width() == 0);
    }

    SECTION("get_width 负宽度（反转矩形）") {
        tTVPRect inverted(110, 20, 10, 220);  // left > right
        // 负宽度不是错误——tTVPRect 允许反转矩形存在
        REQUIRE(inverted.get_width() == -100);
    }

    SECTION("set_width 修改宽度") {
        rect.set_width(50);

        // 验证：宽度变为 50
        REQUIRE(rect.get_width() == 50);
        // 验证：left 没有变化（set_width 只修改 right）
        REQUIRE(rect.left == 10);
        // 验证：right = left + 新宽度 = 10 + 50 = 60
        REQUIRE(rect.right == 60);
    }

    SECTION("set_height 修改高度") {
        rect.set_height(300);

        REQUIRE(rect.get_height() == 300);
        // top 不变
        REQUIRE(rect.top == 20);
        // bottom = top + 新高度 = 20 + 300 = 320
        REQUIRE(rect.bottom == 320);
    }

    SECTION("set_width 零值") {
        rect.set_width(0);
        REQUIRE(rect.get_width() == 0);
        REQUIRE(rect.right == rect.left);  // right 等于 left
    }

    SECTION("set_width 负值创建反转矩形") {
        rect.set_width(-50);
        // right = left + (-50) = 10 + (-50) = -40
        REQUIRE(rect.right == -40);
        REQUIRE(rect.get_width() == -50);
    }
}
```

**注意 `SECTION` 的隔离性**：虽然所有 `SECTION` 共用开头的 `tTVPRect rect(10, 20, 110, 220)` 声明，但每个 `SECTION` 都是独立执行的。`set_width(50)` 在一个 `SECTION` 中修改了 `rect`，但下一个 `SECTION` 的 `rect` 仍然是原始的 `(10, 20, 110, 220)`。这是 Catch2 的核心特性——每个 `SECTION` 就像一个独立的测试，共享初始化代码但不共享状态修改。

### 测试 3：位置方法

```cpp
// ===== 位置方法测试 =====
TEST_CASE("tTVPRect 位置方法", "[rect][position]") {
    tTVPRect rect(10, 20, 110, 220);
    // 初始状态：width=100, height=200

    SECTION("add_offsets 正值偏移") {
        rect.add_offsets(5, 3);

        // 四个坐标各自偏移
        REQUIRE(rect.left == 15);     // 10 + 5
        REQUIRE(rect.top == 23);      // 20 + 3
        REQUIRE(rect.right == 115);   // 110 + 5
        REQUIRE(rect.bottom == 223);  // 220 + 3
        // 宽高不变
        REQUIRE(rect.get_width() == 100);
        REQUIRE(rect.get_height() == 200);
    }

    SECTION("add_offsets 负值偏移") {
        rect.add_offsets(-20, -30);

        REQUIRE(rect.left == -10);    // 10 + (-20)
        REQUIRE(rect.top == -10);     // 20 + (-30)
        REQUIRE(rect.right == 90);    // 110 + (-20)
        REQUIRE(rect.bottom == 190);  // 220 + (-30)
        // 宽高不变
        REQUIRE(rect.get_width() == 100);
        REQUIRE(rect.get_height() == 200);
    }

    SECTION("add_offsets 零值（无操作）") {
        tTVPRect original = rect;  // 保存原始状态
        rect.add_offsets(0, 0);

        // 矩形应该完全不变
        REQUIRE(rect.left == original.left);
        REQUIRE(rect.top == original.top);
        REQUIRE(rect.right == original.right);
        REQUIRE(rect.bottom == original.bottom);
    }

    SECTION("set_offsets 移动到新位置") {
        rect.set_offsets(50, 30);

        // 左上角移到 (50, 30)
        REQUIRE(rect.left == 50);
        REQUIRE(rect.top == 30);
        // 宽高保持不变：width=100, height=200
        REQUIRE(rect.right == 150);    // 50 + 100
        REQUIRE(rect.bottom == 230);   // 30 + 200
        REQUIRE(rect.get_width() == 100);
        REQUIRE(rect.get_height() == 200);
    }

    SECTION("set_offsets 到原点") {
        rect.set_offsets(0, 0);

        REQUIRE(rect.left == 0);
        REQUIRE(rect.top == 0);
        REQUIRE(rect.right == 100);   // 0 + width(100)
        REQUIRE(rect.bottom == 200);  // 0 + height(200)
    }

    SECTION("set_offsets 到负坐标") {
        rect.set_offsets(-100, -50);

        REQUIRE(rect.left == -100);
        REQUIRE(rect.top == -50);
        REQUIRE(rect.get_width() == 100);   // 宽度不变
        REQUIRE(rect.get_height() == 200);  // 高度不变
    }
}
```

### 测试 4：重置与状态查询

```cpp
// ===== 重置与状态查询测试 =====
TEST_CASE("tTVPRect 重置与状态", "[rect][state]") {

    SECTION("clear 将所有字段归零") {
        tTVPRect rect(10, 20, 110, 220);
        rect.clear();

        REQUIRE(rect.left == 0);
        REQUIRE(rect.top == 0);
        REQUIRE(rect.right == 0);
        REQUIRE(rect.bottom == 0);
    }

    SECTION("is_empty 正常矩形返回 false") {
        tTVPRect rect(0, 0, 100, 100);
        REQUIRE_FALSE(rect.is_empty());
    }

    SECTION("is_empty 零宽度返回 true") {
        // left == right → 宽度为 0 → 空矩形
        tTVPRect rect(10, 0, 10, 100);
        REQUIRE(rect.is_empty());
    }

    SECTION("is_empty 零高度返回 true") {
        // top == bottom → 高度为 0 → 空矩形
        tTVPRect rect(0, 10, 100, 10);
        REQUIRE(rect.is_empty());
    }

    SECTION("is_empty 反转矩形返回 true") {
        // left > right 且 top > bottom → 空矩形
        tTVPRect rect(100, 100, 0, 0);
        REQUIRE(rect.is_empty());
    }

    SECTION("is_empty 仅宽度反转返回 true") {
        // left > right 但 top < bottom
        tTVPRect rect(100, 0, 0, 100);
        REQUIRE(rect.is_empty());
    }

    SECTION("clear 后 is_empty 返回 true") {
        tTVPRect rect(10, 20, 110, 220);
        rect.clear();
        // clear 后 left == right == 0，所以 is_empty() 为 true
        REQUIRE(rect.is_empty());
    }

    SECTION("set_size 修改尺寸") {
        tTVPRect rect(10, 20, 0, 0);  // 初始尺寸无关紧要
        rect.set_size(200, 300);

        // right = left + w = 10 + 200 = 210
        REQUIRE(rect.right == 210);
        // bottom = top + h = 20 + 300 = 320
        REQUIRE(rect.bottom == 320);
        // left 和 top 不变
        REQUIRE(rect.left == 10);
        REQUIRE(rect.top == 20);
    }

    SECTION("set_size 零值创建空矩形") {
        tTVPRect rect(10, 20, 110, 220);
        rect.set_size(0, 0);

        // right == left, bottom == top → 空矩形
        REQUIRE(rect.right == rect.left);
        REQUIRE(rect.bottom == rect.top);
        REQUIRE(rect.is_empty());
    }
}
```

> **`REQUIRE_FALSE` 的用法**：`REQUIRE_FALSE(expr)` 等同于 `REQUIRE(!(expr))`，但在测试失败时输出更清晰。当你期望一个布尔表达式返回 `false` 时，优先使用 `REQUIRE_FALSE` 而不是 `REQUIRE(!expr)`。

### 测试 5：集合运算

集合运算是 `tTVPRect` 最复杂也最重要的功能，测试代码相应更长。

```cpp
// ===== 集合运算测试 =====
TEST_CASE("tTVPRect 集合运算 - do_union", "[rect][union]") {

    SECTION("两个重叠矩形的并集") {
        tTVPRect a(0, 0, 100, 100);
        tTVPRect b(50, 50, 150, 150);

        bool result = a.do_union(b);

        REQUIRE(result == true);
        // 并集 = 包围两个矩形的最小矩形
        REQUIRE(a.left == 0);      // min(0, 50)
        REQUIRE(a.top == 0);       // min(0, 50)
        REQUIRE(a.right == 150);   // max(100, 150)
        REQUIRE(a.bottom == 150);  // max(100, 150)
    }

    SECTION("两个不相邻矩形的并集") {
        tTVPRect a(0, 0, 10, 10);
        tTVPRect b(100, 100, 200, 200);

        bool result = a.do_union(b);

        REQUIRE(result == true);
        // 包围盒包含中间的空白区域
        REQUIRE(a.left == 0);
        REQUIRE(a.top == 0);
        REQUIRE(a.right == 200);
        REQUIRE(a.bottom == 200);
    }

    SECTION("与空矩形求并集返回 false") {
        tTVPRect a(10, 20, 110, 220);
        tTVPRect empty_rect(0, 0, 0, 0);  // is_empty() == true

        tTVPRect original = a;  // 保存原始值
        bool result = a.do_union(empty_rect);

        REQUIRE(result == false);
        // 原矩形应该不变
        REQUIRE(a.left == original.left);
        REQUIRE(a.top == original.top);
        REQUIRE(a.right == original.right);
        REQUIRE(a.bottom == original.bottom);
    }

    SECTION("与自身求并集") {
        tTVPRect a(10, 20, 110, 220);
        tTVPRect copy = a;

        bool result = a.do_union(copy);

        REQUIRE(result == true);
        // 与自身合并后应该不变
        REQUIRE(a.left == 10);
        REQUIRE(a.top == 20);
        REQUIRE(a.right == 110);
        REQUIRE(a.bottom == 220);
    }

    SECTION("B 完全包含 A 时的并集") {
        tTVPRect a(20, 20, 80, 80);
        tTVPRect b(0, 0, 100, 100);

        a.do_union(b);

        // 结果应该等于 B（更大的矩形）
        REQUIRE(a.left == 0);
        REQUIRE(a.top == 0);
        REQUIRE(a.right == 100);
        REQUIRE(a.bottom == 100);
    }
}
```

接下来是相交判定和包含判定——这两组测试的边界条件尤其容易写错。

```cpp
// ===== 相交判定测试 =====
TEST_CASE("tTVPRect 集合运算 - intersects_with", "[rect][intersect]") {

    SECTION("两个重叠矩形相交") {
        tTVPRect a(0, 0, 100, 100);
        tTVPRect b(50, 50, 150, 150);

        REQUIRE(a.intersects_with(b));
        REQUIRE(b.intersects_with(a));  // 对称性
    }

    SECTION("两个不重叠矩形不相交") {
        tTVPRect a(0, 0, 10, 10);
        tTVPRect b(20, 20, 30, 30);

        REQUIRE_FALSE(a.intersects_with(b));
        REQUIRE_FALSE(b.intersects_with(a));
    }

    SECTION("仅边缘相邻不算相交（左闭右开）") {
        // A 的右边 == B 的左边
        tTVPRect a(0, 0, 10, 10);
        tTVPRect b(10, 0, 20, 10);

        // right <= ref.left → 10 <= 10 → true → 不相交
        REQUIRE_FALSE(a.intersects_with(b));

        // 上下边缘相邻也不算
        tTVPRect c(0, 0, 10, 10);
        tTVPRect d(0, 10, 10, 20);
        REQUIRE_FALSE(c.intersects_with(d));
    }

    SECTION("空矩形与任何矩形不相交") {
        tTVPRect empty_rect(0, 0, 0, 0);
        tTVPRect normal(0, 0, 100, 100);

        REQUIRE_FALSE(empty_rect.intersects_with(normal));
        REQUIRE_FALSE(normal.intersects_with(empty_rect));
    }

    SECTION("完全相同的矩形相交") {
        tTVPRect a(10, 10, 50, 50);
        tTVPRect b(10, 10, 50, 50);

        REQUIRE(a.intersects_with(b));
    }

    SECTION("一个矩形完全在另一个内部也算相交") {
        tTVPRect outer(0, 0, 100, 100);
        tTVPRect inner(20, 20, 80, 80);

        REQUIRE(outer.intersects_with(inner));
        REQUIRE(inner.intersects_with(outer));
    }

    SECTION("no_empty_check 版本跳过空检查") {
        // 正常情况下行为相同
        tTVPRect a(0, 0, 100, 100);
        tTVPRect b(50, 50, 150, 150);

        REQUIRE(a.intersects_with_no_empty_check(b));
    }
}

// ===== 包含判定测试 =====
TEST_CASE("tTVPRect 集合运算 - included_in", "[rect][include]") {

    SECTION("小矩形在大矩形内部") {
        tTVPRect inner(20, 20, 80, 80);
        tTVPRect outer(0, 0, 100, 100);

        REQUIRE(inner.included_in(outer));
    }

    SECTION("大矩形不在小矩形内部") {
        tTVPRect outer(0, 0, 100, 100);
        tTVPRect inner(20, 20, 80, 80);

        REQUIRE_FALSE(outer.included_in(inner));
    }

    SECTION("完全相同的矩形互相包含") {
        tTVPRect a(10, 10, 50, 50);
        tTVPRect b(10, 10, 50, 50);

        // included_in 使用 <= 和 >=，所以相同矩形返回 true
```

### 测试 7：union 内存布局

这是一个特殊的测试——验证 `tTVPRect` 的 `union` 内存布局，确保通过不同成员名访问到的是同一块内存。

```cpp
// ===== union 内存布局测试 =====
TEST_CASE("tTVPRect union 内存布局", "[rect][layout]") {
    tTVPRect rect(10, 20, 30, 40);

    SECTION("小写和大写字段名是别名") {
        // 小写和大写字段访问同一内存
        REQUIRE(rect.left == rect.Left);
        REQUIRE(rect.top == rect.Top);
        REQUIRE(rect.right == rect.Right);
        REQUIRE(rect.bottom == rect.Bottom);
    }

    SECTION("tTVPPoint 视图与坐标字段对齐") {
        // upper_left 对应 left 和 top
        REQUIRE(rect.upper_left.x == rect.left);
        REQUIRE(rect.upper_left.y == rect.top);
        // bottom_right 对应 right 和 bottom
        REQUIRE(rect.bottom_right.x == rect.right);
        REQUIRE(rect.bottom_right.y == rect.bottom);
    }

    SECTION("数组视图与坐标字段对齐") {
        // array[0]=left, array[1]=top, array[2]=right, array[3]=bottom
        REQUIRE(rect.array[0] == rect.left);
        REQUIRE(rect.array[1] == rect.top);
        REQUIRE(rect.array[2] == rect.right);
        REQUIRE(rect.array[3] == rect.bottom);
    }

    SECTION("通过一种视图修改，其他视图立即可见") {
        rect.array[0] = 999;  // 通过数组修改

        // 所有视图都能看到新值
        REQUIRE(rect.left == 999);
        REQUIRE(rect.Left == 999);
        REQUIRE(rect.upper_left.x == 999);
        REQUIRE(rect.array[0] == 999);
    }

    SECTION("结构体大小验证") {
        // tTVPRect 应该恰好是 4 个 int 的大小
        REQUIRE(sizeof(tTVPRect) >= 4 * sizeof(tjs_int));
        // tTVPPoint 应该是 2 个 int 的大小
        REQUIRE(sizeof(tTVPPoint) == 2 * sizeof(tjs_int));
    }
}
```

> **为什么测试内存布局？** `tTVPRect` 的 `union` 设计意味着它依赖特定的内存布局。如果未来有人重构（Refactor，在不改变外部行为的前提下改善代码内部结构）这个结构体——比如把 `union` 改成普通成员——那些通过 `array[]` 或 `upper_left` 访问的代码会静默失败（Silent Failure，程序不报错但结果错误）。布局测试就是防止这种情况的守卫。

## 常见错误与解决方案

### 错误 1：编译时找不到 ComplexRect.h

```
fatal error: 'ComplexRect.h' file not found
#include "ComplexRect.h"
         ^~~~~~~~~~~~~~~~
```

**原因**：CMakeLists.txt 中没有链接 `krkr2core`，或链接顺序不对。

**解决**：确认 `target_link_libraries` 中有 `krkr2core`：

```cmake
target_link_libraries(${name}
    PRIVATE
        Catch2::Catch2
    PUBLIC
        krkr2core     # ← 必须有这一行
)
```

### 错误 2：链接时找不到 spdlog 符号

```
undefined reference to `spdlog::stdout_color_mt[...]`
```

**原因**：`main.cpp` 中使用了 spdlog，但测试目标没有链接 spdlog 库。

**解决**：`krkr2core` 已经通过传递依赖（Transitive Dependency）链接了 spdlog。如果仍然报错，检查 `krkr2core` 的 CMakeLists.txt 中是否有 `find_package(spdlog REQUIRED)` 和对应的 `target_link_libraries`。

### 错误 3：SOURCES 列表添加了文件名但编译时没有新目标

```
No tests found in project
```

**原因**：CMake 缓存没有更新。修改 CMakeLists.txt 后需要重新配置。

**解决**：

```bash
# 方式 1：删除缓存重新配置
rm -rf out/<平台>/debug
cmake --preset="<平台> Debug Config"

# 方式 2：触发增量重新配置
cmake --preset="<平台> Debug Config"  # 再跑一次就行
cmake --build --preset="<平台> Debug Build"
```

## 本节小结

1. **一个 .cpp 文件对应一个测试目标**：项目的 CMake 模式是为每个 `.cpp` 文件创建独立的可执行文件，通过 `main.cpp` 共享初始化代码
2. **TEST_CASE 按方法组划分**：7 个方法组对应 10 个 `TEST_CASE`（集合运算和比较运算各拆分为多个），保持输出的可读性
3. **SECTION 实现测试隔离**：共享初始化代码但不共享状态修改，每个 SECTION 是一个独立的测试路径
4. **标签用于选择性运行**：`[rect]`、`[constructor]`、`[union]` 等标签让你可以只运行感兴趣的测试子集
5. **REQUIRE vs CHECK**：本节全部使用 `REQUIRE`（失败立即停止），因为每个断言都是后续断言的前提。如果你想看到所有失败而不是只看到第一个，可以用 `CHECK`
6. **58 个 SECTION 覆盖了 7 个方法组**：从构造函数到内存布局，完整覆盖 `tTVPRect` 的所有公开接口
7. **测试名可以用中文**：Catch2 支持 UTF-8 测试名，中文名在输出中更易读

### 题目 2：分析 do_union 对 this 为空矩形的行为

`do_union` 只检查参数 `ref` 是否为空，不检查 `this`。编写一个测试，验证当 `this` 是空矩形但 `ref` 不是空矩形时，`do_union` 的行为。

提示：先在纸上推演代码执行过程，再写测试。

<details>
<summary>查看答案</summary>

```cpp
SECTION("this 为空矩形时 do_union 的行为") {
    // this 是空矩形（left == right == 0）
    tTVPRect a(0, 0, 0, 0);
    REQUIRE(a.is_empty());  // 确认 a 确实是空的

    tTVPRect b(10, 20, 100, 200);

    bool result = a.do_union(b);

    // do_union 不检查 this 是否为空，所以 result 取决于 ref 是否为空
    // b 不是空的，所以 result = true
    REQUIRE(result == true);

    // 逐步推演 do_union 对 a 的修改：
    // a 初始: left=0, top=0, right=0, bottom=0
    // if(0 > 10) left = 10;   → 否，left 保持 0
    // if(0 > 20) top = 20;    → 否，top 保持 0
    // if(0 < 100) right = 100; → 是，right 变为 100
    // if(0 < 200) bottom = 200; → 是，bottom 变为 200
    REQUIRE(a.left == 0);
    REQUIRE(a.top == 0);
    REQUIRE(a.right == 100);
    REQUIRE(a.bottom == 200);

    // 注意：结果的左上角是 (0,0) 而不是 (10,20)！
    // 这是因为 a 的 left(0) 不大于 b.left(10)，所以 left 没有被更新
    // 这意味着"空矩形与非空矩形求并"的结果不等于非空矩形本身
    REQUIRE_FALSE(a == b);
}
```

**关键认识**：`do_union` 的设计假设 `this` 是有效矩形。当 `this` 是空矩形时，结果是一个"包围了原点 (0,0) 和参数矩形"的包围盒，而不是参数矩形本身。这是预期行为——引擎代码中通常会在第一次合并前先用赋值（`a = first_rect`）而不是 `do_union`。

</details>
