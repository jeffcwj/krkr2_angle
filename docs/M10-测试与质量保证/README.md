# M10 — 测试与质量保证

## 模块概述

在大型跨平台 C++ 项目中，**单元测试（Unit Testing）** 是保证代码质量的基石。KrKr2 模拟器使用 **Catch2 v3** 作为测试框架，为核心模块（TJS2 脚本引擎、视频播放器、渲染子系统）和插件系统编写了结构化的单元测试。

本模块将带你从零学习 Catch2 v3 测试框架的核心用法，深入理解 KrKr2 项目的测试架构设计（自定义 `main()` 入口、spdlog 日志集成、`TEST_FILES_PATH` 夹具机制），最终通过实战为项目模块编写完整的单元测试。

无论你是想为已有代码补充测试、为新开发的插件添加回归测试，还是想理解 CI/CD 流水线中的测试环节，本模块都是必经之路。

## 学习目标

完成本模块后，你将能够：

1. 使用 Catch2 v3 编写 `TEST_CASE`、`SECTION`、参数化测试（Generator）和自定义 Matcher（匹配器）
2. 理解 KrKr2 项目的测试目录结构、CMake 测试配置、以及自定义 `main()` 入口的设计原因
3. 使用 `TEST_FILES_PATH` 夹具（Fixture）机制加载测试所需的二进制文件和脚本文件
4. 为 KrKr2 的任意模块（core/plugins）独立编写、编译和运行单元测试
5. 将测试集成到 CTest 和 GitHub Actions CI 流水线中

## 术语预览

本模块将涉及以下术语，先有个印象，后续章节会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 单元测试 | Unit Test | 对程序中最小可测试单元（函数、类方法）进行隔离验证的自动化测试 |
| 测试用例 | Test Case | 一个独立的测试场景，包含输入数据、执行步骤和预期结果 |
| 断言 | Assertion | 测试中用于验证某个条件是否成立的语句，失败则标记测试不通过 |
| 测试节 | Section | Catch2 中用于在同一个 TEST_CASE 内组织多个子场景的机制，每个 SECTION 独立运行 |
| 夹具 | Fixture | 测试运行前准备好的外部资源（文件、数据库、配置等），测试结束后可以清理 |
| 参数化测试 | Parameterized Test | 用不同参数自动重复运行同一段测试逻辑的技术 |
| 匹配器 | Matcher | Catch2 中用于灵活匹配预期值的工具，支持字符串包含、范围比较等复杂断言 |
| CTest | CTest | CMake 自带的测试运行器，可以发现并执行项目中所有注册的测试 |
| BDD | Behavior-Driven Development | 行为驱动开发，用 GIVEN/WHEN/THEN 描述测试场景的风格 |
| spdlog | spdlog | KrKr2 使用的高性能 C++ 日志库，测试中需要初始化日志器才能正确运行引擎代码 |
