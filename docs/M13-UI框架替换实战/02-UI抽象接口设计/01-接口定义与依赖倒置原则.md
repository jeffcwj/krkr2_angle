# 01 — 接口定义与依赖倒置原则

> **所属模块：** M13-UI框架替换实战
> **前置知识：** [01-当前Cocos2d-UI分析](../01-当前Cocos2d-UI分析/)
> **预计阅读时间：** 65 分钟

## 本节目标

读完本节后，你将能够：

1. 解释依赖倒置原则（DIP）在 UI 框架替换场景下的价值
2. 设计覆盖 KrKr2 所有 UI 需求的抽象接口集合（`IDialog`、`IMenu`、`IUIManager`）
3. 描述为什么接口设计要"面向行为，而非面向实现"
4. 在不改变业务逻辑的前提下，编写可接入不同 UI 后端（Cocos2d-x / Flutter / Compose）的代码

## 术语预览

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 依赖倒置原则 | Dependency Inversion Principle (DIP) | SOLID 原则之一：高层模块不依赖低层模块，两者都依赖抽象接口；接口属于高层，实现属于低层 |
| 开闭原则 | Open-Closed Principle (OCP) | 对扩展开放（可以新增 UI 后端），对修改关闭（不需要改动业务逻辑代码） |
| 抽象工厂 | Abstract Factory | 创建一组相关对象（如一套 UI 控件）的工厂接口，不指定具体实现类 |
| 纯虚函数 | Pure Virtual Function | C++ 中以 `= 0` 声明的虚函数，要求所有子类必须实现，使基类成为"接口"（抽象类） |
| 回调接口 | Callback Interface | 通过纯虚函数定义的事件通知机制，业务层实现接口接收 UI 事件，而非直接传递 lambda |
| 指针事件 | Pointer Event | 统一触摸/鼠标/触控笔输入的抽象事件类型，不区分物理输入设备 |

## 一、为什么需要 UI 抽象接口

### 1.1 没有抽象接口时的依赖图

上一章分析了 KrKr2 现有 UI 的问题，其中最根本的是**依赖方向错误**：

```
现有（错误）依赖图：

业务逻辑层
  ↓ 直接依赖
Cocos2d-x 类 (cocos2d::Layer, ui::Button, ui::Slider...)
  ↓ 直接依赖
OpenGL / 平台 SDK
```

这意味着：要替换 UI 框架，必须修改业务逻辑层的所有代码——这正是"紧耦合"的代价。

### 1.2 引入抽象接口后的依赖图

依赖倒置原则（DIP）要求我们将依赖方向"倒置"：

```
正确的依赖图（引入抽象接口后）：

业务逻辑层
  ↓ 依赖（调用）
UI 抽象接口 (IDialog, IMenu, IUIManager)  ← 接口属于业务层
  ↑ 实现（被依赖）
UI 实现层 A: Cocos2d-x 实现
UI 实现层 B: Flutter 实现
UI 实现层 C: Compose 实现
```

现在，要替换 UI 框架只需：
1. 新写一个实现层（例如 `FlutterConfigDialog implements IDialog`）
2. 在工厂/注册点切换实现
3. **业务逻辑层零改动**

### 1.3 设计原则：面向行为，而非面向实现

抽象接口应该描述"做什么"，而不是"怎么做"：

```cpp
// ❌ 错误：接口暴露了实现细节（Cocos2d-x 相关类型）
class IDialog {
    virtual cocos2d::Layer* getLayer() = 0;  // 泄露了 Cocos2d-x 类型
    virtual void setFont(const std::string& ttfPath, int size) = 0;  // 实现细节
};

// ✅ 正确：接口只描述行为
class IDialog {
    virtual void show() = 0;
    virtual void hide() = 0;
    virtual void setTitle(const std::string& title) = 0;
    virtual bool isVisible() const = 0;
    virtual void setCloseCallback(std::function<void()> cb) = 0;
};
```

## 二、KrKr2 UI 抽象接口设计

### 2.1 基础接口：`IUIElement`

所有 UI 元素的公共基础接口：

```cpp
// cpp/core/environ/ui/abstract/IUIElement.h
#pragma once
#include <functional>
#include <string>

namespace krkr2 {
namespace ui {

// 指针事件（统一触摸和鼠标输入）
struct PointerEvent {
    enum class Type { Down, Move, Up, Cancel };
    Type    type;
    float   x;        // 归一化坐标 [0.0, 1.0]，0=左/上
    float   y;
    int     pointerId; // 多点触摸 ID（鼠标为 0）
};

// 所有 UI 元素的基础接口
class IUIElement {
public:
    virtual ~IUIElement() = default;

    // 显示/隐藏
    virtual void show() = 0;
    virtual void hide() = 0;
    virtual bool isVisible() const = 0;

    // 销毁（释放与底层 UI 框架相关的资源）
    virtual void destroy() = 0;
};

} // namespace ui
} // namespace krkr2
```

### 2.2 对话框接口：`IDialog`

KrKr2 的对话框分为两种：**确认型**（OK/Cancel）和**展示型**（纯信息展示）。

```cpp
// cpp/core/environ/ui/abstract/IDialog.h
#pragma once
#include "IUIElement.h"
#include <string>
#include <vector>
#include <functional>

namespace krkr2 {
namespace ui {

// 通用对话框接口
class IDialog : public IUIElement {
public:
    virtual ~IDialog() = default;

    // 设置对话框标题
    virtual void setTitle(const std::string& title) = 0;

    // 设置消息内容
    virtual void setMessage(const std::string& message) = 0;

    // 注册确认/取消回调
    virtual void setOnConfirm(std::function<void()> callback) = 0;
    virtual void setOnCancel(std::function<void()> callback) = 0;
};

// 配置对话框接口（带设置项）
class IConfigDialog : public IDialog {
public:
    // 音量设置
    virtual void setMasterVolume(float value) = 0;  // [0.0, 1.0]
    virtual void setBGMVolume(float value) = 0;
    virtual void setSEVolume(float value) = 0;
    virtual void setVoiceVolume(float value) = 0;

    // 音量变更回调（用户拖动滑块时实时触发）
    virtual void setOnVolumeChanged(
        std::function<void(float master, float bgm, float se, float voice)> cb) = 0;

    // 视频设置
    virtual void setFullscreen(bool fullscreen) = 0;
    virtual void setResolution(int width, int height) = 0;
    virtual void setOnVideoSettingsChanged(
        std::function<void(bool fullscreen, int w, int h)> cb) = 0;
};

// 游戏选择对话框接口
class IGameSelectDialog : public IDialog {
public:
    struct GameEntry {
        std::string name;    // 游戏名称
        std::string path;    // 游戏目录路径
        std::string iconPath; // 封面图路径（可选）
    };

    // 设置游戏列表
    virtual void setGameList(const std::vector<GameEntry>& games) = 0;

    // 用户选中游戏时的回调
    virtual void setOnGameSelected(std::function<void(const GameEntry&)> cb) = 0;

    // "浏览文件夹"按钮回调
    virtual void setOnBrowse(std::function<void()> cb) = 0;
};

// 文件选择对话框接口
class IFilePickerDialog : public IDialog {
public:
    // 设置允许的文件类型过滤器（如 { ".xp3", ".exe" }）
    virtual void setFileFilters(const std::vector<std::string>& extensions) = 0;

    // 设置初始目录
    virtual void setInitialDirectory(const std::string& dir) = 0;

    // 用户选择文件后的回调
    virtual void setOnFileSelected(std::function<void(const std::string& path)> cb) = 0;
};

} // namespace ui
} // namespace krkr2
```

### 2.3 菜单接口：`IMenu`

```cpp
// cpp/core/environ/ui/abstract/IMenu.h
#pragma once
#include "IUIElement.h"
#include <string>
#include <vector>
#include <functional>

namespace krkr2 {
namespace ui {

// 菜单项（支持普通项、分隔符、可勾选项）
struct MenuItem {
    enum class Type { Action, Separator, Toggle };
    Type        type    {Type::Action};
    std::string id;           // 唯一标识符
    std::string label;        // 显示文字
    bool        enabled {true};
    bool        checked {false};  // 仅 Type::Toggle 时有效
    std::string iconPath;     // 图标路径（可选）
};

// 菜单接口（右键菜单 / 顶部菜单栏 / 浮动菜单）
class IMenu : public IUIElement {
public:
    // 设置菜单项列表
    virtual void setItems(const std::vector<MenuItem>& items) = 0;

    // 更新单个菜单项的状态（不需要重建整个菜单）
    virtual void updateItem(const std::string& id, bool enabled, bool checked) = 0;

    // 菜单项被选中时的回调
    virtual void setOnItemSelected(
        std::function<void(const std::string& itemId)> callback) = 0;

    // 在指定位置显示菜单（适用于右键菜单）
    virtual void showAt(float x, float y) = 0;
};

} // namespace ui
} // namespace krkr2
```

### 2.4 UI 管理器接口：`IUIManager`

所有 UI 元素的生命周期都通过 `IUIManager` 管理：

```cpp
// cpp/core/environ/ui/abstract/IUIManager.h
#pragma once
#include "IDialog.h"
#include "IMenu.h"
#include <memory>
#include <functional>

namespace krkr2 {
namespace ui {

// 消息提示类型
enum class ToastType { Info, Warning, Error };

// UI 管理器接口（负责创建和管理所有 UI 元素）
class IUIManager {
public:
    virtual ~IUIManager() = default;

    // 工厂方法：创建各类 UI 元素
    virtual std::unique_ptr<IConfigDialog>     createConfigDialog()     = 0;
    virtual std::unique_ptr<IGameSelectDialog> createGameSelectDialog() = 0;
    virtual std::unique_ptr<IFilePickerDialog> createFilePicker()       = 0;
    virtual std::unique_ptr<IMenu>             createContextMenu()      = 0;

    // 对话框栈管理
    virtual void pushDialog(IDialog* dialog) = 0;
    virtual void popDialog() = 0;
    virtual int  getDialogStackDepth() const = 0;

    // 轻量提示（Toast/Snackbar）
    virtual void showToast(const std::string& message,
                           ToastType type = ToastType::Info,
                           int durationMs = 3000) = 0;

    // 加载进度覆盖层（全屏遮罩 + 进度条/旋转图标）
    virtual void showLoading(const std::string& message = "") = 0;
    virtual void hideLoading() = 0;
    virtual void updateLoadingProgress(float progress) = 0;  // [0.0, 1.0]

    // 处理返回/Escape 键（移动端/桌面通用）
    virtual bool onBackPressed() = 0;  // 返回 true 表示已处理

    // 全局指针事件注入（由渲染层调用，将触摸/鼠标事件传给当前激活 UI）
    virtual void injectPointerEvent(const PointerEvent& event) = 0;

    // 初始化/销毁（在应用启动/退出时调用）
    virtual void initialize() = 0;
    virtual void shutdown() = 0;
};

} // namespace ui
} // namespace krkr2
```

## 三、接口的工厂注册机制

### 3.1 UI 后端工厂

不同 UI 后端通过工厂模式注册：

```cpp
// cpp/core/environ/ui/abstract/UIBackendFactory.h
#pragma once
#include "IUIManager.h"
#include <memory>
#include <functional>
#include <string>
#include <unordered_map>

namespace krkr2 {
namespace ui {

// UI 后端类型
enum class UIBackend {
    Cocos2d,   // 当前实现：Cocos2d-x
    Flutter,   // 新实现：Flutter Engine
    Compose,   // 新实现：Compose Multiplatform
    Native,    // 平台原生 UI（Win32/Android View/UIKit）
};

// UI 后端工厂
class UIBackendFactory {
public:
    // 注册 UI 后端创建函数（在 main() 或平台初始化时调用）
    static void registerBackend(UIBackend type,
                                std::function<std::unique_ptr<IUIManager>()> creator) {
        s_creators[type] = std::move(creator);
    }

    // 创建指定后端的 IUIManager 实例
    static std::unique_ptr<IUIManager> create(UIBackend type) {
        auto it = s_creators.find(type);
        if (it == s_creators.end()) {
            throw std::runtime_error("UI backend not registered: " +
                                     std::to_string(static_cast<int>(type)));
        }
        return it->second();
    }

    // 创建当前平台的最佳后端（可根据编译宏自动选择）
    static std::unique_ptr<IUIManager> createDefault() {
#if defined(USE_FLUTTER_UI)
        return create(UIBackend::Flutter);
#elif defined(USE_COMPOSE_UI)
        return create(UIBackend::Compose);
#else
        return create(UIBackend::Cocos2d);  // 默认使用 Cocos2d-x
#endif
    }

private:
    static inline std::unordered_map<UIBackend,
                                     std::function<std::unique_ptr<IUIManager>()>>
        s_creators;
};

} // namespace ui
} // namespace krkr2
```

### 3.2 如何在 AppDelegate 中切换后端

```cpp
// cpp/core/environ/cocos2d/AppDelegate.cpp（修改后）
#include "ui/abstract/UIBackendFactory.h"
#include "ui/cocos2d/Cocos2dUIManager.h"  // 当前实现

bool AppDelegate::applicationDidFinishLaunching() {
    // 注册 Cocos2d-x UI 后端
    UIBackendFactory::registerBackend(
        UIBackend::Cocos2d,
        []() -> std::unique_ptr<IUIManager> {
            return std::make_unique<Cocos2dUIManager>();
        }
    );

    // 将来可以这样切换到 Flutter 后端：
    // UIBackendFactory::registerBackend(
    //     UIBackend::Flutter,
    //     []() -> std::unique_ptr<IUIManager> {
    //         return std::make_unique<FlutterUIManager>();
    //     }
    // );

    // 创建默认 UI 管理器（由编译宏决定使用哪个后端）
    m_uiManager = UIBackendFactory::createDefault();
    m_uiManager->initialize();

    // 后续代码完全通过 IUIManager 接口操作 UI，与具体实现无关
    return true;
}
```

    bindWidgets();

    auto scene = cocos2d::Director::getInstance()->getRunningScene();
    scene->addChild(m_layer, 1000);  // Z-order 1000

    m_layer->removeFromParent();
}

    return m_layer->getParent() != nullptr;

    hide();
    m_layer->release();
    m_layer = nullptr;
}

    void setOnCancel(std::function<void()> cb) override {}

    void setBGMVolume(float v) override {}
    void setSEVolume(float v) override {}
    void setVoiceVolume(float v) override {}
    void setOnVolumeChanged(std::function<void(float,float,float,float)>) override {}
    void setFullscreen(bool fs) override { m_fullscreen = fs; }
    void setResolution(int w, int h) override {}
    void setOnVideoSettingsChanged(std::function<void(bool,int,int)>) override {}

    void simulateConfirm() { if (m_onConfirm) m_onConfirm(); }

    bool        m_visible     {false};
    bool        m_fullscreen  {false};
    float       m_masterVolume {0.5f};
    std::string m_title;
    std::function<void()> m_onConfirm;
};

// 测试：ConfigManager 正确响应对话框回调
TEST_CASE("ConfigDialog 接口：回调驱动配置更新", "[ui][config]") {
    MockConfigDialog dialog;

    float savedVolume = -1.0f;
    dialog.setOnVolumeChanged([&](float master, float, float, float) {
        savedVolume = master;
    });

    SECTION("setMasterVolume 更新 mock 状态") {
        dialog.setMasterVolume(0.75f);
        REQUIRE(dialog.m_masterVolume == Approx(0.75f));
    }

    SECTION("show/hide 控制可见性") {
        REQUIRE_FALSE(dialog.isVisible());
        dialog.show();
        REQUIRE(dialog.isVisible());
        dialog.hide();
        REQUIRE_FALSE(dialog.isVisible());
    }

    SECTION("确认回调被正确触发") {
        bool confirmed = false;
        dialog.setOnConfirm([&]() { confirmed = true; });
        dialog.simulateConfirm();
        REQUIRE(confirmed);
    }
}

## 本节小结

- 依赖倒置原则要求业务逻辑层依赖抽象接口，而不是具体的 UI 框架类
- KrKr2 的 UI 抽象接口分四层：`IUIElement`（基础）→ `IDialog`/`IMenu`（具体类型）→ `IUIManager`（管理器）
- 接口应面向行为（show/hide/setCallback），不暴露框架细节（不能有 `cocos2d::Layer*` 类型）
- `UIBackendFactory` 通过编译宏或运行时注册，实现 UI 后端的无缝切换
- Mock 实现让 UI 逻辑可以在无图形上下文的环境中单元测试

## 练习题与答案

### 题目 1：接口设计错误识别

以下 `IDialog` 接口设计有问题，请找出所有违反"面向行为"原则的地方并说明修改方案：

```cpp
class IDialog {
    virtual cocos2d::Node* getRootNode() = 0;
    virtual void setTTFFont(const std::string& path, int size) = 0;
    virtual void setBackgroundColor(cocos2d::Color4B color) = 0;
    virtual bool onTouchBegan(cocos2d::Touch* touch, cocos2d::Event* event) = 0;
    virtual void setVisible(bool v) = 0;
};
```

<details>
<summary>查看答案</summary>

**问题 1：`getRootNode()` 返回 `cocos2d::Node*`**
- 错误原因：将 Cocos2d-x 内部节点暴露给接口使用方，导致接口使用方也依赖 Cocos2d-x
- 修改：删除此方法；如果使用方需要将对话框添加到场景，应该通过 `show()` 方法内部处理

**问题 2：`setTTFFont()` 是实现细节**
- 错误原因：字体设置属于 UI 框架的渲染细节，Flutter/Compose 有自己的字体系统
- 修改：删除；字体样式应在各后端的内部实现中处理，或通过主题（Theme）系统统一配置

**问题 3：`setBackgroundColor(cocos2d::Color4B)`**
- 错误原因：`cocos2d::Color4B` 是 Cocos2d-x 专属类型
- 修改：如确实需要设置颜色，改用平台无关的类型：`setBackgroundColor(uint8_t r, uint8_t g, uint8_t b, uint8_t a)`

**问题 4：`onTouchBegan(cocos2d::Touch*, cocos2d::Event*)`**
- 错误原因：Cocos2d-x 的事件类型直接出现在接口中
- 修改：使用统一的 `PointerEvent` 结构体，通过 `IUIManager::injectPointerEvent()` 统一注入

**修复后的接口：**
```cpp
class IDialog : public IUIElement {
    virtual void setTitle(const std::string& title) = 0;
    virtual void setMessage(const std::string& message) = 0;
    virtual void setOnConfirm(std::function<void()> cb) = 0;
    virtual void setOnCancel(std::function<void()> cb) = 0;
    // 不需要字体、颜色、节点等实现细节
};
```

</details>

### 题目 2：工厂注册

请实现一个基于 Cocos2d-x 的 `IGameSelectDialog` 适配类的框架代码，包括：类声明（头文件）、`show()` 方法实现、`setGameList()` 的基本逻辑。

<details>
<summary>查看答案</summary>

```cpp
// cpp/core/environ/ui/cocos2d/Cocos2dGameSelectDialog.h
#pragma once
#include "../abstract/IDialog.h"
#include "cocos2d.h"
#include "ui/CocosGUI.h"
#include "editor-support/cocostudio/CocoStudio.h"

namespace krkr2::ui {

class Cocos2dGameSelectDialog final : public IGameSelectDialog {
public:
    Cocos2dGameSelectDialog() {
        m_layer = cocos2d::Layer::create();
        m_layer->retain();
        auto root = cocostudio::CSLoader::createNode(
            "ui/cocos-studio/GameStartDialog.csb");
        m_layer->addChild(root);
        // 获取列表视图控件引用
        m_listView = root->findChild<cocos2d::ui::ListView*>("GameListView");
    }

    ~Cocos2dGameSelectDialog() override {
        if (m_layer) {
            m_layer->release();
        }
    }

    // IUIElement
    void show() override {
        auto scene = cocos2d::Director::getInstance()->getRunningScene();
        if (!m_layer->getParent()) {
            scene->addChild(m_layer, 1000);
        }
    }

    void hide() override { m_layer->removeFromParent(); }
    bool isVisible() const override { return m_layer->getParent() != nullptr; }
    void destroy() override { hide(); m_layer->release(); m_layer = nullptr; }

    // IDialog（简化实现）
    void setTitle(const std::string& title) override {
        auto label = m_layer->findChild<cocos2d::Label*>("TitleLabel");
        if (label) label->setString(title);
    }
    void setMessage(const std::string&) override {}
    void setOnConfirm(std::function<void()> cb) override { m_onConfirm = cb; }
    void setOnCancel(std::function<void()> cb) override { m_onCancel = cb; }

    // IGameSelectDialog
    void setGameList(const std::vector<GameEntry>& games) override {
        if (!m_listView) return;
        m_listView->removeAllItems();
        for (const auto& game : games) {
            // 创建列表项
            auto item = cocos2d::ui::Layout::create();
            item->setContentSize({400, 60});

            auto nameLabel = cocos2d::Label::createWithTTF(
                game.name, "fonts/arial.ttf", 20);
            nameLabel->setPosition({200, 30});
            item->addChild(nameLabel);

            // 注册点击事件
            auto touchListener = cocos2d::EventListenerTouchOneByOne::create();
            touchListener->setSwallowTouches(true);
            const GameEntry gameCopy = game;
            touchListener->onTouchBegan = [this, gameCopy](
                    cocos2d::Touch*, cocos2d::Event*) -> bool {
                if (m_onGameSelected) m_onGameSelected(gameCopy);
                return true;
            };
            cocos2d::Director::getInstance()->getEventDispatcher()
                ->addEventListenerWithSceneGraphPriority(touchListener, item);

            m_listView->addItem(item);
        }
    }

    void setOnGameSelected(std::function<void(const GameEntry&)> cb) override {
        m_onGameSelected = cb;
    }

    void setOnBrowse(std::function<void()> cb) override {
        m_onBrowse = cb;
    }

private:
    cocos2d::Layer*           m_layer     {nullptr};
    cocos2d::ui::ListView*    m_listView  {nullptr};
    std::function<void()>     m_onConfirm;
    std::function<void()>     m_onCancel;
    std::function<void()>     m_onBrowse;
    std::function<void(const GameEntry&)> m_onGameSelected;
};

} // namespace krkr2::ui
```

</details>

### 题目 3：依赖方向验证

请解释为什么 `UIBackendFactory::createDefault()` 中可以引用 `USE_FLUTTER_UI` 编译宏，而 `IUIManager` 接口本身不能引用任何特定 UI 框架的头文件。这个约束如何在代码层面强制执行？

<details>
<summary>查看答案</summary>

**为什么 `IUIManager` 不能引用特定 UI 框架头文件：**

`IUIManager` 是"高层接口"，属于业务域。如果它 `#include` 了 `flutter/flutter_engine.h` 或 `cocos2d.h`，那么所有包含 `IUIManager` 的文件都会被迫依赖这些框架的头文件，接口就失去了"平台无关"的意义。

**为什么 `UIBackendFactory::createDefault()` 可以有编译宏：**

工厂类是"连接层"，它的职责就是决定创建哪个具体实现。编译宏 `USE_FLUTTER_UI` 控制的是：在编译时选择哪条代码路径——这是工厂职责的一部分，而不是业务逻辑。

**如何在代码层面强制执行：**

通过 CMake 的 `target_include_directories` 和 `target_link_libraries` 进行隔离：

```cmake
# CMakeLists.txt

# 抽象接口库：只允许包含标准库头文件
add_library(ui_abstract INTERFACE)
target_include_directories(ui_abstract INTERFACE cpp/core/environ/ui/abstract)
# 不链接任何 UI 框架！

# Cocos2d-x 实现：链接 Cocos2d-x
add_library(ui_cocos2d STATIC
    cpp/core/environ/ui/cocos2d/Cocos2dUIManager.cpp
    cpp/core/environ/ui/cocos2d/Cocos2dConfigDialog.cpp)
target_link_libraries(ui_cocos2d PRIVATE ui_abstract cocos2d)

# Flutter 实现：链接 Flutter
add_library(ui_flutter STATIC
    cpp/core/environ/ui/flutter/FlutterUIManager.cpp)
target_link_libraries(ui_flutter PRIVATE ui_abstract flutter_engine)

# 主程序：只链接工厂和抽象接口
# 通过编译宏选择链接哪个具体实现
if(USE_FLUTTER_UI)
    target_link_libraries(krkr2 PRIVATE ui_flutter)
else()
    target_link_libraries(krkr2 PRIVATE ui_cocos2d)
endif()
```

这样，如果有人在 `IDialog.h` 中不小心 `#include "cocos2d.h"`，`ui_abstract` 目标会构建失败（因为它没有链接 Cocos2d-x），错误会在编译时被立即发现。

</details>
