## 术语预览

| 术语 | 解释 |
|---|---|
| **外部纹理（External Texture）** | 宿主应用提供的纹理 ID，Flutter/Compose 可以将 UI 内容渲染到其中，或从宿主读取纹理显示 |
| **FBO（Framebuffer Object）** | OpenGL 的离屏渲染目标，与屏幕帧缓冲不同，可以绑定纹理作为颜色附件 |
| **EGLImage** | EGL 扩展（`EGL_KHR_image_base`），允许在不同 EGL 上下文之间共享图像数据（零拷贝）|
| **AHardwareBuffer** | Android 8.0+ 的硬件缓冲区 API，可以在 CPU、GPU、相机之间高效共享内存 |
| **Texture Barrier** | 确保一个 OpenGL 上下文完成写入后，另一个上下文读取时数据已同步的屏障机制 |
| **EGL Fence Sync** | EGL 同步对象，用于跨上下文的操作顺序保证（`eglCreateSyncKHR`） |
| **零拷贝（Zero-Copy）** | 数据不经过 CPU 内存中转，直接在 GPU 内存中传递（仅修改 GPU 纹理句柄的引用关系）|
| **glBlitFramebuffer** | OpenGL 函数，将一个 FBO 的内容（包括纹理）复制到另一个 FBO（非零拷贝，但 GPU 侧完成）|

## 一、纹理共享的必要性

### 1.1 为什么不能直接叠加窗口

前几章介绍了将 Flutter/Compose UI 作为独立窗口叠加的方案。这种方案简单，但有严重局限：

```
┌──────────────────────────────────────────────────────┐
│  独立窗口叠加方案的问题                               │
│                                                      │
│  1. 透明度问题：Flutter 窗口叠加时无法真正半透明      │
│     （半透明窗口通常需要 DWM 合成，有性能开销）       │
│                                                      │
│  2. 视觉撕裂：两个独立渲染管线不同步，VSync 不一致   │
│     可能导致 UI 和游戏内容的刷新不在同一帧           │
│                                                      │
│  3. 剪切区域复杂：对话框的圆角/阴影无法正确裁剪      │
│     游戏背景内容                                     │
│                                                      │
│  4. Android 限制：GLSurfaceView 和 ComposeView 的    │
│     Z-order 在某些 Android 版本上行为不一致          │
└──────────────────────────────────────────────────────┘
```

**理想方案**：Flutter/Compose 渲染到一张 OpenGL 纹理，Cocos2d-x 在游戏渲染完成后将这张纹理叠加绘制到最终画面上。这样两者在同一个 VSync 周期内合成，无撕裂，透明度也由 Cocos2d-x 正确处理。

### 1.2 三种共享模式对比

| 模式 | 原理 | 是否零拷贝 | 跨上下文 | 适用平台 |
|---|---|---|---|---|
| **Shared FBO（共享 FBO）** | 两个上下文共享同一个 GL 上下文（EGL shared context），直接读写同一纹理 | ✅ 是 | 部分支持 | 所有 OpenGL 平台 |
| **EGLImage** | 通过 EGLImage 在不同 EGL 上下文间共享纹理 | ✅ 是 | ✅ 完全支持 | Android/Linux/macOS |
| **AHardwareBuffer** | Android 专用硬件缓冲区，GPU 层面共享，支持 Vulkan/OpenGL | ✅ 是 | ✅ 完全支持 | Android 8.0+ |
| **GPU Blit（GPU 拷贝）** | `glBlitFramebuffer` 在 GPU 内部从一个 FBO 复制到另一个 | ❌ 是 GPU 拷贝 | ✅ 支持 | 所有平台（降级方案）|
| **CPU 回读（Readback）** | `glReadPixels` 读回 CPU 内存，再 `glTexSubImage2D` 上传 | ❌ 最慢 | ✅ 支持 | 所有平台（最低公分母）|

## 二、Flutter ExternalTexture API

### 2.1 Flutter 外部纹理机制

Flutter 的 `FlutterRendererConfig` 允许宿主提供一个 FBO ID，Flutter 将所有 UI 内容渲染到这个 FBO 的颜色纹理附件上，而不是默认帧缓冲：

```c
// 在 FlutterOpenGLRendererConfig 中，fbo_callback 指定渲染目标
config.open_gl.fbo_callback = [](void* user_data) -> uint32_t {
    // 返回非零值 = 渲染到指定 FBO
    // 返回 0    = 渲染到默认帧缓冲（屏幕）
    auto* wrapper = static_cast<FlutterEngineWrapper*>(user_data);
    return wrapper->getFlutterFboId();  // 返回我们创建的离屏 FBO
};
```

Flutter 完成渲染后调用 `present` 回调，此时宿主可以将 Flutter 渲染好的纹理作为普通 OpenGL 纹理使用。

### 2.2 FlutterRenderTarget：管理 FBO 和纹理

```cpp
// cpp/core/environ/ui/flutter/FlutterRenderTarget.h
#pragma once

#include <GLES3/gl3.h>
#include <EGL/egl.h>
#include <EGL/eglext.h>
#include <atomic>
#include <functional>
#include <mutex>

namespace krkr2 {
namespace ui {
namespace flutter_backend {

/**
 * FlutterRenderTarget — 管理 Flutter 的离屏渲染目标
 *
 * 核心职责：
 *   1. 创建和管理 FBO + 颜色纹理
 *   2. 在 Flutter 渲染完成时，通知 Cocos2d-x 使用新纹理
 *   3. 实现双缓冲（Flutter 渲染到 Back Buffer，Cocos2d-x 读取 Front Buffer）
 *
 * 双缓冲策略：
 *   ┌──────────────────────────────────────────┐
 *   │  Back Buffer  ← Flutter 当前帧渲染到此处  │
 *   │  Front Buffer ← Cocos2d-x 读取此处的纹理  │
 *   └──────────────────────────────────────────┘
 *   每帧渲染完成时交换（swap）两个缓冲区
 */
class FlutterRenderTarget {
public:
    explicit FlutterRenderTarget(int width, int height);
    ~FlutterRenderTarget();

    FlutterRenderTarget(const FlutterRenderTarget&) = delete;
    FlutterRenderTarget& operator=(const FlutterRenderTarget&) = delete;

    /** 在 OpenGL 上下文中初始化 FBO 和纹理（必须在 GL 线程调用）*/
    bool initialize();

    /** 释放所有 OpenGL 资源 */
    void destroy();

    /**
     * 获取 Flutter 的渲染目标 FBO ID（Flutter 渲染到这里）
     * 对应 fbo_callback 的返回值
     */
    GLuint getBackFboId() const { return fbo_[back_]; }

    /**
     * Flutter 一帧渲染完成后调用（对应 present 回调）
     * 将 Back Buffer 切换为 Front Buffer
     */
    void onFramePresented();

    /**
     * 获取 Cocos2d-x 可读取的纹理 ID（Front Buffer）
     * 每次调用前确保 Flutter 已经 present 至少一帧
     */
    GLuint getFrontTextureId() const { return textures_[front_]; }

    /** 目标尺寸 */
    int width() const  { return width_; }
    int height() const { return height_; }

    /** 调整渲染目标尺寸（窗口大小变化时调用）*/
    void resize(int new_width, int new_height);

    /** 是否有可用的 Front Buffer（Flutter 至少渲染了一帧）*/
    bool hasFrame() const { return has_frame_.load(); }

private:
    // 创建单个 FBO + 纹理对
    bool createFbo(int idx, int w, int h);
    void destroyFbo(int idx);

    int width_, height_;

    // 双缓冲：0 和 1 两组 FBO + Texture
    GLuint fbo_[2]      = {0, 0};
    GLuint textures_[2] = {0, 0};
    GLuint depth_rbo_[2]= {0, 0};  // 深度缓冲（Flutter 需要）

    // back_ = Flutter 当前写入的缓冲区索引
    // front_ = Cocos2d-x 当前读取的缓冲区索引
    int back_  = 0;
    int front_ = 1;

    std::atomic<bool> has_frame_{false};
    mutable std::mutex swap_mutex_;
};

} // namespace flutter_backend
} // namespace ui
} // namespace krkr2
```

```cpp
// cpp/core/environ/ui/flutter/FlutterRenderTarget.cpp
#include "FlutterRenderTarget.h"
#include <stdexcept>

namespace krkr2 {
namespace ui {
namespace flutter_backend {

FlutterRenderTarget::FlutterRenderTarget(int width, int height)
    : width_(width), height_(height) {}

FlutterRenderTarget::~FlutterRenderTarget() {
    if (fbo_[0] != 0 || fbo_[1] != 0) {
        destroy();
    }
}

bool FlutterRenderTarget::initialize() {
    return createFbo(0, width_, height_) && createFbo(1, width_, height_);
}

bool FlutterRenderTarget::createFbo(int idx, int w, int h) {
    // 1. 创建颜色纹理
    glGenTextures(1, &textures_[idx]);
    glBindTexture(GL_TEXTURE_2D, textures_[idx]);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA8, w, h, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
    // 设置纹理过滤（Flutter UI 不需要 mipmap）
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glBindTexture(GL_TEXTURE_2D, 0);

    // 2. 创建深度 RBO（Flutter 的渲染需要深度缓冲）
    glGenRenderbuffers(1, &depth_rbo_[idx]);
    glBindRenderbuffer(GL_RENDERBUFFER, depth_rbo_[idx]);
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8, w, h);
    glBindRenderbuffer(GL_RENDERBUFFER, 0);

    // 3. 创建 FBO 并附加纹理和深度缓冲
    glGenFramebuffers(1, &fbo_[idx]);
    glBindFramebuffer(GL_FRAMEBUFFER, fbo_[idx]);
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                           GL_TEXTURE_2D, textures_[idx], 0);
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_STENCIL_ATTACHMENT,
                              GL_RENDERBUFFER, depth_rbo_[idx]);

    // 4. 验证 FBO 完整性
    GLenum status = glCheckFramebufferStatus(GL_FRAMEBUFFER);
    glBindFramebuffer(GL_FRAMEBUFFER, 0);

    if (status != GL_FRAMEBUFFER_COMPLETE) {
        return false;
    }
    return true;
}

void FlutterRenderTarget::destroyFbo(int idx) {
    if (fbo_[idx]) {
        glDeleteFramebuffers(1, &fbo_[idx]);
        fbo_[idx] = 0;
    }
    if (textures_[idx]) {
        glDeleteTextures(1, &textures_[idx]);
        textures_[idx] = 0;
    }
    if (depth_rbo_[idx]) {
        glDeleteRenderbuffers(1, &depth_rbo_[idx]);
        depth_rbo_[idx] = 0;
    }
}

void FlutterRenderTarget::destroy() {
    destroyFbo(0);
    destroyFbo(1);
}

void FlutterRenderTarget::onFramePresented() {
    // Flutter 完成渲染后调用：交换前后缓冲区
    std::lock_guard<std::mutex> lock(swap_mutex_);
    std::swap(back_, front_);
    has_frame_.store(true);
}

void FlutterRenderTarget::resize(int new_width, int new_height) {
    if (new_width == width_ && new_height == height_) return;
    
    width_  = new_width;
    height_ = new_height;
    
    // 重建 FBO（需要在 GL 上下文中调用）
    destroy();
    initialize();
    has_frame_.store(false);
}

} // namespace flutter_backend
} // namespace ui
} // namespace krkr2
```

## 三、EGLImage 跨上下文纹理共享

### 3.1 EGLImage 原理

EGLImage 是 EGL 扩展 `EGL_KHR_image_base` 提供的机制，允许将一个 EGL 上下文中的 OpenGL 纹理"导出"为 EGLImage 对象，然后在另一个 EGL 上下文中"导入"（不复制数据）：

```
上下文 A（Flutter 渲染线程）       上下文 B（Cocos2d-x 渲染线程）
        │                                   │
  glGenTextures → textureA                  │
        │                                   │
  eglCreateImageKHR(textureA) → eglImage   │
        │                                   │
        ├───────── eglImage 传递 ───────────►│
                                            │
                          glGenTextures → textureB
                                            │
                    glEGLImageTargetTexture2DOES(textureB, eglImage)
                                            │
                          // textureB 现在指向与 textureA 相同的 GPU 内存！
```

### 3.2 EGLImage 实现（Android/Linux）

```cpp
// cpp/core/environ/ui/flutter/EGLTextureSharing.h
#pragma once

#ifdef __ANDROID__
#include <EGL/egl.h>
#include <EGL/eglext.h>
#include <GLES3/gl3.h>
#include <GLES2/gl2ext.h>  // GL_OES_EGL_image 扩展

namespace krkr2 {
namespace ui {
namespace flutter_backend {

/**
 * EGLTextureSharing — EGL 跨上下文纹理共享实现
 *
 * 使用步骤：
 *   1. 在 Flutter 上下文中：exportTexture(texture_id) → EGLImageKHR
 *   2. 将 EGLImageKHR 传递给 Cocos2d-x 渲染线程
 *   3. 在 Cocos2d-x 上下文中：importTexture(egl_image) → new texture_id
 *   4. 在 Cocos2d-x 的 Sprite 中使用这个 texture_id
 */
class EGLTextureSharing {
public:
    /**
     * 从 Flutter 侧导出纹理为 EGLImage
     * （在 Flutter 的 EGL 上下文中调用）
     * 
     * @param texture_id  Flutter 渲染到的 OpenGL 纹理 ID
     * @return  EGLImage 句柄，NULL 表示失败
     */
    static EGLImageKHR exportTexture(GLuint texture_id) {
        EGLDisplay display = eglGetCurrentDisplay();
        if (display == EGL_NO_DISPLAY) return EGL_NO_IMAGE_KHR;
        
        // EGL_GL_TEXTURE_2D_KHR 表示从 2D 纹理创建 EGLImage
        EGLClientBuffer buffer = reinterpret_cast<EGLClientBuffer>(
            static_cast<uintptr_t>(texture_id)
        );
        
        EGLint attribs[] = {
            EGL_GL_TEXTURE_LEVEL_KHR, 0,  // mip level 0
            EGL_IMAGE_PRESERVED_KHR,  EGL_TRUE,  // 保留现有纹理内容
            EGL_NONE
        };
        
        EGLImageKHR image = eglCreateImageKHR(
            display,
            eglGetCurrentContext(),
            EGL_GL_TEXTURE_2D_KHR,
            buffer,
            attribs
        );
        
        if (image == EGL_NO_IMAGE_KHR) {
            // 检查错误
            // EGL_BAD_ATTRIBUTE  — 属性不支持
            // EGL_BAD_ACCESS     — 纹理未完成（mipmap 不完整等）
        }
        
        return image;
    }
    
    /**
     * 在 Cocos2d-x 侧导入 EGLImage 为本地纹理
     * （在 Cocos2d-x 的 EGL 上下文中调用）
     * 
     * @param egl_image   从 Flutter 侧导出的 EGLImage
     * @return  本地 OpenGL 纹理 ID（与 Flutter 的纹理共享 GPU 内存）
     */
    static GLuint importTexture(EGLImageKHR egl_image) {
        if (egl_image == EGL_NO_IMAGE_KHR) return 0;
        
        GLuint texture_id = 0;
        glGenTextures(1, &texture_id);
        glBindTexture(GL_TEXTURE_EXTERNAL_OES, texture_id);
        
        // GL_OES_EGL_image 扩展：将 EGLImage 绑定到纹理
        // 此后对 texture_id 的读取会直接访问 Flutter 渲染的 GPU 内存
        glEGLImageTargetTexture2DOES(GL_TEXTURE_EXTERNAL_OES, egl_image);
        
        // 必须使用 GL_TEXTURE_EXTERNAL_OES 作为 target，
        // 在 GLSL shader 中也需要 samplerExternalOES
        
        glTexParameteri(GL_TEXTURE_EXTERNAL_OES, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_EXTERNAL_OES, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glBindTexture(GL_TEXTURE_EXTERNAL_OES, 0);
        
        return texture_id;
    }
    
    /**
     * 销毁 EGLImage（不销毁原始纹理，只销毁共享引用）
     */
    static void destroyEGLImage(EGLImageKHR egl_image) {
        if (egl_image != EGL_NO_IMAGE_KHR) {
            EGLDisplay display = eglGetCurrentDisplay();
            eglDestroyImageKHR(display, egl_image);
        }
    }
    
    /**
     * 检查当前 EGL/GL 环境是否支持 EGLImage 纹理共享
     */
    static bool isSupported() {
        const char* egl_extensions = eglQueryString(
            eglGetCurrentDisplay(), EGL_EXTENSIONS
        );
        const char* gl_extensions = (const char*)glGetString(GL_EXTENSIONS);
        
        bool has_egl_image_base = egl_extensions && 
            strstr(egl_extensions, "EGL_KHR_image_base") != nullptr;
        bool has_egl_gl_texture = egl_extensions &&
            strstr(egl_extensions, "EGL_KHR_gl_texture_2D_image") != nullptr;
        bool has_gl_oes_egl_image = gl_extensions &&
            strstr(gl_extensions, "GL_OES_EGL_image") != nullptr;
        
        return has_egl_image_base && has_egl_gl_texture && has_gl_oes_egl_image;
    }
};

} // namespace flutter_backend
} // namespace ui
} // namespace krkr2
} // anonymous namespace

#endif // __ANDROID__
```

## 四、AHardwareBuffer 方案（Android 8.0+）

### 4.1 AHardwareBuffer 与 EGLImage 的区别

`AHardwareBuffer` 是 Android 8.0 引入的原生硬件缓冲区 API，可以理解为更高层的抽象：

```
                    AHardwareBuffer
                          │
              ┌───────────┼───────────┐
              │           │           │
         OpenGL ES     Vulkan      CPU 访问
    (via EGLImage)  (via VkImage)  (AHardwareBuffer_lock)
```

AHardwareBuffer 相比直接使用 EGLImage 的优势：
- 不依赖 EGL 上下文（可在 CPU 线程中创建和传递）
- 统一了 OpenGL 和 Vulkan 的共享接口
- 在 NDK 层有完整的 C API

### 4.2 AHardwareBuffer 实现

```cpp
// cpp/core/environ/ui/flutter/HardwareBufferSharing.h（Android only）
#pragma once

#ifdef __ANDROID__
#include <android/hardware_buffer.h>
#include <GLES3/gl3.h>
#include <EGL/egl.h>
#include <EGL/eglext.h>

namespace krkr2 {
namespace ui {
namespace flutter_backend {

/**
 * HardwareBufferSharing — 基于 AHardwareBuffer 的纹理共享
 *
 * 更适合跨进程共享（如果 Flutter Engine 在独立进程中运行）
 * 对于同进程内的跨上下文共享，与 EGLImage 方案效果相同
 */
class HardwareBufferSharing {
public:
    /**
     * 创建一个 AHardwareBuffer 作为共享渲染目标
     * 
     * @param width, height  纹理尺寸
     * @return  AHardwareBuffer 指针，nullptr 表示失败
     */
    static AHardwareBuffer* createBuffer(int width, int height) {
        AHardwareBuffer_Desc desc = {};
        desc.width  = static_cast<uint32_t>(width);
        desc.height = static_cast<uint32_t>(height);
        desc.layers = 1;
        desc.format = AHARDWAREBUFFER_FORMAT_R8G8B8A8_UNORM;
        desc.usage  = AHARDWAREBUFFER_USAGE_GPU_FRAMEBUFFER    // Flutter 渲染目标
                    | AHARDWAREBUFFER_USAGE_GPU_SAMPLED_IMAGE   // Cocos2d-x 采样
                    | AHARDWAREBUFFER_USAGE_CPU_WRITE_RARELY;   // 可能的 CPU 调试写入
        
        AHardwareBuffer* buffer = nullptr;
        int result = AHardwareBuffer_allocate(&desc, &buffer);
        if (result != 0) {
            return nullptr;
        }
        return buffer;
    }
    
    /**
     * 将 AHardwareBuffer 导入为 EGLImage（用于 OpenGL 绑定）
     * 
     * @param buffer      AHardwareBuffer 指针
     * @return  EGLImage 句柄
     */
    static EGLImageKHR toEGLImage(AHardwareBuffer* buffer) {
        EGLDisplay display = eglGetCurrentDisplay();
        
        // EGL_ANDROID_get_native_client_buffer 扩展
        EGLClientBuffer client_buffer = 
            eglGetNativeClientBufferANDROID(buffer);
        
        EGLint attribs[] = { EGL_NONE };
        
        return eglCreateImageKHR(
            display,
            EGL_NO_CONTEXT,             // 无上下文（buffer 独立于上下文）
            EGL_NATIVE_BUFFER_ANDROID,  // AHardwareBuffer 类型
            client_buffer,
            attribs
        );
    }
    
    /**
     * 从 AHardwareBuffer 创建 OpenGL FBO
     * Flutter 可以将此 FBO 作为渲染目标
     */
    static GLuint createFboFromBuffer(AHardwareBuffer* buffer) {
        // 1. 将 buffer 转为 EGLImage
        EGLImageKHR egl_image = toEGLImage(buffer);
        if (egl_image == EGL_NO_IMAGE_KHR) return 0;
        
        // 2. 创建纹理并绑定 EGLImage
        GLuint texture = 0;
        glGenTextures(1, &texture);
        glBindTexture(GL_TEXTURE_2D, texture);
        glEGLImageTargetTexture2DOES(GL_TEXTURE_2D, egl_image);
        
        // 3. 创建 FBO 并绑定纹理
        GLuint fbo = 0;
        glGenFramebuffers(1, &fbo);
        glBindFramebuffer(GL_FRAMEBUFFER, fbo);
        glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                               GL_TEXTURE_2D, texture, 0);
        
        // 4. EGLImage 已附加到纹理，可以销毁 EGLImage 对象（引用计数会保留数据）
        eglDestroyImageKHR(eglGetCurrentDisplay(), egl_image);
        
        glBindFramebuffer(GL_FRAMEBUFFER, 0);
        glBindTexture(GL_TEXTURE_2D, 0);
        
        return fbo;
    }
    
    /** 释放 AHardwareBuffer */
    static void releaseBuffer(AHardwareBuffer* buffer) {
        if (buffer) AHardwareBuffer_release(buffer);
    }
    
    /** 检查 AHardwareBuffer 是否被当前设备支持 */
    static bool isSupported() {
        // 检查 Android API level
        // 在 NDK 中，可通过 android_get_device_api_level() 获取
        // 或检查 EGL 扩展：EGL_ANDROID_get_native_client_buffer
        const char* egl_ext = eglQueryString(eglGetCurrentDisplay(), EGL_EXTENSIONS);
        return egl_ext && 
               strstr(egl_ext, "EGL_ANDROID_get_native_client_buffer") != nullptr;
    }
};

} // namespace flutter_backend
} // namespace ui
} // namespace krkr2

#endif // __ANDROID__
```

## 五、跨上下文同步

### 5.1 为什么需要同步

纹理共享最大的陷阱是**GPU 读写竞争**。Flutter 渲染线程写入纹理时，Cocos2d-x 渲染线程可能同时在读取同一纹理，导致画面撕裂或程序崩溃。

```
Flutter 渲染线程         Cocos2d-x 渲染线程
  t=0: 开始写入纹理A
  t=1:               ← 开始读取纹理A（撕裂！读到部分新数据）
  t=2: 完成写入纹理A
  t=3:               ← 理想读取时机
```

### 5.2 EGL Fence Sync 实现跨线程同步

```cpp
// cpp/core/environ/ui/flutter/GPUSync.h
#pragma once

#include <EGL/egl.h>
#include <EGL/eglext.h>

namespace krkr2 {
namespace ui {
namespace flutter_backend {

/**
 * GPUSync — EGL Fence Sync 封装
 * 
 * 用法：
 *   1. Flutter 完成渲染后：auto fence = GPUSync::createFence();
 *   2. 将 fence 传给 Cocos2d-x 渲染线程
 *   3. Cocos2d-x 读取纹理前：GPUSync::waitFence(fence);
 *   4. GPUSync::destroyFence(fence);
 */
class GPUSync {
public:
    /**
     * 创建一个 EGL Fence Sync
     * 当前 GL 上下文中所有已提交的命令完成时，fence 会被触发
     * （在 Flutter 的 present 回调中调用）
     */
    static EGLSyncKHR createFence() {
        EGLDisplay display = eglGetCurrentDisplay();
        if (display == EGL_NO_DISPLAY) return EGL_NO_SYNC_KHR;
        
        // 插入 fence：等待 GL 命令队列中当前所有命令完成
        EGLSyncKHR sync = eglCreateSyncKHR(
            display,
            EGL_SYNC_FENCE_KHR,
            nullptr
        );
        
        // 确保 fence 已插入 GPU 命令流（不是可选的！）
        glFlush();
        
        return sync;
    }
    
    /**
     * 等待 fence 触发（阻塞当前线程）
     * （在 Cocos2d-x 读取纹理之前调用）
     * 
     * @param timeout_ns  最大等待时间（纳秒），EGL_FOREVER_KHR 表示无限等待
     * @return  true = fence 已触发，false = 超时或错误
     */
    static bool waitFence(EGLSyncKHR fence, 
                          EGLTimeKHR timeout_ns = EGL_FOREVER_KHR) {
        if (fence == EGL_NO_SYNC_KHR) return false;
        
        EGLDisplay display = eglGetCurrentDisplay();
        EGLint result = eglClientWaitSyncKHR(
            display, fence,
            EGL_SYNC_FLUSH_COMMANDS_BIT_KHR,  // 如果 fence 未触发，先 flush GPU
            timeout_ns
        );
        
        return result == EGL_CONDITION_SATISFIED_KHR;
    }
    
    /**
     * 销毁 fence 对象
     */
    static void destroyFence(EGLSyncKHR fence) {
        if (fence != EGL_NO_SYNC_KHR) {
            EGLDisplay display = eglGetCurrentDisplay();
            eglDestroySyncKHR(display, fence);
        }
    }
};

} // namespace flutter_backend
} // namespace ui
} // namespace krkr2
```

### 5.3 在 FlutterRenderTarget 中集成同步

```cpp
// FlutterRenderTarget::onFramePresented 的增强版（含同步）
void FlutterRenderTarget::onFramePresented() {
    // 1. 创建 fence（在 Flutter 渲染完成后、上下文切换前）
    EGLSyncKHR fence = GPUSync::createFence();
    
    // 2. 交换缓冲区
    {
        std::lock_guard<std::mutex> lock(swap_mutex_);
        std::swap(back_, front_);
        pending_fence_ = fence;  // 保存 fence 给 Cocos2d-x 使用
        has_frame_.store(true);
    }
}

// Cocos2d-x 渲染线程中，在使用 Flutter 纹理之前：
GLuint FlutterRenderTarget::getFrontTextureIdSafe() {
    std::lock_guard<std::mutex> lock(swap_mutex_);
    
    // 等待 Flutter 完成渲染
    if (pending_fence_ != EGL_NO_SYNC_KHR) {
        GPUSync::waitFence(pending_fence_, 10'000'000ULL);  // 10ms 超时
        GPUSync::destroyFence(pending_fence_);
        pending_fence_ = EGL_NO_SYNC_KHR;
    }
    
    return textures_[front_];
}
```

### 6.1 使用 Flutter 纹理创建 Cocos2d-x Sprite

```cpp
// cpp/core/environ/ui/flutter/FlutterOverlayNode.cpp
// Cocos2d-x 节点，将 Flutter 渲染的 UI 覆盖绘制在游戏上方

#include "FlutterOverlayNode.h"
#include "FlutterRenderTarget.h"
#include "cocos2d.h"

USING_NS_CC;

class FlutterOverlayNode : public cocos2d::Node {
public:
    static FlutterOverlayNode* create(FlutterRenderTarget* render_target) {
        auto* node = new (std::nothrow) FlutterOverlayNode();
        if (node && node->init(render_target)) {
            node->autorelease();
            return node;
        }
        CC_SAFE_DELETE(node);
        return nullptr;
    }

    bool init(FlutterRenderTarget* render_target) {
        if (!Node::init()) return false;
        render_target_ = render_target;
        
        // 设置绘制顺序（最高优先级，绘制在游戏内容最上层）
        this->setLocalZOrder(999);
        
        // 调度更新（每帧检查 Flutter 是否有新帧）
        this->scheduleUpdate();
        
        return true;
    }

    void update(float dt) override {
        if (!render_target_ || !render_target_->hasFrame()) return;
        
        // 获取 Flutter 最新渲染的纹理 ID（已同步）
        GLuint tex_id = render_target_->getFrontTextureIdSafe();
        if (tex_id == 0) return;
        
        // 更新 Sprite 的纹理（如果纹理变化了）
        if (tex_id != current_texture_id_) {
            current_texture_id_ = tex_id;
            
            // 用 Flutter 纹理创建 Cocos2d-x Texture2D
            auto* cc_texture = new cocos2d::Texture2D();
            cc_texture->initWithGLtextureHandle(
                tex_id,
                render_target_->width(),
                render_target_->height(),
                cocos2d::Texture2D::PixelFormat::RGBA8888
            );
            
            if (!overlay_sprite_) {
                overlay_sprite_ = cocos2d::Sprite::createWithTexture(cc_texture);
                overlay_sprite_->setAnchorPoint(Vec2::ANCHOR_BOTTOM_LEFT);
                overlay_sprite_->setPosition(Vec2::ZERO);
                // 启用混合（Flutter UI 有 alpha 透明度）
                overlay_sprite_->setBlendFunc(BlendFunc::ALPHA_PREMULTIPLIED);
                this->addChild(overlay_sprite_);
            } else {
                overlay_sprite_->setTexture(cc_texture);
            }
            
            cc_texture->release();
        }
    }

private:
    FlutterRenderTarget* render_target_ = nullptr;
    cocos2d::Sprite*     overlay_sprite_ = nullptr;
    GLuint               current_texture_id_ = 0;
};
```

### 7.1 按平台选择最优共享方案

```cpp
// cpp/core/environ/ui/flutter/TextureSharingStrategy.h

enum class TextureSharingMode {
    EGLImageShared,      // EGLImage 零拷贝（Android/Linux）
    HardwareBuffer,      // AHardwareBuffer（Android 8.0+）
    SharedEGLContext,    // 共享 EGL 上下文（Desktop 优先）
    GPUBlit,             // GPU 侧拷贝（降级方案）
    CPUReadback,         // CPU 回读（最低级降级）
};

TextureSharingMode selectBestMode() {
#ifdef __ANDROID__
    if (HardwareBufferSharing::isSupported()) {
        return TextureSharingMode::HardwareBuffer;
    }
    if (EGLTextureSharing::isSupported()) {
        return TextureSharingMode::EGLImageShared;
    }
#elif defined(__linux__)
    if (EGLTextureSharing::isSupported()) {
        return TextureSharingMode::EGLImageShared;
    }
    // Linux 也可以尝试 Shared EGL Context
    return TextureSharingMode::SharedEGLContext;
#elif defined(_WIN32)
    // Windows 上 Cocos2d-x 通常用 ANGLE（EGL over D3D11）
    // ANGLE 支持 EGL shared context 但不支持 EGL_KHR_image_base
    return TextureSharingMode::SharedEGLContext;
#elif defined(__APPLE__)
    // macOS 使用 Metal，通过 IOSurface 共享
    // 这里简化为 GPU Blit
    return TextureSharingMode::GPUBlit;
#endif
    return TextureSharingMode::GPUBlit;
}
```

## 本节小结

- Flutter 通过 `fbo_callback` 让宿主提供渲染目标，完成后调用 `present`，实现离屏渲染
- 双缓冲（Back/Front Buffer）避免 Flutter 写入和 Cocos2d-x 读取同时发生
- EGLImage 允许在不同 EGL 上下文间**零拷贝**共享纹理；AHardwareBuffer 是 Android 8.0+ 的更高层抽象
- EGL Fence Sync 确保 Flutter 渲染完成后 Cocos2d-x 才读取纹理（解决 GPU 读写竞争）
- 平台降级策略：AHardwareBuffer > EGLImage > 共享 EGL 上下文 > GPU Blit > CPU Readback

## 练习题与答案

**题 1：** 双缓冲方案中，如果 Flutter 渲染速度比 Cocos2d-x 更慢（Flutter 还没准备好新帧），Cocos2d-x 应如何处理？

<details>
<summary>查看答案</summary>

Cocos2d-x 应继续使用上一帧的 Front Buffer（旧帧），而不是等待 Flutter 完成。`FlutterRenderTarget::hasFrame()` 和 `has_frame_` 标志的作用就是：只要 Flutter 至少渲染了一帧，Front Buffer 就是有效的。

具体实现：
- 首次渲染：在 Flutter 完成第一帧之前，`hasFrame()` 返回 false，Cocos2d-x 跳过 Flutter 覆盖层
- 后续帧：`onFramePresented()` 每次交换 back/front 时设置 `has_frame_ = true`，Cocos2d-x 始终使用上一帧已完成渲染的 Front Buffer
- 优先级：游戏流畅度优先，UI 帧可以适当 "跟不上" 游戏帧率

这类似于视频播放器的"最后一帧显示"机制。

</details>

---

**题 2：** 如果 Cocos2d-x 和 Flutter 共享同一个 EGL 上下文（Shared Context），是否还需要 EGL Fence Sync？为什么？

<details>
<summary>查看答案</summary>

**仍然需要同步，但形式不同。**

共享 EGL 上下文意味着两个线程可以操作同一批 GL 对象（纹理、FBO 等），但 OpenGL ES 规范明确指出：**从多个线程并发访问同一个 GL 对象的行为是未定义的**（即使上下文共享）。

对于 Shared Context 方案：
- 不需要 EGLImage（纹理 ID 直接共用）
- 但仍需要应用层同步（互斥锁 + Fence Sync）确保：
  - Flutter 完成对纹理的写入操作（`glFramebufferTexture2D` + 渲染）
  - **再**由 Cocos2d-x 开始对同一纹理的读取（`glBindTexture` + 采样）

最简单的方案：使用互斥锁保护纹理指针的读写，配合 `glFinish()` 确保 GPU 命令执行完毕（但 `glFinish()` 会阻塞等待，性能不如 Fence Sync）。

EGL Fence Sync 优于 `glFinish()` 的原因：Fence 允许 GPU 继续执行后续命令，只有当依赖纹理的读取实际发生时才等待，而不是停止整个 GPU 管线。

</details>

---

**题 3：** `GL_TEXTURE_EXTERNAL_OES` 和普通的 `GL_TEXTURE_2D` 有什么区别？使用 EGLImage 导入的纹理时，Cocos2d-x 的 GLSL Shader 需要做哪些修改？

<details>
<summary>查看答案</summary>

**区别：**
- `GL_TEXTURE_2D`：标准 2D 纹理，坐标范围 [0,1]×[0,1]，在片元着色器中使用 `sampler2D`
- `GL_TEXTURE_EXTERNAL_OES`：外部纹理（来自 EGLImage/Camera/视频等），坐标可能是非标准范围，需要变换矩阵，在片元着色器中使用 `samplerExternalOES`

**Shader 修改：**

原始 Cocos2d-x 默认 Shader（标准纹理）：
```glsl
// 片元着色器
precision mediump float;
uniform sampler2D CC_Texture0;
varying vec2 v_texCoord;

void main() {
    gl_FragColor = texture2D(CC_Texture0, v_texCoord);
}
```

修改为支持外部纹理：
```glsl
// 片元着色器（外部纹理版本）
#extension GL_OES_EGL_image_external : require

precision mediump float;
uniform samplerExternalOES CC_Texture0;  // ← 改为 samplerExternalOES
varying vec2 v_texCoord;

void main() {
    gl_FragColor = texture2D(CC_Texture0, v_texCoord);
    // 注意：texture2D 仍然可用（扩展提供了重载）
    // 也可以使用 texture() （GLSL 3.00 ES）
}
```

在实践中，如果 EGLImage 导入时使用 `GL_TEXTURE_2D` 作为 target（而非 `GL_TEXTURE_EXTERNAL_OES`），则不需要修改 Shader。但某些 EGL 实现（特别是 Android 上的 Camera 纹理）强制要求使用 `GL_TEXTURE_EXTERNAL_OES`。对于 Flutter 渲染纹理，通常可以使用普通 `GL_TEXTURE_2D`。

</details>
