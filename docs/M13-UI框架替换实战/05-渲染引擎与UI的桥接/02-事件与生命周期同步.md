## 术语预览

| 术语 | 含义 |
|------|------|
| `FlutterPointerEvent` | Flutter Embedder API 中描述单次指针输入的 C 结构体，包含 phase、坐标、时间戳等字段 |
| `FlutterPointerPhase` | 指针阶段枚举：`kAdd`、`kDown`、`kMove`、`kUp`、`kRemove`、`kCancel`、`kHover` |
| Hit Test | 判断触摸点落在哪个 Widget 上的过程，由 Flutter 引擎在 UI 线程上完成 |
| Platform Task Runner | Embedder 提供给 Flutter 引擎的"平台线程"任务队列，处理 Platform Channel 消息和指针事件 |
| `AppDelegate` | Cocos2d-x 应用生命周期接口，定义 `applicationDidEnterBackground`、`applicationWillEnterForeground` 等回调 |
| `FlutterEngineNotifyLowMemoryWarning` | Embedder API，通知 Flutter 引擎系统内存不足，触发 Dart 侧 GC |
| `FlutterEngineRunsAOTCompiledDartCode` | 查询当前引擎是否以 AOT 模式运行，影响 snapshot 路径选择 |
| EGL Fence Sync | OpenGL ES 层面的同步原语，确保 GPU 命令在 CPU 侧等待完成后再继续 |

## 1. 输入事件传递架构概览

### 1.1 事件流全景

在 KrKr2 嵌入 Flutter UI 的架构中，用户触摸屏幕后事件的流向如下：

```
用户触摸
  │
  ▼
Android/Windows 原生输入系统
  │  (通过 Cocos2d-x 的 GLSurfaceView / Win32 WM_TOUCH)
  ▼
Cocos2d-x EventDispatcher（onTouchBegan / onTouchMoved / onTouchEnded）
  │
  ├─── [Cocos2d-x 层：游戏 UI 消费] ──► 游戏逻辑（krkr2 脚本交互）
  │
  └─── [Flutter 覆盖层可见时] ──► FlutterEngineWrapper::sendPointerEvent()
                                        │
                                        ▼
                               Flutter Engine（UI 线程）
                                 └─ HitTest → Widget 响应
                                        │
                                        ▼
                               Flutter Dart 侧 GestureDetector 回调
                                        │
                                        ▼
                               Platform Channel 消息 → C++ 处理
```

关键原则：**Cocos2d-x 的触摸系统是事件的"单一来源"**，Flutter 引擎本身不直接接收操作系统输入，需要 Embedder 层主动将触摸数据"喂"给它。这与 Flutter 在 Android/iOS 上作为主框架时的行为完全不同——在那些平台上，Flutter 的 PlatformView 通过 `FlutterView` 直接接收原生事件。

### 1.2 为什么不能让 Flutter 直接接收原生事件

在 KrKr2 的场景下，Flutter 以"嵌入纹理"的方式渲染，其 Surface 并不是原生视图层级的一部分（没有实体 `SurfaceView` 或 `ANativeWindow`）。因此：

- Android 的 `View.onTouchEvent` 不会触发到 Flutter 的 `FlutterView`
- Flutter 引擎无法通过 `InputMethodManager` 收到软键盘输入
- 所有输入**必须**由 Embedder 通过 `FlutterEngineSendPointerEvent` API 主动注入

## 2. FlutterPointerEvent 结构体详解

### 2.1 字段一览

```c
// flutter_embedder.h 中的完整定义（精简注释版）
typedef struct {
  size_t struct_size;            // 必须 = sizeof(FlutterPointerEvent)

  FlutterPointerPhase phase;     // 指针阶段（见下方枚举说明）

  size_t timestamp;              // 事件时间戳，微秒级，从任意固定起点计算

  double x;                      // 逻辑像素坐标（非物理像素）
  double y;                      // 逻辑像素坐标（非物理像素）

  FlutterPointerDeviceKind device_kind;  // touch / mouse / stylus

  int64_t device;                // 多点触控时的 touch ID（0-based）

  double scroll_delta_x;         // 鼠标滚轮（水平）
  double scroll_delta_y;         // 鼠标滚轮（垂直）

  FlutterPointerMouseButtons buttons;   // 鼠标按键状态位掩码

  // Stylus-only 字段
  double pan_x;
  double pan_y;
  double scale;
  double rotation;
} FlutterPointerEvent;
```

### 2.2 Phase 枚举使用规范

`FlutterPointerPhase` 的状态机如下，**不能跳跃**：

```
kAdd ──► kDown ──► kMove（0~N 次）──► kUp ──► kRemove
                                     └──► kCancel（异常中断）
```

- `kAdd`：指针首次出现在屏幕上（手指靠近但未按下，适用于 Hover）
- `kDown`：手指按下
- `kMove`：手指移动（必须在 Down 和 Up 之间）
- `kUp`：手指抬起
- `kCancel`：手势被系统取消（来电打断、屏幕关闭等）
- `kRemove`：指针离开感应区域（Hover 结束）

**常见错误**：直接从 `kDown` 跳到 `kRemove`，或在 `kUp` 后继续发 `kMove`，会导致 Flutter 内部状态机混乱，Widget 无法正确响应后续手势。

### 2.3 坐标系转换

Flutter 使用**逻辑像素**（logical pixel），而 Cocos2d-x 的触摸坐标通常是物理像素（physical pixel）。需要除以设备像素比（DPR）：

```cpp
// 获取 DPR（设备像素比）
float getDpr() {
#if CC_TARGET_PLATFORM == CC_PLATFORM_ANDROID
    // Android JNI 获取
    JNIEnv* env = cocos2d::JniHelper::getEnv();
    jclass cls = env->FindClass("android/content/res/Resources");
    jmethodID getSystem = env->GetStaticMethodID(cls, "getSystem",
        "()Landroid/content/res/Resources;");
    jobject res = env->CallStaticObjectMethod(cls, getSystem);
    jclass resCls = env->GetObjectClass(res);
    jfieldID dmField = env->GetFieldID(resCls, "displayMetrics",
        "Landroid/util/DisplayMetrics;");
    jobject dm = env->GetObjectField(res, dmField);
    jclass dmCls = env->GetObjectClass(dm);
    jfieldID densityField = env->GetFieldID(dmCls, "density", "F");
    return env->GetFloatField(dm, densityField);
#else
    return cocos2d::Director::getInstance()
               ->getContentScaleFactor();
#endif
}

// 物理像素 → 逻辑像素
cocos2d::Vec2 toLogical(const cocos2d::Vec2& physical) {
    float dpr = getDpr();
    return { physical.x / dpr, physical.y / dpr };
}
```

### 2.4 时间戳规范

Flutter 期望 **微秒级**时间戳，且要求单调递增。在 Android 上使用 `CLOCK_MONOTONIC`：

```cpp
static size_t getMonotonicUs() {
    struct timespec ts;
    clock_gettime(CLOCK_MONOTONIC, &ts);
    return static_cast<size_t>(ts.tv_sec) * 1000000ULL
           + static_cast<size_t>(ts.tv_nsec) / 1000ULL;
}
```

在 Windows 上使用 `QueryPerformanceCounter`：

```cpp
static size_t getMonotonicUs() {
    LARGE_INTEGER freq, counter;
    QueryPerformanceFrequency(&freq);
    QueryPerformanceCounter(&counter);
    return static_cast<size_t>(
        counter.QuadPart * 1000000ULL / freq.QuadPart
    );
}
```

## 3. 完整的触摸事件注入实现

### 3.1 TouchForwarder 类设计

```cpp
// cpp/core/environ/ui/flutter/TouchForwarder.h
#pragma once
#include <flutter/flutter_embedder.h>
#include <unordered_map>
#include "cocos2d.h"

class FlutterEngineWrapper;

/// 将 Cocos2d-x 触摸事件转发给 Flutter Engine
class TouchForwarder {
public:
    explicit TouchForwarder(FlutterEngineWrapper* engine,
                            float dpr,
                            float viewportWidth,
                            float viewportHeight);

    /// 注册到 Cocos2d-x EventDispatcher
    void registerWithDispatcher(cocos2d::EventDispatcher* dispatcher,
                                cocos2d::Node* target);
    void unregisterFromDispatcher();

    /// 更新视口尺寸（旋转/resize 时调用）
    void setViewport(float w, float h);

private:
    void onTouchesBegan(const std::vector<cocos2d::Touch*>& touches,
                        cocos2d::Event* event);
    void onTouchesMoved(const std::vector<cocos2d::Touch*>& touches,
                        cocos2d::Event* event);
    void onTouchesEnded(const std::vector<cocos2d::Touch*>& touches,
                        cocos2d::Event* event);
    void onTouchesCancelled(const std::vector<cocos2d::Touch*>& touches,
                            cocos2d::Event* event);

    void sendPointerEvent(cocos2d::Touch* touch,
                          FlutterPointerPhase phase);

    FlutterEngineWrapper* engine_;
    float dpr_;
    float viewportW_;
    float viewportH_;
    cocos2d::EventListenerTouchAllAtOnce* listener_ = nullptr;

    /// touch ID → Flutter device ID 映射（多点触控）
    std::unordered_map<int, int64_t> touchDeviceMap_;
    int64_t nextDeviceId_ = 0;
};
```

### 3.2 实现文件

```cpp
// cpp/core/environ/ui/flutter/TouchForwarder.cpp
#include "TouchForwarder.h"
#include "FlutterEngineWrapper.h"

#if CC_TARGET_PLATFORM == CC_PLATFORM_ANDROID
#include <time.h>
#elif CC_TARGET_PLATFORM == CC_PLATFORM_WIN32
#include <windows.h>
#endif

static size_t monoUs() {
#if CC_TARGET_PLATFORM == CC_PLATFORM_ANDROID
    struct timespec ts;
    clock_gettime(CLOCK_MONOTONIC, &ts);
    return (size_t)ts.tv_sec * 1000000ULL + ts.tv_nsec / 1000ULL;
#elif CC_TARGET_PLATFORM == CC_PLATFORM_WIN32
    LARGE_INTEGER freq, counter;
    QueryPerformanceFrequency(&freq);
    QueryPerformanceCounter(&counter);
    return (size_t)(counter.QuadPart * 1000000ULL / freq.QuadPart);
#else
    struct timespec ts;
    clock_gettime(CLOCK_MONOTONIC, &ts);
    return (size_t)ts.tv_sec * 1000000ULL + ts.tv_nsec / 1000ULL;
#endif
}

TouchForwarder::TouchForwarder(FlutterEngineWrapper* engine,
                               float dpr,
                               float w, float h)
    : engine_(engine), dpr_(dpr), viewportW_(w), viewportH_(h) {}

void TouchForwarder::registerWithDispatcher(
        cocos2d::EventDispatcher* dispatcher,
        cocos2d::Node* target) {

    listener_ = cocos2d::EventListenerTouchAllAtOnce::create();

    listener_->onTouchesBegan = [this](
        const std::vector<cocos2d::Touch*>& t, cocos2d::Event* e) {
        onTouchesBegan(t, e);
    };
    listener_->onTouchesMoved = [this](
        const std::vector<cocos2d::Touch*>& t, cocos2d::Event* e) {
        onTouchesMoved(t, e);
    };
    listener_->onTouchesEnded = [this](
        const std::vector<cocos2d::Touch*>& t, cocos2d::Event* e) {
        onTouchesEnded(t, e);
    };
    listener_->onTouchesCancelled = [this](
        const std::vector<cocos2d::Touch*>& t, cocos2d::Event* e) {
        onTouchesCancelled(t, e);
    };

    dispatcher->addEventListenerWithSceneGraphPriority(listener_, target);
}

void TouchForwarder::unregisterFromDispatcher() {
    if (listener_) {
        cocos2d::Director::getInstance()
            ->getEventDispatcher()
            ->removeEventListener(listener_);
        listener_ = nullptr;
    }
}

void TouchForwarder::setViewport(float w, float h) {
    viewportW_ = w;
    viewportH_ = h;
}

void TouchForwarder::sendPointerEvent(cocos2d::Touch* touch,
                                      FlutterPointerPhase phase) {
    // 分配 / 查找 device ID
    int cocosId = touch->getID();
    if (phase == kDown) {
        touchDeviceMap_[cocosId] = nextDeviceId_++;
    }
    auto it = touchDeviceMap_.find(cocosId);
    if (it == touchDeviceMap_.end()) return;  // 未见过的 ID（异常情况）
    int64_t deviceId = it->second;

    // 坐标：Cocos2d-x 物理像素 → Flutter 逻辑像素
    cocos2d::Vec2 phys = touch->getLocationInView();
    // Cocos2d-x Y 轴从底部向上，Flutter 从顶部向下：翻转
    double lx = phys.x / dpr_;
    double ly = (viewportH_ - phys.y) / dpr_;

    FlutterPointerEvent ev = {};
    ev.struct_size   = sizeof(FlutterPointerEvent);
    ev.phase         = phase;
    ev.timestamp     = monoUs();
    ev.x             = lx;
    ev.y             = ly;
    ev.device_kind   = kFlutterPointerDeviceKindTouch;
    ev.device        = deviceId;

    engine_->sendPointerEvent(ev);

    // Up 或 Cancel 后清理 device ID 映射
    if (phase == kUp || phase == kCancel) {
        touchDeviceMap_.erase(it);
    }
}

void TouchForwarder::onTouchesBegan(
        const std::vector<cocos2d::Touch*>& touches,
        cocos2d::Event*) {
    for (auto* t : touches) sendPointerEvent(t, kDown);
}

void TouchForwarder::onTouchesMoved(
        const std::vector<cocos2d::Touch*>& touches,
        cocos2d::Event*) {
    for (auto* t : touches) sendPointerEvent(t, kMove);
}

void TouchForwarder::onTouchesEnded(
        const std::vector<cocos2d::Touch*>& touches,
        cocos2d::Event*) {
    for (auto* t : touches) sendPointerEvent(t, kUp);
}

void TouchForwarder::onTouchesCancelled(
        const std::vector<cocos2d::Touch*>& touches,
        cocos2d::Event*) {
    for (auto* t : touches) sendPointerEvent(t, kCancel);
}
```

## 4. 生命周期同步

### 4.1 为什么生命周期同步至关重要

Flutter 引擎维护自己的动画调度器（`Animator`）和渲染管线。如果宿主 App 进入后台但 Flutter 引擎继续渲染，会导致：

- **电量浪费**：GPU 渲染被继续触发，CPU 唤醒频繁
- **渲染冲突**：EGL Surface 已被 Android 系统回收，但 Flutter 仍在向其提交帧，导致 `EGL_BAD_SURFACE` 错误
- **内存压力**：Flutter 的图像缓存不会自动回收，持续占用内存
- **ANR 风险**：某些 Platform Task Runner 操作需要 Activity 窗口存在，后台调用可能触发死锁

因此，必须在宿主框架的生命周期事件中**同步通知** Flutter 引擎进入/退出暂停状态。

### 4.2 Flutter Engine 生命周期 API

Flutter Embedder API 提供以下生命周期相关函数：

```c
// 通知引擎进入后台（停止渲染、降低优先级）
FlutterResult FlutterEngineNotifyAppIsResuming(FlutterEngine engine);
FlutterResult FlutterEngineNotifyAppIsInactive(FlutterEngine engine);
FlutterResult FlutterEngineNotifyAppIsPaused(FlutterEngine engine);
FlutterResult FlutterEngineNotifyAppIsDetached(FlutterEngine engine);

// 通知低内存警告（触发 Dart GC 和图像缓存清理）
FlutterResult FlutterEngineNotifyLowMemoryWarning(FlutterEngine engine);
```

这四个状态对应 Dart 侧的 `AppLifecycleState` 枚举：

| C API | Dart AppLifecycleState | 含义 |
|-------|------------------------|------|
| `NotifyAppIsResuming` | `resumed` | App 在前台，可见且可交互 |
| `NotifyAppIsInactive` | `inactive` | App 可见但不可交互（通话中、下拉通知栏等） |
| `NotifyAppIsPaused` | `paused` | App 不可见，在后台 |
| `NotifyAppIsDetached` | `detached` | App 即将销毁 |

### 4.3 在 FlutterEngineWrapper 中封装生命周期方法


### 4.4 与 Cocos2d-x AppDelegate 对接

```cpp
// Classes/AppDelegate.cpp（KrKr2 的宿主 App 生命周期）
#include "FlutterEngineWrapper.h"

// 获取全局 engine 实例（由 UIManagerImpl 持有）
extern FlutterEngineWrapper* gFlutterEngine;

void AppDelegate::applicationDidEnterBackground() {
    // 1. 通知 Cocos2d-x 暂停
    cocos2d::Director::getInstance()->stopAnimation();

    // 2. 通知 Flutter 引擎暂停（必须在 GL 线程上执行）
    cocos2d::Director::getInstance()->getScheduler()->performFunctionInCocosThread([]{
        if (gFlutterEngine) {
            gFlutterEngine->notifyPaused();
        }
    });

    // 3. 暂停音效
    cocos2d::SimpleAudioEngine::getInstance()->pauseBackgroundMusic();
    cocos2d::SimpleAudioEngine::getInstance()->pauseAllEffects();
}

void AppDelegate::applicationWillEnterForeground() {
    // 1. 恢复 Cocos2d-x 渲染
    cocos2d::Director::getInstance()->startAnimation();

    // 2. 通知 Flutter 引擎恢复
    cocos2d::Director::getInstance()->getScheduler()->performFunctionInCocosThread([]{
        if (gFlutterEngine) {
            gFlutterEngine->notifyResumed();
        }
    });

    // 3. 恢复音效
    cocos2d::SimpleAudioEngine::getInstance()->resumeBackgroundMusic();
    cocos2d::SimpleAudioEngine::getInstance()->resumeAllEffects();
}
```

**为什么用 `performFunctionInCocosThread`？**  
Flutter 的 Platform Task Runner 绑定在 Cocos2d-x 主线程（详见 M13/03-01），而 `applicationDidEnterBackground` 在 Android 上可能从 UI 线程以外的线程回调，因此需要将 Flutter API 调用 marshal 回主线程。

## 5. Android 特有的生命周期对接

### 5.1 Activity 生命周期回调链路

Android 的生命周期回调在 Java/Kotlin 层，需要通过 JNI 传递到 C++ 层：

```kotlin
// android/app/src/main/kotlin/org/krkr2/KrKr2Activity.kt
class KrKr2Activity : AppCompatActivity() {

    override fun onResume() {
        super.onResume()
        nativeOnResume()
    }

    override fun onPause() {
        super.onPause()
        nativeOnPause()
    }

    override fun onTrimMemory(level: Int) {
        super.onTrimMemory(level)
        // ComponentCallbacks2.TRIM_MEMORY_RUNNING_CRITICAL = 15
        if (level >= ComponentCallbacks2.TRIM_MEMORY_RUNNING_CRITICAL) {
            nativeOnLowMemory()
        }
    }

    override fun onSaveInstanceState(outState: Bundle) {
        super.onSaveInstanceState(outState)
        // 保存 Flutter 不会自动持久化的状态（如当前打开的对话框类型）
        val dialogState = nativeGetCurrentDialogState()
        outState.putString("flutter_dialog_state", dialogState)
    }

    override fun onRestoreInstanceState(savedInstanceState: Bundle) {
        super.onRestoreInstanceState(savedInstanceState)
        val dialogState = savedInstanceState.getString("flutter_dialog_state")
        if (!dialogState.isNullOrEmpty()) {
            nativeRestoreDialogState(dialogState)
        }
    }

    private external fun nativeOnResume()
    private external fun nativeOnPause()
    private external fun nativeOnLowMemory()
    private external fun nativeGetCurrentDialogState(): String
    private external fun nativeRestoreDialogState(state: String)
}
```

```cpp
JNIEXPORT jstring JNICALL
Java_org_krkr2_KrKr2Activity_nativeGetCurrentDialogState(
        JNIEnv* env, jobject) {
    auto* mgr = UIManagerImpl::getInstance();
    std::string state = mgr ? mgr->serializeDialogState() : "";
    return env->NewStringUTF(state.c_str());
}

JNIEXPORT void JNICALL
Java_org_krkr2_KrKr2Activity_nativeRestoreDialogState(
        JNIEnv* env, jobject, jstring jstate) {
    const char* state = env->GetStringUTFChars(jstate, nullptr);
    auto* mgr = UIManagerImpl::getInstance();
    if (mgr) mgr->restoreDialogState(state);
    env->ReleaseStringUTFChars(jstate, state);
}

} // extern "C"
```

### 5.3 实例状态序列化（对话框状态持久化）

`onSaveInstanceState` 在 Android 系统因内存压力杀死 Activity 前调用，重建 Activity 时需要恢复到之前的对话框状态：

```cpp
// UIManagerImpl.cpp 中的状态序列化
#include <nlohmann/json.hpp>
using json = nlohmann::json;

std::string UIManagerImpl::serializeDialogState() {
    json state;
    state["active_dialog"] = currentDialogType_;  // e.g. "config", "game_select"
    state["dialog_visible"] = isDialogVisible_;
    // 序列化对话框内的表单值（如配置项）
    if (currentDialogType_ == "config" && configDialog_) {
        state["config_values"] = configDialog_->getFormValues();
    }
    return state.dump();
}

void UIManagerImpl::restoreDialogState(const std::string& serialized) {
    try {
        json state = json::parse(serialized);
        std::string dialogType = state.value("active_dialog", "");
        bool visible = state.value("dialog_visible", false);

        if (visible && !dialogType.empty()) {
            // 重新打开对话框并恢复表单值
            if (dialogType == "config") {
                auto dlg = createConfigDialog();
                if (state.contains("config_values")) {
                    dlg->setFormValues(state["config_values"]);
                }
                dlg->show();
            }
        }
    } catch (const json::exception& e) {
        CCLOGERROR("Failed to restore dialog state: %s", e.what());
    }
}
```

## 6. 鼠标与键盘事件注入（桌面平台）

### 6.1 鼠标事件注入

桌面平台（Windows/macOS/Linux）的 Flutter UI 需要支持鼠标 Hover 和点击：

```cpp
// cpp/core/environ/ui/flutter/MouseForwarder.h
#pragma once
#include <flutter/flutter_embedder.h>
#include "cocos2d.h"

class FlutterEngineWrapper;

class MouseForwarder {
public:
    explicit MouseForwarder(FlutterEngineWrapper* engine, float dpr);

    void onMouseMove(cocos2d::EventMouse* event);
    void onMouseDown(cocos2d::EventMouse* event);
    void onMouseUp(cocos2d::EventMouse* event);
    void onMouseScroll(cocos2d::EventMouse* event);

private:
    void sendMouse(double x, double y,
                   FlutterPointerPhase phase,
                   FlutterPointerMouseButtons buttons,
                   double scrollX = 0.0, double scrollY = 0.0);

    FlutterEngineWrapper* engine_;
    float dpr_;
    bool hoverActive_ = false;  // 追踪 Hover 状态（kAdd/kRemove）
};
```

```cpp
// cpp/core/environ/ui/flutter/MouseForwarder.cpp
#include "MouseForwarder.h"
#include "FlutterEngineWrapper.h"

MouseForwarder::MouseForwarder(FlutterEngineWrapper* engine, float dpr)
    : engine_(engine), dpr_(dpr) {}

void MouseForwarder::sendMouse(double x, double y,
                               FlutterPointerPhase phase,
                               FlutterPointerMouseButtons buttons,
                               double scrollX, double scrollY) {
    FlutterPointerEvent ev = {};
    ev.struct_size    = sizeof(FlutterPointerEvent);
    ev.phase          = phase;
    ev.timestamp      = monoUs();  // 同前
    ev.x              = x / dpr_;
    ev.y              = y / dpr_;
    ev.device_kind    = kFlutterPointerDeviceKindMouse;
    ev.device         = 0;  // 鼠标只有一个
    ev.buttons        = buttons;
    ev.scroll_delta_x = scrollX;
    ev.scroll_delta_y = scrollY;
    engine_->sendPointerEvent(ev);
}

void MouseForwarder::onMouseMove(cocos2d::EventMouse* event) {
    double x = event->getCursorX();
    double y = event->getCursorY();
    // 首次移动时发送 kAdd（Hover 开始）
    if (!hoverActive_) {
        sendMouse(x, y, kAdd, kFlutterPointerButtonMousePrimary);
        hoverActive_ = true;
    }
    sendMouse(x, y, kHover, 0);
}

void MouseForwarder::onMouseDown(cocos2d::EventMouse* event) {
    FlutterPointerMouseButtons btn =
        (event->getMouseButton() == cocos2d::EventMouse::MouseButton::BUTTON_LEFT)
        ? kFlutterPointerButtonMousePrimary
        : kFlutterPointerButtonMouseSecondary;
    sendMouse(event->getCursorX(), event->getCursorY(), kDown, btn);
}

void MouseForwarder::onMouseUp(cocos2d::EventMouse* event) {
    FlutterPointerMouseButtons btn =
        (event->getMouseButton() == cocos2d::EventMouse::MouseButton::BUTTON_LEFT)
        ? kFlutterPointerButtonMousePrimary
        : kFlutterPointerButtonMouseSecondary;
    sendMouse(event->getCursorX(), event->getCursorY(), kUp, btn);
}

void MouseForwarder::onMouseScroll(cocos2d::EventMouse* event) {
    sendMouse(event->getCursorX(), event->getCursorY(),
              kMove, 0,
              event->getScrollX() * 10.0,
              -event->getScrollY() * 10.0);  // Flutter Y 轴反向
}
```

### 6.2 键盘事件注入

```cpp
#include <flutter/flutter_embedder.h>
#include "FlutterEngineWrapper.h"
#include "cocos2d.h"

void KeyForwarder::onKeyPressed(cocos2d::EventKeyboard::KeyCode keyCode,
                                cocos2d::Event*) {
    FlutterKeyEvent ev = {};
    ev.struct_size    = sizeof(FlutterKeyEvent);
    ev.timestamp      = monoUs();
    ev.type           = kFlutterKeyEventTypeDown;
    ev.physical       = cocosKeyToFlutterPhysical(keyCode);
    ev.logical        = cocosKeyToFlutterLogical(keyCode);
    ev.character      = nullptr;  // 可选：UTF-8 字符串
    ev.synthesized    = false;

    FlutterEngineSendKeyEvent(engine_->getHandle(), &ev,
        [](bool handled, void* userData) {
            if (!handled) {
                // Flutter 未处理，传给下层（Cocos2d-x 游戏）
            }
        }, nullptr);
}
```

## 本节小结

| 知识点 | 核心要点 |
|--------|----------|
| 输入注入 | Flutter 引擎不接收原生输入，Embedder 必须通过 `FlutterEngineSendPointerEvent` 主动注入 |
| Phase 状态机 | kAdd→kDown→kMove→kUp→kRemove，不可跳跃，否则手势识别失效 |
| 坐标转换 | 物理像素 / DPR = 逻辑像素；Cocos2d-x Y 轴需要翻转 |
| 时间戳 | 必须是微秒级单调递增值（`CLOCK_MONOTONIC`） |
| 生命周期同步 | 进入后台必须调用 `FlutterEngineNotifyAppIsPaused`，否则 GPU 渲染持续、EGL Surface 报错 |
| Android 特有 | `onTrimMemory` → `FlutterEngineNotifyLowMemoryWarning`；`onSaveInstanceState` → 序列化对话框状态 |

## 练习题与答案

**练习 1**：为什么 Flutter 的 `FlutterPointerEvent.timestamp` 必须使用单调时钟（`CLOCK_MONOTONIC`），而不能使用系统壁钟时间（`gettimeofday`/`CLOCK_REALTIME`）？

<details>
<summary>答案</summary>

壁钟时间会因 NTP 同步、时区切换、用户手动调整时间等原因**发生跳变**（可能向后回退）。Flutter 引擎的手势识别器（如 `TapGestureRecognizer`、`LongPressGestureRecognizer`）依赖事件时间戳计算时间间隔（如长按阈值 500ms）。如果时间戳回退，时间差变为负数或超大值，手势识别会完全失效。`CLOCK_MONOTONIC` 保证严格单调递增，不受外部时间调整影响。

</details>

---

**练习 2**：`TouchForwarder` 在发送 `kDown` 事件时分配了一个新的 `deviceId`，但如果用户快速多次点击（多个 `kDown` 快速连续），`nextDeviceId_` 会无限增长。这会导致什么问题？应如何修复？

<details>
<summary>答案</summary>

Flutter 引擎内部维护一个"活跃指针"表，每个 deviceId 对应一条触摸轨迹。如果 `deviceId` 无限增长（例如达到 `INT64_MAX`），**不会**导致崩溃，但会持续向 Flutter 引擎注册新的指针设备，造成轻微内存泄漏。

修复方式：使用循环 ID（`nextDeviceId_ = (nextDeviceId_ + 1) % MAX_TOUCH_POINTS`，`MAX_TOUCH_POINTS = 10`），确保同一时间最多 10 个活跃触摸点，ID 循环复用。同时在 `kUp`/`kCancel` 后立即从 `touchDeviceMap_` 中移除，确保 ID 可被复用。

</details>

---

**练习 3**：在 `AppDelegate::applicationDidEnterBackground` 中，为什么要用 `performFunctionInCocosThread` 而不是直接调用 `gFlutterEngine->notifyPaused()`？直接调用会发生什么？

<details>
<summary>答案</summary>

`applicationDidEnterBackground` 在 Android 上由 `GLThread`（Cocos2d-x 的渲染线程）的 `onPause` 触发，但**不一定在主线程**。而 Flutter 的 Platform Task Runner 绑定到 Cocos2d-x 主线程（见 M13/03-01），`FlutterEngineNotifyAppIsPaused` 要求在 Platform Task Runner 的线程上调用。

直接调用会导致 Flutter 内部线程检查断言失败（Debug 构建下崩溃），或在 Release 构建下产生竞态条件，导致引擎状态不一致（如渲染线程已停止但 Platform Channel 仍活跃）。使用 `performFunctionInCocosThread` 将调用 marshal 到正确线程可避免这一问题。

</details>

**练习 4（进阶）**：实现一个 `InputHitTest` 类，在将触摸事件转发给 Flutter 之前，先判断触摸点是否落在"Flutter 覆盖层矩形"内部。如果触摸点在覆盖层外，应直接将事件传给 Cocos2d-x 游戏逻辑，不转发给 Flutter。

<details>
<summary>答案</summary>

```cpp
class InputHitTest {
public:
    void setFlutterRect(float x, float y, float w, float h) {
        rect_ = cocos2d::Rect(x, y, w, h);
    }

    bool isInsideFlutter(const cocos2d::Vec2& physPoint) const {
        // 注意：rect_ 使用逻辑像素，physPoint 需要先转换
        float dpr = getDpr();
        cocos2d::Vec2 logical(physPoint.x / dpr, physPoint.y / dpr);
        return rect_.containsPoint(logical);
    }

private:
    cocos2d::Rect rect_;
};

// 在 TouchForwarder::onTouchesBegan 中使用：
void TouchForwarder::onTouchesBegan(...) {
    for (auto* t : touches) {
        if (hitTest_.isInsideFlutter(t->getLocationInView())) {
            sendPointerEvent(t, kDown);  // 转发给 Flutter
        }
        // else: 由 Cocos2d-x 的其他 Listener 处理
    }
}
```

这样实现了"分区输入路由"：覆盖层内的触摸给 Flutter，覆盖层外的触摸给游戏。

</details>
