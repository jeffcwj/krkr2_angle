# 01-变量与缓存

> **所属模块：** P01-现代CMake与构建工具链
> **前置知识：** 第1章全部内容
> **预计阅读时间：** 20 分钟

## 本节目标
读完本节后，你将能够：
1. 掌握 CMake 变量的定义、引用与作用域规则
2. 理解缓存变量与普通变量的区别，并学会使用 `option()`
3. 熟练使用常用的 CMake 内置变量来控制项目构建

## 正文内容
在第一章中，你已经成功跑通了一个最简单的 "Hello World" 项目。但在实际的 C++ 开发中，源代码文件可能有成百上千个，编译选项也会根据平台（Windows/Linux/macOS/Android）的不同而变化。如果把所有东西都写死在 `add_executable` 里面，维护起来会非常痛苦。

这时候，你就需要 **变量（Variable）**。

### 1. 普通变量（Normal Variable）
普通变量是 CMake 中最基础的存储单元。你可以把它想象成 C++ 中的局部变量。

#### 定义与引用
使用 `set()` 命令定义变量，使用 `${VAR_NAME}` 引用变量：
```cmake
# 定义一个名为 MY_PROJECT_VERSION 的变量
set(MY_PROJECT_VERSION "1.0.0")

# 引用这个变量并打印
message(STATUS "当前项目版本是: ${MY_PROJECT_VERSION}")
```

#### 列表（List）
在 CMake 中，列表其实就是用分号 `;` 分隔的字符串。你可以通过 `set()` 一次性定义多个值：
```cmake
# 定义一个源文件列表
set(MY_SOURCES main.cpp utils.cpp logger.cpp)

# 这等同于
# set(MY_SOURCES "main.cpp;utils.cpp;logger.cpp")

# 在命令中使用列表
add_executable(my_app ${MY_SOURCES})
```

#### 变量作用域（Scope）
普通变量是有作用域的：
- **目录作用域**：在某个 `CMakeLists.txt` 中定义的变量，对该目录及其所有子目录（通过 `add_subdirectory` 进入）可见。
- **函数作用域**：在 `function()` 块中定义的变量仅在函数内可见。

如果你想在子作用域中修改父作用域的变量，需要使用 `PARENT_SCOPE` 关键字：

```cmake
function(increment_version)
    set(MY_PROJECT_VERSION "1.0.1" PARENT_SCOPE)
endfunction()

increment_version()
# 此时 MY_PROJECT_VERSION 变为了 1.0.1
```

### 2. 缓存变量（Cache Variable）

普通变量在每次运行 `cmake` 配置时都会重置，而 **缓存变量** 则会持久化存储在构建目录下的 `CMakeCache.txt` 文件中。

#### 为什么需要缓存变量？
有些配置我们需要在多次构建之间保持不变，或者希望用户从命令行手动调整。

#### 定义缓存变量
```cmake
set(MY_CACHE_VAR "default_value" CACHE STRING "这是一个描述信息")
```

当你第一次运行 `cmake ..` 时，这个值被写入缓存。之后你再次运行，它会优先读取缓存里的值，而不是重新设置。

#### 命令行覆盖
你可以使用 `-D` 参数在命令行修改缓存变量：
```bash
cmake -DMY_CACHE_VAR="new_value" ..
```
这种方式是控制构建行为（如开启/关闭某个功能）的标准做法。

### 3. option() 命令

`option()` 是专门为 **布尔型（ON/OFF）** 缓存变量准备的快捷方式。它通常用于控制功能开关。

在 KrKr2 项目中，你会经常看到类似的用法：
```cmake
option(ENABLE_TESTS "是否启用单元测试" ON)

if(ENABLE_TESTS)
    add_subdirectory(tests)
endif()
```
### 4. 环境变量（Environment Variable）

CMake 也可以读取系统的环境变量。

- **读取**：使用 `$ENV{NAME}` 引用环境变量。
- **设置**：使用 `set(ENV{VAR} "value")`。
  
注意：CMake 对环境变量的修改是临时性的，仅对当前的构建过程生效，不会改动操作系统的环境变量。

```cmake
# 打印环境变量 PATH
message(STATUS "系统 PATH 为: $ENV{PATH}")

# 设置一个临时环境变量
set(ENV{MY_TEMP_VAR} "123")
```

### 5. 常用内置变量

CMake 预设了许多内置变量。这些变量由 CMake 自动填充，在构建项目时非常有用。

| 变量名 | 含义 |
| --- | --- |
| `CMAKE_SOURCE_DIR` | 项目根目录（`project()` 所在目录）的绝对路径 |
| `CMAKE_BINARY_DIR` | 构建根目录（一般是 `build/`）的绝对路径 |
| `CMAKE_CURRENT_SOURCE_DIR` | 当前正在处理的 `CMakeLists.txt` 所在目录 |
| `CMAKE_CURRENT_BINARY_DIR` | 当前正在处理的 `CMakeLists.txt` 对应的构建目录 |
| `PROJECT_NAME` | 最近一次 `project()` 命令定义的项目名称 |
| `CMAKE_CXX_STANDARD` | 设置 C++ 标准（如 11, 14, 17, 20） |
| `CMAKE_BUILD_TYPE` | 构建类型（Debug, Release, RelWithDebInfo, MinSizeRel） |

### 6. 变量作用域与 PARENT_SCOPE

在 CMake 中，函数（`function`）会创建自己的局部作用域。如果你在函数内修改了变量，这个修改默认不会影响到函数外部。

```cmake
set(MY_VAR "Original")

function(change_var)
    set(MY_VAR "NewValue") # 仅在函数内生效
endfunction()

change_var()
message(STATUS "MY_VAR 依然是: ${MY_VAR}") # 打印 Original
```

如果你希望在函数内修改并让外部生效，需要使用 `PARENT_SCOPE`：

```cmake
function(change_var_for_real)
    set(MY_VAR "NewValue" PARENT_SCOPE) # 将值传递给上一层作用域
endfunction()

change_var_for_real()
message(STATUS "MY_VAR 现在是: ${MY_VAR}") # 打印 NewValue
```

### 7. 动手实践：管理源文件与编译选项

让我们创建一个完整的 `CMakeLists.txt` 示例：

```cmake
cmake_minimum_required(VERSION 3.10)
project(VariableDemo LANGUAGES CXX)

# 1. 设置 C++ 标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 2. 定义源文件列表变量
set(SRC_LIST main.cpp utils.cpp)

# 3. 用户可配置的开关
option(ENABLE_DEBUG_LOG "是否开启调试日志" OFF)

if(ENABLE_DEBUG_LOG)
    # 添加预处理定义
    add_definitions(-DDEBUG_LOG_ENABLED)
endif()

# 4. 使用变量创建可执行文件
add_executable(${PROJECT_NAME} ${SRC_LIST})

# 5. 打印一些关键路径信息
message(STATUS "------------------------------------")
message(STATUS "项目名称: ${PROJECT_NAME}")
message(STATUS "项目源目录: ${CMAKE_SOURCE_DIR}")
message(STATUS "当前构建目录: ${CMAKE_CURRENT_BINARY_DIR}")
message(STATUS "C++ 标准: ${CMAKE_CXX_STANDARD}")
message(STATUS "------------------------------------")
```

### 8. `set()` 命令完整语法速查
`set()` 是本节最核心命令。它既能设置普通变量，也能写缓存变量，还能改当前 CMake 进程内的环境变量。
```cmake
# 形式1：设置普通变量（目录或函数作用域）
set(VAR_NAME "value")

# 形式2：设置列表（多个值会被存成分号分隔字符串）
set(SRC_FILES main.cpp utils.cpp logger.cpp)

# 形式3：清空变量（等价于 unset(VAR_NAME)）
set(VAR_NAME)

# 形式4：设置缓存变量（写入 CMakeCache.txt）
set(ENABLE_DEMO ON CACHE BOOL "是否启用演示模块")
set(LIB_BACKEND "ffmpeg" CACHE STRING "媒体后端类型")
set(RETRY_COUNT 3 CACHE STRING "重试次数")

# 形式5：强制覆盖缓存变量（慎用）
set(ENABLE_DEMO OFF CACHE BOOL "是否启用演示模块" FORCE)

# 形式6：设置环境变量（仅当前 CMake 配置进程有效）
set(ENV{MY_TEMP_KEY} "hello")

# 形式7：从函数向父作用域回传变量
set(OUT_VALUE "done" PARENT_SCOPE)
```

`CACHE` 子句中的 `<type>` 常见值有 `BOOL`、`STRING`、`PATH`、`FILEPATH`、`INTERNAL`。其中 `INTERNAL` 变量默认不会在 GUI 中展示，适合内部状态。

### 9. 普通变量 / 缓存变量 / 环境变量生命周期对比
| 维度 | 普通变量 | 缓存变量 | 环境变量（CMake 内） |
| --- | --- | --- | --- |
| 定义方式 | `set(VAR value)` | `set(VAR value CACHE TYPE "doc")` / `option()` | `set(ENV{VAR} value)` |
| 生命周期 | 本次配置过程 | 跨多次配置保留 | 本次配置过程 |
| 存储位置 | 内存 | `build/CMakeCache.txt` | CMake 进程环境 |
| 适用场景 | 中间计算值 | 用户可调参数 | 外部工具调用上下文 |

### 10. 变量作用域：目录、函数、`PARENT_SCOPE`
- **目录作用域**：父目录变量对子目录可见；子目录修改默认不回写父目录。
- **函数作用域**：`function()` 内变量默认局部。
- **回写父层**：用 `set(VAR value PARENT_SCOPE)`。
```cmake
set(CONFIG_READY OFF)
function(mark_ready)
    set(CONFIG_READY ON PARENT_SCOPE) # 回传到调用者作用域
endfunction()
mark_ready()
message(STATUS "CONFIG_READY=${CONFIG_READY}")
```

### 11. `CMakeCache.txt` 文件结构与手动编辑
缓存文件位于构建目录，核心格式是 `变量名:类型=值`：

```text
ENABLE_TESTS:BOOL=ON
BUILD_TOOLS:BOOL=OFF
CMAKE_INSTALL_PREFIX:PATH=C:/Program Files/krkr2
```

手动编辑建议：
1. 先停止 IDE 的自动配置。
2. 只改项目开关类变量，不改编译器探测变量。
3. 改完立刻重新执行一次 `cmake -S . -B build` 验证。

### 12. `option()` 与布尔缓存变量
```cmake
option(ENABLE_PROFILER "是否启用性能分析" OFF)
if(ENABLE_PROFILER)
    add_compile_definitions(ENABLE_PROFILER=1)
endif()
```

在 KrKr2 中，`ENABLE_TESTS` 与 `BUILD_TOOLS` 就是通过 `option()` 控制 `add_subdirectory(tests/tools)` 的典型案例。

### 13. 列表变量操作：`list()` 命令
```cmake
set(PLUGIN_LIST psdfile psbfile motionplayer)
list(APPEND PLUGIN_LIST fstat)
list(REMOVE_ITEM PLUGIN_LIST motionplayer)
list(LENGTH PLUGIN_LIST PLUGIN_COUNT)
message(STATUS "插件数量=${PLUGIN_COUNT}")

```

### 14. 字符串操作：`string()` 命令
```cmake
set(VER_MAJOR 1)
set(VER_MINOR 2)
set(VER_PATCH 3)
string(CONCAT APP_VERSION "${VER_MAJOR}" "." "${VER_MINOR}" "." "${VER_PATCH}")

message(STATUS "版本=${APP_VERSION}")
message(STATUS "大写名=${APP_NAME_UPPER}")
```

### 15. 变量展开规则与嵌套引用
```cmake
set(KEY "APP")
set(APP_NAME "KrKr2")
message(STATUS "名称=${APP_NAME}")
message(STATUS "嵌套名称=${${KEY}_NAME}")
message(STATUS "未定义='${NOT_DEFINED_VAR}'")
```

说明：未定义变量会展开为空字符串；嵌套引用可实现“动态变量名”，但要控制使用频率，避免可读性下降。

## 常见错误及解决方案

### 错误 1：缓存变量"改不动"

**现象：** 在 `CMakeLists.txt` 中修改了 `set(MY_VAR "new" CACHE STRING "...")` 的默认值，但重新配置后值没变。

**原因：** 缓存变量只在**首次配置**时写入 `CMakeCache.txt`，之后不会被 CMakeLists 中的默认值覆盖。这是设计行为——缓存变量本意是让用户的手动配置持久化。

**解决：**
```bash
# 方法 1：命令行强制覆盖
cmake -DMY_VAR="new_value" -S . -B build

# 方法 2：删除缓存重新配置
rm build/CMakeCache.txt
cmake -S . -B build

# 方法 3：在 CMakeLists 中使用 FORCE（慎用，会忽略用户设置）
set(MY_VAR "new" CACHE STRING "描述" FORCE)
```

### 错误 2：`PARENT_SCOPE` 在当前作用域不生效

**现象：** 函数内写了 `set(VAR "value" PARENT_SCOPE)` 后，紧接着在函数内读取 `${VAR}`，发现还是旧值。

**原因：** `PARENT_SCOPE` 只修改**父作用域**的变量，不影响当前作用域。函数内 `${VAR}` 依然是进入函数时的值。

**解决：**
```cmake
function(update_version)
    set(VERSION "2.0" PARENT_SCOPE)  # 修改父作用域
    set(VERSION "2.0")               # 同时修改当前作用域（如需在函数内使用）
    message(STATUS "函数内 VERSION=${VERSION}")  # 现在是 2.0
endfunction()
```

### 错误 3：列表变量被引号"吃掉"分隔符

**现象：** `set(SRCS "a.cpp;b.cpp;c.cpp")` 后 `add_executable(app ${SRCS})` 只认到一个源文件。

**原因：** 用引号包裹的列表会被当作**单个字符串**。CMake 列表本质是分号分隔的字符串，但在引号内分号不会触发列表展开。

**解决：**
```cmake
# 正确：不加引号，让 CMake 按列表处理
set(SRCS a.cpp b.cpp c.cpp)
add_executable(app ${SRCS})

# 或者显式用分号（不加外层引号）
set(SRCS a.cpp;b.cpp;c.cpp)
```

## 本节小结
1. **普通变量** 使用 `set()` 定义，`${}` 引用。它们遵循目录和函数作用域。
2. **缓存变量** 持久化在 `CMakeCache.txt` 中，是跨平台配置和用户交互的核心。
3. **环境变量** 通过 `$ENV{}` 访问。
4. **内置变量** 如 `CMAKE_SOURCE_DIR` 让你能准确定位项目路径。
5. **`option()`** 提供简单的布尔开关控制。

### 题目 1：
定义一个名为 `SRCS` 的列表变量，包含 `a.cpp` 和 `b.cpp`。然后使用 `list(APPEND ...)` 向其中追加一个 `c.cpp`。
<details><summary>查看答案</summary>


### 题目 2：
在多配置生成器（如 Windows 上的 Visual Studio）中，`CMAKE_BUILD_TYPE` 变量通常是无效的。但对于单配置生成器（如 Ninja 或 Makefile），如何通过命令行将项目设置为 `Release` 模式？
<details><summary>查看答案</summary>
```bash
cmake -DCMAKE_BUILD_TYPE=Release ..
```
</details>

### 题目 3：
编写一个函数 `set_global_config`，在函数内部将变量 `GLOBAL_CONFIG_READY` 设置为 `ON`，并确保函数调用结束后，外部作用域也能读取到这个值。
<details><summary>查看答案</summary>
```cmake
function(set_global_config)
    set(GLOBAL_CONFIG_READY ON PARENT_SCOPE)
endfunction()

set_global_config()
if(GLOBAL_CONFIG_READY)
    message(STATUS "配置已就绪")
endif()
```
</details>

### 题目 4：
写一段配置代码：定义缓存变量 `USE_SYSTEM_FFMPEG`（默认 `OFF`），如果为 `ON` 就打印“使用系统 FFmpeg”，否则打印“使用项目内依赖”。
<details><summary>查看答案</summary>
```cmake
set(USE_SYSTEM_FFMPEG OFF CACHE BOOL "是否使用系统 FFmpeg")

if(USE_SYSTEM_FFMPEG)
    message(STATUS "使用系统 FFmpeg")
else()
    message(STATUS "使用项目内依赖")
endif()
```
</details>
