## 正文内容

### 1. 生成器表达式是什么

生成器表达式（Generator Expression）是 CMake 的一套动态求值语法，外形是 `$<...>`。

它的关键特点不是“写法炫”，而是“求值时机不同”：

- `if()`、`set()` 一般在**配置阶段（configure）**执行。
- `$<...>` 在**生成阶段（generate）**展开。

这意味着你可以把“规则”写在配置文件里，让规则在真正生成工程或构建时再决定具体值。

### 2. 延迟求值为什么是核心

在单配置生成器（常见 Ninja）里，`CMAKE_BUILD_TYPE` 常在配置时就确定。
但在多配置生成器（Visual Studio、Xcode）里，Debug/Release 是构建时切换的。

如果你把“按配置加宏”写成 `if(CMAKE_BUILD_TYPE STREQUAL ...)`，在多配置下往往不稳定。
这正是生成器表达式存在的原因：在最终目标参数处按配置展开。

### 3. 基础语法速览

常见形式有三种：

1. `$<$<条件>:值>`：条件成立时输出值，否则空字符串。
2. `$<IF:条件,真值,假值>`：三目分支。
3. `$<AND:条件1,条件2>`、`$<OR:条件1,条件2>`、`$<NOT:条件>`：逻辑组合。

示例1：按 Debug/Release 写不同日志级别宏（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo01 LANGUAGES CXX)

add_executable(genexp_demo01 main.cpp)

# Debug 下为 3，其他配置为 1
target_compile_definitions(genexp_demo01 PRIVATE
    "KRKR2_LOG_LEVEL=$<IF:$<CONFIG:Debug>,3,1>"
)
```

### 4. 三大类表达式详解

#### 4.1 条件表达式 `$<IF:...>`

当你需要“明确的二选一结果”时，优先用 `$<IF:条件,真值,假值>`。
它比“条件成立才追加某片段”的写法更直观，尤其适合定义宏值、输出路径片段。

示例2：按配置切换资源目录名（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo02 LANGUAGES CXX)

add_executable(genexp_demo02 main.cpp)

target_compile_definitions(genexp_demo02 PRIVATE
    "KRKR2_RES_DIR=\"$<IF:$<CONFIG:Debug>,res_debug,res_release>\""
)
```

#### 4.2 信息查询 `$<TARGET_FILE:...>`

信息查询类表达式用于读取目标最终产物信息，例如：

- `$<TARGET_FILE:目标名>`：目标完整路径。
- `$<TARGET_FILE_DIR:目标名>`：目标目录。

示例3：构建后打印程序绝对路径（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo03 LANGUAGES CXX)

add_executable(genexp_demo03 main.cpp)

add_custom_command(TARGET genexp_demo03 POST_BUILD
    COMMAND ${CMAKE_COMMAND} -E echo "产物路径: $<TARGET_FILE:genexp_demo03>"
    COMMAND ${CMAKE_COMMAND} -E echo "产物目录: $<TARGET_FILE_DIR:genexp_demo03>"
)
```

#### 4.3 逻辑组合 `$<AND:...>`

逻辑表达式负责拼接复杂条件。
典型模式是“语言 + 编译器 + 配置”三条件组合。

示例4：仅在 C++ 且 MSVC 且 Debug 时启用一组选项（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo04 LANGUAGES C CXX)

add_executable(genexp_demo04 main.cpp helper.c)

target_compile_options(genexp_demo04 PRIVATE
    "$<$<AND:$<COMPILE_LANGUAGE:CXX>,$<CXX_COMPILER_ID:MSVC>,$<CONFIG:Debug>>:/W4;/Od;/utf-8>"
    "$<$<AND:$<COMPILE_LANGUAGE:CXX>,$<NOT:$<CXX_COMPILER_ID:MSVC>>>:-Wall;-Wextra;-g>"
)
```

### 5. 常用表达式清单与示例

下面这 5 组是日常最高频：

#### 5.1 `$<CONFIG:...>`

```cmake
target_compile_definitions(my_target PRIVATE
    $<$<CONFIG:Debug>:DEBUG_MODE=1>
    $<$<NOT:$<CONFIG:Debug>>:NDEBUG>
)
```

#### 5.2 `$<PLATFORM_ID:...>`

```cmake
target_link_libraries(my_target PRIVATE
    $<$<PLATFORM_ID:Windows>:ws2_32>
    $<$<PLATFORM_ID:Linux>:pthread>
    $<$<PLATFORM_ID:Darwin>:-framework CoreFoundation>
    $<$<PLATFORM_ID:Android>:log>
)
```

#### 5.3 `$<CXX_COMPILER_ID:...>`

```cmake
target_compile_options(my_target PRIVATE
    "$<$<CXX_COMPILER_ID:MSVC>:/W4;/permissive->"
    "$<$<CXX_COMPILER_ID:GNU>:-Wall;-Wextra;-Wconversion>"
    "$<$<CXX_COMPILER_ID:Clang>:-Wall;-Wextra;-Wshadow>"
)
```

#### 5.4 `$<TARGET_PROPERTY:...>`

```cmake
add_library(core STATIC core.cpp)
set_property(TARGET core PROPERTY KRKR2_COMPONENT_NAME "visual")

add_executable(app main.cpp)
target_compile_definitions(app PRIVATE
    "COMPONENT_NAME=\"$<TARGET_PROPERTY:core,KRKR2_COMPONENT_NAME>\""
)
target_link_libraries(app PRIVATE core)
```

### 6. 嵌套表达式写法

嵌套表达式可以很强，也很容易失控。
建议遵循三条规则：

1. 先拆条件，再组合。
2. 含分号的参数必须加引号。
3. 一行只表达一个意图。

示例5：将嵌套表达式拆成中间变量（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo05 LANGUAGES CXX)

add_library(genexp_demo05 STATIC demo.cpp)

set(IS_MSVC "$<CXX_COMPILER_ID:MSVC>")
set(IS_DEBUG "$<CONFIG:Debug>")
set(IS_MSVC_DEBUG "$<AND:${IS_MSVC},${IS_DEBUG}>")

target_compile_options(genexp_demo05 PRIVATE
    "$<${IS_MSVC_DEBUG}:/W4;/Od>"
    "$<$<AND:${IS_MSVC},$<NOT:${IS_DEBUG}>>:/O2>"
    "$<$<AND:$<NOT:${IS_MSVC}>,${IS_DEBUG}>:-O0;-g3;-Wall>"
    "$<$<AND:$<NOT:${IS_MSVC}>,$<NOT:${IS_DEBUG}>>:-O2>"
)
```

### 7. 典型业务场景

- 按编译器设置警告策略：MSVC 与 GCC/Clang 选项名不同，用表达式在同一目标分发。
- 按平台链接系统库：Windows `ws2_32`、Linux `pthread`、macOS framework、Android `log`。
- 按构建类型切宏：Debug 打开跟踪宏，Release 定义 `NDEBUG`。

### 8. KrKr2 项目实例：把语法映射到真实工程

#### 8.1 `krkr2/CMakeLists.txt`
文件中可见如下表达式：

```cmake
add_compile_definitions(
        $<$<CONFIG:Debug>:DEBUG>
        $<$<CONFIG:Debug>:_DEBUG>
        $<$<NOT:$<CONFIG:Debug>>:NDEBUG>
)
```

这段位于文件前部（约 11-15 行），体现了“按配置定义宏”。同文件还存在：

```cmake
add_compile_options("$<$<COMPILE_LANGUAGE:CXX>:/EHsc>" "/MP" "/utf-8")
```

这段位于 MSVC 分支内（约 41 行），体现“仅 C++ 语言启用指定编译参数”。
#### 8.2 `krkr2/cmake/CocosBuildHelpers.cmake`
文件中可见：

```cmake
-m $<CONFIG>
```

这段在资源脚本调用位置（约 55 行），用于把当前配置名传给 Python 脚本。同文件还可见：

```cmake
"$<TARGET_FILE_DIR:${cocos_target}>/${cc_dll_name}"
```

这段在 DLL 拷贝逻辑（约 172 行），用于定位目标输出目录。
### 9. 调试生成器表达式

#### 9.1 使用 `file(GENERATE)` 输出展开结果
示例6：将关键上下文写入报告文件（可运行）

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_demo06 LANGUAGES CXX)

add_executable(genexp_demo06 main.cpp)

file(GENERATE
    OUTPUT "${CMAKE_BINARY_DIR}/genexp-report-$<CONFIG>.txt"
    CONTENT "config=$<CONFIG>\nplatform=$<PLATFORM_ID>\ncompiler=$<CXX_COMPILER_ID>\nexe=$<TARGET_FILE:genexp_demo06>\n"
)
```

#### 9.2 使用 `cmake --trace` 与 `--trace-expand`

```bash
cmake -S . -B out/debug -G Ninja -DCMAKE_BUILD_TYPE=Debug --trace --trace-expand
```

你会看到每条 CMake 命令的执行轨迹，适合排查条件命中与变量拼接问题。

### 10. 常见错误清单

1. 在 `if()` 中直接写 `$<...>` 并期望按生成器表达式求值。  
2. 在 VS/Xcode 下依赖 `CMAKE_BUILD_TYPE` 做核心分支。  
3. 编译选项里写了分号但忘记引号，导致参数被拆断。  
4. 嵌套过深不拆变量，后期无法维护。

## 动手实践

目标：做一个最小工程，验证“编译器 + 平台 + 构建类型”三维差异。

### 步骤1：创建文件并写入 `CMakeLists.txt`

```cmake
cmake_minimum_required(VERSION 3.28)
project(genexp_lab LANGUAGES CXX)

add_executable(genexp_lab main.cpp)

target_compile_definitions(genexp_lab PRIVATE
    "KRKR2_LOG_LEVEL=$<IF:$<CONFIG:Debug>,3,1>"
    $<$<PLATFORM_ID:Windows>:KRKR2_OS_WINDOWS=1>
    $<$<PLATFORM_ID:Linux>:KRKR2_OS_LINUX=1>
    $<$<PLATFORM_ID:Darwin>:KRKR2_OS_MACOS=1>
    $<$<PLATFORM_ID:Android>:KRKR2_OS_ANDROID=1>
)

target_compile_options(genexp_lab PRIVATE
    "$<$<CXX_COMPILER_ID:MSVC>:/W4;/utf-8>"
    "$<$<NOT:$<CXX_COMPILER_ID:MSVC>>:-Wall;-Wextra>"
)

target_link_libraries(genexp_lab PRIVATE
    $<$<PLATFORM_ID:Windows>:ws2_32>
    $<$<PLATFORM_ID:Linux>:pthread>
    $<$<PLATFORM_ID:Darwin>:-framework CoreFoundation>
    $<$<PLATFORM_ID:Android>:log>
)

file(GENERATE
    OUTPUT "${CMAKE_BINARY_DIR}/lab-$<CONFIG>.txt"
    CONTENT "cfg=$<CONFIG>\nplatform=$<PLATFORM_ID>\ncompiler=$<CXX_COMPILER_ID>\n"
)
```

### 步骤2：写入 `main.cpp`

```cpp
#include <iostream>

int main() {
    std::cout << "KRKR2_LOG_LEVEL=" << KRKR2_LOG_LEVEL << "\n";
    std::cout << "生成器表达式实验成功" << "\n";
    return 0;
}
```

### 步骤3：执行构建

```bash
# Windows / Linux / macOS（Ninja）
cmake -S . -B out/debug -G Ninja -DCMAKE_BUILD_TYPE=Debug
cmake --build out/debug

# Android（需 NDK）
cmake -S . -B out/android-debug -G Ninja -DANDROID=ON -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK/build/cmake/android.toolchain.cmake -DANDROID_ABI=arm64-v8a -DANDROID_PLATFORM=android-24
cmake --build out/android-debug
```

## 本节小结

- 生成器表达式的本质是“生成阶段求值”，解决多配置场景最有效。
- 三大类表达式要会用：`$<IF:...>`、`$<TARGET_FILE:...>`、`$<AND:...>`。
- 高频表达式要熟：`$<CONFIG:...>`、`$<PLATFORM_ID:...>`、`$<CXX_COMPILER_ID:...>`、`$<TARGET_PROPERTY:...>`。
- 写复杂表达式时优先拆变量，调试优先用 `file(GENERATE)`。
- KrKr2 根 CMake 与 CocosBuildHelpers 已给出可直接参考的工程级实践。
