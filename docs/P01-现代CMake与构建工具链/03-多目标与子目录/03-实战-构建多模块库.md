# 03-实战-构建多模块库

> **所属模块：** P01-现代CMake与构建工具链
> **前置知识：** 02-INTERFACE与PUBLIC与PRIVATE
> **预计阅读时间：** 35 分钟

## 本节目标
通过前面的学习，你已经掌握了目录组织和属性传播的核心理论。现在，我们将从零开始构建一个名为 `MiniEngine` 的多模块项目。这个项目的结构将高度模拟 KrKr2 的架构，帮助你理解大型工程是如何被组织起来的。

## 项目架构设计
我们将构建以下模块：
- **`engine/`**：聚合库（类似 `krkr2core`），类型为 `INTERFACE`。
- **`engine/math/`**：数学库，类型为 `STATIC`。
- **`engine/render/`**：渲染桩库（模拟渲染逻辑），类型为 `STATIC`。
- **`plugins/`**：插件库（类似 `krkr2plugin`），依赖 `engine`。
- **`app/`**：最终的可执行文件。

## 完整项目代码

### 1. 根目录配置
**文件：`CMakeLists.txt`**
```cmake
cmake_minimum_required(VERSION 3.10)
project(MiniEngineProject)

# 设置 C++ 标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 添加子目录
add_subdirectory(engine)
add_subdirectory(plugins)
add_subdirectory(app)
```

### 2. 引擎模块配置
**文件：`engine/CMakeLists.txt`**
```cmake
# 这是一个聚合库，不包含源码，只负责链接子模块
add_library(engine INTERFACE)

add_subdirectory(math)
add_subdirectory(render)

# 将子模块链接到聚合库
target_link_libraries(engine INTERFACE engine_math engine_render)
```

**文件：`engine/math/CMakeLists.txt`**
```cmake
add_library(engine_math STATIC math.cpp)
# PUBLIC 包含路径，确保链接 engine_math 的目标能找到头文件
target_include_directories(engine_math PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})
```

**文件：`engine/math/math.h`**
```cpp
#pragma once
namespace math {
    int square(int x);
}
```

**文件：`engine/math/math.cpp`**
```cpp
#include "math.h"
namespace math {
    int square(int x) { return x * x; }
}
```

**文件：`engine/render/CMakeLists.txt`**
```cmake
add_library(engine_render STATIC render.cpp)
target_include_directories(engine_render PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})
# 渲染模块内部可能需要数学库
target_link_libraries(engine_render PRIVATE engine_math)
```

**文件：`engine/render/render.h`**
```cpp
#pragma once
namespace render {
    void draw_frame();
}
```

**文件：`engine/render/render.cpp`**
```cpp
#include "render.h"
#include <iostream>
namespace render {
    void draw_frame() {
        std::cout << "[Render] Drawing a new frame..." << std::endl;
    }
}
```

### 3. 插件模块配置
**文件：`plugins/CMakeLists.txt`**
```cmake
# 插件模块依赖于引擎
add_library(mini_plugin STATIC plugin.cpp)
target_include_directories(mini_plugin PUBLIC ${CMAKE_CURRENT_SOURCE_DIR})

# 链接 engine 聚合库
target_link_libraries(mini_plugin PUBLIC engine)
```

**文件：`plugins/plugin.h`**
```cpp
#pragma once
namespace plugins {
    void run_plugin_logic();
}
```

**文件：`plugins/plugin.cpp`**
```cpp
#include "plugin.h"
#include "math.h"
#include <iostream>

namespace plugins {
    void run_plugin_logic() {
        std::cout << "[Plugin] Running plugin logic..." << std::endl;
        std::cout << "[Plugin] Calling engine math: square(10) = " << math::square(10) << std::endl;
    }
}
```

### 4. 应用程序配置
**文件：`app/CMakeLists.txt`**
```cmake
add_executable(main_app main.cpp)

# 只需要链接插件和引擎聚合库即可
target_link_libraries(main_app PRIVATE mini_plugin engine)
```

**文件：`app/main.cpp`**
```cpp
#include "render.h"
#include "plugin.h"
#include <iostream>

int main() {
    std::cout << "Starting MiniEngine App..." << std::endl;
    
    // 调用引擎的渲染功能
    render::draw_frame();
    
    // 调用插件逻辑
    plugins::run_plugin_logic();
    
    return 0;
}
```

## 构建验证（多平台）
在所有主流平台上，构建流程基本一致：

### 1. Windows (Visual Studio / Ninja)
```powershell
mkdir build
cd build
cmake .. -G "Visual Studio 17 2022" -A x64
cmake --build . --config Release
.\app\Release\main_app.exe
```

### 2. Linux / macOS
```bash
mkdir build
cd build
cmake .. -G Ninja
cmake --build .
./app/main_app
```

### 3. Android
通常通过 Android Studio 自动调用 CMake。如果是手动通过 NDK 构建：
```bash
cmake -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK/build/cmake/android.toolchain.cmake \
      -DANDROID_ABI=arm64-v8a \
      -DANDROID_PLATFORM=android-21 \
      ..
```

## 对照 KrKr2 架构
| 模块类型 | MiniEngine | KrKr2 对应 | 作用 |
| :--- | :--- | :--- | :--- |
| **聚合库 (INTERFACE)** | `engine` | `krkr2core` | 简化依赖，统一暴露 |
| **功能子库 (STATIC)** | `engine_math` | `tjs2`, `visual` | 核心逻辑分拆 |
| **扩展层 (STATIC)** | `mini_plugin` | `krkr2plugin` | 封装可插拔的功能模块 |
| **应用层 (Executable)** | `main_app` | `krkr2` (Win/Linux/Mac) | 系统入口 |

## 本节小结
1. 使用 `INTERFACE` 库作为聚合器，是现代大型 CMake 项目的标配。
2. 合理的目录划分让代码逻辑清晰。
3. `target_link_libraries` 能够自动处理依赖关系和头文件搜索路径。

## 练习题与答案
### 题目 1：依赖链分析
在上述项目中，如果 `engine/math` 改名为 `engine_math_v2`，有哪些文件需要修改？
<details><summary>查看答案</summary>
1. `engine/math/CMakeLists.txt` 中的 `add_library`。
2. `engine/CMakeLists.txt` 中的 `target_link_libraries`。
3. `engine/render/CMakeLists.txt` 中的 `target_link_libraries`。
</details>

### 题目 2：新增功能
如果我们要为 `MiniEngine` 增加一个音频模块 `engine/audio`，并让 `app` 能够直接调用它，需要哪些步骤？
<details><summary>查看答案</summary>
1. 在 `engine/` 下创建 `audio/` 文件夹及源文件、`CMakeLists.txt`。
2. 在 `engine/CMakeLists.txt` 中添加 `add_subdirectory(audio)`。
3. 将 `engine_audio` 库链接到 `engine` INTERFACE 库中。
</details>
