# 01-安装与配置

## vcpkg 是什么

vcpkg 是由微软（Microsoft）开发的开源 C++ 包管理器。它能在 Windows、Linux 和 macOS 等多个平台上运行，帮助开发者轻松获取并管理 C++ 第三方库。

如果你写过 Java 或 Python，你可能熟悉 Maven、Gradle 或 pip。在 C++ 的世界里，长期以来一直缺乏这种统一的包管理工具，导致开发者往往需要手动下载源代码、配置编译选项、解决繁杂的头文件和链接库路径。vcpkg 的出现正是为了解决这些痛点。

## 为什么需要包管理器

在没有包管理器的情况下，如果你想在项目中使用一个第三方库（例如 `nlohmann-json`），你通常需要经历以下步骤：

1. **手动下载**：去 GitHub 搜索并下载对应版本的源码。
2. **环境配置**：确保你的机器上有该库所需的特定编译器版本和构建工具（如 CMake）。
3. **编译安装**：手动运行构建命令，生成静态库（.lib/.a）或动态库（.dll/.so）。
4. **路径管理**：在你的 IDE 或构建脚本中手动添加头文件包含路径和库文件路径。
5. **依赖地狱**：如果该库还依赖于其他库，你需要对每一个依赖库重复上述过程。

vcpkg 将这些繁琐的操作自动化。只需一行命令，它就能为你下载、编译并配置好一切，让你可以专注于业务代码的编写。

## 安装 vcpkg

我们将分别介绍在 Windows、Linux、macOS 上的安装步骤，以及在 Android 平台下的使用建议。

### Windows 平台

在 Windows 上，建议使用 Git 命令行工具。

1. **克隆仓库**：
   找一个你喜欢的目录（例如 `C:\dev`），打开终端运行：
   ```powershell
   git clone https://github.com/microsoft/vcpkg.git
   ```

2. **运行引导脚本**：
   进入 `vcpkg` 目录并执行：
   ```powershell
   .\vcpkg\bootstrap-vcpkg.bat
   ```
   这会下载 vcpkg 自身的可执行文件。

### Linux 平台

在 Linux 上，你需要先安装基本的开发工具（如 `g++`, `make`, `curl`, `zip`, `unzip`, `tar`）。

1. **安装依赖**（以 Ubuntu/Debian 为例）：
   ```bash
   sudo apt-get update
   sudo apt-get install build-essential tar curl zip unzip pkg-config
   ```

2. **克隆与编译**：
   ```bash
   git clone https://github.com/microsoft/vcpkg.git
   cd vcpkg
   ./bootstrap-vcpkg.sh
   ```

### macOS 平台

macOS 用户需要安装 Xcode 命令行工具。

1. **准备环境**：
   ```bash
   xcode-select --install
   ```

2. **克隆与编译**：
   ```bash
   git clone https://github.com/microsoft/vcpkg.git
   cd vcpkg
   ./bootstrap-vcpkg.sh
   ```

### Android 平台

Android 开发通常不会直接在 Android 设备上安装 vcpkg 客户端。相反，我们是在 Windows/Linux/macOS 开发机上使用 vcpkg 来交叉编译出适用于 Android 的库。

在 CI（持续集成）流程中，你会通过配置 `VCPKG_CHAINLOAD_TOOLCHAIN_FILE` 指向 Android NDK 的工具链文件来完成构建。在后续教程中，我们会详细讨论如何针对 Android 目标进行编译。

## 设置 VCPKG_ROOT 环境变量

为了让开发工具（如 CMake 或你的 shell）能够找到 vcpkg，你需要设置环境变量 `VCPKG_ROOT`。

### Windows 平台

1. **设置**：
   在 PowerShell 中运行（仅当前会话有效）：
   ```powershell
   $env:VCPKG_ROOT = "C:\path\to\vcpkg"
   ```
   **永久生效**：搜索“编辑系统环境变量” -> “环境变量” -> “新建用户变量”，变量名为 `VCPKG_ROOT`，值为你的 vcpkg 所在路径。

### Linux / macOS 平台

在你的 `~/.bashrc` 或 `~/.zshrc` 文件中添加以下行：
```bash
export VCPKG_ROOT=$HOME/vcpkg
export PATH=$VCPKG_ROOT:$PATH
```
添加完成后运行 `source ~/.bashrc` 使其立即生效。

### Android 平台（CI 建议）

在持续集成（如 GitHub Actions）中，通常通过环境变量指定 vcpkg 路径：
```yaml
env:
  VCPKG_ROOT: ${{ github.workspace }}/vcpkg
```

## 验证安装

安装完成后，打开一个新的终端窗口并运行：
```bash
vcpkg version
```
如果你看到了类似 `vcpkg package management program version 2024-XX-XX` 的输出，恭喜你，安装成功！

## 工作模式简介：Classic 与 Manifest

vcpkg 提供了两种截然不同的工作模式，根据你的需求选择：

1. **经典模式 (Classic Mode)**：
   - 全局安装。你安装的所有库都存放在 vcpkg 的安装目录下。
   - 适合快速试用库，或者多个项目共享相同的库版本。
   - 命令示例：`vcpkg install zlib`

2. **清单模式 (Manifest Mode)**：
   - 项目级管理。类似于 npm 的 `package.json`。
   - 依赖信息记录在项目的 `vcpkg.json` 文件中。
   - 极力推荐。它能确保你的项目在不同机器上都有相同的库版本。

在接下来的教程中，我们将重点学习如何使用 **Manifest 模式**。

## 练习题与答案

### 题目 1：初识环境变量

在 Windows 系统上，如果不设置 `VCPKG_ROOT` 环境变量，CMake 能否通过 `find_package` 自动定位到 vcpkg 的库？为什么？

<details>
<summary>查看答案</summary>

不能。CMake 本身并不知道 vcpkg 的安装位置。为了让 CMake 找到 vcpkg 库，你通常需要指定 `-DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake`。如果不设置该环境变量，你就需要手动输入完整的长路径。

</details>

### 题目 2：多平台通用安装流程

请简述在任何桌面操作系统上安装 vcpkg 所必需的两个核心步骤。

<details>
<summary>查看答案</summary>

2. **构建二进制文件**：运行平台对应的引导脚本（Windows 下为 `bootstrap-vcpkg.bat`，Linux/macOS 下为 `bootstrap-vcpkg.sh`）以生成 vcpkg 可执行程序。
