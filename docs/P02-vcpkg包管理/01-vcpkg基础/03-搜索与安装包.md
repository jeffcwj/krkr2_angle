## 本节目标

读完本节后，你将能够：
1. 熟练使用 `vcpkg search` 搜索依赖包。
2. 使用 vcpkg.io 验证包信息、平台支持和 features。
3. 掌握 `vcpkg install` 常见完整语法。
4. 正确指定 triplet 与 feature 安装依赖。
5. 查看版本并理解版本选择策略。
6. 使用 `vcpkg depend-info` 查看依赖树。
7. 理解安装过程六阶段。
8. 在失败时通过 `buildtrees/` 日志排查问题。
9. 了解 KrKr2 在 `vcpkg.json` 中使用的全部第三方库。

## 一、搜索与安装的正确工作流
在工程里，搜索与安装是一个闭环。

推荐流程：
1. 搜索候选包。
2. 核对平台与 features。
3. 指定 triplet 安装。
4. 查看依赖树。
5. 失败时看日志。
6. Manifest 模式回写 `vcpkg.json`。

## 二、`vcpkg search` 命令详解与搜索技巧

### 2.1 基础语法

```bash
vcpkg search <关键词>
```

### 2.2 基础示例

Windows：
```powershell
vcpkg search sqlite
```

Linux/macOS：
```bash
vcpkg search json
```

### 2.3 技巧 1：先领域词，再具体库

```bash
vcpkg search image
vcpkg search audio
vcpkg search ffmpeg
vcpkg search libpng
```

### 2.4 技巧 2：拆分关键词

```bash
vcpkg search jpeg
vcpkg search turbo
vcpkg search libjpeg
```

该方法更容易定位 `libjpeg-turbo`。

### 2.5 技巧 3：先看描述

同类库可能用途不同。
先看输出描述可减少误装。

## 三、vcpkg 网站搜索（vcpkg.io）
命令行快查，网页细查。

### 3.1 入口

- `https://vcpkg.io`
- `https://vcpkg.io/en/packages`

### 3.2 重点字段

1. 包名。
2. 描述。
3. Features。
4. 支持平台。
5. 版本记录。

### 3.3 推荐顺序

1. `vcpkg search` 找候选。
2. vcpkg.io 核对 feature 与平台。
3. 回项目维护 `vcpkg.json`。

## 四、`vcpkg install` 命令完整语法

### 4.1 常见语法

```bash
vcpkg install <port>
vcpkg install <port>:<triplet>
vcpkg install <port>[feature1,feature2]:<triplet>
vcpkg install <port> --triplet=<triplet>
```

### 4.2 参数含义

- `<port>`：包名。
- `<triplet>`：平台、架构、链接策略。
- `[feature1,feature2]`：可选功能。

### 4.3 triplet 示例

```bash
vcpkg install zlib:x64-windows-static
vcpkg install zlib:x64-linux
vcpkg install zlib:x64-osx
vcpkg install zlib:arm64-android
```

### 4.4 feature 示例

```bash
vcpkg install opencv4[openmp]:x64-windows
```

## 五、Manifest 模式依赖声明
### 5.1 普通依赖

```json
{
  "dependencies": [
    "zlib",
    "sqlite3"
  ]
}
```

### 5.2 平台过滤

```json
{
  "dependencies": [
    {
      "name": "opencv4",
      "platform": "windows | osx"
    }
  ]
}
```

### 5.3 feature 控制

```json
{
  "dependencies": [
    {
      "name": "opencv4",
      "default-features": false,
      "features": [
        "openmp"
      ]
    }
  ]
}
```

### 5.4 host 依赖

```json
{
  "dependencies": [
    {
      "name": "vcpkg-cmake",
      "host": true
    }
  ]
}
```

## 六、包版本查看和选择

### 6.1 查看版本

```bash
vcpkg list
```

### 6.2 版本策略

建议锁版本的场景：
1. CI 需要可复现。
2. 新版本存在兼容风险。
3. 多平台结果需一致。

### 6.3 KrKr2 的 override 示例

```json
{
  "overrides": [
    {
      "name": "opencv4",
      "version": "4.7.0#6"
    }
  ]
}
```

## 七、依赖树查看：`vcpkg depend-info`

### 7.1 命令示例

```bash
vcpkg depend-info ffmpeg
vcpkg depend-info opencv4 --triplet=x64-windows
```

### 7.2 重点观察

1. 直接依赖数量。
2. 间接依赖深度。
3. 平台特有重依赖。
4. 不必要功能链路。

## 八、安装过程详解
### 8.1 download

下载源码、补丁、缓存。

### 8.2 extract

解压源码到构建目录。

### 8.3 patch

应用端口补丁。

### 8.4 configure

执行 CMake/Meson/Autotools 配置。

### 8.5 build

执行 Ninja/MSBuild/Make 编译。

### 8.6 install

安装到 `installed/<triplet>/`。

## 九、安装失败排查（buildtrees）

### 9.1 日志目录

`<vcpkg根目录>/buildtrees/<port>/`

### 9.2 常见日志

- `config-<triplet>-dbg-out.log`
- `config-<triplet>-dbg-err.log`
- `build-<triplet>-dbg-out.log`
- `build-<triplet>-dbg-err.log`
- `install-<triplet>-dbg-out.log`

### 9.3 排查顺序

1. 先判断失败阶段。
2. 先看 `*-err.log` 首条错误。
3. 再看 `*-out.log` 获取上下文。
4. 核对 triplet、feature、工具链。

## 十一、动手实践
### 步骤 1：搜索

```bash
vcpkg search webp
vcpkg search vorbis
```

### 步骤 2：安装

```bash
vcpkg install libwebp:x64-windows
vcpkg install libvorbis:x64-windows
```

### 步骤 3：依赖树

```bash
vcpkg depend-info libwebp --triplet=x64-windows
```

### 步骤 4：排障

若失败，先看 `buildtrees/libwebp/` 的 `*-err.log`。
再看 `*-out.log`。

### 步骤 5：回写 Manifest

把确认后的依赖写入 `vcpkg.json`。
重新执行 CMake configure。

## 十二、本节小结
- `vcpkg search` 解决包名确认。
- vcpkg.io 解决特性与平台确认。
- `vcpkg install` 需要正确组合 triplet 与 features。
- `vcpkg depend-info` 用于分析依赖复杂度。
- 失败排查优先看 `buildtrees/`。
- KrKr2 依赖覆盖图形、音频、视频、压缩、日志、测试。
