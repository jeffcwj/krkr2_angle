# 03-搜索与安装包

## 使用 `vcpkg search` 搜索包

在你打算为一个项目引入第三方库时，首要任务是确认 vcpkg 是否支持该库，以及该库在 vcpkg 中的正式名称。

### 常用命令语法

```bash
vcpkg search [库名关键字]
```

- **Windows 示例**：
  ```powershell
  vcpkg search sqlite
  ```
  输出通常包含：`sqlite3` (库名) 和对应的简短描述。

- **Linux / macOS 示例**：
  ```bash
  vcpkg search json
  ```
  你会看到一大串与 JSON 相关的库，如 `nlohmann-json`、`rapidjson` 等。

## 使用 `vcpkg install` 安装包 (Classic 模式)

在经典模式下（即没有 `vcpkg.json` 文件的情况下），你直接运行安装指令。

### 全局安装

```bash
vcpkg install [库名]
```
例如：
```bash
vcpkg install zlib
```
这会把 `zlib` 的头文件和库文件安装到你的 vcpkg 根目录下的 `packages` 和 `installed` 文件夹内。

### 指定平台架构 (Triplet)

vcpkg 使用 **Triplet** 来标识目标平台和架构。

- **Windows 64位静态链接**：
  ```bash
  vcpkg install zlib:x64-windows-static
  ```
- **macOS 64位**：
  ```bash
  vcpkg install zlib:x64-osx
  ```
- **Linux 64位**：
  ```bash
  vcpkg install zlib:x64-linux
  ```

## Manifest 模式下如何添加依赖

在实际项目开发（清单模式）中，你**不再运行 `vcpkg install` 命令**。

### 正确流程

1. 打开项目根目录下的 `vcpkg.json`。
2. 将你搜索到的库名添加到 `dependencies` 数组中。

```json
{
  "dependencies": [
    "zlib",
    "sqlite3"
  ]
}
```
3. 重新运行 CMake 配置。CMake 会自动检测到文件变动，并触发 vcpkg 进行增量安装。

## 依赖的平台过滤器 (Platform Filter)

并非所有的库都支持所有平台，或者有些库你只想在特定平台上使用。

### 示例用法

```json
{
  "dependencies": [
    {
      "name": "opencv",
      "platform": "windows | osx"
    },
    {
      "name": "linux-specific-lib",
      "platform": "linux"
    }
  ]
}
```
在这段配置中，`opencv` 只会在 Windows 或 macOS 上安装，而 `linux-specific-lib` 库仅在 Linux 平台上安装。

## 依赖的 Features 控制

许多库提供可选的功能模块。你可以选择关闭默认功能，并按需启用特定功能。

### 示例用法

```json
{
  "dependencies": [
    {
      "name": "curl",
      "default-features": false,
      "features": [
        "openssl",
        "ssh"
      ]
    }
  ]
}
```
通过以上配置，我们安装了一个最小化的 `curl` 库，手动启用了 OpenSSL 支持和 SSH 支持。

## Host 依赖

有时候你需要安装一些构建工具或代码生成器（例如 `protobuf` 编译器 `protoc`）。这些工具必须在你的**开发机**（宿主机，Host Machine）上运行。

### 示例用法

```json
{
  "dependencies": [
    {
      "name": "protobuf",
      "host": true
    }
  ]
}
```
设置 `"host": true` 后，vcpkg 会根据开发机的操作系统（Host Triplet）安装该工具，而不是针对最终的运行平台。

## 查看已安装的包

在日常开发中，查看当前系统已有的库可以帮助你避免重复劳动。

### 常用命令语法

```bash
vcpkg list
```
该命令会列出所有已安装的包及其版本。

- **Windows 示例**：
  ```powershell
  vcpkg list
  ```
- **Linux / macOS / Android 示例**：
  在终端直接输入 `vcpkg list` 即可查看跨平台已安装的库。

## 练习题与答案

### 题目 1：搜索与命名

在 vcpkg 中，大多数知名库的名称与它们在 GitHub 或官网上的名称一致，但有些库有特定的命名约定。如果你想在 Windows 上使用 SQLite 库，你应该搜索什么？

<details>
<summary>查看答案</summary>

你应该执行 `vcpkg search sqlite`。搜索结果会显示 SQLite 库在 vcpkg 中的包名为 `sqlite3`。

</details>

### 题目 2：功能裁剪 (Features)

在清单模式下，默认安装一个包会开启其所有的默认功能（Default Features）。如果你为了减小最终程序包的体积，想安装一个只有基础功能的 `libpng` 库，你应该在 `vcpkg.json` 中如何配置？

<details>
<summary>查看答案</summary>

你应该在 `dependencies` 列表中将 `libpng` 写成对象形式，并显式设置 `"default-features": false`。示例配置如下：

```json
{
  "name": "png-test",
  "version-string": "0.1.0",
  "dependencies": [
    {
      "name": "libpng",
      "default-features": false
    }
  ]
}
```

</details>

### 题目 3：Host 依赖场景

假设你在开发一个 Android 游戏，你需要安装 `flatbuffers` 库用于序列化数据。你既需要 Android 版本的运行库，也需要在开发机上运行 `flatc` 编译器来生成代码。请简述你应如何在 `vcpkg.json` 中定义依赖项。

<details>
<summary>查看答案</summary>

你需要在 `dependencies` 数组中添加两次 `flatbuffers`，一次作为普通依赖项（目标运行库），另一次作为 Host 依赖项（构建工具）：

```json
{
  "dependencies": [
    "flatbuffers",
    {
      "name": "flatbuffers",
      "host": true
    }
  ]
}
```

</details>
