## 字符编码差异

### 各平台默认编码

| 平台 | 文件系统编码 | 控制台编码 | `std::string` 通常编码 |
|------|------------|-----------|----------------------|
| Windows | UTF-16 (NTFS) | 系统区域设置（中文=GBK，日文=Shift-JIS） | 取决于代码页（ANSI） |
| Linux | UTF-8 | UTF-8 | UTF-8 |
| macOS | UTF-8 (APFS) | UTF-8 | UTF-8 |
| Android | UTF-8 | UTF-8 | UTF-8 |

Windows 是唯一一个默认不使用 UTF-8 的主流平台。这对 KrKr2 尤为重要，因为原版 KiriKiri 引擎使用 Shift-JIS 编码（日文 Windows 的默认代码页），而 KrKr2 需要在所有平台上正确处理这些日文文本。

### Windows 的 Wide API vs Narrow API

Windows 系统 API 有两套版本：

```cpp
// Narrow API (A 后缀) — 使用当前代码页的 char*
CreateFileA("file.txt", ...);  // 受代码页影响，非 ASCII 字符可能乱码

// Wide API (W 后缀) — 使用 UTF-16 的 wchar_t*
CreateFileW(L"文件.txt", ...); // 推荐：支持所有 Unicode 字符
```

在 Windows 上，**始终使用 Wide (W) 版本 API** 是跨平台项目的最佳实践。但这意味着你需要在 UTF-8（项目内部编码）和 UTF-16（Windows API）之间转换：

```cpp
#include <string>

#ifdef _WIN32
#include <windows.h>

// UTF-8 → UTF-16 (Windows Wide)
std::wstring utf8ToWide(const std::string& utf8) {
    if (utf8.empty()) return {};
    int wlen = MultiByteToWideChar(CP_UTF8, 0, utf8.c_str(), -1, nullptr, 0);
    std::wstring wide(wlen - 1, L'\0'); // -1 排除 null 终止符
    MultiByteToWideChar(CP_UTF8, 0, utf8.c_str(), -1, wide.data(), wlen);
    return wide;
}

// UTF-16 → UTF-8
std::string wideToUtf8(const std::wstring& wide) {
    if (wide.empty()) return {};
    int len = WideCharToMultiByte(CP_UTF8, 0, wide.c_str(), -1, nullptr, 0, nullptr, nullptr);
    std::string utf8(len - 1, '\0');
    WideCharToMultiByte(CP_UTF8, 0, wide.c_str(), -1, utf8.data(), len, nullptr, nullptr);
    return utf8;
}
#endif
```

### KrKr2 中的编码处理

KrKr2 在 `vcpkg.json` 中依赖了 `icu` 库用于国际化和编码转换。项目编译时定义了 `USE_UNICODE_FSTRING` 宏，表示内部使用 Unicode 字符串。`cpp/core/tjs2/` 中的 TJS2 脚本引擎需要同时处理 Shift-JIS（原版脚本）和 UTF-8/UTF-16 编码。

## 常见错误及解决方案

### 错误 1：硬编码路径分隔符

```cpp
// ❌ 错误：硬编码反斜杠，Linux/macOS/Android 上路径无效
std::string path = "data\\scripts\\startup.tjs";

// ✅ 正确：使用 std::filesystem::path
std::filesystem::path path = std::filesystem::path("data") / "scripts" / "startup.tjs";
```

### 错误 2：用 `long` 存储文件大小

```cpp
// ❌ 错误：Windows 上 long 是 4 字节，文件超过 2GB 就溢出
long fileSize = getFileSize("big_archive.xp3"); // Windows: 最大 2GB

// ✅ 正确：使用 int64_t 或 std::uintmax_t
std::uintmax_t fileSize = std::filesystem::file_size("big_archive.xp3");
```

### 错误 3：假设 `wchar_t` 是 UTF-16

```cpp
// ❌ 错误：在 Linux 上 wchar_t 是 4 字节 UTF-32，不是 UTF-16
std::wstring wstr = L"你好"; // Windows: 4 字节  Linux: 8 字节
size_t bytes = wstr.size() * sizeof(wchar_t); // 结果不同！

// ✅ 正确：使用 char16_t（始终 2 字节 UTF-16）或 char32_t（始终 4 字节 UTF-32）
std::u16string u16str = u"你好"; // 所有平台: 4 字节 (2 × 2)
std::u32string u32str = U"你好"; // 所有平台: 8 字节 (2 × 4)
```
