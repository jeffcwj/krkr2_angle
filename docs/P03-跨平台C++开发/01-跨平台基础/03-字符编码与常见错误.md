## 字符编码基础

### 什么是字符编码

字符编码（Character Encoding）是将人类可读的字符（如字母 'A'、汉字 '你'、日文假名 'あ'）映射为计算机可存储的数字序列的规则。理解编码对跨平台开发至关重要，因为**不同操作系统、不同地区使用不同的默认编码**，处理不当就会出现乱码。

编码的核心概念：

| 术语 | 英文 | 含义 |
|------|------|------|
| 字符集 | Character Set | 一组字符的集合（如 "所有汉字"、"所有拉丁字母"） |
| 码点 | Code Point | 字符集中每个字符的唯一编号（如 Unicode 中 'A' = U+0041） |
| 编码方案 | Encoding Scheme | 将码点转换为字节序列的具体规则（如 UTF-8、UTF-16） |

三者的关系：**字符 → 码点（编号） → 编码（字节）**。

### ASCII：一切的起点

ASCII（American Standard Code for Information Interchange，美国信息交换标准代码）是最早的字符编码标准，使用 7 位二进制表示 128 个字符：

```cpp
#include <cstdio>
#include <cstdint>

// ASCII 编码示例：每个字符占 1 字节
int main() {
    // 可打印字符范围：0x20-0x7E
    char letter = 'A';       // ASCII 0x41 = 65
    char digit  = '0';       // ASCII 0x30 = 48
    char space  = ' ';       // ASCII 0x20 = 32
    char newline = '\n';     // ASCII 0x0A = 10（控制字符）

    // 打印字符的 ASCII 码值
    printf("'%c' = 0x%02X (%d)\n", letter, letter, letter);   // 'A' = 0x41 (65)
    printf("'%c' = 0x%02X (%d)\n", digit, digit, digit);      // '0' = 0x30 (48)

    // ASCII 的大小写转换利用了编码规律：
    // 大写字母 0x41-0x5A，小写字母 0x61-0x7A，差值恒为 0x20
    char lower = letter | 0x20;   // 'A' | 0x20 = 'a'（设置第 5 位）
    char upper = lower & ~0x20;   // 'a' & ~0x20 = 'A'（清除第 5 位）
    printf("大写转小写: '%c' -> '%c'\n", letter, lower);
    printf("小写转大写: '%c' -> '%c'\n", lower, upper);

    return 0;
}
```

ASCII 的局限性：只能表示英文字母、数字和少量符号，**无法表示中文、日文等非拉丁字符**。

### 从代码页到 Unicode

为解决 ASCII 的局限，各国/地区发展了自己的编码标准：

| 编码标准 | 覆盖语言 | 字节数 | KrKr2 相关性 |
|----------|----------|--------|-------------|
| **Shift-JIS** | 日文 | 1-2 字节 | KiriKiri 原版使用此编码存储脚本 |
| **GBK/GB2312** | 简体中文 | 1-2 字节 | Windows 中文版默认代码页（CP936） |
| **Big5** | 繁体中文 | 1-2 字节 | 台湾/香港 Windows 使用 |
| **EUC-KR** | 韩文 | 1-2 字节 | 韩国 Windows 使用 |
| **ISO-8859-1** | 西欧语言 | 1 字节 | 扩展 ASCII，又称 Latin-1 |

这种"各自为政"的编码方案导致了严重的兼容性问题——同一个字节序列在不同编码下会显示为不同的字符（即"乱码"）。

```cpp
#include <cstdio>
#include <cstdint>

// 演示同一字节序列在不同编码下的含义
int main() {
    // 字节序列 0x82 0xB1
    uint8_t bytes[] = {0x82, 0xB1};

    // 在 Shift-JIS 中：这是日文假名 'こ'
    // 在 GBK 中：    这是汉字 '苙'（或其他字符）
    // 在 UTF-8 中：  这是非法序列（0x82 不是合法的 UTF-8 起始字节）

    printf("字节序列: 0x%02X 0x%02X\n", bytes[0], bytes[1]);
    printf("Shift-JIS 解释: こ (日文假名)\n");
    printf("GBK 解释:       （不同的汉字）\n");
    printf("UTF-8 解释:     （非法序列，会显示乱码）\n");
    printf("\n");
    printf("这就是为什么同一个文件在不同系统上打开会出现乱码！\n");

    return 0;
}
```

**Unicode**（统一码）的诞生就是为了解决这个问题。Unicode 为世界上每一个字符分配一个唯一的码点（目前包含超过 14 万个字符），范围从 U+0000 到 U+10FFFF。

### UTF-8 编码

UTF-8（8-bit Unicode Transformation Format）是最流行的 Unicode 编码方案，**也是跨平台项目推荐的内部编码**。它的关键特性：

1. **变长编码**：1-4 字节表示一个字符
2. **向后兼容 ASCII**：ASCII 字符（U+0000-U+007F）用 1 字节表示，与 ASCII 编码完全相同
3. **自同步**：从任意字节位置都能找到字符边界

```
UTF-8 编码规则：
┌──────────────────────┬──────────────────────────────┐
│ 码点范围              │ UTF-8 字节表示                │
├──────────────────────┼──────────────────────────────┤
│ U+0000 - U+007F      │ 0xxxxxxx                     │ ← 1 字节（ASCII 兼容）
│ U+0080 - U+07FF      │ 110xxxxx 10xxxxxx            │ ← 2 字节
│ U+0800 - U+FFFF      │ 1110xxxx 10xxxxxx 10xxxxxx   │ ← 3 字节（CJK 字符）
│ U+10000 - U+10FFFF   │ 11110xxx 10xxxxxx 10xxxxxx   │
│                      │          10xxxxxx             │ ← 4 字节（emoji 等）
└──────────────────────┴──────────────────────────────┘
```

```cpp
#include <cstdio>
#include <cstdint>
#include <cstring>

// 手动编码和解码 UTF-8，理解其二进制结构
int main() {
    // 示例 1：ASCII 字符 'A' (U+0041)
    // 二进制：0100 0001 → UTF-8: 0x41（1 字节，与 ASCII 相同）
    const char ascii[] = "A";
    printf("'A': %zu 字节, 0x%02X\n", strlen(ascii), (uint8_t)ascii[0]);

    // 示例 2：中文 '你' (U+4F60)
    // 二进制：0100 1111 0110 0000
    // UTF-8 编码过程：
    //   U+4F60 在 U+0800-U+FFFF 范围 → 3 字节
    //   1110xxxx 10xxxxxx 10xxxxxx
    //   填入位：0100 | 111101 | 100000
    //   结果：11100100 10111101 10100000
    //   十六进制：0xE4 0xBD 0xA0
    const char ni[] = "你";
    printf("'你': %zu 字节, 0x%02X 0x%02X 0x%02X\n",
           strlen(ni), (uint8_t)ni[0], (uint8_t)ni[1], (uint8_t)ni[2]);

    // 示例 3：emoji '😀' (U+1F600)
    // U+1F600 在 U+10000-U+10FFFF 范围 → 4 字节
    const char emoji[] = "😀";
    printf("'😀': %zu 字节, 0x%02X 0x%02X 0x%02X 0x%02X\n",
           strlen(emoji),
           (uint8_t)emoji[0], (uint8_t)emoji[1],
           (uint8_t)emoji[2], (uint8_t)emoji[3]);

    // 关键结论：strlen() 返回字节数，不是字符数！
    const char mixed[] = "Hello你好";
    printf("'Hello你好': strlen=%zu 字节, 但只有 7 个字符\n", strlen(mixed));

    return 0;
}
```

输出：

```
'A': 1 字节, 0x41
'你': 3 字节, 0xE4 0xBD 0xA0
'😀': 4 字节, 0xF0 0x9F 0x98 0x80
'Hello你好': 11 字节, 但只有 7 个字符
```

### UTF-16 编码

UTF-16 使用 2 或 4 字节表示一个字符。**Windows 系统 API 内部使用 UTF-16**，这是跨平台编码问题的主要来源之一。

```
UTF-16 编码规则：
┌──────────────────────┬────────────────────────────────┐
│ 码点范围              │ UTF-16 表示                     │
├──────────────────────┼────────────────────────────────┤
│ U+0000 - U+FFFF      │ 直接用 2 字节表示（BMP 平面）    │
│ U+10000 - U+10FFFF   │ 代理对：高代理(2字节)+低代理(2字节)│
└──────────────────────┴────────────────────────────────┘
```

```cpp
#include <cstdio>
#include <cstdint>
#include <cstring>

// 演示 UTF-16 编码
int main() {
    // BMP 字符：直接用 2 字节
    char16_t bmp_char = u'你';  // U+4F60 → 0x4F60（2 字节）
    printf("'你' UTF-16: 0x%04X (%zu 字节)\n",
           (unsigned)bmp_char, sizeof(bmp_char));

    // 补充平面字符：需要代理对（4 字节）
    // U+1F600 (😀) 编码过程：
    //   1. 减去 0x10000: 0x1F600 - 0x10000 = 0x0F600
    //   2. 高 10 位: 0x0F600 >> 10 = 0x003D → 高代理: 0xD800 + 0x003D = 0xD83D
    //   3. 低 10 位: 0x0F600 & 0x3FF = 0x0200 → 低代理: 0xDC00 + 0x0200 = 0xDE00
    char16_t surrogate_pair[] = {0xD83D, 0xDE00, 0};  // 😀 的 UTF-16 代理对
    printf("'😀' UTF-16 代理对: 0x%04X 0x%04X (4 字节)\n",
           (unsigned)surrogate_pair[0], (unsigned)surrogate_pair[1]);

    // UTF-8 vs UTF-16 空间效率对比
    printf("\n编码空间对比：\n");
    printf("  ASCII 'A':   UTF-8=1字节  UTF-16=2字节  → UTF-8 更省\n");
    printf("  中文 '你':   UTF-8=3字节  UTF-16=2字节  → UTF-16 更省\n");
    printf("  emoji '😀':  UTF-8=4字节  UTF-16=4字节  → 相同\n");

    return 0;
}
```

### Shift-JIS：KiriKiri 的历史遗产

Shift-JIS（Shift Japanese Industrial Standards）是日本 Windows 的默认编码，对 KrKr2 项目尤为重要，因为**原版 KiriKiri 引擎的所有脚本文件（.ks、.tjs）都使用 Shift-JIS 编码**。

```cpp
#include <cstdio>
#include <cstdint>

// Shift-JIS 编码结构
int main() {
    // Shift-JIS 的编码规则：
    // 1. ASCII 范围 (0x20-0x7E)：1 字节，与 ASCII 相同
    // 2. 半角片假名 (0xA1-0xDF)：1 字节
    // 3. 全角字符：2 字节（首字节 0x81-0x9F 或 0xE0-0xEF）

    // 日文 'こ' 在 Shift-JIS 中的编码：0x82 0xB1
    uint8_t kana_sjis[] = {0x82, 0xB1};

    // 同一个字符 'こ' 在 UTF-8 中的编码：0xE3 0x81 0x93
    uint8_t kana_utf8[] = {0xE3, 0x81, 0x93};

    printf("'こ' Shift-JIS: 0x%02X 0x%02X (2 字节)\n",
           kana_sjis[0], kana_sjis[1]);
    printf("'こ' UTF-8:     0x%02X 0x%02X 0x%02X (3 字节)\n",
           kana_utf8[0], kana_utf8[1], kana_utf8[2]);

    // Shift-JIS 的陷阱：某些双字节字符的第二字节与 ASCII 反斜杠 0x5C 冲突
    // 例如日文汉字 '表' 的 Shift-JIS 编码是 0x95 0x5C
    // 如果 C 编译器不感知 Shift-JIS，会把 0x5C 误解为转义字符 '\'
    uint8_t hyou[] = {0x95, 0x5C};  // '表' in Shift-JIS
    printf("'表' Shift-JIS: 0x%02X 0x%02X\n", hyou[0], hyou[1]);
    printf("注意：第二字节 0x5C 就是 ASCII 的 '\\'！\n");
    printf("这就是 Shift-JIS 的著名'反斜杠陷阱'\n");

    return 0;
}
```

## 各平台默认编码对比

| 平台 | 文件系统编码 | 控制台编码 | `std::string` 通常编码 | `wchar_t` 大小 |
|------|------------|-----------|----------------------|---------------|
| **Windows** | UTF-16 (NTFS) | 系统区域设置（中文=GBK, 日文=Shift-JIS） | 取决于代码页（ANSI） | 2 字节（UTF-16） |
| **Linux** | UTF-8 | UTF-8 | UTF-8 | 4 字节（UTF-32） |
| **macOS** | UTF-8 (APFS/HFS+) | UTF-8 | UTF-8 | 4 字节（UTF-32） |
| **Android** | UTF-8 | UTF-8 | UTF-8 | 4 字节（UTF-32） |

Windows 是唯一一个默认不使用 UTF-8 的主流平台。这对 KrKr2 尤为重要，因为：
1. 原版 KiriKiri 引擎使用 Shift-JIS（日文 Windows 的默认代码页）
2. KrKr2 需要在所有平台上正确处理这些日文脚本文本
3. Windows API 使用 UTF-16，需要额外的转换步骤

## Windows 的 Wide API vs Narrow API

Windows 系统 API 有两套版本，这是跨平台编码处理的核心难点：

```cpp
#include <cstdio>

#ifdef _WIN32
#include <windows.h>
#include <string>

// Narrow API (A 后缀)：使用当前代码页的 char*
// Wide API (W 后缀)：使用 UTF-16 的 wchar_t*
// 无后缀版本：根据 UNICODE 宏决定调用 A 还是 W 版本

void demonstrate_api_versions() {
    // ❌ Narrow API：受代码页影响，非 ASCII 字符可能乱码
    // 在日文 Windows 上能打开日文文件名，在中文 Windows 上可能失败
    HANDLE hA = CreateFileA(
        "data\\scripts\\startup.tjs",  // ANSI 编码的路径
        GENERIC_READ, FILE_SHARE_READ, nullptr,
        OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, nullptr
    );
    if (hA != INVALID_HANDLE_VALUE) CloseHandle(hA);

    // ✅ Wide API：使用 UTF-16，支持所有 Unicode 字符
    // 在任何语言的 Windows 上都能正确工作
    HANDLE hW = CreateFileW(
        L"data\\scripts\\startup.tjs",  // UTF-16 编码的路径
        GENERIC_READ, FILE_SHARE_READ, nullptr,
        OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, nullptr
    );
    if (hW != INVALID_HANDLE_VALUE) CloseHandle(hW);
}

// 跨平台项目的标准做法：内部使用 UTF-8，调用 Windows API 时转换为 UTF-16

// UTF-8 → UTF-16 (Windows Wide)
std::wstring utf8ToWide(const std::string& utf8) {
    if (utf8.empty()) return {};
    // 第一次调用：计算所需的 wchar_t 数量
    int wlen = MultiByteToWideChar(
        CP_UTF8,        // 源编码：UTF-8
        0,              // 标志：无特殊行为
        utf8.c_str(),   // 源字符串
        -1,             // 源长度：-1 表示自动检测（以 null 结尾）
        nullptr,        // 目标缓冲区：nullptr 表示只计算长度
        0               // 目标缓冲区大小：0 表示只计算长度
    );
    if (wlen == 0) return {};  // 转换失败

    std::wstring wide(wlen - 1, L'\0'); // -1 排除 null 终止符
    // 第二次调用：执行实际转换
    MultiByteToWideChar(CP_UTF8, 0, utf8.c_str(), -1, wide.data(), wlen);
    return wide;
}

// UTF-16 → UTF-8
std::string wideToUtf8(const std::wstring& wide) {
    if (wide.empty()) return {};
    int len = WideCharToMultiByte(
        CP_UTF8,        // 目标编码：UTF-8
        0,              // 标志
        wide.c_str(),   // 源 UTF-16 字符串
        -1,             // 源长度
        nullptr, 0,     // 只计算长度
        nullptr,        // 默认字符（UTF-8 不需要）
        nullptr         // 是否使用了默认字符（UTF-8 不需要）
    );
    if (len == 0) return {};

    std::string utf8(len - 1, '\0');
    WideCharToMultiByte(CP_UTF8, 0, wide.c_str(), -1,
                        utf8.data(), len, nullptr, nullptr);
    return utf8;
}

int main() {
    // 使用示例：打开包含日文字符的文件
    std::string utf8Path = "data/scripts/シナリオ.ks";  // UTF-8 内部路径
    std::wstring widePath = utf8ToWide(utf8Path);        // 转换为 UTF-16
    printf("UTF-8 路径: %s (%zu 字节)\n", utf8Path.c_str(), utf8Path.size());
    printf("UTF-16 路径: %zu 个 wchar_t\n", widePath.size());

    // 用 Wide API 打开文件
    HANDLE h = CreateFileW(widePath.c_str(), GENERIC_READ,
                           FILE_SHARE_READ, nullptr,
                           OPEN_EXISTING, FILE_ATTRIBUTE_NORMAL, nullptr);
    if (h != INVALID_HANDLE_VALUE) {
        printf("文件打开成功！\n");
        CloseHandle(h);
    } else {
        printf("文件不存在（预期行为）\n");
    }

    return 0;
}
#else
int main() {
    // Linux/macOS/Android：直接使用 UTF-8，无需转换
    const char* path = "data/scripts/シナリオ.ks";
    FILE* f = fopen(path, "r");
    if (f) {
        printf("文件打开成功！\n");
        fclose(f);
    } else {
        printf("文件不存在（预期行为）\n");
    }
    return 0;
}
#endif
```

## KrKr2 中的编码处理

KrKr2 在编码处理方面有以下关键设计：

```
编码处理架构：
┌─────────────────────────────────────────────────┐
│                  KrKr2 引擎内部                   │
│              统一使用 UTF-16 / Unicode            │
├─────────────────────────────────────────────────┤
│                                                  │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │ 脚本加载  │    │ 文件系统  │    │ UI 显示  │   │
│  │ .ks/.tjs │    │ 路径处理  │    │ 文本渲染  │   │
│  └────┬─────┘    └────┬─────┘    └────┬─────┘   │
│       │               │               │          │
├───────┼───────────────┼───────────────┼──────────┤
│       ↓               ↓               ↓          │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐   │
│  │Shift-JIS │    │ Platform │    │  ICU     │   │
│  │→ Unicode │    │ 编码转换  │    │ 国际化   │   │
│  │ 检测转换  │    │  适配层  │    │  支持    │   │
│  └──────────┘    └──────────┘    └──────────┘   │
└─────────────────────────────────────────────────┘
```

```cpp
// KrKr2 编码相关配置和实现要点

// 1. vcpkg.json 中的 ICU 依赖
// KrKr2 使用 ICU (International Components for Unicode) 进行编码转换
// vcpkg.json 中包含 "icu" 依赖项

// 2. 编译宏定义
// CMakeLists.txt 中定义：
//   target_compile_definitions(... USE_UNICODE_FSTRING ...)
// 这告诉引擎内部使用 Unicode 宽字符串

// 3. TJS2 脚本引擎的字符类型
// cpp/core/tjs2/tjsTypes.h 中定义了脚本引擎的字符类型：
//   typedef char16_t tjs_char;    // TJS2 字符 = UTF-16 单元
//   typedef std::u16string tjs_string;

// 4. 脚本文件编码检测
// KrKr2 的 TJS2 词法器需要处理两种编码的脚本文件：
//   - 原版 KiriKiri 脚本：Shift-JIS 编码
//   - 新版脚本：UTF-8 编码（带或不带 BOM）
// 检测策略：检查文件头部是否有 UTF-8 BOM (0xEF 0xBB 0xBF)
//           有 BOM → UTF-8
//           无 BOM → 尝试 UTF-8 解码，失败则回退到 Shift-JIS

// 5. 平台文件路径处理
// Windows: 内部 UTF-8 → MultiByteToWideChar → CreateFileW
// Linux/macOS: 直接使用 UTF-8 路径
// Android: UTF-8 路径 → JNI 传入 Java 的 String (UTF-16)
```

```cpp
// 模拟 KrKr2 的脚本编码检测逻辑
#include <cstdio>
#include <cstdint>
#include <cstring>
#include <string>
#include <vector>

// BOM (Byte Order Mark) 检测
enum class TextEncoding {
    UTF8,       // UTF-8 (带或不带 BOM)
    UTF16LE,    // UTF-16 小端序 (BOM: 0xFF 0xFE)
    UTF16BE,    // UTF-16 大端序 (BOM: 0xFE 0xFF)
    ShiftJIS,   // Shift-JIS (无 BOM，回退默认)
    Unknown
};

// 通过 BOM 检测文件编码
TextEncoding detectEncoding(const uint8_t* data, size_t size) {
    if (size >= 3 &&
        data[0] == 0xEF && data[1] == 0xBB && data[2] == 0xBF) {
        return TextEncoding::UTF8;        // UTF-8 BOM
    }
    if (size >= 2 && data[0] == 0xFF && data[1] == 0xFE) {
        return TextEncoding::UTF16LE;     // UTF-16 LE BOM
    }
    if (size >= 2 && data[0] == 0xFE && data[1] == 0xFF) {
        return TextEncoding::UTF16BE;     // UTF-16 BE BOM
    }

    // 无 BOM：尝试验证是否为合法 UTF-8
    bool validUtf8 = true;
    bool hasHighByte = false;  // 是否包含非 ASCII 字节
    for (size_t i = 0; i < size && validUtf8; ) {
        uint8_t b = data[i];
        int expectedCont = 0;  // 预期的后续字节数

        if (b <= 0x7F) {
            i++; continue;      // ASCII 字符
        }
        hasHighByte = true;
        if ((b & 0xE0) == 0xC0)      expectedCont = 1;  // 110xxxxx
        else if ((b & 0xF0) == 0xE0) expectedCont = 2;  // 1110xxxx
        else if ((b & 0xF8) == 0xF0) expectedCont = 3;  // 11110xxx
        else { validUtf8 = false; break; }               // 非法起始字节

        // 验证后续字节都是 10xxxxxx
        for (int j = 1; j <= expectedCont && (i + j) < size; j++) {
            if ((data[i + j] & 0xC0) != 0x80) {
                validUtf8 = false; break;
            }
        }
        i += 1 + expectedCont;
    }

    if (validUtf8 && hasHighByte) return TextEncoding::UTF8;
    if (!validUtf8) return TextEncoding::ShiftJIS;  // UTF-8 验证失败，回退 Shift-JIS
    return TextEncoding::UTF8;  // 纯 ASCII 视为 UTF-8
}

int main() {
    // 测试 1：UTF-8 BOM 文件
    uint8_t utf8bom[] = {0xEF, 0xBB, 0xBF, 'H', 'e', 'l', 'l', 'o'};
    auto enc1 = detectEncoding(utf8bom, sizeof(utf8bom));
    printf("UTF-8 BOM 文件: %s\n",
           enc1 == TextEncoding::UTF8 ? "UTF-8 ✓" : "错误");

    // 测试 2：无 BOM 的 UTF-8 中文
    uint8_t utf8nobom[] = {0xE4, 0xBD, 0xA0, 0xE5, 0xA5, 0xBD};  // "你好"
    auto enc2 = detectEncoding(utf8nobom, sizeof(utf8nobom));
    printf("无 BOM UTF-8:   %s\n",
           enc2 == TextEncoding::UTF8 ? "UTF-8 ✓" : "错误");

    // 测试 3：Shift-JIS 日文 "こんにちは"
    uint8_t sjis[] = {0x82, 0xB1, 0x82, 0xF1, 0x82, 0xC9,
                      0x82, 0xBF, 0x82, 0xCD};
    auto enc3 = detectEncoding(sjis, sizeof(sjis));
    printf("Shift-JIS 文件: %s\n",
           enc3 == TextEncoding::ShiftJIS ? "Shift-JIS ✓" : "错误");

    return 0;
}
```

## 常见跨平台编码错误

### 错误 1：硬编码路径分隔符


### 错误 3：假设 `wchar_t` 是固定大小

```cpp

#include <cwchar>
#include <cstring>

    std::u32string u32str = U"你好";
    printf("U\"你好\" 占 %zu 字节\n", u32str.size() * sizeof(char32_t));
```

### 错误 4：忽略 BOM（字节序标记）

```cpp
#include <cstdio>
#include <cstdint>
#include <fstream>
#include <string>

// BOM 会导致文件开头出现"不可见字符"
// UTF-8 BOM: 0xEF 0xBB 0xBF（3 字节）
// 如果不跳过 BOM，解析文件内容时可能出错

// ✅ 正确：读取文件时跳过 UTF-8 BOM
std::string readFileSkipBOM(const std::string& path) {
    std::ifstream file(path, std::ios::binary);
    if (!file.is_open()) return {};

    // 检查前 3 字节是否为 UTF-8 BOM
    char bom[3] = {};
    file.read(bom, 3);

    std::string content;
    if (file.gcount() == 3 &&
        (uint8_t)bom[0] == 0xEF &&
        (uint8_t)bom[1] == 0xBB &&
        (uint8_t)bom[2] == 0xBF) {
        // 有 BOM，从第 4 字节开始读取
        content.assign(std::istreambuf_iterator<char>(file),
                       std::istreambuf_iterator<char>());
    } else {
        // 无 BOM，回到文件开头重新读取
        file.seekg(0);
        content.assign(std::istreambuf_iterator<char>(file),
                       std::istreambuf_iterator<char>());
    }
    return content;
}

int main() {
    // 演示 BOM 的影响
    uint8_t withBOM[] = {0xEF, 0xBB, 0xBF, 'H', 'e', 'l', 'l', 'o'};
    uint8_t noBOM[]   = {'H', 'e', 'l', 'l', 'o'};

    printf("带 BOM 的文件前 3 字节: 0x%02X 0x%02X 0x%02X\n",
           withBOM[0], withBOM[1], withBOM[2]);
    printf("如果不跳过 BOM，解析 JSON/XML/TJS 脚本时会出错！\n");

    return 0;
}
```

### 实践 1：编写跨平台编码转换工具

创建一个完整可编译的跨平台编码检测和转换程序：

```cpp
// encoding_tool.cpp — 跨平台编码检测工具
#include <cstdio>
#include <cstdint>
#include <cstring>
#include <fstream>
#include <string>
#include <vector>

// 编码类型枚举
enum class Encoding { ASCII, UTF8, UTF8_BOM, UTF16_LE, UTF16_BE, SJIS, UNKNOWN };

// 返回编码名称的辅助函数
const char* encodingName(Encoding enc) {
    switch (enc) {
        case Encoding::ASCII:    return "ASCII";
        case Encoding::UTF8:     return "UTF-8 (无 BOM)";
        case Encoding::UTF8_BOM: return "UTF-8 (有 BOM)";
        case Encoding::UTF16_LE: return "UTF-16 LE";
        case Encoding::UTF16_BE: return "UTF-16 BE";
        case Encoding::SJIS:     return "Shift-JIS";
        default:                 return "未知";
    }
}

// 检测字节序列的编码
Encoding detectEncoding(const std::vector<uint8_t>& data) {
    if (data.size() < 2) return Encoding::ASCII;

    // 检查 BOM
    if (data.size() >= 3 &&
        data[0] == 0xEF && data[1] == 0xBB && data[2] == 0xBF) {
        return Encoding::UTF8_BOM;
    }
    if (data[0] == 0xFF && data[1] == 0xFE) return Encoding::UTF16_LE;
    if (data[0] == 0xFE && data[1] == 0xFF) return Encoding::UTF16_BE;

    // 验证 UTF-8 合法性
    bool allAscii = true;
    bool validUtf8 = true;
    for (size_t i = 0; i < data.size() && validUtf8; ) {
        uint8_t b = data[i];
        if (b > 0x7F) allAscii = false;

        if (b <= 0x7F) { i++; continue; }

        int cont = 0;
        if ((b & 0xE0) == 0xC0)      cont = 1;
        else if ((b & 0xF0) == 0xE0) cont = 2;
        else if ((b & 0xF8) == 0xF0) cont = 3;
        else { validUtf8 = false; break; }

        if (i + cont >= data.size()) { validUtf8 = false; break; }
        for (int j = 1; j <= cont; j++) {
            if ((data[i + j] & 0xC0) != 0x80) {
                validUtf8 = false; break;
            }
        }
        i += 1 + cont;
    }

    if (allAscii) return Encoding::ASCII;
    if (validUtf8) return Encoding::UTF8;
    return Encoding::SJIS;  // 非合法 UTF-8，推测为 Shift-JIS
}

int main() {
    // 测试用例
    struct TestCase {
        const char* name;
        std::vector<uint8_t> data;
    };

    TestCase tests[] = {
        {"纯 ASCII",           {'H', 'e', 'l', 'l', 'o'}},
        {"UTF-8 BOM + 内容",   {0xEF, 0xBB, 0xBF, 0xE4, 0xBD, 0xA0}},
        {"UTF-8 无 BOM 中文",  {0xE4, 0xBD, 0xA0, 0xE5, 0xA5, 0xBD}},
        {"UTF-16 LE BOM",      {0xFF, 0xFE, 0x60, 0x4F}},
        {"Shift-JIS 日文",     {0x82, 0xB1, 0x82, 0xF1}},
    };

    printf("=== 编码检测测试 ===\n\n");
    for (const auto& test : tests) {
        Encoding enc = detectEncoding(test.data);
        printf("%-20s → %s\n", test.name, encodingName(enc));
    }

    return 0;
}
```

## 本节小结

- **字符编码**是将字符映射为字节序列的规则，不同编码方案互不兼容
- **ASCII** 只支持 128 个字符；**Unicode** 统一了全球所有字符的编号
- **UTF-8** 是跨平台推荐编码：变长（1-4 字节）、向后兼容 ASCII、自同步
- **UTF-16** 是 Windows API 的内部编码（2 或 4 字节）
- **Shift-JIS** 是 KiriKiri 原版引擎的脚本编码，KrKr2 需要兼容处理
- **Windows 是唯一默认不使用 UTF-8 的主流平台**，需要 Wide API（W 后缀）和 `MultiByteToWideChar`/`WideCharToMultiByte` 转换
- **`wchar_t`** 在 Windows 上是 2 字节（UTF-16），在 Unix 上是 4 字节（UTF-32）——使用 `char16_t`/`char32_t` 保证一致性
- 读取文件时**必须检测并跳过 BOM**，否则解析可能出错

### 题目 1：UTF-8 编码计算

中文字符 '龙' 的 Unicode 码点是 U+9F99。请手动计算它的 UTF-8 编码（写出每个字节的十六进制值），并编写 C++ 代码验证你的计算。

<details>
<summary>查看答案</summary>

U+9F99 落在 U+0800-U+FFFF 范围，使用 3 字节 UTF-8 编码：

```
格式：1110xxxx 10xxxxxx 10xxxxxx
U+9F99 = 1001 1111 1001 1001 (二进制)
填入：  1110 1001  10 111110  10 011001
        = 0xE9     0xBE       0x99
```

验证代码：

```cpp
#include <cstdio>
#include <cstdint>
#include <cstring>

int main() {
    const char dragon[] = "龙";
    printf("'龙' UTF-8 编码: ");
    for (size_t i = 0; i < strlen(dragon); i++) {
        printf("0x%02X ", (uint8_t)dragon[i]);
    }
    printf("\n");
    // 预期输出: 0xE9 0xBE 0x99

    // 手动编码验证
    uint32_t codepoint = 0x9F99;
    uint8_t b1 = 0xE0 | ((codepoint >> 12) & 0x0F);  // 1110xxxx
    uint8_t b2 = 0x80 | ((codepoint >> 6) & 0x3F);   // 10xxxxxx
    uint8_t b3 = 0x80 | (codepoint & 0x3F);           // 10xxxxxx
    printf("手动编码结果: 0x%02X 0x%02X 0x%02X\n", b1, b2, b3);

    return 0;
}
```

### 题目 2：编码检测函数

给定以下字节序列，判断其编码类型并说明理由：
- 序列 A：`0xE4 0xB8 0xAD 0xE6 0x96 0x87`
- 序列 B：`0x92 0x86 0x95 0xB6`
- 序列 C：`0xFE 0xFF 0x4E 0x2D`

<details>
<summary>查看答案</summary>

- **序列 A**：UTF-8 编码的 "中文"
  - `0xE4 0xB8 0xAD` = U+4E2D '中'（1110**0100** 10**111000** 10**101101** → 0100 111000 101101 = 0x4E2D）
  - `0xE6 0x96 0x87` = U+6587 '文'
  - 理由：每个三字节序列都符合 UTF-8 的 `1110xxxx 10xxxxxx 10xxxxxx` 模式

- **序列 B**：Shift-JIS 编码
  - `0x92 0x86` 和 `0x95 0xB6` 都是 Shift-JIS 双字节字符
  - 首字节 0x92 和 0x95 落在 Shift-JIS 双字节首字节范围 (0x81-0x9F, 0xE0-0xEF)
  - 这不是合法 UTF-8（0x92 = 10010010，是 UTF-8 后续字节格式但出现在首位）

- **序列 C**：UTF-16 BE 编码的 "中"
  - `0xFE 0xFF` = UTF-16 大端序 BOM
  - `0x4E 0x2D` = U+4E2D '中' 的 UTF-16 编码（大端序）

```cpp
#include <cstdio>
#include <cstdint>
#include <vector>

int main() {
    // 验证序列 A 是 UTF-8
    uint8_t seqA[] = {0xE4, 0xB8, 0xAD, 0xE6, 0x96, 0x87};
    printf("序列 A (UTF-8): %s\n", (const char*)seqA); // 输出: 中文

    // 验证序列 C 的 BOM
    uint8_t seqC[] = {0xFE, 0xFF, 0x4E, 0x2D};
    printf("序列 C BOM: 0x%02X 0x%02X → UTF-16 BE\n", seqC[0], seqC[1]);
    printf("序列 C 数据: U+%04X = '中'\n",
           (uint16_t)(seqC[2] << 8 | seqC[3]));

    return 0;
}
```

### 题目 3：跨平台文件路径处理

编写一个函数 `std::string normalizePath(const std::string& path)`，将路径中的反斜杠统一替换为正斜杠，并处理 Windows 盘符（如 `C:\`）。该函数需要在四平台上都能正确工作。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>
#include <string>
#include <algorithm>

// 将路径分隔符统一为正斜杠
// 正斜杠在所有平台的文件系统 API 中都能正确工作
std::string normalizePath(const std::string& path) {
    std::string result = path;

    // 统一分隔符为 '/'
    std::replace(result.begin(), result.end(), '\\', '/');

    // 移除尾部多余的分隔符（保留根目录的 '/'）
    while (result.size() > 1 && result.back() == '/') {
        result.pop_back();
    }

    // 处理连续的分隔符 "//" → "/"
    std::string cleaned;
    cleaned.reserve(result.size());
    for (size_t i = 0; i < result.size(); i++) {
        if (result[i] == '/' && i > 0 && result[i - 1] == '/') {
            continue;  // 跳过连续的 '/'
        }
        cleaned += result[i];
    }

    return cleaned;
}

int main() {
    // 测试用例
    printf("%s\n", normalizePath("C:\\Users\\test\\file.txt").c_str());
    // → "C:/Users/test/file.txt"

    printf("%s\n", normalizePath("data//scripts///startup.tjs").c_str());
    // → "data/scripts/startup.tjs"

    printf("%s\n", normalizePath("/home/user/projects/").c_str());
    // → "/home/user/projects"

    printf("%s\n", normalizePath("relative/path/file.cpp").c_str());
    // → "relative/path/file.cpp"

    return 0;
}
```
