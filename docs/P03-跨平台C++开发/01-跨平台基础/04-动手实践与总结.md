## 为什么需要综合实践

前三节我们分别学习了文件系统差异、字节序与数据类型、字符编码三个独立的跨平台主题。但在真实的项目开发中，这三个问题往往**同时出现**。

想象这样一个场景：你正在为 KrKr2 引擎编写一个 XP3 归档读取器。这个任务需要：

- **文件系统知识**：打开归档文件时，路径要跨平台兼容（`/` vs `\`、大小写敏感性）

- **字符编码知识**：归档中的文件名使用 UTF-16LE 编码，但你的程序内部可能使用 UTF-8

看到了吗？一个"简单的"文件读取操作，实际上涉及了本章所有知识点。下面的实验就是让你在一个完整的小项目中把这些知识串起来。

## 实验 1：平台特性检测诊断工具

### 场景

假设你加入了一个跨平台项目团队。你的第一个任务是编写一个"环境检测工具"，让每个开发者在自己的机器上运行，输出完整的平台信息报告。这样当有人报告 Bug 时，只需贴上报告就能快速定位是不是平台差异导致的问题。

### 完整代码

```cpp
// platform_info.cpp
// 平台特性检测诊断工具 — 检测 OS、数据类型、字节序、编码
#include <cstdint>
#include <cstring>
#include <iostream>
#include <string>

// C++20 的 <bit> 头文件提供了 std::endian
// 如果你的编译器不支持 C++20，后面有替代方案
#if __cplusplus >= 202002L || (defined(_MSVC_LANG) && _MSVC_LANG >= 202002L)
    #include <bit>
    #define HAS_CPP20_BIT 1
#else
    #define HAS_CPP20_BIT 0
#endif

// ============================================================
// 第一部分：操作系统检测
// ============================================================
void detectOS() {
    std::cout << "\n========== 操作系统 ==========" << std::endl;

#if defined(_WIN32)
    // _WIN32 在 32 位和 64 位 Windows 上都被定义
    std::cout << "操作系统: Windows" << std::endl;
    #if defined(_WIN64)
        std::cout << "架构:     64-bit" << std::endl;
    #else
        std::cout << "架构:     32-bit" << std::endl;
    #endif
#elif defined(__linux__) && defined(__ANDROID__)
    // Android 也定义了 __linux__，所以必须先检测 __ANDROID__
    std::cout << "操作系统: Android (Linux kernel)" << std::endl;
    #if defined(__aarch64__)
        std::cout << "架构:     ARM64 (aarch64)" << std::endl;
    #elif defined(__x86_64__)
        std::cout << "架构:     x86_64" << std::endl;
    #endif
#elif defined(__linux__)
    std::cout << "操作系统: Linux" << std::endl;
    #if defined(__x86_64__)
        std::cout << "架构:     x86_64" << std::endl;
    #endif
#elif defined(__APPLE__)
    std::cout << "操作系统: macOS" << std::endl;
    #if defined(__aarch64__)
        std::cout << "架构:     ARM64 (Apple Silicon)" << std::endl;
    #elif defined(__x86_64__)
        std::cout << "架构:     x86_64 (Intel)" << std::endl;
    #endif
#else
    std::cout << "操作系统: 未知" << std::endl;
#endif

    // 编译器信息也很重要——不同编译器行为不同
#if defined(_MSC_VER)
    std::cout << "编译器:   MSVC " << _MSC_VER << std::endl;
#elif defined(__clang__)
    std::cout << "编译器:   Clang " << __clang_major__ << "."
              << __clang_minor__ << std::endl;
#elif defined(__GNUC__)
    std::cout << "编译器:   GCC " << __GNUC__ << "."
              << __GNUC_MINOR__ << std::endl;
#endif
}

// ============================================================
// 第二部分：数据类型大小检测
// ============================================================
void detectDataTypes() {
    std::cout << "\n========== 数据类型大小 ==========" << std::endl;

    // 这些类型在所有平台上大小一致
    std::cout << "[固定大小类型]" << std::endl;
    std::cout << "  char:        " << sizeof(char) << " bytes" << std::endl;
    std::cout << "  short:       " << sizeof(short) << " bytes" << std::endl;
    std::cout << "  int:         " << sizeof(int) << " bytes" << std::endl;
    std::cout << "  long long:   " << sizeof(long long) << " bytes" << std::endl;

    // ⚠️ 这些类型在不同平台上大小不同——这是跨平台 Bug 的温床
    std::cout << "\n[⚠️ 平台可变类型 — 跨平台危险区]" << std::endl;
    std::cout << "  long:        " << sizeof(long) << " bytes";
    if (sizeof(long) == 4)
        std::cout << "  (Windows/Android 32-bit 模型)";
    else if (sizeof(long) == 8)
        std::cout << "  (Linux/macOS LP64 模型)";
    std::cout << std::endl;

    std::cout << "  wchar_t:     " << sizeof(wchar_t) << " bytes";
    if (sizeof(wchar_t) == 2)
        std::cout << "  (Windows: UTF-16)";
    else if (sizeof(wchar_t) == 4)
        std::cout << "  (Linux/macOS: UTF-32)";
    std::cout << std::endl;

    std::cout << "  size_t:      " << sizeof(size_t) << " bytes" << std::endl;
    std::cout << "  pointer:     " << sizeof(void*) << " bytes" << std::endl;

    // 推荐：使用 <cstdint> 中的固定宽度类型
    std::cout << "\n[推荐的固定宽度类型]" << std::endl;
    std::cout << "  int8_t:      " << sizeof(int8_t) << " bytes" << std::endl;
    std::cout << "  int16_t:     " << sizeof(int16_t) << " bytes" << std::endl;
    std::cout << "  int32_t:     " << sizeof(int32_t) << " bytes" << std::endl;
    std::cout << "  int64_t:     " << sizeof(int64_t) << " bytes" << std::endl;
}

// ============================================================
// 第三部分：字节序检测
// ============================================================
void detectEndianness() {
    std::cout << "\n========== 字节序 ==========" << std::endl;

    // 方法 1：C++20 标准方法
#if HAS_CPP20_BIT
    if constexpr (std::endian::native == std::endian::little) {
        std::cout << "字节序 (C++20): Little-Endian (小端)" << std::endl;
    } else if constexpr (std::endian::native == std::endian::big) {
        std::cout << "字节序 (C++20): Big-Endian (大端)" << std::endl;
    } else {
        std::cout << "字节序 (C++20): Mixed (混合端序)" << std::endl;
    }
#endif

    // 方法 2：运行时检测（兼容 C++17 及更早标准）
    // 原理：将整数 1 放入内存，看第一个字节是 1 还是 0
    uint32_t testVal = 1;
    uint8_t firstByte = *reinterpret_cast<uint8_t*>(&testVal);
    if (firstByte == 1) {
        std::cout << "字节序 (运行时): Little-Endian (小端)" << std::endl;
    } else {
        std::cout << "字节序 (运行时): Big-Endian (大端)" << std::endl;
    }

    // 验证：显示一个多字节整数在内存中的实际排列
    uint32_t demo = 0x01020304;
    uint8_t* bytes = reinterpret_cast<uint8_t*>(&demo);
    std::cout << "\n验证: 0x01020304 在内存中的字节排列:" << std::endl;
    std::cout << "  地址低→高: ";
    for (int i = 0; i < 4; i++) {
        std::cout << "0x" << std::hex
                  << static_cast<int>(bytes[i]);
        if (i < 3) std::cout << " ";
    }
    std::cout << std::dec << std::endl;
    std::cout << "  (小端应为: 0x4 0x3 0x2 0x1)" << std::endl;
    std::cout << "  (大端应为: 0x1 0x2 0x3 0x4)" << std::endl;
}

// ============================================================
// 第四部分：字符编码环境检测
// ============================================================
void detectEncoding() {
    std::cout << "\n========== 字符编码环境 ==========" << std::endl;

    // 检测 char 的符号性（影响字符比较）
    // char 是 signed 还是 unsigned 取决于平台和编译器
    if (static_cast<char>(0x80) < 0) {
        std::cout << "char 符号性:  signed (有符号)" << std::endl;
        std::cout << "  → 高位字符(>127)会被当作负数" << std::endl;
    } else {
        std::cout << "char 符号性:  unsigned (无符号)" << std::endl;
        std::cout << "  → ARM 平台通常默认 unsigned" << std::endl;
    }

    // UTF-8 字符串长度 vs 字节数
    std::string utf8Hello = u8"你好";
    std::cout << "\nUTF-8 字符串 \"你好\":" << std::endl;
    std::cout << "  strlen/size(): " << utf8Hello.size()
              << " bytes" << std::endl;
    std::cout << "  实际字符数:    2 个汉字" << std::endl;
    std::cout << "  每个汉字占:    " << utf8Hello.size() / 2
              << " bytes (UTF-8 编码)" << std::endl;

    // 显示 UTF-8 编码的字节内容
    std::cout << "  字节内容:      ";
    for (unsigned char c : utf8Hello) {
        std::cout << "0x" << std::hex
                  << static_cast<int>(c) << " ";
    }
    std::cout << std::dec << std::endl;

#if defined(_WIN32)
    std::cout << "\n平台编码提示:" << std::endl;
    std::cout << "  Windows 默认使用 ANSI 代码页" << std::endl;
    std::cout << "  建议使用 Wide API (wchar_t/UTF-16) 处理文件名" << std::endl;
    std::cout << "  或在 Windows 10 1903+ 启用 UTF-8 代码页" << std::endl;
#else
    std::cout << "\n平台编码提示:" << std::endl;
    std::cout << "  当前平台默认使用 UTF-8 编码" << std::endl;
    std::cout << "  文件名、终端输出均为 UTF-8" << std::endl;
#endif
}

// ============================================================
// 主程序
// ============================================================
int main() {
    std::cout << "╔══════════════════════════════════╗" << std::endl;
    std::cout << "║   跨平台特性检测诊断工具 v1.0   ║" << std::endl;
    std::cout << "╚══════════════════════════════════╝" << std::endl;

    detectOS();
    detectDataTypes();
    detectEndianness();
    detectEncoding();

    std::cout << "\n========== 诊断完成 ==========" << std::endl;
    std::cout << "将以上输出保存下来，可用于排查跨平台问题。" << std::endl;

    return 0;
}
```

### 预期输出对比

| 检测项 | Windows x64 | Linux x64 | macOS ARM64 | Android ARM64 |
|--------|------------|-----------|-------------|---------------|
| `sizeof(long)` | 4 | 8 | 8 | 8 |
| `sizeof(wchar_t)` | 2 | 4 | 4 | 4 |
| `sizeof(void*)` | 8 | 8 | 8 | 8 |
| 字节序 | Little | Little | Little | Little |
| `char` 符号性 | signed | signed | signed | **unsigned** |

## 实验 2：跨平台文件操作综合练习

### 场景

你需要编写一个工具，在用户的临时目录下创建测试文件、写入文本和二进制数据、再读回验证。这个看似简单的任务实际上要正确处理路径分隔符、临时目录获取、文本编码和二进制数据的字节序。

### 完整代码

```cpp
// cross_file.cpp
// 跨平台文件操作综合练习
#include <cstdint>
#include <cstring>
#include <filesystem>
#include <fstream>
#include <iostream>
#include <string>
#include <vector>

namespace fs = std::filesystem;

// 安全的小端序写入函数（从第 02 节学到的知识）
void writeLE32(std::ofstream& out, uint32_t value) {
    uint8_t bytes[4];
    bytes[0] = static_cast<uint8_t>(value & 0xFF);         // 最低字节
    bytes[1] = static_cast<uint8_t>((value >> 8) & 0xFF);
    bytes[2] = static_cast<uint8_t>((value >> 16) & 0xFF);
    bytes[3] = static_cast<uint8_t>((value >> 24) & 0xFF);  // 最高字节
    out.write(reinterpret_cast<const char*>(bytes), 4);
}

// 安全的小端序读取函数
uint32_t readLE32(std::ifstream& in) {
    uint8_t bytes[4];
    in.read(reinterpret_cast<char*>(bytes), 4);
    return static_cast<uint32_t>(bytes[0])
         | (static_cast<uint32_t>(bytes[1]) << 8)
         | (static_cast<uint32_t>(bytes[2]) << 16)
         | (static_cast<uint32_t>(bytes[3]) << 24);
}

int main() {
    std::cout << "=== 跨平台文件操作综合练习 ===" << std::endl;

    // --------------------------------------------------------
    // 步骤 1：获取跨平台临时目录
    // --------------------------------------------------------
    // std::filesystem::temp_directory_path() 在各平台返回：
    //   Windows: C:\Users\xxx\AppData\Local\Temp
    //   Linux:   /tmp
    //   macOS:   /var/folders/xx/...T/
    //   Android: /data/local/tmp (需要 root 或 shell 权限)
    fs::path testDir = fs::temp_directory_path() / "kr2_cross_platform_test";
    fs::create_directories(testDir);
    std::cout << "测试目录: " << testDir.string() << std::endl;
    std::cout << "路径分隔符: '" << fs::path::preferred_separator
              << "'" << std::endl;

    // --------------------------------------------------------
    // 步骤 2：写入文本文件（注意编码）
    // --------------------------------------------------------
    fs::path textFile = testDir / "hello.txt";
    {
        // 以文本模式写入 — 注意 Windows 会自动将 \n 转为 \r\n
        std::ofstream out(textFile);
        if (!out.is_open()) {
            std::cerr << "无法创建文件: " << textFile << std::endl;
            return 1;
        }
        out << "Hello, Cross-Platform World!" << std::endl;
        out << u8"你好，跨平台世界！" << std::endl;
        std::cout << "✓ 文本文件已写入: " << textFile.filename() << std::endl;
    }

    // --------------------------------------------------------
    // 步骤 3：写入二进制文件（使用安全的字节序函数）
    // --------------------------------------------------------
    fs::path binFile = testDir / "data.bin";
    {
        // ⚠️ 二进制文件必须用 std::ios::binary 打开
        // 否则 Windows 会进行换行符转换，破坏数据
        std::ofstream out(binFile, std::ios::binary);
        if (!out.is_open()) {
            std::cerr << "无法创建文件: " << binFile << std::endl;
            return 1;
        }

        // 写入一个简单的文件头：魔数 + 版本号 + 数据条目数
        // 全部使用小端序，确保跨平台一致
        writeLE32(out, 0x4B523254);  // 魔数 "KR2T" (ASCII)
        writeLE32(out, 1);           // 版本号 1
        writeLE32(out, 3);           // 3 条数据

        // 写入 3 条数据记录
        writeLE32(out, 100);
        writeLE32(out, 200);
        writeLE32(out, 300);

        std::cout << "✓ 二进制文件已写入: " << binFile.filename() << std::endl;
    }

    // --------------------------------------------------------
    // 步骤 4：读回文本文件并验证
    // --------------------------------------------------------
    {
        std::ifstream in(textFile);
        std::string line;
        std::cout << "\n--- 读回文本文件 ---" << std::endl;
        int lineNum = 0;
        while (std::getline(in, line)) {
            lineNum++;
            std::cout << "  第 " << lineNum << " 行: " << line << std::endl;
        }
    }

    // --------------------------------------------------------
    // 步骤 5：读回二进制文件并验证
    // --------------------------------------------------------
    {
        std::ifstream in(binFile, std::ios::binary);
        std::cout << "\n--- 读回二进制文件 ---" << std::endl;

        uint32_t magic = readLE32(in);
        uint32_t version = readLE32(in);
        uint32_t count = readLE32(in);

        // 验证魔数
        std::cout << "  魔数: 0x" << std::hex << magic << std::dec;
        if (magic == 0x4B523254) {
            std::cout << " ✓ (KR2T)" << std::endl;
        } else {
            std::cout << " ✗ 不匹配!" << std::endl;
        }
        std::cout << "  版本: " << version << std::endl;
        std::cout << "  条目: " << count << std::endl;

        for (uint32_t i = 0; i < count; i++) {
            uint32_t value = readLE32(in);
            std::cout << "  数据[" << i << "]: " << value << std::endl;
        }
    }

    // --------------------------------------------------------
    // 步骤 6：遍历目录并显示文件信息
    // --------------------------------------------------------
    std::cout << "\n--- 目录内容 ---" << std::endl;
    for (const auto& entry : fs::directory_iterator(testDir)) {
        auto fileSize = entry.file_size();
        std::cout << "  " << entry.path().filename().string()
                  << "  (" << fileSize << " bytes)" << std::endl;
    }

    // --------------------------------------------------------
    // 步骤 7：清理
    // --------------------------------------------------------
    fs::remove_all(testDir);
    std::cout << "\n✓ 测试目录已清理" << std::endl;

    return 0;
}
```

## 实验 3：迷你 XP3 文件头解析器

### 场景

KrKr2 引擎使用 XP3 格式来打包游戏资源（脚本、图片、音频等）。XP3 文件的前 11 个字节是一个固定的签名（魔数），后面是索引区偏移量等信息。

这个实验让你综合运用文件操作、字节序读取和编码知识，写一个能读取 XP3 文件头的小工具。即使你手上没有 XP3 文件，我们也会先生成一个模拟文件来测试。

### 完整代码

```cpp
// xp3_header_reader.cpp
// 迷你 XP3 文件头解析器 — 综合练习
#include <cstdint>
#include <cstring>
#include <filesystem>
#include <fstream>
#include <iostream>
#include <string>

namespace fs = std::filesystem;

// ============================================================
// 小端序读写工具函数（复用前面的知识）
// ============================================================
void writeLE32(std::ofstream& out, uint32_t value) {
    uint8_t bytes[4];
    bytes[0] = static_cast<uint8_t>(value & 0xFF);
    bytes[1] = static_cast<uint8_t>((value >> 8) & 0xFF);
    bytes[2] = static_cast<uint8_t>((value >> 16) & 0xFF);
    bytes[3] = static_cast<uint8_t>((value >> 24) & 0xFF);
    out.write(reinterpret_cast<const char*>(bytes), 4);
}

void writeLE64(std::ofstream& out, uint64_t value) {
    // 64 位值：先写低 32 位，再写高 32 位
    writeLE32(out, static_cast<uint32_t>(value & 0xFFFFFFFF));
    writeLE32(out, static_cast<uint32_t>(value >> 32));
}

uint32_t readLE32(std::ifstream& in) {
    uint8_t bytes[4];
    in.read(reinterpret_cast<char*>(bytes), 4);
    return static_cast<uint32_t>(bytes[0])
         | (static_cast<uint32_t>(bytes[1]) << 8)
         | (static_cast<uint32_t>(bytes[2]) << 16)
         | (static_cast<uint32_t>(bytes[3]) << 24);
}

uint64_t readLE64(std::ifstream& in) {
    uint32_t lo = readLE32(in);
    uint32_t hi = readLE32(in);
    return static_cast<uint64_t>(lo)
         | (static_cast<uint64_t>(hi) << 32);
}

// ============================================================
// XP3 文件头结构定义
// ============================================================
// XP3 文件格式的签名（魔数）：11 字节
// 实际内容是 "XP3\r\n \n\x1a\x8b\x67\x01"
static const uint8_t XP3_SIGNATURE[] = {
    0x58, 0x50, 0x33, 0x0D, 0x0A, 0x20, 0x0A,
    0x1A, 0x8B, 0x67, 0x01
};
static const size_t XP3_SIG_SIZE = sizeof(XP3_SIGNATURE);

// ============================================================
// 生成模拟 XP3 文件（用于测试）
// ============================================================
bool createMockXP3(const fs::path& path) {
    std::ofstream out(path, std::ios::binary);
    if (!out.is_open()) return false;

    // 写入 XP3 签名（11 字节）
    out.write(reinterpret_cast<const char*>(XP3_SIGNATURE),
              XP3_SIG_SIZE);

    // 写入索引区偏移量（8 字节，小端序）
    // 假设索引区从文件偏移 0x100 开始
    writeLE64(out, 0x0000000000000100ULL);

    // 写入一些填充数据模拟文件内容
    // 在真实 XP3 中，这里是文件数据段
    for (int i = 0; i < 237; i++) {
        out.put(static_cast<char>(i & 0xFF));
    }

    std::cout << "✓ 已生成模拟 XP3 文件: " << path.filename() << std::endl;
    return true;
}

// ============================================================
// 解析 XP3 文件头
// ============================================================
bool parseXP3Header(const fs::path& path) {
    std::ifstream in(path, std::ios::binary);
    if (!in.is_open()) {
        std::cerr << "无法打开文件: " << path << std::endl;
        return false;
    }

    // 获取文件大小
    in.seekg(0, std::ios::end);
    auto fileSize = in.tellg();
    in.seekg(0, std::ios::beg);

    std::cout << "\n=== XP3 文件头解析 ===" << std::endl;
    std::cout << "文件: " << path.filename() << std::endl;
    std::cout << "大小: " << fileSize << " bytes" << std::endl;

    // 读取并验证签名
    uint8_t sig[XP3_SIG_SIZE];
    in.read(reinterpret_cast<char*>(sig), XP3_SIG_SIZE);

    if (in.gcount() != static_cast<std::streamsize>(XP3_SIG_SIZE)) {
        std::cerr << "文件太小，不是有效的 XP3 文件" << std::endl;
        return false;
    }

    std::cout << "\n签名字节: ";
    for (size_t i = 0; i < XP3_SIG_SIZE; i++) {
        std::cout << "0x" << std::hex
                  << static_cast<int>(sig[i]) << " ";
    }
    std::cout << std::dec << std::endl;

    // 比较签名
    bool validSig = (std::memcmp(sig, XP3_SIGNATURE, XP3_SIG_SIZE) == 0);
    std::cout << "签名验证: " << (validSig ? "✓ 有效" : "✗ 无效")
              << std::endl;

    if (!validSig) return false;

    // 读取索引区偏移量（8 字节小端序）
    uint64_t indexOffset = readLE64(in);
    std::cout << "索引偏移: 0x" << std::hex << indexOffset
              << std::dec << " (" << indexOffset << ")" << std::endl;

    // 验证偏移量是否在文件范围内
    if (static_cast<std::streamoff>(indexOffset) > fileSize) {
        std::cout << "⚠️ 索引偏移超出文件大小（模拟文件正常现象）"
                  << std::endl;
    } else {
        std::cout << "✓ 索引偏移在文件范围内" << std::endl;
    }

    return true;
}

// ============================================================
// 主程序
// ============================================================
int main() {
    fs::path testDir = fs::temp_directory_path() / "kr2_xp3_test";
    fs::create_directories(testDir);
    fs::path xp3Path = testDir / "test.xp3";

    // 生成模拟 XP3 文件
    if (!createMockXP3(xp3Path)) {
        std::cerr << "生成测试文件失败" << std::endl;
        return 1;
    }

    // 解析文件头
    parseXP3Header(xp3Path);

    // 清理
    fs::remove_all(testDir);
    std::cout << "\n✓ 测试完成，临时文件已清理" << std::endl;

    return 0;
}
```

## 本章总结

本章（跨平台基础）覆盖了三大核心主题。以下是每个主题的要点回顾和关键教训：

### 文件系统差异（第 01 节）

| 差异点 | Windows | Linux | macOS | Android |
|--------|---------|-------|-------|---------|
| 路径分隔符 | `\`（也接受 `/`） | `/` | `/` | `/` |
| 大小写敏感 | ❌ 不敏感 | ✅ 敏感 | ❌ 默认不敏感 | ✅ 敏感 |
| 最大路径 | 260 字符 (默认) | 4096 字符 | 1024 字符 | 4096 字符 |
| 文件锁定 | 强制锁定 | 建议锁定 | 建议锁定 | 建议锁定 |

**核心教训**：使用 `std::filesystem` 和 `/` 拼接路径；不要依赖大小写不敏感；二进制文件必须用 `std::ios::binary` 打开。

### 字节序与数据类型（第 02 节）

| 差异点 | Windows x64 | Linux x64 | macOS ARM64 | Android ARM64 |
|--------|------------|-----------|-------------|---------------|
| `sizeof(long)` | 4 | 8 | 8 | 8 |
| `sizeof(wchar_t)` | 2 (UTF-16) | 4 (UTF-32) | 4 (UTF-32) | 4 (UTF-32) |
| 字节序 | 小端 | 小端 | 小端 | 小端 |
| `char` 符号性 | signed | signed | signed | **unsigned** |

**核心教训**：不要用 `long` 和 `wchar_t` 做跨平台数据传输；使用 `<cstdint>` 固定宽度类型；二进制文件格式始终使用显式字节序读写函数。

### 字符编码（第 03 节）

| 差异点 | Windows | Linux | macOS | Android |
|--------|---------|-------|-------|---------|
| 默认编码 | ANSI 代码页 | UTF-8 | UTF-8 | UTF-8 |
| 文件名编码 | UTF-16 (内核) | UTF-8 | UTF-8 | UTF-8 |
| `wchar_t` 编码 | UTF-16 | UTF-32 | UTF-32 | UTF-32 |
| 控制台编码 | 系统代码页 | UTF-8 | UTF-8 | UTF-8 |

**核心教训**：Windows 是编码问题的重灾区；使用 Wide API 或显式 UTF-8 转换；内部统一使用 UTF-8，在 API 边界转换；KrKr2 还需处理日文 Shift-JIS 编码的兼容。

### 跨平台开发的黄金法则

```

│  5. 线程：用 std::thread，平台特殊需求用条件编译               │
│  6. 测试：在所有目标平台上跑测试，不要只在 Windows 上测         │

```

## 练习题与答案

### 题目 1：平台特征判断

以下代码在 Windows x64 (MSVC) 和 Linux x64 (GCC) 上编译运行，分别输出什么？

```cpp
#include <iostream>
int main() {
    std::cout << sizeof(long) << " " << sizeof(wchar_t) << std::endl;
    return 0;
}
```

<details>
<summary>查看答案</summary>

- **Windows x64 (MSVC)**：输出 `4 2`
  - `long` 在 MSVC 中始终是 4 字节（LLP64 数据模型，也叫 ILP32LL）
  - `wchar_t` 在 Windows 上是 2 字节（用于存储 UTF-16 编码单元）
- **Linux x64 (GCC)**：输出 `8 4`
  - `long` 在 64 位 Linux 中是 8 字节（LP64 数据模型）
  - `wchar_t` 在 Linux 上是 4 字节（用于存储 UTF-32 码点）

**教训**：永远不要用 `long` 来存储需要跨平台传输的数据。使用 `int32_t` 保证 4 字节，或 `int64_t` 保证 8 字节。

</details>

### 题目 2：字节序转换函数

编写一个函数 `readLE16`，从 `uint8_t*` 指针读取一个小端序的 16 位无符号整数。要求在大端和小端平台上都能正确工作。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdint>
#include <iostream>

uint16_t readLE16(const uint8_t* data) {
    // 小端序：低字节在低地址
    // data[0] 是最低有效字节，data[1] 是最高有效字节
    return static_cast<uint16_t>(data[0])
         | (static_cast<uint16_t>(data[1]) << 8);
}

// 测试
int main() {
    // 小端序表示的 0x1234：低字节 0x34 在前，高字节 0x12 在后
    uint8_t data[] = {0x34, 0x12};
    uint16_t value = readLE16(data);
    std::cout << "0x" << std::hex << value << std::endl;
    // 无论在大端还是小端平台上，都应输出: 0x1234
    return 0;
}
```

**关键点**：
- **不使用 `reinterpret_cast`**——直接将 `uint8_t*` 强转为 `uint16_t*` 会导致未定义行为（对齐问题 + 字节序依赖）
- **显式按字节组装**——手动控制每个字节的位置，不依赖平台字节序
- **为什么不用 `memcpy`**——`memcpy` 虽然可以避免对齐问题，但仍然依赖平台字节序

</details>

### 题目 3：跨平台路径 Bug 修复

以下代码在 Linux 上运行时打不开文件，但在 Windows 上正常。找出问题并修复。

```cpp
#include <fstream>

int main() {
    std::ifstream file("C:\\Data\\Scripts\\startup.tjs");
    if (file.is_open()) {
        // 处理文件...
    }
    return 0;
}
```

<details>
<summary>查看答案</summary>

**问题分析**：
1. `C:\Data\Scripts\startup.tjs` 是 Windows 绝对路径——Linux 上不存在 `C:` 盘符概念
2. 即使路径存在，反斜杠 `\` 在 Linux 上不是路径分隔符（它是文件名的一部分）
3. Linux 文件系统区分大小写，如果实际文件名是 `Startup.TJS`，用 `startup.tjs` 找不到

**修复方案**：

```cpp
#include <filesystem>
#include <fstream>
#include <iostream>
#include <algorithm>
#include <cctype>

namespace fs = std::filesystem;

// 方案 1：使用 std::filesystem 的 / 操作符拼接路径（推荐）
void solution1() {
    // 使用相对路径 + 跨平台分隔符
    fs::path scriptPath = fs::path("Data") / "Scripts" / "startup.tjs";
    std::ifstream file(scriptPath);
    if (file.is_open()) {
        std::cout << "方案 1: 文件打开成功" << std::endl;
    }
}

// 方案 2：如果文件名大小写可能不匹配，实现不区分大小写的查找
std::ifstream openCaseInsensitive(const fs::path& dir,
                                   const std::string& filename) {
    if (!fs::exists(dir)) return std::ifstream();

    std::string lowerTarget = filename;
    std::transform(lowerTarget.begin(), lowerTarget.end(),
                   lowerTarget.begin(),
                   [](unsigned char c) { return std::tolower(c); });

    for (const auto& entry : fs::directory_iterator(dir)) {
        std::string entryName = entry.path().filename().string();
        std::string lowerEntry = entryName;
        std::transform(lowerEntry.begin(), lowerEntry.end(),
                       lowerEntry.begin(),
                       [](unsigned char c) { return std::tolower(c); });

        if (lowerEntry == lowerTarget) {
            return std::ifstream(entry.path());
        }
    }
    return std::ifstream(); // 未找到
}

int main() {
    solution1();

    // 方案 2 用法：
    auto file = openCaseInsensitive("Data/Scripts", "startup.tjs");
    if (file.is_open()) {
        std::cout << "方案 2: 文件打开成功（大小写不敏感）" << std::endl;
    }
    return 0;
}
```

**总结**：
- 绝不要在代码中硬编码 Windows 绝对路径
- 使用 `std::filesystem::path` 的 `/` 操作符自动处理分隔符
- 如果需要兼容大小写不敏感的文件名引用（比如 KrKr2 就需要），实现一个不区分大小写的文件查找函数

</details>

### 题目 4：二进制文件 Bug 排查

以下代码将一个 `uint32_t` 值 `0x12345678` 写入文件，再读回来。在 Windows 上写入的文件复制到 Linux 上读取，输出了错误的值。为什么？

```cpp
#include <cstdint>
#include <fstream>
#include <iostream>

int main() {
    // 写入
    {
        std::ofstream out("data.bin");  // ← 注意这里
        uint32_t value = 0x12345678;
        out.write(reinterpret_cast<const char*>(&value), sizeof(value));
    }
    // 读取
    {
        std::ifstream in("data.bin");  // ← 注意这里
        uint32_t value;
        in.read(reinterpret_cast<char*>(&value), sizeof(value));
        std::cout << "0x" << std::hex << value << std::endl;
    }
    return 0;
}
```

<details>
<summary>查看答案</summary>

**问题 1：没有使用 `std::ios::binary` 模式**

在 Windows 上，文本模式（默认模式）会做换行符转换：
- 写入时：`0x0A`（LF）→ `0x0D 0x0A`（CR+LF）
- 读取时：`0x0D 0x0A` → `0x0A`

如果 `0x12345678` 的四个字节中恰好包含 `0x0A`（在小端序中，字节排列为 `78 56 34 12`，没有 `0x0A`），那这个特定值可能不受影响。但如果值是 `0x1234560A`，写入时会变成 5 个字节（多了一个 `0x0D`），数据就被破坏了。

**问题 2：直接 `reinterpret_cast` 读写整个值**

虽然 Windows 和 Linux x64 都是小端序（所以这个特定例子中字节序不是问题），但这是不好的习惯。如果将来需要支持大端序平台，这段代码就会产生错误结果。

**正确写法**：

```cpp
#include <cstdint>
#include <fstream>
#include <iostream>

// 显式小端序写入
void writeLE32(std::ofstream& out, uint32_t value) {
    uint8_t bytes[4] = {
        static_cast<uint8_t>(value & 0xFF),
        static_cast<uint8_t>((value >> 8) & 0xFF),
        static_cast<uint8_t>((value >> 16) & 0xFF),
        static_cast<uint8_t>((value >> 24) & 0xFF)
    };
    out.write(reinterpret_cast<const char*>(bytes), 4);
}

// 显式小端序读取
uint32_t readLE32(std::ifstream& in) {
    uint8_t bytes[4];
    in.read(reinterpret_cast<char*>(bytes), 4);
    return static_cast<uint32_t>(bytes[0])
         | (static_cast<uint32_t>(bytes[1]) << 8)
         | (static_cast<uint32_t>(bytes[2]) << 16)
         | (static_cast<uint32_t>(bytes[3]) << 24);
}

int main() {
    {
        // ✅ 使用 binary 模式 + 显式字节序
        std::ofstream out("data.bin", std::ios::binary);
        writeLE32(out, 0x12345678);
    }
    {
        std::ifstream in("data.bin", std::ios::binary);
        uint32_t value = readLE32(in);
        std::cout << "0x" << std::hex << value << std::endl;
        // 在任何平台上都输出: 0x12345678
    }
    return 0;
}
```

**两个关键修复**：
1. 加上 `std::ios::binary` 防止换行符转换
2. 使用显式字节序函数防止大端/小端差异

</details>
