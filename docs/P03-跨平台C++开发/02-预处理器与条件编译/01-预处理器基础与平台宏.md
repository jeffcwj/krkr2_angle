## 预处理器基础回顾

C++ 编译过程的第一步是**预处理**（Preprocessing），由预处理器执行。预处理器处理所有以 `#` 开头的指令，在编译器看到代码之前就完成了文本替换和条件选择。

```
源代码 (.cpp) → [预处理器] → 预处理后代码 → [编译器] → 目标文件 (.o/.obj)
```

### 条件编译指令一览

| 指令 | 含义 | 示例 |
|------|------|------|
| `#define MACRO` | 定义宏（无值） | `#define _DEBUG` |
| `#define MACRO value` | 定义宏（有值） | `#define VERSION 3` |
| `#undef MACRO` | 取消宏定义 | `#undef NDEBUG` |
| `#ifdef MACRO` | 如果宏**已定义** | `#ifdef _WIN32` |
| `#ifndef MACRO` | 如果宏**未定义** | `#ifndef NDEBUG` |
| `#if expr` | 如果表达式为真 | `#if VERSION >= 3` |
| `#elif expr` | 否则如果 | `#elif defined(__linux__)` |
| `#else` | 否则 | `#else` |
| `#endif` | 结束条件块 | `#endif` |

### `#ifdef` vs `#if defined` 的区别

这两种写法功能相同，但 `#if defined` 更灵活：

```cpp
// 方式1: #ifdef — 只能检查单个宏
#ifdef _WIN32
    // Windows 代码
#endif

// 方式2: #if defined — 可以组合多个条件
#if defined(_WIN32)
    // 等价于 #ifdef _WIN32
#endif

// #if defined 的优势：支持逻辑运算
#if defined(_WIN32) && !defined(_WIN64)
    // 32位 Windows
#endif

#if defined(__linux__) || defined(__APPLE__)
    // Unix-like 系统（Linux 或 macOS）
#endif

#if defined(__ANDROID__) && __ANDROID_API__ >= 24
    // Android API 24+ (Android 7.0 Nougat)
#endif
```

**推荐做法：** 简单的单宏判断用 `#ifdef`，需要组合逻辑时用 `#if defined()`。

## 各平台预定义宏速查表

### 操作系统宏

编译器会自动定义这些宏，无需手动 `#define`：

| 宏名 | Windows (MSVC) | Linux (GCC/Clang) | macOS (Clang) | Android (NDK Clang) |
|------|---------------|-------------------|---------------|-------------------|
| `_WIN32` | ✅ | ❌ | ❌ | ❌ |
| `_WIN64` | ✅ (64位) | ❌ | ❌ | ❌ |
| `__linux__` | ❌ | ✅ | ❌ | ✅ |
| `__APPLE__` | ❌ | ❌ | ✅ | ❌ |
| `__MACH__` | ❌ | ❌ | ✅ | ❌ |
| `__ANDROID__` | ❌ | ❌ | ❌ | ✅ |
| `__ANDROID_API__` | ❌ | ❌ | ❌ | ✅ (数字，如 24) |
| `__unix__` | ❌ | ✅ | ✅ | ✅ |
| `__posix__` | ❌ | ✅ | ✅ | ✅ |

**注意：** Android 同时定义了 `__linux__` 和 `__ANDROID__`。因此判断 Android 时必须优先检查 `__ANDROID__`：

```cpp
// ❌ 错误：Linux 分支会匹配 Android
#ifdef __linux__
    // 这段代码在 Android 上也会执行！
#endif

// ✅ 正确：先检查 Android
#if defined(__ANDROID__)
    // Android 专用代码
#elif defined(__linux__)
    // 纯 Linux（非 Android）
#elif defined(__APPLE__)
    // macOS / iOS
#elif defined(_WIN32)
    // Windows
#endif
```

### 编译器宏

| 宏名 | 含义 | 示例值 |
|------|------|--------|
| `_MSC_VER` | MSVC 编译器版本 | `1940`（VS 2022 17.10） |
| `__GNUC__` | GCC 主版本号 | `13` |
| `__GNUC_MINOR__` | GCC 次版本号 | `2` |
| `__clang__` | Clang 编译器 | 已定义 |
| `__clang_major__` | Clang 主版本号 | `17` |

```cpp
// 按编译器选择不同的特性
#if defined(_MSC_VER)
    // MSVC 特有: __declspec, #pragma warning
    #pragma warning(disable: 4996) // 禁用 deprecated 警告
#elif defined(__clang__)
    // Clang 特有: __has_feature, __attribute__
    #pragma clang diagnostic ignored "-Wdeprecated"
#elif defined(__GNUC__)
    // GCC 特有: __attribute__, __builtin_*
    #pragma GCC diagnostic ignored "-Wdeprecated"
#endif
```

### 架构宏

| 宏名 | 含义 |
|------|------|
| `__x86_64__` 或 `_M_X64` | x86-64 架构 |
| `__i386__` 或 `_M_IX86` | x86 32位架构 |
| `__aarch64__` 或 `_M_ARM64` | ARM 64位架构 |
| `__arm__` 或 `_M_ARM` | ARM 32位架构 |

```cpp
// KrKr2 支持的架构
#if defined(__x86_64__) || defined(_M_X64)
    constexpr const char* ARCH_NAME = "x86_64";
#elif defined(__aarch64__) || defined(_M_ARM64)
    constexpr const char* ARCH_NAME = "arm64";
#else
    #error "Unsupported architecture"
#endif
```

### C++ 标准版本宏

| 宏值 | 对应标准 |
|------|---------|
| `__cplusplus >= 201703L` | C++17 |
| `__cplusplus >= 202002L` | C++20 |
| `__cplusplus >= 202302L` | C++23 |

```cpp
#if __cplusplus >= 202002L
    #include <bit>    // C++20: std::endian
    #include <span>   // C++20: std::span
#elif __cplusplus >= 201703L
    #include <optional>  // C++17
    #include <variant>   // C++17
#endif
```

**注意：** MSVC 默认不正确设置 `__cplusplus`（始终返回 `199711L`），需要添加编译选项 `/Zc:__cplusplus`。KrKr2 使用 CMake 的 `CMAKE_CXX_STANDARD 17`，CMake 会自动处理这个问题。
