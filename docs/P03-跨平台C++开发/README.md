## 模块简介

KrKr2 模拟器需要在 Windows、Linux、macOS 和 Android 四个平台上运行同一份核心代码。这意味着开发者必须掌握：操作系统差异（文件系统、线程模型、字节序）、预处理器条件编译、平台抽象层（PAL）设计模式，以及 Android 特有的 NDK/JNI 互操作技术。

本模块从零开始讲解跨平台 C++ 开发的核心知识，并以 KrKr2 项目的 `cpp/core/environ/` 平台抽象层作为贯穿始终的真实案例。

## 与 KrKr2 项目的关联

- **`cpp/core/environ/`** — 项目的平台抽象层（PAL），包含 `win32/`、`linux/`、`android/`、`apple/`、`sdl/` 五个平台实现目录
- **`platforms/`** — 各平台的入口点和构建配置
- **`CMakeLists.txt`** — 使用自定义 `WINDOWS`/`LINUX`/`MACOS`/`ANDROID` 变量做平台条件编译
- **`platforms/android/cpp/`** — JNI 桥接代码，Java↔C++ 互调的实际案例
- **`cmake/vcpkg_android.cmake`** — Android 交叉编译的 toolchain 叠加

学完本模块后，你将能够阅读并理解 KrKr2 中任何与平台相关的代码，并有能力为项目添加新平台支持（如 iOS）。
