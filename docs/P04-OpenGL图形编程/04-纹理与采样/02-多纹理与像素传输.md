## 场景引入：视频播放器需要同时使用三张纹理

想象你在做一个视频播放器——FFmpeg 解码出来的视频帧通常不是 RGBA 格式，而是 **YUV** 格式（一个亮度通道 Y + 两个色度通道 U/V）。你需要把 Y、U、V 三个分量分别上传到三张独立的纹理，然后在片元着色器中同时采样这三张纹理，做 YUV→RGB 转换后输出。

问题来了：一个 `sampler2D` uniform 怎么知道该读哪张纹理？`glBindTexture` 一次只能绑定一张纹理到 `GL_TEXTURE_2D`，那三张纹理怎么同时使用？

答案是**纹理单元（Texture Unit）**——OpenGL 提供了多个"插槽"，每个插槽可以绑定一张纹理，sampler 通过"插槽编号"而非"纹理 ID"来读取数据。

## 一、纹理单元：多纹理采样的核心机制

### 1.1 纹理单元是什么

纹理单元是 GPU 上的"纹理插槽"。你可以把它想象成一排插座：

```
纹理单元：  [单元0]   [单元1]   [单元2]   [单元3]  ...  [单元N]
              ↑         ↑         ↑
           绑定tex_A  绑定tex_B  绑定tex_C

着色器中：
  uniform sampler2D uTexY;  // 值=0 → 从单元0读
  uniform sampler2D uTexU;  // 值=1 → 从单元1读
  uniform sampler2D uTexV;  // 值=2 → 从单元2读
```

**关键理解**：
- `sampler2D` uniform 的值不是纹理对象 ID，而是**纹理单元编号**（0, 1, 2, ...）
- `glBindTexture` 绑定的是"当前激活单元"上的纹理
- 要切换"当前激活单元"，用 `glActiveTexture(GL_TEXTURE0 + i)`

### 1.2 多纹理绑定的三步流程

每绑定一张纹理到一个单元，需要三步：

```
第1步：glActiveTexture(GL_TEXTURE0 + i)   ← 激活第i个单元
第2步：glBindTexture(GL_TEXTURE_2D, texID) ← 在该单元上绑定纹理
第3步：glUniform1i(samplerLoc, i)          ← 告诉着色器的sampler从第i个单元读
```

### 1.3 纹理单元数量限制

| 平台 | 最低保证 | 典型值 | 查询方式 |
|------|----------|--------|----------|
| Desktop GL 3.3+ | 16 | 32-80 | `glGetIntegerv(GL_MAX_TEXTURE_IMAGE_UNITS, &n)` |
| OpenGL ES 3.0 | 16 | 16-32 | 同上 |
| OpenGL ES 2.0 | **8** | 8-16 | 同上 |

ES 2.0 保证至少 8 个纹理单元。对于视觉小说引擎来说绰绰有余（YUV 只需要 3 个）。

### 1.4 示例1：三纹理绑定（YUV 视频播放场景）

```cpp
#include <GLES2/gl2.h>

// 将三张纹理分别绑定到单元 0、1、2
void BindYUVTextures(GLuint program, GLuint texY, GLuint texU, GLuint texV) {
    glUseProgram(program); // 激活着色器程序

    // 查找三个 sampler 的 location
    GLint locY = glGetUniformLocation(program, "uTexY"); // Y亮度采样器
    GLint locU = glGetUniformLocation(program, "uTexU"); // U色度采样器
    GLint locV = glGetUniformLocation(program, "uTexV"); // V色度采样器

    // 绑定 Y 纹理到单元 0
    glActiveTexture(GL_TEXTURE0);              // 激活单元0
    glBindTexture(GL_TEXTURE_2D, texY);        // 在单元0上绑定Y纹理
    glUniform1i(locY, 0);                      // 告诉sampler从单元0读

    // 绑定 U 纹理到单元 1
    glActiveTexture(GL_TEXTURE1);              // 激活单元1
    glBindTexture(GL_TEXTURE_2D, texU);        // 在单元1上绑定U纹理
    glUniform1i(locU, 1);                      // 告诉sampler从单元1读

    // 绑定 V 纹理到单元 2
    glActiveTexture(GL_TEXTURE2);              // 激活单元2
    glBindTexture(GL_TEXTURE_2D, texV);        // 在单元2上绑定V纹理
    glUniform1i(locV, 2);                      // 告诉sampler从单元2读
}
```

对应的片元着色器：

```glsl
#ifdef GL_ES
precision lowp float;
#endif

uniform sampler2D uTexY;  // Y亮度纹理（单元0）
uniform sampler2D uTexU;  // U色度纹理（单元1）
uniform sampler2D uTexV;  // V色度纹理（单元2）

varying vec2 vUV;

void main() {
    // 从三张纹理分别采样 Y、U、V 分量
    float y = texture2D(uTexY, vUV).r - 0.0625;  // Y偏移校正
    float u = texture2D(uTexU, vUV).r - 0.5;     // U偏移校正
    float v = texture2D(uTexV, vUV).r - 0.5;     // V偏移校正

    // BT.601 标准的 YUV→RGB 转换矩阵
    vec3 rgb;
    rgb.r = 1.164 * y + 1.596 * v;               // 红色分量
    rgb.g = 1.164 * y - 0.392 * u - 0.813 * v;   // 绿色分量
    rgb.b = 1.164 * y + 2.017 * u;               // 蓝色分量

    gl_FragColor = vec4(rgb, 1.0);                // 输出不透明RGB
}
```

## 二、纹理格式：内部格式与像素格式

### 2.1 两种格式的区别

上一节提到 `glTexImage2D` 有两个"格式"参数。这里展开详细解释：

```cpp
glTexImage2D(GL_TEXTURE_2D, 0,
    internalformat,  // 第3个参数：GPU内部存储格式（GPU怎么存）
    width, height, 0,
    format,          // 第7个参数：CPU输入像素格式（CPU数据怎么排的）
    type,            // 第8个参数：每个分量的数据类型
    data);
```

类比：你给快递公司寄一箱苹果。
- `format` = 你告诉快递员"箱子里苹果怎么排的"（一排3个还是4个）
- `internalformat` = 快递公司收到后"在仓库里怎么存"（可能重新整理排列）

### 2.2 常见格式对照表

| 数据类型 | internalformat | format | type | 每像素字节 |
|----------|----------------|--------|------|-----------|
| RGBA 彩色图 | `GL_RGBA` | `GL_RGBA` | `GL_UNSIGNED_BYTE` | 4 |
| RGB 彩色图（无alpha） | `GL_RGB` | `GL_RGB` | `GL_UNSIGNED_BYTE` | 3 |
| 灰度图（ES 2.0） | `GL_LUMINANCE` | `GL_LUMINANCE` | `GL_UNSIGNED_BYTE` | 1 |
| Alpha 通道图（ES 2.0） | `GL_ALPHA` | `GL_ALPHA` | `GL_UNSIGNED_BYTE` | 1 |
| 灰度图（GL 3.3+） | `GL_R8` | `GL_RED` | `GL_UNSIGNED_BYTE` | 1 |
| 深度纹理 | `GL_DEPTH_COMPONENT16` | `GL_DEPTH_COMPONENT` | `GL_UNSIGNED_SHORT` | 2 |

### 2.3 ES 2.0 与桌面 GL 的格式差异

这是一个重要的跨平台坑：

| 功能 | ES 2.0 | ES 3.0 | Desktop GL 3.3+ |
|------|--------|--------|-----------------|
| 单通道灰度 | `GL_LUMINANCE` | `GL_R8`/`GL_RED` | `GL_R8`/`GL_RED` |
| 单通道 alpha | `GL_ALPHA` | `GL_R8`/`GL_RED` | `GL_R8`/`GL_RED` |
| 带精确位深的格式 | ❌ 不支持 | `GL_RGBA8`等 | `GL_RGBA8`等 |
| sRGB 格式 | 扩展 | `GL_SRGB8_ALPHA8` | `GL_SRGB8_ALPHA8` |

**关键点**：`GL_LUMINANCE` 和 `GL_ALPHA` 在 ES 3.0+ 和桌面 OpenGL Core Profile 中已被**废弃**。如果你需要跨平台支持，需要做版本检测：

```cpp
GLenum GetSingleChannelFormat(bool isES2) {
    if (isES2) {
        return GL_LUMINANCE;  // ES 2.0 只能用 LUMINANCE
    } else {
        return GL_RED;         // ES 3.0 / Desktop GL 用 RED（需配合 GL_R8）
    }
}
```

### 2.4 示例2：根据源图像类型选择格式

```cpp
#include <GLES2/gl2.h>

// 源图像类型枚举
enum class ImageFormat {
    Gray8,     // 单通道灰度（字体、Y分量）
    RGB24,     // 三通道彩色（无alpha的照片）
    RGBA32     // 四通道彩色（带透明度的UI元素）
};

// 根据源图像格式上传纹理
void UploadByFormat(GLuint tex, ImageFormat fmt,
                    int w, int h, const void* data) {
    GLenum internalFmt = GL_RGBA;  // GPU存储格式
    GLenum pixelFmt    = GL_RGBA;  // CPU输入格式

    switch (fmt) {
        case ImageFormat::Gray8:
            internalFmt = GL_LUMINANCE; // GPU按亮度存（ES 2.0兼容）
            pixelFmt    = GL_LUMINANCE; // 输入也是亮度
            break;
        case ImageFormat::RGB24:
            internalFmt = GL_RGB;       // GPU按RGB存
            pixelFmt    = GL_RGB;       // 输入RGB排列
            break;
        case ImageFormat::RGBA32:
            internalFmt = GL_RGBA;      // GPU按RGBA存
            pixelFmt    = GL_RGBA;      // 输入RGBA排列
            break;
    }

    glBindTexture(GL_TEXTURE_2D, tex);
    glTexImage2D(GL_TEXTURE_2D, 0, internalFmt, w, h, 0,
                 pixelFmt, GL_UNSIGNED_BYTE, data); // 上传像素
}
```

## 三、glPixelStorei：像素对齐与行步长

### 3.1 为什么需要关心像素对齐

当你用 `glTexImage2D` 上传像素数据时，GPU 不是直接按指针地址顺序读取的——它会按照**对齐规则**来确定每行数据的起始位置。

默认的对齐值是 **4 字节**（`GL_UNPACK_ALIGNMENT = 4`）。这意味着 GPU 认为每行像素数据的起始地址是 4 的倍数。

对于 RGBA 格式（每像素 4 字节），任何宽度都满足 4 字节对齐，没有问题。但对于 RGB 格式（每像素 3 字节），宽度不是 4 的倍数时就出问题了。

### 3.2 对齐错误的具体案例

假设一张 RGB 图片，宽度 = 3 像素：
- 每行实际数据 = 3 × 3 = **9 字节**
- 默认对齐 = 4 → GPU 认为每行 = **12 字节**（向上取到 4 的倍数）
- GPU 多读了 3 字节垃圾数据

结果：画面出现**错行、斜条纹、颜色错位**。

```
你传的数据（紧凑排列）：         GPU 认为的数据（4字节对齐）：

行0: [R G B] [R G B] [R G B]     行0: [R G B] [R G B] [R G B] [? ? ?]
行1: [R G B] [R G B] [R G B]     行1: [R G B] [R G B] [R G B] [? ? ?]
       9 字节/行                        12 字节/行（多了3字节padding）
```

### 3.3 解决方案：设置对齐为 1

```cpp
// 上传 RGB 数据前，将对齐设为1（字节对齐，无padding）
glPixelStorei(GL_UNPACK_ALIGNMENT, 1);

// 上传 RGB 纹理
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, w, h, 0,
             GL_RGB, GL_UNSIGNED_BYTE, pixels);

// 用完后恢复默认值（重要！OpenGL是状态机）
glPixelStorei(GL_UNPACK_ALIGNMENT, 4);
```

**黄金法则**：如果你不确定数据格式是否对齐，**无脑设 `GL_UNPACK_ALIGNMENT = 1`** 是最安全的做法。性能差异在现代 GPU 上可以忽略。

### 3.4 GL_UNPACK_ROW_LENGTH：处理有 pitch 的数据

有时候你拿到的像素数据不是"紧凑排列"的——每行之间有额外的间距（pitch/stride）。比如从截屏 API 拿到的数据，每行末尾可能有 padding。

`GL_UNPACK_ROW_LENGTH` 告诉 GPU "源数据每行的像素数"（注意是像素数，不是字节数）。这样 GPU 就知道跳过多少字节到下一行。

```
源数据布局（pitch > width × 像素字节数）：

├── width ──┤ ← GL_UNPACK_ROW_LENGTH
[pixel][pixel][pixel][padding][padding]    ← 实际行宽
[pixel][pixel][pixel][padding][padding]
[pixel][pixel][pixel][padding][padding]
```

### 3.5 跨平台兼容：GL_EXT_unpack_subimage

| 平台 | GL_UNPACK_ROW_LENGTH | 备注 |
|------|---------------------|------|
| Desktop GL 2.0+ | ✅ 内建支持 | 直接使用 |
| OpenGL ES 3.0 | ✅ 内建支持 | 直接使用 |
| OpenGL ES 2.0 | ⚠️ 需要扩展 | 需要 `GL_EXT_unpack_subimage` |

在 ES 2.0 上，如果设备不支持 `GL_EXT_unpack_subimage`，你就**不能**用 `GL_UNPACK_ROW_LENGTH`。此时需要手动逐行拷贝数据去掉 padding。

### 3.6 示例3：带 pitch 的数据上传（扩展检测 + fallback）

```cpp
#include <GLES2/gl2.h>
#include <vector>
#include <cstring>

// 上传有 pitch 的 RGBA 子区域
// src: 源数据指针（指向子区域第一行首字节）
// x, y: 目标纹理中的偏移位置
// w, h: 要上传的区域宽高
// pitchBytes: 源数据每行的总字节数（可能 > w * 4）
// hasUnpackSubimage: 是否支持 GL_EXT_unpack_subimage
void UploadSubRegionRGBA(
    GLuint tex,
    const unsigned char* src,
    int x, int y, int w, int h,
    int pitchBytes,
    bool hasUnpackSubimage
) {
    glBindTexture(GL_TEXTURE_2D, tex);
    glPixelStorei(GL_UNPACK_ALIGNMENT, 1); // 安全对齐

    if (hasUnpackSubimage) {
        // 快速路径：GPU直接按pitch读取，无需CPU侧拷贝
        glPixelStorei(GL_UNPACK_ROW_LENGTH, pitchBytes / 4); // RGBA每像素4字节
        glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                        GL_RGBA, GL_UNSIGNED_BYTE, src);
        glPixelStorei(GL_UNPACK_ROW_LENGTH, 0); // 恢复默认（0=紧凑排列）
    } else {
        // 慢速路径：CPU侧逐行拷贝，去掉padding
        int rowBytes = w * 4;  // 紧凑排列的每行字节数
        std::vector<unsigned char> compact(rowBytes * h);
        for (int i = 0; i < h; ++i) {
            memcpy(compact.data() + i * rowBytes,  // 目标：紧凑排列
                   src + i * pitchBytes,             // 源：有padding
                   rowBytes);                        // 只拷贝有效像素
        }
        glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                        GL_RGBA, GL_UNSIGNED_BYTE, compact.data());
    }

    glPixelStorei(GL_UNPACK_ALIGNMENT, 4); // 恢复默认对齐
}
```

## 四、glTexSubImage2D：纹理局部更新

### 4.1 为什么需要局部更新

`glTexImage2D` 每次调用都会**重新分配整张纹理的显存**。如果你只需要更新纹理中一小块区域（比如字体图集新增了一个字符，或者视频帧只变了一部分），重新分配整张纹理既浪费带宽又浪费时间。

`glTexSubImage2D` 可以只更新已存在纹理的一个子区域，不重新分配显存。

### 4.2 使用前提

1. 纹理必须已经通过 `glTexImage2D` 分配过（定义了尺寸和格式）
2. 更新区域不能超出纹理边界
3. `format` 和 `type` 必须与初始分配时兼容

### 4.3 参数详解

```cpp
void glTexSubImage2D(
    GLenum  target,    // GL_TEXTURE_2D
    GLint   level,     // Mipmap层级（通常0）
    GLint   xoffset,   // 更新区域的X偏移（像素）
    GLint   yoffset,   // 更新区域的Y偏移（像素）
    GLsizei width,     // 更新区域的宽度
    GLsizei height,    // 更新区域的高度
    GLenum  format,    // 像素格式（必须与glTexImage2D时一致）
    GLenum  type,      // 数据类型（必须与glTexImage2D时一致）
    const void* data   // 新像素数据
);
```

```
纹理全图（1024×1024）：
┌──────────────────────────────┐
│                              │
│   ┌──────────┐               │
│   │ 更新区域 │ ← glTexSubImage2D(x=64, y=128, w=256, h=128)
│   │ 256×128  │               │
│   └──────────┘               │
│                              │
└──────────────────────────────┘
```

### 4.4 示例4：先分配空纹理，再逐步填充

这种模式在**字体图集**中非常常见——先分配一张大纹理，然后每渲染一个新字符就把它的像素写入图集的一小块区域。

```cpp
#include <GLES2/gl2.h>

// 第一步：创建空的RGBA纹理（只分配不上传数据）
GLuint CreateEmptyTexture(int w, int h) {
    GLuint tex = 0;
    glGenTextures(1, &tex);
    glBindTexture(GL_TEXTURE_2D, tex);

    // data = nullptr → 只分配显存，不上传像素
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, w, h, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);

    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

    return tex;
}

// 第二步：局部更新——把一个字符的像素写入图集
void UpdateGlyphRegion(GLuint tex, int x, int y, int w, int h,
                       const unsigned char* pixels) {
    glBindTexture(GL_TEXTURE_2D, tex);
    glPixelStorei(GL_UNPACK_ALIGNMENT, 1); // 字体数据通常不是4对齐

    glTexSubImage2D(GL_TEXTURE_2D, 0,
                    x, y,          // 图集中的偏移位置
                    w, h,          // 字符像素的宽高
                    GL_RGBA, GL_UNSIGNED_BYTE,
                    pixels);       // 字符像素数据

    glPixelStorei(GL_UNPACK_ALIGNMENT, 4); // 恢复默认
}
```

### 4.5 示例5：视频帧逐块刷新

视频播放时，如果使用分块刷新可以减少带宽：

```cpp
#include <GLES2/gl2.h>

// 视频帧更新器
struct VideoFrameUpdater {
    GLuint texY, texU, texV;  // Y/U/V 三张纹理
    int frameW, frameH;       // 视频帧宽高

    // 创建三张纹理（Y全尺寸，UV各半尺寸——YUV420格式）
    void Init(int w, int h) {
        frameW = w;
        frameH = h;

        // Y纹理：全分辨率
        glGenTextures(1, &texY);
        glBindTexture(GL_TEXTURE_2D, texY);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_LUMINANCE, w, h, 0,
                     GL_LUMINANCE, GL_UNSIGNED_BYTE, nullptr);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

        // U纹理：半分辨率（YUV420色度子采样）
        glGenTextures(1, &texU);
        glBindTexture(GL_TEXTURE_2D, texU);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_LUMINANCE, w/2, h/2, 0,
                     GL_LUMINANCE, GL_UNSIGNED_BYTE, nullptr);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

        // V纹理：半分辨率
        glGenTextures(1, &texV);
        glBindTexture(GL_TEXTURE_2D, texV);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_LUMINANCE, w/2, h/2, 0,
                     GL_LUMINANCE, GL_UNSIGNED_BYTE, nullptr);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    }

    // 每帧更新：用 glTexSubImage2D 替换全部像素（比重新分配快）
    void UpdateFrame(const unsigned char* yData,
                     const unsigned char* uData,
                     const unsigned char* vData) {
        glPixelStorei(GL_UNPACK_ALIGNMENT, 1); // YUV数据通常1字节对齐

        glBindTexture(GL_TEXTURE_2D, texY);
        glTexSubImage2D(GL_TEXTURE_2D, 0, 0, 0, frameW, frameH,
                        GL_LUMINANCE, GL_UNSIGNED_BYTE, yData);

        glBindTexture(GL_TEXTURE_2D, texU);
        glTexSubImage2D(GL_TEXTURE_2D, 0, 0, 0, frameW/2, frameH/2,
                        GL_LUMINANCE, GL_UNSIGNED_BYTE, uData);

        glBindTexture(GL_TEXTURE_2D, texV);
        glTexSubImage2D(GL_TEXTURE_2D, 0, 0, 0, frameW/2, frameH/2,
                        GL_LUMINANCE, GL_UNSIGNED_BYTE, vData);

        glPixelStorei(GL_UNPACK_ALIGNMENT, 4); // 恢复默认
    }

    // 清理
    void Destroy() {
        GLuint textures[] = { texY, texU, texV };
        glDeleteTextures(3, textures);
        texY = texU = texV = 0;
    }
};
```

## 五、NPOT 纹理：跨平台兼容的最大坑

### 5.1 什么是 NPOT

NPOT（Non-Power-Of-Two）指纹理尺寸**不是** 2 的幂。常见的 POT 尺寸有：1、2、4、8、16、32、64、128、256、512、1024、2048、4096。而 300×200、1920×1080 都是 NPOT。

在早期的 OpenGL 和 OpenGL ES 2.0 中，NPOT 纹理有严格的功能限制。

### 5.2 NPOT 限制完整对照表

| 功能 | POT 纹理 | ES 2.0 NPOT | ES 3.0 NPOT | Desktop GL NPOT |
|------|----------|-------------|-------------|-----------------|
| GL_REPEAT | ✅ | ❌ | ✅ | ✅ |
| GL_MIRRORED_REPEAT | ✅ | ❌ | ✅ | ✅ |
| GL_CLAMP_TO_EDGE | ✅ | ✅ | ✅ | ✅ |
| Mipmap | ✅ | ❌ | ✅ | ✅ |
| glGenerateMipmap | ✅ | ❌ | ✅ | ✅ |
| 基本采样 | ✅ | ✅ | ✅ | ✅ |

### 5.3 安全的 NPOT 兼容策略

```cpp
#include <GLES2/gl2.h>

// 判断是否为2的幂
static bool IsPOT(int x) {
    return (x > 0) && ((x & (x - 1)) == 0);
}

// NPOT安全的纹理配置
void ConfigureTextureSafe(GLuint tex, int w, int h,
                          bool wantRepeat, bool wantMipmap) {
    glBindTexture(GL_TEXTURE_2D, tex);

    bool isPOT = IsPOT(w) && IsPOT(h);

    // Wrap模式：NPOT只能用CLAMP_TO_EDGE
    if (wantRepeat && isPOT) {
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);
    } else {
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
        if (wantRepeat && !isPOT) {
            // 可以在这里打日志提醒开发者
            // LOG("WARNING: NPOT texture %dx%d, REPEAT downgraded to CLAMP", w, h);
        }
    }

    // Mipmap：NPOT不能用
    if (wantMipmap && isPOT) {
        glGenerateMipmap(GL_TEXTURE_2D);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER,
                        GL_LINEAR_MIPMAP_LINEAR);
    } else {
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    }

    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
}
```

### 5.4 将 NPOT 纹理"升级"为 POT

如果你确实需要 REPEAT 和 Mipmap，可以将 NPOT 纹理放入更大的 POT 纹理中：

```cpp
#include <cstring>
#include <cmath>

// 向上取到最近的2的幂
static int NextPOT(int x) {
    int p = 1;
    while (p < x) p <<= 1;  // 逐次翻倍直到 >= x
    return p;
}

// 将 NPOT 图片嵌入 POT 纹理
// 返回 UV 缩放因子（实际图片只占纹理的一部分）
GLuint CreatePOTTexture(const unsigned char* pixels, int w, int h,
                        float* outUScale, float* outVScale) {
    int potW = NextPOT(w);  // 如 300 → 512
    int potH = NextPOT(h);  // 如 200 → 256

    // UV缩放：实际图片只占纹理的 w/potW × h/potH 区域
    *outUScale = (float)w / potW;  // 如 300/512 = 0.586
    *outVScale = (float)h / potH;  // 如 200/256 = 0.781

    GLuint tex = 0;
    glGenTextures(1, &tex);
    glBindTexture(GL_TEXTURE_2D, tex);

    // 先分配完整的POT尺寸纹理（填充0）
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, potW, potH, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);

    // 再用 glTexSubImage2D 只上传实际图片区域
    glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
    glTexSubImage2D(GL_TEXTURE_2D, 0, 0, 0, w, h,
                    GL_RGBA, GL_UNSIGNED_BYTE, pixels);
    glPixelStorei(GL_UNPACK_ALIGNMENT, 4);

    // 现在可以安全使用 REPEAT 和 Mipmap
    glGenerateMipmap(GL_TEXTURE_2D);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER,
                    GL_LINEAR_MIPMAP_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);

    return tex;
}
// 使用时，顶点的UV坐标需要乘以 outUScale/outVScale
```

## 常见错误与解决方案

### ❌ 错误1：sampler uniform 传了纹理 ID 而不是单元编号

```cpp
// ❌ 错误：把纹理对象ID传给sampler（tex可能是73这种大数字）
GLuint tex = 73;
glUniform1i(samplerLoc, tex);  // 错！应该传单元编号（0、1、2...）

// ✅ 修复：传纹理单元编号
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, tex);
glUniform1i(samplerLoc, 0);  // 传0，表示"从纹理单元0读取"
```

### ❌ 错误2：RGB 数据上传后画面斜条纹/错行

```cpp
// ❌ 错误：忘记设置对齐，默认alignment=4
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, 301, 200, 0,  // 宽301不是4的倍数
             GL_RGB, GL_UNSIGNED_BYTE, pixels);
// 结果：画面出现斜条纹

// ✅ 修复：上传非4倍宽度的数据前设置alignment=1
glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, 301, 200, 0,
             GL_RGB, GL_UNSIGNED_BYTE, pixels);
glPixelStorei(GL_UNPACK_ALIGNMENT, 4);  // 恢复默认
```

### ❌ 错误3：glTexSubImage2D 越界但不报错——画面随机花屏

```cpp
// ❌ 错误：更新区域超出纹理边界
glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, 256, 256, 0, ...); // 纹理256×256
glTexSubImage2D(GL_TEXTURE_2D, 0,
                200, 200,   // 偏移(200,200)
                128, 128,   // 区域128×128 → 右下角到 (328,328)
                ...);       // 越界！纹理只有256×256

// ✅ 修复：上传前检查边界
void SafeTexSubImage(int texW, int texH,
                     int x, int y, int w, int h,
                     const void* data) {
    // 裁剪到纹理边界内
    if (x + w > texW) w = texW - x;  // 右边裁剪
    if (y + h > texH) h = texH - y;  // 下边裁剪
    if (w <= 0 || h <= 0) return;     // 完全越界，跳过

    glTexSubImage2D(GL_TEXTURE_2D, 0, x, y, w, h,
                    GL_RGBA, GL_UNSIGNED_BYTE, data);
}
```

## 本节小结

- **纹理单元是插槽**：sampler 不绑定纹理 ID，而是绑定"纹理单元编号"。每个单元可以挂载一张纹理
- **三步绑定**：`glActiveTexture(GL_TEXTURE0+i)` → `glBindTexture(tex)` → `glUniform1i(sampler, i)`
- **两种格式分清**：`internalformat` = GPU 怎么存，`format` = CPU 数据怎么排
- **对齐是隐形杀手**：RGB 等非 4 倍宽度数据必须设 `GL_UNPACK_ALIGNMENT = 1`
- **glTexSubImage2D 省带宽**：只更新子区域，不重新分配整张纹理
- **NPOT 限制记牢**：ES 2.0 上 NPOT 只能用 `CLAMP_TO_EDGE`，不能用 Mipmap 和 REPEAT
- **扩展检测是工程必须**：`GL_EXT_unpack_subimage` 等扩展需要运行时检测并提供 fallback

## 练习题与答案

### 题目1：为什么 sampler uniform 传 0/1/2（单元编号），而不是纹理对象 ID？

<details>
<summary>查看答案</summary>

因为 OpenGL 的纹理系统是**间接绑定**的——sampler 指向的是"纹理单元"，而纹理对象是通过 `glBindTexture` 绑定到纹理单元上的。

这种设计的好处是**解耦**：
1. 同一个 sampler 可以在不同帧读取不同纹理（只需改变单元上绑定的纹理对象）
2. 着色器不需要知道纹理对象 ID（ID 是 GPU 驱动分配的、不确定的数字）
3. 可以在不重新编译着色器的情况下切换纹理

如果 sampler 直接绑定纹理 ID，每次更换纹理都需要调用 `glUniform1i` 更新 sampler 值，而现在只需要在对应单元上 `glBindTexture` 新的纹理即可。

</details>

### 题目2：`GL_UNPACK_ALIGNMENT` 设错会出现什么现象？什么情况下最容易出错？

<details>
<summary>查看答案</summary>

**现象**：画面出现斜条纹、错行、颜色错位。如果偏移量刚好使每行多读/少读几个字节，看起来就像整个图片被"剪切"然后斜着贴上去了。

**最容易出错的情况**：
1. **RGB 格式 + 非 4 倍宽度**：例如宽度 301 的 RGB 图片，每行 301×3=903 字节，不是 4 的倍数。默认 alignment=4 时，GPU 认为每行是 904 字节，多读了 1 字节
2. **单通道灰度格式**：宽度不是 4 的倍数时同样会出问题。例如宽度 100 的灰度图，每行 100 字节，不是 4 的倍数
3. **从其他 API 获取的带 padding 数据**：如 Windows GDI、Android Bitmap 等 API 返回的数据可能每行有额外 padding

**安全做法**：在任何 `glTexImage2D` / `glTexSubImage2D` 调用前，先设 `glPixelStorei(GL_UNPACK_ALIGNMENT, 1)`，用完后恢复为 4。

</details>

### 题目3：你需要在 ES 2.0 设备上播放 1920×1080 的视频（YUV420 格式），请列出创建 Y/U/V 三张纹理时需要注意的所有跨平台要点。

<details>
<summary>查看答案</summary>

1920×1080 是 NPOT，YUV420 意味着 U/V 分量是半尺寸（960×540，也是 NPOT）。需要注意以下要点：

**纹理格式**：
- ES 2.0 上，单通道数据使用 `GL_LUMINANCE`（不能用 `GL_RED`，那是 ES 3.0+ 的）
- `internalformat` 和 `format` 都写 `GL_LUMINANCE`

**Wrap 模式**：
- 必须用 `GL_CLAMP_TO_EDGE`（NPOT 不支持 REPEAT）
- 视频纹理本来也不需要 REPEAT，所以这不影响功能

**Mipmap**：
- 不能生成 Mipmap（NPOT 限制）
- `MIN_FILTER` 只能用 `GL_LINEAR` 或 `GL_NEAREST`

**对齐**：
- YUV 数据通常是紧凑排列的，需要设 `GL_UNPACK_ALIGNMENT = 1`
- 特别注意：如果 FFmpeg 解码输出的 linesize（行步长）大于宽度，需要用 `GL_UNPACK_ROW_LENGTH`（如果支持扩展）或手动逐行拷贝

**每帧更新**：
- 使用 `glTexSubImage2D` 而非 `glTexImage2D` 更新帧数据
- 先用 `glTexImage2D(... nullptr)` 分配一次，之后每帧只 `glTexSubImage2D`

完整创建代码：
```cpp
glPixelStorei(GL_UNPACK_ALIGNMENT, 1);

// Y纹理
glGenTextures(1, &texY);
glBindTexture(GL_TEXTURE_2D, texY);
glTexImage2D(GL_TEXTURE_2D, 0, GL_LUMINANCE, 1920, 1080, 0,
             GL_LUMINANCE, GL_UNSIGNED_BYTE, nullptr);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

// U纹理（半尺寸）
glGenTextures(1, &texU);
glBindTexture(GL_TEXTURE_2D, texU);
glTexImage2D(GL_TEXTURE_2D, 0, GL_LUMINANCE, 960, 540, 0,
             GL_LUMINANCE, GL_UNSIGNED_BYTE, nullptr);
// ... 同样的参数配置

// V纹理（半尺寸）
// ... 同U
```

</details>
