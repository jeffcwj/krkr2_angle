## 帧缓冲基础概念

### 什么是帧缓冲

在前几节中，我们所有的绘制操作都直接输出到屏幕上。OpenGL 在内部维护一个**默认帧缓冲**（Default Framebuffer），由窗口系统创建，包含：

| 缓冲区 | 作用 | 位深（典型） |
|---------|------|-------------|
| 颜色缓冲（Color Buffer） | 存储最终像素颜色 | RGBA8（32位） |
| 深度缓冲（Depth Buffer） | 存储每个像素的深度值 | 24位 |
| 模板缓冲（Stencil Buffer） | 存储模板测试值 | 8位 |

默认帧缓冲的内容最终被显示到屏幕上。但在实际应用中，我们经常需要**先把场景渲染到一个中间"画布"**，对画布做加工（例如模糊、色调调整），最后再把加工结果显示到屏幕——这就是**离屏渲染**，而承载这个中间画布的对象就是**帧缓冲对象（Framebuffer Object, FBO）**。

```
┌──────────────────────────────────────────────┐
│              渲染管线                          │
│                                              │
│  顶点数据 → 顶点着色器 → 光栅化 → 片段着色器  │
│                                              │
│         ┌──── 输出目标 ────┐                  │
│         │                  │                  │
│    默认帧缓冲           自定义 FBO            │
│    (直接上屏)        (离屏渲染到纹理)          │
│                         │                    │
│                    后处理着色器                │
│                         │                    │
│                    默认帧缓冲                  │
│                    (最终上屏)                  │
└──────────────────────────────────────────────┘
```

### FBO 的组成结构

一个 FBO 本身**不存储任何像素数据**，它只是一个"容器"，通过**附件（Attachment）**指向实际存储数据的对象：

```
┌─────────────── FBO ───────────────────┐
│                                       │
│  GL_COLOR_ATTACHMENT0  ──→  纹理/RBO  │
│  GL_COLOR_ATTACHMENT1  ──→  纹理/RBO  │  （MRT：多渲染目标）
│  GL_DEPTH_ATTACHMENT   ──→  纹理/RBO  │
│  GL_STENCIL_ATTACHMENT ──→  纹理/RBO  │
│  GL_DEPTH_STENCIL_ATTACHMENT ──→ RBO  │  （深度+模板合并）
│                                       │
└───────────────────────────────────────┘
```

附件可以是两种类型：

| 附件类型 | 载体 | 优点 | 缺点 | 适用场景 |
|----------|------|------|------|----------|
| 纹理附件 | `GL_TEXTURE_2D` | 可作为后续 pass 的输入纹理采样 | 创建稍复杂 | 颜色附件（需要后处理读取） |
| 渲染缓冲对象 | `GL_RENDERBUFFER` | 针对离屏写入优化，性能更好 | 无法直接采样 | 深度/模板附件（只写不读） |

## 创建和使用帧缓冲对象

### 基础 FBO 创建流程

创建一个带颜色纹理附件的 FBO 的完整代码：

```cpp
#include <glad/glad.h>
#include <GLFW/glfw3.h>
#include <iostream>

// 屏幕尺寸
const int SCR_WIDTH = 800;
const int SCR_HEIGHT = 600;

// 创建 FBO 及其附件
GLuint createFramebuffer(GLuint &colorTexture, GLuint &rbo) {
    GLuint fbo;
    // 第一步：生成 FBO
    glGenFramebuffers(1, &fbo);
    // 第二步：绑定 FBO（后续所有 framebuffer 操作作用于此 FBO）
    glBindFramebuffer(GL_FRAMEBUFFER, fbo);

    // 第三步：创建颜色纹理附件
    glGenTextures(1, &colorTexture);
    glBindTexture(GL_TEXTURE_2D, colorTexture);
    // 分配纹理存储，不传初始数据（data = nullptr）
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA,
                 SCR_WIDTH, SCR_HEIGHT, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
    // 设置过滤参数（FBO 纹理通常用 LINEAR）
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    // 边缘钳制，防止后处理采样越界
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    // 将纹理附加到 FBO 的颜色附件 0
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                           GL_TEXTURE_2D, colorTexture, 0);

    // 第四步：创建 RBO 作为深度+模板附件
    glGenRenderbuffers(1, &rbo);
    glBindRenderbuffer(GL_RENDERBUFFER, rbo);
    // GL_DEPTH24_STENCIL8：24位深度 + 8位模板，最常用的组合
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8,
                          SCR_WIDTH, SCR_HEIGHT);
    // 将 RBO 附加到 FBO 的深度+模板附件
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_STENCIL_ATTACHMENT,
                              GL_RENDERBUFFER, rbo);

    // 第五步：检查 FBO 完整性
    GLenum status = glCheckFramebufferStatus(GL_FRAMEBUFFER);
    if (status != GL_FRAMEBUFFER_COMPLETE) {
        std::cerr << "FBO 不完整！状态码: 0x"
                  << std::hex << status << std::endl;
    }

    // 第六步：解绑 FBO，恢复默认帧缓冲
    glBindFramebuffer(GL_FRAMEBUFFER, 0);
    return fbo;
}
```

### FBO 完整性检查

`glCheckFramebufferStatus` 是创建 FBO 后**必须调用**的函数。常见错误状态码：

| 状态码 | 含义 | 原因 |
|--------|------|------|
| `GL_FRAMEBUFFER_COMPLETE` | 完整，可用 | 正常 |
| `GL_FRAMEBUFFER_INCOMPLETE_ATTACHMENT` | 附件不完整 | 纹理/RBO 未正确创建或格式不对 |
| `GL_FRAMEBUFFER_INCOMPLETE_MISSING_ATTACHMENT` | 缺少附件 | FBO 至少需要一个颜色附件 |
| `GL_FRAMEBUFFER_INCOMPLETE_DIMENSIONS` | 尺寸不一致（ES） | 各附件的宽高不匹配（ES 2.0 特有） |
| `GL_FRAMEBUFFER_UNSUPPORTED` | 格式组合不支持 | 附件格式组合在当前驱动/GPU 不支持 |

### 使用 FBO 进行离屏渲染

有了 FBO 后，离屏渲染的流程如下：

```cpp
// ===== 第一步：渲染场景到 FBO =====
glBindFramebuffer(GL_FRAMEBUFFER, fbo);  // 绑定自定义 FBO
glClearColor(0.1f, 0.1f, 0.1f, 1.0f);
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
glEnable(GL_DEPTH_TEST);  // 离屏渲染也需要深度测试

// 用普通着色器渲染场景
sceneShader.use();
drawScene();  // 绘制所有场景对象

// ===== 第二步：渲染 FBO 纹理到屏幕 =====
glBindFramebuffer(GL_FRAMEBUFFER, 0);    // 切回默认帧缓冲
glClearColor(1.0f, 1.0f, 1.0f, 1.0f);
glClear(GL_COLOR_BUFFER_BIT);
glDisable(GL_DEPTH_TEST);  // 全屏四边形不需要深度测试

// 用后处理着色器绘制全屏四边形
postProcessShader.use();
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, colorTexture);  // 绑定 FBO 颜色纹理
drawFullscreenQuad();  // 绘制覆盖全屏的四边形

glfwSwapBuffers(window);
```

### 全屏四边形（Fullscreen Quad）

后处理的核心是用一个覆盖整个屏幕的四边形来采样 FBO 纹理。这个四边形的顶点数据是固定的：

```cpp
// 全屏四边形顶点数据
// 位置（NDC坐标）      纹理坐标
float quadVertices[] = {
    // 第一个三角形
    -1.0f,  1.0f,    0.0f, 1.0f,  // 左上
    -1.0f, -1.0f,    0.0f, 0.0f,  // 左下
     1.0f, -1.0f,    1.0f, 0.0f,  // 右下
    // 第二个三角形
    -1.0f,  1.0f,    0.0f, 1.0f,  // 左上
     1.0f, -1.0f,    1.0f, 0.0f,  // 右下
     1.0f,  1.0f,    1.0f, 1.0f,  // 右上
};

GLuint quadVAO, quadVBO;

void initFullscreenQuad() {
    glGenVertexArrays(1, &quadVAO);
    glGenBuffers(1, &quadVBO);
    glBindVertexArray(quadVAO);
    glBindBuffer(GL_ARRAY_BUFFER, quadVBO);
    glBufferData(GL_ARRAY_BUFFER, sizeof(quadVertices),
                 quadVertices, GL_STATIC_DRAW);
    // 位置属性：location=0, 2个float
    glEnableVertexAttribArray(0);
    glVertexAttribPointer(0, 2, GL_FLOAT, GL_FALSE,
                          4 * sizeof(float), (void*)0);
    // 纹理坐标属性：location=1, 2个float
    glEnableVertexAttribArray(1);
    glVertexAttribPointer(1, 2, GL_FLOAT, GL_FALSE,
                          4 * sizeof(float),
                          (void*)(2 * sizeof(float)));
    glBindVertexArray(0);
}

void drawFullscreenQuad() {
    glBindVertexArray(quadVAO);
    glDrawArrays(GL_TRIANGLES, 0, 6);  // 2个三角形 = 6个顶点
    glBindVertexArray(0);
}
```

## 渲染缓冲对象（RBO）详解

### RBO vs 纹理附件的选择

渲染缓冲对象（Renderbuffer Object, RBO）是一种不可采样的离屏缓冲区。与纹理附件相比：

```
           纹理附件                    RBO
    ┌──────────────────┐      ┌──────────────────┐
    │  可读可写          │      │  只写（写入优化）  │
    │  可作为采样器输入   │      │  不可采样          │
    │  格式灵活          │      │  格式受限          │
    │  适合：颜色附件     │      │  适合：深度/模板   │
    └──────────────────┘      └──────────────────┘
```

**选择原则**：
- 如果后续需要**读取**该附件的数据（如后处理读取颜色） → 用**纹理**
- 如果只需要 OpenGL 内部**写入**（如深度测试、模板测试） → 用 **RBO**（性能更优）

### 创建深度纹理附件

某些场景需要读取深度信息（如阴影映射），此时要用纹理而非 RBO：

```cpp
GLuint depthTexture;
glGenTextures(1, &depthTexture);
glBindTexture(GL_TEXTURE_2D, depthTexture);
// GL_DEPTH_COMPONENT24：24位深度精度
glTexImage2D(GL_TEXTURE_2D, 0, GL_DEPTH_COMPONENT24,
             SCR_WIDTH, SCR_HEIGHT, 0,
             GL_DEPTH_COMPONENT, GL_FLOAT, nullptr);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_NEAREST);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_NEAREST);
// 深度纹理边缘用白色（最大深度），避免阴影边缘伪影
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_BORDER);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_BORDER);
float borderColor[] = {1.0f, 1.0f, 1.0f, 1.0f};
glTexParameterfv(GL_TEXTURE_2D, GL_TEXTURE_BORDER_COLOR, borderColor);
// 附加到 FBO 深度附件
glFramebufferTexture2D(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT,
                       GL_TEXTURE_2D, depthTexture, 0);
```

> **跨平台注意**：`GL_CLAMP_TO_BORDER` 在 OpenGL ES 2.0 中**不可用**，ES 3.2 才支持。在移动端需要改用 `GL_CLAMP_TO_EDGE` 并在着色器中手动处理边界。

## 后处理效果实战

后处理的核心思路：先把场景渲染到 FBO 的颜色纹理上，再用一个**全屏四边形 + 后处理着色器**读取该纹理并输出到默认帧缓冲。不同的后处理效果只是片段着色器不同。

### 后处理顶点着色器（通用）

所有后处理效果共享同一个顶点着色器：

```glsl
#version 330 core
layout(location = 0) in vec2 aPos;      // NDC 坐标
layout(location = 1) in vec2 aTexCoord; // 纹理坐标

out vec2 TexCoord;

void main() {
    gl_Position = vec4(aPos, 0.0, 1.0);  // 直接使用 NDC 坐标
    TexCoord = aTexCoord;
}
```

### 效果一：灰度化（Grayscale）

将彩色图像转为灰度图。使用人眼感知的加权公式（亮度 = 0.2126R + 0.7152G + 0.0722B）：

```glsl
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D screenTexture;  // FBO 颜色纹理

void main() {
    vec4 color = texture(screenTexture, TexCoord);
    // ITU-R BT.709 亮度系数，人眼对绿色最敏感
    float gray = dot(color.rgb, vec3(0.2126, 0.7152, 0.0722));
    FragColor = vec4(gray, gray, gray, 1.0);
}
```

### 效果二：核卷积（Kernel Convolution）

卷积是图像处理的基石。对每个像素，取其周围 3×3 邻域，用**卷积核**加权求和：

```
  核（Kernel）3×3 示例 — 锐化：
  ┌────┬────┬────┐
  │  0 │ -1 │  0 │
  ├────┼────┼────┤
  │ -1 │  5 │ -1 │
  ├────┼────┼────┤
  │  0 │ -1 │  0 │
  └────┴────┴────┘
```

通用核卷积片段着色器：

```glsl
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D screenTexture;

// 纹理尺寸的倒数，用于计算相邻像素的 UV 偏移
uniform vec2 texelSize;  // = vec2(1.0/width, 1.0/height)

void main() {
    // 定义不同效果的卷积核 — 这里用锐化核
    float kernel[9] = float[](
         0.0, -1.0,  0.0,
        -1.0,  5.0, -1.0,
         0.0, -1.0,  0.0
    );

    // 3×3 邻域的偏移量
    vec2 offsets[9] = vec2[](
        vec2(-1, 1),  vec2(0, 1),  vec2(1, 1),   // 上方一行
        vec2(-1, 0),  vec2(0, 0),  vec2(1, 0),   // 当前行
        vec2(-1,-1),  vec2(0,-1),  vec2(1,-1)    // 下方一行
    );

    vec3 result = vec3(0.0);
    for (int i = 0; i < 9; i++) {
        // 采样相邻像素并乘以对应的核权重
        vec3 sample = texture(screenTexture,
                              TexCoord + offsets[i] * texelSize).rgb;
        result += sample * kernel[i];
    }

    FragColor = vec4(result, 1.0);
}
```

### 效果三：高斯模糊（Gaussian Blur）

高斯模糊是最常用的模糊算法。直接用 N×N 核开销大，实际常用**两步分离式模糊**（先水平后垂直），将 O(N²) 降为 O(2N)：

```
完整高斯模糊 = 水平 pass + 垂直 pass

Pass 1（水平）:        Pass 2（垂直）:
FBO_A → FBO_B         FBO_B → 默认帧缓冲
━━━━━━━━━━━━━         ━━━━━━━━━━━━━━━━━━
对每行做 1D 模糊       对每列做 1D 模糊
```

这需要**两个 FBO**（或一个 FBO 的 Ping-Pong 切换）。水平模糊着色器：

```glsl
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D image;
uniform vec2 texelSize;

// 5-tap 高斯权重（sigma ≈ 1.5）
const float weight[5] = float[](
    0.227027, 0.1945946, 0.1216216, 0.054054, 0.016216
);

void main() {
    vec3 result = texture(image, TexCoord).rgb * weight[0];
    // 水平方向采样
    for (int i = 1; i < 5; i++) {
        vec2 offset = vec2(texelSize.x * float(i), 0.0);
        result += texture(image, TexCoord + offset).rgb * weight[i];
        result += texture(image, TexCoord - offset).rgb * weight[i];
    }
    FragColor = vec4(result, 1.0);
}
```

垂直模糊着色器与上述类似，只需把偏移方向改为 `vec2(0.0, texelSize.y * float(i))`。

### 效果四：边缘检测（Sobel 算子）

使用 Sobel 算子检测图像中的边缘。Sobel 核分水平和垂直两个方向：

```glsl
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D screenTexture;
uniform vec2 texelSize;

void main() {
    // Sobel 水平核 Gx
    float Gx[9] = float[](
        -1, 0, 1,
        -2, 0, 2,
        -1, 0, 1
    );
    // Sobel 垂直核 Gy
    float Gy[9] = float[](
        -1, -2, -1,
         0,  0,  0,
         1,  2,  1
    );

    vec2 offsets[9] = vec2[](
        vec2(-1, 1), vec2(0, 1), vec2(1, 1),
        vec2(-1, 0), vec2(0, 0), vec2(1, 0),
        vec2(-1,-1), vec2(0,-1), vec2(1,-1)
    );

    vec3 sumX = vec3(0.0), sumY = vec3(0.0);
    for (int i = 0; i < 9; i++) {
        vec3 s = texture(screenTexture,
                         TexCoord + offsets[i] * texelSize).rgb;
        sumX += s * Gx[i];
        sumY += s * Gy[i];
    }
    // 梯度幅值 = sqrt(Gx² + Gy²)
    vec3 edge = sqrt(sumX * sumX + sumY * sumY);
    FragColor = vec4(edge, 1.0);
}
```

## 多 Pass 后处理管线

实际项目中经常需要串联多个后处理效果。这就需要**Ping-Pong FBO**技术：

```
场景 ──→ FBO_A ──→ FBO_B ──→ FBO_A ──→ 屏幕
         (渲染)    (效果1)   (效果2)   (最终)
```

```cpp
// Ping-Pong FBO 管线示例
GLuint pingpongFBO[2], pingpongTex[2];

void initPingPongFBOs(int width, int height) {
    glGenFramebuffers(2, pingpongFBO);
    glGenTextures(2, pingpongTex);
    for (int i = 0; i < 2; i++) {
        glBindFramebuffer(GL_FRAMEBUFFER, pingpongFBO[i]);
        glBindTexture(GL_TEXTURE_2D, pingpongTex[i]);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA,
                     width, height, 0,
                     GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
        glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                               GL_TEXTURE_2D, pingpongTex[i], 0);
    }
}

void renderWithMultiPass() {
    // Pass 0: 场景渲染到 FBO_A
    glBindFramebuffer(GL_FRAMEBUFFER, pingpongFBO[0]);
    renderScene();

    // Pass 1: FBO_A → 高斯模糊水平 → FBO_B
    glBindFramebuffer(GL_FRAMEBUFFER, pingpongFBO[1]);
    blurHShader.use();
    glBindTexture(GL_TEXTURE_2D, pingpongTex[0]);
    drawFullscreenQuad();

    // Pass 2: FBO_B → 高斯模糊垂直 → FBO_A
    glBindFramebuffer(GL_FRAMEBUFFER, pingpongFBO[0]);
    blurVShader.use();
    glBindTexture(GL_TEXTURE_2D, pingpongTex[1]);
    drawFullscreenQuad();

    // 最终 Pass: FBO_A → 色调映射 → 屏幕
    glBindFramebuffer(GL_FRAMEBUFFER, 0);
    tonemapShader.use();
    glBindTexture(GL_TEXTURE_2D, pingpongTex[0]);
    drawFullscreenQuad();
}
```

### FBO 尺寸与窗口尺寸

当窗口大小改变时，FBO 也需要重建：

```cpp
void onWindowResize(int newWidth, int newHeight) {
    // 重新分配颜色纹理
    glBindTexture(GL_TEXTURE_2D, colorTexture);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA,
                 newWidth, newHeight, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
    // 重新分配 RBO
    glBindRenderbuffer(GL_RENDERBUFFER, rbo);
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8,
                          newWidth, newHeight);
    // 重新检查完整性
    glBindFramebuffer(GL_FRAMEBUFFER, fbo);
    if (glCheckFramebufferStatus(GL_FRAMEBUFFER)
        != GL_FRAMEBUFFER_COMPLETE) {
        std::cerr << "FBO resize 后不完整！" << std::endl;
    }
    glBindFramebuffer(GL_FRAMEBUFFER, 0);
}
```

## 跨平台 FBO 差异

| 特性 | Desktop GL 3.3+ | OpenGL ES 2.0 | OpenGL ES 3.0+ |
|------|-----------------|---------------|-----------------|
| FBO 核心支持 | ✅ 核心 | ✅ 核心 | ✅ 核心 |
| MRT（多渲染目标） | ✅ 最多 8 个 | ❌（需扩展 `NV_draw_buffers`） | ✅ 最多 4 个 |
| `GL_DEPTH24_STENCIL8` | ✅ | ❌（需 `OES_packed_depth_stencil`） | ✅ |
| 深度纹理 | ✅ | 需 `OES_depth_texture` | ✅ |
| `glBlitFramebuffer` | ✅ | ❌ | ✅ |
| `GL_READ_FRAMEBUFFER` / `GL_DRAW_FRAMEBUFFER` | ✅ | ❌ | ✅ |
| MSAA FBO | ✅ `glRenderbufferStorageMultisample` | 需扩展 | ✅ |

### ES 2.0 兼容写法

在 ES 2.0 中，深度+模板需要分开创建：

```cpp
#ifdef GL_ES_VERSION_2_0
    // ES 2.0：分别创建深度和模板 RBO
    GLuint depthRBO, stencilRBO;
    glGenRenderbuffers(1, &depthRBO);
    glBindRenderbuffer(GL_RENDERBUFFER, depthRBO);
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT16,
                          width, height);
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT,
                              GL_RENDERBUFFER, depthRBO);

    glGenRenderbuffers(1, &stencilRBO);
    glBindRenderbuffer(GL_RENDERBUFFER, stencilRBO);
    glRenderbufferStorage(GL_RENDERBUFFER, GL_STENCIL_INDEX8,
                          width, height);
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_STENCIL_ATTACHMENT,
                              GL_RENDERBUFFER, stencilRBO);
#else
    // Desktop GL / ES 3.0+：合并深度+模板
    GLuint rbo;
    glGenRenderbuffers(1, &rbo);
    glBindRenderbuffer(GL_RENDERBUFFER, rbo);
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8,
                          width, height);
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_STENCIL_ATTACHMENT,
                              GL_RENDERBUFFER, rbo);
#endif
```

## 对照项目源码

KrKr2 项目在 `RenderManager_ogl.cpp` 中使用了 FBO 进行离屏渲染。以下是关键实现：

### FBO 管理（第 469-518 行）

```cpp
// 文件：cpp/core/visual/ogl/RenderManager_ogl.cpp

// 全局 FBO 变量
static bool _CurrentFBOValid = false;
static GLuint _FBO;                    // 所有可渲染纹理共用的 FBO
static GLuint _stencil_FBO;            // 模板缓冲专用 FBO
static GLuint _CurrentRenderTarget = 0; // 当前渲染目标纹理 ID
static GLint _screenFrameBuffer = 0;    // 默认帧缓冲 ID（由系统创建）

// 切换渲染目标
void TVPSetRenderTarget(GLuint t) {
    if (t) {
        if (!_CurrentFBOValid) {
            _CurrentFBOValid = true;
            // 保存当前绑定的帧缓冲（即默认帧缓冲）
            glGetIntegerv(GL_FRAMEBUFFER_BINDING, &_screenFrameBuffer);
            // 绑定离屏 FBO
            glBindFramebuffer(GL_FRAMEBUFFER, _FBO);
        }
        if (_CurrentRenderTarget == t) return;  // 避免重复切换
        // 将目标纹理挂到 FBO 的颜色附件 0
        glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                               GL_TEXTURE_2D, t, 0);
    } else {
        // t=0 表示恢复到默认帧缓冲（屏幕）
        glBindFramebuffer(GL_FRAMEBUFFER, _screenFrameBuffer);
        _CurrentFBOValid = false;
    }
    _CurrentRenderTarget = t;
}
```

**设计要点**：
- KrKr2 使用**单个共享 FBO**，通过 `glFramebufferTexture2D` 动态切换颜色附件，而非为每个渲染目标创建独立 FBO。这种方式减少了 FBO 对象数量
- `_screenFrameBuffer` 在首次切换时通过 `glGetIntegerv(GL_FRAMEBUFFER_BINDING)` 获取，因为 Cocos2d-x 框架可能使用非 0 的默认帧缓冲
- `_RestoreGLStatues()` 函数（第 511-518 行）负责恢复像素存储状态、混合模式和渲染目标

### Shader Framebuffer Fetch 扩展（第 84-108 行）

KrKr2 还检测了 `GL_EXT_shader_framebuffer_fetch` 扩展。这是移动 GPU（特别是 Adreno、Mali、PowerVR）的一个重要优化：允许片段着色器**直接读取**当前帧缓冲中已有的颜色值，无需额外纹理读取。这对 Alpha 混合等操作特别高效。

```cpp
// 检测 shader_framebuffer_fetch 支持（三个厂商的变体）
DEFEXT(EXT_shader_framebuffer_fetch);    // 标准扩展
DEFEXT(ARM_shader_framebuffer_fetch);    // ARM Mali GPU
DEFEXT(NV_shader_framebuffer_fetch);     // NVIDIA Tegra
```

## 本节小结

- FBO 是离屏渲染的核心，它不存储数据，而是通过附件指向纹理或 RBO
- 颜色附件用纹理（后续需要采样），深度/模板用 RBO（只写不读，性能好）
- 创建 FBO 后**必须**检查完整性（`glCheckFramebufferStatus`）
- 后处理的核心模式：场景 → FBO → 全屏四边形 + 后处理着色器 → 屏幕
- 核卷积是大多数图像效果的基础，不同的核实现不同效果（锐化、模糊、边缘检测）
- 高斯模糊用两步分离式（水平 + 垂直）可将复杂度从 O(N²) 降到 O(2N)
- 多 pass 管线用 Ping-Pong FBO 技术串联多个效果
- ES 2.0 的 FBO 功能受限，深度模板需要分开、无 MRT、无 `glBlitFramebuffer`
- KrKr2 使用单个共享 FBO + 动态附件切换的设计，减少了对象管理开销

## 常见错误及解决方案

### 错误一：FBO 渲染结果是黑屏

**症状**：绑定 FBO 后渲染，切回默认帧缓冲绘制全屏四边形时一片黑。

**常见原因及排查**：

1. **忘记绑定 FBO 的颜色纹理**：绘制全屏四边形前没有 `glBindTexture(GL_TEXTURE_2D, colorTexture)`
2. **FBO 不完整**：没有检查 `glCheckFramebufferStatus`，附件格式不兼容
3. **纹理过滤模式不对**：FBO 纹理未生成 mipmap 但 MIN_FILTER 设为 `GL_LINEAR_MIPMAP_LINEAR`，改为 `GL_LINEAR`
4. **视口未重置**：FBO 尺寸与窗口不同时，切回默认帧缓冲后未调用 `glViewport(0, 0, windowW, windowH)`

### 错误二：后处理纹理坐标反转（图像上下颠倒）

**症状**：后处理结果垂直翻转。

**原因**：OpenGL 纹理坐标原点在左下角，而某些框架（如 Cocos2d-x）或 FBO 纹理的 Y 轴方向可能与预期不同。

**解决**：在后处理顶点着色器中翻转 Y 坐标：
```glsl
TexCoord = vec2(aTexCoord.x, 1.0 - aTexCoord.y);
```

### 错误三：ES 2.0 上 FBO 不完整

**症状**：桌面正常，Android ES 2.0 设备报 `GL_FRAMEBUFFER_INCOMPLETE_ATTACHMENT`。

**原因**：使用了 ES 2.0 不支持的格式，如 `GL_DEPTH24_STENCIL8` 或 `GL_DEPTH_COMPONENT24`。

**解决**：
```cpp
#ifdef GL_ES_VERSION_2_0
    // ES 2.0 安全格式
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT16, w, h);
#else
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8, w, h);
#endif
```

## 练习题与答案

### 题目 1：FBO 概念理解

请回答以下问题：
1. FBO 本身是否存储像素数据？
2. 如果只需要深度测试但不需要在着色器中采样深度值，应该用纹理还是 RBO 作为深度附件？为什么？
3. 为什么高斯模糊要分成水平和垂直两个 pass？

<details>
<summary>查看答案</summary>

1. **不存储**。FBO 本身只是一个"容器"（类似 VAO），通过附件（Attachment）指向纹理或 RBO，实际像素数据存储在纹理或 RBO 中。

2. 应该使用 **RBO（渲染缓冲对象）**。原因有两个：
   - RBO 针对离屏写入操作进行了驱动优化，写入性能通常优于纹理
   - 既然不需要在着色器中采样深度值，就不需要纹理的采样能力
   - 只有当需要在后续 pass 中读取深度信息（如阴影映射、SSAO）时才用深度纹理

3. 完整的 N×N 高斯模糊，每个像素需要采样 N² 个邻居。分成两个 pass 后：
   - 水平 pass：每个像素只采样水平方向的 N 个邻居
   - 垂直 pass：每个像素只采样垂直方向的 N 个邻居
   - 总采样次数从 N² 降为 2N，例如 9×9 核从 81 次降为 18 次
   - 数学上，二维高斯函数可分解为两个一维高斯函数的乘积，所以两步分离的结果与一步完整卷积**完全等价**

</details>

### 题目 2：实现反色效果

编写一个后处理片段着色器，实现画面反色效果（每个像素的 RGB 取反：`1.0 - color`）。

<details>
<summary>查看答案</summary>

```glsl
#version 330 core
in vec2 TexCoord;
out vec4 FragColor;

uniform sampler2D screenTexture;

void main() {
    vec4 color = texture(screenTexture, TexCoord);
    // 反色：RGB 各分量取反，Alpha 保持不变
    FragColor = vec4(1.0 - color.rgb, color.a);
}
```

对应的 OpenGL ES 2.0 版本：

```glsl
#version 100
precision mediump float;
varying vec2 TexCoord;

uniform sampler2D screenTexture;

void main() {
    vec4 color = texture2D(screenTexture, TexCoord);
    gl_FragColor = vec4(1.0 - color.rgb, color.a);
}
```

注意 ES 2.0 的差异：`#version 100`、`precision` 声明必需、`texture2D` 代替 `texture`、`varying` 代替 `in`、`gl_FragColor` 代替自定义 `out` 变量。

</details>

### 题目 3：分析 KrKr2 FBO 设计

阅读 KrKr2 的 `TVPSetRenderTarget` 函数，回答：

1. 为什么用 `glGetIntegerv(GL_FRAMEBUFFER_BINDING)` 来获取默认帧缓冲 ID，而不是直接用 0？
2. KrKr2 为什么用一个共享 FBO 而不是为每个纹理创建独立的 FBO？
3. `_CurrentRenderTarget == t` 的提前返回有什么作用？

<details>
<summary>查看答案</summary>

1. **因为 Cocos2d-x 框架可能在 KrKr2 之前就创建了自己的 FBO**。在某些情况下（例如 Cocos2d-x 的 RenderTexture 或 iOS 的 EAGL 层），"默认帧缓冲"的 ID 不是 0。通过 `glGetIntegerv(GL_FRAMEBUFFER_BINDING)` 动态获取当前绑定的帧缓冲 ID，可以正确恢复到框架的帧缓冲，而不是假设默认是 0。

2. **减少 GPU 资源占用和状态切换开销**：
   - 每个 FBO 对象会占用少量 GPU 驱动内存
   - 切换 FBO（`glBindFramebuffer`）会导致 GPU 管线刷新（pipeline flush），开销较大
   - 切换同一 FBO 上的纹理附件（`glFramebufferTexture2D`）开销比切换 FBO 要小
   - KrKr2 作为视觉小说引擎，渲染目标切换频繁（图层混合），共享 FBO 方案更高效

3. **避免重复的 `glFramebufferTexture2D` 调用**。如果当前渲染目标已经是请求的纹理 `t`，就直接返回，跳过不必要的 GL 状态变更。这是一个常见的**状态缓存**优化，减少冗余的 GPU 驱动调用。

</details>
