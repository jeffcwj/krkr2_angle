## 帧缓冲基础概念

### 什么是帧缓冲

在前几节中，我们所有的绘制操作都直接输出到屏幕上。OpenGL 在内部维护一个**默认帧缓冲**（Default Framebuffer），由窗口系统创建，包含：

| 缓冲区 | 作用 | 位深（典型） |
|---------|------|-------------|
| 颜色缓冲（Color Buffer） | 存储最终像素颜色 | RGBA8（32位） |
| 深度缓冲（Depth Buffer） | 存储每个像素的深度值 | 24位 |
| 模板缓冲（Stencil Buffer） | 存储模板测试值 | 8位 |

默认帧缓冲的内容最终被显示到屏幕上。但在实际应用中，我们经常需要**先把场景渲染到一个中间"画布"**，对画布做加工（例如模糊、色调调整），最后再把加工结果显示到屏幕——这就是**离屏渲染**，而承载这个中间画布的对象就是**帧缓冲对象（Framebuffer Object, FBO）**。

```
┌──────────────────────────────────────────────┐
│              渲染管线                          │
│                                              │
│  顶点数据 → 顶点着色器 → 光栅化 → 片段着色器  │
│                                              │
│         ┌──── 输出目标 ────┐                  │
│         │                  │                  │
│    默认帧缓冲           自定义 FBO            │
│    (直接上屏)        (离屏渲染到纹理)          │
│                         │                    │
│                    后处理着色器                │
│                         │                    │
│                    默认帧缓冲                  │
│                    (最终上屏)                  │
└──────────────────────────────────────────────┘
```

### FBO 的组成结构

一个 FBO 本身**不存储任何像素数据**，它只是一个"容器"，通过**附件（Attachment）**指向实际存储数据的对象：

```
┌─────────────── FBO ───────────────────┐
│                                       │
│  GL_COLOR_ATTACHMENT0  ──→  纹理/RBO  │
│  GL_COLOR_ATTACHMENT1  ──→  纹理/RBO  │  （MRT：多渲染目标）
│  GL_DEPTH_ATTACHMENT   ──→  纹理/RBO  │
│  GL_STENCIL_ATTACHMENT ──→  纹理/RBO  │
│  GL_DEPTH_STENCIL_ATTACHMENT ──→ RBO  │  （深度+模板合并）
│                                       │
└───────────────────────────────────────┘
```

附件可以是两种类型：

| 附件类型 | 载体 | 优点 | 缺点 | 适用场景 |
|----------|------|------|------|----------|
| 纹理附件 | `GL_TEXTURE_2D` | 可作为后续 pass 的输入纹理采样 | 创建稍复杂 | 颜色附件（需要后处理读取） |
| 渲染缓冲对象 | `GL_RENDERBUFFER` | 针对离屏写入优化，性能更好 | 无法直接采样 | 深度/模板附件（只写不读） |

## 创建和使用帧缓冲对象

### 基础 FBO 创建流程

创建一个带颜色纹理附件的 FBO 的完整代码：

```cpp
#include <glad/glad.h>
#include <GLFW/glfw3.h>
#include <iostream>

// 屏幕尺寸
const int SCR_WIDTH = 800;
const int SCR_HEIGHT = 600;

// 创建 FBO 及其附件
GLuint createFramebuffer(GLuint &colorTexture, GLuint &rbo) {
    GLuint fbo;
    // 第一步：生成 FBO
    glGenFramebuffers(1, &fbo);
    // 第二步：绑定 FBO（后续所有 framebuffer 操作作用于此 FBO）
    glBindFramebuffer(GL_FRAMEBUFFER, fbo);

    // 第三步：创建颜色纹理附件
    glGenTextures(1, &colorTexture);
    glBindTexture(GL_TEXTURE_2D, colorTexture);
    // 分配纹理存储，不传初始数据（data = nullptr）
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA,
                 SCR_WIDTH, SCR_HEIGHT, 0,
                 GL_RGBA, GL_UNSIGNED_BYTE, nullptr);
    // 设置过滤参数（FBO 纹理通常用 LINEAR）
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    // 边缘钳制，防止后处理采样越界
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    // 将纹理附加到 FBO 的颜色附件 0
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0,
                           GL_TEXTURE_2D, colorTexture, 0);

    // 第四步：创建 RBO 作为深度+模板附件
    glGenRenderbuffers(1, &rbo);
    glBindRenderbuffer(GL_RENDERBUFFER, rbo);
    // GL_DEPTH24_STENCIL8：24位深度 + 8位模板，最常用的组合
    glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH24_STENCIL8,
                          SCR_WIDTH, SCR_HEIGHT);
    // 将 RBO 附加到 FBO 的深度+模板附件
    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_STENCIL_ATTACHMENT,
                              GL_RENDERBUFFER, rbo);

    // 第五步：检查 FBO 完整性
    GLenum status = glCheckFramebufferStatus(GL_FRAMEBUFFER);
    if (status != GL_FRAMEBUFFER_COMPLETE) {
        std::cerr << "FBO 不完整！状态码: 0x"
                  << std::hex << status << std::endl;
    }

    // 第六步：解绑 FBO，恢复默认帧缓冲
    glBindFramebuffer(GL_FRAMEBUFFER, 0);
    return fbo;
}
```

### FBO 完整性检查

`glCheckFramebufferStatus` 是创建 FBO 后**必须调用**的函数。常见错误状态码：

| 状态码 | 含义 | 原因 |
|--------|------|------|
| `GL_FRAMEBUFFER_COMPLETE` | 完整，可用 | 正常 |
| `GL_FRAMEBUFFER_INCOMPLETE_ATTACHMENT` | 附件不完整 | 纹理/RBO 未正确创建或格式不对 |
| `GL_FRAMEBUFFER_INCOMPLETE_MISSING_ATTACHMENT` | 缺少附件 | FBO 至少需要一个颜色附件 |
| `GL_FRAMEBUFFER_INCOMPLETE_DIMENSIONS` | 尺寸不一致（ES） | 各附件的宽高不匹配（ES 2.0 特有） |
| `GL_FRAMEBUFFER_UNSUPPORTED` | 格式组合不支持 | 附件格式组合在当前驱动/GPU 不支持 |

### 使用 FBO 进行离屏渲染

有了 FBO 后，离屏渲染的流程如下：

```cpp
// ===== 第一步：渲染场景到 FBO =====
glBindFramebuffer(GL_FRAMEBUFFER, fbo);  // 绑定自定义 FBO
glClearColor(0.1f, 0.1f, 0.1f, 1.0f);
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
glEnable(GL_DEPTH_TEST);  // 离屏渲染也需要深度测试

// 用普通着色器渲染场景
sceneShader.use();
drawScene();  // 绘制所有场景对象

// ===== 第二步：渲染 FBO 纹理到屏幕 =====
glBindFramebuffer(GL_FRAMEBUFFER, 0);    // 切回默认帧缓冲
glClearColor(1.0f, 1.0f, 1.0f, 1.0f);
glClear(GL_COLOR_BUFFER_BIT);
glDisable(GL_DEPTH_TEST);  // 全屏四边形不需要深度测试

// 用后处理着色器绘制全屏四边形
postProcessShader.use();
glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, colorTexture);  // 绑定 FBO 颜色纹理
drawFullscreenQuad();  // 绘制覆盖全屏的四边形

glfwSwapBuffers(window);
```

### 全屏四边形（Fullscreen Quad）

后处理的核心是用一个覆盖整个屏幕的四边形来采样 FBO 纹理。这个四边形的顶点数据是固定的：

```cpp
// 全屏四边形顶点数据
// 位置（NDC坐标）      纹理坐标
float quadVertices[] = {
    // 第一个三角形
    -1.0f,  1.0f,    0.0f, 1.0f,  // 左上
    -1.0f, -1.0f,    0.0f, 0.0f,  // 左下
     1.0f, -1.0f,    1.0f, 0.0f,  // 右下
    // 第二个三角形
    -1.0f,  1.0f,    0.0f, 1.0f,  // 左上
     1.0f, -1.0f,    1.0f, 0.0f,  // 右下
     1.0f,  1.0f,    1.0f, 1.0f,  // 右上
};

GLuint quadVAO, quadVBO;

void initFullscreenQuad() {
    glGenVertexArrays(1, &quadVAO);
    glGenBuffers(1, &quadVBO);
    glBindVertexArray(quadVAO);
    glBindBuffer(GL_ARRAY_BUFFER, quadVBO);
    glBufferData(GL_ARRAY_BUFFER, sizeof(quadVertices),
                 quadVertices, GL_STATIC_DRAW);
    // 位置属性：location=0, 2个float
    glEnableVertexAttribArray(0);
    glVertexAttribPointer(0, 2, GL_FLOAT, GL_FALSE,
                          4 * sizeof(float), (void*)0);
    // 纹理坐标属性：location=1, 2个float
    glEnableVertexAttribArray(1);
    glVertexAttribPointer(1, 2, GL_FLOAT, GL_FALSE,
                          4 * sizeof(float),
                          (void*)(2 * sizeof(float)));
    glBindVertexArray(0);
}

void drawFullscreenQuad() {
    glBindVertexArray(quadVAO);
    glDrawArrays(GL_TRIANGLES, 0, 6);  // 2个三角形 = 6个顶点
    glBindVertexArray(0);
}
```
