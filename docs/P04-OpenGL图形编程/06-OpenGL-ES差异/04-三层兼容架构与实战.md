## 一、为什么需要三层架构

上一节解决了跨平台的**着色器精度**兼容问题。但真实项目的跨平台渲染不止精度——还有：

- **扩展检测**：同一功能在不同平台有不同的扩展名（EXT/OES/ARB/NV/ARM），需要逐个检查
- **函数指针加载**：GL 扩展函数不在标准头文件中声明，需要在运行时通过 `GetProcAddress` 获取函数指针（函数指针就是一个变量，存储了某个函数在内存中的地址，通过这个变量可以调用对应的函数）
- **NPOT 纹理限制**：ES 2.0 的非 2 的幂纹理不支持 `GL_REPEAT` 和 Mipmap（一种让远处纹理不闪烁的多级缩小技术，第 4 章已详讲）
- **VAO 支持差异**：ES 2.0 的 VAO（Vertex Array Object，顶点数组对象——把顶点属性配置打包存储的对象，第 2 章已详讲）需要扩展
- **数值上限差异**：最大纹理尺寸、最大纹理单元数等在不同 GPU 上不同

如果把这些 `#ifdef` 分支散布在渲染代码的各处，会造成"平台分支爆炸"——每加一个功能都要写 4 个平台的分支，维护成本呈指数增长。

**解决方案**：把所有平台相关的 `#ifdef` 集中在底层，中间层统一输出"能力开关"，上层只读取开关做判断。

## 二、三层架构总览

```
┌─────────────────────────────────────┐
│ 第三层：渲染实现层                    │
│ 读取能力开关决定代码路径，无 #ifdef   │
├─────────────────────────────────────┤
│ 第二层：能力探测层                    │
│ GL 版本/扩展/上限 → 能力结构体        │
├─────────────────────────────────────┤
│ 第一层：上下文与函数入口加载层         │
│ EGL/WGL/GLX + GetProcAddress        │
│ 平台 #ifdef 集中在这一层              │
└─────────────────────────────────────┘
```

**关键设计思想**：

1. **底层（第一层）**——唯一允许 `#ifdef` 的层。Windows 用 `wglGetProcAddress`，Android 用 `eglGetProcAddress`，Linux 用 `glXGetProcAddress`
2. **中间层（第二层）**——把平台差异转化为布尔开关和数值上限。上层只读 `supportsNPOTFull == true` 这样的开关
3. **上层（第三层）**——纯渲染逻辑，零平台代码

这种分层在实际游戏引擎中非常普遍。Unity 的 `SystemInfo` 类、Unreal 的 `FOpenGLBase::SupportsFeature()`、Cocos2d-x 的 `Configuration::supportsNPOT()` 都是同样的思路。

## 三、代码示例 1：能力结构体设计

```cpp
// 文件：GLCapabilities.h
// 将所有平台差异抽象为布尔开关和数值上限
// 在 GL 上下文创建后调用一次 probeGLCapabilities() 填充

#pragma once
#include <string>

struct GLCapabilities {
    // === 基本信息 ===
    bool isGLES = false;           // 是 ES 还是桌面 GL
    int majorVersion = 0;          // 主版本号
    int minorVersion = 0;          // 次版本号

    // === 纹理能力 ===
    int maxTextureSize = 0;        // 最大纹理尺寸（像素）
    bool supportsNPOTFull = false; // NPOT 纹理是否无限制
    //   true：桌面 GL 2.0+ 或 ES 3.0+
    //   false：ES 2.0（NPOT 只能 CLAMP_TO_EDGE，不能 Mipmap）

    // === 扩展能力 ===
    bool hasVAO = false;           // ES 2.0 需 OES_vertex_array_object 扩展
    bool hasCopyImage = false;     // GPU 间纹理拷贝（不经 CPU）
    bool hasUnpackSubimage = false;// 纹理上传行间距控制
    bool hasFramebufferFetch = false; // 片段着色器直读帧缓冲
    bool hasClearTexture = false;  // 快速纹理清除

    // === 着色器能力 ===
    int maxVaryingVectors = 0;     // 最大 varying 向量数
    int maxVertexAttribs = 0;      // 最大顶点属性数
    int maxTextureUnits = 0;       // 片段着色器最大纹理单元数
};

// 在 GL 上下文创建后调用一次，填充所有字段
GLCapabilities probeGLCapabilities();
```

> **设计要点**：结构体只含布尔开关和数值上限，不含函数指针或 `#ifdef`。上层代码做 `if (caps.supportsNPOTFull)` 判断即可。

## 四、代码示例 2：能力探测实现

```cpp
// 文件：GLCapabilities.cpp
// 在 GL 上下文创建后调用一次，探测当前 GPU 的所有能力

#include "GLCapabilities.h"
#include <cstring>
#include <cstdio>

// 辅助函数：检查是否支持某个 GL 扩展
// 参考了 KrKr2 的 TVPCheckGLExtension() 实现
static bool checkExtension(const char* extName) {
    const char* extensions = (const char*)glGetString(GL_EXTENSIONS);
    // ┌─────────────────────────────────────────────┐
    // │ glGetString(GL_EXTENSIONS)                   │
    // │ 功能：返回当前 GL 支持的所有扩展名              │
    // │ 参数：GL_EXTENSIONS                           │
    // │ 返回值：以空格分隔的扩展名字符串，如             │
    // │   "GL_OES_vertex_array_object GL_EXT_..."     │
    // │ 注意：GL 3.0+ 推荐用 glGetStringi 逐个查询     │
    // │   但 ES 2.0 只有此方法                         │
    // └─────────────────────────────────────────────┘
    if (!extensions) return false;
    return strstr(extensions, extName) != nullptr;
}

GLCapabilities probeGLCapabilities() {
    GLCapabilities caps;

    // === 1. 检测 ES 还是桌面 ===
    const char* version = (const char*)glGetString(GL_VERSION);
    // glGetString(GL_VERSION) 返回版本字符串：
    //   桌面："4.6.0 NVIDIA 537.58"  ES："OpenGL ES 3.0 V@..."
    caps.isGLES = (version && strstr(version, "OpenGL ES") != nullptr);

    // === 2. 解析版本号 ===
    if (caps.isGLES) {
        sscanf(version, "OpenGL ES %d.%d",
               &caps.majorVersion, &caps.minorVersion);
    } else {
        sscanf(version, "%d.%d",
               &caps.majorVersion, &caps.minorVersion);
    }

    // === 3. 查询数值上限 ===
    glGetIntegerv(GL_MAX_TEXTURE_SIZE, &caps.maxTextureSize);
    // glGetIntegerv(pname, data)：查询 GL 整数状态值
    //   pname=GL 常量，data=输出指针。常用值见下方
    glGetIntegerv(GL_MAX_VARYING_VECTORS, &caps.maxVaryingVectors);
    glGetIntegerv(GL_MAX_VERTEX_ATTRIBS, &caps.maxVertexAttribs);
    glGetIntegerv(GL_MAX_TEXTURE_IMAGE_UNITS, &caps.maxTextureUnits);

    // === 4. NPOT 支持——桌面 GL 2.0+ 或 ES 3.0+ 完全支持
    caps.supportsNPOTFull = !caps.isGLES || caps.majorVersion >= 3;

    // === 5. VAO 支持——桌面 3.0+/ES 3.0+ 核心，ES 2.0 需扩展
    caps.hasVAO = (!caps.isGLES && caps.majorVersion >= 3)
              || (caps.isGLES && caps.majorVersion >= 3)
              || checkExtension("GL_OES_vertex_array_object");

    // === 6. 其他扩展——同一功能可能有多个扩展名
    caps.hasUnpackSubimage = checkExtension("GL_EXT_unpack_subimage");

    // copy_image：4 种可能的扩展名
    caps.hasCopyImage = checkExtension("GL_EXT_copy_image")
                     || checkExtension("GL_OES_copy_image")
                     || checkExtension("GL_ARB_copy_image")
                     || checkExtension("GL_NV_copy_image");

    // framebuffer_fetch：3 种可能的扩展名
    caps.hasFramebufferFetch =
           checkExtension("GL_EXT_shader_framebuffer_fetch")
        || checkExtension("GL_ARM_shader_framebuffer_fetch")
        || checkExtension("GL_NV_shader_framebuffer_fetch");

    caps.hasClearTexture = checkExtension("GL_EXT_clear_texture")
                        || checkExtension("GL_ARB_clear_texture");

    return caps;
}
```

> **为什么同一功能有这么多扩展名？** GL 扩展体系允许厂商独立提出新功能。同一功能可能先由 NVIDIA 提出（`NV`），再由标准委员会 ARB 采纳（`ARB`），再被 ES 移植（`OES`/`EXT`）。健壮的代码必须检查所有可能的名字。

## 五、代码示例 3：基于能力的渲染路径选择

有了能力结构体，渲染实现层就可以不用任何 `#ifdef`——只读取布尔开关：

```cpp
// 文件：Renderer.cpp
// 渲染实现层——不含任何 #ifdef，只读取能力结构体

class Renderer {
    GLCapabilities mCaps;  // 在初始化时调用 probeGLCapabilities() 填充
public:
    void setupTexture(GLuint tex, int width, int height,
                      const unsigned char* data) {
        glBindTexture(GL_TEXTURE_2D, tex);
        // ┌─────────────────────────────────────────────┐
        // │ glBindTexture(target, texture)               │
        // │ 功能：绑定纹理对象到当前活跃的纹理单元        │
        // │ target：纹理类型                              │
        // │   GL_TEXTURE_2D — 2D 纹理（最常用）           │
        // │   GL_TEXTURE_CUBE_MAP — 立方体贴图            │
        // │ texture：纹理对象 ID（glGenTextures 生成）     │
        // │   传入 0 表示解绑                             │
        // └─────────────────────────────────────────────┘

        glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, width, height, 0,
                     GL_RGBA, GL_UNSIGNED_BYTE, data);
        // ┌─────────────────────────────────────────────┐
        // │ glTexImage2D(target, level, internalformat,  │
        // │   width, height, border, format, type, data) │
        // │ 功能：创建 2D 纹理图像                        │
        // │ level：Mipmap 级别，0 = 原始尺寸               │
        // │ internalformat：GPU 内部格式（GL_RGBA）       │
        // │ border：必须为 0（历史遗留）                   │
        // │ format：输入像素格式                           │
        // │ type：数据类型（GL_UNSIGNED_BYTE）             │
        // │ data：像素数据指针（nullptr = 只分配不上传）    │
        // └─────────────────────────────────────────────┘

        // 根据能力选择纹理参数——不用 #ifdef
        bool isPOT = ((width & (width - 1)) == 0)
                  && ((height & (height - 1)) == 0);
        // 判断是否为 2 的幂：n & (n-1) == 0
        // 例：256 = 0b100000000，255 = 0b011111111
        //     256 & 255 = 0 → 是 2 的幂

        if (mCaps.supportsNPOTFull || isPOT) {
            // 桌面 GL 或 ES 3.0+，或者本身就是 POT 纹理
            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S,
                            GL_REPEAT);
            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T,
                            GL_REPEAT);
            // ┌─────────────────────────────────────────┐
            // │ glTexParameteri(target, pname, param)    │
            // │ 功能：设置纹理参数                        │
            // │ GL_TEXTURE_WRAP_S / _T：纹理坐标超出     │
            // │   [0,1] 范围时的行为                      │
            // │   GL_REPEAT — 重复平铺                    │
            // │   GL_CLAMP_TO_EDGE — 拉伸边缘像素        │
            // │   GL_MIRRORED_REPEAT — 镜像重复           │
            // └─────────────────────────────────────────┘

            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER,
                            GL_LINEAR_MIPMAP_LINEAR);
            // GL_TEXTURE_MIN_FILTER：纹理缩小时的过滤方式
            //   GL_LINEAR_MIPMAP_LINEAR — 三线性过滤（最高质量）
            //   在两个最接近的 Mipmap 级别之间做线性插值

            glGenerateMipmap(GL_TEXTURE_2D);
            // ┌─────────────────────────────────────────┐
            // │ glGenerateMipmap(target)                 │
            // │ 功能：自动生成 Mipmap 链                  │
            // │   从第 0 级纹理逐级缩小（宽高各除 2）     │
            // │   直到 1×1                               │
            // │ 为什么需要：远处物体用低分辨率纹理可       │
            // │   避免闪烁（摩尔纹）并提高性能             │
            // └─────────────────────────────────────────┘
        } else {
            // ES 2.0 + NPOT——必须使用安全的参数
            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S,
                            GL_CLAMP_TO_EDGE);
            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T,
                            GL_CLAMP_TO_EDGE);
            glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER,
                            GL_LINEAR);
            // GL_LINEAR：双线性过滤（不使用 Mipmap）
            // 不生成 Mipmap——ES 2.0 的 NPOT 纹理不允许 Mipmap
        }

        glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER,
                        GL_LINEAR);
        // GL_TEXTURE_MAG_FILTER：纹理放大时的过滤方式
        //   GL_LINEAR — 双线性过滤（平滑，推荐）
        //   GL_NEAREST — 最近邻（像素风格，无平滑）
    }
};
```

> **观察**：整个函数没有一个 `#ifdef`，但正确处理了 ES 2.0 NPOT 纹理限制。把"平台是什么"转化为"能做什么"——这就是能力结构体的价值。

## 常见错误及解决方案

### 错误一：GL 上下文创建前调用探测

**现象**：`glGetString(GL_VERSION)` 返回 `nullptr`，崩溃或全零。**原因**：`gl*` 函数必须在 GL 上下文创建后调用。**修复**——确保正确顺序：

```cpp
glfwInit();                         // 1. 初始化窗口库
window = glfwCreateWindow(...);     // 2. 创建窗口
glfwMakeContextCurrent(window);     // 3. 设置 GL 上下文
gladLoadGLLoader((GLADloadproc)     // 4. 加载函数指针
    glfwGetProcAddress);
GLCapabilities caps =               // 5. 现在才能探测
    probeGLCapabilities();
```

### 错误二：`strstr` 扩展名误匹配

**现象**：检查 `GL_EXT_copy_image` 误匹配 `GL_EXT_copy_image_compressed`。**原因**：子串匹配不是完整单词匹配。**修复**——边界检查：

```cpp
static bool checkExtensionStrict(const char* extName) {
    const char* ext = (const char*)glGetString(GL_EXTENSIONS);
    if (!ext) return false;
    size_t len = strlen(extName);
    const char* pos = ext;
    while ((pos = strstr(pos, extName)) != nullptr) {
        bool startOK = (pos == ext || *(pos - 1) == ' ');
        bool endOK = (pos[len] == ' ' || pos[len] == '\0');
        if (startOK && endOK) return true;
        pos += len;
    }
    return false;
}
```

### 错误三：每帧重复查询扩展

**现象**：性能浪费。**修复**——初始化时查询一次，后续读结构体：

```cpp
// 错误——每帧执行 glGetString + strstr
void render() {
    if (checkExtension("GL_EXT_copy_image")) { ... }
}
// 正确——读布尔值，零开销
GLCapabilities gCaps;
void init() { gCaps = probeGLCapabilities(); }
void render() {
    if (gCaps.hasCopyImage) { ... }  // 零开销
}

## 本节小结

- 三层架构的核心思想：**底层集中 `#ifdef`，中层输出能力开关，上层零平台代码**
- **能力结构体**用布尔开关和数值上限表示 GPU 能力，是中间层和上层的通信接口
- **能力探测**通过 `glGetString`（版本/扩展）和 `glGetIntegerv`（数值上限）在运行时采集 GPU 信息
- 同一 GL 功能可能有**多个扩展名**（EXT/OES/ARB/NV/ARM），健壮的检测必须全部检查
- 基于能力的渲染路径选择让**渲染层不包含任何 `#ifdef`**——只读取布尔开关做分支
- NPOT 纹理在 ES 2.0 上有严格限制（不能 `GL_REPEAT`、不能 Mipmap），能力结构体帮助正确处理

### 题目 2：修复扩展检测 bug

以下代码有两个 bug，找出并修复：

```cpp
bool hasExtension(const char* name) {
    const char* ext = (const char*)glGetString(GL_EXTENSIONS);
    return strstr(ext, name) != nullptr;
}
```

<details>
<summary>查看答案</summary>

**Bug 1**：`glGetString` 可能返回 `nullptr`，`strstr(nullptr, ...)` 是未定义行为。
**Bug 2**：子串匹配，`"GL_OES_texture"` 会误匹配 `"GL_OES_texture_float"`。

```cpp
bool hasExtension(const char* name) {
    const char* ext = (const char*)glGetString(GL_EXTENSIONS);
    if (!ext) return false;          // 修复 Bug 1
    size_t len = strlen(name);
    const char* pos = ext;
    while ((pos = strstr(pos, name)) != nullptr) {
        bool atStart = (pos == ext || *(pos - 1) == ' ');
        bool atEnd = (pos[len] == ' ' || pos[len] == '\0');
        if (atStart && atEnd) return true;  // 修复 Bug 2
        pos += len;
    }
    return false;
}
