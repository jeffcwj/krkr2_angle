## 1. 运行时格式检测

### 1.1 为什么需要运行时检测

上一节介绍了 BC / PVRTC / ETC / ASTC 等格式的特点和选型。但"离线选好格式"只是一半工作——运行时必须验证当前设备是否真的支持该格式。

原因有三：
1. **同一操作系统不同 GPU 差异大**。Android 上高通 Adreno 和 ARM Mali 对 ASTC 的支持版本不同。
2. **驱动更新可能增减格式**。某些旧驱动虽然声称支持 ES3.0，但实际格式列表中缺少 ETC2。
3. **扩展字符串不总可靠**。用 `glGetString(GL_EXTENSIONS)` 能看到扩展名，但不代表所有子格式都在；直接查询格式列表更安全。

### 1.2 查询 API：glGetIntegerv

OpenGL 提供两步查询：

```
┌─────────────────────────────────────────────────────────────────┐
│  GL 函数：glGetIntegerv                                         │
├─────────────────────────────────────────────────────────────────┤
│  void glGetIntegerv(GLenum pname, GLint *params);               │
│                                                                 │
│  参数：                                                         │
│    pname  — 要查询的状态名，常用值：                              │
│             GL_NUM_COMPRESSED_TEXTURE_FORMATS (0x86A3)           │
│               返回驱动支持的压缩格式总数                          │
│             GL_COMPRESSED_TEXTURE_FORMATS (0x86A4)               │
│               返回所有支持格式的 GLenum 数组                      │
│             GL_MAX_TEXTURE_SIZE                                  │
│               返回纹理最大尺寸（单边像素数）                       │
│    params — 接收结果的整数指针/数组                               │
│                                                                 │
│  返回值：无（结果写入 params）                                    │
│  错误：GL_INVALID_ENUM — pname 不合法                           │
└─────────────────────────────────────────────────────────────────┘
```

完整流程如下：

```cpp
// 示例 1：查询当前设备支持的全部压缩纹理格式
#include <cstdio>
#include <set>
#include <vector>

// 需要 OpenGL / OpenGL ES 头文件，此处用条件编译示意
#ifdef __ANDROID__
#include <GLES3/gl3.h>
#else
#include <GL/glew.h>  // 桌面端常用 GLEW
#endif

// 将查询结果存入 std::set，方便 O(log n) 查找
std::set<GLenum> QueryCompressedFormats() {
    GLint count = 0;
    glGetIntegerv(GL_NUM_COMPRESSED_TEXTURE_FORMATS, &count);
    // count 为 0 表示该设备不支持任何压缩格式（极罕见）

    std::set<GLenum> result;
    if (count <= 0) return result;

    std::vector<GLint> formats(static_cast<size_t>(count));
    glGetIntegerv(GL_COMPRESSED_TEXTURE_FORMATS, formats.data());

    for (GLint f : formats) {
        result.insert(static_cast<GLenum>(f));
    }
    return result;
}

int main() {
    // 假设 OpenGL 上下文已初始化
    auto supported = QueryCompressedFormats();
    std::printf("设备支持 %zu 种压缩格式：\n", supported.size());
    for (GLenum fmt : supported) {
        std::printf("  0x%04X\n", fmt);
    }
    return 0;
}
```

### 1.3 KrKr2 源码分析：TVPInitTextureFormatList

KrKr2 的实现在 `RenderManager_ogl.cpp` 第 193-208 行，逻辑与上面的示例几乎一样：

```cpp
// KrKr2 源码（简化注释版）
// 文件：krkr2/cpp/core/visual/ogl/RenderManager_ogl.cpp 第 193-208 行

static std::set<GLenum> TVPTextureFormats;          // 全局格式集合

void TVPInitTextureFormatList() {
    if (TVPTextureFormats.empty()) {                 // 只初始化一次
        GLint nTexFormats;
        glGetIntegerv(GL_NUM_COMPRESSED_TEXTURE_FORMATS, &nTexFormats);
        std::vector<GLint> texFormats;
        texFormats.resize(nTexFormats);
        glGetIntegerv(GL_COMPRESSED_TEXTURE_FORMATS, &texFormats.front());
        for (GLint f : texFormats) {
            TVPTextureFormats.insert(f);             // 存入 std::set
        }
    }
}

bool TVPIsSupportTextureFormat(GLenum fmt) {
    return TVPTextureFormats.end() != TVPTextureFormats.find(fmt);
}
```

**设计要点：**
- 使用 `std::set` 而非 `std::vector`，查找复杂度从 O(n) 降到 O(log n)。
- `empty()` 保护确保只查询一次——GL 查询虽然不贵，但重复调用无意义。
- 暴露 `TVPIsSupportTextureFormat` 作为外部接口，调用方只需传入 GLenum 常量即可判断。

KrKr2 还在 `TVPGetOpenGLInfo()`（第 277-434 行）中维护了一张完整的"GLenum → 人可读名称"映射表，涵盖 S3TC、PVRTC、ETC1/ETC2、ASTC 共计 50+ 种格式枚举，用于调试日志输出。这张表本身也是查阅格式常量的好参考。

## 2. glCompressedTexImage2D 详解

### 2.1 函数签名与参数

这是上传压缩纹理数据到 GPU 的核心函数：

```
┌─────────────────────────────────────────────────────────────────┐
│  GL 函数：glCompressedTexImage2D                                │
├─────────────────────────────────────────────────────────────────┤
│  void glCompressedTexImage2D(                                   │
│      GLenum  target,          // 纹理目标，通常 GL_TEXTURE_2D   │
│      GLint   level,           // Mipmap 层级，0 = 最大层        │
│      GLenum  internalformat,  // 压缩格式枚举（见下表）          │
│      GLsizei width,           // 该 mip 层的像素宽度            │
│      GLsizei height,          // 该 mip 层的像素高度            │
│      GLint   border,          // 必须为 0（历史遗留参数）        │
│      GLsizei imageSize,       // 该 mip 层压缩后的字节数        │
│      const void *data         // 压缩数据指针                   │
│  );                                                             │
│                                                                 │
│  常见 internalformat 值：                                       │
│    0x83F0 — GL_COMPRESSED_RGB_S3TC_DXT1_EXT     (BC1, 4bpp)    │

│    0x8C02 — GL_COMPRESSED_RGBA_PVRTC_4BPPV1_IMG (PVRTC 4bpp)  │
│    0x8D64 — GL_COMPRESSED_RGB8_ETC1             (ETC1, 4bpp)   │
│    0x9278 — GL_COMPRESSED_RGBA8_ETC2_EAC        (ETC2, 8bpp)   │
│    0x93B0 — GL_COMPRESSED_RGBA_ASTC_4x4_KHR     (ASTC 4×4)    │
│    0x93B7 — GL_COMPRESSED_RGBA_ASTC_8x8_KHR     (ASTC 8×8)    │
│                                                                 │
│  错误：                                                         │
│    GL_INVALID_ENUM  — internalformat 不被设备支持               │
│    GL_INVALID_VALUE — imageSize 与 width/height/format 不匹配  │
│    GL_INVALID_VALUE — width 或 height 为负数                    │
│    GL_INVALID_OPERATION — 当前没有绑定纹理对象                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 imageSize 计算公式

**这是最容易出错的参数。** 压缩纹理不像 RGBA8 那样简单地用 `width × height × 4` 算字节。正确公式是：

```
imageSize = blocksX × blocksY × bytesPerBlock

其中：
  blocksX = ceil(width  / blockWidth)     ← 向上取整！
  blocksY = ceil(height / blockHeight)    ← 向上取整！
```

不同格式的参数：

| 格式 | blockWidth | blockHeight | bytesPerBlock | bpp |
|------|-----------|------------|--------------|-----|
| BC1 / DXT1 | 4 | 4 | 8 | 4 |
| BC3 / DXT5 | 4 | 4 | 16 | 8 |
| BC5 | 4 | 4 | 16 | 8 |
| PVRTC 4bpp | — | — | — | 4 (特殊算法) |
| ETC1 | 4 | 4 | 8 | 4 |
| ETC2 RGBA | 4 | 4 | 16 | 8 |
| ASTC 4×4 | 4 | 4 | 16 | 8 |
| ASTC 6×6 | 6 | 6 | 16 | ≈3.56 |
| ASTC 8×8 | 8 | 8 | 16 | 2 |

> **注意：** PVRTC 比较特殊，它的大小计算不完全是块计算，而是 `max(width, 8) × max(height, 8) / 2`（4bpp 时），因为 PVRTC 要求纹理尺寸为 2 的幂次（Power of Two，简称 POT——即宽高必须是 2、4、8、16、32…… 这样的值）。

```cpp
// 示例 2：通用 imageSize 计算函数
#include <algorithm>
#include <cstddef>
#include <cstdio>

struct BlockFormat {
    int blockW;       // 块宽度（像素）
    int blockH;       // 块高度（像素）
    int blockBytes;   // 每个块的字节数
};

// 各格式参数表
static const BlockFormat FMT_BC1      = {4, 4, 8};
static const BlockFormat FMT_BC3      = {4, 4, 16};
static const BlockFormat FMT_ETC1     = {4, 4, 8};
static const BlockFormat FMT_ETC2RGBA = {4, 4, 16};
static const BlockFormat FMT_ASTC4x4  = {4, 4, 16};
static const BlockFormat FMT_ASTC6x6  = {6, 6, 16};
static const BlockFormat FMT_ASTC8x8  = {8, 8, 16};

size_t CalcImageSize(int width, int height, const BlockFormat &fmt) {
    // 向上取整：(width + blockW - 1) / blockW
    int bx = (width  + fmt.blockW - 1) / fmt.blockW;
    int by = (height + fmt.blockH - 1) / fmt.blockH;
    // 最少 1 个块
    bx = std::max(bx, 1);
    by = std::max(by, 1);
    return static_cast<size_t>(bx) * static_cast<size_t>(by)
         * static_cast<size_t>(fmt.blockBytes);
}

int main() {
    // 1024×512 的 ASTC 8×8
    size_t s1 = CalcImageSize(1024, 512, FMT_ASTC8x8);
    std::printf("ASTC 8x8 1024x512: %zu bytes (%.1f KB)\n", s1, s1 / 1024.0);
    // 预期：ceil(1024/8)=128, ceil(512/8)=64, 128×64×16 = 131072 (128 KB)

    // 1024×1024 的 ETC2 RGBA
    size_t s2 = CalcImageSize(1024, 1024, FMT_ETC2RGBA);
    std::printf("ETC2 RGBA 1024x1024: %zu bytes (%.1f KB)\n", s2, s2 / 1024.0);
    // 预期：256×256×16 = 1048576 (1024 KB = 1 MB)

    // 非 4 对齐：100×100 的 BC1
    size_t s3 = CalcImageSize(100, 100, FMT_BC1);
    std::printf("BC1 100x100: %zu bytes\n", s3);
    // 预期：ceil(100/4)=25, 25×25×8 = 5000 bytes

    return 0;
}
```

## 3. 完整压缩纹理加载模块

下面把"检测 → 选择 → 计算 → 上传 → 回退"串成一个完整可运行的模块：

```cpp
// 示例 3：跨平台压缩纹理加载模块
#include <cstdio>
#include <cstdlib>
#include <cstring>
#include <set>
#include <vector>
#include <algorithm>

#ifdef __ANDROID__
#include <GLES3/gl3.h>
#else
#include <GL/glew.h>
#endif

// ---- 格式常量 ----
static const GLenum GL_ASTC_RGBA_4x4  = 0x93B0;
static const GLenum GL_ETC2_RGBA8_EAC = 0x9278;
static const GLenum GL_ETC1_RGB8      = 0x8D64;
static const GLenum GL_BC3_RGBA       = 0x83F3;
static const GLenum GL_BC1_RGB        = 0x83F0;

// ---- 格式检测（参考 KrKr2 TVPInitTextureFormatList）----
static std::set<GLenum> g_supportedFormats;

void InitCompressedFormatList() {
    if (!g_supportedFormats.empty()) return;  // 只初始化一次
    GLint count = 0;
    glGetIntegerv(GL_NUM_COMPRESSED_TEXTURE_FORMATS, &count);
    if (count <= 0) return;
    std::vector<GLint> fmts(static_cast<size_t>(count));
    glGetIntegerv(GL_COMPRESSED_TEXTURE_FORMATS, fmts.data());
    for (GLint f : fmts) {
        g_supportedFormats.insert(static_cast<GLenum>(f));
    }
    std::printf("[TexLoader] 设备支持 %zu 种压缩格式\n",
                g_supportedFormats.size());
}

bool IsFormatSupported(GLenum fmt) {
    return g_supportedFormats.count(fmt) > 0;
}

// ---- 格式选择 ----
struct ChosenFormat {
    GLenum  glEnum;       // 选中的 GL 格式枚举
    int     blockW;       // 块宽
    int     blockH;       // 块高
    int     blockBytes;   // 每块字节数
    bool    isCompressed; // 是否压缩格式（false = 回退到 RGBA8）
};

ChosenFormat PickBestFormat(bool needAlpha) {
    // 优先级：ASTC 4×4 > ETC2 RGBA > BC3 > ETC1 > BC1 > RGBA8 回退
    if (IsFormatSupported(GL_ASTC_RGBA_4x4)) {
        return {GL_ASTC_RGBA_4x4, 4, 4, 16, true};
    }
    if (needAlpha && IsFormatSupported(GL_ETC2_RGBA8_EAC)) {
        return {GL_ETC2_RGBA8_EAC, 4, 4, 16, true};
    }
    if (needAlpha && IsFormatSupported(GL_BC3_RGBA)) {
        return {GL_BC3_RGBA, 4, 4, 16, true};
    }
    if (!needAlpha && IsFormatSupported(GL_ETC1_RGB8)) {
        return {GL_ETC1_RGB8, 4, 4, 8, true};
    }
    if (!needAlpha && IsFormatSupported(GL_BC1_RGB)) {
        return {GL_BC1_RGB, 4, 4, 8, true};
    }
    // 全部不支持，回退到未压缩 RGBA8
    return {GL_RGBA, 0, 0, 0, false};
}

// ---- imageSize 计算 ----
size_t CalcCompressedImageSize(int w, int h, const ChosenFormat &fmt) {
    int bx = std::max(1, (w + fmt.blockW - 1) / fmt.blockW);
    int by = std::max(1, (h + fmt.blockH - 1) / fmt.blockH);
    return static_cast<size_t>(bx) * by * fmt.blockBytes;
}

// ---- 上传 ----
bool UploadCompressedTexture(GLuint texId,
                             const ChosenFormat &fmt,
                             int width, int height,
                             const void *compressedData,
                             size_t dataSize) {
    glBindTexture(GL_TEXTURE_2D, texId);

    if (!fmt.isCompressed) {
        // 回退路径：当作 RGBA8 上传
        std::printf("[TexLoader] 回退到 RGBA8 上传 (%dx%d)\n", width, height);
        glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, width, height,
                     0, GL_RGBA, GL_UNSIGNED_BYTE, compressedData);
    } else {
        size_t expectedSize = CalcCompressedImageSize(width, height, fmt);
        if (dataSize != expectedSize) {
            std::printf("[TexLoader] 错误：数据大小 %zu != 预期 %zu\n",
                        dataSize, expectedSize);
            return false;
        }
        glCompressedTexImage2D(
            GL_TEXTURE_2D,                           // target
            0,                                       // level（mip 0）
            fmt.glEnum,                              // internalformat
            width, height,                           // 尺寸
            0,                                       // border（必须 0）
            static_cast<GLsizei>(dataSize),          // imageSize
            compressedData                           // 数据指针
        );

        // 检查 GL 错误
        GLenum err = glGetError();
        if (err != GL_NO_ERROR) {
            std::printf("[TexLoader] glCompressedTexImage2D 失败: 0x%04X\n", err);
            return false;
        }
    }

    // 设置采样参数
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

    std::printf("[TexLoader] 上传成功: %dx%d, 格式 0x%04X, %zu 字节\n",
                width, height, fmt.glEnum, dataSize);
    return true;
}
```

## 常见错误及解决方案

### 错误 1：imageSize 计算使用像素字节数而非块字节数

**症状：** `glCompressedTexImage2D` 返回 `GL_INVALID_VALUE`。

**错误代码：**
```cpp
// ❌ 错误：把 imageSize 当作未压缩字节数
GLsizei wrongSize = width * height * 4;  // 这是 RGBA8 的大小！
glCompressedTexImage2D(GL_TEXTURE_2D, 0, 0x93B0,
                       width, height, 0, wrongSize, data);
```

**正确做法：** 按块计算。
```cpp
// ✅ 正确：按块数 × 每块字节数
int bx = (width + 3) / 4;
int by = (height + 3) / 4;
GLsizei correctSize = bx * by * 16;  // ASTC 4×4 每块 16 字节
```

### 错误 2：NPOT 纹理不对齐

**症状：** 边缘像素花屏或闪烁，`glGetError()` 可能返回 `GL_INVALID_VALUE`。

**原因：** 例如 100×100 的纹理，块数应为 `ceil(100/4) = 25`，而非 `100/4 = 25`（恰好整除时没问题，但 101×100 就会出错：`ceil(101/4) = 26`，若用整数除法得到 25 则少算了一列块）。

**解决：** 始终用 `(width + blockW - 1) / blockW` 做向上取整。

### 错误 3：格式不支持却强制上传

**症状：** `GL_INVALID_ENUM` 或纹理显示全黑。

**原因：** 没有在上传前检查格式支持。例如在不支持 ASTC 的旧 Android 设备上传 ASTC 数据。

**解决：** 上传前必须查询格式支持集合（参考本节第 1 节的 `QueryCompressedFormats()`），不支持则回退到下一优先级格式或 RGBA8。

## 本节小结

1. 运行时格式检测使用 `glGetIntegerv(GL_NUM_COMPRESSED_TEXTURE_FORMATS)` + `glGetIntegerv(GL_COMPRESSED_TEXTURE_FORMATS)` 两步查询，比只看扩展字符串更可靠。
2. `glCompressedTexImage2D` 的 `imageSize` 参数必须按块计算：`ceil(w/blockW) × ceil(h/blockH) × bytesPerBlock`，不是像素字节数。
3. KrKr2 的 `TVPInitTextureFormatList()` 用 `std::set` 存储格式集合，配合 `TVPIsSupportTextureFormat()` 提供 O(log n) 查询。
4. 完整的纹理加载链路是：启动时检测 → 选择最优格式 → 按块计算 imageSize → 上传 → 错误时回退 RGBA8。
5. 最常见的三类错误是 imageSize 算错、NPOT 块数不对齐、格式不支持——全部可以通过本节的公式和检测逻辑避免。

## 练习题与答案

### 题目 1：为什么"全用 PNG"仍可能导致运行时显存爆炸？

<details>
<summary>查看答案</summary>

PNG 是一种文件压缩格式（基于 DEFLATE 算法），它压缩的是磁盘上的文件体积。但 GPU 不能直接采样 PNG 比特流——上传前必须完全解码为像素数据（通常是 RGBA8，即每像素 4 字节）。因此，即使 PNG 文件只有 200KB，一张 2048×2048 的纹理在显存中仍占 `2048 × 2048 × 4 = 16MB`。如果一屏常驻 20 张这样的纹理，显存占用就可能达到 320MB。

要降低运行时显存和带宽开销，必须使用 GPU 压缩格式（BC / ETC / ASTC / PVRTC），这些格式在显存中也保持压缩状态，GPU 硬件直接采样压缩数据。

</details>

### 题目 2：以下代码有什么错误？如何修复？

```cpp
// 上传 1024×768 的 ASTC 6×6 纹理
int bx = 1024 / 6;   // = 170
int by = 768 / 6;    // = 128
GLsizei imageSize = bx * by * 16;
glCompressedTexImage2D(GL_TEXTURE_2D, 0, 0x93B4,
                       1024, 768, 0, imageSize, data);
```

<details>
<summary>查看答案</summary>

**错误：** 块数计算使用了整数除法（截断），没有向上取整。

- `1024 / 6 = 170`（余 4），正确应为 `ceil(1024/6) = 171`
- `768 / 6 = 128`（整除），这行恰好正确

错误导致少算了 1 列块（128 个块），`imageSize` 偏小，`glCompressedTexImage2D` 会返回 `GL_INVALID_VALUE`。

**修复：**
```cpp
int bx = (1024 + 6 - 1) / 6;  // = 1029 / 6 = 171 ✅
int by = (768  + 6 - 1) / 6;  // = 773  / 6 = 128 ✅（与之前一样）
GLsizei imageSize = bx * by * 16;  // 171 × 128 × 16 = 350208 字节
```

</details>

### 题目 3：压缩纹理上传报 `GL_INVALID_VALUE`，优先排查哪三项？

<details>
<summary>查看答案</summary>

1. **`imageSize` 是否按块计算**——最常见的错误是把 `width × height × 4`（RGBA8 字节数）当作 `imageSize` 传入，正确公式是 `ceil(w/blockW) × ceil(h/blockH) × bytesPerBlock`。

2. **Mip 层尺寸和数据长度是否匹配**——每个 mip 层的 width/height 都是上一层的一半（向下取整，最小为 1），每层的 `imageSize` 必须独立重新计算，不能复用上一层的值。

3. **NPOT 边缘块是否补齐**——对于非对齐尺寸（如 100×100 的纹理用 4×4 块），块数必须向上取整（25×25），而非截断。如果使用了整数除法（`100/4=25`），这里恰好没问题，但 `101/4=25` 就错了（应该是 26）。

</details>
