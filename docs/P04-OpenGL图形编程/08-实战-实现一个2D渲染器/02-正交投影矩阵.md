## 为什么需要投影矩阵

在上一节中，我们的顶点着色器有这样一行代码：

```glsl
gl_Position = uProjection * vec4(aPos, 0.0, 1.0);
```

这里 `aPos` 是屏幕像素坐标（比如 `(400, 300)`），而 GPU 只认识 NDC 坐标（Normalized Device Coordinates，归一化设备坐标，X 和 Y 都在 `[-1, +1]` 范围内）。投影矩阵的作用就是把像素坐标**映射**到 NDC。

如果没有投影矩阵，你需要在 CPU 端手动计算每个顶点的 NDC 坐标：

```cpp
// 手动转换（不推荐，但 KrKr2 就是这么做的）
float ndcX = (pixelX / screenWidth) * 2.0f - 1.0f;   // [0, 800] → [-1, +1]
float ndcY = 1.0f - (pixelY / screenHeight) * 2.0f;   // [0, 600] → [+1, -1]（Y 翻转）
```

有了投影矩阵，顶点着色器可以直接接收像素坐标，矩阵会自动完成转换。这让 CPU 端代码更简洁，也更容易支持窗口缩放。

## 正交投影矩阵的推导

### 目标

我们需要一个 4×4 矩阵，把以下坐标范围映射到 NDC：

| 轴 | 输入范围 | 输出范围 |
|----|---------|---------|
| X | `[left, right]` | `[-1, +1]` |
| Y | `[top, bottom]` | `[-1, +1]` |
| Z | `[near, far]` | `[-1, +1]`（2D 中不关心，固定为 0） |

对于我们的 2D 渲染器（屏幕坐标，左上角原点，Y 轴向下）：
- `left = 0`, `right = screenWidth`
- `top = 0`, `bottom = screenHeight`
- `near = -1`, `far = 1`

### 推导过程

以 X 轴为例。我们需要把 `[0, screenWidth]` 线性映射到 `[-1, +1]`：

```
步骤 1：平移，使范围起点为 0
    x' = x - left

步骤 2：缩放到 [0, 1]
    x'' = x' / (right - left)

步骤 3：缩放到 [0, 2]
    x''' = x'' * 2

步骤 4：平移到 [-1, +1]
    x_ndc = x''' - 1

合并：
    x_ndc = 2 * (x - left) / (right - left) - 1
          = 2x / (right - left) - 2*left / (right - left) - 1
          = 2x / (right - left) - (right + left) / (right - left)
```

Y 轴同理（用 top/bottom 替换 left/right）。

### 矩阵形式

把上面的线性变换写成矩阵乘法的形式：

```
┌                                          ┐   ┌   ┐   ┌                              ┐
│  2/(right-left)    0              0    tx │   │ x │   │ x_ndc                         │
│  0                 2/(top-bottom) 0    ty │ × │ y │ = │ y_ndc                         │
│  0                 0             -2/d  tz │   │ z │   │ z_ndc                         │
│  0                 0              0    1  │   │ 1 │   │ 1                             │
└                                          ┘   └   ┘   └                              ┘

其中：
  tx = -(right + left) / (right - left)
  ty = -(top + bottom) / (top - bottom)
  tz = -(far + near) / (far - near)
  d  = far - near
```

代入我们的 2D 值（left=0, right=W, top=0, bottom=H, near=-1, far=1）：

```
tx = -(W + 0) / (W - 0) = -1
ty = -(0 + H) / (0 - H) = 1
tz = -(1 + (-1)) / (1 - (-1)) = 0

矩阵简化为：
┌              ┐
│  2/W  0  0 -1│
│  0  -2/H 0  1│
│  0    0  -1 0│
│  0    0   0 1│
└              ┘
```

注意 Y 轴系数是**负数** `-2/H`，这正是因为我们选择了"Y 轴向下"的屏幕坐标系——像素 Y 坐标增大时，NDC Y 坐标减小。

## C++ 实现

### 列主序存储约定

在写代码之前，必须理解 OpenGL 的矩阵存储方式。OpenGL 使用**列主序（Column-Major Order）**：矩阵按列存储在一维数组中。

```cpp
// 逻辑矩阵（数学写法，按行阅读）：
//   m00 m01 m02 m03
//   m10 m11 m12 m13
//   m20 m21 m22 m23
//   m30 m31 m32 m33
//
// 列主序存储（OpenGL 约定，按列存储）：
//   out[0]=m00  out[4]=m01  out[8]=m02   out[12]=m03
//   out[1]=m10  out[5]=m11  out[9]=m12   out[13]=m13
//   out[2]=m20  out[6]=m21  out[10]=m22  out[14]=m23
//   out[3]=m30  out[7]=m31  out[11]=m32  out[15]=m33
//
// 公式：out[col * 4 + row] = m[row][col]
//
// 与 C 语言二维数组的行主序（Row-Major）相反！
// C 语言 float m[4][4] 中，m[row][col] 存储在 m[row*4+col]。
// 如果你用 C 二维数组存矩阵，传给 OpenGL 时需要转置（transpose=GL_TRUE）。
// 为避免混乱，推荐直接用一维数组按列主序存储。
```

### 正交投影矩阵生成函数

```cpp
// 生成 2D 正交投影矩阵（列主序，OpenGL 约定）
// 参数：
//   out    — 输出数组（16 个 float，列主序排列）
//   left   — 左边界（通常为 0）
//   right  — 右边界（通常为屏幕宽度）
//   top    — 上边界（通常为 0）
//   bottom — 下边界（通常为屏幕高度）
void createOrthoProjection(float* out,
                           float left, float right,
                           float top, float bottom) {
    // 先清零整个矩阵
    for (int i = 0; i < 16; i++) out[i] = 0.0f;

    // 第一列（X 缩放）
    out[0] = 2.0f / (right - left);          // m[0][0]

    // 第二列（Y 缩放，负号实现 Y 翻转）
    out[5] = 2.0f / (top - bottom);           // m[1][1]

    // 第三列（Z 缩放，2D 中固定为 -1）
    out[10] = -1.0f;                          // m[2][2]

    // 第四列（平移分量）
    out[12] = -(right + left) / (right - left);   // tx
    out[13] = -(top + bottom) / (top - bottom);   // ty
    out[14] = 0.0f;                               // tz
    out[15] = 1.0f;                               // w
}

// 简化版：固定 left=0, top=0（最常用的 2D 场景）
void createOrthoProjection(float* out, float width, float height) {
    createOrthoProjection(out, 0.0f, width, 0.0f, height);
    // 结果：out[0]=2/W, out[5]=-2/H, out[10]=-1
    //       out[12]=-1, out[13]=1, out[15]=1
}
```

### 验证：代入测试点

验证矩阵是否正确，最可靠的方法是代入已知的点，检查输出是否符合预期：

```cpp
#include <cstdio>

// 4×4 矩阵（列主序）× 4D 向量
void mat4MulVec4(const float* m, const float* v, float* out) {
    for (int row = 0; row < 4; row++) {
        out[row] = 0.0f;
        for (int col = 0; col < 4; col++) {
            out[row] += m[col * 4 + row] * v[col];  // 列主序
        }
    }
}

int main() {
    float proj[16];
    createOrthoProjection(proj, 800.0f, 600.0f);

    // 测试 4 个关键点
    struct TestCase { float x, y; float expectX, expectY; const char* name; };
    TestCase tests[] = {
        {0.0f,   0.0f,   -1.0f,  1.0f,  "左上角"},
        {800.0f, 600.0f,  1.0f, -1.0f,  "右下角"},
        {400.0f, 300.0f,  0.0f,  0.0f,  "中心"},
        {200.0f, 150.0f, -0.5f,  0.5f,  "1/4 处"},
    };

    for (auto& t : tests) {
        float in[4] = {t.x, t.y, 0.0f, 1.0f};
        float out[4];
        mat4MulVec4(proj, in, out);
        printf("%-6s (%3.0f,%3.0f) → (%+.2f, %+.2f)  期望 (%+.2f, %+.2f) %s\n",
               t.name, t.x, t.y, out[0], out[1], t.expectX, t.expectY,
               (out[0] == t.expectX && out[1] == t.expectY) ? "✓" : "✗");
    }
    return 0;
}
/* 输出：
左上角  (  0,  0) → (-1.00, +1.00)  期望 (-1.00, +1.00) ✓
右下角  (800,600) → (+1.00, -1.00)  期望 (+1.00, -1.00) ✓
中心    (400,300) → (+0.00, +0.00)  期望 (+0.00, +0.00) ✓
1/4 处  (200,150) → (-0.50, +0.50)  期望 (-0.50, +0.50) ✓
*/
```

## KrKr2 的做法：不用投影矩阵

KrKr2 的顶点着色器（`RenderManager_ogl.cpp` 第 1999 行）直接写的是：

```glsl
gl_Position = vec4(a_position, 0.0, 1.0);
```

没有乘任何投影矩阵！这意味着 `a_position` 必须**已经是 NDC 坐标**。

KrKr2 在 CPU 端完成了坐标转换。绘制前，引擎把像素坐标转换为 `[-1, +1]` 范围：

```cpp
// KrKr2 的 CPU 端坐标转换（ApplyVertex 函数，概念简化版）
void applyVertex(GLVertexInfo& vtx, const tTVPRect& rect) {
    float l = (float)rect.left   / viewW * 2.0f - 1.0f;  // [0,W] → [-1,+1]
    float r = (float)rect.right  / viewW * 2.0f - 1.0f;
    float t = 1.0f - (float)rect.top    / viewH * 2.0f;  // [0,H] → [+1,-1]
    float b = 1.0f - (float)rect.bottom / viewH * 2.0f;

    // 生成两个三角形的 6 个顶点（无索引缓冲）
    vtx.push(l, t);  vtx.push(l, b);  vtx.push(r, t);
    vtx.push(l, b);  vtx.push(r, t);  vtx.push(r, b);
}
```

在纹理恢复（`RestoreNormalSize`，第 2328-2333 行）时，KrKr2 直接用硬编码的全屏 NDC 四边形：

```cpp
// KrKr2 源码：全屏四边形（NDC 坐标）
GLfloat minx = -1, maxx = 1, miny = -1, maxy = 1;
static const GLfloat vertices[12] = {
    minx, miny, maxx, miny, minx, maxy,  // 三角形 1
    maxx, miny, minx, maxy, maxx, maxy   // 三角形 2
};
// 配合 glViewport(0, 0, w, h) 控制实际输出区域
```

### 两种方式的对比

| 特性 | 投影矩阵方式（教学版） | CPU 转换方式（KrKr2） |
|------|----------------------|---------------------|
| 坐标转换位置 | GPU（顶点着色器中矩阵乘法） | CPU（每帧手动计算） |
| 顶点着色器复杂度 | 矩阵乘法（1 次 `mat4 × vec4`） | 直接赋值（无计算） |
| 窗口缩放处理 | 只需更新投影矩阵（16 个 float） | 需要更新所有顶点的 NDC |
| 精灵变换 | 可用模型矩阵，GPU 并行计算 | 必须在 CPU 逐顶点计算 |
| 适用场景 | 大量精灵、频繁变换 | 少量矩形、变换简单 |

KrKr2 选择 CPU 转换是因为：视觉小说引擎每帧矩形数量有限（< 50），且不需要复杂变换，CPU 直接算更简单。

## 常见错误及解决方案

### 错误 1：画面显示上下颠倒

**现象**：精灵的 Y 坐标反了——顶部内容出现在底部。

**原因**：投影矩阵的 top/bottom 参数传反了。

```cpp
// 错误：top=H, bottom=0 → Y 轴向上（数学坐标系）
createOrthoProjection(proj, 0.0f, 800.0f, 600.0f, 0.0f);

// 正确：top=0, bottom=H → Y 轴向下（屏幕坐标系）
createOrthoProjection(proj, 0.0f, 800.0f, 0.0f, 600.0f);
```

### 错误 2：窗口缩放后精灵变形

**现象**：拖拽窗口改变大小后，圆形变成椭圆。

**原因**：窗口大小改变后没有更新投影矩阵和 `glViewport`。

```cpp
// 在窗口大小回调中更新：
void framebufferSizeCallback(GLFWwindow* w, int width, int height) {
    // ========== glViewport ==========
    // 设置渲染区域（视口），告诉 OpenGL 将 NDC 映射到窗口的哪个区域
    // 参数1,2: 视口左下角坐标（像素）
    // 参数3,4: 视口宽高（像素）
    // 通常设置为整个窗口大小
    glViewport(0, 0, width, height);
    
    // 重新生成投影矩阵（用新的窗口尺寸）
    createOrthoProjection(projection, (float)width, (float)height);
    glUseProgram(shaderProgram);
    glUniformMatrix4fv(projLoc, 1, GL_FALSE, projection);
}

// 注册回调（在初始化窗口后）：
glfwSetFramebufferSizeCallback(window, framebufferSizeCallback);
```

### 错误 3：列主序和行主序搞混

**现象**：矩阵运算结果完全不对，但公式本身是正确的。

**原因**：OpenGL 使用列主序，而代码按行主序存储矩阵。

**解决方案**：

```cpp
// 方案 1（推荐）：直接用列主序一维数组
float proj[16];
createOrthoProjection(proj, w, h);
glUniformMatrix4fv(loc, 1, GL_FALSE, proj);  // GL_FALSE = 不转置

// 方案 2：用行主序存储，传入时让 OpenGL 转置
float projRowMajor[4][4] = {
    {2.0f/w, 0,       0,  -1},
    {0,      -2.0f/h, 0,   1},
    {0,      0,       -1,  0},
    {0,      0,       0,   1},
};
glUniformMatrix4fv(loc, 1, GL_TRUE, &projRowMajor[0][0]);  // GL_TRUE = 转置
// 注意：OpenGL ES 2.0 不支持 GL_TRUE！必须用列主序。
```

## 本节小结

- 正交投影矩阵将像素坐标线性映射到 NDC `[-1, +1]` 范围
- 2D 屏幕坐标系（Y 向下）的投影矩阵 Y 分量为负数 `-2/H`
- OpenGL 使用列主序存储矩阵，访问公式：`out[col * 4 + row]`
- `glUniformMatrix4fv` 的 transpose 参数控制输入数据的存储方式（GL_FALSE=列主序）
- KrKr2 不使用投影矩阵，在 CPU 端直接计算 NDC 坐标，适合少量矩形的场景
- 窗口缩放时必须同时更新 `glViewport` 和投影矩阵
- 验证矩阵正确性的最佳方法：代入已知点检查输出

### 题目 2：手写矩阵乘向量函数

不使用任何数学库，手写一个 `mat4MulVec4` 函数（列主序 4×4 矩阵 × 4D 向量），然后用正交投影矩阵和 3 个测试点验证结果。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdio>

void mat4MulVec4(const float* m, const float* v, float* out) {
    // 列主序矩阵：m[col*4 + row]
    // result[row] = sum(m[col*4+row] * v[col], col=0..3)
    for (int row = 0; row < 4; row++) {
        out[row] = 0.0f;
        for (int col = 0; col < 4; col++) {
            out[row] += m[col * 4 + row] * v[col];
        }
    }
}

int main() {
    float proj[16];
    createOrthoProjection(proj, 1024.0f, 768.0f);

    float tests[][2] = {{0, 0}, {1024, 768}, {512, 384}};
    float expects[][2] = {{-1, 1}, {1, -1}, {0, 0}};
    const char* names[] = {"左上角", "右下角", "中心"};

    for (int i = 0; i < 3; i++) {
        float in[4] = {tests[i][0], tests[i][1], 0, 1};
        float out[4];
        mat4MulVec4(proj, in, out);
        printf("%s: (%.0f, %.0f) → (%.2f, %.2f)  期望 (%.2f, %.2f)\n",
               names[i], tests[i][0], tests[i][1],
               out[0], out[1], expects[i][0], expects[i][1]);
    }
    return 0;
}
```

关键理解：列主序中 `m[col*4+row]` 对应数学矩阵的第 row 行第 col 列。乘法时，结果的每一行是矩阵该行与输入向量的点积。

<summary>查看答案</summary>

(1) ES 2.0 的目标是嵌入式/移动设备，规范设计者为了简化 GPU 驱动实现，去掉了矩阵转置功能。驱动只需支持一种内存布局（列主序），不需要在上传时做额外的转置操作。这减少了驱动代码量和潜在的 bug。

(2) 在 CPU 端手动转置矩阵后再上传：

```cpp
// 将行主序矩阵转为列主序
void transposeMatrix(float* out, const float* in) {
    for (int row = 0; row < 4; row++) {
        for (int col = 0; col < 4; col++) {
            out[col * 4 + row] = in[row * 4 + col];
        }
    }
}

// 使用：
float rowMajor[16] = { /* 行主序数据 */ };
float colMajor[16];
transposeMatrix(colMajor, rowMajor);
glUniformMatrix4fv(loc, 1, GL_FALSE, colMajor);  // ES 2.0 安全
```

或者更好的做法：**从一开始就用列主序存储矩阵**，避免转置开销。这也是为什么本教程的矩阵函数都直接输出列主序格式。
