## 什么是仿射变换

**仿射变换（Affine Transform）** 是一类保持"直线仍然是直线、平行线仍然平行"的几何变换。在 2D 渲染中，常用的仿射变换有三种：

| 变换类型 | 效果 | 应用场景 |
|---------|------|---------|
| 平移（Translation） | 移动位置 | 精灵在屏幕上移动 |
| 缩放（Scaling） | 放大或缩小 | 精灵大小变化、镜像翻转 |
| 旋转（Rotation） | 绕某点旋转 | 旋转动画、转盘 UI |

还有一种不太常用的：**错切（Shear）**，将矩形拉成平行四边形（想象把一叠扑克牌推歪）。本节不做详细讲解。

这些变换都可以用矩阵乘法统一表示。把变换写成矩阵的好处是：**多个变换可以通过矩阵相乘合并成一个矩阵**，然后只做一次矩阵×向量运算即可。

## 三种基本变换矩阵

### 辅助函数：单位矩阵

```cpp
// 生成 4×4 单位矩阵（列主序）
// 单位矩阵（Identity Matrix）是矩阵乘法的"1"——任何矩阵乘以单位矩阵等于自身
// 对角线全为 1，其余全为 0
void identity4x4(float* m) {
    for (int i = 0; i < 16; i++) m[i] = 0.0f;
    m[0] = m[5] = m[10] = m[15] = 1.0f;
}
```

### 平移矩阵

将精灵沿 X 和 Y 方向移动 `(tx, ty)` 个像素：

```
数学形式（3×3 简化）：
┌          ┐   ┌   ┐   ┌        ┐
│ 1  0  tx │   │ x │   │ x + tx │
│ 0  1  ty │ × │ y │ = │ y + ty │
│ 0  0  1  │   │ 1 │   │ 1      │
└          ┘   └   ┘   └        ┘
```

```cpp
// 生成 4×4 平移矩阵（列主序）
// tx, ty: 平移量（像素单位）
void makeTranslation(float* out, float tx, float ty) {
    identity4x4(out);
    // 列主序中，平移分量在第四列：out[12]=tx, out[13]=ty
    out[12] = tx;
    out[13] = ty;
}

// 验证：
// 点 (100, 200) 经过 T(50, -30) 后应该变成 (150, 170)
// makeTranslation(m, 50, -30);
// mat4MulVec4(m, {100, 200, 0, 1}, result);
// result = {150, 170, 0, 1} ✓
```

### 缩放矩阵

将精灵的大小乘以缩放因子 `(sx, sy)`：

```
数学形式：
┌          ┐   ┌   ┐   ┌      ┐
│ sx 0   0 │   │ x │   │ x*sx │
│ 0  sy  0 │ × │ y │ = │ y*sy │
│ 0  0   1 │   │ 1 │   │ 1    │
└          ┘   └   ┘   └      ┘
```

```cpp
// 生成 4×4 缩放矩阵（列主序）
// sx, sy: 缩放因子（1.0 = 原始大小，2.0 = 两倍大，0.5 = 一半大）
// 负数可以实现镜像翻转（sx=-1 水平翻转，sy=-1 垂直翻转）
void makeScale(float* out, float sx, float sy) {
    identity4x4(out);
    out[0] = sx;   // X 缩放
    out[5] = sy;   // Y 缩放
}

// 常用场景：
// makeScale(m, -1, 1)  → 水平镜像翻转（角色面朝左↔右）
// makeScale(m, 1, -1)  → 垂直翻转
// makeScale(m, 2, 2)   → 放大 2 倍
// makeScale(m, 0.5, 0.5) → 缩小到一半
```

> **注意**：缩放是以坐标原点为中心的。如果直接对一个位于 `(100, 200)` 的精灵做 2 倍缩放，它会跑到 `(200, 400)`！要保持位置不变地缩放，需要先平移到原点、缩放、再平移回去（类似下面旋转的做法）。

### 旋转矩阵

绕坐标原点旋转 θ 角（弧度）：

```
数学形式：
┌                ┐   ┌   ┐   ┌                    ┐
│ cosθ  -sinθ  0 │   │ x │   │ x·cosθ - y·sinθ   │
│ sinθ   cosθ  0 │ × │ y │ = │ x·sinθ + y·cosθ   │
│ 0      0     1 │   │ 1 │   │ 1                  │
└                ┘   └   ┘   └                    ┘
```

```cpp
#include <cmath>

// 生成 4×4 旋转矩阵（列主序，绕原点逆时针旋转）
// angleRadians: 旋转角度，单位为弧度
// 1 弧度 ≈ 57.3°，π 弧度 = 180°
// 角度转弧度：radians = degrees * π / 180
void makeRotation(float* out, float angleRadians) {
    identity4x4(out);
    float c = cosf(angleRadians);
    float s = sinf(angleRadians);
    // 列主序：
    // out[0] = cos, out[1] = sin    （第一列）
    // out[4] = -sin, out[5] = cos   （第二列）
    out[0] = c;
    out[1] = s;
    out[4] = -s;
    out[5] = c;
}
```

> **为什么 `out[1] = sin` 而不是 `-sin`？** 因为列主序的存储方式：`out[1]` 对应矩阵第一列第二行，在数学矩阵中是 `m[1][0] = sinθ`。如果你不确定，代入一个 90° 旋转 `(cos90°=0, sin90°=1)` 验证：点 `(1,0)` 应该变成 `(0,1)`。

## 绕精灵中心旋转

直接使用旋转矩阵会绕坐标原点（屏幕左上角）旋转，这通常不是我们想要的。要绕精灵自身的中心旋转，需要三步组合变换：

```
1. 平移精灵中心到原点  → T(-cx, -cy)
2. 旋转                → R(θ)
3. 平移回原来的位置    → T(cx, cy)

最终矩阵 = T(cx, cy) × R(θ) × T(-cx, -cy)
```

```
变换过程示意（绕中心旋转 45°）：

步骤 1: T(-cx,-cy)     步骤 2: R(45°)      步骤 3: T(cx,cy)
   ┌──┐                    /\                   /\
   │  │  → 移到原点 →     /  \  → 旋转 →      /  \  → 移回来
   └──┘                  /    \              /    \
 (100,200)              (0,0)处            (100,200)处
```

### C++ 实现

```cpp
// 4×4 矩阵乘法（列主序）
// 结果 = a × b
void mat4Multiply(float* out, const float* a, const float* b) {
    float tmp[16];  // 临时数组，允许 out 与 a 或 b 相同
    for (int col = 0; col < 4; col++) {
        for (int row = 0; row < 4; row++) {
            float sum = 0.0f;
            for (int k = 0; k < 4; k++) {
                sum += a[k * 4 + row] * b[col * 4 + k];
            }
            tmp[col * 4 + row] = sum;
        }
    }
    for (int i = 0; i < 16; i++) out[i] = tmp[i];
}

// 绕指定中心点旋转
void rotateAroundCenter(float* out, float cx, float cy, float angle) {
    float t1[16], r[16], t2[16], temp[16];

    makeTranslation(t1, cx, cy);    // 步骤 3：移回来
    makeRotation(r, angle);         // 步骤 2：旋转
    makeTranslation(t2, -cx, -cy);  // 步骤 1：移到原点

    mat4Multiply(temp, t1, r);      // temp = T(cx,cy) × R(θ)
    mat4Multiply(out, temp, t2);    // out = temp × T(-cx,-cy)
}
```

也可以直接计算合并后的矩阵元素（更高效，省去两次矩阵乘法）：

```cpp
// 优化版：直接计算合并矩阵
void rotateAroundCenterFast(float* out, float cx, float cy, float angle) {
    float c = cosf(angle);
    float s = sinf(angle);

    for (int i = 0; i < 16; i++) out[i] = 0.0f;
    out[0]  = c;
    out[1]  = s;
    out[4]  = -s;
    out[5]  = c;
    out[10] = 1.0f;
    out[12] = cx - cx * c + cy * s;  // 由 T × R × T⁻¹ 展开得到
    out[13] = cy - cx * s - cy * c;
    out[15] = 1.0f;
}
```

## 增强版顶点着色器：支持模型矩阵

在上一节的顶点着色器基础上，添加 `uModel`（模型矩阵）uniform：

```glsl
#version 330 core

layout(location = 0) in vec2 aPos;
layout(location = 1) in vec2 aTexCoord;
layout(location = 2) in vec4 aColor;

uniform mat4 uProjection;  // 投影矩阵（像素坐标 → NDC）
uniform mat4 uModel;       // 模型矩阵（精灵变换：平移/旋转/缩放）

out vec2 vTexCoord;
out vec4 vColor;

void main() {
    // 矩阵乘法顺序从右到左：
    //   1. vec4(aPos, 0.0, 1.0) — 精灵的局部像素坐标
    //   2. uModel × ... — 对精灵做变换（在像素坐标空间中）
    //   3. uProjection × ... — 转换到 NDC
    gl_Position = uProjection * uModel * vec4(aPos, 0.0, 1.0);
    vTexCoord = aTexCoord;
    vColor = aColor;
}
```

### CPU 端使用方法

```cpp
// 初始化时获取 uniform 位置
GLint projLoc = glGetUniformLocation(program, "uProjection");
GLint modelLoc = glGetUniformLocation(program, "uModel");

// 设置投影矩阵（只需在初始化和窗口缩放时更新）
float projection[16];
createOrthoProjection(projection, 800.0f, 600.0f);
glUniformMatrix4fv(projLoc, 1, GL_FALSE, projection);

// --- 渲染循环中 ---

// 绘制旋转的角色（每帧更新模型矩阵）
float angle = (float)glfwGetTime() * 0.5f;  // 每秒旋转 0.5 弧度
float model[16];
// 角色位于 (400, 100)，大小 256×512，绕中心旋转
rotateAroundCenter(model, 400.0f + 128.0f, 100.0f + 256.0f, angle);
glUniformMatrix4fv(modelLoc, 1, GL_FALSE, model);
// ... 绘制角色精灵 ...

// 绘制不需要变换的精灵（背景、UI 等）
float identity[16];
identity4x4(identity);
glUniformMatrix4fv(modelLoc, 1, GL_FALSE, identity);
// ... 绘制背景 ...
```

## 矩阵乘法顺序很重要

矩阵乘法**不满足交换律**（A × B ≠ B × A）。变换的应用顺序是从**右到左**读。

```cpp
// 场景：先缩放 2 倍，再旋转 45°
float s[16], r[16], result[16];
makeScale(s, 2.0f, 2.0f);
makeRotation(r, 3.14159f / 4.0f);  // 45°

// 正确：result = R × S（先缩放后旋转）
mat4Multiply(result, r, s);

// 如果写成 result = S × R（先旋转后缩放），结果不同！
// 当 sx ≠ sy 时，先旋转后缩放会导致图形变形（圆变椭圆）
```

| 变换组合 | 矩阵写法 | 效果 |
|---------|---------|------|
| 先平移后旋转 | `R × T` | 精灵绕原点画圆弧 |
| 先旋转后平移 | `T × R` | 精灵原地旋转后平移 |
| 绕中心旋转 | `T(c) × R × T(-c)` | 精灵绕自身中心旋转 |
| 先缩放后旋转 | `R × S` | 缩放后旋转（保持比例） |
| 先旋转后缩放 | `S × R` | 旋转后缩放（可能变形） |

## 动手实践

### 实现精灵动画系统

结合本节和上一节的知识，实现一个精灵在屏幕上同时做三种变换的完整示例：

```cpp
// 完整的精灵变换示例
#include <cstdio>
#include <cmath>

// （identity4x4, makeTranslation, makeScale, makeRotation,
//   mat4Multiply, rotateAroundCenter 函数定义见上文）

// 生成组合变换：平移 + 绕中心旋转 + 缩放
void makeSpriteTransform(float* out,
                         float x, float y, float w, float h,
                         float scaleX, float scaleY,
                         float rotationDeg) {
    float cx = x + w * 0.5f;  // 精灵中心
    float cy = y + h * 0.5f;
    float angleRad = rotationDeg * 3.14159f / 180.0f;

    // 组合：T(cx,cy) × R(angle) × S(sx,sy) × T(-cx,-cy)
    float c = cosf(angleRad);
    float s = sinf(angleRad);

    for (int i = 0; i < 16; i++) out[i] = 0.0f;
    out[0]  = c * scaleX;
    out[1]  = s * scaleX;
    out[4]  = -s * scaleY;
    out[5]  = c * scaleY;
    out[10] = 1.0f;
    out[12] = cx - cx * c * scaleX + cy * s * scaleY;
    out[13] = cy - cx * s * scaleX - cy * c * scaleY;
    out[15] = 1.0f;
}

// 在渲染循环中使用：
// float time = (float)glfwGetTime();
// float model[16];
//
// // 角色：上下浮动 + 缓慢旋转
// float bobY = sinf(time * 2.0f) * 10.0f;  // 上下浮动 10 像素
// makeSpriteTransform(model,
//     300.0f, 150.0f + bobY, 200.0f, 300.0f,  // 位置和大小
//     1.0f, 1.0f,                               // 不缩放
//     sinf(time) * 5.0f);                       // 左右摆动 ±5°
// glUniformMatrix4fv(modelLoc, 1, GL_FALSE, model);
// batcher.drawSprite(charTex, 300, 150, 200, 300, ...);
```

### KrKr2 的变换处理

KrKr2 的精灵变换方式与我们的教学版本不同。由于 KrKr2 不使用模型矩阵（上一节已解释原因），它的变换完全在 CPU 端完成：

```cpp
// KrKr2 的做法（概念简化版）：
// 不用矩阵，直接修改顶点坐标
void drawSprite(float x, float y, float w, float h) {
    // 将像素坐标直接转为 NDC
    float l = x / viewW * 2.0f - 1.0f;
    float r = (x + w) / viewW * 2.0f - 1.0f;
    float t = 1.0f - y / viewH * 2.0f;
    float b = 1.0f - (y + h) / viewH * 2.0f;
    // 生成 6 个顶点...
}
```

这种方式不支持旋转——KrKr2 的视觉小说场景中确实不需要旋转精灵（角色立绘、背景、文本框都是轴对齐矩形）。如果需要旋转效果（如转盘 UI），KrKr2 会使用预渲染的旋转帧序列而非实时变换。

相关文件：
- `cpp/core/visual/ogl/RenderManager_ogl.cpp` 第 2345-2353 行 — `glVertexAttribPointer` + `glDrawArrays` 直接绘制

### 与教学版的对比总结

| 能力 | 教学版（矩阵方式） | KrKr2（CPU 直接计算） |
|------|-------------------|---------------------|
| 平移 | 模型矩阵 | CPU 修改坐标 |
| 缩放 | 模型矩阵 | CPU 修改宽高 |
| 旋转 | 模型矩阵 | 不支持（用帧动画替代） |
| 组合变换 | 矩阵乘法 | 逐顶点手算 |
| GPU 利用率 | 高（GPU 并行变换） | 低（CPU 逐顶点） |

## 常见错误及解决方案

### 错误 1：旋转后精灵飞出屏幕

**现象**：给精灵添加旋转后，它绕屏幕左上角画大圆。

**原因**：使用了绕原点的旋转矩阵 `makeRotation`，没有先平移到精灵中心。

**解决**：使用 `rotateAroundCenter` 函数，传入精灵中心坐标 `(x + w/2, y + h/2)`。

### 错误 2：缩放后精灵位置偏移

**现象**：精灵放大后，位置向右下方偏移。

**原因**：缩放是以原点为中心的。位于 `(100, 200)` 的精灵做 2 倍缩放后会到 `(200, 400)`。

**解决**：使用绕精灵中心的缩放变换：`T(cx,cy) × S(sx,sy) × T(-cx,-cy)`。或者使用 `makeSpriteTransform` 函数，它已经内置了这个处理。

### 错误 3：变换顺序导致意外结果

**现象**：同样的平移和旋转参数，结果与预期不同。

**原因**：矩阵乘法的顺序搞反了。`R × T` 和 `T × R` 的结果完全不同。

**记忆法**：**从右到左读变换顺序**。`Projection × Model × Vertex` 表示先对顶点做 Model 变换，再做 Projection 变换。

## 本节小结

- 2D 仿射变换包括平移、缩放、旋转，都可以用 4×4 矩阵表示
- 平移矩阵的分量在第四列 `out[12], out[13]`
- 旋转矩阵由 `cos/sin` 组成，注意列主序中的存储位置
- 绕精灵中心旋转 = `T(center) × R(θ) × T(-center)`
- 模型矩阵通过 `uniform mat4 uModel` 传入着色器，实现精灵变换
- 矩阵乘法不满足交换律，变换顺序从右到左读
- KrKr2 不使用变换矩阵，所有变换在 CPU 端逐顶点计算

### 题目 1：实现水平镜像翻转

编写代码，使一个位于 `(200, 100)`、大小 `128×256` 的角色精灵水平翻转（面朝左变面朝右）。提示：水平翻转 = `scaleX = -1`，但需要绕精灵中心做。

<details>
<summary>查看答案</summary>

```cpp
// 绕精灵中心水平翻转
float model[16];
float cx = 200.0f + 64.0f;   // 精灵中心 X = x + w/2
float cy = 100.0f + 128.0f;  // 精灵中心 Y = y + h/2

// 方法 1：使用 makeSpriteTransform
makeSpriteTransform(model,
    200.0f, 100.0f, 128.0f, 256.0f,  // 位置和大小
    -1.0f, 1.0f,                      // sx=-1 水平翻转
    0.0f);                            // 不旋转
glUniformMatrix4fv(modelLoc, 1, GL_FALSE, model);

// 方法 2：手动组合矩阵
// T(cx,cy) × S(-1,1) × T(-cx,-cy)
// 展开：
//   out[0] = -1, out[5] = 1
//   out[12] = cx - cx*(-1) = 2*cx = 528
//   out[13] = cy - cy*(1) = 0
```

### 题目 2：组合变换的计算

给定一个位于 `(0, 0)` 的 `100×100` 精灵，先缩放 2 倍（绕中心），再绕中心旋转 90°。请手动计算精灵四个角的最终坐标。

<details>
<summary>查看答案</summary>

```
精灵中心 = (50, 50)

步骤 1：绕中心缩放 2 倍
  T(50,50) × S(2,2) × T(-50,-50) 作用于四个角：
  (0,0)     → (-50,-50) → (-100,-100) → (-50,-50)    // 左上
  (100,0)   → (50,-50)  → (100,-100)  → (150,-50)    // 右上
  (100,100) → (50,50)   → (100,100)   → (150,150)    // 右下
  (0,100)   → (-50,50)  → (-100,100)  → (-50,150)    // 左下

步骤 2：绕中心旋转 90°（cos90°=0, sin90°=1）
  T(50,50) × R(90°) × T(-50,-50) 作用于步骤 1 的结果：
  (-50,-50)  → (-100,-100) → (100,-100) → (150,-50)  → 旋转后 (150,-50)
  
  实际上直接用组合矩阵计算更清晰：
  最终矩阵 = T(50,50) × R(90°) × S(2,2) × T(-50,-50)
```

### 题目 3：实现弹性缩放动画

编写代码，实现精灵的"弹性缩放"效果：精灵从 0 倍大小放大到 1.2 倍，再回弹到 1.0 倍。使用 `sin` 函数实现衰减振荡。

<details>
<summary>查看答案</summary>

```cpp
// 弹性缩放动画
// startTime: 动画开始时间
// duration: 动画总时长（秒）
float elasticScale(float time, float startTime, float duration) {
    float t = (time - startTime) / duration;  // 归一化到 [0, 1]
    if (t < 0.0f) return 0.0f;  // 动画未开始
    if (t > 1.0f) return 1.0f;  // 动画结束，保持 1.0

    // 衰减振荡公式：
    // scale = 1 + A * sin(ω*t) * exp(-decay*t)
    // A = 0.3 (最大过冲 30%)
    // ω = 3π (振荡 1.5 次)
    // decay = 5 (衰减速度)
    float A = 0.3f;
    float omega = 3.0f * 3.14159f;
    float decay = 5.0f;
    return 1.0f + A * sinf(omega * t) * expf(-decay * t);
}

// 在渲染循环中使用：
float time = (float)glfwGetTime();
float s = elasticScale(time, popupStartTime, 0.5f);
float model[16];
makeSpriteTransform(model,
    spriteX, spriteY, spriteW, spriteH,
    s, s,    // 均匀缩放
    0.0f);   // 不旋转
glUniformMatrix4fv(modelLoc, 1, GL_FALSE, model);

// 效果：精灵在 0.5 秒内从无到有弹出，带有弹性感
```
