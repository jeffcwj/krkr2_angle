## stb_image 简介

**stb_image** 是一个单头文件（single-header）的 C 语言图片解码库，只需一个 `.h` 文件即可解码 PNG、JPEG、BMP、GIF、TGA、HDR 等常见格式。它是 OpenGL 教程和独立游戏中最常用的图片加载方案，因为它零依赖、零配置、跨平台。

### 为什么选择 stb_image

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| stb_image | 零依赖、单文件集成、跨平台 | 不支持 PSD/WebP 等特殊格式 | 教程、小型项目、原型开发 |
| libpng + libjpeg | 功能完整、工业级质量 | 需要链接多个库，配置复杂 | 大型项目、需要高级 PNG 特性 |
| SDL_image | 格式丰富、SDL 生态集成 | 依赖 SDL 库 | 已使用 SDL 的项目 |
| KrKr2 内置加载器 | 支持 TLG 等专有格式 | 与引擎强耦合 | KrKr2 引擎内部 |

### stb_image 集成方式

从 [GitHub](https://github.com/nothings/stb) 下载 `stb_image.h`，放到项目的 `third_party/` 目录。在 **恰好一个** `.cpp` 文件中定义实现宏：

```cpp
// texture.cpp — 纹理加载实现文件
// 必须在 #include 之前定义此宏，否则只声明函数而不生成实现代码
#define STB_IMAGE_IMPLEMENTATION
#include "stb_image.h"
```

> ⚠️ **关键规则**：`STB_IMAGE_IMPLEMENTATION` 只能在**整个项目中定义一次**。如果在多个 `.cpp` 文件中都定义，链接器会报"重复符号"错误。其他需要使用 stb_image 的文件只写 `#include "stb_image.h"`（不加宏）。

stb_image 的核心 API 只有三个函数：

```cpp
// stbi_load — 加载图片文件，返回像素数据指针
// 参数：
//   filename  — 图片文件路径（支持 PNG/JPG/BMP/TGA/GIF/HDR）
//   x         — [输出] 图片宽度（像素）
//   y         — [输出] 图片高度（像素）
//   channels  — [输出] 实际通道数（1=灰度, 2=灰度+Alpha, 3=RGB, 4=RGBA）
//   desired   — 期望通道数（0=保持原样, 4=强制转为 RGBA）
// 返回值：成功返回像素数据指针，失败返回 NULL
unsigned char* stbi_load(const char* filename,
                         int* x, int* y, int* channels,
                         int desired_channels);

// stbi_image_free — 释放 stbi_load 返回的像素数据
// 必须用此函数释放，不能用 free() 或 delete[]
void stbi_image_free(void* data);

// stbi_set_flip_vertically_on_load — 设置是否垂直翻转
// OpenGL 纹理坐标原点在左下角，而大多数图片格式原点在左上角
// 设为 true 可自动翻转，省去手动处理的麻烦
void stbi_set_flip_vertically_on_load(int flag);
```

## 纹理加载函数实现

现在我们把 stb_image 和 OpenGL 纹理创建结合起来，编写一个完整的纹理加载函数。每一步都对应一个 GL 函数调用，我们逐一讲解。

### 完整的 loadTexture 函数

```cpp
#include <glad/glad.h>
#include <iostream>
#include "stb_image.h"

// loadTexture — 从文件加载图片并创建 OpenGL 纹理
// 参数：path — 图片文件路径
// 返回值：成功返回纹理 ID（>0），失败返回 0
GLuint loadTexture(const char* path) {
    // ---- 第一步：用 stb_image 解码图片文件 ----
    int width, height, channels;
    // 翻转 Y 轴：OpenGL 纹理坐标 (0,0) 在左下角
    // 而 PNG/JPG 等格式的 (0,0) 在左上角
    stbi_set_flip_vertically_on_load(true);
    unsigned char* data = stbi_load(path, &width, &height, &channels, 0);
    if (!data) {
        std::cerr << "纹理加载失败: " << path
                  << " 原因: " << stbi_failure_reason() << std::endl;
        return 0;
    }

    // ---- 第二步：根据通道数确定 OpenGL 像素格式 ----
    GLenum internalFormat, pixelFormat;
    // internalFormat — GPU 内部存储格式
    // pixelFormat   — CPU 端像素数据的排列格式
    if (channels == 4) {
        internalFormat = GL_RGBA;  // 4 通道：红绿蓝+透明度
        pixelFormat = GL_RGBA;
    } else if (channels == 3) {
        internalFormat = GL_RGB;   // 3 通道：红绿蓝，无透明度
        pixelFormat = GL_RGB;
    } else if (channels == 1) {
        internalFormat = GL_RED;   // 1 通道：灰度图
        pixelFormat = GL_RED;
    } else {
        std::cerr << "不支持的通道数: " << channels << std::endl;
        stbi_image_free(data);
        return 0;
    }

    // ---- 第三步：设置像素对齐 ----
    // glPixelStorei(GL_UNPACK_ALIGNMENT, n)
    //   告诉 OpenGL 从 CPU 读取像素时，每行数据的起始地址对齐到 n 字节边界
    //   参数 pname: GL_UNPACK_ALIGNMENT
    //   参数 param: 对齐值，只能是 1, 2, 4, 8 之一
    //   默认值是 4，即每行起始地址必须是 4 的倍数
    //   问题：RGB 图片每像素 3 字节，一行 800 像素 = 2400 字节
    //         2400 是 4 的倍数没问题；但 801 像素 = 2403 字节，不是 4 的倍数
    //         OpenGL 会多读 1 字节填充，导致颜色错位（画面斜纹）
    //   解决：对非 4 字节对齐的格式，设为 1（逐字节对齐，无填充）
    if (channels == 3 || channels == 1) {
        glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
    }
    // RGBA（4 字节/像素）天然满足 4 字节对齐，无需修改

    // ---- 第四步：创建 GL 纹理对象 ----
    GLuint texture;
    // glGenTextures(n, textures)
    //   在 GPU 上分配 n 个纹理对象名称（ID）
    //   参数 n:        要生成的纹理数量
    //   参数 textures: 存放生成的纹理 ID 的数组
    //   注意：此时 GPU 并未分配显存，只是分配了一个"名字"
    //         真正的显存分配在 glTexImage2D 调用时发生
    glGenTextures(1, &texture);

    // glBindTexture(target, texture)
    //   将纹理对象绑定到指定的纹理目标
    //   参数 target:  GL_TEXTURE_2D（2D 纹理）、GL_TEXTURE_CUBE_MAP（立方体贴图）等
    //   参数 texture: 纹理 ID（0 表示解绑）
    //   后续所有对 GL_TEXTURE_2D 的操作都作用于此纹理
    glBindTexture(GL_TEXTURE_2D, texture);

    // ---- 第五步：设置纹理参数 ----
    // glTexParameteri(target, pname, param)
    //   设置纹理对象的采样参数
    //   参数 target: GL_TEXTURE_2D
    //   参数 pname:  要设置的参数名
    //   参数 param:  参数值

    // GL_TEXTURE_WRAP_S / GL_TEXTURE_WRAP_T — 纹理坐标超出 [0,1] 范围时的处理方式
    //   GL_REPEAT          — 重复平铺（默认值）
    //   GL_CLAMP_TO_EDGE   — 拉伸边缘像素（2D UI 推荐，避免边缘出现接缝线）
    //   GL_MIRRORED_REPEAT — 镜像重复
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

    // GL_TEXTURE_MIN_FILTER — 纹理被缩小时的采样方式
    //   GL_NEAREST                — 最近邻采样（锯齿感强，速度快）
    //   GL_LINEAR                 — 双线性插值（平滑，轻微模糊）
    //   GL_LINEAR_MIPMAP_LINEAR   — 三线性插值（最平滑，需要 Mipmap）
    //   对 2D 精灵，GL_LINEAR 通常是最佳选择
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);

    // GL_TEXTURE_MAG_FILTER — 纹理被放大时的采样方式
    //   GL_NEAREST — 保留像素边缘（像素风格游戏推荐）
    //   GL_LINEAR  — 平滑插值（常规 2D 推荐）
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

    // ---- 第六步：上传像素数据到 GPU ----
    // glTexImage2D — 将 CPU 端的像素数据传输到 GPU 显存，创建纹理图像
    //   这是纹理创建过程中最核心的一步，也是显存分配实际发生的地方
    //
    // 参数详解（共 9 个参数）：
    //   target         — 纹理目标，GL_TEXTURE_2D
    //   level          — Mipmap 层级。0 = 原始尺寸（基层），1 = 1/2 尺寸，2 = 1/4 尺寸...
    //                    手动指定各层时用 1,2,3...；用 glGenerateMipmap 自动生成时只需传 0
    //   internalformat — GPU 内部存储格式（GL_RGBA, GL_RGB, GL_RED 等）
    //                    决定 GPU 为每个像素分配多少字节的显存
    //   width          — 纹理宽度（像素）
    //   height         — 纹理高度（像素）
    //   border         — 边框宽度，必须为 0（历史遗留参数，现代 OpenGL 已废弃）
    //   format         — CPU 端像素数据的格式（GL_RGBA, GL_RGB, GL_RED 等）
    //   type           — 每个颜色分量的数据类型
    //                    GL_UNSIGNED_BYTE = 每个通道 8 位（0-255），最常用
    //                    GL_FLOAT = 每个通道 32 位浮点（HDR 纹理使用）
    //   data           — 像素数据指针（传 nullptr 可创建空纹理，用于帧缓冲附件）
    glTexImage2D(GL_TEXTURE_2D,    // target
                 0,                 // level: 基层 Mipmap
                 internalFormat,    // GPU 内部格式
                 width, height,     // 纹理尺寸
                 0,                 // border: 必须为 0
                 pixelFormat,       // CPU 数据格式
                 GL_UNSIGNED_BYTE,  // 数据类型: 每通道 8 位
                 data);             // 像素数据指针

    // ---- 第七步：恢复默认对齐 ----
    // 上传完成后恢复默认对齐值，避免影响后续纹理加载
    if (channels == 3 || channels == 1) {
        glPixelStorei(GL_UNPACK_ALIGNMENT, 4);  // 恢复默认值
    }

    // ---- 第八步：释放 CPU 端像素数据 ----
    // 像素数据已经上传到 GPU 显存，CPU 端的副本不再需要
    stbi_image_free(data);

    return texture;
}
```

### GL 纹理函数速查表

下表汇总了纹理加载过程中涉及的所有 GL 函数：

| 函数 | 作用 | 关键参数 | 调用时机 |
|------|------|----------|----------|
| `glGenTextures` | 分配纹理 ID | n=数量 | 首次创建纹理 |
| `glBindTexture` | 绑定纹理到目标 | target=GL_TEXTURE_2D | 每次操作纹理前 |
| `glTexParameteri` | 设置采样参数 | WRAP/FILTER 模式 | 创建后、上传前 |
| `glPixelStorei` | 设置像素传输参数 | UNPACK_ALIGNMENT | 上传非 4 字节对齐数据前 |
| `glTexImage2D` | 上传像素数据到 GPU | 9 个参数（见上） | 创建纹理图像 |
| `glGenerateMipmap` | 自动生成 Mipmap 链 | target | 上传基层后（可选） |
| `glActiveTexture` | 激活纹理单元 | GL_TEXTURE0+n | 多纹理绑定时 |
| `glDeleteTextures` | 释放纹理对象 | n, textures | 不再需要纹理时 |

## 完整渲染循环

现在我们把前四节的所有组件组合起来：着色器编译（第 1 节）、正交投影矩阵（第 2 节）、坐标变换（第 3 节）、批处理器（第 4 节）、纹理加载（本节），构建一个完整可运行的 2D 渲染器。

### 渲染循环结构

一个标准的 OpenGL 渲染循环分为三个阶段：

```
┌─── 初始化阶段 ──────────────────────────────┐
│  1. 创建窗口 + GL 上下文                      │
│  2. 编译着色器                                │
│  3. 设置正交投影矩阵                          │
│  4. 创建 SpriteBatcher（VAO/VBO/EBO）         │
│  5. 加载纹理                                  │
└───────────────────────────────────────────────┘
          │
          ▼
┌─── 主循环（每帧重复）────────────────────────┐
│  1. 处理输入事件（键盘/鼠标）                  │
│  2. 更新逻辑（动画、物理、AI）                 │
│  3. 清屏 → 绘制精灵 → flush → 交换缓冲区      │
└───────────────────────────────────────────────┘
          │
          ▼
┌─── 清理阶段 ────────────────────────────────┐
│  1. 删除纹理、VAO、VBO、着色器程序             │
│  2. 销毁窗口                                  │
└───────────────────────────────────────────────┘
```

### 完整的 main.cpp

```cpp
#include <glad/glad.h>
#include <GLFW/glfw3.h>
#include <iostream>
#include <cmath>

// 前几节定义的函数和类（省略声明，完整代码见各节）
GLuint compileShader(GLenum type, const char* source);
GLuint createProgram(const char* vertSrc, const char* fragSrc);
void ortho(float* mat, float l, float r, float b, float t);
GLuint loadTexture(const char* path);

// SpriteBatcher 类（第 4 节完整实现）
class SpriteBatcher { /* 见第 4 节 */ };

// ----- 着色器源码 -----
const char* vertexShaderSource = R"(
#version 330 core
layout(location = 0) in vec2 aPos;      // 顶点位置（像素坐标）
layout(location = 1) in vec2 aTexCoord; // 纹理坐标
layout(location = 2) in vec4 aColor;    // 顶点颜色（用于调色/透明度）
uniform mat4 uProjection;               // 正交投影矩阵
out vec2 vTexCoord;
out vec4 vColor;
void main() {
    gl_Position = uProjection * vec4(aPos, 0.0, 1.0);
    vTexCoord = aTexCoord;
    vColor = aColor;
}
)";

const char* fragmentShaderSource = R"(
#version 330 core
in vec2 vTexCoord;
in vec4 vColor;
out vec4 FragColor;
uniform sampler2D uTexture;  // 纹理采样器
void main() {
    // texture() — GLSL 内置函数，从纹理中采样颜色
    // 参数 1: sampler2D 类型的纹理采样器
    // 参数 2: vec2 纹理坐标
    FragColor = texture(uTexture, vTexCoord) * vColor;
}
)";

int main() {
    // ===== 1. 初始化 GLFW 和 OpenGL 上下文 =====
    glfwInit();
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);
#ifdef __APPLE__
    glfwWindowHint(GLFW_OPENGL_FORWARD_COMPAT, GL_TRUE);  // macOS 必需
#endif

    const int SCREEN_W = 800, SCREEN_H = 600;
    GLFWwindow* window = glfwCreateWindow(
        SCREEN_W, SCREEN_H, "2D Renderer", nullptr, nullptr);
    glfwMakeContextCurrent(window);
    gladLoadGLLoader((GLADloadproc)glfwGetProcAddress);

    // ===== 2. 编译着色器并获取 uniform 位置 =====
    GLuint program = createProgram(vertexShaderSource, fragmentShaderSource);
    // glUseProgram(program) — 激活着色器程序
    glUseProgram(program);

    // glGetUniformLocation(program, name)
    //   查询 uniform 变量在着色器程序中的位置索引
    //   参数 program: 着色器程序 ID
    //   参数 name:    uniform 变量名（字符串）
    //   返回值：位置索引（≥0），-1 表示未找到（变量不存在或被编译器优化掉）
    GLint projLoc = glGetUniformLocation(program, "uProjection");
    GLint texLoc  = glGetUniformLocation(program, "uTexture");

    // ===== 3. 设置正交投影矩阵 =====
    float projection[16];
    ortho(projection, 0.0f, (float)SCREEN_W, (float)SCREEN_H, 0.0f);
    // glUniformMatrix4fv — 设置 mat4 类型的 uniform 变量
    //   参数 location:  uniform 位置索引
    //   参数 count:     矩阵数量（通常为 1）
    //   参数 transpose: 是否转置（GL_FALSE = 列主序，OpenGL 默认）
    //   参数 value:     矩阵数据指针（16 个 float）
    glUniformMatrix4fv(projLoc, 1, GL_FALSE, projection);
    // glUniform1i — 设置 int/sampler 类型的 uniform 变量
    //   纹理采样器的值是纹理单元编号（0 = GL_TEXTURE0, 1 = GL_TEXTURE1...）
    glUniform1i(texLoc, 0);

    // ===== 4. 创建批处理器 =====
    SpriteBatcher batcher;  // 内部创建 VAO/VBO/EBO

    // ===== 5. 加载纹理 =====
    GLuint bgTex    = loadTexture("assets/background.png");
    GLuint charTex  = loadTexture("assets/character.png");

    // ===== 6. 设置混合（透明度） =====
    // glEnable(GL_BLEND) — 开启 Alpha 混合
    //   不开启的话，半透明像素会覆盖背景而不是混合
    glEnable(GL_BLEND);
    // glBlendFunc(sfactor, dfactor) — 设置混合公式
    //   最终颜色 = 源颜色 × sfactor + 目标颜色 × dfactor
    //   GL_SRC_ALPHA           = 源 Alpha 值（即新像素的透明度）
    //   GL_ONE_MINUS_SRC_ALPHA = 1 - 源 Alpha 值
    //   效果：alpha=1.0 完全不透明，alpha=0.0 完全透明
    glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA);

    // ===== 7. 渲染主循环 =====
    while (!glfwWindowShouldClose(window)) {
        // -- 7a. 处理输入 --
        glfwPollEvents();
        if (glfwGetKey(window, GLFW_KEY_ESCAPE) == GLFW_PRESS)
            glfwSetWindowShouldClose(window, true);

        // -- 7b. 清屏 --
        // glClearColor — 设置清屏颜色（RGBA，范围 0.0-1.0）
        glClearColor(0.1f, 0.1f, 0.15f, 1.0f);
        // glClear — 清除指定缓冲区
        //   GL_COLOR_BUFFER_BIT — 清除颜色缓冲区（用 glClearColor 设置的颜色填充）
        //   可以 OR 组合：GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT
        glClear(GL_COLOR_BUFFER_BIT);

        // -- 7c. 绘制精灵 --
        // 背景（全屏铺满，白色调制 = 原色显示）
        batcher.drawSprite(bgTex,
            0.0f, 0.0f, (float)SCREEN_W, (float)SCREEN_H,   // 位置和尺寸
            0.0f, 0.0f, 1.0f, 1.0f,                          // UV 坐标
            1.0f, 1.0f, 1.0f, 1.0f);                         // RGBA 调制

        // 角色立绘（右侧，带透明度）
        float charW = 200.0f, charH = 300.0f;
        batcher.drawSprite(charTex,
            550.0f, 150.0f, charW, charH,
            0.0f, 0.0f, 1.0f, 1.0f,
            1.0f, 1.0f, 1.0f, 1.0f);

        // 半透明文本框（底部，蓝色调制，50% 透明）
        batcher.drawSprite(bgTex,   // 用背景纹理的一小块当文本框底色
            50.0f, 450.0f, 700.0f, 120.0f,
            0.0f, 0.0f, 0.1f, 0.1f,            // 只取纹理左上角一小块
            0.2f, 0.3f, 0.8f, 0.5f);            // 蓝色调制，alpha=0.5

        // -- 7d. 提交批次并交换缓冲区 --
        batcher.flush();
        // glfwSwapBuffers — 交换前后缓冲区
        //   OpenGL 使用双缓冲：后缓冲区绘制完成后与前缓冲区交换
        //   这样用户看到的始终是完整的一帧，不会看到绘制过程（画面撕裂）
        glfwSwapBuffers(window);
    }

    // ===== 8. 清理资源 =====
    // glDeleteTextures(n, textures) — 删除纹理对象，释放 GPU 显存
    glDeleteTextures(1, &bgTex);
    glDeleteTextures(1, &charTex);
    // glDeleteProgram(program) — 删除着色器程序
    glDeleteProgram(program);
    // batcher 析构函数会自动清理 VAO/VBO/EBO

    glfwDestroyWindow(window);
    glfwTerminate();
    return 0;
}
```

> **代码要点**：每一个 GL 函数调用都有注释说明其作用和参数含义。如果你对某个函数不确定，回到对应章节复习：着色器（第 1 节）、投影矩阵（第 2 节）、坐标变换（第 3 节）、批处理器（第 4 节）。

## 对照项目源码

### KrKr2 的纹理类：`tTVPOGLTexture2D`

**文件**：`cpp/core/visual/ogl/RenderManager_ogl.cpp` 第 862-961 行

KrKr2 的纹理管理比我们的 `loadTexture` 函数复杂得多。下面逐项对比：

#### 1. 纹理创建（构造函数，第 877-890 行）

```cpp
// KrKr2 源码（简化摘录）
tTVPOGLTexture2D(unsigned int w, unsigned int h,
                 TVPTextureFormat::e format,
                 GLint mode = GL_LINEAR) {
    glGenTextures(1, &texture);
    // 注意：这里用的是 Cocos2d-x 的封装函数
    // 它内部维护一个"当前绑定纹理"缓存，避免重复调用 glBindTexture
    cocos2d::GL::bindTexture2D(texture);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, mode);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, mode);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
}
```

**对比我们的实现**：
- KrKr2 使用 `cocos2d::GL::bindTexture2D` 代替直接 `glBindTexture`，因为 Cocos2d-x 缓存了当前绑定的纹理 ID，如果新旧 ID 相同就跳过调用，减少 GPU 状态切换
- KrKr2 始终使用 `GL_CLAMP_TO_EDGE`，与我们的选择一致——2D 游戏不需要纹理平铺

#### 2. 像素对齐策略（第 934-950 行）

```cpp
// KrKr2 的像素对齐——比我们更精细
switch (pitch & 7) {  // pitch = 一行图片数据的字节数
    case 0: glPixelStorei(GL_UNPACK_ALIGNMENT, 8); break;
    case 2: glPixelStorei(GL_UNPACK_ALIGNMENT, 2); break;
    case 4: glPixelStorei(GL_UNPACK_ALIGNMENT, 4); break;
    case 6: glPixelStorei(GL_UNPACK_ALIGNMENT, 2); break;
    default: glPixelStorei(GL_UNPACK_ALIGNMENT, 1); break;
}
```

**为什么 KrKr2 要这样做？**

我们的 `loadTexture` 用简单规则：RGB/灰度 → 对齐 1，RGBA → 默认 4。这在大多数情况下够用，但 KrKr2 处理的图片来源更复杂（TLG 格式、从内存动态生成等），行字节数（pitch）可能不等于 `width × channels`。比如一个 801×600 的 RGBA 图片，每行 3204 字节，`3204 & 7 = 4`，所以用对齐 4 而非默认的 8，既正确又高效。

#### 3. 显存监控（第 893、959 行）

```cpp
// 析构函数中
_totalVMemSize -= internalW * internalH * getPixelSize();
// InternalInit 中
_totalVMemSize += internalW * internalH * getPixelSize();
```

KrKr2 用全局变量 `_totalVMemSize` 跟踪所有纹理占用的显存总量。这在移动设备上很重要——Android 手机的 GPU 显存有限，超过阈值（如内存警告 `TVPCheckMemory`）时需要主动释放低优先级纹理。

#### 4. 格式适配（第 915-932 行）

KrKr2 支持三种格式，使用 `TVPTextureFormat::e` 枚举：
- `Gray` (1 字节/像素) → `GL_LUMINANCE`（注意不是 `GL_RED`，因为 OpenGL ES 2.0 不支持 `GL_RED`）
- `RGB` (3 字节/像素) → `GL_RGB`
- `RGBA` (4 字节/像素) → `GL_RGBA`

> **跨平台注意**：`GL_LUMINANCE` 是 OpenGL ES 2.0 的灰度格式。在桌面 OpenGL 3.3 Core Profile 中已被废弃，应使用 `GL_RED` + Swizzle Mask（纹理通道重映射，一种让单通道数据映射到 RGBA 四通道的技术）。KrKr2 使用 `GL_LUMINANCE` 是因为它需要兼容 OpenGL ES 2.0。

### KrKr2 的渲染提交方式

**文件**：`cpp/core/visual/ogl/RenderManager_ogl.cpp` 第 2328-2353 行

KrKr2 的渲染不使用索引缓冲，而是直接用 `glDrawArrays`：

```cpp
// KrKr2 纹理恢复时的全屏渲染（第 2328-2353 行简化）
// 手工构建 6 个顶点（两个三角形），覆盖 NDC 空间 [-1,1]
static const GLfloat vertices[12] = {
    -1, -1,   1, -1,  -1,  1,   // 三角形 1
     1, -1,  -1,  1,   1,  1    // 三角形 2
};
glVertexAttribPointer(posAttr, 2, GL_FLOAT, GL_FALSE, 0, vertices);
glDrawArrays(GL_TRIANGLES, 0, 6);  // 6 个顶点 = 2 个三角形
```

**与我们的区别**：
- 我们使用 EBO（索引缓冲）+ `glDrawElements`，每精灵 4 个顶点 + 6 个索引
- KrKr2 使用 `glDrawArrays`，每精灵 6 个顶点，不需要索引缓冲
- KrKr2 的坐标是预计算的 NDC（归一化设备坐标，Normalized Device Coordinates），不使用投影矩阵

KrKr2 不做批处理——每次 Draw Call 只画一个矩形。批处理由 Cocos2d-x 框架在上层处理。

## 常见错误及解决方案

### 错误一：纹理显示为黑色矩形

**原因**：纹理创建失败或采样器未正确设置。

**排查清单**：
1. 检查 `stbi_load` 返回值——NULL 表示文件路径错误或格式不支持
2. 确认 `glBindTexture` 在 `glTexImage2D` 之前调用
3. 确认片段着色器中的 `sampler2D` uniform 设置了正确的纹理单元编号（`glUniform1i(texLoc, 0)`）
4. 确认 `glActiveTexture(GL_TEXTURE0)` 被调用（或依赖默认值 GL_TEXTURE0）

### 错误二：图片颜色错位（斜纹/条纹）

**原因**：`GL_UNPACK_ALIGNMENT` 与实际数据对齐不匹配。

```cpp
// 错误：RGB 图片使用默认对齐 4
glTexImage2D(..., GL_RGB, GL_UNSIGNED_BYTE, data);
// 若图片宽度导致每行字节数不是 4 的倍数，颜色会错位

// 正确：先设置对齐为 1
glPixelStorei(GL_UNPACK_ALIGNMENT, 1);
glTexImage2D(..., GL_RGB, GL_UNSIGNED_BYTE, data);
glPixelStorei(GL_UNPACK_ALIGNMENT, 4);  // 恢复默认
```

### 错误三：图片上下颠倒

**原因**：未调用 `stbi_set_flip_vertically_on_load(true)`。

OpenGL 纹理坐标 (0,0) 在**左下角**，而 PNG/JPG 的 (0,0) 在**左上角**。如果不翻转，图片会倒着显示。

## 本节小结

- **stb_image** 是加载图片最简单的方案：三个 API 函数即可完成解码
- 纹理加载流程：解码 → 确定格式 → 设置对齐 → 创建纹理 → 设置参数 → 上传数据 → 释放 CPU 数据
- `glTexImage2D` 是纹理创建的核心，9 个参数各有含义，`border` 参数始终为 0
- `glPixelStorei(GL_UNPACK_ALIGNMENT, ...)` 对 RGB/灰度纹理至关重要，忘记设置会导致颜色错位
- 完整渲染循环 = 初始化 + 每帧（输入→更新→绘制→交换）+ 清理
- KrKr2 的纹理管理更精细：动态对齐选择、显存监控、Cocos2d-x 绑定缓存

## 练习题与答案

### 题目 1：纹理格式判断

以下代码有一个 Bug，会导致加载 PNG 透明图片时透明通道丢失。找出问题并修复：

```cpp
GLuint loadBuggyTexture(const char* path) {
    int w, h, ch;
    unsigned char* data = stbi_load(path, &w, &h, &ch, 3); // 强制 3 通道
    GLuint tex;
    glGenTextures(1, &tex);
    glBindTexture(GL_TEXTURE_2D, tex);
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGB, w, h, 0, GL_RGB,
                 GL_UNSIGNED_BYTE, data);
    stbi_image_free(data);
    return tex;
}
```

<details>
<summary>查看答案</summary>

**问题**：`stbi_load` 的第 5 个参数（desired_channels）被设为 3，强制将图片转为 RGB 格式，丢弃了 Alpha 通道。同时 `glTexImage2D` 使用 `GL_RGB` 格式，GPU 端也没有 Alpha 通道。

**修复**：让 stb_image 保持原始通道数，根据实际通道数选择格式：

```cpp
GLuint loadFixedTexture(const char* path) {
    int w, h, ch;
    stbi_set_flip_vertically_on_load(true);
    // desired_channels = 0，保持图片原始通道数
    unsigned char* data = stbi_load(path, &w, &h, &ch, 0);
    if (!data) return 0;

    GLuint tex;
    GLenum fmt = (ch == 4) ? GL_RGBA : GL_RGB;

    // RGB 需要设置对齐为 1
    if (ch == 3) glPixelStorei(GL_UNPACK_ALIGNMENT, 1);

    glGenTextures(1, &tex);
    glBindTexture(GL_TEXTURE_2D, tex);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    glTexImage2D(GL_TEXTURE_2D, 0, fmt, w, h, 0, fmt,
                 GL_UNSIGNED_BYTE, data);

    if (ch == 3) glPixelStorei(GL_UNPACK_ALIGNMENT, 4);
    stbi_image_free(data);
    return tex;
}
```

</details>

### 题目 2：渲染顺序与透明度

在我们的渲染循环中，如果把角色（charTex，有 Alpha 通道）画在背景（bgTex）之前，会出现什么问题？为什么？

<details>
<summary>查看答案</summary>

**问题**：角色的半透明区域会与**清屏颜色**（深灰色 0.1, 0.1, 0.15）混合，而不是与背景图混合。角色身体是正常的（不透明区域不受影响），但角色周围本该看到背景的地方会显示深灰色。

**原因**：Alpha 混合公式是 `最终颜色 = 源 × α + 目标 × (1-α)`。如果先画角色，此时"目标"是清屏颜色；之后画背景会覆盖角色，因为背景是完全不透明的（α=1.0）。

**结论**：2D 渲染中必须**从后往前**绘制（画家算法）：先画背景 → 再画角色 → 最后画 UI。这样每一层的半透明像素都能正确与下面的图层混合。

</details>

### 题目 3：实现纹理热重载

编写一段代码，在渲染循环中检测图片文件是否被修改（文件时间戳变化），如果修改了就重新加载纹理。提示：使用 `std::filesystem::last_write_time`。

<details>
<summary>查看答案</summary>

```cpp
#include <filesystem>
#include <chrono>
namespace fs = std::filesystem;

struct HotReloadTexture {
    GLuint id = 0;
    std::string path;
    fs::file_time_type lastModified;

    // 初次加载
    void load(const char* filePath) {
        path = filePath;
        id = loadTexture(filePath);
        lastModified = fs::last_write_time(path);
    }

    // 每帧调用，检查文件是否更新
    void checkReload() {
        auto currentTime = fs::last_write_time(path);
        if (currentTime != lastModified) {
            std::cout << "检测到文件变化，重新加载: " << path << std::endl;
            // 删除旧纹理
            glDeleteTextures(1, &id);
            // 重新加载
            id = loadTexture(path.c_str());
            lastModified = currentTime;
        }
    }
};

// 使用示例
HotReloadTexture bgTex;
bgTex.load("assets/background.png");

while (!glfwWindowShouldClose(window)) {
    bgTex.checkReload();  // 每帧检查
    // ... 使用 bgTex.id 绘制 ...
}
```

**优化提示**：在生产环境中，不要每帧都检查文件系统（性能开销大）。可以每 60 帧检查一次，或使用文件系统监视 API（如 inotify/ReadDirectoryChangesW/FSEvents）。

</details>
