## 1. 仿射变换基础

### 1.1 什么是仿射变换

仿射变换是二维图形中最基本的坐标变换族。它包含**平移**、**旋转**、**缩放**、**剪切**四种操作，以及它们的任意组合。仿射变换有两个关键性质：直线变换后仍然是直线；平行线变换后仍然平行。

用一个 2×3 矩阵（或 3×3 齐次矩阵）可以统一表达所有仿射变换：

```text
┌ x' ┐   ┌ a  b  tx ┐ ┌ x ┐
│ y' │ = │ c  d  ty │ │ y │
└  1 ┘   └ 0  0   1 ┘ └ 1 ┘

其中：
  a, d  控制缩放
  b, c  控制旋转和剪切
  tx,ty 控制平移
```

### 1.2 常用变换矩阵

| 变换 | 矩阵 | 说明 |
|------|------|------|
| 平移 (tx, ty) | `[1, 0, tx; 0, 1, ty; 0, 0, 1]` | 所有点偏移 (tx, ty) |
| 缩放 (sx, sy) | `[sx, 0, 0; 0, sy, 0; 0, 0, 1]` | 绕原点缩放 |
| 旋转 θ | `[cosθ, -sinθ, 0; sinθ, cosθ, 0; 0, 0, 1]` | 绕原点逆时针旋转 θ 弧度 |
| 剪切 (kx, ky) | `[1, kx, 0; ky, 1, 0; 0, 0, 1]` | 沿轴方向拉伸 |

组合变换按矩阵乘法从右到左应用。例如"先绕中心旋转再平移"：

```text
M = T(tx,ty) × R(θ) × T(-cx,-cy)
```

### 1.3 仿射矩阵的 C++ 表达

```cpp
#include <cmath>
#include <cstdint>

// 仿射变换矩阵（3×3 齐次矩阵）
struct AffineMatrix {
    float m[6]; // [a, b, tx, c, d, ty]，第三行固定 [0,0,1]

    // 变换一个点：(x,y) → (x',y')
    void transform(float x, float y, float& ox, float& oy) const {
        ox = m[0] * x + m[1] * y + m[2]; // a*x + b*y + tx
        oy = m[3] * x + m[4] * y + m[5]; // c*x + d*y + ty
    }

    // 构造旋转矩阵（弧度）
    static AffineMatrix rotation(float rad) {
        float c = std::cos(rad), s = std::sin(rad);
        return {c, -s, 0.0f, s, c, 0.0f};
    }

    // 构造缩放矩阵
    static AffineMatrix scale(float sx, float sy) {
        return {sx, 0.0f, 0.0f, 0.0f, sy, 0.0f};
    }

    // 构造平移矩阵
    static AffineMatrix translate(float tx, float ty) {
        return {1.0f, 0.0f, tx, 0.0f, 1.0f, ty};
    }

    // 矩阵乘法：this × other
    AffineMatrix multiply(const AffineMatrix& o) const {
        return {
            m[0]*o.m[0] + m[1]*o.m[3],          // a
            m[0]*o.m[1] + m[1]*o.m[4],          // b
            m[0]*o.m[2] + m[1]*o.m[5] + m[2],   // tx
            m[3]*o.m[0] + m[4]*o.m[3],          // c
            m[3]*o.m[1] + m[4]*o.m[4],          // d
            m[3]*o.m[2] + m[4]*o.m[5] + m[5]    // ty
        };
    }
};
```

### 1.4 反向映射与矩阵求逆

在渲染中，我们遍历**目标图像**的每个像素，反推它在**源图像**中的坐标，然后插值采样。这叫**反向映射**（inverse mapping）。正向映射（遍历源像素投射到目标）会导致目标像素遗漏（出现"空洞"）。

反向映射需要仿射矩阵的逆矩阵。对于 2×3 仿射矩阵，逆矩阵公式如下：

```cpp
// 计算仿射矩阵的逆矩阵
// 返回 false 表示矩阵不可逆（行列式为 0）
bool invert(const AffineMatrix& src, AffineMatrix& dst) {
    // 行列式 det = a*d - b*c
    float det = src.m[0] * src.m[4] - src.m[1] * src.m[3];
    if (std::fabs(det) < 1e-12f) return false; // 奇异矩阵（不可逆）
    float inv_det = 1.0f / det;

    dst.m[0] =  src.m[4] * inv_det;   // d / det
    dst.m[1] = -src.m[1] * inv_det;   // -b / det
    dst.m[3] = -src.m[3] * inv_det;   // -c / det
    dst.m[4] =  src.m[0] * inv_det;   // a / det

    // 平移部分：-(inv_rotation × 原平移)
    dst.m[2] = -(dst.m[0] * src.m[2] + dst.m[1] * src.m[5]);
    dst.m[5] = -(dst.m[3] * src.m[2] + dst.m[4] * src.m[5]);
    return true;
}
```

## 2. 仿射重采样实现

### 2.1 旋转 + 双线性采样

下面实现一个完整的仿射变换采样器。对目标图像的每个像素，通过逆矩阵找到源坐标，再用双线性插值：

```cpp
#include <cstdint>
#include <vector>
#include <algorithm>
#include <cmath>

struct ImageBuf {
    int w, h;
    std::vector<uint32_t> px; // ARGB

    uint32_t sample_clamp(int x, int y) const {
        x = std::max(0, std::min(x, w - 1));
        y = std::max(0, std::min(y, h - 1));
        return px[size_t(y) * w + x];
    }
};

// 单通道双线性插值（内联函数，被四通道复用）
static inline uint8_t lerp_channel(
    uint32_t c00, uint32_t c10, uint32_t c01, uint32_t c11,
    float u, float v, int shift)
{
    float p00 = static_cast<float>((c00 >> shift) & 0xFF);
    float p10 = static_cast<float>((c10 >> shift) & 0xFF);
    float p01 = static_cast<float>((c01 >> shift) & 0xFF);
    float p11 = static_cast<float>((c11 >> shift) & 0xFF);
    float top = p00 * (1.0f - u) + p10 * u; // 上边线性
    float bot = p01 * (1.0f - u) + p11 * u; // 下边线性
    float val = top * (1.0f - v) + bot * v;  // 纵向线性
    return static_cast<uint8_t>(std::max(0.0f, std::min(255.0f, val)) + 0.5f);
}

// 仿射变换采样（反向映射 + 双线性插值）
ImageBuf affine_resample(const ImageBuf& src, int dstW, int dstH,
                         const AffineMatrix& fwd)
{
    ImageBuf dst{dstW, dstH, std::vector<uint32_t>(size_t(dstW) * dstH, 0)};

    // 计算逆矩阵：目标坐标 → 源坐标
    AffineMatrix inv;
    if (!invert(fwd, inv)) return dst; // 不可逆则返回空图

    for (int dy = 0; dy < dstH; ++dy) {
        for (int dx = 0; dx < dstW; ++dx) {
            // 反向映射：目标像素中心 → 源图坐标
            float sx, sy;
            inv.transform(dx + 0.5f, dy + 0.5f, sx, sy);
            sx -= 0.5f; // 像素中心校正
            sy -= 0.5f;

            // 越界检查：完全在源图外则跳过（留透明）
            if (sx < -1.0f || sx >= src.w || sy < -1.0f || sy >= src.h)
                continue;

            // 双线性插值的四个邻居
            int x0 = static_cast<int>(std::floor(sx));
            int y0 = static_cast<int>(std::floor(sy));
            float u = sx - x0, v = sy - y0;

            uint32_t c00 = src.sample_clamp(x0,     y0);
            uint32_t c10 = src.sample_clamp(x0 + 1, y0);
            uint32_t c01 = src.sample_clamp(x0,     y0 + 1);
            uint32_t c11 = src.sample_clamp(x0 + 1, y0 + 1);

            uint8_t a = lerp_channel(c00, c10, c01, c11, u, v, 24);
            uint8_t r = lerp_channel(c00, c10, c01, c11, u, v, 16);
            uint8_t g = lerp_channel(c00, c10, c01, c11, u, v, 8);
            uint8_t b = lerp_channel(c00, c10, c01, c11, u, v, 0);

            dst.px[size_t(dy) * dstW + dx] =
                (uint32_t(a) << 24) | (uint32_t(r) << 16) |
                (uint32_t(g) << 8)  | uint32_t(b);
        }
    }
    return dst;
}
```

### 2.2 旋转示例：绕图像中心旋转 30°

```cpp
#include <cstdio>

int main() {
    // 构造变换：绕图像中心旋转 30°
    const int W = 256, H = 256;
    float cx = W / 2.0f, cy = H / 2.0f;
    float rad = 30.0f * 3.14159265f / 180.0f;

    // 变换顺序：先平移到原点 → 旋转 → 再平移回来
    AffineMatrix t1 = AffineMatrix::translate(-cx, -cy);
    AffineMatrix rot = AffineMatrix::rotation(rad);
    AffineMatrix t2 = AffineMatrix::translate(cx, cy);
    AffineMatrix fwd = t2.multiply(rot.multiply(t1));

    // 加载/生成源图（此处用棋盘格代替）
    ImageBuf src{W, H, std::vector<uint32_t>(size_t(W) * H)};
    for (int y = 0; y < H; ++y)
        for (int x = 0; x < W; ++x)
            src.px[y * W + x] = ((x/16 + y/16) % 2) ? 0xFFFFFFFF : 0xFF3366CC;

    // 执行仿射变换
    ImageBuf result = affine_resample(src, W, H, fwd);
    std::printf("旋转完成，输出 %d×%d 像素\n", result.w, result.h);
    return 0;
}
```

## 3. 抗锯齿与 Mipmap

### 3.1 缩小时的混叠问题

当图像被大幅缩小（如 4× 以上）时，直接双线性插值只采样 2×2 邻域，而目标像素实际覆盖了源图上更大的区域。这些未被采样到的高频细节会产生**混叠**（aliasing，即信号欠采样导致的伪影）——视觉上表现为闪烁、摩尔纹（moiré pattern，一种由规则图案干涉产生的波纹假图案）。

### 3.2 Mipmap 的思想

Mipmap 是预计算一系列逐级缩半的图像金字塔，缩小时从接近目标尺寸的那一级采样，避免"跨太多像素"的欠采样问题。

```text
Level 0:  256×256  (原图)
Level 1:  128×128  (每 2×2 取平均)
Level 2:   64×64
Level 3:   32×32
Level 4:   16×16
...
```

选择哪一级取决于像素在源图中的"脚印"大小。如果一个目标像素覆盖源图 4×4 区域，就用 Level 2（64×64）采样。

### 3.3 Mipmap 链生成

```cpp
#include <cstdint>
#include <vector>

struct MipChain {
    struct Level {
        int w, h;
        std::vector<uint32_t> data; // ARGB
    };
    std::vector<Level> levels;
};

// 生成 Mipmap 链：每级宽高减半，像素取 2×2 均值
MipChain generate_mipchain(const uint32_t* src, int w, int h) {
    MipChain chain;
    // Level 0 = 原图
    chain.levels.push_back({w, h, std::vector<uint32_t>(src, src + size_t(w) * h)});

    int cw = w, ch = h;
    while (cw > 1 && ch > 1) {
        int nw = cw / 2, nh = ch / 2;
        if (nw < 1) nw = 1;
        if (nh < 1) nh = 1;

        const auto& prev = chain.levels.back().data;
        std::vector<uint32_t> next(size_t(nw) * nh);

        for (int y = 0; y < nh; ++y) {
            for (int x = 0; x < nw; ++x) {
                int sx = x * 2, sy = y * 2;
                // 取 2×2 区域的四个像素
                uint32_t p00 = prev[sy * cw + sx];
                uint32_t p10 = prev[sy * cw + std::min(sx + 1, cw - 1)];
                uint32_t p01 = prev[std::min(sy + 1, ch - 1) * cw + sx];
                uint32_t p11 = prev[std::min(sy + 1, ch - 1) * cw
                                    + std::min(sx + 1, cw - 1)];

                // 四通道分别取平均
                auto avg = [](uint32_t a, uint32_t b, uint32_t c, uint32_t d,
                              int shift) -> uint32_t {
                    return (((a >> shift) & 0xFF) + ((b >> shift) & 0xFF) +
                            ((c >> shift) & 0xFF) + ((d >> shift) & 0xFF) + 2) / 4;
                };
                uint32_t ra = avg(p00, p10, p01, p11, 24);
                uint32_t rr = avg(p00, p10, p01, p11, 16);
                uint32_t rg = avg(p00, p10, p01, p11, 8);
                uint32_t rb = avg(p00, p10, p01, p11, 0);

                next[size_t(y) * nw + x] =
                    (ra << 24) | (rr << 16) | (rg << 8) | rb;
            }
        }
        chain.levels.push_back({nw, nh, std::move(next)});
        cw = nw;
        ch = nh;
    }
    return chain;
}
```

### 3.4 Mipmap 等级选择

选择公式基于目标像素在源图上的"脚印"大小。对于均匀缩放比 `scale`：

```cpp
// 选择最合适的 Mipmap 等级
// scale: 缩放因子（dstSize / srcSize），< 1.0 表示缩小
int select_mip_level(float scale, int max_level) {
    if (scale >= 1.0f) return 0; // 放大时使用原图
    // LOD = log2(1/scale)，即缩小 4× 时 LOD=2
    float lod = std::log2(1.0f / scale);
    int level = static_cast<int>(std::floor(lod));
    return std::min(level, max_level);
}
```

> **注意**：上面选择最近等级会有"跳变"，高质量实现会在相邻两级之间做**三线性插值**（trilinear interpolation，即在两个 Mipmap 级别的双线性结果之间再做一次线性混合），GPU 纹理采样器默认就是这样做的。

## 4. SIMD 加速双线性插值

双线性插值的核心是 4 个像素的加权混合，非常适合 SSE2 并行化。下面展示如何用 SSE2 intrinsic 加速单个像素的双线性采样：

```cpp
#include <emmintrin.h> // SSE2
#include <cstdint>

// SSE2 加速的双线性插值（处理单个采样点）
// 输入：四个邻居像素 c00/c10/c01/c11，权重 u/v（0~255 整数化）
static inline uint32_t bilinear_sse2(
    uint32_t c00, uint32_t c10, uint32_t c01, uint32_t c11,
    int u8, int v8) // u8 = u*256, v8 = v*256，范围 [0,256]
{
    __m128i zero = _mm_setzero_si128();

    // 加载四个像素并解包到 16 位
    // _mm_cvtsi32_si128: 将 32 位整数放入 128 位寄存器的最低 32 位
    __m128i p00 = _mm_unpacklo_epi8(_mm_cvtsi32_si128(c00), zero);
    __m128i p10 = _mm_unpacklo_epi8(_mm_cvtsi32_si128(c10), zero);
    __m128i p01 = _mm_unpacklo_epi8(_mm_cvtsi32_si128(c01), zero);
    __m128i p11 = _mm_unpacklo_epi8(_mm_cvtsi32_si128(c11), zero);

    // 权重向量
    // _mm_set1_epi16: 将 16 位整数广播到所有 8 个位置
    __m128i wu  = _mm_set1_epi16(static_cast<short>(u8));
    __m128i wiu = _mm_set1_epi16(static_cast<short>(256 - u8));
    __m128i wv  = _mm_set1_epi16(static_cast<short>(v8));
    __m128i wiv = _mm_set1_epi16(static_cast<short>(256 - v8));

    // top  = c00*(256-u) + c10*u （横向插值，上边）
    // _mm_mullo_epi16: 8 个 16 位整数同时相乘，取低 16 位
    __m128i top = _mm_add_epi16(
        _mm_mullo_epi16(p00, wiu), // c00 * (256-u)
        _mm_mullo_epi16(p10, wu)   // c10 * u
    );
    // bot  = c01*(256-u) + c11*u （横向插值，下边）
    __m128i bot = _mm_add_epi16(
        _mm_mullo_epi16(p01, wiu),
        _mm_mullo_epi16(p11, wu)
    );

    // result = top*(256-v) + bot*v （纵向插值）
    // 这里结果范围 0~256*256*255，需要右移 16 位
    // _mm_srli_epi16: 每个 16 位元素逻辑右移 n 位
    top = _mm_srli_epi16(top, 8); // 先缩回 [0,255] 范围
    bot = _mm_srli_epi16(bot, 8);
    __m128i result = _mm_add_epi16(
        _mm_mullo_epi16(top, wiv),
        _mm_mullo_epi16(bot, wv)
    );
    result = _mm_srli_epi16(result, 8); // 最终缩回 [0,255]

    // 打包回 8 位
    // _mm_packus_epi16: 将 16 位饱和到 8 位无符号
    result = _mm_packus_epi16(result, zero);

    // 提取 32 位结果
    // _mm_cvtsi128_si32: 从寄存器最低 32 位提取为普通整数
    return static_cast<uint32_t>(_mm_cvtsi128_si32(result));
}
```

| SSE2 函数 | 作用 | 参数说明 |
|-----------|------|----------|
| `_mm_cvtsi32_si128(x)` | 32 位整数 → 128 位寄存器低 32 位 | x: 普通 int/uint32_t |
| `_mm_unpacklo_epi8(a, zero)` | 低 8 字节与零交错 → 8 个 uint16 | a: 源像素, zero: 全零 |
| `_mm_set1_epi16(x)` | 将 16 位值广播到所有 8 个位置 | x: short 类型权重 |
| `_mm_mullo_epi16(a, b)` | 8 组 16 位乘法取低 16 位 | a, b: 操作数 |
| `_mm_add_epi16(a, b)` | 8 组 16 位加法 | a, b: 操作数 |
| `_mm_srli_epi16(a, n)` | 每个 16 位元素逻辑右移 n 位 | a: 操作数, n: 立即数 |
| `_mm_packus_epi16(a, b)` | 16 位饱和到 8 位无符号 | a: 低 8 通道, b: 高 8 通道 |
| `_mm_cvtsi128_si32(a)` | 从寄存器低 32 位提取为 int | a: 128 位寄存器 |

## 5. KrKr2 ResampleImage 工程化分析

KrKr2 的图像缩放不是教科书式的逐像素遍历，而是一套高度优化的工程化方案。其核心文件是 `cpp/core/visual/gl/ResampleImage.cpp`，采用**6 步流水线**：

### 5.1 六步流水线

```text
┌─────────────────────────────────────────────────────┐
│  步骤 1：setClipping                                │
│  设置源/目标裁剪区域，避免越界                       │
├─────────────────────────────────────────────────────┤
│  步骤 2：setFunctionFromParam                       │
│  根据用户参数选择插值核函数（双线性/双三次/Lanczos等）│
├─────────────────────────────────────────────────────┤
│  步骤 3：AxisParamCalculateAxis                      │
│  对 X 轴和 Y 轴分别预计算权重表                     │
├─────────────────────────────────────────────────────┤
│  步骤 4：samplingVertical + samplingHorizontal       │
│  先垂直采样到中间缓冲，再水平采样到最终结果          │
│  （轴分离卷积：把 O(n²·k²) 降到 O(n²·2k)）         │
├─────────────────────────────────────────────────────┤
│  步骤 5：ResampleMT                                 │
│  多线程分发，每个线程处理若干行                       │
├─────────────────────────────────────────────────────┤
│  步骤 6：TVPResampleImage                            │
│  入口调度函数，协调以上所有步骤                       │
└─────────────────────────────────────────────────────┘
```

### 5.2 轴分离卷积

这是 ResampleImage 最关键的优化思想。如果对每个像素直接应用 k×k 的二维卷积核，计算量是 O(n² × k²)。但由于大多数插值核是**可分离的**（separable，即二维核可以表示为两个一维核的外积），可以拆成两次一维卷积：

```text
原始：对每个像素，查询 k×k 邻域 → 每像素 k² 次乘加
分离：先对每列做 k 点垂直卷积 → 中间结果
      再对每行做 k 点水平卷积 → 最终结果
      每像素只需 2k 次乘加

例：Lanczos-3 核 k=6
  原始：6×6 = 36 次乘加/像素
  分离：6+6 = 12 次乘加/像素 → 3 倍加速
```

### 5.3 WeightFunctor RANGE 常量

KrKr2 在 `WeightFunctor.cpp` 中为每种插值核定义了 RANGE（权重有效半径）常量：

| 插值核 | RANGE | 邻域大小 | 说明 |
|--------|------:|-------:|------|
| BilinearWeight | 1.0 | 2×2 | 最基本的线性插值 |
| BicubicWeight | 2.0 | 4×4 | Keys 三次核 |
| Spline16Weight | 2.0 | 4×4 | 分段三次样条，比双三次更平稳 |
| Spline36Weight | 3.0 | 6×6 | 扩展样条，高质量但较慢 |
| GaussianWeight | 2.0 | 4×4 | 高斯核，平滑但可能过度模糊 |
| BlackmanSincWeight | 4.0 | 8×8 | Blackman 窗 sinc，最高质量 |

RANGE 决定了每个方向需要查询多少个邻居像素。RANGE=3 意味着每个方向查 6 个点（左右各 3），所以是 6×6 邻域。

### 5.4 AffineBlt 与 StretchType

`LayerBitmapIntf.cpp` 中的 `AffineBlt` 函数通过 `StretchType` 枚举选择拉伸路径：

```cpp
// KrKr2 源码中的关键枚举（简化）
enum StretchType {
    stNearest       = 0, // 最近邻
    stFastLinear    = 1, // 快速双线性（内置优化路径）
    stLinear        = 2, // 标准双线性
    stCubic         = 3, // 双三次
    stSpline16      = 4, // Spline16
    stSpline36      = 5, // Spline36
    stLanczos2      = 6, // Lanczos-2
    stLanczos3      = 7, // Lanczos-3
    stAreaAvg       = 8, // 区域平均（缩小专用）
    stGaussian      = 9, // 高斯
    stBlackmanSinc  = 10 // Blackman-Sinc
};
```

`stFastLinear` 是 KrKr2 的特殊优化路径：对于不需要高质量的实时预览（如拖拽缩放时），跳过权重预计算，直接用硬编码的双线性公式，速度显著快于通用 `stLinear`。

### 实践 1：实现一个旋转动画

1. 创建一个 256×256 的棋盘格图像。
2. 用上面的 `affine_resample` 函数，以 5° 步长生成 72 帧旋转序列。
3. 导出为 `frame_000.ppm` 到 `frame_071.ppm`。
4. 对比最近邻和双线性在旋转时的边缘质量。

```cpp
#include <cstdio>

void generate_rotation_sequence() {
    const int W = 256, H = 256;
    ImageBuf src{W, H, std::vector<uint32_t>(size_t(W) * H)};
    // 生成棋盘格
    for (int y = 0; y < H; ++y)
        for (int x = 0; x < W; ++x)
            src.px[y * W + x] = ((x/8 + y/8) % 2) ? 0xFFFFFFFF : 0xFF2266AA;

    float cx = W / 2.0f, cy = H / 2.0f;
    for (int frame = 0; frame < 72; ++frame) {
        float deg = frame * 5.0f;
        float rad = deg * 3.14159265f / 180.0f;
        AffineMatrix t1 = AffineMatrix::translate(-cx, -cy);
        AffineMatrix rot = AffineMatrix::rotation(rad);
        AffineMatrix t2 = AffineMatrix::translate(cx, cy);
        AffineMatrix fwd = t2.multiply(rot.multiply(t1));

        ImageBuf result = affine_resample(src, W, H, fwd);

        char filename[64];
        std::snprintf(filename, sizeof(filename), "frame_%03d.ppm", frame);
        // 导出 PPM（复用前节的 savePPM 函数）
        std::printf("导出 %s (旋转 %.0f°)\n", filename, deg);
    }
}
```

## 常见错误及解决方案

### 错误 1：用正向映射做旋转后出现空洞

**现象：** 旋转后的图像上出现散布的黑色像素点（空洞）。
**原因：** 正向映射遍历源像素并投射到目标，但多个源像素可能映射到同一目标像素，而某些目标像素可能没有源像素映射到。这些遗漏的像素就是空洞。
**解决：** 使用反向映射——遍历目标图像的每个像素，通过逆矩阵反推到源图坐标，保证每个目标像素都有值。

### 错误 2：仿射变换时透明边缘出现黑边

**现象：** 旋转一张带透明通道的图像后，透明区域边缘出现深色或黑色条带。
**原因：** 在非预乘格式下做双线性插值时，透明像素的 RGB 通道（通常是 `(0,0,0,0)`）参与了混合，把边缘颜色拉向黑色。
**解决：** 先将图像转为预乘 Alpha 格式（R×A, G×A, B×A），再做插值，最后需要时再转回非预乘。预乘格式下透明像素的 RGB 也是 0，但混合公式正确处理了权重，不会引入错误的黑色。

### 错误 3：缩小图像时不用 Mipmap 导致闪烁

**现象：** 动画中图像连续缩小时，纹理细节闪烁不稳定。
**原因：** 大幅缩小时一个目标像素覆盖了源图上很大的区域，但只采样了 2×2（双线性）或 4×4（双三次）邻域，高频信息被欠采样导致混叠。
**解决：** 使用 Mipmap 预过滤：根据缩小比例选择合适的 Mipmap 级别，从已经低通过的图像上采样。或者使用 AreaAverage 算法，按实际覆盖面积加权平均。

## 本节小结

- **仿射变换**用 3×3 齐次矩阵统一表达平移、旋转、缩放、剪切。
- **反向映射**是渲染管线的标准做法——遍历目标像素反查源坐标，避免空洞。
- **矩阵求逆**对于 2×3 仿射矩阵有封闭解，计算行列式 `a*d - b*c` 即可。
- **Mipmap** 通过预计算多级缩小图像解决大比例缩小时的混叠问题。
- **SIMD 双线性**将四通道权重混合从 16 次标量乘法压缩为 3 条 SSE2 指令。
- **KrKr2 的 ResampleImage** 采用"权重预计算 + 轴分离卷积 + 多线程"的 6 步流水线，把 O(n²·k²) 降到 O(n²·2k)。
- **StretchType 枚举**提供 11 种插值策略，`stFastLinear` 是实时场景的特殊快速路径。

## 练习题与答案

### 题目 1：为什么反向映射优于正向映射？

<details>
<summary>查看答案</summary>

正向映射遍历源图像素，将每个源像素投射到目标位置。问题有两个：

1. **空洞**：多个源像素可能映射到同一目标像素，而另一些目标像素没有任何源像素映射到，形成空洞（黑点）。
2. **精度**：源像素投射到目标的非整数位置，需要"溅射"（splatting）到周围像素，实现复杂且容易出错。

反向映射遍历目标图像的每个像素，通过逆矩阵计算其在源图上的坐标，然后插值采样。这保证了每个目标像素都会被赋值，不会有空洞，且可以自然地使用各种插值算法（最近邻、双线性、双三次等）。

</details>

### 题目 2：轴分离卷积为什么能加速？它的前提条件是什么？

<details>
<summary>查看答案</summary>

轴分离卷积将一次 k×k 的二维卷积拆成两次 k 点的一维卷积（先水平后垂直，或反过来），计算量从 O(k²) 降到 O(2k)。

前提条件是卷积核必须是**可分离的**（separable），即二维核 H(x,y) 可以表示为两个一维核的外积：`H(x,y) = h_x(x) × h_y(y)`。常见的可分离核包括：高斯核、双线性核、双三次核、Lanczos 核。不可分离的核（如某些方向性滤波器）不能使用这个优化。

以 Lanczos-3（RANGE=3，邻域 6×6）为例：
- 直接二维卷积：每像素 36 次乘加
- 轴分离：每像素 12 次乘加（6 次水平 + 6 次垂直）
- 加速比：3 倍

</details>

### 题目 3：写出绕任意点 (cx, cy) 旋转 θ 角的组合仿射矩阵

<details>
<summary>查看答案</summary>

绕任意点旋转需要三步组合：

1. **平移到原点**：`T(-cx, -cy)`
2. **旋转 θ**：`R(θ)`
3. **平移回去**：`T(cx, cy)`

组合矩阵 `M = T(cx,cy) × R(θ) × T(-cx,-cy)`：

```cpp
// 代码实现
AffineMatrix rotate_around(float cx, float cy, float theta) {
    AffineMatrix t1 = AffineMatrix::translate(-cx, -cy);
    AffineMatrix rot = AffineMatrix::rotation(theta);
    AffineMatrix t2 = AffineMatrix::translate(cx, cy);
    return t2.multiply(rot.multiply(t1));
}

// 展开后的矩阵元素：
// a  = cos(θ)
// b  = -sin(θ)
// tx = cx - cx*cos(θ) + cy*sin(θ)
// c  = sin(θ)
// d  = cos(θ)
// ty = cy - cx*sin(θ) - cy*cos(θ)
```

注意矩阵乘法的顺序：代码中写 `t2 × rot × t1`，但实际变换顺序是从右到左——先执行 t1（平移到原点），再执行 rot（旋转），最后执行 t2（平移回去）。

</details>
