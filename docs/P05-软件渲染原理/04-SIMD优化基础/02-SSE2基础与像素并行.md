## 1. SSE2 数据类型与寄存器

### 1.1 寄存器组

SSE2 提供 **16 个 128 位寄存器**，命名为 `xmm0` 到 `xmm15`（在 32 位模式下只有 `xmm0`-`xmm7`）。在 C/C++ 中通过 intrinsic 函数（编译器内建函数，参见上一节）访问：

```cpp
#include <emmintrin.h>  // SSE2 intrinsic 头文件

// SSE2 提供三种 128 位数据类型
__m128i  reg;   // 128 位整数向量（像素处理最常用）
__m128   freg;  // 128 位单精度浮点向量（4× float）
__m128d  dreg;  // 128 位双精度浮点向量（2× double）
```

`__m128i` 是像素处理的核心类型。它可以被解释为多种整数排列：

```
__m128i 的多种解释方式（同一块 128 位数据）：

  解释为 16× uint8 ：[b15 b14 b13 ... b2 b1 b0]
    → 像素的单个颜色通道（一次操作 16 个字节）

  解释为 8× uint16 ：[h7 h6 h5 h4 | h3 h2 h1 h0]
    → 中间计算用，16 位空间避免 8 位乘法溢出

  解释为 4× uint32 ：[w3 | w2 | w1 | w0]
    → 恰好是 4 个完整的 ARGB 像素

  解释为 2× uint64 ：[q1 | q0]
```

### 1.2 命名约定

SSE2 intrinsic 函数遵循统一的命名规则：

```
_mm_<操作>_<数据类型后缀>

前缀：
  _mm      → 128 位操作（SSE/SSE2）
  _mm256   → 256 位操作（AVX/AVX2）
  _mm512   → 512 位操作（AVX-512）

数据类型后缀：
  epi8     → 有符号 8 位整数（signed int8）
  epu8     → 无符号 8 位整数（unsigned uint8）
  epi16    → 有符号 16 位整数
  epu16    → 无符号 16 位整数
  epi32    → 有符号 32 位整数
  si128    → 不区分元素大小的 128 位整体操作
  ps       → 单精度浮点（packed single）
  pd       → 双精度浮点（packed double）
```

例如 `_mm_adds_epu8` = 128 位 + 饱和加法（add saturated） + 无符号 8 位。

## 2. 基础操作：加载、存储与常量

### 2.1 加载数据到寄存器

```cpp
// 从内存加载 128 位（16 字节）数据到 SSE 寄存器

// 对齐加载（地址必须是 16 的整数倍）—— 速度最快
__m128i pixels = _mm_load_si128((__m128i*)aligned_ptr);

// 非对齐加载（任意地址都可以）—— 更安全
__m128i pixels = _mm_loadu_si128((__m128i*)any_ptr);
// 名字中的 'u' = unaligned（未对齐）
```

### 2.2 存储数据到内存

```cpp
// 对齐存储（目标地址必须 16 字节对齐）
_mm_store_si128((__m128i*)aligned_ptr, pixels);

// 非对齐存储（任意地址）
_mm_storeu_si128((__m128i*)any_ptr, pixels);
```

### 2.3 创建常量向量

```cpp
// 全零
__m128i zero = _mm_setzero_si128();           // 128 位全 0

// 广播一个字节值到 16 个位置
__m128i all_ff = _mm_set1_epi8((char)0xFF);   // 16 个 0xFF

// 广播一个 16 位值到 8 个位置
__m128i val = _mm_set1_epi16(255);            // 8 个 255

// 广播一个 32 位值到 4 个位置
__m128i mask = _mm_set1_epi32(0x00FF00FF);    // 4 个 0x00FF00FF

// 手动设置 4 个不同的 32 位值
__m128i custom = _mm_set_epi32(3, 2, 1, 0);
// 注意：参数顺序是从高到低！结果是 [3, 2, 1, 0]
```

### 2.4 对齐内存分配

当需要使用对齐加载/存储时，必须确保内存地址对齐：

```cpp
// 方法 1：Windows API
uint32_t* buf = (uint32_t*)_aligned_malloc(size_bytes, 16);
// 用完后必须用 _aligned_free(buf) 释放

// 方法 2：POSIX（Linux / macOS）
uint32_t* buf = nullptr;
posix_memalign((void**)&buf, 16, size_bytes);
// 用完后用 free(buf) 释放

// 方法 3：C++17 跨平台（推荐）
#include <cstdlib>
uint32_t* buf = static_cast<uint32_t*>(
    std::aligned_alloc(16, size_bytes)
);
// 注意：size_bytes 必须是 16 的整数倍
// 用完后用 std::free(buf) 释放

// 方法 4：栈上对齐（C++11+）
alignas(16) uint32_t buf[4];  // 编译器保证 16 字节对齐
```

## 3. 整数算术与位操作指令

### 3.1 加法与减法

```cpp
// 按字节（8 位）加法 —— 16 个字节同时加，回绕模式
__m128i sum = _mm_add_epi8(a, b);
// 200 + 100 = 44（溢出回绕，通常不是像素处理想要的）

// 无符号字节饱和加法 —— 像素处理最常用
__m128i sum = _mm_adds_epu8(a, b);
// 200 + 100 = 255（饱和截断，正确的像素行为）

// 无符号字节饱和减法
__m128i diff = _mm_subs_epu8(a, b);
// 50 - 100 = 0（下溢截断到 0，而非回绕到 206）

// 16 位加法（用于中间计算，防止 8 位溢出）
__m128i sum16 = _mm_add_epi16(a, b);

// 16 位减法
__m128i diff16 = _mm_sub_epi16(a, b);
```

### 3.2 乘法

```cpp
// 16 位乘法（取低 16 位）—— Alpha 混合的关键指令
__m128i lo = _mm_mullo_epi16(a, b);
// 每个 16 位元素：result = (a * b) 的低 16 位
// 例：200 * 128 = 25600，取低 16 位 = 25600（未溢出）

// 16 位乘法（取高 16 位）
__m128i hi = _mm_mulhi_epu16(a, b);
// 每个 16 位元素：result = (a * b) >> 16
```

> **为什么没有 8 位乘法？** SSE2 不提供 `_mm_mullo_epi8`。两个 8 位数相乘结果最大 255×255=65025，需要 16 位才能存下。所以像素乘法的标准做法是：先解包到 16 位 → 做 16 位乘法 → 打包回 8 位。

### 3.3 移位

```cpp
// 按 16 位元素逻辑右移（每个 16 位元素独立移位）
__m128i shifted = _mm_srli_epi16(a, 8);
// 每个 uint16 右移 8 位，等效于 ÷256

// 按 32 位元素逻辑右移
__m128i shifted32 = _mm_srli_epi32(a, 24);
// 每个 uint32 右移 24 位，可用于提取 ARGB 的 Alpha

// 按 16 位元素逻辑左移
__m128i shifted_left = _mm_slli_epi16(a, 8);
```

### 3.4 位操作

```cpp
// 按位与（AND）—— 常用于掩码提取通道
__m128i result = _mm_and_si128(a, b);

// 按位或（OR）—— 常用于组装像素
__m128i result = _mm_or_si128(a, b);

// 按位异或（XOR）
__m128i result = _mm_xor_si128(a, b);

// 按位与非（(~a) AND b）—— 注意是先取反 a 再与 b
__m128i result = _mm_andnot_si128(a, b);
```

## 4. 像素打包与解包（核心技巧）

像素处理中最关键的技巧是 **8 位 ↔ 16 位转换**。乘法需要 16 位精度避免溢出，但像素数据是 8 位存储的。

### 4.1 解包（Unpack）：8 位扩展到 16 位

```cpp
// 输入：4 个 ARGB 像素打包在一个 __m128i 中
// [A3 R3 G3 B3 | A2 R2 G2 B2 | A1 R1 G1 B1 | A0 R0 G0 B0]
//  ←—— 高 64 位 ——→             ←—— 低 64 位 ——→

__m128i zero = _mm_setzero_si128();

// 解包低 64 位：将低 8 个字节中的每个字节前插入一个 0 字节
__m128i lo16 = _mm_unpacklo_epi8(pixels, zero);
// 结果（8× uint16）：
// [00 A1 | 00 R1 | 00 G1 | 00 B1 | 00 A0 | 00 R0 | 00 G0 | 00 B0]
// 每个通道值从 uint8 变成了 uint16，高字节补零

// 解包高 64 位：将高 8 个字节同样扩展
__m128i hi16 = _mm_unpackhi_epi8(pixels, zero);
// 结果（8× uint16）：
// [00 A3 | 00 R3 | 00 G3 | 00 B3 | 00 A2 | 00 R2 | 00 G2 | 00 B2]
```

解包后，4 个像素变成了 2 个 `__m128i`（lo16 和 hi16），每个含 2 个像素的 8 个 16 位通道值。这样就有足够的空间做乘法运算而不溢出。

### 4.2 打包（Pack）：16 位压缩回 8 位

```cpp
// 计算完成后，将 16 位结果打包回 8 位
__m128i packed = _mm_packus_epi16(lo16, hi16);
// packus = pack unsigned saturated（带无符号饱和的打包）
// 将 lo16 的 8 个 int16 和 hi16 的 8 个 int16
// 各自饱和截断到 [0, 255]，打包成 16 个 uint8
```

### 4.3 完整的解包→计算→打包流程

```
原始像素（4 个 ARGB，128 位）
  [A3R3G3B3 | A2R2G2B2 | A1R1G1B1 | A0R0G0B0]
            │
            ├── unpacklo_epi8 ──→ 像素 0,1 的 8× uint16
            │                      （可以安全做乘法）
            │
            └── unpackhi_epi8 ──→ 像素 2,3 的 8× uint16
                                   （可以安全做乘法）
            │
            │  做 16 位乘法、加法等运算...
            │
            └── packus_epi16 ──→ 结果打包回 16× uint8
                                  （溢出自动饱和到 255）
```

## 5. SSE2 实现 4 像素并行 Alpha 混合

这是本节的核心。我们将标量 Alpha 混合升级为 SSE2 版本。

### 5.1 混合公式回顾

```
对于每个颜色通道（R、G、B）：
  result = src × src_alpha + dst × (255 - src_alpha)
  result = result / 255   （归一化回 [0,255]）
```

### 5.2 完整实现

```cpp
#include <emmintrin.h>  // SSE2
#include <cstdint>

// SSE2 Alpha 混合：一次处理 4 个 ARGB 像素
// dst: 目标像素数组，src: 源像素数组
// count: 像素数量（必须是 4 的倍数，尾部另行处理）
void alpha_blend_sse2(uint32_t* dst, const uint32_t* src, int count) {
    const __m128i zero = _mm_setzero_si128();
    const __m128i alpha_mask = _mm_set1_epi32(0xFF000000);  // Alpha 通道位掩码
    const __m128i round_add  = _mm_set1_epi16(128);         // 四舍五入偏移量

    for (int i = 0; i < count; i += 4) {
        // ── 步骤 1：加载 4 个源像素和 4 个目标像素 ──
        __m128i s = _mm_loadu_si128((__m128i*)(src + i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst + i));
        // s = [A3 R3 G3 B3 | A2 R2 G2 B2 | A1 R1 G1 B1 | A0 R0 G0 B0]

        // ── 步骤 2：解包到 16 位，为乘法做准备 ──
        __m128i s_lo = _mm_unpacklo_epi8(s, zero);  // 像素 0,1 → 8× uint16
        __m128i s_hi = _mm_unpackhi_epi8(s, zero);  // 像素 2,3 → 8× uint16
        __m128i d_lo = _mm_unpacklo_epi8(d, zero);
        __m128i d_hi = _mm_unpackhi_epi8(d, zero);

        // ── 步骤 3：提取 Alpha 并广播到同一像素的所有通道 ──
        // 像素 0 的 alpha 在 s_lo 的第 3 个 16 位位置
        // 像素 1 的 alpha 在 s_lo 的第 7 个 16 位位置
        // _MM_SHUFFLE(3,3,3,3) = 将 4 个 16 位槽都填为索引 3 的值
        __m128i sa_lo = _mm_shufflelo_epi16(s_lo, _MM_SHUFFLE(3, 3, 3, 3));
        sa_lo = _mm_shufflehi_epi16(sa_lo, _MM_SHUFFLE(3, 3, 3, 3));
        // 现在 sa_lo = [a1 a1 a1 a1 | a0 a0 a0 a0]（每个通道都是该像素的 alpha）

        __m128i sa_hi = _mm_shufflelo_epi16(s_hi, _MM_SHUFFLE(3, 3, 3, 3));
        sa_hi = _mm_shufflehi_epi16(sa_hi, _MM_SHUFFLE(3, 3, 3, 3));
        // sa_hi = [a3 a3 a3 a3 | a2 a2 a2 a2]

        // ── 步骤 4：计算 inv_alpha = 255 - alpha ──
        __m128i all_ff = _mm_set1_epi16(255);
        __m128i inv_lo = _mm_sub_epi16(all_ff, sa_lo);
        __m128i inv_hi = _mm_sub_epi16(all_ff, sa_hi);

        // ── 步骤 5：混合计算 ──
        // result = (src × alpha + dst × inv_alpha + 128) >> 8
        // 加 128 再右移 8 位是 ÷256 的四舍五入近似

        // src × alpha（16 位乘法，取低 16 位）
        __m128i mul_s_lo = _mm_mullo_epi16(s_lo, sa_lo);
        __m128i mul_s_hi = _mm_mullo_epi16(s_hi, sa_hi);

        // dst × inv_alpha
        __m128i mul_d_lo = _mm_mullo_epi16(d_lo, inv_lo);
        __m128i mul_d_hi = _mm_mullo_epi16(d_hi, inv_hi);

        // 两项相加
        __m128i res_lo = _mm_add_epi16(mul_s_lo, mul_d_lo);
        __m128i res_hi = _mm_add_epi16(mul_s_hi, mul_d_hi);

        // 加四舍五入偏移
        res_lo = _mm_add_epi16(res_lo, round_add);
        res_hi = _mm_add_epi16(res_hi, round_add);

        // 右移 8 位（÷256 近似 ÷255）
        res_lo = _mm_srli_epi16(res_lo, 8);
        res_hi = _mm_srli_epi16(res_hi, 8);

        // ── 步骤 6：打包回 8 位并存储 ──
        __m128i result = _mm_packus_epi16(res_lo, res_hi);

        // 强制 Alpha 通道为 0xFF（不透明输出）
        result = _mm_or_si128(result, alpha_mask);

        _mm_storeu_si128((__m128i*)(dst + i), result);
    }
}
```

### 5.3 执行流程可视化

```
像素数据：  [P3][P2][P1][P0]  ← 4 个 ARGB 像素
                 │
        ┌────────┴────────┐
   unpacklo(低 64 位)   unpackhi(高 64 位)
        │                 │
   [P1 的 8×u16]     [P3 的 8×u16]
   [P0 的 8×u16]     [P2 的 8×u16]
        │                 │
   shuffle 提取α     shuffle 提取α
        │                 │
   ×α, ×(255-α)      ×α, ×(255-α)
   相加, +128, >>8    相加, +128, >>8
        │                 │
        └────────┬────────┘
            packus_epi16
                 │
        [P3][P2][P1][P0]  ← 结果 4 个像素
```

## 6. KrKr2 的 ÷255 精确近似

### 6.1 问题：÷256 vs ÷255

上面的代码用 `>> 8`（等价于 ÷256）近似 `÷ 255`，会引入 1/255 ≈ 0.4% 的误差。对于大多数场景够用，但 KrKr2 追求更高精度：

```cpp
// 标准方式：直接除以 255（CPU 整数除法很慢！）
uint32_t r = (sr * sa + dr * inv_sa) / 255;

// 简单近似：右移 8 位 = ÷256（快但有误差）
// 例：255 * 128 = 32640
//     32640 / 255 = 128（精确）
//     32640 >> 8  = 127（偏差 1）

// KrKr2 的近似公式（来自 tvpgl.h）：
// x / 255 ≈ (x + (x >> 8) + 1) >> 8
// 最大误差 < 0.004，对 8 位颜色来说完全精确
```

### 6.2 公式推导

```
设 x 是需要除以 255 的值（范围 [0, 65025]）

精确值：x / 255

近似过程：
  1/255 ≈ 1/256 + 1/256² + 1/256³ + ...
       = (1 + 1/256 + 1/256² + ...) / 256

  取前两项近似：
  x / 255 ≈ x × (1/256 + 1/65536) = (x + x/256) / 256

  加上修正常数 1 后：
  x / 255 ≈ (x + (x >> 8) + 1) >> 8
```

### 6.3 SSE2 实现

```cpp
// KrKr2 的 ÷255 在 SSE2 中的实现
// 输入：8× uint16 的乘法结果
// 输出：8× uint16 的除以 255 的结果
__m128i div255_sse2(__m128i x) {
    __m128i one = _mm_set1_epi16(1);
    // 步骤 1：t = x + (x >> 8)
    __m128i t = _mm_add_epi16(x, _mm_srli_epi16(x, 8));
    // 步骤 2：(t + 1) >> 8
    return _mm_srli_epi16(_mm_add_epi16(t, one), 8);
}

// 使用方法：替换 Alpha 混合中的 >> 8 步骤
// 原来：res_lo = _mm_srli_epi16(res_lo, 8);
// 改为：res_lo = div255_sse2(res_lo);
```

### 6.4 精度验证

```cpp
// 验证 ÷255 近似公式的精度
#include <cstdio>
#include <cstdint>

int main() {
    int max_error = 0;
    int error_count = 0;

    // 遍历所有可能的输入值（0 到 255×255=65025）
    for (uint32_t x = 0; x <= 65025; ++x) {
        uint32_t exact   = x / 255;                        // 精确值
        uint32_t approx  = (x + (x >> 8) + 1) >> 8;       // 近似值
        uint32_t simple  = x >> 8;                         // 简单近似

        int err_approx = (int)approx - (int)exact;
        int err_simple = (int)simple - (int)exact;

        if (err_approx != 0) error_count++;
        if (err_approx > max_error) max_error = err_approx;
    }

    std::printf("KrKr2 公式: 最大误差=%d, 错误计数=%d/65026\n",
                max_error, error_count);
    // 输出：最大误差=0, 错误计数=0/65026
    // 在 [0, 65025] 范围内完全精确！
    return 0;
}
```

## 7. 处理尾部像素

实际图像宽度不一定是 4 的整数倍。SSE2 一次处理 4 个像素，剩余的 0-3 个像素需要用标量代码处理：

```cpp
// 标量版：处理单个像素
static inline void alpha_blend_one(uint32_t* dst, uint32_t src) {
    uint32_t sa = (src >> 24) & 0xFF;
    if (sa == 0) return;          // 完全透明，跳过
    if (sa == 255) {              // 完全不透明，直接覆盖
        *dst = src;
        return;
    }
    uint32_t inv_sa = 255 - sa;
    uint32_t sr = (src >> 16) & 0xFF, sg = (src >> 8) & 0xFF, sb = src & 0xFF;
    uint32_t dr = (*dst >> 16) & 0xFF, dg = (*dst >> 8) & 0xFF, db = *dst & 0xFF;

    uint32_t or_ = (sr * sa + dr * inv_sa + 128) / 255;
    uint32_t og  = (sg * sa + dg * inv_sa + 128) / 255;
    uint32_t ob  = (sb * sa + db * inv_sa + 128) / 255;

    *dst = (0xFFu << 24) | (or_ << 16) | (og << 8) | ob;
}

// 安全版 SSE2 混合：处理任意数量的像素
void alpha_blend_sse2_safe(uint32_t* dst, const uint32_t* src, int count) {
    // 主循环：每次 4 个像素
    int simd_count = count & ~3;  // 向下取整到 4 的倍数
    //   例：count=1921 → simd_count=1920
    //       count=1920 → simd_count=1920
    //       count=  5  → simd_count=4

    alpha_blend_sse2(dst, src, simd_count);

    // 尾部：用标量处理剩余 0~3 个像素
    for (int i = simd_count; i < count; ++i) {
        alpha_blend_one(&dst[i], src[i]);
    }
}
```

> **性能提示**：尾部标量处理的开销通常可以忽略。对于 1920 宽的图像，尾部为 0（1920 是 4 的倍数）。即使宽度为 1921，尾部也只有 1 个像素，占总量的 0.05%。

## 8. AVX2 扩展：8 像素并行

### 8.1 AVX2 基本用法

AVX2 将寄存器宽度从 128 位扩展到 256 位，使用 `__m256i` 类型和 `_mm256_` 前缀的 intrinsic 函数：

```cpp
#include <immintrin.h>  // AVX2 头文件（同时包含 SSE2）

void alpha_blend_avx2(uint32_t* dst, const uint32_t* src, int count) {
    const __m256i zero      = _mm256_setzero_si256();
    const __m256i round_add = _mm256_set1_epi16(128);
    const __m256i all_ff    = _mm256_set1_epi16(255);
    const __m256i alpha_mask = _mm256_set1_epi32(0xFF000000);

    for (int i = 0; i < count; i += 8) {
        // 一次加载 8 个像素（256 位 = 8 × 32 位）
        __m256i s = _mm256_loadu_si256((__m256i*)(src + i));
        __m256i d = _mm256_loadu_si256((__m256i*)(dst + i));

        // 解包到 16 位
        __m256i s_lo = _mm256_unpacklo_epi8(s, zero);
        __m256i s_hi = _mm256_unpackhi_epi8(s, zero);
        __m256i d_lo = _mm256_unpacklo_epi8(d, zero);
        __m256i d_hi = _mm256_unpackhi_epi8(d, zero);

        // 提取并广播 Alpha
        __m256i sa_lo = _mm256_shufflelo_epi16(s_lo, _MM_SHUFFLE(3,3,3,3));
        sa_lo = _mm256_shufflehi_epi16(sa_lo, _MM_SHUFFLE(3,3,3,3));
        __m256i sa_hi = _mm256_shufflelo_epi16(s_hi, _MM_SHUFFLE(3,3,3,3));
        sa_hi = _mm256_shufflehi_epi16(sa_hi, _MM_SHUFFLE(3,3,3,3));

        // inv_alpha = 255 - alpha
        __m256i inv_lo = _mm256_sub_epi16(all_ff, sa_lo);
        __m256i inv_hi = _mm256_sub_epi16(all_ff, sa_hi);

        // 混合计算：src×α + dst×(1-α) + 128，然后 >> 8
        __m256i res_lo = _mm256_add_epi16(
            _mm256_mullo_epi16(s_lo, sa_lo),
            _mm256_mullo_epi16(d_lo, inv_lo)
        );
        __m256i res_hi = _mm256_add_epi16(
            _mm256_mullo_epi16(s_hi, sa_hi),
            _mm256_mullo_epi16(d_hi, inv_hi)
        );

        res_lo = _mm256_srli_epi16(_mm256_add_epi16(res_lo, round_add), 8);
        res_hi = _mm256_srli_epi16(_mm256_add_epi16(res_hi, round_add), 8);

        // 打包回 8 位并存储
        __m256i result = _mm256_packus_epi16(res_lo, res_hi);
        result = _mm256_or_si256(result, alpha_mask);

        _mm256_storeu_si256((__m256i*)(dst + i), result);
    }
}
```

### 8.2 AVX2 的"lane 陷阱"（重要！）

AVX2 的 256 位寄存器**不是**简单地把两个 128 位拼接在一起。它内部分为两个 **128 位 lane**（车道），许多指令在 lane 内独立操作：

```
AVX2 的 __m256i 内部结构：

  [  高 128 位 lane (lane 1)  |  低 128 位 lane (lane 0)  ]
  [h15 h14 ... h8             | l7  l6  ... l0            ]

_mm256_unpacklo_epi8 的实际行为：
  ❌ 不是：取整个 256 位的低 128 位来解包
  ✅ 而是：分别在 lane 0 和 lane 1 内各取低 64 位来解包

这意味着 AVX2 解包后的像素顺序可能与预期不同！
需要用 _mm256_permute2x128_si256 等跨 lane 指令来重排。
```

这是从 SSE2 代码迁移到 AVX2 时**最常见的 bug 来源**。如果你的 AVX2 Alpha 混合结果像素顺序错乱，几乎一定是 lane 边界问题。

### 实践 1：SSE2 Alpha 混合完整测试程序

```cpp
// 文件：sse2_blend_test.cpp
#include <chrono>
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <vector>
#include <emmintrin.h>

// 标量版（参考实现）
static void blend_scalar(uint32_t* dst, const uint32_t* src, int n) {
    for (int i = 0; i < n; ++i) {
        uint32_t s = src[i], d = dst[i];
        uint32_t sa = (s >> 24) & 0xFF, inv = 255 - sa;
        uint32_t sr = (s>>16)&0xFF, sg = (s>>8)&0xFF, sb = s&0xFF;
        uint32_t dr = (d>>16)&0xFF, dg = (d>>8)&0xFF, db = d&0xFF;
        uint32_t r = (sr*sa + dr*inv) / 255;
        uint32_t g = (sg*sa + dg*inv) / 255;
        uint32_t b = (sb*sa + db*inv) / 255;
        dst[i] = (0xFFu<<24)|(r<<16)|(g<<8)|b;
    }
}

// SSE2 版（上面讲过的完整版）
static void blend_sse2(uint32_t* dst, const uint32_t* src, int n) {
    const __m128i zero = _mm_setzero_si128();
    const __m128i amask = _mm_set1_epi32(0xFF000000);
    const __m128i radd = _mm_set1_epi16(128);

    int i = 0;
    for (; i + 3 < n; i += 4) {
        __m128i s = _mm_loadu_si128((__m128i*)(src+i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst+i));
        __m128i sl = _mm_unpacklo_epi8(s,zero), sh = _mm_unpackhi_epi8(s,zero);
        __m128i dl = _mm_unpacklo_epi8(d,zero), dh = _mm_unpackhi_epi8(d,zero);

        __m128i al = _mm_shufflelo_epi16(sl, 0xFF);
        al = _mm_shufflehi_epi16(al, 0xFF);
        __m128i ah = _mm_shufflelo_epi16(sh, 0xFF);
        ah = _mm_shufflehi_epi16(ah, 0xFF);

        __m128i ff = _mm_set1_epi16(255);
        __m128i il = _mm_sub_epi16(ff,al), ih = _mm_sub_epi16(ff,ah);

        __m128i rl = _mm_add_epi16(_mm_mullo_epi16(sl,al),_mm_mullo_epi16(dl,il));
        __m128i rh = _mm_add_epi16(_mm_mullo_epi16(sh,ah),_mm_mullo_epi16(dh,ih));
        rl = _mm_srli_epi16(_mm_add_epi16(rl,radd),8);
        rh = _mm_srli_epi16(_mm_add_epi16(rh,radd),8);

        __m128i out = _mm_packus_epi16(rl,rh);
        _mm_storeu_si128((__m128i*)(dst+i), _mm_or_si128(out,amask));
    }
    // 尾部标量
    for (; i < n; ++i) {
        uint32_t s = src[i], d = dst[i];
        uint32_t sa = (s>>24)&0xFF, inv = 255-sa;
        uint32_t r = ((s>>16&0xFF)*sa+(d>>16&0xFF)*inv)/255;
        uint32_t g = ((s>>8&0xFF)*sa+(d>>8&0xFF)*inv)/255;
        uint32_t b = ((s&0xFF)*sa+(d&0xFF)*inv)/255;
        dst[i] = (0xFFu<<24)|(r<<16)|(g<<8)|b;
    }
}

int main() {
    const int W = 1920, H = 1080, N = W * H;
    std::vector<uint32_t> src(N), d1(N), d2(N);

    // 填充测试数据
    for (int i = 0; i < N; ++i) {
        src[i] = 0x80336699u;   // 半透明蓝
        d1[i] = d2[i] = 0xFFEEDDCCu;  // 不透明暖色背景
    }

    const int ITERS = 100;

    // 基准测试：标量
    auto t0 = std::chrono::high_resolution_clock::now();
    for (int r = 0; r < ITERS; ++r) blend_scalar(d1.data(), src.data(), N);
    auto t1 = std::chrono::high_resolution_clock::now();

    // 基准测试：SSE2
    for (int r = 0; r < ITERS; ++r) blend_sse2(d2.data(), src.data(), N);
    auto t2 = std::chrono::high_resolution_clock::now();

    double ms_scalar = std::chrono::duration<double,std::milli>(t1-t0).count();
    double ms_sse2   = std::chrono::duration<double,std::milli>(t2-t1).count();

    std::printf("标量: %.1f ms (%.1f M像素/秒)\n", ms_scalar,
                (double)N*ITERS/ms_scalar*1000.0/1e6);
    std::printf("SSE2: %.1f ms (%.1f M像素/秒)\n", ms_sse2,
                (double)N*ITERS/ms_sse2*1000.0/1e6);
    std::printf("加速比: %.2fx\n", ms_scalar / ms_sse2);

    // 验证正确性：比较两个结果
    int diff_count = 0;
    for (int i = 0; i < N; ++i) {
        if (d1[i] != d2[i]) {
            if (diff_count < 5) {
                std::printf("差异 @%d: 标量=0x%08X SSE2=0x%08X\n", i, d1[i], d2[i]);
            }
            diff_count++;
        }
    }
    std::printf("像素差异数: %d/%d (÷256 vs ÷255 的舍入差异是正常的)\n", diff_count, N);

    return 0;
}
```

## 常见错误及解决方案

### 错误 1：未对齐访问导致段错误（Segmentation Fault）

**现象**：程序在 `_mm_load_si128` 处崩溃，错误信息为 SIGSEGV 或 Access Violation。

**原因**：`_mm_load_si128` 要求指针地址是 16 的整数倍，但传入的 `malloc` 分配的内存可能没有 16 字节对齐。

**解决**：

```cpp
// ❌ 危险：malloc 不保证 16 字节对齐
uint32_t* buf = (uint32_t*)malloc(size);
__m128i data = _mm_load_si128((__m128i*)buf);  // 可能崩溃！

// ✅ 方案 A：使用非对齐加载（简单安全）
__m128i data = _mm_loadu_si128((__m128i*)buf);

// ✅ 方案 B：分配对齐内存（性能最优）
uint32_t* buf = (uint32_t*)_aligned_malloc(size, 16);  // Windows
```

### 错误 2：忘记处理尾部像素

**现象**：图像最右侧几列像素颜色异常或程序越界崩溃。

**原因**：SSE2 每次处理 4 个像素，如果图像宽度不是 4 的倍数，最后一批会读取越界内存。

**解决**：总是先计算 SIMD 可处理的像素数 `count & ~3`，剩余用标量循环处理。参见本节第 7 节。

### 错误 3：÷256 精度不足导致颜色偏差

**现象**：SSE2 混合结果与标量参考实现差 1（某些像素的某个通道值差 1）。

**原因**：`>> 8` 是 ÷256，而正确结果应该是 ÷255。当 src×α 的乘积恰好落在 256 和 255 的舍入分界点时就会出现差异。

**解决**：使用 KrKr2 的 `div255_sse2` 公式替代简单的 `>> 8`。参见本节第 6 节。

## 本节小结

| 概念 | 要点 |
|------|------|
| SSE2 数据类型 | `__m128i`（128 位整数向量），可解释为 16×u8 / 8×u16 / 4×u32 |
| 加载/存储 | `_mm_loadu_si128`（安全） / `_mm_load_si128`（需对齐） |
| 像素解包 | `_mm_unpacklo/hi_epi8(pixels, zero)` 将 4 像素拆为 2 组 16 位 |
| 像素打包 | `_mm_packus_epi16(lo, hi)` 将 16 位结果饱和回 8 位 |
| Alpha 混合流程 | 加载 → 解包 → 提取α → 乘法 → 加法 → 打包 → 存储 |
| ÷255 近似 | `(x + (x >> 8) + 1) >> 8`，KrKr2 使用，完全精确 |
| 尾部处理 | `count & ~3` 取 SIMD 可处理数，剩余标量处理 |
| AVX2 lane | 256 位寄存器分两个 128 位 lane，解包等操作在 lane 内独立 |

## 练习题与答案

### 题目 1：SSE2 指令选择

**问题**：以下场景分别应该用哪条 SSE2 intrinsic？

1. 将两层图像的像素颜色值相加，超过 255 的截断为 255
2. 将 `__m128i` 中的 16 个字节（uint8）解包为 8 个 uint16，只取低 8 字节
3. 将 8 个 uint16 的计算结果打包回 16 个 uint8，超出 [0,255] 的饱和截断

<details>
<summary>查看答案</summary>

1. **`_mm_adds_epu8(a, b)`** — 无符号字节饱和加法。`adds` = add saturated，`epu8` = unsigned packed 8-bit。超过 255 自动截断到 255。

2. **`_mm_unpacklo_epi8(a, _mm_setzero_si128())`** — 将 `a` 的低 8 字节与零交错排列。每个 uint8 前面插入一个 0 字节，变成 uint16。例如字节 `[R0]` 变成 `[00 R0]`。

3. **`_mm_packus_epi16(lo, hi)`** — 将两个 `__m128i`（各含 8 个 int16）打包为一个 `__m128i`（16 个 uint8），带无符号饱和。`packus` = pack unsigned saturated。负值截断到 0，超过 255 截断到 255。

</details>

### 题目 2：Alpha 广播原理

**问题**：在 SSE2 Alpha 混合中，为什么需要使用 `_mm_shufflelo_epi16` 和 `_mm_shufflehi_epi16` 来"广播" Alpha 值？能否用更简单的方法？

<details>
<summary>查看答案</summary>

**为什么需要广播**：混合公式要求每个颜色通道（R、G、B）都乘以**同一个** Alpha 值。但解包后 Alpha 只在特定位置（每 4 个 uint16 中的最后一个），R、G、B 通道需要访问的 Alpha 值分散在不同位置。

例如解包后像素 0 的数据是 `[A0, R0, G0, B0]`（4 个 uint16），乘法时需要：
- `R0 × A0`
- `G0 × A0`
- `B0 × A0`

如果不广播，就需要逐元素提取 Alpha 再分别乘，效率很低。广播后变成 `[A0, A0, A0, A0]`，一条 `_mm_mullo_epi16` 就能同时完成所有通道的乘法。

**替代方法**：SSSE3 的 `_mm_shuffle_epi8`（PSHUFB）可以用单条指令完成任意字节重排，比 `shufflelo` + `shufflehi` 两条指令更高效。但 SSSE3 不是所有 CPU 都支持（SSE2 是基线），所以 KrKr2 在 SSE2 路径中使用两步 shuffle。

</details>

### 题目 3：修改混合公式

**问题**：上面实现的是 `result = src × α + dst × (1-α)` 的标准 Alpha 混合。如果要实现**预乘 Alpha**（Premultiplied Alpha）的混合公式 `result = src + dst × (1-α)`（源像素已预乘，不再乘 α），SSE2 代码需要修改哪些步骤？

<details>
<summary>查看答案</summary>

只需修改步骤 5 中的乘法部分：

```cpp
// 原来（非预乘）：
__m128i mul_s_lo = _mm_mullo_epi16(s_lo, sa_lo);  // src × α
__m128i mul_d_lo = _mm_mullo_epi16(d_lo, inv_lo);  // dst × (1-α)
__m128i res_lo = _mm_add_epi16(mul_s_lo, mul_d_lo);

// 改为（预乘）：
// src 已经预乘过 α，直接用原值，不再乘 α
// 但 src 的值范围仍是 [0,255]，需要左移 8 位对齐乘法结果的量级
__m128i mul_d_lo = _mm_mullo_epi16(d_lo, inv_lo);  // dst × (1-α)
__m128i res_lo = _mm_add_epi16(
    _mm_slli_epi16(s_lo, 8),  // src << 8（对齐量级）
    mul_d_lo
);
// 后续的 +128 >> 8 不变
```

核心变化：去掉了 `src × α` 的乘法，因为预乘格式的源像素已经包含了 Alpha 信息。这减少了一半的乘法运算，是预乘 Alpha 的性能优势之一。

</details>
