## 1. 什么是饱和运算

### 1.1 回绕 vs 饱和

在 C/C++ 中，`uint8_t` 的取值范围是 [0, 255]。当运算结果超出这个范围时，有两种处理方式：

- **回绕（Wrapping）**：结果对 256 取模。`200 + 100 = 300 → 300 % 256 = 44`
- **饱和（Saturating）**：结果钳制到边界。`200 + 100 = 300 → min(300, 255) = 255`

```cpp
#include <cstdint>
#include <cstdio>

int main() {
    uint8_t a = 200, b = 100;

    // C/C++ 默认行为：回绕
    uint8_t wrap = a + b;  // = 44 （回绕到 0-255 之间）

    // 手动饱和
    int sum = a + b;
    uint8_t sat = (sum > 255) ? 255 : (uint8_t)sum;  // = 255

    printf("回绕: %d + %d = %u\n", a, b, wrap);    // 输出: 回绕: 200 + 100 = 44
    printf("饱和: %d + %d = %u\n", a, b, sat);      // 输出: 饱和: 200 + 100 = 255
    return 0;
}
```

### 1.2 像素处理为什么需要饱和

在图像混合中，**加法混合**（Additive Blend，将两个像素的颜色值直接相加）用于实现光照叠加、火焰、爆炸等发光效果。如果使用回绕加法：

```
前景红色通道 = 200
背景红色通道 = 100
回绕结果 = 44  ← 本应更亮，结果反而变暗了！
饱和结果 = 255 ← 正确，亮度饱和到最大值
```

回绕会导致**颜色突变**——本应越来越亮的效果突然变暗，在游戏中表现为闪烁的色块。KrKr2 的视觉小说引擎需要大量光效叠加（场景过渡、光晕、粒子效果），所以饱和运算至关重要。

同理，**减法混合**（Subtractive Blend，从背景中减去前景的颜色值）用于阴影、暗角等效果：

```
背景红色通道 = 50
前景红色通道 = 100
回绕结果 = 50 - 100 = -50 → 对 uint8 回绕 = 206  ← 本应更暗，结果变亮了！
饱和结果 = max(50 - 100, 0) = 0  ← 正确，暗到纯黑
```

## 2. 标量饱和加法：KrKr2 的位运算技巧

### 2.1 朴素实现（有分支）

最直白的饱和加法需要对每个通道做条件判断：

```cpp
// 朴素版：4 次条件分支
uint32_t saturated_add_naive(uint32_t a, uint32_t b) {
    // ARGB 像素，每个通道 8 位
    uint8_t a_b = a & 0xFF, b_b = b & 0xFF;                // Blue
    uint8_t a_g = (a >> 8) & 0xFF, b_g = (b >> 8) & 0xFF;  // Green
    uint8_t a_r = (a >> 16) & 0xFF, b_r = (b >> 16) & 0xFF; // Red
    uint8_t a_a = (a >> 24) & 0xFF, b_a = (b >> 24) & 0xFF; // Alpha

    int rb = a_b + b_b; if (rb > 255) rb = 255;  // 分支 1
    int rg = a_g + b_g; if (rg > 255) rg = 255;  // 分支 2
    int rr = a_r + b_r; if (rr > 255) rr = 255;  // 分支 3
    int ra = a_a + b_a; if (ra > 255) ra = 255;  // 分支 4

    return (ra << 24) | (rr << 16) | (rg << 8) | rb;
}
```

这段代码正确但有两个性能问题：
1. **4 次条件分支**：CPU 的分支预测器（Branch Predictor，一种硬件机制用于猜测 if/else 走哪条路径）可能频繁预测失败，因为每个像素的溢出情况不同
2. **4 次拆包 + 1 次组装**：每个通道单独处理然后重新拼装

### 2.2 KrKr2 的无分支位运算

KrKr2 在 `tvpgl.h` / `tvpgl.cpp` 中使用了一个巧妙的位运算公式，**零分支**完成 4 通道并行饱和加法：

```cpp
// KrKr2 的 TVPSaturatedAdd 位运算核心
inline uint32_t TVPSaturatedAdd(uint32_t a, uint32_t b) {
    // 步骤 1：检测每个通道是否溢出
    uint32_t tmp = (((a & b) + (((a ^ b) >> 1) & 0x7F7F7F7F)) & 0x80808080);

    // 步骤 2：将溢出标志转换为饱和掩码
    tmp = (tmp << 1) - (tmp >> 7);

    // 步骤 3：应用掩码
    return (a + b - tmp) | tmp;
}
```

这段代码令人困惑，让我们逐步拆解：

#### 步骤 1：检测溢出

```cpp
uint32_t tmp = (((a & b) + (((a ^ b) >> 1) & 0x7F7F7F7F)) & 0x80808080);
```

这实际上是在**同时判断 4 个 8 位通道的加法是否溢出**。核心原理：

- `a & b`：两个数都为 1 的位——这些位的"进位"一定会产生
- `a ^ b`：两个数不同的位——这些位可能产生进位（取决于低位进位）
- `(a ^ b) >> 1`：将不同位右移——模拟进位传播
- `& 0x7F7F7F7F`：清除每个字节的最高位——防止跨字节进位污染
- 最终 `& 0x80808080`：只保留每个字节的最高位——如果该位为 1，表示该通道溢出了

```
假设某通道 a=200(0xC8), b=100(0x64):
  a & b       = 0x40        (0100 0000)
  a ^ b       = 0xAC        (1010 1100)
  (a^b)>>1    = 0x56        (0101 0110)
  &0x7F       = 0x56        (0101 0110)  不变（最高位已经是 0）
  0x40 + 0x56 = 0x96        (1001 0110)
  & 0x80      = 0x80        ← 最高位为 1，表示溢出！

假设某通道 a=50(0x32), b=100(0x64):
  a & b       = 0x20        (0010 0000)
  a ^ b       = 0x56        (0101 0110)
  (a^b)>>1    = 0x2B        (0010 1011)
  &0x7F       = 0x2B        (0010 1011)
  0x20 + 0x2B = 0x4B        (0100 1011)
  & 0x80      = 0x00        ← 最高位为 0，未溢出
```

#### 步骤 2：生成饱和掩码

```cpp
tmp = (tmp << 1) - (tmp >> 7);
```

如果某通道溢出（`tmp` 中该字节的最高位 = 1，即 `0x80`）：
- `0x80 << 1 = 0x00`（溢出到上一字节，本字节清零——但这里的运算恰好利用了这个溢出）
- `0x80 >> 7 = 0x01`
- `0x00 - 0x01 = 0xFF`（对 uint8 回绕）→ 整个字节变成全 1

如果某通道未溢出（`0x00`）：
- `0x00 << 1 = 0x00`
- `0x00 >> 7 = 0x00`
- `0x00 - 0x00 = 0x00` → 整个字节保持全 0

所以 `tmp` 现在是：溢出的通道 = `0xFF`，未溢出的通道 = `0x00`。

#### 步骤 3：应用掩码

```cpp
return (a + b - tmp) | tmp;
```

- 溢出通道：`(a + b - 0xFF) | 0xFF = 任意值 | 0xFF = 0xFF = 255`  ✅ 饱和
- 未溢出通道：`(a + b - 0x00) | 0x00 = a + b`  ✅ 正常加法

> **精妙之处**：整个过程没有任何 `if`，对 32 位整数（4 个通道）同时操作，等效于 4 路并行饱和加法。这种技巧在 SIMD 普及之前（2000 年代初）是像素处理的标准优化方式。

### 2.3 完整标量饱和加法混合

```cpp
#include <cstdint>

// 标量饱和加法——无分支，4 通道并行
inline uint32_t saturated_add(uint32_t a, uint32_t b) {
    uint32_t tmp = (((a & b) + (((a ^ b) >> 1) & 0x7F7F7F7F)) & 0x80808080);
    tmp = (tmp << 1) - (tmp >> 7);
    return (a + b - tmp) | tmp;
}

// 加法混合函数
void blend_add_scalar(uint32_t* dst, const uint32_t* src, int count) {
    for (int i = 0; i < count; ++i) {
        dst[i] = saturated_add(dst[i], src[i]);
    }
}

// 验证程序
#include <cstdio>

int main() {
    // 测试用例
    uint32_t a = 0xFF'C8'64'32;  // A=255, R=200, G=100, B=50
    uint32_t b = 0x80'64'C8'E0;  // A=128, R=100, G=200, B=224

    uint32_t result = saturated_add(a, b);

    // 预期：A=min(255+128,255)=255, R=min(200+100,255)=255,
    //       G=min(100+200,255)=255, B=min(50+224,255)=255
    // 全部溢出，结果应为 0xFFFFFFFF
    printf("0x%08X + 0x%08X = 0x%08X\n", a, b, result);
    // 输出: 0xFFC86432 + 0x8064C8E0 = 0xFFFFFFFF

    // 不溢出的测试
    uint32_t c = 0xFF'10'20'30;
    uint32_t d = 0x80'05'0A'0F;
    uint32_t r2 = saturated_add(c, d);
    printf("0x%08X + 0x%08X = 0x%08X\n", c, d, r2);
    // 预期: A=255, R=21, G=42, B=63 → 0xFF152A3F
    return 0;
}
```

## 3. SIMD 饱和运算——一条指令

### 3.1 SSE2 饱和加法与减法

标量版本的位运算技巧虽然精妙，但 SSE2 提供了**硬件级饱和运算指令**——一条指令同时处理 16 个字节（4 个像素的 16 个通道）：

```cpp
#include <emmintrin.h>  // SSE2
#include <cstdint>
#include <cstdio>

// SSE2 饱和加法混合 —— 一条核心指令
void blend_add_sse2(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;  // 4 像素对齐

    for (int i = 0; i < simd_end; i += 4) {
        // 加载 4 像素（16 字节）
        __m128i s = _mm_loadu_si128((__m128i*)(src + i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst + i));

        // _mm_adds_epu8: 16 字节无符号饱和加法
        // adds = add saturated, epu8 = unsigned packed 8-bit
        // 每个字节独立做 min(a+b, 255)
        __m128i result = _mm_adds_epu8(s, d);

        // 存储结果
        _mm_storeu_si128((__m128i*)(dst + i), result);
    }

    // 尾部标量
    for (int i = simd_end; i < count; ++i) {
        dst[i] = saturated_add(dst[i], src[i]);
    }
}

// SSE2 饱和减法混合 —— 用于阴影/暗角效果
void blend_sub_sse2(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;

    for (int i = 0; i < simd_end; i += 4) {
        __m128i s = _mm_loadu_si128((__m128i*)(src + i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst + i));

        // _mm_subs_epu8: 16 字节无符号饱和减法
        // 每个字节独立做 max(d-s, 0)
        __m128i result = _mm_subs_epu8(d, s);

        _mm_storeu_si128((__m128i*)(dst + i), result);
    }

    // 尾部标量
    for (int i = simd_end; i < count; ++i) {
        uint8_t* dp = (uint8_t*)(dst + i);
        const uint8_t* sp = (const uint8_t*)(src + i);
        for (int c = 0; c < 4; ++c) {
            int diff = dp[c] - sp[c];
            dp[c] = (diff < 0) ? 0 : (uint8_t)diff;
        }
    }
}
```

> **对比标量**：`TVPSaturatedAdd` 的位运算需要 ~8 条指令处理 1 个像素（4 通道）。`_mm_adds_epu8` 用 **1 条指令**处理 4 个像素（16 通道）。性能差距约为 **8×4 = 32 倍**理论加速比（实际受限于内存带宽，通常为 4-8 倍）。

### 3.2 NEON 饱和加法与减法

NEON 同样提供硬件饱和运算，命名中的 `q` 前缀表示 "saturating"（饱和的）：

```cpp
#include <arm_neon.h>
#include <cstdint>

// NEON 饱和加法混合
void blend_add_neon(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;

    for (int i = 0; i < simd_end; i += 4) {
        // 按字节加载 16 字节
        uint8x16_t s = vld1q_u8((const uint8_t*)(src + i));
        uint8x16_t d = vld1q_u8((const uint8_t*)(dst + i));

        // vqaddq_u8: 16 字节无符号饱和加法
        // vq = vector saturating, add = 加法, q = 128 位, u8 = unsigned 8-bit
        uint8x16_t result = vqaddq_u8(s, d);

        vst1q_u8((uint8_t*)(dst + i), result);
    }

    // 尾部标量
    for (int i = simd_end; i < count; ++i) {
        dst[i] = saturated_add(dst[i], src[i]);
    }
}

// NEON 饱和减法混合
void blend_sub_neon(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;

    for (int i = 0; i < simd_end; i += 4) {
        uint8x16_t s = vld1q_u8((const uint8_t*)(src + i));
        uint8x16_t d = vld1q_u8((const uint8_t*)(dst + i));

        // vqsubq_u8: 饱和减法，每字节 max(d-s, 0)
        uint8x16_t result = vqsubq_u8(d, s);

        vst1q_u8((uint8_t*)(dst + i), result);
    }

    // 尾部标量（同上）
    for (int i = simd_end; i < count; ++i) {
        uint8_t* dp = (uint8_t*)(dst + i);
        const uint8_t* sp = (const uint8_t*)(src + i);
        for (int c = 0; c < 4; ++c) {
            int diff = dp[c] - sp[c];
            dp[c] = (diff < 0) ? 0 : (uint8_t)diff;
        }
    }
}
```

### 3.3 SSE2 vs NEON 饱和指令对照

| 操作 | SSE2 | NEON | 说明 |
|------|------|------|------|
| 无符号字节饱和加法 | `_mm_adds_epu8(a, b)` | `vqaddq_u8(a, b)` | min(a+b, 255) |
| 无符号字节饱和减法 | `_mm_subs_epu8(a, b)` | `vqsubq_u8(a, b)` | max(a-b, 0) |
| 有符号字节饱和加法 | `_mm_adds_epi8(a, b)` | `vqaddq_s8(a, b)` | 钳制到 [-128, 127] |
| 有符号 16 位饱和加法 | `_mm_adds_epi16(a, b)` | `vqaddq_s16(a, b)` | 钳制到 [-32768, 32767] |
| 无符号 16 位饱和减法 | `_mm_subs_epu16(a, b)` | `vqsubq_u16(a, b)` | max(a-b, 0)，16 位版本 |

> **命名规律**：SSE2 中 `adds`/`subs` 的 `s` 后缀表示 saturated；NEON 中 `q` 前缀（如 `vqadd`）表示 saturating。这个命名差异是 SSE2 和 NEON 代码对照时的常见混淆点。

## 4. 完整跨平台饱和混合函数

把前面的知识整合成一个完整的、可编译运行的跨平台示例：

```cpp
// ── saturated_blend.cpp —— 完整跨平台饱和混合 ──
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <chrono>
#include <vector>

// ── 标量实现 ──
static inline uint32_t sat_add(uint32_t a, uint32_t b) {
    uint32_t tmp = (((a & b) + (((a ^ b) >> 1) & 0x7F7F7F7F)) & 0x80808080);
    tmp = (tmp << 1) - (tmp >> 7);
    return (a + b - tmp) | tmp;
}

static void blend_add_c(uint32_t* dst, const uint32_t* src, int count) {
    for (int i = 0; i < count; ++i)
        dst[i] = sat_add(dst[i], src[i]);
}

// ── SSE2 实现 ──
#if defined(__x86_64__) || defined(_M_X64) || defined(__i386__) || defined(_M_IX86)
#include <emmintrin.h>
static void blend_add_sse2(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;
    for (int i = 0; i < simd_end; i += 4) {
        __m128i s = _mm_loadu_si128((__m128i*)(src + i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst + i));
        _mm_storeu_si128((__m128i*)(dst + i), _mm_adds_epu8(s, d));
    }
    for (int i = simd_end; i < count; ++i)
        dst[i] = sat_add(dst[i], src[i]);
}
#endif

// ── NEON 实现 ──
#if defined(__aarch64__) || defined(__ARM_NEON__) || defined(__ARM_NEON)
#include <arm_neon.h>
static void blend_add_neon(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;
    for (int i = 0; i < simd_end; i += 4) {
        uint8x16_t s = vld1q_u8((const uint8_t*)(src + i));
        uint8x16_t d = vld1q_u8((const uint8_t*)(dst + i));
        vst1q_u8((uint8_t*)(dst + i), vqaddq_u8(s, d));
    }
    for (int i = simd_end; i < count; ++i)
        dst[i] = sat_add(dst[i], src[i]);
}
#endif

// ── 函数指针分发 ──
typedef void (*BlendAddFunc)(uint32_t*, const uint32_t*, int);
BlendAddFunc g_blend_add = nullptr;

void init_blend_add() {
#if defined(__aarch64__) || defined(__ARM_NEON__)
    g_blend_add = blend_add_neon;
    printf("[Dispatch] blend_add → NEON\n");
#elif defined(__x86_64__) || defined(_M_X64)
    g_blend_add = blend_add_sse2;
    printf("[Dispatch] blend_add → SSE2\n");
#else
    g_blend_add = blend_add_c;
    printf("[Dispatch] blend_add → Scalar C\n");
#endif
}

// ── 基准测试 ──
void benchmark(const char* name, BlendAddFunc func, int w, int h, int iters) {
    std::vector<uint32_t> src(w * h, 0x80FF8040);
    std::vector<uint32_t> dst(w * h, 0xFFCCBBAA);

    // 预热
    func(dst.data(), src.data(), w * h);

    auto t0 = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < iters; ++i) {
        func(dst.data(), src.data(), w * h);
    }
    auto t1 = std::chrono::high_resolution_clock::now();

    double ms = std::chrono::duration<double, std::milli>(t1 - t0).count();
    double mpix = (double)w * h * iters / ms / 1000.0;
    printf("%-12s: %.2f ms (%d iters), %.1f M 像素/秒\n", name, ms, iters, mpix);
}

int main() {
    init_blend_add();

    const int W = 1920, H = 1080, N = 200;
    printf("基准测试：%d × %d，%d 次迭代\n\n", W, H, N);

    benchmark("标量 C", blend_add_c, W, H, N);

#if defined(__x86_64__) || defined(_M_X64)
    benchmark("SSE2", blend_add_sse2, W, H, N);
#elif defined(__aarch64__)
    benchmark("NEON", blend_add_neon, W, H, N);
#endif

    return 0;
}
```

### 5.2 查找表 vs SIMD：两种优化策略

KrKr2 并不是所有地方都用 SIMD。对于**简单的逐像素运算**（加法、减法、Alpha 混合），SIMD 指令效率最高。但对于**复杂的 Photoshop 混合模式**（如 SoftLight、ColorDodge），公式复杂且包含除法，SIMD 实现困难。KrKr2 对这类模式使用**查找表（Look-Up Table, LUT）**：

```cpp
// tvpgl.cpp 中的查找表（简化）
// TVPDivTable: 256×256 的除法表，将整数除法转为查表
// 大小: 256 × 256 = 65536 字节 = 64KB
extern uint8_t TVPDivTable[256 * 256];

// 使用示例：a / b 转为 TVPDivTable[a * 256 + b]
// 避免了昂贵的整数除法指令

// TVPOpacityOnOpacityTable: 不透明度叠加表
// 用于计算两层半透明叠加后的结果不透明度
extern uint8_t TVPOpacityOnOpacityTable[256 * 256];

// TVPPsTableSoftLight: SoftLight 混合模式查找表
// Photoshop 的 SoftLight 公式太复杂，用 LUT 预计算
extern uint8_t TVPPsTableSoftLight[256 * 256];
```

| 优化策略 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| SIMD 指令 | 简单算术（加减乘、位操作、饱和运算） | 吞吐量极高（4-16 像素/指令） | 复杂公式难以向量化 |
| 查找表 (LUT) | 复杂公式（除法、非线性函数） | 实现简单、结果精确 | 占用内存（64KB 起）、可能 cache miss |
| 标量位运算 | 无 SIMD 支持的平台 | 无依赖、可移植 | 吞吐量最低 |

## 6. ARGB → 灰度转换实践

作为本章 SIMD 学习的综合练习，来实现一个常用操作——将 ARGB 像素转为灰度值。

### 6.1 灰度公式

标准灰度转换公式（ITU-R BT.601，一种国际电信联盟定义的标准）：

```
Gray = 0.299 × R + 0.587 × G + 0.114 × B
```

整数近似（避免浮点运算）：

```
Gray = (77 × R + 150 × G + 29 × B) >> 8
```

其中 77/256 ≈ 0.301, 150/256 ≈ 0.586, 29/256 ≈ 0.113，误差 < 0.5%。

### 6.2 标量实现

```cpp
void grayscale_scalar(uint32_t* pixels, int count) {
    for (int i = 0; i < count; ++i) {
        uint32_t p = pixels[i];
        uint32_t a = (p >> 24) & 0xFF;
        uint32_t r = (p >> 16) & 0xFF;
        uint32_t g = (p >>  8) & 0xFF;
        uint32_t b = p & 0xFF;

        uint32_t gray = (77 * r + 150 * g + 29 * b) >> 8;
        pixels[i] = (a << 24) | (gray << 16) | (gray << 8) | gray;
    }
}
```

### 6.3 SSE2 实现

```cpp
#if defined(__x86_64__) || defined(_M_X64)
#include <emmintrin.h>

void grayscale_sse2(uint32_t* pixels, int count) {
    // 灰度系数（16 位）
    __m128i coeff_r = _mm_set1_epi16(77);   // 0.299 × 256
    __m128i coeff_g = _mm_set1_epi16(150);  // 0.587 × 256
    __m128i coeff_b = _mm_set1_epi16(29);   // 0.114 × 256
    __m128i zero = _mm_setzero_si128();

    int simd_end = count & ~3;

    for (int i = 0; i < simd_end; i += 4) {
        __m128i px = _mm_loadu_si128((__m128i*)(pixels + i));

        // 解包到 16 位：[B0 G0 R0 A0 B1 G1 R1 A1]
        __m128i lo = _mm_unpacklo_epi8(px, zero);  // 前 2 像素
        __m128i hi = _mm_unpackhi_epi8(px, zero);  // 后 2 像素

        // 提取 R/G/B 通道（ARGB 内存布局: B G R A）
        // lo = [B0 G0 R0 A0 B1 G1 R1 A1]，各占 16 位
        // 需要分别乘以系数再累加

        // 简化方案：逐像素处理（每个 __m128i 4 像素）
        // 更高效的方法是使用 SSSE3 的 _mm_maddubs_epi16
        // 但 SSE2 没有这个指令，这里用手动方式

        // 像素 0-1 的 B/G/R 通道
        __m128i b_lo = _mm_and_si128(lo, _mm_set1_epi32(0x00FF));       // B
        __m128i g_lo = _mm_and_si128(_mm_srli_epi32(lo, 16), _mm_set1_epi32(0x00FF)); // G
        // 对于 BGRA 内存序，R 在 32 位字的第 16-23 位
        // 但解包后 16 位通道排列不同，需要调整

        // 实际上更简单的做法：直接在 32 位层面操作
        // 这里给出简化的逐像素版本
        for (int j = 0; j < 4 && (i + j) < count; ++j) {
            uint32_t p = pixels[i + j];
            uint32_t a = (p >> 24) & 0xFF;
            uint32_t r = (p >> 16) & 0xFF;
            uint32_t g = (p >>  8) & 0xFF;
            uint32_t b = p & 0xFF;
            uint32_t gray = (77 * r + 150 * g + 29 * b) >> 8;
            pixels[i + j] = (a << 24) | (gray << 16) | (gray << 8) | gray;
        }
    }
    // 注意：上面的 SSE2 灰度转换因为通道提取较复杂，
    // 真正高效的实现需要 SSSE3 的 _mm_maddubs_epi16（乘加指令）
    // 或者手动解交错每个通道。这是留给读者的进阶练习。

    for (int i = simd_end; i < count; ++i) {
        uint32_t p = pixels[i];
        uint32_t a = (p >> 24) & 0xFF;
        uint32_t r = (p >> 16) & 0xFF;
        uint32_t g = (p >>  8) & 0xFF;
        uint32_t b = p & 0xFF;
        uint32_t gray = (77 * r + 150 * g + 29 * b) >> 8;
        pixels[i] = (a << 24) | (gray << 16) | (gray << 8) | gray;
    }
}
#endif
```

> **为什么灰度转换比饱和加法复杂？** 饱和加法对所有通道做相同操作（加法），可以把像素当作纯字节流处理。灰度转换需要**分离每个通道**、用**不同系数**相乘、再**累加**回去——这需要更多的数据重排指令。这就是 SSSE3 的 `_mm_maddubs_epi16`（Multiply and ADD Unsigned Bytes to Signed 16-bit，无符号字节与有符号字节逐对相乘再两两累加到 16 位）被频繁使用的原因。

### 练习 1：验证位运算饱和加法

编写一个测试程序，对所有可能的 `uint8_t` 组合 (0-255 × 0-255 = 65536 种) 验证 `TVPSaturatedAdd` 的位运算结果与朴素 `min(a+b, 255)` 完全一致：

```cpp
#include <cstdint>
#include <cstdio>

// 位运算版本（单字节测试需要把字节放到 32 位里）
uint8_t sat_add_bitwise(uint8_t a, uint8_t b) {
    uint32_t a32 = a, b32 = b;
    uint32_t tmp = (((a32 & b32) + (((a32 ^ b32) >> 1) & 0x7F)) & 0x80);
    tmp = (tmp << 1) - (tmp >> 7);
    return (uint8_t)((a32 + b32 - tmp) | tmp);
}

int main() {
    int errors = 0;
    for (int a = 0; a < 256; ++a) {
        for (int b = 0; b < 256; ++b) {
            uint8_t expected = (a + b > 255) ? 255 : (uint8_t)(a + b);
            uint8_t actual = sat_add_bitwise((uint8_t)a, (uint8_t)b);
            if (actual != expected) {
                printf("错误: %d + %d = %d (应为 %d)\n", a, b, actual, expected);
                ++errors;
            }
        }
    }
    printf("测试完成: %d 错误 / 65536 组合\n", errors);
    return 0;
}
```

### 练习 2：实现 SSE2 取最大值混合

使用 `_mm_max_epu8`（SSE2 无符号字节取最大值指令）实现一个"Lighten"（变亮）混合模式——对每个通道取两个像素中的较大值：

```cpp
// 目标: dst[i] = max(dst[i], src[i])  对每个字节通道
void blend_lighten_sse2(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;
    for (int i = 0; i < simd_end; i += 4) {
        __m128i s = _mm_loadu_si128((__m128i*)(src + i));
        __m128i d = _mm_loadu_si128((__m128i*)(dst + i));
        __m128i result = _mm_max_epu8(s, d);  // 逐字节取最大值
        _mm_storeu_si128((__m128i*)(dst + i), result);
    }
    for (int i = simd_end; i < count; ++i) {
        uint8_t* dp = (uint8_t*)(dst + i);
        const uint8_t* sp = (const uint8_t*)(src + i);
        for (int c = 0; c < 4; ++c)
            dp[c] = (sp[c] > dp[c]) ? sp[c] : dp[c];
    }
}
```

## 常见错误及解决方案

### 错误 1：混淆回绕加法和饱和加法

```cpp
// ❌ 使用 _mm_add_epi8 做加法混合——结果会回绕！
__m128i result = _mm_add_epi8(s, d);  // 200 + 100 = 44，颜色突变

// ✅ 使用 _mm_adds_epu8 做加法混合——结果饱和
__m128i result = _mm_adds_epu8(s, d);  // 200 + 100 = 255，正确
```

**区分记忆**：`add` = 回绕，`adds` = 饱和（s = saturated）。像素混合几乎总是用 `adds`/`subs`。

### 错误 2：饱和减法方向弄反

```cpp
// ❌ src - dst（错误：应该是从背景减去前景）
__m128i result = _mm_subs_epu8(s, d);  // 从前景减背景，结果意义不对

// ✅ dst - src（正确：从背景中减去前景的颜色贡献）
__m128i result = _mm_subs_epu8(d, s);  // 背景减前景，阴影效果
```

**记忆口诀**：减法混合 = "背景被前景削暗"，所以是 `dst - src`。

### 错误 3：忘记尾部处理

```cpp
// ❌ 不处理非 4 倍数的尾部像素
void blend_add(uint32_t* dst, const uint32_t* src, int count) {
    for (int i = 0; i < count; i += 4) {  // 如果 count=1023，最后 3 像素被跳过！
        // SSE2 操作...
    }
}

// ✅ 先 SIMD 处理对齐部分，再标量处理尾部
void blend_add(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;  // 向下对齐到 4 的倍数
    for (int i = 0; i < simd_end; i += 4) {
        // SSE2 操作...
    }
    for (int i = simd_end; i < count; ++i) {
        // 标量处理剩余 0-3 个像素
    }
}
```

## 本节小结

| 主题 | 要点 |
|------|------|
| 回绕 vs 饱和 | 回绕 = `(a+b) % 256`，饱和 = `min(a+b, 255)`；像素处理必须用饱和 |
| TVPSaturatedAdd | KrKr2 的无分支 4 通道并行饱和加法，核心是位运算溢出检测 + 掩码生成 |
| SSE2 饱和指令 | `_mm_adds_epu8`（加法）、`_mm_subs_epu8`（减法），1 条指令 = 16 通道 |
| NEON 饱和指令 | `vqaddq_u8`（加法）、`vqsubq_u8`（减法），前缀 `q` = saturating |
| 性能对比 | 标量位运算 ~8 条指令/像素，SIMD ~0.25 条指令/像素，理论加速 32× |
| 查找表 vs SIMD | 简单运算用 SIMD，复杂公式（如 SoftLight）用 LUT |
| KrKr2 架构 | tvpgl.h 声明 → tvpgl.cpp 默认 C → tvpgl_asm_init.h 注册 SIMD |

## 练习题与答案

### 题目 1：分析位运算

KrKr2 的 `TVPSaturatedAdd` 中，如果两个像素的某通道值分别为 `a=127` 和 `b=128`，手动计算步骤 1 的 `tmp` 值和最终结果。

<details>
<summary>查看答案</summary>

```
a = 127 = 0x7F, b = 128 = 0x80

步骤 1：
  a & b       = 0x7F & 0x80 = 0x00
  a ^ b       = 0x7F ^ 0x80 = 0xFF
  (a^b) >> 1  = 0xFF >> 1   = 0x7F
  & 0x7F      = 0x7F & 0x7F = 0x7F
  0x00 + 0x7F = 0x7F
  & 0x80      = 0x7F & 0x80 = 0x00  ← 最高位为 0，未溢出

步骤 2：tmp = (0x00 << 1) - (0x00 >> 7) = 0x00

步骤 3：result = (127 + 128 - 0) | 0 = 255

127 + 128 = 255，恰好不溢出，结果正确！
```

如果 `a=128, b=128`：
```
  a & b = 0x80, a ^ b = 0x00
  步骤 1: (0x80 + 0x00) & 0x80 = 0x80  ← 溢出！
  步骤 2: tmp = (0x80<<1) - (0x80>>7) = 0x00 - 0x01 = 0xFF (回绕)
  步骤 3: (128+128-255) | 255 = 1 | 255 = 255 ✅
```

</details>

### 题目 2：SIMD 指令选择

为以下像素操作选择最合适的 SSE2 指令：

1. 将两层图像的每个通道取较小值（Darken 变暗混合）
2. 检测哪些像素的 Alpha 通道 > 128（半透明阈值判断）
3. 将 ARGB 像素的所有通道取反（反色效果，但保留 Alpha）

<details>
<summary>查看答案</summary>

   ```cpp
   // 方法 1：用 XOR 将无符号转为有符号再比较
   __m128i bias = _mm_set1_epi8(-128);  // 0x80
   __m128i threshold = _mm_set1_epi8(128 - 128);  // = 0
   __m128i alpha = /* 提取 Alpha 通道后 */;
   __m128i biased = _mm_xor_si128(alpha, bias);
   __m128i mask = _mm_cmpgt_epi8(biased, threshold);

   // 方法 2（更简单）：饱和减法
   __m128i thresh = _mm_set1_epi8(128);
   __m128i result = _mm_subs_epu8(alpha, thresh);
   // result 中非零的位置就是 Alpha > 128 的像素
   ```

3. 使用 **`_mm_xor_si128`** 与掩码：
   ```cpp
   __m128i mask = _mm_set1_epi32(0x00FFFFFF);  // RGB 取反，Alpha 不变
   __m128i result = _mm_xor_si128(pixel, mask);
   // XOR 0xFF = 取反, XOR 0x00 = 不变
   ```

### 题目 3：完整实现 NEON 饱和减法混合

为 ARM 平台实现一个完整的饱和减法混合函数，包含 NEON 主循环和标量尾部处理。给出完整可编译代码。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdint>
#include <cstdio>

#if defined(__aarch64__) || defined(__ARM_NEON__) || defined(__ARM_NEON)
#include <arm_neon.h>

void blend_sub_neon(uint32_t* dst, const uint32_t* src, int count) {
    int simd_end = count & ~3;  // 向下对齐到 4 的倍数

    // NEON 主循环：每次处理 4 像素（16 字节）
    for (int i = 0; i < simd_end; i += 4) {
        uint8x16_t s = vld1q_u8((const uint8_t*)(src + i));
        uint8x16_t d = vld1q_u8((const uint8_t*)(dst + i));

        // vqsubq_u8: 无符号字节饱和减法
        // 每字节: result = max(d - s, 0)
        uint8x16_t result = vqsubq_u8(d, s);

        vst1q_u8((uint8_t*)(dst + i), result);
    }

    // 标量尾部：处理剩余 0-3 个像素
    for (int i = simd_end; i < count; ++i) {
        uint8_t* dp = (uint8_t*)(dst + i);
        const uint8_t* sp = (const uint8_t*)(src + i);
        for (int c = 0; c < 4; ++c) {
            int diff = (int)dp[c] - (int)sp[c];
            dp[c] = (diff < 0) ? 0 : (uint8_t)diff;
        }
    }
}

#else
// 非 ARM 平台的纯 C 回退
void blend_sub_neon(uint32_t* dst, const uint32_t* src, int count) {
    for (int i = 0; i < count; ++i) {
        uint8_t* dp = (uint8_t*)(dst + i);
        const uint8_t* sp = (const uint8_t*)(src + i);
        for (int c = 0; c < 4; ++c) {
            int diff = (int)dp[c] - (int)sp[c];
            dp[c] = (diff < 0) ? 0 : (uint8_t)diff;
        }
    }
}
#endif

int main() {
    // 测试数据
    uint32_t dst[5] = {0xFF'C8'64'32, 0xFF'50'50'50, 0xFF'00'00'00, 0xFF'FF'FF'FF, 0xFF'10'20'30};
    uint32_t src[5] = {0x00'64'32'64, 0x00'A0'30'60, 0x00'01'01'01, 0x00'FE'FE'FE, 0x00'20'10'30};

        printf("dst[%d] = 0x%08X\n", i, dst[i]);

    // 预期输出:
    // dst[0] = 0xFF640000   (200-100=100, 100-50=50→64-32=50, 50-100→0)
    // dst[1] = 0xFF002000   (80-160→0, 80-48=32→50-30=20→0x14, 80-96→0)
    // ... 具体值取决于字节序
    return 0;
}
```
