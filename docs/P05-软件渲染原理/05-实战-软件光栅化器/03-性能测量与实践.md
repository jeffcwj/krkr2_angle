## 九、性能测量：标量 vs SIMD

性能测量一定要“同输入、同输出、重复足够次数”。

代码示例 10（完整）：

```cpp

static double benchmarkBlendScalar(std::vector<uint32_t>& dst, const std::vector<uint32_t>& src, int rounds) {
```

## 十、常见错误及解决方案

### 错误 1：三角形边缘出现“裂缝”或重复填充

**现象：** 相邻三角形共享边时出现亮线或黑线。  
**原因：** 左右边取整规则不一致（一个用 `floor`，另一个用 `ceil`）。  
**解决：** 统一采用 Top-Left 规则，或统一 `[ceil(xL), floor(xR)]` 并保证共享边只归属一侧。

### 错误 2：纹理在透视角度下被“拉花”

**现象：** 近大远小时，纹理线条扭曲。  
**原因：** 直接线性插值 `u,v`，没有做透视校正。  
**解决：** 插值 `u/w、v/w、1/w`，最后除回 `u,v`。

### 错误 3：Alpha 混合后整体偏暗或发灰

**现象：** 半透明图层叠加后颜色不对。  
**原因：** 把预乘 Alpha 与非预乘 Alpha 路径混用。  
**解决：** 明确输入格式；若使用非预乘，按 `src*sa + dst*(1-sa)`；若预乘，则混合公式要对应修改。

### 错误 4：SSE2 路径比标量还慢

**现象：** 开 SIMD 后耗时不降反升。  
**原因：** 数据量太小、频繁分支、内存不连续、反复装箱拆箱。  
**解决：** 只在长扫描线启用 SIMD，保证连续内存，减少路径切换。

## 对照项目源码

## 本节小结

- 你已经从帧缓冲开始，完整走通了“线框→填充→纹理→混合→SIMD→MVP 管线”。

## 练习题与答案

### 题目 1：把最近邻采样替换为双线性采样，如何验证结果确实生效？
<details><summary>查看答案</summary>放大一张 16×16 棋盘纹理到 512×512：最近邻会有明显台阶，双线性会边缘变软；可同时输出两张图做像素对比。</details>

### 题目 2：若你使用预乘 Alpha 纹理，`alphaOver` 应如何改？
<details><summary>查看答案</summary>预乘输入下颜色项直接 `outRGB = srcRGB + dstRGB*(1-sa)`，不再乘 `sa`；Alpha 仍是 `sa + da*(1-sa)`。</details>

### 题目 3：为什么 KrKr2 要做 `TVPOpacityOnOpacityTable` 这类查表？
<details><summary>查看答案</summary>因为像素循环里除法昂贵且难向量化，查表把复杂函数离线化，运行时变成一次索引访问，吞吐更稳定。</details>
