## 一、为什么需要学容器与编码？

如果你打算理解 KrKr2 模拟器中的视频播放（`cpp/core/movie/ffmpeg/`）和音频解码（`cpp/core/sound/`），那么**容器格式**和**编码格式**是最基础的两个概念——它们决定了"数据以什么方式存在磁盘上"以及"数据用什么算法被压缩/还原"。

一个视觉小说引擎需要处理多种媒体：

| 媒体类型 | KrKr2 中的用途 | 典型格式 |
|----------|---------------|---------|
| OP/ED 动画 | 游戏片头/片尾视频 | MPEG-1/H.264 视频 |
| 背景音乐（BGM） | 循环播放的背景乐 | Ogg Vorbis |
| 角色语音 | 对话时的配音 | Ogg Vorbis / WAV |
| 音效（SE） | 按钮点击、环境音 | WAV（PCM，一种不压缩直接存储原始音频数据的方式） |

这些媒体的底层都涉及同一个问题：**原始数据太大，需要压缩存储，播放时再解压还原**。容器和编码就是解决这个问题的两层机制。

> **类比**：容器格式好比快递包装盒——它决定盒子的结构（能放几样东西、有没有分隔层、贴不贴时间标签）；编码格式好比打包方式——它决定物品如何被折叠压缩以节省空间。同一种打包方式（编码）可以装进不同品牌的盒子（容器），同一个盒子也能装用不同方式打包的物品。

## 二、容器格式（Container Format）

### 2.1 什么是容器格式

容器格式定义了媒体数据的**存储结构**。它规定了视频流（Video Stream，连续的视频画面数据）、音频流（Audio Stream，连续的音频数据）、字幕流等如何被组织在一个文件中。容器本身**不关心**数据用什么编码压缩，它只负责"装箱"和"编目录"。

一个容器文件通常包含：

```
┌──────────────────────────────────────────────────┐
│  容器文件头（File Header）                        │
│  ├── 格式标识（Magic Bytes，用于识别文件类型）      │
│  ├── 流信息表（有几路视频/音频/字幕）               │
│  └── 元数据（标题、作者、时长等）                   │
├──────────────────────────────────────────────────┤
│  数据区域                                         │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐    │
│  │视频包 1 │ │音频包 1 │ │视频包 2 │ │音频包 2 │    │
│  │(含时间戳)│ │(含时间戳)│ │(含时间戳)│ │(含时间戳)│   │
│  └────────┘ └────────┘ └────────┘ └────────┘    │
│  ... 交错排列的数据包 ...                          │
├──────────────────────────────────────────────────┤
│  索引表（可选）—— 记录关键帧位置，加速跳转          │
└──────────────────────────────────────────────────┘
```

容器的核心职责：
- **多路复用（Muxing）**：将多个流（视频+音频+字幕）交错写入同一个文件
- **时间戳管理**：为每个数据包（Packet，容器中最小的数据单元）打上时间戳，确保播放时音画同步
- **随机访问支持**：通过索引表让播放器能快速跳转到指定时间点

### 2.2 常见容器格式

| 容器格式 | 扩展名 | 特点 | 常见用途 |
|----------|--------|------|---------|
| **MP4** (MPEG-4 Part 14) | `.mp4`, `.m4a`, `.m4v` | 最通用的现代容器，索引在文件头或文件尾 | 网络视频、移动端、流媒体 |
| **MKV** (Matroska) | `.mkv`, `.mka` | 开放格式，支持几乎所有编码，可内嵌多字幕 | 高清视频收藏、字幕组发布 |
| **AVI** (Audio Video Interleave) | `.avi` | 微软早期格式，结构简单但功能有限 | 旧版视频、早期游戏 |
| **OGG** | `.ogg`, `.ogv` | 开源容器，常搭配 Vorbis/Opus 编码 | 游戏音频（**KrKr2 广泛使用**） |
| **WAV** (Waveform Audio) | `.wav` | 通常存储未压缩 PCM 音频的容器 | 音效、录音中间文件 |
| **FLV** (Flash Video) | `.flv` | 为 Flash 设计的轻量容器 | RTMP 直播推流 |
| **MPEG-TS** (Transport Stream) | `.ts`, `.m2ts` | 传输流，数据包固定 188 字节，抗丢包 | 电视广播、蓝光光盘 |
| **MPEG-PS** (Program Stream) | `.mpg`, `.mpeg` | 节目流，用于本地存储 | DVD、早期游戏 OP |

> **KrKr2 实际使用**：KiriKiri 引擎的 OP 动画通常封装在 `.mpg`（MPEG-PS 容器）或 `.avi` 容器中；BGM 和语音主要使用 `.ogg`（OGG 容器 + Vorbis 编码）。

### 2.3 容器格式在 FFmpeg 中的表示

FFmpeg 使用 `AVFormatContext`（一个 C 结构体，代表已打开的媒体文件的所有信息）来表示一个已打开的容器文件。通过 `avformat_open_input()`（FFmpeg 的文件打开函数）读取容器头信息：

```cpp
#include <libavformat/avformat.h>  // FFmpeg 的封装/解封装库头文件

int main() {
    AVFormatContext *fmt_ctx = NULL;  // 格式上下文指针，初始为空

    // 打开文件并读取容器头信息
    // 参数：&fmt_ctx = 输出指针，"input.mp4" = 文件路径
    //       NULL = 不指定格式（自动检测），NULL = 无额外选项
    int ret = avformat_open_input(&fmt_ctx, "input.mp4", NULL, NULL);
    if (ret < 0) {
        // av_err2str() 将错误码转为可读字符串
        printf("无法打开文件: %s\n", av_err2str(ret));
        return -1;
    }

    // 打印容器格式名称
    // iformat 是输入格式的描述结构，name 是格式的短名称
    printf("容器格式: %s\n", fmt_ctx->iformat->name);
    // 输出示例: "容器格式: mov,mp4,m4a,3gp,3g2,mj2"

    // 打印文件中有多少路流（视频流、音频流、字幕流等）
    // nb_streams 是流的数量
    printf("流数量: %d\n", fmt_ctx->nb_streams);

    // 打印时长（duration 单位是微秒，除以 AV_TIME_BASE 得到秒）
    // AV_TIME_BASE = 1000000，即 FFmpeg 内部用微秒作为时间单位
    if (fmt_ctx->duration != AV_NOPTS_VALUE) {
        double duration_sec = (double)fmt_ctx->duration / AV_TIME_BASE;
        printf("时长: %.2f 秒\n", duration_sec);
    }

    // 关闭文件，释放 fmt_ctx 占用的内存
    avformat_close_input(&fmt_ctx);
    return 0;
}
```

## 三、编码格式（Codec）

### 3.1 什么是编码格式

编码格式定义了数据的**压缩算法**。它规定了原始像素数据（视频）或原始采样数据（音频）如何被压缩成更小的体积，以及如何从压缩数据中还原出原始数据。

为什么压缩是必需的？以 1080p 视频为例：

```
未压缩 1080p 视频每秒数据量：
= 1920（宽）× 1080（高）× 24bit（每像素颜色深度）× 24fps（每秒帧数）
= 1,194,393,600 bit/s
≈ 149 MB/s
≈ 8.9 GB/min

→ 一部 90 分钟的电影未压缩约需 800 GB！
```

H.264 编码可以将同样的视频压缩到 5-20 Mbps（约 38-150 MB/min），压缩比高达 **100-200 倍**，而画质几乎无损。

### 3.2 常见视频编码

| 编码 | 全称 | 压缩效率 | 解码复杂度 | 典型用途 |
|------|------|---------|-----------|---------|
| **MPEG-1** | — | 低 | 极低 | VCD、**KiriKiri 游戏 OP** |
| **MPEG-2** | — | 中低 | 低 | DVD、数字电视广播 |
| **H.264** (AVC) | Advanced Video Coding | 高 | 中 | 当前最广泛使用的视频编码 |
| **H.265** (HEVC) | High Efficiency Video Coding | 极高 | 高 | 4K/8K 视频、新一代流媒体 |
| **VP9** | — | 高 | 中 | YouTube、WebM 格式 |
| **AV1** | AOMedia Video 1 | 极高 | 极高 | 新一代开源编码（Netflix、YouTube） |

> **KrKr2 源码对照**：KrKr2 的 `cpp/core/movie/ffmpeg/` 目录下的视频播放器基于 Kodi/XBMC 架构，使用 FFmpeg 解码，**理论上支持 FFmpeg 支持的所有视频编码**。但 KiriKiri 原版游戏最常用的是 MPEG-1，因为它解码极快、CPU 占用低，适合 2000 年代的硬件。

### 3.3 常见音频编码

| 编码 | 全称 | 压缩类型 | 典型用途 |
|------|------|---------|---------|
| **PCM** | Pulse Code Modulation（脉冲编码调制） | 无压缩 | WAV 内部、原始音频、音效 |
| **MP3** | MPEG-1 Audio Layer 3 | 有损 | 老牌压缩音频格式 |
| **AAC** | Advanced Audio Coding | 有损 | MP4 默认音频、流媒体 |
| **Vorbis** | — | 有损 | OGG 容器，**KrKr2 BGM/语音主力** |
| **Opus** | — | 有损 | 低延迟语音通话、新一代音频 |
| **FLAC** | Free Lossless Audio Codec（自由无损音频编码） | 无损 | 高品质音乐收藏 |

> **有损 vs 无损**：有损压缩（Lossy Compression）会丢弃人耳不易察觉的信息来获得更高压缩比，解压后的数据与原始数据**不完全相同**。无损压缩（Lossless Compression）不丢弃任何信息，解压后的数据与原始数据**完全一致**，但压缩比较低（通常只有 2-3 倍）。

### 3.4 编码在 FFmpeg 中的表示

FFmpeg 使用 `AVCodecParameters`（编码参数结构体）描述每一路流的编码信息：

```cpp
#include <libavformat/avformat.h>
#include <libavcodec/avcodec.h>  // FFmpeg 的编解码库头文件

int main() {
    AVFormatContext *fmt_ctx = NULL;
    avformat_open_input(&fmt_ctx, "input.mp4", NULL, NULL);

    // 读取流信息（分析文件内容，填充每路流的编码参数）
    // 不调用此函数的话，某些容器格式的流信息可能不完整
    avformat_find_stream_info(fmt_ctx, NULL);

    // 遍历所有流，打印编码信息
    for (unsigned i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream *stream = fmt_ctx->streams[i];  // 第 i 路流
        AVCodecParameters *par = stream->codecpar; // 该流的编码参数

        // codec_type: 流类型（视频/音频/字幕等）
        // avcodec_get_name(): 根据 codec_id 返回编码名称字符串
        printf("流 #%d: 类型=%s, 编码=%s\n",
               i,
               av_get_media_type_string(par->codec_type),
               avcodec_get_name(par->codec_id));

        if (par->codec_type == AVMEDIA_TYPE_VIDEO) {
            // 视频流特有信息
            printf("  分辨率: %dx%d\n", par->width, par->height);
            printf("  像素格式: %s\n",
                   av_get_pix_fmt_name((enum AVPixelFormat)par->format));
        } else if (par->codec_type == AVMEDIA_TYPE_AUDIO) {
            // 音频流特有信息
            printf("  采样率: %d Hz\n", par->sample_rate);
            printf("  声道数: %d\n", par->ch_layout.nb_channels);
        }
    }

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

## 四、容器与编码的组合关系

容器和编码是**独立的两个维度**——一个容器可以装不同编码的数据，一种编码也可以装在不同容器里。但并非所有组合都合法，每种容器有其支持的编码列表。

### 4.1 组合示例

```
┌───────────────────────────────────────────────────┐
│  MP4 容器                                          │
│  ┌───────────────────┐  ┌───────────────────────┐ │
│  │ 视频流: H.264      │  │ 音频流: AAC           │ │
│  │ 1920×1080, 24fps   │  │ 44100Hz, 立体声       │ │
│  └───────────────────┘  └───────────────────────┘ │
│  ┌───────────────────┐                             │
│  │ 字幕流: mov_text   │                            │
│  └───────────────────┘                             │
└───────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────┐
│  OGG 容器                                          │
│  ┌────────────────────────────────────────────┐   │
│  │ 音频流: Vorbis                              │   │
│  │ 44100Hz, 立体声, 192kbps                    │   │
│  └────────────────────────────────────────────┘   │
└───────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────┐
│  MPEG-PS 容器（KrKr2 OP 动画）                      │
│  ┌───────────────────┐  ┌───────────────────────┐ │
│  │ 视频流: MPEG-1     │  │ 音频流: MPEG-1 Layer 2│ │
│  │ 640×480, 29.97fps  │  │ 44100Hz, 立体声       │ │
│  └───────────────────┘  └───────────────────────┘ │
└───────────────────────────────────────────────────┘
```

### 4.2 常见容器与编码的兼容性

| 容器 | 支持的视频编码 | 支持的音频编码 |
|------|--------------|--------------|
| MP4 | H.264, H.265, MPEG-4, VP9 | AAC, MP3, AC-3, Opus |
| MKV | 几乎所有编码 | 几乎所有编码 |
| AVI | MPEG-4, H.264, DivX, Xvid | MP3, PCM, AC-3 |
| OGG | Theora, VP8 | Vorbis, Opus, FLAC |
| WAV | — | PCM, ADPCM, GSM |
| FLV | H.264, VP6, Sorenson | AAC, MP3, Speex |
| MPEG-PS | MPEG-1, MPEG-2 | MPEG-1 Layer 2, AC-3 |

> **常见误解**：".mp4 文件一定是 H.264 编码"——错！MP4 容器可以装 H.265 甚至 VP9 编码的视频。文件扩展名只能告诉你容器格式，编码格式必须用工具分析才能确定。

## 五、使用 ffprobe 分析媒体文件

`ffprobe` 是 FFmpeg 自带的媒体分析命令行工具，用来查看文件的容器格式、编码信息、流数量、时长等参数。

### 5.1 基本用法

```bash
# 查看文件的完整信息（容器+所有流）
ffprobe -v quiet -show_format -show_streams input.mp4

# 精简输出：只看容器名和每路流的类型+编码名
ffprobe -v error \
  -show_entries format=format_name \
  -show_entries stream=codec_type,codec_name \
  input.mp4
```

输出示例：

```
[FORMAT]
format_name=mov,mp4,m4a,3gp,3g2,mj2    ← 容器格式（MP4 族）
[/FORMAT]
[STREAM]
codec_type=video                         ← 第 1 路流：视频
codec_name=h264                          ← 编码：H.264
[/STREAM]
[STREAM]
codec_type=audio                         ← 第 2 路流：音频
codec_name=aac                           ← 编码：AAC
[/STREAM]
```

### 5.2 生成测试文件并分析

如果手边没有视频文件，可以用 FFmpeg 生成一个测试文件：

```bash
# 生成 5 秒测试视频（H.264 视频 + AAC 音频，640×480，30fps）
# -f lavfi -i testsrc=...  使用 FFmpeg 内置的测试源生成彩色条纹画面
# -f lavfi -i sine=...     生成 440Hz 正弦波音频（标准 A 音）
# -c:v libx264             视频编码器选择 H.264
# -c:a aac                 音频编码器选择 AAC
# -pix_fmt yuv420p         像素格式选择 YUV 4:2:0（下一节详解）
ffmpeg -f lavfi -i testsrc=duration=5:size=640x480:rate=30 \
       -f lavfi -i sine=frequency=440:duration=5:sample_rate=44100 \
       -c:v libx264 -c:a aac -pix_fmt yuv420p \
       test_output.mp4

# 用 ffprobe 分析刚生成的文件
ffprobe -v error -show_format -show_streams test_output.mp4
```

### 5.3 查看关键帧分布

```bash
# 只查看视频流的帧类型和时间戳
# -select_streams v   只选视频流
# pict_type           帧类型（I/P/B）
# pts_time            显示时间（秒）
ffprobe -v error -select_streams v \
  -show_frames \
  -show_entries frame=pict_type,pts_time \
  test_output.mp4
```

输出示例（节选）：

```
[FRAME]
pict_type=I      ← 关键帧（独立完整帧，详见下一节）
pts_time=0.000
[/FRAME]
[FRAME]
pict_type=P      ← 预测帧（存储与前一帧的差异，详见下一节）
pts_time=0.033
[/FRAME]
```

### 5.4 用 C++ 实现类似 ffprobe 的功能

```cpp
#include <cstdio>
#include <libavformat/avformat.h>
#include <libavcodec/avcodec.h>

// 打印媒体文件的容器和编码信息
void print_media_info(const char *filename) {
    AVFormatContext *fmt_ctx = NULL;

    // 打开文件
    if (avformat_open_input(&fmt_ctx, filename, NULL, NULL) < 0) {
        printf("无法打开: %s\n", filename);
        return;
    }

    // 分析流信息
    if (avformat_find_stream_info(fmt_ctx, NULL) < 0) {
        printf("无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return;
    }

    // 打印容器信息
    printf("=== 文件: %s ===\n", filename);
    printf("容器格式: %s (%s)\n",
           fmt_ctx->iformat->name,
           fmt_ctx->iformat->long_name);
    if (fmt_ctx->duration != AV_NOPTS_VALUE) {
        printf("时长: %.2f 秒\n",
               (double)fmt_ctx->duration / AV_TIME_BASE);
    }
    printf("总比特率: %lld bps\n", fmt_ctx->bit_rate);
    printf("流数量: %d\n\n", fmt_ctx->nb_streams);

    // 遍历每路流
    for (unsigned i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream *st = fmt_ctx->streams[i];
        AVCodecParameters *par = st->codecpar;

        printf("--- 流 #%d ---\n", i);
        printf("  类型: %s\n",
               av_get_media_type_string(par->codec_type));
        printf("  编码: %s\n", avcodec_get_name(par->codec_id));
        printf("  比特率: %lld bps\n", par->bit_rate);

        if (par->codec_type == AVMEDIA_TYPE_VIDEO) {
            printf("  分辨率: %d × %d\n", par->width, par->height);
            // 帧率从流的 avg_frame_rate 获取
            if (st->avg_frame_rate.den != 0) {
                printf("  帧率: %.2f fps\n",
                       av_q2d(st->avg_frame_rate));
            }
        } else if (par->codec_type == AVMEDIA_TYPE_AUDIO) {
            printf("  采样率: %d Hz\n", par->sample_rate);
            printf("  声道数: %d\n", par->ch_layout.nb_channels);
            printf("  采样格式: %s\n",
                   av_get_sample_fmt_name(
                       (enum AVSampleFormat)par->format));
        }
        printf("\n");
    }

    avformat_close_input(&fmt_ctx);
}

int main() {
    print_media_info("test_output.mp4");
    return 0;
}
```

## 常见错误及解决方案

### 错误 1：混淆容器格式和编码格式

**症状**：认为 `.mp4` 文件一定是 H.264 编码，或者认为 H.264 只能放在 MP4 容器里。

**正确理解**：容器和编码是独立的两个维度。MP4 可以装 H.264 也可以装 H.265；H.264 可以放在 MP4、MKV、FLV 等不同容器中。文件扩展名只反映容器格式。

**验证方法**：养成用 `ffprobe` 分析文件的习惯，不要根据扩展名猜测编码。

### 错误 2：不调用 avformat_find_stream_info 就读取流参数

**症状**：打开文件后直接访问 `streams[i]->codecpar`，发现某些字段（如 `width`、`height`、`sample_rate`）为 0 或不正确。

**原因**：某些容器格式（如 MPEG-TS）的头部不包含完整的流信息，`avformat_open_input()` 只读取文件头，不会深入分析每路流。

**正确做法**：在 `avformat_open_input()` 之后、访问流参数之前，必须调用 `avformat_find_stream_info()`。这个函数会读取一些实际数据包来推断流参数。

### 错误 3：忘记释放 AVFormatContext

**症状**：程序运行时内存持续增长，尤其在循环打开多个文件时。

**正确做法**：每次 `avformat_open_input()` 成功后，用完必须调用 `avformat_close_input(&fmt_ctx)` 释放资源。这个函数会同时释放内部分配的所有流和缓冲区。

## 本节小结

- **容器格式**（MP4/MKV/OGG/AVI 等）定义数据的存储结构——负责多路复用、时间戳管理、随机访问支持
- **编码格式**（H.264/Vorbis/AAC/PCM 等）定义数据的压缩算法——负责将原始数据压缩/还原
- 容器和编码是**独立的两个维度**：一个容器可以装多种编码，一种编码可以放入多种容器
- KrKr2 游戏主要使用 **MPEG-PS + MPEG-1** 播放 OP 动画，**OGG + Vorbis** 播放 BGM 和语音
- `ffprobe` 是分析媒体文件的必备工具——它能告诉你真实的容器和编码信息
- FFmpeg 中 `AVFormatContext` 代表容器，`AVCodecParameters` 描述编码参数

### 题目 1：容器与编码分类

以下名词中，哪些是容器格式，哪些是编码格式？

`MP4`、`H.264`、`OGG`、`Vorbis`、`MKV`、`AAC`、`WAV`、`FLAC`、`AVI`、`Opus`、`MPEG-TS`、`PCM`

<details>
<summary>查看答案</summary>

**容器格式**：MP4、OGG、MKV、WAV、AVI、MPEG-TS

**编码格式**：H.264、Vorbis、AAC、FLAC、Opus、PCM

**易混淆点**：
- **WAV** 是容器格式，不是编码。WAV 文件通常包含 PCM 未压缩数据，但 WAV 容器也可以包含 ADPCM、GSM 等其他编码
- **FLAC** 是编码格式（无损压缩算法）。`.flac` 文件虽然有自己的原生容器，但 FLAC 编码也可以封装在 OGG 或 MKV 容器中
- **PCM** 是编码格式（严格说是"无压缩编码"），它只是把模拟信号数字化，不做任何压缩

</details>

### 题目 3：编写转封装命令

用一条 FFmpeg 命令将 `game_op.mpg`（MPEG-PS 容器 + MPEG-1 视频 + MPEG-1 Layer 2 音频）转封装为 MKV 格式，要求**不重新编码**（直接复制流数据）。请写出命令并解释每个参数。

<details>
<summary>查看答案</summary>

```bash
ffmpeg -i game_op.mpg -c copy game_op.mkv
```

参数解释：
- `-i game_op.mpg`：指定输入文件
- `-c copy`：`-c` 是 `-codec` 的缩写，`copy` 表示对所有流（视频+音频）直接复制数据包，不进行重新编码。这样速度极快（秒级完成），且不会损失任何质量
- `game_op.mkv`：输出文件。FFmpeg 根据输出文件的扩展名 `.mkv` 自动选择 Matroska 容器格式

也可以分别指定视频和音频的编码策略：
```bash
ffmpeg -i game_op.mpg -c:v copy -c:a copy game_op.mkv
# -c:v copy  视频流直接复制
# -c:a copy  音频流直接复制
```

</details>
