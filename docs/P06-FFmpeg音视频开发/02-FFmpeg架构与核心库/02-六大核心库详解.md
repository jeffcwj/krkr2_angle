## 一、libavutil —— 基础工具库

libavutil 是 FFmpeg 所有其他库的**基础依赖**，提供通用工具函数。其他五个库都链接了 libavutil，你可以把它理解为 FFmpeg 的"标准库"。

### 1.1 核心功能一览

| 功能域 | 关键类型/函数 | 说明 |
|--------|-------------|------|
| 内存管理 | `av_malloc()`, `av_free()`, `av_freep()` | FFmpeg 自己的内存分配器（保证地址对齐，Alignment，即内存地址是某个特定字节数的整数倍，有利于 CPU 和 SIMD 指令高效访问） |
| 错误处理 | `av_strerror()`, `AVERROR()`, `AVERROR_EOF` | 统一的错误码系统（所有 FFmpeg 函数返回负值表示错误） |
| 像素格式 | `AVPixelFormat`, `av_get_pix_fmt_name()` | 枚举所有像素格式（YUV420P、RGB24、BGRA 等） |
| 采样格式 | `AVSampleFormat`, `av_get_sample_fmt_name()` | 枚举所有音频采样格式（S16、FLTP 等） |
| 数学工具 | `AVRational`, `av_rescale_q()` | 有理数运算——时间基（time_base，用分数表示的时间单位）转换的核心 |
| 日志系统 | `av_log()`, `av_log_set_level()` | 统一日志输出，可设置不同级别过滤 |
| 字典 | `AVDictionary`, `av_dict_set()`, `av_dict_get()` | 键值对存储（用于 metadata 元数据和选项传递） |
| 缓冲区 | `AVBuffer`, `AVBufferRef`, `av_buffer_ref()` | 引用计数缓冲区（实现零拷贝的基础，多个指针共享同一块数据） |

### 1.2 AVRational 与时间基——最重要的概念

FFmpeg 用 `AVRational`（有理数，即分数 num/den）表示时间基。所有时间戳（PTS、DTS）都是以时间基为单位的**整数**，需要乘以时间基才能转换为秒：

```cpp
#include <cstdio>
extern "C" {
#include <libavutil/rational.h>
#include <libavutil/mathematics.h>
}

int main() {
    // AVRational 是一个分数：{分子(num), 分母(den)}
    AVRational time_base = {1, 90000};  // 常见的 MPEG 时间基：1/90000 秒

    // 假设一个帧的 PTS（Presentation Time Stamp，显示时间戳）是 180000
    int64_t pts = 180000;

    // 将 PTS 转换为秒：pts × time_base = 180000 × (1/90000) = 2.0 秒
    // av_q2d() 将 AVRational 转为 double 浮点数
    double seconds = pts * av_q2d(time_base);
    printf("PTS %lld = %.2f 秒\n", pts, seconds);
    // 输出：PTS 180000 = 2.00 秒

    // ─── 在不同时间基之间转换 PTS ───
    AVRational src_tb = {1, 90000};   // 源时间基（MPEG 标准）
    AVRational dst_tb = {1, 1000};    // 目标时间基（毫秒精度）

    // av_rescale_q() 进行时间基转换，内部使用 128 位运算避免溢出
    // 等价于：pts * src_tb.num * dst_tb.den / (src_tb.den * dst_tb.num)
    // 但直接算可能溢出 int64_t（因为中间结果可能超过 2^63）
    int64_t dst_pts = av_rescale_q(pts, src_tb, dst_tb);
    printf("转换后 PTS = %lld（毫秒时间基）\n", dst_pts);
    // 输出：转换后 PTS = 2000（毫秒时间基）

    return 0;
}
```

> **KrKr2 中的使用**：`Clock.cpp` 中的 `CDVDClock`（同步时钟类）大量使用 `av_rescale_q()` 进行时间基转换，将不同流的 PTS 统一到同一时钟域（clock domain，即使用同一个时间基作为参考的时间空间）。

### 1.3 错误处理模式

FFmpeg 所有函数返回负值表示错误，使用 `av_strerror()` 将错误码转为可读字符串：

```cpp
#include <cstdio>
extern "C" {
#include <libavutil/error.h>
}

// 封装一个错误检查辅助函数（推荐在项目中统一使用）
void check_error(int ret, const char* operation) {
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE]; // 错误字符串缓冲区（128 字节）
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "%s 失败: %s\n", operation, errbuf);
    }
}

// ─── 常见错误码速查 ───
// AVERROR_EOF          —— 已到达文件/流末尾
// AVERROR(EAGAIN)      —— 暂时无数据，需要送入更多输入（解码器常返回这个）
// AVERROR(ENOMEM)      —— 内存分配失败
// AVERROR(EINVAL)      —— 参数无效
// AVERROR_INVALIDDATA  —— 输入数据损坏或格式不对
// AVERROR(ENOENT)      —— 文件/资源不存在
```

### 1.4 内存管理函数

```cpp
extern "C" {
#include <libavutil/mem.h>
}

// av_malloc：分配对齐的内存（通常 32 字节对齐，便于 SIMD 指令访问）
uint8_t* buffer = (uint8_t*)av_malloc(1024);  // 分配 1024 字节

// av_mallocz：分配并清零
uint8_t* zeros = (uint8_t*)av_mallocz(1024);  // 所有字节为 0

// av_calloc：分配数组（等价于 av_mallocz(n * size) 但有溢出检查）
float* samples = (float*)av_calloc(1024, sizeof(float));

// av_free：释放内存
av_free(buffer);

// av_freep：释放内存并将指针置为 nullptr（推荐使用，防止野指针）
av_freep(&zeros);    // zeros 现在是 nullptr
av_freep(&samples);  // samples 现在是 nullptr
```

> **为什么不用 `malloc`/`free`？** FFmpeg 的 `av_malloc` 保证返回的地址是 32 字节对齐的，这对 SIMD（Single Instruction Multiple Data，单指令多数据，一条 CPU 指令同时处理多个数据）优化至关重要。对齐的内存可以使用 AVX、SSE 等高性能指令集直接操作。

## 二、libavformat —— 封装/解封装库

libavformat 负责处理容器格式。它能识别文件格式、拆分出各个流（视频流、音频流、字幕流），并读取原始数据包（AVPacket）。

### 2.1 核心数据结构层次

```
AVFormatContext（格式上下文 —— 最顶层容器，代表一个打开的媒体文件）
├── nb_streams: 流数量（unsigned int）
├── streams[]: AVStream 数组
│   ├── streams[0]: AVStream（视频流）
│   │   ├── codecpar: AVCodecParameters（编解码参数：宽高、码率等）
│   │   ├── time_base: AVRational（该流的时间基）
│   │   └── duration: 流时长（以 time_base 为单位）
│   ├── streams[1]: AVStream（音频流）
│   └── streams[2]: AVStream（字幕流）
├── duration: 总时长（以 AV_TIME_BASE 即 1/1000000 秒为单位）
├── bit_rate: 总比特率（int64_t）
├── pb: AVIOContext*（I/O 上下文，自定义读取的关键入口）
├── iformat: AVInputFormat*（输入格式描述，如 "mov,mp4"）
└── metadata: AVDictionary*（文件级元数据：标题、艺术家等）
```

### 2.2 解封装（Demux）的标准流程

```cpp
#include <cstdio>
extern "C" {
#include <libavformat/avformat.h>
}

int main() {
    AVFormatContext* fmt_ctx = nullptr;  // 必须初始化为 nullptr！

    // ─── 第 1 步：打开文件 ───
    // avformat_open_input 会自动探测文件格式（通过读取文件头的魔数）
    // 参数：&ctx, 文件路径, 强制格式(nullptr=自动), 选项(nullptr=默认)
    int ret = avformat_open_input(&fmt_ctx, "test.mp4", nullptr, nullptr);
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开文件: %s\n", errbuf);
        return 1;
    }
    // 此时 fmt_ctx 已被分配并填充了基本信息

    // ─── 第 2 步：读取流信息 ───
    // avformat_find_stream_info 会读取一部分数据来探测编码参数
    // 对于某些容器（如 MPEG-TS），这一步是必须的
    ret = avformat_find_stream_info(fmt_ctx, nullptr);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // ─── 第 3 步：遍历所有流 ───
    printf("文件包含 %u 个流：\n", fmt_ctx->nb_streams);
    for (unsigned i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream* stream = fmt_ctx->streams[i];
        // av_get_media_type_string: 返回 "video"/"audio"/"subtitle" 等
        const char* type_name = av_get_media_type_string(
            stream->codecpar->codec_type);
        // avcodec_get_name: 返回编解码器名称如 "h264"/"aac"
        const char* codec_name = avcodec_get_name(
            stream->codecpar->codec_id);
        printf("  流 #%u: 类型=%s, 编码=%s, 时间基=%d/%d\n",
               i,
               type_name ? type_name : "unknown",
               codec_name,
               stream->time_base.num, stream->time_base.den);
    }

    // ─── 第 4 步：逐包读取 ───
    AVPacket* pkt = av_packet_alloc();  // 分配数据包结构
    int packet_count = 0;
    while (av_read_frame(fmt_ctx, pkt) >= 0) {
        packet_count++;
        if (packet_count <= 5) {  // 只打印前 5 个包的信息
            printf("  包 #%d: 流=%d, PTS=%lld, 大小=%d 字节\n",
                   packet_count,
                   pkt->stream_index,
                   pkt->pts,
                   pkt->size);
        }
        av_packet_unref(pkt);  // 释放包内的数据缓冲区（重要！否则内存泄漏）
    }
    printf("共读取 %d 个数据包\n", packet_count);

    // ─── 第 5 步：清理 ───
    av_packet_free(&pkt);            // 释放 AVPacket 结构本身
    avformat_close_input(&fmt_ctx);  // 关闭文件并释放 fmt_ctx

    return 0;
}
```

### 2.3 KrKr2 中的使用——自定义 I/O

KrKr2 的 `CDVDDemuxFFmpeg` 类封装了上述解封装流程，但有一个重要差异：它使用**自定义 I/O 回调**而非直接打开文件路径。这是因为 KrKr2 的媒体数据存储在 XP3 归档包（KiriKiri 引擎的专有资源包格式）中：

```cpp
// 来源：cpp/core/movie/ffmpeg/DemuxFFmpeg.h（简化）
class CDVDDemuxFFmpeg : public CDVDDemux {
    AVFormatContext* m_pFormatContext = nullptr;  // 格式上下文
    AVIOContext* m_ioContext = nullptr;           // 自定义 I/O 上下文

    // 自定义读取回调——从 COM IStream 接口读取数据
    static int dvd_file_read(void* h, uint8_t* buf, int size);
    static int64_t dvd_file_seek(void* h, int64_t pos, int whence);
};
```

> **为什么需要自定义 I/O？** 标准的 `avformat_open_input("path")` 使用操作系统的文件 I/O。但 XP3 归档中的文件不在文件系统中——它们是归档内的虚拟文件。通过 `avio_alloc_context()` 注册自定义的 read/seek 回调，FFmpeg 就能从任意数据源读取数据。

## 三、libavcodec —— 编解码库

libavcodec 是 FFmpeg 最核心的库，包含了上百种音视频编解码器的实现。

### 3.1 核心数据结构

```
AVCodec（编解码器描述——静态信息，描述一种编解码器的能力）
├── name: 名称（如 "h264", "aac", "vp9"）
├── type: 类型（AVMEDIA_TYPE_VIDEO / AVMEDIA_TYPE_AUDIO / AVMEDIA_TYPE_SUBTITLE）
├── id: AVCodecID 枚举值（如 AV_CODEC_ID_H264）
└── capabilities: 能力标志位（如支持多线程、支持硬件加速等）

AVCodecContext（编解码器上下文——运行时状态，一个正在工作的编解码器实例）
├── codec: 指向 AVCodec（说明用的是哪个编解码器）
├── width, height: 视频帧的宽高（像素）
├── pix_fmt: 像素格式（AVPixelFormat 枚举，如 AV_PIX_FMT_YUV420P）
├── sample_rate: 音频采样率（Hz）
├── ch_layout: 通道布局（AVChannelLayout，如立体声）
├── time_base: 时间基
├── extradata: 编解码器额外数据（如 H.264 的 SPS/PPS 参数集）
└── opaque: 用户自定义数据指针（可附加任意数据）

AVPacket（压缩数据包——编码后的数据）
├── data: 压缩数据的指针（uint8_t*）
├── size: 数据大小（字节）
├── pts: 显示时间戳（Presentation Time Stamp）
├── dts: 解码时间戳（Decode Time Stamp，有 B 帧时 DTS ≠ PTS）
├── stream_index: 所属流的索引号
└── flags: 标志位（如 AV_PKT_FLAG_KEY 表示关键帧）

AVFrame（原始数据帧——解码后的数据）
├── data[8]: 数据平面指针数组（YUV 有 3 个平面：Y/U/V）
├── linesize[8]: 每行字节数（含对齐填充，可能大于 width×bytes_per_pixel）
├── width, height: 视频帧尺寸
├── format: 像素格式（视频）或采样格式（音频）
├── pts: 显示时间戳
├── nb_samples: 音频帧的采样点数量
└── ch_layout: 音频通道布局
```

### 3.2 完整的视频解码示例

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
}

int decode_video(const char* filename) {
    AVFormatContext* fmt_ctx = nullptr;
    int ret;

    // 打开文件 + 读取流信息
    if ((ret = avformat_open_input(&fmt_ctx, filename, nullptr, nullptr)) < 0)
        return ret;
    if ((ret = avformat_find_stream_info(fmt_ctx, nullptr)) < 0)
        return ret;

    // ─── 找到视频流 ───
    int video_idx = -1;
    for (unsigned i = 0; i < fmt_ctx->nb_streams; i++) {
        if (fmt_ctx->streams[i]->codecpar->codec_type == AVMEDIA_TYPE_VIDEO) {
            video_idx = i;
            break;
        }
    }
    if (video_idx < 0) {
        fprintf(stderr, "找不到视频流\n");
        return -1;
    }

    AVStream* video_stream = fmt_ctx->streams[video_idx];

    // ─── 查找并打开解码器 ───
    // avcodec_find_decoder: 根据 codec_id 找到对应的解码器实现
    const AVCodec* decoder = avcodec_find_decoder(
        video_stream->codecpar->codec_id);
    if (!decoder) {
        fprintf(stderr, "找不到解码器: %s\n",
                avcodec_get_name(video_stream->codecpar->codec_id));
        return -1;
    }
    printf("使用解码器: %s (%s)\n", decoder->name, decoder->long_name);

    // avcodec_alloc_context3: 分配解码器上下文
    AVCodecContext* codec_ctx = avcodec_alloc_context3(decoder);

    // avcodec_parameters_to_context: 从流参数复制到解码器上下文
    // 这一步把宽高、像素格式、extradata 等信息设置到 codec_ctx 中
    avcodec_parameters_to_context(codec_ctx, video_stream->codecpar);

    // avcodec_open2: 初始化解码器（分配内部缓冲区、启动解码线程等）
    if ((ret = avcodec_open2(codec_ctx, decoder, nullptr)) < 0) {
        fprintf(stderr, "无法打开解码器\n");
        return ret;
    }

    printf("视频: %dx%d, 像素格式: %s\n",
           codec_ctx->width, codec_ctx->height,
           av_get_pix_fmt_name(codec_ctx->pix_fmt));

    // ─── 解码循环 ───
    AVPacket* pkt = av_packet_alloc();
    AVFrame* frame = av_frame_alloc();
    int frame_count = 0;

    while (av_read_frame(fmt_ctx, pkt) >= 0) {
        if (pkt->stream_index != video_idx) {
            av_packet_unref(pkt);
            continue;  // 跳过非视频包（音频、字幕等）
        }

        // 发送压缩包到解码器
        ret = avcodec_send_packet(codec_ctx, pkt);
        av_packet_unref(pkt);  // 发送后立即释放包数据
        if (ret < 0) {
            fprintf(stderr, "发送包失败\n");
            break;
        }

        // 从解码器接收解码后的帧
        // 一个 packet 可能产生 0 个或多个 frame（B 帧重排序）
        while (ret >= 0) {
            ret = avcodec_receive_frame(codec_ctx, frame);
            if (ret == AVERROR(EAGAIN)) break;   // 需要更多输入
            if (ret == AVERROR_EOF) goto end;     // 全部解码完毕
            if (ret < 0) {
                fprintf(stderr, "接收帧失败\n");
                goto end;
            }

            frame_count++;
            if (frame_count <= 3) {
                // av_get_picture_type_char: 返回帧类型字符
                // 'I'=关键帧, 'P'=预测帧, 'B'=双向预测帧
                printf("帧 #%d: PTS=%lld, 类型=%c, 尺寸=%dx%d\n",
                       frame_count,
                       frame->pts,
                       av_get_picture_type_char(frame->pict_type),
                       frame->width, frame->height);
            }

            av_frame_unref(frame);  // 释放帧内的数据缓冲区
        }
    }

end:
    printf("共解码 %d 帧\n", frame_count);

    av_frame_free(&frame);
    av_packet_free(&pkt);
    avcodec_free_context(&codec_ctx);
    avformat_close_input(&fmt_ctx);

    return 0;
}

int main() {
    return decode_video("test.mp4");
}
```

## 四、libswscale —— 图像缩放与格式转换

libswscale 用于在不同像素格式之间转换，以及缩放视频帧。在播放器中这是必需的——解码器输出的格式（通常是 YUV420P）往往不是渲染器需要的格式（通常是 RGB 或 BGRA）。

### 4.1 核心 API 示例

```cpp
#include <cstdio>
extern "C" {
#include <libswscale/swscale.h>
#include <libavutil/imgutils.h>
#include <libavutil/frame.h>
}

int main() {
    int src_w = 1920, src_h = 1080;  // 源尺寸
    int dst_w = 960,  dst_h = 540;   // 目标尺寸（缩小一半）

    // ─── sws_getContext: 创建转换上下文 ───
    // 参数：源宽, 源高, 源像素格式,
    //       目标宽, 目标高, 目标像素格式,
    //       缩放算法, (后三个参数通常为 nullptr)
    SwsContext* sws_ctx = sws_getContext(
        src_w, src_h, AV_PIX_FMT_YUV420P,   // 源：YUV420P
        dst_w, dst_h, AV_PIX_FMT_BGRA,       // 目标：BGRA
        SWS_BILINEAR,                          // 双线性插值算法
        nullptr, nullptr, nullptr);

    if (!sws_ctx) {
        fprintf(stderr, "无法创建 SwsContext\n");
        return 1;
    }

    // ─── 分配源缓冲区（模拟 YUV420P 帧） ───
    // av_image_alloc: 为指定像素格式分配图像缓冲区
    // 参数：数据指针数组, 行字节数数组, 宽, 高, 格式, 对齐字节数
    uint8_t* src_data[4];
    int src_linesize[4];
    av_image_alloc(src_data, src_linesize, src_w, src_h,
                   AV_PIX_FMT_YUV420P, 32);  // 32 字节对齐

    // 分配目标缓冲区（BGRA 只有一个平面）
    uint8_t* dst_data[4];
    int dst_linesize[4];
    av_image_alloc(dst_data, dst_linesize, dst_w, dst_h,
                   AV_PIX_FMT_BGRA, 32);

    // ─── sws_scale: 执行转换 ───
    // 参数：上下文, 源数据, 源行大小, 起始行, 处理行数, 目标数据, 目标行大小
    int output_height = sws_scale(
        sws_ctx,
        src_data, src_linesize,   // 源
        0, src_h,                  // 从第 0 行开始，处理全部 src_h 行
        dst_data, dst_linesize);   // 目标

    printf("转换完成，输出 %d 行\n", output_height);
    printf("源 linesize: Y=%d, U=%d, V=%d\n",
           src_linesize[0], src_linesize[1], src_linesize[2]);
    printf("目标 linesize: BGRA=%d\n", dst_linesize[0]);

    // 清理
    av_freep(&src_data[0]);
    av_freep(&dst_data[0]);
    sws_freeContext(sws_ctx);

    return 0;
}
```

### 4.2 缩放算法选择

| 算法 | 常量 | 速度 | 质量 | 适用场景 |
|------|------|------|------|---------|
| 最近邻 | `SWS_POINT` | ★★★★★ | ★ | 像素风游戏、整数倍缩放（不做插值，直接取最近的像素） |
| 双线性 | `SWS_BILINEAR` | ★★★★ | ★★★ | 实时播放的首选（KrKr2 的选择） |
| 双三次 | `SWS_BICUBIC` | ★★★ | ★★★★ | 高质量缩放（对边缘更锐利） |
| Lanczos | `SWS_LANCZOS` | ★★ | ★★★★★ | 最高质量（视频转码时推荐） |
| 面积平均 | `SWS_AREA` | ★★★ | ★★★★ | 缩小图像（保留更多细节） |

> **KrKr2 中的使用**：`VideoCodecFFmpeg.cpp` 使用 libswscale 将解码器输出的 YUV 帧转换为渲染器所需的像素格式。具体转换路径取决于渲染后端——OpenGL 渲染器可以直接接受 YUV420P 并在 shader（着色器，在 GPU 上运行的小程序）中转换，软件渲染器则需要先转为 BGRA。

## 五、libswresample —— 音频重采样库

libswresample 用于音频采样率转换、通道布局转换和采样格式转换。在播放器中，解码器输出的音频格式可能与音频设备期望的格式不同，需要用 libswresample 进行转换。

### 5.1 核心 API 示例

```cpp
#include <cstdio>
extern "C" {
#include <libswresample/swresample.h>
#include <libavutil/channel_layout.h>
#include <libavutil/samplefmt.h>
#include <libavutil/mem.h>
#include <libavutil/mathematics.h>
}

int main() {
    SwrContext* swr_ctx = nullptr;
    int ret;

    // ─── 设置源和目标通道布局 ───
    AVChannelLayout src_ch_layout = AV_CHANNEL_LAYOUT_STEREO;  // 源：立体声
    AVChannelLayout dst_ch_layout = AV_CHANNEL_LAYOUT_STEREO;  // 目标：立体声

    // ─── swr_alloc_set_opts2: 创建并配置重采样上下文 ───
    ret = swr_alloc_set_opts2(
        &swr_ctx,
        &dst_ch_layout,              // 目标通道布局
        AV_SAMPLE_FMT_S16,           // 目标格式：16 位有符号整数（interleaved）
        48000,                        // 目标采样率：48kHz
        &src_ch_layout,              // 源通道布局
        AV_SAMPLE_FMT_FLTP,         // 源格式：32 位浮点（planar，每通道独立数组）
        44100,                        // 源采样率：44.1kHz
        0, nullptr);

    if (ret < 0 || !swr_ctx) {
        fprintf(stderr, "无法创建 SwrContext\n");
        return 1;
    }

    // swr_init: 初始化重采样器（内部准备好转换需要的滤波器系数）
    if ((ret = swr_init(swr_ctx)) < 0) {
        fprintf(stderr, "无法初始化 SwrContext\n");
        return 1;
    }

    // 模拟转换 1024 个采样点
    int src_nb_samples = 1024;

    // ─── 计算目标采样数 ───
    // 采样率不同，输出的采样数也不同（48000/44100 ≈ 1.088 倍）
    // swr_get_delay: 获取重采样器内部的延迟采样数
    int dst_nb_samples = av_rescale_rnd(
        swr_get_delay(swr_ctx, 44100) + src_nb_samples,
        48000,   // 目标采样率
        44100,   // 源采样率
        AV_ROUND_UP);  // 向上取整

    printf("源采样数: %d, 目标采样数: %d\n",
           src_nb_samples, dst_nb_samples);

    // 分配源缓冲区（FLTP 是 planar 格式，每个通道独立数组）
    uint8_t* src_data[2];  // 2 个通道
    src_data[0] = (uint8_t*)av_malloc(src_nb_samples * sizeof(float));
    src_data[1] = (uint8_t*)av_malloc(src_nb_samples * sizeof(float));

    // 分配目标缓冲区（S16 是 interleaved 格式，通道数据交织）
    uint8_t* dst_data[1];
    dst_data[0] = (uint8_t*)av_malloc(
        dst_nb_samples * 2 * sizeof(int16_t));  // 2 通道 × 2 字节/采样

    // ─── swr_convert: 执行转换 ───
    int converted = swr_convert(
        swr_ctx,
        dst_data, dst_nb_samples,                      // 目标缓冲区和最大采样数
        (const uint8_t**)src_data, src_nb_samples);    // 源数据和采样数

    printf("转换了 %d 个采样\n", converted);

    // 清理
    av_free(src_data[0]);
    av_free(src_data[1]);
    av_free(dst_data[0]);
    swr_free(&swr_ctx);

    return 0;
}
```

### 5.2 Planar vs Interleaved 格式

这是 FFmpeg 音频处理中最重要的概念之一：

```
Interleaved（交织/交错）格式，如 AV_SAMPLE_FMT_S16：
  内存布局：[L0][R0][L1][R1][L2][R2]...
  所有通道的采样交替存放在一个数组中
  AVFrame::data[0] 包含所有数据，data[1] 未使用

Planar（平面/分离）格式，如 AV_SAMPLE_FMT_FLTP：
  通道 0：[L0][L1][L2][L3]...    ← AVFrame::data[0]
  通道 1：[R0][R1][R2][R3]...    ← AVFrame::data[1]
  每个通道是独立的连续数组
```

FFmpeg 解码器通常输出 **planar** 格式（格式名以 P 结尾，如 FLTP、S16P），因为这样有利于编解码器内部的 SIMD 优化。而音频设备（如 SDL、OpenAL、Android AudioTrack）通常期望 **interleaved** 格式（如 S16、FLT）。libswresample 能在两种格式之间高效转换。

> **KrKr2 中的使用**：`AudioCodecFFmpeg.cpp` 和 `FFWaveDecoder.cpp` 都使用 libswresample 将解码器输出转换为引擎音频系统期望的格式（16 位立体声交织格式，即 `AV_SAMPLE_FMT_S16`）。

## 六、libavfilter —— 滤镜框架

libavfilter 提供音视频滤镜处理能力，通过构建**滤镜图**（Filter Graph，由多个滤镜节点组成的处理链路）来对音视频数据进行变换。

### 6.1 滤镜图概念

```
                    滤镜图（Filter Graph）
┌──────────────────────────────────────────────────────┐
│                                                      │
│  [buffer] ──→ [yadif] ──→ [scale] ──→ [buffersink]  │
│   (输入)     (反交错)    (缩放)      (输出)          │
│                                                      │
└──────────────────────────────────────────────────────┘

buffer:     输入端——将 AVFrame 送入滤镜图
yadif:      反交错滤镜（Yet Another DeInterlacing Filter）
            将隔行扫描视频转为逐行扫描
scale:      缩放滤镜——改变视频帧的尺寸或像素格式
buffersink: 输出端——从滤镜图中取出处理后的 AVFrame
```

### 6.2 基础用法

```cpp
#include <cstdio>
extern "C" {
#include <libavfilter/avfilter.h>
#include <libavfilter/buffersrc.h>
#include <libavfilter/buffersink.h>
#include <libavutil/opt.h>
}

// 创建一个简单的视频滤镜图：缩放到 640x360
int create_filter_graph(int src_w, int src_h, AVPixelFormat src_fmt) {
    // avfilter_graph_alloc: 分配一个空的滤镜图
    AVFilterGraph* graph = avfilter_graph_alloc();
    if (!graph) return -1;

    // ─── 创建输入滤镜（buffer） ───
    // buffer 滤镜是滤镜图的"入口"，负责接收外部送入的帧
    char args[512];
    snprintf(args, sizeof(args),
             "video_size=%dx%d:pix_fmt=%d:time_base=1/25",
             src_w, src_h, src_fmt);

    const AVFilter* buffersrc = avfilter_get_by_name("buffer");
    AVFilterContext* src_ctx = nullptr;
    // avfilter_graph_create_filter: 创建滤镜实例并添加到滤镜图
    int ret = avfilter_graph_create_filter(
        &src_ctx, buffersrc, "in", args, nullptr, graph);
    if (ret < 0) {
        fprintf(stderr, "无法创建 buffer 滤镜\n");
        avfilter_graph_free(&graph);
        return ret;
    }

    // ─── 创建输出滤镜（buffersink） ───
    // buffersink 是滤镜图的"出口"，处理后的帧从这里取出
    const AVFilter* buffersink = avfilter_get_by_name("buffersink");
    AVFilterContext* sink_ctx = nullptr;
    ret = avfilter_graph_create_filter(
        &sink_ctx, buffersink, "out", nullptr, nullptr, graph);
    if (ret < 0) {
        fprintf(stderr, "无法创建 buffersink 滤镜\n");
        avfilter_graph_free(&graph);
        return ret;
    }

    // ─── 创建缩放滤镜 ───
    const AVFilter* scale_filter = avfilter_get_by_name("scale");
    AVFilterContext* scale_ctx = nullptr;
    ret = avfilter_graph_create_filter(
        &scale_ctx, scale_filter, "scale",
        "640:360", nullptr, graph);  // 参数："宽:高"
    if (ret < 0) {
        fprintf(stderr, "无法创建 scale 滤镜\n");
        avfilter_graph_free(&graph);
        return ret;
    }

    // ─── 连接滤镜 ───
    // avfilter_link: 将一个滤镜的输出端口连接到另一个滤镜的输入端口
    avfilter_link(src_ctx, 0, scale_ctx, 0);    // buffer → scale
    avfilter_link(scale_ctx, 0, sink_ctx, 0);   // scale → buffersink

    // avfilter_graph_config: 验证并配置滤镜图
    ret = avfilter_graph_config(graph, nullptr);
    if (ret < 0) {
        fprintf(stderr, "滤镜图配置失败\n");
        avfilter_graph_free(&graph);
        return ret;
    }

    printf("滤镜图创建成功\n");

    // ─── 使用方式 ───
    // av_buffersrc_add_frame(src_ctx, decoded_frame);    // 送入解码后的帧
    // av_buffersink_get_frame(sink_ctx, filtered_frame); // 取出处理后的帧

    avfilter_graph_free(&graph);
    return 0;
}

int main() {
    create_filter_graph(1920, 1080, AV_PIX_FMT_YUV420P);
    return 0;
}
```

> **KrKr2 中的使用**：`VideoCodecFFmpeg.cpp` 使用 libavfilter 进行视频后处理。类成员 `m_pFilteredFrame` 存储滤镜处理后的帧。典型用途包括使用 yadif 滤镜进行反交错处理（将隔行扫描的老式视频转为现代的逐行扫描格式）。

## 常见错误及解决方案

### 错误 1：忘记 `av_packet_unref` 导致内存泄漏

```cpp
while (av_read_frame(fmt_ctx, pkt) >= 0) {
    // 处理 pkt...
    // ❌ 忘记调用 av_packet_unref(pkt) → 每次循环泄漏一个包的数据！
}
```

**现象**：程序运行一段时间后内存暴涨，最终崩溃。

**解决**：每次 `av_read_frame` 成功后，无论是否使用了该包，都必须调用 `av_packet_unref(pkt)` 释放包内的数据缓冲区。同理，`avcodec_receive_frame` 成功后必须在用完帧数据后调用 `av_frame_unref(frame)`。

### 错误 2：`sws_scale` 输出全黑

**现象**：格式转换后输出的图像全是黑色。

**原因**：通常是 `linesize`（行步长，每行数据在内存中实际占用的字节数，可能大于 `width × bytes_per_pixel` 因为对齐填充）设置错误。

**解决**：使用 `av_image_alloc()` 或 `av_image_fill_arrays()` 让 FFmpeg 自动计算正确的 linesize，不要手动计算：

```cpp
// ✅ 让 FFmpeg 计算 linesize
av_image_alloc(dst_data, dst_linesize, dst_w, dst_h, AV_PIX_FMT_BGRA, 32);

// ❌ 手动计算可能出错（忘记对齐）
dst_linesize[0] = dst_w * 4;  // 可能不是 32 字节对齐的！
```

### 错误 3：Planar/Interleaved 格式混淆

**现象**：音频解码后播放出来是噪音。

**原因**：解码器输出 planar 格式（如 `AV_SAMPLE_FMT_FLTP`），但代码按 interleaved 方式处理了 `data[0]`。

**解决**：检查 `AVCodecContext::sample_fmt`，如果是以 P 结尾的格式（S16P、S32P、FLTP），说明是 planar 格式，需要用 `libswresample` 转换为 interleaved 格式后再送到音频设备。

## 本节小结

- **libavutil**：地基库，提供内存管理（`av_malloc`/`av_freep`）、时间基转换（`AVRational`/`av_rescale_q`）、错误处理（`av_strerror`）和格式定义

- **libswscale**：图像缩放和像素格式转换，核心 API 是 `sws_getContext` + `sws_scale`
- **libswresample**：音频重采样和格式转换，核心 API 是 `swr_alloc_set_opts2` + `swr_init` + `swr_convert`
- **libavfilter**：滤镜框架，通过滤镜图（buffer → 处理节点 → buffersink）对音视频进行变换
- KrKr2 中，视频模块用 6 个库，音频模块用 4 个库，所有模块都依赖 libavutil

### 题目 2：修复错误的代码

以下代码有 3 处错误，找出并修正：

```cpp
#include <libavformat/avformat.h>

int main() {
    AVFormatContext* fmt_ctx;  // 错误 1
    avformat_open_input(&fmt_ctx, "test.mp4", NULL, NULL);
    avformat_find_stream_info(fmt_ctx, NULL);

    AVPacket pkt;  // 错误 2
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        printf("stream=%d size=%d\n", pkt.stream_index, pkt.size);
        // 错误 3：缺少什么？
    }

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

<details>
<summary>查看答案</summary>

**错误 1**：`fmt_ctx` 未初始化为 `nullptr`。`avformat_open_input` 的第一个参数是 `AVFormatContext**`，如果 `*fmt_ctx` 不为 `nullptr`，它会尝试使用已有的上下文而非分配新的，可能导致崩溃。

```cpp
AVFormatContext* fmt_ctx = nullptr;  // ✅ 必须初始化为 nullptr
```

**错误 2**：`AVPacket` 不应该在栈上直接声明。现代 FFmpeg 推荐使用 `av_packet_alloc()` 在堆上分配。栈上的 `AVPacket` 需要用 `av_init_packet` 初始化，而该函数在新版 FFmpeg 中已被废弃。

```cpp
AVPacket* pkt = av_packet_alloc();  // ✅ 使用堆分配
// 循环中使用 pkt 而非 &pkt
av_packet_free(&pkt);  // 最后释放
```

**错误 3**：循环内缺少 `av_packet_unref(pkt)` 调用。每次 `av_read_frame` 成功后，`pkt` 内部会分配数据缓冲区引用，如果不释放就会持续泄漏内存。

```cpp
while (av_read_frame(fmt_ctx, pkt) >= 0) {
    printf("stream=%d size=%d\n", pkt->stream_index, pkt->size);
    av_packet_unref(pkt);  // ✅ 释放包内数据的引用
}
```

完整修正代码：

```cpp
#include <cstdio>
extern "C" {
#include <libavformat/avformat.h>
}

int main() {
    AVFormatContext* fmt_ctx = nullptr;
    int ret = avformat_open_input(&fmt_ctx, "test.mp4", nullptr, nullptr);
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "打开失败: %s\n", errbuf);
        return 1;
    }

    if (avformat_find_stream_info(fmt_ctx, nullptr) < 0) {
        fprintf(stderr, "获取流信息失败\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    AVPacket* pkt = av_packet_alloc();
    while (av_read_frame(fmt_ctx, pkt) >= 0) {
        printf("stream=%d size=%d\n", pkt->stream_index, pkt->size);
        av_packet_unref(pkt);
    }

    av_packet_free(&pkt);
    avformat_close_input(&fmt_ctx);
    return 0;
}
```

</details>

### 题目 3：设计题——自定义 I/O

KrKr2 的媒体文件存储在 XP3 归档包中，无法用文件路径直接打开。请回答：

1. FFmpeg 提供了什么机制来支持自定义数据源？
2. 需要实现哪些回调函数？
3. 使用哪个 API 创建自定义 I/O 上下文？

<details>
<summary>查看答案</summary>

**1. 自定义 I/O 机制**

FFmpeg 通过 `AVIOContext` 支持自定义 I/O。用户可以注册回调函数，FFmpeg 在需要读取数据时调用这些回调，而不是直接操作文件系统。

**2. 需要实现的回调函数**

```cpp
// 读取回调：从自定义数据源读取 buf_size 字节到 buf
int read_callback(void* opaque, uint8_t* buf, int buf_size);
// 返回：实际读取的字节数，到达末尾返回 AVERROR_EOF

// Seek 回调：在数据源中定位
int64_t seek_callback(void* opaque, int64_t offset, int whence);
// whence 可以是 SEEK_SET、SEEK_CUR、SEEK_END 或
// AVSEEK_SIZE（返回数据源总大小，不实际移动位置）
```

- `opaque`：用户自定义指针，指向你的数据源对象（KrKr2 中是 `IStream*`）
- 读取回调必须实现，Seek 回调可选但强烈建议实现（否则无法拖动进度条）

**3. 创建自定义 I/O 上下文的 API**

```cpp
// 分配 I/O 缓冲区
int buffer_size = 32768;  // 32KB
uint8_t* buffer = (uint8_t*)av_malloc(buffer_size);

// avio_alloc_context: 创建自定义 AVIOContext
AVIOContext* avio_ctx = avio_alloc_context(
    buffer,          // I/O 缓冲区（FFmpeg 用来缓冲读取的数据）
    buffer_size,     // 缓冲区大小
    0,               // 0 = 只读，1 = 可写
    my_stream,       // opaque 指针（传给回调函数的第一个参数）
    read_callback,   // 读取回调
    nullptr,         // 写入回调（只读时传 nullptr）
    seek_callback);  // Seek 回调

// 将自定义 I/O 设置到格式上下文
AVFormatContext* fmt_ctx = avformat_alloc_context();
fmt_ctx->pb = avio_ctx;  // 设置 I/O 上下文

// 然后正常调用 avformat_open_input，URL 参数传 nullptr
avformat_open_input(&fmt_ctx, nullptr, nullptr, nullptr);
```

KrKr2 的 `DemuxFFmpeg.cpp` 正是这样实现的——`opaque` 指向 COM 的 `IStream` 接口，回调函数通过 `IStream::Read()` 和 `IStream::Seek()` 从 XP3 归档中读取媒体数据。

</details>
