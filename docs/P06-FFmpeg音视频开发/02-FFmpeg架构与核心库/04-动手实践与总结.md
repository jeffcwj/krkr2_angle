## 一、实践项目：ffprobe 迷你工具

### 1.1 项目目标

FFmpeg 自带一个命令行工具 `ffprobe`，可以查看任意媒体文件的详细信息（容器格式、流数量、编解码器、分辨率、帧率、比特率等）。我们将使用 FFmpeg 的 C API 实现一个简化版的 ffprobe，作为本章的综合实践。

这个项目会用到本章学过的三个核心概念：
- **libavformat**：打开文件、解析容器格式、获取流信息
- **libavcodec**：查询编解码器名称和参数
- **libavutil**：时间基转换（`av_q2d`，将 `AVRational`（有理数，即分数形式的数值）转为浮点数）、错误码转文字（`av_strerror`）

### 1.4 完整源码（main.cpp）

```cpp
#include <cstdio>
#include <cstdlib>

extern "C" {
#include <libavformat/avformat.h>  // 容器格式操作
#include <libavcodec/avcodec.h>    // 编解码器查询
#include <libavutil/avutil.h>      // 工具函数
}

// 辅助函数：将媒体类型枚举转为中文字符串
// AVMediaType 是 FFmpeg 定义的流类型枚举，
// 包含 AVMEDIA_TYPE_VIDEO、AVMEDIA_TYPE_AUDIO 等值
const char* media_type_to_string(enum AVMediaType type) {
    switch (type) {
        case AVMEDIA_TYPE_VIDEO:    return "视频";
        case AVMEDIA_TYPE_AUDIO:    return "音频";
        case AVMEDIA_TYPE_SUBTITLE: return "字幕";
        case AVMEDIA_TYPE_DATA:     return "数据";
        default:                    return "未知";
    }
}

// 打印单个流的详细信息
// AVStream 代表容器中的一路流（视频流、音频流等）
// 每个流有自己的编解码参数（AVCodecParameters）和时间基（time_base）
void print_stream_info(AVFormatContext* fmt_ctx, int stream_idx) {
    // fmt_ctx->streams 是一个 AVStream* 数组
    AVStream* stream = fmt_ctx->streams[stream_idx];

    // AVCodecParameters 存储流的编解码参数（分辨率、采样率等）
    // 注意：这里不是解码器本身，只是参数描述
    AVCodecParameters* par = stream->codecpar;

    // av_get_media_type_string 将枚举转为英文字符串（如 "video"）
    const char* type = media_type_to_string(par->codec_type);

    // avcodec_get_name 通过编解码器 ID 获取名称（如 "h264"）
    // AVCodecID 是 FFmpeg 定义的编解码器标识符枚举
    const char* codec_name = avcodec_get_name(par->codec_id);

    printf("\n  ┌── 流 #%d ──────────────────────────\n", stream_idx);
    printf("  │ 类型:       %s\n", type);
    printf("  │ 编解码器:   %s\n", codec_name);

    // time_base（时间基）是一个分数，代表时间戳的单位
    // 例如 time_base = 1/90000 表示每个时间戳单位 = 1/90000 秒
    printf("  │ 时间基:     %d/%d\n",
           stream->time_base.num,   // 分子
           stream->time_base.den);  // 分母

    // duration 以 time_base 为单位存储
    // AV_NOPTS_VALUE 表示"未知"（Not a Presentation Time Stamp）
    if (stream->duration != AV_NOPTS_VALUE) {
        // av_q2d 将 AVRational（分数）转为 double
        // duration * av_q2d(time_base) = 秒数
        double duration_sec = stream->duration * av_q2d(stream->time_base);
        printf("  │ 时长:       %.2f 秒\n", duration_sec);
    } else {
        printf("  │ 时长:       未知\n");
    }

    // 视频流特有信息
    if (par->codec_type == AVMEDIA_TYPE_VIDEO) {
        printf("  │ 分辨率:     %dx%d\n", par->width, par->height);

        // AVPixelFormat 是像素格式枚举（如 AV_PIX_FMT_YUV420P）
        // av_get_pix_fmt_name 转为可读字符串（如 "yuv420p"）
        const char* pix_fmt = av_get_pix_fmt_name(
            (AVPixelFormat)par->format);
        printf("  │ 像素格式:   %s\n", pix_fmt ? pix_fmt : "未知");

        // avg_frame_rate 是平均帧率，以 AVRational 表示
        if (stream->avg_frame_rate.den != 0) {
            printf("  │ 帧率:       %.2f fps\n",
                   av_q2d(stream->avg_frame_rate));
        }

        if (par->bit_rate > 0) {
            printf("  │ 比特率:     %lld kbps\n",
                   (long long)par->bit_rate / 1000);
        }
    }

    // 音频流特有信息
    if (par->codec_type == AVMEDIA_TYPE_AUDIO) {
        printf("  │ 采样率:     %d Hz\n", par->sample_rate);

        // ch_layout 是声道布局结构体（FFmpeg 5.1+ 引入）
        // nb_channels 表示声道数量
        printf("  │ 声道数:     %d\n", par->ch_layout.nb_channels);

        // AVSampleFormat 是音频采样格式枚举（如 AV_SAMPLE_FMT_FLTP）
        const char* sample_fmt = av_get_sample_fmt_name(
            (AVSampleFormat)par->format);
        printf("  │ 采样格式:   %s\n", sample_fmt ? sample_fmt : "未知");

        if (par->bit_rate > 0) {
            printf("  │ 比特率:     %lld kbps\n",
                   (long long)par->bit_rate / 1000);
        }
    }

    // 字幕流特有信息
    if (par->codec_type == AVMEDIA_TYPE_SUBTITLE) {
        printf("  │ 字幕格式:   %s\n", codec_name);
    }

    printf("  └─────────────────────────────────\n");
}

int main(int argc, char* argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件路径>\n", argv[0]);
        fprintf(stderr, "示例: %s video.mp4\n", argv[0]);
        return 1;
    }

    const char* filename = argv[1];

    // ========== 步骤 1：打开文件 ==========
    // AVFormatContext 是 libavformat 的核心结构体
    // 代表一个多媒体文件的完整上下文（格式、流、元数据等）
    AVFormatContext* fmt_ctx = nullptr;  // 必须初始化为 nullptr

    // avformat_open_input 打开文件并解析容器头部
    // 参数：上下文指针的指针、文件路径、输入格式(nullptr=自动检测)、选项(nullptr=默认)
    // 返回值：0 表示成功，负值表示错误码
    int ret = avformat_open_input(&fmt_ctx, filename, nullptr, nullptr);
    if (ret < 0) {
        // av_strerror 将错误码转为可读字符串
        // AV_ERROR_MAX_STRING_SIZE 是错误字符串的最大长度（64字节）
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "❌ 无法打开 '%s': %s\n", filename, errbuf);
        return 1;
    }

    // ========== 步骤 2：获取流信息 ==========
    // avformat_find_stream_info 读取足够多的数据来分析每个流的参数
    // 这一步会实际读取部分数据包来确定编解码器参数
    // 对于某些格式（如 MPEG-TS），这一步是必须的，否则流参数不完整
    ret = avformat_find_stream_info(fmt_ctx, nullptr);
    if (ret < 0) {
        fprintf(stderr, "❌ 无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // ========== 步骤 3：打印文件级信息 ==========
    printf("╔══════════════════════════════════════╗\n");
    printf("║          媒体文件信息                ║\n");
    printf("╠══════════════════════════════════════╣\n");
    printf("  文件:   %s\n", filename);

    // iformat 指向输入格式描述符（AVInputFormat）
    // name 是短名称（如 "mov,mp4,m4a,3gp,3g2,mj2"）
    // long_name 是可读名称（如 "QuickTime / MOV"）
    printf("  格式:   %s (%s)\n",
           fmt_ctx->iformat->name,
           fmt_ctx->iformat->long_name);

    // duration 以 AV_TIME_BASE（1000000，即微秒）为单位
    if (fmt_ctx->duration != AV_NOPTS_VALUE) {
        double total_sec = fmt_ctx->duration / (double)AV_TIME_BASE;
        int hours   = (int)(total_sec / 3600);
        int minutes = ((int)(total_sec / 60)) % 60;
        int seconds = ((int)total_sec) % 60;
        printf("  总时长: %02d:%02d:%02d (%.2f 秒)\n",
               hours, minutes, seconds, total_sec);
    }

    if (fmt_ctx->bit_rate > 0) {
        printf("  比特率: %lld kbps\n",
               (long long)fmt_ctx->bit_rate / 1000);
    }

    // nb_streams 是流的数量
    printf("  流数量: %u\n", fmt_ctx->nb_streams);

    // ========== 步骤 4：打印每个流的信息 ==========
    for (unsigned i = 0; i < fmt_ctx->nb_streams; i++) {
        print_stream_info(fmt_ctx, i);
    }

    // ========== 步骤 5：打印元数据 ==========
    printf("\n  ┌── 元数据 ──────────────────────────\n");
    // AVDictionary 是 FFmpeg 的键值对容器
    // av_dict_iterate 遍历字典中的所有条目
    const AVDictionaryEntry* tag = nullptr;
    int meta_count = 0;
    while ((tag = av_dict_iterate(fmt_ctx->metadata, tag))) {
        printf("  │ %-12s = %s\n", tag->key, tag->value);
        meta_count++;
    }
    if (meta_count == 0) {
        printf("  │ (无元数据)\n");
    }
    printf("  └─────────────────────────────────\n");

    printf("╚══════════════════════════════════════╝\n");

    // ========== 步骤 6：清理资源 ==========
    // avformat_close_input 关闭文件并释放 AVFormatContext
    // 传入指针的指针，函数内部会将 fmt_ctx 置为 nullptr
    avformat_close_input(&fmt_ctx);

    return 0;
}
```

### 1.7 API 函数速查表

| 函数 | 所属库 | 作用 | 参数说明 |
|------|--------|------|---------|
| `avformat_open_input` | libavformat | 打开媒体文件 | `(ctx**, url, fmt, opts)` — ctx 必须初始化为 nullptr |

| `av_q2d` | libavutil | AVRational → double | `(q)` → num/den |

## 二、对照 KrKr2 项目源码

### 2.1 六个库的使用分布

KrKr2 项目中 FFmpeg 各库的使用情况，按源文件整理：

| 源文件 | 使用的 FFmpeg 库 | 主要功能 |
|--------|-----------------|----------|
| `cpp/core/movie/ffmpeg/DemuxFFmpeg.cpp` (1656行) | libavformat, libavutil | 解封装：通过自定义 AVIOContext 从 XP3 归档读取媒体数据，解析容器格式，逐包读取 AVPacket |
| `cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp` (1112行) | libavcodec, libswscale, libavfilter, libavutil | 视频解码：H.264/VP9 等解码 + YUV→BGRA 像素格式转换 + 反交错（Deinterlace）滤镜 |
| `cpp/core/movie/ffmpeg/AudioCodecFFmpeg.cpp` (359行) | libavcodec, libswresample, libavutil | 音频解码：AAC/MP3 等解码 + 重采样为统一输出格式 |
| `cpp/core/movie/ffmpeg/VideoPlayer.cpp` (2906行) | libavutil | 主播放循环：协调解封装、视频解码、音频解码、渲染的多线程工作流 |
| `cpp/core/movie/ffmpeg/Clock.cpp` (275行) | libavutil | 时钟管理：使用 `av_rescale_q`（时间基缩放函数）实现 PTS 时间转换 |
| `cpp/core/sound/FFWaveDecoder.cpp` (451行) | libavformat, libavcodec, libswresample, libavutil | 独立音频解码器：用于 BGM（背景音乐）和 SE（音效）的解码播放 |

### 2.2 关键发现

**库使用模式**：
- **libavutil** 被所有模块使用——它是基础设施层（错误处理、时间转换、内存管理）
- **视频播放**使用了全部 6 个库（不含 libavdevice）
- **音频解码**使用了 4 个库（不需要 libswscale 图像缩放和 libavfilter 滤镜）
- **VideoPlayer.cpp** 是最大的文件（2906行），但只直接使用 libavutil——它通过调用其他类间接使用其他库

**API 风格**：
- KrKr2 全部使用新式 API（`avcodec_send_packet` / `avcodec_receive_frame`），不使用已废弃的 `avcodec_decode_video2` 等旧 API
- 自定义 I/O 通过 `avio_alloc_context` 实现，callback 函数读取 XP3 归档中的 IStream 数据

### 3.2 API 层级与依赖关系

```
              ┌────────────┐
              │ libavfilter│ ← 最高层：滤镜处理
              └──────┬─────┘
                     │ 依赖
              ┌──────┴─────┐
              │ libavformat│ ← 容器格式读写
              └──────┬─────┘
                     │ 依赖
              ┌──────┴─────┐
              │ libavcodec │ ← 编解码器
              └──────┬─────┘
                     │ 依赖
         ┌───────────┼───────────┐
         │           │           │
   ┌─────┴──────┐ ┌──┴───────┐ ┌┴──────────┐
   │ libswscale │ │libsw-    │ │ libavutil  │ ← 最底层
   │ (图像缩放) │ │resample  │ │ (基础工具) │
   └────────────┘ │(音频重采)│ └────────────┘
                  └──────────┘
```

**链接顺序规则**：静态链接时，高层在前，底层在后。即：
`avfilter → avformat → avcodec → swscale → swresample → avutil`

### 3.3 关键数据结构一览

| 结构体 | 所属库 | 作用 | 类比 |
|--------|--------|------|------|
| `AVFormatContext` | libavformat | 多媒体文件的完整上下文 | 一本书的目录和元信息 |
| `AVStream` | libavformat | 文件中的一路流（视频/音频/字幕） | 书的一个章节 |
| `AVPacket` | libavcodec | 一个压缩数据包 | 一页密文 |
| `AVFrame` | libavutil | 一帧解码后的原始数据 | 密文翻译后的明文 |
| `AVCodecContext` | libavcodec | 编解码器的工作上下文 | 翻译器的工作台 |
| `AVCodecParameters` | libavcodec | 流的编解码参数（只读描述） | 翻译器的说明书 |
| `AVIOContext` | libavformat | 自定义 I/O 上下文 | 自定义的书页翻页器 |
| `AVRational` | libavutil | 有理数（分数），用于时间基 | 分数 num/den |
| `SwsContext` | libswscale | 图像缩放/转换上下文 | 图片格式转换器 |
| `SwrContext` | libswresample | 音频重采样上下文 | 音频格式转换器 |

## 常见错误及解决方案

### 错误 1：AVFormatContext 未初始化为 nullptr

```cpp
AVFormatContext* fmt_ctx;  // ❌ 未初始化
avformat_open_input(&fmt_ctx, "test.mp4", nullptr, nullptr);
```

**现象**：程序崩溃（Segmentation Fault）或返回奇怪的错误码。

**原因**：`avformat_open_input` 检查 `*fmt_ctx` 是否为 nullptr。如果不是，它认为你已经分配了一个上下文并尝试使用，但未初始化的指针指向随机内存。

**解决**：

```cpp
AVFormatContext* fmt_ctx = nullptr;  // ✅ 始终初始化为 nullptr
```

### 错误 2：使用栈上的 AVPacket 并且不 unref

```cpp
AVPacket pkt;                         // ❌ 栈上分配（旧式用法）
while (av_read_frame(fmt_ctx, &pkt) >= 0) {
    // 处理数据...
    // ❌ 缺少 av_packet_unref(&pkt)——每次循环都内存泄漏
}
```

**现象**：程序内存持续增长，最终 OOM（Out of Memory，内存不足）崩溃。

**原因**：每次 `av_read_frame` 成功后，`pkt` 内部持有一个数据缓冲区的引用。不调用 `av_packet_unref` 会导致该缓冲区永远无法释放。

**解决**：使用堆分配 + 循环内 unref：

```cpp
AVPacket* pkt = av_packet_alloc();     // ✅ 堆分配（现代推荐方式）
while (av_read_frame(fmt_ctx, pkt) >= 0) {
    // 处理数据...
    av_packet_unref(pkt);              // ✅ 释放包内数据
}
av_packet_free(&pkt);                  // ✅ 释放 AVPacket 结构本身
```

### 错误 3：不检查 avformat_open_input 的返回值

```cpp
avformat_open_input(&fmt_ctx, filename, nullptr, nullptr);
avformat_find_stream_info(fmt_ctx, nullptr);  // ❌ 如果上一步失败，这里崩溃
```

**现象**：文件不存在或格式不支持时直接崩溃，没有任何错误提示。

**解决**：始终检查返回值并优雅处理错误：

```cpp
int ret = avformat_open_input(&fmt_ctx, filename, nullptr, nullptr);
if (ret < 0) {
    char errbuf[AV_ERROR_MAX_STRING_SIZE];
    av_strerror(ret, errbuf, sizeof(errbuf));
    fprintf(stderr, "打开失败: %s\n", errbuf);
    return 1;
}
```

### 题目 2：修复有三处错误的代码

以下代码有 3 处错误，找出并修正：

```cpp
#include <libavformat/avformat.h>

int main() {
    AVFormatContext* fmt_ctx;                                // 错误 1
    avformat_open_input(&fmt_ctx, "test.mp4", NULL, NULL);
    avformat_find_stream_info(fmt_ctx, NULL);

    AVPacket pkt;                                            // 错误 2
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        printf("stream=%d size=%d\n", pkt.stream_index, pkt.size);
        // 错误 3：缺少什么？
    }

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

<details>
<summary>查看答案</summary>

**错误 1**：`fmt_ctx` 未初始化为 `nullptr`。

```cpp
AVFormatContext* fmt_ctx = nullptr;  // ✅ 必须初始化
```

**错误 2**：`AVPacket` 在栈上声明是旧式用法。现代 FFmpeg（≥4.0）推荐使用 `av_packet_alloc` 在堆上分配，因为 `av_init_packet`（栈上 AVPacket 的初始化函数）在新版本中已废弃。

```cpp
AVPacket* pkt = av_packet_alloc();  // ✅ 堆分配
// 循环中使用 pkt 而非 &pkt
// 循环后调用 av_packet_free(&pkt);
```

**错误 3**：循环内缺少 `av_packet_unref(pkt)`。每次 `av_read_frame` 成功后，`pkt` 持有数据缓冲区的引用，不释放会持续泄漏内存。

完整修正版：

```cpp
#include <cstdio>
extern "C" {
#include <libavformat/avformat.h>
}

int main() {
    AVFormatContext* fmt_ctx = nullptr;

    int ret = avformat_open_input(&fmt_ctx, "test.mp4", nullptr, nullptr);
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "打开失败: %s\n", errbuf);
        return 1;
    }

    if (avformat_find_stream_info(fmt_ctx, nullptr) < 0) {
        fprintf(stderr, "获取流信息失败\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    AVPacket* pkt = av_packet_alloc();
    while (av_read_frame(fmt_ctx, pkt) >= 0) {
        printf("stream=%d size=%d\n", pkt->stream_index, pkt->size);
        av_packet_unref(pkt);  // ✅ 每次循环释放包数据
    }

    av_packet_free(&pkt);
    avformat_close_input(&fmt_ctx);
    return 0;
}
```

</details>

### 题目 3：设计题——KrKr2 自定义 I/O

KrKr2 的媒体文件存储在 XP3 归档包（KiriKiri 引擎的专有打包格式）中，无法用文件路径直接打开。请回答：

1. FFmpeg 提供了什么机制来支持自定义数据源？
2. 需要实现哪些回调函数？
3. 使用哪个 API 创建自定义 I/O 上下文？
4. 创建后如何与 `avformat_open_input` 配合使用？

<details>
<summary>查看答案</summary>

**1. 自定义 I/O 机制**

FFmpeg 通过 `AVIOContext`（自定义 I/O 上下文）支持从任意数据源读取。用户注册回调函数，FFmpeg 调用这些回调来读写数据，完全绕过文件系统。

**2. 需要实现的回调函数**

```cpp
// 读取回调：从自定义数据源读取 buf_size 字节到 buf
// 返回值：实际读取的字节数，EOF 时返回 AVERROR_EOF
int read_callback(void* opaque, uint8_t* buf, int buf_size);

// Seek 回调：在数据源中定位
// whence 可以是 SEEK_SET/SEEK_CUR/SEEK_END 或 AVSEEK_SIZE（返回总大小）
// 返回值：新的位置，或错误码
int64_t seek_callback(void* opaque, int64_t offset, int whence);
```

- `opaque`：用户自定义指针，通常指向你的数据源对象
- 读取回调必须实现；Seek 回调可选但强烈建议实现（否则无法随机访问）

**3. 创建自定义 I/O 上下文**

```cpp
// 分配 I/O 缓冲区（FFmpeg 通过此缓冲区与回调交互）
int buffer_size = 32768;  // 32KB，典型大小
uint8_t* buffer = (uint8_t*)av_malloc(buffer_size);

// 创建 AVIOContext
AVIOContext* avio_ctx = avio_alloc_context(
    buffer,          // I/O 缓冲区
    buffer_size,     // 缓冲区大小
    0,               // 0 = 只读，1 = 可写
    my_stream_ptr,   // opaque（传给回调的 void*，指向你的数据源）
    read_callback,   // 读取回调函数指针
    nullptr,         // 写入回调（只读时为 nullptr）
    seek_callback    // Seek 回调函数指针
);
```

**4. 与 avformat_open_input 配合**

```cpp
// 手动分配 AVFormatContext（不能让 avformat_open_input 自动分配）
AVFormatContext* fmt_ctx = avformat_alloc_context();
fmt_ctx->pb = avio_ctx;  // 设置自定义 I/O

// 调用 avformat_open_input，URL 传 nullptr（因为数据来自回调而非文件路径）
int ret = avformat_open_input(&fmt_ctx, nullptr, nullptr, nullptr);
```

KrKr2 的 `DemuxFFmpeg.cpp` 正是这样做的——`opaque` 指向 COM 的 `IStream` 接口，回调函数通过 `IStream::Read()` 和 `IStream::Seek()` 读取 XP3 归档中的数据。

</details>
