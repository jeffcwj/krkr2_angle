## 3.1 什么是解封装（Demuxing）

### 3.1.1 容器与编码的关系

第1章我们学过，一个视频文件通常包含多个**流**（Stream）：视频流、音频流、
字幕流等。这些流被"打包"在一个**容器格式**（Container Format）中。

```
┌─────────────────────────────────────────────────────┐
│                  MP4 容器文件                         │
│                                                      │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐│
│  │ 视频流 #0    │  │ 音频流 #1    │  │ 字幕流 #2    ││
│  │ H.264 编码   │  │ AAC 编码     │  │ SRT 文本     ││
│  │              │  │              │  │              ││
│  │ [pkt][pkt]   │  │ [pkt][pkt]   │  │ [pkt][pkt]  ││
│  │ [pkt][pkt]   │  │ [pkt][pkt]   │  │             ││
│  └─────────────┘  └─────────────┘  └──────────────┘│
│                                                      │
│  文件头：流信息、时间信息、元数据                       │
│  索引表：快速定位到任意时间位置                         │
└─────────────────────────────────────────────────────┘
```

**解封装**（Demuxing / Demultiplexing）就是从容器文件中把各个流的数据包
（Packet）**分离提取**出来的过程：

```
MP4文件 ──解封装──→ 视频 AVPacket 序列
                  → 音频 AVPacket 序列
                  → 字幕 AVPacket 序列
```

### 3.1.2 常见容器格式对比

| 容器格式 | 扩展名 | 视频编码支持 | 音频编码支持 | 特点 |
|----------|--------|-------------|-------------|------|
| MP4/M4V | .mp4, .m4v | H.264, H.265, VP9 | AAC, MP3, Opus | 最通用，浏览器/移动端首选 |
| MKV | .mkv | 几乎所有 | 几乎所有 | 功能最全，字幕支持好 |
| AVI | .avi | 几乎所有 | 几乎所有 | 古老格式，索引在文件尾部 |
| FLV | .flv | H.264, VP6 | AAC, MP3 | 网络流媒体常用 |
| TS | .ts | H.264, H.265 | AAC, AC3, MP2 | 广播/直播标准，可边录边播 |
| OGG | .ogg, .ogv | Theora, VP8 | Vorbis, Opus | 开源格式 |
| WebM | .webm | VP8, VP9, AV1 | Vorbis, Opus | 网页视频标准 |

> **KrKr2 中常见的视频格式**：游戏 OP/ED 通常使用 **AVI（MPEG4/WMV）** 或 **MP4（H.264）**。
> KrKr2 的 `CDVDDemuxFFmpeg` 通过 FFmpeg 的 `libavformat` 自动探测格式，
> 无需硬编码支持列表。

### 3.1.3 为什么需要解封装？

你可能会问：为什么不把视频数据直接存为原始流？

1. **多流复用**：一个文件同时包含音视频，播放器只需打开一个文件
2. **时间戳同步**：容器记录每个包的 PTS/DTS，播放器据此同步音视频
3. **随机访问**：容器的索引表允许快速跳转到任意时间位置（Seek）
4. **元数据**：标题、作者、专辑封面等信息存储在容器头部
5. **流选择**：多音轨/多字幕轨，用户可以动态切换

## 3.2 解封装核心数据结构

### 3.2.1 AVFormatContext —— 格式上下文

`AVFormatContext` 是 `libavformat` 的核心结构体，代表一个打开的媒体文件：

```c
// AVFormatContext 关键字段（简化版）
typedef struct AVFormatContext {
    const AVInputFormat  *iformat;     // 输入格式（如 "mov,mp4,m4a,3gp"）
    const AVOutputFormat *oformat;     // 输出格式（封装时使用）

    AVIOContext  *pb;                  // I/O 上下文（底层读写）

    unsigned int  nb_streams;          // 流的数量
    AVStream    **streams;             // 流数组（下标即 stream_index）

    int64_t       duration;            // 文件总时长（AV_TIME_BASE 单位，即微秒）
    int64_t       bit_rate;            // 总码率（bits/second）
    int64_t       start_time;          // 第一帧的时间戳

    AVDictionary *metadata;            // 元数据（标题、作者等）
} AVFormatContext;
```

> **生命周期**：
> `avformat_alloc_context()` → `avformat_open_input()` → 使用 → `avformat_close_input()`

### 3.2.2 AVStream —— 流信息

每个 `AVStream` 描述文件中的一路流：

```c
typedef struct AVStream {
    int            index;              // 流索引号
    AVCodecParameters *codecpar;       // 编解码参数（codec_type, codec_id, width, height...）
    AVRational     time_base;          // 时间基准（如 {1, 90000} 表示 1/90000 秒）
    int64_t        duration;           // 流时长（time_base 单位）
    int64_t        nb_frames;          // 帧数（可能为 0，表示未知）
    AVRational     avg_frame_rate;     // 平均帧率
    AVDictionary  *metadata;           // 流级元数据（语言、标题等）
} AVStream;
```

**`codecpar` 中的关键字段**：

```c
typedef struct AVCodecParameters {
    enum AVMediaType codec_type;       // AVMEDIA_TYPE_VIDEO / AVMEDIA_TYPE_AUDIO / ...
    enum AVCodecID   codec_id;         // AV_CODEC_ID_H264 / AV_CODEC_ID_AAC / ...
    int              width, height;    // 视频分辨率
    int              sample_rate;      // 音频采样率
    int              channels;         // 音频声道数
    uint64_t         channel_layout;   // 声道布局
    enum AVSampleFormat format;        // 音频采样格式（仅音频）
    enum AVPixelFormat  format;        // 像素格式（仅视频，与上面共用字段名）
    uint8_t         *extradata;        // 解码器初始化数据（SPS/PPS 等）
    int              extradata_size;
} AVCodecParameters;
```

### 3.2.3 AVPacket —— 压缩数据包

`AVPacket` 是解封装输出的基本单元，包含一个流的一段压缩数据：

```c
typedef struct AVPacket {
    int64_t  pts;              // 显示时间戳（Presentation Timestamp）
    int64_t  dts;              // 解码时间戳（Decoding Timestamp）
    uint8_t *data;             // 压缩数据指针
    int      size;             // 数据大小（字节）
    int      stream_index;     // 所属流的索引号
    int      flags;            // 标志位（AV_PKT_FLAG_KEY = 关键帧）
    int64_t  duration;         // 包的时长（stream time_base 单位）
    int64_t  pos;              // 在文件中的字节偏移
} AVPacket;
```

> **PTS vs DTS**：
> - **PTS**（Presentation Timestamp）：帧应该被**显示**的时间
> - **DTS**（Decoding Timestamp）：帧应该被**解码**的时间
> - 对于无 B 帧的视频（如很多游戏视频），PTS == DTS
> - 有 B 帧时，DTS ≤ PTS（因为 B 帧需要先解码后面的 P 帧）

### 3.2.4 三者的关系

```
AVFormatContext
├── streams[0]: AVStream (视频, H.264)
│   └── codecpar: AVCodecParameters
├── streams[1]: AVStream (音频, AAC)
│   └── codecpar: AVCodecParameters
└── av_read_frame() → AVPacket
    ├── stream_index = 0 → 视频包
    └── stream_index = 1 → 音频包
```

## 3.3 解封装完整流程

### 3.3.1 流程图

```
┌──────────────────────────────────────────────────┐
│            解封装（Demuxing）标准流程               │
│                                                   │
│  1. avformat_open_input()     ← 打开文件/URL      │
│         ↓                                         │
│  2. avformat_find_stream_info() ← 探测流信息      │
│         ↓                                         │
│  3. 遍历 streams[]            ← 找到目标流        │
│         ↓                                         │
│  4. av_read_frame() 循环      ← 逐包读取          │
│     ├── stream_index == video → 送视频解码器      │
│     ├── stream_index == audio → 送音频解码器      │
│     └── 其他                  → 忽略/字幕处理     │
│         ↓                                         │
│  5. av_packet_unref()         ← 释放包引用        │
│         ↓                                         │
│  6. avformat_close_input()    ← 关闭文件          │
└──────────────────────────────────────────────────┘
```

### 3.3.2 打开文件

```c
#include <libavformat/avformat.h>
#include <libavutil/avutil.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    const char *filename = "test.mp4";

    // 步骤 1：打开输入文件
    AVFormatContext *fmt_ctx = NULL;
    int ret = avformat_open_input(&fmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));   // FFmpeg 错误码转字符串
        fprintf(stderr, "无法打开文件 '%s': %s\n", filename, errbuf);
        return 1;
    }

    printf("成功打开文件: %s\n", filename);
    printf("格式: %s (%s)\n", fmt_ctx->iformat->name, fmt_ctx->iformat->long_name);

    // ... 后续操作

    avformat_close_input(&fmt_ctx);  // 步骤 6：关闭文件
    return 0;
}
```

> **`avformat_open_input` 做了什么？**
> 1. 根据文件扩展名或内容探测格式（probe）
> 2. 打开文件 I/O（本地文件/HTTP/RTSP 等）
> 3. 读取文件头部，解析基本信息
> 4. 分配并填充 `AVFormatContext`
>
> 第二个参数可以传 `NULL`（自动探测）或指定格式（如 `av_find_input_format("mp4")`）。
> 第四个参数是选项字典，可以设置超时、缓冲区大小等。

### 3.3.3 探测流信息

```c
    // 步骤 2：探测流信息
    ret = avformat_find_stream_info(fmt_ctx, NULL);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // 打印详细信息（调试利器，等同于 ffprobe 输出）
    av_dump_format(fmt_ctx, 0, filename, 0);

    // 显示基本信息
    printf("\n流的数量: %u\n", fmt_ctx->nb_streams);
    printf("总时长: %.2f 秒\n", fmt_ctx->duration / (double)AV_TIME_BASE);
    printf("总码率: %"PRId64" kbps\n", fmt_ctx->bit_rate / 1000);
```

> **为什么需要 `avformat_find_stream_info`？**
>
> `avformat_open_input` 只读取文件头部，对于某些格式（如 MPEG-TS），
> 头部信息不完整，需要实际读取几秒数据来"探测"：
> - 实际的编码器参数（分辨率、采样率）
> - 准确的流时长
> - 帧率估算
>
> 这个函数可能耗时较长（尤其是网络流），因为它会读取并解码前几帧。

### 3.3.4 遍历流

```c
    // 步骤 3：遍历所有流，找到音视频流
    int video_stream_idx = -1;
    int audio_stream_idx = -1;

    for (unsigned int i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream *stream = fmt_ctx->streams[i];
        AVCodecParameters *par = stream->codecpar;

        // 根据类型分类
        const char *type_name;
        switch (par->codec_type) {
        case AVMEDIA_TYPE_VIDEO:
            type_name = "视频";
            if (video_stream_idx < 0) video_stream_idx = i;
            printf("  流 #%u [%s]: %s, %dx%d, %.2f fps\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id),
                   par->width, par->height,
                   av_q2d(stream->avg_frame_rate));
            break;

        case AVMEDIA_TYPE_AUDIO:
            type_name = "音频";
            if (audio_stream_idx < 0) audio_stream_idx = i;
            printf("  流 #%u [%s]: %s, %d Hz, %d 声道\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id),
                   par->sample_rate, par->channels);
            break;

        case AVMEDIA_TYPE_SUBTITLE:
            type_name = "字幕";
            printf("  流 #%u [%s]: %s\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id));
            break;

        default:
            printf("  流 #%u [其他]: type=%d\n", i, par->codec_type);
            break;
        }

        // 打印时间基准
        printf("         time_base: %d/%d\n",
               stream->time_base.num, stream->time_base.den);
    }

    if (video_stream_idx < 0) {
        fprintf(stderr, "警告：未找到视频流\n");
    }
    if (audio_stream_idx < 0) {
        fprintf(stderr, "警告：未找到音频流\n");
    }
```

> **`av_find_best_stream` 快捷方式**：
> FFmpeg 提供了一个辅助函数，自动选择"最佳"流：
> ```c
> int idx = av_find_best_stream(fmt_ctx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
> ```
> "最佳"的标准是：分辨率最高的视频流、声道数最多的音频流。
> KrKr2 的 `CDVDDemuxFFmpeg::Open()` 内部使用类似逻辑选择默认流。

## 3.4 逐包读取 —— av_read_frame

### 3.4.1 读取循环

```c
    // 步骤 4：逐包读取
    AVPacket pkt;
    int packet_count = 0;
    int video_packets = 0, audio_packets = 0;
    int64_t video_bytes = 0, audio_bytes = 0;

    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        packet_count++;

        if (pkt.stream_index == video_stream_idx) {
            video_packets++;
            video_bytes += pkt.size;

            // 前 5 个视频包的详细信息
            if (video_packets <= 5) {
                AVStream *s = fmt_ctx->streams[video_stream_idx];
                double pts_sec = pkt.pts * av_q2d(s->time_base);
                printf("视频包 #%d: size=%d, pts=%.3fs, dts=%.3fs, key=%s\n",
                       video_packets, pkt.size,
                       pts_sec,
                       pkt.dts * av_q2d(s->time_base),
                       (pkt.flags & AV_PKT_FLAG_KEY) ? "是" : "否");
            }
        } else if (pkt.stream_index == audio_stream_idx) {
            audio_packets++;
            audio_bytes += pkt.size;
        }

        // 步骤 5：释放包引用（重要！否则内存泄漏）
        av_packet_unref(&pkt);
    }

    printf("\n── 统计 ──\n");
    printf("总包数: %d\n", packet_count);
    printf("视频包: %d (%.2f MB)\n", video_packets, video_bytes / 1048576.0);
    printf("音频包: %d (%.2f MB)\n", audio_packets, audio_bytes / 1048576.0);
```

### 3.4.2 AVPacket 生命周期

```
┌─────────────────────────────────────────┐
│         AVPacket 生命周期                 │
│                                          │
│  av_read_frame(&pkt)                     │
│      │  ← pkt 现在持有数据引用           │
│      ↓                                   │
│  使用 pkt.data / pkt.size / pkt.pts      │
│      │  ← 可以送给解码器                 │
│      ↓                                   │
│  av_packet_unref(&pkt)                   │
│      │  ← 释放引用，计数-1               │
│      ↓                                   │
│  下一轮 av_read_frame()                  │
│                                          │
│  ⚠️ 如果需要保存包（如放入队列）：        │
│  av_packet_clone(&pkt) 或                │
│  av_packet_ref(dst, &pkt)                │
│      └─ 引用计数+1，数据不会被释放       │
└─────────────────────────────────────────┘
```

> **常见内存泄漏**：忘记调用 `av_packet_unref`。每个 `av_read_frame` 成功后
> 都**必须**配对一个 `av_packet_unref`（或将所有权转移给其他对象）。
>
> KrKr2 中 `CDVDDemuxFFmpeg::Read()` 在读取后将包的所有权转移给
> `CDVDDemuxerPacket`，由消息队列负责最终释放。

### 3.4.3 时间戳转换

FFmpeg 中的时间戳不是秒，而是以各流的 `time_base` 为单位的整数值：

```c
// 时间戳转换公式
// 秒 = pts * time_base.num / time_base.den
// 简化为：
double seconds = pkt.pts * av_q2d(stream->time_base);

// av_q2d 等价于：
// double av_q2d(AVRational a) { return a.num / (double)a.den; }
```

**常见 time_base 值**：

| 格式 | 视频 time_base | 音频 time_base |
|------|---------------|---------------|
| MP4 | 1/90000 | 1/44100 |
| MKV | 1/1000 | 1/1000 |
| AVI | 1/fps (如 1/30) | 1/sample_rate |
| TS | 1/90000 | 1/90000 |

```c
// 示例：将 time_base 不同的两个时间戳做比较
int64_t video_pts = 90000;   // time_base = 1/90000 → 1秒
int64_t audio_pts = 44100;   // time_base = 1/44100 → 1秒

// 用 av_rescale_q 安全转换（避免溢出）
int64_t audio_in_video_tb = av_rescale_q(
    audio_pts,
    fmt_ctx->streams[audio_stream_idx]->time_base,  // 源 time_base
    fmt_ctx->streams[video_stream_idx]->time_base    // 目标 time_base
);
printf("音频时间在视频时间基准下: %"PRId64"\n", audio_in_video_tb);
// 输出：90000（即 1 秒）
```

> **`av_rescale_q` 是安全的**：它内部使用 128 位整数运算避免溢出，
> 比手动计算 `pts * src.num * dst.den / (src.den * dst.num)` 安全得多。

## 3.5 完整示例：媒体信息探测器

以下是一个完整可编译的程序，类似简化版的 `ffprobe`：

### 3.5.1 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.16)
project(media_probe C)

set(CMAKE_C_STANDARD 11)

find_package(PkgConfig REQUIRED)
pkg_check_modules(LIBAV REQUIRED libavformat libavcodec libavutil)

add_executable(media_probe media_probe.c)
target_include_directories(media_probe PRIVATE ${LIBAV_INCLUDE_DIRS})
target_link_libraries(media_probe PRIVATE ${LIBAV_LIBRARIES})

# Windows 补充链接库
if(WIN32)
    target_link_libraries(media_probe PRIVATE ws2_32 Secur32 Bcrypt)
endif()
```

### 3.5.2 media_probe.c

```c
#include <libavformat/avformat.h>
#include <libavcodec/avcodec.h>
#include <libavutil/avutil.h>
#include <libavutil/dict.h>
#include <stdio.h>
#include <inttypes.h>

// 辅助：格式化时长为 HH:MM:SS.ms
static void format_duration(int64_t duration_us, char *buf, size_t buf_size) {
    if (duration_us == AV_NOPTS_VALUE) {
        snprintf(buf, buf_size, "N/A");
        return;
    }
    int64_t total_sec = duration_us / AV_TIME_BASE;
    int ms = (int)((duration_us % AV_TIME_BASE) * 1000 / AV_TIME_BASE);
    int h = (int)(total_sec / 3600);
    int m = (int)((total_sec % 3600) / 60);
    int s = (int)(total_sec % 60);
    snprintf(buf, buf_size, "%02d:%02d:%02d.%03d", h, m, s, ms);
}

// 辅助：打印元数据
static void print_metadata(AVDictionary *metadata, const char *indent) {
    AVDictionaryEntry *tag = NULL;
    while ((tag = av_dict_get(metadata, "", tag, AV_DICT_IGNORE_SUFFIX))) {
        printf("%s%s: %s\n", indent, tag->key, tag->value);
    }
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件>\n", argv[0]);
        return 1;
    }

    const char *filename = argv[1];

    // 旧版 FFmpeg 需要注册（新版自动注册）
    #if LIBAVFORMAT_VERSION_INT < AV_VERSION_INT(58, 9, 100)
    av_register_all();
    avformat_network_init();  // 支持网络协议
    #endif

    // ── 打开文件 ──
    AVFormatContext *fmt_ctx = NULL;
    int ret = avformat_open_input(&fmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "错误: 无法打开 '%s': %s\n", filename, errbuf);
        return 1;
    }

    // ── 探测流信息 ──
    if (avformat_find_stream_info(fmt_ctx, NULL) < 0) {
        fprintf(stderr, "错误: 无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // ── 文件级信息 ──
    char duration_str[64];
    format_duration(fmt_ctx->duration, duration_str, sizeof(duration_str));

    printf("═══════════════════════════════════════\n");
    printf("文件: %s\n", filename);
    printf("格式: %s (%s)\n", fmt_ctx->iformat->name,
           fmt_ctx->iformat->long_name);
    printf("时长: %s\n", duration_str);
    printf("码率: %"PRId64" kbps\n",
           fmt_ctx->bit_rate > 0 ? fmt_ctx->bit_rate / 1000 : 0);
    printf("流数: %u\n", fmt_ctx->nb_streams);

    // 文件级元数据
    if (fmt_ctx->metadata) {
        printf("\n── 元数据 ──\n");
        print_metadata(fmt_ctx->metadata, "  ");
    }

    // ── 逐流信息 ──
    printf("\n── 流信息 ──\n");
    for (unsigned int i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream *stream = fmt_ctx->streams[i];
        AVCodecParameters *par = stream->codecpar;

        printf("\n流 #%u:\n", i);

        switch (par->codec_type) {
        case AVMEDIA_TYPE_VIDEO:
            printf("  类型: 视频\n");
            printf("  编码: %s (profile: %d)\n",
                   avcodec_get_name(par->codec_id), par->profile);
            printf("  分辨率: %dx%d\n", par->width, par->height);
            printf("  像素格式: %s\n",
                   av_get_pix_fmt_name((enum AVPixelFormat)par->format));
            printf("  帧率: %.2f fps\n", av_q2d(stream->avg_frame_rate));
            printf("  码率: %"PRId64" kbps\n", par->bit_rate / 1000);
            break;

        case AVMEDIA_TYPE_AUDIO:
            printf("  类型: 音频\n");
            printf("  编码: %s\n", avcodec_get_name(par->codec_id));
            printf("  采样率: %d Hz\n", par->sample_rate);
            printf("  声道数: %d\n", par->channels);
            printf("  采样格式: %s\n",
                   av_get_sample_fmt_name((enum AVSampleFormat)par->format));
            printf("  码率: %"PRId64" kbps\n", par->bit_rate / 1000);
            break;

        case AVMEDIA_TYPE_SUBTITLE:
            printf("  类型: 字幕\n");
            printf("  编码: %s\n", avcodec_get_name(par->codec_id));
            break;

        default:
            printf("  类型: 其他 (%d)\n", par->codec_type);
        }

        printf("  time_base: %d/%d\n",
               stream->time_base.num, stream->time_base.den);

        char stream_dur[64];
        if (stream->duration != AV_NOPTS_VALUE) {
            int64_t dur_us = av_rescale_q(stream->duration,
                                          stream->time_base, AV_TIME_BASE_Q);
            format_duration(dur_us, stream_dur, sizeof(stream_dur));
        } else {
            snprintf(stream_dur, sizeof(stream_dur), "N/A");
        }
        printf("  时长: %s\n", stream_dur);

        if (stream->metadata) {
            printf("  元数据:\n");
            print_metadata(stream->metadata, "    ");
        }
    }

    // ── 包统计 ──
    printf("\n── 包统计 ──\n");
    AVPacket pkt;
    int total = 0;
    int per_stream[64] = {0};    // 假设最多 64 个流
    int key_frames[64] = {0};

    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        total++;
        if (pkt.stream_index < 64) {
            per_stream[pkt.stream_index]++;
            if (pkt.flags & AV_PKT_FLAG_KEY) {
                key_frames[pkt.stream_index]++;
            }
        }
        av_packet_unref(&pkt);
    }

    printf("总包数: %d\n", total);
    for (unsigned int i = 0; i < fmt_ctx->nb_streams && i < 64; i++) {
        printf("  流 #%u: %d 包 (关键帧: %d)\n",
               i, per_stream[i], key_frames[i]);
    }

    printf("═══════════════════════════════════════\n");

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

## 3.6 随机访问（Seek）

### 3.6.1 基本 Seek

```c
// 跳转到第 30 秒位置
int64_t target_ts = 30 * AV_TIME_BASE;  // 30秒，AV_TIME_BASE 单位
int ret = av_seek_frame(fmt_ctx, -1, target_ts, AVSEEK_FLAG_BACKWARD);
if (ret < 0) {
    fprintf(stderr, "Seek 失败\n");
}
```

**参数说明**：
- `stream_index = -1`：使用默认时间基准（AV_TIME_BASE = 微秒）
- `AVSEEK_FLAG_BACKWARD`：跳转到目标时间之前最近的关键帧

### 3.6.2 Seek 标志

| 标志 | 含义 |
|------|------|
| `AVSEEK_FLAG_BACKWARD` | 跳到目标之前的关键帧（最常用） |
| `AVSEEK_FLAG_BYTE` | 按字节偏移 Seek |
| `AVSEEK_FLAG_ANY` | 允许跳到非关键帧（可能导致花屏） |
| `AVSEEK_FLAG_FRAME` | 按帧号 Seek |

### 3.6.3 精确 Seek

`av_seek_frame` 只能跳到关键帧，如果需要精确到某一帧，需要：

```c
// 精确 Seek 到 target_sec 秒
void seek_precise(AVFormatContext *fmt_ctx, int stream_idx,
                  double target_sec) {
    AVStream *stream = fmt_ctx->streams[stream_idx];

    // 1. 转换目标时间到流的 time_base
    int64_t target_ts = (int64_t)(target_sec / av_q2d(stream->time_base));

    // 2. Seek 到目标之前的关键帧
    av_seek_frame(fmt_ctx, stream_idx, target_ts, AVSEEK_FLAG_BACKWARD);

    // 3. 读取并跳过帧，直到到达目标时间
    AVPacket pkt;
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        if (pkt.stream_index == stream_idx) {
            if (pkt.pts >= target_ts) {
                // 到达目标位置
                printf("精确 Seek 成功: pts=%"PRId64" (目标=%"PRId64")\n",
                       pkt.pts, target_ts);
                av_packet_unref(&pkt);
                break;
            }
            // 还没到目标，需要解码（不显示）来推进解码器状态
        }
        av_packet_unref(&pkt);
    }
}
```

> **为什么不能直接跳到非关键帧？**
> P/B 帧依赖前面的关键帧（I 帧）作为参考。如果直接从 P 帧开始解码，
> 解码器没有参考帧，会输出花屏（马赛克/绿屏）。
>
> KrKr2 的 `BasePlayer::SeekTime()`（VideoPlayer.cpp）在 Seek 后会调用
> `FlushBuffers()` 清空解码器缓冲区，然后从关键帧重新解码。

## 3.7 自定义 I/O —— 从内存/归档读取

FFmpeg 默认从文件系统读取，但 KrKr2 的游戏资源存储在 XP3 归档中，
需要自定义 I/O 来让 FFmpeg 从归档中读取视频数据。

### 3.7.1 AVIOContext 自定义

```c
#include <libavformat/avio.h>

// 自定义读取回调
static int custom_read(void *opaque, uint8_t *buf, int buf_size) {
    FILE *fp = (FILE *)opaque;
    int ret = (int)fread(buf, 1, buf_size, fp);
    if (ret == 0) return AVERROR_EOF;
    return ret;
}

// 自定义 Seek 回调
static int64_t custom_seek(void *opaque, int64_t offset, int whence) {
    FILE *fp = (FILE *)opaque;

    if (whence == AVSEEK_SIZE) {
        // 返回文件大小
        long pos = ftell(fp);
        fseek(fp, 0, SEEK_END);
        long size = ftell(fp);
        fseek(fp, pos, SEEK_SET);
        return size;
    }

    if (fseek(fp, (long)offset, whence) < 0) return -1;
    return ftell(fp);
}

int open_custom_io(const char *filename) {
    FILE *fp = fopen(filename, "rb");
    if (!fp) return -1;

    // 分配 I/O 缓冲区（4KB）
    const int buf_size = 4096;
    uint8_t *io_buf = (uint8_t *)av_malloc(buf_size);

    // 创建自定义 I/O 上下文
    AVIOContext *avio_ctx = avio_alloc_context(
        io_buf,          // 缓冲区
        buf_size,        // 缓冲区大小
        0,               // write_flag = 0（只读）
        fp,              // opaque（传给回调的用户数据）
        custom_read,     // 读回调
        NULL,            // 写回调（不需要）
        custom_seek      // Seek 回调
    );

    // 分配格式上下文并关联自定义 I/O
    AVFormatContext *fmt_ctx = avformat_alloc_context();
    fmt_ctx->pb = avio_ctx;    // 关键：替换 I/O 上下文

    // 打开（不传文件名，因为 I/O 已经自定义）
    int ret = avformat_open_input(&fmt_ctx, NULL, NULL, NULL);
    if (ret < 0) {
        // 错误处理...
    }

    // ... 使用 fmt_ctx ...

    // 清理
    avformat_close_input(&fmt_ctx);
    av_free(avio_ctx->buffer);
    avio_context_free(&avio_ctx);
    fclose(fp);
    return 0;
}
```

## 3.9 常见错误及解决方案

### 错误 1：av_read_frame 返回 AVERROR(EAGAIN)

**现象**：循环中 `av_read_frame` 返回 `-11`（EAGAIN）

**原因**：非阻塞 I/O 模式下数据尚未就绪（常见于网络流）

**解决**：
```c
ret = av_read_frame(fmt_ctx, &pkt);
if (ret == AVERROR(EAGAIN)) {
    // 数据暂时不可用，稍后重试
    SDL_Delay(10);
    continue;
}
```

### 错误 2：avformat_find_stream_info 超时

**现象**：调用 `avformat_find_stream_info` 阻塞数十秒

**原因**：网络流或损坏文件导致探测时间过长

**解决**：
```c
AVDictionary *opts = NULL;
// 限制探测时长为 5 秒
av_dict_set(&opts, "analyzeduration", "5000000", 0);  // 微秒
// 限制探测数据量为 5MB
av_dict_set(&opts, "probesize", "5000000", 0);        // 字节
avformat_find_stream_info(fmt_ctx, &opts);
av_dict_free(&opts);
```

### 错误 3：Seek 后画面花屏

**现象**：`av_seek_frame` 后解码出的前几帧有马赛克

**原因**：Seek 到了关键帧之前，但解码器缓冲区中还有旧数据

**解决**：
```c
av_seek_frame(fmt_ctx, -1, target_ts, AVSEEK_FLAG_BACKWARD);
// Seek 后必须刷新解码器缓冲区
avcodec_flush_buffers(video_codec_ctx);
avcodec_flush_buffers(audio_codec_ctx);
```

### 错误 4：跨平台路径问题

**Windows 特有**：
```c
// Windows 路径使用反斜杠，但 FFmpeg 接受两种
// 推荐统一使用正斜杠
const char *path = "C:/Users/test/video.mp4";  // ✅ 推荐
const char *path2 = "C:\\Users\\test\\video.mp4";  // ✅ 也可以
```

**Android 特有**：
```c
// Android 上从 assets 读取需要自定义 I/O
// 不能直接传 "assets://video.mp4"
// 需要通过 AAssetManager 打开，再用 AVIOContext 包装
```

## 3.10 本节小结

本章学习了 FFmpeg 解封装的完整流程：

| 步骤 | API | 作用 |
|------|-----|------|
| 打开文件 | `avformat_open_input()` | 探测格式，读取文件头 |
| 探测流信息 | `avformat_find_stream_info()` | 获取完整的编码参数 |
| 遍历流 | 访问 `fmt_ctx->streams[]` | 找到目标音视频流 |
| 读取包 | `av_read_frame()` | 逐包读取压缩数据 |
| 释放包 | `av_packet_unref()` | 防止内存泄漏 |
| 随机访问 | `av_seek_frame()` | 跳转到指定时间位置 |
| 关闭文件 | `avformat_close_input()` | 释放所有资源 |

### 关键要点

1. **容器 ≠ 编码**：MP4 是容器，H.264 是编码。解封装分离容器，解码处理编码
2. **时间戳单位**：每个流有独立的 `time_base`，用 `av_rescale_q` 安全转换
3. **引用计数**：`AVPacket` 使用引用计数管理内存，必须配对 `unref`
4. **Seek 限制**：只能 Seek 到关键帧，精确 Seek 需要额外解码
5. **自定义 I/O**：通过 `AVIOContext` 可以从任意数据源（归档、内存、网络）读取

## 3.11 练习题与答案

### 题目 1：解封装和解码有什么区别？为什么需要分成两步？

<details>
<summary>查看答案</summary>

**解封装（Demuxing）**：
- 从容器文件（MP4/MKV/AVI）中提取各个流的压缩数据包（AVPacket）
- 不涉及数据的解压缩
- 输出：仍然是压缩的数据（H.264 NALU、AAC 帧等）
- 使用 `libavformat` 库

**解码（Decoding）**：
- 将压缩数据包还原为原始像素/采样数据（AVFrame）
- 涉及复杂的数学运算（DCT 逆变换、熵解码等）
- 输出：未压缩的原始数据（YUV 像素、PCM 采样）
- 使用 `libavcodec` 库

**为什么分开？**

1. **解耦**：同一种编码可以放在不同容器中（H.264 可以在 MP4/MKV/TS 中），
   解封装和解码的组合是 M×N 的关系，分开实现避免代码爆炸
2. **流水线**：解封装在前，解码在后，可以用不同线程并行处理
3. **灵活性**：可以只解封装不解码（如 remux 转封装），
   或者直接解码裸流（如 H.264 Annex B 格式）

</details>

### 题目 2：写一个程序提取视频文件中的第一个关键帧，保存其压缩数据到文件

<details>
<summary>查看答案</summary>

```c
#include <libavformat/avformat.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    if (argc < 3) {
        fprintf(stderr, "用法: %s <输入视频> <输出文件>\n", argv[0]);
        return 1;
    }

    AVFormatContext *fmt_ctx = NULL;
    if (avformat_open_input(&fmt_ctx, argv[1], NULL, NULL) < 0) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }
    avformat_find_stream_info(fmt_ctx, NULL);

    // 找到视频流
    int video_idx = av_find_best_stream(
        fmt_ctx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
    if (video_idx < 0) {
        fprintf(stderr, "没有视频流\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // 读取第一个关键帧
    AVPacket pkt;
    int found = 0;
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        if (pkt.stream_index == video_idx &&
            (pkt.flags & AV_PKT_FLAG_KEY)) {
            // 保存到文件
            FILE *fp = fopen(argv[2], "wb");
            if (fp) {
                fwrite(pkt.data, 1, pkt.size, fp);
                fclose(fp);
                printf("已保存关键帧: %d 字节\n", pkt.size);
                found = 1;
            }
            av_packet_unref(&pkt);
            break;
        }
        av_packet_unref(&pkt);
    }

    if (!found) printf("未找到关键帧\n");

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

</details>

### 题目 3：time_base 为 {1, 90000} 的流中，pts=450000 对应多少秒？如果要转换到 time_base 为 {1, 1000} 的流，新的 pts 是多少？

<details>
<summary>查看答案</summary>

**计算秒数**：
```
秒 = pts × time_base
   = 450000 × (1/90000)
   = 5.0 秒
```

**转换到 {1, 1000}**：
```
新 pts = 原 pts × (原 time_base / 新 time_base)
       = 450000 × (1/90000) / (1/1000)
       = 450000 × 1000 / 90000
       = 5000
```

用 FFmpeg API：
```c
AVRational src_tb = {1, 90000};
AVRational dst_tb = {1, 1000};
int64_t src_pts = 450000;
int64_t dst_pts = av_rescale_q(src_pts, src_tb, dst_tb);
// dst_pts = 5000
```

</details>
