## 3.1 什么是解封装（Demuxing）

### 3.1.1 容器与编码的关系

第1章我们学过，一个视频文件通常包含多个**流**（Stream）：视频流、音频流、
字幕流等。这些流被"打包"在一个**容器格式**（Container Format）中。

```
┌─────────────────────────────────────────────────────┐
│                  MP4 容器文件                         │
│                                                      │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐│
│  │ 视频流 #0    │  │ 音频流 #1    │  │ 字幕流 #2    ││
│  │ H.264 编码   │  │ AAC 编码     │  │ SRT 文本     ││
│  │              │  │              │  │              ││
│  │ [pkt][pkt]   │  │ [pkt][pkt]   │  │ [pkt][pkt]  ││
│  │ [pkt][pkt]   │  │ [pkt][pkt]   │  │             ││
│  └─────────────┘  └─────────────┘  └──────────────┘│
│                                                      │
│  文件头：流信息、时间信息、元数据                       │
│  索引表：快速定位到任意时间位置                         │
└─────────────────────────────────────────────────────┘
```

**解封装**（Demuxing / Demultiplexing）就是从容器文件中把各个流的数据包
（Packet）**分离提取**出来的过程：

```
MP4文件 ──解封装──→ 视频 AVPacket 序列
                  → 音频 AVPacket 序列
                  → 字幕 AVPacket 序列
```

### 3.1.2 常见容器格式对比

| 容器格式 | 扩展名 | 视频编码支持 | 音频编码支持 | 特点 |
|----------|--------|-------------|-------------|------|
| MP4/M4V | .mp4, .m4v | H.264, H.265, VP9 | AAC, MP3, Opus | 最通用，浏览器/移动端首选 |
| MKV | .mkv | 几乎所有 | 几乎所有 | 功能最全，字幕支持好 |
| AVI | .avi | 几乎所有 | 几乎所有 | 古老格式，索引在文件尾部 |
| FLV | .flv | H.264, VP6 | AAC, MP3 | 网络流媒体常用 |
| TS | .ts | H.264, H.265 | AAC, AC3, MP2 | 广播/直播标准，可边录边播 |
| OGG | .ogg, .ogv | Theora, VP8 | Vorbis, Opus | 开源格式 |
| WebM | .webm | VP8, VP9, AV1 | Vorbis, Opus | 网页视频标准 |

> **KrKr2 中常见的视频格式**：游戏 OP/ED 通常使用 **AVI（MPEG4/WMV）** 或 **MP4（H.264）**。
> KrKr2 的 `CDVDDemuxFFmpeg` 通过 FFmpeg 的 `libavformat` 自动探测格式，
> 无需硬编码支持列表。

### 3.1.3 为什么需要解封装？

你可能会问：为什么不把视频数据直接存为原始流？

1. **多流复用**：一个文件同时包含音视频，播放器只需打开一个文件
2. **时间戳同步**：容器记录每个包的 PTS/DTS，播放器据此同步音视频
3. **随机访问**：容器的索引表允许快速跳转到任意时间位置（Seek）
4. **元数据**：标题、作者、专辑封面等信息存储在容器头部
5. **流选择**：多音轨/多字幕轨，用户可以动态切换

## 3.2 解封装核心数据结构

### 3.2.1 AVFormatContext —— 格式上下文

`AVFormatContext` 是 `libavformat` 的核心结构体，代表一个打开的媒体文件：

```c
// AVFormatContext 关键字段（简化版）
typedef struct AVFormatContext {
    const AVInputFormat  *iformat;     // 输入格式（如 "mov,mp4,m4a,3gp"）
    const AVOutputFormat *oformat;     // 输出格式（封装时使用）

    AVIOContext  *pb;                  // I/O 上下文（底层读写）

    unsigned int  nb_streams;          // 流的数量
    AVStream    **streams;             // 流数组（下标即 stream_index）

    int64_t       duration;            // 文件总时长（AV_TIME_BASE 单位，即微秒）
    int64_t       bit_rate;            // 总码率（bits/second）
    int64_t       start_time;          // 第一帧的时间戳

    AVDictionary *metadata;            // 元数据（标题、作者等）
} AVFormatContext;
```

> **生命周期**：
> `avformat_alloc_context()` → `avformat_open_input()` → 使用 → `avformat_close_input()`

### 3.2.2 AVStream —— 流信息

每个 `AVStream` 描述文件中的一路流：

```c
typedef struct AVStream {
    int            index;              // 流索引号
    AVCodecParameters *codecpar;       // 编解码参数（codec_type, codec_id, width, height...）
    AVRational     time_base;          // 时间基准（如 {1, 90000} 表示 1/90000 秒）
    int64_t        duration;           // 流时长（time_base 单位）
    int64_t        nb_frames;          // 帧数（可能为 0，表示未知）
    AVRational     avg_frame_rate;     // 平均帧率
    AVDictionary  *metadata;           // 流级元数据（语言、标题等）
} AVStream;
```

**`codecpar` 中的关键字段**：

```c
typedef struct AVCodecParameters {
    enum AVMediaType codec_type;       // AVMEDIA_TYPE_VIDEO / AVMEDIA_TYPE_AUDIO / ...
    enum AVCodecID   codec_id;         // AV_CODEC_ID_H264 / AV_CODEC_ID_AAC / ...
    int              width, height;    // 视频分辨率
    int              sample_rate;      // 音频采样率
    int              channels;         // 音频声道数
    uint64_t         channel_layout;   // 声道布局
    enum AVSampleFormat format;        // 音频采样格式（仅音频）
    enum AVPixelFormat  format;        // 像素格式（仅视频，与上面共用字段名）
    uint8_t         *extradata;        // 解码器初始化数据（SPS/PPS 等）
    int              extradata_size;
} AVCodecParameters;
```

### 3.2.3 AVPacket —— 压缩数据包

`AVPacket` 是解封装输出的基本单元，包含一个流的一段压缩数据：

```c
typedef struct AVPacket {
    int64_t  pts;              // 显示时间戳（Presentation Timestamp）
    int64_t  dts;              // 解码时间戳（Decoding Timestamp）
    uint8_t *data;             // 压缩数据指针
    int      size;             // 数据大小（字节）
    int      stream_index;     // 所属流的索引号
    int      flags;            // 标志位（AV_PKT_FLAG_KEY = 关键帧）
    int64_t  duration;         // 包的时长（stream time_base 单位）
    int64_t  pos;              // 在文件中的字节偏移
} AVPacket;
```

> **PTS vs DTS**：
> - **PTS**（Presentation Timestamp）：帧应该被**显示**的时间
> - **DTS**（Decoding Timestamp）：帧应该被**解码**的时间
> - 对于无 B 帧的视频（如很多游戏视频），PTS == DTS
> - 有 B 帧时，DTS ≤ PTS（因为 B 帧需要先解码后面的 P 帧）

### 3.2.4 三者的关系

```
AVFormatContext
├── streams[0]: AVStream (视频, H.264)
│   └── codecpar: AVCodecParameters
├── streams[1]: AVStream (音频, AAC)
│   └── codecpar: AVCodecParameters
└── av_read_frame() → AVPacket
    ├── stream_index = 0 → 视频包
    └── stream_index = 1 → 音频包
```
