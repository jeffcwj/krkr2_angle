# 3.3 解封装完整流程
## 3.3 解封装完整流程

### 3.3.1 流程图

```
┌──────────────────────────────────────────────────┐
│            解封装（Demuxing）标准流程               │
│                                                   │
│  1. avformat_open_input()     ← 打开文件/URL      │
│         ↓                                         │
│  2. avformat_find_stream_info() ← 探测流信息      │
│         ↓                                         │
│  3. 遍历 streams[]            ← 找到目标流        │
│         ↓                                         │
│  4. av_read_frame() 循环      ← 逐包读取          │
│     ├── stream_index == video → 送视频解码器      │
│     ├── stream_index == audio → 送音频解码器      │
│     └── 其他                  → 忽略/字幕处理     │
│         ↓                                         │
│  5. av_packet_unref()         ← 释放包引用        │
│         ↓                                         │
│  6. avformat_close_input()    ← 关闭文件          │
└──────────────────────────────────────────────────┘
```

### 3.3.2 打开文件

```c
#include <libavformat/avformat.h>
#include <libavutil/avutil.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    const char *filename = "test.mp4";

    // 步骤 1：打开输入文件
    AVFormatContext *fmt_ctx = NULL;
    int ret = avformat_open_input(&fmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));   // FFmpeg 错误码转字符串
        fprintf(stderr, "无法打开文件 '%s': %s\n", filename, errbuf);
        return 1;
    }

    printf("成功打开文件: %s\n", filename);
    printf("格式: %s (%s)\n", fmt_ctx->iformat->name, fmt_ctx->iformat->long_name);

    // ... 后续操作

    avformat_close_input(&fmt_ctx);  // 步骤 6：关闭文件
    return 0;
}
```

> **`avformat_open_input` 做了什么？**
> 1. 根据文件扩展名或内容探测格式（probe）
> 2. 打开文件 I/O（本地文件/HTTP/RTSP 等）
> 3. 读取文件头部，解析基本信息
> 4. 分配并填充 `AVFormatContext`
>
> 第二个参数可以传 `NULL`（自动探测）或指定格式（如 `av_find_input_format("mp4")`）。
> 第四个参数是选项字典，可以设置超时、缓冲区大小等。

### 3.3.3 探测流信息

```c
    // 步骤 2：探测流信息
    ret = avformat_find_stream_info(fmt_ctx, NULL);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // 打印详细信息（调试利器，等同于 ffprobe 输出）
    av_dump_format(fmt_ctx, 0, filename, 0);

    // 显示基本信息
    printf("\n流的数量: %u\n", fmt_ctx->nb_streams);
    printf("总时长: %.2f 秒\n", fmt_ctx->duration / (double)AV_TIME_BASE);
    printf("总码率: %"PRId64" kbps\n", fmt_ctx->bit_rate / 1000);
```

> **为什么需要 `avformat_find_stream_info`？**
>
> `avformat_open_input` 只读取文件头部，对于某些格式（如 MPEG-TS），
> 头部信息不完整，需要实际读取几秒数据来"探测"：
> - 实际的编码器参数（分辨率、采样率）
> - 准确的流时长
> - 帧率估算
>
> 这个函数可能耗时较长（尤其是网络流），因为它会读取并解码前几帧。

### 3.3.4 遍历流

```c
    // 步骤 3：遍历所有流，找到音视频流
    int video_stream_idx = -1;
    int audio_stream_idx = -1;

    for (unsigned int i = 0; i < fmt_ctx->nb_streams; i++) {
        AVStream *stream = fmt_ctx->streams[i];
        AVCodecParameters *par = stream->codecpar;

        // 根据类型分类
        const char *type_name;
        switch (par->codec_type) {
        case AVMEDIA_TYPE_VIDEO:
            type_name = "视频";
            if (video_stream_idx < 0) video_stream_idx = i;
            printf("  流 #%u [%s]: %s, %dx%d, %.2f fps\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id),
                   par->width, par->height,
                   av_q2d(stream->avg_frame_rate));
            break;

        case AVMEDIA_TYPE_AUDIO:
            type_name = "音频";
            if (audio_stream_idx < 0) audio_stream_idx = i;
            printf("  流 #%u [%s]: %s, %d Hz, %d 声道\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id),
                   par->sample_rate, par->channels);
            break;

        case AVMEDIA_TYPE_SUBTITLE:
            type_name = "字幕";
            printf("  流 #%u [%s]: %s\n",
                   i, type_name,
                   avcodec_get_name(par->codec_id));
            break;

        default:
            printf("  流 #%u [其他]: type=%d\n", i, par->codec_type);
            break;
        }

        // 打印时间基准
        printf("         time_base: %d/%d\n",
               stream->time_base.num, stream->time_base.den);
    }

    if (video_stream_idx < 0) {
        fprintf(stderr, "警告：未找到视频流\n");
    }
    if (audio_stream_idx < 0) {
        fprintf(stderr, "警告：未找到音频流\n");
    }
```

> **`av_find_best_stream` 快捷方式**：
> FFmpeg 提供了一个辅助函数，自动选择"最佳"流：
> ```c
> int idx = av_find_best_stream(fmt_ctx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
> ```
> "最佳"的标准是：分辨率最高的视频流、声道数最多的音频流。
> KrKr2 的 `CDVDDemuxFFmpeg::Open()` 内部使用类似逻辑选择默认流。

## 3.4 逐包读取 —— av_read_frame

### 3.4.1 读取循环

```c
    // 步骤 4：逐包读取
    AVPacket pkt;
    int packet_count = 0;
    int video_packets = 0, audio_packets = 0;
    int64_t video_bytes = 0, audio_bytes = 0;

    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        packet_count++;

        if (pkt.stream_index == video_stream_idx) {
            video_packets++;
            video_bytes += pkt.size;

            // 前 5 个视频包的详细信息
            if (video_packets <= 5) {
                AVStream *s = fmt_ctx->streams[video_stream_idx];
                double pts_sec = pkt.pts * av_q2d(s->time_base);
                printf("视频包 #%d: size=%d, pts=%.3fs, dts=%.3fs, key=%s\n",
                       video_packets, pkt.size,
                       pts_sec,
                       pkt.dts * av_q2d(s->time_base),
                       (pkt.flags & AV_PKT_FLAG_KEY) ? "是" : "否");
            }
        } else if (pkt.stream_index == audio_stream_idx) {
            audio_packets++;
            audio_bytes += pkt.size;
        }

        // 步骤 5：释放包引用（重要！否则内存泄漏）
        av_packet_unref(&pkt);
    }

    printf("\n── 统计 ──\n");
    printf("总包数: %d\n", packet_count);
    printf("视频包: %d (%.2f MB)\n", video_packets, video_bytes / 1048576.0);
    printf("音频包: %d (%.2f MB)\n", audio_packets, audio_bytes / 1048576.0);
```

### 3.4.2 AVPacket 生命周期

```
┌─────────────────────────────────────────┐
│         AVPacket 生命周期                 │
│                                          │
│  av_read_frame(&pkt)                     │
│      │  ← pkt 现在持有数据引用           │
│      ↓                                   │
│  使用 pkt.data / pkt.size / pkt.pts      │
│      │  ← 可以送给解码器                 │
│      ↓                                   │
│  av_packet_unref(&pkt)                   │
│      │  ← 释放引用，计数-1               │
│      ↓                                   │
│  下一轮 av_read_frame()                  │
│                                          │
│  ⚠️ 如果需要保存包（如放入队列）：        │
│  av_packet_clone(&pkt) 或                │
│  av_packet_ref(dst, &pkt)                │
│      └─ 引用计数+1，数据不会被释放       │
└─────────────────────────────────────────┘
```

> **常见内存泄漏**：忘记调用 `av_packet_unref`。每个 `av_read_frame` 成功后
> 都**必须**配对一个 `av_packet_unref`（或将所有权转移给其他对象）。
>
> KrKr2 中 `CDVDDemuxFFmpeg::Read()` 在读取后将包的所有权转移给
> `CDVDDemuxerPacket`，由消息队列负责最终释放。

### 3.4.3 时间戳转换

FFmpeg 中的时间戳不是秒，而是以各流的 `time_base` 为单位的整数值：

```c
// 时间戳转换公式
// 秒 = pts * time_base.num / time_base.den
// 简化为：
double seconds = pkt.pts * av_q2d(stream->time_base);

// av_q2d 等价于：
// double av_q2d(AVRational a) { return a.num / (double)a.den; }
```

**常见 time_base 值**：

| 格式 | 视频 time_base | 音频 time_base |
|------|---------------|---------------|
| MP4 | 1/90000 | 1/44100 |
| MKV | 1/1000 | 1/1000 |
| AVI | 1/fps (如 1/30) | 1/sample_rate |
| TS | 1/90000 | 1/90000 |

```c
// 示例：将 time_base 不同的两个时间戳做比较
int64_t video_pts = 90000;   // time_base = 1/90000 → 1秒
int64_t audio_pts = 44100;   // time_base = 1/44100 → 1秒

// 用 av_rescale_q 安全转换（避免溢出）
int64_t audio_in_video_tb = av_rescale_q(
    audio_pts,
    fmt_ctx->streams[audio_stream_idx]->time_base,  // 源 time_base
    fmt_ctx->streams[video_stream_idx]->time_base    // 目标 time_base
);
printf("音频时间在视频时间基准下: %"PRId64"\n", audio_in_video_tb);
// 输出：90000（即 1 秒）
```

> **`av_rescale_q` 是安全的**：它内部使用 128 位整数运算避免溢出，
> 比手动计算 `pts * src.num * dst.den / (src.den * dst.num)` 安全得多。
