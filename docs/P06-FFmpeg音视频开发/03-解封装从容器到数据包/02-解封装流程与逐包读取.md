## 3.1 解封装六步流程

上一节我们学习了四个核心数据结构。本节把它们串成完整的操作流程——从打开文件到读完所有数据包。这六个步骤是 FFmpeg 解封装的标准模式，几乎所有基于 FFmpeg 的播放器（包括 KrKr2）都遵循这个模式。

```
┌──────────────────────────────────────────────────┐
│            解封装（Demuxing）标准六步流程           │
│                                                   │
│  1. avformat_open_input()     ← 打开文件/URL      │
│         ↓                                         │
│  2. avformat_find_stream_info() ← 探测流信息      │
│         ↓                                         │
│  3. 遍历 streams[]            ← 找到目标流        │
│     或 av_find_best_stream()                      │
│         ↓                                         │
│  4. av_read_frame() 循环      ← 逐包读取          │
│     ├── stream_index == video → 送视频解码器      │
│     ├── stream_index == audio → 送音频解码器      │
│     └── 其他                  → 忽略/字幕处理     │
│         ↓                                         │
│  5. av_packet_unref()         ← 释放包引用        │
│         ↓                                         │
│  6. avformat_close_input()    ← 关闭文件          │
└──────────────────────────────────────────────────┘
```

接下来我们逐步详解每一步。

## 3.2 步骤 1：打开文件——`avformat_open_input()`

### 3.2.1 函数签名与参数

```c
int avformat_open_input(
    AVFormatContext **ps,       // [输入/输出] 指向 AVFormatContext 指针的指针
                               //   传入 NULL 则自动分配；传入已分配的则复用
    const char *url,           // [输入] 文件路径或 URL
                               //   本地文件："test.mp4"、"C:/videos/op.avi"
                               //   网络流："http://example.com/stream.m3u8"
                               //   RTSP 流："rtsp://camera/live"
    const AVInputFormat *fmt,  // [输入] 强制指定输入格式，NULL 表示自动探测
                               //   大多数情况传 NULL 即可
    AVDictionary **options     // [输入/输出] 额外选项（超时、缓冲区大小等）
                               //   传 NULL 表示使用默认值
);
// 返回值：0 表示成功，负数表示错误（使用 av_strerror 转为可读字符串）
```

### 3.2.2 自动探测 vs 强制格式

绝大多数情况下，让 FFmpeg 自动探测格式即可（第三个参数传 `NULL`）。FFmpeg 会依次尝试：

1. 根据文件扩展名猜测格式（如 `.mp4` → MP4 解析器）
2. 读取文件头部的魔数（Magic Number，文件开头几个字节的固定标识，如 MP4 文件以 `ftyp` 开头）进行验证
3. 如果扩展名和魔数都无法确定，则用探针函数（Probe Function）逐个尝试所有已注册的格式解析器

只有在特殊情况下才需要强制指定格式，例如：

```c
// 强制以原始 H.264 格式打开（无容器包装的裸流）
const AVInputFormat *fmt = av_find_input_format("h264");
avformat_open_input(&fmt_ctx, "raw_video.h264", fmt, NULL);

// 强制以 MPEG-TS 格式打开（扩展名可能不是 .ts）
const AVInputFormat *fmt2 = av_find_input_format("mpegts");
avformat_open_input(&fmt_ctx, "recording.dat", fmt2, NULL);
```

### 3.2.3 完整示例：打开文件并检查错误

```c
#include <libavformat/avformat.h>
#include <libavutil/avutil.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    const char *filename = "test.mp4";

    // 步骤 1：打开输入文件
    AVFormatContext *fmt_ctx = NULL;  // 初始化为 NULL，让函数自动分配
    int ret = avformat_open_input(&fmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        // 错误处理：将 FFmpeg 错误码转为可读字符串
        char errbuf[AV_ERROR_MAX_STRING_SIZE];  // AV_ERROR_MAX_STRING_SIZE = 64
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开文件 '%s': %s (错误码: %d)\n",
                filename, errbuf, ret);
        return 1;
    }

    // 打开成功——此时 fmt_ctx 已经被分配并填充了基本信息
    printf("成功打开文件: %s\n", filename);
    printf("格式名称: %s\n", fmt_ctx->iformat->name);       // 如 "mov,mp4,m4a,3gp"
    printf("格式全称: %s\n", fmt_ctx->iformat->long_name);  // 如 "QuickTime / MOV"

    // ... 后续步骤 ...

    avformat_close_input(&fmt_ctx);  // 步骤 6：关闭文件
    return 0;
}
```

### 3.2.4 使用选项字典

第四个参数 `options` 可以传入额外选项，这在处理网络流或特殊文件时很有用：

```c
AVDictionary *opts = NULL;

// 设置超时（微秒），避免打开网络流时无限等待
av_dict_set(&opts, "timeout", "5000000", 0);      // 5 秒超时

// 设置探测数据量上限（字节），加快打开速度
av_dict_set(&opts, "probesize", "5000000", 0);     // 最多读 5MB 用于探测

// 设置 User-Agent（HTTP 流媒体时可能需要）
av_dict_set(&opts, "user_agent", "KrKr2Player/1.0", 0);

int ret = avformat_open_input(&fmt_ctx, "http://example.com/stream.m3u8",
                               NULL, &opts);

// 注意：avformat_open_input 会消费掉它识别的选项
// 函数返回后 opts 中剩余的是未被识别的选项
// 应当检查并释放
AVDictionaryEntry *e = NULL;
while ((e = av_dict_get(opts, "", e, AV_DICT_IGNORE_SUFFIX))) {
    fprintf(stderr, "警告：未识别的选项 '%s'\n", e->key);
}
av_dict_free(&opts);  // 释放字典内存
```

> **`avformat_open_input` 内部做了什么？**
> 1. 如果 `*ps` 为 NULL，分配一个新的 `AVFormatContext`
> 2. 根据文件扩展名或内容探测容器格式（probe），设置 `iformat`
> 3. 打开底层 I/O（本地文件：`fopen()`；HTTP：建立 TCP 连接）
> 4. 读取容器头部，解析基本信息（流的数量、容器级元数据等）
> 5. 分配 `streams[]` 数组，但此时流的详细参数可能还不完整

## 3.3 步骤 2：探测流信息——`avformat_find_stream_info()`

### 3.3.1 为什么需要这一步？

`avformat_open_input()` 只读取了容器的头部信息。对于某些容器格式（特别是 MPEG-TS 这种广播格式），头部信息非常少，可能缺少：

- 视频分辨率和帧率
- 音频采样率和声道数
- 准确的流时长
- 编码器的 extradata（SPS/PPS 等初始化参数）

`avformat_find_stream_info()` 通过实际读取并**试解码**前几秒的数据来补全这些信息：

```c
int avformat_find_stream_info(
    AVFormatContext *ic,       // [输入] 已打开的格式上下文
    AVDictionary **options     // [输入] 每个流的额外选项（通常传 NULL）
);
// 返回值：≥0 表示成功，负数表示错误
```

### 3.3.2 使用示例

```c
// 步骤 2：探测流信息
ret = avformat_find_stream_info(fmt_ctx, NULL);
if (ret < 0) {
    fprintf(stderr, "无法获取流信息\n");
    avformat_close_input(&fmt_ctx);
    return 1;
}

// 现在 streams[] 和 codecpar 的信息已经完整了
printf("流的数量: %u\n", fmt_ctx->nb_streams);
printf("总时长  : %.2f 秒\n", fmt_ctx->duration / (double)AV_TIME_BASE);
printf("总码率  : %"PRId64" kbps\n", fmt_ctx->bit_rate / 1000);

// av_dump_format() 是调试利器——一行代码打印全部信息（类似 ffprobe 输出）
av_dump_format(fmt_ctx,    // 格式上下文
               0,          // 流索引（0 = 显示所有流）
               filename,   // 文件名（仅用于打印标题）
               0);         // 0 = 输入, 1 = 输出
```

`av_dump_format()` 的输出类似这样：

```
Input #0, mov,mp4,m4a,3gp, from 'test.mp4':
  Metadata:
    major_brand     : isom
    minor_version   : 512
    compatible_brands: isomiso2avc1mp41
    encoder         : Lavf58.76.100
  Duration: 00:01:30.50, start: 0.000000, bitrate: 2500 kb/s
  Stream #0:0(und): Video: h264 (High), yuv420p, 1920x1080, 2200 kb/s, 29.97 fps
  Stream #0:1(und): Audio: aac (LC), 44100 Hz, stereo, fltp, 128 kb/s
```

### 3.3.3 性能注意事项

`avformat_find_stream_info()` 可能很慢，因为它需要：

1. 读取前几秒的数据
2. 尝试解码若干帧来确定编码参数
3. 估算帧率和时长

对于本地文件通常很快（毫秒级），但对于网络流可能需要数秒。可以通过选项限制探测范围：

```c
// 方式一：通过 AVFormatContext 的字段限制
fmt_ctx->max_analyze_duration = 5 * AV_TIME_BASE;  // 最多分析 5 秒
fmt_ctx->probesize = 5 * 1024 * 1024;              // 最多读取 5MB

// 方式二：通过选项字典（在 avformat_open_input 时设置）
AVDictionary *opts = NULL;
av_dict_set(&opts, "analyzeduration", "5000000", 0);  // 5 秒（微秒单位）
av_dict_set(&opts, "probesize", "5000000", 0);         // 5MB（字节单位）
avformat_open_input(&fmt_ctx, filename, NULL, &opts);
av_dict_free(&opts);
```

## 3.4 步骤 3：找到目标流

一个媒体文件通常有多条流。我们需要找到感兴趣的视频流和音频流的索引号（stream_index），以便在后续读包时按类型分发。

### 3.4.1 方式一：手动遍历 streams[]

```c
int video_stream_idx = -1;   // -1 表示"未找到"
int audio_stream_idx = -1;

for (unsigned int i = 0; i < fmt_ctx->nb_streams; i++) {
    AVStream *stream = fmt_ctx->streams[i];
    AVCodecParameters *par = stream->codecpar;

    switch (par->codec_type) {
    case AVMEDIA_TYPE_VIDEO:
        if (video_stream_idx < 0) {          // 取第一条视频流
            video_stream_idx = (int)i;
        }
        printf("流 #%u [视频]: %s, %dx%d, %.2f fps, time_base=%d/%d\n",
               i,
               avcodec_get_name(par->codec_id),  // 编码名称，如 "h264"
               par->width, par->height,
               av_q2d(stream->avg_frame_rate),    // AVRational → double
               stream->time_base.num, stream->time_base.den);
        break;

    case AVMEDIA_TYPE_AUDIO:
        if (audio_stream_idx < 0) {          // 取第一条音频流
            audio_stream_idx = (int)i;
        }
        printf("流 #%u [音频]: %s, %d Hz, %d 声道, time_base=%d/%d\n",
               i,
               avcodec_get_name(par->codec_id),
               par->sample_rate,
               par->channels,
               stream->time_base.num, stream->time_base.den);
        break;

    case AVMEDIA_TYPE_SUBTITLE:
        printf("流 #%u [字幕]: %s\n", i, avcodec_get_name(par->codec_id));
        break;

    default:
        printf("流 #%u [其他]: codec_type=%d\n", i, par->codec_type);
        break;
    }
}

if (video_stream_idx < 0) {
    fprintf(stderr, "警告：未找到视频流\n");
}
if (audio_stream_idx < 0) {
    fprintf(stderr, "警告：未找到音频流\n");
}
```

### 3.4.2 方式二：`av_find_best_stream()` 快捷方式

FFmpeg 提供了一个辅助函数，自动选择"最佳"流。所谓"最佳"是指：分辨率最高的视频流、声道数最多的音频流。

```c
int av_find_best_stream(
    AVFormatContext *ic,       // 格式上下文
    enum AVMediaType type,     // 要找的流类型（AVMEDIA_TYPE_VIDEO / AUDIO / SUBTITLE）
    int wanted_stream_number,  // 希望的流编号（-1 = 自动选择最佳）
    int related_stream,        // 关联流（-1 = 不关联；传视频流索引可找到匹配的音频流）
    const AVCodec **decoder_ret, // [输出] 找到的解码器（可传 NULL）
    int flags                  // 保留参数，传 0
);
// 返回值：≥0 = 流索引号，负数 = 错误（如 AVERROR_STREAM_NOT_FOUND）
```

使用示例：

```c
// 自动选择最佳视频流
int video_idx = av_find_best_stream(fmt_ctx, AVMEDIA_TYPE_VIDEO,
                                     -1,   // wanted: 自动选择
                                     -1,   // related: 无关联
                                     NULL,  // decoder: 不需要
                                     0);    // flags: 保留
if (video_idx < 0) {
    fprintf(stderr, "没有视频流\n");
}

// 自动选择最佳音频流（关联到视频流，优先选同一节目的音频）
int audio_idx = av_find_best_stream(fmt_ctx, AVMEDIA_TYPE_AUDIO,
                                     -1, video_idx, NULL, 0);
if (audio_idx < 0) {
    fprintf(stderr, "没有音频流\n");
}

printf("最佳视频流: #%d\n", video_idx);
printf("最佳音频流: #%d\n", audio_idx);
```

> **KrKr2 中的做法**：`CDVDDemuxFFmpeg::Open()` 内部使用类似 `av_find_best_stream()` 的逻辑来选择默认的视频流和音频流，并将结果存储为 `m_streams` 成员变量。

## 3.5 步骤 4：逐包读取——`av_read_frame()`

### 3.5.1 函数签名

```c
int av_read_frame(
    AVFormatContext *s,    // 格式上下文
    AVPacket *pkt          // [输出] 读取到的数据包
);
// 返回值：0 = 成功，AVERROR(EAGAIN) = 暂时无数据（非阻塞 I/O），
//         AVERROR_EOF = 文件结束，其他负数 = 错误
```

> **函数名的历史遗留**：`av_read_frame()` 的名字可能让人以为它读的是"帧"（Frame），但实际上它读的是"包"（Packet）。这是 FFmpeg 历史 API 命名不规范导致的。在 FFmpeg 的早期版本中，"frame"和"packet"的概念没有严格区分。现在请记住：`av_read_frame()` 返回的是 `AVPacket`（压缩数据包），不是 `AVFrame`（原始帧）。

### 3.5.2 读取循环的标准写法

```c
AVPacket pkt;               // 栈上分配 AVPacket
int packet_count = 0;       // 总包计数
int video_packets = 0;      // 视频包计数
int audio_packets = 0;      // 音频包计数
int64_t video_bytes = 0;    // 视频数据总字节
int64_t audio_bytes = 0;    // 音频数据总字节

// 循环读取，直到文件结束或出错
while (av_read_frame(fmt_ctx, &pkt) >= 0) {
    packet_count++;

    // 根据 stream_index 分发到不同处理逻辑
    if (pkt.stream_index == video_stream_idx) {
        video_packets++;
        video_bytes += pkt.size;

        // 获取该流的信息，用于时间戳转换
        AVStream *s = fmt_ctx->streams[video_stream_idx];
        double pts_sec = pkt.pts * av_q2d(s->time_base);   // 转为秒
        double dts_sec = pkt.dts * av_q2d(s->time_base);

        // 打印前 5 个视频包的详细信息
        if (video_packets <= 5) {
            printf("视频包 #%d: size=%d 字节, pts=%.3f秒, dts=%.3f秒, "
                   "关键帧=%s, 文件偏移=%"PRId64"\n",
                   video_packets, pkt.size,
                   pts_sec, dts_sec,
                   (pkt.flags & AV_PKT_FLAG_KEY) ? "是" : "否",
                   pkt.pos);
        }

        // 实际播放器中：这里会将 pkt 送入视频解码器
        // avcodec_send_packet(video_codec_ctx, &pkt);

    } else if (pkt.stream_index == audio_stream_idx) {
        audio_packets++;
        audio_bytes += pkt.size;

        // 实际播放器中：这里会将 pkt 送入音频解码器
        // avcodec_send_packet(audio_codec_ctx, &pkt);
    }
    // 其他流（字幕等）在这个示例中忽略

    // 步骤 5：释放包引用（必须！否则内存泄漏）
    av_packet_unref(&pkt);
}

// 打印统计
printf("\n── 统计信息 ──\n");
printf("总包数  : %d\n", packet_count);
printf("视频包  : %d 个 (%.2f MB)\n", video_packets, video_bytes / 1048576.0);
printf("音频包  : %d 个 (%.2f MB)\n", audio_packets, audio_bytes / 1048576.0);
```

### 3.5.3 `av_read_frame` 返回值处理

在实际播放器中，不能简单地用 `>= 0` 判断，还需要处理特殊返回值：

```c
while (1) {
    int ret = av_read_frame(fmt_ctx, &pkt);

    if (ret == 0) {
        // 成功读到包，正常处理
        process_packet(&pkt);
        av_packet_unref(&pkt);

    } else if (ret == AVERROR_EOF) {
        // 文件已经读完（End Of File）
        printf("文件读取完毕\n");
        break;

    } else if (ret == AVERROR(EAGAIN)) {
        // 非阻塞模式下数据暂时不可用（常见于网络流、实时源）
        // 等待一段时间后重试
        #ifdef _WIN32
        Sleep(10);        // Windows：毫秒
        #else
        usleep(10000);    // Linux/macOS：微秒
        #endif
        continue;

    } else {
        // 其他错误（损坏的文件、I/O 错误等）
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "读取包出错: %s\n", errbuf);
        break;
    }
}
```

> **KrKr2 中的处理**：`CDVDDemuxFFmpeg::Read()` 方法内部调用 `av_read_frame()`，并对 `EAGAIN` 做了特殊处理——返回 NULL 让上层循环重试，而不是让线程忙等（Busy-Wait，CPU 空转等待）。

## 3.6 时间戳与 time_base 转换

### 3.6.1 time_base 的含义

FFmpeg 中的时间戳不是秒，而是以各流独立的 `time_base`（时间基准）为单位的**整数值**。每条流有自己的 `time_base`，不同流的 `time_base` 通常不同。

**转换公式**：

```
秒 = pts × time_base.num / time_base.den
简写为：
秒 = pts × av_q2d(stream->time_base)
```

其中 `av_q2d()` 将 `AVRational` 转为 `double`：

```c
// av_q2d 的等价实现：
static inline double av_q2d(AVRational a) {
    return a.num / (double)a.den;
}
```

**常见 time_base 值**：

| 容器格式 | 视频流 time_base | 音频流 time_base | 说明 |
|---------|-----------------|-----------------|------|
| MP4 | {1, 90000} | {1, 44100} | 视频用 90kHz 时钟，音频用采样率 |
| MKV | {1, 1000} | {1, 1000} | 统一毫秒精度 |
| AVI | {1, fps} 如 {1, 30} | {1, sample_rate} | 按帧率/采样率 |
| TS | {1, 90000} | {1, 90000} | 统一 90kHz（MPEG 标准） |

### 3.6.2 不同 time_base 之间的转换

当需要比较来自不同流的时间戳时，必须先统一 time_base。FFmpeg 提供了 `av_rescale_q()` 函数来安全转换：

```c
int64_t av_rescale_q(
    int64_t a,         // 要转换的时间戳值
    AVRational bq,     // 源 time_base
    AVRational cq      // 目标 time_base
);
// 返回值：转换后的时间戳值（目标 time_base 单位）
```

**为什么要用 `av_rescale_q` 而不是手动计算？**

手动转换公式是 `result = a * bq.num * cq.den / (bq.den * cq.num)`，但中间乘法可能溢出 `int64_t` 的范围。`av_rescale_q()` 内部使用 128 位整数运算来避免溢出，安全可靠。

```c
// 示例：将视频流的 PTS 转换到音频流的 time_base
AVStream *video_stream = fmt_ctx->streams[video_stream_idx];
AVStream *audio_stream = fmt_ctx->streams[audio_stream_idx];

int64_t video_pts = 90000;   // time_base={1,90000} → 1.0 秒
int64_t audio_in_video_tb = av_rescale_q(
    video_pts,
    video_stream->time_base,   // 源：{1, 90000}
    audio_stream->time_base    // 目标：{1, 44100}
);
printf("视频 1 秒处在音频 time_base 下: %"PRId64"\n", audio_in_video_tb);
// 输出：44100（即 1 秒）

// 转为统一的秒数用于比较
double video_sec = video_pts * av_q2d(video_stream->time_base);
double audio_sec = audio_in_video_tb * av_q2d(audio_stream->time_base);
printf("视频时间: %.3f 秒, 音频时间: %.3f 秒\n", video_sec, audio_sec);
// 输出：视频时间: 1.000 秒, 音频时间: 1.000 秒
```

### 3.6.3 使用 AV_TIME_BASE 作为通用时间基准

FFmpeg 定义了一个全局时间基准 `AV_TIME_BASE`（= 1000000，即微秒）和对应的 `AVRational` 常量 `AV_TIME_BASE_Q`（= {1, 1000000}）。当需要一个与流无关的统一时间表示时，可以转换到 `AV_TIME_BASE`：

```c
// 将流的 duration 转为微秒（AV_TIME_BASE 单位）
int64_t stream_dur_us = av_rescale_q(
    stream->duration,
    stream->time_base,
    AV_TIME_BASE_Q           // {1, 1000000}
);

// 转为秒
double stream_dur_sec = stream_dur_us / (double)AV_TIME_BASE;
printf("流时长: %.2f 秒\n", stream_dur_sec);

// 注意：AVFormatContext::duration 本身就是 AV_TIME_BASE 单位
double file_dur_sec = fmt_ctx->duration / (double)AV_TIME_BASE;
printf("文件时长: %.2f 秒\n", file_dur_sec);
```

## 3.7 动手实践：统计各流的包分布

以下完整程序综合运用六步流程，统计一个媒体文件中每条流的包数量、关键帧（Keyframe，也称 I 帧——不依赖其他帧即可独立解码的帧）数量和数据量：

```c
// stream_stats.c —— 统计媒体文件的包分布
#include <libavformat/avformat.h>
#include <libavcodec/avcodec.h>
#include <libavutil/avutil.h>
#include <stdio.h>
#include <inttypes.h>
#include <string.h>

// 每条流的统计数据
typedef struct {
    int packet_count;          // 包总数
    int keyframe_count;        // 关键帧数量
    int64_t total_bytes;       // 总字节数
    int64_t min_pts;           // 最小 PTS
    int64_t max_pts;           // 最大 PTS
    int pts_initialized;       // 是否已初始化 min/max
} StreamStats;

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件>\n", argv[0]);
        return 1;
    }
    const char *filename = argv[1];

    // ── 步骤 1：打开文件 ──
    AVFormatContext *fmt_ctx = NULL;
    int ret = avformat_open_input(&fmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开 '%s': %s\n", filename, errbuf);
        return 1;
    }

    // ── 步骤 2：探测流信息 ──
    if (avformat_find_stream_info(fmt_ctx, NULL) < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // ── 步骤 3：显示所有流 ──
    printf("文件: %s\n", filename);
    printf("格式: %s\n", fmt_ctx->iformat->long_name);
    printf("时长: %.2f 秒\n", fmt_ctx->duration / (double)AV_TIME_BASE);
    printf("流数: %u\n\n", fmt_ctx->nb_streams);

    // 初始化统计数组
    StreamStats stats[32];
    memset(stats, 0, sizeof(stats));
    unsigned int num_streams = fmt_ctx->nb_streams;
    if (num_streams > 32) num_streams = 32;  // 安全上限

    // ── 步骤 4 & 5：逐包读取并统计 ──
    AVPacket pkt;
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        if ((unsigned)pkt.stream_index < num_streams) {
            StreamStats *ss = &stats[pkt.stream_index];
            ss->packet_count++;
            ss->total_bytes += pkt.size;

            if (pkt.flags & AV_PKT_FLAG_KEY) {
                ss->keyframe_count++;
            }

            // 记录 PTS 范围
            if (pkt.pts != AV_NOPTS_VALUE) {
                if (!ss->pts_initialized) {
                    ss->min_pts = ss->max_pts = pkt.pts;
                    ss->pts_initialized = 1;
                } else {
                    if (pkt.pts < ss->min_pts) ss->min_pts = pkt.pts;
                    if (pkt.pts > ss->max_pts) ss->max_pts = pkt.pts;
                }
            }
        }
        av_packet_unref(&pkt);  // 必须释放引用！
    }

    // ── 打印统计结果 ──
    printf("%-6s %-8s %-10s %-10s %-10s %-12s %-12s\n",
           "流号", "类型", "包数量", "关键帧", "数据量", "起始时间", "结束时间");
    printf("────── ──────── ────────── ────────── ────────── ──────────── ────────────\n");

    for (unsigned int i = 0; i < num_streams; i++) {
        AVStream *s = fmt_ctx->streams[i];
        StreamStats *ss = &stats[i];

        const char *type = "其他";
        switch (s->codecpar->codec_type) {
        case AVMEDIA_TYPE_VIDEO:    type = "视频"; break;
        case AVMEDIA_TYPE_AUDIO:    type = "音频"; break;
        case AVMEDIA_TYPE_SUBTITLE: type = "字幕"; break;
        default: break;
        }

        double start_sec = ss->pts_initialized ?
            ss->min_pts * av_q2d(s->time_base) : 0.0;
        double end_sec = ss->pts_initialized ?
            ss->max_pts * av_q2d(s->time_base) : 0.0;

        printf("#%-5u %-8s %-10d %-10d %-8.2f MB %8.2f 秒  %8.2f 秒\n",
               i, type, ss->packet_count, ss->keyframe_count,
               ss->total_bytes / 1048576.0, start_sec, end_sec);
    }

    // ── 步骤 6：关闭文件 ──
    avformat_close_input(&fmt_ctx);
    return 0;
}
```

## 常见错误及解决方案

### 错误 1：`av_read_frame` 返回 AVERROR(EAGAIN)

**现象**：循环中 `av_read_frame()` 反复返回 `-11`（EAGAIN），程序卡住或 CPU 占用飙高

**原因**：非阻塞 I/O 模式下数据尚未就绪（常见于实时网络流、USB 摄像头等）

**解决**：不要忽略也不要死循环，应该等待一小段时间后重试：

```c
ret = av_read_frame(fmt_ctx, &pkt);
if (ret == AVERROR(EAGAIN)) {
    // 数据暂时不可用，短暂休眠后重试
    #ifdef _WIN32
    Sleep(10);          // Windows：10 毫秒
    #else
    usleep(10000);      // Linux/macOS：10000 微秒 = 10 毫秒
    #endif
    continue;
}
```

### 错误 2：`avformat_find_stream_info` 阻塞数十秒

**现象**：打开网络流或损坏的本地文件时，`avformat_find_stream_info()` 长时间不返回

**原因**：FFmpeg 默认会读取前 5 秒的数据来探测流信息。如果是网络流且带宽低，或文件损坏导致无法正确识别帧，就会阻塞很久

**解决**：限制探测范围：

```c
// 在调用 avformat_open_input 之前设置
AVDictionary *opts = NULL;
av_dict_set(&opts, "analyzeduration", "3000000", 0);  // 最多分析 3 秒
av_dict_set(&opts, "probesize", "2000000", 0);         // 最多读取 2MB
avformat_open_input(&fmt_ctx, url, NULL, &opts);
av_dict_free(&opts);

// 或者在 avformat_open_input 之后、avformat_find_stream_info 之前设置
fmt_ctx->max_analyze_duration = 3 * AV_TIME_BASE;
fmt_ctx->probesize = 2 * 1024 * 1024;
avformat_find_stream_info(fmt_ctx, NULL);
```

### 错误 3：时间戳转换得到错误结果

**现象**：手动计算 `pts * src_tb.num * dst_tb.den / (src_tb.den * dst_tb.num)` 得到负数或极大值

**原因**：中间乘法溢出了 `int64_t` 的范围。例如 `90000 * 44100 = 3969000000`，再乘以大 PTS 很容易超过 2^63

**解决**：始终使用 `av_rescale_q()` 代替手动计算，它内部使用 128 位整数运算：

```c
// ❌ 危险：可能溢出
int64_t result = pts * src_tb.num * dst_tb.den / (src_tb.den * dst_tb.num);

// ✅ 安全：使用 av_rescale_q
int64_t result = av_rescale_q(pts, src_tb, dst_tb);
```

## 本节小结

- FFmpeg 解封装遵循**六步标准流程**：open → find_stream_info → 找流 → read_frame 循环 → unref → close
- `avformat_open_input()` 打开文件并自动探测格式，支持本地文件和网络 URL
- `avformat_find_stream_info()` 通过实际读取和试解码来补全流的详细参数，可能较慢
- `av_find_best_stream()` 是定位目标流的快捷方式，自动选择"最佳"的视频/音频流
- `av_read_frame()` 逐包读取数据，通过 `stream_index` 区分包属于哪条流
- 每次 `av_read_frame()` 成功后必须配对 `av_packet_unref()` 释放引用
- 时间戳以各流的 `time_base` 为单位，跨流比较时必须用 `av_rescale_q()` 安全转换

## 练习题与答案

### 题目 1：`avformat_open_input` 和 `avformat_find_stream_info` 各做了什么？为什么不能合并成一个函数？

<details>
<summary>查看答案</summary>

**`avformat_open_input` 做了什么**：
1. 分配 `AVFormatContext`（如果传入 NULL）
2. 探测容器格式（通过扩展名和文件头魔数）
3. 打开底层 I/O（文件/网络连接）
4. 读取容器头部，获取基本信息（流的数量等）

**`avformat_find_stream_info` 做了什么**：
1. 实际读取前几秒的数据
2. 尝试解码若干帧来确定编码参数（分辨率、帧率等）
3. 填充 `AVStream::codecpar` 的详细字段
4. 估算流时长

**为什么不合并**：
1. **性能**：`find_stream_info` 可能很慢（几秒），有些场景不需要完整的流信息（比如只想知道容器格式和流数量）
2. **灵活性**：在两个函数之间可以配置探测参数（`probesize`、`analyzeduration`），控制探测行为
3. **自定义 I/O**：有时需要在 `open_input` 之后、`find_stream_info` 之前替换 I/O 上下文
4. **分步错误处理**：可以分别处理"文件打不开"和"流信息获取失败"两种不同错误

</details>

### 题目 2：一个 TS 格式的文件，视频流 time_base = {1, 90000}，某个视频包的 pts = 2700000。这个包应该在什么时间显示？如果要转换到 MKV 的 time_base = {1, 1000}，新的 pts 是多少？请写出代码。

<details>
<summary>查看答案</summary>

**计算秒数**：
```
秒 = pts × time_base = 2700000 × (1/90000) = 30.0 秒
```

**转换到 {1, 1000}**（毫秒精度）：
```
新 pts = 原 pts × (原 time_base / 新 time_base)
       = 2700000 × (1/90000) / (1/1000)
       = 2700000 × 1000 / 90000
       = 30000
```

代码：
```c
#include <libavutil/mathematics.h>
#include <stdio.h>
#include <inttypes.h>

int main(void) {
    AVRational ts_video_tb = {1, 90000};   // TS 视频流的 time_base
    AVRational mkv_tb = {1, 1000};         // MKV 的 time_base
    int64_t pts = 2700000;

    // 方式 1：手动计算秒数
    double seconds = pts * av_q2d(ts_video_tb);
    printf("显示时间: %.3f 秒\n", seconds);  // 30.000 秒

    // 方式 2：用 av_rescale_q 转换 time_base
    int64_t mkv_pts = av_rescale_q(pts, ts_video_tb, mkv_tb);
    printf("MKV time_base 下的 pts: %"PRId64"\n", mkv_pts);  // 30000

    // 验证：两个 pts 表示的秒数应该相同
    double verify = mkv_pts * av_q2d(mkv_tb);
    printf("验证: %.3f 秒\n", verify);  // 30.000 秒

    return 0;
}
```

</details>

### 题目 3：为什么 `av_read_frame` 的名字叫"read frame"但返回的是 `AVPacket`？如何区分 `AVPacket` 和 `AVFrame`？

<details>
<summary>查看答案</summary>

**命名历史**：
这是 FFmpeg API 的历史遗留问题。在 FFmpeg 早期版本中，"frame"和"packet"的概念没有严格区分——当时 `AVPacket` 甚至不存在，数据直接用 `AVFrame` 表示。后来 FFmpeg 引入了 `AVPacket` 来区分压缩数据和原始数据，但函数名 `av_read_frame` 已经被广泛使用，出于向后兼容性没有改名。

**`AVPacket` vs `AVFrame` 的区别**：

| 特性 | AVPacket | AVFrame |
|------|----------|---------|
| 数据状态 | **压缩**数据（编码后） | **原始**数据（解码后） |
| 产生方式 | `av_read_frame()` 从容器读取 | `avcodec_receive_frame()` 从解码器获取 |
| 数据内容 | H.264 NALU、AAC 帧等 | YUV 像素数据、PCM 音频采样 |
| 使用的库 | `libavformat`（封装/解封装） | `libavcodec`（编码/解码） |
| 流向 | 解封装输出 → 解码器输入 | 解码器输出 → 渲染/播放 |

流水线关系：
```
容器文件 → [解封装] → AVPacket → [解码] → AVFrame → [渲染/播放]
```

</details>
