## 3.8 对照项目源码

### CDVDDemuxFFmpeg 类

KrKr2 的解封装逻辑封装在 `CDVDDemuxFFmpeg` 类中：

**文件**：`cpp/core/movie/ffmpeg/DemuxFFmpeg.h`（157 行）

```cpp
// DemuxFFmpeg.h 关键接口（简化）
class CDVDDemuxFFmpeg : public CDVDDemux {
public:
    bool Open(CDVDInputStream *pInput);    // 打开流
    void Dispose();                         // 关闭释放
    CDVDDemuxerPacket *Read();             // 读取下一个包
    bool SeekTime(double time);            // Seek 到指定时间（秒）
    int GetNrOfStreams() const;            // 流数量
    CDVDDemuxStream *GetStream(int idx);   // 获取流信息

private:
    AVFormatContext *m_pFormatContext;      // 核心：FFmpeg 格式上下文
    AVIOContext     *m_ioContext;           // 自定义 I/O 上下文
    CDVDInputStream *m_pInput;             // 输入流（KrKr2 的流接口）
};
```

**与我们代码的对应关系**：

| 我们的代码 | KrKr2 对应 | 说明 |
|-----------|-----------|------|
| `avformat_open_input()` | `CDVDDemuxFFmpeg::Open()` | 封装在类方法中 |
| `av_read_frame()` | `CDVDDemuxFFmpeg::Read()` | 返回 CDVDDemuxerPacket |
| `av_seek_frame()` | `CDVDDemuxFFmpeg::SeekTime()` | 封装了精确 Seek |
| `avformat_close_input()` | `CDVDDemuxFFmpeg::Dispose()` | 释放所有资源 |
| `AVPacket` | `CDVDDemuxerPacket` | 包装类，添加了引用计数 |

> **设计模式差异**：我们的代码是过程式 C 代码，KrKr2 使用面向对象封装。
> 但底层的 FFmpeg API 调用完全相同——`Open()` 内部调用 `avformat_open_input()`，
> `Read()` 内部调用 `av_read_frame()`。这就是为什么学好 FFmpeg API 能直接
> 看懂 KrKr2 源码。

## 3.9 常见错误及解决方案

### 错误 1：av_read_frame 返回 AVERROR(EAGAIN)

**现象**：循环中 `av_read_frame` 返回 `-11`（EAGAIN）

**原因**：非阻塞 I/O 模式下数据尚未就绪（常见于网络流）

**解决**：
```c
ret = av_read_frame(fmt_ctx, &pkt);
if (ret == AVERROR(EAGAIN)) {
    // 数据暂时不可用，稍后重试
    SDL_Delay(10);
    continue;
}
```

### 错误 2：avformat_find_stream_info 超时

**现象**：调用 `avformat_find_stream_info` 阻塞数十秒

**原因**：网络流或损坏文件导致探测时间过长

**解决**：
```c
AVDictionary *opts = NULL;
// 限制探测时长为 5 秒
av_dict_set(&opts, "analyzeduration", "5000000", 0);  // 微秒
// 限制探测数据量为 5MB
av_dict_set(&opts, "probesize", "5000000", 0);        // 字节
avformat_find_stream_info(fmt_ctx, &opts);
av_dict_free(&opts);
```

### 错误 3：Seek 后画面花屏

**现象**：`av_seek_frame` 后解码出的前几帧有马赛克

**原因**：Seek 到了关键帧之前，但解码器缓冲区中还有旧数据

**解决**：
```c
av_seek_frame(fmt_ctx, -1, target_ts, AVSEEK_FLAG_BACKWARD);
// Seek 后必须刷新解码器缓冲区
avcodec_flush_buffers(video_codec_ctx);
avcodec_flush_buffers(audio_codec_ctx);
```

### 错误 4：跨平台路径问题

**Windows 特有**：
```c
// Windows 路径使用反斜杠，但 FFmpeg 接受两种
// 推荐统一使用正斜杠
const char *path = "C:/Users/test/video.mp4";  // ✅ 推荐
const char *path2 = "C:\\Users\\test\\video.mp4";  // ✅ 也可以
```

**Android 特有**：
```c
// Android 上从 assets 读取需要自定义 I/O
// 不能直接传 "assets://video.mp4"
// 需要通过 AAssetManager 打开，再用 AVIOContext 包装
```

## 3.10 本节小结

本章学习了 FFmpeg 解封装的完整流程：

| 步骤 | API | 作用 |
|------|-----|------|
| 打开文件 | `avformat_open_input()` | 探测格式，读取文件头 |
| 探测流信息 | `avformat_find_stream_info()` | 获取完整的编码参数 |
| 遍历流 | 访问 `fmt_ctx->streams[]` | 找到目标音视频流 |
| 读取包 | `av_read_frame()` | 逐包读取压缩数据 |
| 释放包 | `av_packet_unref()` | 防止内存泄漏 |
| 随机访问 | `av_seek_frame()` | 跳转到指定时间位置 |
| 关闭文件 | `avformat_close_input()` | 释放所有资源 |

### 关键要点

1. **容器 ≠ 编码**：MP4 是容器，H.264 是编码。解封装分离容器，解码处理编码
2. **时间戳单位**：每个流有独立的 `time_base`，用 `av_rescale_q` 安全转换
3. **引用计数**：`AVPacket` 使用引用计数管理内存，必须配对 `unref`
4. **Seek 限制**：只能 Seek 到关键帧，精确 Seek 需要额外解码
5. **自定义 I/O**：通过 `AVIOContext` 可以从任意数据源（归档、内存、网络）读取

## 3.11 练习题与答案

### 题目 1：解封装和解码有什么区别？为什么需要分成两步？

<details>
<summary>查看答案</summary>

**解封装（Demuxing）**：
- 从容器文件（MP4/MKV/AVI）中提取各个流的压缩数据包（AVPacket）
- 不涉及数据的解压缩
- 输出：仍然是压缩的数据（H.264 NALU、AAC 帧等）
- 使用 `libavformat` 库

**解码（Decoding）**：
- 将压缩数据包还原为原始像素/采样数据（AVFrame）
- 涉及复杂的数学运算（DCT 逆变换、熵解码等）
- 输出：未压缩的原始数据（YUV 像素、PCM 采样）
- 使用 `libavcodec` 库

**为什么分开？**

1. **解耦**：同一种编码可以放在不同容器中（H.264 可以在 MP4/MKV/TS 中），
   解封装和解码的组合是 M×N 的关系，分开实现避免代码爆炸
2. **流水线**：解封装在前，解码在后，可以用不同线程并行处理
3. **灵活性**：可以只解封装不解码（如 remux 转封装），
   或者直接解码裸流（如 H.264 Annex B 格式）

</details>

### 题目 2：写一个程序提取视频文件中的第一个关键帧，保存其压缩数据到文件

<details>
<summary>查看答案</summary>

```c
#include <libavformat/avformat.h>
#include <stdio.h>

int main(int argc, char *argv[]) {
    if (argc < 3) {
        fprintf(stderr, "用法: %s <输入视频> <输出文件>\n", argv[0]);
        return 1;
    }

    AVFormatContext *fmt_ctx = NULL;
    if (avformat_open_input(&fmt_ctx, argv[1], NULL, NULL) < 0) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }
    avformat_find_stream_info(fmt_ctx, NULL);

    // 找到视频流
    int video_idx = av_find_best_stream(
        fmt_ctx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
    if (video_idx < 0) {
        fprintf(stderr, "没有视频流\n");
        avformat_close_input(&fmt_ctx);
        return 1;
    }

    // 读取第一个关键帧
    AVPacket pkt;
    int found = 0;
    while (av_read_frame(fmt_ctx, &pkt) >= 0) {
        if (pkt.stream_index == video_idx &&
            (pkt.flags & AV_PKT_FLAG_KEY)) {
            // 保存到文件
            FILE *fp = fopen(argv[2], "wb");
            if (fp) {
                fwrite(pkt.data, 1, pkt.size, fp);
                fclose(fp);
                printf("已保存关键帧: %d 字节\n", pkt.size);
                found = 1;
            }
            av_packet_unref(&pkt);
            break;
        }
        av_packet_unref(&pkt);
    }

    if (!found) printf("未找到关键帧\n");

    avformat_close_input(&fmt_ctx);
    return 0;
}
```

</details>

### 题目 3：time_base 为 {1, 90000} 的流中，pts=450000 对应多少秒？如果要转换到 time_base 为 {1, 1000} 的流，新的 pts 是多少？

<details>
<summary>查看答案</summary>

**计算秒数**：
```
秒 = pts × time_base
   = 450000 × (1/90000)
   = 5.0 秒
```

**转换到 {1, 1000}**：
```
新 pts = 原 pts × (原 time_base / 新 time_base)
       = 450000 × (1/90000) / (1/1000)
       = 450000 × 1000 / 90000
       = 5000
```

用 FFmpeg API：
```c
AVRational src_tb = {1, 90000};
AVRational dst_tb = {1, 1000};
int64_t src_pts = 450000;
int64_t dst_pts = av_rescale_q(src_pts, src_tb, dst_tb);
// dst_pts = 5000
```

</details>
