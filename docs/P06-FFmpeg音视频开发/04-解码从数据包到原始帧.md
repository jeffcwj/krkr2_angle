## 4.1 解码器基础概念

### 什么是解码（Decoding）

在第三节中，我们通过解封装（demuxing）从容器文件中提取了压缩的数据包（`AVPacket`）。这些数据包包含的是经过编码器（encoder）压缩后的数据——视频帧是 H.264/H.265/VP9 等编码格式，音频帧是 AAC/MP3/Vorbis 等编码格式。

**解码**（decoding）就是将这些压缩数据还原为原始数据的过程：

- **视频解码**：将压缩的视频包还原为原始像素数据（如 YUV420P、NV12 等格式）
- **音频解码**：将压缩的音频包还原为原始 PCM 采样数据（如 S16、Float 等格式）

```
压缩数据包 (AVPacket)          原始帧 (AVFrame)
┌─────────────────────┐       ┌─────────────────────┐
│ H.264 NAL Units     │──────>│ YUV420P 像素数据     │
│ (几 KB ~ 几十 KB)    │ 视频  │ (几百 KB ~ 几 MB)    │
│                     │ 解码  │ width × height × 1.5 │
└─────────────────────┘       └─────────────────────┘

┌─────────────────────┐       ┌─────────────────────┐
│ AAC 压缩帧          │──────>│ Float PCM 采样       │
│ (几百字节)           │ 音频  │ (几 KB)             │
│                     │ 解码  │ samples × channels   │
└─────────────────────┘       └─────────────────────┘
```

### 解码器的核心数据结构

FFmpeg 解码过程涉及三个核心数据结构：

| 结构体 | 头文件 | 作用 |
|--------|--------|------|
| `AVCodec` | `libavcodec/avcodec.h` | 编解码器描述（名称、ID、能力） |
| `AVCodecContext` | `libavcodec/avcodec.h` | 解码器上下文（配置参数、运行状态） |
| `AVFrame` | `libavutil/frame.h` | 解码后的原始帧（像素/采样数据） |

```cpp
// AVCodec — 只读的编解码器描述
// 通过 codec_id 或 name 查找
AVCodec *codec = avcodec_find_decoder(AV_CODEC_ID_H264);
// codec->name       = "h264"
// codec->long_name  = "H.264 / AVC / MPEG-4 AVC / MPEG-4 part 10"
// codec->type       = AVMEDIA_TYPE_VIDEO
// codec->id         = AV_CODEC_ID_H264
// codec->capabilities 包含 AV_CODEC_CAP_DELAY 等能力标志

// AVCodecContext — 可读写的解码器配置
// 每个解码器实例需要独立的 context
AVCodecContext *ctx = avcodec_alloc_context3(codec);
// ctx->width / ctx->height     — 视频尺寸
// ctx->pix_fmt                 — 像素格式 (如 AV_PIX_FMT_YUV420P)
// ctx->sample_rate             — 音频采样率
// ctx->channels                — 音频通道数
// ctx->sample_fmt              — 音频采样格式 (如 AV_SAMPLE_FMT_FLTP)
// ctx->thread_count            — 多线程解码的线程数
// ctx->extradata               — 编解码器特定数据 (SPS/PPS 等)

// AVFrame — 解码输出的原始帧
AVFrame *frame = av_frame_alloc();
// 视频: frame->data[0..3], frame->linesize[0..3], frame->width, frame->height
// 音频: frame->data[0..7], frame->nb_samples, frame->sample_rate, frame->channels
// 通用: frame->pts — 显示时间戳
```

### 新旧 API 对比

FFmpeg 解码 API 经历了重大演进。旧版 API 使用同步的 `avcodec_decode_video2` / `avcodec_decode_audio4`，新版 API（FFmpeg 3.1+）使用异步的 send/receive 模式：

| 特性 | 旧 API | 新 API（推荐） |
|------|--------|---------------|
| 视频解码 | `avcodec_decode_video2()` | `avcodec_send_packet()` + `avcodec_receive_frame()` |
| 音频解码 | `avcodec_decode_audio4()` | `avcodec_send_packet()` + `avcodec_receive_frame()` |
| 返回方式 | 一次调用返回一帧 | 一次 send 可能产生多帧 |
| 刷新方式 | 发送空 packet | 发送 `NULL` packet |
| 线程安全 | 较差 | 更好 |

> **注意**：KrKr2 项目源码中同时使用了两种 API。`CDVDVideoCodecFFmpeg` 使用旧版 `avcodec_decode_video2`，`CDVDAudioCodecFFmpeg` 使用旧版 `avcodec_decode_audio4`，而 sound 模块的 `FFWaveDecoder` 也使用 `avcodec_decode_audio4`。这是因为代码继承自 Kodi/XBMC 较早版本。本节同时讲解两种 API，新项目推荐使用新 API。

## 4.2 视频解码完整流程

### 4.2.1 查找并打开解码器

视频解码的第一步是根据流信息找到对应的解码器，并配置解码器上下文：

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libavutil/imgutils.h>
}

int main() {
    // 第一步：打开文件并获取流信息（参见第三节）
    AVFormatContext *fmtCtx = nullptr;
    if (avformat_open_input(&fmtCtx, "input.mp4", nullptr, nullptr) < 0) {
        fprintf(stderr, "无法打开文件\n");
        return -1;
    }
    if (avformat_find_stream_info(fmtCtx, nullptr) < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmtCtx);
        return -1;
    }

    // 第二步：找到视频流
    int videoIdx = -1;
    for (unsigned i = 0; i < fmtCtx->nb_streams; i++) {
        if (fmtCtx->streams[i]->codecpar->codec_type == AVMEDIA_TYPE_VIDEO) {
            videoIdx = i;
            break;
        }
    }
    if (videoIdx < 0) {
        fprintf(stderr, "未找到视频流\n");
        avformat_close_input(&fmtCtx);
        return -1;
    }

    // 第三步：查找解码器
    AVCodecParameters *codecPar = fmtCtx->streams[videoIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(codecPar->codec_id);
    if (!codec) {
        fprintf(stderr, "未找到解码器: codec_id=%d\n", codecPar->codec_id);
        avformat_close_input(&fmtCtx);
        return -1;
    }
    printf("使用解码器: %s (%s)\n", codec->name, codec->long_name);

    // 第四步：分配并配置解码器上下文
    AVCodecContext *codecCtx = avcodec_alloc_context3(codec);
    if (!codecCtx) {
        fprintf(stderr, "无法分配解码器上下文\n");
        avformat_close_input(&fmtCtx);
        return -1;
    }

    // 从流参数复制配置到解码器上下文
    // 这一步非常重要！它复制了 width/height/pix_fmt/extradata 等关键参数
    if (avcodec_parameters_to_context(codecCtx, codecPar) < 0) {
        fprintf(stderr, "无法复制编解码器参数\n");
        avcodec_free_context(&codecCtx);
        avformat_close_input(&fmtCtx);
        return -1;
    }

    // 第五步：配置多线程解码（可选但推荐）
    codecCtx->thread_count = 4;  // 使用 4 个线程解码
    // thread_type 默认为 FF_THREAD_SLICE | FF_THREAD_FRAME

    // 第六步：打开解码器
    if (avcodec_open2(codecCtx, codec, nullptr) < 0) {
        fprintf(stderr, "无法打开解码器\n");
        avcodec_free_context(&codecCtx);
        avformat_close_input(&fmtCtx);
        return -1;
    }

    printf("视频: %dx%d, 像素格式: %s\n",
           codecCtx->width, codecCtx->height,
           av_get_pix_fmt_name(codecCtx->pix_fmt));

    // ... 开始解码循环 ...

    // 清理
    avcodec_free_context(&codecCtx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

### 4.2.2 新 API 解码循环（推荐）

新 API 使用 `avcodec_send_packet` + `avcodec_receive_frame` 的异步模式：

```cpp
// 分配 AVFrame 和 AVPacket
AVFrame *frame = av_frame_alloc();
AVPacket *pkt = av_packet_alloc();
int frameCount = 0;

// 解码主循环
while (av_read_frame(fmtCtx, pkt) >= 0) {
    if (pkt->stream_index != videoIdx) {
        av_packet_unref(pkt);  // 跳过非视频包
        continue;
    }

    // 发送压缩包给解码器
    int ret = avcodec_send_packet(codecCtx, pkt);
    if (ret < 0) {
        fprintf(stderr, "发送数据包失败: %d\n", ret);
        av_packet_unref(pkt);
        break;
    }

    // 循环接收解码后的帧（一个 packet 可能产生多个 frame）
    while (ret >= 0) {
        ret = avcodec_receive_frame(codecCtx, frame);
        if (ret == AVERROR(EAGAIN)) {
            // 解码器需要更多数据，继续读取下一个 packet
            break;
        }
        if (ret == AVERROR_EOF) {
            // 解码器已完全刷新，没有更多帧
            break;
        }
        if (ret < 0) {
            fprintf(stderr, "接收帧失败: %d\n", ret);
            break;
        }

        // 成功获取一帧！
        frameCount++;
        printf("帧 #%d: pts=%lld, size=%dx%d, format=%s, key=%d\n",
               frameCount,
               (long long)frame->pts,
               frame->width, frame->height,
               av_get_pix_fmt_name((AVPixelFormat)frame->format),
               frame->key_frame);

        // 帧数据在 frame->data[0..3] 中
        // YUV420P: data[0]=Y, data[1]=U, data[2]=V
        // linesize[i] 是每行的字节数（可能包含 padding）

        av_frame_unref(frame);  // 释放帧数据，为下一帧做准备
    }

    av_packet_unref(pkt);  // 释放包数据
}

// 刷新解码器：发送 NULL packet 通知解码器没有更多数据
avcodec_send_packet(codecCtx, nullptr);
while (avcodec_receive_frame(codecCtx, frame) >= 0) {
    frameCount++;
    printf("刷新帧 #%d: pts=%lld\n", frameCount, (long long)frame->pts);
    av_frame_unref(frame);
}

printf("共解码 %d 帧\n", frameCount);

// 清理
av_frame_free(&frame);
av_packet_free(&pkt);
```

> **关键点**：`avcodec_send_packet` 返回 `AVERROR(EAGAIN)` 时表示解码器内部缓冲区已满，需要先调用 `avcodec_receive_frame` 取走已解码的帧后再送入新的 packet。正确的做法是在 send 之前先 drain 所有可用帧。

### 4.2.3 旧 API 解码（KrKr2 使用的方式）

KrKr2 的 `CDVDVideoCodecFFmpeg::Decode` 方法使用的是旧版 API `avcodec_decode_video2`：

```cpp
// 旧 API 风格（仅供理解 KrKr2 源码，新项目请勿使用）
AVPacket avpkt;
av_init_packet(&avpkt);
avpkt.data = pData;       // 压缩数据指针
avpkt.size = iSize;       // 压缩数据大小
avpkt.dts = dts_value;    // 解码时间戳
avpkt.pts = pts_value;    // 显示时间戳

int iGotPicture = 0;
// 返回值 len 是消耗的字节数
int len = avcodec_decode_video2(codecCtx, decodedFrame, &iGotPicture, &avpkt);

if (len < 0) {
    // 解码错误
    return VC_ERROR;
}

if (!iGotPicture) {
    // 本次未产生帧（B帧参考尚未就绪、需要更多数据等）
    return VC_BUFFER;  // 告诉调用者继续喂数据
}

// iGotPicture != 0 表示 decodedFrame 中有有效数据
// 可以通过 av_frame_get_best_effort_timestamp 获取最佳时间戳
int64_t framePTS = av_frame_get_best_effort_timestamp(decodedFrame);
```

### 4.2.4 解码返回值与状态机

解码过程是一个状态机，理解各返回值的含义至关重要：

```
                    ┌──────────────────┐
                    │  等待数据         │
    send_packet()   │  (EAGAIN/BUFFER)  │  receive_frame()
    ┌──────────────>│                  │<──────────────┐
    │               └─────────┬────────┘               │
    │                         │                        │
    │  ┌──────────────────────┼──────────────────────┐ │
    │  │ 新 API               │                      │ │
    │  │ send_packet 返回:    │  receive_frame 返回: │ │
    │  │  0      → 成功       │   0      → 有帧      │ │
    │  │  EAGAIN → 先取帧     │   EAGAIN → 需更多包  │ │
    │  │  EOF    → 已关闭     │   EOF    → 全部刷完  │ │
    │  │  其他   → 错误       │   其他   → 错误      │ │
    │  └──────────────────────┼──────────────────────┘ │
    │                         │                        │
    │  ┌──────────────────────┼──────────────────────┐ │
    │  │ 旧 API               │                      │ │
    │  │ decode_video2 返回:  │                      │ │
    │  │  >0 且 got=1 → 有帧  │                      │ │
    │  │  >0 且 got=0 → 无帧  │                      │ │
    │  │  <0         → 错误   │                      │ │
    │  └──────────────────────┴──────────────────────┘ │
    │                                                  │
    └──────────────────────────────────────────────────┘
```

## 4.3 音频解码完整流程

### 4.3.1 音频解码与视频解码的差异

音频解码在 API 层面与视频解码几乎相同（新 API 共用 `send_packet`/`receive_frame`），但在数据组织上有重要差异：

| 特性 | 视频 | 音频 |
|------|------|------|
| 输出数据 | 像素矩阵 (width × height) | PCM 采样序列 (nb_samples × channels) |
| 数据布局 | 平面式 YUV（Y/U/V 分离）或打包式 RGB | 交错式或平面式 |
| 帧大小 | 固定（取决于分辨率） | 可变（每帧 samples 数不同） |
| `frame->data` | `[0]=Y [1]=U [2]=V` | 交错：`[0]=全部`；平面：`[i]=第i声道` |
| 典型大小 | 几百 KB ~ 几 MB | 几 KB ~ 几十 KB |

### 4.3.2 音频采样格式详解

FFmpeg 定义了多种音频采样格式（`AVSampleFormat`），分为**交错**（interleaved）和**平面**（planar）两大类：

```cpp
// 交错格式 — 所有声道数据交替存储
// [L0][R0][L1][R1][L2][R2]...
AV_SAMPLE_FMT_U8      // unsigned 8-bit
AV_SAMPLE_FMT_S16     // signed 16-bit (CD 质量)
AV_SAMPLE_FMT_S32     // signed 32-bit
AV_SAMPLE_FMT_FLT     // 32-bit float
AV_SAMPLE_FMT_DBL     // 64-bit double

// 平面格式 — 每个声道数据分别存储
// data[0]: [L0][L1][L2]...
// data[1]: [R0][R1][R2]...
AV_SAMPLE_FMT_U8P     // unsigned 8-bit planar
AV_SAMPLE_FMT_S16P    // signed 16-bit planar
AV_SAMPLE_FMT_S32P    // signed 32-bit planar
AV_SAMPLE_FMT_FLTP    // 32-bit float planar（最常见！AAC/Vorbis 等默认输出）
AV_SAMPLE_FMT_DBLP    // 64-bit double planar
```

> **重要**：大多数现代编解码器（AAC、Vorbis、Opus）默认输出**平面格式**（planar），但大多数音频 API（OpenAL、SDL、Windows WASAPI）期望**交错格式**。因此通常需要使用 libswresample 进行格式转换。

### 4.3.3 完整音频解码示例

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libavutil/opt.h>
}

int main() {
    AVFormatContext *fmtCtx = nullptr;
    avformat_open_input(&fmtCtx, "input.mp4", nullptr, nullptr);
    avformat_find_stream_info(fmtCtx, nullptr);

    // 找到音频流
    int audioIdx = av_find_best_stream(fmtCtx, AVMEDIA_TYPE_AUDIO, -1, -1, nullptr, 0);
    if (audioIdx < 0) {
        fprintf(stderr, "未找到音频流\n");
        return -1;
    }

    // 查找解码器并打开
    AVCodecParameters *par = fmtCtx->streams[audioIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);
    avcodec_open2(ctx, codec, nullptr);

    printf("音频: %d Hz, %d 声道, 格式: %s\n",
           ctx->sample_rate, ctx->channels,
           av_get_sample_fmt_name(ctx->sample_fmt));

    // 解码循环
    AVPacket *pkt = av_packet_alloc();
    AVFrame *frame = av_frame_alloc();
    int totalSamples = 0;

    while (av_read_frame(fmtCtx, pkt) >= 0) {
        if (pkt->stream_index != audioIdx) {
            av_packet_unref(pkt);
            continue;
        }

        avcodec_send_packet(ctx, pkt);
        while (avcodec_receive_frame(ctx, frame) >= 0) {
            totalSamples += frame->nb_samples;

            // 判断是平面还是交错格式
            int isPlanar = av_sample_fmt_is_planar(ctx->sample_fmt);
            int bytesPerSample = av_get_bytes_per_sample(ctx->sample_fmt);

            printf("  音频帧: %d 采样, pts=%lld, %s, %d bytes/sample\n",
                   frame->nb_samples,
                   (long long)frame->pts,
                   isPlanar ? "平面" : "交错",
                   bytesPerSample);

            if (isPlanar) {
                // 平面格式：每个声道在 frame->data[i] 中
                for (int ch = 0; ch < ctx->channels; ch++) {
                    // frame->data[ch] 指向第 ch 个声道的数据
                    // 数据长度 = nb_samples * bytesPerSample
                    uint8_t *channelData = frame->data[ch];
                    int dataSize = frame->nb_samples * bytesPerSample;
                    // 处理 channelData...
                    (void)channelData;
                    (void)dataSize;
                }
            } else {
                // 交错格式：所有声道数据在 frame->data[0] 中
                uint8_t *interleavedData = frame->data[0];
                int dataSize = frame->nb_samples * ctx->channels * bytesPerSample;
                // 处理 interleavedData...
                (void)interleavedData;
                (void)dataSize;
            }

            av_frame_unref(frame);
        }
        av_packet_unref(pkt);
    }

    // 刷新解码器
    avcodec_send_packet(ctx, nullptr);
    while (avcodec_receive_frame(ctx, frame) >= 0) {
        totalSamples += frame->nb_samples;
        av_frame_unref(frame);
    }

    double duration = (double)totalSamples / ctx->sample_rate;
    printf("共解码 %d 采样 (%.2f 秒)\n", totalSamples, duration);

    av_frame_free(&frame);
    av_packet_free(&pkt);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

## 4.4 像素格式转换（libswscale）与音频重采样（libswresample）

### 4.4.1 为什么需要格式转换

解码器输出的原始数据格式通常不是应用程序直接需要的：

- **视频**：解码器输出 YUV420P，但渲染器可能需要 RGB24 或 RGBA
- **音频**：解码器输出 FLTP（float planar），但播放器 API 需要 S16（signed 16-bit interleaved）

### 4.4.2 视频像素格式转换 — sws_scale

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavutil/imgutils.h>
#include <libswscale/swscale.h>
}

// 假设 codecCtx 和 frame 已经通过解码获得

void convertYUVtoRGB(AVCodecContext *codecCtx, AVFrame *yuvFrame) {
    int width = codecCtx->width;
    int height = codecCtx->height;

    // 第一步：创建转换上下文
    struct SwsContext *swsCtx = sws_getContext(
        width, height, codecCtx->pix_fmt,     // 源：解码器输出格式
        width, height, AV_PIX_FMT_RGB24,      // 目标：RGB24
        SWS_BILINEAR,                          // 缩放算法（此处无缩放）
        nullptr, nullptr, nullptr
    );
    if (!swsCtx) {
        fprintf(stderr, "无法创建 SwsContext\n");
        return;
    }

    // 第二步：分配目标缓冲区
    uint8_t *dstData[4] = {nullptr};
    int dstLinesize[4] = {0};
    int bufSize = av_image_alloc(
        dstData, dstLinesize,
        width, height, AV_PIX_FMT_RGB24,
        1  // 对齐值
    );
    if (bufSize < 0) {
        fprintf(stderr, "无法分配图像缓冲区\n");
        sws_freeContext(swsCtx);
        return;
    }

    // 第三步：执行转换
    sws_scale(
        swsCtx,
        yuvFrame->data,        // 源数据
        yuvFrame->linesize,    // 源行宽
        0, height,             // 源切片：从第 0 行到第 height 行
        dstData,               // 目标数据
        dstLinesize            // 目标行宽
    );

    printf("转换完成: %dx%d RGB24, 行宽=%d, 总大小=%d bytes\n",
           width, height, dstLinesize[0], bufSize);

    // dstData[0] 现在包含 RGB24 像素数据
    // 可以写入文件、传给 OpenGL 纹理等

    // 清理
    av_freep(&dstData[0]);
    sws_freeContext(swsCtx);
}
```

> **性能提示**：`SwsContext` 应当复用而非每帧重新创建。在 KrKr2 中，渲染管线通过 `CRenderManager` 统一管理格式转换。

### 4.4.3 音频重采样 — swr_convert

```cpp
#include <cstdio>
extern "C" {
#include <libswresample/swresample.h>
#include <libavutil/opt.h>
#include <libavutil/channel_layout.h>
}

// 将解码器输出的 FLTP 44100Hz 立体声
// 转换为 S16 交错格式 48000Hz 立体声
void resampleAudio(AVFrame *inputFrame) {
    SwrContext *swrCtx = swr_alloc();
    if (!swrCtx) {
        fprintf(stderr, "无法分配 SwrContext\n");
        return;
    }

    // 设置输入参数
    av_opt_set_int(swrCtx, "in_channel_layout",  AV_CH_LAYOUT_STEREO, 0);
    av_opt_set_int(swrCtx, "in_sample_rate",      44100, 0);
    av_opt_set_sample_fmt(swrCtx, "in_sample_fmt", AV_SAMPLE_FMT_FLTP, 0);

    // 设置输出参数
    av_opt_set_int(swrCtx, "out_channel_layout", AV_CH_LAYOUT_STEREO, 0);
    av_opt_set_int(swrCtx, "out_sample_rate",     48000, 0);
    av_opt_set_sample_fmt(swrCtx, "out_sample_fmt", AV_SAMPLE_FMT_S16, 0);

    // 初始化
    if (swr_init(swrCtx) < 0) {
        fprintf(stderr, "无法初始化 SwrContext\n");
        swr_free(&swrCtx);
        return;
    }

    // 计算输出采样数（考虑采样率变换）
    int outSamples = av_rescale_rnd(
        swr_get_delay(swrCtx, 44100) + inputFrame->nb_samples,
        48000,   // 目标采样率
        44100,   // 源采样率
        AV_ROUND_UP
    );

    // 分配输出缓冲区
    uint8_t *outBuf = nullptr;
    int outBufSize = av_samples_alloc(
        &outBuf, nullptr,
        2,              // 声道数
        outSamples,     // 采样数
        AV_SAMPLE_FMT_S16,
        0               // 对齐
    );

    // 执行重采样
    int convertedSamples = swr_convert(
        swrCtx,
        &outBuf,                  // 输出缓冲区
        outSamples,               // 输出最大采样数
        (const uint8_t **)inputFrame->data,  // 输入数据
        inputFrame->nb_samples    // 输入采样数
    );

    printf("重采样: %d -> %d 采样 (44100Hz FLTP -> 48000Hz S16)\n",
           inputFrame->nb_samples, convertedSamples);

    // outBuf 现在包含 S16 交错格式的 PCM 数据
    // 可以直接传给 OpenAL/SDL 等音频 API

    av_freep(&outBuf);
    swr_free(&swrCtx);
}
```

### 4.4.4 常见错误与解决方案

**错误 1：解码后画面偏绿/偏色**

```
原因：直接将 YUV 数据当作 RGB 使用，或 linesize 处理错误
解决：必须通过 sws_scale 正确转换；注意 linesize 可能大于 width × bytesPerPixel
```

**错误 2：音频播放有杂音/噪声**

```
原因：平面格式数据未转为交错格式就直接送给播放器
解决：检查 av_sample_fmt_is_planar()，若为平面格式则用 swr_convert 转换
     或手动交错（如 KrKr2 的 FFWaveDecoder::CopySmaples 所做）
```

**错误 3：sws_getContext 返回 NULL**

```
原因：不支持的像素格式组合
解决：先用 sws_isSupportedInput/sws_isSupportedOutput 检查格式支持
     在 Windows 上确保链接了 libswscale
```

## 4.5 硬件加速解码

### 4.5.1 什么是硬件加速解码

现代 GPU 和专用芯片可以承担视频解码任务，比 CPU 软件解码效率高数倍且功耗更低：

| 平台 | 硬件加速 API | 对应 FFmpeg 像素格式 |
|------|-------------|---------------------|
| Windows | DXVA2 / D3D11VA | `AV_PIX_FMT_DXVA2_VLD` / `AV_PIX_FMT_D3D11` |
| Linux | VAAPI / VDPAU | `AV_PIX_FMT_VAAPI` / `AV_PIX_FMT_VDPAU` |
| macOS/iOS | VideoToolbox | `AV_PIX_FMT_VIDEOTOOLBOX` |
| Android | MediaCodec | `AV_PIX_FMT_MEDIACODEC` |
| 树莓派 | MMAL | 通过 `AV_PIX_FMT_YUV420P` 拦截 |

### 4.5.2 KrKr2 的硬件加速架构

KrKr2 通过 `CDVDVideoCodecFFmpeg::GetFormat` 回调实现硬件加速协商：

```
avcodec 需要选择像素格式时回调 GetFormat
    │
    ├─ 检查当前 decoderState
    │   └─ STATE_HW_SINGLE → 尝试硬件解码
    │   └─ 其他状态 → 直接返回默认软解格式
    │
    ├─ 遍历 FFmpeg 提供的候选格式列表
    │   ├─ VDPAU 格式 → 创建 VDPAU::CDecoder，尝试 Open()
    │   ├─ DXVA2 格式 → 创建 DXVA::CDecoder，尝试 Open()
    │   ├─ VAAPI 格式 → 创建 VAAPI::CDecoder，尝试 Open()
    │   ├─ VTB 格式   → 创建 VTB::CDecoder，尝试 Open()
    │   └─ MMAL 格式  → 创建 MMAL::CDecoder，尝试 Open()
    │
    ├─ 硬件解码器 Open 成功
    │   └─ 设置 m_pHardware，返回硬件像素格式
    │
    └─ 全部失败
        └─ m_decoderState = STATE_HW_FAILED
        └─ 返回 avcodec_default_get_format（回退到软解）
```

对应源码（`VideoCodecFFmpeg.cpp` 第 104-230 行）中的关键逻辑：

```cpp
// 来源：krkr2/cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 104-230 行
enum AVPixelFormat CDVDVideoCodecFFmpeg::GetFormat(
    struct AVCodecContext *avctx, const AVPixelFormat *fmt)
{
    CDVDVideoCodecFFmpeg *ctx = (CDVDVideoCodecFFmpeg *)avctx->opaque;

    // 非硬件单解码状态 → 直接返回默认格式（软解）
    if (ctx->m_decoderState != STATE_HW_SINGLE || ...) {
        return avcodec_default_get_format(avctx, fmt);
    }

    // 清理旧的硬件解码器
    if (ctx->GetHardware()) {
        ctx->SetHardware(nullptr);
        avctx->get_buffer2 = avcodec_default_get_buffer2;
        // ...
    }

    // 遍历候选格式，尝试各平台硬件解码器
    const AVPixelFormat *cur = fmt;
    while (*cur != AV_PIX_FMT_NONE) {
#ifdef HAS_DX
        if (DXVA::CDecoder::Supports(*cur) && /* 设置已启用 */) {
            DXVA::CDecoder *dec = new DXVA::CDecoder(ctx->m_processInfo);
            if (dec->Open(avctx, ctx->m_pCodecContext, *cur,
                          ctx->m_uSurfacesCount)) {
                ctx->SetHardware(dec);  // 硬件解码器绑定成功
                return *cur;
            } else {
                dec->Release();  // 打开失败，释放并继续尝试下一个
            }
        }
#endif
        // ... 类似处理 VDPAU, VAAPI, VTB, MMAL
        cur++;
    }

    // 所有硬件解码器都失败了
    ctx->m_decoderState = STATE_HW_FAILED;
    return avcodec_default_get_format(avctx, fmt);  // 回退软解
}
```

// 立体声优化：避免通道循环的开销
template <typename T>
static unsigned char *_CopySmaples2(unsigned char *dst, AVFrame *frame,
                                    int samples, int buf_index) {
    int buf_pos = buf_index * sizeof(T);
    T *pDst = (T *)dst;
    for (int i = 0; i < samples; ++i, buf_pos += sizeof(T)) {
        *pDst++ = *(T *)(frame->data[0] + buf_pos);  // 左声道
        *pDst++ = *(T *)(frame->data[1] + buf_pos);  // 右声道
    }
    return (unsigned char *)pDst;
}

## 4.7 动手实践

### 实践 1：编写完整的视频帧提取器

目标：提取视频前 10 帧并保存为 PPM 图片文件。

```cpp
// save_frames.cpp — 提取视频帧并保存为 PPM 格式
#include <cstdio>
#include <cstdlib>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libavutil/imgutils.h>
#include <libswscale/swscale.h>
}

// 将 RGB 帧保存为 PPM 文件
static void savePPM(AVFrame *frame, int width, int height, int idx) {
    char filename[64];
    snprintf(filename, sizeof(filename), "frame_%03d.ppm", idx);

    FILE *fp = fopen(filename, "wb");
    if (!fp) return;

    // PPM 文件头
    fprintf(fp, "P6\n%d %d\n255\n", width, height);

    // 逐行写入（注意 linesize 可能有 padding）
    for (int y = 0; y < height; y++) {
        fwrite(frame->data[0] + y * frame->linesize[0],
               1, width * 3, fp);
    }
    fclose(fp);
    printf("保存: %s\n", filename);
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <视频文件>\n", argv[0]);
        return 1;
    }

    // 打开文件
    AVFormatContext *fmtCtx = nullptr;
    if (avformat_open_input(&fmtCtx, argv[1], nullptr, nullptr) < 0) {
        fprintf(stderr, "无法打开: %s\n", argv[1]);
        return 1;
    }
    avformat_find_stream_info(fmtCtx, nullptr);

    // 找视频流
    int videoIdx = av_find_best_stream(fmtCtx, AVMEDIA_TYPE_VIDEO,
                                        -1, -1, nullptr, 0);
    if (videoIdx < 0) {
        fprintf(stderr, "未找到视频流\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 打开解码器
    AVCodecParameters *par = fmtCtx->streams[videoIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);
    ctx->thread_count = 4;
    avcodec_open2(ctx, codec, nullptr);

    // 准备格式转换
    struct SwsContext *swsCtx = sws_getContext(
        ctx->width, ctx->height, ctx->pix_fmt,
        ctx->width, ctx->height, AV_PIX_FMT_RGB24,
        SWS_BILINEAR, nullptr, nullptr, nullptr
    );

    AVFrame *frame = av_frame_alloc();
    AVFrame *rgbFrame = av_frame_alloc();
    av_image_alloc(rgbFrame->data, rgbFrame->linesize,
                   ctx->width, ctx->height, AV_PIX_FMT_RGB24, 1);

    AVPacket *pkt = av_packet_alloc();
    int savedFrames = 0;

    // 解码循环
    while (av_read_frame(fmtCtx, pkt) >= 0 && savedFrames < 10) {
        if (pkt->stream_index != videoIdx) {
            av_packet_unref(pkt);
            continue;
        }

        avcodec_send_packet(ctx, pkt);
        while (avcodec_receive_frame(ctx, frame) >= 0 && savedFrames < 10) {
            // YUV → RGB 转换
            sws_scale(swsCtx, frame->data, frame->linesize,
                      0, ctx->height, rgbFrame->data, rgbFrame->linesize);

            savePPM(rgbFrame, ctx->width, ctx->height, ++savedFrames);
            av_frame_unref(frame);
        }
        av_packet_unref(pkt);
    }

    printf("共保存 %d 帧\n", savedFrames);

    // 清理（严格按顺序）
    av_freep(&rgbFrame->data[0]);
    av_frame_free(&rgbFrame);
    av_frame_free(&frame);
    av_packet_free(&pkt);
    sws_freeContext(swsCtx);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

### 实践 2：统计音频帧信息

```cpp
// audio_info.cpp — 统计音频流的详细信息
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件>\n", argv[0]);
        return 1;
    }

    AVFormatContext *fmtCtx = nullptr;
    avformat_open_input(&fmtCtx, argv[1], nullptr, nullptr);
    avformat_find_stream_info(fmtCtx, nullptr);

    int audioIdx = av_find_best_stream(fmtCtx, AVMEDIA_TYPE_AUDIO,
                                        -1, -1, nullptr, 0);
    if (audioIdx < 0) { fprintf(stderr, "无音频流\n"); return 1; }

    AVCodecParameters *par = fmtCtx->streams[audioIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);
    avcodec_open2(ctx, codec, nullptr);

    AVPacket *pkt = av_packet_alloc();
    AVFrame *frame = av_frame_alloc();

    int frameCount = 0;
    int64_t totalSamples = 0;
    int minSamples = INT32_MAX, maxSamples = 0;

    while (av_read_frame(fmtCtx, pkt) >= 0) {
        if (pkt->stream_index != audioIdx) { av_packet_unref(pkt); continue; }
        avcodec_send_packet(ctx, pkt);
        while (avcodec_receive_frame(ctx, frame) >= 0) {
            frameCount++;
            totalSamples += frame->nb_samples;
            if (frame->nb_samples < minSamples) minSamples = frame->nb_samples;
            if (frame->nb_samples > maxSamples) maxSamples = frame->nb_samples;
            av_frame_unref(frame);
        }
        av_packet_unref(pkt);
    }

    printf("=== 音频流统计 ===\n");
    printf("编解码器: %s\n", codec->long_name);
    printf("采样率:   %d Hz\n", ctx->sample_rate);
    printf("声道数:   %d\n", ctx->channels);
    printf("采样格式: %s\n", av_get_sample_fmt_name(ctx->sample_fmt));
    printf("比特率:   %lld kbps\n", (long long)ctx->bit_rate / 1000);
    printf("总帧数:   %d\n", frameCount);
    printf("总采样:   %lld (%.2f 秒)\n", (long long)totalSamples,
           (double)totalSamples / ctx->sample_rate);
    printf("每帧采样: %d ~ %d\n", minSamples, maxSamples);

    av_frame_free(&frame);
    av_packet_free(&pkt);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

## 4.8 本节小结

- **解码**是将 `AVPacket`（压缩数据）转换为 `AVFrame`（原始像素/PCM 数据）的过程
- **新 API**（`avcodec_send_packet` / `avcodec_receive_frame`）是推荐方式，支持异步和多帧输出
- **旧 API**（`avcodec_decode_video2` / `avcodec_decode_audio4`）仍在 KrKr2 中广泛使用
- **视频解码**输出 YUV 格式像素，通常需要通过 `sws_scale`（libswscale）转为 RGB 供渲染
- **音频解码**输出 PCM 采样，多数编解码器输出平面格式，需要转为交错格式供播放 API 使用
- **硬件加速**通过 `get_format` 回调协商，KrKr2 支持 DXVA2/VAAPI/VDPAU/VideoToolbox/MMAL
- **多线程软解**是硬件加速的回退方案，KrKr2 使用 `核心数 × 1.5`（上限 16）个线程
- **KrKr2 的解码架构**分为 movie 模块（`CDVDVideoCodecFFmpeg` / `CDVDAudioCodecFFmpeg`）和 sound 模块（`FFWaveDecoder`），前者用于视频播放，后者用于 BGM/SE 音频

## 4.9 练习题与答案

### 题目 1：新旧 API 的关键区别

请回答以下问题：
1. 新 API `avcodec_send_packet` 返回 `AVERROR(EAGAIN)` 时应该怎么处理？
2. 为什么新 API 中一次 `send_packet` 后需要循环调用 `receive_frame`？
3. KrKr2 为什么仍使用旧 API？

<details>
<summary>查看答案</summary>

1. 返回 `AVERROR(EAGAIN)` 表示解码器内部缓冲区已满，应先循环调用 `avcodec_receive_frame` 取走所有已解码帧，然后再重新调用 `avcodec_send_packet` 发送该包。

2. 因为某些编解码器（特别是音频编解码器）可能从一个 packet 中解码出多个 frame。例如，一个 AAC packet 可能包含多个音频帧。此外，B 帧重排也可能导致之前缓存的帧在此次 send 后变为可用。

3. KrKr2 的 movie 模块移植自 Kodi/XBMC 较早版本（约 2015-2016 年），当时新 API 尚未普及或尚不稳定。由于代码已经稳定运行，迁移到新 API 的收益不高，且需要大量测试验证。

</details>

### 题目 2：修复音频杂音问题

以下代码从 FFmpeg 解码音频后直接送给 OpenAL 播放，但听到的是杂音。请找出问题并修复：

```cpp
// 有 Bug 的代码
AVFrame *frame = av_frame_alloc();
avcodec_send_packet(ctx, pkt);
avcodec_receive_frame(ctx, frame);

// 直接将 frame->data[0] 送给 OpenAL
alBufferData(buffer, AL_FORMAT_STEREO16,
             frame->data[0],
             frame->nb_samples * 4,  // 2 channels * 2 bytes
             ctx->sample_rate);
```

<details>
<summary>查看答案</summary>

问题在于大多数现代音频编解码器（AAC、Vorbis 等）输出**平面格式**（如 `AV_SAMPLE_FMT_FLTP`），此时 `frame->data[0]` 只包含左声道数据，`frame->data[1]` 包含右声道数据。直接使用 `frame->data[0]` 会导致：
1. 只有左声道数据，缺少右声道
2. 数据格式可能是 float 而非 int16

修复代码：

```cpp
#include <libswresample/swresample.h>

// 创建重采样器（初始化一次，复用）
SwrContext *swrCtx = swr_alloc_set_opts(
    nullptr,
    AV_CH_LAYOUT_STEREO,   // 输出布局
    AV_SAMPLE_FMT_S16,     // 输出格式：16-bit 交错
    ctx->sample_rate,       // 输出采样率
    ctx->channel_layout,    // 输入布局
    ctx->sample_fmt,        // 输入格式（可能是 FLTP）
    ctx->sample_rate,       // 输入采样率
    0, nullptr
);
swr_init(swrCtx);

// 解码
AVFrame *frame = av_frame_alloc();
avcodec_send_packet(ctx, pkt);
avcodec_receive_frame(ctx, frame);

// 重采样/格式转换
uint8_t *outBuf = nullptr;
int outSamples = frame->nb_samples;
av_samples_alloc(&outBuf, nullptr, 2, outSamples, AV_SAMPLE_FMT_S16, 0);
swr_convert(swrCtx,
            &outBuf, outSamples,
            (const uint8_t **)frame->data, frame->nb_samples);

// 现在 outBuf 是 S16 交错格式，可以正确送给 OpenAL
alBufferData(buffer, AL_FORMAT_STEREO16,
             outBuf,
             outSamples * 2 * 2,  // samples * channels * bytesPerSample
             ctx->sample_rate);

av_freep(&outBuf);
av_frame_unref(frame);
```

</details>

### 题目 3：分析 KrKr2 的丢帧策略

阅读 `CDVDVideoCodecFFmpeg::CDropControl` 的源码（`VideoCodecFFmpeg.cpp` 第 61-102 行），回答：
1. `CDropControl` 有几个状态？各状态的含义是什么？
2. 从 INIT 切换到 VALID 需要满足什么条件？
3. 从 VALID 回退到 INIT 需要满足什么条件？

<details>
<summary>查看答案</summary>

1. **两个状态**：
   - `INIT`：初始化/学习阶段，正在收集帧间 PTS 差值数据
   - `VALID`：有效阶段，已经学习到稳定的帧间隔，可以用于检测丢帧

2. **INIT → VALID 的条件**（第 79-87 行）：
   - 收集了超过 **10 帧**有效的 PTS 差值（`m_count > 10`）
   - 计算出的平均帧间隔 `m_diffPTS` 大于 0（`m_diffPTS > 0`）
   
   计算方式是累加所有相邻帧的 PTS 差值，然后除以帧数得到平均值。

3. **VALID → INIT 的条件**（第 89-97 行）：
   - 在非丢帧状态下（`!drop`），连续检测到 **超过 5 次**实际 PTS 差值与平均值偏离超过 **20%**
   - 即：`|pts - m_lastPTS - m_diffPTS| > m_diffPTS * 0.2` 连续出现 5 次以上
   
   这意味着视频流的帧率发生了实质性变化（如从 30fps 切换到 24fps），需要重新学习帧间隔。

</details>
