## 解码的概念

### 什么是解码

在第 3 章中，我们通过**解封装**（Demuxing，从容器文件中分离各个流的压缩数据包）得到了 `AVPacket`。这些 `AVPacket` 中存储的是经过**编码器**（Encoder，将原始数据压缩为体积更小的格式的程序）压缩后的数据——视频可能是 H.264、H.265、VP9 等格式，音频可能是 AAC、MP3、Vorbis 等格式。

**解码**（Decoding）就是将这些压缩数据还原为原始数据的过程：

- **视频解码：** 将压缩的视频包还原为原始像素数据。例如，H.264 编码的几 KB 数据，解码后变成一帧完整的 YUV420P（一种像素格式，Y 是亮度，U/V 是色度，4:2:0 表示色度采样率是亮度的 1/4）像素矩阵，大小可达几百 KB 甚至几 MB
- **音频解码：** 将压缩的音频包还原为原始 PCM（Pulse Code Modulation，脉冲编码调制——将模拟声波按固定间隔采样并量化为数字值）采样数据

```
压缩数据包 (AVPacket)              原始帧 (AVFrame)
┌─────────────────────┐           ┌─────────────────────┐
│ H.264 NAL Units     │──────────>│ YUV420P 像素数据     │
│ (几 KB ~ 几十 KB)    │   视频    │ (几百 KB ~ 几 MB)    │
│                     │   解码    │ width × height × 1.5 │
└─────────────────────┘           └─────────────────────┘

┌─────────────────────┐           ┌─────────────────────┐
│ AAC 压缩帧          │──────────>│ Float PCM 采样       │
│ (几百字节)           │   音频    │ (几 KB)             │
│                     │   解码    │ samples × channels   │
└─────────────────────┘           └─────────────────────┘
```

### 为什么解封装和解码要分成两步

你可能会问：为什么不直接"打开文件 → 得到原始帧"，中间为什么要分成解封装和解码两步？

原因是**解耦合**。同一种编码格式可以放在不同容器中：
- H.264 编码的视频可以放在 MP4、MKV、AVI、TS 容器中
- AAC 编码的音频可以放在 MP4、MKV、ADTS 容器中

如果把解封装和解码写在一起，M 种容器 × N 种编码需要 M×N 份代码。分开后只需 M + N 份——解封装器只关心容器格式，解码器只关心编码格式，彼此独立。

在 FFmpeg 中：
- **`libavformat`** 负责解封装（操作 `AVFormatContext`、`AVPacket`）
- **`libavcodec`** 负责解码（操作 `AVCodecContext`、`AVFrame`）

## 解码器的核心数据结构

FFmpeg 解码过程涉及三个核心数据结构：

### AVCodec——编解码器描述

`AVCodec` 是一个**只读**的编解码器描述结构，包含编解码器的名称、ID、支持的能力等。你不需要创建它，只需要通过 `avcodec_find_decoder()` 查找：

```c
#include <libavcodec/avcodec.h>

// 方法 1：通过 codec_id 查找（最常用）
// codec_id 通常从 AVStream 的 codecpar 中获取
AVCodec *codec = avcodec_find_decoder(AV_CODEC_ID_H264);

// 方法 2：通过名称查找（用于指定特定实现）
AVCodec *codec2 = avcodec_find_decoder_by_name("h264");

// AVCodec 的关键字段（只读）
printf("名称: %s\n", codec->name);        // "h264"
printf("全名: %s\n", codec->long_name);    // "H.264 / AVC / MPEG-4 AVC"
printf("类型: %d\n", codec->type);          // AVMEDIA_TYPE_VIDEO = 0
printf("ID:   %d\n", codec->id);            // AV_CODEC_ID_H264 = 27

// 能力标志（capabilities）
// AV_CODEC_CAP_DELAY: 解码器有延迟，需要在结束时刷新
// AV_CODEC_CAP_FRAME_THREADS: 支持帧级多线程
// AV_CODEC_CAP_SLICE_THREADS: 支持切片级多线程
if (codec->capabilities & AV_CODEC_CAP_DELAY) {
    printf("该解码器有延迟，需要结尾刷新\n");
}
```

### AVCodecContext——解码器上下文

`AVCodecContext` 是**解码器的运行实例**，持有该解码器的所有配置和运行状态。每个同时工作的解码器实例需要独立的 `AVCodecContext`：

```c
// 分配解码器上下文（传入 AVCodec 作为模板）
AVCodecContext *ctx = avcodec_alloc_context3(codec);

// 关键字段——视频解码器
// ctx->width        — 视频宽度（像素）
// ctx->height       — 视频高度（像素）
// ctx->pix_fmt      — 像素格式（如 AV_PIX_FMT_YUV420P）
// ctx->framerate    — 帧率（AVRational 类型，如 {30, 1} 表示 30fps）
// ctx->extradata    — 编解码器特定数据（H.264 的 SPS/PPS 参数集）
// ctx->thread_count — 多线程解码的线程数

// 关键字段——音频解码器
// ctx->sample_rate  — 采样率（如 44100、48000）
// ctx->channels     — 声道数（如 2 表示立体声）
// ctx->sample_fmt   — 采样格式（如 AV_SAMPLE_FMT_FLTP）
// ctx->channel_layout — 声道布局（如 AV_CH_LAYOUT_STEREO）

// 从流参数复制配置——这一步至关重要！
// 它将 AVStream 中记录的宽高、格式、extradata 等信息复制到解码器上下文
AVCodecParameters *codecPar = stream->codecpar;
int ret = avcodec_parameters_to_context(ctx, codecPar);
// 如果跳过这一步，解码器不知道输入数据的格式，无法正确解码

// 配置多线程（可选但推荐）
ctx->thread_count = 4;  // 使用 4 个线程

// 打开解码器——此后才能开始解码
ret = avcodec_open2(ctx, codec, NULL);
if (ret < 0) {
    fprintf(stderr, "无法打开解码器\n");
}

// 使用完毕后释放
avcodec_free_context(&ctx);  // 配对 avcodec_alloc_context3
```

> **注意：** `avcodec_alloc_context3()` 和 `avcodec_free_context()` 是配对的。
> 就像 `avformat_open_input()` 和 `avformat_close_input()` 一样。

### AVFrame——解码后的原始帧

`AVFrame` 保存解码后的原始数据。视频帧包含像素矩阵，音频帧包含 PCM 采样：

```c
#include <libavutil/frame.h>

// 分配 AVFrame（只分配结构体，不分配数据缓冲区）
AVFrame *frame = av_frame_alloc();

// 解码成功后，frame 的关键字段：
// --- 视频帧 ---
// frame->data[0]     — Y 平面数据指针（YUV420P 格式）
// frame->data[1]     — U 平面数据指针
// frame->data[2]     — V 平面数据指针
// frame->linesize[0] — Y 平面每行的字节数（可能 > width，因为有对齐填充）
// frame->linesize[1] — U 平面每行的字节数
// frame->linesize[2] — V 平面每行的字节数
// frame->width       — 帧宽度
// frame->height      — 帧高度
// frame->format      — 像素格式（int，对应 AVPixelFormat 枚举）
// frame->key_frame   — 是否关键帧（1=是，0=否）

// --- 音频帧 ---
// frame->data[0]     — 第 0 声道数据（平面格式）或全部数据（交错格式）
// frame->data[1]     — 第 1 声道数据（仅平面格式）
// frame->nb_samples  — 本帧包含的采样数
// frame->sample_rate — 采样率
// frame->channels    — 声道数
// frame->format      — 采样格式（int，对应 AVSampleFormat 枚举）

// --- 通用字段 ---
// frame->pts         — 显示时间戳（单位由所属流的 time_base 决定）

// 释放帧数据（保留结构体，可复用）
av_frame_unref(frame);

// 释放结构体本身
av_frame_free(&frame);  // 配对 av_frame_alloc
```

### 三者的关系

```
AVCodec（只读模板）
  │
  │ avcodec_find_decoder() 查找
  │
  ▼
AVCodecContext（运行实例）
  │
  │ avcodec_parameters_to_context() 配置
  │ avcodec_open2() 打开
  │
  │ avcodec_send_packet(ctx, pkt) 送入压缩包
  │ avcodec_receive_frame(ctx, frame) 取出原始帧
  │
  ▼
AVFrame（解码输出）
  │
  │ frame->data[] 包含原始像素/采样数据
  │ frame->pts 包含时间戳
  │
  ▼
渲染/播放
```

## 新 API 与旧 API 对比

FFmpeg 的解码 API 在 3.1 版本经历了重大改进。旧版使用同步的 `avcodec_decode_video2` / `avcodec_decode_audio4`，新版使用**异步的 send/receive 模式**：

| 特性 | 旧 API（FFmpeg < 3.1） | 新 API（FFmpeg ≥ 3.1，推荐） |
|------|----------------------|---------------------------|
| 视频解码 | `avcodec_decode_video2()` | `avcodec_send_packet()` + `avcodec_receive_frame()` |
| 音频解码 | `avcodec_decode_audio4()` | 同上（音频视频共用） |
| 一次调用 | 输入一个包，最多输出一帧 | 一次 send 后可能 receive 多帧 |
| 刷新方式 | 发送空 `AVPacket` | 发送 `NULL` 指针 |
| 状态 | FFmpeg 3.4 标记废弃 | 当前推荐 |

> **KrKr2 的情况：** KrKr2 的 movie 模块移植自 Kodi/XBMC 较早版本（约 2015-2016 年），
> 使用的是旧 API `avcodec_decode_video2` 和 `avcodec_decode_audio4`。
> 本教程**同时讲解两种 API**——新项目应使用新 API，但理解旧 API 对阅读 KrKr2 源码至关重要。

## 视频解码完整流程——新 API

### 第一步：查找并打开解码器

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libavutil/imgutils.h>
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <视频文件>\n", argv[0]);
        return 1;
    }

    // === 解封装阶段（第 3 章内容）===
    AVFormatContext *fmtCtx = NULL;
    if (avformat_open_input(&fmtCtx, argv[1], NULL, NULL) < 0) {
        fprintf(stderr, "无法打开文件: %s\n", argv[1]);
        return 1;
    }
    if (avformat_find_stream_info(fmtCtx, NULL) < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 找到视频流
    int videoIdx = -1;
    for (unsigned i = 0; i < fmtCtx->nb_streams; i++) {
        if (fmtCtx->streams[i]->codecpar->codec_type == AVMEDIA_TYPE_VIDEO) {
            videoIdx = (int)i;
            break;
        }
    }
    if (videoIdx < 0) {
        fprintf(stderr, "未找到视频流\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // === 解码准备阶段（本章新内容）===

    // 1. 获取流的编解码器参数
    AVCodecParameters *codecPar = fmtCtx->streams[videoIdx]->codecpar;

    // 2. 查找解码器
    AVCodec *codec = avcodec_find_decoder(codecPar->codec_id);
    if (!codec) {
        fprintf(stderr, "未找到解码器: codec_id=%d\n", codecPar->codec_id);
        avformat_close_input(&fmtCtx);
        return 1;
    }
    printf("解码器: %s (%s)\n", codec->name, codec->long_name);

    // 3. 分配解码器上下文
    AVCodecContext *codecCtx = avcodec_alloc_context3(codec);
    if (!codecCtx) {
        fprintf(stderr, "无法分配解码器上下文\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 4. 复制编解码器参数到上下文
    // 这一步将 width/height/pix_fmt/extradata 等关键参数复制过来
    if (avcodec_parameters_to_context(codecCtx, codecPar) < 0) {
        fprintf(stderr, "无法复制编解码器参数\n");
        avcodec_free_context(&codecCtx);
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 5. 配置多线程解码（可选但推荐）
    codecCtx->thread_count = 4;   // 使用 4 个线程
    // thread_type 默认为 FF_THREAD_SLICE | FF_THREAD_FRAME
    // FF_THREAD_FRAME: 多帧并行解码（每个线程解一帧）
    // FF_THREAD_SLICE: 单帧内多切片并行（适合 H.264）

    // 6. 打开解码器
    if (avcodec_open2(codecCtx, codec, NULL) < 0) {
        fprintf(stderr, "无法打开解码器\n");
        avcodec_free_context(&codecCtx);
        avformat_close_input(&fmtCtx);
        return 1;
    }

    printf("视频: %dx%d, 像素格式: %s\n",
           codecCtx->width, codecCtx->height,
           av_get_pix_fmt_name(codecCtx->pix_fmt));

    // ... 下一步：解码循环 ...

    // 清理
    avcodec_free_context(&codecCtx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

**API 详解：**

| API | 参数 | 返回值 | 作用 |
|-----|------|--------|------|
| `avcodec_find_decoder(id)` | `enum AVCodecID` | `AVCodec*` 或 `NULL` | 根据 ID 查找解码器 |
| `avcodec_alloc_context3(codec)` | `AVCodec*` | `AVCodecContext*` 或 `NULL` | 分配解码器上下文 |
| `avcodec_parameters_to_context(ctx, par)` | 上下文 + 参数 | `0` 成功 / `<0` 错误 | 复制流参数到上下文 |
| `avcodec_open2(ctx, codec, opts)` | 上下文 + 解码器 + 选项 | `0` 成功 / `<0` 错误 | 打开解码器（此后可开始解码） |
| `avcodec_free_context(&ctx)` | 上下文指针的地址 | 无 | 释放解码器上下文（配对 alloc） |

### 第二步：解码循环

```c
// 分配 AVFrame 和 AVPacket
AVFrame *frame = av_frame_alloc();   // 用于接收解码输出
AVPacket pkt;                         // 用于存储压缩数据包
av_init_packet(&pkt);
pkt.data = NULL;
pkt.size = 0;

int frameCount = 0;

// 主循环：逐包读取 → 送入解码器 → 取出解码帧
while (av_read_frame(fmtCtx, &pkt) >= 0) {
    // 只处理视频流的包
    if (pkt.stream_index != videoIdx) {
        av_packet_unref(&pkt);
        continue;
    }

    // === 送入压缩包 ===
    // avcodec_send_packet 将一个 AVPacket 送入解码器的输入队列
    // 解码器内部会缓存数据，可能不会立即产生输出帧
    int ret = avcodec_send_packet(codecCtx, &pkt);
    if (ret < 0) {
        fprintf(stderr, "send_packet 失败: %d\n", ret);
        av_packet_unref(&pkt);
        break;
    }

    // === 取出解码帧 ===
    // 一个 packet 可能产生 0、1 或多个 frame
    // 必须循环调用 receive_frame 直到返回 EAGAIN
    while (ret >= 0) {
        ret = avcodec_receive_frame(codecCtx, frame);

        if (ret == AVERROR(EAGAIN)) {
            // EAGAIN: 解码器需要更多输入数据
            // 跳出内层循环，继续读下一个 packet
            break;
        }
        if (ret == AVERROR_EOF) {
            // EOF: 解码器已完全刷新，没有更多帧
            break;
        }
        if (ret < 0) {
            fprintf(stderr, "receive_frame 失败: %d\n", ret);
            break;
        }

        // === 成功获取一帧！ ===
        frameCount++;
        printf("帧 #%d: pts=%lld, %dx%d, format=%s, key=%d\n",
               frameCount,
               (long long)frame->pts,
               frame->width, frame->height,
               av_get_pix_fmt_name((enum AVPixelFormat)frame->format),
               frame->key_frame);

        // 帧数据在 frame->data[] 中：
        // YUV420P 格式:
        //   data[0] = Y 平面 (width × height 字节)
        //   data[1] = U 平面 (width/2 × height/2 字节)
        //   data[2] = V 平面 (width/2 × height/2 字节)
        // linesize[i] 是每行的字节数（可能因对齐而 > width）

        // 释放帧数据，为下一帧腾出空间
        av_frame_unref(frame);
    }

    av_packet_unref(&pkt);
}

// === 刷新解码器 ===
// 发送 NULL 通知解码器：没有更多输入数据了
// 解码器会输出内部缓存的所有帧（B 帧重排延迟的帧）
avcodec_send_packet(codecCtx, NULL);
while (avcodec_receive_frame(codecCtx, frame) >= 0) {
    frameCount++;
    printf("刷新帧 #%d: pts=%lld\n", frameCount, (long long)frame->pts);
    av_frame_unref(frame);
}

printf("共解码 %d 帧\n", frameCount);

// 清理
av_frame_free(&frame);
```

> **为什么需要"刷新"？** 视频编码中的 B 帧（Bidirectional frame，双向预测帧——同时参考前后两帧来压缩）
> 会导致解码延迟。解码器需要缓存若干帧才能输出正确顺序的帧。
> 发送 `NULL` 告诉解码器"输入结束了"，让它输出所有缓存帧。

### 第三步：完整可运行示例

将上述代码组合为完整的可编译文件：

```c
// video_decode_demo.c — 完整的视频解码演示
#include <stdio.h>

#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <视频文件>\n", argv[0]);
        return 1;
    }

    // 解封装
    AVFormatContext *fmtCtx = NULL;
    avformat_open_input(&fmtCtx, argv[1], NULL, NULL);
    avformat_find_stream_info(fmtCtx, NULL);

    // 找视频流
    int videoIdx = av_find_best_stream(
        fmtCtx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
    if (videoIdx < 0) {
        fprintf(stderr, "未找到视频流\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 打开解码器
    AVCodecParameters *par = fmtCtx->streams[videoIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);
    ctx->thread_count = 4;
    avcodec_open2(ctx, codec, NULL);

    printf("视频: %s %dx%d %s\n",
           codec->name, ctx->width, ctx->height,
           av_get_pix_fmt_name(ctx->pix_fmt));

    // 解码循环
    AVFrame *frame = av_frame_alloc();
    AVPacket pkt;
    int count = 0;

    while (av_read_frame(fmtCtx, &pkt) >= 0) {
        if (pkt.stream_index == videoIdx) {
            avcodec_send_packet(ctx, &pkt);
            while (avcodec_receive_frame(ctx, frame) >= 0) {
                count++;
                printf("帧 #%d: pts=%lld %s\n",
                       count, (long long)frame->pts,
                       frame->key_frame ? "[关键帧]" : "");
                av_frame_unref(frame);
            }
        }
        av_packet_unref(&pkt);
    }

    // 刷新
    avcodec_send_packet(ctx, NULL);
    while (avcodec_receive_frame(ctx, frame) >= 0) {
        count++;
        printf("刷新帧 #%d: pts=%lld\n", count, (long long)frame->pts);
        av_frame_unref(frame);
    }

    printf("共解码 %d 帧\n", count);

    av_frame_free(&frame);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

## 旧 API 解码——理解 KrKr2 源码

KrKr2 的 `CDVDVideoCodecFFmpeg::Decode` 方法（位于 `VideoCodecFFmpeg.cpp` 第 555-726 行）使用旧版 API `avcodec_decode_video2`。理解旧 API 对阅读 KrKr2 源码至关重要：

```c
// 旧 API 风格（仅供理解 KrKr2 源码，新项目请勿使用）

AVPacket avpkt;
av_init_packet(&avpkt);
avpkt.data = pData;       // 压缩数据指针
avpkt.size = iSize;       // 压缩数据大小
avpkt.dts = dts_value;    // 解码时间戳
avpkt.pts = pts_value;    // 显示时间戳

int iGotPicture = 0;  // 输出参数：是否获得了一帧

// avcodec_decode_video2:
//   参数 1: 解码器上下文
//   参数 2: 输出帧（AVFrame*）
//   参数 3: 输出标志（&iGotPicture，非零表示有帧）
//   参数 4: 输入包（AVPacket*）
//   返回值: 消耗的字节数，或负数表示错误
int len = avcodec_decode_video2(codecCtx, decodedFrame,
                                 &iGotPicture, &avpkt);

if (len < 0) {
    // 解码错误——数据损坏或格式不支持
    return VC_ERROR;
}

if (!iGotPicture) {
    // 本次未产生帧——可能的原因：
    // 1. B 帧需要后续参考帧才能输出
    // 2. 解码器内部缓冲未满
    // 3. 输入数据不完整
    return VC_BUFFER;  // 告诉调用者继续喂数据
}

// iGotPicture != 0 表示 decodedFrame 中有有效数据
// 获取最佳时间戳（综合考虑 PTS、DTS 和推测值）
int64_t framePTS = av_frame_get_best_effort_timestamp(decodedFrame);

printf("解码成功: %dx%d, pts=%lld\n",
       decodedFrame->width, decodedFrame->height,
       (long long)framePTS);
```

### 新旧 API 的关键区别

```
旧 API：一次调用 = 一个输入包 + 最多一个输出帧
┌─────────────┐    avcodec_decode_video2     ┌─────────────┐
│  AVPacket   │ ──────────────────────────> │  AVFrame    │
│  (输入)     │   返回: 消耗字节 + got_pic   │  (输出)     │
└─────────────┘                              └─────────────┘

新 API：send/receive 解耦，一次 send 可以多次 receive
┌─────────────┐    avcodec_send_packet       ┌─────────────┐
│  AVPacket   │ ──────────────────────────> │  解码器     │
│  (输入)     │                              │  内部队列   │
└─────────────┘                              │             │
                   avcodec_receive_frame     │             │
┌─────────────┐ <────────────────────────── │             │
│  AVFrame    │   可以调用多次               │             │
│  (输出)     │   直到返回 EAGAIN           └─────────────┘
└─────────────┘
```

## 解码器状态机与返回值

理解各返回值的含义对正确编写解码循环至关重要：

### 新 API 返回值表

| 函数 | 返回值 | 含义 | 处理方式 |
|------|--------|------|---------|
| `send_packet` | `0` | 成功送入 | 继续 receive |
| `send_packet` | `AVERROR(EAGAIN)` | 解码器输入满 | 先 receive 取帧，再重试 send |
| `send_packet` | `AVERROR_EOF` | 解码器已关闭 | 不再 send |
| `send_packet` | 其他负值 | 错误 | 记录错误，停止 |
| `receive_frame` | `0` | 成功取帧 | 处理帧，继续 receive |
| `receive_frame` | `AVERROR(EAGAIN)` | 需要更多输入 | 跳出 receive 循环，send 下一个包 |
| `receive_frame` | `AVERROR_EOF` | 刷新完毕 | 所有帧已取完 |
| `receive_frame` | 其他负值 | 错误 | 记录错误 |

### 状态机图示

```
                    ┌──────────────────┐
                    │  等待输入         │
    send_packet(pkt)│  (初始状态)       │  receive_frame(frame)
    ┌──────────────>│                  │<──────────────┐
    │   返回 0      └─────────┬────────┘   返回 EAGAIN  │
    │                         │                        │
    │               ┌─────────▼────────┐               │
    │               │  内部有缓存帧    │               │
    │   send 返回    │                  │  receive 返回 0│
    │   EAGAIN      │  send → EAGAIN   │───────────────┘
    │ ◄─────────────│  receive → 0     │
    │               └─────────┬────────┘
    │                         │
    │               ┌─────────▼────────┐
    │               │  刷新模式        │
    │ send(NULL)    │  send(NULL) 触发  │  receive 返回 0
    │ ──────────────│                  │───────────────┘
    │               │  直到 receive    │
    │               │  返回 EOF        │
    │               └──────────────────┘
```

## 动手实践

### 练习：统计视频的关键帧分布

编写一个程序，解码视频文件并统计关键帧（I 帧）的分布间隔：

```c
// keyframe_stats.c — 关键帧分布统计
#include <stdio.h>
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <视频文件>\n", argv[0]);
        return 1;
    }

    AVFormatContext *fmtCtx = NULL;
    avformat_open_input(&fmtCtx, argv[1], NULL, NULL);
    avformat_find_stream_info(fmtCtx, NULL);

    int videoIdx = av_find_best_stream(
        fmtCtx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
    if (videoIdx < 0) return 1;

    AVCodecParameters *par = fmtCtx->streams[videoIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);
    ctx->thread_count = 4;
    avcodec_open2(ctx, codec, NULL);

    AVFrame *frame = av_frame_alloc();
    AVPacket pkt;
    int totalFrames = 0, keyFrames = 0;
    int lastKeyFrame = 0;
    int maxGap = 0, minGap = 999999;
    long long totalGap = 0;

    while (av_read_frame(fmtCtx, &pkt) >= 0) {
        if (pkt.stream_index == videoIdx) {
            avcodec_send_packet(ctx, &pkt);
            while (avcodec_receive_frame(ctx, frame) >= 0) {
                totalFrames++;
                if (frame->key_frame) {
                    if (keyFrames > 0) {
                        int gap = totalFrames - lastKeyFrame;
                        if (gap > maxGap) maxGap = gap;
                        if (gap < minGap) minGap = gap;
                        totalGap += gap;
                    }
                    keyFrames++;
                    lastKeyFrame = totalFrames;
                }
                av_frame_unref(frame);
            }
        }
        av_packet_unref(&pkt);
    }

    // 刷新
    avcodec_send_packet(ctx, NULL);
    while (avcodec_receive_frame(ctx, frame) >= 0) {
        totalFrames++;
        if (frame->key_frame) keyFrames++;
        av_frame_unref(frame);
    }

    printf("=== 关键帧统计 ===\n");
    printf("总帧数:     %d\n", totalFrames);
    printf("关键帧数:   %d\n", keyFrames);
    printf("关键帧比例: %.1f%%\n",
           (double)keyFrames / totalFrames * 100);
    if (keyFrames > 1) {
        printf("关键帧间隔: 最小=%d 最大=%d 平均=%.1f 帧\n",
               minGap, maxGap,
               (double)totalGap / (keyFrames - 1));
    }

    av_frame_free(&frame);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

**运行输出示例：**

```
=== 关键帧统计 ===
总帧数:     3600
关键帧数:   120
关键帧比例: 3.3%
关键帧间隔: 最小=24 最大=36 平均=30.0 帧
```

> 这个结果表示视频编码时的 GOP（Group of Pictures，一组图片——从一个关键帧到下一个关键帧之间的所有帧）大小约为 30 帧。

## 常见错误及解决方案

### 错误 1：avcodec_open2 返回 -22（EINVAL）

**现象：** `avcodec_open2()` 返回 `-22`。

**原因：** 忘记调用 `avcodec_parameters_to_context()`，解码器上下文没有必要的编解码器参数（宽高、extradata 等）。

```c
// ❌ 错误：跳过了参数复制
AVCodecContext *ctx = avcodec_alloc_context3(codec);
avcodec_open2(ctx, codec, NULL);  // 返回 -22！

// ✅ 正确：先复制参数再打开
AVCodecContext *ctx = avcodec_alloc_context3(codec);
avcodec_parameters_to_context(ctx, stream->codecpar);  // 关键步骤
avcodec_open2(ctx, codec, NULL);  // 成功
```

### 错误 2：解码出的帧数比预期少

**现象：** 一个 120 帧的视频只解码出 116 帧。

**原因：** 忘记在循环结束后刷新解码器。B 帧解码器有内部延迟，最后几帧缓存在解码器中。

```c
// ❌ 错误：循环结束后直接退出
while (av_read_frame(fmtCtx, &pkt) >= 0) { ... }
// 少了 4 帧！

// ✅ 正确：循环结束后刷新解码器
while (av_read_frame(fmtCtx, &pkt) >= 0) { ... }
avcodec_send_packet(ctx, NULL);  // 发送 NULL 触发刷新
while (avcodec_receive_frame(ctx, frame) >= 0) {
    // 处理缓存帧
    av_frame_unref(frame);
}
```

### 错误 3：linesize 与 width 不一致

**现象：** 将 `frame->data[0]` 按 `width × height` 拷贝后图像错位。

**原因：** `linesize`（每行字节数）通常大于 `width × bytesPerPixel`，因为 FFmpeg 为了 SIMD（Single Instruction Multiple Data，一条指令处理多个数据——CPU 的向量化加速技术） 优化会在每行末尾添加对齐填充。

```c
// ❌ 错误：按 width 拷贝
for (int y = 0; y < height; y++) {
    memcpy(dst + y * width, frame->data[0] + y * width, width);
}

// ✅ 正确：按 linesize 偏移源数据
for (int y = 0; y < height; y++) {
    // 源数据按 linesize 对齐，目标按 width 紧凑排列
    memcpy(dst + y * width,
           frame->data[0] + y * frame->linesize[0],
           width);
}
```

## 本节小结

- **解码**是将 `AVPacket`（压缩数据包）转换为 `AVFrame`（原始像素/PCM 数据）的过程
- FFmpeg 解码涉及三个核心结构：`AVCodec`（编解码器描述）、`AVCodecContext`（运行实例）、`AVFrame`（输出帧）
- **新 API**（`avcodec_send_packet` / `avcodec_receive_frame`）是推荐方式，支持异步和多帧输出
- **旧 API**（`avcodec_decode_video2`）仍在 KrKr2 的 Kodi 移植代码中使用
- 解码器是一个状态机——必须正确处理 `EAGAIN`、`EOF` 等返回值
- 解码循环结束后**必须刷新解码器**（发送 `NULL`），否则会丢失最后几帧
- `linesize` ≠ `width`——对齐填充是常见的坑

## 练习题与答案

### 题目 1：为什么 avcodec_send_packet 可能返回 EAGAIN？应该怎么处理？

<details>
<summary>查看答案</summary>

`AVERROR(EAGAIN)` 表示解码器的**输入缓冲区已满**——解码器内部已经有足够的数据可以产生输出帧了，但输出帧还没有被取走。

**正确处理方式：**

1. 先循环调用 `avcodec_receive_frame()` 取走所有已解码的帧
2. 然后重试 `avcodec_send_packet()` 送入之前被拒绝的包

```c
int ret = avcodec_send_packet(ctx, &pkt);
if (ret == AVERROR(EAGAIN)) {
    // 先取走所有可用帧
    AVFrame *frame = av_frame_alloc();
    while (avcodec_receive_frame(ctx, frame) >= 0) {
        process_frame(frame);
        av_frame_unref(frame);
    }
    av_frame_free(&frame);

    // 重试 send
    ret = avcodec_send_packet(ctx, &pkt);
}
```

在实践中，如果你在每次 `send_packet` 之后都正确地循环调用 `receive_frame` 直到返回 `EAGAIN`，通常不会遇到 `send_packet` 返回 `EAGAIN` 的情况。

</details>

### 题目 2：旧 API `avcodec_decode_video2` 的 `got_picture` 参数为 0 时，是否表示解码失败？

<details>
<summary>查看答案</summary>

**不是。** `got_picture = 0` 有多种正常原因：

1. **B 帧延迟：** 视频编码中的 B 帧需要参考前后两帧。解码器收到 B 帧后可能需要等待后续的参考帧才能输出。此时 `got_picture = 0`，但解码器内部已经缓存了数据。

2. **数据不完整：** 一个 `AVPacket` 可能不包含完整的一帧数据（某些容器格式会将一帧拆分为多个包）。

3. **解码器预填充：** 某些解码器在收到前几个包时需要积累足够的参考帧才能开始输出。

**正确做法：** 返回 `VC_BUFFER`（告诉调用者继续喂数据），而不是报错。这就是 KrKr2 的 `CDVDVideoCodecFFmpeg::Decode` 方法的处理方式。

```cpp
// KrKr2 的处理逻辑
if (!iGotPicture) {
    return VC_BUFFER;  // 继续喂数据，不是错误
}
```

</details>

### 题目 3：编写一个程序，解码视频的前 5 帧并打印每帧的 YUV 数据统计信息（Y 平面的平均亮度值）。

<details>
<summary>查看答案</summary>

```c
#include <stdio.h>
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>

int main(int argc, char *argv[]) {
    if (argc < 2) return 1;

    AVFormatContext *fmtCtx = NULL;
    avformat_open_input(&fmtCtx, argv[1], NULL, NULL);
    avformat_find_stream_info(fmtCtx, NULL);

    int vi = av_find_best_stream(fmtCtx, AVMEDIA_TYPE_VIDEO, -1, -1, NULL, 0);
    if (vi < 0) return 1;

    AVCodec *codec = avcodec_find_decoder(
        fmtCtx->streams[vi]->codecpar->codec_id);
    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, fmtCtx->streams[vi]->codecpar);
    avcodec_open2(ctx, codec, NULL);

    AVFrame *frame = av_frame_alloc();
    AVPacket pkt;
    int count = 0;

    while (av_read_frame(fmtCtx, &pkt) >= 0 && count < 5) {
        if (pkt.stream_index == vi) {
            avcodec_send_packet(ctx, &pkt);
            while (avcodec_receive_frame(ctx, frame) >= 0 && count < 5) {
                count++;
                // 计算 Y 平面平均亮度
                long long sum = 0;
                int w = frame->width;
                int h = frame->height;
                for (int y = 0; y < h; y++) {
                    uint8_t *row = frame->data[0] + y * frame->linesize[0];
                    for (int x = 0; x < w; x++) {
                        sum += row[x];
                    }
                }
                double avgY = (double)sum / (w * h);
                printf("帧 #%d: %dx%d, 平均亮度 Y=%.1f (0=黑 255=白)"
                       " pts=%lld %s\n",
                       count, w, h, avgY,
                       (long long)frame->pts,
                       frame->key_frame ? "[关键帧]" : "");
                av_frame_unref(frame);
            }
        }
        av_packet_unref(&pkt);
    }

    av_frame_free(&frame);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

**输出示例：**
```
帧 #1: 1920x1080, 平均亮度 Y=45.2 (0=黑 255=白) pts=0 [关键帧]
帧 #2: 1920x1080, 平均亮度 Y=46.1 (0=黑 255=白) pts=3600
帧 #3: 1920x1080, 平均亮度 Y=47.3 (0=黑 255=白) pts=7200
帧 #4: 1920x1080, 平均亮度 Y=48.0 (0=黑 255=白) pts=10800
帧 #5: 1920x1080, 平均亮度 Y=48.5 (0=黑 255=白) pts=14400
```

平均亮度值 45-48 说明画面偏暗（视频可能是暗色调场景）。明亮的户外场景通常在 120-180 范围。

</details>
