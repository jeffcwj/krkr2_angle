## 音频解码基础

### 音频解码与视频解码的异同

在上一节中，我们学习了视频解码的完整流程。音频解码在 API 层面几乎完全相同——都使用 `avcodec_send_packet()` 发送压缩数据包，然后用 `avcodec_receive_frame()` 接收解码后的原始帧。但在数据的本质和组织方式上，音频与视频有显著差异：

| 特性 | 视频帧 | 音频帧 |
|------|--------|--------|
| 数据含义 | 一张完整的图片（像素矩阵） | 一段时间的声音波形（采样序列） |
| 输出格式 | 像素格式，如 YUV420P、RGB24 | 采样格式，如 S16（16位有符号整数）、FLTP（32位浮点平面） |
| 帧大小 | 固定（取决于分辨率，如 1920×1080） | 可变（每帧采样数不同，通常 960~4096 个采样） |
| `frame->data` 含义 | `[0]=Y平面 [1]=U平面 [2]=V平面`（YUV420P 时） | 交错模式下 `[0]=全部数据`；平面模式下 `[i]=第i声道` |
| 典型大小 | 几百 KB 到几 MB | 几 KB 到几十 KB |
| 每秒帧数 | 24~60 帧 | 音频帧率由编解码器决定（AAC 约 43 帧/秒，Opus 约 50 帧/秒） |

理解这些差异非常重要，因为它们直接影响你如何读取和处理解码后的数据。

### `AVSampleFormat` — 音频采样格式枚举

FFmpeg 通过 `AVSampleFormat` 枚举定义所有支持的音频采样格式。这个枚举是音频处理的核心，每个解码器都会通过 `AVCodecContext::sample_fmt` 告诉你它输出的是哪种格式：

```cpp
// FFmpeg 音频采样格式枚举（定义在 libavutil/samplefmt.h）
enum AVSampleFormat {
    AV_SAMPLE_FMT_NONE = -1,  // 无效格式

    // ===== 交错格式（Interleaved）=====
    // 所有声道的采样交替排列在 frame->data[0] 中
    // 内存布局：[L0][R0][L1][R1][L2][R2]...
    AV_SAMPLE_FMT_U8,     // unsigned 8-bit（无符号 8 位整数，范围 0~255）
    AV_SAMPLE_FMT_S16,    // signed 16-bit（有符号 16 位整数，范围 -32768~32767，CD 质量）
    AV_SAMPLE_FMT_S32,    // signed 32-bit（有符号 32 位整数，高精度）
    AV_SAMPLE_FMT_FLT,    // 32-bit float（IEEE 754 单精度浮点，范围 -1.0~1.0）
    AV_SAMPLE_FMT_DBL,    // 64-bit double（IEEE 754 双精度浮点，极高精度）

    // ===== 平面格式（Planar）=====
    // 每个声道的采样分别存储在 frame->data[i] 中
    // data[0]: [L0][L1][L2]...（左声道）
    // data[1]: [R0][R1][R2]...（右声道）
    AV_SAMPLE_FMT_U8P,    // unsigned 8-bit planar
    AV_SAMPLE_FMT_S16P,   // signed 16-bit planar
    AV_SAMPLE_FMT_S32P,   // signed 32-bit planar
    AV_SAMPLE_FMT_FLTP,   // 32-bit float planar（★最常见：AAC、Vorbis、Opus 默认输出此格式）
    AV_SAMPLE_FMT_DBLP,   // 64-bit double planar
    AV_SAMPLE_FMT_S64,    // signed 64-bit（FFmpeg 3.x+ 新增）
    AV_SAMPLE_FMT_S64P,   // signed 64-bit planar
    AV_SAMPLE_FMT_NB      // 格式总数（内部使用）
};
```

**关键事实：** 大多数现代音频编解码器（AAC、Vorbis、Opus、MP3 的某些解码器）默认输出**平面格式**（通常是 `AV_SAMPLE_FMT_FLTP`），但几乎所有音频播放 API（OpenAL、SDL_audio、Windows WASAPI、ALSA）都期望接收**交错格式**（通常是 `AV_SAMPLE_FMT_S16` 或 `AV_SAMPLE_FMT_FLT`）。这个格式不匹配是音频编程中最常见的坑之一。

### 音频格式查询辅助函数

FFmpeg 提供了一组实用函数来查询采样格式信息，在调试和开发中非常有用：

```cpp
#include <cstdio>
extern "C" {
#include <libavutil/samplefmt.h>
}

int main() {
    // 1. 获取格式名称字符串
    // av_get_sample_fmt_name: 将枚举值转换为人类可读的字符串
    // 参数: AVSampleFormat 枚举值
    // 返回: 格式名称字符串（如 "fltp"、"s16"），失败返回 NULL
    const char *name = av_get_sample_fmt_name(AV_SAMPLE_FMT_FLTP);
    printf("格式名: %s\n", name);  // 输出: "fltp"

    // 2. 从名称反查枚举值
    // av_get_sample_fmt: 将格式名称字符串转换回枚举值
    // 参数: 格式名称字符串（如 "s16"、"fltp"）
    // 返回: 对应的 AVSampleFormat 枚举值，失败返回 AV_SAMPLE_FMT_NONE
    enum AVSampleFormat fmt = av_get_sample_fmt("s16");
    printf("枚举值: %d\n", fmt);  // 输出: 1（AV_SAMPLE_FMT_S16）

    // 3. 查询每个采样占多少字节
    // av_get_bytes_per_sample: 返回指定格式中单个采样的字节数
    // 参数: AVSampleFormat 枚举值
    // 返回: 字节数（U8=1, S16=2, S32/FLT=4, DBL=8），失败返回 0
    int bytes = av_get_bytes_per_sample(AV_SAMPLE_FMT_S16);
    printf("S16 每采样字节: %d\n", bytes);  // 输出: 2

    bytes = av_get_bytes_per_sample(AV_SAMPLE_FMT_FLTP);
    printf("FLTP 每采样字节: %d\n", bytes);  // 输出: 4

    // 4. 判断是否为平面格式
    // av_sample_fmt_is_planar: 判断指定格式是否为平面格式
    // 参数: AVSampleFormat 枚举值
    // 返回: 1 表示平面格式，0 表示交错格式
    int isPlanar = av_sample_fmt_is_planar(AV_SAMPLE_FMT_FLTP);
    printf("FLTP 是平面格式: %s\n", isPlanar ? "是" : "否");  // 输出: "是"

    isPlanar = av_sample_fmt_is_planar(AV_SAMPLE_FMT_S16);
    printf("S16 是平面格式: %s\n", isPlanar ? "是" : "否");  // 输出: "否"

    // 5. 获取平面格式对应的交错格式（反之亦然）
    // av_get_packed_sample_fmt: 将平面格式转换为对应的交错格式
    // 参数: AVSampleFormat 枚举值（平面或交错均可）
    // 返回: 对应的交错格式；如果已经是交错格式则原样返回
    enum AVSampleFormat packed = av_get_packed_sample_fmt(AV_SAMPLE_FMT_FLTP);
    printf("FLTP 对应的交错格式: %s\n",
           av_get_sample_fmt_name(packed));  // 输出: "flt"

    // av_get_planar_sample_fmt: 将交错格式转换为对应的平面格式
    // 参数: AVSampleFormat 枚举值（平面或交错均可）
    // 返回: 对应的平面格式；如果已经是平面格式则原样返回
    enum AVSampleFormat planar = av_get_planar_sample_fmt(AV_SAMPLE_FMT_S16);
    printf("S16 对应的平面格式: %s\n",
           av_get_sample_fmt_name(planar));  // 输出: "s16p"

    return 0;
}
```

## 交错格式与平面格式深度解析

### 内存布局对比

交错格式和平面格式的本质区别在于多声道数据在内存中的排列方式。以立体声（左 L、右 R 两个声道）、4 个采样为例：

```
交错格式（Interleaved）：所有声道数据交替存储在一块连续内存中
┌──────────────────────────────────────────────────┐
│ frame->data[0]:                                  │
│ [L0][R0][L1][R1][L2][R2][L3][R3]                │
│  ^   ^   ^   ^   ^   ^   ^   ^                  │
│  │   │   │   │   │   │   │   │                  │
│  └─时刻0─┘   └─时刻1─┘   └─时刻2─┘   └─时刻3─┘  │
└──────────────────────────────────────────────────┘
frame->data[1] = NULL（未使用）

平面格式（Planar）：每个声道数据分别存储在独立的内存块中
┌──────────────────────────────┐
│ frame->data[0]（左声道）：    │
│ [L0][L1][L2][L3]            │
└──────────────────────────────┘
┌──────────────────────────────┐
│ frame->data[1]（右声道）：    │
│ [R0][R1][R2][R3]            │
└──────────────────────────────┘
```

### 为什么现代编解码器偏好平面格式

平面格式看起来更复杂，那为什么 AAC、Vorbis、Opus 等现代编解码器都选择输出平面格式？原因有三：

1. **SIMD 友好**：现代 CPU 的 SIMD（Single Instruction Multiple Data，单指令多数据——一条指令同时处理多个数据）指令集（SSE、AVX、NEON）在处理连续的同类型数据时效率最高。平面格式让每个声道的数据连续存放，可以直接用 SIMD 批量处理。

2. **解码器内部计算方便**：频域变换（如 MDCT，Modified Discrete Cosine Transform，改进离散余弦变换——音频压缩的核心数学工具）是逐声道独立进行的。平面格式避免了在频域运算完成后还要交错排列数据的额外步骤。

3. **灵活性**：平面格式天然支持任意声道数，而不需要硬编码声道交错顺序。

### 实际读取平面格式数据

理解了内存布局后，让我们看看如何正确读取平面格式和交错格式的音频数据：

```cpp
#include <cstdio>
#include <cstdint>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavutil/samplefmt.h>
}

// 演示如何根据采样格式正确读取 AVFrame 中的音频数据
void printAudioData(AVCodecContext *ctx, AVFrame *frame) {
    int isPlanar = av_sample_fmt_is_planar(ctx->sample_fmt);
    int bytesPerSample = av_get_bytes_per_sample(ctx->sample_fmt);
    int channels = ctx->channels;
    int nbSamples = frame->nb_samples;

    printf("采样格式: %s (%s)\n",
           av_get_sample_fmt_name(ctx->sample_fmt),
           isPlanar ? "平面" : "交错");
    printf("声道数: %d, 采样数: %d, 每采样 %d 字节\n",
           channels, nbSamples, bytesPerSample);

    if (isPlanar) {
        // ===== 平面格式 =====
        // 每个声道在 frame->data[ch] 中，长度 = nb_samples * bytesPerSample
        for (int ch = 0; ch < channels; ch++) {
            uint8_t *channelData = frame->data[ch];
            int channelSize = nbSamples * bytesPerSample;
            printf("  声道 %d: data[%d], 大小 %d 字节, 地址 %p\n",
                   ch, ch, channelSize, (void *)channelData);

            // 如果是 FLTP 格式，可以直接当 float 数组读取
            if (ctx->sample_fmt == AV_SAMPLE_FMT_FLTP) {
                float *samples = (float *)channelData;
                printf("    前 4 个采样: %.4f, %.4f, %.4f, %.4f\n",
                       samples[0], samples[1], samples[2], samples[3]);
            }
        }

        // 注意：frame->linesize[0] 是每个平面的大小（字节），
        // 不是总大小。所有平面的 linesize 相同
        printf("  linesize[0] = %d\n", frame->linesize[0]);
    } else {
        // ===== 交错格式 =====
        // 所有声道数据在 frame->data[0] 中
        uint8_t *interleavedData = frame->data[0];
        int totalSize = nbSamples * channels * bytesPerSample;
        printf("  交错数据: data[0], 总大小 %d 字节\n", totalSize);

        // 如果是 S16 格式，可以直接当 int16_t 数组读取
        if (ctx->sample_fmt == AV_SAMPLE_FMT_S16) {
            int16_t *samples = (int16_t *)interleavedData;
            // 对于立体声：samples[0]=L0, samples[1]=R0, samples[2]=L1, ...
            for (int i = 0; i < 4 && i < nbSamples; i++) {
                printf("    时刻 %d:", i);
                for (int ch = 0; ch < channels; ch++) {
                    printf(" ch%d=%d", ch, samples[i * channels + ch]);
                }
                printf("\n");
            }
        }

        // 注意：frame->linesize[0] 是全部数据的总大小（字节）
        printf("  linesize[0] = %d\n", frame->linesize[0]);
    }
}
```

## 完整音频解码示例

下面是一个完整的音频解码程序，它打开一个媒体文件，找到音频流，解码所有音频帧，并统计输出信息：

```cpp
// audio_decode_full.cpp — 完整音频解码示例
#include <cstdio>
#include <cstdint>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libavutil/opt.h>
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件>\n", argv[0]);
        return 1;
    }

    // 第一步：打开文件并获取流信息
    AVFormatContext *fmtCtx = nullptr;
    int ret = avformat_open_input(&fmtCtx, argv[1], nullptr, nullptr);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开文件: %s (%s)\n", argv[1], errbuf);
        return 1;
    }
    // avformat_find_stream_info: 读取部分数据包来分析流信息
    // 参数1: AVFormatContext 指针
    // 参数2: 每个流的额外选项（通常传 NULL）
    // 返回: >= 0 成功，< 0 失败
    avformat_find_stream_info(fmtCtx, nullptr);

    // 第二步：找到最佳音频流
    // av_find_best_stream: 自动选择"最佳"的指定类型流
    // 参数1: AVFormatContext 指针
    // 参数2: 媒体类型（AVMEDIA_TYPE_AUDIO 表示音频）
    // 参数3: 希望的流索引（-1 表示自动选择）
    // 参数4: 相关流索引（-1 表示无偏好）
    // 参数5: 输出找到的解码器指针（可以传 NULL）
    // 参数6: 标志位（通常传 0）
    // 返回: 流索引 >= 0 成功，< 0 失败
    int audioIdx = av_find_best_stream(
        fmtCtx, AVMEDIA_TYPE_AUDIO, -1, -1, nullptr, 0);
    if (audioIdx < 0) {
        fprintf(stderr, "未找到音频流\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // 第三步：查找解码器并打开
    AVCodecParameters *par = fmtCtx->streams[audioIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    if (!codec) {
        fprintf(stderr, "未找到解码器: codec_id=%d\n", par->codec_id);
        avformat_close_input(&fmtCtx);
        return 1;
    }

    AVCodecContext *ctx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(ctx, par);

    ret = avcodec_open2(ctx, codec, nullptr);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开解码器: %s\n", errbuf);
        avcodec_free_context(&ctx);
        avformat_close_input(&fmtCtx);
        return 1;
    }

    printf("=== 音频流信息 ===\n");
    printf("编解码器: %s (%s)\n", codec->name, codec->long_name);
    printf("采样率:   %d Hz\n", ctx->sample_rate);
    printf("声道数:   %d\n", ctx->channels);
    printf("采样格式: %s (%s)\n",
           av_get_sample_fmt_name(ctx->sample_fmt),
           av_sample_fmt_is_planar(ctx->sample_fmt) ? "平面" : "交错");
    printf("比特率:   %lld kbps\n", (long long)ctx->bit_rate / 1000);
    printf("==================\n\n");

    // 第四步：解码循环
    AVPacket *pkt = av_packet_alloc();
    AVFrame *frame = av_frame_alloc();
    int64_t totalSamples = 0;
    int frameCount = 0;
    int minSamples = INT32_MAX, maxSamples = 0;

    while (av_read_frame(fmtCtx, pkt) >= 0) {
        if (pkt->stream_index != audioIdx) {
            av_packet_unref(pkt);
            continue;
        }

        ret = avcodec_send_packet(ctx, pkt);
        if (ret < 0) {
            av_packet_unref(pkt);
            continue;  // 跳过无法解码的包
        }

        while (avcodec_receive_frame(ctx, frame) >= 0) {
            frameCount++;
            totalSamples += frame->nb_samples;
            if (frame->nb_samples < minSamples)
                minSamples = frame->nb_samples;
            if (frame->nb_samples > maxSamples)
                maxSamples = frame->nb_samples;

            // 每解码 100 帧打印一次进度
            if (frameCount % 100 == 0) {
                printf("  已解码 %d 帧, 当前帧 %d 采样, pts=%" PRId64 "\n",
                       frameCount, frame->nb_samples, frame->pts);
            }
            av_frame_unref(frame);
        }
        av_packet_unref(pkt);
    }

    // 第五步：刷新解码器（获取缓存中的剩余帧）
    // 发送 NULL packet 表示输入结束，解码器应输出所有缓存帧
    avcodec_send_packet(ctx, nullptr);
    while (avcodec_receive_frame(ctx, frame) >= 0) {
        frameCount++;
        totalSamples += frame->nb_samples;
        av_frame_unref(frame);
    }

    // 第六步：输出统计信息
    double duration = (double)totalSamples / ctx->sample_rate;
    printf("\n=== 解码统计 ===\n");
    printf("总帧数:   %d\n", frameCount);
    printf("总采样:   %" PRId64 " (%.2f 秒)\n", totalSamples, duration);
    printf("每帧采样: %d ~ %d\n", minSamples, maxSamples);
    printf("================\n");

    // 第七步：清理资源（严格按分配的逆序）
    av_frame_free(&frame);
    av_packet_free(&pkt);
    avcodec_free_context(&ctx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

## 音频重采样与格式转换（libswresample）

### 为什么需要格式转换

音频格式转换是将一种 PCM 格式转换为另一种的过程。在实际开发中，你几乎总是需要进行格式转换，原因有三：

1. **采样格式不匹配**：解码器输出 FLTP（32 位浮点平面），但播放 API 要求 S16（16 位整数交错）
2. **采样率不匹配**：源文件是 44100 Hz，但音频设备工作在 48000 Hz
3. **声道布局不匹配**：源文件是 5.1 环绕声（6 声道），但输出设备只有立体声（2 声道）

libswresample 库（头文件 `<libswresample/swresample.h>`）可以**同时**处理以上三种转换。它的核心类型是 `SwrContext`，代表一个重采样上下文，记录了输入格式、输出格式和转换算法等配置。

### SwrContext 初始化 — 两种方式

FFmpeg 提供两种方式创建和配置 `SwrContext`：

**方式一：`swr_alloc_set_opts()` 一步到位**

```cpp
#include <cstdio>
extern "C" {
#include <libswresample/swresample.h>
#include <libavutil/channel_layout.h>
}

void example_alloc_set_opts() {
    // swr_alloc_set_opts: 分配 SwrContext 并设置所有参数
    // 参数1: 已有的 SwrContext（传 NULL 表示创建新的）
    // 参数2: 输出声道布局（AV_CH_LAYOUT_STEREO = 左+右）
    // 参数3: 输出采样格式（S16 = 16位有符号整数交错）
    // 参数4: 输出采样率（48000 Hz）
    // 参数5: 输入声道布局
    // 参数6: 输入采样格式（FLTP = 32位浮点平面）
    // 参数7: 输入采样率（44100 Hz）
    // 参数8: 日志偏移量（通常传 0）
    // 参数9: 日志上下文（通常传 NULL）
    // 返回: SwrContext 指针，失败返回 NULL
    SwrContext *swrCtx = swr_alloc_set_opts(
        nullptr,
        AV_CH_LAYOUT_STEREO,    // 输出：立体声
        AV_SAMPLE_FMT_S16,      // 输出：16位整数交错格式
        48000,                   // 输出：48kHz
        AV_CH_LAYOUT_STEREO,    // 输入：立体声
        AV_SAMPLE_FMT_FLTP,     // 输入：32位浮点平面格式
        44100,                   // 输入：44.1kHz
        0, nullptr
    );
    if (!swrCtx) {
        fprintf(stderr, "swr_alloc_set_opts 失败\n");
        return;
    }

    // swr_init: 初始化 SwrContext，使配置生效
    // 参数: SwrContext 指针
    // 返回: 0 成功，< 0 失败（通常是参数无效）
    int ret = swr_init(swrCtx);
    if (ret < 0) {
        fprintf(stderr, "swr_init 失败\n");
        swr_free(&swrCtx);
        return;
    }

    printf("SwrContext 初始化成功\n");

    // 使用完毕后释放
    // swr_free: 释放 SwrContext 及其所有内部资源
    // 参数: SwrContext 指针的指针（释放后自动置 NULL）
    swr_free(&swrCtx);
}
```

**方式二：`swr_alloc()` + `av_opt_set_*()` 逐项配置**

```cpp
void example_alloc_then_set() {
    // swr_alloc: 分配一个空的 SwrContext
    // 返回: SwrContext 指针，失败返回 NULL
    SwrContext *swrCtx = swr_alloc();
    if (!swrCtx) {
        fprintf(stderr, "swr_alloc 失败\n");
        return;
    }

    // 通过 av_opt_set_* 系列函数逐项设置参数
    // 这种方式更灵活，可以动态决定参数
    // av_opt_set_int: 设置整数类型选项
    // 参数1: 目标对象指针
    // 参数2: 选项名称字符串
    // 参数3: 选项值
    // 参数4: 搜索标志（0 表示在当前对象中搜索）
    av_opt_set_int(swrCtx, "in_channel_layout",  AV_CH_LAYOUT_STEREO, 0);
    av_opt_set_int(swrCtx, "in_sample_rate",      44100, 0);
    // av_opt_set_sample_fmt: 设置采样格式类型选项
    // 参数同 av_opt_set_int，但专门用于 AVSampleFormat 类型
    av_opt_set_sample_fmt(swrCtx, "in_sample_fmt", AV_SAMPLE_FMT_FLTP, 0);

    av_opt_set_int(swrCtx, "out_channel_layout", AV_CH_LAYOUT_STEREO, 0);
    av_opt_set_int(swrCtx, "out_sample_rate",     48000, 0);
    av_opt_set_sample_fmt(swrCtx, "out_sample_fmt", AV_SAMPLE_FMT_S16, 0);

    int ret = swr_init(swrCtx);
    if (ret < 0) {
        fprintf(stderr, "swr_init 失败\n");
        swr_free(&swrCtx);
        return;
    }

    printf("逐项配置初始化成功\n");
    swr_free(&swrCtx);
```

### 执行重采样 — swr_convert

配置好 `SwrContext` 后，就可以用 `swr_convert()` 执行实际的格式转换了：

```cpp
#include <cstdio>
extern "C" {
#include <libswresample/swresample.h>
#include <libavutil/opt.h>
#include <libavutil/channel_layout.h>
#include <libavcodec/avcodec.h>
}

// 将解码后的音频帧从 FLTP 44100Hz 转换为 S16 48000Hz
// 返回转换后的采样数，失败返回 -1
int convertAudioFrame(SwrContext *swrCtx, AVFrame *inputFrame,
                       uint8_t **outBuf, int maxOutSamples) {
    // 第一步：计算输出采样数
    // 当输入和输出采样率不同时，输出的采样数与输入不同
    // av_rescale_rnd: 带四舍五入的整数缩放运算
    // 公式: (a * b + rnd) / c，这里用于按采样率比例换算采样数
    // swr_get_delay: 获取重采样器内部缓冲的延迟采样数
    // 参数1: SwrContext 指针
    // 参数2: 采样率基准（用于确定延迟的时间单位）
    // 返回: 延迟的采样数（以参数2指定的采样率为单位）
    int64_t delay = swr_get_delay(swrCtx, 44100);
    int outSamples = (int)av_rescale_rnd(
        delay + inputFrame->nb_samples,  // 输入采样数 + 延迟
        48000,                            // 输出采样率
        44100,                            // 输入采样率
        AV_ROUND_UP                       // 向上取整，确保缓冲区够大
    );

    if (outSamples > maxOutSamples) {
        fprintf(stderr, "输出缓冲区不够大: 需要 %d, 只有 %d\n",
                outSamples, maxOutSamples);
        return -1;
    }

    // 第二步：执行转换
    // swr_convert: 执行音频重采样和格式转换
    // 参数1: SwrContext 指针
    // 参数2: 输出缓冲区指针数组（交错格式只需 [0]，平面格式需 [0..channels-1]）
    // 参数3: 输出缓冲区最大采样数
    // 参数4: 输入数据指针数组（来自 AVFrame::data）
    // 参数5: 输入采样数
    // 返回: 实际输出的采样数（>= 0），或负数表示错误
    int converted = swr_convert(
        swrCtx,
        outBuf,                                    // 输出
        outSamples,                                // 输出最大采样数
        (const uint8_t **)inputFrame->data,        // 输入（AVFrame 的 data 数组）
        inputFrame->nb_samples                     // 输入采样数
    );

    if (converted < 0) {
        fprintf(stderr, "swr_convert 失败\n");
        return -1;
    }

    return converted;
}
```

### 完整的解码 + 重采样示例

将解码和重采样组合起来，以下是一个完整的程序，它将任意音频文件解码为 S16 交错 48kHz 立体声 PCM 数据，并写入 raw 文件：

```cpp
// audio_resample.cpp — 解码音频并重采样为标准格式
#include <cstdio>
#include <cstdint>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavformat/avformat.h>
#include <libswresample/swresample.h>
#include <libavutil/opt.h>
#include <libavutil/channel_layout.h>
}

int main(int argc, char *argv[]) {
    if (argc < 3) {
        fprintf(stderr, "用法: %s <输入文件> <输出.raw>\n", argv[0]);
        fprintf(stderr, "输出格式: S16LE, 48000Hz, 立体声\n");
        return 1;
    }

    // 打开输入文件
    AVFormatContext *fmtCtx = nullptr;
    if (avformat_open_input(&fmtCtx, argv[1], nullptr, nullptr) < 0) {
        fprintf(stderr, "无法打开: %s\n", argv[1]);
        return 1;
    }
    avformat_find_stream_info(fmtCtx, nullptr);

    // 找到音频流并打开解码器
    int audioIdx = av_find_best_stream(
        fmtCtx, AVMEDIA_TYPE_AUDIO, -1, -1, nullptr, 0);
    if (audioIdx < 0) {
        fprintf(stderr, "未找到音频流\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    AVCodecParameters *par = fmtCtx->streams[audioIdx]->codecpar;
    AVCodec *codec = avcodec_find_decoder(par->codec_id);
    AVCodecContext *decCtx = avcodec_alloc_context3(codec);
    avcodec_parameters_to_context(decCtx, par);
    avcodec_open2(decCtx, codec, nullptr);

    printf("输入: %s, %d Hz, %d 声道, 格式 %s\n",
           codec->name, decCtx->sample_rate, decCtx->channels,
           av_get_sample_fmt_name(decCtx->sample_fmt));

    // 创建重采样上下文
    // 输入参数从解码器上下文获取，输出参数固定为我们需要的格式
    SwrContext *swrCtx = swr_alloc_set_opts(
        nullptr,
        AV_CH_LAYOUT_STEREO, AV_SAMPLE_FMT_S16, 48000,   // 输出
        decCtx->channel_layout ? decCtx->channel_layout
            : av_get_default_channel_layout(decCtx->channels),
        decCtx->sample_fmt, decCtx->sample_rate,           // 输入
        0, nullptr
    );
    if (!swrCtx || swr_init(swrCtx) < 0) {
        fprintf(stderr, "无法初始化重采样器\n");
        return 1;
    }

    // 打开输出文件
    FILE *outFile = fopen(argv[2], "wb");
    if (!outFile) {
        fprintf(stderr, "无法创建输出文件: %s\n", argv[2]);
        return 1;
    }

    // 分配输出缓冲区（足够大以容纳重采样后的数据）
    // 公式：最大输出采样数 = 输入采样数 * (输出采样率/输入采样率) + 256
    int maxOutSamples = 8192;
    uint8_t *outBuf = nullptr;
    // av_samples_alloc: 为音频采样分配缓冲区
    // 参数1: 输出缓冲区指针数组的首元素指针
    // 参数2: 输出行大小（可以传 NULL）
    // 参数3: 声道数
    // 参数4: 采样数
    // 参数5: 采样格式
    // 参数6: 对齐值（0 表示默认对齐）
    // 返回: 分配的字节数，< 0 表示失败
    av_samples_alloc(&outBuf, nullptr, 2, maxOutSamples,
                     AV_SAMPLE_FMT_S16, 0);

    // 解码 + 重采样循环
    AVPacket *pkt = av_packet_alloc();
    AVFrame *frame = av_frame_alloc();
    int64_t totalOutSamples = 0;

    while (av_read_frame(fmtCtx, pkt) >= 0) {
        if (pkt->stream_index != audioIdx) {
            av_packet_unref(pkt);
            continue;
        }

        avcodec_send_packet(decCtx, pkt);
        while (avcodec_receive_frame(decCtx, frame) >= 0) {
            // 计算本次输出采样数
            int outCount = (int)av_rescale_rnd(
                swr_get_delay(swrCtx, decCtx->sample_rate) + frame->nb_samples,
                48000, decCtx->sample_rate, AV_ROUND_UP);

            if (outCount > maxOutSamples) {
                // 缓冲区不够，重新分配
                av_freep(&outBuf);
                maxOutSamples = outCount + 256;
                av_samples_alloc(&outBuf, nullptr, 2, maxOutSamples,
                                 AV_SAMPLE_FMT_S16, 0);
            }

            // 执行重采样
            int converted = swr_convert(
                swrCtx,
                &outBuf, outCount,
                (const uint8_t **)frame->data, frame->nb_samples);

            if (converted > 0) {
                // S16 立体声：每个采样 2 字节 × 2 声道 = 4 字节
                int dataSize = converted * 2 * sizeof(int16_t);
                fwrite(outBuf, 1, dataSize, outFile);
                totalOutSamples += converted;
            }
            av_frame_unref(frame);
        }
        av_packet_unref(pkt);
    }

    // 刷新解码器
    avcodec_send_packet(decCtx, nullptr);
    while (avcodec_receive_frame(decCtx, frame) >= 0) {
        int outCount = (int)av_rescale_rnd(
            swr_get_delay(swrCtx, decCtx->sample_rate) + frame->nb_samples,
            48000, decCtx->sample_rate, AV_ROUND_UP);
        int converted = swr_convert(
            swrCtx, &outBuf, outCount,
            (const uint8_t **)frame->data, frame->nb_samples);
        if (converted > 0) {
            fwrite(outBuf, 1, converted * 4, outFile);
            totalOutSamples += converted;
        }
        av_frame_unref(frame);
    }

    // 刷新重采样器内部缓存
    // 传入 NULL/0 作为输入，取出重采样器内部残留的数据
    int flushed = swr_convert(swrCtx, &outBuf, maxOutSamples, nullptr, 0);
    if (flushed > 0) {
        fwrite(outBuf, 1, flushed * 4, outFile);
        totalOutSamples += flushed;
    }

    double duration = (double)totalOutSamples / 48000.0;
    printf("输出: S16LE 48000Hz 立体声, %" PRId64 " 采样 (%.2f 秒)\n",
           totalOutSamples, duration);
    printf("文件: %s (%lld 字节)\n", argv[2],
           (long long)totalOutSamples * 4);

    // 清理
    fclose(outFile);
    av_freep(&outBuf);
    av_frame_free(&frame);
    av_packet_free(&pkt);
    swr_free(&swrCtx);
    avcodec_free_context(&decCtx);
    avformat_close_input(&fmtCtx);
    return 0;
}
```

## 视频像素格式转换（libswscale）

### 为什么视频也需要格式转换

视频解码器输出的像素格式通常是 YUV420P（Y/U/V 三个平面分离，U/V 水平和垂直方向各降采样一半），但渲染 API 通常需要 RGB 或 RGBA 格式。libswscale 库（头文件 `<libswscale/swscale.h>`）负责这种转换，同时也支持分辨率缩放。

| 源格式 | 目标格式 | 常见场景 |
|--------|----------|----------|
| YUV420P | RGB24 | 软件渲染、保存为图片（PPM/BMP） |
| YUV420P | RGBA | OpenGL 纹理上传（`glTexImage2D`） |
| YUV420P | BGR24 | OpenCV 图像处理 |
| NV12 | RGB24 | 硬件解码器输出（Android MediaCodec、iOS VideoToolbox） |

### SwsContext 创建与使用

```cpp
#include <cstdio>
extern "C" {
#include <libavcodec/avcodec.h>
#include <libavutil/imgutils.h>
#include <libswscale/swscale.h>
}

// 将解码后的 YUV 帧转换为 RGB24
// 返回: 0 成功, -1 失败
int convertYUVtoRGB(AVCodecContext *codecCtx, AVFrame *yuvFrame,
                     uint8_t **outData, int *outLinesize) {
    int width = codecCtx->width;
    int height = codecCtx->height;

    // 第一步：创建转换上下文
    // sws_getContext: 创建像素格式转换/缩放上下文
    // 参数1-3: 源宽度、源高度、源像素格式
    // 参数4-6: 目标宽度、目标高度、目标像素格式
    // 参数7: 缩放算法标志
    //   SWS_BILINEAR  — 双线性插值（速度与质量均衡，最常用）
    //   SWS_BICUBIC   — 双三次插值（更高质量，稍慢）
    //   SWS_FAST_BILINEAR — 快速双线性（最快，质量略低）
    //   SWS_POINT      — 最近邻插值（最快，有锯齿）
    // 参数8-10: 源/目标滤波器和参数（通常传 NULL）
    // 返回: SwsContext 指针，失败返回 NULL
    struct SwsContext *swsCtx = sws_getContext(
        width, height, codecCtx->pix_fmt,   // 源
        width, height, AV_PIX_FMT_RGB24,    // 目标（此处无缩放）
        SWS_BILINEAR,                        // 缩放算法
        nullptr, nullptr, nullptr
    );
    if (!swsCtx) {
        fprintf(stderr, "sws_getContext 失败\n");
        return -1;
    }

    // 第二步：分配目标图像缓冲区
    // av_image_alloc: 为图像数据分配缓冲区
    // 参数1: 输出数据指针数组（最多 4 个平面）
    // 参数2: 输出行宽数组（最多 4 个平面）
    // 参数3-4: 图像宽度和高度
    // 参数5: 像素格式
    // 参数6: 对齐值（1 = 不对齐，16/32 = SIMD 对齐）
    // 返回: 分配的总字节数，< 0 失败
    int bufSize = av_image_alloc(
        outData, outLinesize,
        width, height, AV_PIX_FMT_RGB24,
        1  // 对齐值
    );
    if (bufSize < 0) {
        fprintf(stderr, "av_image_alloc 失败\n");
        sws_freeContext(swsCtx);
        return -1;
    }

    // 第三步：执行像素格式转换
    // sws_scale: 执行图像缩放和/或像素格式转换
    // 参数1: SwsContext 指针
    // 参数2: 源数据指针数组（AVFrame::data）
    // 参数3: 源行宽数组（AVFrame::linesize）
    // 参数4: 源图像中要处理的起始行号（通常为 0）
    // 参数5: 源图像中要处理的行数（通常为 height）
    // 参数6: 目标数据指针数组
    // 参数7: 目标行宽数组
    // 返回: 实际输出的行数
    int outputHeight = sws_scale(
        swsCtx,
        yuvFrame->data,        // 源数据（YUV 三个平面）
        yuvFrame->linesize,    // 源行宽（注意：可能有 padding）
        0, height,             // 从第 0 行到第 height 行
        outData,               // 目标数据
        outLinesize            // 目标行宽
    );

    printf("YUV→RGB 转换完成: %dx%d, linesize=%d, 总 %d 字节\n",
           width, outputHeight, outLinesize[0], bufSize);

    // 释放 SwsContext（生产代码中应复用，不要每帧重建！）
    // sws_freeContext: 释放 SwsContext 及其所有内部资源
    // 参数: SwsContext 指针
    sws_freeContext(swsCtx);
    return 0;
}
```

**性能要点**：`SwsContext` 的创建成本较高（内部会预计算查找表和滤波器系数），因此在循环中应当**复用**而不是每帧重建。只有当源或目标的宽高/格式发生变化时，才需要重新创建。在 KrKr2 中，渲染管线通过 `CRenderManager` 统一管理 SwsContext 的生命周期。

### linesize 与 padding

一个容易忽略的细节是 `AVFrame::linesize`（行宽/步长）。它表示图像每一行在内存中占用的字节数，这个值**通常大于** `width × bytesPerPixel`，因为 FFmpeg 会在每行末尾添加 padding（填充字节）以满足 SIMD 对齐要求：

```
假设图像宽度 = 100 像素，RGB24 格式（每像素 3 字节）
理论行宽 = 100 × 3 = 300 字节
实际 linesize = 320 字节（对齐到 32 字节边界）

内存布局：
行0: [300 字节像素数据][20 字节 padding]  ← linesize = 320
行1: [300 字节像素数据][20 字节 padding]  ← linesize = 320
行2: [300 字节像素数据][20 字节 padding]  ← linesize = 320
```

如果你直接用 `width * bytesPerPixel * height` 来计算总数据量或逐行拷贝数据，就会出错。正确做法是用 `linesize` 来计算行间距：

```cpp
// 正确的逐行拷贝（考虑 linesize padding）
void copyFrameToBuffer(uint8_t *dst, AVFrame *frame,
                        int width, int height) {
    int rowBytes = width * 3;  // RGB24: 每像素 3 字节
    for (int y = 0; y < height; y++) {
        // 源数据起始位置用 linesize 计算，不是 width*3
        uint8_t *srcRow = frame->data[0] + y * frame->linesize[0];
        uint8_t *dstRow = dst + y * rowBytes;
        memcpy(dstRow, srcRow, rowBytes);  // 只拷贝有效像素
    }
}

// 错误的做法（会导致画面错位/花屏）
void copyFrameWrong(uint8_t *dst, AVFrame *frame,
                     int width, int height) {
    // ❌ 如果 linesize != width*3，数据就对不上
    memcpy(dst, frame->data[0], width * 3 * height);
}
```

7. 刷新解码器和重采样器

9. 清理所有资源

## 对照项目源码

### FFWaveDecoder — KrKr2 的手动格式转换

KrKr2 的 `FFWaveDecoder`（位于 `krkr2/cpp/core/sound/FFWaveDecoder.cpp`，452 行）没有使用 libswresample 进行格式转换，而是**手动实现**了平面到交错的转换。这是因为 KiriKiri 引擎的音频管线要求特定格式的 PCM 数据，且不想引入额外的库依赖。

**`CopySmaples` 模板函数**（第 112-125 行）——通用的平面转交错：

```cpp
// 来源：krkr2/cpp/core/sound/FFWaveDecoder.cpp 第 112-125 行
// 模板函数：将平面格式数据交错排列到目标缓冲区
// T 是采样类型（int16_t、float 等），由运行时的采样格式决定
template <typename T>
static unsigned char *_CopySmaples(unsigned char *dst, AVFrame *frame,
                                    int channels, int samples,
                                    int buf_index) {
    int buf_pos = buf_index * sizeof(T);  // 在帧内的起始位置
    T *pDst = (T *)dst;
    for (int i = 0; i < samples; ++i, buf_pos += sizeof(T)) {
        for (int ch = 0; ch < channels; ++ch) {
            // 从每个声道平面取一个采样，依次写入目标
            *pDst++ = *(T *)(frame->data[ch] + buf_pos);
        }
    }
    return (unsigned char *)pDst;
}
```

**`_CopySmaples2` 模板函数**（第 126-136 行）——立体声优化版：

```cpp
// 来源：krkr2/cpp/core/sound/FFWaveDecoder.cpp 第 126-136 行
// 立体声特化版本：避免内层 channels 循环，直接展开左右声道
template <typename T>
static unsigned char *_CopySmaples2(unsigned char *dst, AVFrame *frame,
                                     int samples, int buf_index) {
    int buf_pos = buf_index * sizeof(T);
    T *pDst = (T *)dst;
    for (int i = 0; i < samples; ++i, buf_pos += sizeof(T)) {
        *pDst++ = *(T *)(frame->data[0] + buf_pos);  // 左声道
        *pDst++ = *(T *)(frame->data[1] + buf_pos);  // 右声道
    }
    return (unsigned char *)pDst;
}
```

**为什么需要立体声优化？** 音频处理是性能敏感的——在 48000 Hz 立体声的场景下，每秒要处理 96000 个采样（48000 × 2 声道）。消除内层循环的分支判断能显著减少 CPU 分支预测失败的开销。

**格式选择调度**（第 138-158 行）——根据运行时格式选择对应的模板实例：

```cpp
// 来源：krkr2/cpp/core/sound/FFWaveDecoder.cpp 第 138-158 行（简化）
static unsigned char *CopySmaples(unsigned char *dst, AVFrame *frame,
                                   int channels, int samples,
                                   int buf_index, int format) {
    if (channels == 2) {
        // 立体声走优化路径
        switch (format) {
            case AV_SAMPLE_FMT_S16P:
                return _CopySmaples2<int16_t>(dst, frame, samples, buf_index);
            case AV_SAMPLE_FMT_S32P:
                return _CopySmaples2<int32_t>(dst, frame, samples, buf_index);
            case AV_SAMPLE_FMT_FLTP:
                return _CopySmaples2<float>(dst, frame, samples, buf_index);
            // ... 其他格式
        }
    } else {
        // 多声道走通用路径
        switch (format) {
            case AV_SAMPLE_FMT_S16P:
                return _CopySmaples<int16_t>(dst, frame, channels,
                                              samples, buf_index);
            // ... 其他格式
        }
    }
    return dst;
}
```

**与 libswresample 的对比：**

| 特性 | FFWaveDecoder 手动转换 | libswresample |
|------|----------------------|---------------|
| 格式转换 | 仅平面→交错 | 任意格式互转 |
| 采样率转换 | ❌ 不支持 | ✅ 支持 |
| 声道混合 | ❌ 不支持 | ✅ 支持（如 5.1→立体声） |
| 性能 | 略快（无库调用开销） | 略慢（但功能更全） |
| 代码复杂度 | 简单直接 | 需要初始化和配置 |
| 适用场景 | 已知格式、不需要重采样 | 通用场景 |

### CDVDAudioCodecFFmpeg — movie 模块的音频解码

KrKr2 的 movie 模块（`krkr2/cpp/core/movie/ffmpeg/AudioCodecFFmpeg.cpp`，360 行）使用旧 API `avcodec_decode_audio4` 进行音频解码。它有一个值得注意的设计——`GetDataFormat()` 方法将 FFmpeg 的 `AVSampleFormat` 映射到内部的 `AEDataFormat` 枚举：

```cpp
// 来源：krkr2/cpp/core/movie/ffmpeg/AudioCodecFFmpeg.cpp 第 234-263 行
// 将 FFmpeg 采样格式映射到 KrKr2 内部音频格式
enum AEDataFormat CDVDAudioCodecFFmpeg::GetDataFormat() {
    switch (m_pCodecContext->sample_fmt) {
        case AV_SAMPLE_FMT_U8:   return AE_FMT_U8;    // 无符号8位→内部U8
        case AV_SAMPLE_FMT_U8P:  return AE_FMT_U8P;   // 无符号8位平面
        case AV_SAMPLE_FMT_S16:  return AE_FMT_S16NE;  // 有符号16位→本机字节序
        case AV_SAMPLE_FMT_S16P: return AE_FMT_S16NEP; // 有符号16位平面
        case AV_SAMPLE_FMT_S32:  return AE_FMT_S32NE;  // 有符号32位
        case AV_SAMPLE_FMT_S32P: return AE_FMT_S32NEP; // 有符号32位平面
        case AV_SAMPLE_FMT_FLT:  return AE_FMT_FLOAT;  // 32位浮点
        case AV_SAMPLE_FMT_FLTP: return AE_FMT_FLOATP; // 32位浮点平面
        case AV_SAMPLE_FMT_DBL:  return AE_FMT_DOUBLE; // 64位双精度
        case AV_SAMPLE_FMT_DBLP: return AE_FMT_DOUBLEP;// 64位双精度平面
        default: return AE_FMT_INVALID;                 // 不支持的格式
    }
}
```

这个映射表明 movie 模块**不做格式转换**——它把原始格式直接传递给下游的音频渲染管线（`CAudioSinkAE`），由渲染管线自己决定是否需要转换。这与 sound 模块的 `FFWaveDecoder`（手动转换为交错格式）形成了有趣的对比。

## 常见错误及解决方案

### 错误 1：解码后音频有杂音或噪声

**症状**：播放解码后的音频时听到刺耳的杂音，或者声音听起来像"机器人"。

**原因**：将平面格式的 `frame->data[0]` 直接当作交错格式送给播放 API。此时播放器只收到了左声道数据，而且采样格式也可能不匹配（例如 float 被当作 int16 解读）。

**解决方案**：

```cpp
// 检查采样格式是否为平面
if (av_sample_fmt_is_planar(ctx->sample_fmt)) {
    // 必须使用 swr_convert 转换为交错格式
    // 或者手动交错（如 KrKr2 的 CopySmaples）
    printf("警告：解码器输出平面格式 %s，需要转换\n",
           av_get_sample_fmt_name(ctx->sample_fmt));
}
```

### 错误 2：sws_scale 后画面偏绿或颜色错误

**症状**：转换后的 RGB 图像颜色明显偏绿，或者整体色调不对。

**原因**：常见两个原因——（1）没有正确处理 `linesize` 的 padding，导致数据偏移；（2）源像素格式判断错误（比如 NV12 被当成 YUV420P 处理）。

**解决方案**：

```cpp
// 确认源格式
printf("解码器输出格式: %s\n",
       av_get_pix_fmt_name(ctx->pix_fmt));

// 确保 SwsContext 使用正确的源格式
struct SwsContext *swsCtx = sws_getContext(
    width, height, ctx->pix_fmt,  // ← 用 ctx->pix_fmt，不要硬编码
    width, height, AV_PIX_FMT_RGB24,
    SWS_BILINEAR, nullptr, nullptr, nullptr);

// 写入文件时考虑 linesize
for (int y = 0; y < height; y++) {
    fwrite(rgbFrame->data[0] + y * rgbFrame->linesize[0],
           1, width * 3, fp);  // ← 只写有效像素，不写 padding
}
```

### 错误 3：swr_convert 输出采样数为 0 或不完整

**症状**：`swr_convert()` 返回的采样数比预期少，甚至返回 0。

**原因**：当输入和输出采样率不同时，重采样器内部有缓冲区。前几帧可能因为缓冲区还没填满而输出较少的采样。最后一帧后需要"刷新"重采样器。

**解决方案**：

```cpp
// 在解码循环结束后，刷新重采样器
// 传入 NULL/0 作为输入
int remaining = swr_convert(swrCtx, &outBuf, maxSamples,
                             nullptr, 0);  // ← 输入为空
while (remaining > 0) {
    // 处理残留数据
    fwrite(outBuf, 1, remaining * bytesPerFrame, outFile);
    remaining = swr_convert(swrCtx, &outBuf, maxSamples,
                             nullptr, 0);
}
```

## 本节小结

- **音频解码**在 API 层面与视频解码相同（`send_packet`/`receive_frame`），但数据组织方式不同：音频帧包含可变数量的采样，而非固定大小的像素矩阵
- **采样格式**分为交错（Interleaved，所有声道交替存储在 `data[0]`）和平面（Planar，每个声道独立存储在 `data[ch]`）两大类
- **大多数现代编解码器输出平面格式**（如 `AV_SAMPLE_FMT_FLTP`），但播放 API 通常需要交错格式——这是初学者最常踩的坑
- **libswresample**（`SwrContext` + `swr_convert`）可以同时完成采样格式转换、采样率重采样和声道布局混合
- **libswscale**（`SwsContext` + `sws_scale`）用于视频像素格式转换和分辨率缩放，核心注意事项是 `linesize` 可能包含 padding
- **KrKr2 的 `FFWaveDecoder`** 手动实现了平面→交错转换（`CopySmaples` 模板），并对立体声做了特化优化
- **KrKr2 的 `CDVDAudioCodecFFmpeg`** 不做格式转换，直接将 AV 格式映射为内部 AE 格式传递给下游

## 练习题与答案

### 题目 1：判断采样格式和计算数据大小

假设解码器输出了一帧音频，参数如下：
- 采样格式：`AV_SAMPLE_FMT_FLTP`（32位浮点平面）
- 声道数：2（立体声）
- `nb_samples`：1024

请回答：
1. `frame->data[0]` 指向什么数据？大小是多少字节？
2. `frame->data[1]` 指向什么数据？大小是多少字节？
3. 如果要转换为 `AV_SAMPLE_FMT_S16`（16位整数交错），输出缓冲区至少需要多少字节？
4. `av_sample_fmt_is_planar(AV_SAMPLE_FMT_FLTP)` 返回什么？

<details>
<summary>查看答案</summary>

1. `frame->data[0]` 指向**左声道**的浮点采样数据。大小 = `1024 × sizeof(float)` = `1024 × 4` = **4096 字节**。

2. `frame->data[1]` 指向**右声道**的浮点采样数据。大小同样为 **4096 字节**。

3. S16 交错格式：每个采样 2 字节，两声道交错。需要 `1024 × 2（声道）× 2（字节/采样）` = **4096 字节**。注意虽然数值巧合相同，但含义不同——FLTP 是每声道 4096 字节（共两块），S16 是一整块 4096 字节。

4. 返回 **1**（true），因为 FLTP 是平面格式。

</details>

### 题目 2：修复格式不匹配导致的杂音

以下代码将 AAC 音频解码后直接送给 SDL 播放，但听到的是杂音。分析原因并修复：

```cpp
// 有 Bug 的代码
avcodec_send_packet(ctx, pkt);
avcodec_receive_frame(ctx, frame);

// 假设 SDL 已配置为 S16, 44100Hz, 立体声
SDL_QueueAudio(deviceId, frame->data[0],
               frame->nb_samples * 2 * sizeof(int16_t));
```

<details>
<summary>查看答案</summary>

**问题分析**：AAC 解码器默认输出 `AV_SAMPLE_FMT_FLTP`（32位浮点平面格式）。直接把 `frame->data[0]` 送给 SDL 有两个问题：
1. `data[0]` 只包含左声道数据，右声道在 `data[1]` 中
2. 数据是 float 类型，但 SDL 期望 int16 类型

**修复代码**：

```cpp
// 初始化时创建 SwrContext（只做一次）
SwrContext *swrCtx = swr_alloc_set_opts(
    nullptr,
    AV_CH_LAYOUT_STEREO, AV_SAMPLE_FMT_S16, 44100,  // 输出
    ctx->channel_layout, ctx->sample_fmt, ctx->sample_rate,  // 输入
    0, nullptr);
swr_init(swrCtx);

// 分配输出缓冲区（只做一次，循环复用）
int maxSamples = 8192;
uint8_t *outBuf = nullptr;
av_samples_alloc(&outBuf, nullptr, 2, maxSamples, AV_SAMPLE_FMT_S16, 0);

// 解码循环中
avcodec_send_packet(ctx, pkt);
while (avcodec_receive_frame(ctx, frame) >= 0) {
    int outSamples = swr_convert(swrCtx,
        &outBuf, maxSamples,
        (const uint8_t **)frame->data, frame->nb_samples);
    if (outSamples > 0) {
        SDL_QueueAudio(deviceId, outBuf,
                       outSamples * 2 * sizeof(int16_t));
    }
    av_frame_unref(frame);
}
```

</details>

### 题目 3：分析 KrKr2 的手动格式转换设计

阅读 `FFWaveDecoder.cpp` 中的 `CopySmaples` 函数族，回答：
1. 为什么要为立体声单独写一个 `_CopySmaples2` 函数？
2. 如果输入是 `AV_SAMPLE_FMT_S16`（交错格式而非平面格式），`CopySmaples` 需要做什么？
3. `FFWaveDecoder` 为什么不使用 libswresample？

<details>
<summary>查看答案</summary>

1. **性能优化**。通用版本 `_CopySmaples` 有一个内层循环 `for (int ch = 0; ch < channels; ++ch)`，对于立体声来说这个循环只执行 2 次，但循环本身有分支判断和迭代开销。`_CopySmaples2` 直接展开为两条赋值语句，消除了循环开销。在 48000Hz 立体声下，每秒调用约 43 次（每帧 1024 采样），每次循环 1024 次，累计减少了约 86000 次分支判断。

2. 如果输入已经是交错格式（如 `AV_SAMPLE_FMT_S16`），则 `frame->data[0]` 已经包含了正确排列的数据，**不需要做任何转换**——直接 `memcpy` 即可。在 `CopySmaples` 的 switch 语句中，交错格式的分支应该执行简单的内存拷贝而不是逐采样交错。

3. 两个原因：（a）**减少依赖**——sound 模块只需要 libavcodec 和 libavformat，不使用 libswresample 就可以少链接一个库；（b）**功能够用**——FFWaveDecoder 只需要平面→交错的转换，不需要采样率转换或声道混合，手动实现比引入整个库更简单直接。这是一种典型的"够用就好"的工程决策。

</details>
