# 4.5 硬件加速解码
## 4.5 硬件加速解码

### 4.5.1 什么是硬件加速解码

现代 GPU 和专用芯片可以承担视频解码任务，比 CPU 软件解码效率高数倍且功耗更低：

| 平台 | 硬件加速 API | 对应 FFmpeg 像素格式 |
|------|-------------|---------------------|
| Windows | DXVA2 / D3D11VA | `AV_PIX_FMT_DXVA2_VLD` / `AV_PIX_FMT_D3D11` |
| Linux | VAAPI / VDPAU | `AV_PIX_FMT_VAAPI` / `AV_PIX_FMT_VDPAU` |
| macOS/iOS | VideoToolbox | `AV_PIX_FMT_VIDEOTOOLBOX` |
| Android | MediaCodec | `AV_PIX_FMT_MEDIACODEC` |
| 树莓派 | MMAL | 通过 `AV_PIX_FMT_YUV420P` 拦截 |

### 4.5.2 KrKr2 的硬件加速架构

KrKr2 通过 `CDVDVideoCodecFFmpeg::GetFormat` 回调实现硬件加速协商：

```
avcodec 需要选择像素格式时回调 GetFormat
    │
    ├─ 检查当前 decoderState
    │   └─ STATE_HW_SINGLE → 尝试硬件解码
    │   └─ 其他状态 → 直接返回默认软解格式
    │
    ├─ 遍历 FFmpeg 提供的候选格式列表
    │   ├─ VDPAU 格式 → 创建 VDPAU::CDecoder，尝试 Open()
    │   ├─ DXVA2 格式 → 创建 DXVA::CDecoder，尝试 Open()
    │   ├─ VAAPI 格式 → 创建 VAAPI::CDecoder，尝试 Open()
    │   ├─ VTB 格式   → 创建 VTB::CDecoder，尝试 Open()
    │   └─ MMAL 格式  → 创建 MMAL::CDecoder，尝试 Open()
    │
    ├─ 硬件解码器 Open 成功
    │   └─ 设置 m_pHardware，返回硬件像素格式
    │
    └─ 全部失败
        └─ m_decoderState = STATE_HW_FAILED
        └─ 返回 avcodec_default_get_format（回退到软解）
```

对应源码（`VideoCodecFFmpeg.cpp` 第 104-230 行）中的关键逻辑：

```cpp
// 来源：krkr2/cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 104-230 行
enum AVPixelFormat CDVDVideoCodecFFmpeg::GetFormat(
    struct AVCodecContext *avctx, const AVPixelFormat *fmt)
{
    CDVDVideoCodecFFmpeg *ctx = (CDVDVideoCodecFFmpeg *)avctx->opaque;

    // 非硬件单解码状态 → 直接返回默认格式（软解）
    if (ctx->m_decoderState != STATE_HW_SINGLE || ...) {
        return avcodec_default_get_format(avctx, fmt);
    }

    // 清理旧的硬件解码器
    if (ctx->GetHardware()) {
        ctx->SetHardware(nullptr);
        avctx->get_buffer2 = avcodec_default_get_buffer2;
        // ...
    }

    // 遍历候选格式，尝试各平台硬件解码器
    const AVPixelFormat *cur = fmt;
    while (*cur != AV_PIX_FMT_NONE) {
#ifdef HAS_DX
        if (DXVA::CDecoder::Supports(*cur) && /* 设置已启用 */) {
            DXVA::CDecoder *dec = new DXVA::CDecoder(ctx->m_processInfo);
            if (dec->Open(avctx, ctx->m_pCodecContext, *cur,
                          ctx->m_uSurfacesCount)) {
                ctx->SetHardware(dec);  // 硬件解码器绑定成功
                return *cur;
            } else {
                dec->Release();  // 打开失败，释放并继续尝试下一个
            }
        }
#endif
        // ... 类似处理 VDPAU, VAAPI, VTB, MMAL
        cur++;
    }

    // 所有硬件解码器都失败了
    ctx->m_decoderState = STATE_HW_FAILED;
    return avcodec_default_get_format(avctx, fmt);  // 回退软解
}
```

### 4.5.3 多线程软解的配置

当硬件解码不可用时，KrKr2 使用多线程软解（`VideoCodecFFmpeg.cpp` 第 311-348 行）：

```cpp
// 来源：krkr2/cpp/core/movie/ffmpeg/VideoCodecFFmpeg.cpp 第 336-348 行
if (tryhw && m_decoderState == STATE_NONE) {
    m_decoderState = STATE_HW_SINGLE;  // 优先尝试硬件解码
} else {
    // 软解：根据 CPU 核心数设置线程数
    int num_threads = TVPGetProcessorNum() * 3 / 2;  // 核心数的 1.5 倍
    num_threads = std::max(1, std::min(num_threads, 16));  // 限制 1-16
    m_pCodecContext->thread_count = num_threads;
    m_pCodecContext->thread_safe_callbacks = 1;
    m_decoderState = STATE_SW_MULTI;
}
```

```
  └── CDVDVideoCodecFFmpeg
        ├── Open()         — 查找解码器、配置线程模型、打开解码器
        ├── Decode()       — 核心解码逻辑（旧 API avcodec_decode_video2）
        ├── GetPicture()   — 从解码帧提取 DVDVideoPicture

        ├── FilterProcess()— 将帧送入滤镜处理
        └── IHardwareDecoder — 硬件解码器接口（内部类）
              ├── DXVA::CDecoder    (Windows)

              ├── VAAPI::CDecoder   (Linux Intel/AMD)
              ├── VTB::CDecoder     (macOS)
              └── MMAL::CDecoder    (Raspberry Pi)
```

**Open 方法关键步骤**（第 266-435 行）：

1. `avcodec_find_decoder(hints.codec)` — 根据 codec_id 查找解码器
2. `avcodec_alloc_context3(pCodec)` — 分配解码器上下文
3. 设置 `get_format` 回调为 `GetFormat`（硬件加速协商）
4. 复制 extradata（SPS/PPS 等编解码器特定数据）
5. 配置线程模型（硬件单线程 or 软件多线程）
6. `avcodec_open2()` — 打开解码器（可能重试 lowres）
7. 分配三个 AVFrame（m_pFrame、m_pDecodedFrame、m_pFilterFrame）

**Decode 方法核心流程**（第 555-726 行）：

```cpp
int CDVDVideoCodecFFmpeg::Decode(uint8_t *pData, int iSize,
                                 double dts, double pts) {
    // 1. 硬件解码器优先处理
    if (m_pHardware) {
        result = m_pHardware->Check/Decode(m_pCodecContext);
        if (result) return result;
    }

    // 2. 构造 AVPacket 并调用旧版 API 解码
    AVPacket avpkt;
    av_init_packet(&avpkt);
    avpkt.data = pData;
    avpkt.size = iSize;
    int len = avcodec_decode_video2(m_pCodecContext, m_pDecodedFrame,
                                    &iGotPicture, &avpkt);

    // 3. 处理帧丢弃逻辑（CDropControl 自适应）
    m_dropCtrl.Process(framePTS, ...);

    // 4. 根据解码模式分发结果
    if (m_pHardware) {
        result = m_pHardware->Decode(m_pCodecContext, m_pFrame);
    } else if (m_pFilterGraph) {
        result = FilterProcess(m_pDecodedFrame);  // 滤镜处理
    } else {
        av_frame_move_ref(m_pFrame, m_pDecodedFrame);  // 直接输出
        result = VC_PICTURE | VC_BUFFER;
    }
    return result;
}
```

**CDropControl — 自适应丢帧控制**（第 61-102 行）：

这是一个智能丢帧控制器，通过统计帧间 PTS 差值来判断是否可以安全丢帧：

```cpp
// 初始化阶段：收集 10 帧以上的 PTS 差值，计算平均帧间隔
// 有效阶段：如果实际 PTS 差值偏离平均值超过 20%，说明有帧被跳过
```

### 4.6.3 FFWaveDecoder — sound 模块的 FFmpeg 桥接

**文件路径**：`krkr2/cpp/core/sound/FFWaveDecoder.cpp`（452 行）

这个类与 movie 模块独立，是 KiriKiri 原生音频系统与 FFmpeg 的桥接层：

```
tTVPWaveDecoder (KiriKiri 音频解码器接口)
  └── FFWaveDecoder
        ├── SetStream()       — 打开文件、创建自定义 IO、查找音频流、打开解码器
        ├── Render()          — 解码并输出 PCM 到缓冲区
        ├── SetPosition()     — 定位到指定采样位置
        └── audio_decode_frame() — 内部解码一帧音频
```

**FFWaveDecoder 的特殊之处**：

1. **直接使用旧版 `avcodec_decode_audio4`**（第 378 行）
2. **手动处理平面格式转交错格式**（第 112-158 行的 `CopySmaples` 模板函数）
3. **自定义 I/O 回调**通过 `tTJSBinaryStream` 读取 KiriKiri 虚拟文件系统中的文件
4. **优化的立体声路径**：`_CopySmaples2` 模板针对双声道做了特化优化

```cpp
// 来源：krkr2/cpp/core/sound/FFWaveDecoder.cpp 第 126-136 行
// 立体声优化：避免通道循环的开销
template <typename T>
static unsigned char *_CopySmaples2(unsigned char *dst, AVFrame *frame,
                                    int samples, int buf_index) {
    int buf_pos = buf_index * sizeof(T);
    T *pDst = (T *)dst;
    for (int i = 0; i < samples; ++i, buf_pos += sizeof(T)) {
        *pDst++ = *(T *)(frame->data[0] + buf_pos);  // 左声道
        *pDst++ = *(T *)(frame->data[1] + buf_pos);  // 右声道
    }
    return (unsigned char *)pDst;
}
```
