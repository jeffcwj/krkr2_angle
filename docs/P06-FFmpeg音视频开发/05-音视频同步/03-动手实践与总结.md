## 实践 1：音频主时钟同步框架

本实践实现一个简化但完整的音视频同步框架。它模拟了真实播放器中视频线程如何根据音频时钟决定每一帧的处理方式：显示（DISPLAY）、丢弃（SKIP）、或等待（WAIT）。

### 设计思路

在真实播放器中，音频线程以固定速率将 PCM（Pulse Code Modulation，脉冲编码调制——原始数字音频格式）数据送入声卡，声卡硬件时钟驱动播放。视频线程则需要根据音频时钟的进度来决定当前帧的命运：

```
音频时钟（主时钟）        视频帧队列
    │                    ┌─────┬─────┬─────┬─────┐
    │ audioClock = 1.5s  │ 1.3s│ 1.5s│ 1.7s│ 1.9s│
    │                    └──┬──┴──┬──┴──┬──┴─────┘
    │                       │     │     │
    │ diff = pts - audio    │     │     │
    │                       │     │     │
    ▼                       ▼     ▼     ▼
                         丢弃   显示   等待
                        (落后) (同步) (超前)
```

### 完整代码

```cpp
// av_sync_demo.cpp — 音频主时钟同步演示
// 功能：模拟 30fps 视频与音频时钟的同步决策过程
// 编译：g++ -std=c++17 -o av_sync_demo av_sync_demo.cpp -lpthread
#include <cstdio>
#include <cstdint>
#include <cmath>
#include <chrono>
#include <thread>
#include <vector>

// ============================================================
// 简化版时钟（模拟 CDVDClock 的核心功能）
// ============================================================
class SimpleClock {
public:
    void start() {
        m_startTime = now();
        m_paused = false;
    }

    void pause(bool p) {
        if (p && !m_paused) {
            m_pauseTime = now();   // 冻结时间
            m_paused = true;
        } else if (!p && m_paused) {
            // 恢复时补偿暂停期间的时间流逝
            m_startTime += (now() - m_pauseTime);
            m_paused = false;
        }
    }

    // 获取当前播放时间（秒）
    double getClock() const {
        if (m_paused)
            return (m_pauseTime - m_startTime) / 1e6;
        return (now() - m_startTime) / 1e6;
    }

    // 跳变修正（修改起始时间来偏移播放时间）
    void adjust(double seconds) {
        m_startTime -= (int64_t)(seconds * 1e6);
    }

private:
    // 获取当前系统时间（微秒）
    static int64_t now() {
        auto t = std::chrono::steady_clock::now().time_since_epoch();
        return std::chrono::duration_cast<
            std::chrono::microseconds>(t).count();
    }

    int64_t m_startTime = 0;   // 播放起始的系统时间
    int64_t m_pauseTime = 0;   // 暂停时刻的系统时间
    bool m_paused = false;     // 是否处于暂停状态
};

// ============================================================
// 视频帧结构
// ============================================================
struct VideoFrame {
    double pts;         // 显示时间戳（秒）
    int frameNumber;    // 帧序号
    bool isKeyFrame;    // 是否为关键帧（I帧，可独立解码的帧）
};

// ============================================================
// 音频主时钟同步器
// ============================================================
class AudioMasterSync {
public:
    // syncThreshold: 同步容差（秒），默认 40ms
    // 小于此值的偏差被认为是"同步的"
    AudioMasterSync(double syncThreshold = 0.04)
        : m_syncThreshold(syncThreshold) {}

    // 同步决策结果
    enum Action {
        DISPLAY,  // 显示这一帧（在同步范围内）
        SKIP,     // 丢弃这一帧（视频落后于音频）
        WAIT      // 等待一段时间再显示（视频超前于音频）
    };

    // 根据视频帧 PTS 和当前音频时钟，决定帧的处理方式
    Action decide(const VideoFrame &frame, double audioClock) {
        // diff > 0: 视频帧的显示时间在音频时钟之后（视频超前）
        // diff < 0: 视频帧的显示时间在音频时钟之前（视频落后）
        double diff = frame.pts - audioClock;

        if (diff < -m_syncThreshold) {
            // 视频严重落后 → 丢弃该帧以追赶音频
            m_droppedFrames++;
            m_lastAction = SKIP;
            return SKIP;
        } else if (diff > m_syncThreshold) {
            // 视频超前 → 计算需要等待的时间
            m_waitMs = (int)(diff * 1000);
            m_lastAction = WAIT;
            return WAIT;
        } else {
            // 在同步范围内 → 正常显示
            m_displayedFrames++;
            m_lastAction = DISPLAY;
            return DISPLAY;
        }
    }

    int getWaitMs() const { return m_waitMs; }
    int getDroppedFrames() const { return m_droppedFrames; }
    int getDisplayedFrames() const { return m_displayedFrames; }

    // 获取同步质量评分（0-100）
    int getSyncQuality() const {
        int total = m_displayedFrames + m_droppedFrames;
        if (total == 0) return 100;
        return (int)(100.0 * m_displayedFrames / total);
    }

private:
    double m_syncThreshold;       // 同步容差（秒）
    int m_waitMs = 0;             // 上次等待的毫秒数
    int m_droppedFrames = 0;      // 累计丢弃帧数
    int m_displayedFrames = 0;    // 累计显示帧数
    Action m_lastAction = DISPLAY;
};

int main() {
    SimpleClock clock;
    AudioMasterSync sync(0.04);  // 40ms 同步容差

    // 模拟 30fps 视频，播放 5 秒 = 150 帧
    const int totalFrames = 150;
    const double fps = 30.0;
    const double frameDuration = 1.0 / fps;  // 每帧 33.33ms

    printf("=== 音频主时钟同步演示 ===\n");
    printf("视频: %dfps, 总帧数: %d, 时长: %.1f秒\n",
           (int)fps, totalFrames, totalFrames / fps);
    printf("同步容差: %.0fms\n\n", sync.getSyncQuality() > 0 ? 40.0 : 0);

    clock.start();

    int skipCount = 0, waitCount = 0, displayCount = 0;

    for (int i = 0; i < totalFrames; i++) {
        VideoFrame frame;
        frame.pts = i * frameDuration;  // 理想显示时间
        frame.frameNumber = i;
        frame.isKeyFrame = (i % 30 == 0);  // 每 30 帧一个 I 帧

        // 获取当前音频时钟（这里用 SimpleClock 模拟）
        double audioClock = clock.getClock();

        auto action = sync.decide(frame, audioClock);

        // 每 30 帧打印一次详细信息，避免输出过多
        bool verbose = (i % 30 == 0) || action != AudioMasterSync::DISPLAY;

        switch (action) {
            case AudioMasterSync::DISPLAY:
                displayCount++;
                if (verbose) {
                    printf("帧 #%03d [%.3fs]: 显示 (diff=%+.1fms)%s\n",
                           i, frame.pts,
                           (frame.pts - audioClock) * 1000,
                           frame.isKeyFrame ? " [关键帧]" : "");
                }
                // 模拟渲染耗时（真实播放器中这里是 SDL_RenderPresent 等）
                std::this_thread::sleep_for(
                    std::chrono::milliseconds((int)(frameDuration * 1000)));
                break;

            case AudioMasterSync::SKIP:
                skipCount++;
                printf("帧 #%03d [%.3fs]: ★丢弃★ (落后 %.1fms)%s\n",
                       i, frame.pts,
                       (audioClock - frame.pts) * 1000,
                       frame.isKeyFrame ? " [关键帧!]" : "");
                // 丢弃帧不等待，立即处理下一帧
                break;

            case AudioMasterSync::WAIT: {
                waitCount++;
                int waitMs = sync.getWaitMs();
                if (verbose) {
                    printf("帧 #%03d [%.3fs]: 等待 %dms (超前 %.1fms)\n",
                           i, frame.pts, waitMs,
                           (frame.pts - audioClock) * 1000);
                }
                std::this_thread::sleep_for(
                    std::chrono::milliseconds(waitMs));
                break;
            }
        }
    }

    // 输出统计信息
    printf("\n=== 同步统计 ===\n");
    printf("总帧数:   %d\n", totalFrames);
    printf("显示:     %d (%.1f%%)\n",
           displayCount, 100.0 * displayCount / totalFrames);
    printf("丢弃:     %d (%.1f%%)\n",
           skipCount, 100.0 * skipCount / totalFrames);
    printf("等待:     %d (%.1f%%)\n",
           waitCount, 100.0 * waitCount / totalFrames);
    printf("实际时长: %.3f 秒（期望 %.3f 秒）\n",
           clock.getClock(), totalFrames / fps);
    printf("同步质量: %d%%\n", sync.getSyncQuality());

    return 0;
}
```

## 实践 2：PTS 时间基转换工具

本实践使用 FFmpeg API 读取真实媒体文件，展示不同流的时间基（time_base）差异和 PTS 转换过程。

### 设计思路

不同容器格式和编解码器（Codec，编码解码器的缩写）使用不同的 time_base。例如：
- **MPEG-TS** 容器：视频流通常用 `{1, 90000}`（1 tick = 11.11μs）
- **MP4** 容器：视频流通常用 `{1, 帧率的整数倍}`
- **音频流**：通常用 `{1, 采样率}`（如 `{1, 48000}`）

跨流传递 PTS 时必须正确转换时间基，否则会导致同步错误。

### 完整代码

```cpp
// timebase_converter.cpp — PTS 时间基转换工具
// 功能：读取媒体文件，展示各流的时间基和 PTS 转换
// 编译：需要 FFmpeg 开发库
// g++ -std=c++17 -o timebase_conv timebase_converter.cpp \
//     $(pkg-config --cflags --libs libavformat libavutil)
#include <cstdio>
#include <cstdlib>
#include <cinttypes>

extern "C" {
#include <libavformat/avformat.h>
#include <libavutil/mathematics.h>
}

// 安全地将 PTS 从一个时间基转换到另一个时间基
// 使用 av_rescale_q_rnd 避免溢出
int64_t safe_rescale(int64_t pts, AVRational src_tb, AVRational dst_tb) {
    if (pts == AV_NOPTS_VALUE)
        return AV_NOPTS_VALUE;

    // AV_ROUND_NEAR_INF: 四舍五入到最近值
    // AV_ROUND_PASS_MINMAX: 保留 INT64_MIN/MAX 不溢出
    return av_rescale_q_rnd(pts, src_tb, dst_tb,
        (AVRounding)(AV_ROUND_NEAR_INF | AV_ROUND_PASS_MINMAX));
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "用法: %s <媒体文件>\n", argv[0]);
        fprintf(stderr, "示例: %s video.mp4\n", argv[0]);
        return 1;
    }

    // 打开媒体文件
    AVFormatContext *fmtCtx = nullptr;
    int ret = avformat_open_input(&fmtCtx, argv[1], nullptr, nullptr);
    if (ret < 0) {
        char errbuf[256];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开文件 '%s': %s\n", argv[1], errbuf);
        return 1;
    }

    // 读取流信息（分析文件头部和部分数据）
    ret = avformat_find_stream_info(fmtCtx, nullptr);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    // === 第一部分：展示各流的时间基信息 ===
    printf("=== 文件: %s ===\n", argv[1]);
    printf("格式: %s (%s)\n", fmtCtx->iformat->name,
           fmtCtx->iformat->long_name);
    printf("总时长: %.3f 秒\n",
           fmtCtx->duration > 0
               ? fmtCtx->duration / (double)AV_TIME_BASE : -1.0);
    printf("流数量: %u\n\n", fmtCtx->nb_streams);

    // 定义统一的目标时间基：毫秒和微秒
    AVRational tb_ms = {1, 1000};        // 毫秒时间基
    AVRational tb_us = {1, 1000000};     // 微秒时间基（= DVD_TIME_BASE）

    printf("=== 各流时间基信息 ===\n");
    for (unsigned i = 0; i < fmtCtx->nb_streams; i++) {
        AVStream *st = fmtCtx->streams[i];
        const char *type = av_get_media_type_string(
            st->codecpar->codec_type);
        const char *codecName = avcodec_get_name(st->codecpar->codec_id);

        printf("\n流 #%u (%s, %s):\n", i,
               type ? type : "未知", codecName);
        printf("  time_base: %d/%d",
               st->time_base.num, st->time_base.den);
        printf(" (1 tick = %.6f 秒 = %.3f 毫秒)\n",
               av_q2d(st->time_base),
               av_q2d(st->time_base) * 1000);

        // 展示时长转换
        if (st->duration != AV_NOPTS_VALUE) {
            double durSec = st->duration * av_q2d(st->time_base);
            int64_t durMs = safe_rescale(st->duration,
                                         st->time_base, tb_ms);
            int64_t durUs = safe_rescale(st->duration,
                                         st->time_base, tb_us);
            printf("  duration: %" PRId64 " ticks = %.3f 秒"
                   " = %" PRId64 " ms = %" PRId64 " μs\n",
                   st->duration, durSec, durMs, durUs);
        }

        // 展示起始时间转换
        if (st->start_time != AV_NOPTS_VALUE) {
            double startSec = st->start_time * av_q2d(st->time_base);
            printf("  start_time: %" PRId64 " ticks = %.3f 秒\n",
                   st->start_time, startSec);
        }

        // 展示特定编解码器参数
        if (st->codecpar->codec_type == AVMEDIA_TYPE_VIDEO) {
            printf("  分辨率: %dx%d\n",
                   st->codecpar->width, st->codecpar->height);
            if (st->avg_frame_rate.den > 0) {
                printf("  帧率: %.3f fps\n",
                       av_q2d(st->avg_frame_rate));
            }
        } else if (st->codecpar->codec_type == AVMEDIA_TYPE_AUDIO) {
            printf("  采样率: %d Hz, 声道数: %d\n",
                   st->codecpar->sample_rate,
                   st->codecpar->ch_layout.nb_channels);
        }
    }

    // === 第二部分：读取前 20 个包，展示 PTS/DTS 转换 ===
    printf("\n=== 前 20 个数据包的时间戳转换 ===\n");
    printf("%-4s %-6s %-12s %-10s %-10s %-12s %-10s\n",
           "序号", "流/类型", "PTS(原始)", "PTS(秒)",
           "PTS(毫秒)", "DTS(原始)", "DTS(秒)");
    printf("──────────────────────────────────────────"
           "──────────────────────────────\n");

    AVPacket *pkt = av_packet_alloc();
    if (!pkt) {
        fprintf(stderr, "无法分配 AVPacket\n");
        avformat_close_input(&fmtCtx);
        return 1;
    }

    int count = 0;
    while (av_read_frame(fmtCtx, pkt) >= 0 && count < 20) {
        AVStream *st = fmtCtx->streams[pkt->stream_index];
        const char *type = av_get_media_type_string(
            st->codecpar->codec_type);

        // 将 PTS/DTS 转换为秒和毫秒
        double ptsSec = (pkt->pts != AV_NOPTS_VALUE)
            ? pkt->pts * av_q2d(st->time_base) : -1.0;
        double dtsSec = (pkt->dts != AV_NOPTS_VALUE)
            ? pkt->dts * av_q2d(st->time_base) : -1.0;
        int64_t ptsMs = safe_rescale(pkt->pts, st->time_base, tb_ms);

        printf("%-4d %u/%-5s %-12" PRId64 " %-10.3f %-10" PRId64
               " %-12" PRId64 " %-10.3f\n",
               count, pkt->stream_index,
               type ? type : "?",
               pkt->pts, ptsSec, ptsMs,
               pkt->dts, dtsSec);

        av_packet_unref(pkt);
        count++;
    }

    // === 第三部分：跨流时间基转换示例 ===
    if (fmtCtx->nb_streams >= 2) {
        printf("\n=== 跨流时间基转换示例 ===\n");
        AVRational tb0 = fmtCtx->streams[0]->time_base;
        AVRational tb1 = fmtCtx->streams[1]->time_base;

        // 假设流 0 的 PTS = 10000，转换到流 1 的时间基
        int64_t sample_pts = 10000;
        int64_t converted = safe_rescale(sample_pts, tb0, tb1);
        double seconds = sample_pts * av_q2d(tb0);

        printf("流0 time_base: %d/%d\n", tb0.num, tb0.den);
        printf("流1 time_base: %d/%d\n", tb1.num, tb1.den);
        printf("流0 PTS=%" PRId64 " → %.6f 秒 → 流1 PTS=%" PRId64 "\n",
               sample_pts, seconds, converted);
    }

    av_packet_free(&pkt);
    avformat_close_input(&fmtCtx);

    printf("\n=== 完成 ===\n");
    return 0;
}
```

## 实践 3：带漂移模拟的同步测试

真实播放中，声卡的实际采样率与标称值总有微小差异，导致长时间播放后音视频逐渐不同步。本实践模拟这种漂移并展示修正策略。

```cpp
// drift_demo.cpp — 时钟漂移模拟与修正演示
// 编译：g++ -std=c++17 -o drift_demo drift_demo.cpp -lpthread
#include <cstdio>
#include <cmath>
#include <chrono>
#include <thread>

// 带漂移模拟的时钟
class DriftingClock {
public:
    // driftPpm: 漂移量（PPM，百万分之一）
    // 例如 +50 PPM = 每秒快 50 微秒
    DriftingClock(double driftPpm = 0)
        : m_driftRate(1.0 + driftPpm / 1e6) {}

    void start() { m_startTime = now(); }

    // 模拟有漂移的时钟读取
    double getClock() const {
        double elapsed = (now() - m_startTime) / 1e6;  // 真实经过时间
        return elapsed * m_driftRate;  // 应用漂移：略快或略慢
    }

private:
    static int64_t now() {
        auto t = std::chrono::steady_clock::now().time_since_epoch();
        return std::chrono::duration_cast<
            std::chrono::microseconds>(t).count();
    }

    int64_t m_startTime = 0;
    double m_driftRate;  // 漂移率：1.0 = 无漂移
};

int main() {
    // 声卡时钟有 +50 PPM 漂移（实际比标称快 0.005%）
    // 这意味着每 20 秒漂移 1 毫秒
    DriftingClock audioClock(50);
    DriftingClock videoClock(0);  // 视频使用理想时钟

    audioClock.start();
    videoClock.start();

    printf("=== 时钟漂移模拟 ===\n");
    printf("音频时钟漂移: +50 PPM (每秒快 50μs)\n");
    printf("视频时钟漂移: 0 PPM (理想时钟)\n\n");

    printf("%-10s %-12s %-12s %-12s %-8s\n",
           "经过时间", "音频时钟", "视频时钟", "偏差", "状态");
    printf("────────────────────────────────────────────────\n");

    double correctionTotal = 0;
    const double threshold = 0.02;  // 20ms 修正阈值

    // 模拟 60 秒播放，每秒检查一次
    for (int sec = 1; sec <= 60; sec++) {
        std::this_thread::sleep_for(std::chrono::seconds(1));

        double audio = audioClock.getClock();
        double video = videoClock.getClock();
        double drift = audio - video;  // 正值 = 音频快
        const char *status = "同步";

        // 当漂移超过阈值时修正
        if (fabs(drift) > threshold) {
            correctionTotal += drift;
            status = "★修正★";
        }

        // 每 5 秒或有修正时打印
        if (sec % 5 == 0 || fabs(drift) > threshold) {
            printf("%-10ds %-12.6f %-12.6f %+.3fms    %s\n",
                   sec, audio, video,
                   drift * 1000, status);
        }
    }

    printf("\n=== 漂移统计 ===\n");
    printf("理论漂移: +50 PPM × 60s = +%.3f ms\n", 50.0 * 60 / 1000);
    printf("实际测量偏差: %+.3f ms\n",
           (audioClock.getClock() - videoClock.getClock()) * 1000);
    printf("累计修正量: %+.3f ms\n", correctionTotal * 1000);

    return 0;
}
```

```bash
# Windows (MSVC)
```

## 实践 4：手动时间基计算器

```cpp
// timebase_calc.cpp — 手动时间基计算器
// 编译：g++ -std=c++17 -o timebase_calc timebase_calc.cpp
#include <cstdio>
#include <cstdint>
#include <cmath>

// 模拟 FFmpeg 的 AVRational
struct Rational {
    int num;  // 分子
    int den;  // 分母
};

// 将 PTS 转换为秒
double pts_to_seconds(int64_t pts, Rational tb) {
    return (double)pts * tb.num / tb.den;
}

// 秒转换为 PTS
int64_t seconds_to_pts(double seconds, Rational tb) {
    return (int64_t)(seconds * tb.den / tb.num + 0.5);  // 四舍五入
}

// 跨时间基转换（等价于 av_rescale_q）
// src_pts * (src_tb.num / src_tb.den) = dst_pts * (dst_tb.num / dst_tb.den)
// dst_pts = src_pts * src_tb.num * dst_tb.den / (src_tb.den * dst_tb.num)
int64_t rescale_q(int64_t src_pts, Rational src_tb, Rational dst_tb) {
    // 使用 __int128 或 double 避免溢出
    // 简化版使用 double：
    double seconds = (double)src_pts * src_tb.num / src_tb.den;
    return (int64_t)(seconds * dst_tb.den / dst_tb.num + 0.5);
}

int main() {
    printf("=== 时间基转换计算器 ===\n\n");

    // 定义常见时间基
    Rational tb_90k  = {1, 90000};     // MPEG-TS 视频
    Rational tb_44k  = {1, 44100};     // CD 音频采样率
    Rational tb_48k  = {1, 48000};     // 专业音频采样率
    Rational tb_ms   = {1, 1000};      // 毫秒
    Rational tb_us   = {1, 1000000};   // 微秒 (DVD_TIME_BASE)
    Rational tb_30fps = {1, 30};       // 30fps 帧号

    // 示例 1：MPEG-TS PTS 转换
    printf("--- 示例 1：MPEG-TS PTS 转换 ---\n");
    int64_t ts_pts = 8100000;  // MPEG-TS 的 PTS 值
    double ts_sec = pts_to_seconds(ts_pts, tb_90k);
    printf("MPEG-TS PTS = %lld (time_base=1/90000)\n",
           (long long)ts_pts);
    printf("  → %.3f 秒\n", ts_sec);
    printf("  → %lld 毫秒 (time_base=1/1000)\n",
           (long long)rescale_q(ts_pts, tb_90k, tb_ms));
    printf("  → %lld 微秒 (DVD_TIME_BASE=1/1000000)\n",
           (long long)rescale_q(ts_pts, tb_90k, tb_us));
    printf("  → 第 %lld 帧 (30fps)\n",
           (long long)rescale_q(ts_pts, tb_90k, tb_30fps));

    // 示例 2：音频采样位置转换
    printf("\n--- 示例 2：音频采样位置转换 ---\n");
    int64_t audio_pts = 441000;  // 44100Hz 下的第 441000 个采样点
    printf("音频 PTS = %lld (time_base=1/44100)\n",
           (long long)audio_pts);
    printf("  → %.3f 秒\n", pts_to_seconds(audio_pts, tb_44k));
    printf("  → %lld 毫秒\n",
           (long long)rescale_q(audio_pts, tb_44k, tb_ms));
    printf("  → %lld 微秒 (DVD_TIME_BASE)\n",
           (long long)rescale_q(audio_pts, tb_44k, tb_us));

    // 示例 3：帧号到时间戳
    printf("\n--- 示例 3：帧号到时间戳 ---\n");
    int64_t frame_num = 90;  // 第 90 帧
    printf("第 %lld 帧 (30fps)\n", (long long)frame_num);
    printf("  → %.3f 秒\n", pts_to_seconds(frame_num, tb_30fps));
    printf("  → %lld MPEG-TS ticks\n",
           (long long)rescale_q(frame_num, tb_30fps, tb_90k));
    printf("  → %lld 毫秒\n",
           (long long)rescale_q(frame_num, tb_30fps, tb_ms));

    // 示例 4：验证溢出风险
    printf("\n--- 示例 4：大值 PTS 溢出检查 ---\n");
    int64_t big_pts = 7200LL * 90000;  // 2 小时的 MPEG-TS PTS
    printf("2小时 MPEG-TS PTS = %lld\n", (long long)big_pts);
    printf("  直接整数乘法: %lld * 1000000 / 90000 = %lld (正确)\n",
           (long long)big_pts,
           (long long)(big_pts * 1000000LL / 90000));
    printf("  注意: 如果 PTS 更大（如 48小时），\n");
    printf("        big_pts * 1000000 可能溢出 int64！\n");
    printf("        48h: %lld * 1000000 = ",
           (long long)(48LL * 3600 * 90000));
    // 检查是否溢出
    int64_t huge_pts = 48LL * 3600 * 90000;
    double check = (double)huge_pts * 1000000.0;
    printf("%.0f\n", check);
    printf("        INT64_MAX = %lld\n", (long long)INT64_MAX);
    printf("        %s\n",
           check > (double)INT64_MAX ? "★会溢出★ → 必须用 double 中间值"
                                     : "不会溢出");

    printf("\n=== 计算完成 ===\n");
    return 0;
}
```

## 常见错误与解决方案

### 错误 1：seek 后音画不同步

```
现象：用户拖动进度条后，画面和声音不对齐，持续 1-2 秒后恢复
原因：seek 后没有正确 flush（清空）解码器缓冲和重置同步状态

正确的 seek 流程（KrKr2 实际做法）：
1. CDVDDemuxFFmpeg::SeekTime()    — 调用 av_seek_frame 定位
2. CDVDClock::Discontinuity()     — 重置时钟到目标位置
3. 向音频线程发送 GENERAL_FLUSH   — 清空音频解码器和输出缓冲
4. 向视频线程发送 GENERAL_FLUSH   — 清空视频解码器和输出缓冲
5. avcodec_flush_buffers()        — 重置 FFmpeg 内部状态
6. 音频线程回到 SYNC_STARTING     — 重新走同步流程
7. STARTING → WAITSYNC → INSYNC   — 重建同步
```

```cpp
// 错误示例：seek 后不 flush
void badSeek(double targetSec) {
    demuxer->seek(targetSec);
    clock->discontinuity(targetSec * DVD_TIME_BASE, ...);
    // ❌ 忘记 flush 解码器 → 旧数据残留 → 画面闪烁
}

// 正确示例
void goodSeek(double targetSec) {
    demuxer->seek(targetSec);
    clock->discontinuity(targetSec * DVD_TIME_BASE, ...);
    audioPlayer->sendMessage(GENERAL_FLUSH);   // 清空音频
    videoPlayer->sendMessage(GENERAL_FLUSH);   // 清空视频
    // 各线程在处理 FLUSH 时会调用 avcodec_flush_buffers()
}
```

### 错误 2：长时间播放后逐渐不同步（漂移）

```
现象：播放开始时完美同步，30 分钟后明显听到音画不齐
原因：声卡采样率漂移（Drift）——标称 48000Hz 实际可能是 47998Hz

漂移量计算：
  偏差 = 2Hz / 48000Hz = 0.0000417 = 41.7 PPM
  30 分钟后：0.0000417 × 1800s = 0.075s = 75ms（可感知！）

解决方案：
1. 持续监测同步误差（每帧或每秒检测一次）
2. 使用渐进式速度微调（CDVDClock::SetSpeedAdjust）
3. 微调系数通常极小（如 0.0001），让时钟速率略快/慢以跟上声卡
4. 不要用大幅跳变——频繁跳变会造成画面卡顿
```

### 错误 3：av_rescale_q 整数溢出

```
现象：长时间视频（>24小时）的 PTS 转换得到负数或异常值
原因：av_rescale_q 内部计算 a * b / c，当 a 和 b 都很大时乘积溢出 int64

危险计算示例：
  PTS = 48h × 90000 = 15,552,000,000
  转换到微秒：15,552,000,000 × 1,000,000 = 1.555 × 10^16
  INT64_MAX = 9.22 × 10^18 → 这次没溢出
  
  但如果 PTS = 720h × 90000 = 233,280,000,000
  233,280,000,000 × 1,000,000 = 2.33 × 10^17 → 还没溢出
  
  真正危险的是非标准时间基，如 time_base = {1, 1000000000}（纳秒）
  1h × 10^9 = 3.6 × 10^12
  3.6 × 10^12 × 10^6 = 3.6 × 10^18 → 接近 INT64_MAX！

解决：始终使用 av_rescale_q_rnd 并加上 AV_ROUND_PASS_MINMAX 标志
      或使用 double 中间变量进行转换
```

```cpp
// 安全的时间基转换
int64_t safe_convert(int64_t pts, AVRational src, AVRational dst) {
    // 方法 1：使用 FFmpeg 的安全函数
    return av_rescale_q_rnd(pts, src, dst,
        (AVRounding)(AV_ROUND_NEAR_INF | AV_ROUND_PASS_MINMAX));

    // 方法 2：使用 double（精度稍低但不会溢出）
    // double seconds = (double)pts * src.num / src.den;
    // return (int64_t)(seconds * dst.den / dst.num);
}
```

## 本节小结

本章（音视频同步）的核心知识回顾：

- **音视频同步的本质**是解决两个独立硬件时钟（声卡和显示器）之间的漂移问题
- **PTS/DTS/time_base** 三者共同定义帧的时间位置。PTS 决定显示时间，DTS 决定解码顺序，time_base 是将整数时间戳转换为实际秒数的缩放因子
- 三种主时钟策略中，**音频主时钟**最常用（保证音频连续性，视频通过丢帧/等待跟随）
- KrKr2 使用 **SYNC_DISCON** 策略：音频主时钟 + 不连续跳变修正
- **CDVDClock** 通过三层时间模型（系统→绝对→播放）将不可控的硬件时钟转化为可暂停、可变速、可跳变的播放时钟
- **ErrorAdjust** 在 VSync 模式下使用不对称阈值（20ms 超前 / 27ms 落后）按帧步进修正
- **CVideoPlayerAudio** 的三状态同步状态机（STARTING→WAITSYNC→INSYNC）确保启动和 seek 后的平滑同步重建
- 音频时钟采用 **PTS 精确更新 + duration 预测递增** 的双源策略
- 长期漂移通过 `m_speedAdjust` 渐进修正，短期偏差通过 `ErrorAdjust` 跳变修正

2. 如果要转换到 `time_base = {1, 1000}`（毫秒），新的 PTS 值是多少？
3. 如果要转换到 KrKr2 的 `DVD_TIME_BASE`（1/1000000），新的 PTS 值是多少？
4. 如果视频是 24fps，这是第几帧？

<summary>查看答案</summary>

   ```
   new_pts = 8100000 × (1/90000) / (1/1000)
   ```

   PTS = 90000000（即 90.0 秒 × 1000000）✓

### 题目 2：同步策略选择

阅读以下场景，判断应该使用哪种同步策略，并说明理由：

- 场景 A：用户在电脑上观看一部 2 小时的电影
- 场景 B：直播推流软件（OBS 等），需要与真实时间对齐
- 场景 C：视觉小说游戏中播放过场动画（KrKr2 的典型场景）
- 场景 D：音乐播放器的可视化效果（频谱图跟随音乐节拍）

<details>
<summary>查看答案</summary>

- **场景 A：音频主时钟**
  - 理由：2 小时长时间播放，音频连续性最重要——中断音频比丢视频帧更令人不适。声卡硬件时钟稳定，适合作为长期基准。偶尔丢帧观众几乎感知不到。

- **场景 B：外部主时钟**
  - 理由：直播需要与真实时间（墙钟/NTP 时钟）对齐，不能让音频或视频流"自主"决定时间。音频和视频都需要根据墙钟独立调整速率。

- **场景 C：音频主时钟（SYNC_DISCON）**
  - 理由：视觉小说的过场动画通常较短（几十秒到几分钟），用户对画面和音频的配合非常敏感（对话配音对口型）。音频主时钟保证语音连续流畅。KrKr2 默认使用 SYNC_DISCON 正是因为这个原因。

- **场景 D：音频主时钟（严格跟随）**
  - 理由：可视化效果必须精确跟随音频节拍。音频是绝对权威——频谱数据直接来源于音频 PCM。可视化渲染（视频）必须完全服从音频时钟，任何偏差都会导致"节奏不对"的糟糕体验。

</details>

### 题目 3：CDVDClock 状态追踪

假设 CDVDClock 初始状态如下：

```
m_systemFrequency = 1000000 (1 tick = 1μs)
m_startClock = 1000000      (系统时间 1.0s 时开始播放)
m_systemUsed = 1000000      (1x 速度)
m_iDisc = 0
m_systemAdjust = 0
m_pauseClock = 0
```

按顺序执行以下操作，追踪每步后 `SystemToPlaying(current)` 的返回值：

1. `current = 3000000`（系统时间 3.0s）→ 播放时间 = ?
2. `SetSpeed(2000)`（2x 速度），然后 `current = 3500000`（0.5s 后）→ 播放时间 = ?
3. `SetSpeed(0)`（暂停），然后 `current = 5000000`（1.5s 后）→ 播放时间 = ?
4. `SetSpeed(1000)`（恢复 1x），然后 `current = 6000000`（1s 后）→ 播放时间 = ?

<details>
<summary>查看答案</summary>

**步骤 1**：`current = 3000000`
```
playing = 1000000 × (3000000 - 1000000 + 0) / 1000000 + 0
        = 1000000 × 2000000 / 1000000
        = 2000000.0  → 2.0 秒
```

**步骤 2**：`SetSpeed(2000)` 然后 `current = 3500000`
- SetSpeed 计算：
  ```
  newfreq = 1000000 × 1000 / 2000 = 500000
  new_startClock = 3000000 - (3000000 - 1000000) × 500000 / 1000000
                 = 3000000 - 1000000 = 2000000
  m_systemUsed = 500000
  ```
- SystemToPlaying(3500000)：
  ```
  playing = 1000000 × (3500000 - 2000000 + 0) / 500000 + 0
          = 1000000 × 1500000 / 500000
          = 3000000.0  → 3.0 秒
  ```
  验证：切换时是 2.0 秒，过了 0.5 秒真实时间，2x 速度增加 1.0 秒 → 3.0 秒 ✓

**步骤 3**：`SetSpeed(0)` 暂停，然后 `current = 5000000`
- 暂停：`m_pauseClock = 3500000`（暂停时刻的系统时间）
- SystemToPlaying(5000000)：
  ```
  current = m_pauseClock = 3500000（使用冻结值）
  playing = 1000000 × (3500000 - 2000000 + 0) / 500000 + 0
          = 3000000.0  → 3.0 秒（冻结在暂停时刻）
  ```

**步骤 4**：`SetSpeed(1000)` 恢复，然后 `current = 6000000`
- SetSpeed 计算：
  ```
  newfreq = 1000000 × 1000 / 1000 = 1000000
  // 从暂停恢复：m_startClock += 5000000 - 3500000 = 1500000
  //   → m_startClock = 2000000 + 1500000 = 3500000
  // m_pauseClock = 0
  // 速度切换：new_start = 5000000 - (5000000 - 3500000) × 1000000 / 500000
  //         = 5000000 - 3000000 = 2000000
  // 等等，需要更仔细地追踪...
  
  实际顺序：
  1. current_now = 5000000（SetSpeed 被调用时的系统时间）
  2. 暂停恢复：m_startClock += 5000000 - 3500000 = 1500000
     → m_startClock = 2000000 + 1500000 = 3500000
     m_pauseClock = 0
  3. 速度切换：
     new_start = 5000000 - (5000000 - 3500000) × 1000000 / 500000
               = 5000000 - 3000000 = 2000000
     m_systemUsed = 1000000
  ```
- SystemToPlaying(6000000)：
  ```
  playing = 1000000 × (6000000 - 2000000 + 0) / 1000000 + 0
          = 4000000.0  → 4.0 秒
  ```
  验证：暂停时是 3.0 秒，恢复后过了 1.0 秒真实时间，1x 速度增加 1.0 秒 → 4.0 秒 ✓

</details>
