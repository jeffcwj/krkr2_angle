# P06 — FFmpeg 音视频开发

> **前置知识：** C++ 基础（指针、内存管理、多线程基础）
> **难度：** ★★★★
> **目标：** 掌握 FFmpeg C API，能理解 KrKr2 的完整媒体播放器栈

## 模块简介

FFmpeg 是全球使用最广泛的开源音视频处理框架，KrKr2 项目中的视频播放器（`cpp/core/movie/ffmpeg/`）和部分音频解码器（`cpp/core/sound/`）均基于 FFmpeg 构建。本模块从零教你理解音视频基础概念，掌握 FFmpeg 的 C API，最终能读懂并修改项目中的媒体播放器代码。

## 术语预览

本模块涉及大量 FFmpeg 和音视频领域术语，先在这里建立印象，各章正文会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 容器格式 | Container Format | 把音频、视频、字幕等多路数据打包在一起的文件格式，如 MP4、AVI、MKV |
| 编码格式 | Codec | 音视频数据的压缩/解压算法，如 H.264（视频）、AAC（音频） |
| 采样率 | Sample Rate | 音频每秒采集的样本数（如 44100 Hz），决定音频能表达的最高频率 |
| 帧率 | Frame Rate | 视频每秒显示的画面数（如 30 FPS），决定画面流畅度 |
| PTS | Presentation Time Stamp | 显示时间戳——每一帧/每个音频样本应该在什么时刻呈现给用户 |
| 像素格式 | Pixel Format | 描述一个像素如何存储颜色数据（如 YUV420P、RGB24），决定颜色精度和内存占用 |
| AVFormatContext | — | FFmpeg 中表示"已打开的媒体文件"的核心结构体，包含所有流信息和文件元数据 |
| AVCodecContext | — | FFmpeg 中表示"一个解码器实例"的结构体，存储编解码参数和内部状态 |
| 解封装 | Demuxing | 从容器格式中拆分出独立的音频流、视频流的过程（由 `avformat` 库完成） |
| 解码 | Decoding | 将压缩的音视频数据还原为原始像素/采样数据的过程（由 `avcodec` 库完成） |
| swscale | — | FFmpeg 的图像缩放和像素格式转换库，常用于将解码输出的 YUV 转为 RGB |
| swresample | — | FFmpeg 的音频重采样库，用于转换采样率、通道数、采样格式 |
| avutil | — | FFmpeg 的基础工具库，提供内存管理、数学函数、像素格式定义等公共设施 |
| 音视频同步 | A/V Sync | 确保视频画面与音频声音在时间上对齐——通常以音频时钟为基准调整视频显示时机 |

## 环境准备

学习本模块需要安装 FFmpeg 开发库：

```bash
# Windows (vcpkg)
vcpkg install ffmpeg[core,avcodec,avformat,swscale,swresample]:x64-windows

# Linux (Ubuntu/Debian)
sudo apt install libavcodec-dev libavformat-dev libswscale-dev libswresample-dev libavutil-dev

# macOS (Homebrew)
brew install ffmpeg

# 验证安装
pkg-config --modversion libavcodec  # Linux/macOS
```
