## 一、声音的物理本质

### 1.1 声音是什么？

声音是一种**机械波**——由物体振动引起的空气分子疏密变化，以纵波的形式向四周传播。

关键物理参数：

| 参数 | 符号 | 单位 | 含义 |
|------|------|------|------|
| 频率（Frequency） | f | Hz（赫兹） | 每秒振动次数，决定**音高** |
| 振幅（Amplitude） | A | Pa（帕斯卡） | 压力变化幅度，决定**音量** |
| 波长（Wavelength） | λ | m（米） | 一个完整周期的空间长度 |
| 声速（Speed of Sound） | c | m/s | 约 343 m/s（20°C 空气中） |

它们之间的关系：`c = f × λ`（例如 A4 音 440 Hz，波长 = 343/440 ≈ 0.78 m）

### 1.2 人耳的听觉范围

人耳能感知的频率范围约为 **20 Hz - 20000 Hz（20 kHz）**：

```
 20 Hz          200 Hz         2 kHz          20 kHz
  |───────────────|───────────────|───────────────|
  低频（低音）     中低频          中高频          高频（极高音）
  地板震动         人声基频        乐器泛音        蝙蝠/犬笛
```

这个范围直接决定了数字音频的核心参数——采样率需要多高。

### 1.3 声音信号的数学表示

最简单的声音是**纯音（正弦波）**：

```
y(t) = A × sin(2π × f × t)
```

其中：
- `A` = 振幅（0.0 ~ 1.0 归一化）
- `f` = 频率（Hz）
- `t` = 时间（秒）

现实中的声音几乎都不是纯正弦波，而是多个频率的叠加（**傅里叶分解**）。例如小提琴拉 440 Hz 的 A 音时，包含基频 + 880 Hz 二次谐波 + 1320 Hz 三次谐波等，这些谐波组合决定了**音色（Timbre）**。

```cpp
// 用 C++ 表示纯正弦波信号
#include <cmath>
#include <vector>

// 生成指定频率和持续时间的正弦波采样数据
// sample_rate: 每秒采样次数
// frequency:   正弦波频率 (Hz)
// duration:    持续时间 (秒)
// amplitude:   振幅 (0.0 ~ 1.0)
std::vector<float> generateSineWave(int sample_rate, float frequency,
                                     float duration, float amplitude) {
    int total_samples = static_cast<int>(sample_rate * duration);
    std::vector<float> samples(total_samples);

    for (int i = 0; i < total_samples; ++i) {
        float t = static_cast<float>(i) / sample_rate;   // 当前时间点
        samples[i] = amplitude * std::sin(2.0f * M_PI * frequency * t);
    }
    return samples;
}
```

> **注意：** 上面的代码生成的是 **float 格式的 PCM 数据**。KrKr2 的音频系统默认使用 16-bit 整数 PCM（`TVPPrimarySBBits = 16`，见 `cpp/core/sound/win32/WaveImpl.cpp` 第 63 行），需要做格式转换才能播放。

## 二、从模拟到数字：ADC 与采样

### 2.1 为什么需要数字化？

计算机无法直接存储连续的模拟信号。模拟信号是**时间连续、值连续**的，而计算机只能存储**离散**的数字值。因此需要一个转换过程——**ADC（Analog-to-Digital Conversion，模数转换）**。

ADC 的三个核心步骤：

```
模拟信号 ──→ ① 采样(Sampling) ──→ ② 量化(Quantization) ──→ ③ 编码(Encoding) ──→ 数字信号
              时间离散化              值离散化                二进制表示
```

本节重点讲**采样**，下一节《位深与动态范围》详解量化和编码。

### 2.2 采样（Sampling）

采样是在时间轴上**等间隔地测量**模拟信号的瞬时值。每次测量叫做一个**采样点（Sample）**，单位时间内的采样次数叫做**采样率（Sample Rate）**。

```
模拟信号（连续正弦波）：
    ╱╲      ╱╲      ╱╲
   ╱  ╲    ╱  ╲    ╱  ╲
──╱────╲──╱────╲──╱────╲──
  ╲    ╱  ╲    ╱  ╲    ╱
   ╲  ╱    ╲  ╱    ╲  ╱
    ╲╱      ╲╱      ╲╱

采样后（离散采样点，采样率=每周期8个点）：
    ·        ·        ·
   · ·      · ·      · ·
──·───·──·───·──·───·──·──
  ·   ·  ·   ·  ·   ·
   · ·    · ·    · ·
    ·      ·      ·
```

采样率的单位是 **Hz（赫兹）** 或 **samples/second（采样/秒）**。例如 44100 Hz 表示每秒采集 44100 个采样点。

### 2.3 奈奎斯特-香农采样定理

> **定理：** 要无失真地重建一个带限模拟信号，采样率必须**大于**信号最高频率的两倍。

数学表示：`f_s > 2 × f_max`。其中 `f_s` 是采样率，`f_max` 是信号中的最高频率分量。`2 × f_max` 也叫**奈奎斯特频率**。

**为什么 CD 音质是 44100 Hz？** 人耳听觉上限约 20 kHz → 奈奎斯特频率 = 2 × 20000 = 40000 Hz。再加上抗混叠滤波器的过渡带宽需求（约 10%），最终选定 44100 Hz。该数字还与早期 PCM 录音使用 NTSC/PAL 制式视频录像带有关。

### 2.4 欠采样与混叠（Aliasing）

当采样率不满足奈奎斯特条件时，会发生**混叠**——高频信号在数字化后"伪装"成低频信号。

```
原始: 15 kHz, 采样率: 20 kHz (不满足 2×15=30 kHz)
重建: 5 kHz (错误！混叠频率 = |20000-15000| = 5000 Hz)
```

**防止混叠：** ① 提高采样率确保 f_s > 2 × f_max；② 在 ADC 前用低通滤波器（Anti-aliasing Filter）切掉超过 f_s/2 的频率分量。

```cpp
// 演示混叠：用低采样率采样高频信号
#include <cmath>
#include <cstdio>
int main() {
    const float freq = 15000.0f;   // 15 kHz 原始信号
    const int sr = 20000;          // 20 kHz 采样率 (< 2×15kHz)
    printf("信号 %.0fHz, 采样率 %dHz, 奈奎斯特要求 %.0fHz\n", freq, sr, 2*freq);
    printf("混叠频率: %.0f Hz\n\n", std::abs(sr - freq));
    for (int i = 0; i < 10; ++i) {
        float t = float(i) / sr;
        float orig = std::sin(2.0f * 3.14159f * freq * t);
        float alias = std::sin(2.0f * 3.14159f * 5000.0f * t);
        printf("t=%.5f: original=%.4f, aliased_5kHz=%.4f %s\n",
               t, orig, alias, (std::abs(orig-alias)<0.001f) ? "✓" : "");
    }  // 两列数值几乎相同——15kHz 在 20kHz 采样下与 5kHz 不可区分
    return 0;
}
```

## 三、常见采样率与应用场景

### 3.1 采样率对照表

| 采样率 | 典型应用 | 频率范围 | 数据量（16-bit 立体声每秒） |
|--------|----------|----------|---------------------------|
| 8000 Hz | 电话语音 | 0-4 kHz | 32 KB/s |
| 22050 Hz | 低品质音效 | 0-11 kHz | 88 KB/s |
| 44100 Hz | CD 音质 | 0-20 kHz | 176 KB/s |
| 48000 Hz | 专业音频/视频 | 0-20 kHz | 192 KB/s |
| 96000 Hz | 高清音频 | 0-48 kHz | 384 KB/s |
| 192000 Hz | 录音棚母带 | 0-96 kHz | 768 KB/s |

### 3.2 KrKr2 的采样率选择

KrKr2 的 `WaveImpl.cpp` 第 62 行定义全局默认 44100 Hz，但渲染器实际输出 48000 Hz（`WaveMixer.cpp` 第 245 行）。这意味着 44.1 kHz 的音频源会被**重采样**到 48 kHz——一个有损但通常无感知差异的过程。详细源码分析见本节第六部分。

### 3.3 为什么 48000 Hz 成为事实标准？

- **视频兼容性：** 48000 是视频帧率（24/25/30/60 fps）的整数倍或简单分数倍
- **USB 音频：** 大多数 USB 声卡原生支持 48 kHz
- **系统默认：** Windows（WASAPI）、Android（AAudio/Oboe）都默认 48 kHz

## 四、从数字回到模拟：DAC

播放数字音频时需要进行反向转换——**DAC（Digital-to-Analog Conversion，数模转换）**。

```
数字信号 ──→ ① 解码 ──→ ② 插值重建 ──→ ③ 低通滤波 ──→ 模拟信号 ──→ 扬声器
              二进制→数值    离散→连续       去除阶梯噪声
```

DAC 的核心任务是将离散的采样点"连接"成连续的波形。理论上，如果采样满足奈奎斯特条件，通过理想的 sinc 插值可以**完美重建**原始信号。

完整的音频播放链路：

```
┌──────────┐    ┌────────────┐    ┌────────────┐    ┌─────────┐    ┌──────────┐
│ 音频文件  │───→│ 软件解码器  │───→│ 音频API     │───→│ DAC芯片  │───→│ 扬声器    │
│ (WAV/OGG)│    │ (FFmpeg)   │    │ (OpenAL)   │    │ (声卡)   │    │ (耳机)    │
└──────────┘    └────────────┘    └────────────┘    └─────────┘    └──────────┘
   磁盘存储        PCM数据           音频缓冲区         模拟信号        声波
```

> **KrKr2 的播放链路：** OGG/WAV 文件 → FFWaveDecoder（Vorbis/PCM 解码） → tTVPSoundBufferAL（OpenAL Buffer Queue） → OpenAL Soft（软件混音） → 操作系统音频 API → 声卡 DAC → 扬声器

## 五、动手实践：生成正弦波 WAV 文件

下面我们用纯 C++ 代码生成一个正弦波 WAV 文件。这是理解数字音频最直观的方式——你可以用任何音频播放器听到你亲手"制造"的声音。

### 5.1 完整代码：正弦波 WAV 生成器

> **提示：** WAV 文件格式的详细解析见本章第 4 节《PCM 数据格式详解》。此处只需理解 WAV = 44 字节文件头 + PCM 原始数据。

```cpp
// sine_wave_generator.cpp
// 编译: g++ -std=c++17 -o sine_wave sine_wave_generator.cpp -lm
#include <cmath>
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <vector>

#ifndef M_PI
#define M_PI 3.14159265358979323846
#endif

// WAV 文件头（44 字节，PCM 格式固定结构）
#pragma pack(push, 1)
struct WavHeader {
    char riff[4]{'R','I','F','F'}; uint32_t file_size;
    char wave[4]{'W','A','V','E'};
    char fmt[4]{'f','m','t',' '};  uint32_t fmt_size = 16;
    uint16_t audio_fmt = 1;        uint16_t channels;     // 1=PCM
    uint32_t sample_rate;          uint32_t byte_rate;
    uint16_t block_align;          uint16_t bits = 16;
    char data[4]{'d','a','t','a'}; uint32_t data_size;
};
#pragma pack(pop)

bool writeWav(const char* filename, const std::vector<int16_t>& samples,
              int sr, int ch) {
    FILE* fp = fopen(filename, "wb");
    if (!fp) return false;
    uint32_t dsz = samples.size() * sizeof(int16_t);
    WavHeader h;
    h.file_size = 36 + dsz;  h.channels = ch;  h.sample_rate = sr;
    h.byte_rate = sr * ch * 2;  h.block_align = ch * 2;  h.data_size = dsz;
    fwrite(&h, 44, 1, fp);
    fwrite(samples.data(), 2, samples.size(), fp);
    fclose(fp);
    printf("已生成: %s (%.1fKB, %dHz, %.1f秒)\n",
           filename, (44+dsz)/1024.f, sr, float(samples.size())/ch/sr);
    return true;
}

int main() {
    // === 参数设置 ===
    const int sample_rate = 44100;     // CD 音质采样率
    const int num_channels = 1;        // 单声道
    const float duration = 2.0f;       // 2 秒
    const float frequency = 440.0f;    // A4 音（标准音高）
    const float amplitude = 0.8f;      // 80% 音量（留 headroom 避免削波）

    int total_samples = static_cast<int>(sample_rate * duration);
    std::vector<int16_t> samples(total_samples);

    // === 生成正弦波 PCM 数据 ===
    for (int i = 0; i < total_samples; ++i) {
        float t = static_cast<float>(i) / sample_rate;
        // 生成 float 采样值 (-1.0 ~ 1.0)
        float sample = amplitude * std::sin(2.0f * M_PI * frequency * t);
        // 转换为 16-bit 有符号整数 (-32768 ~ 32767)
        samples[i] = static_cast<int16_t>(sample * 32767.0f);
    }

    // === 写入 WAV 文件 ===
    writeWav("sine_440hz.wav", samples, sample_rate, num_channels);

    // === 也可以生成不同频率用于对比 ===
    float freqs[] = {261.63f, 329.63f, 392.0f};  // C4, E4, G4 大三和弦
    const char* names[] = {"sine_C4.wav", "sine_E4.wav", "sine_G4.wav"};
    for (int f = 0; f < 3; ++f) {
        for (int i = 0; i < total_samples; ++i) {
            float t = static_cast<float>(i) / sample_rate;
            samples[i] = static_cast<int16_t>(
                amplitude * std::sin(2.0f * M_PI * freqs[f] * t) * 32767.0f);
        }
        writeWav(names[f], samples, sample_rate, num_channels);
    }
    printf("提示：用音频播放器打开 .wav 即可听到正弦波音调\n");
    return 0;
}
```

## 七、常见错误与解决方案

### 错误 1：生成的 WAV 文件有杂音/爆音

**原因：** 采样值超出范围导致**削波（Clipping）**。16-bit PCM 的有效范围是 -32768 ~ 32767，超出此范围的值会被截断。

```cpp
// 错误：amplitude 设为 1.0 时，sin 值可能精确等于 1.0
//       1.0 * 32768 = 32768 > 32767，溢出！
samples[i] = static_cast<int16_t>(1.0f * 32768.0f * std::sin(...));

// 正确：使用 32767.0f 并留出余量
samples[i] = static_cast<int16_t>(0.8f * 32767.0f * std::sin(...));
```

### 错误 2：WAV 文件大小不对 / 播放器无法识别

**原因：** WAV 文件头的 `file_size` 或 `data_size` 字段计算错误。

```cpp
// 错误：file_size 应该是文件总大小减去 RIFF 头的 8 字节
header.file_size = data_size;  // ❌ 漏了文件头的大小

// 正确：
header.file_size = 36 + data_size;  // ✅ 36 = 44 (总头大小) - 8 (RIFF标签+大小字段)
```

### 错误 3：采样率和频率搞混

**原因：** 把信号频率当作采样率，或反过来。

```cpp
// 错误：用 440 作为采样率？那每秒只有 440 个采样点
// 最大可表示频率 = 440/2 = 220 Hz，连人声都装不下
int sample_rate = 440;  // ❌ 这是音高频率，不是采样率！

// 正确：采样率至少 8000 Hz 以上
int sample_rate = 44100;   // ✅ CD 品质
float frequency = 440.0f;  // ✅ A4 音高
```

## 本节小结

- 声音是空气分子的疏密波，由频率（音高）、振幅（音量）、波形（音色）三要素决定
- ADC 将模拟信号数字化：采样（时间离散化）→ 量化（值离散化）→ 编码（二进制存储）
- **奈奎斯特定理**：采样率必须大于信号最高频率的 2 倍，否则产生混叠
- CD 音质 44100 Hz 覆盖人耳全频段，专业音频/视频领域多用 48000 Hz
- DAC 将数字信号还原为模拟波形，通过插值和滤波重建连续信号
- KrKr2 默认使用 48 kHz / 16-bit 的 PCM 格式，通过 OpenAL Soft 输出

## 练习题与答案

### 题目 1：计算 WAV 文件大小

一段 3 分钟的 CD 品质音频（44100 Hz、16-bit、立体声）未压缩 PCM 数据有多大？加上 WAV 文件头后总文件大小是多少？

<details>
<summary>查看答案</summary>

```
PCM 数据大小计算：
  每个采样点大小 = 16 bit / 8 = 2 bytes
  每帧大小（立体声）= 2 bytes × 2 channels = 4 bytes
  每秒数据量 = 4 bytes × 44100 = 176400 bytes ≈ 172.3 KB
  3 分钟 = 180 秒
  PCM 总大小 = 176400 × 180 = 31752000 bytes ≈ 30.28 MB

WAV 文件总大小：
  文件头 = 44 bytes
  总大小 = 44 + 31752000 = 31752044 bytes ≈ 30.28 MB
  （文件头相对于音频数据可忽略不计）
```

这就是为什么 CD 一张 700 MB 只能放约 80 分钟音乐（700 MB / 180 秒 × 60 ≈ 77 分钟）。

</details>

### 题目 2：判断混叠

某录音设备采样率为 32000 Hz，录制了一段包含 18000 Hz 高频泛音的小提琴演奏。请问：
1. 这个采样率能否正确记录 18000 Hz 的信号？
2. 如果不能，会产生什么频率的混叠？
3. 如何解决？

<details>
<summary>查看答案</summary>

1. **不能。** 奈奎斯特要求采样率 > 2 × 18000 = 36000 Hz，而 32000 Hz < 36000 Hz。

2. **混叠频率 = f_s - f_signal = 32000 - 18000 = 14000 Hz。** 原本 18 kHz 的泛音会被"折叠"成 14 kHz 的虚假信号，混入真实音频中无法去除。

3. **两种解决方案：**
   - 提高采样率到 ≥ 36001 Hz（例如 44100 Hz 或 48000 Hz）
   - 在录音设备的 ADC 前增加低通滤波器，截止频率设为 16000 Hz（f_s/2），在数字化之前就去掉 18 kHz 分量

</details>

### 题目 3：修改正弦波生成器

修改本节的正弦波生成器代码，生成一个从 200 Hz 线性扫频到 2000 Hz 的"啾啾"声（Chirp Signal），时长 3 秒，采样率 44100 Hz。

<details>
<summary>查看答案</summary>

复用本节正弦波生成器的 `WavHeader` 结构体和 `writeWav()` 函数，只需修改 `main()` 中的采样生成逻辑：

```cpp
// 在 main() 中替换正弦波生成循环为以下扫频逻辑
const float f_start = 200.0f;   // 起始频率
const float f_end = 2000.0f;    // 结束频率
const float duration = 3.0f;
const int total_samples = static_cast<int>(sample_rate * duration);
std::vector<int16_t> samples(total_samples);

// 线性扫频：瞬时频率 f(t) = f_start + (f_end - f_start) * t / duration
// 瞬时相位 = 2π × ∫f(t)dt = 2π × (f_start*t + (f_end-f_start)*t²/(2*duration))
for (int i = 0; i < total_samples; ++i) {
    float t = static_cast<float>(i) / sample_rate;
    float phase = 2.0f * M_PI * (f_start * t +
                  (f_end - f_start) * t * t / (2.0f * duration));
    samples[i] = static_cast<int16_t>(0.8f * std::sin(phase) * 32767.0f);
}

writeWav("chirp_200_2000.wav", samples, sample_rate, 1);
```

运行后会听到一个从低音到高音的"啾啾"声。核心数学：相位是瞬时频率对时间的积分。
