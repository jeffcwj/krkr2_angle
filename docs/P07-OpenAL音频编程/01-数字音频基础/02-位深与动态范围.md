## 一、什么是位深？

上一节讲了采样（时间轴上的离散化），本节讲**量化（值轴上的离散化）**。

位深（Bit Depth）决定了每个采样点用多少个二进制位来表示其数值。位数越多，能表示的数值越精细，音频质量越高。

```
模拟信号值: 0.0 ─────── 0.5 ─────── 1.0
             │            │            │
3-bit量化:   000  001  010  011  100  101  110  111
(8级)        │    │    │    │    │    │    │    │
             0   0.14 0.29 0.43 0.57 0.71 0.86 1.0

8-bit量化:   256 级 (00000000 ~ 11111111)
16-bit量化:  65536 级
24-bit量化:  16777216 级
```

**核心公式：**

| 量 | 公式 | 说明 |
|----|------|------|
| 量化级数 | 2^n | n 为位深 |
| 量化步长 | 满量程 / 2^n | 最小可分辨变化 |
| 动态范围 | ≈ 6.02 × n dB | 每增加 1 bit，动态范围增加约 6 dB |

## 二、量化过程详解

### 2.1 从连续到离散

ADC 采样得到的瞬时电压值是连续的（如 0.3728...V），但数字系统只能存储有限精度的数值。**量化**就是将连续值映射到最近的离散级。

```
          原始采样值: 0.3728
          ├─── 0.375 (最近的 8-bit 级) ← 量化后取这个值
          │
3-bit:    ├─── 0.286 (010)
          ├─── 0.429 (011)  ← 差距较大
          误差 = |0.3728 - 0.286| = 0.087 (大)

8-bit:    ├─── 0.3725 (01011111)
          ├─── 0.3765 (01100000)  ← 非常接近
          误差 = |0.3728 - 0.3725| = 0.0003 (小)
```

### 2.2 量化误差与量化噪声

量化误差 = 原始值 - 量化后的值。这个误差均匀分布在 `[-Δ/2, +Δ/2]` 之间（Δ 是量化步长），听起来像白噪声，称为**量化噪声（Quantization Noise）**。

```cpp
// 演示不同位深的量化误差
#include <cmath>
#include <cstdio>
#include <cstdint>

// 将 float (-1.0~1.0) 量化为指定位深的整数再还原回 float
float quantize(float value, int bits) {
    // 有符号整数范围: -(2^(n-1)) ~ (2^(n-1) - 1)
    float max_val = static_cast<float>((1 << (bits - 1)) - 1);
    int quantized = static_cast<int>(value * max_val);           // 量化
    return static_cast<float>(quantized) / max_val;              // 还原
}

int main() {
    float original = 0.3728f;
    printf("原始值: %.6f\n\n", original);
    printf("位深    量化级数      量化后       误差          SNR(理论)\n");
    printf("────────────────────────────────────────────────────────\n");

    int bits_list[] = {4, 8, 12, 16, 24};
    for (int b : bits_list) {
        float q = quantize(original, b);
        float err = std::abs(original - q);
        float snr = 6.02f * b + 1.76f;  // 理论 SNR 公式
        printf("%2d-bit   %8d     %.6f    %.6f     %.1f dB\n",
               b, 1 << b, q, err, snr);
    }
    return 0;
}
```

**预期输出（近似）：**

```
位深    量化级数      量化后       误差          SNR(理论)
────────────────────────────────────────────────────────
 4-bit         16     0.428571    0.055271     25.8 dB
 8-bit        256     0.370079    0.002721     49.9 dB
12-bit       4096     0.372803    0.000003     74.0 dB
16-bit      65536     0.372803    0.000000     98.1 dB
24-bit   16777216     0.372800    0.000000    146.2 dB
```

## 三、信噪比（SNR）与动态范围

### 3.1 理论 SNR 公式

对于 n-bit 均匀量化的正弦波信号，理论最大信噪比为：

```
SNR_max = 6.02 × n + 1.76  (dB)
```

| 位深 | SNR | 动态范围 | 等效描述 |
|------|-----|----------|----------|
| 8-bit | 49.9 dB | ~50 dB | 能听到量化噪声（"嘶嘶"声）|
| 16-bit | 98.1 dB | ~96 dB | CD 音质，噪声不可闻 |
| 24-bit | 146.2 dB | ~144 dB | 超过人耳极限（~120 dB） |
| 32-bit float | ~1528 dB | 理论无限 | 浮点精度，用于处理阶段 |

### 3.2 动态范围的直觉理解

**动态范围 = 最大音量与最小可辨音量之比（dB）。**

```
                           8-bit (~50 dB)
安静房间    ├────────────────────┤
                  16-bit (~96 dB)
呼吸声      ├────────────────────────────────────────┤
                         24-bit (~144 dB)
           ├────────────────────────────────────────────────────────────┤
           │                                                           │
     针落地声(10dB)                                           摇滚演唱会(120dB)
```

- **8-bit**：50 dB 动态范围，安静段能听到明显的量化噪声底噪
- **16-bit**：96 dB，足以覆盖从极安静到极响亮的全部范围
- **24-bit**：144 dB，超过人耳物理极限，用于录音/混音保留余量

## 四、常见位深格式对比

### 4.1 整数 PCM 格式

| 格式 | 范围 | 存储方式 | 典型用途 |
|------|------|----------|----------|
| uint8（无符号 8-bit） | 0 ~ 255，静音=128 | 1 byte | WAV 8-bit、老游戏音效 |
| int16（有符号 16-bit） | -32768 ~ 32767 | 2 bytes，小端序 | CD、游戏、**KrKr2 默认** |
| int24（有符号 24-bit） | -8388608 ~ 8388607 | 3 bytes，小端序 | 专业录音 |
| int32（有符号 32-bit） | -2^31 ~ 2^31-1 | 4 bytes | 音频处理中间格式 |

> **注意 8-bit 是无符号的！** 这是历史遗留——WAV 规范中 8-bit PCM 使用 unsigned，静音值是 128 而非 0。16-bit 及以上使用 signed，静音值是 0。

```cpp
// 8-bit 无符号 vs 16-bit 有符号的转换
#include <cstdint>

// 8-bit unsigned (0~255, 静音=128) → float (-1.0~1.0)
float uint8_to_float(uint8_t sample) {
    return (static_cast<float>(sample) - 128.0f) / 128.0f;
}

// 16-bit signed (-32768~32767, 静音=0) → float (-1.0~1.0)
float int16_to_float(int16_t sample) {
    return static_cast<float>(sample) / 32768.0f;
}

// float → 16-bit signed (带削波保护)
int16_t float_to_int16(float sample) {
    if (sample > 1.0f) sample = 1.0f;     // 防止正向削波
    if (sample < -1.0f) sample = -1.0f;   // 防止负向削波
    return static_cast<int16_t>(sample * 32767.0f);
}
```

### 4.2 浮点 PCM 格式

| 格式 | 范围 | 精度 | 用途 |
|------|------|------|------|
| float32 | ±3.4×10^38 | 24-bit 尾数 | 音频处理、DAW 内部 |
| float64 | ±1.8×10^308 | 53-bit 尾数 | 高精度科学计算 |

浮点 PCM 的优势：
- **不会溢出**：信号超过 1.0 不会立即削波，后续可以再缩放回来
- **处理精度高**：多次运算（混音、滤波、效果器）不累积量化误差
- **动态范围极大**：理论上无限（实际受浮点精度限制）

浮点的劣势：
- **体积大**：float32 占 4 字节/采样（比 int16 大一倍）
- **兼容性**：并非所有硬件/API 原生支持（OpenAL 基础规范只支持整数）

```cpp
// 浮点 PCM 的超量程处理：混音后可能超过 1.0
float track_a = 0.8f;   // 音轨 A
float track_b = 0.7f;   // 音轨 B
float mixed = track_a + track_b;  // = 1.5，超过 1.0！

// 如果用 int16，直接溢出产生噪声
// 如果用 float32，可以保持 1.5 不失真，最后再归一化：
float normalized = mixed / 2.0f;  // = 0.75，安全范围内
```

## 五、Dithering（抖动）：降低位深时的必备技术

当把高位深音频转换为低位深（如 24-bit → 16-bit）时，直接截断低位会产生**谐波失真**。Dithering 通过在截断前加入微小的随机噪声，将谐波失真转化为均匀的白噪声，听感更自然。

```cpp
// 24-bit → 16-bit 转换（带 Dithering）
#include <cstdint>
#include <cstdlib>
#include <cmath>

// 三角概率密度分布（TPDF）Dithering：两个均匀随机数之和
float tpdf_dither() {
    float r1 = static_cast<float>(rand()) / RAND_MAX - 0.5f;  // -0.5~0.5
    float r2 = static_cast<float>(rand()) / RAND_MAX - 0.5f;
    return r1 + r2;  // 三角分布 -1.0~1.0
}

int16_t convert_24bit_to_16bit(int32_t sample_24bit) {
    // 24-bit 有效范围: -8388608 ~ 8388607
    // 需要右移 8 位变成 16-bit
    float normalized = static_cast<float>(sample_24bit) / 8388608.0f;
    float dithered = normalized + tpdf_dither() / 32768.0f;  // 加入 1 LSB 级别的抖动
    return static_cast<int16_t>(std::round(dithered * 32767.0f));
}
```

## 六、动手实践：位深对比听音测试

```cpp
// bit_depth_comparison.cpp — 生成不同位深的同一正弦波用于对比
#include <cmath>
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <vector>

#ifndef M_PI
#define M_PI 3.14159265358979323846
#endif

#pragma pack(push, 1)
struct WavHeader {
    char riff[4]{'R','I','F','F'}; uint32_t file_size;
    char wave[4]{'W','A','V','E'};
    char fmt[4]{'f','m','t',' '};  uint32_t fmt_size = 16;
    uint16_t audio_fmt = 1;        uint16_t channels = 1;
    uint32_t sample_rate;          uint32_t byte_rate;
    uint16_t block_align;          uint16_t bits;
    char data[4]{'d','a','t','a'}; uint32_t data_size;
};
#pragma pack(pop)

int main() {
    const int sr = 44100;
    const float duration = 2.0f;
    const float freq = 440.0f;
    const int n = static_cast<int>(sr * duration);

    // 先生成 float 精度的原始数据
    std::vector<float> original(n);
    for (int i = 0; i < n; ++i) {
        float t = float(i) / sr;
        original[i] = 0.5f * std::sin(2.0f * M_PI * freq * t);
    }

    // 模拟不同位深量化后输出为 16-bit WAV
    int bit_depths[] = {4, 8, 12, 16};
    for (int bits : bit_depths) {
        std::vector<int16_t> samples(n);
        int levels = 1 << (bits - 1);  // 有符号正半轴级数
        for (int i = 0; i < n; ++i) {
            // 量化到指定位深再还原
            int q = static_cast<int>(original[i] * levels);
            float restored = float(q) / levels;
            samples[i] = static_cast<int16_t>(restored * 32767.0f);
        }
        char fname[64];
        snprintf(fname, sizeof(fname), "sine_%dbit.wav", bits);
        uint32_t dsz = n * 2;
        WavHeader h;
        h.file_size = 36 + dsz; h.sample_rate = sr;
        h.byte_rate = sr * 2;   h.block_align = 2;
        h.bits = 16;             h.data_size = dsz;  // 输出始终为 16-bit WAV
        FILE* fp = fopen(fname, "wb");
        fwrite(&h, 44, 1, fp);
        fwrite(samples.data(), 2, n, fp);
        fclose(fp);
        printf("已生成 %s (模拟 %d-bit 量化)\n", fname, bits);
    }
    printf("\n用播放器对比：4-bit 有明显阶梯噪声，16-bit 几乎完美\n");
    return 0;
}
```

## 八、常见错误与解决方案

### 错误 1：8-bit 和 16-bit 的符号规则搞反

```cpp
// ❌ 错误：8-bit WAV 用 signed 解析
int8_t sample_8bit = buffer[i];               // 静音时得到 -128！
float f = static_cast<float>(sample_8bit) / 128.0f;  // 静音不是 0.0

// ✅ 正确：8-bit WAV 是 unsigned，静音=128
uint8_t sample_8bit = buffer[i];
float f = (static_cast<float>(sample_8bit) - 128.0f) / 128.0f;  // 静音=0.0
```

### 错误 2：位深转换时精度丢失

```cpp
// ❌ 错误：16-bit → 8-bit 直接截断高位
uint8_t bad = static_cast<uint8_t>(sample_16bit >> 8);  // 丢失低 8 位

// ✅ 正确：先归一化到 float，再映射到目标范围
float f = static_cast<float>(sample_16bit) / 32768.0f;
uint8_t good = static_cast<uint8_t>((f + 1.0f) * 127.5f);  // -1~1 → 0~255
```

### 错误 3：混音后整数溢出

```cpp
// ❌ 两个 int16 直接相加可能溢出
int16_t a = 20000, b = 25000;
int16_t sum = a + b;  // 45000 > 32767，溢出为负数！

// ✅ 先转 int32 或 float 再混合
int32_t mix = static_cast<int32_t>(a) + static_cast<int32_t>(b);
if (mix > 32767) mix = 32767;    // 硬限幅（Hard Clipping）
if (mix < -32768) mix = -32768;
int16_t result = static_cast<int16_t>(mix);
```

## 本节小结

- 位深决定每个采样点的精度，n-bit 有 2^n 个量化级
- 理论 SNR = 6.02n + 1.76 dB，每增加 1 bit 动态范围增加约 6 dB
- 8-bit PCM 是无符号（0-255，静音=128），16-bit 及以上是有符号（静音=0）
- 整数 PCM 适合存储和传输，浮点 PCM 适合处理和混音
- Dithering 可以改善降位深时的听感质量
- KrKr2 使用 16-bit 整数 PCM，OpenAL 基础规范不支持浮点格式

### 题目 1：计算动态范围

一个 12-bit 的 ADC 芯片，其理论最大信噪比和动态范围是多少 dB？如果录制古典音乐（动态范围约 60 dB），这个 ADC 是否够用？

<details>
<summary>查看答案</summary>

```
SNR = 6.02 × 12 + 1.76 = 74.0 dB
动态范围 ≈ 6.02 × 12 = 72.24 dB

古典音乐动态范围约 60 dB，12-bit ADC 提供 72 dB，有约 12 dB 余量，够用。
但现代专业录音通常使用 24-bit (144 dB) 以获得更大的安全余量。
```

</details>

### 题目 2：格式转换代码

编写一个函数 `convert_stereo_16bit_to_mono_8bit()`，将 16-bit 有符号立体声 PCM 数据转换为 8-bit 无符号单声道 PCM。要求：左右声道取平均。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdint>
#include <vector>

// 输入：16-bit 立体声交错数据 [L0, R0, L1, R1, ...]
// 输出：8-bit 单声道数据
std::vector<uint8_t> convert_stereo_16bit_to_mono_8bit(
    const std::vector<int16_t>& stereo) {
    size_t frame_count = stereo.size() / 2;  // 每帧 2 个采样 (L+R)
    std::vector<uint8_t> mono(frame_count);

    for (size_t i = 0; i < frame_count; ++i) {
        int16_t left = stereo[i * 2];
        int16_t right = stereo[i * 2 + 1];
        // 1. 左右声道平均（用 int32 防溢出）
        int32_t avg = (static_cast<int32_t>(left) + right) / 2;
        // 2. 16-bit signed → float
        float f = static_cast<float>(avg) / 32768.0f;  // -1.0 ~ 1.0
        // 3. float → 8-bit unsigned (0~255, 静音=128)
        int val = static_cast<int>((f + 1.0f) * 127.5f);
        if (val < 0) val = 0;
        if (val > 255) val = 255;
        mono[i] = static_cast<uint8_t>(val);
    }
    return mono;
}
```

</details>

### 题目 3：为什么不用 32-bit 整数做默认格式？

从存储效率、硬件支持、人耳感知三个角度，解释为什么 CD 和游戏引擎（如 KrKr2）选择 16-bit 而非 32-bit 整数 PCM。

<details>
<summary>查看答案</summary>

1. **存储效率**：32-bit 每秒数据量是 16-bit 的两倍（44100Hz 立体声：16-bit = 176 KB/s，32-bit = 352 KB/s），CD 容量直接减半。

2. **硬件支持**：大多数 DAC 芯片、音频接口和 API（包括 OpenAL 基础规范）原生支持 16-bit，32-bit 支持有限或需要软件转换。

3. **人耳感知**：16-bit 提供 96 dB 动态范围，已超过大多数听音环境的需求（普通房间环境噪声约 30 dB，16-bit 可以覆盖从 30 dB 到 126 dB 的范围）。人耳的瞬态动态范围约 120 dB，16-bit 已非常接近这个极限。

综上，16-bit 是存储效率、兼容性和音质的最佳平衡点。

</details>
