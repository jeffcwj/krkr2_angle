## 一、声道（Channel）的基本概念

### 1.1 什么是声道？

声道是独立的音频信号通路。每个声道对应一个扬声器位置，多个声道组合形成空间音场。

| 声道配置 | 声道数 | 说明 |
|----------|--------|------|
| Mono（单声道） | 1 | 所有扬声器播放同一信号，无方向感 |
| Stereo（立体声） | 2 | 左(L) + 右(R)，有左右方向感 |
| 2.1 | 3 | L + R + 低音炮(LFE) |
| 5.1 环绕声 | 6 | FL + FR + C + LFE + SL + SR |
| 7.1 环绕声 | 8 | 5.1 + 后左(BL) + 后右(BR) |

### 1.2 5.1 环绕声布局详解

```
                 中央 (C)
                   ●
                  ╱ ╲
        前左 (FL)●   ●前右 (FR)
                 │   │
                 │ 🧑 │    ← 听者位置
                 │   │
       侧左 (SL)●   ●侧右 (SR)
                  ╲ ╱
                   ●
              低音炮 (LFE)
              （位置任意）
```

各声道的频率分工：

| 声道 | 缩写 | 频率范围 | 内容 |
|------|------|----------|------|
| 前左/前右 | FL/FR | 全频段 | 主要音乐、音效 |
| 中央 | C | 全频段 | 人声对白、中央定位 |
| 低音炮 | LFE | 20-120 Hz | 超低频效果（爆炸、雷声） |
| 侧左/侧右 | SL/SR | 全频段 | 环境音效、空间感 |

> **".1" 的含义：** LFE 声道只传输低频（20-120 Hz），带宽仅为全频段声道的 1/10，因此计为 0.1 而非 1。

### 1.3 声道布局标识

不同标准使用不同的声道标识方式：

```cpp
// WAV 文件的声道掩码 (WAVEFORMATEXTENSIBLE)
#define SPEAKER_FRONT_LEFT          0x00000001
#define SPEAKER_FRONT_RIGHT         0x00000002
#define SPEAKER_FRONT_CENTER        0x00000004
#define SPEAKER_LOW_FREQUENCY       0x00000008
#define SPEAKER_BACK_LEFT           0x00000010
#define SPEAKER_BACK_RIGHT          0x00000020
#define SPEAKER_SIDE_LEFT           0x00000200
#define SPEAKER_SIDE_RIGHT          0x00000400

// 常见组合
const uint32_t STEREO_MASK = 0x03;        // FL + FR
const uint32_t SURROUND_51 = 0x3F;        // FL+FR+C+LFE+BL+BR
const uint32_t SURROUND_71 = 0x63F;       // 5.1 + SL + SR
```

## 二、交错存储与平面存储

### 2.1 交错存储（Interleaved）

交错存储是最常见的格式——所有声道的采样**按帧交替排列**。一帧（Frame）包含同一时刻所有声道各一个采样点。

```
立体声交错存储（每个方块是一个采样值）：
  帧0         帧1         帧2         帧3
┌────┬────┐┌────┬────┐┌────┬────┐┌────┬────┐
│ L0 │ R0 ││ L1 │ R1 ││ L2 │ R2 ││ L3 │ R3 │
└────┴────┘└────┴────┘└────┴────┘└────┴────┘
内存地址 →

5.1 环绕声交错存储（每帧 6 个采样）：
┌────┬────┬────┬────┬────┬────┐┌────┬────┬...
│ FL │ FR │ C  │LFE│ SL │ SR ││ FL │ FR │
└────┴────┴────┴────┴────┴────┘└────┴────┴...
```

**优势：** 按帧读取时内存连续，对实时播放友好（DAC 逐帧输出）。

### 2.2 平面存储（Planar / Non-interleaved）

平面存储将每个声道的数据**连续存放**在各自的缓冲区中。

```
立体声平面存储：
Buffer 0 (左声道):  [L0] [L1] [L2] [L3] [L4] ...
Buffer 1 (右声道):  [R0] [R1] [R2] [R3] [R4] ...
```

**优势：** 对 DSP 处理友好（SIMD 可以一次处理一整个声道的数据）。FFmpeg 内部广泛使用平面格式（如 `AV_SAMPLE_FMT_FLTP`）。

### 2.3 格式对比与转换

```
交错 vs 平面对比：
┌─────────────┬─────────────────────────┬──────────────────────┐
│             │ 交错 (Interleaved)       │ 平面 (Planar)        │
├─────────────┼─────────────────────────┼──────────────────────┤
│ 内存布局     │ LRLRLRLR...             │ LLLL... | RRRR...    │
│ 按帧访问     │ ✅ 直接索引              │ ❌ 需跨缓冲区        │
│ 按声道处理   │ ❌ 步长跳跃              │ ✅ 连续内存           │
│ 实时播放     │ ✅ DAC 友好              │ ❌ 需先交错化         │
│ SIMD 处理   │ ❌ 需解交错              │ ✅ 向量化友好         │
│ 典型使用者   │ WAV、OpenAL、ALSA       │ FFmpeg、AAudio       │
└─────────────┴─────────────────────────┴──────────────────────┘
```

```cpp
// 交错转平面（立体声 16-bit）
#include <cstdint>
#include <vector>

void interleaved_to_planar(const std::vector<int16_t>& interleaved,
                           std::vector<int16_t>& left,
                           std::vector<int16_t>& right) {
    size_t frames = interleaved.size() / 2;
    left.resize(frames);
    right.resize(frames);
    for (size_t i = 0; i < frames; ++i) {
        left[i] = interleaved[i * 2];       // 偶数位置 = 左声道
        right[i] = interleaved[i * 2 + 1];  // 奇数位置 = 右声道
    }
}

// 平面转交错
void planar_to_interleaved(const std::vector<int16_t>& left,
                           const std::vector<int16_t>& right,
                           std::vector<int16_t>& interleaved) {
    size_t frames = left.size();
    interleaved.resize(frames * 2);
    for (size_t i = 0; i < frames; ++i) {
        interleaved[i * 2] = left[i];
        interleaved[i * 2 + 1] = right[i];
    }
}
```

## 三、比特率计算

### 3.1 未压缩 PCM 比特率公式

```
比特率 (bps) = 采样率 × 位深 × 声道数
```

| 配置 | 计算 | 比特率 | 每秒数据量 |
|------|------|--------|-----------|
| 电话语音 (8kHz/8bit/Mono) | 8000×8×1 | 64 kbps | 8 KB/s |
| CD (44.1kHz/16bit/Stereo) | 44100×16×2 | 1411.2 kbps | 176.4 KB/s |
| DVD (48kHz/24bit/5.1) | 48000×24×6 | 6912 kbps | 864 KB/s |
| Hi-Res (96kHz/24bit/Stereo) | 96000×24×2 | 4608 kbps | 576 KB/s |
| KrKr2 默认 (48kHz/16bit/Stereo) | 48000×16×2 | 1536 kbps | 192 KB/s |

### 3.2 文件大小计算

```
文件大小 = 文件头大小 + (比特率 / 8) × 时长(秒)
```

```cpp
// 计算 PCM 文件大小的实用函数
#include <cstdio>
#include <cstdint>

struct PCMConfig {
    uint32_t sample_rate;
    uint16_t bits_per_sample;
    uint16_t channels;
};

void printPCMInfo(const PCMConfig& cfg, float duration_sec) {
    uint64_t bitrate = (uint64_t)cfg.sample_rate * cfg.bits_per_sample * cfg.channels;
    uint64_t bytes_per_sec = bitrate / 8;
    uint64_t total_bytes = bytes_per_sec * static_cast<uint64_t>(duration_sec);
    uint64_t wav_size = total_bytes + 44;  // WAV 文件头 44 字节

    printf("配置: %u Hz / %u-bit / %u ch\n",
           cfg.sample_rate, cfg.bits_per_sample, cfg.channels);
    printf("比特率: %.1f kbps (%.1f KB/s)\n",
           bitrate / 1000.0, bytes_per_sec / 1024.0);
    printf("%.0f秒 PCM: %.2f MB, WAV: %.2f MB\n\n",
           duration_sec, total_bytes / 1048576.0, wav_size / 1048576.0);
}

int main() {
    // CD 音质 5 分钟
    printPCMInfo({44100, 16, 2}, 300.0f);
    // KrKr2 默认 5 分钟
    printPCMInfo({48000, 16, 2}, 300.0f);
    // DVD 5.1 环绕声 5 分钟
    printPCMInfo({48000, 24, 6}, 300.0f);
    return 0;
}
```

### 3.3 压缩音频的比特率

对比压缩格式，理解为什么游戏引擎使用 Ogg Vorbis 而非 WAV：

| 格式 | 比特率 | 5分钟文件大小 | 压缩比 |
|------|--------|-------------|--------|
| WAV (CD) | 1411 kbps | 51.5 MB | 1:1 |
| FLAC (无损) | ~800 kbps | 29 MB | 1.8:1 |
| Ogg Vorbis (q5) | ~160 kbps | 5.9 MB | 8.8:1 |
| MP3 (128kbps) | 128 kbps | 4.7 MB | 11:1 |
| Opus (高质量) | 128 kbps | 4.7 MB | 11:1 |

> **KrKr2 的选择：** BGM 使用 Ogg Vorbis（高压缩比、开源免版税），短音效可用 WAV（无解码延迟）。

## 四、OpenAL 的声道限制

### 4.1 OpenAL 基础规范支持的格式

OpenAL 1.1 基础规范**仅支持 Mono 和 Stereo**：

```cpp
// OpenAL 定义的 4 种格式
AL_FORMAT_MONO8      // 8-bit 单声道
AL_FORMAT_MONO16     // 16-bit 单声道
AL_FORMAT_STEREO8    // 8-bit 立体声
AL_FORMAT_STEREO16   // 16-bit 立体声
```

**为什么只支持单声道和立体声？**

OpenAL 的设计哲学是**3D 空间音频**：每个 Source（声源）是空间中的一个点，由 OpenAL 根据 Source/Listener 的位置关系自动计算声道分配。因此：
- **Mono**：用于 3D 定位声源（可以自由放置在 3D 空间中）
- **Stereo**：用于背景音乐等不需要 3D 定位的音频（直接输出到左右声道）

## 五、动手实践：立体声声像扫描

生成一个从左到右移动的声音（Pan Sweep），直观感受声道的作用：

```cpp
// pan_sweep.cpp — 生成左右声像扫描的立体声 WAV
#include <cmath>
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <vector>

#ifndef M_PI
#define M_PI 3.14159265358979323846
#endif

#pragma pack(push, 1)
struct WavHeader {
    char riff[4]{'R','I','F','F'}; uint32_t file_size;
    char wave[4]{'W','A','V','E'};
    char fmt[4]{'f','m','t',' '};  uint32_t fmt_size = 16;
    uint16_t audio_fmt = 1;        uint16_t channels = 2;  // 立体声
    uint32_t sample_rate = 44100;  uint32_t byte_rate = 44100*4;
    uint16_t block_align = 4;      uint16_t bits = 16;
    char data[4]{'d','a','t','a'}; uint32_t data_size;
};
#pragma pack(pop)

int main() {
    const int sr = 44100;
    const float duration = 4.0f;         // 4 秒
    const float freq = 440.0f;
    const int frames = sr * (int)duration;

    std::vector<int16_t> stereo(frames * 2);  // 交错存储: L,R,L,R,...

    for (int i = 0; i < frames; ++i) {
        float t = float(i) / sr;
        float mono = 0.7f * std::sin(2.0f * M_PI * freq * t);

        // Pan 从 -1 (全左) 线性扫到 +1 (全右)
        float pan = -1.0f + 2.0f * t / duration;  // -1.0 → +1.0

        // 等功率声像法（Equal Power Panning）
        float angle = (pan + 1.0f) * M_PI / 4.0f;  // 0 ~ π/2
        float gain_L = std::cos(angle);   // 左声道增益
        float gain_R = std::sin(angle);   // 右声道增益

        stereo[i * 2]     = int16_t(mono * gain_L * 32767.0f);  // L
        stereo[i * 2 + 1] = int16_t(mono * gain_R * 32767.0f);  // R
    }

    uint32_t dsz = frames * 4;  // 2 channels * 2 bytes
    WavHeader h; h.file_size = 36 + dsz; h.data_size = dsz;
    FILE* fp = fopen("pan_sweep.wav", "wb");
    fwrite(&h, 44, 1, fp);
    fwrite(stereo.data(), 2, frames * 2, fp);
    fclose(fp);
    printf("已生成 pan_sweep.wav (%.1f KB, 声像从左到右扫描)\n",
           (44+dsz)/1024.0f);
    printf("用耳机听效果最佳！\n");
    return 0;
}
```

> **等功率声像法（Equal Power Panning）** 使用 cos/sin 曲线而非线性比例，因为两个扬声器同时播放时，人耳感知到的总音量 ∝ √(L² + R²)。cos²+sin²=1 保证了扫描过程中总功率恒定。

## 六、常见错误与解决方案

### 错误 1：立体声帧数算错

```cpp
// ❌ 错误：把总采样数当帧数
int total_samples = buffer.size();  // 立体声时这是帧数的 2 倍！
for (int i = 0; i < total_samples; ++i) {
    process_frame(buffer[i*2], buffer[i*2+1]);  // 越界！
}

// ✅ 正确：帧数 = 总采样数 / 声道数
int frames = buffer.size() / num_channels;
for (int i = 0; i < frames; ++i) {
    process_frame(buffer[i*2], buffer[i*2+1]);
}
```

### 错误 2：线性声像导致中间位置变小声

```cpp
// ❌ 线性声像：center 时 L=R=0.5，总功率 = 0.5²+0.5² = 0.5（-3dB）
float gain_L = (1.0f - pan) / 2.0f;
float gain_R = (1.0f + pan) / 2.0f;

// ✅ 等功率声像：center 时总功率 = cos²(π/4)+sin²(π/4) = 1.0
float angle = (pan + 1.0f) * M_PI / 4.0f;
float gain_L = std::cos(angle);
float gain_R = std::sin(angle);
```

## 本节小结

- 声道数决定空间音频的维度：Mono(1)、Stereo(2)、5.1(6)、7.1(8)
- 交错存储（LRLRLR）适合实时播放，平面存储（LLL...RRR...）适合 DSP 处理
- PCM 比特率 = 采样率 × 位深 × 声道数，CD 音质约 1411 kbps
- OpenAL 基础规范仅支持 Mono/Stereo，Mono 用于 3D 定位声源
- KrKr2 只使用 Mono 和 Stereo 两种格式

### 题目 2：声道格式转换

编写函数将 5.1 环绕声（6声道交错 16-bit）降混为立体声（2声道交错 16-bit）。使用 ITU-R BS.775 标准公式：`L = FL + 0.707*C + 0.707*SL`，`R = FR + 0.707*C + 0.707*SR`。

<details>
<summary>查看答案</summary>

```cpp
#include <cstdint>
#include <cmath>
#include <vector>

// 5.1 声道顺序: FL, FR, C, LFE, SL, SR
// 输出: L, R (交错)
std::vector<int16_t> downmix_51_to_stereo(const std::vector<int16_t>& surround) {
    size_t frames = surround.size() / 6;
    std::vector<int16_t> stereo(frames * 2);
    const float k = 0.707f;  // ≈ 1/√2 ≈ -3dB

    for (size_t i = 0; i < frames; ++i) {
        float FL  = surround[i*6 + 0] / 32768.0f;
        float FR  = surround[i*6 + 1] / 32768.0f;
        float C   = surround[i*6 + 2] / 32768.0f;
        // LFE (i*6+3) 通常丢弃或低通滤波后混入
        float SL  = surround[i*6 + 4] / 32768.0f;
        float SR  = surround[i*6 + 5] / 32768.0f;

        float L = FL + k * C + k * SL;
        float R = FR + k * C + k * SR;

        // 限幅防溢出
        L = std::fmax(-1.0f, std::fmin(1.0f, L));
        R = std::fmax(-1.0f, std::fmin(1.0f, R));

        stereo[i*2]     = static_cast<int16_t>(L * 32767.0f);
        stereo[i*2 + 1] = static_cast<int16_t>(R * 32767.0f);
    }
    return stereo;
}
```

</details>
