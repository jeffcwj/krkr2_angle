# 坐标系与空间音频

> **所属模块：** P07-OpenAL音频编程 | **前置知识：** [Device-Context-Source-Buffer](02-Device-Context-Source-Buffer.md) | **阅读时间：** 25 分钟

## 本节目标

读完本节后，你将能够：

1. 理解 OpenAL 的右手坐标系及 Listener/Source 的 3D 定位
2. 使用六种距离衰减模型控制音量随距离变化的行为

## OpenAL 坐标系

OpenAL 使用**右手坐标系**，与 OpenGL 默认一致：

```
     +Y（上）
      |
      |
      +---→ +X（右）
     /
    /
   +Z（朝向屏幕外/观察者）
```

所有位置、速度、方向向量都在此坐标系中表示。单位由应用自定义（通常 1 单位 = 1 米）。

## Listener（听者）

每个 Context 有且仅有一个 Listener，代表"耳朵"的位置。Listener 不需要创建/销毁，直接设置属性：

```cpp
// 位置：世界坐标中的听者位置
float listenerPos[] = {0.0f, 0.0f, 0.0f};
alListenerfv(AL_POSITION, listenerPos);

// 速度：用于多普勒效应计算
float listenerVel[] = {0.0f, 0.0f, 0.0f};
alListenerfv(AL_VELOCITY, listenerVel);

// 朝向：6 个浮点数 = forward(3) + up(3)
float listenerOri[] = {
    0.0f, 0.0f, -1.0f,  // forward：面朝 -Z（屏幕内）
    0.0f, 1.0f,  0.0f   // up：头顶朝 +Y
};
alListenerfv(AL_ORIENTATION, listenerOri);

// 全局增益（主音量）
alListenerf(AL_GAIN, 1.0f);  // 0.0=静音，1.0=正常
```

| 属性 | 函数 | 维度 | 默认值 | 说明 |
|------|------|------|--------|------|
| `AL_POSITION` | `alListenerfv` | 3 | {0,0,0} | 听者位置 |
| `AL_VELOCITY` | `alListenerfv` | 3 | {0,0,0} | 听者速度（多普勒） |
| `AL_ORIENTATION` | `alListenerfv` | 6 | {0,0,-1, 0,1,0} | forward + up 向量 |
| `AL_GAIN` | `alListenerf` | 1 | 1.0 | 主音量 |

## Source 的 3D 定位

Source 的位置决定了声音从哪个方向传来、音量多大：

```cpp
ALuint source;
alGenSources(1, &source);

// 设置 Source 在听者右前方 5 米处
float srcPos[] = {3.0f, 0.0f, -4.0f};  // x=3, z=-4
alSourcefv(source, AL_POSITION, srcPos);

// Source 速度（用于多普勒效应）
float srcVel[] = {0.0f, 0.0f, 0.0f};
alSourcefv(source, AL_VELOCITY, srcVel);
```

### AL_SOURCE_RELATIVE

默认情况下 Source 位置是世界坐标。设置 `AL_SOURCE_RELATIVE = AL_TRUE` 后，位置变为**相对于 Listener 的偏移**：

```cpp
// 相对定位：始终在听者正前方（不随听者朝向旋转）
alSourcei(source, AL_SOURCE_RELATIVE, AL_TRUE);
float relPos[] = {0.0f, 0.0f, -1.0f};  // Listener 前方 1 米
alSourcefv(source, AL_POSITION, relPos);
```

> **KrKr2 用法：** 视觉小说通常不需要 3D 空间感，KrKr2 将所有 Source 设为 `AL_SOURCE_RELATIVE`，用 x 分量模拟左右声像（pan）。

### 示例 1：3D 环绕音效

```cpp
// surround_demo.cpp — Source 绕 Listener 旋转
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <cmath>
#include <vector>
#include <thread>
#include <chrono>

int main() {
    ALCdevice* dev = alcOpenDevice(nullptr);
    ALCcontext* ctx = alcCreateContext(dev, nullptr);
    alcMakeContextCurrent(ctx);

    // Listener 在原点，面朝 -Z
    float ori[] = {0,0,-1, 0,1,0};
    alListenerfv(AL_ORIENTATION, ori);

    // 生成 2 秒 440Hz 正弦波
    constexpr int kRate = 44100, kN = kRate * 2;
    std::vector<int16_t> pcm(kN);
    for (int i = 0; i < kN; ++i)
        pcm[i] = int16_t(0.4f * 32767 * std::sin(6.2832f * 440.f * i / kRate));

    ALuint buf, src;
    alGenBuffers(1, &buf);
    alBufferData(buf, AL_FORMAT_MONO16, pcm.data(), kN * 2, kRate);
    alGenSources(1, &src);
    alSourcei(src, AL_BUFFER, buf);
    alSourcei(src, AL_LOOPING, AL_TRUE);

    alSourcePlay(src);
    std::puts("Source 绕听者旋转（半径 3 米）...");

    // 每 50ms 更新位置，绕 Y 轴旋转
    for (int step = 0; step < 120; ++step) {  // 6 秒
        float angle = step * 0.05f * 6.2832f;  // 每秒约 1 圈
        float pos[] = {3.0f * std::sin(angle), 0.0f, 3.0f * std::cos(angle)};
        alSourcefv(src, AL_POSITION, pos);
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
    }

    alSourceStop(src);
    alDeleteSources(1, &src); alDeleteBuffers(1, &buf);
    alcMakeContextCurrent(nullptr); alcDestroyContext(ctx); alcCloseDevice(dev);
}
```

运行时戴上耳机，能感受到声音从左到右再到后方不断旋转。

## 距离衰减模型

距离模型决定音量如何随 Source 与 Listener 间距离变化。通过 `alDistanceModel()` 设置：

```cpp
alDistanceModel(AL_INVERSE_DISTANCE_CLAMPED);  // 默认
```

### 六种距离模型对比

| 模型 | 公式 | 特点 |
|------|------|------|
| `AL_NONE` | gain = 1 | 无衰减，任何距离音量不变 |
| `AL_INVERSE_DISTANCE` | gain = ref / (ref + rolloff × (dist - ref)) | 不截断，可能负值 |
| `AL_INVERSE_DISTANCE_CLAMPED` | 同上，dist 截断到 [ref, max] | **默认**，最常用 |
| `AL_LINEAR_DISTANCE` | gain = 1 - rolloff × (dist - ref) / (max - ref) | 线性衰减 |
| `AL_LINEAR_DISTANCE_CLAMPED` | 同上，dist 截断 | 平滑截断 |
| `AL_EXPONENT_DISTANCE` | gain = (dist / ref) ^ (-rolloff) | 指数衰减 |
| `AL_EXPONENT_DISTANCE_CLAMPED` | 同上，dist 截断 | 最真实 |

### Source 距离属性

```cpp
alSourcef(source, AL_REFERENCE_DISTANCE, 1.0f);  // ref：全音量的距离
alSourcef(source, AL_MAX_DISTANCE, 50.0f);        // max：听不到的距离
alSourcef(source, AL_ROLLOFF_FACTOR, 1.0f);       // rolloff：衰减速度
```

> **建议：** 大多数游戏使用 `AL_INVERSE_DISTANCE_CLAMPED`（默认值），`ref=1.0`、`max=50.0`、`rolloff=1.0` 是合理的起始值。

### 示例 2：距离衰减对比

```cpp
// distance_models.cpp — 对比不同距离模型的音量变化
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <cmath>
#include <vector>
#include <thread>
#include <chrono>

int main() {
    ALCdevice* dev = alcOpenDevice(nullptr);
    ALCcontext* ctx = alcCreateContext(dev, nullptr);
    alcMakeContextCurrent(ctx);

    // 生成 1 秒 440Hz
    constexpr int kRate = 44100;
    std::vector<int16_t> pcm(kRate);
    for (int i = 0; i < kRate; ++i)
        pcm[i] = int16_t(0.5f * 32767 * std::sin(6.2832f * 440.f * i / kRate));

    ALuint buf;
    alGenBuffers(1, &buf);
    alBufferData(buf, AL_FORMAT_MONO16, pcm.data(), kRate * 2, kRate);

    // 对比三种模型
    ALenum models[] = {AL_INVERSE_DISTANCE_CLAMPED, AL_LINEAR_DISTANCE_CLAMPED,
                       AL_EXPONENT_DISTANCE_CLAMPED};
    const char* names[] = {"INVERSE", "LINEAR", "EXPONENT"};

    for (int m = 0; m < 3; ++m) {
        alDistanceModel(models[m]);
        std::printf("\n=== %s ===\n", names[m]);

        ALuint src;
        alGenSources(1, &src);
        alSourcei(src, AL_BUFFER, buf);
        alSourcef(src, AL_REFERENCE_DISTANCE, 1.0f);
        alSourcef(src, AL_MAX_DISTANCE, 20.0f);
        alSourcei(src, AL_LOOPING, AL_TRUE);
        alSourcePlay(src);

        // Source 从 1 米逐渐远离到 20 米
        for (int d = 1; d <= 20; d += 2) {
            float pos[] = {0.0f, 0.0f, -(float)d};
            alSourcefv(src, AL_POSITION, pos);
            std::printf("  距离 %2dm\n", d);
            std::this_thread::sleep_for(std::chrono::milliseconds(500));
        }
        alSourceStop(src);
        alDeleteSources(1, &src);
    }

    alDeleteBuffers(1, &buf);
    alcMakeContextCurrent(nullptr); alcDestroyContext(ctx); alcCloseDevice(dev);
}
```

## 多普勒效应

当 Source 或 Listener 相对运动时，OpenAL 自动计算音调偏移（靠近音高升高，远离音高降低）。

### 全局参数

```cpp
alDopplerFactor(1.0f);    // 多普勒强度倍率（0=关闭，>1=夸张）
alSpeedOfSound(343.3f);   // 声速（米/秒），默认 343.3
```

### 多普勒公式

```
SS = AL_SPEED_OF_SOUND × AL_DOPPLER_FACTOR
freq_shift = freq × (SS - listener_vel_proj) / (SS + source_vel_proj)
```

其中 `vel_proj` 是速度在 Source→Listener 方向上的投影。

### 示例 3：模拟汽车驶过

```cpp
// doppler_demo.cpp — 汽车从左到右驶过
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <cmath>
#include <vector>
#include <thread>
#include <chrono>

int main() {
    ALCdevice* dev = alcOpenDevice(nullptr);
    ALCcontext* ctx = alcCreateContext(dev, nullptr);
    alcMakeContextCurrent(ctx);

    alDopplerFactor(2.0f);    // 夸大多普勒效果便于感知
    alSpeedOfSound(343.3f);

    // 引擎声：200Hz 低频
    constexpr int kRate = 44100, kN = kRate * 2;
    std::vector<int16_t> pcm(kN);
    for (int i = 0; i < kN; ++i)
        pcm[i] = int16_t(0.5f * 32767 * std::sin(6.2832f * 200.f * i / kRate));

    ALuint buf, src;
    alGenBuffers(1, &buf);
    alBufferData(buf, AL_FORMAT_MONO16, pcm.data(), kN * 2, kRate);
    alGenSources(1, &src);
    alSourcei(src, AL_BUFFER, buf);
    alSourcei(src, AL_LOOPING, AL_TRUE);
    alSourcef(src, AL_REFERENCE_DISTANCE, 1.0f);
    alSourcef(src, AL_MAX_DISTANCE, 100.0f);

    // 汽车从 x=-50 以 30m/s 向右驶过
    float speed = 30.0f;
    float vel[] = {speed, 0.0f, 0.0f};
    alSourcefv(src, AL_VELOCITY, vel);

    alSourcePlay(src);
    std::puts("汽车从左向右驶过（听音调变化）...");

    for (int step = 0; step < 100; ++step) {
        float x = -50.0f + speed * step * 0.05f;
        float pos[] = {x, 0.0f, -5.0f};  // 在听者前方 5 米
        alSourcefv(src, AL_POSITION, pos);
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
    }

    alSourceStop(src);
    alDeleteSources(1, &src); alDeleteBuffers(1, &buf);
    alcMakeContextCurrent(nullptr); alcDestroyContext(ctx); alcCloseDevice(dev);
}
```

## 常见错误与解决方案

### 错误 1：立体声 Buffer 无法 3D 定位

```cpp
// ❌ STEREO16 Buffer 不参与 3D 计算，始终以原始左右声道播放
alBufferData(buf, AL_FORMAT_STEREO16, data, size, rate);
alSourcefv(src, AL_POSITION, pos);  // 无效！
// ✅ 3D 定位必须用 MONO 格式 Buffer
alBufferData(buf, AL_FORMAT_MONO16, data, size, rate);
```

### 错误 2：Velocity 设置了但没感觉到多普勒

检查：(1) `alDopplerFactor` 是否 > 0；(2) 速度值是否相对于 `alSpeedOfSound` 有意义（速度太小效果不明显）；(3) 速度方向是否沿 Source→Listener 方向有分量。

### 错误 3：AL_ORIENTATION 只传了 3 个浮点数

`AL_ORIENTATION` 需要 6 个浮点数（forward + up），传 3 个会导致 `AL_INVALID_VALUE`。

## 动手实践

### 练习 A：声像控制器（2D 游戏风格）

用 `AL_SOURCE_RELATIVE` 模式，创建一个 Source，通过键盘（或定时循环）改变 x 位置从 -1.0 到 +1.0，模拟左右声像滑动。

### 练习 B：3D 音频场景

创建 3 个 Source 分别放在听者的左方（-5,0,0）、正前方（0,0,-5）、右方（5,0,0），用不同频率正弦波，验证佩戴耳机时能分辨方向。

## 对照项目源码

KrKr2 的声像（pan）实现在 `WaveMixer.cpp` 第 654-656 行：

```cpp
float pos[3] = { pan, 0.0f, 0.0f };
alSourcefv(_alSource, AL_POSITION, pos);
```

其中 `pan` ∈ [-1.0, 1.0]，-1=全左，+1=全右，0=居中。配合 `AL_SOURCE_RELATIVE = AL_TRUE`，这等效于在 Listener 左右两侧移动 Source。KrKr2 不使用 3D 距离衰减（视觉小说不需要），距离模型保持默认。

## 本节小结

- OpenAL 使用右手坐标系，与 OpenGL 一致
- Listener 是唯一的"耳朵"，有 Position/Velocity/Orientation/Gain 四属性
- `AL_SOURCE_RELATIVE` 让 Source 位置相对于 Listener（适合 2D 音频/UI 音效）
- 3D 定位**必须用 MONO 格式** Buffer，STEREO 不参与空间计算
- 六种距离模型控制衰减曲线，默认 `AL_INVERSE_DISTANCE_CLAMPED` 适合大多数场景
- 多普勒效应由 Velocity 自动计算，`alDopplerFactor` 控制强度

## 练习题与答案

### 题目 1：坐标系与方向

Listener 的 `AL_ORIENTATION` 默认值 `{0,0,-1, 0,1,0}` 含义是什么？如果要让 Listener 面朝 +X 方向，应设置什么值？

<details>
<summary>查看答案</summary>

默认值表示：forward = (0,0,-1) 即面朝 -Z 方向，up = (0,1,0) 即头顶朝 +Y。

面朝 +X 方向：`{1,0,0, 0,1,0}`（forward = +X，up 不变）。

</details>

### 题目 2：为什么 STEREO Buffer 无法 3D 定位？

<details>
<summary>查看答案</summary>

STEREO 音频已经包含左右声道信息，OpenAL 会直接将左声道送左耳、右声道送右耳，跳过 HRTF/3D 定位计算。只有 MONO 格式的 Buffer 才会参与空间化处理——OpenAL 根据 Source 和 Listener 的相对位置计算左右耳的增益和延迟差异。

</details>

### 题目 3：计算多普勒频率偏移

一个 Source 发出 1000Hz 声音，以 34.33m/s 直接朝 Listener 移动（Listener 静止），`AL_SPEED_OF_SOUND = 343.3`，`AL_DOPPLER_FACTOR = 1.0`。实际听到的频率是多少？

<details>
<summary>查看答案</summary>

公式：`freq_heard = freq × SS / (SS + source_vel_proj)`

Source 朝 Listener 移动，`source_vel_proj` 为负值（约定：靠近为负）：

`freq_heard = 1000 × 343.3 / (343.3 - 34.33) = 1000 × 343.3 / 308.97 ≈ 1111 Hz`

靠近时音调升高约 11%。远离时 `source_vel_proj` 为正，频率降低。

</details>
