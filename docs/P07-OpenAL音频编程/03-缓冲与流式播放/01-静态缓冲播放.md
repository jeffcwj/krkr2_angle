## 什么是静态缓冲播放

**静态缓冲播放**（Static Buffer Playback）是 OpenAL 最基础的音频播放方式——将完整的 PCM 数据一次性加载到一个 Buffer 对象中，再绑定到 Source 进行播放：

```
┌──────────┐   alBufferData    ┌──────────┐   alSourcei    ┌──────────┐
│ PCM 数据  │ ───────────────> │  Buffer   │ ────────────> │  Source   │
│ (内存中)  │   一次性拷贝      │ (AL内部)  │  AL_BUFFER    │  播放控制  │
└──────────┘                   └──────────┘               └──────────┘
```

**与流式播放的核心区别：**

| 特性 | 静态缓冲 | 流式队列 |
|------|----------|----------|
| 数据加载 | 一次性全部加载 | 分块逐步填充 |
| 内存占用 | 整个音频驻留内存 | 仅缓冲区大小 |
| 适用场景 | 短音效（< 5秒） | 长音乐、语音 |
| API 复杂度 | 简单 | 需要持续管理队列 |

**经验法则：** 短于 2-5 秒的音效（按钮点击、脚步声、爆炸声）用静态缓冲；超过 5 秒的背景音乐、语音对话用流式播放。

## alBufferData 详解

### 函数签名

```cpp
void alBufferData(
    ALuint   buffer,    // Buffer 对象 ID（alGenBuffers 生成）
    ALenum   format,    // PCM 格式标识符
    const ALvoid *data, // 指向 PCM 数据的指针
    ALsizei  size,      // PCM 数据的字节数
    ALsizei  freq       // 采样率（Hz）
);
```

调用后，OpenAL 将 `data` 的 `size` 字节**拷贝**到内部存储，调用返回后可安全释放原始指针。若 Buffer 已含数据，旧数据被替换（但 Buffer 不能处于被 Source 绑定的状态）。

### 格式标识符

OpenAL 定义了四种基本 PCM 格式：

```cpp
AL_FORMAT_MONO8      // 单声道 8-bit（无符号，0-255，128 静音）
AL_FORMAT_STEREO8    // 立体声 8-bit
AL_FORMAT_MONO16     // 单声道 16-bit（有符号，-32768~32767，小端序）
AL_FORMAT_STEREO16   // 立体声 16-bit
```

KrKr2 中的格式选择（`WaveMixer.cpp` 第 523-535 行）：

```cpp
ALenum alFormat;
if (format->Channels == 1)
    alFormat = (format->BitsPerSample == 8) ? AL_FORMAT_MONO8 : AL_FORMAT_MONO16;
else
    alFormat = (format->BitsPerSample == 8) ? AL_FORMAT_STEREO8 : AL_FORMAT_STEREO16;
```

### 数据对齐要求

`size` 必须是**帧大小**（`声道数 × 每样本字节数`）的整数倍。MONO8=1、MONO16=2、STEREO8=2、STEREO16=4。不对齐时行为未定义（截断/噪音/`AL_INVALID_VALUE`）。

### 示例 1：最简静态播放

下面是将 PCM 数据加载到 Buffer 并播放的最小完整示例：

```cpp
// static_play_minimal.cpp — 生成正弦波并通过静态缓冲播放
#include <AL/al.h>
#include <AL/alc.h>
#include <cmath>
#include <cstdio>
#include <cstdint>
#include <vector>
#include <thread>
#include <chrono>

#define AL_CHECK(msg) do { \
    ALenum err = alGetError(); \
    if (err != AL_NO_ERROR) { \
        std::fprintf(stderr, "AL error [%s]: 0x%X\n", msg, err); \
        return 1; \
    } \
} while(0)

int main() {
    // 1. 打开设备与创建上下文
    ALCdevice* device = alcOpenDevice(nullptr);
    if (!device) { std::fprintf(stderr, "无法打开设备\n"); return 1; }
    ALCcontext* context = alcCreateContext(device, nullptr);
    alcMakeContextCurrent(context);

    // 2. 生成 440Hz 正弦波（16-bit Mono, 44100 Hz, 1秒）
    const ALsizei sampleRate = 44100;
    const int numSamples = sampleRate;  // 1 秒
    std::vector<int16_t> pcm(numSamples);
    for (int i = 0; i < numSamples; ++i) {
        float t = static_cast<float>(i) / sampleRate;
        pcm[i] = static_cast<int16_t>(
            0.8f * 32767.0f * std::sin(2.0f * 3.14159265f * 440.0f * t));
    }

    // 3. 创建 Buffer 并填充
    ALuint buffer;
    alGenBuffers(1, &buffer);
    alBufferData(buffer, AL_FORMAT_MONO16, pcm.data(),
                 numSamples * sizeof(int16_t), sampleRate);
    AL_CHECK("alBufferData");
    // pcm 可安全释放——OpenAL 已内部拷贝

    // 4. 创建 Source 绑定 Buffer 并播放
    ALuint source;
    alGenSources(1, &source);
    alSourcei(source, AL_BUFFER, buffer);
    alSourcePlay(source);
    AL_CHECK("alSourcePlay");

    // 5. 等待播放结束并清理
    ALint state;
    do {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        alGetSourcei(source, AL_SOURCE_STATE, &state);
    } while (state == AL_PLAYING);

    alDeleteSources(1, &source);
    alDeleteBuffers(1, &buffer);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(context);
    alcCloseDevice(device);
    return 0;
}
```

**CMakeLists.txt：**

```cmake
cmake_minimum_required(VERSION 3.20)
project(static_play_minimal)
find_package(OpenAL CONFIG REQUIRED)
add_executable(static_play_minimal static_play_minimal.cpp)
target_link_libraries(static_play_minimal PRIVATE OpenAL::OpenAL)
```

编译运行后会听到 1 秒 440Hz 正弦波（A4 音符）。

**注意：** 如果 Buffer 当前绑定到某个 Source，调用 `alBufferData` 会返回 `AL_INVALID_OPERATION`。必须先用 `alSourcei(source, AL_BUFFER, 0)` 解绑。

## WAV 文件格式解析

WAV 是最常用的无压缩音频格式，本质是 RIFF 容器包裹的 PCM 数据。

### WAV 文件结构

```
偏移  大小  内容            说明
0x00  4     "RIFF"          RIFF 标识
0x04  4     fileSize-8      文件总大小减 8
0x08  4     "WAVE"          WAVE 标识
0x0C  4     "fmt "          fmt 子块标识（注意末尾空格）
0x10  4     16              fmt 块大小（PCM 固定 16）
0x14  2     1               音频格式（1=PCM）
0x16  2     channels        声道数
0x18  4     sampleRate      采样率（Hz）
0x1C  4     byteRate        字节率
0x20  2     blockAlign      块对齐 = channels × bitsPerSample/8
0x22  2     bitsPerSample   位深度（8 或 16）
0x24  4     "data"          data 子块标识
0x28  4     dataSize        PCM 数据字节数
0x2C  ...   PCM data        原始样本数据
```

### WAV 解析器实现

```cpp
// wav_parser.h — 最小 WAV 文件解析器
#pragma once
#include <cstdint>
#include <cstdio>
#include <cstring>
#include <vector>

struct WavData {
    uint16_t channels;       // 声道数
    uint32_t sampleRate;     // 采样率
    uint16_t bitsPerSample;  // 位深
    std::vector<uint8_t> pcm; // 原始 PCM 数据
};

inline uint32_t readU32LE(const uint8_t* p) {  // 小端序读 4 字节
    return p[0] | (p[1] << 8) | (p[2] << 16) | (p[3] << 24);
}
inline uint16_t readU16LE(const uint8_t* p) {  // 小端序读 2 字节
    return p[0] | (p[1] << 8);
}

inline bool parseWav(const char* path, WavData& out) {
    std::FILE* f = std::fopen(path, "rb");
    if (!f) return false;
    std::fseek(f, 0, SEEK_END);
    long fileSize = std::ftell(f);
    std::fseek(f, 0, SEEK_SET);
    std::vector<uint8_t> raw(fileSize);
    std::fread(raw.data(), 1, fileSize, f);
    std::fclose(f);

    if (fileSize < 44) return false;
    if (std::memcmp(raw.data(), "RIFF", 4) != 0) return false;
    if (std::memcmp(raw.data() + 8, "WAVE", 4) != 0) return false;

    size_t pos = 12;
    bool fmtFound = false, dataFound = false;
    while (pos + 8 <= static_cast<size_t>(fileSize)) {
        char id[5] = {};
        std::memcpy(id, raw.data() + pos, 4);
        uint32_t sz = readU32LE(raw.data() + pos + 4);

        if (std::strcmp(id, "fmt ") == 0 && sz >= 16) {
            if (readU16LE(raw.data() + pos + 8) != 1) return false; // 仅 PCM
            out.channels     = readU16LE(raw.data() + pos + 10);
            out.sampleRate   = readU32LE(raw.data() + pos + 12);
            out.bitsPerSample = readU16LE(raw.data() + pos + 22);
            fmtFound = true;
        } else if (std::strcmp(id, "data") == 0) {
            out.pcm.assign(raw.data() + pos + 8,
                           raw.data() + pos + 8 + sz);
            dataFound = true;
        }
        pos += 8 + sz + (sz % 2);  // RIFF 子块按 2 字节对齐
    }
    return fmtFound && dataFound;
}
```

**关键细节：** WAV 使用小端序（与 x86/ARM 一致，无需转换）。遍历子块比固定偏移更健壮——`fmt` 和 `data` 之间可能夹着 `LIST`、`fact` 等块。子块按 2 字节对齐。

### 示例 2：加载 WAV 并播放

结合 WAV 解析器和 OpenAL，实现完整的文件到播放流程：

```cpp
// play_wav.cpp — 加载 WAV 文件并通过 OpenAL 静态缓冲播放
#include "wav_parser.h"
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <thread>
#include <chrono>

// 根据 WAV 参数选择 OpenAL 格式
ALenum getALFormat(uint16_t channels, uint16_t bitsPerSample) {
    if (channels == 1 && bitsPerSample == 8)  return AL_FORMAT_MONO8;
    if (channels == 1 && bitsPerSample == 16) return AL_FORMAT_MONO16;
    if (channels == 2 && bitsPerSample == 8)  return AL_FORMAT_STEREO8;
    if (channels == 2 && bitsPerSample == 16) return AL_FORMAT_STEREO16;
    return 0;  // 不支持的格式组合
}

int main(int argc, char* argv[]) {
    if (argc < 2) { std::printf("用法: play_wav <file.wav>\n"); return 1; }

    // 1. 解析 WAV
    WavData wav;
    if (!parseWav(argv[1], wav)) return 1;
    ALenum format = getALFormat(wav.channels, wav.bitsPerSample);
    if (format == 0) { std::fprintf(stderr, "不支持的格式\n"); return 1; }

    float dur = static_cast<float>(wav.pcm.size()) /
                (wav.sampleRate * wav.channels * wav.bitsPerSample / 8);
    std::printf("%d Hz, %dch, %dbit, %.2fs\n",
                wav.sampleRate, wav.channels, wav.bitsPerSample, dur);

    // 2. 初始化 OpenAL → 创建 Buffer → 加载 → 播放
    ALCdevice* device = alcOpenDevice(nullptr);
    ALCcontext* ctx   = alcCreateContext(device, nullptr);
    alcMakeContextCurrent(ctx);

    ALuint buffer;
    alGenBuffers(1, &buffer);
    alBufferData(buffer, format, wav.pcm.data(),
                 static_cast<ALsizei>(wav.pcm.size()),
                 static_cast<ALsizei>(wav.sampleRate));

    ALuint source;
    alGenSources(1, &source);
    alSourcei(source, AL_BUFFER, buffer);
    alSourcePlay(source);

    ALint state;
    do {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        alGetSourcei(source, AL_SOURCE_STATE, &state);
    } while (state == AL_PLAYING);

    // 清理
    alDeleteSources(1, &source);
    alDeleteBuffers(1, &buffer);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(ctx);
    alcCloseDevice(device);
    return 0;
}
```

填充 Buffer 后可用 `alGetBufferi` 查询属性验证数据正确性：

```cpp
ALint freq, bits, channels, size;
alGetBufferi(buffer, AL_FREQUENCY, &freq);   // 采样率
alGetBufferi(buffer, AL_BITS, &bits);        // 位深
alGetBufferi(buffer, AL_CHANNELS, &channels);// 声道数
alGetBufferi(buffer, AL_SIZE, &size);        // 字节数
float duration = static_cast<float>(size) / (freq * channels * bits / 8);
std::printf("%d Hz, %dbit, %dch, %d bytes (%.2fs)\n",
            freq, bits, channels, size, duration);
```

## 常见错误与解决方案

### 错误 1：Buffer 绑定时更新数据

```cpp
alSourcei(source, AL_BUFFER, buffer);
alBufferData(buffer, AL_FORMAT_MONO16, data, size, 44100);
// ❌ AL_INVALID_OPERATION — Buffer 正在被 Source 使用
```

**解决：** 先解绑再更新：

```cpp
alSourceStop(source);                  // 确保停止播放
alSourcei(source, AL_BUFFER, 0);       // 解绑
alBufferData(buffer, ...);             // 安全更新
alSourcei(source, AL_BUFFER, buffer);  // 重新绑定
```

### 错误 2：数据大小未对齐

```cpp
// 假设有 1001 字节数据，但用 STEREO16（帧大小 4）
alBufferData(buffer, AL_FORMAT_STEREO16, data, 1001, 44100);
// ❌ 1001 不是 4 的倍数 → AL_INVALID_VALUE 或噪音
```

**解决：** 截断到帧对齐边界：

```cpp
int frameSize = channels * (bitsPerSample / 8);
ALsizei alignedSize = (dataSize / frameSize) * frameSize;
alBufferData(buffer, format, data, alignedSize, sampleRate);
```

### 错误 3：8-bit 与 16-bit 数据混淆

8-bit PCM 是**无符号**（0-255，128 静音），16-bit 是**有符号**（-32768 到 32767，0 静音）。格式不匹配会导致噪音：

```cpp
// ❌ 将 16-bit 有符号数据当作 8-bit 无符号加载
alBufferData(buffer, AL_FORMAT_MONO8, pcm16Data, size, 44100);
// 结果：刺耳的噪音

// ✅ 正确匹配格式
alBufferData(buffer, AL_FORMAT_MONO16, pcm16Data, size, 44100);
```

## 本节小结

- `alBufferData` 一次性拷贝 PCM 到 Buffer 内部，调用后可释放原始指针
- 四种格式：MONO8/MONO16/STEREO8/STEREO16；数据大小必须帧对齐
- WAV = RIFF 容器 + PCM 数据，解析时应遍历子块而非固定偏移
- 静态缓冲适合短音效（< 5秒），长音频用流式播放

## 练习题与答案

### 题目 1：格式参数计算

一个 WAV 文件参数为：采样率 22050 Hz、立体声、16-bit。请计算：
1. 帧大小（frame size）
2. 每秒数据量（byte rate）
3. 10 秒音频的 PCM 数据总字节数
4. 应使用哪个 `AL_FORMAT_*` 常量

<details>
<summary>查看答案</summary>

1. **帧大小** = 声道数 × 每样本字节数 = 2 × 2 = **4 字节**
2. **字节率** = 采样率 × 帧大小 = 22050 × 4 = **88200 字节/秒**
3. **10 秒数据量** = 88200 × 10 = **882000 字节 ≈ 861 KB**
4. **格式常量** = `AL_FORMAT_STEREO16`（立体声 + 16-bit）

</details>

### 题目 2：找出代码错误

以下代码有 3 处错误，请指出并修正：

```cpp
ALuint buf, src;
alGenBuffers(1, &buf);
alGenSources(1, &src);
alSourcei(src, AL_BUFFER, buf);

int16_t samples[1000];
// ... 填充 samples ...
alBufferData(buf, AL_FORMAT_MONO16, samples, 1000, 44100);
alSourcePlay(src);
```

<details>
<summary>查看答案</summary>

**错误 1：** 在 `alSourcei(src, AL_BUFFER, buf)` 之后才调用 `alBufferData`。Buffer 已绑定到 Source，会触发 `AL_INVALID_OPERATION`。应先填充数据再绑定。

**错误 2：** `alBufferData` 的 `size` 参数应为**字节数**，不是样本数。1000 个 `int16_t` 是 2000 字节，应传 `1000 * sizeof(int16_t)` 即 `2000`。

**错误 3：** 传 `1000` 时虽然对齐正确（MONO16 帧大小为 2，1000 是 2 的倍数），但只加载了前 500 个样本的数据。

修正后代码：

```cpp
ALuint buf, src;
alGenBuffers(1, &buf);
alGenSources(1, &src);

int16_t samples[1000];
// ... 填充 samples ...
alBufferData(buf, AL_FORMAT_MONO16, samples,
             1000 * sizeof(int16_t), 44100);  // 2000 字节
alSourcei(src, AL_BUFFER, buf);               // 填充后再绑定
alSourcePlay(src);
```

</details>

### 题目 3：实现 Buffer 复用

编写一个函数，安全地更新已有 Buffer 的数据（即使它可能正绑定在某个 Source 上）：

```cpp
bool safeBufferUpdate(ALuint source, ALuint buffer,
                      ALenum format, const void* data,
                      ALsizei size, ALsizei freq);
```

<details>
<summary>查看答案</summary>

```cpp
bool safeBufferUpdate(ALuint source, ALuint buffer,
                      ALenum format, const void* data,
                      ALsizei size, ALsizei freq) {
    ALint state;
    alGetSourcei(source, AL_SOURCE_STATE, &state);
    bool wasPlaying = (state == AL_PLAYING);
    if (state == AL_PLAYING || state == AL_PAUSED) alSourceStop(source);

    alSourcei(source, AL_BUFFER, 0);           // 解绑
    if (alGetError() != AL_NO_ERROR) return false;
    alBufferData(buffer, format, data, size, freq); // 更新
    if (alGetError() != AL_NO_ERROR) return false;
    alSourcei(source, AL_BUFFER, buffer);      // 重新绑定
    if (alGetError() != AL_NO_ERROR) return false;

    if (wasPlaying) alSourcePlay(source);      // 恢复播放
    return true;
}
```

</details>
