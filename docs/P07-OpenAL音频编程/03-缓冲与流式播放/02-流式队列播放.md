## 为什么需要流式播放

静态缓冲将完整音频一次性加载到内存，对于短音效足够高效。但对于长音频（背景音乐、语音对话），完全加载会带来严重问题：

| 场景 | 44100Hz Stereo 16-bit | 内存占用 |
|------|----------------------|----------|
| 3 秒音效 | 3 × 176400 | **517 KB** |
| 3 分钟音乐 | 180 × 176400 | **30.3 MB** |
| 30 分钟语音 | 1800 × 176400 | **303 MB** |

流式播放的核心思想：**只在内存中保留即将播放的一小段数据，边解码边播放**。

```
解码器                         OpenAL
┌──────┐    填充     ┌────┐ Queue  ┌────────┐
│ 文件  │ ────────> │ B1 │ ────> │        │
│ 解码  │            ├────┤       │ Source │ ──> 扬声器
│      │    填充     │ B2 │ ────> │        │
└──────┘ <──回收──── ├────┤       └────────┘
                     │ B3 │ (已播放，回收重填)
                     └────┘
```

## Buffer 队列 API

### alSourceQueueBuffers — 入队

```cpp
void alSourceQueueBuffers(
    ALuint  source,     // Source 对象
    ALsizei nb,         // 要入队的 Buffer 数量
    const ALuint *buffers // Buffer ID 数组
);
```

将一个或多个**已填充数据**的 Buffer 追加到 Source 的播放队列尾部。入队后 Buffer 归 Source 所有，不能被其他 Source 使用或直接调用 `alBufferData` 更新。

### alSourceUnqueueBuffers — 出队

```cpp
void alSourceUnqueueBuffers(
    ALuint  source,     // Source 对象
    ALsizei nb,         // 要出队的 Buffer 数量
    ALuint *buffers     // 接收 Buffer ID 的数组
);
```

从队列头部移除已经**播放完毕**的 Buffer。只有 `AL_BUFFERS_PROCESSED` 个 Buffer 可以出队，尝试出队更多会触发 `AL_INVALID_VALUE`。

### 关键状态查询

```cpp
ALint queued, processed;
alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);       // 队列中总 Buffer 数
alGetSourcei(source, AL_BUFFERS_PROCESSED, &processed); // 已播放完的 Buffer 数
// 正在播放 + 等待播放的数量 = queued - processed
```

### 静态绑定 vs 队列绑定

OpenAL 中一个 Source 只能处于两种模式之一：

```
静态模式：alSourcei(source, AL_BUFFER, buffer)
  - 绑定单个 Buffer，可重复播放
  - 不能调用 Queue/Unqueue

队列模式：alSourceQueueBuffers(source, ...)
  - 使用 Buffer 队列
  - 不能调用 alSourcei(AL_BUFFER)

切换模式：先 alSourcei(source, AL_BUFFER, 0) 清空，再切换
```

## 流式播放循环

### 示例 1：基础流式播放框架

```cpp
// stream_play.cpp — 流式播放框架（伪代码+完整结构）
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <vector>
#include <thread>
#include <chrono>

constexpr int NUM_BUFFERS = 3;        // 缓冲区数量
constexpr int BUFFER_SIZE = 4096 * 4; // 每块 16KB（约 93ms @44100Hz Mono16）

// 模拟解码器：生成指定频率的正弦波
struct SineDecoder {
    int sampleRate = 44100;
    float frequency = 440.0f;
    int position = 0;      // 当前采样位置
    int totalSamples;      // 总采样数

    SineDecoder(float dur) : totalSamples(static_cast<int>(sampleRate * dur)) {}

    // 解码一块数据，返回实际字节数（0 = 结束）
    int decode(int16_t* out, int maxSamples) {
        int remaining = totalSamples - position;
        int count = (remaining < maxSamples) ? remaining : maxSamples;
        for (int i = 0; i < count; ++i) {
            float t = static_cast<float>(position + i) / sampleRate;
            out[i] = static_cast<int16_t>(
                0.8f * 32767.0f * std::sin(2.0f * 3.14159265f * frequency * t));
        }
        position += count;
        return count * sizeof(int16_t);  // 返回字节数
    }

    bool finished() const { return position >= totalSamples; }
};

int main() {
    // 初始化 OpenAL
    ALCdevice* device = alcOpenDevice(nullptr);
    ALCcontext* ctx = alcCreateContext(device, nullptr);
    alcMakeContextCurrent(ctx);

    // 创建 Source 和 Buffer 池
    ALuint source;
    alGenSources(1, &source);
    ALuint buffers[NUM_BUFFERS];
    alGenBuffers(NUM_BUFFERS, buffers);

    SineDecoder decoder(5.0f);  // 5 秒正弦波
    std::vector<int16_t> pcm(BUFFER_SIZE / sizeof(int16_t));

    // ---- 1. 预填充所有 Buffer 并入队 ----
    for (int i = 0; i < NUM_BUFFERS && !decoder.finished(); ++i) {
        int bytes = decoder.decode(pcm.data(),
                                   static_cast<int>(pcm.size()));
        if (bytes > 0) {
            alBufferData(buffers[i], AL_FORMAT_MONO16,
                         pcm.data(), bytes, 44100);
            alSourceQueueBuffers(source, 1, &buffers[i]);
        }
    }

    // ---- 2. 开始播放 ----
    alSourcePlay(source);

    // ---- 3. 流式循环：回收已播完 Buffer → 重填 → 入队 ----
    while (true) {
        ALint processed = 0;
        alGetSourcei(source, AL_BUFFERS_PROCESSED, &processed);

        while (processed-- > 0) {
            ALuint buf;
            alSourceUnqueueBuffers(source, 1, &buf);  // 回收

            if (!decoder.finished()) {
                int bytes = decoder.decode(pcm.data(),
                                           static_cast<int>(pcm.size()));
                if (bytes > 0) {
                    alBufferData(buf, AL_FORMAT_MONO16,
                                 pcm.data(), bytes, 44100);
                    alSourceQueueBuffers(source, 1, &buf); // 重新入队
                }
            }
        }

        // 检查是否播放完毕
        ALint state;
        alGetSourcei(source, AL_SOURCE_STATE, &state);
        if (state != AL_PLAYING) {
            ALint queued;
            alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);
            if (queued == 0) break;  // 队列空且停止 → 播放完毕
            alSourcePlay(source);    // Buffer Underrun 恢复
        }

        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }

    std::printf("流式播放完毕\n");

    // 清理
    alDeleteSources(1, &source);
    alDeleteBuffers(NUM_BUFFERS, buffers);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(ctx);
    alcCloseDevice(device);
    return 0;
}
```

**关键流程分析：**

```
┌─────────────┐
│  预填充阶段   │  填充 N 个 Buffer → 全部 Queue → Play
└──────┬──────┘
       ▼
┌─────────────┐
│  流式循环    │  查询 processed → Unqueue → 解码 → BufferData → Queue
│  (每10ms)   │  如果 state != PLAYING 且队列非空 → 恢复播放
└──────┬──────┘
       ▼
┌─────────────┐
│  结束条件    │  解码器结束 + 队列清空 + Source 停止
└─────────────┘
```

## Buffer 数量与大小调优

Buffer 数量和大小直接影响延迟和稳定性：

| 参数 | 较小值 | 较大值 |
|------|--------|--------|
| Buffer 数量 | 延迟低，但容易 Underrun | 容错高，延迟增加 |
| Buffer 大小 | 更新频率高，CPU 占用高 | 更新频率低，但延迟高 |

**推荐配置：**

```
               延迟计算
Buffer 大小 B = 每块样本数 × 帧大小
延迟        L = B × N / (采样率 × 帧大小)

示例：4096 样本 × 2 字节 × 3 个 Buffer / (44100 × 2) ≈ 139ms
```

| 场景 | Buffer 数 | 每块大小 | 延迟 |
|------|-----------|----------|------|
| 实时音效 | 2 | 2048 样本 | ~93ms |
| 音乐播放 | 3-4 | 4096 样本 | ~186ms |
| 后台语音 | 4-8 | 8192 样本 | ~743ms |

### 示例 2：可配置的流式播放器

```cpp
// configurable_stream.cpp — 可调参数的流式播放器
struct StreamConfig {
    int numBuffers  = 3;           // Buffer 数量
    int bufferSize  = 4096 * 2;    // 每块字节数
    int sampleRate  = 44100;
    ALenum format   = AL_FORMAT_MONO16;
};

class StreamPlayer {
    ALuint source_;
    std::vector<ALuint> buffers_;
    StreamConfig config_;

public:
    StreamPlayer(const StreamConfig& cfg) : config_(cfg) {
        alGenSources(1, &source_);
        buffers_.resize(cfg.numBuffers);
        alGenBuffers(cfg.numBuffers, buffers_.data());
    }

    ~StreamPlayer() {
        alSourceStop(source_);
        // 必须先清空队列再删除 Buffer
        ALint queued;
        alGetSourcei(source_, AL_BUFFERS_QUEUED, &queued);
        while (queued-- > 0) {
            ALuint buf;
            alSourceUnqueueBuffers(source_, 1, &buf);
        }
        alDeleteSources(1, &source_);
        alDeleteBuffers(config_.numBuffers, buffers_.data());
    }

    // 预填充所有 Buffer
    template<typename Decoder>
    void prefill(Decoder& dec) {
        std::vector<int16_t> pcm(config_.bufferSize / sizeof(int16_t));
        for (int i = 0; i < config_.numBuffers; ++i) {
            int bytes = dec.decode(pcm.data(), static_cast<int>(pcm.size()));
            if (bytes <= 0) break;
            alBufferData(buffers_[i], config_.format,
                         pcm.data(), bytes, config_.sampleRate);
            alSourceQueueBuffers(source_, 1, &buffers_[i]);
        }
    }

    void play() { alSourcePlay(source_); }

    // 更新循环，返回 false 表示播放结束
    template<typename Decoder>
    bool update(Decoder& dec) {
        ALint processed;
        alGetSourcei(source_, AL_BUFFERS_PROCESSED, &processed);

        std::vector<int16_t> pcm(config_.bufferSize / sizeof(int16_t));
        while (processed-- > 0) {
            ALuint buf;
            alSourceUnqueueBuffers(source_, 1, &buf);
            if (!dec.finished()) {
                int bytes = dec.decode(pcm.data(),
                                       static_cast<int>(pcm.size()));
                if (bytes > 0) {
                    alBufferData(buf, config_.format,
                                 pcm.data(), bytes, config_.sampleRate);
                    alSourceQueueBuffers(source_, 1, &buf);
                }
            }
        }

        // Underrun 检测与恢复
        ALint state;
        alGetSourcei(source_, AL_SOURCE_STATE, &state);
        if (state != AL_PLAYING) {
            ALint queued;
            alGetSourcei(source_, AL_BUFFERS_QUEUED, &queued);
            if (queued == 0) return false;  // 播放结束
            alSourcePlay(source_);           // Underrun 恢复
        }
        return true;
    }
};
```

## Buffer Underrun 处理

**Buffer Underrun** 发生在 Source 播放完所有队列中的 Buffer、但新数据还没填充上来时。此时 Source 自动变为 `AL_STOPPED` 状态。

```
正常流程：B1→B2→B3→(B1已回收重填)→B1→B2→...
Underrun：B1→B2→B3→(解码太慢，没有新Buffer)→STOPPED!
```

检测与恢复模式：

```cpp
ALint state;
alGetSourcei(source, AL_SOURCE_STATE, &state);
if (state == AL_STOPPED) {
    ALint queued;
    alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);
    if (queued > 0) {
        // Underrun：队列非空但播放停止 → 恢复
        std::fprintf(stderr, "警告: Buffer Underrun，恢复播放\n");
        alSourcePlay(source);
    } else {
        // 正常结束：队列为空
    }
}
```

**预防 Underrun 的策略：**
- 增加 Buffer 数量（3 → 4-6）
- 增加每块 Buffer 大小
- 降低流式循环的休眠间隔（10ms → 5ms）
- 在单独的线程中运行解码，避免主线程阻塞

## 常见错误与解决方案

### 错误 1：混用静态绑定和队列模式

```cpp
alSourcei(source, AL_BUFFER, buffer);        // 静态模式
alSourceQueueBuffers(source, 1, &buffer2);   // ❌ AL_INVALID_OPERATION
```

**解决：** 切换模式前先清空：`alSourcei(source, AL_BUFFER, 0)`

### 错误 2：Unqueue 数量超过 processed

```cpp
ALint processed;
alGetSourcei(source, AL_BUFFERS_PROCESSED, &processed);
// processed = 1，但尝试 Unqueue 3 个
ALuint bufs[3];
alSourceUnqueueBuffers(source, 3, bufs);  // ❌ AL_INVALID_VALUE
```

**解决：** 始终先查询 processed，只 Unqueue 不超过 processed 的数量。

### 错误 3：忘记在销毁前清空队列

```cpp
alDeleteBuffers(NUM_BUFFERS, buffers);  // ❌ Buffer 还在队列中
```

**解决：** 销毁前必须停止 Source 并 Unqueue 所有 Buffer：

```cpp
alSourceStop(source);
ALint queued;
alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);
std::vector<ALuint> unqueued(queued);
alSourceUnqueueBuffers(source, queued, unqueued.data());
alDeleteBuffers(NUM_BUFFERS, buffers);  // 现在安全
```

## 本节小结

- 流式播放通过 Buffer 队列实现"边解码边播放"，内存占用恒定
- `alSourceQueueBuffers` 入队、`alSourceUnqueueBuffers` 出队已播完的 Buffer
- 流式循环三步：查询 processed → Unqueue → 重填 BufferData → Queue
- Buffer Underrun 时 Source 自动停止，需检测并恢复
- 静态绑定与队列模式互斥，切换前需清空

## 练习题与答案

### 题目 1：延迟计算

假设流式播放配置为：4 个 Buffer，每个 Buffer 8192 个样本，采样率 48000 Hz，Stereo 16-bit。计算最大播放延迟。

<details>
<summary>查看答案</summary>

每个 Buffer 的时长 = 8192 / 48000 ≈ 0.1707 秒
最大延迟 = 4 × 0.1707 ≈ **0.683 秒 ≈ 683ms**

注意：延迟按**样本数÷采样率**计算，与声道数和位深无关（它们影响字节数但不影响时间）。

</details>

### 题目 2：实现安全的队列清空函数

编写 `drainQueue(ALuint source)` 函数，安全地清空 Source 的 Buffer 队列（无论 Source 处于什么状态）：

<details>
<summary>查看答案</summary>

```cpp
void drainQueue(ALuint source) {
    alSourceStop(source);  // 停止播放（使所有 Buffer 变为 processed）
    ALint queued;
    alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);
    for (int i = 0; i < queued; ++i) {
        ALuint buf;
        alSourceUnqueueBuffers(source, 1, &buf);
    }
}
```

关键：`alSourceStop` 后，所有队列中的 Buffer 都变为 processed 状态，可以安全 Unqueue。

</details>

### 题目 3：Underrun 判断

以下状态中，哪些表示 Buffer Underrun？

| 序号 | AL_SOURCE_STATE | AL_BUFFERS_QUEUED | AL_BUFFERS_PROCESSED |
|------|-----------------|-------------------|---------------------|
| A | PLAYING | 3 | 1 |
| B | STOPPED | 2 | 2 |
| C | STOPPED | 0 | 0 |
| D | PAUSED | 3 | 0 |

<details>
<summary>查看答案</summary>

- **A：正常播放**。3 个 Buffer 在队列中，1 个已播完可回收。
- **B：Buffer Underrun**。Source 停止但队列非空（还有 2 个 Buffer），说明播放中断。
- **C：正常结束**。队列为空且已停止，播放自然结束。
- **D：正常暂停**。用户主动 Pause，不是 Underrun。

</details>
