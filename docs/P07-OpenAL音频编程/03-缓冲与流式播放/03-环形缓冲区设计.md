## 为什么音频需要环形缓冲区

在上一节的流式播放中，我们用 OpenAL Buffer 队列实现了"边解码边播放"。但实际音频引擎中，解码线程和播放线程运行在不同线程上：

```
解码线程：读文件 → 解码为 PCM → 写入缓冲区
播放线程：从缓冲区读 PCM → 填充 AL Buffer → 提交给 Source
```

两个线程以不同速率访问同一块内存——经典的**生产者-消费者问题**。普通数组有致命缺陷：

| 问题 | 普通数组 | 环形缓冲区 |
|------|----------|------------|
| 空间回收 | 已读数据需 memmove 搬移 | 读指针前进后自动可写 |
| 内存分配 | 频繁 new/delete 或不断增长 | 固定大小，零分配 |
| 线程安全 | 需互斥锁保护整个数组 | 单生产者-单消费者可无锁 |

环形缓冲区是音频引擎的标配数据结构，PortAudio、JACK、PulseAudio、CoreAudio 都依赖它。

## 环形缓冲区原理

### 内存布局与回绕

环形缓冲区是一块**固定大小的线性内存**，通过读指针 `R` 和写指针 `W` 模拟"环形"：

```
初始:  [  |  |  |  |  |  |  |  ]  R=W=0, DS=0
       R,W

写5个: [ A| B| C| D| E|  |  |  ]  R=0,W=5, DS=5
       R              W

读3个: [  |  |  | D| E|  |  |  ]  R=3,W=5, DS=2
                R     W

再写5(回绕): [ H| I|  | D| E| F| G| H]  R=3,W=2, DS=7
                  W  R    ← 数据跨物理边界 [3..7]+[0..1]
```

游标回绕核心是取模：`WritePos = (WritePos + count) % Size`。若 Size 是 2 的幂可用位与加速：`(WritePos + count) & Mask`。KrKr2 用减法（假设 advance < Size）：`if (WritePos >= Size) WritePos -= Size;`

### 数据量追踪策略

| 策略 | 实现 | 优缺点 |
|------|------|--------|
| 独立计数器（KrKr2 用） | 维护 `DataSize` | 空/满判断简单，需原子操作 |
| 指针差值 | `(W-R+Size)%Size` | 无额外变量，无法区分全满/全空 |
| 浪费一位置 | 满=`(W+1)%Size==R` | 可用容量减一 |

## 从零实现：分段指针接口

当数据跨越物理边界时，一次读取对应**两段不连续内存**。直接返回指针（零拷贝）比 memcpy 更高效。KrKr2 的 `tRisaRingBuffer` 正是这种模式。

### 示例 1：分段指针环形缓冲区

```cpp
#include <cstddef>
#include <cstring>
#include <algorithm>

template <typename T>
class RingBuffer {
    T* buf_;
    size_t cap_, rpos_, wpos_, used_;
public:
    explicit RingBuffer(size_t capacity)
        : cap_(capacity), rpos_(0), wpos_(0), used_(0) {
        buf_ = new T[cap_];
    }
    ~RingBuffer() { delete[] buf_; }
    RingBuffer(const RingBuffer&) = delete;
    RingBuffer& operator=(const RingBuffer&) = delete;

    size_t capacity()  const { return cap_; }
    size_t dataSize()  const { return used_; }
    size_t freeSize()  const { return cap_ - used_; }
    bool   empty()     const { return used_ == 0; }
    bool   full()      const { return used_ == cap_; }

    // 获取可读数据指针（最多两段）
    void getReadPointer(size_t count,
                        const T*& p1, size_t& s1,
                        const T*& p2, size_t& s2) const {
        if (count + rpos_ > cap_) {
            p1 = buf_ + rpos_;  s1 = cap_ - rpos_;
            p2 = buf_;          s2 = count - s1;
        } else {
            p1 = buf_ + rpos_;  s1 = count;
            p2 = nullptr;       s2 = 0;
        }
    }

    // 获取可写空间指针（最多两段）
    void getWritePointer(size_t count,
                         T*& p1, size_t& s1,
                         T*& p2, size_t& s2) {
        if (count + wpos_ > cap_) {
            p1 = buf_ + wpos_;  s1 = cap_ - wpos_;
            p2 = buf_;          s2 = count - s1;
        } else {
            p1 = buf_ + wpos_;  s1 = count;
            p2 = nullptr;       s2 = 0;
        }
    }

    void advanceRead(size_t n) {
        rpos_ += n; if (rpos_ >= cap_) rpos_ -= cap_;
        used_ -= n;
    }
    void advanceWrite(size_t n) {
        wpos_ += n; if (wpos_ >= cap_) wpos_ -= cap_;
        used_ += n;
    }

    // 批量写入（用 memcpy）
    size_t write(const T* src, size_t count) {
        count = std::min(count, freeSize());
        if (count == 0) return 0;
        T* p1; size_t s1; T* p2; size_t s2;
        getWritePointer(count, p1, s1, p2, s2);
        std::memcpy(p1, src, s1 * sizeof(T));
        if (s2 > 0) std::memcpy(p2, src + s1, s2 * sizeof(T));
        advanceWrite(count);
        return count;
    }

    // 批量读取（用 memcpy）
    size_t read(T* dst, size_t count) {
        count = std::min(count, used_);
        if (count == 0) return 0;
        const T* p1; size_t s1; const T* p2; size_t s2;
        getReadPointer(count, p1, s1, p2, s2);
        std::memcpy(dst, p1, s1 * sizeof(T));
        if (s2 > 0) std::memcpy(dst + s1, p2, s2 * sizeof(T));
        advanceRead(count);
        return count;
    }
};
```

### 示例 2：测试回绕行为

```cpp
#include <cstdio>
int main() {
    RingBuffer<int16_t> rb(8);
    int16_t samples[] = {100, 200, 300, 400, 500};
    rb.write(samples, 5);           // 写入5个, 已用=5
    int16_t out[3];
    rb.read(out, 3);                // 读取3个: 100 200 300, 已用=2
    int16_t more[] = {600, 700, 800, 900, 1000, 1100};
    rb.write(more, 6);              // 触发回绕, 已用=8
    int16_t all[8];
    size_t r = rb.read(all, 8);     // 读取: 400 500 600 700 800 900 1000 1100
    for (size_t i = 0; i < r; ++i) printf("%d ", all[i]);
    return 0;
}
```

### 用分段指针填充 OpenAL Buffer

```cpp
void fillALBuffer(ALuint alBuf, RingBuffer<int16_t>& ring,
                  size_t samples, ALenum fmt, ALsizei rate) {
    const int16_t* p1; size_t s1;
    const int16_t* p2; size_t s2;
    ring.getReadPointer(samples, p1, s1, p2, s2);
    if (s2 == 0) {
        // 数据连续，直接提交（零拷贝）
        alBufferData(alBuf, fmt, p1, s1 * sizeof(int16_t), rate);
    } else {
        // 跨边界，拼接到临时缓冲区
        std::vector<int16_t> tmp(samples);
        std::memcpy(tmp.data(), p1, s1 * sizeof(int16_t));
        std::memcpy(tmp.data() + s1, p2, s2 * sizeof(int16_t));
        alBufferData(alBuf, fmt, tmp.data(),
                     samples * sizeof(int16_t), rate);
    }
    ring.advanceRead(samples);
}
```

## KrKr2 tRisaRingBuffer 源码解析

KrKr2 在 `cpp/core/sound/RingBuffer.h` 中提供 `tRisaRingBuffer`，来自吉里吉里3（Risa）架构。成员变量与我们的 `RingBuffer` 完全一致：`Buffer`（数组）、`Size`（容量）、`WritePos`/`ReadPos`（读写指针）、`DataSize`（独立计数器）。

### GetReadPointer — 分段读取（第 77-97 行）

```cpp
void GetReadPointer(size_t readsize,
                    const T*& p1, size_t& p1size,
                    const T*& p2, size_t& p2size,
                    ptrdiff_t offset = 0) {
    size_t pos = ReadPos + offset;
    while (pos >= Size) pos -= Size;  // offset 可能导致多次回绕
    if (readsize + pos > Size) {
        p1 = pos + Buffer;  p1size = Size - pos;
        p2 = Buffer;        p2size = readsize - p1size;
    } else {
        p1 = pos + Buffer;  p1size = readsize;
        p2 = nullptr;       p2size = 0;
    }
}
```

`offset` 参数可窥视 `ReadPos + offset` 处数据（不消费），用于 DSP 滤波。**不检查越界**，调用者须先确认数据充足。

### AdvanceWritePosWithDiscard — 溢出丢弃（第 184-192 行）

```cpp
void AdvanceWritePosWithDiscard(size_t advance = 1) {
    WritePos += advance;
    if (WritePos >= Size) WritePos -= Size;
    DataSize += advance;
    if (DataSize > Size) {
        AdvanceReadPos(DataSize - Size);  // 丢弃最旧数据
    }
}
```

当解码速度 > 播放速度、缓冲区满时，**丢弃最旧数据**保证写入不阻塞。

### 设计总结

| 特点 | tRisaRingBuffer | 说明 |
|------|-----------------|------|
| 模板参数 | `template <typename T>` | int16_t / float / 结构体均可 |
| 溢出策略 | `WithDiscard` | 覆盖旧数据，不阻塞 |
| 线程安全 | 无 | 调用者负责同步 |
| 指针接口 | 分段两指针 | 零拷贝设计 |
| 随机访问 | `GetAt(n)` | 不移动读指针，DSP 滤波用 |
| 大小限制 | 任意值 | 不要求 2 的幂 |

## 双线程音频管线实战

将环形缓冲区插入解码-播放管线：解码线程 → `write()` → RingBuffer → `read()` → 播放线程填充 AL Buffer Queue。

### 示例 3：无锁 SPSC 环形缓冲区 + 双线程播放

```cpp
#include <AL/al.h>
#include <AL/alc.h>
#include <atomic>
#include <thread>
#include <cstring>
#include <cstdio>
#include <cmath>
#include <algorithm>

template <typename T>
class SPSCRingBuffer {
    T* buf_;
    size_t cap_;
    std::atomic<size_t> used_{0};  // 原子计数：唯一跨线程同步点
    size_t rpos_ = 0, wpos_ = 0;  // 各自仅由一个线程修改
public:
    explicit SPSCRingBuffer(size_t c) : cap_(c) { buf_ = new T[c]; }
    ~SPSCRingBuffer() { delete[] buf_; }
    size_t dataSize() const { return used_.load(std::memory_order_acquire); }
    size_t freeSize() const { return cap_ - dataSize(); }

    size_t write(const T* src, size_t n) {  // 仅生产者调用
        n = std::min(n, freeSize());
        size_t first = std::min(n, cap_ - wpos_);
        std::memcpy(buf_ + wpos_, src, first * sizeof(T));
        if (n > first)
            std::memcpy(buf_, src + first, (n - first) * sizeof(T));
        wpos_ = (wpos_ + n) % cap_;
        used_.fetch_add(n, std::memory_order_release);
        return n;
    }
    size_t read(T* dst, size_t n) {  // 仅消费者调用
        n = std::min(n, dataSize());
        size_t first = std::min(n, cap_ - rpos_);
        std::memcpy(dst, buf_ + rpos_, first * sizeof(T));
        if (n > first)
            std::memcpy(dst + first, buf_, (n - first) * sizeof(T));
        rpos_ = (rpos_ + n) % cap_;
        used_.fetch_sub(n, std::memory_order_release);
        return n;
    }
};

constexpr int RATE = 44100, CHUNK = 4096, AL_BUFS = 4;
static std::atomic<bool> g_done{false};

void decoderThread(SPSCRingBuffer<int16_t>& ring) {
    double phase = 0.0;
    int remain = RATE * 3;
    int16_t chunk[1024];
    while (remain > 0) {
        int n = std::min(1024, remain);
        for (int i = 0; i < n; ++i) {
            chunk[i] = (int16_t)(sin(phase * 2.0 * M_PI) * 16000);
            phase += 440.0 / RATE;
        }
        while (ring.freeSize() < (size_t)n && !g_done)
            std::this_thread::sleep_for(std::chrono::milliseconds(1));
        ring.write(chunk, n);
        remain -= n;
    }
    g_done = true;
}

int main() {
    ALCdevice* dev = alcOpenDevice(nullptr);
    ALCcontext* ctx = alcCreateContext(dev, nullptr);
    alcMakeContextCurrent(ctx);
    ALuint src; alGenSources(1, &src);
    ALuint bufs[AL_BUFS]; alGenBuffers(AL_BUFS, bufs);
    SPSCRingBuffer<int16_t> ring(RATE / 2);  // 0.5秒缓冲
    std::thread decoder(decoderThread, std::ref(ring));

    while (ring.dataSize() < CHUNK * AL_BUFS)  // 等预填充
        std::this_thread::sleep_for(std::chrono::milliseconds(5));
    int16_t tmp[CHUNK];
    for (int i = 0; i < AL_BUFS; ++i) {
        ring.read(tmp, CHUNK);
        alBufferData(bufs[i], AL_FORMAT_MONO16, tmp,
                     CHUNK * sizeof(int16_t), RATE);
        alSourceQueueBuffers(src, 1, &bufs[i]);
    }
    alSourcePlay(src);

    while (!g_done || ring.dataSize() > 0) {  // 播放循环
        ALint processed;
        alGetSourcei(src, AL_BUFFERS_PROCESSED, &processed);
        while (processed-- > 0) {
            ALuint buf; alSourceUnqueueBuffers(src, 1, &buf);
            size_t got = ring.read(tmp, CHUNK);
            if (got > 0) {
                alBufferData(buf, AL_FORMAT_MONO16, tmp,
                             got * sizeof(int16_t), RATE);
                alSourceQueueBuffers(src, 1, &buf);
            }
        }
        ALint state; alGetSourcei(src, AL_SOURCE_STATE, &state);
        if (state != AL_PLAYING) {
            ALint queued; alGetSourcei(src, AL_BUFFERS_QUEUED, &queued);
            if (queued > 0) alSourcePlay(src);
        }
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }
    decoder.join();
    ALint state;
    do { std::this_thread::sleep_for(std::chrono::milliseconds(50));
         alGetSourcei(src, AL_SOURCE_STATE, &state);
    } while (state == AL_PLAYING);
    alDeleteSources(1, &src); alDeleteBuffers(AL_BUFS, bufs);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(ctx); alcCloseDevice(dev);
    printf("播放完成\n");
    return 0;
}
```

`rpos_` 仅消费者修改、`wpos_` 仅生产者修改，`used_` 用 `std::atomic` 保证跨线程可见性——无需互斥锁。

## 常见错误与解决方案

### 错误 1：忘记处理回绕导致越界

```cpp
// 错误：直接 memcpy 可能越界（rpos_+count > cap_）
memcpy(dst, buf_ + rpos_, count * sizeof(T));
```

**解决：** 判断是否跨边界，分两次拷贝。参考示例 1 的 `read` 实现。

### 错误 2：多线程竞争 DataSize

```cpp
void write(...) { used_ += n; }  // 写线程
void read(...)  { used_ -= n; }  // 读线程 → 数据竞争！
```

**解决：** 用 `std::atomic` 或 `std::mutex`。SPSC 场景推荐原子操作（示例 3）。

### 错误 3：满/空状态混淆

`ReadPos == WritePos` 时无法区分全满/全空。**解决：** 使用独立 `DataSize` 计数器（KrKr2 的方案），不靠指针差值判断。

### 题目 1：容量计算

环形缓冲区需缓存 200ms 的 Stereo 16-bit 48000Hz PCM。用 `tRisaRingBuffer<int16_t>` 时 size 参数应为多少？

<details>
<summary>查看答案</summary>

每秒样本数 = 48000 × 2（声道）= 96000 个 int16_t
200ms = 0.2s → 96000 × 0.2 = **19200**
字节数 = 19200 × 2 = 38400 字节。传入 `tRisaRingBuffer<int16_t>(19200)`。

</details>

### 题目 2：分段读取

用 `tRisaRingBuffer::GetReadPointer` 将环形缓冲区中的数据拷贝到连续数组：

<details>
<summary>查看答案</summary>

```cpp
void copyFromRing(tRisaRingBuffer<int16_t>& ring,
                  int16_t* out, size_t count) {
    if (count > ring.GetDataSize()) count = ring.GetDataSize();
    if (count == 0) return;
    const int16_t* p1; size_t s1;
    const int16_t* p2; size_t s2;
    ring.GetReadPointer(count, p1, s1, p2, s2);
    std::memcpy(out, p1, s1 * sizeof(int16_t));
    if (p2 && s2 > 0)
        std::memcpy(out + s1, p2, s2 * sizeof(int16_t));
    ring.AdvanceReadPos(count);
}
```

</details>

### 题目 3：溢出丢弃分析

```cpp
tRisaRingBuffer<int> ring(4);
// 写入 3 个
ring.GetLast() = 10; ring.AdvanceWritePosWithDiscard();
ring.GetLast() = 20; ring.AdvanceWritePosWithDiscard();
ring.GetLast() = 30; ring.AdvanceWritePosWithDiscard();
// 再写入 3 个
ring.GetLast() = 40; ring.AdvanceWritePosWithDiscard();
ring.GetLast() = 50; ring.AdvanceWritePosWithDiscard();
ring.GetLast() = 60; ring.AdvanceWritePosWithDiscard();
```

最终 DataSize、ReadPos 各是多少？缓冲区逻辑内容是什么？

<details>
<summary>查看答案</summary>

- 写40：W=3→0, DS=4（满），R=0
- 写50：W=0→1, DS=5>4 → AdvanceReadPos(1)，R→1, DS=4
- 写60：W=1→2, DS=5>4 → AdvanceReadPos(1)，R→2, DS=4

**DataSize=4, ReadPos=2, WritePos=2**。物理 `[50,60,30,40]`，从 R=2 读逻辑顺序：**30, 40, 50, 60**（10、20 被丢弃）。

</details>
