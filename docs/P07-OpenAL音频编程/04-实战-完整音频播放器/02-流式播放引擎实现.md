## 流式播放引擎架构

静态缓冲（Ch02）适合短音效，大文件（BGM、语音）必须流式播放。流式引擎的核心思路：

```
┌─────────────┐    环形缓冲区    ┌─────────────┐    OpenAL 队列    ┌──────────┐
│  解码线程    │ ──→ RingBuffer ──→│  填充线程    │ ──→ Buffer Queue ──→│  硬件输出 │
│ (WAV/FFmpeg) │                  │ (主循环)     │                    │ (扬声器)  │
└─────────────┘                  └─────────────┘                    └──────────┘
```

**为什么需要两级缓冲？**

1. **解码速度不均匀** — FFmpeg 解码一帧可能瞬间产出大量 PCM，也可能因 I/O 阻塞暂停。环形缓冲区吸收这种波动
2. **OpenAL 队列有限** — 通常只排队 3-4 个 Buffer，填满就要等处理完。如果解码直接写队列，解码线程会频繁阻塞
3. **线程解耦** — 解码线程只管往 RingBuffer 写，填充线程只管从 RingBuffer 读并提交给 OpenAL，两者互不阻塞

### 与 KrKr2 的对照

KrKr2 的 `tTVPSoundBufferAL`（`WaveMixer.cpp` 第 489-701 行）采用了简化方案：没有单独的解码线程，而是在 `AppendBuffer` 回调中直接从解码器读取数据填入 OpenAL Buffer。这对于 KrKr2 的使用场景（视觉小说的 BGM/SE）足够高效，但通用播放器需要更健壮的架构。

## 示例1：最简流式引擎骨架

先实现一个不带环形缓冲区的最简版本，直接在主循环中从文件读取并填入 OpenAL 队列：

```cpp
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <vector>
#include <thread>
#include <chrono>

// WavHeader 和 getALFormat 定义见上一节（01-WAV文件加载与PCM解码）
struct WavHeader {
    char riff[4]; uint32_t fileSize; char wave[4]; char fmt[4];
    uint32_t fmtSize; uint16_t audioFormat; uint16_t channels;
    uint32_t sampleRate; uint32_t byteRate; uint16_t blockAlign;
    uint16_t bitsPerSample; char data[4]; uint32_t dataSize;
};

ALenum getALFormat(uint16_t ch, uint16_t bits) {
    if (ch == 1 && bits == 8)  return AL_FORMAT_MONO8;
    if (ch == 1 && bits == 16) return AL_FORMAT_MONO16;
    if (ch == 2 && bits == 8)  return AL_FORMAT_STEREO8;
    if (ch == 2 && bits == 16) return AL_FORMAT_STEREO16;
    return 0;
}

static constexpr int NUM_BUFFERS = 4;       // 队列中缓冲区数量
static constexpr int BUFFER_SAMPLES = 8192; // 每个缓冲区的采样帧数

int main() {
    FILE* file = fopen("test.wav", "rb");
    if (!file) { fprintf(stderr, "无法打开 test.wav\n"); return 1; }

    WavHeader hdr;
    fread(&hdr, sizeof(WavHeader), 1, file);
    ALenum format = getALFormat(hdr.channels, hdr.bitsPerSample);
    int bytesPerFrame = hdr.channels * (hdr.bitsPerSample / 8);
    int bufferBytes   = BUFFER_SAMPLES * bytesPerFrame;

    // 初始化 OpenAL
    ALCdevice*  device  = alcOpenDevice(nullptr);
    ALCcontext* context = alcCreateContext(device, nullptr);
    alcMakeContextCurrent(context);
    ALuint source;
    alGenSources(1, &source);
    ALuint buffers[NUM_BUFFERS];
    alGenBuffers(NUM_BUFFERS, buffers);

    // 预填充所有缓冲区
    std::vector<char> pcm(bufferBytes);
    bool eof = false;
    for (int i = 0; i < NUM_BUFFERS && !eof; ++i) {
        size_t bytesRead = fread(pcm.data(), 1, bufferBytes, file);
        if (bytesRead == 0) { eof = true; break; }
        alBufferData(buffers[i], format, pcm.data(), (ALsizei)bytesRead, hdr.sampleRate);
        alSourceQueueBuffers(source, 1, &buffers[i]);
    }
    alSourcePlay(source);
    printf("开始流式播放...\n");

    // 流式填充主循环
    while (true) {
        ALint processed = 0;
        alGetSourcei(source, AL_BUFFERS_PROCESSED, &processed);
        while (processed-- > 0) {
            ALuint buf;
            alSourceUnqueueBuffers(source, 1, &buf);
            if (!eof) {
                size_t bytesRead = fread(pcm.data(), 1, bufferBytes, file);
                if (bytesRead > 0) {
                    alBufferData(buf, format, pcm.data(), (ALsizei)bytesRead, hdr.sampleRate);
                    alSourceQueueBuffers(source, 1, &buf);
                } else { eof = true; }
            }
        }
        ALint state;
        alGetSourcei(source, AL_SOURCE_STATE, &state);
        if (eof && state != AL_PLAYING) break;

        // Underrun 恢复
        ALint queued = 0;
        alGetSourcei(source, AL_BUFFERS_QUEUED, &queued);
        if (state != AL_PLAYING && queued > 0 && !eof) {
            printf("检测到 underrun，重新启动播放\n");
            alSourcePlay(source);
        }
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
    }
    printf("播放结束\n");

    // 清理
    alDeleteSources(1, &source);
    alDeleteBuffers(NUM_BUFFERS, buffers);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(context);
    alcCloseDevice(device);
    fclose(file);
    return 0;
}
```

**关键流程解析：**

| 步骤 | 说明 |
|------|------|
| 预填充 | 启动前把 4 个 Buffer 全部装满数据并入队，避免播放开始时的空白 |
| 轮询回收 | `AL_BUFFERS_PROCESSED` 获取已播放完毕的 Buffer 数量 |
| 重填入队 | `alSourceUnqueueBuffers` → 读文件 → `alBufferData` → `alSourceQueueBuffers` |
| Underrun 恢复 | 如果所有 Buffer 都播完但文件还没读完，说明填充太慢，需要重新 `alSourcePlay` |
| 结束判断 | 文件读完（`eof`）且 Source 不再播放，退出循环 |

### 常见错误1：忘记 Underrun 恢复

OpenAL 的 Source 在队列中所有 Buffer 都播放完后会自动切换到 `AL_STOPPED` 状态。如果填充循环因 I/O 延迟暂时跟不上播放速度，队列清空 → Source 停止 → 后续 `alSourceQueueBuffers` 不会自动恢复播放。**必须** 在检测到"有排队 Buffer 但 Source 已停止"时手动调用 `alSourcePlay`。

### 常见错误2：Unqueue 正在播放的 Buffer

`alSourceUnqueueBuffers` 只能回收 `AL_BUFFERS_PROCESSED`（已播放完毕）的 Buffer。如果试图回收尚在播放中的 Buffer，OpenAL 会返回 `AL_INVALID_VALUE` 错误。始终先检查 `processed` 数量。

## 示例2：带环形缓冲区的双线程引擎

将解码与播放分离到两个线程，用 SPSC（Single-Producer Single-Consumer）环形缓冲区连接：

```
解码线程（Producer）            环形缓冲区              填充线程（Consumer）
┌───────────────┐         ┌──────────────┐         ┌───────────────┐
│ fread(wav)    │ ──写──→ │ RingBuffer   │ ──读──→ │ alBufferData  │
│ 或 FFmpeg解码 │         │ (lock-free)  │         │ alSourceQueue │
└───────────────┘         └──────────────┘         └───────────────┘
```

```cpp
#include <AL/al.h>
#include <AL/alc.h>
#include <cstdio>
#include <cstdint>
#include <atomic>
#include <thread>
#include <vector>
#include <chrono>

// ===== SPSC 环形缓冲区（完整实现见 Ch03-03，此处精简版）=====
class SPSCRingBuffer {
    std::vector<char> buf_;
    size_t cap_;
    std::atomic<size_t> head_{0}, tail_{0};
public:
    explicit SPSCRingBuffer(size_t capacity) : buf_(capacity), cap_(capacity) {}
    size_t writeAvailable() const {
        return cap_ - 1 - ((head_.load(std::memory_order_relaxed)
                          - tail_.load(std::memory_order_acquire) + cap_) % cap_);
    }
    size_t readAvailable() const {
        return (head_.load(std::memory_order_acquire)
              - tail_.load(std::memory_order_relaxed) + cap_) % cap_;
    }
    size_t write(const char* data, size_t len) {
        size_t avail = writeAvailable();
        if (len > avail) len = avail;
        size_t h = head_.load(std::memory_order_relaxed);
        for (size_t i = 0; i < len; ++i) buf_[(h + i) % cap_] = data[i];
        head_.store((h + len) % cap_, std::memory_order_release);
        return len;
    }
    size_t read(char* dst, size_t len) {
        size_t avail = readAvailable();
        if (len > avail) len = avail;
        size_t t = tail_.load(std::memory_order_relaxed);
        for (size_t i = 0; i < len; ++i) dst[i] = buf_[(t + i) % cap_];
        tail_.store((t + len) % cap_, std::memory_order_release);
        return len;
    }
};

// ===== WAV 头部（同示例1）=====
struct WavHeader {
    char riff[4]; uint32_t fileSize; char wave[4];
    char fmt[4];  uint32_t fmtSize;  uint16_t audioFormat;
    uint16_t channels; uint32_t sampleRate; uint32_t byteRate;
    uint16_t blockAlign; uint16_t bitsPerSample;
    char data[4]; uint32_t dataSize;
};

ALenum getALFormat(uint16_t ch, uint16_t bits) {
    if (ch == 1 && bits == 8)  return AL_FORMAT_MONO8;
    if (ch == 1 && bits == 16) return AL_FORMAT_MONO16;
    if (ch == 2 && bits == 8)  return AL_FORMAT_STEREO8;
    if (ch == 2 && bits == 16) return AL_FORMAT_STEREO16;
    return 0;
}

static constexpr int NUM_BUFFERS    = 4;
static constexpr int BUFFER_SAMPLES = 4096;
// 环形缓冲区大小 = 能容纳约 0.5 秒的 PCM 数据（44100Hz × 2ch × 2byte × 0.5s ≈ 88200）
static constexpr size_t RING_SIZE   = 131072; // 128KB，2的幂便于取模

int main() {
    FILE* file = fopen("test.wav", "rb");
    if (!file) { fprintf(stderr, "无法打开文件\n"); return 1; }
    WavHeader hdr;
    fread(&hdr, sizeof(hdr), 1, file);

    ALenum format = getALFormat(hdr.channels, hdr.bitsPerSample);
    int bytesPerFrame = hdr.channels * (hdr.bitsPerSample / 8);
    int bufferBytes   = BUFFER_SAMPLES * bytesPerFrame;

    // 共享状态
    SPSCRingBuffer ring(RING_SIZE);
    std::atomic<bool> decodeDone{false};

    // --- 解码线程 ---
    std::thread decoder([&]() {
        std::vector<char> tmp(4096);
        while (true) {
            size_t n = fread(tmp.data(), 1, tmp.size(), file);
            if (n == 0) break; // EOF
            size_t written = 0;
            while (written < n) {
                size_t w = ring.write(tmp.data() + written, n - written);
                written += w;
                if (w == 0) {
                    // 缓冲区满，等待消费者读取
                    std::this_thread::sleep_for(std::chrono::milliseconds(1));
                }
            }
        }
        decodeDone.store(true, std::memory_order_release);
        fclose(file);
        printf("[解码线程] 解码完成\n");
    });

    // OpenAL 初始化
    ALCdevice*  device  = alcOpenDevice(nullptr);
    ALCcontext* context = alcCreateContext(device, nullptr);
    alcMakeContextCurrent(context);
    ALuint source;
    alGenSources(1, &source);
    ALuint buffers[NUM_BUFFERS];
    alGenBuffers(NUM_BUFFERS, buffers);

    // 预填充：等待环形缓冲区有足够数据
    std::vector<char> pcm(bufferBytes);
    int queued = 0;
    while (queued < NUM_BUFFERS) {
        if (ring.readAvailable() >= (size_t)bufferBytes) {
            ring.read(pcm.data(), bufferBytes);
            alBufferData(buffers[queued], format, pcm.data(), bufferBytes, hdr.sampleRate);
            alSourceQueueBuffers(source, 1, &buffers[queued]);
            ++queued;
        } else if (decodeDone.load(std::memory_order_acquire)) {
            size_t remain = ring.readAvailable();
            if (remain > 0) {
                ring.read(pcm.data(), remain);
                alBufferData(buffers[queued], format, pcm.data(), (ALsizei)remain, hdr.sampleRate);
                alSourceQueueBuffers(source, 1, &buffers[queued++]);
            }
            break;
        } else { std::this_thread::sleep_for(std::chrono::milliseconds(1)); }
    }
    alSourcePlay(source);
    printf("[主线程] 预填充 %d 个 Buffer，开始播放\n", queued);

    // 流式填充循环
    bool streamDone = false;
    while (true) {
        ALint processed = 0;
        alGetSourcei(source, AL_BUFFERS_PROCESSED, &processed);
        while (processed-- > 0 && !streamDone) {
            ALuint buf;
            alSourceUnqueueBuffers(source, 1, &buf);
            if (ring.readAvailable() >= (size_t)bufferBytes) {
                ring.read(pcm.data(), bufferBytes);
                alBufferData(buf, format, pcm.data(), bufferBytes, hdr.sampleRate);
                alSourceQueueBuffers(source, 1, &buf);
            } else if (decodeDone.load(std::memory_order_acquire)) {
                size_t remain = ring.readAvailable();
                if (remain > 0) {
                    ring.read(pcm.data(), remain);
                    alBufferData(buf, format, pcm.data(), (ALsizei)remain, hdr.sampleRate);
                    alSourceQueueBuffers(source, 1, &buf);
                }
                streamDone = true;
            }
            // 数据不够且解码未完成 → 不入队，下轮重试
        }
        ALint state;
        alGetSourcei(source, AL_SOURCE_STATE, &state);
        if (streamDone && state != AL_PLAYING) break;
        // Underrun 恢复
        ALint q = 0;
        alGetSourcei(source, AL_BUFFERS_QUEUED, &q);
        if (state != AL_PLAYING && q > 0 && !streamDone) {
            printf("[主线程] underrun 恢复\n");
            alSourcePlay(source);
        }
        std::this_thread::sleep_for(std::chrono::milliseconds(5));
    }
    printf("[主线程] 播放结束\n");
    decoder.join();

    alDeleteSources(1, &source);
    alDeleteBuffers(NUM_BUFFERS, buffers);
    alcMakeContextCurrent(nullptr);
    alcDestroyContext(context);
    alcCloseDevice(device);
    return 0;
}
```

### 双线程引擎的时序

```
时间 ──→

解码线程:  [读WAV][读WAV][读WAV]...[读WAV][EOF] ──→ decodeDone=true
              ↓写     ↓写     ↓写         ↓写
环形缓冲:  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░  (▓=有数据 ░=空闲)
              ↓读     ↓读         ↓读       ↓读
主线程:    [预填4][play][回收→填→入队][回收→填→入队]...[结束]
```

两个线程通过环形缓冲区完全解耦：解码线程全速读文件写缓冲；主线程以 OpenAL 的节奏从缓冲区取数据。即使解码线程因磁盘 I/O 暂停几十毫秒，只要环形缓冲区有足够余量，播放不会中断。

### 常见错误3：环形缓冲区大小不对齐采样帧

如果 `RING_SIZE` 不是 `bytesPerFrame`（如 16-bit 立体声为 4 字节）的整数倍，可能读出半个采样帧提交给 `alBufferData`，导致噪音或错误。解决方案：

```cpp
// 保证缓冲区容量是帧大小的整数倍
size_t ringSize = 131072;
ringSize -= ringSize % bytesPerFrame; // 向下对齐
```

## 对照项目源码

KrKr2 的流式播放实现在 `cpp/core/sound/win32/WaveMixer.cpp` 中：

**`tTVPSoundBufferAL::AppendBuffer`（第 582-625 行）核心逻辑：**

```cpp
// 简化后的 AppendBuffer 伪代码
void tTVPSoundBufferAL::AppendBuffer() {
    ALint processed;
    alGetSourcei(_alSource, AL_BUFFERS_PROCESSED, &processed);
    while (processed-- > 0) {
        ALuint buf;
        alSourceUnqueueBuffers(_alSource, 1, &buf);
        int decoded = decoder->Decode(tmpBuffer, bufferSize);
        if (decoded > 0) {
            alBufferData(buf, _alFormat, tmpBuffer, decoded, sampleRate);
            alSourceQueueBuffers(_alSource, 1, &buf);
        }
    }
    // underrun 恢复
    ALint state;
    alGetSourcei(_alSource, AL_SOURCE_STATE, &state);
    if (state != AL_PLAYING) {
        ALint queued;
        alGetSourcei(_alSource, AL_BUFFERS_QUEUED, &queued);
        if (queued > 0) alSourcePlay(_alSource);
    }
}
```

**对比我们的实现与 KrKr2：**

| 特性 | 双线程引擎 | KrKr2 |
|------|-----------|-------|
| 线程模型 | 解码线程 + 主线程 | 单线程回调 |
| 缓冲策略 | 环形缓冲区解耦 | 直接解码到 AL Buffer |
| Underrun 恢复 | ✅ | ✅（同样逻辑）|
| 适用场景 | 通用/高开销解码 | 视觉小说 BGM/SE |

KrKr2 的简化方案对视觉小说足够，但高开销解码或网络流场景下双线程架构更可靠。

## 本节小结

- 流式播放的核心循环：**轮询 → 回收 → 填充 → 入队**，配合 underrun 恢复
- 单线程方案简单直接，适合解码速度远超播放速度的场景（如 WAV/PCM）
- 双线程 + 环形缓冲区方案将解码与播放完全解耦，适合高开销解码或网络流
- KrKr2 采用单线程回调方案，`AppendBuffer` 中直接从解码器读取数据填入 OpenAL 队列
- Underrun 恢复是**必须**实现的——没有它，任何短暂的填充延迟都会导致播放永久停止

## 练习题与答案

### 题目1：为什么预填充阶段要用 4 个 Buffer 而不是 1 个？

<details>
<summary>查看答案</summary>

预填充多个 Buffer 实现**流水线效率**：OpenAL 播放 Buffer 0 时，1/2/3 在队列中等待，给主线程留出回收和重填的时间。只用 1 个 Buffer 时，播放完毕后 Source 立即停止，主线程必须在极短时间窗口（4096 采样 ÷ 44100Hz ≈ 93ms）内完成"回收→读→填→入队"全流程，非常容易 underrun。4 个 Buffer 窗口 ≈ 372ms，足以应对 I/O 波动。

</details>

### 题目2：如果解码速度远慢于播放速度（如网络流），环形缓冲区应设多大？

<details>
<summary>查看答案</summary>

环形缓冲区大小 = `最大延迟(秒) × 采样率 × 声道数 × 每采样字节数`。例如 44100Hz 立体声 16-bit、最大延迟 2 秒：`2 × 44100 × 2 × 2 = 352800 字节 ≈ 345KB`，取 2 的幂向上对齐为 512KB。经验法则：0.5-2 秒的音频容量。太小频繁 underrun，太大浪费内存且增加暂停/跳转时的丢弃开销。

</details>

### 题目3：请修改双线程引擎，增加一个 `std::atomic<bool> paused` 标志，实现暂停/恢复功能

<details>
<summary>查看答案</summary>

```cpp
std::atomic<bool> paused{false}; // 在 main() 中添加

// 在流式填充循环中添加暂停检测（sleep_for 之前）
if (paused.load(std::memory_order_relaxed)) {
    ALint state;
    alGetSourcei(source, AL_SOURCE_STATE, &state);
    if (state == AL_PLAYING) alSourcePause(source);
    std::this_thread::sleep_for(std::chrono::milliseconds(50));
    continue;
}
// 恢复检测
ALint state;
alGetSourcei(source, AL_SOURCE_STATE, &state);
if (state == AL_PAUSED) alSourcePlay(source);
```

关键：用 `alSourcePause`（不是 `alSourceStop`），恢复时从中断点继续。解码线程不需暂停——继续填缓冲区，恢复时立即有数据。`alSourceStop` 会重置队列状态，需重新预填充。
