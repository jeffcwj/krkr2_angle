## 什么是递归下降

**递归下降解析器（Recursive Descent Parser）** 是最直观的语法分析方法：为文法中的每个**非终结符**写一个函数，函数体中按照产生式规则消耗 Token。函数之间的相互调用形成递归，天然处理嵌套结构。

核心思想极其简单——**EBNF 中的每个元素对应一种代码结构**：

| EBNF 元素 | 代码结构 |
|-----------|----------|
| 终结符 `"+"` | `expect(TOKEN_PLUS)` |
| 非终结符 `<expr>` | `parseExpr()` 函数调用 |
| 选择 `A \| B` | `if/else` 或 `switch` |
| 可选 `[A]` | `if (当前Token匹配A)` |
| 重复 `{A}` | `while (当前Token匹配A)` |
| 序列 `A B C` | 依次调用 |

### 最小示例：解析括号嵌套

文法：`S → '(' S ')' | 'x'`（生成 `x`、`(x)`、`((x))`...）

```cpp
#include <iostream>
#include <string>
#include <stdexcept>

class ParenParser {
    std::string input_;
    size_t pos_ = 0;
    char peek() const { return pos_ < input_.size() ? input_[pos_] : '\0'; }
    char advance() { return input_[pos_++]; }
    void expect(char ch) {
        if(advance() != ch) throw std::runtime_error(std::string("期望 '") + ch + "'");
    }
public:
    explicit ParenParser(std::string input) : input_(std::move(input)) {}
    void parseS() {             // S → '(' S ')' | 'x'
        if(peek() == '(') { advance(); parseS(); expect(')'); }  // 递归处理嵌套
        else { expect('x'); }                                     // 基础情况
    }
    void parse() {
        parseS();
        if(pos_ != input_.size()) throw std::runtime_error("输入未完全消耗");
        std::cout << "解析成功！\n";
    }
};

int main() {
    ParenParser("((x))").parse();   // 输出：解析成功！
    // ParenParser("(x").parse();   // 抛出：期望 ')'
    return 0;
}
```

注意 `parseS()` 通过查看**当前字符**（`peek()`）决定走哪个分支——这就是**预测分析**的核心：**用一个前看符号（Lookahead）决定产生式选择**。

## 左递归问题

递归下降有一个致命弱点：**无法处理左递归**。

考虑算术表达式的经典文法：

```
Expr → Expr + Term | Term
```

翻译成代码：

```cpp
// ⚠️ 这段代码会无限递归！
void parseExpr() {
    parseExpr();    // 第一步就递归调用自身，永不终止
    expect('+');
    parseTerm();
}
```

`parseExpr()` 在没有消耗任何 Token 的情况下就调用了自身——死循环。

### 消除左递归

将左递归 `A → A α | β` 转换为右递归：`A → β A'`、`A' → α A' | ε`。用 EBNF 更简洁：`A → β { α }`。应用到算术表达式：

```
原始（左递归）：  Expr → Expr + Term | Expr - Term | Term
EBNF（无左递归）：Expr → Term { ("+" | "-") Term }
```

同理 `Term → Factor { ("*" | "/") Factor }`，`Factor → "(" Expr ")" | number`。`{ }` 映射为 `while` 循环——天然消除左递归。

## 从 EBNF 到代码：四则运算计算器

这个例子实现一个完整的四则运算解析器，支持 `+`、`-`、`*`、`/`、括号和整数。

### 词法器（复用上一章的设计）

```cpp
#include <iostream>
#include <string>
#include <stdexcept>
#include <cctype>

enum class TokenType { Number, Plus, Minus, Star, Slash, LParen, RParen, Eof };

struct Token { TokenType type; int value; };  // value 仅 Number 有效

class Lexer {
    std::string src_;
    size_t pos_ = 0;
public:
    explicit Lexer(std::string src) : src_(std::move(src)) {}
    Token next() {
        while(pos_ < src_.size() && std::isspace(src_[pos_])) pos_++;
        if(pos_ >= src_.size()) return {TokenType::Eof, 0};
        char ch = src_[pos_++];
        switch(ch) {
            case '+': return {TokenType::Plus, 0};   case '-': return {TokenType::Minus, 0};
            case '*': return {TokenType::Star, 0};   case '/': return {TokenType::Slash, 0};
            case '(': return {TokenType::LParen, 0}; case ')': return {TokenType::RParen, 0};
            default:
                if(std::isdigit(ch)) {
                    int val = ch - '0';
                    while(pos_ < src_.size() && std::isdigit(src_[pos_]))
                        val = val * 10 + (src_[pos_++] - '0');
                    return {TokenType::Number, val};
                }
                throw std::runtime_error(std::string("非法字符: ") + ch);
        }
    }
};
```

### 递归下降解析器

```cpp
class Parser {
    Lexer lexer_;
    Token current_;
    void advance() { current_ = lexer_.next(); }
    void expect(TokenType type) {
        if(current_.type != type) throw std::runtime_error("Token 类型不匹配");
        advance();
    }
public:
    explicit Parser(Lexer lexer) : lexer_(std::move(lexer)) { advance(); }

    int parseExpr() {                                      // Expr → Term { ("+" | "-") Term }
        int result = parseTerm();
        while(current_.type == TokenType::Plus || current_.type == TokenType::Minus) {
            TokenType op = current_.type;  advance();
            int right = parseTerm();
            result = (op == TokenType::Plus) ? result + right : result - right;
        }
        return result;
    }

    int parseTerm() {                                      // Term → Factor { ("*" | "/") Factor }
        int result = parseFactor();
        while(current_.type == TokenType::Star || current_.type == TokenType::Slash) {
            TokenType op = current_.type;  advance();
            int right = parseFactor();
            if(op == TokenType::Star) result *= right;
            else { if(right == 0) throw std::runtime_error("除以零"); result /= right; }
        }
        return result;
    }

    int parseFactor() {                                    // Factor → "(" Expr ")" | number
        if(current_.type == TokenType::LParen) {
            advance();  int r = parseExpr();  expect(TokenType::RParen);  return r;
        }
        if(current_.type == TokenType::Number) { int v = current_.value; advance(); return v; }
        throw std::runtime_error("期望数字或 '('");
    }

    int parse() {
        int result = parseExpr();
        if(current_.type != TokenType::Eof) throw std::runtime_error("输入未完全消耗");
        return result;
    }
};

int main() {
    auto test = [](const char* expr, int expected) {
        Parser p(Lexer(expr));
        int result = p.parse();
        std::cout << expr << " = " << result
                  << (result == expected ? " ✓" : " ✗") << "\n";
    };
    test("2 + 3 * 4", 14);       // 乘法优先级高
    test("(2 + 3) * 4", 20);     // 括号改变优先级
    test("10 - 3 - 2", 5);       // 左结合：(10-3)-2 = 5
    test("100 / 10 / 2", 5);     // 左结合：(100/10)/2 = 5
    return 0;
}
```

输出：
```
2 + 3 * 4 = 14 ✓
(2 + 3) * 4 = 20 ✓
10 - 3 - 2 = 5 ✓
100 / 10 / 2 = 5 ✓
```

> **关键观察：** `parseExpr` 中的 `while` 循环天然实现了**左结合**——`1 - 2 - 3` 的计算顺序是 `(1 - 2) - 3`，因为每次循环把之前的结果作为左操作数。

## FIRST 集与 LL(1) 分析

递归下降解析器在每个产生式的选择点需要"看一个 Token 就知道走哪条路"。这种能力的形式化基础是 **FIRST 集**和 **FOLLOW 集**。

### FIRST 集

`FIRST(α)` 是从串 α 能推导出的所有**首终结符**的集合。如果 α 能推导出空串 ε，则 ε ∈ FIRST(α)。

计算规则：
1. 终结符 a：`FIRST(a) = {a}`
2. 非终结符 A 有产生式 `A → α₁ | α₂ | ...`：`FIRST(A) = FIRST(α₁) ∪ FIRST(α₂) ∪ ...`
3. 串 `X₁X₂...Xₙ`：从 X₁ 开始，如果 ε ∈ FIRST(Xᵢ)，继续加入 FIRST(Xᵢ₊₁)

### 示例：计算四则运算文法的 FIRST 集

```
Expr   → Term { ("+" | "-") Term }
Term   → Factor { ("*" | "/") Factor }
Factor → "(" Expr ")" | number
```

```
FIRST(Factor) = { "(", number }
FIRST(Term)   = FIRST(Factor) = { "(", number }
FIRST(Expr)   = FIRST(Term) = { "(", number }
```

### LL(1) 条件

一个文法是 **LL(1)** 的，当且仅当：对于同一非终结符的任意两条产生式 `A → α` 和 `A → β`，`FIRST(α) ∩ FIRST(β) = ∅`（考虑 FOLLOW 集的情况这里省略）。

直观含义：**看一个前看符号就能唯一确定选哪条产生式**——正是递归下降解析器需要的。

上面的四则运算文法是 LL(1) 的——`Factor` 的两个产生式的 FIRST 集 `{(}` 和 `{number}` 不相交。

> **常见错误：** 左递归文法**永远不是 LL(1)**。`Expr → Expr + Term | Term` 中两条产生式的 FIRST 集都包含 `FIRST(Term)`，完全重叠。所以递归下降必须先消除左递归。

## 左因子提取

当两条产生式有**公共前缀**时，预测分析器无法区分：

```
Stmt → if Expr then Stmt else Stmt
     | if Expr then Stmt
```

两条都以 `if` 开头——看到 `if` 时不知道选哪条。

**左因子提取**将公共前缀提出来：

```
Stmt     → if Expr then Stmt StmtTail
StmtTail → else Stmt | ε
```

现在选择点推迟到了 `StmtTail`——看到 `else` 就走第一条，否则走 ε。

## Pratt 解析法：优雅的运算符优先级处理

前面的递归下降解析器为每个优先级层写了一个函数（`parseExpr`、`parseTerm`、`parseFactor`）。如果语言有 15 级运算符（像 C/TJS2），就需要 15 个几乎一样的函数。

**Pratt 解析法（Top-Down Operator Precedence Parsing）** 用一个函数 + 优先级表替代所有运算符层。

核心思想：每个 Token 有两个属性——**前缀解析器**（Null Denotation, nud）和**中缀解析器**（Left Denotation, led），加上一个**绑定力**（Binding Power, bp）。

```cpp
// Pratt 解析器核心（约 30 行）
int parseExpr(int minBp = 0) {
    // 1. 处理前缀（数字、一元运算符、括号）
    Token t = current_;
    advance();
    int left = nud(t);  // 前缀解析

    // 2. 循环处理中缀运算符，直到绑定力不够
    while(bp(current_) > minBp) {
        Token op = current_;
        advance();
        left = led(left, op);  // 中缀解析
    }
    return left;
}

int nud(Token t) {                             // 前缀解析
    if(t.type == TokenType::Number) return t.value;
    if(t.type == TokenType::Minus)  return -parseExpr(90); // 一元负号，高绑定力
    if(t.type == TokenType::LParen) {
        int r = parseExpr(0);                  // 括号内重置绑定力
        expect(TokenType::RParen);
        return r;
    }
    throw std::runtime_error("非法前缀");
}

int led(int left, Token op) {                  // 中缀解析
    int right = parseExpr(bp(op));             // 右操作数的最低绑定力 = 当前运算符的绑定力
    switch(op.type) {
        case TokenType::Plus:  return left + right;
        case TokenType::Minus: return left - right;
        case TokenType::Star:  return left * right;
        case TokenType::Slash: return left / right;
        default: throw std::runtime_error("非法中缀");
    }
}

int bp(Token t) {                              // 绑定力（优先级）
    switch(t.type) {
        case TokenType::Plus: case TokenType::Minus: return 10;
        case TokenType::Star: case TokenType::Slash: return 20;
        default: return 0;                     // 非运算符，绑定力为 0
    }
}
```

**优势：** 添加新运算符只需在 `bp()` 和 `led()` 中加一行——不需要新函数。15 级运算符仍然只有一个 `parseExpr` 函数。

**右结合运算符**（如赋值 `=`）：在 `led` 中递归调用 `parseExpr(bp(op) - 1)` 即可——减 1 使得同级运算符优先右结合。

## 错误处理与恢复

生产级解析器不能在第一个语法错误处就崩溃。常见的错误恢复策略：

### 恐慌模式（Panic Mode）

遇到错误时，跳过 Token 直到找到"同步点"（通常是 `;`、`}`、`)` 等分隔符）：

```cpp
void synchronize() {
    advance();
    while(current_.type != TokenType::Eof) {
        if(current_.type == TokenType::Semicolon) { advance(); return; }
        switch(current_.type) {          // 同步到新语句的开头关键字
            case TokenType::If: case TokenType::While: case TokenType::For:
            case TokenType::Return: case TokenType::Var: return;
            default: advance();
        }
    }
}
```

另一种策略是**错误产生式**——在解析中插入容错逻辑，例如缺少分号时报警但不中断：

```cpp
void parseStmt() {
    parseExprStmt();
    if(current_.type == TokenType::Semicolon) advance();
    else reportError("缺少分号 ';'");  // 报错但假设分号存在，继续
}
```

> **常见错误：** 在错误恢复中消耗了太多或太少 Token。太多会跳过有效代码，产生"幽灵错误"；太少会导致同一个错误被报告多次。TJS2 的做法比较简单——遇到严重错误直接抛出异常终止编译。

## 动手实践

用本节的 `ExprParser` 做以下实验：

1. **扩展文法**：在 `parsePrimary()` 中添加对负数前缀 `-` 的支持（一元运算符）。例如 `-3 + 5` 应返回 `Add(Neg(3), 5)` 的 AST。提示：在 `parsePrimary()` 开头检查 `-` Token，如果匹配就消耗它并递归调用 `parsePrimary()`。

3. **Pratt vs 递归下降对比**：将本节的 `ExprParser` 用 Pratt 解析法重写。对比两种实现的代码行数和新增运算符的难度。Pratt 版本新增一个运算符只需在绑定力表中加一行，递归下降版本需要新加一个函数。

4. **错误恢复测试**：故意给解析器输入错误的表达式（如 `3 + + 5`、`(3 +`、`3 * ()`），观察报错信息。然后实现恐慌模式错误恢复：遇到错误时跳到下一个 `)` 或语句结束符。

## 对照项目源码

TJS2 **没有**使用递归下降——它用 Bison 生成的 LALR(1) 解析器。但理解递归下降对理解 Bison 至关重要：

1. **对比：** 递归下降是自顶向下（从 `Expr` 展开到 `number`），Bison 是自底向上（从 `number` 归约到 `Expr`）。两者识别的语言能力不同——LALR(1) 能处理左递归文法，而递归下降必须先消除。

2. **实际应用：** 很多现代编译器（GCC、Clang、V8、Go）选择手写递归下降而非生成器。原因是递归下降的错误信息更好控制，且用 Pratt 解析法可以高效处理表达式。

3. **TJS2 的选择：** TJS2 选择 Bison 的原因可能是——2000 年代初期 Bison 是主流选择，且 TJS2 的文法相对标准，Bison 能直接处理。

相关文件：

- `cpp/core/tjs2/tjsLex.cpp` `GetNext()` — 词法器产出 Token，供解析器消耗

## 本节小结

- **递归下降**将每个非终结符映射为一个函数，EBNF 的 `|`/`[]`/`{}` 分别对应 `if-else`/`if`/`while`
- **左递归必须消除**——`A → Aα | β` 改写为 `A → β {α}`（EBNF 的 `while` 循环）
- **LL(1)** 要求每个选择点的 FIRST 集不相交——看一个 Token 就能选产生式
- **左因子提取**解决公共前缀导致的预测冲突
- **Pratt 解析法**用绑定力表替代多层函数，一个函数处理所有优先级——适合运算符多的语言
- **错误恢复**用恐慌模式跳到同步点，避免一个错误引发雪崩式报错

### 题目 1：消除左递归

将以下左递归文法转换为 EBNF（适合递归下降的形式）：

```
A → A "," B | A ";" B | B
B → B "." C | C
C → id | "(" A ")"
```

<details>
<summary>查看答案</summary>

```ebnf
A = B { ("," | ";") B } ;
B = C { "." C } ;
C = id | "(" A ")" ;
```

推导过程：
- `A → A "," B | A ";" B | B`：基础 β = B，递归 α₁ = `","B`、α₂ = `";"B` → `A = B { ("," | ";") B }`
- `B → B "." C | C`：基础 β = C，递归 α = `"."C` → `B = C { "." C }`
- `C` 无左递归，直接转换

</details>

### 题目 2：Pratt 解析器扩展

在前面的 Pratt 解析器基础上，添加对 `**`（幂运算，右结合，优先级高于 `*`）的支持。写出需要修改的三个函数。

```cpp
// 1. bp() 中增加幂运算的绑定力
int bp(Token t) {
    switch(t.type) {
        case TokenType::Plus: case TokenType::Minus: return 10;
        case TokenType::Star: case TokenType::Slash: return 20;
        case TokenType::Power: return 30;  // 高于 * /
        default: return 0;
    }
}

// 2. led() 中增加幂运算的处理
int led(int left, Token op) {
    if(op.type == TokenType::Power) {
        int right = parseExpr(bp(op) - 1);  // -1 实现右结合！
        return static_cast<int>(std::pow(left, right));
    }
    int right = parseExpr(bp(op));           // 左结合：不减 1
    // ... 其他运算符同前
}

// 3. Lexer 中识别 ** Token
// 在 case '*' 后检查下一个字符是否也是 '*'
```

关键点：`parseExpr(bp(op) - 1)` 让同级运算符右结合。`2 ** 3 ** 2` 被解析为 `2 ** (3 ** 2) = 512` 而非 `(2 ** 3) ** 2 = 64`。

### 题目 3：FIRST 集计算

计算以下文法中所有非终结符的 FIRST 集：

```
S → A B | c
A → a | ε
B → b | d
```

<details>
<summary>查看答案</summary>

```
FIRST(A) = { a, ε }       — A → a 给出 a，A → ε 给出 ε
FIRST(B) = { b, d }       — B → b 给出 b，B → d 给出 d
FIRST(S):
  S → A B：FIRST(AB) 首先包含 FIRST(A)\{ε} = {a}
           因为 ε ∈ FIRST(A)，还要加入 FIRST(B) = {b, d}
           所以 FIRST(AB) = { a, b, d }
  S → c：FIRST(c) = {c}
  FIRST(S) = { a, b, c, d }
```

验证：看到 `a` → 选 `S → AB`（A 匹配 a）；看到 `b` 或 `d` → 选 `S → AB`（A 选 ε，B 匹配）；看到 `c` → 选 `S → c`。四个首符号各不重叠——是 LL(1) 文法。

</details>
