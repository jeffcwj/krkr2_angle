## 声明部分（第 1-191 行）

### Bison 指令与三大宏

```yacc
// 第 1-6 行：Bison 配置
%require "3.8.2"              // 最低 Bison 版本
%language "C++"               // 生成 C++ 而非 C
%expect 1                     // 精确断言 1 个移进-归约冲突（悬挂 else）
%define api.namespace {TJS}   // 解析器类 → TJS::parser
```

语义动作中反复出现三个宏（第 19-26 行）：

```cpp
#define cc (ptr->GetCurrentContext())   // 当前编译上下文（InterCodeGenerator）
#define cn (cc->GetCurrentNode())       // 当前 AST 节点（用于数组/字典字面量构建）
#define lx (ptr->GetLexicalAnalyzer())  // 词法器实例
```

`ptr` 就是 `%parse-param` 传入的 `tTJSScriptBlock*`——它是整个编译流程的中枢，持有词法器、编译上下文栈、错误计数器等。

### %union 与 Token 声明

```yacc
// 第 32-35 行
%union {
    tjs_int        num;     // 整数（Token 的索引值）
    tTJSExprNode * np;      // AST 节点指针
}
```

只有**两个字段**——这比大多数教科书示例都简洁。`num` 用于存储 Token 在字面量表中的索引（而非 Token 的实际值），`np` 用于所有表达式节点。

Token 声明分三大类：

| 类别 | 行号 | 数量 | 示例 |
|------|------|------|------|
| 运算符（带字符串别名） | 44-98 | ~55 | `T_PLUS "+"`、`T_EQUALEQUAL "=="` |
| 关键字（带字符串别名） | 100-156 | ~40 | `T_IF "if"`、`T_CLASS "class"` |
| 内部 Token（无别名） | 159-176 | ~18 | `T_UPLUS`、`T_UMINUS`、`T_INLINEARRAY` |
| 带值 Token | 177-180 | 4 | `T_CONSTVAL`、`T_SYMBOL`、`T_REGEXP`、`T_VARIANT` |

内部 Token（如 `T_UPLUS`、`T_POSTINCREMENT`）不由词法器产生，而是在语法规则的语义动作中使用——用于区分二元加和一元加。

## 语句层文法（第 193-525 行）

### 顶层结构

```yacc
program : global_list ;

global_list
    :                { ptr->PushContextStack(TJS_W("global"), ctTopLevel); }
      def_list       { ptr->PopContextStack(); }
    ;

def_list
    :                                          /* 空：允许空程序 */
    | def_list block_or_statement              /* 递归：语句序列 */
    | def_list error ";"                       /* 错误恢复 */
    ;
```

入口点 `program` → `global_list` → `def_list`。语义动作在解析开始时推入全局上下文（`ctTopLevel`），解析结束时弹出。`def_list` 是左递归列表——Bison 的 LALR(1) 天然支持。

### statement 规则——TJS2 的所有语句类型

```yacc
statement
    : ";"                                  // 空语句
    | expr ";"         { cc->CreateExprCode($1); }  // 表达式语句
    | if %prec LOWER_ELSE                  // if（无 else）
    | if_else                              // if-else
    | while | do_while | for               // 循环
    | "break" ";"      { cc->DoBreak(); }
    | "continue" ";"   { cc->DoContinue(); }
    | variable_def | func_def | property_def | class_def
    | return | switch | with | case | try | throw
    ;
```

注意 `if %prec LOWER_ELSE`——通过 `%prec` 伪指令将 `if`（无 else）的归约优先级设为 `LOWER_ELSE`，低于 `T_ELSE` 的移进优先级，从而解决悬挂 else 歧义。

### 控制流语句的语义动作模式

所有控制流语句遵循**Enter/Exit 配对**模式：

```yacc
while
    : "while"              { cc->EnterWhileCode(false); }   // 1. 进入
      "(" expr ")"         { cc->CreateWhileExprCode($4, false); } // 2. 条件
      block_or_statement   { cc->ExitWhileCode(false); }    // 3. 退出
    ;
```

`EnterWhileCode` 记录循环起始位置（用于 continue 跳转目标），`CreateWhileExprCode` 生成条件判断的字节码，`ExitWhileCode` 回填 break 跳转地址。这是编译器的**一趟式代码生成**——边解析边生成字节码，无需单独的代码生成遍历。

其他控制流语句完全相同的模式：

| 语句 | Enter | 中间 | Exit |
|------|-------|------|------|
| `while` | `EnterWhileCode(false)` | `CreateWhileExprCode` | `ExitWhileCode(false)` |
| `do-while` | `EnterWhileCode(true)` | `CreateWhileExprCode` | `ExitWhileCode(true)` |
| `if` | `EnterIfCode` | `CreateIfExprCode` | `ExitIfCode` |
| `if-else` | （继承 if）| `EnterElseCode` | `ExitElseCode` |
| `for` | `EnterForCode` | `CreateForExprCode` | `ExitForCode` |
| `switch` | `EnterSwitchCode` | — | `ExitSwitchCode` |
| `try` | `EnterTryCode` | `EnterCatchCode` | `ExitTryCode` |

### 函数与类的上下文栈

函数和类定义使用 `PushContextStack` / `PopContextStack` 创建新的编译上下文：

```yacc
func_def
    : "function" T_SYMBOL      { ptr->PushContextStack(
                                     lx->GetString($2), ctFunction);
                                   cc->EnterBlock(); }
      func_decl_arg_opt variable_type
      block                    { cc->ExitBlock();
                                 ptr->PopContextStack(); }
    ;
```

上下文类型（Context Type）决定了字节码生成的目标：

| 类型 | 含义 | 对应 TJS2 语法 |
|------|------|---------------|
| `ctTopLevel` | 全局脚本 | 文件顶层 |
| `ctFunction` | 命名函数 | `function foo() {}` |
| `ctExprFunction` | 匿名函数 | `function() {}` |
| `ctProperty` | 属性 | `property name {}` |
| `ctPropertySetter` | setter | `setter(v) {}` |
| `ctPropertyGetter` | getter | `getter() {}` |
| `ctClass` | 类 | `class Foo {}` |

每个上下文拥有独立的字节码缓冲区、局部变量表和常量池。`PopContextStack` 时，编译好的字节码被封装为可调用对象。

## 表达式层次链（第 527-716 行）

TJS2 表达式优先级通过**文法分层**实现——13 层非终结符形成从低优先级到高优先级的链：

```
expr
  └─ comma_expr              逗号（最低优先级）
       └─ assign_expr        赋值 =, +=, -=, ...（右结合）
            └─ cond_expr     三元 ? :
                 └─ logical_or_expr    ||
                      └─ logical_and_expr   &&
                           └─ inclusive_or_expr   |
                                └─ exclusive_or_expr  ^
                                     └─ and_expr      &
                                          └─ identical_expr  ==, !=, ===, !==
                                               └─ compare_expr  <, >, <=, >=
                                                    └─ shift_expr  >>, <<, >>>
                                                         └─ add_sub_expr  +, -
                                                              └─ mul_div_expr  *, /, %, \
                                                                   └─ unary_expr  !, ~, ++, --, new, ...
                                                                        └─ incontextof_expr
                                                                             └─ priority_expr  [], ., ()
                                                                                  └─ factor_expr  字面量
```

每一层的产生式模式完全一致——"要么向下传递，要么在本层做运算"：

```yacc
// 加减法层（第 624-628 行）
add_sub_expr
    : mul_div_expr                              { $$ = $1; }         // 向下
    | add_sub_expr "+" mul_div_expr             { $$ = cc->MakeNP2(token::T_PLUS, $1, $3); }
    | add_sub_expr "-" mul_div_expr             { $$ = cc->MakeNP2(token::T_MINUS, $1, $3); }
    ;
```

左递归 `add_sub_expr "+" mul_div_expr` 保证**左结合**——`1+2+3` 解析为 `(1+2)+3`。赋值表达式则用右递归保证右结合：

```yacc
// 赋值层（第 544-562 行）——右递归
assign_expr
    : cond_expr                                 { $$ = $1; }
    | cond_expr "=" assign_expr                 { $$ = cc->MakeNP2(token::T_EQUAL, $1, $3); }
    | cond_expr "+=" assign_expr                { $$ = cc->MakeNP2(token::T_PLUSEQUAL, $1, $3); }
    // ... 共 16 种赋值运算符
    ;
```

`cond_expr "=" assign_expr`（而非 `assign_expr "=" cond_expr`）实现右递归——`a = b = c` 解析为 `a = (b = c)`。

### AST 节点工厂方法

所有语义动作都通过四个工厂方法构建 AST：

| 方法 | 参数 | 用途 | 示例 |
|------|------|------|------|
| `MakeNP0(op)` | 0 个子节点 | 叶节点 | 常量、符号、this、global |
| `MakeNP1(op, child)` | 1 个子节点 | 一元运算 | `!x`、`++x`、`typeof x` |
| `MakeNP2(op, left, right)` | 2 个子节点 | 二元运算 | `a+b`、`a=b`、`a.b` |
| `MakeNP3(op, a, b, c)` | 3 个子节点 | 三元运算 | `a ? b : c` |

每个节点存储**操作码**（Token 枚举值）和子节点指针。这些节点在归约时立即创建，形成自底向上的 AST 构建过程。

### factor_expr——叶节点层

```yacc
// 第 695-716 行（简化）
factor_expr
    : T_CONSTVAL       { $$ = cc->MakeNP0(token::T_CONSTVAL);
                         $$->SetValue(lx->GetValue($1)); }  // 常量：数字、字符串
    | T_SYMBOL         { $$ = cc->MakeNP0(token::T_SYMBOL);
                         $$->SetValue(tTJSVariant(lx->GetString($1))); } // 标识符
    | "this"           { $$ = cc->MakeNP0(token::T_THIS); }
    | "super"          { $$ = cc->MakeNP0(token::T_SUPER); }
    | "global"         { $$ = cc->MakeNP0(token::T_GLOBAL); }
    | func_expr_def    { $$ = $1; }             // 匿名函数
    | inline_array     { $$ = $1; }             // [a, b, c]
    | inline_dic       { $$ = $1; }             // %["key" => val]
    | "/" ...           { ... T_REGEXP ... }     // /pattern/flags
    ;
```

注意 `T_CONSTVAL` 和 `T_SYMBOL` 的语义值是**索引**（`$1` 是 `tjs_int`），通过 `lx->GetValue($1)` 或 `lx->GetString($1)` 从词法器的字面量表中取出实际值。

## TJS2 特有语法结构

### property 定义（第 412-452 行）

TJS2 的 `property` 是一等语法结构（而非像 C++ 的 getter/setter 约定）：

```javascript
// TJS2 代码
property width {
    getter() { return _width; }
    setter(v) { _width = v; onResize(); }
}
```

对应文法：

```yacc
property_def
    : "property" T_SYMBOL "{" property_handler_def_list "}" ;
property_handler_def_list
    : property_handler_setter
    | property_handler_getter
    | property_handler_setter property_handler_getter
    | property_handler_getter property_handler_setter
    ;
```

getter 和 setter 可以以**任意顺序**出现，且至少要有一个。每个 handler 创建独立的编译上下文（`ctPropertySetter` / `ctPropertyGetter`）。

### incontextof 表达式（第 670-674 行）

```yacc
incontextof_expr
    : priority_expr
    | priority_expr "incontextof" incontextof_expr
        { $$ = cc->MakeNP2(token::T_INCONTEXTOF, $1, $3); }
    ;
```

`incontextof` 是 TJS2 独有的运算符——改变方法调用的 `this` 上下文，类似 JavaScript 的 `call`/`apply`，但作为中缀运算符。例如：`func incontextof obj` 表示"以 `obj` 为 this 调用 `func`"。

### 字典字面量（第 760-785 行）

```yacc
inline_dic
    : "%" "["  { ... MakeNP0(token::T_INLINEDIC) ... }
      dic_elm_list  dic_dummy_elm_opt
      "]"      { $$ = cn; cc->PopCurrentNode(); }
    ;
dic_elm
    : expr_no_comma "," expr_no_comma   // 位置形式：%[ key, value ]
    | T_SYMBOL ":" expr_no_comma        // 简写形式：%[ name: value ]
    ;
```

`%[...]` 语法——百分号+方括号创建字典。支持两种键值语法：逗号分隔（`key, value`）和冒号分隔（`name: value`）。冒号形式中键自动转为字符串常量。

> **与 JavaScript 对比：** JS 用 `{key: value}` 创建对象，TJS2 用 `%[key: value]`。TJS2 的 `{...}` 保留给代码块使用。

### 常量数组与字典（第 795-858 行）

```yacc
const_inline_array
    : "(" "const" ")" "[" ... "]"     // (const)[1, 2, 3]
    ;
const_inline_dic
    : "(" "const" ")" "%" "[" ... "]" // (const)%["a", 1, "b", 2]
    ;
```

`(const)` 前缀告诉编译器这些字面量的内容在**编译期完全确定**，可以预先构建并缓存，避免每次执行时重新创建。

## 动手实践：追踪 TJS2 代码的解析过程

以一段简单的 TJS2 代码为例，追踪从源码到 AST 的完整过程：

```javascript
var x = 1 + 2 * 3;
```

**词法器输出的 Token 流：**

```
T_VAR  T_SYMBOL("x")  T_EQUAL  T_CONSTVAL(1)  T_PLUS
T_CONSTVAL(2)  T_ASTERISK  T_CONSTVAL(3)  T_SEMICOLON
```

**解析器的移进-归约过程（简化）：**

```
步骤  栈                              动作
───  ─────────────────────            ──────
1    var x = 1                        归约 factor→T_CONSTVAL, 调用 MakeNP0(T_CONSTVAL)
2    var x = [np:1]                   向上传递 13 层 → assign_expr 层
3    var x = [np:1] +                 移进 +
4    var x = [np:1] + 2               归约 factor→T_CONSTVAL
5    var x = [np:1] + [np:2]          向上传递到 mul_div 层
6    var x = [np:1] + [np:2] *        移进 *（优先级高于 +）
7    var x = [np:1] + [np:2] * 3      归约 factor→T_CONSTVAL
8    var x = [np:1] + [np:2*3]        归约 mul→mul*unary, 调用 MakeNP2(T_ASTERISK, np:2, np:3)
9    var x = [np:1+2*3]               归约 add→add+mul, 调用 MakeNP2(T_PLUS, np:1, np:2*3)
10   var x = [np:assign]              向上到 assign_expr 层（无赋值→透传）
11   variable_id ; → variable_def     调用 cc->InitLocalVariable("x", np:assign)
```

最终生成的 AST：

```
        InitLocalVariable("x")
              │
         T_PLUS (+)
        ┌──┴──┐
    T_CONSTVAL  T_ASTERISK (*)
      (1)      ┌──┴──┐
          T_CONSTVAL  T_CONSTVAL
            (2)         (3)
```

注意第 11 步：`var x = expr` 不是通过赋值表达式（`assign_expr`）处理的，而是通过 `variable_id` 规则直接调用 `cc->InitLocalVariable`——变量声明和赋值表达式是**不同的文法路径**。

## 常见错误与解决方案

### 混淆 `var x = ...` 与 `x = ...` 的文法路径

`var x = expr;` 走 `variable_def → variable_id` 规则（调用 `InitLocalVariable`）；`x = expr;` 走 `statement → expr → assign_expr` 规则（调用 `CreateExprCode`）。两者生成完全不同的字节码。

### 忘记 `variable_type` 是可选的

TJS2 支持 `var x : int = 5;` 这样的类型标注语法（第 339-347 行），但实际上**类型被忽略**（注释写着 "It is not currently effect"）。文法中 `variable_type` 可以为空，所以 `var x = 5;` 也合法。

### `mul_div_expr_and_asterisk` 的特殊处理

乘法产生式不是简单的 `mul_div_expr "*" unary_expr`，而是拆成了两步：

```yacc
mul_div_expr_and_asterisk
    : mul_div_expr "*"              { $$ = $1; }
    ;
// 然后在 mul_div_expr 中：
    | mul_div_expr_and_asterisk unary_expr  { $$ = cc->MakeNP2(T_ASTERISK, $1, $2); }
```

为什么？因为 `*` 还可以作为一元运算符（属性直接访问：`*prop`），以及函数参数中的展开运算符。将 `mul_div_expr "*"` 先归约为 `mul_div_expr_and_asterisk`，让解析器能在后续决策中区分这些用法。

### 题目 1：表达式层次追踪

TJS2 代码 `a || b && c` 会被解析为 `a || (b && c)` 还是 `(a || b) && c`？为什么？请从文法层次链角度解释。

<details>
<summary>查看答案</summary>

解析为 `a || (b && c)`。

在表达式层次链中，`logical_or_expr` 引用 `logical_and_expr`：

```yacc
logical_or_expr
    : logical_and_expr
    | logical_or_expr "||" logical_and_expr
    ;
```

`||` 的操作数是 `logical_and_expr`，这意味着 `&&` 的优先级高于 `||`。解析过程：
1. `b && c` 先在 `logical_and_expr` 层归约为一个节点
2. 然后 `a || [b&&c]` 在 `logical_or_expr` 层归约

这与 C/Java/JavaScript 的优先级规则一致。

### 题目 2：识别 TJS2 特有语法

以下 TJS2 代码中，哪些语法在标准 JavaScript 中不存在？分别对应 `tjs.y` 的哪些产生式？

```javascript
var obj = %["name": "test", "value": 42];
var result = func incontextof obj;
property count {
    getter() { return _count; }
    setter(v) { _count = v; }
}
```

<details>
<summary>查看答案</summary>

三处 TJS2 特有语法：

1. **`%[...]` 字典字面量**（第 1 行）→ `inline_dic` 产生式（第 760-768 行）。JS 使用 `{name: "test", value: 42}`。

2. **`incontextof` 运算符**（第 2 行）→ `incontextof_expr` 产生式（第 670-674 行）。JS 使用 `func.call(obj)` 或 `func.apply(obj, args)`。

3. **`property` 声明**（第 3-6 行）→ `property_def` 产生式（第 412-420 行）。JS 使用 `Object.defineProperty()` 或 class 的 `get`/`set` 语法。

### 题目 3：`%expect 1` 的来源

TJS2 的 `%expect 1` 声明有一个移进-归约冲突。请指出该冲突发生在哪两条产生式之间，以及 Bison 选择了哪个动作、为什么这是正确的。

<details>
<summary>查看答案</summary>

冲突发生在 `if`（第 270-273 行）和 `if_else`（第 277-279 行）之间：

```yacc
if      : "if" "(" expr ")" block_or_statement ;
if_else : if "else" block_or_statement ;
```

当栈上已有 `if (expr) stmt`，下一个 Token 是 `else` 时：
- **移进** `else` → 继续匹配 `if_else`（else 配近 if）
- **归约** `if` → 结束当前 if，else 配外层 if

Bison 默认选择**移进**——`else` 与最近的 `if` 配对，这是几乎所有语言的约定行为。TJS2 配合 `%prec LOWER_ELSE` 使 if 归约的优先级低于 else 的移进优先级，在声明层面固化了这个选择。`%expect 1` 精确断言只有这一个冲突——如果以后修改文法引入了新冲突，Bison 会立即报错。

</details>
