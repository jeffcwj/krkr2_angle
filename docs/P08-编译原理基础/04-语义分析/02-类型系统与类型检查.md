## 静态类型 vs 动态类型

### 静态类型：编译期确定

C++ 是静态类型语言——每个变量的类型在**编译期**确定：

```cpp
int x = 10;           // x 的类型是 int，编译期已知
std::string s = x;    // ❌ 编译错误：int 不能赋值给 string
double d = x;         // ✓ 隐式转换：int → double
```

编译器在生成代码前就能捕获类型错误。代价是灵活性低——同一个变量不能在运行时改变类型。

### 动态类型：运行时决定

TJS2、JavaScript、Python 是动态类型语言——变量没有固定类型：

```javascript
var x = 10;           // x 当前是整数
x = "hello";          // x 变成了字符串——合法！
x = %[key: "value"];  // x 变成了字典——仍然合法！
```

灵活性高，但类型错误要到运行时才能发现。

### 对比总结

| 特性 | 静态类型 | 动态类型 |
|------|---------|---------|
| 类型确定时机 | 编译期 | 运行时 |
| 类型错误检测 | 编译时报错 | 运行时抛异常 |
| 代表语言 | C/C++, Java, Rust | JavaScript, Python, TJS2 |
| 性能 | 高（无运行时类型检查） | 较低（每次操作需检查类型） |
| 灵活性 | 低（类型固定） | 高（类型可变） |
| 内存布局 | 紧凑（类型已知） | 需额外存储类型标签 |

> **C++ 程序员注意：** C++ 的 `std::variant`（C++17）和 `std::any`（C++17）就是静态语言中模拟动态类型的工具。TJS2 的 `tTJSVariant` 与 `std::variant` 的思路非常相似，只是诞生得更早。

## 动态类型的实现——tagged union

动态类型语言的核心问题：一个变量可能存储任意类型的值，如何在内存中表示？

答案：**tagged union**——用一个枚举标签记录当前类型，用 union 存储实际值：

```
┌──────────────────────────────────┐
│  tTJSVariant                     │
│  ┌──────┐  ┌──────────────────┐  │
│  │ vt   │  │     union        │  │
│  │ enum │  │  Integer (int64) │  │
│  │      │  │  Real    (double)│  │
│  │      │  │  String  (ptr)   │  │
│  │      │  │  Object  (ptr)   │  │
│  │      │  │  Octet   (ptr)   │  │
│  └──────┘  └──────────────────┘  │
└──────────────────────────────────┘
```

## 对照项目源码——TJS2 的 tTJSVariant

TJS2 的类型系统定义在 `cpp/core/tjs2/tjsVariant.h`。

### 六种类型标签

```cpp
// tjsVariant.h 第 103-110 行
enum tTJSVariantType {
    tvtVoid,      // 空值（类似 undefined/null）
    tvtObject,    // 对象引用（函数、类实例、数组、字典）
    tvtString,    // 字符串（引用计数的 tTJSVariantString*）
    tvtOctet,     // 二进制数据（引用计数的 tTJSVariantOctet*）
    tvtInteger,   // 64 位有符号整数（tjs_int64）
    tvtReal       // 64 位浮点数（double）
};
```

> **对比 JavaScript：** JS 有 7 种基本类型（undefined, null, boolean, number, string, symbol, bigint）+ object。TJS2 更简洁——boolean 用 integer 0/1 表示，null 和 undefined 合并为 void，新增了 octet（二进制数据类型，JS 无此类型）。

### union 存储

```cpp
// tjsVariant.h 第 136-153 行（简化）
struct tTJSVariant_S {
    union {
        tTJSVariantClosure_S Object;   // Object+ObjThis 两个指针
        tTVInteger Integer;             // tjs_int64 (int64_t)
        tTVReal Real;                   // double
        tTJSVariantString *String;      // 引用计数字符串指针
        tTJSVariantOctet *Octet;        // 引用计数二进制指针
    };
    tTJSVariantType vt;                 // 类型标签
};
```

整个 `tTJSVariant` 的大小 = max(union 成员) + vt 枚举 = **16 + 4 = 20 字节**（按 4 字节对齐后可能为 24 字节）。每个 TJS2 变量在运行时都占这么多空间，无论存的是整数还是字符串。

### 类型转换方法

TJS2 的类型转换遵循一套明确的规则。以 `AsInteger()` 为例：

```cpp
// tjsVariant.h 第 862-879 行（简化）
tTVInteger tTJSVariant::AsInteger() {
    switch (vt) {
        case tvtVoid:    return 0;                    // void → 0
        case tvtInteger: return Integer;              // 原样返回
        case tvtReal:    return (tTVInteger)Real;     // double → int64（截断）
        case tvtString:  return String->ToInteger();  // "123" → 123
        case tvtObject:  // ← 抛异常：对象不能转为整数
        case tvtOctet:   TJSThrowVariantConvertError(*this, tvtInteger);
    }
    return 0;
}
```

完整的转换规则矩阵：

| 源类型 → 目标 | Integer | Real | String | Object | Octet |
|--------------|---------|------|--------|--------|-------|
| **void** | 0 | 0.0 | `""` (null) | ❌ 异常 | null |
| **Integer** | 原样 | int→double | `"123"` | ❌ 异常 | ❌ 异常 |
| **Real** | 截断 | 原样 | `"3.14"` | ❌ 异常 | ❌ 异常 |
| **String** | 解析 | 解析 | 原样 | ❌ 异常 | ❌ 异常 |
| **Object** | ❌ 异常 | ❌ 异常 | 调用 toString | 原样 | ❌ 异常 |
| **Octet** | ❌ 异常 | ❌ 异常 | ❌ 异常 | ❌ 异常 | 原样 |

> **关键发现：** TJS2 的类型转换比 JavaScript 更严格。JS 中 `{} + []` 返回 `0`（对象→字符串→数字），而 TJS2 中对象→整数直接抛异常。

### 引用计数内存管理

Object、String、Octet 三种类型使用引用计数：

```cpp
// 构造——增加引用
tTJSVariant(iTJSDispatch2* ref) {
    if (ref) ref->AddRef();       // 引用 +1
    vt = tvtObject;
    Object.Object = ref;
}

// 析构——释放引用
void ReleaseContent() {
    if (vt == tvtObject) {
        if (Object.Object)  Object.Object->Release();   // 引用 -1
        if (Object.ObjThis) Object.ObjThis->Release();
    } else if (vt == tvtString) {
        if (String) String->Release();
    } else if (vt == tvtOctet) {
        if (Octet) Octet->Release();
    }
}
```

Integer 和 Real 是值类型，直接存在 union 中，无需引用计数。

## 动手实践——实现简化版 Variant

下面用 C++17 实现一个支持 void/int/double/string 四种类型的 Variant：

```cpp
#include <iostream>
#include <string>
#include <cstring>
#include <cstdlib>
#include <stdexcept>

enum class VarType { Void, Int, Real, Str };

class Variant {
    VarType type_ = VarType::Void;
    union {
        int64_t intVal;
        double  realVal;
        char*   strVal;     // 手动管理内存（模拟引用计数字符串）
    } data_{};

    void freeStr() {
        if (type_ == VarType::Str && data_.strVal) {
            delete[] data_.strVal;
            data_.strVal = nullptr;
        }
    }
    void copyStr(const char* s) {
        size_t len = std::strlen(s);
        data_.strVal = new char[len + 1];
        std::memcpy(data_.strVal, s, len + 1);
    }

public:
    Variant() = default;                                     // void
    explicit Variant(int64_t v) : type_(VarType::Int)   { data_.intVal = v; }
    explicit Variant(double v)  : type_(VarType::Real)  { data_.realVal = v; }
    explicit Variant(const char* s) : type_(VarType::Str) { copyStr(s); }

    ~Variant() { freeStr(); }

    // 拷贝构造
    Variant(const Variant& o) : type_(o.type_) {
        if (type_ == VarType::Str) copyStr(o.data_.strVal);
        else data_ = o.data_;
    }

    Variant& operator=(const Variant& o) {
        if (this != &o) { freeStr(); type_ = o.type_;
            if (type_ == VarType::Str) copyStr(o.data_.strVal);
            else data_ = o.data_;
        }
        return *this;
    }

    VarType type() const { return type_; }

    // === 类型转换方法（模仿 TJS2 AsInteger / AsString） ===
    int64_t asInt() const {
        switch (type_) {
            case VarType::Void: return 0;               // void → 0
            case VarType::Int:  return data_.intVal;
            case VarType::Real: return (int64_t)data_.realVal;  // 截断
            case VarType::Str:  return std::atoll(data_.strVal); // "123" → 123
        }
        return 0;
    }

    double asReal() const {
        switch (type_) {
            case VarType::Void: return 0.0;
            case VarType::Int:  return (double)data_.intVal;
            case VarType::Real: return data_.realVal;
            case VarType::Str:  return std::atof(data_.strVal);
        }
        return 0.0;
    }

    std::string asString() const {
        switch (type_) {
            case VarType::Void: return "";
            case VarType::Int:  return std::to_string(data_.intVal);
            case VarType::Real: return std::to_string(data_.realVal);
            case VarType::Str:  return data_.strVal;
        }
        return "";
    }

    // === 运算符（展示动态类型的隐式转换） ===
    Variant operator+(const Variant& rhs) const {
        // 规则：任一操作数为字符串 → 字符串拼接；否则数值加法
        if (type_ == VarType::Str || rhs.type_ == VarType::Str) {
            std::string result = asString() + rhs.asString();
            return Variant(result.c_str());
        }
        // 任一操作数为 Real → Real 加法
        if (type_ == VarType::Real || rhs.type_ == VarType::Real)
            return Variant(asReal() + rhs.asReal());
        // 否则 Int 加法
        return Variant(asInt() + rhs.asInt());
    }

    friend std::ostream& operator<<(std::ostream& os, const Variant& v) {
        switch (v.type_) {
            case VarType::Void: os << "void"; break;
            case VarType::Int:  os << v.data_.intVal; break;
            case VarType::Real: os << v.data_.realVal; break;
            case VarType::Str:  os << "\"" << v.data_.strVal << "\""; break;
        }
        return os;
    }
};

int main() {
    Variant a(int64_t(100));
    Variant b(3.14);
    Variant c("hello ");
    Variant d("world");
    Variant e;              // void

    std::cout << "a + b = " << (a + b) << "\n";     // 103.14 (Real)
    std::cout << "c + d = " << (c + d) << "\n";     // "hello world" (Str)
    std::cout << "a + c = " << (a + c) << "\n";     // "100hello " (Str)
    std::cout << "e + a = " << (e + a) << "\n";     // 100 (Int, void→0)

    // 类型转换演示
    Variant num("456");
    std::cout << "\"456\".asInt() = " << num.asInt() << "\n";  // 456
    std::cout << "3.14.asInt() = " << b.asInt() << "\n";       // 3（截断）
    return 0;
}
```

编译运行：

```bash
g++ -std=c++17 -o variant variant.cpp && ./variant
# 输出：
# a + b = 103.14
# c + d = "hello world"
# a + c = "100hello "
# e + a = 100
# "456".asInt() = 456
# 3.14.asInt() = 3
```

## 追踪 TJS2 中 `"123" + 456` 的执行

TJS2 中执行 `var result = "123" + 456;` 的完整过程：

```
1. 编译器生成字节码：
   CONST_STR  r0, "123"     // r0 = Variant(tvtString, "123")
   CONST_INT  r1, 456       // r1 = Variant(tvtInteger, 456)
   ADD        r2, r0, r1    // r2 = r0 + r1

2. VM 执行 ADD 指令：
   a) 检查 r0.vt = tvtString → 走字符串拼接路径
   b) 调用 r1.AsString() → "456"（Integer→String 转换）
   c) 拼接："123" + "456" = "123456"
   d) r2 = Variant(tvtString, "123456")

3. 结果：result 的值为字符串 "123456"
```

> **注意：** TJS2 的 `+` 对字符串的行为和 JavaScript 相同：只要有一个操作数是字符串，另一个就隐式转为字符串。这与 Python 不同——Python 的 `"123" + 456` 会直接抛 TypeError。

## 编译期能做的类型推断

虽然 TJS2 是动态类型，编译器仍然可以做一些优化：

| 场景 | 编译器推断 | 优化 |
|------|-----------|------|
| `var x = 10` | x 初始为 Integer | 后续 `x + 1` 直接用整数加法指令 |
| `var s = "hi"` | s 初始为 String | 后续拼接跳过类型检查 |
| `x = someFunc()` | 返回类型未知 | 无法优化，必须运行时检查 |
| `if (typeof x == "Integer")` | 分支内 x 为 Integer | 理论上可优化（TJS2 未实现） |

TJS2 的编译器**不做**这些高级优化——它总是生成运行时类型检查的字节码。V8、SpiderMonkey 等现代 JS 引擎会通过 JIT 编译和"内联缓存"（Inline Cache）做激进的类型推断优化。

## 常见错误与解决方案

### 错误 1：忘记处理 void 类型

```cpp
// ❌ 没有处理 void
int64_t asInt() {
    if (type_ == VarType::Int)  return data_.intVal;
    if (type_ == VarType::Real) return (int64_t)data_.realVal;
    throw std::runtime_error("无法转为整数");
}
// void 类型会抛异常——但 TJS2 中 void → 0 是合法的
```

**解决：** 始终为 void 提供默认值（0、0.0、空字符串），这是动态语言的惯例。

### 错误 2：tagged union 忘记更新类型标签

```cpp
// ❌ 赋值时只更新了 union 值，忘了更新 vt
void setInt(int64_t v) {
    data_.intVal = v;       // 值更新了
    // 忘了：type_ = VarType::Int;
}
// 后续 type() 仍返回旧类型 → AsString() 可能读到垃圾数据
```

**解决：** 每次修改 union 值都**必须同步更新类型标签**。TJS2 的赋值运算符中每个分支都先设 `vt`。

### 错误 3：未释放旧值就赋新值导致内存泄漏

```cpp
// ❌ 直接覆盖，旧字符串泄漏
Variant& operator=(int64_t v) {
    type_ = VarType::Int;
    data_.intVal = v;       // 如果之前是 Str，旧 strVal 没有 delete → 泄漏！
    return *this;
}
```

**解决：** 赋值前**先释放旧内容**（调用 `freeStr()` 或 `ReleaseContent()`）。TJS2 的赋值运算符总是先 `ReleaseContent()` 再设新值。

## 本节小结

- **静态类型**在编译期确定，**动态类型**在运行时通过 tagged union 实现
- TJS2 的 `tTJSVariant` 有 6 种类型：void、object、string、octet、integer、real
- union 存储 + `vt` 枚举标签 = 每个变量 ~20-24 字节
- 类型转换规则：void→0 合法，string↔number 合法，object/octet 与数值间不可转换
- 引用计数管理 object/string/octet 的生命周期；integer/real 是值类型
- 编译器理论上可做类型推断优化，但 TJS2 未实现（V8 等现代引擎通过 JIT 实现）

### 题目 2：扩展 Variant 支持比较

为上面的 Variant 类添加 `operator==`，规则：
- 相同类型直接比较值
- 不同类型时都转为 Real 再比较（void 转为 0.0）

<details>
<summary>查看答案</summary>

```cpp
bool operator==(const Variant& rhs) const {
    if (type_ == rhs.type_) {
        switch (type_) {
            case VarType::Void: return true;          // void == void
            case VarType::Int:  return data_.intVal == rhs.data_.intVal;
            case VarType::Real: return data_.realVal == rhs.data_.realVal;
            case VarType::Str:  return std::strcmp(data_.strVal, rhs.data_.strVal) == 0;
        }
    }
    // 不同类型——都转为 Real 比较
    return asReal() == rhs.asReal();
}
```

注意：TJS2 有两种比较——`==`（NormalCompare，会做类型转换）和 `===`（DiscernCompare，类型不同直接返回 false）。上面的实现对应 `==`。

</details>

### 题目 3：分析 tTJSVariant 的内存布局

假设运行在 64 位系统上，`sizeof(tjs_int64) = 8`, `sizeof(double) = 8`, `sizeof(void*) = 8`, `sizeof(tTJSVariantType) = 4`：

1. `tTJSVariantClosure_S` 包含两个指针，大小是多少？
2. union 中最大成员是哪个？整个 union 大小？
3. `tTJSVariant_S` 在 `#pragma pack(push, 4)` 下总大小是多少？

<details>
<summary>查看答案</summary>

1. `tTJSVariantClosure_S` = 2 × 8 = **16 字节**（两个 `iTJSDispatch2*` 指针）
2. union 最大成员是 `Object`（tTJSVariantClosure_S = 16 字节）。Integer = 8, Real = 8, String = 8, Octet = 8 都更小。union 大小 = **16 字节**。
3. `tTJSVariant_S` = union(16) + vt(4) = 20 字节。在 `pack(4)` 下，对齐到 4 字节边界，总大小 = **20 字节**。

</details>
