## 语义分析在编译流程中的位置

```
源代码 → [词法分析] → tokens → [语法分析] → AST → [语义分析] → 标注AST → [代码生成]
                                                    ↑ 我们在这里
```

语法分析确保代码结构正确（括号匹配、运算符优先级），语义分析确保代码**含义**正确：

| 语法正确但语义错误的代码 | 错误类型 |
|------------------------|---------|
| `var x = 1; var x = 2;` | 重复声明 |
| `y = 10;`（未声明 y） | 未定义变量 |
| `"hello" / 3` | 类型不匹配（TJS2 中会运行时报错） |
| `return x;`（不在函数内） | 控制流错误 |

## 动手实践——构建完整的语义分析器

我们将从零构建一个语义分析器，处理以下语法的迷你语言：

```
program    → statement*
statement  → var_decl | assignment | block | expr_stmt
var_decl   → "var" IDENT "=" expr ";"
assignment → IDENT "=" expr ";"
block      → "{" statement* "}"
expr       → term (("+"|"-") term)*
term       → factor (("*"|"/") factor)*
factor     → NUMBER | STRING | IDENT | "(" expr ")"
```

### 第一步：定义 AST 节点

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <memory>
#include <unordered_map>
#include <stdexcept>
#include <variant>
#include <cassert>

// AST 节点类型
enum class NodeType {
    Program, VarDecl, Assign, Block, BinaryOp,
    NumberLit, StringLit, Identifier, ExprStmt
};

struct ASTNode {
    NodeType type;
    std::string value;                          // 标识符名 / 运算符 / 字面值
    std::vector<std::unique_ptr<ASTNode>> children;

    ASTNode(NodeType t, const std::string& v = "") : type(t), value(v) {}

    void addChild(std::unique_ptr<ASTNode> child) {
        children.push_back(std::move(child));
    }
};

using ASTPtr = std::unique_ptr<ASTNode>;

// 辅助工厂函数
ASTPtr makeNode(NodeType t, const std::string& v = "") {
    return std::make_unique<ASTNode>(t, v);
}
```

### 第二步：符号表（复用上一节的设计）

```cpp
enum class SymKind { Variable, Function };
enum class ValType { Unknown, Int, Real, Str, Void };

struct Symbol {
    std::string name;
    SymKind kind;
    ValType type;
    int scopeDepth;
};

class SymbolTable {
    struct Scope {
        std::unordered_map<std::string, Symbol> symbols;
    };
    std::vector<Scope> scopes_;
    int depth_ = 0;
public:
    void enterScope() { scopes_.push_back(Scope{}); depth_++; }
    void exitScope()  { scopes_.pop_back(); depth_--; }

    bool declareInCurrentScope(const std::string& name, ValType type) {
        auto& current = scopes_.back().symbols;
        if (current.count(name)) return false;  // 当前作用域已有同名 → 重复声明
        current[name] = {name, SymKind::Variable, type, depth_};
        return true;
    }

    Symbol* lookup(const std::string& name) {
        for (int i = (int)scopes_.size() - 1; i >= 0; i--) {
            auto it = scopes_[i].symbols.find(name);
            if (it != scopes_[i].symbols.end()) return &it->second;
        }
        return nullptr;  // 未找到
    }

    int depth() const { return depth_; }
};
```

### 第三步：语义分析器

```cpp
class SemanticAnalyzer {
    SymbolTable symtab_;
    std::vector<std::string> errors_;

    void error(const std::string& msg) {
        errors_.push_back(msg);
    }

public:
    // 分析入口
    void analyze(ASTNode* node) {
        symtab_.enterScope();  // 全局作用域
        analyzeNode(node);
        symtab_.exitScope();
    }

    void analyzeNode(ASTNode* node) {
        if (!node) return;
        switch (node->type) {
            case NodeType::Program:
                for (auto& child : node->children) analyzeNode(child.get());
                break;
            case NodeType::VarDecl:   analyzeVarDecl(node); break;
            case NodeType::Assign:    analyzeAssign(node); break;
            case NodeType::Block:     analyzeBlock(node); break;
            case NodeType::ExprStmt:  analyzeExpr(node->children[0].get()); break;
            default: break;
        }
    }

    void analyzeVarDecl(ASTNode* node) {
        // children[0] = 标识符, children[1] = 初始化表达式
        const std::string& name = node->children[0]->value;
        ValType initType = analyzeExpr(node->children[1].get());

        if (!symtab_.declareInCurrentScope(name, initType)) {
            error("重复声明变量 '" + name + "'");
        }
    }

    void analyzeAssign(ASTNode* node) {
        const std::string& name = node->children[0]->value;
        analyzeExpr(node->children[1].get());

        Symbol* sym = symtab_.lookup(name);
        if (!sym) {
            error("赋值给未声明的变量 '" + name + "'");
        }
    }

    void analyzeBlock(ASTNode* node) {
        symtab_.enterScope();
        for (auto& child : node->children) analyzeNode(child.get());
        symtab_.exitScope();
    }

    ValType analyzeExpr(ASTNode* node) {
        if (!node) return ValType::Void;
        switch (node->type) {
            case NodeType::NumberLit:
                return node->value.find('.') != std::string::npos
                    ? ValType::Real : ValType::Int;
            case NodeType::StringLit:
                return ValType::Str;
            case NodeType::Identifier: {
                Symbol* sym = symtab_.lookup(node->value);
                if (!sym) {
                    error("使用未声明的变量 '" + node->value + "'");
                    return ValType::Unknown;
                }
                return sym->type;
            }
            case NodeType::BinaryOp:
                return analyzeBinaryOp(node);
            default:
                return ValType::Unknown;
        }
    }

    ValType analyzeBinaryOp(ASTNode* node) {
        ValType left  = analyzeExpr(node->children[0].get());
        ValType right = analyzeExpr(node->children[1].get());
        const std::string& op = node->value;

        // 字符串 + 字符串 → 字符串拼接
        if (op == "+" && (left == ValType::Str || right == ValType::Str))
            return ValType::Str;
        // 字符串参与 - * / → 报错
        if ((left == ValType::Str || right == ValType::Str)
            && (op == "-" || op == "*" || op == "/")) {
            error("字符串不能参与 " + op + " 运算");
            return ValType::Unknown;
        }
        // Real 参与 → Real
        if (left == ValType::Real || right == ValType::Real)
            return ValType::Real;
        return ValType::Int;
    }

    bool hasErrors() const { return !errors_.empty(); }
    const std::vector<std::string>& getErrors() const { return errors_; }
};
```

### 第四步：构造 AST 并运行分析

```cpp
// 辅助：快速构造 VarDecl 节点
ASTPtr makeVarDecl(const std::string& name, ASTPtr init) {
    auto decl = makeNode(NodeType::VarDecl);
    decl->addChild(makeNode(NodeType::Identifier, name));
    decl->addChild(std::move(init));
    return decl;
}

int main() {
    // 测试代码：
    //   var x = 10; var y = "hello";
    //   { var x = 20; z = x + 1; }   // z 未声明
    //   var x = 30;                    // 外层已有 x
    //   "hello" - 1;                   // 字符串减法
    auto prog = makeNode(NodeType::Program);
    prog->addChild(makeVarDecl("x", makeNode(NodeType::NumberLit, "10")));
    prog->addChild(makeVarDecl("y", makeNode(NodeType::StringLit, "hello")));

    // 块作用域
    auto block = makeNode(NodeType::Block);
    block->addChild(makeVarDecl("x", makeNode(NodeType::NumberLit, "20")));
    auto assign = makeNode(NodeType::Assign);
    assign->addChild(makeNode(NodeType::Identifier, "z"));
    auto addExpr = makeNode(NodeType::BinaryOp, "+");
    addExpr->addChild(makeNode(NodeType::Identifier, "x"));
    addExpr->addChild(makeNode(NodeType::NumberLit, "1"));
    assign->addChild(std::move(addExpr));
    block->addChild(std::move(assign));
    prog->addChild(std::move(block));

    prog->addChild(makeVarDecl("x", makeNode(NodeType::NumberLit, "30")));

    auto exprStmt = makeNode(NodeType::ExprStmt);
    auto subExpr = makeNode(NodeType::BinaryOp, "-");
    subExpr->addChild(makeNode(NodeType::StringLit, "hello"));
    subExpr->addChild(makeNode(NodeType::NumberLit, "1"));
    exprStmt->addChild(std::move(subExpr));
    prog->addChild(std::move(exprStmt));

    // 运行分析
    SemanticAnalyzer analyzer;
    analyzer.analyze(prog.get());
    for (auto& err : analyzer.getErrors())
        std::cout << "[错误] " << err << std::endl;
    return 0;
}
```

输出：

```
[错误] 赋值给未声明的变量 'z'
[错误] 重复声明变量 'x'
[错误] 字符串不能参与 - 运算
```

三种语义错误全部检出：`z` 未声明、外层 `x` 重复声明（内层的遮蔽已随块作用域退出而消失）、字符串不支持减法。

## 对照项目源码——TJS2 的单遍编译策略

我们的迷你分析器采用"先建 AST、再遍历"的两遍方案。TJS2 则更高效——**边解析语法边做语义分析**，一遍完成。

TJS2 的 `tTJSInterCodeGenerator`（`tjsInterCodeGen.h` 第 404 行）内嵌 `tTJSLocalNamespace Namespace` 作为符号表。当 Bison 在 `tjs.y` 中归约 `var x = expr;` 时，语义动作直接调用 `Namespace.Add()` 注册变量并同时生成赋值字节码——不需要中间 AST 节点。

| 方面 | 我们的实现（两遍） | TJS2（单遍） |
|------|-------------------|-------------|
| 流程 | 解析 → AST → 遍历 | 解析 + 分析同步 |
| 内存 | 需要完整 AST | 不需要 |
| 前向引用 | 可处理 | 难处理 |
| 适用场景 | 静态类型、优化编译器 | 动态类型脚本语言 |

TJS2 选择单遍策略是合理的——动态类型语言的类型检查推迟到运行时，编译时只需做变量作用域解析。

## 常量折叠——编译期优化入门

常量折叠（Constant Folding）是最简单也最常见的编译优化：在编译期计算常量表达式的值。

### 原理

```
源代码：   var x = 1 + 2 * 3;

未优化 AST：                优化后 AST：
    VarDecl                     VarDecl
    ├─ x                        ├─ x
    └─ BinaryOp(+)              └─ NumberLit(7)
       ├─ NumberLit(1)
       └─ BinaryOp(*)
          ├─ NumberLit(2)
          └─ NumberLit(3)

优化省去了运行时的 2 次算术运算！
```

### 实现

在语义分析阶段添加常量折叠，只需要在 `analyzeBinaryOp` 中增加判断：

```cpp
// 在 SemanticAnalyzer 类中添加
ASTPtr foldConstants(ASTNode* node) {
    if (node->type != NodeType::BinaryOp) return nullptr;
    // 递归折叠子树
    auto lf = foldConstants(node->children[0].get());
    auto rf = foldConstants(node->children[1].get());
    if (lf) node->children[0] = std::move(lf);
    if (rf) node->children[1] = std::move(rf);

    auto* left  = node->children[0].get();
    auto* right = node->children[1].get();
    if (left->type != NodeType::NumberLit || right->type != NodeType::NumberLit)
        return nullptr;  // 非常量 → 不折叠

    double lv = std::stod(left->value), rv = std::stod(right->value), res = 0;
    if      (node->value == "+") res = lv + rv;
    else if (node->value == "-") res = lv - rv;
    else if (node->value == "*") res = lv * rv;
    else if (node->value == "/") { if (rv == 0) return nullptr; res = lv / rv; }

    if (res == (int)res) return makeNode(NodeType::NumberLit, std::to_string((int)res));
    return makeNode(NodeType::NumberLit, std::to_string(res));
}
```

测试：对 `1 + 2 * 3` 的 AST 调用 `foldConstants()`，先将 `2 * 3` 折叠为 `6`，再将 `1 + 6` 折叠为 `7`，最终整棵子树变为一个 `NumberLit(7)` 节点。

TJS2 在 `tTJSInterCodeGenerator` 中也做常量折叠——当两个操作数都是字面量时直接计算结果，生成 `const` 指令而非运算指令。脚本中 `var SCREEN_W = 800 + 24;` 这类表达式受益显著。

## 常见错误及解决方案

### 错误 1：退出作用域后使用悬垂符号指针

```cpp
Symbol* sym = symtab_.lookup("x");
symtab_.exitScope();         // 底层 vector 被 pop
sym->type = ValType::Int;    // ❌ 悬垂指针！
```

**解决：** 退出作用域前完成所有符号操作，或先拷贝：`Symbol copy = *symtab_.lookup("x");`

### 错误 2：switch 遗漏节点类型

忘记在 `analyzeNode` 中处理 `NodeType::Block` → 内层声明/赋值全部跳过。**解决：** 使用 `-Wswitch` 编译选项，在 `default` 中 `assert(false)`。

### 错误 3：常量折叠整数溢出

`int result = 2000000000 + 2000000000;` 触发未定义行为。**解决：** 用 `double` 做中间计算（如我们的实现），或使用 `long long` 并检查范围。

## 本节小结

- 语义分析在 AST 上检查**含义正确性**：变量声明、作用域、类型兼容
- 核心数据结构是**符号表**（嵌套作用域栈），核心算法是**AST 递归遍历**
- 我们的分析器检测三类错误：未定义变量、重复声明、类型不匹配
- TJS2 采用**单遍编译**——Bison 归约时直接操作 `tTJSLocalNamespace`，不建中间 AST
- **常量折叠**是最简单的编译优化，在编译期计算常量子表达式
- 两遍 vs 单遍各有优劣：两遍灵活可多次分析，单遍节省内存适合脚本语言

### 题目 1：添加函数调用检查

扩展语义分析器，新增 `NodeType::FuncDecl` 和 `NodeType::FuncCall`。要求：函数声明时登记 `SymKind::Function` 并记录参数个数；调用时检查函数是否存在、实参个数是否匹配。

<details>
<summary>查看答案</summary>

```cpp
// Symbol 添加 paramCount 字段
int paramCount = 0;  // 仅 kind == Function 时有效

void analyzeFuncDecl(ASTNode* node) {
    // children[0]=名字, children[1]=参数列表, children[2]=函数体
    const std::string& name = node->children[0]->value;
    int paramCount = (int)node->children[1]->children.size();
    if (!symtab_.declareInCurrentScope(name, ValType::Void)) {
        error("重复声明函数 '" + name + "'"); return;
    }
    Symbol* sym = symtab_.lookup(name);
    sym->kind = SymKind::Function;
    sym->paramCount = paramCount;
    symtab_.enterScope();  // 函数体新作用域
    for (auto& param : node->children[1]->children)
        symtab_.declareInCurrentScope(param->value, ValType::Unknown);
    analyzeNode(node->children[2].get());
    symtab_.exitScope();
}

void analyzeFuncCall(ASTNode* node) {
    // children[0]=名字, children[1...]=实参
    const std::string& name = node->children[0]->value;
    Symbol* sym = symtab_.lookup(name);
    if (!sym) { error("调用未声明的函数 '" + name + "'"); return; }
    if (sym->kind != SymKind::Function) { error("'" + name + "' 不是函数"); return; }
    int argCount = (int)node->children.size() - 1;
    if (argCount != sym->paramCount) {
        error("函数 '" + name + "' 需要 " + std::to_string(sym->paramCount)
              + " 个参数，但传入了 " + std::to_string(argCount) + " 个");
    }
}
```

</details>

### 题目 2：实现 `const` 变量检查

添加 `NodeType::ConstDecl`，在符号表中标记 `isConst = true`。当对 `const` 变量赋值时报语义错误 "不能对常量 'x' 赋值"。

<details>
<summary>查看答案</summary>

```cpp
// Symbol 添加: bool isConst = false;

void analyzeConstDecl(ASTNode* node) {
    const std::string& name = node->children[0]->value;
    ValType initType = analyzeExpr(node->children[1].get());
    if (!symtab_.declareInCurrentScope(name, initType)) {
        error("重复声明常量 '" + name + "'"); return;
    }
    symtab_.lookup(name)->isConst = true;
}

// 修改 analyzeAssign —— 增加 const 检查
void analyzeAssign(ASTNode* node) {
    const std::string& name = node->children[0]->value;
    analyzeExpr(node->children[1].get());
    Symbol* sym = symtab_.lookup(name);
    if (!sym) error("赋值给未声明的变量 '" + name + "'");
    else if (sym->isConst) error("不能对常量 '" + name + "' 赋值");
}
```

测试：`const PI = 3.14; PI = 3;` → "不能对常量 'PI' 赋值"。

### 题目 3：思考题——变量提升

TJS2 单遍编译无法处理前向引用。JavaScript 的 `var` 声明会被提升（hoisting）到函数顶部。如果要为 TJS2 添加变量提升，需要怎样修改编译流程？

<details>
<summary>查看答案</summary>

采用"一遍半"策略：在每个函数体/块归约前，先快速扫描所有 `var` 语句，预注册变量名（类型 `Unknown`）到 `Namespace` 中。正式编译时变量已存在，不会报"未声明"。具体步骤：

1. Bison 进入函数体前添加预扫描动作，只找 `T_VAR T_SYMBOL` 模式
2. 预注册的变量初始值为 `void`（TJS2 中 `void` 等价于 undefined）
3. 正式编译到 `var x = expr;` 时只生成赋值字节码，不重复注册
4. 这比完整两遍高效——预扫描只看声明语句，跳过表达式和其他语句

V8 引擎早期版本使用类似策略。

</details>
