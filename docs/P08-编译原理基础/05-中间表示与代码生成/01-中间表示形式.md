## 为什么需要中间表示？

编译器的前端（词法、语法、语义分析）与后端（优化、代码生成）之间需要一个"通用语言"：

```
                    ┌─────────┐
   C++ 源码 ───→   │         │ ───→ x86 机器码
   Java 源码 ───→  │   IR    │ ───→ ARM 机器码
   TJS2 源码 ───→  │         │ ───→ TJS2 字节码
                    └─────────┘
         M 个前端 × N 个后端 → M+N 个适配器（而非 M×N）
```

没有 IR 的话，每种源语言到每种目标平台都要写一个完整编译器。有了 IR：
- **前端**只需要把源码翻译成 IR
- **后端**只需要把 IR 翻译成目标码
- **优化器**在 IR 上工作，对所有语言和平台通用

LLVM 就是这个思想的典范——Clang（C/C++）、Rust、Swift 的前端都输出 LLVM IR，然后由统一的后端生成 x86/ARM/RISC-V 等目标码。

## 三种 IR 层次

| IR 形式 | 结构 | 抽象级别 | 代表 |
|---------|------|---------|------|
| 树形 IR | AST / DAG | 高 | GCC GENERIC/GIMPLE |
| 线性 IR | 指令序列 | 中 | 三地址码、字节码、LLVM IR |
| 图形 IR | 控制流图 | 低 | SSA 形式的 CFG |

大多数编译器在不同阶段使用不同层次的 IR。例如 GCC：AST → GIMPLE（高层 IR）→ RTL（低层 IR）→ 机器码。

## 三地址码——最经典的线性 IR

三地址码（Three-Address Code, TAC）中每条指令最多包含三个地址（操作数）：

```
result = operand1 op operand2
```

### 将表达式转换为三地址码

源代码：`a = b * c + d / e`

```
t1 = b * c          // 乘法
t2 = d / e          // 除法
t3 = t1 + t2        // 加法
a  = t3             // 赋值
```

每条指令只做一件事，临时变量 `t1`、`t2`、`t3` 保存中间结果。

### 控制流的三地址码

`if (x > 0) y = 1; else y = -1;` 转换为：

```
    t1 = x > 0
    if_false t1 goto L1
    y = 1
    goto L2
L1: y = -1
L2:
```

### 循环的三地址码

`while (i < 10) { sum = sum + i; i = i + 1; }` 转换为：

```
L1: t1 = i < 10
    if_false t1 goto L2
    t2 = sum + i
    sum = t2
    t3 = i + 1
    i = t3
    goto L1
L2:
```

### 动手实践——AST 到三地址码翻译器

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <memory>
#include <sstream>

// 复用上一节的 AST 定义（简化版）
enum class NodeType {
    Program, VarDecl, Assign, BinaryOp,
    NumberLit, StringLit, Identifier, ExprStmt,
    If, While, Block
};

struct ASTNode {
    NodeType type;
    std::string value;
    std::vector<std::unique_ptr<ASTNode>> children;
    ASTNode(NodeType t, const std::string& v = "") : type(t), value(v) {}
    void addChild(std::unique_ptr<ASTNode> c) { children.push_back(std::move(c)); }
};
using ASTPtr = std::unique_ptr<ASTNode>;
ASTPtr makeNode(NodeType t, const std::string& v = "") {
    return std::make_unique<ASTNode>(t, v);
}

// 三地址码指令
struct TACInstruction {
    std::string result;   // 目标（变量名或临时变量）
    std::string op;       // 操作符：+, -, *, /, =, >, <, if_false, goto, label
    std::string arg1;     // 操作数 1
    std::string arg2;     // 操作数 2（可选）
};

class TACGenerator {
    int tempCount_ = 0;
    int labelCount_ = 0;
    std::vector<TACInstruction> code_;

    std::string newTemp() { return "t" + std::to_string(tempCount_++); }
    std::string newLabel() { return "L" + std::to_string(labelCount_++); }

    void emit(const std::string& result, const std::string& op,
              const std::string& arg1, const std::string& arg2 = "") {
        code_.push_back({result, op, arg1, arg2});
    }

public:
    // 表达式翻译 — 返回结果所在的临时变量名
    std::string translateExpr(ASTNode* node) {
        if (!node) return "";
        switch (node->type) {
            case NodeType::NumberLit:
            case NodeType::StringLit:
                return node->value;  // 字面量直接返回
            case NodeType::Identifier:
                return node->value;  // 变量名直接返回
            case NodeType::BinaryOp: {
                std::string left  = translateExpr(node->children[0].get());
                std::string right = translateExpr(node->children[1].get());
                std::string temp  = newTemp();
                emit(temp, node->value, left, right);  // temp = left op right
                return temp;
            }
            default: return "";
        }
    }

    // 语句翻译
    void translateStmt(ASTNode* node) {
        if (!node) return;
        switch (node->type) {
            case NodeType::Program:
            case NodeType::Block:
                for (auto& c : node->children) translateStmt(c.get());
                break;
            case NodeType::VarDecl: {
                // var x = expr;  →  x = translateExpr(expr)
                std::string val = translateExpr(node->children[1].get());
                emit(node->children[0]->value, "=", val);
                break;
            }
            case NodeType::Assign: {
                std::string val = translateExpr(node->children[1].get());
                emit(node->children[0]->value, "=", val);
                break;
            }
            case NodeType::ExprStmt:
                translateExpr(node->children[0].get());
                break;
            default: break;
        }
    }

    void print() const {
        for (auto& inst : code_) {
            if (inst.op == "label") {
                std::cout << inst.result << ":" << std::endl;
            } else if (inst.op == "=") {
                std::cout << "    " << inst.result << " = " << inst.arg1 << std::endl;
            } else if (!inst.arg2.empty()) {
                std::cout << "    " << inst.result << " = "
                          << inst.arg1 << " " << inst.op << " " << inst.arg2 << std::endl;
            } else {
                std::cout << "    " << inst.op << " " << inst.arg1 << std::endl;
            }
        }
    }
};
```

测试：翻译 `var x = a * b + c;`

```cpp
int main() {
    // 构造 AST: var x = a * b + c
    auto mul = makeNode(NodeType::BinaryOp, "*");
    mul->addChild(makeNode(NodeType::Identifier, "a"));
    mul->addChild(makeNode(NodeType::Identifier, "b"));
    auto add = makeNode(NodeType::BinaryOp, "+");
    add->addChild(std::move(mul));
    add->addChild(makeNode(NodeType::Identifier, "c"));
    auto decl = makeNode(NodeType::VarDecl);
    decl->addChild(makeNode(NodeType::Identifier, "x"));
    decl->addChild(std::move(add));
    auto prog = makeNode(NodeType::Program);
    prog->addChild(std::move(decl));

    TACGenerator gen;
    gen.translateStmt(prog.get());
    gen.print();
    return 0;
}
```

输出：

```
    t0 = a * b
    t1 = t0 + c
    x = t1
```

## SSA 形式——每个变量只赋值一次

SSA（Static Single Assignment）是一种特殊的 IR 约束：**每个变量只被赋值一次**。需要重新赋值时，创建新版本的变量。

### 普通三地址码 vs SSA

```
// 普通三地址码              // SSA 形式
x = 1                       x₁ = 1
x = x + 2                   x₂ = x₁ + 2
y = x * 3                   y₁ = x₂ * 3
```

SSA 中 `x₁`、`x₂` 是同一个变量的不同版本。

### φ 函数——处理控制流汇合

当两条路径汇合时，不确定变量是哪个版本，需要 φ（phi）函数：

```cpp
// 源码
if (cond)
    x = 1;
else
    x = 2;
y = x + 3;
```

```
// SSA 形式
    if_false cond goto L1
    x₁ = 1
    goto L2
L1: x₂ = 2
L2: x₃ = φ(x₁, x₂)    // 选择来自哪条路径的值
    y₁ = x₃ + 3
```

### SSA 的优势

| 优化类型 | SSA 如何帮助 |
|---------|------------|
| 常量传播 | 变量只有一个定义点，直接替换 |
| 死代码消除 | 没有使用者的定义可以安全删除 |
| 公共子表达式消除 | 相同表达式的结果可以复用 |
| 寄存器分配 | 变量生命周期清晰，易于着色 |

LLVM IR 就是 SSA 形式。不过对于脚本语言的字节码编译器（如 TJS2），通常不使用 SSA——开销不值得。

## 控制流图（CFG）

控制流图将代码划分为**基本块**（Basic Block），每个基本块内没有分支，块之间用有向边连接：

```
        ┌──────────┐
        │ Entry    │
        │ x = 1    │
        │ y = 0    │
        └────┬─────┘
             │
        ┌────▼─────┐
    ┌──►│ L1:      │
    │   │ t = x<10 │
    │   └────┬─────┘
    │        │
    │   ┌────▼─────┐    ┌──────────┐
    │   │ if t     ├───►│ L3:      │
    │   └────┬─────┘    │ return y │
    │        │ true     └──────────┘
    │   ┌────▼─────┐
    │   │ L2:      │
    │   │ y = y+x  │
    │   │ x = x+1  │
    └───┤ goto L1  │
        └──────────┘
```

CFG 是做数据流分析（活跃变量分析、到达定义分析）的基础。

## 对照项目源码——TJS2 的 IR 选择

TJS2 没有使用三地址码或 SSA，而是直接从语法动作生成**基于寄存器的字节码**。这是脚本语言的常见选择：

| IR 方案 | 代表 | 适用场景 |
|---------|------|---------|
| 三地址码 + SSA | LLVM, GCC | 需要重度优化的编译型语言 |
| 栈式字节码 | Java, Python, Lua 5.0 | 编译简单，生成快 |
| 寄存器式字节码 | TJS2, Lua 5.1+ | 执行快（减少栈操作） |
| 直接机器码 | V8 TurboFan, LuaJIT | JIT 编译，极致性能 |

TJS2 选择寄存器式字节码的原因：
1. **执行效率**——操作数在"寄存器"（实际是数组槽位）中，不需要反复压栈弹栈
2. **编译简单**——单遍编译时，Bison 语义动作直接生成寄存器指令
3. **与动态类型配合好**——每个寄存器槽位是一个 `tTJSVariant`，天然支持多类型

下一节我们将详细分析 TJS2 的字节码指令集设计。

## 常见错误及解决方案

### 错误 1：临时变量编号冲突

```cpp
std::string newTemp() { return "t0"; }  // ❌ 永远返回 t0！
```

**解决：** 使用递增计数器（如我们实现的 `tempCount_++`），确保每个临时变量唯一。

### 错误 2：忘记翻译子表达式

```cpp
case NodeType::BinaryOp: {
    std::string temp = newTemp();
    // ❌ 直接用 children 的 value，没有递归翻译
    emit(temp, node->value, node->children[0]->value, node->children[1]->value);
    return temp;
}
```

如果子节点也是 `BinaryOp`，其 `value` 是运算符（如 `"+"`）而非结果。**必须**先递归调用 `translateExpr` 获得子表达式的结果。

### 错误 3：标签跳转目标错误

在生成 `if-else` 或循环的三地址码时，容易搞混 `goto` 的目标标签。**解决：** 画出控制流图再编码；先生成标签名，再在正确位置插入。

## 本节小结

- IR 是前端与后端之间的桥梁，使 M 种语言 × N 种平台只需 M+N 个适配器
- 三地址码是最经典的线性 IR，每条指令最多三个地址
- SSA 形式要求每个变量只赋值一次，通过 φ 函数处理控制流汇合
- 控制流图（CFG）将代码划分为基本块，是数据流分析的基础
- TJS2 直接生成基于寄存器的字节码，跳过了传统的三地址码/SSA 阶段

### 题目 2：画出 CFG

为以下代码画出控制流图（列出每个基本块的内容和边的方向）：

```
x = 0;
while (x < 5) {
    if (x % 2 == 0) {
        y = y + x;
    }
    x = x + 1;
}

```

<details>
<summary>查看答案</summary>

```
BB1 (入口):         BB2 (循环头):      BB3 (if 判断):
  x = 0               t1 = x < 5         t2 = x % 2
  goto BB2             if_false t1         t3 = t2 == 0
                       goto BB5            if_false t3
                       goto BB3            goto BB4
                                           goto BB4_skip

BB4 (if 体):        BB4_skip:           BB5 (出口):
  y = y + x            x = x + 1          print(y)
  goto BB4_skip        goto BB2
```

边：BB1→BB2, BB2→BB3(true), BB2→BB5(false), BB3→BB4(true), BB3→BB4_skip(false), BB4→BB4_skip, BB4_skip→BB2

</details>

### 题目 3：SSA 转换

将以下代码转换为 SSA 形式（包含 φ 函数）：

```
x = 1;
if (flag) x = 2;
y = x + 3;
```

<details>
<summary>查看答案</summary>

```
    x₁ = 1
    if_false flag goto L1
    x₂ = 2
    goto L2
L1: // (不修改 x)
L2: x₃ = φ(x₁, x₂)
    y₁ = x₃ + 3
```

φ(x₁, x₂) 表示：如果从 if 分支来，取 x₂=2；如果从 else 分支来，取 x₁=1。φ 函数在编译器的 SSA 构造阶段自动插入。

</details>
