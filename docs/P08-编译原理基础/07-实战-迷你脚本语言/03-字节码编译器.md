## 本节目标

读完本节后，你将能够：

1. 理解 AST → 字节码的编译流程——递归遍历每个节点、发射（emit）对应指令

## 编译器的整体架构

编译器（Compiler）接收 Parser 输出的 `std::vector<StmtPtr>`，输出 `std::vector<BytecodeChunk>`。

### 核心设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 寄存器分配 | 线性递增 + 块退出回退 | 简单可靠 |
| 常量处理 | 常量池 + `CONST` 指令 | 避免编码浮点数进指令 |
| 函数编译 | 每函数独立 Chunk | 与 VM `CallFrame` 对应 |
| 跳转 | 先占位、后回填（Backpatch） | 编译时不知目标地址 |
| 变量查找 | 作用域链逐层向上 | 支持嵌套和变量遮蔽 |

## 基础设施——寄存器分配器与作用域

### 寄存器分配器

MiniScript 使用**线性递增**策略：每需要一个新寄存器就递增计数器，块作用域退出时回退。

```cpp
// 复用前节的 #include（cstdint, string, vector, unordered_map, variant, memory, iostream）
// 复用 Ch05/Ch06 定义的 OpCode 和 Instruction
enum class OpCode : uint8_t {
    CONST, MOV, ADD, SUB, MUL, DIV, MOD, NEG,
    EQ, LT, LE, NOT, JMP, JF, CALL, RET, PRINT, HALT
};
struct Instruction { OpCode op; uint8_t a, b, c; };

struct BytecodeChunk {
    std::string name;                    // 函数名（顶层为 "<main>"）
    std::vector<Instruction> code;       // 指令序列
    std::vector<double> constants;       // 常量池
    int registerCount = 0;               // 使用的最大寄存器数
    int paramCount    = 0;               // 形参数量
};

// ============ 寄存器分配器 ============
class RegisterAllocator {
    int next_ = 0, peak_ = 0;
public:
    uint8_t alloc() {                              // 分配新寄存器
        if (next_ >= 256) throw std::runtime_error("寄存器溢出");
        uint8_t r = static_cast<uint8_t>(next_++);
        if (next_ > peak_) peak_ = next_;
        return r;
    }
    void resetTo(int level) { next_ = level; }     // 退出作用域时回退
    int current() const { return next_; }
    int peak() const { return peak_; }             // 最大使用量
};
```

**设计要点：** `alloc()` 只递增不复用；退出块作用域时 `resetTo()` 回退水位，内层寄存器即可被外层复用；`peak()` 最终写入 `BytecodeChunk::registerCount`。

### 作用域链

变量名到寄存器的映射用**作用域链**（Scope Chain）管理——每进入一个块 `{}`，压入新作用域；退出时弹出：

```cpp
struct Scope {
    std::unordered_map<std::string, uint8_t> vars;  // 变量名 → 寄存器
    int savedLevel;                                  // 进入时的寄存器水位
};

class ScopeChain {
    std::vector<Scope> chain_;
public:
    void push(int regLevel) { chain_.push_back({{}, regLevel}); }
    int pop() { int l = chain_.back().savedLevel; chain_.pop_back(); return l; }
    void define(const std::string& name, uint8_t reg) { chain_.back().vars[name] = reg; }
    uint8_t resolve(const std::string& name) const {  // 从内层向外层逐层查找
        for (int i = static_cast<int>(chain_.size()) - 1; i >= 0; --i) {
            auto it = chain_[i].vars.find(name);
            if (it != chain_[i].vars.end()) return it->second;
        }
        throw std::runtime_error("未定义的变量: " + name);
    }

    bool empty() const { return chain_.empty(); }
};
```

**查找顺序示例：**

```
全局作用域:  { x → R0, y → R1 }
  │
  └─ if 块:  { z → R2 }
       │
       └─ 嵌套块: { x → R3 }    ← x 遮蔽（shadow）全局 x
```

当嵌套块内引用 `x` 时，`resolve("x")` 从最内层开始找到 `R3`——**变量遮蔽**（Variable Shadowing）。

## 编译器核心——Compiler 类

Compiler 持有 `BytecodeChunk` 列表，每编译一个函数就添加一个新 Chunk：

```cpp
// AST 类型复用上一节定义（完整定义见 02-语法分析与AST构建.md）

class Compiler {
    std::vector<BytecodeChunk> chunks_;
    BytecodeChunk* current_ = nullptr;
    RegisterAllocator regs_;
    ScopeChain scopes_;

    size_t emit(OpCode op, uint8_t a=0, uint8_t b=0, uint8_t c=0) {
        current_->code.push_back({op, a, b, c});
        return current_->code.size() - 1;  // 返回索引，用于回填
    }
    uint8_t addConstant(double value) {
        current_->constants.push_back(value);
        if (current_->constants.size() > 256) throw std::runtime_error("常量池溢出");
        return static_cast<uint8_t>(current_->constants.size() - 1);
    }
    void patchJump(size_t idx) {  // 回填跳转目标地址
        size_t target = current_->code.size();
        current_->code[idx].a = static_cast<uint8_t>(target & 0xFF);
        current_->code[idx].b = static_cast<uint8_t>((target >> 8) & 0xFF);
    }
public:
    std::vector<BytecodeChunk> compile(const std::vector<StmtPtr>& program) {
        chunks_.push_back({"<main>", {}, {}, 0, 0});
        current_ = &chunks_.back();
        scopes_.push(regs_.current());
        for (auto& stmt : program) compileStmt(*stmt);
        emit(OpCode::HALT);
        current_->registerCount = regs_.peak();
        scopes_.pop();
        return std::move(chunks_);
    }
    void compileStmt(const Stmt& stmt);

    // ---- 表达式编译 ----
    // 返回结果存放的寄存器编号
    uint8_t compileExpr(const Expr& expr);
};
```

**关键点：**

- `emit()` 返回指令索引，用于之后的跳转回填
- `addConstant()` 把常量放入池中，返回索引供 `CONST` 指令引用
- `patchJump()` 实现了**回填**（Backpatching）技术——先发射一条跳转指令（目标未知），等编译完分支体后再填入实际地址
- `compile()` 是对外接口：创建 `<main>` Chunk → 编译所有语句 → 发射 `HALT`

## 表达式编译

表达式编译的核心任务是：**计算表达式的值，将结果存入一个寄存器，返回该寄存器编号**。

### 编译数值和布尔字面量

```cpp
uint8_t compileNumberLit(double value) {
    uint8_t reg = regs_.alloc();
    emit(OpCode::CONST, reg, addConstant(value));  // CONST R(reg), K(idx)
    return reg;
}
uint8_t compileBoolLit(bool value) {
    return compileNumberLit(value ? 1.0 : 0.0);   // 布尔用 1.0/0.0 表示
}
```

### 编译标识符（变量引用）

直接在作用域链中查找变量对应的寄存器：

```cpp
uint8_t compileIdentifier(const std::string& name) {
    return scopes_.resolve(name);  // 直接返回变量所在寄存器
}
```

> **注意**：因为变量就存在寄存器里，"读取变量"不需要生成任何指令——这正是寄存器式虚拟机的优势。栈式 VM 需要 `LOAD` 指令把变量压栈。

### 编译二元运算

对于 `a + b`，先编译左右操作数，再发射运算指令：

```cpp
uint8_t compileBinaryExpr(const std::string& op,
                          const Expr& left, const Expr& right) {
    uint8_t lreg = compileExpr(left);   // 编译左操作数
    uint8_t rreg = compileExpr(right);  // 编译右操作数
    uint8_t dst  = regs_.alloc();       // 分配结果寄存器

    // 算术运算符 → 对应 OpCode
    if      (op == "+")  emit(OpCode::ADD, dst, lreg, rreg);
    else if (op == "-")  emit(OpCode::SUB, dst, lreg, rreg);
    else if (op == "*")  emit(OpCode::MUL, dst, lreg, rreg);
    else if (op == "/")  emit(OpCode::DIV, dst, lreg, rreg);
    else if (op == "%")  emit(OpCode::MOD, dst, lreg, rreg);
    // 比较运算符
    else if (op == "==") emit(OpCode::EQ,  dst, lreg, rreg);
    else if (op == "<")  emit(OpCode::LT,  dst, lreg, rreg);
    else if (op == "<=") emit(OpCode::LE,  dst, lreg, rreg);
    // 逻辑运算符（MiniScript 用 and/or 关键字）
    // and/or 需要短路求值——见下文单独处理
    else throw std::runtime_error("未知运算符: " + op);

    return dst;
}
```

生成的指令序列示例（`3 + 4 * 5`）：

```
CONST R0, K0     ; K0 = 3.0
CONST R1, K1     ; K1 = 4.0
CONST R2, K2     ; K2 = 5.0
MUL   R3, R1, R2 ; R3 = 4.0 * 5.0 = 20.0
ADD   R4, R0, R3 ; R4 = 3.0 + 20.0 = 23.0
```

### 编译逻辑运算——短路求值

`and` 和 `or` 需要**短路求值**（Short-Circuit Evaluation）——不能先算两边再运算：

```cpp
uint8_t compileAnd(const Expr& left, const Expr& right) {
    uint8_t lreg = compileExpr(left);
    size_t jumpFalse = emit(OpCode::JF, 0, 0, lreg);  // 左为假 → 跳过右侧
    uint8_t rreg = compileExpr(right);
    emit(OpCode::MOV, lreg, rreg);
    patchJump(jumpFalse);
    return lreg;
}

uint8_t compileOr(const Expr& left, const Expr& right) {
    uint8_t lreg = compileExpr(left);
    uint8_t tmp = regs_.alloc();
    emit(OpCode::NOT, tmp, lreg);                       // 取反
    size_t jumpTrue = emit(OpCode::JF, 0, 0, tmp);     // !left 为假(left 为真) → 跳过
    uint8_t rreg = compileExpr(right);
    emit(OpCode::MOV, lreg, rreg);
    patchJump(jumpTrue);
    return lreg;
}
```

### 编译一元运算、赋值与函数调用

```cpp
uint8_t compileUnaryExpr(const std::string& op, const Expr& operand) {
    uint8_t src = compileExpr(operand);
    uint8_t dst = regs_.alloc();
    if (op == "-")        emit(OpCode::NEG, dst, src);
    else if (op == "not") emit(OpCode::NOT, dst, src);
    else throw std::runtime_error("未知一元运算符: " + op);
    return dst;
}

uint8_t compileAssignExpr(const std::string& name, const Expr& value) {
    uint8_t valReg = compileExpr(value);
    uint8_t varReg = scopes_.resolve(name);   // 查找目标变量
    emit(OpCode::MOV, varReg, valReg);        // 赋值 = MOV
    return varReg;
}

uint8_t compileCallExpr(const std::string& callee,
                        const std::vector<ExprPtr>& args) {
    // 在 chunks_ 中按名字查找函数索引
    int funcIdx = -1;
    for (int i = 0; i < static_cast<int>(chunks_.size()); ++i)
        if (chunks_[i].name == callee) { funcIdx = i; break; }
    if (funcIdx < 0) throw std::runtime_error("未定义的函数: " + callee);
    // 编译参数并 MOV 到连续寄存器区域
    std::vector<uint8_t> argRegs;
    for (auto& arg : args) argRegs.push_back(compileExpr(*arg));
    uint8_t dst = regs_.alloc();
    for (int i = 0; i < static_cast<int>(argRegs.size()); ++i)
        emit(OpCode::MOV, static_cast<uint8_t>(dst + 1 + i), argRegs[i]);
    emit(OpCode::CALL, static_cast<uint8_t>(funcIdx), dst,
         static_cast<uint8_t>(argRegs.size()));
    return dst;  // VM 将返回值写入 dst
}
```

### 完整的 compileExpr——std::visit 分发

用 `std::visit` + `if constexpr` 对 variant 进行类型安全分发，每种 AST 节点调用对应编译函数：

```cpp
uint8_t Compiler::compileExpr(const Expr& expr) {
    return std::visit([this](const auto& node) -> uint8_t {
        using T = std::decay_t<decltype(node)>;
        if constexpr (std::is_same_v<T, NumberLit>)
            return compileNumberLit(node.value);
        else if constexpr (std::is_same_v<T, BoolLit>)
            return compileBoolLit(node.value);
        else if constexpr (std::is_same_v<T, Identifier>)
            return compileIdentifier(node.name);
        else if constexpr (std::is_same_v<T, std::unique_ptr<UnaryExpr>>)
            return compileUnaryExpr(node->op, *node->operand);
        else if constexpr (std::is_same_v<T, std::unique_ptr<BinaryExpr>>) {
            if (node->op == "and") return compileAnd(*node->left, *node->right);
            if (node->op == "or")  return compileOr(*node->left, *node->right);
            return compileBinaryExpr(node->op, *node->left, *node->right);
        }
        else if constexpr (std::is_same_v<T, std::unique_ptr<AssignExpr>>)
            return compileAssignExpr(node->name, *node->value);
        else if constexpr (std::is_same_v<T, std::unique_ptr<CallExpr>>)
            return compileCallExpr(node->callee, node->args);
        else throw std::runtime_error("compileExpr: 未处理的表达式类型");
    }, expr.node);
}
```

## 语句编译——控制流与函数

### VarDecl、Print、Block

```cpp
void compileVarDecl(const VarDecl& decl) {
    uint8_t reg = regs_.alloc();              // 为变量分配寄存器
    scopes_.define(decl.name, reg);           // 登记到作用域
    if (decl.init) {
        uint8_t val = compileExpr(*decl.init);
        emit(OpCode::MOV, reg, val);
    } else {
        emit(OpCode::CONST, reg, addConstant(0.0));  // 默认值 0
    }
}

void compilePrintStmt(const PrintStmt& s) {
    emit(OpCode::PRINT, compileExpr(*s.expr));
}

void compileBlock(const Block& block) {
    scopes_.push(regs_.current());
    for (auto& stmt : block.stmts) compileStmt(*stmt);
    regs_.resetTo(scopes_.pop());             // 弹出并回退寄存器
}
```

### if/else——跳转回填

编译时不知道跳转距离，先发射占位指令，后用 `patchJump()` 填入实际地址：

```
  [编译条件 → R(cond)]
  JF   else_addr, R(cond)     ← 待回填
  [then 分支]
  JMP  end_addr               ← 待回填
else_addr:  [else 分支]
end_addr:
```

```cpp
void compileIfStmt(const IfStmt& stmt) {
    uint8_t cond = compileExpr(*stmt.cond);
    size_t jmpFalse = emit(OpCode::JF, 0, 0, cond);
    compileStmt(*stmt.then_);
    if (stmt.else_) {
        size_t jmpEnd = emit(OpCode::JMP);
        patchJump(jmpFalse);
        compileStmt(*stmt.else_);
        patchJump(jmpEnd);
    } else {
        patchJump(jmpFalse);
    }
}
```

### while——前跳与回跳

```cpp
void compileWhileStmt(const WhileStmt& stmt) {
    size_t loopStart = current_->code.size();
    uint8_t cond = compileExpr(*stmt.cond);
    size_t jmpFalse = emit(OpCode::JF, 0, 0, cond);
    compileStmt(*stmt.body);
    emit(OpCode::JMP, static_cast<uint8_t>(loopStart & 0xFF),
         static_cast<uint8_t>((loopStart >> 8) & 0xFF));
    patchJump(jmpFalse);
}
```

### 函数编译——独立 Chunk

每个函数编译为独立的 `BytecodeChunk`，编译器保存/恢复当前状态来切换上下文：

```cpp
void compileFnDecl(const FnDecl& decl) {
    // 保存外层状态
    BytecodeChunk* outerChunk = current_;
    RegisterAllocator outerRegs = regs_;
    ScopeChain outerScopes = std::move(scopes_);
    // 创建新 Chunk
    chunks_.push_back({decl.name, {}, {}, 0, static_cast<int>(decl.params.size())});
    current_ = &chunks_.back();
    regs_ = RegisterAllocator{};
    scopes_ = ScopeChain{};
    // 形参绑定到 R0, R1, R2, ...
    scopes_.push(regs_.current());
    for (auto& p : decl.params) { scopes_.define(p, regs_.alloc()); }
    // 编译函数体
    for (auto& stmt : decl.body->stmts) compileStmt(*stmt);
    emit(OpCode::RET, 0, 0);  // 隐式 return
    current_->registerCount = regs_.peak();
    // 恢复外层
    scopes_.pop();
    current_ = outerChunk;
    regs_ = outerRegs;
    scopes_ = std::move(outerScopes);
}
```

### compileStmt 分发

```cpp
void Compiler::compileStmt(const Stmt& stmt) {
    std::visit([this](const auto& node) {
        using T = std::decay_t<decltype(node)>;
        if constexpr (std::is_same_v<T, VarDecl>)          compileVarDecl(node);
        else if constexpr (std::is_same_v<T, PrintStmt>)   compilePrintStmt(node);
        else if constexpr (std::is_same_v<T, ExprStmt>)    compileExpr(*node.expr);
        else if constexpr (std::is_same_v<T, ReturnStmt>) {
            if (node.value) emit(OpCode::RET, compileExpr(*node.value), 1);
            else emit(OpCode::RET, 0, 0);
        }
        else if constexpr (std::is_same_v<T, std::unique_ptr<Block>>)     compileBlock(*node);
        else if constexpr (std::is_same_v<T, std::unique_ptr<IfStmt>>)    compileIfStmt(*node);
        else if constexpr (std::is_same_v<T, std::unique_ptr<WhileStmt>>) compileWhileStmt(*node);
        else if constexpr (std::is_same_v<T, std::unique_ptr<FnDecl>>)    compileFnDecl(*node);
    }, stmt.node);
}
```

## 常见错误与解决方案

### 错误 1：跳转回填遗漏

编译 if/while 后忘记 `patchJump()` → VM 跳转到地址 0 → 无限循环或越界崩溃。

**解决：** 每个 `emit(JF/JMP)` 必须有对应的 `patchJump()`。编译结束后可扫描验证：

```cpp
void verifyJumps(const BytecodeChunk& chunk) {
    for (size_t i = 0; i < chunk.code.size(); ++i) {
        auto& c = chunk.code[i];
        if (c.op == OpCode::JMP || c.op == OpCode::JF) {
            size_t target = static_cast<size_t>(c.a) | (static_cast<size_t>(c.b) << 8);
            if (target > chunk.code.size())
                std::cerr << "指令 " << i << " 跳转越界: " << target << "\n";
        }
    }
}
```

### 错误 2：块作用域寄存器泄漏

`compileBlock` 中只调用 `scopes_.pop()` 但忘记 `regs_.resetTo()` → 寄存器永远递增 → 256 溢出。

**解决：** 始终成对：`scopes_.push(regs_.current())` + `regs_.resetTo(scopes_.pop())`。

### 错误 3：函数编译破坏外层状态

编译函数体时没保存 `current_`、`regs_`、`scopes_`，导致函数指令写入外层 Chunk。

**解决：** `compileFnDecl` 入口保存三件套，出口恢复。

## 对照项目源码

TJS2 的编译器位于以下文件：

- **`cpp/core/tjs2/tjsInterCodeGen.h`**（872 行）—— `tTJSInterCodeGenerator` 类，负责将 AST 编译为字节码

TJS2 的编译方式与我们的 MiniScript 编译器类似，但更复杂：

```
MiniScript Compiler             TJS2 InterCodeGenerator
─────────────────────         ───────────────────────────
compileExpr() 递归遍历         GenNodeCode() 递归遍历
线性寄存器分配                 帧偏移分配（FrameBase + offset）
scope chain                    Namespace + Context
BytecodeChunk                  tTJSInterCodeContext
```

TJS2 使用**帧内偏移**而非绝对寄存器编号——每个函数帧有自己的局部变量区，通过 `FrameBase` 偏移访问。这本质上与我们的"函数独立 Chunk + 寄存器"方案等价。

## 本节小结

- **编译器 = AST 递归遍历器 + 指令发射器**——对每种 AST 节点生成对应的字节码指令
- **线性寄存器分配**：`alloc()` 递增、块退出 `resetTo()` 回退，简单可靠
- **作用域链**管理变量名 → 寄存器映射，支持嵌套作用域和变量遮蔽
- **回填技术**解决了编译时不知道跳转目标的问题——先留空、后填入
- **每个函数编译为独立 Chunk**，编译器通过保存/恢复状态切换上下文
- `std::visit` + `if constexpr` 模式对 `std::variant` AST 进行类型安全的分发

## 练习题与答案

### 题目 1：添加 `>` 和 `>=` 运算符

当前 OpCode 只有 `LT` 和 `LE`。不增加新 OpCode，如何在编译器层面支持 `>` 和 `>=`？

<details>
<summary>查看答案</summary>

**思路：** 交换操作数——`a > b` 等价于 `b < a`，`a >= b` 等价于 `b <= a`。

```cpp
// 在 compileBinaryExpr 中添加：
else if (op == ">")  emit(OpCode::LT, dst, rreg, lreg);   // 交换操作数
else if (op == ">=") emit(OpCode::LE, dst, rreg, lreg);   // 交换操作数
```

这样不需要新增 OpCode，VM 也不需要修改——编译器通过操作数交换实现语义转换。这种技巧在实际编译器中很常见。

</details>

### 题目 2：为编译器添加反汇编器

实现一个 `disassemble()` 函数，将 `BytecodeChunk` 反汇编为可读文本，用于调试编译结果。

<details>
<summary>查看答案</summary>

```cpp
void disassemble(const BytecodeChunk& chunk) {
    const char* names[] = {
        "CONST","MOV","ADD","SUB","MUL","DIV","MOD","NEG",
        "EQ","LT","LE","NOT","JMP","JF","CALL","RET","PRINT","HALT"
    };
    std::cout << "=== " << chunk.name << " (regs="
              << chunk.registerCount << " params=" << chunk.paramCount << ") ===\n";
    for (size_t i = 0; i < chunk.code.size(); ++i) {
        auto& c = chunk.code[i];
        std::cout << "[" << i << "] " << names[static_cast<int>(c.op)]
                  << " " << (int)c.a << " " << (int)c.b << " " << (int)c.c << "\n";
    }
}
```

可根据 OpCode 类型细化输出格式（如 JMP/JF 显示跳转目标、CONST 显示常量值等），核心思路不变。

</details>

### 题目 3：实现 for 循环的语法糖编译

如果添加 `for (init; cond; incr) body` 语法，如何在编译器层将其转换为等价的 while？

<details>
<summary>查看答案</summary>

`for (init; cond; incr) body` 等价于 `{ init; while (cond) { body; incr; } }`。编译器实现：

```cpp
void compileForStmt(const ForStmt& stmt) {
    scopes_.push(regs_.current());
    if (stmt.init) compileStmt(*stmt.init);
    size_t loopStart = current_->code.size();
    size_t jmpFalse = 0;
    if (stmt.cond) {
        uint8_t cond = compileExpr(*stmt.cond);
        jmpFalse = emit(OpCode::JF, 0, 0, cond);
    }
    compileStmt(*stmt.body);
    if (stmt.incr) compileExpr(*stmt.incr);
    emit(OpCode::JMP, static_cast<uint8_t>(loopStart & 0xFF),
         static_cast<uint8_t>((loopStart >> 8) & 0xFF));
    if (stmt.cond) patchJump(jmpFalse);
    int level = scopes_.pop();
    regs_.resetTo(level);
}
```

语法糖编译的好处：前端增加语法，VM 无需任何改动。

</details>
