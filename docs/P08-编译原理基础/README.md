# P08 — 编译原理基础

> **前置知识：** [P03-跨平台C++开发](../P03-跨平台C++开发/README.md)（C++17 基础）
> **难度：** ★★★★（中高级）
> **预计总学时：** 20-28 小时

## 模块概述

编译原理是理解任何脚本引擎的基石。KrKr2 项目内置了完整的 TJS2（T Visual Presenter JavaScript 2）脚本引擎，其实现涵盖了从词法分析到字节码虚拟机的完整编译流水线。TJS2 位于 `cpp/core/tjs2/` 目录，包含手写词法器（`tjsLex.cpp`）、Bison 语法分析器（`bison/tjs.y`）、字节码编译器和基于栈的虚拟机。

本模块从编译器的五个经典阶段出发，逐步讲解词法分析、语法分析、语义分析、中间表示生成和虚拟机执行，最终实战从零构建一个迷你脚本语言。每一章都会对照 TJS2 的实际源码，让你不仅掌握理论，更能读懂真实编译器的实现。

## 学习目标

完成本模块后，你将能够：

1. 理解编译器/解释器的五阶段流水线，能区分编译型、解释型和混合型语言实现
2. 手写一个词法分析器，能将源代码转化为 Token 序列，处理字符串、数字、注释等各类词素
3. 使用 BNF/EBNF 描述语法规则，能编写 Bison 语法文件并生成解析器
4. 理解语义分析的作用，能实现符号表管理、类型检查和作用域规则
5. 设计字节码指令集，能将 AST 编译为字节码序列
6. 实现基于栈的虚拟机，能执行字节码指令、管理调用栈和垃圾回收
7. 阅读并理解 KrKr2 项目中 `cpp/core/tjs2/` 的 TJS2 引擎完整实现

## 项目源码对照

本模块将频繁引用以下 KrKr2 源文件：

| 文件 | 内容 |
|------|------|
| `cpp/core/tjs2/tjsLex.cpp` / `tjsLex.h` | TJS2 手写词法分析器 — Token 生成、字符串/数字解析 |
| `cpp/core/tjs2/bison/tjs.y` | TJS2 Bison 语法文件 — 完整文法规则、AST 构建动作 |
| `cpp/core/tjs2/tjsInterCodeGen.cpp` / `.h` | TJS2 中间代码生成器 — AST→字节码编译 |
| `cpp/core/tjs2/tjsInterCodeExec.cpp` | TJS2 字节码执行器 — VM 主循环、指令分派 |
| `cpp/core/tjs2/tjsObject.cpp` / `.h` | TJS2 对象系统 — 基类、属性访问、方法调用 |
| `cpp/core/tjs2/tjsVariant.cpp` / `.h` | TJS2 值类型 — Variant 联合体、类型转换 |
| `cpp/core/tjs2/tjsByteCodeLoader.cpp` | 字节码加载器 — 序列化/反序列化字节码 |
| `cpp/core/tjs2/tjspp.cpp` | TJS2 预处理器 — 宏展开、条件编译 |
| `cpp/core/tjs2/tjsGlobalStringMap.cpp` | 全局字符串池 — 字符串 intern 机制 |

## 环境准备

```bash
# 本模块的示例代码只需要标准 C++17 编译器
# Windows
cl /std:c++17 /EHsc example.cpp

# Linux / macOS
g++ -std=c++17 -o example example.cpp
# 或
clang++ -std=c++17 -o example example.cpp

# 如需使用 Bison（第3章）
# Windows — 需要 winflexbison（KrKr2 项目已依赖）
choco install winflexbison3
# 或手动下载：https://github.com/lexxmark/winflexbison/releases

# Linux
sudo apt install bison flex

# macOS
brew install bison flex
```
