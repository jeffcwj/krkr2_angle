- 掌握 x86（IA-32）和 x86-64（AMD64）通用寄存器的名称、大小和用途

- 熟练阅读 MOV、LEA、ADD、SUB、CMP、TEST、JMP/Jcc 等常见指令

## 1. 寄存器总览

### 1.1 x86（32 位）通用寄存器

| 寄存器 | 低 16 位 | 高 8 位 / 低 8 位 | 传统用途 |
|---------|---------|-------------------|---------|
| **EAX** | AX | AH / AL | 累加器，函数返回值 |
| **EBX** | BX | BH / BL | 基址寄存器（被调用者保存） |
| **ECX** | CX | CH / CL | 计数器（循环、移位） |
| **EDX** | DX | DH / DL | 数据寄存器（乘除扩展） |
| **ESI** | SI | — | 源索引（字符串操作） |
| **EDI** | DI | — | 目的索引（字符串操作） |
| **EBP** | BP | — | 帧指针（栈帧基址） |
| **ESP** | SP | — | 栈指针（栈顶） |

> **逆向要点**：在 IDA 反汇编中，`eax` 几乎总是函数返回值；`ecx` 在 `thiscall` 约定下传递 `this` 指针——这在分析 KiriKiri2 C++ 代码时极为常见。

```nasm
; 典型的 KiriKiri2 thiscall 调用模式
mov     ecx, [ebp+var_this]    ; this 指针 → ECX
call    ?GetCount@TJS::tTJSArrayObject@@QAEHXZ
; 返回值在 EAX 中
mov     [ebp+var_count], eax
```

### 1.2 x86-64（64 位）通用寄存器

64 位模式扩展了寄存器宽度并新增了 R8–R15：

```
┌────────────────────────────────────────────────────────────┐
│ 63                    31          15      7       0        │
├────────────────────────┬───────────┬───────┬───────┬───────┤
│         RAX            │    EAX    │  AX   │  AH   │  AL  │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RBX            │    EBX    │  BX   │  BH   │  BL  │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RCX            │    ECX    │  CX   │  CH   │  CL  │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RDX            │    EDX    │  DX   │  DH   │  DL  │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RSI            │    ESI    │  SI   │       │  SIL │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RDI            │    EDI    │  DI   │       │  DIL │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RBP            │    EBP    │  BP   │       │  BPL │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         RSP            │    ESP    │  SP   │       │  SPL │
├────────────────────────┼───────────┼───────┼───────┼───────┤
│         R8             │    R8D    │  R8W  │       │  R8B │
│         ...            │    ...    │  ...  │       │  ... │
│         R15            │   R15D    │ R15W  │       │ R15B │
└────────────────────────┴───────────┴───────┴───────┴───────┘
```

**关键差异**：

| 特性 | x86 (32-bit) | x86-64 (64-bit) |
|------|-------------|-----------------|
| 通用寄存器数量 | 8 个 | 16 个 (多 R8–R15) |
| 指针大小 | 4 字节 | 8 字节 |
| 默认函数参数传递 | 全部通过栈 | 前 4/6 个通过寄存器 |
| `this` 指针 | ECX (thiscall) | RCX (MS x64) |

### 1.3 特殊用途寄存器

```nasm
; EIP / RIP — 指令指针（不可直接赋值）
; 控制流跳转实际就是修改 EIP/RIP

; 段寄存器（现代保护模式下几乎不直接使用）
; CS, DS, SS, ES, FS, GS
; 但 FS/GS 在 SEH（结构化异常处理）和 TLS 中仍有用:
mov     eax, fs:[0]          ; x86 SEH 链头
mov     rax, gs:[0x30]       ; x64 TEB (Thread Environment Block)
```

> **实战提示**：在 KiriKiri2 反汇编中看到 `fs:[0]` 访问，通常是异常处理安装（`__try/__except`）的标志。

## 2. 标志寄存器（EFLAGS / RFLAGS）

### 2.1 关键标志位

标志寄存器是条件跳转的核心依据：

```
Bit  名称    缩写   含义
────────────────────────────────────────────────────────
 0   Carry    CF    无符号运算溢出（进位/借位）
 2   Parity   PF    结果低 8 位中 1 的个数为偶数
 6   Zero     ZF    运算结果为零
 7   Sign     SF    结果的最高位（符号位）
11   Overflow OF    有符号运算溢出
10   Direction DF   字符串操作方向（0=递增，1=递减）
```

### 2.2 标志位与条件跳转的对应关系

```
┌──────────────┬──────────────┬──────────────────────────┐
│   指令       │  跳转条件     │ 检查的标志位              │
├──────────────┼──────────────┼──────────────────────────┤
│ JZ  / JE     │ 相等/为零     │ ZF = 1                   │
│ JNZ / JNE    │ 不等/非零     │ ZF = 0                   │
│ JL  / JNGE   │ 小于(有符号)  │ SF ≠ OF                  │
│ JGE / JNL    │ ≥(有符号)     │ SF = OF                  │
│ JG  / JNLE   │ 大于(有符号)  │ ZF=0 且 SF=OF            │
│ JLE / JNG    │ ≤(有符号)     │ ZF=1 或 SF≠OF            │
│ JB  / JNAE   │ 小于(无符号)  │ CF = 1                   │
│ JAE / JNB    │ ≥(无符号)     │ CF = 0                   │
│ JA  / JNBE   │ 大于(无符号)  │ CF=0 且 ZF=0             │
│ JBE / JNA    │ ≤(无符号)     │ CF=1 或 ZF=1             │
│ JS           │ 负数          │ SF = 1                   │
│ JO           │ 溢出          │ OF = 1                   │
└──────────────┴──────────────┴──────────────────────────┘
```

> **记忆技巧**：`B`elow/`A`bove = 无符号比较，`L`ess/`G`reater = 有符号比较。

## 3. 数据传送指令

### 3.1 MOV — 数据复制

`MOV` 是最常见的指令，将源操作数复制到目的操作数：

```nasm
; 寄存器 ← 立即数
mov     eax, 42              ; eax = 42

; 寄存器 ← 寄存器
mov     ecx, edx             ; ecx = edx

; 寄存器 ← 内存
mov     eax, [ebp-8]         ; eax = *(int*)(ebp-8)  // 读取局部变量

; 内存 ← 寄存器
mov     [ebp-0Ch], eax       ; *(int*)(ebp-0xC) = eax // 写入局部变量

; 内存 ← 立即数
mov     dword ptr [esi+10h], 0 ; *(int*)(esi+0x10) = 0
```

**常见错误与解析**：

```nasm
; ❌ 错误理解：以下两条指令功能相同
mov     eax, [ebx]           ; eax = *ebx    （读取 ebx 指向的内存）
mov     eax, ebx             ; eax = ebx     （复制 ebx 的值）
; ✅ 方括号表示内存解引用，没有方括号是寄存器间复制
```

### 3.2 LEA — 加载有效地址

`LEA`（Load Effective Address）计算地址但**不访问内存**：

```nasm
; LEA 的地址计算能力
lea     eax, [ebx+ecx*4+10h]  ; eax = ebx + ecx*4 + 0x10
; 等效 C: eax = (char*)ebx + ecx*4 + 16;

; 编译器常用 LEA 做快速算术（不影响标志位！）
lea     eax, [eax+eax*2]      ; eax = eax * 3
lea     eax, [eax*4+eax]      ; eax = eax * 5
lea     eax, [eax*8]          ; eax = eax * 8
```

**MOV vs LEA 对比**：

```nasm
; 假设 EBX = 0x00401000，该地址处存储的值为 0xDEADBEEF
mov     eax, [ebx+4]    ; eax = *(0x00401004) = 某个内存值
lea     eax, [ebx+4]    ; eax = 0x00401004    = 地址本身

; 逆向中的典型场景：获取结构体成员地址
; C++ 源码: &obj.member_at_offset_0x1C
lea     ecx, [esi+1Ch]  ; ecx = &(this->member)
push    ecx              ; 作为参数传递
call    SomeFunction
```

### 3.3 MOVZX / MOVSX — 零扩展与符号扩展

```nasm
; MOVZX: 零扩展（无符号）
movzx   eax, byte ptr [esi]   ; 读取 1 字节，高位补 0
movzx   ecx, word ptr [edi]   ; 读取 2 字节，高位补 0

; MOVSX: 符号扩展（有符号）
movsx   eax, byte ptr [esi]   ; 读取 1 字节，高位用符号位填充
; 若 [esi] = 0xFF (即 -1)，则 eax = 0xFFFFFFFF

; KiriKiri2 中常见：TJS2 字符用 tjs_char (wchar_t = 2 字节)
movzx   eax, word ptr [ebx+ecx*2]  ; 读取 TJS 字符串中第 ecx 个字符
```

## 4. 算术与逻辑指令

### 4.1 ADD / SUB / INC / DEC

```nasm
add     eax, ebx         ; eax += ebx     (影响 CF, ZF, SF, OF)
sub     ecx, 10h         ; ecx -= 0x10    (影响 CF, ZF, SF, OF)
inc     edx              ; edx++          (不影响 CF!)
dec     esi              ; esi--          (不影响 CF!)

; 逆向陷阱：inc/dec 不影响 CF
; 如果后续有 JB/JAE（检查 CF），inc 的结果不会触发它们
```

### 4.2 AND / OR / XOR / NOT

```nasm
and     eax, 0FFh        ; eax &= 0xFF   保留低 8 位（掩码操作）
or      ecx, 80000000h   ; ecx |= 0x80000000  设置最高位
xor     eax, eax         ; eax = 0        （最常见的清零手法）
not     edx              ; edx = ~edx     按位取反

; XOR 清零比 MOV EAX, 0 更短（2 字节 vs 5 字节）
; 在反汇编中看到 xor reg, reg 几乎一定是清零
```

### 4.3 SHL / SHR / SAR — 移位操作

```nasm
shl     eax, 2           ; eax <<= 2   (等效 eax *= 4)
shr     ebx, 3           ; ebx >>= 3   逻辑右移（高位补 0）
sar     ecx, 1           ; ecx >>= 1   算术右移（保留符号位）

; 编译器优化：乘除法常被替换为移位
; n * 8  → shl n, 3
; n / 4  → sar n, 2  (有符号) 或 shr n, 2 (无符号)
```

### 4.4 IMUL / MUL / IDIV / DIV

```nasm
; 双操作数 IMUL（有符号乘法，最常见形式）
imul    eax, ecx         ; eax = eax * ecx
imul    edx, [ebp+8], 14h  ; edx = *(ebp+8) * 0x14

; 单操作数形式（结果在 EDX:EAX）
mul     ecx              ; EDX:EAX = EAX * ECX (无符号)
imul    ecx              ; EDX:EAX = EAX * ECX (有符号)

; 除法（被除数在 EDX:EAX 中）
cdq                      ; 将 EAX 符号扩展到 EDX:EAX
idiv    ecx              ; EAX = EDX:EAX / ECX, EDX = 余数

; ⚠ 常见逆向陷阱：除法前的 CDQ/XOR EDX,EDX
; 有符号除法前用 CDQ，无符号除法前用 XOR EDX, EDX
xor     edx, edx         ; EDX = 0 (无符号除法准备)
div     ebx              ; EAX = EAX / EBX (无符号)
```

## 5. 比较与测试指令

### 5.1 CMP — 比较（隐式减法）

`CMP` 执行减法但**不保存结果**，仅设置标志位：

```nasm
cmp     eax, 5           ; 计算 eax - 5，设置 ZF/SF/CF/OF
je      label_equal      ; 若 eax == 5 (ZF=1) 则跳转
jl      label_less       ; 若 eax < 5 (有符号) 则跳转
jb      label_below      ; 若 eax < 5 (无符号) 则跳转

; 对应 C 代码:
; if (value == 5) goto label_equal;
; if (value < 5)  goto label_less;   // signed
; if (value < 5u) goto label_below;  // unsigned
```

**实际反汇编示例**——KiriKiri2 类型检查：

```nasm
; C++: if (type == tvtString) { ... }
; tvtString 的枚举值可能是 4
mov     eax, [esi+0Ch]       ; eax = variant.type
cmp     eax, 4               ; type == tvtString?
jnz     short loc_NotString  ; 不等则跳过
; --- 处理字符串类型 ---
mov     ecx, [esi+10h]       ; ecx = variant.string_ptr
...
loc_NotString:
```

### 5.2 TEST — 按位与测试

`TEST` 执行 AND 但**不保存结果**，仅设置标志位：

```nasm
; 最常见用法：检查寄存器是否为 0 / NULL
test    eax, eax         ; eax & eax，设置 ZF
jz      label_null       ; 若 eax == 0 则跳转
; 对应 C: if (ptr == NULL)

; 检查特定位
test    ecx, 1           ; 检查最低位（奇偶判断）
jnz     label_odd        ; 若最低位=1 (奇数) 则跳转
; 对应 C: if (n % 2 != 0) 或 if (n & 1)

; 检查标志位
test    eax, 80000000h   ; 检查最高位（符号位）
jnz     label_negative   ; 若为负数则跳转

; KiriKiri2 中的标志位检查:
; flags & TJS_MEMBERENSURE
test    dword ptr [ebp+arg_flags], 200h
jz      short loc_NoEnsure
```

**CMP vs TEST 使用场景**：

```
┌──────────────────────────────────────────────────────────┐
│  目的              │ 使用指令      │ 原因               │
├──────────────────────────────────────────────────────────┤
│ 与具体值比较       │ CMP a, imm    │ 减法比较            │
│ 两个值比较         │ CMP a, b      │ 减法比较            │
│ 是否为零/NULL      │ TEST a, a     │ 比 CMP a, 0 更短   │
│ 检查特定位         │ TEST a, mask  │ 位操作              │
└──────────────────────────────────────────────────────────┘
```

## 6. 跳转指令

### 6.1 无条件跳转

```nasm
jmp     loc_Target       ; 直接跳转（常量目标地址）
jmp     eax              ; 间接跳转（跳转到 eax 指向的地址）
jmp     [eax+ecx*4]      ; 跳转表（switch-case 实现）

; switch-case 跳转表实现示例
cmp     eax, 5           ; case 数量检查
ja      loc_Default      ; > 5 跳转到 default
jmp     ds:jpt_401000[eax*4]  ; 跳转表
; IDA 会标注为 "switch jump" 并列出所有 case
```

### 6.2 条件跳转模式识别

**if-else 模式**：

```nasm
; C: if (a > b) { X } else { Y }
cmp     eax, ebx
jle     short loc_Else   ; a <= b 跳到 else
; --- if 分支 (X) ---
...
jmp     short loc_End    ; 跳过 else
loc_Else:
; --- else 分支 (Y) ---
...
loc_End:
```

**循环模式**（for/while）：

```nasm
; C: for (int i = 0; i < count; i++) { body }
xor     ecx, ecx         ; i = 0
jmp     short loc_Cond   ; 先检查条件
loc_Body:
; --- 循环体 ---
...
inc     ecx              ; i++
loc_Cond:
cmp     ecx, [ebp+count] ; i < count?
jl      short loc_Body   ; 是则继续循环

; do-while 模式（条件在尾部）:
loc_Body:
; --- 循环体 ---
...
dec     ecx              ; count--
jnz     short loc_Body   ; count != 0 则继续
```

## 7. 栈操作指令

### 7.1 PUSH / POP

```nasm
push    eax              ; ESP -= 4; [ESP] = EAX
pop     ebx              ; EBX = [ESP]; ESP += 4

; 函数调用参数传递 (cdecl/stdcall, 32 位)
push    0                ; 参数 3 (从右到左)
push    dword ptr [esi]  ; 参数 2
push    ecx              ; 参数 1
call    SomeFunction
; cdecl: 调用者清栈
add     esp, 0Ch         ; 清理 3 个参数 (3 * 4 = 12 = 0Ch)
```

### 7.2 CALL / RET

```nasm
; CALL = push EIP; jmp target
call    sub_401000       ; 直接调用
call    [eax+8]          ; 虚函数调用：*(vtable + 8)
call    dword ptr [edx]  ; 间接调用

; RET = pop EIP
ret                      ; 返回（cdecl）
ret     8                ; 返回并从栈弹出 8 字节（stdcall，2 个参数）
```

**虚函数调用识别**：

```nasm
; C++: obj->VirtualMethod()
; 虚函数调用的典型模式:
mov     eax, [ecx]       ; eax = vtable 指针 (*this 的前 4/8 字节)
call    [eax+1Ch]        ; 调用 vtable[7] (0x1C / 4 = 7)

; KiriKiri2 中大量使用虚函数:
; iTJSDispatch2 接口的 FuncCall:
mov     edx, [ecx]           ; vtable
push    0                     ; result
push    0                     ; numparams
push    offset aOnCreate      ; member name "onCreate"
push    0                     ; flag
call    dword ptr [edx+44h]  ; iTJSDispatch2::FuncCall (vtable 偏移)
```

## 8. 动手实践

### 练习 1：手动反汇编阅读

分析以下反汇编代码，写出等效的 C 代码：

```nasm
; 函数 sub_401000
push    ebp
mov     ebp, esp
sub     esp, 8
mov     eax, [ebp+8]         ; 参数 1
mov     ecx, [ebp+0Ch]       ; 参数 2
xor     edx, edx             ; edx = 0
test    ecx, ecx
jle     short loc_Exit        ; if (ecx <= 0) goto exit
loc_Loop:
add     edx, [eax]           ; edx += *eax
add     eax, 4               ; eax += 4 (下一个 int)
dec     ecx                   ; ecx--
jnz     short loc_Loop        ; if (ecx != 0) continue
loc_Exit:
mov     eax, edx              ; return edx
mov     esp, ebp
pop     ebp
ret
```

<details>
<summary>参考答案</summary>

```c
// int sum_array(int* arr, int count)
int sub_401000(int* arr, int count) {
    int sum = 0;               // xor edx, edx
    if (count <= 0)            // test ecx, ecx; jle
        return sum;
    for (; count > 0; count--) { // dec ecx; jnz
        sum += *arr;            // add edx, [eax]
        arr++;                  // add eax, 4
    }
    return sum;                 // mov eax, edx
}
```

这是一个典型的数组求和函数。编译器将 `for(int i=0; i<count; i++)` 优化为倒计数循环。

</details>

### 练习 2：用 NASM 编写并反汇编

```nasm
; exercise_registers.asm
; nasm -f win32 exercise_registers.asm -o exercise_registers.obj

; int test_arithmetic(int a, int b)

    mov     ebp, esp
    mov     eax, [ebp+8]       ; eax = a
    lea     eax, [eax+eax*2]   ; eax = a * 3

    ret
```

```bash
# Linux
```

### 练习 3：编译器输出对比

用不同优化级别编译同一函数，观察指令差异：

```c
// compare_opt.c
int find_max(int* arr, int n) {
    int max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max)
            max = arr[i];
    }
    return max;
}
```

```bash
# 无优化 — 逐行对应源码，易于理解
gcc -m32 -O0 -S -masm=intel compare_opt.c -o compare_O0.s

# 中等优化 — 使用寄存器变量，减少内存访问
gcc -m32 -O1 -S -masm=intel compare_opt.c -o compare_O1.s

# 高度优化 — 可能使用 CMOV（条件传送）避免分支
gcc -m32 -O2 -S -masm=intel compare_opt.c -o compare_O2.s

# 对比关键差异
diff compare_O0.s compare_O2.s
```

<details>
<summary>优化差异要点</summary>

| 优化级别 | 循环变量 | max 存储位置 | 条件分支 |
|---------|---------|------------|---------|
| -O0 | 栈 `[ebp-4]` | 栈 `[ebp-8]` | `cmp` + `jle` |
| -O1 | 寄存器 `ecx` | 寄存器 `eax` | `cmp` + `jle` |
| -O2 | 寄存器 `ecx` | 寄存器 `eax` | `cmovg`（无分支） |

`-O2` 下编译器可能使用 `cmovg`（条件传送，大于时移动）替代分支：
```nasm
cmp     edx, eax       ; arr[i] > max?
cmovg   eax, edx       ; 若大于，max = arr[i]（无分支预测开销）
```

</details>

### 练习 4：KiriKiri2 二进制片段分析

分析以下从 KiriKiri2 `tTJSVariant` 类型转换中提取的反汇编片段：

```nasm
; 假设 ESI = tTJSVariant* this
mov     eax, [esi+0Ch]     ; eax = this->vt (variant type)
cmp     eax, 2             ; tvtInteger = 2
jz      short loc_IsInt
cmp     eax, 3             ; tvtReal = 3
jz      short loc_IsReal
cmp     eax, 4             ; tvtString = 4
jz      short loc_IsString
; ... 其他类型处理 ...
xor     eax, eax           ; 默认返回 0
jmp     short loc_Return

loc_IsInt:
    mov     eax, [esi+14h]  ; 直接取整数值
    jmp     short loc_Return

loc_IsReal:
    fld     qword ptr [esi+14h]  ; 加载 double
    call    __ftol2              ; double → int
    jmp     short loc_Return

loc_IsString:
    mov     ecx, [esi+14h]      ; 字符串指针
    push    ecx
    call    _wtoi                ; wchar_t* → int
    ; 返回值在 eax 中

loc_Return:
    ...
```

<details>
<summary>参考答案</summary>

```cpp
// tTJSVariant::AsInteger() 的简化还原
tjs_int tTJSVariant::AsInteger() const {
    switch (this->vt) {     // [esi+0Ch] = variant type
    case tvtInteger:        // type == 2
        return this->IntValue;    // [esi+14h] 直接返回
    case tvtReal:           // type == 3
        return (tjs_int)this->RealValue;  // [esi+14h] double → int
    case tvtString:         // type == 4
        return _wtoi(this->StringValue);  // wchar_t* → int
    default:
        return 0;
    }
}
```

**关键识别要素**：
- `[esi+0Ch]`：对象偏移 12 字节处是类型字段
- `[esi+14h]`：偏移 20 字节处是值联合体（union）
- `fld + __ftol2`：浮点转整数的标准模式
- `_wtoi`：宽字符串转整数

</details>

## 9. 常见错误与解决方案

### 错误 1：混淆 MOV 和 LEA

```nasm
; 新手常犯：把 LEA 当作内存读取
lea     eax, [ebx+10h]    ; eax = ebx + 0x10 （计算地址）
; 不是 eax = *(ebx + 0x10)!

; 解决方案：记住 LEA = "计算地址，不访问内存"
; 看到 LEA + 无方括号结果 → 纯算术
; 看到 MOV + 方括号 → 内存访问
```

### 错误 2：忽略有符号/无符号跳转差异

```nasm
cmp     eax, ebx
ja      label            ; unsigned: eax > ebx
jg      label            ; signed:   eax > ebx
; 当 eax=0xFFFFFFFF, ebx=1 时:
;   ja → 跳转 (4294967295 > 1 无符号)
;   jg → 不跳转 (-1 < 1 有符号)

; 解决方案：检查上下文中变量是有符号还是无符号
; 数组索引/长度通常用 JB/JAE (无符号)
; 算术比较通常用 JL/JGE (有符号)
```

### 错误 3：遗忘 x64 下 32 位操作的零扩展

```nasm
; x86-64 特性：写入 32 位寄存器会自动清零高 32 位
mov     eax, 1           ; RAX = 0x0000000000000001 (高 32 位清零)
mov     ax, 1            ; RAX 高 48 位不变！仅改变低 16 位

; 这导致 IDA 中看到的代码可能令人困惑:
xor     eax, eax         ; 在 x64 下等效 xor rax, rax (清零 64 位)
; 编译器故意使用 32 位形式因为编码更短
```

## 本节小结

| 概念 | 要点 |
|------|------|
| 通用寄存器 | x86: 8 个 (EAX–ESP)；x64: 16 个 (+R8–R15) |
| 返回值 | EAX / RAX |
| this 指针 | x86 thiscall: ECX；x64 MS: RCX |
| MOV vs LEA | MOV 读内存，LEA 计算地址 |
| CMP vs TEST | CMP 隐式减法，TEST 隐式 AND |
| 条件跳转 | B/A = 无符号，L/G = 有符号 |
| 虚函数调用 | `mov reg, [ecx]; call [reg+offset]` |
| 编译器优化 | `xor eax,eax` 清零，LEA 快速乘法，SHL/SHR 替代乘除 |

## 练习题与答案

### 题目 1：寄存器用途

在 x86 的 `thiscall` 调用约定中，`this` 指针通过哪个寄存器传递？在 x64（MSVC）中呢？

<details>
<summary>答案</summary>

- x86 thiscall: `ECX`
- x64 MSVC: `RCX`（第一个参数寄存器，this 被当作隐含的第一个参数）

</details>

### 题目 2：指令语义

写出以下三条指令序列执行后 EAX 的值（假设初始 EAX=10, EBX=3）：

```nasm
lea     eax, [eax+eax*4]   ; ?
add     eax, ebx            ; ?
shl     eax, 1              ; ?
```

<details>
<summary>答案</summary>

```
lea eax, [eax+eax*4]  → eax = 10 + 10*4 = 50
add eax, ebx           → eax = 50 + 3 = 53
shl eax, 1             → eax = 53 * 2 = 106
```

整个序列等效 C: `(a * 5 + b) * 2`

</details>

### 题目 3：反汇编→C 还原

将以下反汇编还原为 C 代码：

```nasm
push    ebp
mov     ebp, esp
mov     eax, [ebp+8]
mov     ecx, [ebp+0Ch]
test    ecx, ecx
jz      short loc_ret_zero
xor     edx, edx
loc_loop:
    test    ecx, 1
    jz      short loc_skip
    add     edx, eax
loc_skip:
    shl     eax, 1
    shr     ecx, 1
    jnz     short loc_loop
    mov     eax, edx
    pop     ebp
    ret
loc_ret_zero:
    xor     eax, eax
    pop     ebp
    ret
```

<details>
<summary>答案</summary>

```c
// 俄罗斯农民乘法（二进制乘法）
int multiply(int a, int b) {
    if (b == 0) return 0;
    int result = 0;
    while (b != 0) {
        if (b & 1)           // test ecx, 1; jz
            result += a;     // add edx, eax
        a <<= 1;            // shl eax, 1
        b >>= 1;            // shr ecx, 1 (无符号右移)
    }
    return result;
}
```

这是不使用 MUL 指令的纯移位加法乘法实现。编译器在某些嵌入式场景或无硬件乘法器的平台上会生成类似代码。

</details>
