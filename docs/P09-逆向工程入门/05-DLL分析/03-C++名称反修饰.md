# C++ 名称反修饰

## 本節目標

- 理解 C++ 名称修饰（Name Mangling）的必要性和原理
- 掌握 MSVC 修饰名的完整编码规则
- 掌握 GCC/Clang（Itanium ABI）修饰名的编码规则
- 熟练使用 undname、c++filt、Ghidra 等工具进行反修饰
- 能够手动解读 KiriKiri2 导出的 C++ 修饰名

## 5.3.1 名称修饰的必要性

### 为什么需要名称修饰

C++ 支持函数重载、命名空间、类成员函数等特性，这些在链接层面会造成名称冲突：

```cpp
// 以下四个函数在 C++ 中是合法的（重载）
// 但链接器需要用唯一的符号名区分它们

int    calc(int a);              // 函数 1
double calc(double a);           // 函数 2
int    calc(int a, int b);       // 函数 3

namespace math {
    int calc(int a);             // 函数 4（不同命名空间）
}

class Calculator {
    int calc(int a);             // 函数 5（成员函数）
    static int calc(int a, int b); // 函数 6（静态成员函数）
};
```

编译器通过**名称修饰**（Name Mangling）将函数签名编码进符号名：

```
MSVC 修饰结果:
  函数 1: ?calc@@YAHH@Z
  函数 2: ?calc@@YANN@Z
  函数 3: ?calc@@YAHHH@Z
  函数 4: ?calc@math@@YAHH@Z
  函数 5: ?calc@Calculator@@QAEHH@Z
  函数 6: ?calc@Calculator@@SAHHH@Z

GCC/Clang 修饰结果:
  函数 1: _Z4calci
  函数 2: _Z4calcd
  函数 3: _Z4calcii
  函数 4: _ZN4math4calcEi
  函数 5: _ZN10Calculator4calcEi
  函数 6: _ZN10Calculator4calcEii
```

### extern "C" 抑制修饰

```cpp
// extern "C" 告诉编译器使用 C 链接规则（不修饰名称）
extern "C" {
    HRESULT __stdcall V2Link(iTVPFunctionExporter* exporter);
    HRESULT __stdcall V2Unlink();
}

// 导出名称:
//   V2Link    （无修饰，或 stdcall: _V2Link@4）
//   V2Unlink  （无修饰，或 stdcall: _V2Unlink@0）

// 这就是为什么 KiriKiri2 插件的导出函数使用 extern "C"
// 否则不同编译器生成的修饰名不同，无法跨编译器加载
```

## 5.3.2 MSVC 修饰规则详解

### 基本格式

MSVC 修饰名以 `?` 开头，格式为：

```
?<函数名>@<限定名>@<修饰符><调用约定><返回类型><参数列表>@Z

其中:
  ?         → 修饰名标识符
  <函数名>  → 原始函数名
  @         → 分隔符
  <限定名>  → 命名空间/类名（从内到外，@@ 结尾）
  <修饰符>  → 访问权限 + 成员类型
  <调用约定> → 调用约定编码
  <返回类型> → 返回值类型编码
  <参数列表> → 参数类型编码列表
  @Z        → 参数列表结尾标记
```

### 类型编码表

```
基本类型编码:
  C → signed char       D → char            E → unsigned char
  F → short             G → unsigned short
  H → int               I → unsigned int
  J → long              K → unsigned long
  M → float             N → double
  _J → __int64           _K → unsigned __int64
  _N → bool
  X → void
  _W → wchar_t

指针/引用修饰:
  PA → 指针 (*)          PB → const 指针
  AA → 引用 (&)          AB → const 引用

类类型:
  V<类名>@@ → class 类名
  U<类名>@@ → struct 类名
```

### 调用约定编码

```
  A → __cdecl
  B → __cdecl (with __declspec(dllexport))
  C → __pascal
  E → __thiscall
  G → __stdcall
  I → __fastcall
```

### 成员函数修饰符

```
  Q → public:           (非 const)
  R → public: const
  S → public: static
  A → private:          (非 const)
  B → private: const
  I → protected:        (非 const)
  J → protected: const
```

### 手动解码示例

```
?GetInstance@CFactory@@SAPAV1@XZ

分解:
  ?GetInstance              → 函数名: GetInstance
  @CFactory                → 限定: CFactory 类
  @@                       → 限定结束
  S                        → public: static
  A                        → __cdecl
  PAV1@                    → 返回: 指针(PA) + class(V) + 反向引用1(CFactory) 
  X                        → 参数: void（无参数）
  Z                        → 结束

还原: public: static CFactory* __cdecl CFactory::GetInstance(void)
```

```
?PropGet@tTJSCustomObject@@UAGJKPAV?$tTJSVariant@@ABVtTJSString@@@Z

分解（简化）:
  ?PropGet                 → 函数名: PropGet
  @tTJSCustomObject        → 限定: tTJSCustomObject 类
  @@                       → 限定结束
  U                        → public: virtual
  A                        → __thiscall
  G → J                    → 返回: long (__stdcall 修饰中的 HRESULT)
  K                        → 参数1: unsigned long (flag)
  PAV?$tTJSVariant@@       → 参数2: tTJSVariant* (result)
  ABVtTJSString@@          → 参数3: const tTJSString& (name)
  @Z                       → 结束

还原（概念）: 
  public: virtual long __thiscall tTJSCustomObject::PropGet(
      unsigned long flag, tTJSVariant* result, const tTJSString& name)
```

### 反向引用机制

MSVC 使用数字 0-9 来反向引用已出现过的类型/名称，减少重复：

```
?func@@YAHPAV1@PAV1@@Z

分解:
  ?func@@     → 全局函数 func
  Y           → 非成员函数（全局）
  A           → __cdecl
  H           → 返回 int
  PAV1@       → 参数1: 指针 + class + 反向引用 1
  PAV1@       → 参数2: 同类型
  @Z          → 结束

; 反向引用 0-9 指向之前出现过的类名
; 这里 "1" 是一个编号，不是名称
```

## 5.3.3 GCC/Clang（Itanium ABI）修饰规则

### 基本格式

```
_Z<长度><函数名><参数编码>

嵌套名称:
_ZN<限定名1><限定名2>...<函数名>E<参数编码>

其中:
  _Z  → 修饰名标识符
  N   → 开始嵌套名称
  E   → 结束嵌套名称
  数字 → 后续名称的字符长度
```

### 类型编码表

```
基本类型:
  v → void          b → bool
  c → char          a → signed char     h → unsigned char
  s → short         t → unsigned short
  i → int           j → unsigned int
  l → long          m → unsigned long
  x → long long     y → unsigned long long
  f → float         d → double
  w → wchar_t       e → long double

指针/引用:
  P → 指针           R → 引用
  K → const          O → 右值引用 (&&)
```

### 手动解码示例

```
_ZN10Calculator4calcEi

分解:
  _Z                 → 修饰名标识
  N                  → 嵌套名称开始
  10Calculator       → "Calculator"（10 个字符）
  4calc              → "calc"（4 个字符）
  E                  → 嵌套名称结束
  i                  → 参数: int

还原: Calculator::calc(int)
```

```
_ZNK8tTJSString6GetLenEv

分解:
  _Z                 → 修饰名标识
  N                  → 嵌套名称开始
  K                  → const 成员函数
  8tTJSString        → "tTJSString"（10... wait, 8 chars? → 正确: tTJSStri = 8? 不对）
  
; 实际上 "tTJSString" 是 10 个字符
; 正确写法应该是: _ZNK10tTJSString6GetLenEv

分解:
  _ZNK               → const 成员函数
  10tTJSString       → 类 "tTJSString"
  6GetLen            → 函数 "GetLen"
  E                  → 嵌套结束
  v                  → 参数: void

还原: tTJSString::GetLen() const
```

## 5.3.4 反修饰工具

### undname（MSVC 自带）

```cmd
:: 在 Developer Command Prompt 中使用
undname "?GetInstance@CFactory@@SAPAV1@XZ"
:: 输出: public: static class CFactory * __cdecl CFactory::GetInstance(void)

:: 批量反修饰
undname "?PropGet@tTJSCustomObject@@UAEJAAH@Z"

:: 从文件反修饰
type mangled_names.txt | undname

:: 实际操作：提取 DLL 导出名后反修饰
dumpbin /exports example.dll | findstr "?" > mangled.txt
for /f "tokens=4" %%a in (mangled.txt) do @undname "%%a"
```

### c++filt（GCC/MinGW 自带）

```bash
# 反修饰 Itanium ABI 名称
echo "_ZN10Calculator4calcEi" | c++filt
# 输出: Calculator::calc(int)

# 批量反修饰
c++filt < mangled_names.txt

# 指定编译器格式
c++filt --format=gnu "_ZN10Calculator4calcEi"
c++filt --format=auto "?calc@@YAHH@Z"  # 自动检测格式
```

### Python 脚本反修饰

```python
import subprocess
import re
import os

def undname_msvc(mangled_name):
    """使用 undname.exe 反修饰 MSVC 名称"""
    try:
        result = subprocess.run(
            ['undname', mangled_name],
            capture_output=True, text=True, timeout=5
        )
        # undname 输出格式: "mangled_name == demangled_name"
        output = result.stdout.strip()
        if '==' in output:
            return output.split('==')[1].strip()
        return output
    except (FileNotFoundError, subprocess.TimeoutExpired):
        return f"[undname not available: {mangled_name}]"

def cppfilt(mangled_name):
    """使用 c++filt 反修饰 GCC/Clang 名称"""
    try:
        result = subprocess.run(
            ['c++filt', mangled_name],
            capture_output=True, text=True, timeout=5
        )
        return result.stdout.strip()
    except (FileNotFoundError, subprocess.TimeoutExpired):
        return f"[c++filt not available: {mangled_name}]"

def auto_demangle(mangled_name):
    """自动检测修饰格式并反修饰"""
    if mangled_name.startswith('?'):
        return undname_msvc(mangled_name)
    elif mangled_name.startswith('_Z'):
        return cppfilt(mangled_name)
    elif mangled_name.startswith('_') and '@' in mangled_name:
        # stdcall: _FuncName@N
        match = re.match(r'_(.+)@(\d+)', mangled_name)
        if match:
            return f"{match.group(1)} [stdcall, {match.group(2)} bytes params]"
    return mangled_name  # 可能是 extern "C" 或未修饰名称

# 批量处理 DLL 导出
def demangle_dll_exports(dll_path):
    """反修饰 DLL 的所有导出函数名"""
    result = subprocess.run(
        ['dumpbin', '/exports', dll_path],
        capture_output=True, text=True
    )
    
    print(f"\n{'='*70}")
    print(f"Demangled exports for: {os.path.basename(dll_path)}")
    print(f"{'='*70}")
    
    # 解析 dumpbin 输出
    for line in result.stdout.split('\n'):
        # 匹配: ordinal hint RVA name
        match = re.match(r'\s+\d+\s+[0-9A-F]+\s+[0-9A-F]+\s+(\S+)', line)
        if match:
            mangled = match.group(1)
            demangled = auto_demangle(mangled)
            if mangled != demangled:
                print(f"  {mangled}")
                print(f"    → {demangled}")
            else:
                print(f"  {mangled} (C linkage)")

# 使用示例
demangle_dll_exports(r"C:\game\plugin\extrans.dll")
```

## 5.3.5 MSVC 与 GCC 修饰名对照

同一函数在不同编译器下的修饰名：

```
// 源码: class tTJSString { int GetLen() const; };

MSVC:  ?GetLen@tTJSString@@QBEHXZ
GCC:   _ZNK10tTJSString6GetLenEv

// 源码: namespace TJS { void* AllocMem(size_t size); }

MSVC:  ?AllocMem@TJS@@YAPAXI@Z
GCC:   _ZN3TJS8AllocMemEm

// 源码: template<class T> class vector { void push_back(const T&); };
//       vector<int> 的 push_back

MSVC:  ?push_back@?$vector@HV?$allocator@H@std@@@std@@QAEXABH@Z
GCC:   _ZNSt6vectorIiSaIiEE9push_backERKi
```

### 快速识别表

```
特征                    MSVC              GCC/Clang
──────────             ──────             ──────────
修饰前缀               ?                 _Z
嵌套标记               @类名@            N...E
const 成员             Q→R (public)      K
模板                   ?$模板名@参数@@    I...E
std:: 缩写             @std@@            St / Ss / Sa
```

## 5.3.6 KiriKiri2 中的修饰名实例

### TJS2 引擎核心类

```
修饰名: ?FuncCall@tTJSCustomObject@@UAGJKPAV?$tTJSVariant@@VtTJSString@@HPAVtTJSVariant@@2@Z
反修饰: public: virtual long __thiscall tTJSCustomObject::FuncCall(
            unsigned long, tTJSVariant*, tTJSString, int, tTJSVariant*, tTJSVariant*)

修饰名: ?PropGet@tTJSCustomObject@@UAEGJPAV?$tTJSVariant@@ABVtTJSString@@PAV2@@Z
反修饰: public: virtual long __thiscall tTJSCustomObject::PropGet(
            unsigned long, tTJSVariant*, const tTJSString&, tTJSVariant*)

修饰名: ?CreateNew@tTJSCustomObject@@UAEGJKPAV?$tTJSVariant@@ABVtTJSString@@HPAV2@PAV3@@Z
反修饰: public: virtual long __thiscall tTJSCustomObject::CreateNew(
            unsigned long, unsigned long, tTJSVariant*, const tTJSString&,
            int, tTJSVariant*, tTJSVariant*)
```

### 从修饰名推断类层次结构

```
修饰名列表:
  ?AddRef@tTJSCustomObject@@UAGKXZ
  ?Release@tTJSCustomObject@@UAGKXZ
  ?PropGet@tTJSCustomObject@@...
  ?PropSet@tTJSCustomObject@@...
  ?FuncCall@tTJSCustomObject@@...
  ?FuncCallByNum@tTJSCustomObject@@...

推断:
1. tTJSCustomObject 有 AddRef/Release → COM 风格引用计数
2. 所有方法都是 virtual (U) → 继承自接口
3. PropGet/PropSet → 属性访问器
4. FuncCall/FuncCallByNum → 方法调用（按名/按序号）
5. 这些函数构成了 iTJSDispatch2 接口的实现

// 推断出的接口:
class iTJSDispatch2 {
public:
    virtual unsigned long AddRef() = 0;
    virtual unsigned long Release() = 0;
    virtual long PropGet(...) = 0;
    virtual long PropSet(...) = 0;
    virtual long FuncCall(...) = 0;
    virtual long FuncCallByNum(...) = 0;
    // ...
};

class tTJSCustomObject : public iTJSDispatch2 {
    // 实现所有虚函数
};
```

## 5.3.7 常见错误与解决方案

### 错误 1：undname 无法识别修饰名

```
症状: undname 输出与输入相同，没有反修饰
原因: 
1. 不是 MSVC 修饰名（可能是 GCC 格式）
2. 修饰名不完整（被截断）
3. 非标准修饰（自定义编译器或混淆）

解决方案:
# 1. 检查格式：? 开头 = MSVC，_Z 开头 = GCC
echo "_ZN10Calculator4calcEi" | c++filt
# 输出: Calculator::calc(int)

# 2. 确保名称完整（必须以 @Z 结尾表示 MSVC）
# 如果从 dumpbin 复制时被截断，重新提取

# 3. 使用在线工具
# https://demangler.com/ 支持多种格式
```

### 错误 2：模板名称过长导致截断

```
症状: 反修饰后名称极长，包含大量模板参数
原因: STL 容器和模板在修饰后会产生非常长的名称

示例:
?push_back@?$vector@V?$basic_string@_WU?$char_traits@_W@std@@V?$allocator@_W@2@@std@@V?$allocator@V?$basic_string@_WU?$char_traits@_W@std@@V?$allocator@_W@2@@std@@@2@@std@@QAEXABV?$basic_string@_WU?$char_traits@_W@std@@V?$allocator@_W@2@@2@@Z

反修饰:
public: void __thiscall std::vector<
    std::basic_string<wchar_t, std::char_traits<wchar_t>, std::allocator<wchar_t>>,
    std::allocator<std::basic_string<wchar_t, std::char_traits<wchar_t>, std::allocator<wchar_t>>>
>::push_back(
    std::basic_string<wchar_t, std::char_traits<wchar_t>, std::allocator<wchar_t>> const &
)

简化理解: std::vector<std::wstring>::push_back(const std::wstring&)

解决方案:
# 使用 undname 的简化模式（如果支持）
# 或在 IDA 中设置:
# Options → Demangled names → Short form
```

### 错误 3：混淆修饰名和函数名

```
症状: 将修饰名的一部分当成函数名
原因: 不了解修饰名的结构

常见误解:
  修饰名: ?calc@Calculator@@QAEHH@Z
  
  误解: 函数名是 "calc@Calculator"
  正确: 函数名是 "calc"，"Calculator" 是类名
  
  误解: "@Z" 是函数名的一部分
  正确: "@Z" 是参数列表结束标记

规则:
  ? 到第一个 @ 之间 = 函数名
  @ 之后到 @@ = 限定名（类/命名空间）
  @@ 之后 = 修饰符 + 类型信息
```

## 本節小結

1. **名称修饰的必要性**：C++ 的重载、命名空间、成员函数需要唯一的链接符号
2. **MSVC 格式**：以 `?` 开头，使用 `@` 分隔，编码包含类名、调用约定、返回类型、参数类型
3. **GCC/Clang 格式**：以 `_Z` 开头，使用长度前缀编码名称，`N...E` 表示嵌套
4. **反修饰工具**：MSVC 用 undname，GCC 用 c++filt，IDA/Ghidra 内置支持
5. **实际应用**：通过反修饰可以直接从导出表获取完整的函数原型（类名、参数类型、返回类型、调用约定）

### 题目 1：手动解码 MSVC 修饰名

**问题**：手动解码 `?AddRef@tTJSCustomObject@@UAGKXZ`，写出完整的 C++ 函数签名。

<details>
<summary>查看答案</summary>

```
?AddRef@tTJSCustomObject@@UAGKXZ

分解步骤：
1. ? → MSVC 修饰名标识
2. AddRef → 函数名: AddRef
3. @ → 分隔符
4. tTJSCustomObject → 限定: tTJSCustomObject 类
5. @@ → 限定结束
6. U → public: virtual
7. A → __thiscall（成员函数默认调用约定）
8. G → 返回类型修饰符（与 __stdcall 的返回类型编码有关）
9. K → unsigned long
10. X → 参数列表: void（无参数）
11. Z → 修饰名结束

完整签名:
public: virtual unsigned long __thiscall tTJSCustomObject::AddRef(void)

等价 C++ 声明:
class tTJSCustomObject {
public:
    virtual unsigned long AddRef();
};
```

</details>

### 题目 2：MSVC 与 GCC 修饰名互译

**问题**：以下 GCC 修饰名对应什么 C++ 函数？写出等价的 MSVC 修饰名的大致结构。

```
_ZN10tTJSString6AppendEPKwi
```

<details>
<summary>查看答案</summary>

GCC 修饰名解码：

```
_Z                → Itanium ABI 修饰标识
N                 → 嵌套名称开始
10tTJSString      → "tTJSString"（10 个字符）
6Append           → "Append"（6 个字符）
E                 → 嵌套名称结束
PKw               → const wchar_t* (P=指针, K=const, w=wchar_t)
i                 → int

C++ 函数: tTJSString::Append(const wchar_t*, int)
```

等价 MSVC 修饰名（大致结构）：

```
?Append@tTJSString@@QAEXPB_WH@Z

分解:
  ?Append           → 函数名
  @tTJSString       → 类名
  @@                → 限定结束
  Q                 → public: (非 virtual)
  A                 → __thiscall
  E → X             → 返回 void
  PB_W              → const wchar_t* (P=指针, B=const, _W=wchar_t)
  H                 → int
  @Z                → 结束
```

注意：MSVC 修饰名的具体编码取决于编译器版本，上述为近似结构。

</details>

### 题目 3：从修饰名推断类接口

**问题**：给定以下一组修饰名，推断该类的接口（列出所有公共方法及其签名）。

```
?AddRef@MyObject@@UAGKXZ
?Release@MyObject@@UAGKXZ
?GetName@MyObject@@QBEPB_WXZ
?SetValue@MyObject@@QAEXH@Z
?Calculate@MyObject@@QAENN@Z
```

<details>
<summary>查看答案</summary>

逐一反修饰：

```
1. ?AddRef@MyObject@@UAGKXZ
   → public: virtual unsigned long MyObject::AddRef(void)

2. ?Release@MyObject@@UAGKXZ
   → public: virtual unsigned long MyObject::Release(void)

3. ?GetName@MyObject@@QBEPB_WXZ
   → public: const wchar_t* MyObject::GetName(void) const
   (QB = public const, E = thiscall, PB_W = const wchar_t*)

4. ?SetValue@MyObject@@QAEXH@Z
   → public: void MyObject::SetValue(int)
   (QA = public non-const, E = thiscall, X = void return, H = int)

5. ?Calculate@MyObject@@QAENN@Z
   → public: double MyObject::Calculate(double)
   (QA = public, E = thiscall, N = double return, N = double param)
```

推断的类接口：

```cpp
class MyObject {
public:
    // COM 风格引用计数（virtual）
    virtual unsigned long AddRef();
    virtual unsigned long Release();
    
    // 属性访问
    const wchar_t* GetName() const;
    void SetValue(int value);
    
    // 计算方法
    double Calculate(double input);
};
```

类特征分析：
- 有 AddRef/Release → COM 风格的引用计数对象
- 都是 public → 设计为外部使用的接口
- GetName 是 const → 只读访问器
- Calculate 接收和返回 double → 数值计算功能

</details>
