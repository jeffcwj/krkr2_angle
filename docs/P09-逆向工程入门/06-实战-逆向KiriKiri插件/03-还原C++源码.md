## 6.3.2 步骤一：类型还原

### 基本类型映射

反编译器输出的类型通常是泛化的（int、void* 等），需要映射回 KiriKiri2 的类型系统：

```cpp
// 反编译器类型 → KiriKiri2 实际类型
// ──────────────────────────────────────
// int              → tjs_int        (有符号 32 位)
// unsigned int     → tjs_uint       (无符号 32 位)
// unsigned int     → tjs_uint32     (明确 32 位)
// __int64          → tjs_int64      (64 位)
// wchar_t*         → tjs_char*      (TJS 宽字符)
// wchar_t*         → const ttstr&   (TJS 字符串引用，需判断)
// int              → tjs_error      (函数返回值)
// void*            → iTJSDispatch2* (TJS 对象指针)
// void*            → tTJSVariant*   (TJS 变量)
```

### 判断真实类型的方法

```cpp
// 方法 1: 根据使用上下文判断
// 伪代码:
//   if (a1 < 0) return a1;   ← a1 与负数比较 → tjs_error (HRESULT)
//   result = a2 + a3;         ← 算术运算 → tjs_int

// 方法 2: 根据函数签名推断
// 伪代码中如果看到:
//   v1 = (*(int (__stdcall **)(int, wchar_t*, int, int, int))(*(int *)a1 + 0x18))
//                                                                          ^^^
//   +0x18 = PropSet → 第一个参数是 iTJSDispatch2*

// 方法 3: 根据常量值推断
//   if (v1 == 0x80040005)  ← TJS_E_MEMBERNOTFOUND
//   if (v1 & 0x10000)      ← TJS_MEMBERENSURE 标志位
//   push 0x00000006        ← tTJSVariant.type = tvtObject (0x06)
```

### 结构体还原实例

继续使用前一节分析的 `NativeImageFilter`：

```cpp
// ── 反编译器看到的访问模式 ──
// mov eax, [ecx+04h]     ; 指针类型（后续被解引用为 DWORD 数组）
// mov edx, [ecx+08h]     ; 整数（与循环终止条件比较）
// mov esi, [ecx+0Ch]     ; 整数（与另一个整数相乘）
// fild dword ptr [ecx+14h] ; 整数（被加载为浮点数参与运算）

// ── 还原后的结构体 ──
struct NativeImageFilter {
    // 偏移  类型           名称          推断依据
    /* +00 */ void* vtbl;                // 第一个 DWORD 被间接调用（虚函数）
    /* +04 */ tjs_uint32* pixel_buffer;  // 被解引用后按 4 字节步进访问
    /* +08 */ tjs_int width;             // 与 height 相乘得到像素总数
    /* +0C */ tjs_int height;            // 作为外层循环终止条件
    /* +10 */ tjs_int stride;            // 计算行偏移时使用
    /* +14 */ tjs_int radius;            // 高斯模糊核大小参数
    /* +18 */ tjs_int bpp;               // 恒为 32，用于分配缓冲区
    
    NativeImageFilter();
    ~NativeImageFilter();
    
    // 方法签名从 ncbind 注册回调的包装层推断
    void adjustBrightness(double brightness);
    void adjustContrast(double contrast);
    void gaussianBlur();
    
    tjs_int getRadius() const { return radius; }
    void setRadius(tjs_int r) { radius = r; }
};
```

### 方法还原——adjustBrightness 完整版

```cpp
// 从反编译伪代码清洗后的 C++ 实现
void NativeImageFilter::adjustBrightness(double brightness) {
    // 参数校验（从反编译中的 if-throw 模式还原）
    if (brightness < -1.0 || brightness > 1.0) {
        TVPThrowExceptionMessage(
            TJS_W("brightness must be between -1.0 and 1.0"));
        return;
    }
    
    if (!pixel_buffer || width <= 0 || height <= 0) return;
    
    tjs_int total = width * height;
    tjs_int adjustment = static_cast<tjs_int>(brightness * 255.0);
    
    for (tjs_int i = 0; i < total; i++) {
        tjs_uint32 px = pixel_buffer[i];
        
        // ARGB 分量提取（从位运算模式还原）
        tjs_int a = (px >> 24) & 0xFF;
        tjs_int r = (px >> 16) & 0xFF;
        tjs_int g = (px >> 8)  & 0xFF;
        tjs_int b =  px        & 0xFF;
        
        // clamp 到 [0, 255]（反编译中可能是条件移动指令 cmov）
        r = std::min(std::max(r + adjustment, 0), 255);
        g = std::min(std::max(g + adjustment, 0), 255);
        b = std::min(std::max(b + adjustment, 0), 255);
        
        pixel_buffer[i] = (a << 24) | (r << 16) | (g << 8) | b;
    }
}
```

## 6.3.4 步骤三：逻辑清洗

### 常见优化伪影及还原方法

反编译器的输出经常包含编译器优化的残余，需要识别并清洗：

```cpp
// ── 伪影 1: 乘法 → 移位 ──
// 反编译: v3 = v2 << 2;       还原: index = i * 4;

// ── 伪影 2: 除法 → 魔数乘法 ──
// 反编译: v5 = (unsigned __int64)(0xAAAAAAAB * v4) >> 33;
// 还原:   result = v4 / 3;    (0xAAAAAAAB ≈ ceil(2^33/3))

// ── 伪影 3: clamp 展开 ──
// 反编译: v6=v4; if(v4>255) v6=255; if(v6<0) v6=0;
// 还原:   v6 = std::clamp(v4, 0, 255);

// ── 伪影 4: 循环展开 ──
// 反编译中处理 pixels[i], pixels[i+1], pixels[i+2], pixels[i+3]
// 还原:   单次迭代处理 pixels[i]（编译器展开了 4 次）

// ── 伪影 5: switch → 跳转表 ──
// 反编译: *(code*)(&off_10003000 + v1*4)();
// 还原:   switch(v1) { case 0:...; case 1:...; }
```

### 字符串还原

```cpp
// 反编译中的宽字符串:
// push offset aAdjustbrightne  ; "adjustBrightness"（ANSI 引用名）
// 实际在数据段中:
// .rdata:10005000  dw 'a','d','j','u','s','t','B','r','i','g','h','t',...,0

// 还原为 C++ 代码:
// L"adjustBrightness"   — 如果是函数名/成员名
// TJS_W("...")          — 如果传递给 TJS API
```

## 6.3.5 步骤四：框架集成

### 完整插件源码还原

将所有还原的部分组装为完整的 KiriKiri2 插件：

```cpp
// imageEx.cpp — 还原后的完整插件源码

#include "ncbind.hpp"
#include <algorithm>
#include <cmath>
#include <cstdlib>
#include <cstring>

// ────────────────────────────────────────────
// NativeImageFilter 类定义
// ────────────────────────────────────────────
class NativeImageFilter {
private:
    tjs_uint32* pixel_buffer_;
    tjs_int     width_;
    tjs_int     height_;
    tjs_int     stride_;
    tjs_int     radius_;
    tjs_int     bpp_;

public:
    NativeImageFilter()
        : pixel_buffer_(nullptr), width_(0), height_(0)
        , stride_(0), radius_(3), bpp_(32) {}
    
    ~NativeImageFilter() {
        if (pixel_buffer_) {
            free(pixel_buffer_);
            pixel_buffer_ = nullptr;
        }
    }

    // ── 属性 ──
    tjs_int getRadius() const { return radius_; }
    void setRadius(tjs_int r) {
        if (r < 0) r = 0;
        if (r > 100) r = 100;
        radius_ = r;
    }
    
    // ── 方法: 亮度调整 ──
    void adjustBrightness(double brightness) {
        if (brightness < -1.0 || brightness > 1.0) {
            TVPThrowExceptionMessage(
                TJS_W("brightness must be between -1.0 and 1.0"));
            return;
        }
        if (!pixel_buffer_ || width_ <= 0 || height_ <= 0) return;
        
        tjs_int total = width_ * height_;
        tjs_int adj = static_cast<tjs_int>(brightness * 255.0);
        
        for (tjs_int i = 0; i < total; i++) {
            tjs_uint32 px = pixel_buffer_[i];
            tjs_int a = (px >> 24) & 0xFF;
            tjs_int r = std::clamp(((px >> 16) & 0xFF) + adj, 0, 255);
            tjs_int g = std::clamp(((px >> 8)  & 0xFF) + adj, 0, 255);
            tjs_int b = std::clamp(( px        & 0xFF) + adj, 0, 255);
            pixel_buffer_[i] = (a << 24) | (r << 16) | (g << 8) | b;
        }
    }
    
    // ── 方法: 对比度调整（逻辑类似亮度，使用非线性 factor 映射）──
    void adjustContrast(double contrast) {
        if (contrast < -1.0 || contrast > 1.0) {
            TVPThrowExceptionMessage(TJS_W("contrast out of range"));
            return;
        }
        if (!pixel_buffer_ || width_ <= 0 || height_ <= 0) return;
        double factor = (1.0 + contrast) * (1.0 + contrast); // fmul st(0),st(0)
        tjs_int total = width_ * height_;
        for (tjs_int i = 0; i < total; i++) {
            tjs_uint32 px = pixel_buffer_[i];
            tjs_int a = (px >> 24) & 0xFF;
            tjs_int r = std::clamp((int)(factor*(((px>>16)&0xFF)-128)+128),0,255);
            tjs_int g = std::clamp((int)(factor*(((px>>8)&0xFF)-128)+128),0,255);
            tjs_int b = std::clamp((int)(factor*((px&0xFF)-128)+128),0,255);
            pixel_buffer_[i] = (a << 24) | (r << 16) | (g << 8) | b;
        }
    }
    
    // ── 方法: 高斯模糊（分离式二趟卷积）──
    // 完整实现参见前一节 6.2.4 的伪代码分析
    void gaussianBlur() {
        if (radius_ <= 0 || !pixel_buffer_) return;
        tjs_int w = width_, h = height_;
        tjs_uint32* temp = (tjs_uint32*)malloc(w * h * sizeof(tjs_uint32));
        if (!temp) return;
        
        // 生成高斯核: kernel[k] = exp(-x²/(2σ²)), σ = radius/3
        tjs_int ksize = radius_ * 2 + 1;
        double* kernel = new double[ksize];
        double sigma = radius_ / 3.0, sum = 0;
        for (tjs_int k = 0; k < ksize; k++) {
            double x = k - radius_;
            kernel[k] = std::exp(-(x*x) / (2*sigma*sigma));
            sum += kernel[k];
        }
        for (tjs_int k = 0; k < ksize; k++) kernel[k] /= sum;
        
        // 水平方向卷积: pixel_buffer_ → temp
        // 垂直方向卷积: temp → pixel_buffer_
        // （每个方向对 ARGB 四通道分别加权求和）
        apply_1d_convolution(pixel_buffer_, temp, w, h, kernel, radius_, true);
        apply_1d_convolution(temp, pixel_buffer_, w, h, kernel, radius_, false);
        
        delete[] kernel;
        free(temp);
    }
};

// ────────────────────────────────────────────
// ncbind 注册
// ────────────────────────────────────────────
NCB_REGISTER_CLASS(NativeImageFilter) {
    NCB_CONSTRUCTOR(());
    NCB_METHOD(adjustBrightness);
    NCB_METHOD(adjustContrast);
    NCB_METHOD(gaussianBlur);
    NCB_PROPERTY(radius, getRadius, setRadius);
}
```

## 6.3.6 步骤五：还原质量评估

### 检查清单

| 检查项 | 说明 |
|--------|------|
| 函数签名一致 | 参数个数、类型、返回值与反编译匹配 |
| 算法逻辑等价 | 数值计算结果相同 |
| 边界条件匹配 | 参数校验、空指针检查 |
| 默认值正确 | 构造函数初始值与反编译一致 |
| 内存管理一致 | malloc/free 或 new/delete 配对正确 |
| TJS 接口一致 | 类名、方法名、属性名大小写完全相同 |

### 常见还原陷阱

```cpp
// 陷阱 1: 缺少调用约定 — V2Link 必须是 __stdcall
// 陷阱 2: 结构体对齐不匹配 — 可能需要 #pragma pack(push, 4)
// 陷阱 3: 浮点精度差异 — x87 FPU 用 80 位，C++ double 64 位（通常可接受）
// 陷阱 4: 字符串编码 — 始终用 tjs_char 而非 wchar_t（跨平台兼容）
```

## 6.3.7 常见错误与解决方案

### 错误 1：编译时类型不匹配

```
症状: error: cannot convert 'int' to 'tjs_error' 等类型错误
原因: 反编译中的泛化类型未正确映射
解决: 查看 tp_stub.h 中的 typedef（typedef int tjs_error; typedef unsigned int
      tjs_uint32;），统一使用 KiriKiri2 类型别名
```

### 错误 2：ncbind 注册后 TJS 找不到方法

```
症状: TJS 脚本调用 obj.method() 抛出 "member not found"
原因: ncbind 注册名与反编译中的字符串不一致
解决: 对照反编译中 TJSNativeClassRegisterNCM 的第二个参数（宽字符串），
      确保类名/方法名大小写完全一致（"adjustBrightness" ≠ "AdjustBrightness"）
```

### 错误 3：运行时崩溃（访问违例）

```
症状: 插件加载后在 V2Link 中崩溃
原因: tp_stub.h 版本不匹配导致虚函数表错位
解决: 使用项目仓库中的 tp_stub.h（cpp/core/plugin/tp_stub.h），确保编译时
      定义了正确的宏（_WIN32 等），DLL 和 krkr.exe 调用约定一致（__stdcall）
```

## 本節小結

1. **类型还原是基础**——将反编译器的泛化类型正确映射回 KiriKiri2 类型系统
2. **结构体从访问模式推断**——多个方法中对 this 指针的偏移访问模式揭示结构体布局
3. **编译器优化伪影需要识别**——移位代替乘法、魔数乘法代替除法、循环展开等

5. **编译验证是最终关卡**——能编译通过不等于行为正确，需要结合下一节的集成测试

## 練習題与答案

### 题目 1：还原 setPixelData 方法

**问题**：将动手实践中的伪代码还原为 NativeImageFilter 的 C++ 方法。

<details>
<summary>查看答案</summary>

```cpp
// 参数推断:
//   a2 → width (存入 +0x08)
//   a3 → height (存入 +0x0C)
//   a5 → 源像素数据指针 (memcpy 的源)
//   a4 → 未使用（可能是 bpp，被忽略）

tjs_error NativeImageFilter::setPixelData(
    tjs_int width, tjs_int height, tjs_int bpp,
    const tjs_uint32* src_pixels)
{
    // 释放旧缓冲区
    if (pixel_buffer_) {
        free(pixel_buffer_);
    }
    
    // 设置尺寸
    width_  = width;
    height_ = height;
    stride_ = width * 4;  // 每像素 4 字节 (ARGB)
    
    // 分配新缓冲区
    pixel_buffer_ = (tjs_uint32*)malloc(width * height * sizeof(tjs_uint32));
    if (!pixel_buffer_) {
        return TJS_E_FAIL;  // -1 对应 E_FAIL
    }
    
    // 复制像素数据
    memcpy(pixel_buffer_, src_pixels, width * height * sizeof(tjs_uint32));
    
    return TJS_S_OK;  // 0 对应 S_OK
}
```

关键推断:
- `a2 * 4` 出现在 stride 赋值 → a2 = width，4 字节/像素
- `a2 * a3 * 4` = 总字节数 → ARGB 格式
- `a5` 作为 memcpy 源 → 外部像素数据指针
- 返回 -1 / 0 → tjs_error 类型 (HRESULT 风格)

</details>

### 题目 2：识别优化伪影

**问题**：以下反编译代码包含编译器优化。还原其原始 C++ 逻辑。

```c
v3 = (unsigned __int64)(0x55555556i64 * v2) >> 32;
v4 = v3 + (v3 >> 31);
```

<details>
<summary>查看答案</summary>

```
分析:
0x55555556 = 1431655766（十进制）
编译器用乘法+移位替代整数除法的经典优化。

计算: 0x55555556 / 2^32 ≈ 1/3 → v3 ≈ v2 / 3

第二行: v3 + (v3 >> 31)
  v3 >> 31 提取符号位（0 或 -1），修正负数的向零取整

还原: v4 = v2 / 3;

验证: v2=9 → 0x55555556*9 = 0x0000_0003_0000_0002(64位)
      >> 32 = 3, +(3>>31=0) = 3 = 9/3 ✓
```

</details>

### 题目 3：ncbind 属性还原

**问题**：反编译中看到两个 TJSNativeClassRegisterNCM 调用共用同一个成员名 `L"opacity"`，标志位分别是 `0x20` 和 `0x40`。写出对应的 ncbind 注册代码和 C++ 类方法。

<details>
<summary>查看答案</summary>

```cpp
// 0x20 = TJS_NCM_PROPERTY_GET → getter
// 0x40 = TJS_NCM_PROPERTY_SET → setter
// 同名 + getter + setter = 读写属性

// ncbind 注册:
NCB_REGISTER_CLASS(NativeLayerEx) {
    // ...
    NCB_PROPERTY(opacity, getOpacity, setOpacity);
    // ...
}

// C++ 类方法:
class NativeLayerEx {
private:
    double opacity_;  // 0.0 ~ 1.0
public:
    double getOpacity() const { return opacity_; }
    void setOpacity(double value) {
        if (value < 0.0) value = 0.0;
        if (value > 1.0) value = 1.0;
        opacity_ = value;
    }
};

// TJS: layer.opacity = 0.5 → setOpacity; var o = layer.opacity → getOpacity
```
