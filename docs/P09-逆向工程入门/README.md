# P09 — 逆向工程入门

> **前置依赖：** C++ 基础
> **目标：** 掌握二进制逆向基础，能分析无源码的 KiriKiri DLL 插件

## 模块概述

KrKr2 项目需要兼容原版 KiriKiri/KiriKiriZ 的全部插件。部分插件只有编译后的 DLL 二进制文件，没有源代码（参见 `doc/krkr2_plugins.md` 中标记为 `nan` 的插件）。要还原这些插件的功能，开发者需要掌握二进制逆向分析的基本技能。

本模块从零开始介绍逆向工程的核心概念、工具和方法论，最终通过实战演练逆向一个 KiriKiri 插件 DLL 并还原为可编译的 C++ 源码。

## 学习路径

```
逆向概述（合法性 + 方法论）
  │
  ▼

  │
  ▼
x86/x64 反汇编（寄存器 + 调用约定 + 栈帧）
  │
  ▼
工具实操（IDA/Ghidra/x64dbg）
  │
  ▼
DLL 分析（导出表 + 签名推断 + 名称反修饰）
  │
  ▼
实战：逆向 KiriKiri 插件 → 还原 C++ 源码
```

## 与其他模块的关系

- **P08-编译原理基础** — 理解编译流程有助于理解反编译原理
- **M07-TJS2脚本引擎** — 理解 TJS2 对象模型有助于分析插件的 TJS2 绑定
- **M09-插件系统与开发** — 理解插件接口是逆向插件的前提
- **M12-插件逆向与实现** — 本模块是 M12 的前置知识，M12 将实战逆向多个无源码插件

## 适合谁学

- 需要还原无源码 KiriKiri 插件的开发者
- 想理解"从二进制到源码"全流程的 C++ 开发者
- 对安全研究和漏洞分析感兴趣的开发者

## 学习建议

1. 建议先完成 P01（CMake）和 P08（编译原理）再学习本模块
2. 准备一台 Windows 环境用于 PE 分析和 x64dbg 调试
3. 安装 Ghidra（免费）作为主力反编译工具
4. 本模块的实战部分以 KrKr2 项目中的实际插件为目标
5. 逆向分析需要耐心——反复对照反编译输出和已知代码模式
