## Cocos2d-x 坐标系

### 左下角原点的右手坐标系

Cocos2d-x 使用**数学标准坐标系**（与 OpenGL 一致），原点在**左下角**，X 轴向右，Y 轴向上：

```
Y ↑
  │
  │    (480, 320)  ← 屏幕中心（960×640）
  │
  │
  └──────────────→ X
(0,0)
左下角原点
```

这与大多数 UI 框架（原点在左上角、Y 轴向下）相反：

| 坐标系 | 原点 | Y 轴方向 | 典型框架 |
|--------|------|----------|----------|
| Cocos2d-x / OpenGL | 左下角 | ↑ 向上 | Cocos2d-x, OpenGL, 数学 |
| 屏幕坐标 | 左上角 | ↓ 向下 | Windows GDI, HTML DOM, UIKit |
| DirectX | 左上角 | ↓ 向下 | Direct3D, WPF |

> **注意：** 触摸事件的坐标已经被 Cocos2d-x 自动转换为 GL 坐标系（左下角原点）。你不需要手动转换触摸坐标。

### 三种坐标空间

Cocos2d-x 中有三种坐标空间需要区分：

```
┌─ 世界坐标（World Space）
│   以场景左下角为原点的绝对坐标
│
├─ 本地坐标（Local / Node Space）
│   以父节点左下角为原点的相对坐标
│
└─ GL 坐标（GL Space）
    OpenGL 视口坐标（通常与世界坐标一致）
```

```cpp
#include "cocos2d.h"
USING_NS_CC;

// 坐标空间演示
auto parent = Node::create();
parent->setPosition(Vec2(100, 50));

auto child = Sprite::create("icon.png");
child->setPosition(Vec2(30, 20)); // 本地坐标：相对于 parent
parent->addChild(child);

// child 的世界坐标 = parent位置 + child本地位置
// = (100+30, 50+20) = (130, 70)
// （锚点为默认 (0.5,0.5) 时，是 child 中心的世界坐标）
```

### 坐标转换 API

```cpp
// 世界坐标 → 本地坐标（某节点的坐标空间）
Vec2 localPos = node->convertToNodeSpace(worldPos);

// 本地坐标 → 世界坐标
Vec2 worldPos = node->convertToWorldSpace(localPos);

// 带锚点的转换（以节点锚点为原点）
Vec2 localAR = node->convertToNodeSpaceAR(worldPos);
Vec2 worldAR = node->convertToWorldSpaceAR(localPos);
```

#### 不带 AR 与带 AR 的区别

```cpp
auto parent = Node::create();
parent->setPosition(Vec2(100, 100));
parent->setContentSize(Size(200, 200));
parent->setAnchorPoint(Vec2(0.5f, 0.5f)); // 中心锚点

Vec2 worldPoint(150, 150);

// convertToNodeSpace —— 以节点左下角为原点
Vec2 local = parent->convertToNodeSpace(worldPoint);
// parent 左下角在世界坐标 (100-100, 100-100) = (0, 0)
// 所以 local = (150-0, 150-0) = (150, 150)
// 实际计算考虑锚点偏移：local = (150, 150)（取决于变换矩阵）

// convertToNodeSpaceAR —— 以节点锚点为原点
Vec2 localAR = parent->convertToNodeSpaceAR(worldPoint);
// parent 锚点在世界坐标 (100, 100)
// localAR = (150-100, 150-100) = (50, 50)
```

### 完整坐标转换示例

```cpp
#include "cocos2d.h"
USING_NS_CC;

class CoordDemo : public Scene {
public:
    static CoordDemo* create() {
        auto s = new(std::nothrow) CoordDemo();
        if (s && s->init()) { s->autorelease(); return s; }
        CC_SAFE_DELETE(s);
        return nullptr;
    }

    bool init() override {
        if (!Scene::init()) return false;

        auto visSize = Director::getInstance()->getVisibleSize();

        // 父节点在屏幕中央
        auto parent = Node::create();
        parent->setPosition(visSize / 2); // (480, 320)
        parent->setContentSize(Size(200, 200));
        addChild(parent);

        // 子节点在父的 (50, 50) 处
        auto child = Sprite::create();
        child->setTextureRect(Rect(0, 0, 20, 20));
        child->setColor(Color3B::RED);
        child->setPosition(Vec2(50, 50)); // 本地坐标
        parent->addChild(child);

        // 查询子节点的世界坐标
        Vec2 childWorld = parent->convertToWorldSpace(
            child->getPosition());
        CCLOG("child 本地坐标: (50, 50)");
        CCLOG("child 世界坐标: (%.0f, %.0f)",
              childWorld.x, childWorld.y);
        // 输出: child 世界坐标: (530, 370)

        // 将一个世界坐标转到 parent 的本地空间
        Vec2 worldPoint(600, 400);
        Vec2 inParent = parent->convertToNodeSpace(worldPoint);
        CCLOG("世界 (600,400) → parent本地: (%.0f, %.0f)",
              inParent.x, inParent.y);
        // 输出: parent本地: (120, 80)

        return true;
    }
};
```

## 变换矩阵（Mat4）

### 节点变换的矩阵表示

Cocos2d-x 使用 4×4 矩阵（`Mat4`，即 `float[16]`）表示节点的变换。每个节点的最终世界变换 = 父节点变换 × 自身局部变换。

```
世界变换 = M_scene × M_parent × M_self

其中 M_self 由以下属性组合：
  平移（position）→ 平移矩阵 T
  旋转（rotation）→ 旋转矩阵 R
  缩放（scale）    → 缩放矩阵 S

M_self = T × R × S
```

### Mat4 基本操作

```cpp
#include "cocos2d.h"
USING_NS_CC;

// 单位矩阵
Mat4 identity = Mat4::IDENTITY;

// 平移矩阵
Mat4 translate;
Mat4::createTranslation(Vec3(100, 200, 0), &translate);

// 旋转矩阵（绕 Z 轴旋转 45 度）
Mat4 rotate;
float angle = CC_DEGREES_TO_RADIANS(45.0f);
Mat4::createRotationZ(angle, &rotate);

// 缩放矩阵
Mat4 scale;
Mat4::createScale(Vec3(2.0f, 2.0f, 1.0f), &scale);

// 组合变换：先缩放，再旋转，再平移
Mat4 combined = translate * rotate * scale;

// 变换一个点
Vec3 point(10, 20, 0);
Vec3 transformed;
combined.transformPoint(point, &transformed);
CCLOG("变换后: (%.1f, %.1f)", transformed.x, transformed.y);
```

### 节点的 getNodeToWorldTransform

```cpp
// 获取节点的完整变换矩阵（本地→世界）
Mat4 nodeToWorld = sprite->getNodeToWorldTransform();

// 获取逆矩阵（世界→本地）
Mat4 worldToNode = sprite->getWorldToNodeTransform();

// 用矩阵变换点
Vec3 localPoint(0, 0, 0);
Vec3 worldPoint;
nodeToWorld.transformPoint(localPoint, &worldPoint);
CCLOG("节点原点的世界坐标: (%.1f, %.1f)",
      worldPoint.x, worldPoint.y);
```

### 变换继承链

```cpp
// 场景图的变换传播
// Scene(位移=0) → Parent(位移=100,50, 缩放=2) → Child(位移=30,20)
//
// Child 的世界位置计算：
// 1. Child 本地位置 (30, 20)
// 2. 经 Parent 缩放2倍 → (60, 40)
// 3. 经 Parent 平移 → (60+100, 40+50) = (160, 90)

auto parent = Node::create();
parent->setPosition(Vec2(100, 50));
parent->setScale(2.0f);

auto child = Node::create();
child->setPosition(Vec2(30, 20));
parent->addChild(child);

scene->addChild(parent);

Vec2 worldPos = parent->convertToWorldSpace(
    child->getPosition());
CCLOG("Child 世界坐标: (%.0f, %.0f)",
      worldPos.x, worldPos.y);
// 输出：(160, 90)
```

## 多分辨率适配

### 设计分辨率与 ResolutionPolicy

不同设备有不同的物理分辨率（720p、1080p、4K 等）。Cocos2d-x 通过**设计分辨率**（Design Resolution）和**适配策略**（ResolutionPolicy）解决这个问题。

```cpp
auto glView = Director::getInstance()->getOpenGLView();

// 设置设计分辨率 960×640，使用 SHOW_ALL 策略
glView->setDesignResolutionSize(960, 640, ResolutionPolicy::SHOW_ALL);
```

五种适配策略对比：

| 策略 | 行为 | 优点 | 缺点 |
|------|------|------|------|
| `EXACT_FIT` | 拉伸到屏幕，不保持比例 | 填满屏幕 | 画面变形 |
| `SHOW_ALL` | 保持比例，可能有黑边 | 无变形、无裁剪 | 有黑边 |
| `NO_BORDER` | 保持比例，填满屏幕，裁剪溢出 | 无黑边 | 边缘被裁剪 |
| `FIXED_WIDTH` | 固定宽度，高度自适应 | 宽度精确 | 高度不可控 |
| `FIXED_HEIGHT` | 固定高度，宽度自适应 | 高度精确 | 宽度不可控 |

```
设备分辨率 1920×1080，设计分辨率 960×640

SHOW_ALL（比例 960:640=3:2，设备 1920:1080=16:9）
┌─────────────────────────┐
│ ███████████████████████ │  ← 上下黑边
│ █   960×640 内容区    █ │
│ ███████████████████████ │
└─────────────────────────┘

NO_BORDER
┌───────────────────┐
│                   │
│  可见区 960×640    │← 左右被裁剪
│                   │
└───────────────────┘

FIXED_WIDTH
┌─────────────────┐
│   960 宽度固定   │
│                 │  ← 高度可能 > 640
│                 │
│                 │
└─────────────────┘
```

### getVisibleSize 与 getVisibleOrigin

```cpp
auto director = Director::getInstance();

// 设计分辨率（你设定的逻辑尺寸）
Size designSize = glView->getDesignResolutionSize();
// = (960, 640)

// 可见区域大小（实际可用的逻辑尺寸，SHOW_ALL 时等于设计分辨率）
Size visibleSize = director->getVisibleSize();
// SHOW_ALL: (960, 640)
// FIXED_WIDTH: (960, 可能 > 或 < 640)

// 可见区域原点（黑边偏移）
Vec2 origin = director->getVisibleOrigin();
// SHOW_ALL 无黑边时: (0, 0)
// 有黑边时: 可能有偏移

// 安全布局：总是用 origin + visibleSize
auto label = Label::createWithSystemFont("Hi", "Arial", 24);
label->setPosition(Vec2(
    origin.x + visibleSize.width / 2,
    origin.y + visibleSize.height / 2
));
```

## 动手实践

### 实验：坐标转换可视化

```cpp
#include "cocos2d.h"
USING_NS_CC;

class CoordVisualizer : public Scene {
public:
    static CoordVisualizer* create() {
        auto s = new(std::nothrow) CoordVisualizer();
        if (s && s->init()) { s->autorelease(); return s; }
        CC_SAFE_DELETE(s);
        return nullptr;
    }

    bool init() override {
        if (!Scene::init()) return false;

        auto vs = Director::getInstance()->getVisibleSize();
        auto origin = Director::getInstance()->getVisibleOrigin();

        // 创建一个带旋转的父节点
        auto parent = Node::create();
        parent->setPosition(Vec2(
            origin.x + vs.width / 2,
            origin.y + vs.height / 2
        ));
        parent->setRotation(30.0f); // 旋转 30 度
        parent->setContentSize(Size(200, 200));
        addChild(parent, 0, "parent");

        // 创建红色标记（父节点原点）
        auto originMark = Sprite::create();
        originMark->setTextureRect(Rect(0, 0, 10, 10));
        originMark->setColor(Color3B::RED);
        originMark->setPosition(Vec2::ZERO);
        parent->addChild(originMark);

        // 创建绿色标记（本地坐标 100,0）
        auto xMark = Sprite::create();
        xMark->setTextureRect(Rect(0, 0, 10, 10));
        xMark->setColor(Color3B::GREEN);
        xMark->setPosition(Vec2(100, 0));
        parent->addChild(xMark);

        // 创建蓝色标记（本地坐标 0,100）
        auto yMark = Sprite::create();
        yMark->setTextureRect(Rect(0, 0, 10, 10));
        yMark->setColor(Color3B::BLUE);
        yMark->setPosition(Vec2(0, 100));
        parent->addChild(yMark);

        // 打印世界坐标
        Vec2 worldOrigin = parent->convertToWorldSpace(Vec2::ZERO);
        Vec2 worldX = parent->convertToWorldSpace(Vec2(100, 0));
        Vec2 worldY = parent->convertToWorldSpace(Vec2(0, 100));

        CCLOG("父节点旋转30度后:");
        CCLOG("  原点 世界坐标: (%.1f, %.1f)",
              worldOrigin.x, worldOrigin.y);
        CCLOG("  X轴端 世界坐标: (%.1f, %.1f)",
              worldX.x, worldX.y);
        CCLOG("  Y轴端 世界坐标: (%.1f, %.1f)",
              worldY.x, worldY.y);

        // 注册触摸监听：点击屏幕显示坐标
        auto listener = EventListenerTouchOneByOne::create();
        listener->onTouchBegan = [parent](Touch* touch, Event*) {
            Vec2 worldPos = touch->getLocation(); // GL坐标
            Vec2 localPos = parent->convertToNodeSpace(worldPos);

            CCLOG("触摸: 世界(%.0f,%.0f) → parent本地(%.0f,%.0f)",
                  worldPos.x, worldPos.y,
                  localPos.x, localPos.y);
            return true;
        };
        _eventDispatcher->addEventListenerWithSceneGraphPriority(
            listener, this);

        return true;
    }
};
```

## 常见错误及解决方案

### 错误 1：将屏幕坐标当作 GL 坐标使用

```cpp
// Windows 消息的鼠标坐标是左上角原点、Y 向下
POINT screenPos = {mouseX, mouseY};

// 错误：直接用作 Cocos2d-x 坐标
sprite->setPosition(Vec2(screenPos.x, screenPos.y));
// Y 轴反了！

// 正确：翻转 Y 轴
auto winSize = Director::getInstance()->getWinSize();
sprite->setPosition(Vec2(
    screenPos.x,
    winSize.height - screenPos.y // 翻转 Y
));
```

### 错误 2：忽略 visibleOrigin 导致多分辨率下位置偏移

```cpp
// 错误：不考虑 origin，SHOW_ALL 策略下有黑边时位置偏了
sprite->setPosition(Vec2(480, 320)); // 不一定是屏幕中心

// 正确：总是加上 origin
auto vs = Director::getInstance()->getVisibleSize();
auto origin = Director::getInstance()->getVisibleOrigin();
sprite->setPosition(Vec2(
    origin.x + vs.width / 2,
    origin.y + vs.height / 2
));
```

### 错误 3：convertToWorldSpace 在节点未加入场景时调用

```cpp
auto node = Node::create();
node->setPosition(Vec2(100, 200));
// 还没 addChild 到任何父节点

Vec2 world = node->convertToWorldSpace(Vec2::ZERO);
// 结果不正确！因为节点没有父链，变换矩阵未计算
```

**解决方案：** 确保节点已经被加入场景图后再调用坐标转换。

## 本节小结

- Cocos2d-x 使用**左下角原点**的坐标系，Y 轴向上（与 OpenGL 一致）
- 每个节点的 `position` 是**相对于父节点**的本地坐标
- `convertToNodeSpace` / `convertToWorldSpace` 用于坐标空间转换
- `Mat4` 变换矩阵沿场景图传播：世界变换 = 父变换 × 本地变换
- 设计分辨率通过 `setDesignResolutionSize` 配置，5 种 `ResolutionPolicy` 适配不同需求
- 布局时总用 `getVisibleSize()` + `getVisibleOrigin()` 确保多分辨率兼容

## 练习题与答案

### 题目 1：坐标转换计算

Parent 位置 (200, 100)，缩放 (1.5, 1.5)。Child 本地位置 (40, 60)。求 Child 的世界坐标。

<details>
<summary>查看答案</summary>

Child 世界坐标 = Parent位置 + Child本地位置 × Parent缩放
= (200 + 40×1.5, 100 + 60×1.5)
= (200 + 60, 100 + 90)
= **(260, 190)**

验证代码：

```cpp
auto parent = Node::create();
parent->setPosition(Vec2(200, 100));
parent->setScale(1.5f);
scene->addChild(parent);

auto child = Node::create();
child->setPosition(Vec2(40, 60));
parent->addChild(child);

Vec2 world = parent->convertToWorldSpace(child->getPosition());
CCLOG("世界坐标: (%.0f, %.0f)", world.x, world.y);
// 输出: 世界坐标: (260, 190)
```

</details>

### 题目 2：选择合适的分辨率策略

游戏设计分辨率为 1280×720。目标设备包括 16:9 和 18:9 手机。需求：
- 核心游戏内容在 1280×720 内
- 18:9 设备上利用额外高度显示更多内容（不留黑边）
- 16:9 设备上完整显示

应该选择哪个 ResolutionPolicy？为什么？

<details>
<summary>查看答案</summary>

应选择 **`FIXED_WIDTH`**。

原因：
1. `FIXED_WIDTH` 固定宽度为 1280，高度根据设备比例自适应
2. 16:9 设备（如 1920×1080）：高度 = 1280 / (1920/1080) = 720 → 刚好
3. 18:9 设备（如 2160×1080）：高度 = 1280 / (2160/1080) = 640 → 实际高度 = 1280 × (1080/2160) × ... 等等

更准确的计算：
- 设备 1920×1080 (16:9): visibleSize = (1280, 720)
- 设备 2340×1080 (19.5:9): visibleSize = (1280, 591) — 实际高度变小

如果想要 18:9 设备显示更多**高度**内容，应该用 **`FIXED_WIDTH`**（宽度固定，高度随设备变化）。

如果 18:9 设备实际宽高比更窄（更高），`FIXED_WIDTH` 确实会给出更多高度。但如果设备更宽，则高度会减少。

**最终结论：** 根据需求"利用额外高度"，选 `FIXED_WIDTH`。布局时用 `getVisibleSize().height` 获取实际可用高度，核心内容放在 720 高度内，额外空间显示装饰或扩展内容。

</details>
