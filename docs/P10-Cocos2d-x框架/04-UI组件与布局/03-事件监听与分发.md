## 1. EventDispatcher 架构

### 1.1 核心设计

Cocos2d-x 的事件系统基于**观察者模式**（Observer Pattern）：

```
事件源（Touch/Key/Mouse/Custom）
         │
         ▼
  EventDispatcher（全局单例）
         │
         ├── 按优先级排序监听器
         │
         ├── EventListenerTouch          ← 触摸
         ├── EventListenerKeyboard       ← 键盘
         ├── EventListenerMouse          ← 鼠标
         ├── EventListenerCustom         ← 自定义
         ├── EventListenerAcceleration   ← 加速计
         └── EventListenerFocus          ← 焦点
```

### 1.2 监听器注册方式

```cpp
auto dispatcher = Director::getInstance()->getEventDispatcher();

// 方式1：与场景图优先级关联（推荐）
// 优先级由节点的 localZOrder 和在节点树中的位置决定
dispatcher->addEventListenerWithSceneGraphPriority(listener, node);

// 方式2：固定优先级（数值越小越先处理）
dispatcher->addEventListenerWithFixedPriority(listener, priority);
// priority < 0：在场景图之前处理
// priority > 0：在场景图之后处理
// priority == 0：保留给场景图优先级，不可使用
```

### 1.3 事件分发流程

```
EventDispatcher::dispatchEvent(event)
  │
  ├── 收集所有该事件类型的监听器
  │
  ├── 排序：
  │   FixedPriority < 0  （先处理）
  │   SceneGraphPriority  （按节点树从前到后）
  │   FixedPriority > 0  （后处理）
  │
  ├── 依次调用监听器的回调
  │   if (listener->onEvent 返回 true 且 swallow) {
  │       停止分发（事件被"吞没"）
  │   }
  │
  └── 清理已标记移除的监听器
```

## 2. 触摸事件

### 2.1 单点触摸（EventListenerTouchOneByOne）

```cpp
auto listener = EventListenerTouchOneByOne::create();

// 是否吞没事件（true = 低优先级的监听器收不到）
listener->setSwallowTouches(true);

// 触摸开始——必须返回 true 才会收到后续的 MOVED/ENDED
listener->onTouchBegan = [](Touch* touch, Event* event) -> bool {
    auto target = event->getCurrentTarget();
    auto locationInNode = target->convertToNodeSpace(
        touch->getLocation()
    );
    auto size = target->getContentSize();
    Rect rect(0, 0, size.width, size.height);

    if (rect.containsPoint(locationInNode)) {
        CCLOG("触摸在节点范围内");
        return true;   // 接受此触摸，后续事件会发给我
    }
    return false;  // 不接受，让其他监听器处理
};

listener->onTouchMoved = [](Touch* touch, Event* event) {
    // touch->getDelta() 返回本次移动的偏移量
    auto delta = touch->getDelta();
    auto target = event->getCurrentTarget();
    target->setPosition(target->getPosition() + delta);
};

listener->onTouchEnded = [](Touch* touch, Event* event) {
    CCLOG("触摸结束");
};

listener->onTouchCancelled = [](Touch* touch, Event* event) {
    CCLOG("触摸被取消（如来电中断）");
};

// 绑定到节点（场景图优先级）
Director::getInstance()->getEventDispatcher()
    ->addEventListenerWithSceneGraphPriority(listener, myNode);
```

### 2.2 多点触摸（EventListenerTouchAllAtOnce）

```cpp
auto listener = EventListenerTouchAllAtOnce::create();

listener->onTouchesBegan = [](
    const std::vector<Touch*>& touches, Event* event
) {
    CCLOG("多点触摸开始，触摸点数: %zu", touches.size());
    for (auto* touch : touches) {
        CCLOG("  Touch ID=%d, pos=(%.1f, %.1f)",
              touch->getID(),
              touch->getLocation().x,
              touch->getLocation().y);
    }
};

listener->onTouchesMoved = [](
    const std::vector<Touch*>& touches, Event* event
) {
    if (touches.size() == 2) {
        // 双指缩放检测
        auto touch1 = touches[0];
        auto touch2 = touches[1];

        float prevDist = touch1->getPreviousLocation()
            .distance(touch2->getPreviousLocation());
        float currDist = touch1->getLocation()
            .distance(touch2->getLocation());

        float scale = currDist / prevDist;
        CCLOG("缩放比例: %.2f", scale);
    }
};

listener->onTouchesEnded = [](
    const std::vector<Touch*>& touches, Event* event
) {
    CCLOG("多点触摸结束");
};

Director::getInstance()->getEventDispatcher()
    ->addEventListenerWithSceneGraphPriority(listener, myNode);
```

### 2.3 常见错误：监听器复用

```cpp
// ❌ 错误——同一个 listener 注册到多个节点
auto listener = EventListenerTouchOneByOne::create();
listener->onTouchBegan = [](Touch* t, Event* e) { return true; };

node1->getEventDispatcher()->addEventListenerWithSceneGraphPriority(
    listener, node1
);
// 第二次注册会断言失败！
node2->getEventDispatcher()->addEventListenerWithSceneGraphPriority(
    listener, node2
);

// ✅ 正确——用 clone() 复制
node1->getEventDispatcher()->addEventListenerWithSceneGraphPriority(
    listener, node1
);
node2->getEventDispatcher()->addEventListenerWithSceneGraphPriority(
    listener->clone(), node2
);
```

## 4. 自定义事件

### 4.1 基本用法

自定义事件是跨模块通信的利器，避免了紧耦合。

```cpp
// === 监听方 ===
auto customListener = EventListenerCustom::create(
    "game_score_changed",   // 事件名称（字符串标识）
    [](EventCustom* event) {
        // 从事件中取出附带数据
        int* score = static_cast<int*>(event->getUserData());
        CCLOG("收到分数变化事件: %d", *score);
    }
);
Director::getInstance()->getEventDispatcher()
    ->addEventListenerWithFixedPriority(customListener, 1);

// === 发送方 ===
int newScore = 1000;
EventCustom event("game_score_changed");
event.setUserData(&newScore);
Director::getInstance()->getEventDispatcher()
    ->dispatchEvent(&event);
```

### 4.2 用自定义事件实现解耦

```cpp
// 事件名称常量——避免拼写错误
namespace GameEvents {
    const char* PLAYER_DIED     = "event_player_died";
    const char* LEVEL_COMPLETE  = "event_level_complete";
    const char* PAUSE_GAME      = "event_pause_game";
    const char* RESUME_GAME     = "event_resume_game";
}

// 战斗系统——发送事件，不关心谁来处理
class BattleSystem {
    void onPlayerHealthZero() {
        EventCustom event(GameEvents::PLAYER_DIED);
        Director::getInstance()->getEventDispatcher()
            ->dispatchEvent(&event);
    }
};

// UI 系统——监听事件，不关心谁发送的
class GameUI : public Node {
    bool init() override {
        auto listener = EventListenerCustom::create(
            GameEvents::PLAYER_DIED,
            [this](EventCustom*) {
                showGameOverPanel();
            }
        );
        getEventDispatcher()->addEventListenerWithSceneGraphPriority(
            listener, this
        );
        return true;
    }

    void showGameOverPanel() {
        // 显示 Game Over 面板
    }
};

// 音效系统——也监听同一事件
class AudioSystem : public Node {
    bool init() override {
        auto listener = EventListenerCustom::create(
            GameEvents::PLAYER_DIED,
            [](EventCustom*) {
                AudioEngine::play2d("gameover.mp3");
            }
        );
        getEventDispatcher()->addEventListenerWithSceneGraphPriority(
            listener, this
        );
        return true;
    }
};
```

### 4.4 常见错误：事件数据生命周期

```cpp
// ❌ 危险——异步使用事件数据
auto listener = EventListenerCustom::create(
    "game_data",
    [](EventCustom* event) {
        auto* data = static_cast<MyData*>(event->getUserData());

        // 延迟使用——此时 data 可能已被销毁
        Director::getInstance()->getScheduler()->schedule(
            [data](float dt) {
                data->process();  // 悬垂指针！
            }, "delayed_process", 1.0f, false
        );
    }
);

// ✅ 安全——拷贝数据
auto listener = EventListenerCustom::create(
    "game_data",
    [](EventCustom* event) {
        auto* data = static_cast<MyData*>(event->getUserData());
        MyData localCopy = *data;  // 值拷贝

        Director::getInstance()->getScheduler()->schedule(
            [localCopy](float dt) {  // lambda 捕获拷贝
                localCopy.process();  // 安全
            }, "delayed_process", 1.0f, false
        );
    }
);
```

## 5. 事件优先级与吞没

### 5.1 优先级规则

```
处理顺序（从先到后）：
1. FixedPriority < 0  的监听器（数值越小越先）
2. SceneGraphPriority 的监听器（节点树从后往前遍历）
3. FixedPriority > 0  的监听器（数值越小越先）
```

场景图优先级的遍历顺序：**从最前面的节点开始**（zOrder 最大、最后绘制的），这样视觉上"最上层"的节点最先接收到触摸事件。

```
节点树：
  Scene
  ├── Layer (zOrder=0)
  │   ├── Sprite A (zOrder=0)
  │   └── Sprite B (zOrder=1)  ← 绘制在 A 上面
  └── UI Panel (zOrder=10)     ← 绘制在最上面

触摸事件处理顺序：
  UI Panel → Sprite B → Sprite A → Layer → Scene
```

### 5.2 吞没机制（Swallow）

```cpp
auto frontListener = EventListenerTouchOneByOne::create();
frontListener->setSwallowTouches(true);  // 启用吞没

frontListener->onTouchBegan = [](Touch* touch, Event* event) -> bool {
    auto target = event->getCurrentTarget();
    // 如果触摸在目标范围内，吞没事件
    auto pos = target->convertToNodeSpace(touch->getLocation());
    if (Rect(0, 0, target->getContentSize()).containsPoint(pos)) {
        CCLOG("前景节点处理触摸——后续节点收不到");
        return true;   // 返回 true + swallow = 吞没
    }
    return false;  // 返回 false = 不接受，继续分发
};

// 后面的节点
auto backListener = EventListenerTouchOneByOne::create();
backListener->onTouchBegan = [](Touch* touch, Event* event) -> bool {
    CCLOG("背景节点收到触摸");  // 如果前景吞没了，这里不会执行
    return true;
};
```

```cpp

// 恢复
```

## 6. 监听器生命周期管理

### 6.1 自动移除

当使用 `addEventListenerWithSceneGraphPriority` 注册时，监听器与节点绑定。**节点被移除时，监听器自动移除**。

```cpp
// 节点销毁时自动清理——不需要手动 remove
auto sprite = Sprite::create("target.png");
auto listener = EventListenerTouchOneByOne::create();
listener->onTouchBegan = [](Touch*, Event*) { return true; };

sprite->getEventDispatcher()
    ->addEventListenerWithSceneGraphPriority(listener, sprite);

// 后续 sprite 被 removeFromParent 时，listener 自动移除
sprite->removeFromParent();  // listener 也没了
```

### 6.2 FixedPriority 需要手动管理

```cpp
auto listener = EventListenerKeyboard::create();
listener->onKeyPressed = [](EventKeyboard::KeyCode, Event*) {};

// FixedPriority 注册——不绑定节点
Director::getInstance()->getEventDispatcher()
    ->addEventListenerWithFixedPriority(listener, 1);

// ⚠️ 必须手动移除，否则节点销毁后回调中的捕获变量成为悬垂指针
// 在析构函数或 onExit 中移除
void MyScene::onExit() {
    Director::getInstance()->getEventDispatcher()
        ->removeEventListener(_keyListener);
    Scene::onExit();
}
```

## 动手实践

### 实践1：拖拽排序

```cpp
#include "cocos2d.h"
USING_NS_CC;

class DragSortScene : public Scene {
    Node* _dragTarget = nullptr;
    Vec2 _dragOffset;

public:
    CREATE_FUNC(DragSortScene);

    bool init() override {
        if (!Scene::init()) return false;

        // 创建 5 个可拖拽的色块
        Color4B colors[] = {
            Color4B::RED, Color4B::GREEN, Color4B::BLUE,
            Color4B::YELLOW, Color4B::MAGENTA
        };

        for (int i = 0; i < 5; i++) {
            auto block = LayerColor::create(colors[i], 100, 60);
            block->setPosition(Vec2(170, 50 + i * 70));
            block->setTag(i);
            this->addChild(block, i);
        }

        // 单个触摸监听器管理所有拖拽
        auto listener = EventListenerTouchOneByOne::create();
        listener->setSwallowTouches(true);

        listener->onTouchBegan = [this](Touch* touch, Event*) -> bool {
            auto pos = touch->getLocation();

            // 从最上层开始检测（逆序遍历子节点）
            auto& children = this->getChildren();
            for (auto it = children.rbegin(); it != children.rend(); ++it) {
                auto child = *it;
                if (child->getBoundingBox().containsPoint(pos)) {
                    _dragTarget = child;
                    _dragOffset = child->getPosition() - pos;

                    // 提升 zOrder 到最前
                    child->setLocalZOrder(100);
                    child->setOpacity(200);
                    return true;
                }
            }
            return false;
        };

        listener->onTouchMoved = [this](Touch* touch, Event*) {
            if (_dragTarget) {
                _dragTarget->setPosition(
                    touch->getLocation() + _dragOffset
                );
            }
        };

        listener->onTouchEnded = [this](Touch* touch, Event*) {
            if (_dragTarget) {
                _dragTarget->setOpacity(255);
                // 吸附到最近的槽位
                float targetY = roundf(
                    (_dragTarget->getPositionY() - 50) / 70
                ) * 70 + 50;
                targetY = clampf(targetY, 50, 50 + 4 * 70);

                _dragTarget->runAction(
                    EaseBackOut::create(
                        MoveTo::create(0.2f, Vec2(170, targetY))
                    )
                );
                _dragTarget->setLocalZOrder(
                    static_cast<int>((targetY - 50) / 70)
                );
                _dragTarget = nullptr;
            }
        };

        getEventDispatcher()
            ->addEventListenerWithSceneGraphPriority(listener, this);

        return true;
    }
};
```

### 实践2：简易事件总线

```cpp
// 类型安全的事件总线——比 EventCustom + void* 更安全
class EventBus {
public:
    using EventId = std::string;
    using Callback = std::function<void(void*)>;

    struct Subscription {
        EventId eventId;
        Callback callback;
        int id;
    };

    static EventBus& getInstance() {
        static EventBus instance;
        return instance;
    }

    // 订阅事件——返回订阅 ID，用于取消
    int subscribe(const EventId& eventId, Callback callback) {
        int subId = _nextId++;
        _subscriptions.push_back({eventId, callback, subId});
        return subId;
    }

    // 取消订阅
    void unsubscribe(int subscriptionId) {
        _subscriptions.erase(
            std::remove_if(_subscriptions.begin(), _subscriptions.end(),
                [subscriptionId](const Subscription& s) {
                    return s.id == subscriptionId;
                }),
            _subscriptions.end()
        );
    }

    // 发布事件
    void publish(const EventId& eventId, void* data = nullptr) {
        for (auto& sub : _subscriptions) {
            if (sub.eventId == eventId) {
                sub.callback(data);
            }
        }
    }

private:
    std::vector<Subscription> _subscriptions;
    int _nextId = 0;
};

// 使用
int subId = EventBus::getInstance().subscribe("player_died", [](void*) {
    CCLOG("玩家死亡——显示 Game Over");
});

// 触发
EventBus::getInstance().publish("player_died");

// 清理
EventBus::getInstance().unsubscribe(subId);
```

### 实践3：组合键检测

```cpp
// 检测 Ctrl+S、Ctrl+Z 等组合键
class HotkeyManager : public Node {
    std::set<EventKeyboard::KeyCode> _pressedKeys;

    struct Hotkey {
        std::set<EventKeyboard::KeyCode> keys;
        std::function<void()> callback;
    };
    std::vector<Hotkey> _hotkeys;

public:
    CREATE_FUNC(HotkeyManager);

    bool init() override {
        if (!Node::init()) return false;

        auto listener = EventListenerKeyboard::create();

        listener->onKeyPressed = [this](
            EventKeyboard::KeyCode code, Event*
        ) {
            _pressedKeys.insert(code);
            checkHotkeys();
        };

        listener->onKeyReleased = [this](
            EventKeyboard::KeyCode code, Event*
        ) {
            _pressedKeys.erase(code);
        };

        getEventDispatcher()
            ->addEventListenerWithSceneGraphPriority(listener, this);

        return true;
    }

    void registerHotkey(
        std::initializer_list<EventKeyboard::KeyCode> keys,
        std::function<void()> callback
    ) {
        _hotkeys.push_back({
            std::set<EventKeyboard::KeyCode>(keys),
            callback
        });
    }

private:
    void checkHotkeys() {
        for (auto& hotkey : _hotkeys) {
            if (_pressedKeys == hotkey.keys) {
                hotkey.callback();
            }
        }
    }
};

// 使用
auto hkManager = HotkeyManager::create();
this->addChild(hkManager);

hkManager->registerHotkey(
    {EventKeyboard::KeyCode::KEY_LEFT_CTRL,
     EventKeyboard::KeyCode::KEY_S},
    []() { CCLOG("Ctrl+S: 保存"); }
);

hkManager->registerHotkey(
    {EventKeyboard::KeyCode::KEY_LEFT_CTRL,
     EventKeyboard::KeyCode::KEY_Z},
    []() { CCLOG("Ctrl+Z: 撤销"); }
);
```

### KrKr2 的触摸→鼠标转换

KrKr2 运行的是 KiriKiri 游戏引擎，原本是 Windows 桌面应用，使用鼠标输入。在移动端需要将触摸事件转换为鼠标事件：

```cpp
// MainScene.cpp 中 TVPWindowLayer 的触摸处理
// TVPWindowLayer 继承自 ScrollView，内置触摸支持

// 单点触摸→鼠标左键
void TVPWindowLayer::onTouchBegan(Touch* touch, Event* event) {
    Vec2 pos = convertToNodeSpace(touch->getLocation());
    // 坐标转换：Cocos2d-x 坐标 → KiriKiri 窗口坐标
    // Cocos2d-x: 左下角原点，Y 轴向上
    // KiriKiri:  左上角原点，Y 轴向下
    int x = static_cast<int>(pos.x);
    int y = static_cast<int>(getContentSize().height - pos.y);

    // 转发为 KiriKiri 的鼠标按下事件
    OnMouseDown(x, y, mbLeft, 0);
}

void TVPWindowLayer::onTouchMoved(Touch* touch, Event* event) {
    Vec2 pos = convertToNodeSpace(touch->getLocation());
    int x = static_cast<int>(pos.x);
    int y = static_cast<int>(getContentSize().height - pos.y);
    OnMouseMove(x, y, 0);
}

void TVPWindowLayer::onTouchEnded(Touch* touch, Event* event) {
    Vec2 pos = convertToNodeSpace(touch->getLocation());
    int x = static_cast<int>(pos.x);
    int y = static_cast<int>(getContentSize().height - pos.y);
    OnMouseUp(x, y, mbLeft, 0);
}
```

## 本节小结

| 概念 | 说明 |
|------|------|
| **EventDispatcher** | 全局单例，管理所有事件监听器的注册和分发 |
| **SceneGraphPriority** | 与节点绑定，按场景图顺序分发，节点销毁时自动移除 |
| **FixedPriority** | 不绑定节点，按数值排序，需手动移除 |
| **Swallow** | 吞没事件，阻止低优先级监听器收到事件 |
| **TouchOneByOne** | 单点触摸，onTouchBegan 返回值决定是否接受 |
| **TouchAllAtOnce** | 多点触摸，所有触摸点一起回调 |
| **EventListenerKeyboard** | 键盘事件，onKeyPressed/onKeyReleased |
| **EventListenerMouse** | 鼠标事件（桌面端），支持按钮/移动/滚轮 |
| **EventCustom** | 自定义事件，字符串标识，void* 数据 |
| **clone()** | 监听器不可复用，第二个节点需 clone |

### 练习1：实现长按检测

**题目**：实现一个函数，为指定节点添加长按检测。按住超过 1 秒触发回调，如果中途移动超过 10 像素则取消。

<details>
<summary>参考答案</summary>

```cpp
void addLongPressDetector(Node* node, float duration,
                           std::function<void(Node*)> onLongPress) {
    bool* pressing = new bool(false);
    bool* cancelled = new bool(false);
    Vec2* startPos = new Vec2();

    auto listener = EventListenerTouchOneByOne::create();
    listener->setSwallowTouches(false);

    listener->onTouchBegan = [=](Touch* touch, Event* event) -> bool {
        auto target = event->getCurrentTarget();
        auto pos = target->convertToNodeSpace(touch->getLocation());
        auto size = target->getContentSize();
        if (!Rect(0, 0, size.width, size.height).containsPoint(pos)) {
            return false;
        }

        *pressing = true;
        *cancelled = false;
        *startPos = touch->getLocation();

        // 延迟调度
        target->scheduleOnce([=](float) {
            if (*pressing && !*cancelled) {
                onLongPress(target);
            }
        }, duration, "long_press_timer");

        return true;
    };

    listener->onTouchMoved = [=](Touch* touch, Event*) {
        if (touch->getLocation().distance(*startPos) > 10.0f) {
            *cancelled = true;
        }
    };

    listener->onTouchEnded = [=](Touch*, Event* event) {
        *pressing = false;
        event->getCurrentTarget()->unschedule("long_press_timer");
    };

    listener->onTouchCancelled = [=](Touch*, Event* event) {
        *pressing = false;
        event->getCurrentTarget()->unschedule("long_press_timer");
    };

    node->getEventDispatcher()
        ->addEventListenerWithSceneGraphPriority(listener, node);
}

// 使用
addLongPressDetector(myButton, 1.0f, [](Node* node) {
    CCLOG("长按触发！节点: %s", node->getName().c_str());
});
```

</details>

### 练习2：事件吞没问题排查

**题目**：以下场景中，Dialog（zOrder=10）覆盖在 GameLayer（zOrder=0）上。Dialog 有一个"关闭"按钮。问题：点击 Dialog **外部区域**时，触摸穿透到了 GameLayer。如何修复？

<details>
<summary>参考答案</summary>

**原因**：Dialog 上只有按钮设置了 `setSwallowTouches(true)`，但按钮外的 Dialog 背景没有吞没触摸事件。

**修复方案**：给 Dialog 的背景层添加一个全屏触摸监听器来拦截触摸。

```cpp
// 给 Dialog 添加透明遮罩层
auto mask = LayerColor::create(Color4B(0, 0, 0, 0));  // 全透明
mask->setContentSize(Director::getInstance()->getVisibleSize());
dialog->addChild(mask, -1);  // 放在 dialog 子节点最底层

auto blocker = EventListenerTouchOneByOne::create();
blocker->setSwallowTouches(true);  // 吞没所有触摸

blocker->onTouchBegan = [](Touch*, Event*) -> bool {
    return true;  // 无条件接受并吞没
};

mask->getEventDispatcher()
    ->addEventListenerWithSceneGraphPriority(blocker, mask);
```

或者更优雅的方式——使用 `ui::Layout` 作为遮罩：

```cpp
auto mask = Layout::create();
mask->setContentSize(Director::getInstance()->getVisibleSize());
mask->setTouchEnabled(true);     // 启用触摸
mask->setSwallowTouches(true);   // 吞没事件
dialog->addChild(mask, -1);
```

</details>
