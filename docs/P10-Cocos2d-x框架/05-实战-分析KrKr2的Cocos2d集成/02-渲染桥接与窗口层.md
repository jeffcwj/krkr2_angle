## TVPWindowLayer 架构总览

### 双重继承设计

TVPWindowLayer 是 KrKr2 最核心的桥接类，它同时继承自两个完全不同世界的接口：

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 365-366 行
class TVPWindowLayer : public cocos2d::extension::ScrollView,
                       public iWindowLayer {
```

| 基类 | 来源 | 职责 |
|------|------|------|
| `cocos2d::extension::ScrollView` | Cocos2d-x 扩展库 | 提供可滚动/可缩放的视图容器，支持触摸拖拽和惯性滚动 |
| `iWindowLayer` | KiriKiri 引擎 | 抽象窗口接口，KiriKiri 通过它设置窗口大小、更新画面、处理输入 |

这种桥接模式的精妙之处在于：KiriKiri 引擎完全不知道 Cocos2d-x 的存在，它只通过 `iWindowLayer` 接口操作窗口。而 `TVPWindowLayer` 把这些操作翻译成 Cocos2d-x 的节点操作。

```
架构关系图：

KiriKiri 引擎                    Cocos2d-x
┌──────────────┐                ┌──────────────┐
│  tTJSNI_     │                │  Director    │
│  Window      │                │  Scene       │
│              │ iWindowLayer   │  GameNode    │
│  SetSize()  ─┼───────────────▶│    ├─ ScrollView (TVPWindowLayer)
│  Update()   ─┼───────────────▶│    │    └─ PrimaryLayerArea
│  OnMouse()  ◀┼───────────────┤│    │         └─ DrawSprite
│              │                │    ├─ ConsoleWindow
│              │                │    └─ GameMainMenu
└──────────────┘                └──────────────┘
```

## 初始化与节点树构建

### init 方法详解

```cpp

```

节点树结构：

```
TVPWindowLayer (ScrollView)
│
└── _container (ScrollView 内部容器)
    └── PrimaryLayerArea (Node) — 缩放控制
        └── DrawSprite (Sprite) — 锚点(0,1)左上角
            └── Texture2D — KiriKiri 渲染结果
```

**为什么 DrawSprite 的锚点设在左上角 `(0, 1)`？** KiriKiri 引擎使用左上角为原点的坐标系（y 轴向下），而 Cocos2d-x 使用左下角为原点（y 轴向上）。将锚点设为 `(0, 1)` 后，DrawSprite 的位置设为 `(0, height)` 就能正确对齐。

### 窗口链表管理

TVPWindowLayer 使用双向链表管理多个窗口实例：

```cpp
// 构造函数：插入链表尾部
TVPWindowLayer(tTJSNI_Window *w) : TJSNativeInstance(w) {
    _nextWindow = nullptr;
    _prevWindow = _lastWindowLayer;  // 连接到上一个窗口
    _lastWindowLayer = this;         // 自己成为最后一个
    ActualZoomDenom = 1;
    ActualZoomNumer = 1;
    if(_prevWindow) {
        _prevWindow->_nextWindow = this;  // 双向链接
    }
}

// 析构函数：从链表中移除
~TVPWindowLayer() override {
    if(_lastWindowLayer == this)
        _lastWindowLayer = _prevWindow;
    if(_nextWindow) _nextWindow->_prevWindow = _prevWindow;
    if(_prevWindow) _prevWindow->_nextWindow = _nextWindow;

    // 如果当前窗口正在显示，切换到另一个可见窗口
    if(_currentWindowLayer == this) {
        TVPWindowLayer *anotherWin = _lastWindowLayer;
        while(anotherWin && !anotherWin->isVisible()) {
            anotherWin = anotherWin->_prevWindow;
        }
        _currentWindowLayer = anotherWin;
    }
}
```

```
窗口链表示例（游戏运行时可能有多个窗口）：

_lastWindowLayer ──▶ WindowC
                      │ _prevWindow
                      ▼
                    WindowB ◀── _currentWindowLayer（当前活动窗口）
                      │ _prevWindow
                      ▼
                    WindowA
                      │ _prevWindow
                      ▼
                    nullptr
```

## 纹理更新：KiriKiri 渲染到 Cocos2d-x

### UpdateDrawBuffer 核心流程

这是整个桥接的核心——KiriKiri 引擎每帧渲染完成后，调用此方法将渲染结果传递给 Cocos2d-x：

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 983-1030 行
void UpdateDrawBuffer(iTVPTexture2D *tex) override {
    if(!tex) return;

    // 获取 DrawSprite 当前持有的纹理
    Texture2D *tex2d = DrawSprite->getTexture();
    // 从 KiriKiri 纹理适配器获取 Cocos2d-x 兼容纹理
    Texture2D *newtex = tex->GetAdapterTexture(tex2d);

    if(tex2d != newtex) {
        // 纹理对象发生变化（首次或尺寸改变时）
        DrawSprite->setTexture(newtex);

        // 获取纹理内部缩放因子
        float sw, sh;
        tex->GetScale(_drawTextureScaleX, _drawTextureScaleY);

        if(_drawTextureScaleX == 1.f)
            sw = LayerWidth;
        else {
            // 纹理实际尺寸与逻辑尺寸不同（如 NPOT 纹理），需要缩放
            sw = tex->GetInternalWidth() *
                ((float)LayerWidth / tex->GetWidth());
            _drawTextureScaleX = 1 / _drawTextureScaleX;
        }
        if(_drawTextureScaleY == 1.f)
            sh = LayerHeight;
        else {
            sh = tex->GetInternalHeight() *
                ((float)LayerHeight / tex->GetHeight());
            _drawTextureScaleY = 1 / _drawTextureScaleY;
        }

        DrawSprite->setTextureRect(cocos2d::Rect(0, 0, sw, sh));
        DrawSprite->setBlendFunc(BlendFunc::DISABLE);  // 不需要混合
        ResetDrawSprite();  // 重新计算位置和缩放
    }
}
```

整个纹理传递流程：

```
┌─────────────────────────────────────────────────────────────┐
│ KiriKiri 引擎渲染流程                                        │
│                                                             │
│ 1. TJS2 脚本触发图层更新                                     │
│ 2. visual/ 模块执行图像混合/变换                              │
│ 3. LayerBitmapImpl 生成 RGBA 像素数据                        │
│ 4. iTVPTexture2D 封装为纹理对象                               │
│    ↓                                                        │
│ 5. 调用 iWindowLayer::UpdateDrawBuffer(texture)              │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│ TVPWindowLayer 桥接层                                        │
│                                                             │
│ 6. GetAdapterTexture() — 获取/创建 cocos2d::Texture2D       │
│ 7. DrawSprite->setTexture(newtex) — 更新精灵纹理             │
│ 8. 调整 TextureRect 和 Scale — 处理 NPOT 纹理缩放            │
│ 9. ResetDrawSprite() — 重计算精灵位置                         │
│    ↓                                                        │
│ Cocos2d-x 在下一帧 Director::drawScene() 时绘制 DrawSprite   │
└─────────────────────────────────────────────────────────────┘
```

**关键概念：NPOT（Non-Power-Of-Two）纹理处理**

某些旧 GPU 不支持非 2 的幂次纹理（如 800×600），需要将其填充到最近的 2 的幂次尺寸（1024×1024）。此时纹理的实际大小（`GetInternalWidth`）与逻辑大小（`GetWidth`）不同，`_drawTextureScaleX/Y` 就用于补偿这个差异。

### ResetDrawSprite 缩放与定位

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 882-911 行
void ResetDrawSprite() {
    if(DrawSprite) {
        cocos2d::Size size = getContentSize();
        float scale = (float)ActualZoomNumer / ActualZoomDenom;
        size = size / scale;

        // 设置纹理缩放补偿
        DrawSprite->setScale(_drawTextureScaleX, _drawTextureScaleY);
        // 设置纹理矩形为完整图层尺寸
        DrawSprite->setTextureRect(
            cocos2d::Rect(0, 0, LayerWidth, LayerHeight));
        // 定位到左上角（锚点为 (0,1)，所以 y = height）
        DrawSprite->setPosition(Vec2(0, size.height));

        // 设置图层区域的内容尺寸和缩放
        PrimaryLayerArea->setContentSize(size);
        PrimaryLayerArea->setScale(scale);
    }
}
```

缩放关系：

```
ActualZoomNumer / ActualZoomDenom = 用户请求的缩放比例

示例：ZoomNumer=2, ZoomDenom=1 → 2倍放大
  ContentSize = (800, 600)
  size = (800, 600) / 2 = (400, 300)    // 逻辑尺寸缩小
  PrimaryLayerArea->setScale(2)          // 实际绘制时放大回来
  效果：画面以 2 倍大小显示
```

## 触摸→鼠标事件转换

KiriKiri 引擎原生只支持鼠标事件（左键点击、右键点击、鼠标移动等）。在移动端，TVPWindowLayer 需要将触摸事件转换为鼠标事件。

### 触摸手势映射表

```
┌────────────────────┬─────────────────────────┐
│  触摸手势          │  映射的鼠标事件          │
├────────────────────┼─────────────────────────┤
│  单指轻触          │  鼠标左键点击            │
│  单指长按并拖动     │  鼠标左键按下 + 移动     │
│  双指触摸          │  鼠标右键点击            │
│  三指触摸          │  鼠标中键点击            │
│  滚轮滚动          │  鼠标滚轮事件            │
└────────────────────┴─────────────────────────┘
```

### onTouchBegan 事件分发

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 566-593 行
bool onTouchBegan(Touch *touch, Event *unused_event) override {
    if(_windowMgrOverlay)
        return inherit::onTouchBegan(touch, unused_event); // 窗口管理模式

    // 维护触摸列表
    if(std::find(_touches.begin(), _touches.end(), touch) ==
       _touches.end()) {
        _touches.push_back(touch);
    }

    switch(_touches.size()) {
        case 1:  // 单指 → 左键
            _touchPoint = touch->getLocation();
            _touchMoved = false;
            _touchLength = 0.0f;
            _touchBeginTick = TVPGetRoughTickCount32();
            _mouseBtn = ::mbLeft;
            break;
        case 2:  // 双指 → 右键
            _mouseBtn = ::mbRight;
            _touchPoint = (_touchPoint + touch->getLocation()) / 2;
            break;
        case 3:  // 三指 → 中键
            _mouseBtn = ::mbMiddle;
        default:
            break;
    }
    return true;
}
```

### 触摸移动的阈值判断

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 595-632 行（简化）
void onTouchMoved(Touch *touch, Event *unused_event) override {
    if(_touches.size() == 1) {
        // 判断是否超过移动阈值
        if(!_touchMoved &&
           (TVPGetRoughTickCount32() - _touchBeginTick > 150 ||    // 时间阈值 150ms
            _touchPoint.getDistanceSq(touch->getLocation()) >
                _touchMoveThresholdSq)) {                          // 距离阈值
            // 首次确认为拖动：发送 MouseDown 事件
            Vec2 nsp = PrimaryLayerArea->convertToNodeSpace(_touchPoint);
            _LastMouseX = nsp.x;
            _LastMouseY = PrimaryLayerArea->getContentSize().height - nsp.y;
            TVPPostInputEvent(new tTVPOnMouseDownInputEvent(
                TJSNativeInstance, _LastMouseX, _LastMouseY,
                _mouseBtn, TVPGetCurrentShiftKeyState()));
            _touchMoved = true;
        } else if(_touchMoved) {
            // 后续拖动：发送 MouseMove 事件
            Vec2 nsp = PrimaryLayerArea->convertTouchToNodeSpace(touch);
            _LastMouseX = nsp.x;
            _LastMouseY = PrimaryLayerArea->getContentSize().height - nsp.y;
            TVPPostInputEvent(new tTVPOnMouseMoveInputEvent(
                TJSNativeInstance, _LastMouseX, _LastMouseY,
                TVPGetCurrentShiftKeyState()),
                TVP_EPT_DISCARDABLE);  // 可丢弃的事件（性能优化）
        }
    }
}
```

**坐标转换的关键步骤：**

```
1. touch->getLocation()
   ↓  屏幕坐标（Cocos2d-x 世界坐标，左下角原点）

2. PrimaryLayerArea->convertToNodeSpace(pt)
   ↓  转换到图层区域的本地坐标（左下角原点）

3. y = ContentSize.height - nsp.y
   ↓  翻转 Y 轴：Cocos2d-x (左下原点) → KiriKiri (左上原点)

4. 最终 (_LastMouseX, _LastMouseY)
   = KiriKiri 窗口坐标系中的鼠标位置
```

## YUV 视频精灵

KrKr2 使用 FFmpeg 解码视频，输出 YUV420P 格式数据。TVPYUVSprite 通过自定义 GLSL 着色器在 GPU 上完成 YUV→RGB 转换。

### GLSL 片段着色器解析

```glsl
// 文件：cpp/core/environ/cocos2d/YUVSprite.cpp 第 7-38 行
#ifdef GL_ES
precision lowp float;    // 移动端使用低精度（性能优先）
#endif

varying vec4 v_fragmentColor;
varying vec2 v_texCoord;

void main() {
    vec3 yuv;
    // 从三个独立纹理采样 Y/U/V 分量
    yuv.x = texture2D(CC_Texture0, v_texCoord).r - 0.0625;  // Y（亮度）
    yuv.y = texture2D(CC_Texture1, v_texCoord).r - 0.5;     // U（蓝色色差）
    yuv.z = texture2D(CC_Texture2, v_texCoord).r - 0.5;     // V（红色色差）

    // BT.601 标准 YUV→RGB 转换矩阵
    vec3 rgb = mat3(
        1.164,  1.164, 1.164,   // Y 对 R/G/B 的贡献
        0.0,   -0.392, 2.017,   // U 对 R/G/B 的贡献
        1.596, -0.813, 0.0      // V 对 R/G/B 的贡献
    ) * yuv;

    gl_FragColor = vec4(rgb, 1.0) * v_fragmentColor;
}
```

**为什么使用三个独立纹理？** YUV420P 格式中，Y 分量（亮度）的分辨率是 U/V 分量（色度）的 2 倍。如果打包到一个纹理中会浪费空间并增加 shader 复杂度。用三个 `I8`（单通道灰度）纹理分别存储，GPU 的纹理采样器自动处理上采样。

### 纹理初始化与更新

```cpp
// 文件：cpp/core/environ/cocos2d/YUVSprite.cpp 第 162-171 行
bool TVPYUVSprite::init() {
    initWithTexture(new cocos2d::Texture2D);  // Y 纹理（CC_Texture0）
    setupVBOAndVAO();

    _textureU->retain();                       // 手动持有引用

    _textureV->retain();

// YUV420P 三平面数据更新
void TVPYUVSprite::updateTextureData(
    const void *Y, int YW, int YH,    // Y 平面（全分辨率）
    const void *U, int UW, int UH,    // U 平面（1/4 分辨率）
    const void *V, int VW, int VH) {  // V 平面（1/4 分辨率）
    // 分别更新三个纹理，格式为 I8（单通道 8 位）
    updateTextureDataInternal(getTexture(), Y, YW, YH,
                              Texture2D::PixelFormat::I8);
    updateTextureDataInternal(_textureU, U, UW, UH,
                              Texture2D::PixelFormat::I8);
    updateTextureDataInternal(_textureV, V, VW, VH,
                              Texture2D::PixelFormat::I8);
    if(size.width != YW || size.height != YH) {
        setTextureRect(cocos2d::Rect(0, 0, YW, YH));
    }
}
```

### 自定义渲染命令

```cpp
// 文件：cpp/core/environ/cocos2d/YUVSprite.cpp 第 259-264 行
void TVPYUVSprite::draw(Renderer *renderer,
                        const Mat4 &transform, uint32_t flags) {
    _mv = transform;                              // 保存变换矩阵
    _drawCommand.init(_globalZOrder);             // 设置渲染顺序
    renderer->addCommand(&_drawCommand);          // 添加到渲染队列
}

// 实际绘制：绑定三个纹理并绘制四边形
void TVPYUVSprite::onDraw() {
    // 绑定 Y/U/V 三个纹理到纹理单元 0/1/2
    GL::bindTexture2DN(0, _texture->getName());   // CC_Texture0 = Y
    GL::bindTexture2DN(1, _textureU->getName());  // CC_Texture1 = U
    GL::bindTexture2DN(2, _textureV->getName());  // CC_Texture2 = V

    GL::blendFunc(_blendFunc.src, _blendFunc.dst);
    _glProgramState->apply(_mv);

    // 设置顶点属性（位置、颜色、纹理坐标）
    GL::enableVertexAttribs(GL::VERTEX_ATTRIB_FLAG_POS_COLOR_TEX);
    // ... 提交 4 个顶点的数据 ...

    // 绘制三角形带（2 个三角形组成矩形）
    glDrawArrays(GL_TRIANGLE_STRIP, 0, 4);
}
```

**为什么用 CustomCommand 而不是标准 Sprite 渲染？** 标准 Sprite 只支持单纹理。YUV 需要同时绑定三个纹理到不同的纹理单元（`GL_TEXTURE0/1/2`），必须用 CustomCommand 手动控制 GL 状态。

## 缩放与自适应

### RecalcPaintBox 自适应算法

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 913-937 行
void RecalcPaintBox() {
    if(!LayerWidth || !LayerHeight) return;
    ResetDrawSprite();

    cocos2d::Size size = getViewSize();         // 可视区域大小
    cocos2d::Size contSize = getContentSize();  // 内容大小
    float r = size.width / size.height;         // 视图宽高比
    float R = contSize.width / contSize.height; // 内容宽高比

    float scale;
    Vec2 offset;
    if(R > r) {
        // 内容比视图更宽 → 以宽度为基准缩放
        scale = size.width / contSize.width;
        offset.x = 0;
        offset.y = (size.height - contSize.height * scale) / 2; // 垂直居中
    } else {
        // 内容比视图更高 → 以高度为基准缩放
        scale = size.height / contSize.height;
        offset.x = (size.width - contSize.width * scale) / 2;   // 水平居中
        offset.y = 0;
    }

    setMinScale(scale);          // 最小缩放 = 刚好填充
    setMaxScale(scale * 2);      // 最大缩放 = 2 倍放大
    setZoomScale(scale);         // 当前缩放
    setContentOffset(offset);    // 居中偏移
    updateInset();
}
```

这就是经典的 "letterbox"（信箱模式）或 "pillarbox"（柱箱模式）适配算法：

```
情况 1：内容比视图更宽（R > r）→ Letterbox
┌─────────────────────────┐
│        黑边(offset.y)    │
├─────────────────────────┤
│                         │
│       游戏画面           │
│                         │
├─────────────────────────┤
│        黑边(offset.y)    │
└─────────────────────────┘

情况 2：内容比视图更高（R < r）→ Pillarbox
┌────┬─────────────┬────┐
│黑边│             │黑边│
│    │  游戏画面    │    │
│    │             │    │
│    │             │    │
└────┴─────────────┴────┘
 offset.x         offset.x
```

```glsl
// 简化版 YUV 着色器（用于理解原理，只用一个纹理）
// 假设 YUV 数据已打包为 RGBA 纹理（Y=R, U=G, V=B）
#ifdef GL_ES
precision mediump float;
#endif

varying vec2 v_texCoord;

void main() {
    vec4 packed = texture2D(CC_Texture0, v_texCoord);
    float y = packed.r - 0.0625;
    float u = packed.g - 0.5;
    float v = packed.b - 0.5;

    // BT.601 转换
    float r = 1.164 * y + 1.596 * v;
    float g = 1.164 * y - 0.392 * u - 0.813 * v;
    float b = 1.164 * y + 2.017 * u;

    gl_FragColor = vec4(r, g, b, 1.0);
}
```

### 题目 1：坐标转换推导

假设游戏画面 `PrimaryLayerArea` 的 ContentSize 为 `(800, 600)`，用户触摸了 Cocos2d-x 世界坐标 `(400, 450)` 的位置。经过 `convertToNodeSpace` 转换后得到节点坐标 `(400, 450)`。那么传递给 KiriKiri 引擎的鼠标坐标 `(_LastMouseX, _LastMouseY)` 是多少？

<details>
<summary>查看答案</summary>

```
_LastMouseX = nsp.x = 400
_LastMouseY = ContentSize.height - nsp.y = 600 - 450 = 150

最终坐标：(400, 150)
```

推导过程：
1. 触摸点的世界坐标 `(400, 450)` 经过 `convertToNodeSpace` 后为 `(400, 450)`（假设 PrimaryLayerArea 没有偏移和旋转）
2. Y 轴翻转：`600 - 450 = 150`
3. 在 KiriKiri 坐标系中，`(0, 0)` 是左上角，所以 `(400, 150)` 表示距左边 400 像素、距顶部 150 像素

验证：在 Cocos2d-x 中 y=450 表示偏上方的位置（y 轴向上），翻转后 y=150 在 KiriKiri 中也是偏上方的位置（y 轴向下），逻辑一致。

</details>

### 题目 2：分析 YUV 着色器矩阵

KrKr2 的 YUV→RGB 着色器使用以下转换矩阵：

```
mat3(1.164, 1.164, 1.164,
     0.0,  -0.392, 2.017,
     1.596, -0.813, 0.0)
```

问：
1. 为什么 Y 的系数是 1.164 而不是 1.0？
2. 如果 U 和 V 都为 0（灰色画面），R=G=B 的值分别是多少（假设 Y 信号值为 128）？

<details>
<summary>查看答案</summary>

**问题 1 解答：**

Y 的系数是 1.164 而不是 1.0，是因为 BT.601 标准中 Y 的有效范围不是 `[0, 255]` 而是 `[16, 235]`（"studio swing" 或 "limited range"）。

- 代码中先减去了 0.0625（即 16/256），将 Y 偏移到从 0 开始
- 然后乘以 1.164（≈ 256/(235-16) ≈ 1.1644）将范围从 `[0, 219/256]` 拉伸到 `[0, 1.0]`

如果直接用 1.0 作为系数，灰色画面会偏暗（因为 Y=16 映射到黑色而非略高于黑色）。

**问题 2 解答：**

```
Y 信号值 = 128/255 ≈ 0.502
减去偏移：yuv.x = 0.502 - 0.0625 = 0.4395
U = V = 0 → yuv.y = 0 - 0.5 = -0.5, yuv.z = 0 - 0.5 = -0.5

R = 1.164 × 0.4395 + 0.0 × (-0.5) + 1.596 × (-0.5)
  = 0.5116 + 0 - 0.798
  = -0.2864（被截断为 0）

G = 1.164 × 0.4395 + (-0.392) × (-0.5) + (-0.813) × (-0.5)
  = 0.5116 + 0.196 + 0.4065
  = 1.114（被截断为 1.0）

B = 1.164 × 0.4395 + 2.017 × (-0.5) + 0.0 × (-0.5)
  = 0.5116 - 1.0085
  = -0.4969（被截断为 0）
```

结果 RGB=(0, 1.0, 0)，画面偏绿。这说明 U=V=0 并不代表灰色——在 YUV 空间中，灰色对应 U=128、V=128（经过偏移后 u=0, v=0）。原始信号值 U=V=0 实际代表极端色偏。

</details>

### 题目 3：设计一个简化版窗口桥接

使用 Cocos2d-x API 编写一个简化版的窗口桥接类，要求：
1. 继承 `cocos2d::Node`
2. 包含一个 `Sprite` 作为渲染目标
3. 提供 `updateTexture(const void *data, int w, int h)` 方法更新纹理
4. 提供 `convertTouchToEngineCoord(Vec2 touchPos)` 方法做坐标转换（Y 轴翻转）

<details>
<summary>查看答案</summary>

```cpp
#include <cocos2d.h>
USING_NS_CC;

class SimpleWindowBridge : public cocos2d::Node {
    Sprite *_drawSprite = nullptr;
    int _contentWidth = 0;
    int _contentHeight = 0;

public:
    static SimpleWindowBridge *create(int width, int height) {
        auto *bridge = new SimpleWindowBridge;
        if(bridge->initWithSize(width, height)) {
            bridge->autorelease();
            return bridge;
        }
        delete bridge;
        return nullptr;
    }

    bool initWithSize(int width, int height) {
        if(!Node::init()) return false;
        _contentWidth = width;
        _contentHeight = height;
        setContentSize(Size(width, height));

        // 创建渲染精灵，锚点左上角（匹配引擎坐标系）
        _drawSprite = Sprite::create();
        _drawSprite->setAnchorPoint(Vec2(0, 1));       // 左上角锚点
        _drawSprite->setPosition(Vec2(0, height));      // 定位到节点左上角
        addChild(_drawSprite);

        return true;
    }

    // 更新纹理数据（模拟 KiriKiri 的 UpdateDrawBuffer）
    void updateTexture(const void *data, int w, int h) {
        if(!data) return;
        Texture2D *tex = _drawSprite->getTexture();
        if(!tex || tex->getContentSize().width != w ||
           tex->getContentSize().height != h) {
            // 首次或尺寸变化：重新创建纹理
            tex = new Texture2D;
            tex->initWithData(data, w * h * 4,
                              Texture2D::PixelFormat::RGBA8888,
                              w, h, Size::ZERO);
            _drawSprite->setTexture(tex);
            _drawSprite->setTextureRect(Rect(0, 0, w, h));
            tex->release();  // Sprite 已 retain
        } else {
            // 尺寸不变：仅更新数据
            tex->updateWithData(data, 0, 0, w, h);
        }
    }

    // 坐标转换：Cocos2d-x 触摸坐标 → 引擎坐标（Y 轴翻转）
    Vec2 convertTouchToEngineCoord(const Vec2 &touchWorldPos) {
        Vec2 localPos = convertToNodeSpace(touchWorldPos);
        // Y 轴翻转：Cocos2d-x (左下原点) → 引擎 (左上原点)
        float engineY = _contentHeight - localPos.y;
        return Vec2(localPos.x, engineY);
    }
};

// 使用示例
auto bridge = SimpleWindowBridge::create(800, 600);
scene->addChild(bridge);

// 模拟渲染更新
std::vector<uint8_t> pixels(800 * 600 * 4, 128); // 灰色画面
bridge->updateTexture(pixels.data(), 800, 600);

// 触摸坐标转换
Vec2 enginePos = bridge->convertTouchToEngineCoord(
    Vec2(400, 450));  // → (400, 150) 在引擎坐标系
```

</details>
