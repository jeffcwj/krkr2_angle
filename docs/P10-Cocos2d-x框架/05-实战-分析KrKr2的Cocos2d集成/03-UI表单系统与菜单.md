## CSBReader：带节点索引的资源加载器

### CSB 加载的核心问题

Cocos Studio 导出的 `.csb` 文件通过 `CSLoader::createNode()` 加载后，得到的是一棵完整的节点树。但在业务代码中，我们需要快速访问特定的子节点——比如名为 `"btn_ok"` 的按钮或 `"title"` 的标签。逐层 `getChildByName()` 不仅代码冗长，还容易因层级结构变化而失效。

KrKr2 的解决方案是 `CSBReader` 类：在加载 `.csb` 的同时，遍历整棵节点树，建立 `名称 → 节点指针` 的扁平映射。

### NodeMap 基类：名称到节点的扁平索引

```cpp
// 文件：cpp/core/environ/ui/BaseForm.h 第 44-71 行
class NodeMap : public std::unordered_map<std::string, cocos2d::Node *> {
protected:
    const char *FileName;               // 记录来源文件名，用于错误提示
    void onLoadError(const std::string &name) const;

public:
    NodeMap();
    NodeMap(const char *filename, cocos2d::Node *node);

    // 模板查找方法，支持类型安全的 dynamic_cast
    template <typename T = cocos2d::Node>
    [[nodiscard]] T *findController(const std::string &name,
                                    bool notice = true) const {
        auto *node = findController<cocos2d::Node>(name, notice);
        if(node) {
            return dynamic_cast<T *>(node);  // 安全向下转型
        }
        return nullptr;
    }

    // 便捷方法：直接查找 Widget 类型
    [[nodiscard]] cocos2d::ui::Widget *findWidget(
        const std::string &name, bool notice = true) const {
        return findController<cocos2d::ui::Widget>(name, notice);
    }

    // 递归遍历节点树，建立扁平索引
    void initFromNode(cocos2d::Node *node);
};
```

关键设计要点：

| 特性 | 实现细节 | 设计意图 |
|------|----------|----------|
| 扁平索引 | `unordered_map<string, Node*>` | O(1) 按名称查找，无需知道层级结构 |
| 类型安全 | `findController<T>` 使用 `dynamic_cast` | 编译期指定期望类型，运行时检查 |
| 错误报告 | `notice` 参数控制是否弹窗 | 加载 UI 时缺少节点会立即报错，便于定位 |
| 递归构建 | `initFromNode` 遍历所有子节点 | 深层嵌套的节点也能直接按名称访问 |

`initFromNode` 的实现——递归遍历子节点，将有名称的节点加入哈希表：

```cpp
// 文件：cpp/core/environ/ui/BaseForm.cpp 第 40-48 行
void NodeMap::initFromNode(cocos2d::Node *node) {
    const Vector<Node *> &childlist = node->getChildren();
    for(auto child : childlist) {
        std::string name = child->getName();
        if(!name.empty())
            (*this)[name] = child;   // 扁平化存储：忽略层级
        initFromNode(child);         // 递归处理子节点
    }
}
```

> **注意**：如果不同层级的节点同名，后遍历到的会覆盖先前的。这要求 `.csb` 设计时避免重名——这是 KrKr2 项目的隐含约定。

### CSBReader：加载+索引一步完成

```cpp
// 文件：cpp/core/environ/ui/BaseForm.cpp 第 56-77 行
Node *CSBReader::Load(const char *filename) {
    clear();                          // 清除上一次加载的索引
    FileName = filename;              // 记录文件名（错误提示用）
    Node *ret = CSLoader::createNode(filename, [this](Ref *p) {
        // 回调：每个节点创建时触发
        Node *node = dynamic_cast<Node *>(p);
        std::string name = node->getName();
        if(!name.empty())
            operator[](name) = node;  // 直接写入 unordered_map

        // 检测并自动播放 "autoplay" 动画
        int nAction = node->getNumberOfRunningActions();
        if(nAction == 1) {
            auto *action = dynamic_cast<
                cocostudio::timeline::ActionTimeline *>(
                node->getActionByTag(node->getTag()));
            if(action && action->IsAnimationInfoExists("autoplay")) {
                action->play("autoplay", true);   // 循环播放
            }
        }
    });
    if(!ret) {
        TVPShowSimpleMessageBox(filename, "Fail to load ui file");
    }
    return ret;
}
```

加载流程时序：

```
CSBReader::Load("ui/GameMenu.csb")
    │
    ├─ clear()  ← 清空旧索引
    │
    ├─ CSLoader::createNode(filename, callback)
    │      │
    │      ├─ 解析 .csb 二进制 → 创建节点树
    │      │
    │      └─ 每创建一个节点 → 调用 callback(node)
    │             │
    │             ├─ 有名称？→ 写入 unordered_map
    │             │
    │             └─ 有 ActionTimeline 且含 "autoplay"？→ 自动播放
    │
    └─ 返回根节点（同时 this 已经充满了索引数据）
```

## iTVPBaseForm：三段式布局基类

### 架构设计

KrKr2 的所有 UI 表单都遵循同一个布局模式：顶部导航栏（NaviBar, 10%）+ 中间内容区域（Body, 80%）+ 底部操作栏（Footer, 10%）。这个模式由 `iTVPBaseForm` 基类强制约束：

```
┌────────────────────────────────┐
│         NaviBar (10%)          │  ← 返回按钮 + 标题 + 右侧菜单
│  [←]    Page Title       [≡]  │
├────────────────────────────────┤
│                                │
│         Body (80%)             │  ← 主内容区：列表/表单/自定义内容
│                                │
├────────────────────────────────┤
│         Footer (10%)           │  ← 操作按钮/状态栏（可选）
│      [Cancel]    [OK]         │
└────────────────────────────────┘
```

### initFromFile：构建三段式布局

```cpp
// 文件：cpp/core/environ/ui/BaseForm.cpp 第 83-135 行
bool iTVPBaseForm::initFromFile(
    const Csd::NodeBuilderFn &naviBarCall,   // NaviBar 构建函数
    const Csd::NodeBuilderFn &bodyCall,      // Body 构建函数
    const Csd::NodeBuilderFn &bottomBarCall, // Footer 构建函数
    Node *parent)
{
    const bool ret = Node::init();
    const auto scale = TVPMainScene::GetInstance()->getUIScale();

    // 调用构建函数，传入各区域的目标尺寸
    auto *naviBar   = naviBarCall(rearrangeHeaderSize(parent), scale);
    auto *body      = bodyCall(rearrangeBodySize(parent), scale);
    auto *bottomBar = bottomBarCall(rearrangeFooterSize(parent), scale);

    RootNode = body;
    if(!RootNode) return false;
    if(!parent)   parent = this;

    // NaviBar → 顶部对齐
    if(naviBar) {
        NaviBar.Root  = naviBar->getChildByName("background");
        NaviBar.Left  = NaviBar.Root->getChildByName<Button *>("left");
        NaviBar.Right = NaviBar.Root->getChildByName<Button *>("right");
        bindHeaderController(NaviBar.Root);

        auto *param = LinearLayoutParameter::create();
        param->setGravity(LinearLayoutParameter::LinearGravity::TOP);
        naviBar->setLayoutParameter(param);
        parent->addChild(naviBar);
    }

    // Footer → 底部对齐
    if(bottomBar) {
        BottomBar.Root = bottomBar;
        bindFooterController(bottomBar);
        auto *param = LinearLayoutParameter::create();
        param->setGravity(LinearLayoutParameter::LinearGravity::BOTTOM);
        bottomBar->setLayoutParameter(param);
        parent->addChild(BottomBar.Root);
    }

    // Body → 垂直居中
    auto *param = LinearLayoutParameter::create();
    param->setGravity(
        LinearLayoutParameter::LinearGravity::CENTER_VERTICAL);
    body->setLayoutParameter(param);
    parent->addChild(RootNode);
    bindBodyController(RootNode);
    return ret;
}
```

`NodeBuilderFn` 类型定义为：

```cpp
// 文件：cpp/core/environ/ui/csd/CsdUIFactory.h 第 17-18 行
using NodeBuilderFn =
    std::function<Widget *(const cocos2d::Size &size, float scale)>;
```

这是一个工厂函数类型——接收目标尺寸和缩放比，返回构建好的 Widget 节点。子类可以传入 CSB 加载函数，也可以传入纯代码构建函数。

### iTVPFloatForm：浮动表单变体

`iTVPFloatForm` 是 `iTVPBaseForm` 的变体，不使用全屏三段式布局，而是占屏幕 75% 的居中浮动窗口：

```cpp
// 文件：cpp/core/environ/ui/BaseForm.cpp 第 147-163 行
void iTVPFloatForm::rearrangeLayout() {
    float scale = TVPMainScene::GetInstance()->getUIScale();
    cocos2d::Size sceneSize =
        TVPMainScene::GetInstance()->getUINodeSize();
    setContentSize(sceneSize);
    Vec2 center = sceneSize / 2;
    sceneSize.height *= 0.75f;        // 高度取 75%
    sceneSize.width *= 0.75f;         // 宽度取 75%
    if(RootNode) {
        sceneSize.width /= scale;
        sceneSize.height /= scale;
        RootNode->setContentSize(sceneSize);
        ui::Helper::doLayout(RootNode);
        RootNode->setScale(scale);
        RootNode->setAnchorPoint(Vec2(0.5f, 0.5f));  // 居中锚点
        RootNode->setPosition(center);
    }
}
```

## pushUIForm / popUIForm：动画表单栈

### UINode 层级与表单栈

回顾第一节的双节点树架构，`TVPMainScene` 拥有独立的 `UINode` 承载 UI 表单：

```
MainScene (zOrder 排序)
├── GameNode   (GAME_SCENE_ORDER = 0)    ← 游戏内容
│   ├── WindowLayer / ConsoleWindow
│   ├── WinMgrOverlay (zOrder=15)
│   └── GameMainMenu  (zOrder=20)
└── UINode     (UI_NODE_ORDER = 100)     ← UI 表单栈
    ├── [Form A] ← 最底层表单
    ├── [Form B] ← 上层表单（覆盖 A）
    └── [Form C] ← 最顶层（当前可见）
```

### pushUIForm：三种进入动画

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 1827-1871 行
void TVPMainScene::pushUIForm(cocos2d::Node *ui, eEnterAni ani) {
    TVPControlAdDialog(0x10002, 1, 0);  // 通知广告系统（Android）
    int n = UINode->getChildrenCount();

    if(ani == eEnterAniNone) {
        UINode->addChild(ui);           // 无动画：直接添加
    } else if(ani == eEnterAniOverFromRight) {
        if(n > 0) {
            cocos2d::Size size = UINode->getContentSize();
            // 旧表单向左移动 1/5 宽度（视差效果）
            cocos2d::Node *lastui = UINode->getChildren().back();
            lastui->runAction(EaseQuadraticActionOut::create(
                MoveTo::create(UI_CHANGE_DURATION,
                               Vec2(size.width / -5, 0))));
            // 半透明遮罩：从全透明渐变到 128
            cocos2d::Node *ColorMask = MaskLayer::create(
                Color4B(0, 0, 0, 0), size.width, size.height);
            ColorMask->setPosition(Vec2(-size.width, 0));
            ui->addChild(ColorMask);
            ColorMask->runAction(
                FadeTo::create(UI_CHANGE_DURATION, 128));
            // 新表单从右侧滑入
            ui->setPosition(size.width, 0);
            ui->runAction(EaseQuadraticActionOut::create(
                MoveTo::create(UI_CHANGE_DURATION, Vec2::ZERO)));
            // 动画结束后移除遮罩
            runAction(Sequence::createWithTwoActions(
                DelayTime::create(UI_CHANGE_DURATION),
                CallFunc::create([=]() {
                    ColorMask->removeFromParent();
                })));
        }
        UINode->addChild(ui);
    } else if(ani == eEnterFromBottom) {
        // 从底部弹出（用于浮动面板）
        cocos2d::Size size = UINode->getContentSize();
        cocos2d::Node *ColorMask = MaskLayer::create(
            Color4B(0, 0, 0, 0), size.width, size.height);
        ColorMask->runAction(
            FadeTo::create(UI_CHANGE_DURATION, 128));
        ui->setPositionY(-ui->getContentSize().height);
        ColorMask->addChild(ui);
        UINode->addChild(ColorMask);
        ui->runAction(EaseQuadraticActionOut::create(
            MoveTo::create(UI_CHANGE_DURATION, Vec2::ZERO)));
    }
}
```

三种动画模式对比：

| 动画类型 | 枚举值 | 视觉效果 | 典型场景 |
|----------|--------|----------|---------|
| `eEnterAniNone` | 0 | 无动画 | 初始表单 |
| `eEnterAniOverFromRight` | 1 | 新表单从右滑入，旧表单左移 1/5 + 遮罩 | 页面导航（最常用） |
| `eEnterFromBottom` | 2 | 从底部弹出 + 背景遮罩 | 浮动面板 |

### popUIForm：配对的离开动画

```cpp
// 文件：cpp/core/environ/cocos2d/MainScene.cpp 第 1874-1924 行
void TVPMainScene::popUIForm(cocos2d::Node *form, eLeaveAni ani) {
    int n = UINode->getChildrenCount();
    if(n <= 0) return;
    if(n == 1) TVPControlAdDialog(0x10002, 0, 0);

    auto children = UINode->getChildren();
    if(ani == eLeaveAniNone) {
        if(n > 1) {
            Node *lastui = children.at(n - 2);
            lastui->setPosition(0, 0);   // 恢复下层表单位置
        }
        children.back()->removeFromParent();
    } else if(ani == eLeaveAniLeaveFromLeft) {
        Node *ui = children.back();
        cocos2d::Size size = UINode->getContentSize();
        if(n > 1) {
            Node *lastui = children.at(n - 2);
            lastui->setPosition(size.width / -5, 0);
            lastui->runAction(EaseQuadraticActionOut::create(
                MoveTo::create(UI_CHANGE_DURATION, Vec2::ZERO)));
        }
        // 遮罩淡出 + 当前表单向右滑出
        cocos2d::Node *ColorMask = MaskLayer::create(
            Color4B(0, 0, 0, 128), size.width, size.height);
        ColorMask->setPosition(Vec2(-size.width, 0));
        ui->addChild(ColorMask);
        ColorMask->runAction(FadeOut::create(UI_CHANGE_DURATION));
        ui->runAction(EaseQuadraticActionOut::create(
            MoveTo::create(UI_CHANGE_DURATION,
                           Vec2(size.width, 0))));
        this->runAction(Sequence::createWithTwoActions(
            DelayTime::create(UI_CHANGE_DURATION),
            CallFunc::create([ui] {
                ui->removeFromParent();   // 动画完成后移除
            })));
    } else if(ani == eLeaveToBottom) {
        cocos2d::Node *ColorMask = children.back();
        ColorMask->runAction(
            FadeOut::create(UI_CHANGE_DURATION));
        Node *ui = ColorMask->getChildren().at(0);
        ui->runAction(EaseQuadraticActionIn::create(
            MoveTo::create(UI_CHANGE_DURATION,
                Vec2(0, -ui->getContentSize().height))));
        runAction(Sequence::createWithTwoActions(
            DelayTime::create(UI_CHANGE_DURATION),
            CallFunc::create([=]() {
                ColorMask->removeFromParent();
            })));
    }
}
```

动画持续时间统一为 `UI_CHANGE_DURATION = 0.3f` 秒。缓动函数使用 `EaseQuadraticActionOut`（先快后慢）和 `EaseQuadraticActionIn`（先慢后快），符合 Material Design 运动规范。

## 常见错误与解决方案

### 错误 1：CSB 文件路径找不到

```
[ERROR] Fail to load ui file: ui/MyCustomForm.csb
```

**原因**：CSB 文件没有放在搜索路径下。Cocos2d-x 的 `FileUtils` 按搜索路径列表查找文件。

**解决方案**：
```cpp
// 检查当前搜索路径
auto paths = FileUtils::getInstance()->getSearchPaths();
for(auto &p : paths) CCLOG("SearchPath: %s", p.c_str());

// 确保 CSB 文件在 ui/cocos-studio/ 目录下
// KrKr2 在 AppDelegate 中添加了搜索路径
FileUtils::getInstance()->addSearchPath("ui/cocos-studio");
```

### 错误 2：findController 返回 nullptr 导致崩溃

```
[FATAL] Node btn_submit not exist in ui/MyForm.csb
```

**原因**：CSB 文件中的节点名称与代码中的字符串不匹配。

**解决方案**：
```cpp
// 使用 notice=false 安全查找，避免弹窗和崩溃
auto *btn = reader.findWidget("btn_submit", false);
if(btn) {
    btn->addClickEventListener([](cocos2d::Ref *) {
        CCLOG("Clicked!");
    });
} else {
    CCLOG("btn_submit not found, skip binding");
}
```

### 错误 3：表单动画叠加导致视觉异常

**现象**：快速连续 push/pop 表单时，动画叠加产生闪烁或位置错误。

**原因**：`UI_CHANGE_DURATION = 0.3f` 秒内连续操作，前一个动画未完成就开始新动画。

**解决方案**：KrKr2 通过 `stopAllActions()` 和延迟 `CallFunc` 处理。如需严格互斥：
```cpp
bool _isAnimating = false;
void pushUIForm(Node *ui, eEnterAni ani) {
    if(_isAnimating) return;       // 动画期间拒绝操作
    _isAnimating = true;
    // ... 执行动画 ...
    runAction(Sequence::createWithTwoActions(
        DelayTime::create(UI_CHANGE_DURATION),
        CallFunc::create([this]() {
            _isAnimating = false;  // 动画完成解锁
        })));
}
```

### 实践 1：创建自定义表单

基于 `iTVPBaseForm` 创建一个"关于"对话框：

// AboutForm.h — 完整可编译示例

#include "BaseForm.h"

class AboutForm : public iTVPBaseForm {
public:
    static AboutForm *create() {
        auto *ret = new AboutForm;
        ret->initUI();
        ret->autorelease();
        return ret;
    }

private:
    void initUI() {
        initFromFile(
            Csd::createNaviBarWithMenu,   // NaviBar
            [](const cocos2d::Size &size, float scale)
                -> cocos2d::ui::Widget * {

                body->setBackGroundColorType(
                    cocos2d::ui::Layout::BackGroundColorType::SOLID);
                body->setBackGroundColor(cocos2d::Color3B(30, 30, 30));
                body->setBackGroundColorOpacity(255);
                // 居中文本
                auto *label = cocos2d::ui::Text::create(
                    "KrKr2 Emulator v1.0\n"
                    "Cross-platform KiriKiri2 Engine",
                    "NotoSansCJK-Regular.ttc", 28 * scale);
                label->setName("about_text");
                label->setPosition(cocos2d::Vec2(
                    size.width / 2, size.height / 2));
                body->addChild(label);
                return body;
            },
            Csd::createEmpty   // 无 Footer
        );
    }

    void bindHeaderController(const Node *) override {
        NaviBar.Left->addClickEventListener([this](cocos2d::Ref *) {
            TVPMainScene::GetInstance()->popUIForm(
                this, TVPMainScene::eLeaveAniLeaveFromLeft);
        });
    }
    void bindBodyController(const Node *) override {}
    void bindFooterController(const Node *) override {}

void loadAndBindUI() {

        });

    if(cancelBtn) {
        cancelBtn->addClickEventListener([root](cocos2d::Ref *) {
            root->removeFromParent();  // 关闭对话框
        });
    }

    // 设置标题文本
    if(titleLabel) {
        titleLabel->setString("My Custom Dialog");
    }

    // 添加到当前场景
    this->addChild(root);
}

## 本节小结

- **CSBReader** 继承 `NodeMap`，在加载 `.csb` 的同时建立名称→节点的扁平哈希索引，实现 O(1) 按名称查找
- **iTVPBaseForm** 强制约束三段式布局（NaviBar 10% + Body 80% + Footer 10%），子类通过三个 `bind*Controller` 纯虚函数绑定具体控件
- **pushUIForm/popUIForm** 实现了带动画的表单栈：从右滑入/向左滑出用于页面导航，从底部弹出/收起用于浮动面板
- **Form 继承体系**包含 12+ 个子类，覆盖偏好设置、文件选择、游戏菜单、调试视图等所有 UI 场景
- `NodeBuilderFn` 工厂函数模式让 CSB 加载和纯代码构建可以混用，提供最大灵活性

## 练习题与答案

### 题目 1：NodeMap 的扁平索引有什么限制？如果两个不同层级的子节点同名会发生什么？

<details>
<summary>查看答案</summary>

`NodeMap` 使用 `unordered_map<string, Node*>` 存储索引。`initFromNode` 采用深度优先递归遍历，如果两个不同层级的节点同名，后遍历到的会覆盖先前的。

具体行为：`getChildren()` 返回按添加顺序排列的子节点，递归遍历先处理当前节点再递归进入子树。因此同名时，层级更深或更靠后的节点会覆盖之前的。

```cpp
// 验证实验
NodeMap map;
auto *root = Node::create();
auto *child1 = Node::create();
child1->setName("btn");          // 第一个 "btn"
auto *child2 = Node::create();
child2->setName("btn");          // 第二个 "btn"
root->addChild(child1);
root->addChild(child2);
map.initFromNode(root);
// map["btn"] → child2（后遍历覆盖了 child1）
```

这是 O(1) 查找效率与命名唯一性之间的设计取舍。项目通过 Cocos Studio 设计规范保证节点命名唯一。

</details>

### 题目 2：为什么 `pushUIForm` 的 `eEnterAniOverFromRight` 要把旧表单移动 -1/5 宽度而不是完全移出屏幕？

<details>
<summary>查看答案</summary>

这是**视差效果（Parallax Effect）**——新表单从右侧完整滑入（移动整个屏幕宽度），旧表单只向左移动 1/5 宽度。这种速度差产生深度感——新表单移动快（在前面），旧表单移动慢（在后面），营造"覆盖"而非"替换"的视觉效果。

同时旧表单被半透明遮罩覆盖（`FadeTo(128)`），进一步强化层次关系。如果旧表单也完全移出，看起来像两个表单同时平移，缺乏层次感。

```
视差时间线（0.3s）：
t=0.0s  旧表单 x=0          新表单 x=width
t=0.15s 旧表单 x=-width/10  新表单 x=width/2
t=0.3s  旧表单 x=-width/5   新表单 x=0
```

这种模式与 iOS/Android 原生页面导航动画一致。

</details>

### 题目 3：`iTVPBaseForm` 的 `initFromFile` 为什么使用 `NodeBuilderFn` 函数而不是直接接收 `Node*` 参数？

<details>
<summary>查看答案</summary>

使用工厂函数而非直接传入节点有三个核心优势：

1. **延迟构建**：节点在 `initFromFile` 内部才创建，此时 `parent` 的尺寸信息已确定，工厂函数能接收到精确的目标尺寸和缩放比，实现自适应布局。

2. **统一接口**：CSB 加载和纯代码构建共享同一个 `NodeBuilderFn` 签名。传 CSB 加载的 lambda、传 `CsdUIFactory` 的静态函数、或传自定义构建逻辑，调用方式完全一致：

```cpp
// CSB 加载
initFromFile(
    [](const Size &s, float sc) -> Widget* {
        CSBReader r; return dynamic_cast<Widget*>(r.Load("ui/NaviBar.csb"));
    },
    bodyBuilder,
    Csd::createEmpty       // 纯代码的空构建函数
);

// 纯代码构建
initFromFile(
    Csd::createNaviBarWithMenu,   // CsdUIFactory 静态函数
    bodyBuilder,
    Csd::createEmpty
);
```

3. **可选区域**：传 `Csd::createEmpty`（返回 nullptr）即可跳过某个区域，比如不需要 Footer 的表单。如果直接传 `Node*`，调用方必须自己处理 nullptr 的构建逻辑。

</details>
