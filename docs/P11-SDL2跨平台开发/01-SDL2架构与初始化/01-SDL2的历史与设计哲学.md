## SDL2 是什么

### 一句话定义

SDL2（Simple DirectMedia Layer 2）是一个用 **C 语言**编写的跨平台多媒体库，它在操作系统的底层 API 之上提供了一层统一的抽象，让开发者用一套代码就能在多个平台上访问窗口管理、图形渲染、音频输出、输入设备（键盘、鼠标、触摸屏、手柄）等硬件能力。

### 它解决了什么问题

假设你要在 Windows 上创建一个窗口，你需要调用 Win32 API：

```cpp
// Windows 上创建窗口 —— 需要大量平台特定代码
#include <windows.h>

LRESULT CALLBACK WindowProc(HWND hwnd, UINT uMsg,
                            WPARAM wParam, LPARAM lParam) {
    switch (uMsg) {
    case WM_DESTROY:
        PostQuitMessage(0);  // 收到销毁消息，退出消息循环
        return 0;
    case WM_PAINT: {
        PAINTSTRUCT ps;
        HDC hdc = BeginPaint(hwnd, &ps);  // 获取设备上下文
        FillRect(hdc, &ps.rcPaint,
                 (HBRUSH)(COLOR_WINDOW + 1));  // 填充背景
        EndPaint(hwnd, &ps);
        return 0;
    }
    }
    return DefWindowProc(hwnd, uMsg, wParam, lParam);
}

int WINAPI WinMain(HINSTANCE hInst, HINSTANCE, LPSTR, int) {
    // 注册窗口类
    WNDCLASS wc = {};
    wc.lpfnWndProc = WindowProc;
    wc.hInstance = hInst;
    wc.lpszClassName = L"MyWindowClass";
    RegisterClass(&wc);

    // 创建窗口
    HWND hwnd = CreateWindowEx(
        0, L"MyWindowClass", L"Hello Windows",
        WS_OVERLAPPEDWINDOW, CW_USEDEFAULT, CW_USEDEFAULT,
        800, 600, NULL, NULL, hInst, NULL);
    ShowWindow(hwnd, SW_SHOW);

    // 消息循环
    MSG msg;
    while (GetMessage(&msg, NULL, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }
    return 0;
}
```

在 Linux 上做同样的事情，你需要完全不同的 X11 代码：

```cpp
// Linux 上创建窗口 —— 需要 X11 API
#include <X11/Xlib.h>

int main() {
    Display* display = XOpenDisplay(NULL);  // 连接 X 服务器
    int screen = DefaultScreen(display);

    // 创建窗口
    Window window = XCreateSimpleWindow(
        display, RootWindow(display, screen),
        10, 10, 800, 600, 1,
        BlackPixel(display, screen),    // 边框颜色
        WhitePixel(display, screen));   // 背景颜色

    XSelectInput(display, window,
                 ExposureMask | KeyPressMask);  // 选择事件
    XMapWindow(display, window);  // 显示窗口

    // 事件循环
    XEvent event;
    while (true) {
        XNextEvent(display, &event);  // 阻塞等待事件
        if (event.type == KeyPress) break;
    }

    XCloseDisplay(display);
    return 0;
}
```

macOS 又是另一套（Cocoa/AppKit），Android 更是完全不同（JNI + Activity 生命周期）。如果你要支持四个平台，就要写四套窗口管理代码，再加上四套输入处理、四套音频输出……

**SDL2 的价值就在于：它帮你写好了这四套（实际上更多，SDL2 支持 20+ 个平台）代码，然后给你一个统一的 C API。** 用 SDL2 创建窗口只需要几行代码：

```cpp
// 用 SDL2 创建窗口 —— 同样的代码在所有平台上运行
#include <SDL.h>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);  // 初始化视频子系统

    // 创建窗口 —— 一个函数调用搞定
    SDL_Window* window = SDL_CreateWindow(
        "Hello SDL2",                    // 窗口标题
        SDL_WINDOWPOS_CENTERED,          // x 坐标（居中）
        SDL_WINDOWPOS_CENTERED,          // y 坐标（居中）
        800, 600,                        // 宽度和高度
        SDL_WINDOW_SHOWN);               // 窗口标志

    // 事件循环
    SDL_Event event;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&event)) {  // 非阻塞轮询事件
            if (event.type == SDL_QUIT) {
                running = false;         // 用户关闭窗口
            }
        }
    }

    SDL_DestroyWindow(window);  // 销毁窗口
    SDL_Quit();                 // 清理 SDL
    return 0;
}
```

这段代码在 Windows、Linux、macOS、Android 上都能编译运行，完全不需要修改。

### SDL2 不做什么

理解 SDL2 的定位，同样重要的是知道它 **不做什么**：

| SDL2 提供 | SDL2 不提供 |
|-----------|------------|
| 窗口创建和管理 | 3D 渲染引擎（它只帮你创建 OpenGL 上下文，但不提供 3D 场景管理） |
| 事件循环和输入处理 | GUI 组件库（没有按钮、文本框、菜单等 UI 控件） |
| OpenGL 上下文创建 | 游戏引擎功能（没有物理引擎、粒子系统、动画系统） |
| 基本的 2D 渲染（SDL_Renderer） | 高级 2D 渲染（没有精灵表、骨骼动画、粒子效果） |
| 音频设备访问 | 音频解码器（不能直接播放 MP3/OGG，需要 SDL_mixer 扩展库或自己解码） |
| 手柄/触摸/键盘统一输入 | 网络通信（需要 SDL_net 扩展库或其他网络库） |
| 文件 I/O 抽象 | 数据库访问 |
| 定时器 | 脚本语言集成 |

这就是"薄抽象层"的含义——SDL2 只在操作系统和你的应用程序之间放了一层很薄的胶水，让你能跨平台访问硬件，但不替你做任何更高层的事情。

## SDL 的发展历史

### SDL 1.2 时代（1998-2012）

SDL 最初由 Sam Lantinga（后来成为 Valve 员工）在 1998 年创建，目的是让游戏开发者能更容易地将游戏从 Windows 移植到 Linux。SDL 1.2 的核心设计非常简洁：

- 一个窗口就是一个全屏 **Surface**（一块 CPU 内存中的像素缓冲区）
- 所有渲染都是软件渲染（CPU 画像素）
- 不直接支持 OpenGL 硬件加速（需要额外设置）
- 每个应用只能有一个窗口

SDL 1.2 在 Linux 游戏社区非常流行，许多知名游戏和模拟器都使用它，包括《文明：权力的召唤》、DOSBox、ZSNES 等。

### SDL2 的诞生（2013 至今）

2013 年，SDL2 发布，这是一次彻底的重写，带来了现代化的架构：

| 特性 | SDL 1.2 | SDL2 |
|------|---------|------|
| 硬件加速 | 无（纯软件渲染） | 内置 SDL_Renderer（底层用 OpenGL/D3D/Metal） |
| 多窗口 | 不支持 | 支持多个窗口 |
| OpenGL 支持 | 基础 | 完整（含 OpenGL ES 2.0/3.0） |
| 手柄支持 | 基础的 Joystick API | Game Controller API（标准化映射） |
| 触摸屏 | 不支持 | 完整的多点触控支持 |
| Android/iOS | 不支持 | 完整支持 |
| 音频 | 回调式，固定格式 | 支持多种格式，可混音 |
| 线程安全 | 有限 | 大部分 API 线程安全 |

SDL2 的架构设计是 KrKr2 项目选择它的关键原因：它支持移动平台（Android），支持 OpenGL ES，而且足够轻量，不会与 Cocos2d-x 渲染框架冲突。

### SDL3 的展望

SDL3 于 2024 年末进入预览阶段，带来了 GPU API 抽象（SDL_gpu，类似 WebGPU 的现代图形 API）、更好的属性系统、更一致的命名约定等改进。但 KrKr2 目前使用的是 SDL2，因为 SDL3 仍在迭代中，且项目的 vcpkg 依赖已经配置好了 SDL2。

## SDL2 的设计哲学

### 用 C 语言编写

SDL2 使用纯 C（C99 标准）编写，这个选择有深刻的技术原因：

1. **ABI（Application Binary Interface，应用程序二进制接口）稳定性** — C 语言的 ABI 在所有主流平台上都是标准化的，任何语言都能通过 FFI（Foreign Function Interface，外部函数接口）调用 C 函数。如果用 C++ 编写，名称修饰（Name Mangling）会导致不同编译器编译的二进制不兼容
2. **最大兼容性** — C 编译器在所有嵌入式、移动、桌面、主机平台上都可用，而 C++ 编译器在某些平台上可能受限或版本老旧
3. **语言绑定容易** — SDL2 有超过 50 种语言的绑定（Python、Rust、Go、C#、Java 等），这正是因为 C API 最容易被其他语言调用

这对 KrKr2 项目来说是个优势——KrKr2 是 C++17 项目，调用 C 语言的 SDL2 API 完全没有障碍，只需 `#include <SDL.h>` 即可。

### 子系统架构

SDL2 将功能划分为多个独立的 **子系统（Subsystem）**，每个子系统可以单独初始化和销毁：

```cpp
// 只初始化你需要的子系统
SDL_Init(SDL_INIT_VIDEO | SDL_INIT_AUDIO);  // 只初始化视频和音频

// 稍后按需初始化更多子系统
SDL_InitSubSystem(SDL_INIT_GAMECONTROLLER);  // 需要手柄时再初始化

// 可以单独销毁某个子系统
SDL_QuitSubSystem(SDL_INIT_AUDIO);  // 不再需要音频

// 最终清理所有子系统
SDL_Quit();
```

SDL2 的子系统包括：

| 子系统标志 | 功能 | 典型用途 |
|-----------|------|---------|
| `SDL_INIT_VIDEO` | 窗口管理、OpenGL 上下文、屏幕信息 | 几乎所有应用都需要 |
| `SDL_INIT_AUDIO` | 音频设备访问、PCM 数据播放 | 游戏/多媒体应用 |
| `SDL_INIT_EVENTS` | 事件队列（被 VIDEO 自动初始化） | 通常不需要单独初始化 |
| `SDL_INIT_TIMER` | 高精度定时器 | 帧率控制、延时 |
| `SDL_INIT_JOYSTICK` | 手柄/摇杆原始输入 | 低级手柄访问 |
| `SDL_INIT_GAMECONTROLLER` | 标准化手柄映射 | 推荐用于手柄支持 |
| `SDL_INIT_HAPTIC` | 力反馈/振动 | 手柄震动效果 |
| `SDL_INIT_SENSOR` | 加速度计、陀螺仪 | 移动设备传感器 |
| `SDL_INIT_EVERYTHING` | 初始化所有子系统 | 快速原型开发 |

这种按需初始化的设计有两个好处：

1. **启动速度** — 如果你的应用不需要音频，就不初始化音频子系统，省去音频设备探测的时间
2. **权限控制** — 在 Android 等移动平台上，初始化某些子系统可能需要特定权限，按需初始化可以避免请求不必要的权限

### 平台后端架构

SDL2 的内部实现采用编译时（Compile-time）平台选择策略。每个子系统在不同平台上有不同的 **后端（Backend）** 实现：

```
SDL2 公共 API（SDL_CreateWindow、SDL_PollEvent 等）
        │
        ▼
┌─────────────────────────────┐
│    SDL2 平台抽象层            │
├──────┬──────┬──────┬────────┤
│Win32 │ X11  │Cocoa │Android │  ← 编译时选择
│后端  │后端  │后端  │ 后端   │
│      │Wayland│     │        │
└──────┴──────┴──────┴────────┘
        │
        ▼
   操作系统原生 API
```

编译 SDL2 时，CMake 会检测当前平台并只编译对应的后端代码。这意味着在 Windows 上编译的 SDL2 库只包含 Win32 后端，在 Linux 上编译的只包含 X11/Wayland 后端。

## SDL2 与同类库的对比

在选择跨平台窗口/输入库时，开发者通常会考虑以下几个选项：

| 特性 | SDL2 | GLFW | SFML | Qt |
|------|------|------|------|-----|
| **语言** | C | C | C++ | C++ |
| **定位** | 多媒体抽象层 | OpenGL 窗口管理 | 多媒体库 | 应用框架 |
| **窗口管理** | ✅ | ✅ | ✅ | ✅ |
| **音频** | ✅ | ❌ | ✅ | ✅ |
| **手柄** | ✅（含标准映射） | ✅（基础） | ✅ | ❌ |
| **触摸屏** | ✅ | ❌ | ❌ | ✅ |
| **Android/iOS** | ✅ | ❌ | ❌ | ✅ |
| **2D 渲染** | ✅（SDL_Renderer） | ❌ | ✅（内置） | ✅ |
| **GUI 组件** | ❌ | ❌ | ❌ | ✅ |
| **库大小** | 中等（~2-5MB） | 小（~1MB） | 中等 | 大（~50MB+） |
| **许可证** | zlib（非常宽松） | zlib | zlib | LGPL/商业 |

KrKr2 选择 SDL2 而非 GLFW 或 SFML 的原因：

1. **Android 支持** — GLFW 和 SFML 都不支持 Android，而 KrKr2 的核心目标之一是 Android 平台
2. **不与 Cocos2d-x 冲突** — SDL2 作为薄抽象层，不会试图接管渲染管线，能和 Cocos2d-x 共存
3. **手柄支持** — SDL2 的 Game Controller API 提供了标准化的手柄映射，对视觉小说游戏的手柄操作很有用
4. **成熟稳定** — SDL2 被大量商业游戏使用（包括 Valve 的 Steam Deck），久经考验

### 步骤 1：使用 vcpkg 安装 SDL2

如果你已经按照 [P02 — vcpkg 包管理](../../P02-vcpkg包管理/) 配置好了 vcpkg，安装 SDL2 非常简单：

```bash

vcpkg search sdl2

# 安装 SDL2（vcpkg 会自动选择当前平台的 triplet）
vcpkg install sdl2
```

### 步骤 2：创建 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.16)
project(SDL2Hello LANGUAGES CXX)

# 查找 SDL2 包（vcpkg 会自动配置查找路径）
find_package(SDL2 CONFIG REQUIRED)

add_executable(sdl2_hello main.cpp)

# 链接 SDL2
# SDL2::SDL2 是动态库版本
# SDL2::SDL2-static 是静态库版本
# SDL2::SDL2main 提供平台入口点（Windows 上将 WinMain 转换为 main）
target_link_libraries(sdl2_hello PRIVATE
    SDL2::SDL2
    SDL2::SDL2main)
```

### 步骤 3：编写 main.cpp

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 初始化视频子系统
    if (SDL_Init(SDL_INIT_VIDEO) < 0) {
        printf("SDL 初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 打印 SDL2 版本信息
    SDL_version compiled;
    SDL_version linked;
    SDL_VERSION(&compiled);            // 编译时版本
    SDL_GetVersion(&linked);           // 运行时版本

    printf("SDL2 编译时版本: %d.%d.%d\n",
           compiled.major, compiled.minor, compiled.patch);
    printf("SDL2 运行时版本: %d.%d.%d\n",
           linked.major, linked.minor, linked.patch);

    // 列出可用的视频驱动
    int numDrivers = SDL_GetNumVideoDrivers();
    printf("\n可用的视频驱动 (%d 个):\n", numDrivers);
    for (int i = 0; i < numDrivers; i++) {
        printf("  [%d] %s\n", i, SDL_GetVideoDriver(i));
    }

    // 打印当前使用的视频驱动
    printf("\n当前视频驱动: %s\n", SDL_GetCurrentVideoDriver());

    // 列出显示器信息
    int numDisplays = SDL_GetNumVideoDisplays();
    printf("\n检测到 %d 个显示器:\n", numDisplays);
    for (int i = 0; i < numDisplays; i++) {
        SDL_DisplayMode mode;
        SDL_GetCurrentDisplayMode(i, &mode);
        printf("  显示器 %d: %dx%d @ %dHz\n",
               i, mode.w, mode.h, mode.refresh_rate);
    }

    SDL_Quit();
    return 0;
}
```

## 常见错误及解决方案

### 错误 1：`SDL.h: No such file or directory`

**原因：** CMake 没有找到 SDL2 的头文件路径。

**解决方案：**

```bash
# 确保 vcpkg 工具链文件路径正确
cmake -B build -S . \
    -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake

# 或者检查 vcpkg 是否已安装 SDL2
vcpkg list | grep sdl2
```

### 错误 2：链接时 `undefined reference to SDL_main`

**原因：** SDL2 在 Windows 上会将 `main` 重定义为 `SDL_main`，需要链接 `SDL2::SDL2main`。

**解决方案：** 确保 CMakeLists.txt 中链接了 `SDL2::SDL2main`：

```cmake
target_link_libraries(sdl2_hello PRIVATE
    SDL2::SDL2
    SDL2::SDL2main)  # 不要忘记这行
```

### 错误 3：运行时找不到 SDL2.dll

**原因：** 使用动态链接时，SDL2.dll 不在可执行文件旁边。

**解决方案：** 用静态链接替代，或者把 DLL 复制到输出目录：

```cmake
# 使用静态链接（推荐，避免 DLL 问题）
target_link_libraries(sdl2_hello PRIVATE
    SDL2::SDL2-static  # 换成静态版本
    SDL2::SDL2main)
```

## 本节小结

- SDL2 是一个用 C 语言编写的跨平台多媒体抽象库，核心功能是提供统一的窗口管理、事件处理和多媒体硬件访问接口
- SDL2 采用"薄抽象层"设计哲学——只做最少量的封装，不替开发者做更高层的决定
- SDL2 按子系统划分功能，支持按需初始化，内部采用编译时平台后端选择
- 相比 GLFW（不支持移动端和音频）和 SFML（不支持移动端），SDL2 的跨平台覆盖最广
- KrKr2 选择 SDL2 是因为它支持 Android、能与 Cocos2d-x 共存、手柄支持完善且足够轻量

## 练习题与答案

### 题目 1：SDL2 的子系统初始化

假设你需要开发一个不需要音频的图形应用程序，但需要支持手柄输入。请写出正确的 SDL2 初始化代码，只初始化必要的子系统。

<details>
<summary>查看答案</summary>

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 只初始化视频（窗口管理）和手柄子系统
    // 不初始化 SDL_INIT_AUDIO，因为不需要音频
    if (SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER) < 0) {
        printf("SDL 初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    printf("视频和手柄子系统初始化成功\n");

    // 检查哪些子系统已初始化
    if (SDL_WasInit(SDL_INIT_VIDEO)) {
        printf("视频子系统: 已初始化\n");
    }
    if (SDL_WasInit(SDL_INIT_AUDIO)) {
        printf("音频子系统: 已初始化\n");
    } else {
        printf("音频子系统: 未初始化（按计划）\n");
    }
    if (SDL_WasInit(SDL_INIT_GAMECONTROLLER)) {
        printf("手柄子系统: 已初始化\n");
    }

    SDL_Quit();
    return 0;
}
```

**关键点：** 使用位或运算符 `|` 组合多个子系统标志。`SDL_WasInit()` 可以检查某个子系统是否已经初始化。不初始化 `SDL_INIT_AUDIO` 可以避免在 Android 上请求不必要的音频权限。

</details>

### 题目 2：SDL2 vs GLFW 选型分析

你的团队正在开发一个跨平台应用，需要支持 Windows、Linux 和 Android，需要 OpenGL 渲染和手柄输入，但不需要音频功能。在 SDL2 和 GLFW 之间你会选择哪个？请给出理由。

<details>
<summary>查看答案</summary>

**应选择 SDL2**，理由如下：

1. **Android 支持（决定性因素）** — GLFW 不支持 Android 平台。如果需求包含 Android，GLFW 直接被排除。SDL2 原生支持 Android，包括触摸屏输入和 OpenGL ES 上下文创建。

2. **手柄支持更完善** — SDL2 的 Game Controller API 提供标准化的手柄映射（将不同品牌的手柄按键映射为统一的 Xbox 风格布局），而 GLFW 只提供基础的 Joystick API，需要开发者自己处理手柄映射。

3. **不需要音频也不影响** — 虽然 GLFW 更轻量（因为没有音频功能），但 SDL2 的子系统是按需初始化的，不初始化 `SDL_INIT_AUDIO` 就不会有任何音频相关的开销。

4. **唯一可能选 GLFW 的场景** — 如果只需要 Windows + Linux + macOS（不需要 Android），且只需要窗口和 OpenGL 上下文（不需要手柄），那么 GLFW 更简洁。但本题明确要求 Android 和手柄支持，所以 SDL2 是唯一的选择。

</details>

### 题目 3：阅读 SDL2 初始化代码并找出问题

以下代码有两个问题，请找出它们：

```cpp
#include <SDL.h>

int main() {
    SDL_Init(SDL_INIT_VIDEO);
    SDL_Window* window = SDL_CreateWindow("Test",
        0, 0, 800, 600, SDL_WINDOW_SHOWN);
    // ... 使用窗口 ...
    SDL_Quit();
    return 0;
}
```

<details>
<summary>查看答案</summary>

**问题 1：`main` 函数签名不正确**

SDL2 在某些平台上（特别是 Windows）会通过宏将 `main` 重定义为 `SDL_main`，要求 `main` 函数的签名必须是 `int main(int argc, char* argv[])`。省略参数可能导致链接错误。

**问题 2：没有检查 `SDL_Init` 的返回值**

`SDL_Init` 在初始化失败时会返回负数，但代码没有检查返回值。如果初始化失败（例如在无图形界面的服务器上），后续的 `SDL_CreateWindow` 调用会导致未定义行为。

**问题 3（额外）：没有检查 `SDL_CreateWindow` 的返回值**

如果窗口创建失败，`window` 会是 `NULL`，但代码没有检查。

**修正后的代码：**

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {  // 修正：完整的 main 签名
    if (SDL_Init(SDL_INIT_VIDEO) < 0) {  // 修正：检查返回值
        printf("SDL 初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    SDL_Window* window = SDL_CreateWindow("Test",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED,
        800, 600, SDL_WINDOW_SHOWN);
    if (!window) {  // 修正：检查窗口创建结果
        printf("窗口创建失败: %s\n", SDL_GetError());
        SDL_Quit();
        return 1;
    }

    // ... 使用窗口 ...

    SDL_DestroyWindow(window);  // 修正：先销毁窗口
    SDL_Quit();
    return 0;
}
```

</details>
