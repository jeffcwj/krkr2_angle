# 子系统详解与初始化流程

> **所属模块：** P11-SDL2 跨平台开发
> **前置知识：** [01-SDL2 的历史与设计哲学](./01-SDL2的历史与设计哲学.md)
> **预计阅读时间：** 25 分钟

## 本节目标

读完本节后，你将能够：

1. **说出 SDL2 的全部子系统** — 知道每个子系统负责什么功能，何时需要初始化它
2. **正确编写初始化和清理代码** — 掌握 `SDL_Init`、`SDL_InitSubSystem`、`SDL_QuitSubSystem`、`SDL_Quit` 的完整用法和错误处理
3. **理解子系统之间的依赖关系** — 知道哪些子系统会自动初始化其他子系统，以及这对初始化顺序的影响
4. **应用按需初始化的最佳实践** — 能在实际项目中根据功能需求选择最小化的子系统组合，避免不必要的资源消耗和权限请求

## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 子系统 | Subsystem | SDL2 内部按功能划分的独立模块，每个模块管理一类硬件资源（如视频、音频） |
| 初始化标志 | Init Flag | 传给 `SDL_Init()` 的位掩码常量（如 `SDL_INIT_VIDEO`），用于指定要初始化哪些子系统 |
| 位掩码 | Bitmask | 用二进制位来表示多个开关状态的编程技巧，通过按位或 `\|` 组合多个标志 |
| 引用计数 | Reference Counting | SDL2 内部记录每个子系统被初始化了多少次，只有当计数降为零时才真正释放资源 |
| 隐式依赖 | Implicit Dependency | 某些子系统在初始化时会自动初始化它依赖的其他子系统（如 VIDEO 自动初始化 EVENTS） |
| SDL_GetError | — | SDL2 的错误查询函数，返回最后一次 SDL 调用失败时的错误描述字符串 |

## SDL2 子系统全景图

SDL2 将所有功能划分为 **9 个子系统**，每个子系统由一个 `SDL_INIT_*` 常量标识。下面这张表列出了所有子系统及其对应的功能范围：

| 子系统标志 | 十六进制值 | 功能范围 | 典型使用场景 |
|-----------|-----------|---------|-------------|
| `SDL_INIT_TIMER` | `0x00000001` | 高精度定时器（`SDL_GetTicks`、`SDL_Delay`、`SDL_AddTimer`） | 帧率控制、延时等待、定时回调 |
| `SDL_INIT_AUDIO` | `0x00000010` | 音频设备打开/关闭、PCM 数据播放和录制 | 游戏音效、背景音乐、语音播放 |
| `SDL_INIT_VIDEO` | `0x00000020` | 窗口创建/管理、OpenGL 上下文、显示器信息、剪贴板 | 几乎所有图形应用程序 |
| `SDL_INIT_JOYSTICK` | `0x00000200` | 手柄/摇杆的原始输入（按钮、轴、帽键） | 需要手柄原始数据的场景 |
| `SDL_INIT_HAPTIC` | `0x00001000` | 力反馈/振动设备控制 | 手柄震动效果 |
| `SDL_INIT_GAMECONTROLLER` | `0x00002000` | 标准化手柄映射（Xbox 风格布局） | 推荐的手柄支持方式 |
| `SDL_INIT_EVENTS` | `0x00004000` | 事件队列管理（`SDL_PollEvent`、`SDL_PushEvent`） | 事件驱动程序 |
| `SDL_INIT_SENSOR` | `0x00008000` | 加速度计、陀螺仪等传感器 | 移动设备体感操作 |
| `SDL_INIT_EVERYTHING` | `0x0000F231` | 以上所有子系统的组合 | 快速原型、不关心性能/权限时 |

> **注意：** `SDL_INIT_EVERYTHING` 的值是上面所有标志的按位或结果（`0x00000001 | 0x00000010 | 0x00000020 | 0x00000200 | 0x00001000 | 0x00002000 | 0x00004000 | 0x00008000 = 0x0000F231`）。它会初始化所有可用的子系统。

### 位掩码组合原理

SDL2 使用位掩码来允许你在一次调用中初始化多个子系统。每个 `SDL_INIT_*` 常量占据不同的二进制位，因此可以用按位或运算符 `|` 任意组合：

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 方式 1：一次性初始化多个子系统
    // 使用按位或 | 组合多个标志
    Uint32 flags = SDL_INIT_VIDEO | SDL_INIT_AUDIO | SDL_INIT_TIMER;
    if (SDL_Init(flags) < 0) {
        printf("SDL 初始化失败: %s\n", SDL_GetError());
        return 1;
    }
    printf("视频 + 音频 + 定时器初始化成功\n");

    // 方式 2：先初始化一部分，后面按需追加
    // SDL_InitSubSystem 可以在 SDL_Init 之后追加新的子系统
    if (SDL_InitSubSystem(SDL_INIT_GAMECONTROLLER) < 0) {
        printf("手柄子系统初始化失败: %s\n", SDL_GetError());
        // 注意：这里只是手柄初始化失败，视频和音频仍然正常
        // 根据业务需求决定是否继续运行
    } else {
        printf("手柄子系统追加初始化成功\n");
    }

    SDL_Quit();
    return 0;
}
```

这段代码演示了两种初始化策略：一次性初始化和分步追加。在实际项目中，分步追加更常见——例如在进入手柄设置界面时才初始化手柄子系统。

## 各子系统详解

### TIMER 子系统（`SDL_INIT_TIMER`）

定时器子系统提供三类功能：

1. **时间查询** — `SDL_GetTicks()` 返回 SDL 初始化以来经过的毫秒数，`SDL_GetPerformanceCounter()` 提供更高精度的时间戳
2. **延时等待** — `SDL_Delay(ms)` 让当前线程休眠指定毫秒数（精度取决于操作系统调度器）
3. **定时回调** — `SDL_AddTimer()` 注册一个在指定间隔后执行的回调函数

```cpp
#include <SDL.h>
#include <cstdio>

// SDL_AddTimer 的回调函数
// interval: 上次回调到这次回调的间隔（毫秒）
// param:    传递给 SDL_AddTimer 的用户数据指针
// 返回值:   下一次回调的间隔（毫秒），返回 0 表示取消定时器
Uint32 timerCallback(Uint32 interval, void* param) {
    int* count = static_cast<int*>(param);
    (*count)++;
    printf("[定时器] 第 %d 次触发，间隔 %u ms\n", *count, interval);

    if (*count >= 5) {
        printf("[定时器] 达到 5 次，停止定时器\n");
        return 0;  // 返回 0 取消定时器
    }
    return interval;  // 返回原间隔，继续定时
}

int main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_TIMER) < 0) {
        printf("定时器初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 获取启动时间
    Uint32 startTicks = SDL_GetTicks();
    printf("SDL 启动时间戳: %u ms\n", startTicks);

    // 高精度计时器
    Uint64 perfFreq = SDL_GetPerformanceFrequency();
    Uint64 perfStart = SDL_GetPerformanceCounter();
    printf("高精度计时器频率: %llu Hz\n",
           static_cast<unsigned long long>(perfFreq));

    // 注册定时回调（每 500ms 触发一次）
    int callCount = 0;
    SDL_TimerID timer = SDL_AddTimer(500, timerCallback, &callCount);
    if (timer == 0) {
        printf("定时器创建失败: %s\n", SDL_GetError());
        SDL_Quit();
        return 1;
    }

    // 等待定时器完成（5 次 × 500ms = 2.5 秒）
    SDL_Delay(3000);

    // 测量经过的精确时间
    Uint64 perfEnd = SDL_GetPerformanceCounter();
    double elapsed = static_cast<double>(perfEnd - perfStart)
                     / static_cast<double>(perfFreq);
    printf("高精度测量: 经过了 %.6f 秒\n", elapsed);

    SDL_Quit();
    return 0;
}
```

**跨平台差异：**
- **Windows** — `SDL_GetPerformanceCounter` 使用 `QueryPerformanceCounter`，精度通常在微秒级
- **Linux** — 使用 `clock_gettime(CLOCK_MONOTONIC)`，精度为纳秒级
- **macOS** — 使用 `mach_absolute_time()`，精度为纳秒级
- **Android** — 使用 `clock_gettime(CLOCK_MONOTONIC)`，但实际精度可能因设备而异

> **注意：** `SDL_Delay()` 只保证休眠 **至少** 指定的毫秒数，实际可能更久。在 Windows 上默认调度精度是 15.6ms，SDL2 会在内部调用 `timeBeginPeriod(1)` 来提高到 1ms 精度，但这也只是尽力而为。不要用 `SDL_Delay` 做精确的帧率控制——应使用 `SDL_GetPerformanceCounter` 配合忙等待或 VSync。

### AUDIO 子系统（`SDL_INIT_AUDIO`）

音频子系统负责打开音频设备并以 PCM（Pulse Code Modulation，脉冲编码调制——将声波数字化为离散采样点的标准方式）格式播放或录制音频数据。

```cpp
#include <SDL.h>
#include <cmath>
#include <cstdio>

// 音频回调函数 —— SDL2 在需要音频数据时调用
// userdata: 传递给 SDL_OpenAudioDevice 的用户数据
// stream:   需要填充的音频缓冲区
// len:      缓冲区长度（字节）
void audioCallback(void* userdata, Uint8* stream, int len) {
    static double phase = 0.0;
    double* frequency = static_cast<double*>(userdata);

    // 生成正弦波
    // len / 2 是因为每个采样点是 16 位（2 字节）
    Sint16* samples = reinterpret_cast<Sint16*>(stream);
    int sampleCount = len / sizeof(Sint16);

    for (int i = 0; i < sampleCount; i++) {
        // 生成 440Hz 正弦波（A4 音符）
        samples[i] = static_cast<Sint16>(
            sin(phase) * 16000);  // 振幅 16000（最大 32767）
        phase += 2.0 * M_PI * (*frequency) / 44100.0;

        // 防止 phase 值无限增大导致浮点精度下降
        if (phase > 2.0 * M_PI) {
            phase -= 2.0 * M_PI;
        }
    }
}

int main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_AUDIO) < 0) {
        printf("音频初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 列出可用的音频设备
    int numDevices = SDL_GetNumAudioDevices(0);  // 0 = 播放设备
    printf("可用的音频播放设备 (%d 个):\n", numDevices);
    for (int i = 0; i < numDevices; i++) {
        printf("  [%d] %s\n", i, SDL_GetAudioDeviceName(i, 0));
    }

    // 配置音频参数
    SDL_AudioSpec desired = {};
    desired.freq = 44100;           // 采样率 44100Hz（CD 音质）
    desired.format = AUDIO_S16SYS;  // 16 位有符号整数，系统字节序
    desired.channels = 1;           // 单声道
    desired.samples = 1024;         // 缓冲区大小（采样点数）
    desired.callback = audioCallback;

    double frequency = 440.0;       // A4 音符频率
    desired.userdata = &frequency;

    SDL_AudioSpec obtained;
    // 打开默认音频设备（设备名传 NULL）
    SDL_AudioDeviceID device = SDL_OpenAudioDevice(
        NULL, 0, &desired, &obtained, 0);
    if (device == 0) {
        printf("打开音频设备失败: %s\n", SDL_GetError());
        SDL_Quit();
        return 1;
    }

    printf("\n实际音频参数:\n");
    printf("  采样率: %d Hz\n", obtained.freq);
    printf("  声道数: %d\n", obtained.channels);
    printf("  缓冲区: %d 采样点\n", obtained.samples);

    // 开始播放（设备默认是暂停状态）
    SDL_PauseAudioDevice(device, 0);  // 0 = 取消暂停 = 开始播放
    printf("\n正在播放 440Hz 正弦波，持续 2 秒...\n");
    SDL_Delay(2000);

    // 停止并关闭
    SDL_CloseAudioDevice(device);
    SDL_Quit();
    return 0;
}
```

**平台注意事项：**
- **Android** — 初始化 `SDL_INIT_AUDIO` 需要应用具有 `RECORD_AUDIO` 权限（如果要录音的话）。纯播放不需要额外权限，但初始化过程会触发 Android 的音频焦点（Audio Focus）机制
- **Linux** — SDL2 支持 PulseAudio、ALSA、JACK 等多种音频后端，通过环境变量 `SDL_AUDIODRIVER` 可以强制选择：`export SDL_AUDIODRIVER=pulseaudio`
- **macOS** — 使用 Core Audio 后端，延迟通常最低
- **Windows** — 默认使用 WASAPI 后端（Windows Audio Session API），也支持 DirectSound 后端

### VIDEO 子系统（`SDL_INIT_VIDEO`）

视频子系统是最常用的子系统，负责：

- 窗口创建和管理（`SDL_CreateWindow`、`SDL_SetWindowSize` 等）
- OpenGL/OpenGL ES 上下文创建（`SDL_GL_CreateContext`）
- 显示器信息查询（分辨率、刷新率、DPI）
- 剪贴板操作（`SDL_GetClipboardText`、`SDL_SetClipboardText`）
- 消息框（`SDL_ShowSimpleMessageBox`）

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // VIDEO 子系统会自动初始化 EVENTS 子系统
    if (SDL_Init(SDL_INIT_VIDEO) < 0) {
        printf("视频初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 查询视频驱动信息
    printf("当前视频驱动: %s\n", SDL_GetCurrentVideoDriver());

    // 查询显示器信息
    int numDisplays = SDL_GetNumVideoDisplays();
    printf("检测到 %d 个显示器:\n", numDisplays);
    for (int i = 0; i < numDisplays; i++) {
        SDL_DisplayMode mode;
        SDL_GetCurrentDisplayMode(i, &mode);

        // 获取显示器名称
        const char* name = SDL_GetDisplayName(i);
        printf("  [%d] %s: %dx%d @ %dHz\n",
               i, name ? name : "未知", mode.w, mode.h,
               mode.refresh_rate);

        // 获取 DPI 信息（在高 DPI 屏幕上很重要）
        float ddpi, hdpi, vdpi;
        if (SDL_GetDisplayDPI(i, &ddpi, &hdpi, &vdpi) == 0) {
            printf("      DPI: 对角线=%.1f 水平=%.1f 垂直=%.1f\n",
                   ddpi, hdpi, vdpi);
        }

        // 获取可用显示区域（排除任务栏等）
        SDL_Rect usable;
        SDL_GetDisplayUsableBounds(i, &usable);
        printf("      可用区域: x=%d y=%d w=%d h=%d\n",
               usable.x, usable.y, usable.w, usable.h);
    }

    // 列出所有支持的显示模式
    int numModes = SDL_GetNumDisplayModes(0);  // 第一个显示器
    printf("\n显示器 0 支持的显示模式 (%d 种):\n", numModes);
    for (int i = 0; i < numModes && i < 5; i++) {  // 只显示前 5 个
        SDL_DisplayMode mode;
        SDL_GetDisplayMode(0, i, &mode);
        printf("  [%d] %dx%d @ %dHz (格式: 0x%X)\n",
               i, mode.w, mode.h, mode.refresh_rate, mode.format);
    }
    if (numModes > 5) {
        printf("  ... 还有 %d 种模式\n", numModes - 5);
    }

    SDL_Quit();
    return 0;
}
```

> **重要：** 初始化 `SDL_INIT_VIDEO` 时，SDL2 会 **自动初始化** `SDL_INIT_EVENTS` 子系统。这是一个隐式依赖——视频子系统需要事件队列来传递窗口事件（如关闭、调整大小、焦点变化等）。你不需要也不应该再单独初始化 `SDL_INIT_EVENTS`。

### JOYSTICK 子系统（`SDL_INIT_JOYSTICK`）

手柄/摇杆子系统提供对游戏手柄的 **原始访问**——直接读取按钮状态、轴值、帽键（Hat，手柄上的方向键/十字键，返回上下左右等方向状态）方向。数据没有经过标准化处理，不同品牌的手柄按钮编号和轴编号可能完全不同。

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 初始化 JOYSTICK 子系统
    // 注意：JOYSTICK 会自动初始化 EVENTS
    if (SDL_Init(SDL_INIT_JOYSTICK) < 0) {
        printf("手柄初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    int numJoysticks = SDL_NumJoysticks();
    printf("检测到 %d 个手柄/摇杆:\n", numJoysticks);

    for (int i = 0; i < numJoysticks; i++) {
        printf("  [%d] %s\n", i, SDL_JoystickNameForIndex(i));
        printf("      GUID: ");
        SDL_JoystickGUID guid = SDL_JoystickGetDeviceGUID(i);
        char guidStr[33];
        SDL_JoystickGetGUIDString(guid, guidStr, sizeof(guidStr));
        printf("%s\n", guidStr);
    }

    // 打开第一个手柄（如果有的话）
    if (numJoysticks > 0) {
        SDL_Joystick* joystick = SDL_JoystickOpen(0);
        if (joystick) {
            printf("\n已打开手柄: %s\n", SDL_JoystickName(joystick));
            printf("  按钮数: %d\n", SDL_JoystickNumButtons(joystick));
            printf("  轴数:   %d\n", SDL_JoystickNumAxes(joystick));
            printf("  帽键数: %d\n", SDL_JoystickNumHats(joystick));
            printf("  球数:   %d\n", SDL_JoystickNumBalls(joystick));

            // 读取 3 秒的输入
            printf("\n读取手柄输入 3 秒...\n");
            Uint32 start = SDL_GetTicks();
            SDL_Event event;
            while (SDL_GetTicks() - start < 3000) {
                while (SDL_PollEvent(&event)) {
                    switch (event.type) {
                    case SDL_JOYBUTTONDOWN:
                        printf("  按钮 %d 按下\n",
                               event.jbutton.button);
                        break;
                    case SDL_JOYAXISMOTION:
                        // 轴值范围: -32768 到 32767
                        if (abs(event.jaxis.value) > 8000) {
                            printf("  轴 %d = %d\n",
                                   event.jaxis.axis,
                                   event.jaxis.value);
                        }
                        break;
                    }
                }
                SDL_Delay(16);
            }

            SDL_JoystickClose(joystick);
        }
    }

    SDL_Quit();
    return 0;
}
```

### GAMECONTROLLER 子系统（`SDL_INIT_GAMECONTROLLER`）

Game Controller 子系统建立在 Joystick 之上，提供 **标准化的手柄映射**。它将不同品牌手柄的按钮/轴映射为统一的 Xbox 风格布局（A/B/X/Y、左/右摇杆、左/右扳机等），开发者不需要为每款手柄编写不同的映射代码。

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // GAMECONTROLLER 会自动初始化 JOYSTICK 和 EVENTS
    if (SDL_Init(SDL_INIT_GAMECONTROLLER) < 0) {
        printf("初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 检查哪些手柄支持 Game Controller 映射
    int numJoysticks = SDL_NumJoysticks();
    printf("检测到 %d 个输入设备:\n", numJoysticks);
    for (int i = 0; i < numJoysticks; i++) {
        if (SDL_IsGameController(i)) {
            printf("  [%d] %s (支持标准映射)\n",
                   i, SDL_GameControllerNameForIndex(i));
        } else {
            printf("  [%d] %s (仅支持原始 Joystick)\n",
                   i, SDL_JoystickNameForIndex(i));
        }
    }

    // 打开第一个支持映射的手柄
    SDL_GameController* controller = NULL;
    for (int i = 0; i < numJoysticks; i++) {
        if (SDL_IsGameController(i)) {
            controller = SDL_GameControllerOpen(i);
            if (controller) {
                printf("\n已打开手柄: %s\n",
                       SDL_GameControllerName(controller));
                // 打印映射字符串
                const char* mapping =
                    SDL_GameControllerMapping(controller);
                if (mapping) {
                    printf("映射: %s\n", mapping);
                }
                break;
            }
        }
    }

    if (controller) {
        printf("\n标准化按钮名称示例:\n");
        printf("  SDL_CONTROLLER_BUTTON_A — Xbox A / PS ×\n");
        printf("  SDL_CONTROLLER_BUTTON_B — Xbox B / PS ○\n");
        printf("  SDL_CONTROLLER_BUTTON_X — Xbox X / PS □\n");
        printf("  SDL_CONTROLLER_BUTTON_Y — Xbox Y / PS △\n");

        printf("\n等待输入 3 秒...\n");
        Uint32 start = SDL_GetTicks();
        SDL_Event event;
        while (SDL_GetTicks() - start < 3000) {
            while (SDL_PollEvent(&event)) {
                if (event.type == SDL_CONTROLLERBUTTONDOWN) {
                    // 获取按钮的标准名称
                    const char* btnName =
                        SDL_GameControllerGetStringForButton(
                            static_cast<SDL_GameControllerButton>(
                                event.cbutton.button));
                    printf("  按下: %s\n",
                           btnName ? btnName : "未知");
                }
            }
            SDL_Delay(16);
        }

        SDL_GameControllerClose(controller);
    }

    SDL_Quit();
    return 0;
}
```

**Joystick vs Game Controller 的选择建议：**

| 场景 | 推荐使用 |
|------|---------|
| 需要支持各种品牌手柄并统一映射 | Game Controller |
| 需要访问手柄上的非标准按钮 | Joystick |
| 需要支持飞行摇杆、方向盘等特殊设备 | Joystick |
| 一般游戏开发 | Game Controller |
| KrKr2 视觉小说的手柄操作 | Game Controller |

### HAPTIC 子系统（`SDL_INIT_HAPTIC`）

力反馈/振动子系统让你能控制手柄的震动马达。在视觉小说引擎中不常用，但某些游戏可能需要：

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_JOYSTICK | SDL_INIT_HAPTIC) < 0) {
        printf("初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 检查力反馈设备数量
    int numHaptics = SDL_NumHaptics();
    printf("检测到 %d 个力反馈设备:\n", numHaptics);
    for (int i = 0; i < numHaptics; i++) {
        printf("  [%d] %s\n", i, SDL_HapticName(i));
    }

    // 如果有手柄，尝试通过手柄打开力反馈
    if (SDL_NumJoysticks() > 0) {
        SDL_Joystick* joy = SDL_JoystickOpen(0);
        if (joy && SDL_JoystickIsHaptic(joy)) {
            SDL_Haptic* haptic = SDL_HapticOpenFromJoystick(joy);
            if (haptic) {
                // 简单的震动效果（rumble）
                if (SDL_HapticRumbleInit(haptic) == 0) {
                    printf("开始震动 1 秒...\n");
                    SDL_HapticRumblePlay(haptic, 0.75f, 1000);
                    // 0.75f = 75% 强度，1000 = 持续 1 秒
                    SDL_Delay(1000);
                    SDL_HapticRumbleStop(haptic);
                }
                SDL_HapticClose(haptic);
            }
        }
        if (joy) SDL_JoystickClose(joy);
    }

    SDL_Quit();
    return 0;
}
```

### SENSOR 子系统（`SDL_INIT_SENSOR`）

传感器子系统主要用于移动设备，提供加速度计和陀螺仪数据。在桌面平台上通常没有传感器可用：

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    if (SDL_Init(SDL_INIT_SENSOR) < 0) {
        printf("传感器初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    int numSensors = SDL_NumSensors();
    printf("检测到 %d 个传感器:\n", numSensors);
    for (int i = 0; i < numSensors; i++) {
        printf("  [%d] %s (类型: %d)\n",
               i, SDL_SensorGetDeviceName(i),
               SDL_SensorGetDeviceType(i));
    }

    if (numSensors == 0) {
        printf("当前平台无可用传感器（桌面系统通常如此）\n");
    }

    SDL_Quit();
    return 0;
}
```

### EVENTS 子系统（`SDL_INIT_EVENTS`）

事件子系统是 SDL2 的核心基础设施，管理一个线程安全的事件队列。大多数情况下你不需要单独初始化它，因为 `SDL_INIT_VIDEO` 和 `SDL_INIT_JOYSTICK` 都会自动初始化事件子系统。

只有在极少数场景下（例如你需要事件机制但不需要任何窗口或手柄）才需要单独初始化：

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 只初始化事件子系统（罕见用法）
    if (SDL_Init(SDL_INIT_EVENTS) < 0) {
        printf("事件初始化失败: %s\n", SDL_GetError());
        return 1;
    }

    // 推送一个自定义事件
    SDL_Event userEvent = {};
    userEvent.type = SDL_USEREVENT;
    userEvent.user.code = 42;
    userEvent.user.data1 = nullptr;
    userEvent.user.data2 = nullptr;

    if (SDL_PushEvent(&userEvent) < 0) {
        printf("推送事件失败: %s\n", SDL_GetError());
    } else {
        printf("成功推送自定义事件 (code=42)\n");
    }

    // 读取事件
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        if (event.type == SDL_USEREVENT) {
            printf("收到自定义事件: code=%d\n", event.user.code);
        }
    }

    SDL_Quit();
    return 0;
}
```

## 本节小结

- SDL2 将功能划分为 9 个子系统（TIMER、AUDIO、VIDEO、JOYSTICK、HAPTIC、GAMECONTROLLER、EVENTS、SENSOR），通过位掩码标志组合初始化
- 每个子系统管理一类硬件资源，可以按需单独初始化和销毁
- TIMER 提供高精度计时和定时回调，AUDIO 提供 PCM 音频设备访问，VIDEO 提供窗口和 OpenGL 上下文
- JOYSTICK 提供手柄原始输入，GAMECONTROLLER 在其上添加标准化映射，推荐优先使用 GAMECONTROLLER
- HAPTIC 控制振动马达，SENSOR 访问移动设备传感器，EVENTS 管理线程安全的事件队列
- 各子系统在四个平台（Windows/Linux/macOS/Android）上的后端实现不同，但 SDL2 API 保持一致

### 题目 1：子系统功能匹配

以下应用场景分别需要初始化哪些子系统？请给出最小化的初始化标志组合。

A. 一个只需要播放音频的命令行工具（无窗口）
B. 一个需要窗口显示和键盘输入的图形程序
C. 一个需要窗口、手柄输入和振动反馈的游戏

<details>
<summary>查看答案</summary>

**A:** `SDL_Init(SDL_INIT_AUDIO)`

只需要音频子系统。不需要窗口，也不需要事件循环（音频通过回调函数工作）。

**B:** `SDL_Init(SDL_INIT_VIDEO)`

只需要显式初始化 VIDEO。VIDEO 会自动初始化 EVENTS 子系统，而键盘输入通过 EVENTS 的事件队列获取，所以不需要额外初始化任何东西。

**C:** `SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER | SDL_INIT_HAPTIC)`

需要显式初始化三个子系统。GAMECONTROLLER 会自动初始化 JOYSTICK 和 EVENTS，VIDEO 也会自动初始化 EVENTS（引用计数为 2），HAPTIC 独立初始化。

</details>

### 题目 2：分析音频回调代码

阅读本节中 AUDIO 子系统的正弦波生成代码，回答以下问题：

1. 如果把 `desired.samples` 从 1024 改为 256，音频播放的延迟会怎么变化？
2. 为什么代码中要检查 `phase > 2.0 * M_PI` 并减去 `2.0 * M_PI`？如果不做这个检查会怎样？
3. 如果要生成立体声（双声道）音频，需要修改哪些参数？

<details>
<summary>查看答案</summary>

**1.** 延迟会降低。`samples` 参数指定音频缓冲区的大小（采样点数）。缓冲区越小，SDL2 调用回调函数的频率越高，音频数据更快到达音频设备，延迟更低。但缓冲区太小（如 64）可能导致音频卡顿（CPU 来不及填充缓冲区）。
- 1024 采样点 ÷ 44100Hz = 约 23ms 延迟
- 256 采样点 ÷ 44100Hz = 约 5.8ms 延迟

**2.** `phase` 变量是正弦波的相位角。随着时间推移，`phase` 会不断累加。如果不做取模处理，`phase` 的值会趋向无穷大，导致浮点精度下降（`double` 在极大值时无法精确表示小的增量）。最终正弦波的频率会出现漂移或产生杂音。减去 `2π` 保证 `phase` 始终在 `[0, 2π)` 范围内，维持浮点精度。

**3.** 需要修改：
```cpp
desired.channels = 2;  // 改为双声道
```
同时修改回调函数，交替填充左右声道的数据：
```cpp
for (int i = 0; i < sampleCount; i += 2) {
    Sint16 sample = static_cast<Sint16>(sin(phase) * 16000);
    samples[i]     = sample;  // 左声道
    samples[i + 1] = sample;  // 右声道（同样的数据 = 单声道模拟）
    phase += 2.0 * M_PI * (*frequency) / 44100.0;
    if (phase > 2.0 * M_PI) phase -= 2.0 * M_PI;
}
```

</details>
