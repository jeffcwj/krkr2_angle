## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 分层初始化 | Layered Initialization | 把子系统分成"必须成功"的核心层和"失败也能降级"的可选层，分两步初始化 |

| 降级运行 | Graceful Degradation | 当某个非必要功能不可用时，程序跳过它继续运行，而非直接崩溃 |

## 分层初始化策略

上一节介绍了单个子系统的初始化和错误处理。但在真实项目中，你通常需要同时初始化多个子系统，而且不同子系统的重要程度不同。例如：

- **VIDEO + EVENTS**：没有这俩就没法显示窗口、接收输入——必须成功，否则程序根本没法运行
- **AUDIO**：没有声音可以凑合（静音模式），但不是致命错误
- **GAMECONTROLLER**：手柄支持是加分项，没有也不影响键鼠操作
- **HAPTIC**（触觉反馈，即控制器/设备的振动功能）：纯粹的锦上添花

把子系统按重要程度分层，就是"分层初始化"的核心思想。

### KrKr2 项目的初始化策略

KrKr2 项目本身并不直接调用 `SDL_Init()`——这个职责交给了 Cocos2d-x 框架。Cocos2d-x 在其内部初始化渲染引擎时，会自动初始化 SDL2 的 VIDEO 和 EVENTS 子系统（这两个是 Cocos2d-x 创建窗口和事件循环的基础）。KrKr2 引擎的代码只在 Cocos2d-x 之上处理自己的逻辑（脚本引擎、图层系统、音视频解码等），不干涉底层的 SDL2 初始化。

这是一种典型的职责分离设计：框架管底层（SDL2、OpenGL），引擎管上层（游戏逻辑、脚本执行）。我们在后面的"对照项目源码"环节会详细看这段代码。

### 能力结构体设计

在实现分层初始化之前，我们需要一个结构来记录"哪些子系统初始化成功了"。这样后续代码可以根据这个能力记录来决定是否使用某个功能：

```cpp
#include <SDL.h>
#include <cstdio>

// 记录各子系统的可用状态
// 初始化后通过这个结构体查询功能是否可用
struct SDLCapabilities {
    bool hasVideo;           // 视频子系统（窗口、渲染）
    bool hasAudio;           // 音频子系统
    bool hasGameController;  // 游戏控制器（手柄）
    bool hasHaptic;          // 触觉反馈（振动）
    bool hasSensor;          // 传感器（加速度计、陀螺仪等）
    bool hasTimer;           // 定时器
};
```

每个字段对应一个子系统。初始化时设为 `true` 或 `false`，后续代码就可以这样写：

```cpp
// 只在音频可用时播放音效
if (capabilities.hasAudio) {
    playSound("click.wav");
} else {
    printf("音频不可用，跳过音效播放\n");
}
```

### 完整的分层初始化函数

下面是将分层思想落地的完整代码。核心层失败时直接返回错误（调用方应退出程序），可选层失败时只记录警告并继续：

```cpp
#include <SDL.h>
#include <cstdio>
#include <cstring>

struct SDLCapabilities {
    bool hasVideo;
    bool hasAudio;
    bool hasGameController;
    bool hasHaptic;
    bool hasSensor;
    bool hasTimer;
};

// 尝试初始化单个子系统，返回是否成功
// subsystemName 用于日志输出，让错误信息可读
static bool tryInitSubSystem(Uint32 flag, const char* subsystemName) {
    if (SDL_InitSubSystem(flag) < 0) {
        printf("[WARN] 子系统 %s 初始化失败: %s\n",
               subsystemName, SDL_GetError());
        return false;
    }
    printf("[OK]   子系统 %s 初始化成功\n", subsystemName);
    return true;
}

// 分层初始化 SDL2
// 返回 true 表示核心子系统全部就绪，可以运行
// 返回 false 表示核心子系统缺失，应退出程序
// capabilities 会被填充为各子系统的实际可用状态
bool initSDLLayered(SDLCapabilities& caps) {
    // 清零所有能力标记
    memset(&caps, 0, sizeof(caps));

    // ========================================
    // 第 1 层：核心子系统（全部必须成功）
    // ========================================

    // VIDEO 是最基本的——没有它就没法创建窗口
    // SDL_Init 会自动拉起 EVENTS（VIDEO 的依赖）
    if (SDL_Init(SDL_INIT_VIDEO) < 0) {
        printf("[FATAL] VIDEO 子系统初始化失败: %s\n", SDL_GetError());
        printf("        没有 VIDEO 子系统，程序无法运行。\n");
        return false;
    }
    caps.hasVideo = true;
    printf("[OK]   核心子系统 VIDEO + EVENTS 初始化成功\n");

    // TIMER 用于帧率控制和延时
    if (!tryInitSubSystem(SDL_INIT_TIMER, "TIMER")) {
        printf("[FATAL] TIMER 子系统是核心依赖，程序无法运行。\n");
        SDL_Quit();  // 清理已初始化的 VIDEO
        return false;
    }
    caps.hasTimer = true;

    // ========================================
    // 第 2 层：可选子系统（失败不影响核心功能）
    // ========================================

    // AUDIO：没有声音照样能运行，只是静音
    caps.hasAudio = tryInitSubSystem(SDL_INIT_AUDIO, "AUDIO");

    // GAMECONTROLLER：手柄支持，自动拉起 JOYSTICK + EVENTS
    // 注意：EVENTS 已经在 VIDEO 初始化时拉起，这里只是增加引用计数
    caps.hasGameController = tryInitSubSystem(
        SDL_INIT_GAMECONTROLLER, "GAMECONTROLLER");

    // HAPTIC：触觉反馈（控制器振动）
    // 在没有触觉设备的环境（如无手柄的桌面系统）会失败，这完全正常
    caps.hasHaptic = tryInitSubSystem(SDL_INIT_HAPTIC, "HAPTIC");

    // SENSOR：加速度计、陀螺仪等
    // 桌面系统通常没有传感器，移动设备通常有
    caps.hasSensor = tryInitSubSystem(SDL_INIT_SENSOR, "SENSOR");

    return true;
}

int main(int argc, char* argv[]) {
    SDLCapabilities caps;

    if (!initSDLLayered(caps)) {
        printf("核心初始化失败，程序退出。\n");
        return 1;
    }

    // 打印能力摘要
    printf("\n===== SDL2 能力摘要 =====\n");
    printf("视频:   %s\n", caps.hasVideo ? "可用" : "不可用");
    printf("音频:   %s\n", caps.hasAudio ? "可用" : "不可用");
    printf("手柄:   %s\n", caps.hasGameController ? "可用" : "不可用");
    printf("振动:   %s\n", caps.hasHaptic ? "可用" : "不可用");
    printf("传感器: %s\n", caps.hasSensor ? "可用" : "不可用");
    printf("定时器: %s\n", caps.hasTimer ? "可用" : "不可用");
    printf("=========================\n");

    // 后续代码根据 caps 决定是否使用某功能
    // ...

    SDL_Quit();
    return 0;
}
```

```
[OK]   子系统 AUDIO 初始化成功
```

## 动手实践：子系统探测工具

让我们把分层初始化的思想进一步扩展，编写一个完整的"子系统探测工具"。这个工具不仅检测子系统是否可用，还会输出每个子系统的依赖关系信息，帮助你理解当前运行环境的 SDL2 能力全貌。

### 探测工具完整代码

```cpp
#include <SDL.h>
#include <cstdio>

// 子系统信息结构
// 每个条目描述一个子系统的名称、初始化标志、依赖说明
struct SubsystemInfo {
    const char* name;        // 子系统显示名称
    Uint32 flag;             // SDL_INIT_* 标志
    const char* description; // 用途说明
    const char* dependency;  // 依赖关系说明
};

int main(int argc, char* argv[]) {
    printf("====================================\n");
    printf("  SDL2 子系统探测工具\n");
    printf("====================================\n\n");

    // 打印 SDL2 版本信息
    SDL_version compiled;  // 编译时链接的 SDL2 版本
    SDL_version linked;    // 运行时加载的 SDL2 动态库版本
    SDL_VERSION(&compiled);
    SDL_GetVersion(&linked);
    printf("编译时 SDL 版本: %d.%d.%d\n",
           compiled.major, compiled.minor, compiled.patch);
    printf("运行时 SDL 版本: %d.%d.%d\n",
           linked.major, linked.minor, linked.patch);
    printf("视频驱动: %s\n", SDL_GetCurrentVideoDriver()
                              ? SDL_GetCurrentVideoDriver()
                              : "(未初始化)");
    printf("音频驱动: %s\n", SDL_GetCurrentAudioDriver()
                              ? SDL_GetCurrentAudioDriver()
                              : "(未初始化)");
    printf("\n");

    // 定义所有要探测的子系统
    SubsystemInfo subsystems[] = {
        {"TIMER",          SDL_INIT_TIMER,
         "定时器与延时",    "无依赖"},
        {"AUDIO",          SDL_INIT_AUDIO,
         "音频播放与录制",  "无依赖"},
        {"VIDEO",          SDL_INIT_VIDEO,
         "窗口与渲染",     "自动拉起 EVENTS"},
        {"JOYSTICK",       SDL_INIT_JOYSTICK,
         "摇杆输入",       "自动拉起 EVENTS"},
        {"HAPTIC",         SDL_INIT_HAPTIC,
         "触觉反馈(振动)", "无依赖"},
        {"GAMECONTROLLER", SDL_INIT_GAMECONTROLLER,
         "游戏手柄",       "自动拉起 JOYSTICK → EVENTS"},
        {"EVENTS",         SDL_INIT_EVENTS,
         "事件队列",       "无依赖（被 VIDEO/JOYSTICK 依赖）"},
        {"SENSOR",         SDL_INIT_SENSOR,
         "传感器输入",     "无依赖"},
    };
    int count = sizeof(subsystems) / sizeof(subsystems[0]);

    // 逐个探测
    printf("%-18s %-8s %-20s %s\n",
           "子系统", "状态", "用途", "依赖关系");
    printf("--------------------------------------------------------------\n");

    int successCount = 0;
    int failCount = 0;

    for (int i = 0; i < count; i++) {
        // 每次探测前先完全清理，确保各子系统独立测试
        SDL_Quit();

        bool ok = (SDL_InitSubSystem(subsystems[i].flag) == 0);
        printf("%-18s %-8s %-20s %s\n",
               subsystems[i].name,
               ok ? "✓ 可用" : "✗ 不可用",
               subsystems[i].description,
               subsystems[i].dependency);

        if (ok) {
            successCount++;

            // 对 VIDEO 子系统，额外输出驱动信息
            if (subsystems[i].flag == SDL_INIT_VIDEO) {
                int numDrivers = SDL_GetNumVideoDrivers();
                printf("  └─ 可用视频驱动(%d个):", numDrivers);
                for (int d = 0; d < numDrivers; d++) {
                    printf(" %s", SDL_GetVideoDriver(d));
                }
                printf("\n  └─ 当前视频驱动: %s\n",
                       SDL_GetCurrentVideoDriver());
            }

            // 对 AUDIO 子系统，额外输出驱动信息
            if (subsystems[i].flag == SDL_INIT_AUDIO) {
                int numDrivers = SDL_GetNumAudioDrivers();
                printf("  └─ 可用音频驱动(%d个):", numDrivers);
                for (int d = 0; d < numDrivers; d++) {
                    printf(" %s", SDL_GetAudioDriver(d));
                }
                printf("\n  └─ 当前音频驱动: %s\n",
                       SDL_GetCurrentAudioDriver());
            }
        } else {
            failCount++;
            printf("  └─ 错误: %s\n", SDL_GetError());
        }
    }

    // 汇总报告
    printf("\n====================================\n");
    printf("  探测结果: %d 可用 / %d 不可用 / %d 总计\n",
           successCount, failCount, count);
    printf("====================================\n");

    SDL_Quit();
    return 0;
}
```

### 编译与运行

下面给出四个平台的编译命令。这个工具只依赖 SDL2，不需要其他库。

**CMakeLists.txt：**

```cmake
cmake_minimum_required(VERSION 3.16)
project(sdl2_probe LANGUAGES C CXX)

# 查找 SDL2——vcpkg、系统包管理器安装的都能找到
find_package(SDL2 REQUIRED)

add_executable(sdl2_probe main.cpp)

# SDL2 的 CMake 包在不同平台提供的 target 名称可能不同
# SDL2::SDL2 是最通用的写法（vcpkg 和大多数发行版都支持）
target_link_libraries(sdl2_probe PRIVATE SDL2::SDL2 SDL2::SDL2main)
```

**Windows（需要 vcpkg 或手动安装 SDL2）：**

```bash
cmake -B build -G Ninja -DCMAKE_TOOLCHAIN_FILE=%VCPKG_ROOT%/scripts/buildsystems/vcpkg.cmake
cmake --build build
.\build\sdl2_probe.exe
```

**Linux（Ubuntu/Debian，通过系统包安装 SDL2）：**

```bash
# 安装 SDL2 开发库
sudo apt install libsdl2-dev

cmake -B build -G Ninja
cmake --build build
./build/sdl2_probe
```

**macOS（通过 Homebrew 安装 SDL2）：**

```bash
brew install sdl2
cmake -B build -G Ninja
cmake --build build
./build/sdl2_probe
```

**Android（在 KrKr2 项目中 SDL2 通过 vcpkg 管理）：**

Android 上 SDL2 不能直接运行命令行程序，但你可以将探测逻辑嵌入到 Activity 的 `onCreate()` 中，通过 `__android_log_print()` 输出到 Logcat：

```cpp
// 替换 printf 为 LOGI 即可
```

## 本节小结

- **分层初始化**将子系统分为核心层（VIDEO、EVENTS、TIMER——失败即退出）和可选层（AUDIO、GAMECONTROLLER、HAPTIC、SENSOR——失败则降级），通过 `SDLCapabilities` 结构体记录各子系统的实际可用状态
- **能力结构体** `SDLCapabilities` 让后续代码可以安全地查询某功能是否可用，而不是假设所有子系统都初始化成功
- **子系统探测工具**可以在运行时检测当前平台支持哪些子系统并输出驱动信息，帮助排查跨平台兼容性问题
- 探测工具通过 `SDL_GetNumVideoDrivers()` / `SDL_GetNumAudioDrivers()` 列出所有可用驱动，配合 `SDL_GetCurrentVideoDriver()` / `SDL_GetCurrentAudioDriver()` 显示当前选中的驱动

## 练习题与答案

### 题目 1：分层策略设计

假设你在开发一个同时支持键鼠和手柄的 SDL2 游戏。请设计你的分层初始化方案：
1. 哪些子系统属于核心层？
2. 哪些子系统属于可选层？
3. 如果 AUDIO 初始化失败，你的程序应该如何处理？

<details>
<summary>查看答案</summary>

**1. 核心层子系统：**
- `SDL_INIT_VIDEO`——游戏必须有画面
- `SDL_INIT_EVENTS`——必须能接收输入（VIDEO 会自动拉起 EVENTS，不需要单独初始化）
- `SDL_INIT_TIMER`——帧率控制和游戏逻辑计时需要

**2. 可选层子系统：**
- `SDL_INIT_AUDIO`——没有声音游戏照样能玩（静音模式）
- `SDL_INIT_GAMECONTROLLER`——手柄支持是加分项，键鼠可以兜底
- `SDL_INIT_HAPTIC`——手柄振动是锦上添花
- `SDL_INIT_JOYSTICK`——如果不用 GAMECONTROLLER 而用原始摇杆 API，也属于可选

**3. AUDIO 失败的处理方案：**

```cpp
caps.hasAudio = tryInitSubSystem(SDL_INIT_AUDIO, "AUDIO");
if (!caps.hasAudio) {
    // 记录日志，通知用户（可选）
    printf("[INFO] 音频不可用，游戏将以静音模式运行。\n");
    // 设置一个全局标志，让音频相关的函数跳过执行
```
