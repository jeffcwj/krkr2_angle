## SDL_CreateWindow 详解

`SDL_CreateWindow()` 是 SDL2 中创建窗口的核心函数。它的签名：

```cpp
SDL_Window* SDL_CreateWindow(
    const char* title,    // 窗口标题（UTF-8 编码的字符串）
    int x,                // 窗口左上角 X 坐标
    int y,                // 窗口左上角 Y 坐标
    int w,                // 窗口宽度（像素）
    int h,                // 窗口高度（像素）
    Uint32 flags          // 窗口标志（位掩码，可用 | 组合多个）
);
// 返回值：成功返回 SDL_Window 指针，失败返回 NULL
// 失败时调用 SDL_GetError() 获取错误原因
```

### 位置参数

`x` 和 `y` 参数指定窗口在屏幕上的位置。SDL2 提供两个特殊宏，避免你手动计算居中坐标：

- `SDL_WINDOWPOS_CENTERED` — 窗口在主显示器上居中
- `SDL_WINDOWPOS_UNDEFINED` — 让操作系统决定位置（通常是屏幕中央附近）

对于多显示器系统，可以指定在哪个显示器上居中：

```cpp
// 示例 1：在不同显示器上创建窗口
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    // 查询可用显示器数量
    int numDisplays = SDL_GetNumVideoDisplays();
    printf("检测到 %d 个显示器\n", numDisplays);

    for (int i = 0; i < numDisplays; i++) {
        SDL_Rect bounds;
        SDL_GetDisplayBounds(i, &bounds);  // 获取显示器的位置和分辨率
        printf("  显示器 %d: %dx%d 位于 (%d, %d)\n",
               i, bounds.w, bounds.h, bounds.x, bounds.y);
    }

    // 在主显示器（索引 0）居中创建窗口
    SDL_Window* w1 = SDL_CreateWindow("主显示器窗口",
        SDL_WINDOWPOS_CENTERED_DISPLAY(0),  // X: 在显示器 0 上居中
        SDL_WINDOWPOS_CENTERED_DISPLAY(0),  // Y: 在显示器 0 上居中
        800, 600, SDL_WINDOW_SHOWN);

    // 如果有第二个显示器，在上面也创建一个窗口
    if (numDisplays > 1) {
        SDL_Window* w2 = SDL_CreateWindow("副显示器窗口",
            SDL_WINDOWPOS_CENTERED_DISPLAY(1),
            SDL_WINDOWPOS_CENTERED_DISPLAY(1),
            640, 480, SDL_WINDOW_SHOWN);
        // 使用 w2 ...
        SDL_DestroyWindow(w2);
    }

    SDL_Delay(3000);  // 显示 3 秒
    SDL_DestroyWindow(w1);
    SDL_Quit();
    return 0;
}
```

### 窗口标志（flags）

flags 参数是位掩码，用 `|` 运算符组合多个标志。常用标志一览：

| 标志 | 值 | 说明 |
|------|-----|------|
| `SDL_WINDOW_SHOWN` | 0x04 | 窗口创建后立即可见（默认行为） |
| `SDL_WINDOW_HIDDEN` | 0x08 | 窗口创建后不可见，需调用 `SDL_ShowWindow()` 显示 |
| `SDL_WINDOW_RESIZABLE` | 0x20 | 允许用户拖拽边框调整窗口大小 |
| `SDL_WINDOW_FULLSCREEN` | 0x01 | 真全屏——改变显示器分辨率以匹配窗口大小 |
| `SDL_WINDOW_FULLSCREEN_DESKTOP` | 0x1001 | 桌面全屏——铺满屏幕但不改变分辨率 |
| `SDL_WINDOW_OPENGL` | 0x02 | 创建 OpenGL 上下文所需（上一章已学） |
| `SDL_WINDOW_BORDERLESS` | 0x10 | 无边框窗口（没有标题栏和关闭按钮） |
| `SDL_WINDOW_ALLOW_HIGHDPI` | 0x2000 | 启用 HiDPI 支持（macOS Retina / Windows DPI 缩放） |
| `SDL_WINDOW_MAXIMIZED` | 0x80 | 窗口创建后立即最大化 |
| `SDL_WINDOW_MINIMIZED` | 0x40 | 窗口创建后立即最小化 |

```cpp
// 示例 2：组合多个窗口标志
SDL_Window* window = SDL_CreateWindow(
    "可调整大小的 OpenGL 窗口",
    SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED,
    1280, 720,
    SDL_WINDOW_OPENGL          // 用于 OpenGL 渲染
    | SDL_WINDOW_RESIZABLE     // 允许调整大小
    | SDL_WINDOW_ALLOW_HIGHDPI // 支持高 DPI 显示器
    | SDL_WINDOW_SHOWN         // 创建后立即显示
);
```

## 全屏模式详解

SDL2 提供三种显示模式，理解它们的区别对于游戏和模拟器开发至关重要：

### 窗口模式 vs 桌面全屏 vs 真全屏

| 特性 | 窗口模式 | 桌面全屏 | 真全屏 |
|------|---------|---------|--------|
| 标志 | 无 | `SDL_WINDOW_FULLSCREEN_DESKTOP` | `SDL_WINDOW_FULLSCREEN` |
| 改变分辨率 | 否 | 否 | **是** |
| 覆盖任务栏 | 否 | **是** | **是** |
| 切换速度 | — | 快（~50ms） | 慢（~500ms） |
| Alt+Tab 切换 | 流畅 | 流畅 | 可能闪烁/黑屏 |
| 性能 | 最低 | 中等 | 最高（独占 GPU） |
| 推荐场景 | 开发调试 | **大多数游戏/模拟器** | 性能敏感的 3A 游戏 |

**KrKr2 项目建议使用桌面全屏**——切换快、不打乱用户桌面、性能足够。

```cpp
// 示例 3：运行时切换全屏模式
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    SDL_Window* window = SDL_CreateWindow("全屏切换演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED,
        800, 600, SDL_WINDOW_SHOWN | SDL_WINDOW_RESIZABLE);

    bool running = true;
    bool isFullscreen = false;
    SDL_Event event;

    while (running) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) {
                running = false;
            }
            // 按 F11 切换全屏
            if (event.type == SDL_KEYDOWN &&
                event.key.keysym.sym == SDLK_F11) {
                isFullscreen = !isFullscreen;
                // 0 = 窗口模式，SDL_WINDOW_FULLSCREEN_DESKTOP = 桌面全屏
                SDL_SetWindowFullscreen(window,
                    isFullscreen ? SDL_WINDOW_FULLSCREEN_DESKTOP : 0);
                printf("切换到 %s 模式\n",
                       isFullscreen ? "全屏" : "窗口");
            }
        }
        SDL_Delay(16);  // ~60 FPS
    }

    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}
```

## 动态修改窗口属性

窗口创建后，SDL2 提供一系列函数动态修改属性：

```cpp
// 示例 4：动态修改窗口属性
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);
    SDL_Window* window = SDL_CreateWindow("初始标题",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED,
        800, 600, SDL_WINDOW_SHOWN | SDL_WINDOW_RESIZABLE);

    // 修改标题
    SDL_SetWindowTitle(window, "新标题 - SDL2 演示");

    // 修改大小并设置限制
    SDL_SetWindowSize(window, 1024, 768);
    SDL_SetWindowMinimumSize(window, 320, 240);
    SDL_SetWindowMaximumSize(window, 1920, 1080);

    // 居中显示
    SDL_SetWindowPosition(window, SDL_WINDOWPOS_CENTERED,
                          SDL_WINDOWPOS_CENTERED);

    // 设置窗口图标（Windows/Linux 有效，macOS 用 Info.plist，Android 用 Manifest）
    SDL_Surface* icon = SDL_LoadBMP("icon.bmp");
    if (icon) { SDL_SetWindowIcon(window, icon); SDL_FreeSurface(icon); }

    // 设置透明度（Windows/macOS 支持，Linux 取决于窗口管理器）
    SDL_SetWindowOpacity(window, 0.8f);

    SDL_Delay(5000);
    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}
```

## HiDPI 处理

高 DPI 显示器（macOS Retina、Windows 150%/200% 缩放）上，逻辑像素和物理像素不一致。如果不处理，渲染结果会模糊或大小不对。

```cpp
// 示例 5：正确处理 HiDPI 的完整流程
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    // 关键：必须加 SDL_WINDOW_ALLOW_HIGHDPI 才能获得物理像素分辨率
    SDL_Window* window = SDL_CreateWindow("HiDPI 演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED,
        800, 600,
        SDL_WINDOW_OPENGL | SDL_WINDOW_ALLOW_HIGHDPI);

    // SDL_GetWindowSize 返回逻辑尺寸（与创建时一致）
    int logicalW, logicalH;
    SDL_GetWindowSize(window, &logicalW, &logicalH);
    printf("逻辑尺寸: %dx%d\n", logicalW, logicalH);

    // SDL_GL_GetDrawableSize 返回实际物理像素尺寸
    // 在 Retina 显示器上可能是逻辑尺寸的 2 倍
    int physicalW, physicalH;
    SDL_GL_GetDrawableSize(window, &physicalW, &physicalH);
    printf("物理尺寸: %dx%d\n", physicalW, physicalH);

    // 计算缩放比例
    float scaleX = (float)physicalW / logicalW;
    float scaleY = (float)physicalH / logicalH;
    printf("DPI 缩放: %.1fx%.1f\n", scaleX, scaleY);

    // 使用物理尺寸设置 OpenGL 视口——这是关键！
    // 如果用逻辑尺寸，渲染结果只占屏幕左下角 1/4（Retina 上）
    // glViewport(0, 0, physicalW, physicalH);

    // 处理窗口大小变化事件时也需要更新视口
    SDL_Event event;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) running = false;
            if (event.type == SDL_WINDOWEVENT &&
                event.window.event == SDL_WINDOWEVENT_SIZE_CHANGED) {
                // 窗口大小变化时重新计算物理尺寸
                SDL_GL_GetDrawableSize(window, &physicalW, &physicalH);
                // glViewport(0, 0, physicalW, physicalH);
                printf("新物理尺寸: %dx%d\n", physicalW, physicalH);
            }
        }
        SDL_Delay(16);
    }

    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}
```

**各平台 HiDPI 行为：** macOS Retina 下逻辑 800×600 → 物理 1600×1200（2x）；Windows 150% DPI 下 → 物理 1200×900；Linux Wayland 类似 macOS，X11 通常无缩放；Android 自动使用物理分辨率。

## 窗口生命周期管理

良好的窗口管理需要处理创建、配置、运行和销毁的完整流程。当窗口状态变化时，SDL2 产生 `SDL_WINDOWEVENT` 类型的事件，通过 `event.window.event` 字段区分具体类型：

| 事件 | 触发条件 | 典型用途 |
|------|---------|---------|
| `SDL_WINDOWEVENT_SHOWN` | 窗口变为可见 | 恢复渲染 |
| `SDL_WINDOWEVENT_HIDDEN` | 窗口被隐藏 | 暂停渲染，节省 GPU |
| `SDL_WINDOWEVENT_RESIZED` | 用户手动调整大小 | 更新视口 |
| `SDL_WINDOWEVENT_SIZE_CHANGED` | 大小因任何原因改变 | 更新渲染目标 |
| `SDL_WINDOWEVENT_MINIMIZED` | 窗口最小化 | 暂停游戏 |
| `SDL_WINDOWEVENT_RESTORED` | 从最小化恢复 | 恢复游戏 |
| `SDL_WINDOWEVENT_FOCUS_GAINED` | 窗口获得焦点 | 恢复输入处理 |
| `SDL_WINDOWEVENT_FOCUS_LOST` | 窗口失去焦点 | 暂停输入/降低帧率 |
| `SDL_WINDOWEVENT_CLOSE` | 用户点击关闭按钮 | 弹出保存确认 |

`RESIZED` 和 `SIZE_CHANGED` 的区别：`RESIZED` 仅在用户手动拖拽边框时触发；`SIZE_CHANGED` 在任何大小变化时都触发（包括代码调用 `SDL_SetWindowSize`、切换全屏等）。通常监听 `SIZE_CHANGED` 更安全。

```cpp
// 示例 6：窗口事件处理模板
void handleWindowEvent(const SDL_WindowEvent& we) {
    switch (we.event) {
    case SDL_WINDOWEVENT_SIZE_CHANGED:
        printf("窗口大小变为 %dx%d\n", we.data1, we.data2);
        // 更新 glViewport 和投影矩阵
        break;
    case SDL_WINDOWEVENT_MINIMIZED:
        printf("窗口最小化——暂停渲染\n");
        break;
    case SDL_WINDOWEVENT_RESTORED:
        printf("窗口恢复——继续渲染\n");
        break;
    case SDL_WINDOWEVENT_FOCUS_LOST:
        printf("失去焦点——降低帧率节省资源\n");
        break;
    case SDL_WINDOWEVENT_CLOSE:
        printf("用户请求关闭窗口\n");
        // 可以在此弹出"是否保存"对话框
        break;
    }
}
```

## 常见错误与排查

### 错误 1：忘记初始化 SDL_INIT_VIDEO

调用 `SDL_CreateWindow` 前必须先 `SDL_Init(SDL_INIT_VIDEO)`，否则窗口创建返回 NULL。

### 错误 2：HiDPI 下渲染模糊

忘记加 `SDL_WINDOW_ALLOW_HIGHDPI` 标志，或者用 `SDL_GetWindowSize` 的逻辑尺寸而非 `SDL_GL_GetDrawableSize` 的物理尺寸设置 `glViewport`。

### 错误 3：全屏切换后窗口位置丢失

从全屏切回窗口模式后，窗口可能跑到屏幕外。解决方案：切换前保存位置，切换后恢复：

```cpp
// 保存窗口位置
int savedX, savedY;
SDL_GetWindowPosition(window, &savedX, &savedY);
// 切换到全屏
SDL_SetWindowFullscreen(window, SDL_WINDOW_FULLSCREEN_DESKTOP);
// ... 使用全屏 ...
// 切回窗口模式并恢复位置
SDL_SetWindowFullscreen(window, 0);
SDL_SetWindowPosition(window, savedX, savedY);
```

## 本节小结

- `SDL_CreateWindow` 的 6 个参数：标题、X/Y 位置、宽高、标志位掩码
- 位置用 `SDL_WINDOWPOS_CENTERED` 或 `SDL_WINDOWPOS_CENTERED_DISPLAY(n)` 指定居中
- 全屏推荐桌面全屏（`SDL_WINDOW_FULLSCREEN_DESKTOP`），切换快且不改变分辨率
- HiDPI 处理关键：用 `SDL_GL_GetDrawableSize` 获取物理像素，而非 `SDL_GetWindowSize` 的逻辑像素
- 窗口事件中 `SIZE_CHANGED` 比 `RESIZED` 覆盖范围更广，通常用前者
- Android 上窗口始终全屏，标题/大小/透明度等修改无效

编写程序：创建 1024×768 的可调整大小窗口，支持 HiDPI，按 F11 切换全屏。窗口大小变化时打印逻辑和物理尺寸。

<details>
<summary>查看答案</summary>

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);
    SDL_Window* w = SDL_CreateWindow("HiDPI 窗口",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 1024, 768,
        SDL_WINDOW_SHOWN | SDL_WINDOW_RESIZABLE
        | SDL_WINDOW_OPENGL | SDL_WINDOW_ALLOW_HIGHDPI);

    bool running = true, fullscreen = false;
    SDL_Event e;
    while (running) {
        while (SDL_PollEvent(&e)) {
            if (e.type == SDL_QUIT) running = false;
            if (e.type == SDL_KEYDOWN && e.key.keysym.sym == SDLK_F11) {
                fullscreen = !fullscreen;
                SDL_SetWindowFullscreen(w,
                    fullscreen ? SDL_WINDOW_FULLSCREEN_DESKTOP : 0);
            }
            if (e.type == SDL_WINDOWEVENT &&
                e.window.event == SDL_WINDOWEVENT_SIZE_CHANGED) {
                int lw, lh, pw, ph;
                SDL_GetWindowSize(w, &lw, &lh);
                SDL_GL_GetDrawableSize(w, &pw, &ph);
                printf("逻辑 %dx%d  物理 %dx%d  缩放 %.1fx\n",
                       lw, lh, pw, ph, (float)pw / lw);
            }
        }
        SDL_Delay(16);
    }
    SDL_DestroyWindow(w);
    SDL_Quit();
}
```

### 题目 2：实现窗口位置记忆功能

编写程序：窗口关闭前将位置和大小保存到文件，下次启动时恢复。

<details>
<summary>查看答案</summary>

```cpp
#include <SDL.h>
#include <cstdio>

void saveState(SDL_Window* w, const char* f) {
    int x, y, width, height;
    SDL_GetWindowPosition(w, &x, &y);
    SDL_GetWindowSize(w, &width, &height);
    FILE* fp = fopen(f, "w");
    if (fp) { fprintf(fp, "%d %d %d %d\n", x, y, width, height); fclose(fp); }
}

bool loadState(const char* f, int* x, int* y, int* w, int* h) {
    FILE* fp = fopen(f, "r");
    if (fp && fscanf(fp, "%d %d %d %d", x, y, w, h) == 4) { fclose(fp); return true; }
    if (fp) fclose(fp);
    return false;
}

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);
    int x = SDL_WINDOWPOS_CENTERED, y = SDL_WINDOWPOS_CENTERED, w = 800, h = 600;
    loadState("window_state.txt", &x, &y, &w, &h);
    SDL_Window* win = SDL_CreateWindow("位置记忆",
        x, y, w, h, SDL_WINDOW_SHOWN | SDL_WINDOW_RESIZABLE);
    SDL_Event e;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&e)) { if (e.type == SDL_QUIT) running = false; }
        SDL_Delay(16);
    }
    saveState(win, "window_state.txt");
    SDL_DestroyWindow(win);
    SDL_Quit();
}
```

### 题目 3：列举真全屏和桌面全屏的三个关键区别

<details>
<summary>查看答案</summary>

1. **分辨率**：真全屏会改变显示器分辨率以匹配窗口大小（可能导致画面拉伸），桌面全屏保持当前分辨率不变
2. **切换速度**：真全屏切换需要 ~500ms（分辨率切换耗时），桌面全屏仅需 ~50ms（不改变分辨率）
3. **Alt+Tab 行为**：真全屏切出时可能闪烁或短暂黑屏（GPU 独占被打断），桌面全屏切换流畅无闪烁

额外区别：真全屏性能略高（GPU 独占输出），桌面全屏性能稍低但差距通常可忽略。
