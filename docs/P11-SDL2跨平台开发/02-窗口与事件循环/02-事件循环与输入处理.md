## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 事件队列 | Event Queue | SDL2 内部维护的 FIFO 缓冲区，操作系统产生的事件按顺序存入，程序按顺序取出处理 |
| 轮询 | Polling | 用 `SDL_PollEvent` 不断检查事件队列，有事件就处理，没有就立即返回继续循环 |
| 扫描码 | Scancode | 键盘上每个物理按键的唯一编号，与键盘布局无关（同一物理位置在 QWERTY 和 AZERTY 上扫描码相同） |
| 键码 | Keycode | 按键代表的逻辑字符，取决于键盘布局（QWERTY 的 Z 键在 AZERTY 上是 W） |
| 修饰键 | Modifier Key | Ctrl、Shift、Alt、Meta（Windows键/Command键）等用于组合快捷键的辅助按键 |
| 手柄映射 | Game Controller Mapping | SDL2 将不同品牌手柄的按键/摇杆映射为统一 Xbox 风格布局的机制 |
| 死区 | Dead Zone | 摇杆在中心附近的小范围内忽略微小偏移，避免"漂移"导致角色缓慢移动 |

## 事件循环基础

SDL2 的事件系统是整个输入处理的核心。操作系统产生的所有输入事件（键盘、鼠标、手柄、窗口变化等）先进入 SDL2 的内部事件队列，程序再从队列中逐个取出处理。

### 两种事件获取模式

```cpp
// 模式 1：轮询——游戏/模拟器的标准选择，队列空时返回 0
while (SDL_PollEvent(&event)) { handleEvent(event); }

// 模式 2：等待——工具/编辑器的选择，阻塞直到有事件
SDL_WaitEvent(&event);
handleEvent(event);

// 折衷方案：最多等 N 毫秒
SDL_WaitEventTimeout(&event, 100);
```

**KrKr2 使用轮询模式**——视觉小说需要持续渲染动画和文字效果。

### 完整的事件循环结构

```cpp
// 示例 1：标准事件循环骨架
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER);
    SDL_Window* window = SDL_CreateWindow("事件循环演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 800, 600, SDL_WINDOW_SHOWN);
    bool running = true;
    SDL_Event event;
    while (running) {
        while (SDL_PollEvent(&event)) {  // 一帧内可能有多个事件
            switch (event.type) {
            case SDL_QUIT:              running = false; break;
            case SDL_KEYDOWN:
            case SDL_KEYUP:             /* 键盘 */  break;
            case SDL_MOUSEMOTION:
            case SDL_MOUSEBUTTONDOWN:
            case SDL_MOUSEBUTTONUP:
            case SDL_MOUSEWHEEL:        /* 鼠标 */  break;
            case SDL_FINGERDOWN:
            case SDL_FINGERUP:
            case SDL_FINGERMOTION:      /* 触摸 */  break;
            case SDL_CONTROLLERAXISMOTION:
            case SDL_CONTROLLERBUTTONDOWN: /* 手柄 */ break;
            }
        }
        // 更新逻辑 → 渲染画面
        SDL_Delay(16);
    }
    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}
```

## 键盘事件处理

键盘事件是最常用的输入类型。SDL2 把键盘事件分为 `SDL_KEYDOWN`（按下）和 `SDL_KEYUP`（释放）。

### 扫描码 vs 键码

这是 SDL2 键盘处理中最重要的概念区分：

| 属性 | 扫描码 (Scancode) | 键码 (Keycode) |
|------|-------------------|----------------|
| 含义 | 物理按键位置 | 逻辑字符 |
| 字段 | `event.key.keysym.scancode` | `event.key.keysym.sym` |
| 枚举前缀 | `SDL_SCANCODE_*` | `SDLK_*` |
| 受布局影响 | **否** | **是** |
| 适用场景 | WASD 移动、游戏按键 | 文本输入、快捷键 |

**实际例子**：法语 AZERTY 键盘上，QWERTY 布局中 W 键的位置实际输出 Z。如果用键码 `SDLK_w` 做移动，法语用户需要按 Z 键。用扫描码 `SDL_SCANCODE_W` 则始终对应物理上方第二排左起第二个键。

```cpp
// 示例 2：扫描码 vs 键码的对比演示
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);
    SDL_Window* w = SDL_CreateWindow("键盘演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 640, 480, SDL_WINDOW_SHOWN);
    SDL_Event event;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) running = false;
            if (event.type == SDL_KEYDOWN) {
                printf("按下: scancode=%s(%d) keycode=%s(%d)\n",
                    SDL_GetScancodeName(event.key.keysym.scancode), event.key.keysym.scancode,
                    SDL_GetKeyName(event.key.keysym.sym), event.key.keysym.sym);
            }
        }
        SDL_Delay(16);
    }
    SDL_DestroyWindow(w);
    SDL_Quit();
}
```

### 按键重复与修饰键

按住一个键时，操作系统会在短暂延迟后开始快速重复发送按键事件。SDL2 通过 `event.key.repeat` 字段区分首次按下和重复：

```cpp
// 示例 3：处理按键重复和修饰键组合
void handleKeyboard(const SDL_KeyboardEvent& key) {
    // 忽略重复按键——游戏移动通常用状态查询而非事件
    if (key.repeat) return;

    // 检查修饰键状态
    SDL_Keymod mod = (SDL_Keymod)key.keysym.mod;
    bool ctrl  = (mod & KMOD_CTRL) != 0;   // Ctrl 键
    bool shift = (mod & KMOD_SHIFT) != 0;  // Shift 键
    bool alt   = (mod & KMOD_ALT) != 0;    // Alt 键

    // 快捷键处理：Ctrl+S 保存
    if (ctrl && key.keysym.sym == SDLK_s) {
        printf("Ctrl+S: 保存\n");
        return;
    }
    // Ctrl+Shift+S 另存为
    if (ctrl && shift && key.keysym.sym == SDLK_s) {
        printf("Ctrl+Shift+S: 另存为\n");
        return;
    }
    // ESC 退出
    if (key.keysym.sym == SDLK_ESCAPE) {
        printf("ESC: 退出\n");
    }
}
```

### 按键状态查询

事件方式适合一次性动作（跳跃、开火），持续性操作（移动）用状态查询更方便：

```cpp
// 示例 4：用键盘状态实现 WASD 移动
float playerX = 400, playerY = 300;
const float speed = 200.0f;  // 像素/秒

void updateMovement(float deltaTime) {
    const Uint8* state = SDL_GetKeyboardState(nullptr);  // 所有按键的当前状态数组
    // 用扫描码——不受键盘布局影响
    if (state[SDL_SCANCODE_W]) playerY -= speed * deltaTime;
    if (state[SDL_SCANCODE_S]) playerY += speed * deltaTime;
    if (state[SDL_SCANCODE_A]) playerX -= speed * deltaTime;
    if (state[SDL_SCANCODE_D]) playerX += speed * deltaTime;
}
```

## 鼠标事件处理

SDL2 提供四种鼠标事件：移动、按钮按下/释放、滚轮。

```cpp
// 示例 5：完整的鼠标事件处理
#include <SDL.h>
#include <cstdio>

void handleMouse(const SDL_Event& event) {
    switch (event.type) {
    case SDL_MOUSEMOTION: {
        // x, y 是当前坐标；xrel, yrel 是相对上次的偏移量
        auto& m = event.motion;
        printf("鼠标移动: (%d, %d) 偏移(%d, %d) 按钮状态=0x%x\n",
               m.x, m.y, m.xrel, m.yrel, m.state);
        // m.state 是位掩码：SDL_BUTTON_LMASK / RMASK / MMASK
        break;
    }
    case SDL_MOUSEBUTTONDOWN:
    case SDL_MOUSEBUTTONUP: {
        auto& b = event.button;
        const char* action = (b.state == SDL_PRESSED) ? "按下" : "释放";
        const char* name;
        switch (b.button) {
            case SDL_BUTTON_LEFT:   name = "左键"; break;
            case SDL_BUTTON_RIGHT:  name = "右键"; break;
            case SDL_BUTTON_MIDDLE: name = "中键"; break;
            default: name = "其他"; break;
        }
        printf("鼠标%s %s 位置(%d, %d) %s\n",
               name, action, b.x, b.y,
               b.clicks == 2 ? "双击" : "单击");
        break;
    }
    case SDL_MOUSEWHEEL: {
        auto& w = event.wheel;
        int scrollY = w.y;  // y > 0 向上，y < 0 向下
        if (w.direction == SDL_MOUSEWHEEL_FLIPPED) scrollY = -scrollY;  // macOS 自然滚动
        printf("滚轮: 垂直=%d 水平=%d\n", scrollY, w.x);
        break;
    }
    }
}
```

### 鼠标捕获与相对模式

相对鼠标模式隐藏光标并报告相对偏移量，适用于第一人称视角控制：

```cpp
SDL_SetRelativeMouseMode(SDL_TRUE);   // 光标隐藏，xrel/yrel 有效
SDL_SetRelativeMouseMode(SDL_FALSE);  // 恢复正常光标
```

## 触摸事件处理

Android 和触摸屏设备上，SDL2 提供触摸事件（`SDL_FINGER*`）。触摸坐标是**归一化坐标**（0.0 到 1.0），需乘窗口尺寸得到像素坐标：

```cpp
// 示例 6：触摸事件处理（Android 适用）
void handleTouch(const SDL_Event& event, int windowW, int windowH) {
    switch (event.type) {
    case SDL_FINGERDOWN: {
        auto& f = event.tfinger;
        int px = (int)(f.x * windowW), py = (int)(f.y * windowH);
        printf("触摸 #%lld 按下: (%d, %d) 压力=%.2f\n",
               (long long)f.fingerId, px, py, f.pressure);
        break;
    }
    case SDL_FINGERUP:
        printf("触摸 #%lld 释放\n", (long long)event.tfinger.fingerId);
        break;
    case SDL_FINGERMOTION: {
        auto& f = event.tfinger;
        printf("触摸 #%lld 移动: (%d, %d)\n",
               (long long)f.fingerId, (int)(f.x * windowW), (int)(f.y * windowH));
        break;
    }
    }
}
```

**触摸与鼠标的关系：** SDL2 默认在触摸设备上也产生鼠标事件（第一根手指模拟鼠标）。若需区分真实鼠标和触摸模拟，检查 `event.button.which == SDL_TOUCH_MOUSEID`。

## 手柄/游戏控制器

SDL2 的 Game Controller API 将各品牌手柄映射为统一的 Xbox 风格布局（A/B/X/Y、左右摇杆、扳机等）。

```cpp
// 示例 7：手柄初始化与输入处理
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER);
    SDL_Window* window = SDL_CreateWindow("手柄演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 800, 600, SDL_WINDOW_SHOWN);

    // 打开第一个可用手柄
    SDL_GameController* controller = nullptr;
    for (int i = 0; i < SDL_NumJoysticks(); i++) {
        if (SDL_IsGameController(i)) {
            controller = SDL_GameControllerOpen(i);
            printf("已连接: %s\n", SDL_GameControllerName(controller));
            break;
        }
    }

    bool running = true;
    SDL_Event event;
    const int DEAD_ZONE = 8000;  // 死区阈值（最大 32767）
    while (running) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) running = false;
            if (event.type == SDL_CONTROLLERBUTTONDOWN) {
                printf("手柄按钮 %s 按下\n",
                    SDL_GameControllerGetStringForButton((SDL_GameControllerButton)event.cbutton.button));
            }
            if (event.type == SDL_CONTROLLERAXISMOTION) {
                auto& a = event.caxis;
                if (a.value > -DEAD_ZONE && a.value < DEAD_ZONE) continue;  // 死区过滤
                printf("手柄轴 %s = %d\n",
                    SDL_GameControllerGetStringForAxis((SDL_GameControllerAxis)a.axis), a.value);
            }
            if (event.type == SDL_CONTROLLERDEVICEADDED) {
                controller = SDL_GameControllerOpen(event.cdevice.which);
                printf("手柄接入: %s\n", SDL_GameControllerName(controller));
            }
            if (event.type == SDL_CONTROLLERDEVICEREMOVED) {
                printf("手柄断开\n");
                controller = nullptr;
            }
        }
        SDL_Delay(16);
    }
    if (controller) SDL_GameControllerClose(controller);
    SDL_DestroyWindow(window);
    SDL_Quit();
}
```

**死区（Dead Zone）说明：** 手柄摇杆在物理中心位置时可能不会精确返回 0，而是小幅抖动（如 ±200）。如果不设死区，角色会在玩家没推摇杆时缓慢漂移。死区阈值 8000（约 25%）是常用值。

## 跨平台输入差异

| 输入类型 | Windows | Linux | macOS | Android |
|---------|---------|-------|-------|---------|
| 键盘 | ✅ 完整支持 | ✅ 完整支持 | ✅ 需注意 Cmd 键 | ⚠️ 外接键盘 |
| 鼠标 | ✅ | ✅ | ✅ | ⚠️ 模拟触摸 |
| 触摸 | ⚠️ 触摸屏设备 | ⚠️ 触摸屏设备 | ⚠️ 触控板 | ✅ 原生 |
| 手柄 | ✅ XInput/DInput | ✅ evdev | ✅ MFi/蓝牙 | ✅ 蓝牙 |
| 滚轮方向 | 传统方向 | 传统方向 | 默认自然滚动 | 触摸滑动 |

macOS 注意事项：Cmd 键映射为 `KMOD_GUI`（不是 `KMOD_CTRL`）。macOS 用户习惯用 Cmd+C/Cmd+V 而非 Ctrl+C/Ctrl+V，编写跨平台快捷键时需要两者都检测。

## 常见错误与排查

### 错误 1：事件循环中只处理一个事件

```cpp
// ❌ 错误——每帧只处理一个事件，事件堆积导致延迟
if (SDL_PollEvent(&event)) { handleEvent(event); }

// ✅ 正确——用 while 循环处理所有待处理事件
while (SDL_PollEvent(&event)) { handleEvent(event); }
```

### 错误 2：WASD 用键码导致布局问题

```cpp
// ❌ 法语用户需要按 ZQSD（AZERTY 布局）
if (event.key.keysym.sym == SDLK_w) moveUp();

// ✅ 用扫描码——始终对应物理位置
if (state[SDL_SCANCODE_W]) moveUp();
```

### 错误 3：忘记初始化 SDL_INIT_GAMECONTROLLER

```cpp
// ❌ 只初始化了 VIDEO，手柄不工作
SDL_Init(SDL_INIT_VIDEO);

// ✅ 手柄需要单独初始化
SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER);
```

## 本节小结

- 事件循环用 `while (SDL_PollEvent)` 每帧处理所有待处理事件
- 键盘：扫描码用于游戏移动（不受布局影响），键码用于快捷键和文本
- 鼠标：`SDL_MOUSEMOTION` 提供绝对和相对坐标，相对模式适合视角控制
- 触摸：坐标是 0-1 归一化值，需乘窗口尺寸；默认模拟鼠标事件
- 手柄：Game Controller API 统一映射为 Xbox 布局，需设死区防漂移
- macOS 的 Cmd 键映射为 `KMOD_GUI`，跨平台快捷键需同时检测 Ctrl 和 Cmd

### 题目 3：扫描码和键码的区别是什么？何时用哪个？

<details>
<summary>查看答案</summary>

**扫描码**（Scancode）代表键盘上的物理按键位置，不受键盘布局影响。QWERTY 键盘上的 W 键和 AZERTY 键盘上同一位置的 Z 键有相同的扫描码。适用于：游戏中的 WASD 移动、需要固定物理位置的按键绑定。

**键码**（Keycode）代表按键产生的逻辑字符，取决于当前键盘布局。同一物理键在不同布局下产生不同键码。适用于：快捷键（Ctrl+C）、文本输入、菜单加速键。

**总结**：需要固定物理位置用扫描码，需要逻辑字符用键码。

</details>
