## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| 用户事件 | User Event | SDL2 预留给开发者自定义的事件类型，编号从 `SDL_USEREVENT` 开始 |
| 事件过滤器 | Event Filter | 在事件进入队列前拦截的回调函数，可丢弃、修改或立即处理事件 |
| 事件监视器 | Event Watch | 与过滤器类似的回调，但不能阻止事件进入队列，只能"旁听" |
| 窗口 ID | Window ID | 每个 SDL_Window 的唯一数字标识，事件通过此 ID 关联到具体窗口 |
| 推送事件 | Push Event | 用 `SDL_PushEvent` 将自定义事件手动放入事件队列 |

## 自定义事件（用户事件）

SDL2 的事件类型是一个 `Uint32` 整数。系统事件（键盘、鼠标等）使用了较小的编号，而 `SDL_USEREVENT`（值为 0x8000）到 `SDL_LASTEVENT`（值为 0xFFFF）这段范围预留给开发者自定义使用。你可以用自定义事件实现：

- **定时器回调通知**：定时器线程触发后，发送自定义事件通知主线程
- **跨模块通信**：音频解码完成后通知渲染模块
- **异步任务完成**：网络请求完成后将结果通过事件传递到主循环

### 注册自定义事件类型

```cpp
// 示例 1：注册和使用自定义事件
#include <SDL.h>
#include <cstdio>
#include <cstring>

// SDL_RegisterEvents 分配 N 个连续事件编号，返回第一个编号
// 失败返回 (Uint32)-1（编号耗尽时）
static Uint32 MY_TIMER_EVENT = 0;
static Uint32 MY_DATA_READY_EVENT = 0;

bool initCustomEvents() {
    // 一次性申请 2 个自定义事件编号
    Uint32 base = SDL_RegisterEvents(2);
    if (base == (Uint32)-1) {
        printf("自定义事件注册失败\n");
        return false;
    }
    MY_TIMER_EVENT = base;       // 第 1 个：定时器事件
    MY_DATA_READY_EVENT = base + 1; // 第 2 个：数据就绪事件
    printf("注册事件: timer=%u data_ready=%u\n", MY_TIMER_EVENT, MY_DATA_READY_EVENT);
    return true;
}
```

### 派发自定义事件

自定义事件通过 `SDL_PushEvent` 手动放入事件队列。`SDL_UserEvent` 结构体提供了 `code`（整数标记）、`data1` 和 `data2`（两个 `void*` 指针）三个自由字段：

```cpp
// 示例 2：从工作线程发送自定义事件
struct TaskResult {
    int taskId;
    char message[64];
};

// 可在任意线程调用——SDL_PushEvent 是线程安全的
void notifyDataReady(int taskId, const char* msg) {
    // 堆分配数据——事件可能异步处理，栈变量会失效
    TaskResult* result = new TaskResult;
    result->taskId = taskId;
    strncpy(result->message, msg, 63);
    result->message[63] = '\0';

    SDL_Event event;
    SDL_memset(&event, 0, sizeof(event));
    event.type = MY_DATA_READY_EVENT;
    event.user.code = taskId;       // 整数标记
    event.user.data1 = result;      // 指向堆数据
    event.user.data2 = nullptr;     // 未使用
    SDL_PushEvent(&event);          // 线程安全地放入队列
}

// 主循环中处理
void handleCustomEvent(const SDL_Event& event) {
    if (event.type == MY_TIMER_EVENT) {
        printf("定时器触发: code=%d\n", event.user.code);
    }
    if (event.type == MY_DATA_READY_EVENT) {
        TaskResult* result = (TaskResult*)event.user.data1;
        printf("数据就绪: task=%d msg=%s\n", result->taskId, result->message);
        delete result;  // 接收方负责释放堆内存
    }
}
```

**内存管理要点：** `data1`/`data2` 中的堆内存由接收方负责释放。如果事件被过滤器丢弃，这块内存会泄漏——后文的事件过滤器部分会讲如何处理。

### 用 SDL 定时器触发自定义事件

SDL2 的定时器在独立线程中运行，不能直接修改游戏状态。正确做法是在定时器回调中发送自定义事件，让主循环统一处理：

```cpp
// 示例 3：定时器 + 自定义事件的完整示例
#include <SDL.h>
#include <cstdio>

static Uint32 TIMER_EVENT = 0;

// 定时器回调——在定时器线程中执行，不能访问渲染资源
Uint32 timerCallback(Uint32 interval, void* param) {
    SDL_Event event;
    SDL_memset(&event, 0, sizeof(event));
    event.type = TIMER_EVENT;
    event.user.code = *(int*)param;  // 传入的计数器
    SDL_PushEvent(&event);           // 安全地通知主线程
    return interval;                 // 返回值 = 下次触发间隔（0 = 停止）
}

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO | SDL_INIT_TIMER);
    SDL_Window* window = SDL_CreateWindow("定时器演示",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 400, 300, SDL_WINDOW_SHOWN);

    TIMER_EVENT = SDL_RegisterEvents(1);
    int counter = 0;
    // 每 1000ms 触发一次，param 指向 counter
    SDL_TimerID timer = SDL_AddTimer(1000, timerCallback, &counter);

    SDL_Event e;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&e)) {
            if (e.type == SDL_QUIT) running = false;
            if (e.type == TIMER_EVENT) {
                counter++;
                printf("定时器 #%d 触发\n", counter);
            }
        }
        SDL_Delay(16);
    }
    SDL_RemoveTimer(timer);  // 退出前移除定时器
    SDL_DestroyWindow(window);
    SDL_Quit();
}
```

## 事件过滤器与监视器

SDL2 允许你在事件进入队列**之前**拦截它们。这有两种机制：

- **事件过滤器**（`SDL_SetEventFilter`）：全局唯一，回调返回 0 表示丢弃事件，返回 1 表示放行。注意：在某些平台（如 Windows）上，窗口大小改变时过滤器在系统线程中调用，不在主线程
- **事件监视器**（`SDL_AddEventWatch`）：可注册多个，回调的返回值被忽略，事件始终进入队列。适合日志记录、统计等旁路逻辑

```cpp
// 示例 4：事件过滤器——阻止窗口被关闭
int myEventFilter(void* userdata, SDL_Event* event) {
    if (event->type == SDL_QUIT) {
        bool* allowQuit = (bool*)userdata;
        if (!*allowQuit) {
            printf("关闭被阻止！请先保存数据\n");
            return 0;  // 返回 0 = 丢弃此事件
        }
    }
    // 自定义事件被丢弃时需释放堆内存
    // （如果你的过滤器丢弃了携带 data1/data2 的用户事件）
    return 1;  // 返回 1 = 放行
}

// 安装过滤器
bool allowQuit = false;
SDL_SetEventFilter(myEventFilter, &allowQuit);
// 之后设置 allowQuit = true 就允许关闭了

// 事件监视器——记录所有键盘事件到日志
int keyLogger(void* userdata, SDL_Event* event) {
    if (event->type == SDL_KEYDOWN) {
        FILE* log = (FILE*)userdata;
        fprintf(log, "KEY: %s\n", SDL_GetKeyName(event->key.keysym.sym));
    }
    return 1;  // 返回值被忽略，但习惯返回 1
}

FILE* logFile = fopen("keys.log", "w");
SDL_AddEventWatch(keyLogger, logFile);
// 不再需要时移除：SDL_DelEventWatch(keyLogger, logFile);
```

### 事件队列管理

除了过滤器，SDL2 还提供直接操作事件队列的函数：

```cpp
// 清空队列中所有特定类型的事件
SDL_FlushEvent(SDL_MOUSEMOTION);  // 清除所有积压的鼠标移动事件

// 清空一个范围内的所有事件
SDL_FlushEvents(SDL_USEREVENT, SDL_LASTEVENT);  // 清除所有自定义事件

// 查看队列中是否有某类事件（不取出）
if (SDL_HasEvent(SDL_QUIT)) {
    printf("队列中有退出事件\n");
}

// 禁用/启用某类事件的入队
SDL_EventState(SDL_MOUSEMOTION, SDL_IGNORE);  // 忽略鼠标移动
SDL_EventState(SDL_MOUSEMOTION, SDL_ENABLE);  // 重新启用
```

## 多窗口管理

SDL2 支持同时创建多个窗口，每个窗口有唯一的 **窗口 ID**（`Uint32`，由 `SDL_GetWindowID` 获取）。所有窗口相关事件（`SDL_WINDOWEVENT`、`SDL_KEYDOWN` 等）通过 `event.window.windowID` 或 `event.key.windowID` 字段标识事件属于哪个窗口。

### 创建与管理多窗口

```cpp
// 示例 5：多窗口应用
#include <SDL.h>
#include <cstdio>
#include <map>

struct WindowInfo {
    SDL_Window* window;
    SDL_Renderer* renderer;
    Uint32 id;
    const char* name;
};

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    // 创建主窗口
    SDL_Window* mainWin = SDL_CreateWindow("主窗口",
        100, 100, 640, 480, SDL_WINDOW_SHOWN);
    SDL_Renderer* mainRen = SDL_CreateRenderer(mainWin, -1, SDL_RENDERER_ACCELERATED);

    // 创建工具窗口
    SDL_Window* toolWin = SDL_CreateWindow("工具面板",
        800, 100, 300, 400, SDL_WINDOW_SHOWN);
    SDL_Renderer* toolRen = SDL_CreateRenderer(toolWin, -1, SDL_RENDERER_ACCELERATED);

    // 用 map 管理窗口——通过 windowID 快速查找
    std::map<Uint32, WindowInfo> windows;
    windows[SDL_GetWindowID(mainWin)] = {mainWin, mainRen, SDL_GetWindowID(mainWin), "主窗口"};
    windows[SDL_GetWindowID(toolWin)] = {toolWin, toolRen, SDL_GetWindowID(toolWin), "工具面板"};

    SDL_Event event;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) {
                running = false;
                break;
            }
            // 窗口事件：通过 windowID 路由
            if (event.type == SDL_WINDOWEVENT) {
                Uint32 wid = event.window.windowID;
                auto it = windows.find(wid);
                if (it == windows.end()) continue;
                if (event.window.event == SDL_WINDOWEVENT_CLOSE) {
                    printf("%s 关闭\n", it->second.name);
                    SDL_DestroyRenderer(it->second.renderer);
                    SDL_DestroyWindow(it->second.window);
                    windows.erase(it);
                    if (windows.empty()) running = false;  // 所有窗口关闭
                }
            }
            // 键盘事件也带 windowID
            if (event.type == SDL_KEYDOWN) {
                Uint32 wid = event.key.windowID;
                auto it = windows.find(wid);
                if (it != windows.end()) {
                    printf("[%s] 按键: %s\n", it->second.name,
                           SDL_GetKeyName(event.key.keysym.sym));
                }
            }
        }
        // 每个窗口独立渲染
        for (auto& [id, info] : windows) {
            SDL_SetRenderDrawColor(info.renderer, 30, 30, 30, 255);
            SDL_RenderClear(info.renderer);
            SDL_RenderPresent(info.renderer);
        }
        SDL_Delay(16);
    }
    // 清理残留窗口
    for (auto& [id, info] : windows) {
        SDL_DestroyRenderer(info.renderer);
        SDL_DestroyWindow(info.window);
    }
    SDL_Quit();
}
```

### 多窗口注意事项

| 问题 | 说明 |
|------|------|
| 窗口焦点 | 键盘事件只发送到有焦点的窗口。用 `SDL_GetKeyboardFocus()` 查询当前焦点窗口 |
| 鼠标位置 | `SDL_GetMouseState` 返回的是相对焦点窗口的坐标。`SDL_GetGlobalMouseState` 获取屏幕绝对坐标 |
| Android 限制 | Android 上只支持**一个窗口**，创建第二个窗口会失败 |
| macOS 限制 | macOS 上多窗口需在主线程操作，跨线程创建窗口会崩溃 |
| 渲染器绑定 | 每个窗口必须有独立的 `SDL_Renderer`，不能共用 |

### 窗口间通信

多窗口间可通过自定义事件通信。发送事件时在 `user.windowID` 字段指定目标窗口：

```cpp
// 示例 6：窗口间发送消息
void sendToWindow(Uint32 targetWindowID, int messageCode, void* data) {
    SDL_Event event;
    SDL_memset(&event, 0, sizeof(event));
    event.type = MY_DATA_READY_EVENT;
    event.user.windowID = targetWindowID;  // 指定目标窗口
    event.user.code = messageCode;
    event.user.data1 = data;
    SDL_PushEvent(&event);
}

// 接收时检查 windowID
if (event.type == MY_DATA_READY_EVENT) {
    if (event.user.windowID == myWindowID) {
        // 这条消息是给我的
    }
}
```

## 跨平台差异

| 特性 | Windows | Linux | macOS | Android |
|------|---------|-------|-------|---------|
| 多窗口 | ✅ 完整支持 | ✅ 完整支持 | ✅ 主线程限制 | ❌ 仅一个窗口 |
| 自定义事件 | ✅ | ✅ | ✅ | ✅ |
| 事件过滤器线程 | ⚠️ 窗口调整时在系统线程 | ✅ 主线程 | ⚠️ 类似 Windows | ✅ 主线程 |
| 定时器精度 | ~15ms（系统调度） | ~1ms | ~1ms | ~10ms |
| SDL_QUIT 触发 | Alt+F4 / 关闭按钮 | 关闭按钮 | Cmd+Q / 关闭按钮 | Activity 销毁 |

**Windows/macOS 事件过滤器陷阱：** 当用户拖拽窗口边框调整大小时，系统进入模态循环（Modal Loop），SDL 的事件循环被阻塞。此时事件过滤器在**系统线程**中调用，而非主线程。如果过滤器中访问非线程安全的资源，会导致竞态条件。解决方案是在过滤器中只发送自定义事件，不直接修改共享状态。

## 常见错误与排查

### 错误 1：忘记初始化 SDL_INIT_TIMER

```cpp
// ❌ 只初始化了 VIDEO，SDL_AddTimer 静默失败
SDL_Init(SDL_INIT_VIDEO);
SDL_AddTimer(1000, callback, nullptr);  // 定时器不触发

// ✅ 需要显式初始化定时器子系统
SDL_Init(SDL_INIT_VIDEO | SDL_INIT_TIMER);
```

### 错误 2：自定义事件堆内存泄漏

```cpp
// ❌ 事件被过滤器丢弃，data1 指向的内存永远无法释放
event.user.data1 = new LargeData();
SDL_PushEvent(&event);  // 如果过滤器返回 0，LargeData 泄漏

// ✅ 在过滤器中检测并释放
int myFilter(void* ud, SDL_Event* e) {
    if (shouldDrop(e)) {
        if (e->type >= SDL_USEREVENT && e->user.data1) {
            delete (LargeData*)e->user.data1;  // 手动释放
        }
        return 0;
    }
    return 1;
}
```

### 错误 3：多窗口销毁后继续访问

```cpp
// ❌ 窗口关闭事件后继续使用已销毁的窗口指针
if (event.window.event == SDL_WINDOWEVENT_CLOSE) {
    SDL_DestroyWindow(win);
}
SDL_RenderClear(renderer);  // 崩溃：renderer 已随窗口销毁

// ✅ 销毁后立即从管理容器中移除，渲染循环自动跳过
```

## 本节小结

- 自定义事件用 `SDL_RegisterEvents` 分配编号，`SDL_PushEvent` 线程安全地派发
- `SDL_UserEvent` 提供 `code`（整数）和 `data1`/`data2`（指针）三个自由字段
- 堆内存通过 `data1`/`data2` 传递时，接收方负责释放
- 事件过滤器在事件进队前拦截（可丢弃），监视器只旁听（不可丢弃）
- 多窗口通过 `windowID` 路由事件，每个窗口需独立渲染器
- Android 不支持多窗口，macOS 多窗口必须在主线程操作
- 定时器回调在后台线程，不能直接修改游戏状态，应通过自定义事件通知主线程

### 题目 1：实现一个倒计时程序

用 SDL 定时器每秒发送自定义事件，主循环显示倒计时数字（10、9、8...0），到 0 时自动退出。

<details>
<summary>查看答案</summary>

```cpp
#include <SDL.h>
#include <cstdio>

static Uint32 COUNTDOWN_EVENT = 0;

Uint32 countdownCallback(Uint32 interval, void* param) {
    int* count = (int*)param;
    SDL_Event event;
    SDL_memset(&event, 0, sizeof(event));
    event.type = COUNTDOWN_EVENT;
    event.user.code = *count;
    SDL_PushEvent(&event);
    (*count)--;
    if (*count < 0) return 0;  // 停止定时器
    return interval;
}

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO | SDL_INIT_TIMER);
    SDL_Window* w = SDL_CreateWindow("倒计时",
        SDL_WINDOWPOS_CENTERED, SDL_WINDOWPOS_CENTERED, 300, 200, SDL_WINDOW_SHOWN);
    COUNTDOWN_EVENT = SDL_RegisterEvents(1);
    int count = 10;
    SDL_AddTimer(1000, countdownCallback, &count);
    SDL_Event e;
    bool running = true;
    while (running) {
        while (SDL_PollEvent(&e)) {
            if (e.type == SDL_QUIT) running = false;
            if (e.type == COUNTDOWN_EVENT) {
                printf("倒计时: %d\n", e.user.code);
                if (e.user.code <= 0) running = false;
            }
        }
        SDL_Delay(16);
    }
    SDL_DestroyWindow(w);
    SDL_Quit();
}
```

</details>

### 题目 2：SDL_SetEventFilter 和 SDL_AddEventWatch 的区别是什么？

<details>
<summary>查看答案</summary>

**事件过滤器**（`SDL_SetEventFilter`）：
- 全局只能设置**一个**（后设置的会替换前一个）
- 回调返回 0 时事件**被丢弃**，不进入队列
- 在某些平台（Windows/macOS）的窗口调整期间，回调在**系统线程**而非主线程中执行
- 适合：阻止特定事件（如禁止关闭窗口、忽略某些输入）

**事件监视器**（`SDL_AddEventWatch`）：
- 可以注册**多个**，互不干扰
- 回调的返回值被**忽略**，事件始终进入队列
- 需要用 `SDL_DelEventWatch` 手动移除
- 适合：日志记录、统计分析、调试监控等旁路逻辑

**总结**：过滤器是"门卫"（可以拒绝入场），监视器是"摄像头"（只记录，不阻拦）。

</details>

### 题目 3：为什么 Android 不支持多窗口？如何处理？

<details>
<summary>查看答案</summary>

Android 的窗口系统基于 Activity 和 Surface。SDL2 在 Android 上将整个 Activity 映射为一个 SDL_Window，底层直接使用 ANativeWindow。Android 系统不支持一个 Activity 内创建多个独立的 Surface（不同于桌面系统的窗口管理器），因此 `SDL_CreateWindow` 第二次调用会返回 `nullptr`。

**处理方案**：
1. **条件编译**：用 `#ifdef __ANDROID__` 跳过多窗口逻辑
2. **虚拟多窗口**：在单一窗口内用不同的渲染区域模拟多窗口效果
3. **运行时检测**：创建窗口后检查返回值是否为 `nullptr`

```cpp
SDL_Window* toolWin = SDL_CreateWindow("工具面板", 0, 0, 300, 400, SDL_WINDOW_SHOWN);
if (!toolWin) {
    // Android 上会走到这里——改为在主窗口内渲染工具面板
    printf("多窗口不可用: %s\n", SDL_GetError());
}
```

</details>
