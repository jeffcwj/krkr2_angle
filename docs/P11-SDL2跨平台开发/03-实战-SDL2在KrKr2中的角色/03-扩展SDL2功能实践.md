## 术语预览

本节将涉及以下术语，先有个印象，正文中会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| Hint | SDL Hint | SDL2 的运行时配置机制，通过键值对字符串调整底层行为（如渲染驱动选择、VSync 策略），类似环境变量 |
| Game Controller API | Game Controller API | SDL2 提供的手柄抽象层，将各品牌手柄映射为统一的 Xbox 风格按键布局，免去开发者自己处理差异 |

| 渲染驱动 | Render Driver | SDL2 内部 2D 渲染器的后端实现（OpenGL、Direct3D、Metal、Vulkan 等），决定了渲染指令如何转换为 GPU 操作 |
| VSync | Vertical Synchronization | 垂直同步，让渲染帧率与显示器刷新率对齐，消除画面撕裂但可能引入延迟 |

## 一、SDL2 Hints 系统：免改代码的行为调优

### 1.1 什么是 Hints？

SDL2 的 Hints 系统是一种运行时配置机制。你可以把它理解为 SDL2 内部的"环境变量"——通过设置键值对字符串，可以在不修改源码、不重新编译的情况下，改变 SDL2 底层的行为。

为什么需要这个机制？因为 SDL2 支持数十个平台，每个平台上的最佳默认行为不同。Hints 让开发者能够针对特定场景微调行为，同时保持 API 的简洁性——核心 API 函数的参数不会因为这些可选配置而膨胀。

### 1.2 基本用法

```cpp
#include <SDL.h>
#include <cstdio>

int main(int argc, char* argv[]) {
    // 在 SDL_Init 之前设置 Hints 效果最佳
    // 大多数 Hints 在子系统初始化时读取

    // 选择渲染驱动：优先使用 OpenGL
    // "opengl" = 使用 OpenGL 后端
    // "direct3d" = 使用 Direct3D（仅 Windows）
    // "metal" = 使用 Metal（仅 macOS/iOS）
    SDL_SetHint(SDL_HINT_RENDER_DRIVER, "opengl");

    // 启用 VSync（垂直同步）
    // "1" = 开启，"0" = 关闭
    SDL_SetHint(SDL_HINT_RENDER_VSYNC, "1");

    // 设置视频窗口缩放质量
    // "0" = 最近邻（像素风格，锐利但有锯齿）
    // "1" = 线性过滤（平滑）
    // "2" = 各向异性过滤（最高质量）
    SDL_SetHint(SDL_HINT_RENDER_SCALE_QUALITY, "1");

    if (SDL_Init(SDL_INIT_VIDEO) != 0) {
        std::printf("SDL_Init 失败: %s\n", SDL_GetError());
        return 1;
    }

    // 初始化后也可以查询当前 Hint 值
    const char* driver = SDL_GetHint(SDL_HINT_RENDER_DRIVER);
    std::printf("当前渲染驱动 Hint: %s\n",
                driver ? driver : "(未设置，使用默认)");

    SDL_Quit();
    return 0;
}
```

### 1.3 Hints 的优先级机制

SDL2 的 Hints 有三个优先级层次，从高到低：

```
┌─────────────────────────────────────────┐
│  优先级 1：环境变量（最高）               │
│  SDL_HINT_RENDER_DRIVER=opengl          │
│  在程序启动前通过 shell 设置             │
├─────────────────────────────────────────┤
│  优先级 2：SDL_SetHintWithPriority()     │
│  SDL_HINT_OVERRIDE 参数                 │
│  代码中强制设置，覆盖环境变量             │
├─────────────────────────────────────────┤
│  优先级 3：SDL_SetHint()（最低）         │
│  普通设置，可被环境变量覆盖              │
└─────────────────────────────────────────┘
```

```cpp
#include <SDL.h>
#include <cstdio>
#include <cstdlib>

int main(int argc, char* argv[]) {
    // 方式 1：普通设置（可被环境变量覆盖）
    SDL_SetHint(SDL_HINT_RENDER_DRIVER, "opengl");

    // 方式 2：强制覆盖（即使环境变量设置了也生效）
    SDL_SetHintWithPriority(
        SDL_HINT_RENDER_VSYNC, "1",
        SDL_HINT_OVERRIDE  // 最高优先级，覆盖环境变量
    );

    // 方式 3：通过环境变量设置（程序外部配置）
    // Windows:  set SDL_RENDER_DRIVER=direct3d
    // Linux:    export SDL_RENDER_DRIVER=opengl
    // macOS:    export SDL_RENDER_DRIVER=metal

    // 实际开发建议：
    // - 调试阶段用环境变量（方便切换，无需重新编译）
    // - 发布版本用 SDL_SetHintWithPriority + SDL_HINT_OVERRIDE
    //   确保行为一致
    // - SDL_SetHint 适合"有默认偏好但允许用户覆盖"的场景

    if (SDL_Init(SDL_INIT_VIDEO) != 0) {
        std::printf("SDL_Init 失败: %s\n", SDL_GetError());
        return 1;
    }

    std::printf("渲染驱动: %s\n",
                SDL_GetHint(SDL_HINT_RENDER_DRIVER));

    SDL_Quit();
    return 0;
}
```

### 1.4 Hint 变更回调

SDL2 允许你监听 Hint 值的变化，当某个 Hint 被修改时收到通知。这在插件系统中特别有用——一个插件修改了 Hint，其他插件需要知道。

```cpp
#include <SDL.h>
#include <cstdio>

// Hint 变更回调函数
// 参数说明：
//   userdata - 你传入的自定义数据指针
//   name     - 被修改的 Hint 名称
//   oldValue - 修改前的值（可能为 NULL）
//   newValue - 修改后的值（可能为 NULL）
void onHintChanged(void* userdata,
                   const char* name,
                   const char* oldValue,
                   const char* newValue) {
    std::printf("[Hint 变更] %s: \"%s\" -> \"%s\"\n",
                name,
                oldValue ? oldValue : "(null)",
                newValue ? newValue : "(null)");

    // userdata 可以用来传递应用状态
    int* changeCount = static_cast<int*>(userdata);
    (*changeCount)++;
}

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    int changeCount = 0;

    // 注册回调：监听 RENDER_VSYNC Hint 的变化
    SDL_AddHintCallback(
        SDL_HINT_RENDER_VSYNC,
        onHintChanged,
        &changeCount  // 传递自定义数据
    );

    // 修改 Hint 会触发回调
    SDL_SetHint(SDL_HINT_RENDER_VSYNC, "1");  // 触发
    SDL_SetHint(SDL_HINT_RENDER_VSYNC, "0");  // 触发
    SDL_SetHint(SDL_HINT_RENDER_VSYNC, "0");  // 值没变，不触发

    std::printf("总变更次数: %d\n", changeCount);

    // 移除回调
    SDL_DelHintCallback(
        SDL_HINT_RENDER_VSYNC,
        onHintChanged,
        &changeCount
    );

    SDL_Quit();
    return 0;
}
```

### 1.5 KrKr2 中实用的 Hints

以下是在 KrKr2 项目中最有实用价值的 Hints 列表：

| Hint 名称 | 作用 | KrKr2 推荐值 | 原因 |
|-----------|------|-------------|------|
| `SDL_HINT_RENDER_DRIVER` | 选择 2D 渲染后端 | `"opengl"` | KrKr2 使用 OpenGL 渲染，保持一致 |
| `SDL_HINT_RENDER_VSYNC` | VSync 开关 | `"1"` | 视觉小说不需要极低延迟，VSync 防止撕裂 |
| `SDL_HINT_ANDROID_SEPARATE_MOUSE_AND_TOUCH` | Android 上区分鼠标和触摸 | `"1"` | 模拟器可能同时接收两种输入 |
| `SDL_HINT_MOUSE_TOUCH_EVENTS` | 鼠标事件是否生成触摸事件 | `"0"` | 桌面端不需要，避免重复事件 |
| `SDL_HINT_ORIENTATIONS` | 允许的屏幕方向 | `"LandscapeLeft LandscapeRight"` | 视觉小说通常横屏显示 |
| `SDL_HINT_ACCELEROMETER_AS_JOYSTICK` | 加速度计是否作为手柄 | `"0"` | 视觉小说不需要倾斜操作 |
| `SDL_HINT_VIDEO_MINIMIZE_ON_FOCUS_LOSS` | 失焦时是否最小化 | `"0"` | 视觉小说不需要独占焦点 |

```cpp
#include <SDL.h>

// 在 KrKr2 初始化前调用此函数配置 SDL2 行为
void configureSDL2ForKrKr2() {
    // === 渲染相关 ===
    // 使用 OpenGL 后端（Cocos2d-x 默认也是 OpenGL）
    SDL_SetHint(SDL_HINT_RENDER_DRIVER, "opengl");
    // 开启 VSync，视觉小说对延迟不敏感
    SDL_SetHint(SDL_HINT_RENDER_VSYNC, "1");

    // === 输入相关 ===
    // Android 上区分鼠标和触摸（模拟器环境可能两者并存）
    SDL_SetHint(SDL_HINT_ANDROID_SEPARATE_MOUSE_AND_TOUCH, "1");
    // 桌面端鼠标不生成假的触摸事件
    SDL_SetHint(SDL_HINT_MOUSE_TOUCH_EVENTS, "0");
    // 禁用加速度计作为手柄输入
    SDL_SetHint(SDL_HINT_ACCELEROMETER_AS_JOYSTICK, "0");

    // === 窗口行为 ===
    // 失焦时不最小化（让用户能边玩边做其他事）
    SDL_SetHint(SDL_HINT_VIDEO_MINIMIZE_ON_FOCUS_LOSS, "0");

    // === 移动平台 ===
    // 锁定横屏方向（支持左右横屏自动旋转）
    SDL_SetHint(SDL_HINT_ORIENTATIONS,
                "LandscapeLeft LandscapeRight");
}
```

## 二、手柄（Game Controller）支持

### 2.1 为什么视觉小说需要手柄？

你可能觉得视觉小说是纯键鼠操作的——确实，传统 PC 上是这样。但 KrKr2 面向多平台，包括 Android TV 和接了手柄的平板。在这些设备上，手柄是主要甚至唯一的输入方式。此外，Steam Deck 等掌机上运行视觉小说时，手柄操作的体验远好于虚拟键盘。

SDL2 的 Game Controller API 提供了一个关键抽象：无论用户接的是 Xbox、PlayStation、Switch Pro 还是国产手柄，你的代码都只需要处理统一的按键名称（A/B/X/Y、左摇杆、右摇杆等）。

### 2.2 手柄初始化与设备发现

```cpp
#include <SDL.h>
#include <cstdio>
#include <vector>

// 管理已连接手柄的结构
struct ControllerInfo {
    SDL_GameController* controller; // SDL2 手柄句柄
    SDL_JoystickID id;             // 手柄的唯一 ID
    const char* name;              // 手柄名称（如 "Xbox Controller"）
};

std::vector<ControllerInfo> g_controllers;

// 初始化手柄子系统
bool initControllerSupport() {
    // SDL_INIT_GAMECONTROLLER 会自动包含 SDL_INIT_JOYSTICK
    if (SDL_Init(SDL_INIT_GAMECONTROLLER) != 0) {
        std::printf("手柄子系统初始化失败: %s\n",
                    SDL_GetError());
        return false;
    }

    // 加载内置的手柄映射数据库
    // SDL2 内置了数千种手柄的按键映射
    // 也可以从文件加载额外映射：
    // SDL_GameControllerAddMappingsFromFile("gamecontrollerdb.txt");

    // 枚举当前已连接的手柄
    int numJoysticks = SDL_NumJoysticks();
    std::printf("检测到 %d 个游戏输入设备\n", numJoysticks);

    for (int i = 0; i < numJoysticks; i++) {
        // 检查这个设备是否被识别为 Game Controller
        // （而不仅仅是通用 Joystick）
        if (SDL_IsGameController(i)) {
            SDL_GameController* ctrl = SDL_GameControllerOpen(i);
            if (ctrl) {
                ControllerInfo info;
                info.controller = ctrl;
                // 获取底层 Joystick ID（用于匹配事件）
                info.id = SDL_JoystickInstanceID(
                    SDL_GameControllerGetJoystick(ctrl));
                info.name = SDL_GameControllerName(ctrl);
                g_controllers.push_back(info);

                std::printf("  手柄 %d: %s (ID=%d)\n",
                            i, info.name, info.id);
            }
        } else {
            std::printf("  设备 %d: 通用手柄（未映射）: %s\n",
                        i, SDL_JoystickNameForIndex(i));
        }
    }

    return true;
}

// 清理手柄资源
void cleanupControllers() {
    for (auto& info : g_controllers) {
        SDL_GameControllerClose(info.controller);
    }
    g_controllers.clear();
}
```

### 2.3 手柄热插拔事件

现代系统支持手柄的热插拔——用户可以在游戏运行时插入或拔出手柄。SDL2 通过事件机制通知你：

```cpp
#include <SDL.h>
#include <cstdio>

// 处理手柄连接/断开事件
void handleControllerEvent(const SDL_Event& event) {
    switch (event.type) {
    case SDL_CONTROLLERDEVICEADDED: {
        // 新手柄连接
        int deviceIndex = event.cdevice.which;
        SDL_GameController* ctrl =
            SDL_GameControllerOpen(deviceIndex);
        if (ctrl) {
            SDL_JoystickID id = SDL_JoystickInstanceID(
                SDL_GameControllerGetJoystick(ctrl));
            std::printf("手柄已连接: %s (ID=%d)\n",
                        SDL_GameControllerName(ctrl), id);

            // 将手柄添加到管理列表
            ControllerInfo info{ctrl, id,
                                SDL_GameControllerName(ctrl)};
            g_controllers.push_back(info);
        }
        break;
    }
    case SDL_CONTROLLERDEVICEREMOVED: {
        // 手柄断开
        SDL_JoystickID id = event.cdevice.which;
        std::printf("手柄已断开: ID=%d\n", id);

        // 从管理列表中移除
        for (auto it = g_controllers.begin();
             it != g_controllers.end(); ++it) {
            if (it->id == id) {
                SDL_GameControllerClose(it->controller);
                g_controllers.erase(it);
                break;
            }
        }
        break;
    }
    case SDL_CONTROLLERDEVICEREMAPPED:
        // 手柄的映射被更新（较少见）
        std::printf("手柄映射已更新: ID=%d\n",
                    event.cdevice.which);
        break;
    }
}
```

### 2.4 手柄输入映射到视觉小说操作

视觉小说的操作比动作游戏简单得多，只需要几个按键映射：

```cpp
#include <SDL.h>
#include <cstdio>

// 视觉小说的操作类型
enum class VNAction {
    None,
    Advance,        // 推进对话
    Skip,           // 快进
    AutoMode,       // 自动模式
    ShowLog,        // 显示历史记录
    ShowMenu,       // 打开菜单
    QuickSave,      // 快速存档
    QuickLoad,      // 快速读档
    CursorUp,       // 选项光标上移
    CursorDown,     // 选项光标下移
    Confirm,        // 确认选项
    Cancel,         // 取消/返回
};

// 将手柄按钮映射为视觉小说操作
VNAction mapControllerButton(SDL_GameControllerButton button) {
    switch (button) {
    case SDL_CONTROLLER_BUTTON_A:
        return VNAction::Advance;   // A 键 = 推进对话
    case SDL_CONTROLLER_BUTTON_B:
        return VNAction::Cancel;    // B 键 = 取消/返回
    case SDL_CONTROLLER_BUTTON_X:
        return VNAction::AutoMode;  // X 键 = 自动模式
    case SDL_CONTROLLER_BUTTON_Y:
        return VNAction::ShowLog;   // Y 键 = 历史记录
    case SDL_CONTROLLER_BUTTON_START:
        return VNAction::ShowMenu;  // Start = 菜单
    case SDL_CONTROLLER_BUTTON_LEFTSHOULDER:
        return VNAction::QuickSave; // LB = 快速存档
    case SDL_CONTROLLER_BUTTON_RIGHTSHOULDER:
        return VNAction::QuickLoad; // RB = 快速读档
    case SDL_CONTROLLER_BUTTON_DPAD_UP:
        return VNAction::CursorUp;  // 方向键上 = 光标上移
    case SDL_CONTROLLER_BUTTON_DPAD_DOWN:
        return VNAction::CursorDown; // 方向键下 = 光标下移
    default:
        return VNAction::None;
    }
}

// 处理右扳机（RT）作为快进键
// 扳机是模拟轴（0-32767），需要设定死区阈值
VNAction mapControllerAxis(SDL_GameControllerAxis axis,
                           int16_t value) {
    // 扳机死区：25% 以上才算"按下"
    const int16_t TRIGGER_THRESHOLD = 8192;

    if (axis == SDL_CONTROLLER_AXIS_TRIGGERRIGHT
        && value > TRIGGER_THRESHOLD) {
        return VNAction::Skip;  // 右扳机 = 快进
    }
    return VNAction::None;
}

// 完整的手柄事件处理循环
void processControllerInput() {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
        switch (event.type) {
        case SDL_CONTROLLERBUTTONDOWN: {
            // 按钮按下
            auto button = static_cast<SDL_GameControllerButton>(
                event.cbutton.button);
            VNAction action = mapControllerButton(button);
            if (action != VNAction::None) {
                std::printf("手柄操作: %d\n",
                            static_cast<int>(action));
                // 将 action 传递给引擎的输入系统
                // engine->handleAction(action);
            }
            break;
        }
        case SDL_CONTROLLERAXISMOTION: {
            // 轴移动（摇杆/扳机）
            auto axis = static_cast<SDL_GameControllerAxis>(
                event.caxis.axis);
            VNAction action = mapControllerAxis(
                axis, event.caxis.value);
            if (action != VNAction::None) {
                // 处理持续性输入（快进）
            }
            break;
        }
        case SDL_CONTROLLERDEVICEADDED:
        case SDL_CONTROLLERDEVICEREMOVED:
            handleControllerEvent(event);
            break;
        }
    }
}
```

## 三、平台特化 API 实践

SDL2 在通用跨平台 API 之外，还为每个平台提供了特化的函数。这些函数只在特定平台上可用，需要条件编译来使用。

### 3.1 平台特化 API 概览

```
┌────────────────────────────────────────────────┐
│                SDL2 通用 API                    │
│  SDL_Init / SDL_CreateWindow / SDL_PollEvent   │
│  所有平台一致，代码无需改动                       │
├─────────┬──────────┬──────────┬────────────────┤
│ Windows │  Linux   │  macOS   │   Android      │
│ SDL_win │ SDL_x11  │ SDL_cocoa│ SDL_android    │
│ ···     │ SDL_wayl │ ···      │ ···            │
│ Direct  │ X11/Wayl │ Cocoa/   │ JNI/NativeWin  │
│ 3D/DXGI │ and 接口 │ Metal    │ Activity 管理  │
└─────────┴──────────┴──────────┴────────────────┘
```

### 3.2 Android 平台特化

Android 上 SDL2 提供了大量 JNI（Java Native Interface，Java 与 C/C++ 互调的桥接机制）桥接函数，允许 C++ 代码直接访问 Android Java 层功能：

```cpp
#include <SDL.h>

#ifdef __ANDROID__
#include <SDL_system.h>  // Android 特化头文件

// 获取 Android 应用信息
void printAndroidInfo() {
    // 获取内部存储路径（应用私有目录）
    // 返回类似 "/data/data/com.example.krkr2/files"
    const char* internalPath = SDL_AndroidGetInternalStoragePath();
    SDL_Log("内部存储: %s", internalPath);

    // 获取外部存储路径（SD 卡等）
    // 返回类似 "/storage/emulated/0/Android/data/com.example.krkr2/files"
    const char* externalPath = SDL_AndroidGetExternalStoragePath();
    SDL_Log("外部存储: %s", externalPath);

    // 检查外部存储状态
    // SDL_ANDROID_EXTERNAL_STORAGE_READ  = 可读
    // SDL_ANDROID_EXTERNAL_STORAGE_WRITE = 可写
    int storageState = SDL_AndroidGetExternalStorageState();
    if (storageState & SDL_ANDROID_EXTERNAL_STORAGE_WRITE) {
        SDL_Log("外部存储可读写");
    } else if (storageState & SDL_ANDROID_EXTERNAL_STORAGE_READ) {
        SDL_Log("外部存储只读");
    } else {
        SDL_Log("外部存储不可用");
    }

    // 获取 Android API 版本（如 API 30 = Android 11）
    int apiLevel = SDL_GetAndroidSDKVersion();
    SDL_Log("Android API Level: %d", apiLevel);
}
```

### 4.2 帧率统计与瓶颈定位

```cpp
#include <SDL.h>
#include <cstdio>

// 帧率统计器
struct FrameStats {
    Uint64 lastFrameTime;  // 上一帧的起始时间
    Uint64 frequency;      // 计时器频率
    double frameTimeMs;    // 当前帧耗时（毫秒）
    double fps;            // 当前 FPS
    double avgFps;         // 平均 FPS
    int frameCount;        // 总帧数
    double totalTime;      // 总时间（秒）

    // 各阶段耗时
    double eventTimeMs;    // 事件处理耗时
    double updateTimeMs;   // 逻辑更新耗时
    double renderTimeMs;   // 渲染耗时

    void init() {
        frequency = SDL_GetPerformanceFrequency();
        lastFrameTime = SDL_GetPerformanceCounter();
        frameCount = 0;
        totalTime = 0.0;
        fps = 0.0;
        avgFps = 0.0;
    }

    // 每帧开始时调用
    void beginFrame() {
        Uint64 now = SDL_GetPerformanceCounter();
        frameTimeMs =
            static_cast<double>(now - lastFrameTime) * 1000.0
            / static_cast<double>(frequency);
        lastFrameTime = now;

        if (frameTimeMs > 0.0) {
            fps = 1000.0 / frameTimeMs;
        }

        frameCount++;
        totalTime += frameTimeMs / 1000.0;
        if (totalTime > 0.0) {
            avgFps = frameCount / totalTime;
        }
    }

    // 测量某个阶段的耗时
    Uint64 startTimer() {
        return SDL_GetPerformanceCounter();
    }

    double endTimer(Uint64 start) {
        Uint64 end = SDL_GetPerformanceCounter();
        return static_cast<double>(end - start) * 1000.0
               / static_cast<double>(frequency);
    }

    // 每 60 帧输出一次统计
    void printIfNeeded() {
        if (frameCount % 60 == 0) {
            std::printf(
                "[帧统计] FPS=%.1f 平均=%.1f "
                "帧耗时=%.2fms "
                "(事件=%.2f 逻辑=%.2f 渲染=%.2f)\n",
                fps, avgFps, frameTimeMs,
                eventTimeMs, updateTimeMs, renderTimeMs);
        }
    }
};

// 使用示例
void gameLoop() {
    FrameStats stats;
    stats.init();

    bool running = true;
    while (running) {
        stats.beginFrame();

        // 阶段 1：事件处理
        Uint64 t = stats.startTimer();
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) {
                running = false;
            }
        }
        stats.eventTimeMs = stats.endTimer(t);

        // 阶段 2：逻辑更新
        t = stats.startTimer();
        // ... 游戏逻辑 ...
        stats.updateTimeMs = stats.endTimer(t);

        // 阶段 3：渲染
        t = stats.startTimer();
        // ... 渲染操作 ...
        stats.renderTimeMs = stats.endTimer(t);

        stats.printIfNeeded();
    }
}
```

### 4.3 常见性能问题与解决方案

以下是 SDL2 应用中常见的性能问题及对应解决方案：

| 问题 | 症状 | 原因 | 解决方案 |
|------|------|------|----------|
| 帧率不稳定 | FPS 在 30-60 之间跳动 | VSync 与帧逻辑不匹配 | 固定逻辑更新步长，渲染与逻辑解耦 |
| 事件处理卡顿 | 输入有明显延迟 | 事件队列积压 | 限制单帧处理事件数量，或用 `SDL_PeepEvents` 过滤 |
| 窗口调整大小闪烁 | 调整窗口大小时白屏 | 调整期间未渲染 | 监听 `SDL_WINDOWEVENT_SIZE_CHANGED` 立即重绘 |
| 高 CPU 占用 | 空闲时 CPU 100% | 事件循环无限制 | 在空闲时用 `SDL_WaitEvent` 替代 `SDL_PollEvent` |

```cpp
#include <SDL.h>
#include <cstdio>

// 问题 4 的修复：视觉小说的低功耗事件循环
// 视觉小说大部分时间在"等待玩家点击"，不需要疯狂轮询
void lowPowerEventLoop(SDL_Window* window,
                       SDL_Renderer* renderer) {
    bool running = true;
    bool needsRedraw = true;  // 是否需要重绘

    while (running) {
        SDL_Event event;

        if (needsRedraw) {
            // 有内容变化时用 PollEvent（不阻塞）
            while (SDL_PollEvent(&event)) {
                if (event.type == SDL_QUIT) {
                    running = false;
                }
                // 处理其他事件...
            }
        } else {
            // 画面静止时用 WaitEventTimeout（阻塞等待）
            // 每 100ms 醒来一次检查动画等
            if (SDL_WaitEventTimeout(&event, 100)) {
                if (event.type == SDL_QUIT) {
                    running = false;
                }
                // 有事件时标记需要重绘
                needsRedraw = true;
            }
        }

        if (needsRedraw) {
            // 执行渲染
            SDL_SetRenderDrawColor(renderer, 0, 0, 0, 255);
            SDL_RenderClear(renderer);
            // ... 绘制场景 ...
            SDL_RenderPresent(renderer);
            needsRedraw = false;  // 绘制完毕
        }
    }
}
```

## 五、在 Cocos2d-x 框架约束下扩展 SDL2

### 5.1 约束分析

前两节我们了解到，KrKr2 中 SDL2 被 Cocos2d-x 完全封装。这带来一个关键约束：**你不能随意调用 SDL2 函数，因为可能与 Cocos2d-x 的内部状态冲突**。

安全与危险操作分类：

| 操作类型 | 安全性 | 说明 |
|----------|--------|------|
| `SDL_GetHint` / `SDL_SetHint` | ✅ 安全 | Hints 是独立的配置系统，不影响渲染状态 |
| `SDL_GetPerformanceCounter` | ✅ 安全 | 只读取系统计时器，无副作用 |
| `SDL_GameControllerOpen` | ✅ 安全 | 手柄管理独立于渲染管线 |
| `SDL_GetWindowWMInfo` | ⚠️ 谨慎 | 只读操作，但依赖 Cocos2d-x 的窗口句柄 |
| `SDL_GL_SetAttribute` | ❌ 危险 | 会破坏 Cocos2d-x 管理的 GL 上下文配置 |
| `SDL_CreateWindow` | ❌ 危险 | Cocos2d-x 只管理一个窗口，创建第二个会混乱 |
| `SDL_GL_SwapWindow` | ❌ 危险 | Cocos2d-x 自己管理缓冲交换时机 |

### 5.2 安全扩展模式：事件过滤器

在 Cocos2d-x 管理的事件循环中，添加自定义事件处理的安全方式是使用 SDL2 的事件过滤器（Event Filter）。事件过滤器在事件进入队列之前被调用，可以拦截、修改或放行事件，而不会干扰 Cocos2d-x 的正常事件处理流程：

```cpp
#include <SDL.h>
#include <cstdio>

// 自定义事件过滤器
// 返回 1 = 事件保留在队列中（交给 Cocos2d-x 处理）
// 返回 0 = 事件被过滤掉（Cocos2d-x 看不到它）
int customEventFilter(void* userdata, SDL_Event* event) {
    // 拦截手柄事件，转换为键盘事件
    // Cocos2d-x 默认不处理手柄输入，但支持键盘
    if (event->type == SDL_CONTROLLERBUTTONDOWN) {
        SDL_GameControllerButton button =
            static_cast<SDL_GameControllerButton>(
                event->cbutton.button);

        SDL_Event keyEvent;
        keyEvent.type = SDL_KEYDOWN;
        keyEvent.key.state = SDL_PRESSED;
        keyEvent.key.repeat = 0;
        keyEvent.key.keysym.mod = KMOD_NONE;

        // 手柄 A 键 → 回车键（确认/推进）
        if (button == SDL_CONTROLLER_BUTTON_A) {
            keyEvent.key.keysym.sym = SDLK_RETURN;
            keyEvent.key.keysym.scancode = SDL_SCANCODE_RETURN;
            SDL_PushEvent(&keyEvent);
            return 0;  // 消费原始手柄事件
        }
        // 手柄 B 键 → Escape（取消/返回）
        if (button == SDL_CONTROLLER_BUTTON_B) {
            keyEvent.key.keysym.sym = SDLK_ESCAPE;
            keyEvent.key.keysym.scancode = SDL_SCANCODE_ESCAPE;
            SDL_PushEvent(&keyEvent);
            return 0;
        }
    }

    return 1;  // 其他事件正常传递给 Cocos2d-x
}

// 在应用初始化时注册
void setupEventFilter() {
    SDL_SetEventFilter(customEventFilter, nullptr);
}
```

### 5.3 安全扩展模式：SDL2 线程工具

SDL2 的线程、互斥锁、条件变量等同步原语是完全独立于渲染系统的，可以安全使用：

```cpp
#include <SDL.h>
#include <SDL_thread.h>
#include <cstdio>

// SDL2 互斥锁封装（RAII 风格）
class SDLMutex {
    SDL_mutex* m_mutex;
public:
    SDLMutex() : m_mutex(SDL_CreateMutex()) {}
    ~SDLMutex() { SDL_DestroyMutex(m_mutex); }

    void lock() { SDL_LockMutex(m_mutex); }
    void unlock() { SDL_UnlockMutex(m_mutex); }

    // RAII 锁守卫
    class Guard {
        SDLMutex& m_mtx;
    public:
        Guard(SDLMutex& mtx) : m_mtx(mtx) { m_mtx.lock(); }
        ~Guard() { m_mtx.unlock(); }
    };
};

// 使用场景：在后台线程加载游戏资源
// KrKr2 加载 XP3 归档时可以在后台线程完成
struct LoadTask {
    const char* archivePath;
    SDLMutex* mutex;
    bool done;
    bool success;
};

int backgroundLoader(void* data) {
    auto* task = static_cast<LoadTask*>(data);
    SDL_Log("后台加载: %s", task->archivePath);

    // 模拟加载工作
    SDL_Delay(2000);  // 假设加载需要 2 秒

    // 线程安全地更新状态
    SDLMutex::Guard guard(*task->mutex);
    task->done = true;
    task->success = true;

    return 0;
}

void startBackgroundLoad(const char* path) {
    static SDLMutex mutex;
    static LoadTask task;
    task.archivePath = path;
    task.mutex = &mutex;
    task.done = false;
    task.success = false;

    // 创建后台线程
    SDL_Thread* thread = SDL_CreateThread(
        backgroundLoader,  // 线程函数
        "loader",          // 线程名（调试用）
        &task              // 传递给线程的数据
    );

    // 主线程可以继续运行（不阻塞）
    // 定期检查 task.done 来获取加载结果
    SDL_DetachThread(thread);
}
```

## 六、跨平台路径与存储

SDL2 提供了统一的路径获取函数，避免了各平台路径格式不同的问题：

```cpp
#include <SDL.h>
#include <cstdio>
#include <string>

// 跨平台路径管理
struct PathManager {
    std::string basePath;   // 程序所在目录
    std::string prefPath;   // 偏好设置目录（存档/配置）

    bool init() {
        // SDL_GetBasePath() 返回程序可执行文件所在目录
        // Windows: C:\Games\KrKr2\
        // Linux:   /usr/local/bin/
        // macOS:   /Applications/KrKr2.app/Contents/Resources/
        // Android: (不可用，返回 NULL)
        char* base = SDL_GetBasePath();
        if (base) {
            basePath = base;
            SDL_free(base);
        } else {
            basePath = "./";
        }

        // SDL_GetPrefPath() 返回应用专属配置目录
        // 会自动创建目录（如果不存在）
        // Windows: C:\Users\<user>\AppData\Roaming\KrKr2\emulator\
        // Linux:   /home/<user>/.local/share/KrKr2/emulator/
        // macOS:   /Users/<user>/Library/Application Support/KrKr2/emulator/
        // Android: /data/data/<package>/files/KrKr2/emulator/
        char* pref = SDL_GetPrefPath("KrKr2", "emulator");
        if (pref) {
            prefPath = pref;
            SDL_free(pref);
        } else {
            prefPath = "./";
            SDL_Log("警告: 无法获取偏好路径, 使用当前目录");
        }

        std::printf("程序目录: %s\n", basePath.c_str());
        std::printf("偏好目录: %s\n", prefPath.c_str());
        return true;
    }

    // 获取存档文件路径
    std::string getSavePath(int slot) {
        return prefPath + "save_" + std::to_string(slot) + ".dat";
    }

    // 获取配置文件路径
    std::string getConfigPath() {
        return prefPath + "config.ini";
    }

    // 获取游戏数据路径（XP3 归档等）
    std::string getGameDataPath(const std::string& filename) {
        return basePath + "data/" + filename;
    }
};

int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO);

    PathManager paths;
    paths.init();

    std::printf("存档路径: %s\n",
                paths.getSavePath(1).c_str());
    std::printf("配置路径: %s\n",
                paths.getConfigPath().c_str());
    std::printf("游戏数据: %s\n",
                paths.getGameDataPath("game.xp3").c_str());

    SDL_Quit();
    return 0;
}
```

1. 初始化 Game Controller 子系统
2. 监听手柄连接/断开
3. 每帧轮询所有按钮和轴的状态
4. 在终端打印实时状态

## 本节小结

- **Hints 系统** 是 SDL2 免改代码调优的利器——通过键值对字符串即可调整渲染驱动、VSync、输入行为等底层参数
- **Game Controller API** 将数千种手柄映射为统一的 Xbox 布局，在视觉小说中可以映射为"推进对话""打开菜单"等操作
- **平台特化 API** 在通用 API 之外提供 Android JNI 桥接、Win32 窗口句柄、macOS Bundle 路径等平台专属功能，需要条件编译使用
- **性能诊断** 使用 `SDL_GetPerformanceCounter` 进行高精度计时，配合分阶段统计可以定位瓶颈在事件处理、逻辑更新还是渲染
- **Cocos2d-x 约束下的安全扩展** 限制为：Hints、计时器、手柄、事件过滤器、线程工具等不涉及渲染状态的功能
- **跨平台路径** 使用 `SDL_GetBasePath()` 和 `SDL_GetPrefPath()` 统一四平台的目录管理

## 练习题与答案

### 题目 1：Hint 优先级

假设你在代码中写了 `SDL_SetHint(SDL_HINT_RENDER_DRIVER, "opengl")`，但在运行程序前设置了环境变量 `SDL_RENDER_DRIVER=direct3d`。最终 SDL2 会使用哪个渲染驱动？如果你想让代码中的设置一定生效，应该怎么做？

<details>
<summary>查看答案</summary>

最终会使用 **direct3d**，因为环境变量的优先级高于 `SDL_SetHint()`。

如果想让代码设置一定生效，应使用 `SDL_SetHintWithPriority()`：

```cpp
SDL_SetHintWithPriority(
    SDL_HINT_RENDER_DRIVER,
    "opengl",
    SDL_HINT_OVERRIDE  // 使用 OVERRIDE 优先级，高于环境变量
);
```

三个优先级从高到低：
1. `SDL_HINT_OVERRIDE`（代码强制覆盖）
2. 环境变量
3. `SDL_HINT_NORMAL`（`SDL_SetHint` 的默认优先级）

</details>

### 题目 2：在 Cocos2d-x 中安全使用 SDL2

以下操作哪些是安全的，哪些会导致问题？请逐一判断并解释原因。

A. 调用 `SDL_GetPerformanceCounter()` 测量帧耗时
B. 调用 `SDL_CreateWindow()` 创建第二个窗口
C. 使用 `SDL_SetEventFilter()` 注册事件过滤器
D. 调用 `SDL_GL_SetAttribute()` 修改 OpenGL 属性
E. 使用 `SDL_CreateThread()` 创建后台加载线程

<details>
<summary>查看答案</summary>

- **A. `SDL_GetPerformanceCounter()` — ✅ 安全**。这是只读操作，只访问系统计时器，不影响任何渲染状态。

- **B. `SDL_CreateWindow()` — ❌ 危险**。Cocos2d-x 的 `Director` 和 `GLView` 假设全局只有一个窗口。创建第二个窗口会导致 OpenGL 上下文混乱，渲染输出可能到错误的窗口。

- **C. `SDL_SetEventFilter()` — ✅ 安全（需谨慎）**。事件过滤器在事件进入队列前调用，不影响 Cocos2d-x 的渲染。但过滤器不能阻塞太久，否则会影响事件处理的实时性。

- **D. `SDL_GL_SetAttribute()` — ❌ 危险**。Cocos2d-x 在创建 GLViewImpl 时已经设置了 OpenGL 属性。在运行时修改这些属性不会立即生效（需要重新创建上下文），而且会导致 Cocos2d-x 的内部状态与实际 GL 上下文不一致。

- **E. `SDL_CreateThread()` — ✅ 安全**。SDL2 的线程 API 完全独立于渲染系统。但要注意：后台线程**不能调用任何 OpenGL 函数**（GL 上下文是线程绑定的），也不能直接操作 Cocos2d-x 的节点树（非线程安全）。

</details>

```cpp
const int16_t STICK_DEAD_ZONE = 8000;
```

<details>
<summary>查看答案</summary>

```cpp
#include <SDL.h>
#include <cstdio>

const int16_t STICK_DEAD_ZONE = 8000;
const int16_t TRIGGER_THRESHOLD = 8192;

void processControllerInput(const SDL_Event& event) {
    switch (event.type) {
    case SDL_CONTROLLERBUTTONDOWN: {
        SDL_GameControllerButton btn =
            static_cast<SDL_GameControllerButton>(
                event.cbutton.button);

        if (btn == SDL_CONTROLLER_BUTTON_A) {
            // A 键 → 回车键
            SDL_Event keyEvent;
            SDL_zero(keyEvent);  // 清零所有字段
            keyEvent.type = SDL_KEYDOWN;
            keyEvent.key.state = SDL_PRESSED;
            keyEvent.key.keysym.sym = SDLK_RETURN;
            keyEvent.key.keysym.scancode = SDL_SCANCODE_RETURN;
            SDL_PushEvent(&keyEvent);
            std::printf("手柄 A → 回车\n");
        } else if (btn == SDL_CONTROLLER_BUTTON_B) {
            // B 键 → Escape
            SDL_Event keyEvent;
            SDL_zero(keyEvent);
            keyEvent.type = SDL_KEYDOWN;
            keyEvent.key.state = SDL_PRESSED;
            keyEvent.key.keysym.sym = SDLK_ESCAPE;
            keyEvent.key.keysym.scancode = SDL_SCANCODE_ESCAPE;
            SDL_PushEvent(&keyEvent);
            std::printf("手柄 B → Escape\n");
        }
        break;
    }
    case SDL_CONTROLLERAXISMOTION: {
        SDL_GameControllerAxis axis =
            static_cast<SDL_GameControllerAxis>(
                event.caxis.axis);
        int16_t value = event.caxis.value;

        // 左摇杆 Y 轴（上下）
        if (axis == SDL_CONTROLLER_AXIS_LEFTY) {
            if (value < -STICK_DEAD_ZONE) {
                // 摇杆推上 → 上方向键
                SDL_Event keyEvent;
                SDL_zero(keyEvent);
                keyEvent.type = SDL_KEYDOWN;
                keyEvent.key.state = SDL_PRESSED;
                keyEvent.key.keysym.sym = SDLK_UP;
                keyEvent.key.keysym.scancode = SDL_SCANCODE_UP;
                SDL_PushEvent(&keyEvent);
                std::printf("摇杆上\n");
            } else if (value > STICK_DEAD_ZONE) {
                // 摇杆推下 → 下方向键
                SDL_Event keyEvent;
                SDL_zero(keyEvent);
                keyEvent.type = SDL_KEYDOWN;
                keyEvent.key.state = SDL_PRESSED;
                keyEvent.key.keysym.sym = SDLK_DOWN;
                keyEvent.key.keysym.scancode = SDL_SCANCODE_DOWN;
                SDL_PushEvent(&keyEvent);
                std::printf("摇杆下\n");
            }
        }

        // 右扳机（按住 = 快进）
        if (axis == SDL_CONTROLLER_AXIS_TRIGGERRIGHT) {
            if (value > TRIGGER_THRESHOLD) {
                // 扳机超过阈值 → 按下 Ctrl 键（KrKr2 的快进键）
                SDL_Event keyEvent;
                SDL_zero(keyEvent);
                keyEvent.type = SDL_KEYDOWN;
                keyEvent.key.state = SDL_PRESSED;
                keyEvent.key.keysym.sym = SDLK_LCTRL;
                keyEvent.key.keysym.scancode = SDL_SCANCODE_LCTRL;
                SDL_PushEvent(&keyEvent);
            } else {
                // 扳机释放 → 释放 Ctrl 键
                SDL_Event keyEvent;
                SDL_zero(keyEvent);
                keyEvent.type = SDL_KEYUP;
                keyEvent.key.state = SDL_RELEASED;
                keyEvent.key.keysym.sym = SDLK_LCTRL;
                keyEvent.key.keysym.scancode = SDL_SCANCODE_LCTRL;
                SDL_PushEvent(&keyEvent);
            }
        }
        break;
    }
    }
}

// 完整的主循环
int main(int argc, char* argv[]) {
    SDL_Init(SDL_INIT_VIDEO | SDL_INIT_GAMECONTROLLER);

    SDL_Window* window = SDL_CreateWindow(
        "手柄测试", SDL_WINDOWPOS_CENTERED,
        SDL_WINDOWPOS_CENTERED, 640, 480, SDL_WINDOW_SHOWN);

    // 打开已连接的手柄
    for (int i = 0; i < SDL_NumJoysticks(); i++) {
        if (SDL_IsGameController(i)) {
            SDL_GameController* ctrl = SDL_GameControllerOpen(i);
            if (ctrl) {
                std::printf("手柄已连接: %s\n",
                            SDL_GameControllerName(ctrl));
            }
        }
    }

    bool running = true;
    while (running) {
        SDL_Event event;
        while (SDL_PollEvent(&event)) {
            if (event.type == SDL_QUIT) {
                running = false;
            }
            processControllerInput(event);
        }
        SDL_Delay(16);  // ~60 FPS
    }

    SDL_DestroyWindow(window);
    SDL_Quit();
    return 0;
}
```

注意摇杆输入需要处理死区（dead zone）——摇杆在物理中心位置时不会精确归零，通常有 ±3000 左右的漂移值。设定阈值 8000（约 25%行程）可以有效过滤这种噪声。

</details>
