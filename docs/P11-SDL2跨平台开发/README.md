# P11 — SDL2 跨平台开发

## 模块概述

SDL2（Simple DirectMedia Layer 2）是一个用 C 语言编写的跨平台多媒体抽象库，提供了统一的窗口管理、事件处理、音频输出、输入设备访问等底层系统接口。它让开发者只需编写一套代码，就能在 Windows、Linux、macOS、Android、iOS 等平台上运行，而不必为每个平台单独编写窗口创建、键盘鼠标处理、手柄输入等繁琐的系统调用代码。

在 KrKr2 模拟器项目中，SDL2 扮演了一个关键但容易被忽视的角色——它是平台抽象层（PAL）的一部分，负责在非 Cocos2d-x 原生支持的场景下提供窗口管理和输入事件分发。具体来说，`cpp/core/environ/sdl/` 目录下的代码利用 SDL2 来处理平台差异，使得引擎核心不需要直接与操作系统的窗口 API（如 Win32 的 `CreateWindow`、Linux 的 X11/Wayland、macOS 的 Cocoa）打交道。

本模块将带你从零学习 SDL2 的核心概念和用法，最终理解它在 KrKr2 项目中的具体角色。

## 学习目标

完成本模块后，你将能够：

1. **理解 SDL2 的架构设计** — 知道 SDL2 如何将操作系统差异隐藏在统一的 C API 后面，以及它的子系统（视频、音频、事件、手柄等）如何协作
2. **使用 SDL2 创建窗口和处理事件** — 能独立编写一个基于 SDL2 的跨平台程序，处理键盘、鼠标、触摸、手柄等输入事件
3. **将 SDL2 与 OpenGL 结合** — 理解如何用 SDL2 创建 OpenGL 上下文并在其中渲染图形，这正是 KrKr2 使用 SDL2 的方式之一
4. **理解 SDL2 在 KrKr2 中的角色** — 能读懂 `cpp/core/environ/sdl/tvpsdl.cpp` 的代码，明白 SDL2 如何与 Cocos2d-x 桥接层配合工作

## 术语预览

本模块将涉及以下术语，先有个印象，后续章节会逐一详细讲解：

| 术语 | 英文 | 一句话解释 |
|------|------|-----------|
| SDL2 | Simple DirectMedia Layer 2 | 一个用 C 语言编写的跨平台多媒体库，提供窗口、输入、音频等统一接口 |
| 子系统 | Subsystem | SDL2 内部按功能划分的独立模块（视频、音频、事件、手柄等），需要单独初始化 |
| 事件循环 | Event Loop | 程序持续运行并轮询操作系统事件（键盘按下、鼠标移动等）的主循环机制 |
| Surface | SDL_Surface | SDL2 中用 CPU 内存表示的一张图像/像素缓冲区，适合软件渲染 |
| Renderer | SDL_Renderer | SDL2 内置的 2D 硬件加速渲染器，底层使用 OpenGL/DirectX/Metal |
| GL 上下文 | OpenGL Context | OpenGL 渲染所需的状态容器，SDL2 可以创建并管理它，让你直接用 OpenGL 绑定渲染 |
| 手柄映射 | Game Controller Mapping | SDL2 将不同品牌手柄的按键/摇杆映射为统一的 Xbox 风格布局的机制 |
| 平台后端 | Platform Backend | SDL2 内部针对每个操作系统的具体实现代码（如 Win32 后端、X11 后端、Cocoa 后端） |
